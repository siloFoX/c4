# Patch 1.5.0 - Auto-resume idle queue check (4.17)

## Date: 2026-04-04

## Summary
auto-mgr이 태스크 완료 후 idle 상태가 되면 _taskQueue에서 다음 태스크를 자동으로 받아 실행하는 로직 추가.
기존에는 idle 콜백이 스냅샷 캡처만 하고 종료하여 연속 작업이 불가했음.

## Root Cause
idle 타이머 콜백(pty-manager.js:2246)에서 `_pendingTask` 확인 후 태스크가 없으면 아무 동작 없이 스냅샷만 캡처.
`_processQueue()`는 새 워커 생성만 처리하고, 기존 idle 워커에 태스크 재할당 로직이 없었음.

## Changes

### src/pty-manager.js
- idle 콜백(line 2258 부근)에 큐 확인 로직 추가
  - `_pendingTask` 없고 `_taskText` 없을 때 (완전 idle) `_taskQueue`에서 `name` 매칭 태스크 검색
  - 매칭 태스크 발견 시 `splice()`로 큐에서 제거 후 `_buildTaskText()` + `_chunkedWrite()`로 전송
  - `_taskText`, `_taskStartedAt` 설정, `_saveState()` 호출
- `_processQueue()`에 idle 워커 감지 로직 추가
  - 큐 순회 시 해당 이름의 워커가 이미 존재하고 alive이며 idle(`_pendingTask` 없고 `_taskText` 없음)이면 직접 전송
  - healthCheck에서 `_processQueue()` 호출 시 idle 워커에도 자동 할당 가능

### tests/auto-resume.test.js (new)
- 13개 유닛 테스트
  - idle 콜백 큐 매칭 로직 6개: 워커명 매칭, 불일치, _pendingTask/busy 스킵, splice, buildTaskText
  - _processQueue idle 워커 감지 7개: idle 전송, busy 스킵, _pendingTask 스킵, 새 워커 생성, 혼합 큐, proc 쓰기, _taskStartedAt 설정

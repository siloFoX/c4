## 5.19: PreToolUse compound blocking fix

### 원인 분석
1. `_createAndSendTask()`에서 워커를 먼저 스폰(home dir에서)한 후 worktree + settings를 생성
2. Claude Code는 시작 디렉토리의 `.claude/settings.json`만 로드
3. worktree의 settings.json에 있는 PreToolUse hook이 로드되지 않음
4. 추가로 inline `node -e "..."` 스크립트는 shell escaping으로 regex가 깨질 가능성 있음

### 수정 내용
1. `src/compound-check.js`: standalone 스크립트로 분리. shell escaping 문제 원천 차단
2. `_buildCompoundBlockCommand()`: absolute path로 standalone 스크립트 참조
3. `create()`: `options.cwd` 지원 추가 (로컬 타겟에서 home dir 대신 지정된 cwd 사용)
4. `_createAndSendTask()`: worktree + settings 생성을 워커 스폰 전으로 이동. 워커가 worktree 디렉토리에서 시작하여 `.claude/settings.json` 정상 로드

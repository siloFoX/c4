# Patch 1.5.0 - Worktree Complete Hook Set

## Problem
`_buildWorkerSettings()`가 worktree settings.json에 hook을 조각별로 생성할 때, daemon 통신 hook(PowerShell)이 PreToolUse[0]에 배치되고 복합 명령 차단 hook이 PreToolUse[1]에 배치되어, daemon hook 실패 시 복합 명령 차단이 실행되지 않을 수 있었다. Claude Code의 설정 병합 동작에 의존하여 홈/프로젝트 settings.json의 hook이 워커에 정확히 적용되는지 보장할 수 없었다.

## Root Cause
- daemon POST hook(PowerShell)이 PreToolUse 첫 번째에 배치됨
- PowerShell 실행 실패/타임아웃 시 후속 hook 체인 중단 가능
- 복합 명령 차단이 daemon hook 뒤에 있어 차단 우선순위가 낮음
- 홈/프로젝트 settings.json의 hook이 worktree settings에 포함되지 않아 설정 병합에 의존

## Changes

### src/pty-manager.js
- `_buildWorkerSettings()`: 완전한 hook 세트를 직접 생성하도록 재구성

  - PostToolUse: daemon 통신 hook

  - `_buildHookCommands()` 한 번만 호출하여 PreToolUse/PostToolUse 동시 추출

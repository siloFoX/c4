# Patch 1.5.0 - merge-homedir config fallback (4.18)

## Date: 2026-04-04

## Summary
cli.js merge 핸들러에서 git rev-parse 실패 시 config.json의 worktree.projectRoot를 폴백으로 사용.
홈디렉토리에서 c4 merge 실행이 가능해짐.

## Root Cause
cli.js:559에서 `git rev-parse --show-toplevel`을 cwd 지정 없이 실행.
홈디렉토리는 git 저장소가 아니므로 실패하고, catch 블록에서 즉시 process.exit(1).
pty-manager.js의 _detectRepoRoot()는 3단계 폴백이 있지만 cli.js는 폴백 없이 종료했음.

## Changes

### src/cli.js
- merge 핸들러(line 556-573)의 repoRoot 탐지에 config.json 폴백 추가
  - git rev-parse 실패 시 `path.resolve(__dirname, '..', 'config.json')` 로드
  - `config.worktree.projectRoot`가 있으면 `path.resolve()`로 절대 경로 변환 후 repoRoot로 사용
  - config 로드 실패 또는 projectRoot 미설정 시 기존과 동일하게 에러 출력 후 종료
  - 에러 메시지 개선: 'config.worktree.projectRoot not set' 안내 추가

### tests/merge-homedir.test.js (new)
- 11개 유닛 테스트
  - 폴백 로직: null 반환 (config 없음), git 우선, config 폴백, 빈 projectRoot, worktree 섹션 없음
  - 엣지 케이스: 잘못된 JSON, 상대 경로 resolve, git 우선순위, null projectRoot
  - 코드 검증: cli.js에 config.worktree.projectRoot 참조 존재 확인

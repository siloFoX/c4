# 1.8.0 - Agent SDK (TODO 9.3)

## Summary

TODO 9.3 (Agent SDK): new top-level `sdk/` package `c4-sdk` v0.1.0 that
wraps the c4 daemon HTTP API in a dependency-free JavaScript client.
Callers can now spawn workers, stream output, send tasks, and merge
branches from application code instead of shelling out to the `c4`
CLI.

- `sdk/package.json` - `c4-sdk` v0.1.0, CommonJS `main: lib/index.js`,
  `types: lib/index.d.ts`, engines `node >= 18`, no dependencies
- `sdk/lib/index.js` - `C4Client` + `C4Error` + `DEFAULT_BASE`, uses
  global `fetch` (Node 18+) with `opts.fetch` escape hatch
- `sdk/lib/index.d.ts` - hand-written TypeScript declarations
  (no build step)
- `sdk/examples/basic.js` - end-to-end spawn -> task -> watch ->
  wait-read -> close walkthrough
- `sdk/README.md` - install, auth, API summary, watch events,
  TypeScript usage
- `tests/agent-sdk.test.js` - 27 node:test assertions across 4 suites
  driving every method against an in-process `http.createServer`
  mock daemon

## Install

```bash
npm install c4-sdk
```

Requires Node.js 18 or newer. No transitive dependencies.

## Example

```js
const { C4Client } = require('c4-sdk');

const c4 = new C4Client({
  base: 'http://localhost:3456',
  token: process.env.C4_TOKEN, // required when auth.enabled in config.json
});

await c4.createWorker('w1', { target: 'local' });
await c4.sendTask('w1', 'analyze src/ and write NOTES.md', { autoMode: true });

for await (const ev of c4.watch('w1')) {
  if (ev.type === 'output') process.stdout.write(ev.dataText);
  if (ev.type === 'complete') break;
}

const result = await c4.readOutput('w1', { wait: true });
console.log(result.output);
```

## Method map

| Method | HTTP route |
| --- | --- |
| `health()` | `GET /health` |
| `listWorkers()` | `GET /list` |
| `getWorker(name)` | filter over `/list` |
| `createWorker(name, opts)` | `POST /create` |
| `sendTask(name, task, opts)` | `POST /task` |
| `sendInput(name, text)` | `POST /send` |
| `sendKey(name, key)` | `POST /key` |
| `readOutput(name, opts)` | `GET /read` or `/read-now` or `/wait-read` |
| `watch(name, opts)` | `GET /watch` (SSE, AsyncIterable) |
| `merge(name, opts)` | `POST /merge` |
| `close(name)` | `POST /close` |
| `fleetOverview(opts)` | `GET /fleet/overview` |

Each non-2xx response throws `C4Error` with `.status` (HTTP code) and
parsed `.body`. Transport failures preserve the original error as
`.cause`.

## Auth (JWT)

When the daemon is started with `auth.enabled: true` (see 8.14), every
API call except `/health` and `/auth/login` requires a bearer token.
The SDK sends `Authorization: Bearer <jwt>` on JSON endpoints and also
appends `?token=<jwt>` on the SSE watch URL so EventSource-style
clients that cannot set custom headers still authenticate (matches the
`auth.extractBearerToken` fallback in `src/auth.js`).

Obtain a token by POSTing to `/auth/login` with your configured user
and password, then pass it into the client:

```js
const loginRes = await fetch('http://localhost:3456/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ user: 'admin', password: '...' }),
});
const { token } = await loginRes.json();
const c4 = new C4Client({ base: 'http://localhost:3456', token });
```

Tokens are signed with HS256 and expire after 24h by default. When a
token has expired the SDK throws `C4Error` with `status === 401`;
catch that and re-login.

## Watch events

`c4.watch(name)` returns an async iterable. Each yielded value is the
parsed SSE payload; for `type === 'output'` the SDK decodes the base64
`data` field into `dataText` for convenience:

| type | purpose |
| --- | --- |
| `connected` | Server accepted the subscription. |
| `output` | PTY output. `data` is base64, `dataText` is decoded UTF-8. |
| `complete` | Worker exited. Iterator ends after this event. |
| `error` | Worker emitted an error or escalation. |

Pass an `AbortSignal` via `{ signal }` to cancel the stream, or break
out of the `for await` loop to clean up the underlying fetch.

## Tests

```
node --test tests/agent-sdk.test.js
```

The test harness boots an ephemeral `http.createServer` that mimics
the daemon's route surface (create / task / send / key / read variants
/ merge / close / fleet / watch SSE) and drives every method end to
end, including JWT auth, error responses, and synchronous argument
validation. Full c4 suite is 76 / 76 pass.

## Scope / follow-ups

- No CLI-only concerns in the SDK (fleet pin file, interactive
  approval UI, config reload) -- those live in `src/cli.js`.
- No WebSocket transport -- SSE via fetch is sufficient for the
  existing daemon and portable to browsers.
- Future work: optional high-level helpers (`waitIdle`,
  `createAndRunTask`), browser-targeted ESM build, token refresh
  helper once the daemon exposes a refresh endpoint.

# Patch 1.3.1 - Hook Event JSONL Persistence (4.2)

## Date: 2026-04-04

## Summary
Hook 이벤트(PreToolUse/PostToolUse)를 JSONL 파일로 영속화하는 기능 추가.

## Changes

### src/pty-manager.js
- `_appendEventLog(workerName, hookEntry)` 메서드 추가
  - `logs/events-<workerName>.jsonl`에 한 줄씩 JSON append
  - 디렉토리 자동 생성 (`fs.mkdirSync recursive`)
  - 잘못된 입력(null/undefined/비문자열/비객체) 무시
  - 쓰기 실패 시 에러 격리 (hook 처리 중단 방지)
- `hookEvent()` 메서드에서 SSE emit 직후 `_appendEventLog()` 호출 추가

### tests/hook-event-log.test.js (new)
- 16개 유닛 테스트
  - 정상 이벤트 기록 (5개): 파일 생성, 다중 append, 워커별 분리, 필드 보존, JSONL 형식 검증
  - 잘못된 입력 처리 (7개): null/undefined/빈문자열/비문자열 workerName, null/undefined/비객체 hookEntry
  - 파일 생성/추가 동작 (4개): 디렉토리 자동 생성, 기존 파일 보존, 쓰기 실패 안전성, 빈 객체 처리

## File Format
```jsonl
{"hook_type":"PreToolUse","tool_name":"Bash","tool_input":{"command":"ls"},"receivedAt":1712200000000}
{"hook_type":"PostToolUse","tool_name":"Write","tool_input":{"file_path":"/app/index.js"},"receivedAt":1712200001000}
```

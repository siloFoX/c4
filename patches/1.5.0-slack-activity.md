# Patch 1.5.0 - slack-activity hook 디버깅 (BF-2)

## Date: 2026-04-04

## Summary
hook 이벤트가 daemon에 도달하지 않아 events.jsonl이 생성되지 않는 문제의 디버깅 로그 추가. `_getLastActivity()`는 events JSONL 우선, 없으면 _taskText 요약으로 폴백.

## Root Cause
events.jsonl 기반 아키텍처 자체는 올바르지만, hook 이벤트가 daemon까지 도달하는 전체 경로에서 단절이 발생. 디버깅 로그가 없어 단절 지점 파악이 불가능했음.

## Changes

### hook 이벤트 수신 경로 디버깅

#### src/daemon.js
- `/hook-event` 핸들러: 요청 수신 로그 추가
  - `[DAEMON] /hook-event received: worker=<name> hook_type=<type> tool=<tool>`
  - `[DAEMON] /hook-event rejected: missing worker name`

#### src/pty-manager.js
- `hookEvent()`: 진입 시 로그 추가
  - `[C4] hookEvent: worker=<name> hook_type=<type> tool=<tool>`
- `_appendEventLog()`: 파일 경로 로그 추가
  - `[C4] _appendEventLog: <logFile> tool=<tool>`
  - `[C4] _appendEventLog error: <message>` (기존 silent catch 대체)
- `_getLastActivity()`: PTY raw.log 파싱 fallback 제거, events JSONL -> _taskText 요약으로 직접 폴백

### tests/slack-activity.test.js
- 8개 유닛 테스트 (raw.log 관련 테스트 제거)
  - events.jsonl에서 tool 활동 반환 확인
  - events.jsonl 없을 때 _taskText fallback 확인
  - events.jsonl 비어있을 때 _taskText fallback 확인
  - events.jsonl에 tool 이벤트 없을 때 _taskText fallback 확인
  - 데이터 없을 때 빈 문자열 확인
  - daemon.js/pty-manager.js에 디버깅 로그 존재 확인 (3개)

# Patch 1.5.0 - slack-activity hook 디버깅 + PTY fallback (BF-2)

## Date: 2026-04-04

## Summary
hook 이벤트가 daemon에 도달하지 않아 events.jsonl이 생성되지 않는 문제의 디버깅 로그 추가. events.jsonl이 없을 때 PTY raw.log에서 tool 사용 패턴을 파싱하는 fallback 추가.

## Root Cause
events.jsonl 기반 아키텍처 자체는 올바르지만, hook 이벤트가 daemon까지 도달하는 전체 경로에서 단절이 발생. 디버깅 로그가 없어 단절 지점 파악이 불가능했음. 또한 events.jsonl이 없을 때 _taskText 첫 줄만 반복 표시되어 워커의 실제 활동을 파악할 수 없었음.

## Changes

### 1단계: hook 이벤트 수신 경로 디버깅

#### src/daemon.js
- `/hook-event` 핸들러 (line 214-222): 요청 수신 로그 추가
  - `[DAEMON] /hook-event received: worker=<name> hook_type=<type> tool=<tool>`
  - `[DAEMON] /hook-event rejected: missing worker name`

#### src/pty-manager.js
- `hookEvent()` (line 286): 진입 시 로그 추가
  - `[C4] hookEvent: worker=<name> hook_type=<type> tool=<tool>`
- `_appendEventLog()` (line 467): 파일 경로 로그 추가
  - `[C4] _appendEventLog: <logFile> tool=<tool>`
  - `[C4] _appendEventLog error: <message>` (기존 silent catch 대체)

### 2단계: PTY raw.log fallback 활동 감지

#### src/pty-manager.js
- `_getLastActivity()` (line 754 이후): PTY raw.log 파싱 fallback 추가
  - events.jsonl이 없거나 비어있거나 tool 이벤트가 없을 때 실행
  - `logs/<workerName>.raw.log` 파일 읽기
  - ANSI escape sequence 제거 (`\x1b\[[0-9;]*[a-zA-Z]`)
  - tool 사용 패턴 정규식: `\b(Edit|Write|Read|Bash|Glob|Grep|Agent|NotebookEdit|WebFetch|WebSearch)\b[:\s]+...`
  - 파일명 추출: `[a-zA-Z0-9_.-]+\.[a-zA-Z0-9]+`
  - 중복 제거, 최근 5개, 120자 제한
  - 기존 `_taskText` fallback보다 우선 적용

### tests/slack-activity.test.js (new)
- 16개 유닛 테스트
  - events.jsonl이 raw.log보다 우선되는지 확인

  - ANSI escape 제거 확인
  - 중복 제거 확인

  - 최근 5개 추출 확인

  - 데이터 없을 때 빈 문자열 확인
  - raw.log가 _taskText보다 우선되는지 확인
  - 빈 raw.log 처리 확인
  - tool 패턴 없는 raw.log 처리 확인

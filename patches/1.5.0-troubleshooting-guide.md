## Summary
C4 운영 중 발생하는 주요 문제에 대한 트러블슈팅 가이드 문서 신규 작성.

## Changes

### docs/troubleshooting.md (new file)

5개 섹션으로 구성된 트러블슈팅 가이드:

#### 1. Zombie Daemon (좀비 데몬)
- 증상: PID 파일 존재하지만 HTTP 무응답
- 원인: uncaughtException, node-pty 메모리 누수, Windows SIGTERM 미처리
- 해결: taskkill + PID 정리 + daemon restart, 또는 `c4 daemon restart`
- 예방: watchdog.sh, daemon.log 모니터링
- 참조: daemon-manager.js stop() (line 118-160)

#### 2. Worktree 잔여물
- 증상: stale worktree 잔존, 새 워커 생성 시 충돌
- 원인: 비정상 종료로 _removeWorktree() 미호출, LOST DIRTY 보존
- 해결: 데몬 재시작(자동 정리), git worktree remove/prune, dirty worktree 수동 복구
- 예방: c4 close 사용, c4 daemon stop 사용
- 참조: pty-manager.js _cleanupLostWorktrees(), _isWorktreeDirty(), _notifyLostDirty()

#### 3. STALL 반복
- 증상: [STALL] 알림 반복, busy 상태 장기 유지
- 원인: 권한 프롬프트 대기, escalation(3회 연속 실패), API timeout, SSH 끊김
- 해결: c4 read-now + key Enter, c4 send 응답, c4 close + 재시작
- 예방: autoApprove 규칙, --auto-mode, healthCheck.autoRestart
- 참조: pty-manager.js healthCheck() stall detection (line 3015-3026), notifications.js notifyStall()

#### 4. Lost Worker 복구
- 증상: c4 list에 LOST 섹션 표시
- 원인: 데몬 재시작으로 PTY 연결 끊어짐, _loadState()에서 lostWorkers 기록
- 해결: c4 resume (--resume 세션 복구), 새 워커로 브랜치 이어받기, c4 close 정리
- 예방: watchdog.sh, healthCheck.autoRestart
- 참조: daemon.js POST /resume (line 319-339), pty-manager.js _loadState()

#### 5. CLI 에러
- ECONNREFUSED: 데몬 미실행
- Request timeout: 좀비 데몬
- Worker not found / already exists: 워커 이름 충돌
- Git Bash 경로 변환: MSYS_NO_PATHCONV=1
- 데몬 로그 확인 방법
- 참조: cli.js fixMsysArgs() (line 9-33)

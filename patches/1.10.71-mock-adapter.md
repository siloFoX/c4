## Summary

Adds a deterministic in-memory adapter to the agent framework as both
test infrastructure and the canonical reference implementation for
new backend authors (codex, claude-agent-sdk, future locals). Pure
addition; no production behavior change.

- `src/agents/mock.js` — full Adapter contract without a PTY/LLM
- `src/agents/index.js` — registers `'mock'` alongside the four
  prior keys
- `tests/agent-mock.test.js` — 20 cases across 6 suites covering
  the contract, the queueing semantics, the listener-error
  isolation, factory registration, and `init()` mutability
- `tests/local-llm.test.js` — REGISTRY canary widened to expect
  `'mock'` alongside the four prior keys

## Why this exists

Three reasons, in order:

1. **Test infra.** Lets agent-aware code (PtyManager state machine,
   hooks, scope guard) be exercised without a live PTY or LLM.
   Existing tests that need an Adapter shape today either spin up
   the real Claude Code adapter (heavy, requires the binary) or
   reach into private state of one of the local-LLM adapters
   (brittle, couples to backend internals). MockAdapter gives those
   tests a first-class fixture.

2. **Reference implementation.** A minimal but correct adapter so
   new backend authors writing `CodexAdapter` /
   `ClaudeAgentSdkAdapter` / etc. have a template that doesn't drag
   in claude-code-specific helpers. The claude-code adapter has 200+
   lines of pattern matching for Claude Code's UI; the local-LLM
   adapter has 400+ lines of HTTP streaming logic. MockAdapter is
   ~150 lines that hit every contract requirement and nothing else.

3. **Validation harness.** `validateAdapter()` now returns true for
   something other than the production claude-code adapter — locks
   in the contract surface even if Claude Code's adapter drifts
   (which it has, repeatedly, as Claude Code's UI changes ship).

## Listener queueing — the subtle part

Production backends sometimes emit output before the daemon
attaches its listener. If MockAdapter dropped those chunks, tests
that exercise "first chunk handling" would have to work around the
fixture. Instead, MockAdapter queues:

```js
pushOutput(chunk) {
  if (this._outputHandlers.length === 0) {
    this._scriptedChunks.push(chunk);
    return;
  }
  for (const h of this._outputHandlers) {
    try { h(chunk); } catch { /* swallow */ }
  }
}

onOutput(cb) {
  this._outputHandlers.push(cb);
  this._maybeFlush();   // replays queued chunks
  return () => { /* unsubscribe */ };
}
```

`setScript([chunks])` exists for the common test pattern where the
caller wants to script output before wiring up the consumer.

## Listener error isolation

A misbehaving consumer must not break the rest of the listener
chain. Each handler call is wrapped per-handler:

```js
for (const h of this._outputHandlers) {
  try { h(chunk); } catch { /* swallow per-handler */ }
}
```

This matches the production claude-code adapter's behavior (its
PTY data handler does the same) and the agent framework now
encodes this as a contract test rather than a coincidence.

## detectIdle semantics

```js
detectIdle(_chunk) { return this._idle === true; }
setIdle(value)     { this._idle = value === true; }
```

Strict `=== true` only — tests verified that `setIdle(1)` /
`setIdle('yes')` do not flip the flag. The reason: the daemon polls
`detectIdle` thousands of times per task; coercing truthy values
(say, a non-empty error message string) to "idle" would silently
declare tasks done while they're still running.

## Mock-only test surface

These methods are not part of the Adapter contract — they're how
tests drive the fixture:

| Method | Purpose |
|--------|---------|
| `setIdle(bool)` | flip the flag `detectIdle` returns |
| `pushOutput(chunk)` | feed one chunk to listeners (or queue) |
| `setScript([chunks])` | feed a series; flushed on first attach |
| `trace()` | snapshot `{inputs, keys, idle, pending}` for assertions |
| `reset()` | clear inputs/keys/pending; idle flag stays |

`reset()` keeps the idle flag deliberately — tests that flip a mock
to idle and reset between assertions usually want the idle state to
survive the reset.

## Constructor options

```js
new MockAdapter(_patterns, {
  name: 'codex-fixture',     // default 'mock'
  version: '0.1.0',           // default '1.0.0'
  supportsPause: true,        // default false
})
```

Lets a mock pose as a different backend in tests that exercise
metadata-sensitive code paths (e.g. audit-log entries that record
adapter name).

## Backwards compatibility

None broken. The `'mock'` key was unused before this patch;
existing callers that pass `agent.type = 'claude-code'` (or omit it)
get the same behavior they did pre-1.10.71.

`tests/local-llm.test.js`'s REGISTRY canary widened from 4 keys to
5; this is the only test that asserted on the exact key set.

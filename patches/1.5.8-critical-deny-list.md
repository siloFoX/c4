# 1.5.8 L4 Critical Deny List

## 5.13: L4 Critical Deny List

L4(full autonomy)에서 모든 deny를 approve로 override하던 로직에 예외 추가. 파괴적 명령은 L4에서도 절대 차단.

### CRITICAL_DENY_PATTERNS
- `rm -rf /` (루트 삭제)
- `git push --force`, `git push -f` (강제 푸시)
- `DROP TABLE`, `DROP DATABASE` (DB 삭제)
- `sudo rm` (관리자 삭제)
- `shutdown`, `reboot` (시스템 종료/재시작)
- `mkfs` (파일시스템 포맷)
- `dd if=` (디스크 직접 쓰기)
- `git reset --hard origin` (원격 기준 하드 리셋)

### 변경 파일
- `src/pty-manager.js`: 상수 `CRITICAL_DENY_PATTERNS` 추가, `_classifyPermission()`에서 L4 override 전에 critical 체크. 차단 시 스냅샷 로그 + `_notifications.notifyStall()` 호출

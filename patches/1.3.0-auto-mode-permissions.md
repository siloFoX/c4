# Patch 1.3.0 - Global Auto Mode + Permissions Overhaul

## Problem
Workers stall overnight on permission prompts for commands not in autoApprove rules (e.g., `where`, `which`, `mkdir`). After context compaction, workers lose CLAUDE.md context and stop using c4 commands.

## Changes

### pty-manager.js
- `autoStart()`: Sets `_globalAutoMode = true` on daemon
- `_isAutoModeEnabled()`: Checks `_globalAutoMode` flag
- `_classifyPermission(screenText, worker)`: Accepts worker context; auto workers default to 'approve' for unmatched commands
- `_buildWorkerSettings()`: Adds PostCompact hook that injects CLAUDE.md + session-context.md
- `healthCheck()`: Includes `task` and `taskStarted` in worker results

### notifications.js
- `notifyHealthCheck()`: Shows per-worker task description + elapsed time. Suppresses empty daemon messages.

### cli.js
- `c4 init`: 4 permissions -> 30+ allow + 7 deny rules

### CLAUDE.md
- Added full c4 CLI command reference
- Added manager worker operation pattern (new/task/wait/read/merge workflow)

- `src/pty-manager.js`
- `src/notifications.js`
- `src/cli.js`
- `CLAUDE.md`

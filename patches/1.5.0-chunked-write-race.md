# Patch 1.5.0 - _chunkedWrite() Race Condition Fix

## Problem
`_chunkedWrite()`가 setTimeout 기반으로 청크를 지연 전송할 때, 500자 초과 텍스트의 `\r`(Enter)이 두 번째 이후 청크에 포함되어 유실될 수 있었다. PTY 프로세스 종료, 이벤트 루프 블로킹, 타이밍 경합 등으로 setTimeout 콜백이 실행되지 않으면 명령이 입력창에 쌓이기만 하고 실행되지 않는 버그.

## Root Cause
- setTimeout은 전송 완료를 보장하지 않음
- `\r`이 텍스트 끝에 붙으므로 항상 마지막 청크에 포함
- 마지막 청크가 setTimeout으로 전송되면 레이스 컨디션 발생
- proc.write() 실패 시 에러 처리 없음

## Changes

### src/pty-manager.js
- `_chunkedWrite()`: async/await + drain 이벤트 기반 순차 전송으로 교체
  - setTimeout 대신 `await new Promise(resolve => setTimeout(resolve, delayMs))` 사용
  - `proc.write()` 반환값 확인, false면 drain 이벤트 대기 (백프레셔 처리)
  - 각 청크 전송 완료를 구조적으로 보장
- 호출처 5곳 async 대응:
  - Line 971 (`_completeSetupPhase1` fallback): setTimeout 콜백을 async로 변경, await 추가
  - Line 2010 (`_reuseWorker`): fire-and-forget (원래 동작 유지)
  - Line 2251 (`_onScreenText` pending task): setTimeout 콜백을 async로 변경, await 추가
  - Line 2447 (`_onScreenText` routine skip): setTimeout 콜백을 async로 변경, await 추가
  - Line 2563 (`send`): fire-and-forget (원래 동작 유지)

# 1.11.2 ‚Äî daemon API bolsters (8.5)

## Summary

Close the last Web-UI-vs-CLI behavioural gap. The Web UI used to drive
special keys through `POST /send {keys:true}` (ambiguous contract that
let any raw string reach the PTY) and had a merge button that was a
pure stub ‚Äî the pre-merge gate only lived in `c4 merge`'s CLI path.

This patch:

1. Extracts merge logic into `src/merge-core.js` so CLI and HTTP use the
   same `runPreMergeChecks` + `performMerge` pipeline.
2. Rewrites `POST /key` with a 18-entry allow-list and a
   `{success, key}` response shape.
3. Rewrites `POST /merge` to accept `{branch}` or `{name}`, to resolve
   worker names to branches via the worktree HEAD probe, and to return
   structured `reasons[]` so the Web UI can render per-check failures.
4. Adds `KEY_WRITE` + `MERGE_WRITE` RBAC actions. Manager role gets both
   by default, viewer gets neither.
5. Wires new Web UI buttons into `WorkerDetail.tsx`: an "Esc / Ctrl-C /
   Ctrl-D / Tab / arrow" row for keys plus a "Merge" button with a
   `window.confirm` guard.

## API contracts

### `POST /key {name, key}`

- `name`: worker name (required).
- `key`: one of
  `Enter` / `Escape` / `Tab` / `Backspace` / `Up` / `Down` / `Left` /
  `Right` / `C-a` / `C-b` / `C-c` / `C-d` / `C-e` / `C-l` / `C-n` /
  `C-p` / `C-r` / `C-z`.
- Success: `200 {success: true, key}`.
- Missing name/key: `400 {error: 'Missing name or key'}`.
- Unknown key: `400 {error: 'Unknown key: <key>', allowed: [...]}`.
- RBAC: requires `key.write`.

### `POST /merge {branch?, name?, skipChecks?}`

- At least one of `branch` or `name` must be set. `name` resolves via
  `manager.workers.get(name)` first, then
  `mergeCore.resolveBranchForWorker(name, repoRoot)`.
- `skipChecks: true` bypasses the doc-diff checks but still enforces
  branch existence + on-main + not-merging-main-into-itself.
- Success: `200 {success: true, branch, sha, summary, reasons, resolvedFrom}`.
- Missing input: `400 {error: 'Missing branch'}`.
- Branch does not exist: `404` with `reasons[]` including a
  `branch-exists / FAIL` entry.
- Pre-merge check failure: `409` with `reasons[]`.
- `git merge` error: `500 {error, branch, reasons}`.
- RBAC: requires `merge.write`.

## Why merge-core.js instead of adding code to merge-guard.js

`merge-guard.js` owns dirty-tree detection and stash push/pop ‚Äî it is
scoped to "what to do with uncommitted changes before a merge" and does
not know about branch existence, main-branch enforcement, or actual
`git merge` invocation. Sharing those last three across CLI + HTTP is
the 8.5 mandate, so a new module keeps the two concerns decoupled.

The CLI keeps ownership of `npm test`, the `validation.test_count`
comparison, and the `package-deps-installed` probe because those have
real side effects (install dependencies, execute tests) that the daemon
context must not run on an HTTP request.

## Test strategy

The test file spins up a minimal in-process HTTP server that wires the
real `merge-core` and `rbac` modules plus a stub PtyManager. Compared
to spawning a full `node src/daemon.js` subprocess, this keeps state
out of the project's real `state.json`, avoids port allocation, and
still exercises the endpoint contract end-to-end via `http.request`.
Source-inspection assertions on `src/daemon.js` guarantee the wiring
stays in place when the main handler moves.

39 assertions across 8 suites:

- `runPreMergeChecks` ‚Äî 9 cases (input validation, branch-exists,
  not-main, PASS path, TODO/CHANGELOG FAIL paths, skipChecks path,
  off-main rejection).
- `performMerge` ‚Äî 4 cases (success path, nonexistent branch, input
  validation).
- `resolveBranchForWorker` ‚Äî 3 cases (no worktree, real worktree via
  `git worktree add`, bad inputs).
- `/key` endpoint ‚Äî 8 cases (Enter/Escape/Ctrl-C/arrow PASS, unknown
  key 400, missing name 400, missing key 400, worker-not-found error
  passthrough).
- `/key` RBAC ‚Äî viewer 403 on `key.write`.
- `/merge` endpoint ‚Äî 6 cases (missing input 400, missing branch 404,
  failing check 409, successful merge 200, simulated git-error 500,
  worker-name resolution, unresolvable worker 404).
- `/merge` RBAC ‚Äî viewer 403 on `merge.write`, unauthenticated 401.
- Daemon source integration ‚Äî 5 greps confirming `merge-core` import,
  `KEY_ALLOWLIST`, and both RBAC gate references.

Full suite pre-existing count was 93; adding `daemon-api.test.js` keeps
that baseline and the worker's validation file records
`test_count: 93` for the pre-merge gate.

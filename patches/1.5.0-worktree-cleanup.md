# Patch 1.5.0 - LOST worker worktree 안전 정리

## Date: 2026-04-04

## Summary
healthCheck에서 LOST worker worktree를 자동 정리하되, dirty 상태(uncommitted changes)인 worktree는 보존하고 알림을 보내는 안전 정리 기능 추가.

## Changes

### src/pty-manager.js

#### _cleanupLostWorktrees()
- lostWorkers 맵에서 LOST 상태 worktree 수집
- orphan c4-worktree-* 디렉토리 스캔
- 삭제 전 `_isWorktreeDirty()` 호출로 dirty 상태 확인
- clean worktree: `git worktree remove --force`로 삭제
- dirty worktree: 보존 + `_notifyLostDirty()` 알림
- 반환값: `{ cleaned, preserved }` 객체
- startHealthCheck()에서 데몬 시작 시 1회 실행
- healthCheck() 주기마다 실행

#### _isWorktreeDirty(worktreePath)
- `git status --porcelain`으로 uncommitted changes 확인
- staged, unstaged, untracked 파일 전부 감지

#### _notifyLostDirty(workerName, worktreePath)
- `[LOST DIRTY]` 알림을 모든 활성 채널(Slack/Discord/Telegram)에 즉시 전송
- `sendImmediate()` 사용 (버퍼 미사용)

### tests/worktree-cleanup.test.js
- 18개 유닛 테스트
  - clean worktree 삭제 확인
  - dirty worktree 보존 확인
  - mixed (clean + dirty) 처리 확인
  - orphan 스캔 확인
  - lostWorkers 중복 처리 방지 확인
  - _notifyLostDirty 알림 전송 확인

# 1.6.1 Hybrid Safety Mode, Auto-Approval Block, Resume Re-orientation

## 5.21 Hybrid Safety Mode
- Critical deny 시 worker `_interventionState`를 `critical_deny`로 설정
- `_criticalCommand`에 차단된 명령 저장
- Slack에 `[CRITICAL DENY]` 승인 요청 메시지 전송 (`pushAll` + `_flushSlack`)
- `c4 approve <name>` CLI 명령으로 승인: `_interventionState` 해제 + `y\r` 전송
- daemon.js에 `/approve` POST 라우트 추가

### 변경 파일
- `src/pty-manager.js`: `_classifyPermission()` critical deny 블록 수정, `approve()` 메서드 추가
- `src/cli.js`: `case 'approve'` 추가
- `src/daemon.js`: `/approve` 라우트 추가

## 5.28 Auto-Approval Block
- `send()` 함수 시작부에 `critical_deny` 상태 체크 추가
- Enter 키 (`isSpecialKey && input === 'Enter'`) 차단
- 'y'/'Y' 텍스트 입력 차단
- 다른 키/텍스트는 통과 허용 (C-c 등으로 취소 가능)

### 변경 파일
- `src/pty-manager.js`: `send()` 메서드에 가드 추가

## 5.14 Resume Re-orientation
- `create()` 메서드에서 `options.resume` 시 5초 후 scrollback 읽기
- `screen.getVisibleText()` 마지막 20줄 캡처
- `[RESUMED]` 태그로 스냅샷 생성
- Slack에 resume 알림 전송

### 변경 파일
- `src/pty-manager.js`: `create()` 메서드 resume 후 re-orientation 로직 추가

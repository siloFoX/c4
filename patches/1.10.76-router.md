## Summary

Multi-tier alternative to the binary `'hybrid'` heuristic.
Backwards-compatible: existing `'hybrid'` callers keep their
length+keyword behavior unchanged. The new `'router'` type is
additive.

- `src/agents/index.js` — `pickRoutedType(task, agentConfig)` +
  `'router'` type wired into `createAdapter()`
- `tests/agent-router.test.js` — 23 cases / 4 suites
- Suite 163 → 164

## Why router and not "extend hybrid"

`'hybrid'` splits binary — simple → local, complex → claude-code.
That fits two-tier setups but loses fidelity when an operator
wants three or more tiers (e.g. cheap-local for short, mid-tier
local for medium, claude-code for complex / keyword-tagged).
Adding a third tier to `'hybrid'` would have required changing a
stable signature, breaking every caller that already configured
`agent.local` / `agent.complex`.

`'router'` lives next to `'hybrid'`. Both are exported, both
tested, operators pick the shape that fits.

## Rule shape

```js
{
  if?: {
    lengthLte?: number,    // task.length <= n
    lengthGte?: number,    // task.length >= n
    matches?: string,      // regex source, /i, must match
    notMatches?: string,   // regex source, /i, must NOT match
  },
  default?: true,          // catch-all (typically last)
  use: string,             // REGISTRY key — required, non-empty
}
```

Rules are evaluated in order; first match wins. Multiple keys in
`if` combine with AND semantics — e.g. `{ lengthGte: 5, lengthLte:
10 }` matches a 5-10 char range.

Empty `if: {}` does NOT match — at least one criterion must be
specified. This catches an operator misconfig that would otherwise
silently match every task.

## Silent skipping

Bad rule entries are skipped without crashing:

- `use` missing or non-string or empty string
- `if.matches` / `if.notMatches` is not a valid regex source
- entire entry is `null` / not an object
- non-array `rules` field

The worst case is the router falling through to `fallback`
(default `claude-code`). Operator gets adapter selection but no
exception — which matches how the rest of the agent factory
handles config errors (silent fallback rather than daemon crash).

## Configuration example

```json
{
  "agent": {
    "type": "router",
    "fallback": "claude-code",
    "rules": [
      { "if": { "lengthLte": 200 }, "use": "local-ollama" },
      { "if": { "matches": "\\bdesign\\b" }, "use": "claude-code" },
      { "if": { "lengthGte": 3000 }, "use": "claude-code" },
      { "default": true, "use": "local-llama-cpp" }
    ],
    "options": {
      "local-ollama":   { "model": "llama3.1" },
      "local-llama-cpp": { "model": "qwen2.5-coder" }
    }
  }
}
```

After resolution, the chosen adapter sees its per-type sub-bag
(e.g. `options['local-ollama']`) per the existing
`resolveAdapterOptions()` semantics — no change to that path.

## Test coverage

```
pickRoutedType — basic dispatch (6)
  - returns fallback when no rules array
  - returns fallback when rules empty
  - honors agentConfig.fallback
  - first matching rule wins
  - default rule catches anything
  - default rule short-circuits even if upstream matches would have caught

pickRoutedType — Condition keys (7)
  - lengthLte
  - lengthGte
  - lengthLte AND lengthGte (range gate)
  - matches uses regex source, case-insensitive
  - notMatches inverts
  - matches AND notMatches combine
  - empty if:{} does NOT match (operator misconfig guard)

pickRoutedType — silent skipping (4)
  - non-string use
  - empty use string
  - invalid regex source
  - null / non-object entries

createAdapter — type:'router' wiring (6)
  - matching rule instantiates the right adapter
  - long task falls through default
  - no match + no default falls back to fallback
  - options forwarded to resolved adapter
  - 'hybrid' still works (backwards compat)
  - unknown `use` throws via createAdapter
```

## Backwards compatibility

None broken. `'hybrid'` keeps the binary length+keyword heuristic.
`pickHybridType` keeps its export. `DEFAULT_HYBRID_*` constants
unchanged. The new code is purely additive: new function
(`pickRoutedType`), new exported function name, new type key
(`'router'`).

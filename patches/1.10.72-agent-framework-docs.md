# 1.10.72 — Agent Framework adapter authoring guide

## Summary

Documentation-only patch closing the gap left after 1.10.71 shipped
MockAdapter as "the canonical reference implementation for new
backend authors" but no actual reference document existed.

- `docs/agent-framework.md` (new) — operator-facing guide for
  writing a new adapter

## What's in the doc

Pulls the contract directly from `src/agents/adapter.js` and points
at `mock.js` as the minimal correct template. Walks through the
four traps that consistently bite new authors:

1. **Listener-before-output queueing.** Drop the first chunk and
   the daemon loses the start of every task. MockAdapter shows the
   queue + flush-on-attach pattern.

2. **Listener error isolation.** One throwing consumer must not
   break the rest of the chain. Wrap each handler call in
   try/catch.

3. **`detectIdle` cheapness + strict `=== true` semantics.** Called
   thousands of times per task; no I/O, no truthy coercion.

4. **`init(workerCtx)` mutability.** Including `init(null)` clear.
   Never assume single-use.

Also includes:
- One-line factory registration walkthrough
- Hybrid routing as a free fallback to claude-code (no need to
  build your own router)
- 11-step contract test checklist mirrored from
  `tests/agent-mock.test.js`
- "Common mistakes" table capturing the bugs that show up when
  authors copy from `claude-code.js` instead of `mock.js`

## Why a separate patch

1.10.71's MockAdapter shipped without the doc — the "reference
implementation" framing in that release implies a doc exists, so
publishing the doc as its own version makes the dependency
explicit. New authors can land on either patch and find their way
to the other.

No code change, no test change, no SDK change. SDK spec version
field auto-bumped 1.10.71 → 1.10.72 to keep the generated client in
sync with package.json.

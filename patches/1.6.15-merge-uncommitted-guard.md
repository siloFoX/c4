# Patch 1.6.15 - c4 merge uncommitted-changes guard (7.28)

## Date: 2026-04-17

## Summary

`c4 merge` now refuses to merge when the target branch (main) has
uncommitted changes, and exposes a new `--auto-stash` flag that wraps the
stash / merge / pop dance with a clean error path when pop conflicts. The
goal is to remove the manual stash/merge/pop sequence the manager had to
run on every dirty merge, and to fail loudly (instead of silently halting)
when a pop conflict would otherwise need interactive resolution.

## Root cause

`src/cli.js` merge handler called `git merge` directly without inspecting
the working tree. When main was dirty (TODO.md, package-lock.json, etc.)
the merge failed late or required the manager to:

1. `git stash push`
2. `c4 merge <name>`
3. `git stash pop`
4. resolve pop conflict if any

That sequence was repeated twice on 2026-04-17, and pop conflicts halted
the automated nightly run.

## Changes

### src/merge-guard.js (new)

Pure-JS helpers, spawn injectable for tests:

- `getDirtyEntries(repoRoot, opts)` — runs `git -C <repo> status --porcelain`
  via `spawnSync` and returns each non-empty line. Throws on git failure.
- `isDirty(repoRoot, opts)` — boolean wrapper.
- `formatEntry(line)` — turns `" M TODO.md"` into `"TODO.md (M)"`.
- `buildDirtyMessage(entries)` — operator-facing error listing the modified
  paths and pointing to `git status`, `c4 cleanup`, and `--auto-stash`.
- `stashPush(repoRoot, label, opts)` — `git stash push -m <label>`. Throws
  on non-zero exit.
- `stashPop(repoRoot, opts)` — returns `{ status, stdout, stderr }` so the
  caller can detect conflict without an exception.
- `buildPopConflictMessage(label, popResult)` — operator-facing conflict
  message that names the stash label and explains it is still in
  `git stash list`.

### src/cli.js (merge handler, additive)

- New `--auto-stash` flag parsed alongside `--skip-checks`.
- After branch/main validation and before the existing pre-merge checks,
  call `mergeGuard.getDirtyEntries(repoRoot)`. If non-empty:
  - Without `--auto-stash`: print `buildDirtyMessage`, exit 1.
  - With `--auto-stash`: stash with label `c4-merge-autostash-<target>`,
    record the label, continue.
- After the existing merge succeeds (and after the existing diff/log
  output), if a label was recorded, call `mergeGuard.stashPop`. On
  non-zero status, print `buildPopConflictMessage`, exit 1. On success,
  print `Stashed changes restored from "<label>".`
- If the merge itself throws after `--auto-stash` already pushed, the
  outer catch now also prints the stash label so the operator can recover
  with `git stash list` + `git stash pop`.
- Help banner and the `Usage:` line for `merge` mention `--auto-stash`.

The clean-tree happy path (no flag, no dirty entries) is unchanged: same
checks, same merge command, same `Merge complete:` line.

### tests/merge-uncommitted-guard.test.js (new)

30 assertions across 9 suites:

- `getDirtyEntries` clean / dirty / git-error / null-spawn cases.
- `isDirty` boolean cases.
- `formatEntry` modified / untracked / staged-add / short-input cases.
- `buildDirtyMessage` lists files and suggests fixes.
- `stashPush` failure / success / argv-quoting cases.
- `stashPop` clean / conflict / null-spawn cases.
- `buildPopConflictMessage` named label and "no data lost" line.
- End-to-end mocked flows for clean / dirty-no-flag / dirty+pop-clean /
  dirty+pop-conflict, including stash-list survival.
- `cli.js` source-level integration: `require('./merge-guard')`,
  `--auto-stash` parsing, helper calls, help banner mention.

## Verification

- `npm test` — 55 suites pass (was 54 before this patch).
- New tests run in 60 ms; full suite in ~4.3 s.
- `git status --porcelain` is the only new command introduced; no compound
  shell strings; all new git invocations go through `spawnSync` with argv
  arrays so paths/labels with spaces are safe.

## Operator notes

- `--auto-stash` is opt-in. The default behavior is to surface the dirty
  tree as an error so automated runs do not silently rewrite the working
  copy.
- `--skip-checks` still skips the test/TODO/CHANGELOG checks but does not
  bypass the dirty guard. Combining `--skip-checks --auto-stash` is the
  fastest path for "I know main is dirty and I just want this merged."

# 1.11.8 - Web redesign (2026-04)

## Goal

Finish the ARPS-inspired redesign of the C4 Web UI. Behaviour, data
flow, and API contract stay identical; every visible screen adopts
the token palette, primitives, layout shell, and lucide-react icon
vocabulary introduced by the earlier workers on the c4/web-redesign
train. Addresses TODO 8.1 (React SPA) and 8.9 (visual redesign).

## Dependency order

The redesign shipped as four sequential worker branches. Each branch
only extended what the previous one left on main:

1. **theme** (`c4/web-theme`) - HSL CSS-variable token set in
   `web/src/index.css` + `web/tailwind.config.js`, `lucide-react` +
   `clsx` + `tailwind-merge` + `class-variance-authority` pinned
   into `web/package.json`, `web/src/lib/cn.ts` helper.
2. **components** (`c4/web-components`) - shadcn-style primitives
   under `web/src/components/ui/` (`button`, `card`, `panel`,
   `input`, `label`, `badge`, `icon-button`) + barrel `index.ts`.
   Also swapped the App.tsx sidebar toggle glyphs for lucide
   `Menu` / `X` as a smoke test for the new primitive layer.
3. **layout** (`c4/web-layout`) - `web/src/components/layout/`
   dashboard shell (`AppHeader` / `TopTabs` / `Sidebar` /
   `DetailTabs` / `EmptyState`). `App.tsx` recomposed onto the
   shell and dropped from ~353 LOC to 172 LOC; Login outer
   background swapped to `bg-background` but the form JSX was left
   for the page worker.
4. **pages** (this branch, `c4/web-pages`) - every remaining
   component re-skinned onto the primitives + lucide icons:
   `Login`, `WorkerList`, `WorkerDetail`, `ChatView`,
   `ControlPanel`, `Chat`, `HierarchyTree`, `HistoryView`,
   `WorkflowEditor`, `Toast`, `WorkerActions`.

The ordering is load-bearing: pages import the primitives, which
import `cn()`, which relies on tokens defined by the theme layer.

passed, 0 failed** on main (the web-layout merge commit added one
new assertion to `tests/history-view.test.js` that caused the count
to climb to 100 on integration). Running the same suite against

preserves every existing assertion including the regex-based

## Operator notes

- **Dark-only.** The app still pins `class="dark"` on `<html>` and
  only the `.dark` token block is consumed. No light-mode toggle
  shipped with this redesign.
- **Dev server URL unchanged.** `npm --prefix web run dev` still
  serves at http://localhost:5173 (vite default) and the daemon
  keeps serving the built bundle at http://localhost:3456. External
  binding behaviour from 8.10 is untouched.
- **No new runtime deps.** The redesign uses what the theme and
  components workers already added (`lucide-react`, `clsx`,
  `tailwind-merge`, `class-variance-authority`). No Radix, no full
  shadcn/ui.
- **API layer untouched.** `web/src/lib/api.ts`, `eventSourceUrl`,
  SSE wiring, resize POST, auth flow, and every `/api/*` call site
  remain identical. The re-skin is visual only.
- **No existing tests required changes.** Only `HistoryView.tsx`'s
  Scribe button needed a whitespace-level tweak to keep the
  pre-existing regex (`\n\s*Scribe\s*\n`) in
  `tests/history-view.test.js` happy after the lucide `NotebookText`
  icon was added next to the label.
- **Validation.** `.c4-validation.json` records `test_passed: true`
  + `test_count: 100` + `build: "pass"` + the branch name so the
  manager can short-circuit pre-merge validation.

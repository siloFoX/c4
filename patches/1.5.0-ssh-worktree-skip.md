# Patch 1.5.0 - SSH target worktree 생성 방지 (4.21)

## Date: 2026-04-04

## Summary
SSH target(dgx 등 remote) worker에 불필요한 로컬 worktree 생성을 방지. SSH worker는 remote 머신에서 실행되므로 로컬에 worktree를 만들 필요가 없고, 오히려 오류를 유발할 수 있음.

## Root Cause
`sendTask()`와 `_createAndSendTask()` 모두 worktree 생성 여부를 판단할 때 `useWorktree` 옵션과 `config.worktree.enabled`만 확인하고, target이 SSH인지는 고려하지 않았음. SSH target worker에 task를 보내면 로컬에 불필요한 worktree 디렉토리가 생성되고, remote에서는 해당 worktree 경로가 존재하지 않아 `cd` 명령이 실패할 수 있었음.

## Changes

### src/pty-manager.js
- `_createAndSendTask()` (line 900-904): worktree 생성 전 `_resolveTarget()`으로 target type 확인, `type === 'ssh'`이면 `useWorktree = false` 강제
- `sendTask()` (line 2076-2080): 동일한 SSH target 검사 추가

### tests/pending-task-worktree.test.js
- MockPtyManager에 `_resolveTarget()` 메서드 추가
- mock `_createAndSendTask`에 SSH target 검사 로직 반영
- 3개 테스트 추가:
  - SSH target일 때 worktree 생성 스킵 확인
  - local target일 때 worktree 정상 생성 확인
  - useWorktree 미지정이어도 SSH면 스킵 확인

# Patch 1.5.0 - pending-task worktree 미생성 버그 수정 (BF-1)

## Date: 2026-04-04

## Summary
`_createAndSendTask()`에서 worktree 생성 로직이 누락되어, 새 워커 생성과 동시에 task 전달 시 worktree 없이 원본 repo에서 작업이 실행되던 버그 수정.

## Root Cause
`sendTask()`(기존 워커에 task 전달)에는 worktree 생성 로직이 있지만, `_createAndSendTask()`(새 워커 생성+task 전달)에는 해당 로직이 없었음. `_pendingTask`에 `useWorktree: true`만 저장하고 실제 worktree를 생성하지 않아, idle handler나 30초 fallback에서 `_buildTaskText()` 호출 시 `worker.worktree`가 undefined여서 `cd worktree` 명령이 누락됨.

## Changes

### src/pty-manager.js
- `_createAndSendTask()` (line 924 이후): `create()` 호출과 `_pendingTask` 저장 사이에 worktree 생성 로직 추가
  - `sendTask()`의 line 1944-1981 패턴 복제
  - `entry.useWorktree`가 true이고 `config.worktree.enabled`가 false가 아닐 때:
    - `_detectRepoRoot(entry.projectRoot)` 호출
    - `_worktreePath(repoRoot, entry.name)` 으로 경로 생성
    - `_createWorktree(repoRoot, worktreePath, branch)` 호출
    - `w.worktree`, `w.worktreeRepoRoot`, `w.branch` 할당
    - `_writeWorkerSettings()` 호출로 .claude/settings.json 생성
  - 에러 처리: `_createWorktree` 실패 시 snapshots에 경고 기록, task 전달은 계속 진행
  - `_writeWorkerSettings` 실패 시 snapshots에 경고 기록, worktree 자체는 유지

### tests/pending-task-worktree.test.js (new)
- 13개 유닛 테스트

  - _buildTaskText에 cd 명령 포함 확인
  - useWorktree=false, useBranch=false, config.enabled=false 시 스킵 확인
  - repoRoot 미감지 시 스킵 확인
  - branch 기본값 c4/<name> 확인
  - _writeWorkerSettings 호출 확인
  - _createWorktree 실패 시 graceful 처리 확인
  - _writeWorkerSettings 실패 시 worktree 유지 확인
  - worktree가 _pendingTask보다 먼저 설정되는지 확인
  - entry.projectRoot 사용 확인

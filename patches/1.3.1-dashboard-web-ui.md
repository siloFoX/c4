# Patch 1.3.1 - Dashboard Web UI (4.3)

## Problem
No web-based visibility into worker states. Users had to rely on CLI `c4 list` to check worker status, which is not accessible from mobile devices.

## Changes

### daemon.js
- `escapeHtml(str)`: XSS protection for HTML rendering
- `renderDashboard(listData)`: Pure HTML string renderer for dashboard
  - Worker table: name, status (with color dot), target, branch, phase, intervention, snapshots, PID
  - Stats bar: total workers, busy, idle, exited, queued counts
  - Queued tasks section (conditional)
  - Lost workers section (conditional)
  - Health check timestamp
  - Dark theme, responsive viewport
  - 30-second auto-refresh via inline script
- `GET /dashboard` route: serves HTML response with Content-Type text/html

### tests/dashboard.test.js
- 17 unit tests: escapeHtml (3 tests), renderDashboard (14 tests)
- Tests cover: empty state, worker rendering, queued/lost sections, XSS, stats, health check, responsiveness

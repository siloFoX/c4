# Patch 1.5.0 - 좀비 데몬 정리 (4.21)

## Date: 2026-04-04

## Summary
`daemon stop` 명령이 프로세스를 확실히 죽이도록 수정. SIGKILL 후에도 종료 확인 없이 `{ ok: true }` 반환하던 버그 해결.

## Root Cause
- `stop()` 함수가 SIGKILL 전송 후 500ms만 대기하고 프로세스 생존 여부를 확인하지 않음
- 프로세스가 살아있어도 `removePid()` 후 `{ ok: true }` 반환 -> 좀비 프로세스 발생
- Windows에서 taskkill /F 실패 후 무의미한 SIGKILL 시도

## Changes

### src/daemon-manager.js
- `stop()`: SIGTERM 후 매 반복마다 종료 확인, 죽으면 즉시 `removePid()` + 반환
- kill 에러 발생 시 race condition 처리 (프로세스가 kill 호출 사이에 죽는 경우)
- SIGKILL 에스컬레이션을 Unix 전용으로 분리 (Windows는 taskkill /F로 충분)
- SIGKILL 후 최대 2초(10x200ms) 동안 종료 확인 루프
- 모든 kill이 실패하면 `{ error: "process survived SIGTERM and SIGKILL" }` 반환

### tests/daemon-stop.test.js (NEW)
- 9개 유닛 테스트: 정상 종료, SIGKILL 에스컬레이션, 좀비 생존 에러, Windows 경로, race condition, PID 없음 케이스

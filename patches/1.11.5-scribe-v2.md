# 1.11.5 — Scribe v2: structured event log + query API (10.9)

## Why

`src/scribe.js` summarises Claude Code session transcripts into
`docs/session-context.md`, which is great for surviving context
compaction but terrible for "what happened to worker `w3` between
10:05 and 10:08?" — the output is prose, categorised by heuristic
Korean/English keyword matching, and has no stable shape a tool can
filter. When an L4 autonomous run splits across 30 workers and 24
hours, an operator needs an event-driven record that survives
restarts and answers time-/worker-/type-scoped queries without
replaying an entire transcript.

10.9 adds a second, parallel layer — Scribe v2 — that records typed
JSONL events from the daemon's event points. `src/scribe.js` is left
untouched; v2 is strictly additive so existing
`c4 scribe start/stop/status/scan` keeps working.

## What lands

### `src/scribe-v2.js` (new)

- **File layout.** One append-only JSONL file per UTC day at
  `~/.c4/events-YYYY-MM-DD.jsonl`. UTC so DST shifts never split one
  logical day across two files. `config.scribeV2.logDir` overrides
  the directory for test clusters without touching the operator's
  real log.
- **Event shape.** Each line is one event:
  ```json
  {"id":"lmx9k0n8-a1b2c3d4","ts":"2026-04-18T10:05:00.000Z","type":"task_start","worker":"w1","task_id":null,"payload":{"branch":"c4/foo","task":"fix X","tier":"worker","model":"sonnet"}}
  ```
  `id` = `Date.now().toString(36) + '-' + 8 hex` so ids are roughly
  sorted yet collision-safe within one ms. `ts` is ISO 8601 UTC.
  `worker` and `task_id` default to `null` when not supplied so the
  shape is stable.
- **Canonical 11-type enum** (`EVENT_TYPES`):
  `task_start`, `task_complete`, `worker_spawn`, `worker_close`,
  `tool_call`, `approval_request`, `approval_grant`, `merge_attempt`,
  `merge_success`, `halt`, `error`. Unknown type strings are rejected
  at `record()` so typos surface at the call site.
- **`class ScribeV2`**:
  - `record(event)` — validates type, writes one line via
    `appendFileSync` wrapped in try/catch so a broken disk never
    breaks the daemon request path. Returns the stored event on
    success (with `id`/`ts` back-filled) or `null` on rejection.
    Using `appendFileSync` keeps ordering deterministic under bursts
    because JS is single-threaded and the OS-level O_APPEND flag
    gives per-line atomicity.
  - `query({from, to, types, workers, limit, reverse})` — accepts
    `from`/`to` as ISO string, numeric ms, or `Date`; `types` /
    `workers` as string or array; `limit > 0` caps the result count;
    `reverse=true` returns newest-first. Prunes non-overlapping day
    files without reading them and survives corrupt JSONL lines
    (JSON.parse errors are skipped, never thrown).
  - `contextAround(target, minutesBefore=5, minutesAfter=5)` —
    resolves `target` from an event id, ISO timestamp, `Date`, or
    numeric ms and returns every event in the window regardless of
    type/worker so the caller sees the full cross-section.
  - `findById(id)` — walks day files newest-first so recent ids
    resolve without scanning ancient days.
  - `listDays()` — every discovered `YYYY-MM-DD` newest-first.
- **Module helpers.** `EVENT_TYPES`, `FILE_PREFIX` / `FILE_SUFFIX` /
  `FILE_PATTERN`, `defaultLogDir`, `isValidEventType`, `formatYMD`,
  `parseYMD`, `nextId`, `normalizePayload`, `getShared`,
  `resetShared`.

### Daemon wiring (`src/daemon.js`)

- Shared `scribeLog` singleton via
  `scribeV2Mod.getShared({logDir: cfg.scribeV2?.logDir})`.
- `safeRecord(type, {worker, task_id, payload})` wrapper runs
  alongside every existing `safeEmit(...)` call. Failures are
  swallowed the same way Slack emit failures are.
- **Wired points**:
  - `worker_spawn` on /create — payload
    `{target, command, tier, pid}`.
  - `task_start` on /task — payload
    `{branch, task, profile, autoMode, tier, model}`.
  - `approval_request` / `approval_grant` on /approve — split on the
    existing `granted` flag; payload
    `{optionNumber, granted}`.
  - `merge_attempt` before `runPreMergeChecks` — payload
    `{branch, skipChecks, resolvedFrom}`.
  - `merge_success` after `performMerge` returns success — payload
    `{branch, sha, summary}`.
  - `error` on merge failure — payload
    `{source:'merge', branch, message}`.
  - `worker_close` on /close — empty payload.
  - `halt` on the `notifyStall` bridge — payload `{reason}`.
- **New endpoints**:
  - `GET /events/query?from&to&types&workers&limit&reverse` (types /
    workers comma-separated) -> `{events, count}`.
  - `GET /events/context?target&minutesBefore&minutesAfter` ->
    `{events, count, target}`.

### CLI wiring (`src/cli.js`)

- `c4 events [--from ISO] [--to ISO] [--type a,b] [--worker x,y]
  [--limit N] [--reverse] [--json]` — filters the structured log.
  Default output is one event per line, aligned
  (`ts type worker key=value ...`), so it stays tail-able. `--json`
  mode emits raw JSONL so callers can pipe through `jq`.
- `c4 events --around <id|ISO> [--window MINUTES]` — pulls the +/-
  window around an event id or timestamp; default window is 5
  minutes. Useful for "show everything around the halt at 10:05".

## Tests

`tests/scribe-v2.test.js` — 38 assertions across six suites:

1. **Helpers** — `EVENT_TYPES` membership (all 11 canonical types),
   `defaultLogDir` shape, `isValidEventType` rejection of unknowns /
   non-strings, `formatYMD` UTC correctness, `parseYMD` round-trip +
   rejection of invalid strings, `FILE_PATTERN` match, `nextId`
   uniqueness, `normalizePayload` coercion of null / array /
   primitive -> `{}`.
2. **`record()`** — canonical JSONL shape, unknown-type rejection,
   null/missing-object rejection, default null worker/task_id,
   caller-supplied `ts` drops the event into the right day file,
   auto-creates the log dir on first write, preserves append order
   within a file, writes to different files when `ts` straddles a
   day boundary.
3. **`query()`** — no-filter returns every event chronologically,
   time range across multiple day files, single-type filter, string
   form of `types`, single-worker filter, limit cap, reverse+limit,
   empty-array filter behaves as no filter, unknown types yield
   zero, corrupt JSONL lines skipped without throwing, query on an
   empty log dir returns `[]`, from/to prunes non-overlapping day
   files without reading them.
4. **`contextAround()`** — ISO target, event id target, default +/-
   5-minute window, distant target returns empty, unresolvable
   target returns `[]`, `Date` target.
5. **`findById` + `listDays`** — locates a recorded event by id,
   `listDays` returns every day newest-first, unrelated files in
   the log dir are ignored.
6. **Shared instance** — `getShared` returns the same singleton
   until `resetShared`.

Full suite: **97 / 97 pass** (96 pre-existing + 1 scribe-v2).

## Limitations

- The Web UI timeline dashboard is deferred (explicit scope cut in
  the 10.9 task spec).
- `tool_call` is declared in the canonical enum but not wired from
  the daemon — it needs hook integration into `hook-relay.js` /
  `hook-event-log.js` so individual Edit/Write/Bash calls flow
  through. That's a follow-up.
- `task_complete` is likewise declared but not emitted — the daemon
  has no synchronous "task done" signal today (workers go idle
  instead). A stall-watcher variant could synthesise it, but that's
  out of scope for the observability layer.
- Retention / archival / compaction — the log grows one file per
  day forever. Not a problem today (a single day fits in a few MB
  even for a busy manager), but eventually an operator will want
  to compress or archive. Separate task.

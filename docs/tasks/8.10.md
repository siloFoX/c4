# TODO 8.10: Web UI 외부 접근 지원 (vite + daemon host + c4 init 통합)

## 배경
현재 웹 UI가 외부 IP에서 접근 불가.
- `vite.config.ts`에 `server.host` 없음 → vite 시작해도 localhost만 바인딩
- c4 daemon도 `127.0.0.1:3456` 하드코딩 → 원격 브라우저에서 /api 프록시 호출 불가
- 40 서버에서 `curl http://192.168.10.40:5173/` → Connection refused 재현 확인

## 목표
- vite dev server가 외부 IP로 바인딩
- c4 daemon이 config.json 기반으로 bind host 선택
- c4 init이 대화형으로 외부 접근 세팅 지원
- 데몬 재시작 포함

## 수정 범위

### 1. vite.config.ts
- `server.host: '0.0.0.0'` 추가 (또는 `true`)
- 필요 시 `server.port: 5173` 명시

### 2. c4 daemon bind host 설정화
- config.json schema에 `daemon.bindHost` 필드 추가 (기본값 `127.0.0.1`)
- src/daemon/server.js (또는 해당 http 서버 파일) 에서 bindHost 읽어서 listen
- localhost 기본 유지 (backward compat)
- `0.0.0.0` 또는 특정 LAN IP 옵션 지원

### 3. src/cli.js c4 init 통합
- init 후반부에 대화형 질문 추가: `"외부(LAN) 접근 활성화하시겠습니까? (y/N): "`
- yes 응답 시:
  - vite.config.ts에 `server.host: '0.0.0.0'` 자동 삽입 (Read→Edit pattern)
  - config.json의 daemon.bindHost를 `0.0.0.0`으로 설정
  - 현재 머신 IP 자동 감지 (`os.networkInterfaces()` 에서 외부 IPv4)
  - 안내 출력: `Web UI: http://<detected-ip>:5173/  (vite dev server 시작 필요)`
  - 보안 경고: `⚠ 외부 접근 활성화됨. 방화벽/인증(JWT 8.1 예정) 검토 필요`
- no (기본) 응답 시: 현 설정 유지

### 4. c4 daemon restart 자동화 (중요)
- bindHost 변경 후 자동으로 `c4 daemon restart` 실행 (또는 안내)
- 사용자가 config 편집 후 `c4 daemon reload` 또는 `restart` 해야 반영됨을 명시

### 5. README / docs/handoff.md
- "외부 접근 시작" 섹션 추가
  - `c4 init` 으로 세팅 가능
  - 수동: `npm --prefix web run dev -- --host 0.0.0.0`
  - daemon bind: `c4 config` 에서 daemon.bindHost 확인

## 5.20 CI 피드백 루프

### 목표
worker가 커밋하면 자동으로 npm test 실행. 실패 시 worker에 자동 피드백.

### 수정: src/pty-manager.js

데이터 핸들러(약 2500번 줄 부근)에서 커밋 감지 후 자동 테스트:
git commit 성공 패턴을 감지하면 자동으로 npm test 실행 지시를 보내는 것은 복잡하므로, 대신 PostToolUse hook 방식으로 구현.

_buildWorkerSettings()의 hooks 섹션에 PostToolUse hook 추가:
커밋 감지 후 npm test를 자동 실행하는 hook은 복잡하므로, 더 간단한 접근:

healthCheck()에서 worker의 worktree에 새 커밋이 있는지 확인하고, 있으면 npm test 실행 결과를 확인:

```js
// CI feedback loop (5.20): check for new commits and run tests
_checkWorkerCI(name, w) {
  if (!w.worktree || !w.alive) return null;
  const gitDir = w.worktree.replace(/\\/g, '/');
  try {
    const head = execSyncSafe(`git -C "${gitDir}" rev-parse HEAD`, {
      encoding: 'utf8', timeout: 5000, stdio: 'pipe'
    }).trim();
    if (head === w._lastCICommit) return null; // Already checked
    w._lastCICommit = head;
    
    // Check if package.json has test script
    const pkgPath = path.join(w.worktree, 'package.json');
    if (!fs.existsSync(pkgPath)) return null;
    
    // Run npm test silently
    try {
      execSyncSafe(`npm test --prefix "${gitDir}"`, {
        encoding: 'utf8', timeout: 120000, stdio: 'pipe'
      });
      return { commit: head, status: 'pass' };
    } catch (e) {
      // Test failed — send feedback to worker
      const failMsg = (e.stderr || e.stdout || '').substring(0, 500);
      this._chunkedWrite(w.proc, `\nnpm test가 실패했어. 커밋 ${head.substring(0,7)} 이후. 실패 내용:\n${failMsg}\n수정해줘.\r`);
      if (this._notifications) {
        this._notifications.pushAll(`[CI FAIL] ${name}: test failed after commit ${head.substring(0,7)}`);
      }
      return { commit: head, status: 'fail', error: failMsg };
    }
  } catch { return null; }
}
```

healthCheck()에서 호출:
```js
// CI feedback loop (5.20)
for (const [name, w] of this.workers) {
  if (w.alive && w.worktree) {
    this._checkWorkerCI(name, w);
  }
}
```

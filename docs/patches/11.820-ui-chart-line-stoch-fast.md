# 11.820 UI: chart-line-stoch-fast primitive (v1.11.838)

## Summary
Pure-SVG dual-panel React/TS primitive plotting a **Fast Stochastic
Oscillator** on the lower panel: the raw `%K` value without any SMA
smoothing. `%K` maps the close into `[0, 100]` within the rolling
high-low band across the lookback.

## Formula
```
highestHigh[i] = max(high[i - length + 1 .. i])
lowestLow[i]   = min(low[i - length + 1 .. i])
percentK[i]    = (highestHigh[i] > lowestLow[i])
                  ? (close[i] - lowestLow[i])
                    / (highestHigh[i] - lowestLow[i]) * 100
                  : null
```
`percentK[i]` is `null` during warmup or when the rolling window is
flat (`highestHigh == lowestLow`). The result is bounded in `[0,
100]`.

## Defaults
- `length = 14`
- `overbought = 80`, `oversold = 20`
- `width = 720`, `height = 460`, `panelGap = 12`

## Zones and crosses
- `overbought`: `%K >= overbought`
- `oversold`: `%K <= oversold`
- `neutral`: in between
- `none`: warmup / flat window
- `crossed`: `'up'` entering overbought, `'down'` entering oversold.
  Markers fire on threshold transitions only.

## Bit-exact testability anchors
- **CONST h=l=close=K**: degenerate window, `percentK = null`
  everywhere across `K in {0, 1, 5, 100, -3}` and `length in {3, 4,
  7, 10}`.
- **LINEAR UP** (`close[i] = i + 1`, `high = low = close`): at any
  bar `i >= length - 1` we have `lowestLow = i - length + 2`,
  `highestHigh = i + 1`, `close = i + 1`. The numerator and
  denominator both equal `length - 1`, so `percentK = 100` exactly
  (integer dyadic arithmetic). Verified for `length in {3, 4, 7,
  10}`.
- **LINEAR DOWN** (`close[i] = N - i`, mirror): `close = lowestLow`
  -> numerator = 0 -> `percentK = 0` exactly.
- **ALTERNATING** `close[i] = i % 2`: at bars where `close = 1`,
  `percentK = 100` (close at the high); at bars where `close = 0`,
  `percentK = 0` (close at the low). Tested explicitly at multiple
  bars.

## Design notes
- The LINEAR anchors exploit a property of the stochastic %K formula:
  when `high = low = close` with unit-step linear movement, both
  endpoints of the rolling window are saturated -- so the close is
  always either exactly at the high (LINEAR UP) or exactly at the
  low (LINEAR DOWN), driving %K to its bounds with no rounding
  ambiguity.
- Stoch axis is **fixed at [0, 100]** since the metric is bounded by
  construction.
- Overbought/oversold thresholds dashed `#9333ea`, midline at 50
  dashed `#475569` with a different stroke pattern.
- Default colors: %K line `#4f46e5` (indigo), thresholds `#9333ea`,
  bullish `#16a34a`, bearish `#dc2626`.
- `noUncheckedIndexedAccess` satisfied via explicit `?? null` and
  defensive guards.

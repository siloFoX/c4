# Patch 11.830 -- ChartLineHlRange primitive (v1.11.848)

## Summary

Add `<ChartLineHlRange />`, a pure-SVG dual-panel line chart that
renders a SMA-smoothed High-Low Range beneath a price line. The
smoother first computes the per-bar range `high - low` and then
averages it across the lookback:

```
range[i]    = high[i] - low[i]
avgRange[i] = mean(range[i - length + 1 .. i])
```

By the linearity of SMA, this is algebraically equivalent to the
`SMA(high) - SMA(low)` decomposition implemented by the HL
Oscillator primitive (11.829). The two primitives differ only in
operation order; for the bit-exact anchors listed below they
produce byte-identical output.

## Defaults

| Knob          | Value     |
|---------------|-----------|
| length        | 14        |
| highThreshold | 1         |
| lowThreshold  | 0         |
| rangeColor    | `#ca8a04` (mustard) |

The mustard color keeps the range panel visually distinct from the
orange HL Oscillator and amber Volume Ratio panels shipped just
before this one.

## Bit-exact anchors

- `CONST high = low = K`: per-bar range = 0, so `avgRange = 0`
  bit-exact post-warmup.
- `CONSTANT SPREAD high - low = D` (any baseLow): per-bar range
  = `D`, so `avgRange = SMA(D) = D` bit-exact across `(K, D, length)`.
- `LINEAR + CONSTANT SPREAD high = i + 1 + D, low = i + 1`:
  per-bar range = `D`, so `avgRange = D` bit-exact (drift in the
  underlying values cancels at the difference step before SMA sees
  the data).

Compared with 11.829 hl-osc, the anchors are the same because the
SMA-linearity identity holds exactly under IEEE 754 when both
inputs are well-conditioned. The narrative distinction is that
11.830 computes per-bar range first (suitable when the per-bar
spread is a useful tooltip field), and 11.829 keeps the two SMA
channels separate (suitable when the high/low SMAs themselves are
displayed as bands).

## Surface

Standard primitive surface:

- `ChartLineHlRangePoint = { x, high, low, close }`
- `runLineHlRange(data, options?) -> Run` with `ok = series.length
  >= length`.
- `computeLineHlRangeLayout({ data, width?, height?, padding?, panelGap?, length?, highThreshold?, lowThreshold? }) -> Layout`.
  Range axis auto-fits but always includes 0 and both threshold lines.
- `describeLineHlRangeChart(data, opts?) -> string` for the ARIA
  description.
- `<ChartLineHlRange />` `forwardRef<HTMLDivElement>` with legend
  toggle (`price`, `range`), config badge, threshold + zero-line
  overlays, markers on cross events, and a tooltip surfacing
  `close / high / low / range / avgRange / zone / cross`.

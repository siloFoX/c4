# Patch 11.831 -- ChartLineTrixDoubleSmoothed primitive (v1.11.849)

## Summary

Add `<ChartLineTrixDoubleSmoothed />`, a pure-SVG dual-panel line
chart that plots an additional EMA pass on top of the standard TRIX
indicator (triple-EMA-smoothed rate of change of log close). The
log transform turns multiplicative price ratios into additive
differences, so the standard TRIX formula `ema3[i] - ema3[i-1]`
expressed in basis points is the natural log-return of the smoothed
series. An additional EMA pass on TRIX (the "doubly smoothed"
variant, equivalent to a signal-line smoothing) dampens noise so
zero-line crossings are clearer trade triggers.

## Formula

```
logClose[i] = ln(close[i])  (null when close <= 0)
ema1[i]     = EMA(logClose, length)[i]
ema2[i]     = EMA(ema1,     length)[i]
ema3[i]     = EMA(ema2,     length)[i]
trix[i]     = (ema3[i] - ema3[i - 1]) * 10000   (basis points)
double[i]   = EMA(trix, signalLength)[i]
```

- Warmup: `3 * (length - 1) + 1 + signalLength - 1` indices are
  `null` (combined: three EMA passes, one difference step, one more
  EMA pass).
- `null` propagates from any `close[i] <= 0`.
- `-0` is normalized to `+0` on every arithmetic result.

## Defaults

| Knob              | Value     |
|-------------------|-----------|
| length            | 15        |
| signalLength      | 9         |
| bullishThreshold  | 0         |
| bearishThreshold  | 0         |
| trixColor         | `#6366f1` (indigo) |

Indigo distinguishes the TRIX panel from the orange/teal/mustard
volatility oscillators shipped earlier in this batch.

## EMA seed precision fix

The SMA-seeded EMA helper now performs a `min === max` check across
the seed window. When all values in the window are identical the
EMA is seeded with that exact value rather than `sum / length`,
which would otherwise drift by 1 ULP when `length` is not a power
of 2 and the constant is non-dyadic (e.g. `ln(100)`). The CONST
short-circuit `next = v === prev ? v : alpha*v + (1-alpha)*prev`
then preserves the constant through all three triple-smoothing
passes and the signal EMA, so the difference of consecutive `ema3`
values is exactly 0 and the final `doubleSmoothed` is exactly 0.

## Bit-exact anchor

- `CONST close = K > 0`: `logClose = ln(K)` constant. The min/max
  seed check installs `ln(K)` exactly. The CONST short-circuit
  preserves it through all four EMA passes. The pair-wise
  difference of consecutive `ema3` values is `ln(K) - ln(K) = 0`.
  `EMA(0) = 0`. Verified across `K in {1, 5, 100}` and
  `(length, signalLength) in {4, 7} x {2, 3}`.
- `CONST close <= 0`: `null` everywhere (log undefined).

Standard primitive surface:

- `ChartLineTrixDoubleSmoothedPoint = { x, close }`

  `close / logClose / ema3 / trix / double / zone / cross`.

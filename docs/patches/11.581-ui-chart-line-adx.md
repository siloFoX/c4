## Summary

New `ChartLineAdx` UI primitive at
`web/src/components/ui/chart-line-adx.tsx` -- a pure-SVG line chart
with an Average Directional Index (ADX) panel. The value line
occupies the top panel; the bottom panel renders the ADX
trend-strength line alongside the +DI and -DI directional indicators,
with a dashed reference line at the conventional trend threshold.
Both panels share one x-axis.

## Indicator math

### Directional movement

`computeLineAdxDirectionalMovement(values)` is Wilder's directional
movement for a single-value series. For each period the change
`value[i] - value[i-1]` splits into:

```
plusDM[i]    = change > 0 ? change  : 0
minusDM[i]   = change < 0 ? -change : 0
trueRange[i] = |change|
```

Index 0 has no prior value and reads `null` for all three series. The
plus and minus movements are mutually exclusive, and together they
partition the true range (`plusDM + minusDM = trueRange`).

### Average Directional Index (Welles Wilder)

`computeLineAdx(values, period)` runs Wilder's full ADX system. The
three directional-movement series are Wilder-smoothed over `period`
(simple-mean seed at index `period`, then
`(prev*(period-1) + x)/period`). The directional indicators are:

```
+DI = 100 * smoothedPlusDM  / smoothedTR
-DI = 100 * smoothedMinusDM / smoothedTR
DX  = 100 * |+DI - -DI| / (+DI + -DI)
ADX = Wilder-smoothed DX
```

`+DI` / `-DI` / `DX` are defined from index `period`; the ADX, after
a second smoothing window over `DX`, from index `2*period - 1`. For a
single-value series `+DI + -DI = 100` exactly, since each period's
plus and minus movement partition the true range.

### Worked fixture

`values = [10, 12, 11, 14, 13, 16]`, `period = 2`:


## API

- `getLineAdxFinitePoints(points)` -- drop points with a non-finite
  `x` or `value`.
- `normalizeLineAdxPeriod(period, fallback)` -- coerce a period to a
  positive integer; non-finite or sub-1 falls back, fractional floors.
- `computeLineAdxDirectionalMovement(values)` -- the +DM / -DM / TR
  triple.
- `computeLineAdx(values, period)` -- the `{ plusDI, minusDI, dx, adx }`
  series.
- `runLineAdx(points, options?)` -- sort finite points by `x`, compute
  the four series, return `{ series, period, plusDI, minusDI, dx, adx,
  samples, adxFinal, plusDiFinal, minusDiFinal, ok }`. `ok` is false
  for fewer than two points.
- `computeLineAdxLayout(options)` -- stack a value panel above a fixed
  0-100 ADX panel sharing one x-axis. Returns panels, ticks, the value
  path, ADX / +DI / -DI paths, ADX markers, and the projected
  threshold reference line `y`.
- `describeLineAdxChart(data, options?)` -- ARIA description string.

## Component

`ChartLineAdx` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge (period + final ADX), a four-series legend (Value / ADX / +DI /
-DI) with toggle, a dashed threshold reference line, ADX markers on
the ADX line, a hover tooltip exposing the value, +DI, -DI, DX, and
ADX for the hovered period, and a `motion-safe:animate-fade-in`
entrance. Every node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-atr** -- chart-line-atr plots the Wilder ATR alone
  as a volatility study. chart-line-adx uses the true range only as
  the denominator that normalizes the directional movements into a
  0-100 trend-strength reading; it plots three derived lines (ADX,
  +DI, -DI), none of which is the ATR itself.
- **vs chart-line-macd / -rsi / -stochastic** -- those oscillators
  measure price *direction* and momentum. The ADX is direction-
  agnostic: it measures the *strength* of a trend regardless of
  whether the trend is up or down. The +DI / -DI lines carry the
  direction separately.

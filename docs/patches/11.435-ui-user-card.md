## Summary

Add a new `<UserCard>` UI primitive in
`web/src/components/ui/user-card.tsx` -- avatar + name +
role display with an opt-in hover-card popover that shows
an extended profile (bio, email, location, joined-at, link
list). The name row exposes an online-status dot in the
bottom-right of the avatar. Both the inline chip and the
popover are keyboard-reachable + screen-reader-accessible.

## API

```ts
type UserCardStatus =
  | 'online' | 'away' | 'busy' | 'offline' | 'unknown';

interface UserCardLink {
  label: ReactNode;
  href: string;
  external?: boolean;
}

interface UserCardProfile {
  email?: string;
  bio?: ReactNode;
  location?: ReactNode;
  joinedAt?: string | number | Date;
  links?: readonly UserCardLink[];
}

interface UserCardProps {
  name: string;
  role?: string;
  avatarSrc?: string;
  avatarAlt?: string;
  status?: UserCardStatus;          // default 'unknown'
  statusLabel?: string;
  profile?: UserCardProfile;
  open?: boolean;                   // controlled
  defaultOpen?: boolean;
  onOpenChange?: (open: boolean) => void;
  hoverDelay?: number;              // default 150 ms
  size?: 'sm' | 'md' | 'lg';        // default 'md'
  href?: string;
  onClick?: () => void;
  showRole?: boolean;               // default true
  showStatus?: boolean;             // default true
  showHoverCard?: boolean;          // default true
  ariaLabel?: string;
  className?: string;
}
```

## Behaviour

- Avatar renders either the supplied `avatarSrc` or a
  fallback chip with the user's initials (computed from the
  first + last word of `name`).
- Status dot anchors to the bottom-right of the avatar with
  a `ring-2 ring-card` outline so it stays readable on any
  background; carries `aria-label="Status: <Label>"` for
  screen readers.
- Trigger swaps between a static `<span>`, a `<button>`
  (when `onClick` is supplied) and an `<a>` (when `href` is
  supplied) so adopters can drop the chip into different
  call sites without changing the rendered structure.
- Hover-card popover opens on mouse-enter / focus + closes
  on mouse-leave / blur with a symmetric `hoverDelay` (150
  ms default). Re-entering the trigger or the popover
  before the close timer fires cancels the close so the
  popover does not flicker when the user moves their cursor
  through the gap.
- Controlled `open` always wins for the rendered state;
  `onOpenChange` still fires on every schedule transition.
- The popover only renders when `profile` is supplied and
  `showHoverCard` is true (the default) -- adopters that
  only need the chip render zero overlay surface.
- Popover content: header (avatar + name + role + status
  label), optional bio paragraph, detail list (email,
  location, joined-at) with per-row icons, optional link
  list with external-link hint.

## Pure helpers (exported)

```ts
DEFAULT_USER_CARD_HOVER_DELAY = 150
getInitials(name): string
getStatusLabel(status): string
getStatusDotClass(status): string
formatJoinedDate(value): string
```

- `getInitials` returns an empty string for empty input,
  the first char of a single word, or `first + last`
  initials for multi-word names. Extra whitespace
  collapses.
- `formatJoinedDate` returns `YYYY-MM-DD` or an empty
  string for falsy / unparseable input. Accepts Date /
  number / string.

## ARIA + data attributes

- Region: `role="region"` + `aria-label` (default:
  `"<name> card"`).
- Trigger: `aria-describedby` points at the popover id
  when the hover card is enabled.
- Avatar status dot: `aria-label="Status: <Label>"`.
- Popover: `role="tooltip"` +
  `id="user-card-popover-<name>"` (name spaces replaced

## Tests (42 cases)

  first+last; 3+ words still first+last; whitespace
  collapse.
- `getStatusLabel` per status.
- `getStatusDotClass` per status non-empty.
- `formatJoinedDate` undefined / unparseable; ISO string;
  Date; number.
- Constants - default hover delay 150.
- Component: region + default + custom `ariaLabel`; name +
  role render; initials fallback; avatarSrc + alt; default
  alt fall-back to name.
- Status dot renders + aria-label + data-status;
  `showStatus=false` hides it; `showRole=false` hides
  role.
- Root data attrs reflect size / status / interactive.
- `href` produces anchor; `onClick` (no href) produces
  button; neither produces a span.
- Hover schedule opens after delay; mouse-leave closes
  after delay; focus opens immediately; blur schedules
  close.
- `open=true` forces popover; popover hidden when profile
  missing.
- `showHoverCard=false` suppresses the popover entirely.
- Popover renders bio + email + location + joined.
- Links render with external attrs when `external:true`;
  internal links keep `target` absent.
- Email link uses `mailto:`.
- Popover status label reflects status.
- Controlled `open` pins rendered state.
- `hoverDelay` prop overrides the default delay.
- Size data attr swaps the avatar pixel dimension.
- Stable `displayName` + `forwardRef` to the root region.
- `aria-describedby` points at the popover id when the
  hover card is enabled.

## Pairs with existing primitives

- `<AvatarGroup>` for stacked avatars (this primitive
  ships the single-user card).
- `<EnvBadge>` (11.434) for the chrome strip beside the
  signed-in user chip.
- `<Tooltip>` for inline textual tooltips when no extended
  profile is needed.

## Out of scope

- Live status subscription (host owns the source of truth
  + passes the current `status`).
- Click-to-message / DM flow (host wires `onClick`).
- Profile editing inside the card (read-only; host
  composes with `<Drawer>` for edit flows).
- Multi-user comparison / diff view (host composes
  multiple chips).

# 11.535 -- web/src/components/ui/chart-line-burndown.tsx

Pure-SVG **burndown chart** -- the canonical agile / project-
management view of remaining work over time. Two specific lines
share an x axis: the **ideal** trajectory (a dashed straight
line from the initial scope at the start date down to zero at
the deadline, computed entirely from `startTotal` + `startX` +
`endX`) and the **actual** trajectory (the observed remaining
work, supplied as `data`). The region between them is shaded:
green where the team is **ahead** of schedule (actual below
ideal -- less remaining than planned) and red where the team is
**behind** (actual above ideal -- more remaining than planned).
A status badge in the top-left calls out the current status
(`ahead` / `behind` / `on-track`) with the delta from ideal.

## Distinct from related primitives

- `<ChartLineTarget>` (11.518): vs a FLAT user-supplied target
  with over / under shading. Burndown's reference is a
  DIAGONAL ideal trajectory computed from `startTotal` +
  deadline.
- `<ChartLineForecast>` (11.519): historical solid + forecast
  dashed split on the TIME axis. Burndown overlays ideal AND
  actual on the SAME time axis as two distinct lines.
- `<ChartLineCumulative>` (11.520): running cumulative SUM
  RISING over time. Burndown is the opposite -- remaining work
  DECREASING (ideally) over time.
- `<ChartLineComparison>` (11.511): two arbitrary curves with
  difference shading. Burndown's "ideal" is computed from
  inputs, and the shading is colour-coded by which side wins
  (ahead vs behind).
- `<ChartLinePeriodCompare>` (11.531): two periods of the SAME
  metric. Burndown overlays REMAINING WORK vs an IDEAL plan,
  not two recurrences of one metric.
- `<ChartLineBaseline>` (11.510): single flat horizontal
  reference. Burndown's reference is a DIAGONAL rule.

The canonical "sprint health" / "release tracking" visualisation
used by every agile team -- answering "are we tracking the
plan?" at a glance.

## Public surface

- `<ChartLineBurndown data={...} />`.
- `data: ChartLineBurndownPoint[]` -- `{x, y}` where x is the
  time index (day, hour, etc.) and y is the **remaining work**
  at that time.
- `startTotal` -- initial scope. Defaults to the first sample's
  y value.
- `startX` / `endX` -- start of sprint + deadline x values.
  Defaults to first / last sample x.
- `flatEpsilon` (default `0`) -- equality band for the
  on-track classification.
- `aheadColor` / `behindColor` / `onTrackColor` -- per-status
  shading + badge + tooltip + dot tinting (where applicable).
- `actualColor` / `idealColor` -- line colours.
- `idealDashArray` (default `'6 4'`).
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showStatusBadge`, `showAheadFill`,
  `showBehindFill`, `showIdeal`, `animate`.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX` / `formatDelta`.
- `actualLabel` (default `'Actual'`) / `idealLabel` (default
  `'Ideal'`).
- `onPointClick({point})`.

## Pure helpers

- `getBurndownFinitePoints(points)` -- finite-only filter.
- `computeBurndownIdealY(x, startX, endX, startTotal)` --
  canonical ideal-trajectory helper. Returns `startTotal` for
  `x <= startX`, `0` for `x >= endX`, and a linear
  interpolation between for any x in the open interval.
  Non-finite inputs -> NaN. Collapse (`startX === endX`) holds
  the curve as a step at the seam.
- `classifyBurndownStatus(delta, flatEpsilon?)` -- `'ahead' |
  'behind' | 'on-track'`. Positive delta (actual > ideal) ->
  behind; negative -> ahead; |delta| <= epsilon -> on-track.
  Non-finite delta -> on-track.
- `computeBurndownLayout({data, ...})` -- returns
  `{ok, ...projection metadata..., actualPoints, actualPath,
  idealStartPx, idealStartPy, idealEndPx, idealEndPy, idealPath,
  aheadFillPath, behindFillPath, startTotal, startX, endX,
  currentRemaining, currentDelta, currentStatus, aheadCount,
  behindCount, onTrackCount, finiteCount, totalCount,
  daysRemaining}`. The y range is always expanded to include
  both `0` and `startTotal` so the ideal endpoints sit at the
  axis extremes. Per-point carries `{index, x, y, px, py,
  idealY, idealPy, delta, status}`. Sorts ascending by x. Drops
  non-finite from `data`. `currentStatus` derived from the last
  sample; `daysRemaining = max(0, endX - lastX)`.
- `describeBurndownChart(data, options?)` -- builds the screen-
  reader description (scope + current remaining + current
  status + delta).

## Shading construction

The ahead / behind shading is built as a list of contiguous run
segments where the per-point status matches the target side.
Each run yields one closed polygon walking the actual curve
forward from runStart to runEnd, then the ideal curve backward
from runEnd to runStart, closed with `Z`. Empty / single-run
segments still emit valid SVG so partial sprints render cleanly.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Ideal line + actual line + every dot is
  `role="graphics-symbol"` + `tabIndex={0}`. Dot aria-label
  surfaces the day, remaining, ideal, status, and absolute
  delta.
- Every node carries `data-section="chart-line-burndown-..."`
  for test selectors.
- Ideal line has `data-kind="ideal"`; actual line has
  `data-kind="actual"`.
- Shading paths expose `data-status` (`'ahead'` | `'behind'`).
- Dots expose `data-ideal-y`, `data-delta`, `data-status`.
- Root mirrors `data-total-points`, `data-start-total`,
  `data-current-remaining`, `data-current-delta`,
  `data-status`, `data-ahead-count`, `data-behind-count`,
  `data-on-track-count`, `data-days-remaining`, `data-animate`.
- Badge mirrors the same status + delta + remaining +
  daysRemaining.

## Tooltip

Hovering a dot shows the actual label in actual-colour, x, bold
`remaining: <y>` row, `ideal: <idealY>` row, and a coloured
`<status> (+/-delta)` row tinted by status.

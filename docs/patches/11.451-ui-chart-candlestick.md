## Summary

Add a new `<ChartCandlestick>` UI primitive in
`web/src/components/ui/chart-candlestick.tsx` -- pure-SVG
OHLC candlestick chart. Each candle paints a solid body
from open->close and a thin wick from low to high. Bullish
candles (close > open) take the up colour, bearish candles
(close < open) take the down colour, dojis (close == open)
collapse to a hairline at the same y. Hovering a candle
opens a tooltip with all four OHLC values + the formatted
date plus an optional volume row. Per-candle click handler.

## API

```ts
type ChartCandlestickDirection = 'up' | 'down' | 'doji';

interface ChartCandlestickPoint {
  date: string | number;       // ISO string or epoch ms
  open: number;
  high: number;
  low: number;
  close: number;
  volume?: number;
}

interface ChartCandlestickProps {
  data: readonly ChartCandlestickPoint[];
  width?: number;                  // default 640
  height?: number;                 // default 320
  padding?: number;                // default 36
  upColor?: string;                // default '#16a34a'
  downColor?: string;              // default '#dc2626'
  dojiColor?: string;              // default '#475569'
  wickColor?: string;              // default: candle colour
  showTooltip?: boolean;           // default true
  showAxisTicks?: boolean;         // default true
  showGrid?: boolean;              // default true
  animate?: boolean;               // default true
  className?: string;
  ariaLabel?: string;              // default 'Candlestick chart'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  formatDate?: (d: string | number) => string;
  tickCount?: number;              // default 5
  candleWidth?: number;
  candleGap?: number;              // default 2
  onCandleClick?: (args: {
    point: ChartCandlestickPoint;
    index: number;
    direction: ChartCandlestickDirection;
  }) => void;
}
```

## Behaviour

- Renders a `<div role="region">` wrapping `<svg
  role="img">` carrying the same `aria-label`.
- Vertical scale: `getCandlestickBounds` returns
  `min(low)` / `max(high)` across the series; collapsed
  range auto-expands by `+/- 0.5`; empty data falls
  back to `(0, 1)`.
- Candle geometry: bodies sit at
  `[min(yOpen, yClose), |yOpen - yClose|]`, clamped to a
  minimum 1 px so a doji (close == open) still renders
  as a hairline. Wicks paint as a thin vertical line
  from `yHigh` to `yLow` through the body centre.
- Direction logic via `getCandlestickDirection(open,
  close)`: `up` for close > open, `down` for close <
  open, `doji` for close == open or non-finite inputs.
- Colour resolution via `getCandleColor`. Per-prop
  `wickColor` overrides the candle colour for the wick
  if supplied; otherwise wick matches candle.
- Candle width: `candleWidth` prop wins; otherwise
  derived from `(innerWidth - gap * (n - 1)) / n` (min
  1 px). `candleGap` controls horizontal spacing.
- Hover / focus: enlarges the body (stroke + full
  opacity) and opens a tooltip with date + O / H / L / C
  + optional V row. Tooltip suppressed via
  `showTooltip=false`.
- Mount animation: `motion-safe:animate-fade-in` per
  candle group. `animate=false` removes the class.
- Default ARIA description: `"Candlestick chart, N
  sessions, M bullish. <first date> open <open> ->
  <last date> close <close>."` -- `ariaDescription`
  overrides; empty data returns `"No data"`.

## Pure helpers (exported)

```ts
DEFAULT_CHART_CANDLESTICK_WIDTH            = 640
DEFAULT_CHART_CANDLESTICK_HEIGHT           = 320
DEFAULT_CHART_CANDLESTICK_PADDING          = 36
DEFAULT_CHART_CANDLESTICK_UP_COLOR         = '#16a34a'
DEFAULT_CHART_CANDLESTICK_DOWN_COLOR       = '#dc2626'
DEFAULT_CHART_CANDLESTICK_DOJI_COLOR       = '#475569'
DEFAULT_CHART_CANDLESTICK_TICK_COUNT       = 5
DEFAULT_CHART_CANDLESTICK_CANDLE_GAP       = 2
DEFAULT_CHART_CANDLESTICK_MIN_BODY_HEIGHT  = 1

getCandlestickBounds(data): { min, max }
getCandlestickDirection(open, close): 'up' | 'down' | 'doji'
getCandleColor(direction, up, down, doji): string
getCandlestickTicks(min, max, count?): number[]
formatCandleDate(date, formatter?): string
describeCandlestickChart(data, formatValue?, formatDate?): string
```

- `getCandlestickBounds` ignores non-finite candle
  values; empty / non-finite-only data returns `(0, 1)`;
  collapsed range expands `+/- 0.5`.
- `getCandlestickDirection` is the canonical mapping;
  non-finite inputs collapse to `doji`.
- `getCandleColor` mirrors the component's colour
  decision so adopters can paint their own overlay
  using the same palette.
- `getCandlestickTicks` is the standard chart-tick
  helper used across the chart family (evenly-spaced
  inclusive endpoints; `[min]` for collapsed ranges;
  min count 2).
- `formatCandleDate` is a small date helper: passes
  through strings, converts epoch numbers to
  `YYYY-MM-DD`, falls back to `String(value)` when the
  epoch is non-finite. Custom formatter wins.
- `describeCandlestickChart` produces the default ARIA
  summary; honours both `formatValue` and `formatDate`.

## ARIA + data attributes

- Root: `<div role="region">` + `aria-label`.
- SVG: `role="img"` + same `aria-label`.
- sr-only span:
  `data-section="chart-candlestick-aria-desc"` carries
  the auto-summary or `ariaDescription` override.
- Each body rect: `role="graphics-symbol"` +
  `tabIndex={0}`. Aria-label per body:
  `"<date> OHLC: open <o>, high <h>, low <l>, close <c>"`.
- `data-section` on every node:
  - `chart-candlestick` (root)
  - `chart-candlestick-aria-desc`
  - `chart-candlestick-svg`
  - `chart-candlestick-axis-y`
  - `chart-candlestick-tick` + `-tick-label`
  - `chart-candlestick-candle` (per-candle group)
  - `chart-candlestick-wick`
  - `chart-candlestick-body`
  - `chart-candlestick-tooltip` + `-tooltip-date` /
    `-tooltip-open` / `-tooltip-high` / `-tooltip-low` /
    `-tooltip-close` / `-tooltip-volume`
- Root mirrors `data-candle-count`, `data-animate`.
- Candle groups mirror `data-candle-index`,
  `data-candle-date`, `data-candle-direction`,
  `data-candle-color`, `data-hovered`.

## Out of scope

- Volume bars under the candles (host stacks a
  `<ChartBar>` underneath when needed).
- Moving averages / Bollinger / RSI overlays (host
  wraps with a custom SVG layer using the exported
  scale helpers).
- Date-axis ticks (host renders its own time-axis
  underneath; the primitive is value-only so a
  monthly summary or per-tick label can be wired
  without re-doing geometry).
- Animated value transitions on data change (snap v1;
  mount fade only).
- Streaming / live tick updates (host owns the data
  pipeline and re-renders on update).

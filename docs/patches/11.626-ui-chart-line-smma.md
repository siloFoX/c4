## Summary

New `ChartLineSmma` UI primitive at
`web/src/components/ui/chart-line-smma.tsx` -- a pure-SVG line chart
with a Smoothed Moving Average overlay using Wilder running-mean
smoothing.

## Indicator math

Welles Wilder's Smoothed Moving Average:

```
SMMA[period-1] = mean of the first `period` prices       (the seed)
SMMA[i]        = (SMMA[i-1] * (period - 1) + price[i]) / period
```

The first value is the simple mean of the first `period` prices;
each later bar folds in by the **running-mean recursion**. That is
an exponential average with the slow weight `1 / period` -- every
past value still contributes with an exponentially decaying weight,
so no value is ever fully dropped and the average stays
exceptionally smooth. The SMMA is defined from index `period - 1`
onward.

### Worked fixture

`values = [3, 6, 9, 12, 11, 9, 6]`, `period 3`. The prices are
chosen so every step of the running mean lands on an integer:

```
SMMA[2] = (3 + 6 + 9) / 3   = 6
SMMA[3] = (6*2 + 12) / 3    = 8
SMMA[4] = (8*2 + 11) / 3    = 9
SMMA[5] = (9*2 + 9) / 3     = 9
SMMA[6] = (9*2 + 6) / 3     = 8
SMMA = [null, null, 6, 8, 9, 9, 8]
```

Every value is exact, so the tests assert the series with
`toEqual`.

## API

- `getLineSmmaFinitePoints(points)` -- drop points with a non-finite
  `x` or `value`.
- `normalizeLineSmmaPeriod(period, fallback)` -- coerce to a
  positive integer; non-finite or sub-1 falls back, fractional
  floors.
- `computeLineSmma(values, period)` -- the SMMA series, null through
  the warm-up.
- `runLineSmma(points, options?)` -- sort finite points by `x`,
  compute the series, return `{ series, period, smma, samples,
  smmaFinal, smmaMin, smmaMax, aboveCount, belowCount, ok }`. Each
  sample carries the price's `above` / `below` / `on` position
  versus the SMMA. `ok` is false for fewer than two points.
- `computeLineSmmaLayout(options)` -- project the price and SMMA
  onto one shared panel. Returns the panel, ticks, the price path,
  the SMMA path, and SMMA markers.
- `describeLineSmmaChart(data, options?)` -- ARIA description.

## Component

`ChartLineSmma` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, axis ticks and grid, a
config badge, a two-series legend (Price / SMMA) with toggle, a
hover/focus tooltip exposing the value, SMMA and position, and a
`motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs a plain exponential moving average** -- an EMA uses the
  faster weight `2 / (period + 1)`. The SMMA's slower `1 / period`
  weight makes it equivalent to an EMA of period `2*period - 1`, so
  it lags more and stays smoother.
- **vs a simple moving average** -- the SMA is a finite windowed
  mean that fully drops the oldest value each bar. The SMMA is a
  running mean -- every past value still contributes with a
  decaying weight, so it never fully drops a value.
- **vs chart-line-wma** -- the WMA weights a finite window by a
  linear ramp. The SMMA is an infinite running mean with Wilder's
  slow exponential decay.
- **vs chart-line-dema / chart-line-tema** -- those *reduce* lag.
  The SMMA deliberately lags more in exchange for heavy smoothing;
  it is the smoothing Wilder built his RSI, ATR and ADX on.

## API

```ts
interface ListVirtualizerItem<T = unknown> {
  id: string | number;
  data: T;
  type?: 'row' | 'header';
  estimatedHeight?: number;
  disabled?: boolean;
}

interface ListVirtualizerHandle {
  scrollToIndex(
    index: number,
    behavior?: ScrollBehavior,
  ): void;
  scrollToTop(behavior?: ScrollBehavior): void;
  getScrollTop(): number;
}

interface ListVirtualizerProps<T = unknown> {
  items: ListVirtualizerItem<T>[];
  renderItem: (
    item: ListVirtualizerItem<T>,
    context: { index, isActive, isSticky },
  ) => ReactNode;
  estimatedRowHeight?: number;   // default 48
  overscan?: number;              // default 4
  height?: number | string;       // default '100%'
  ariaLabel?: string;             // default 'List'
  className?: string;

  activeIndex?: number;
  defaultActiveIndex?: number;
  onActiveIndexChange?: (idx: number) => void;

  stickyHeaders?: boolean;        // default false
  scrollBehavior?: ScrollBehavior; // default 'auto'

  onEndReached?: () => void;
  endReachedThreshold?: number;   // default 64 px

  enableKeyboardNav?: boolean;    // default true
}
```

## Behaviour

### Dynamic row heights

- `estimatedRowHeight` (prop) is the
  fallback when no measurement
  exists for a row.
- `item.estimatedHeight` (per-row
  override) shadows the prop
  default.
- The component installs a ref
  callback on every rendered row
  that captures
  `getBoundingClientRect().height`
  on mount. The captured value is
  written into the measurement Map
  and triggers a re-layout; the
  cumulative offset array recomputes
  via `computeItemOffsets`.

The measurement Map drops stale
entries when `items.length` shrinks,
preventing the offset array from
holding ghost heights for items
that no longer exist.

### Smooth scroll

- `scrollBehavior` prop (default
  `auto`) drives the default behavior
  for auto-scroll-into-view and the
  imperative API.
- `scrollToIndex(idx, behavior?)`
  accepts a per-call override.

### Sticky headers

- Items marked `type: 'header'`
  render in the spacer body
  alongside rows.
- When `stickyHeaders={true}`, the
  LAST header whose offset is <=
  `scrollTop` pins to the top of
  the viewport via
  `position: sticky` + a
  `translateY(scrollTop)` correction
  so the overlay tracks the scroll
  smoothly.
- The body copy of the currently-
  sticky header is omitted to
  avoid duplicate rendering; the
  spacer math is unaffected.

### Keyboard navigation

| key | effect |
| --- | --- |
| ArrowDown | activeIndex + 1 (clamped) |
| ArrowUp | activeIndex - 1 (clamped) |
| PageDown | active + ceil(viewportH / estRowH) |
| PageUp | active - ceil(viewportH / estRowH) |
| Home | 0 |
| End | items.length - 1 |

Active row auto-scrolls into view
(top-aligned when above viewport,
bottom-aligned when below). Disabled
via `enableKeyboardNav={false}`.

### End-reached

On every scroll, the component
computes `scrollHeight - scrollTop -
clientHeight`. When the result is
<= `endReachedThreshold`,
`onEndReached()` fires. Use for
cursor-paginated feeds.

### ResizeObserver

Container viewport height is
measured via `clientHeight` on
mount; a `ResizeObserver` re-
measures on container resize.
When `ResizeObserver` is not
available (older runtimes), the
initial measurement still applies.

## ARIA + data attributes

Root (`role="region"`,
`tabIndex={0}` (or `-1` when nav
disabled), `aria-label`):

- `data-section="list-virtualizer"`
- `data-item-count`
- `data-active-index`
- `data-sticky-headers`

Spacer:

- `data-section="list-virtualizer-spacer"`
  (full document height)

Per-row:

- `data-section="list-virtualizer-item"`
- `data-item-id` (stringified)
- `data-item-index`
- `data-item-type` (`row` /
  `header`)
- `data-active` (true/false)
- `data-disabled` (true/false)

Sticky overlay:

- `data-section="list-virtualizer-sticky"`
- `data-sticky-index`
- Inner item carries
  `data-sticky="true"`.

## Out of scope (deferred)

- Horizontal virtualization.
  Single column only for v1.
- Adaptive overscan based on
  scroll velocity.
- Anchored / pinned scroll
  position when items prepend
  (chat-window pattern).
- Row recycling pool. React's
  key-based reconciliation is
  sufficient.
- Primitive-level onRowClick /
  onRowHover. Hosts can attach
  via the `renderItem` body.
- `role="list"` / `listitem` on
  every row. The primitive uses
  `role="region"` since grouped
  rows + headers do not always
  cleanly map to a list. Hosts
  can add the role through their
  renderItem markup.

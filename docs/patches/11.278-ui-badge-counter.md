# 11.278 -- BadgeCounter primitive (v1.11.296)

Component-scope-only addition. New BadgeCounter primitive that
supersedes the inline count chips that TopTabs, Queue, and
WorkerListGroupHeader had reimplemented locally.

## Why

Three surfaces (`TopTabs.tsx`, `WorkerListGroupHeader.tsx`, the
ad-hoc Queue todo chip) shipped their own inline rounded-pill
count badges. Each picked its own tone vocabulary (legacy `amber`
vs `accent`, `destructive` vs `danger`, etc) and its own overflow
threshold. Patch 11.278 consolidates the contract into one
primitive so a future palette migration only has to touch one
file.

## What changed

### `web/src/components/ui/badge-counter.tsx` (new file)

- `count?: number`, `max?: number` (default 99) -- overflow renders
  `<max>+` (so 250 -> "99+"). Boundary preserves the literal at
  exactly max.
- `showZero?: boolean` -- numeric variant renders nothing at
  count=0 by default; opt in for "0 errors" steady-state chips.
- `variant: 'numeric' | 'dot'`.
- `tone: 'neutral' | 'accent' | 'danger' | 'success' | 'primary'`.
- `size: 'sm' | 'md' | 'lg'` (14 / 16 / 20 px tall).
- `pulse?: boolean` -- gated by `useReducedMotion()`.
- `srLabel?: string` -- screen-reader sentence override.
- `role="status"` so SR announces count changes.
- Data-attribute selectors: `data-section="badge-counter"`,
  `data-variant`, `data-tone`, `data-pulse`, `data-count`,
  `data-overflow`.

### Adopted

- `web/src/components/layout/TopTabs.tsx` -- inline per-tab count
  chip migrated. Legacy tones mapped: `amber` -> `accent`,
  `destructive` -> `danger`, `muted` -> `neutral`. `srLabel` reads
  `<count> <tab name>`. `cn` import removed.
- `web/src/pages/Queue.tsx` -- new accent BadgeCounter in the
  page actions slot showing remaining `todo` rows. Hidden at
  todoCount=0.
- `web/src/components/WorkerListGroupHeader.tsx` --
  - Group total migrated to BadgeCounter (tone=primary for
    managers, tone=neutral for workers).
  - New optional `busyCount` prop renders a second BadgeCounter
    (tone=accent, size=sm, pulse=true) when any worker is busy.
- `web/src/components/WorkerList.tsx` -- partitioner pass also
  tallies `managersBusy` + `regularBusy` (single pass, no extra
  subscription) and threads each into the matching header.

### Tests

- 24 vitest cases on the primitive (count=0 hidden, count=0
  shown with showZero, overflow at default max, custom max
  ceiling, max-boundary preserves literal number, data-overflow
  flip, selectors, dot variant renders no text, dot ALWAYS
  renders even at count=0, role=status, aria-label defaults to
  displayed count, aria-label override via srLabel, overflow
  aria-label, pulse class + data-pulse toggle, reduced-motion
  mutes the pulse, sm / lg size, danger / success / primary
  tone, className pass-through, stable displayName, undefined
  count renders null, forwarded HTML attributes). 98 / 98
  across the four targeted suites.

## Versions

- 1.11.295 -> 1.11.296 across root + web package.json + both
  lockfiles. CHANGELOG.md entry under `## [1.11.296]`.

## Notes

- `BadgeCounter` is intentionally non-clickable. Click affordances
  belong on the surrounding nav row (the tab button, the group
  header button); the badge is decorative on top of that
  interactive surface.

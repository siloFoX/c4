## Summary

Adds the `ChartLineFisherMidCross` primitive
(`web/src/components/ui/chart-line-fisher-mid-cross.tsx`): a pure
SVG dual-panel chart that visualizes John Ehlers' Fisher Transform
crossings through the 0 midline for Gaussian-normalised momentum
centerline regime transitions with bias coloring. Companion to
`chart-line-fisher-zero-cross` (TODO 11.947): shares the same
Fisher pipeline (rolling min/max -> `[-1, 1]` channel + 0.33/0.67
smoother with clamp -> half-EMA on inverse-tanh) and defaults
(length = 10, threshold = 0); reframes the ARIA / describe text
from "baseline regime transition" to "centerline regime
transitions with bias coloring" so it slots into the broader
mid-cross family (RSI / Stoch / CCI / MFI / Stoch-RSI / Williams
%R / UO / CMO / RMI / DPO / QStick / STC / Choppiness). Together
with `chart-line-fisher-overbought-cross` (11.968) and
`chart-line-fisher-oversold-cross` (11.969) this rounds out the
canonical Fisher threshold layer (overbought + oversold + zero /
midline).

## Pipeline

Same close-only Fisher Transform pipeline as the zero / overbought
/ oversold variants:

1. Rolling min/max on a sliding `length` window:
   - `hi[i] = max(close[i-length+1 .. i])`
   - `lo[i] = min(close[i-length+1 .. i])`
2. Normalize the current close into a `[-1, 1]` channel:
   - `raw[i] = (hi == lo) ? 0 : 2 * (close - lo) / (hi - lo) - 1`
3. Recursive smoother on the channel with weights `0.33 / 0.67`,
   clamped to `[-0.999, 0.999]`:
   - `x[i] = clamp(0.33 * raw[i] + 0.67 * x[i-1], -0.999, 0.999)`
4. Inverse hyperbolic tangent (Fisher transform) and another
   recursive half-EMA blend:
   - `f[i] = 0.5 * ln((1 + x[i]) / (1 - x[i])) + 0.5 * f[i-1]`
5. Regime classifier (strict `>=`):
   - `f >= 0` -> bullish
   - `f <  0` -> bearish
6. Cross detector emits a `bullish` cross when
   `f[i-1] <= 0 && f[i] > 0` and a `bearish` cross when
   `f[i-1] >= 0 && f[i] < 0`. Strict inequality on the current
   side means the boundary value 0 does not double-fire.

## Defaults

- `length = 10` (Ehlers' canonical window)
- `threshold = 0` (midline / centerline)
- Warmup boundary: bar `length - 1 = 9`, so the first 9 samples
  (40-bar run) are null and the remaining 31 are populated.

## Bit-exact anchors

- CONST `close = K` for all `K in {0, 1, 42, 100, 1234}` ->
  `fisher[i] === 0` literal (NOT `-0`; verified via `Object.is`)
  for every `i >= 9`. Regime classifies as bullish because 0
  sits on the threshold (`fisher >= threshold`).
- LINEAR UP `close = i` -> the channel converges to the +clamp
  0.999, fisher grows to a positive steady state (~7.6). All
  bullish (well above 0), 0 crosses.
- LINEAR DOWN `close = -i` -> the channel converges to -0.999,
  fisher to ~ -7.6, all bearish, 0 crosses.
- CONST K full-run counters under defaults, 40 bars:
  `noneCount = 9`, `bullishCount = 31`, `bearishCount = 0`,
  `crosses.length = 0`.
- `pricePath` and `fisherPath` are byte-identical across
  repeated layout calls, with `.toFixed(2)` precision and `M
  ... L ...` command shapes. The fisher path uses a single `M`
  for post-warmup data.
- Cross detector boundary: strict inequality (`> T`, `< T`).
- Layout auto-fit: when fisher collapses to a single value
  (CONST case, fisher = 0), the panel uses the `threshold +/-
  1` fallback (oscMin = -1, oscMax = 1).
- Decline-then-rise trajectories produce confirmed bullish
  centerline crosses (Fisher captures the transient as it
  crosses 0 from negative).

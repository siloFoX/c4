# Patch 11.952 - UI: chart-line-tsi-zero-cross primitive

Version: 1.11.970

## Summary

Add `web/src/components/ui/chart-line-tsi-zero-cross.tsx` -- a pure-SVG dual
panel line chart primitive that visualizes price alongside William Blau's
True Strength Index (TSI) panel, with explicit cross markers and overlay
arrows at every TSI zero crossover. TSI applies two passes of EMA
smoothing to the raw momentum (`close - close_prev`) and its absolute
value, then divides the doubly-smoothed signed momentum by the
doubly-smoothed absolute momentum, scaling to [-100, 100]. Zero
crossovers mark double-smoothed momentum baseline regime transitions
-- a sharper trigger than a single-EMA momentum because the double
smoothing rejects noise without losing direction.

## Pipeline

1. Raw momentum (i >= 1):
   - `mom[i] = close[i] - close[i-1]`
   - `abs_mom[i] = |mom[i]|`
2. First-pass EMAs over `long` (SMA-seeded at i = long):
   - `sm1_mom = EMA(mom, long)`
   - `sm1_abs = EMA(abs_mom, long)`
3. Second-pass EMAs over `short` (SMA-seeded at i = long + short - 1):
   - `sm2_mom = EMA(sm1_mom, short)`
   - `sm2_abs = EMA(sm1_abs, short)`
4. TSI:
   - `sm2_abs == 0` -> `tsi = 0` (CONST fallback)
   - else -> `tsi = 100 * sm2_mom / sm2_abs`
5. `classifyLineTsiZeroCrossRegime(tsi, T)`: null -> none, tsi >= T ->
   bullish, tsi < T -> bearish.
6. `detectLineTsiZeroCrossCrosses(series, tsi, T)`: strict
   `prev <= T && cur > T` for bullish, strict `prev >= T && cur < T`
   for bearish. Boundary equality does not fire.

## Bit-exact anchors

| Pattern                  | TSI (after warmup) | Regime  |
|--------------------------|--------------------|---------|
| CONST close = K          | 0 (0/0 -> 0)       | bullish |
| LINEAR UP (close = i)    | 100                | bullish |
| LINEAR DOWN (close = -i) | -100               | bearish |

CONST anchor tested bit-exactly across K in `{0, 1, 42, 100, 1234}`.
LINEAR UP/DOWN reach the saturation limits because `mom = +/- 1` is
constant and `abs_mom = 1` is constant, so both double-EMAs converge to
those values exactly, giving `tsi = +/- 100`.

## Layout

* Default dimensions: 720 x 460.
* Two panels: price panel on top (~55% of plot area), TSI panel below.
* TSI panel uses a fixed `[-100, 100]` range (TSI is bounded by
  construction) so the threshold band sits exactly at the midpoint when
  `threshold = 0`.
* Threshold band rendered as a horizontal rule at `y = thresholdY` in
  the TSI panel.
* Cross markers drawn as circles on the TSI panel at `(cx, cyOsc)` and
  as arrows on the price panel at `(cx, cyPrice)`.
* SVG path strings use `.toFixed(2)` for deterministic snapshot tests.

## Accessibility

* Wrapping `<div role="region" aria-label="TSI Zero Cross chart">`.
* `<svg role="img">` with an aria-described-by linked `<desc>` sr-only
  text.
* Each cross marker: `role="graphics-symbol"` + `tabIndex={0}` +
  descriptive `aria-label` ("bullish cross at x=...").
* Legend toggles: `<button role="switch" aria-pressed="...">`.
* Motion-safe fade-in via `motion-safe:animate-fade-in` (opt-out with
  `animate={false}`).

## Summary

New `ChartLineHighpass` UI primitive in
`web/src/components/ui/chart-line-highpass.tsx` -- a pure-SVG two-panel
line chart with an **Ehlers Highpass Filter** panel removing the
low-frequency trend component of the price. The top panel plots the
price; the bottom panel plots the highpass and its trigger with a zero
line.

## The computation

John Ehlers' Highpass Filter removes the trend (low-frequency) content
of a price series, passing only the faster cyclic swings. It is a
two-pole filter. `ChartLineHighpassPoint` carries `{x, value}`.

- `computeLineHighpassCoefficients(period)` -- turns a single cutoff
  `period` into the two-pole filter constants:
  `a1 = exp(-1.414*pi / period)`, `b1 = 2*a1*cos(1.414*pi / period)`,
  `c2 = b1`, `c3 = -a1*a1` and `c1 = (1 + c2 - c3) / 4`.
- `computeLineHighpass(values, period)` -- drives the price's **second
  difference** through the filter:
  `hp[i] = c1*(price[i] - 2*price[i-1] + price[i-2]) + c2*hp[i-1] + c3*hp[i-2]`,
  seeding the first two bars at zero. The trigger is the highpass
  delayed one bar.
- `classifyLineHighpassZone(hp)` -- marks a bar **up** above zero,
  **down** below, **flat** at zero.

## Worked fixture

The `exp` / `cos` coefficients make the highpass transcendental for a
general series, but the highpass has two exact anchors that are also its
**defining property**.

The second difference of any straight line is identically zero:
`(a + k*b) - 2*(a + (k-1)*b) + (a + (k-2)*b) = 0`. So a **constant
series** (slope zero) and -- more tellingly -- a **linear ramp** (any
non-zero slope) both leave every driving term at zero. With the first
two bars seeded at zero the recursion `c1*0 + c2*0 + c3*0` stays exactly
at zero for every later bar.

The test asserts the constant series `[50,50,...]`, the ascending ramp
`[10,14,18,22,26,30,34,38]` and the descending ramp
`[100,93,86,79,72,65,58,51]` are **all** driven to exactly zero (the
ascending ramp with `toEqual([0,...])`). That is what makes this a
highpass: the trend is removed regardless of its slope. The coefficients
are checked structurally -- `a1` against its defining identity
`exp(-1.414*pi / period)`, `a1` inside the open unit interval,
`c3 = -a1*a1`, `c1 = (1 + c2 - c3) / 4` -- and a two-cycle sinusoid
fixture (which has curvature) confirms the highpass passes the cyclic
content with both signs.

## API

`data` (a `ChartLineHighpassPoint[]`), `period` (default 20),
`pricePanelRatio` / `gap`, sizing/format/a11y props, `strokeWidth` /
`dotRadius`, the colour props, the toggle flags (`showAxis` / `showGrid`
/ `showDots` default false / `showHighpass` / `showTrigger` /
`showZeroLine` / `showMarkers` / `showTooltip` / `showConfigBadge` /
`showLegend` / `animate`), controlled + uncontrolled `hiddenSeries`
(keys `price` / `highpass` / `trigger`), `formatValue` / `formatX`,
`onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineHighpassFinitePoints`,
`normalizeLineHighpassPeriod` (coerces the period to an integer of at
least 2), `computeLineHighpassCoefficients`, `computeLineHighpass`,
`classifyLineHighpassZone`, `runLineHighpass`,
`computeLineHighpassLayout`, `describeLineHighpassChart`, plus the
`DEFAULT_CHART_LINE_HIGHPASS_*` constants and the
`ChartLineHighpassZone` / `ChartLineHighpassCoefficients` types. Every
exported name carries a `LineHighpass` infix so the `export *` barrel
line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen
`data-section="chart-line-highpass-aria-desc"` description span; the
price line, the highpass line, the trigger line and every dot / marker
are `role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-period`, `data-hp-final`,
`data-up-count`, `data-down-count`, `data-flat-count`,
`data-total-points` and `data-animate`.

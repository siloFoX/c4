# Patch 11.847 -- ChartLineAtrCross primitive (v1.11.865)

## Summary

Add `<ChartLineAtrCross />`, a pure-SVG dual-panel chart with the
close on top and a short-vs-long ATR crossover oscillator on the
bottom that flags volatility-regime shifts:

```
trueRange[i] = max(h-l, |h - prevC|, |l - prevC|);  TR[0] = h - l
shortAtr[i]  = Wilder(trueRange, shortLength)[i]
longAtr[i]   = Wilder(trueRange, longLength)[i]
expanding   = prev shortAtr <= prev longAtr  AND  curr shortAtr > curr longAtr
contracting = prev shortAtr >= prev longAtr  AND  curr shortAtr < curr longAtr
```

When short ATR rises above long ATR the market is entering an
**expanding** volatility regime; when short ATR falls below long
ATR the market is **contracting**. The strict-inequality + prev
guard ensures the cross fires only on the bar that newly flips the
relation.

## Wilder helper precision

The Wilder smoother carries the `min === max` seed precision fix
plus the CONST short-circuit
`next = v === smoothed ? v : (smoothed * (n - 1) + v) / n` so
constant inputs collapse to the exact value through both the short
and long passes.

## Bit-exact anchors

- `CONST h = l = close = K`: TR = 0 everywhere -> both ATRs collapse
  to 0 -> relation never strictly flips -> `expandingCrossCount =
  contractingCrossCount = 0`. Verified across `(K, short, long)`
  sweeps.
- `CONSTANT-SPREAD h = K + D, l = K, close = K + D / 2`: TR = `D`
  from `i = 0` (`|h - prevC| = D/2 <= D` and same for low). Both
  Wilder ATRs collapse to `D` exactly (min/max seed plus the
  short-circuit). Relation stays equal -> zero crosses. Verified
  across `(K, D, short, long)` sweeps.

LINEAR data is intentionally NOT used as an anchor because
TR[0] = 0 in `h = l = close = i + 1` breaks the constant-TR-from-
index-0 invariant the seed needs. The CONSTANT-SPREAD anchor
restores it.

## Surface

Dual-panel primitive surface:

- `ChartLineAtrCrossPoint = { x, high, low, close }`
- `runLineAtrCross(data, options?) -> Run` with
  `ok = series.length >= max(shortLength, longLength)`.
- `computeLineAtrCrossLayout({ data, width?, height?, padding?, panelGap?, shortLength?, longLength? }) -> Layout`.
  Bottom panel y-axis starts at 0 and auto-fits up to the larger
  ATR.
- `describeLineAtrCrossChart(data, opts?) -> string` for the ARIA
  description.
- `<ChartLineAtrCross />` `forwardRef<HTMLDivElement>` with three
  legend toggles (`price`, `short`, `long`), config badge with live
  cross counts, scatter markers at each crossover event, and a
  tooltip surfacing `close / tr / short / long / relation / cross`.
  Live counts also exposed as `data-expanding-cross-count` /
  `data-contracting-cross-count` attributes.

## Behaviour

### Modes

| mode | hour range | extras |
| --- | --- | --- |
| `24` (default) | 0..23 | no period |
| `12` | 1..12 | AM/PM toggle rendered |

In `'12'` mode the period button
renders to the right of the
seconds field; `aria-pressed`
mirrors `period === 'PM'`.

### Spinbutton inputs

Each field is a native `<input
type="text" inputMode="numeric"
role="spinbutton">` with:

- `aria-valuemin` /
  `aria-valuemax` /
  `aria-valuenow` /
  `aria-valuetext` reflecting
  the current padded value.
- `aria-label` per field
  (overridable via the
  `hourAriaLabel` /
  `minuteAriaLabel` /
  `secondAriaLabel` props).

Mid-typing the input shows the
raw digit string. On blur the
input snaps back to the
canonical 2-digit padding.

### Keyboard

| key | effect |
| --- | --- |
| ArrowUp | increment focused field by `step` (hours always by 1) |
| ArrowDown | decrement focused field by `step` (hours always by 1) |
| ArrowUp / ArrowDown on AM/PM toggle | flip period |

Increment / decrement wraps at
the field bounds.

### Paste

`onPaste` on the root passes
the clipboard text through
`parseTimeString(text, mode)`.
Accepted shapes:

| input | mode | result |
| --- | --- | --- |
| `'14:30'` | '24' | parses HH:MM |
| `'14:30:45'` | '24' | parses HH:MM:SS |
| `'14.30'` | (any) | dot separator allowed |
| `'2:30 PM'` | '12' | period parsed |
| `'2:30 am'` | '12' | case-insensitive |
| `'2:30 p.m.'` | '12' | dotted period |
| `'15:30'` | '12' | auto-derived PM (hours: 3) |
| `'00:00'` | '12' | -> 12:00 AM |
| `'25:00'` | '24' | null |
| `'14:60'` | (any) | null (minute out of range) |
| `'hello'` | (any) | null |
| `''` | (any) | null |
| `null` / `undefined` | (any) | null |

Invalid pastes silently fall
through to the browser's
default paste handling (the
event is not preventDefault'd
when parsing fails).

### Step

`step` (default 1) applies to
MINUTES and SECONDS arrow nav
only. Hours always step by 1
regardless of `step` -- a
non-1 step on hours rarely
matches operator intent.

### Wrap-around math

Pure helper:

```ts
export function stepTimeField(
  current: number,
  delta: number,
  min: number,
  max: number,
): number;
```

Normalizes via modulo across
`range = max - min + 1`:

- `stepTimeField(23, 1, 0, 23)`
  -> `0`
- `stepTimeField(0, -1, 0, 23)`
  -> `23`
- `stepTimeField(59, 5, 0, 59)`
  -> `4` (`59 + 5 = 64`;
  `64 % 60 = 4` in the
  60-value range)
- `stepTimeField(12, 1, 1, 12)`
  -> `1` (12-hour hour wrap)
- non-finite `current` -> `min`

### Disabled / readOnly

- `disabled` -> inputs disabled
  + AM/PM toggle disabled;
  root carries
  `data-disabled="true"`.
- `readOnly` -> inputs read-
  only + AM/PM toggle
  disabled; root carries
  `data-read-only="true"`.

## ARIA + data attributes

Root (`<div role="group">`,
`aria-label`):

- `data-section="time-picker"`
- `data-mode` ('12' / '24')
- `data-show-seconds`
  ('true' / 'false')
- `data-disabled`
- `data-read-only`

Inputs:

- `role="spinbutton"`
- `aria-valuemin` /
  `aria-valuemax` /
  `aria-valuenow` /
  `aria-valuetext`
- `aria-label`
- `data-section="time-picker-hours"`
  / `-minutes` / `-seconds`
- `data-time-field` mirror

Separator:

- `aria-hidden`
- `data-section="time-picker-separator"`

AM/PM toggle:

- `<button>` with
  `aria-pressed` +
  `aria-label`
- `data-section="time-picker-period"`
- `data-period` ('AM' / 'PM')

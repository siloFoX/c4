## Summary

Add a new `<ChartScatter>` UI primitive in
`web/src/components/ui/chart-scatter.tsx` -- pure-SVG scatter /
bubble plot. Multi-series with per-series colour + shape
(circle / square / triangle / diamond), hover tooltip with
series label + x / y / size lines, optional bubble-size
mapping, click handler, axis grid + ticks, optional legend
(right / bottom placement), and a `motion-safe:animate-fade-in`
mount animation. Re-uses the linear scale + tick helpers from
`<ChartLine>` (11.439) and the default colour palette from
`<ChartBar>` (11.438).

## API

```ts
type ChartScatterShape =
  | 'circle' | 'square' | 'triangle' | 'diamond';

interface ChartScatterPoint {
  x: number;
  y: number;
  size?: number;       // optional bubble-size value
  label?: string;      // optional ARIA / tooltip label
}

interface ChartScatterSeries {
  id: string;
  label: string;
  data: readonly ChartScatterPoint[];
  color?: string;
  shape?: ChartScatterShape;
}

interface ChartScatterProps {
  series: readonly ChartScatterSeries[];
  width?: number;                       // default 520
  height?: number;                      // default 280
  padding?: number;                     // default 36
  axisLabel?: { x?: string; y?: string };
  showGrid?: boolean;                   // default true
  showTooltip?: boolean;                // default true
  showAxisTicks?: boolean;              // default true
  showLegend?: boolean;                 // default true
  animate?: boolean;                    // default true
  dotSize?: number;                     // default 6
  minDotSize?: number;                  // default 3
  maxDotSize?: number;                  // default 18
  className?: string;
  ariaLabel?: string;                   // default 'Scatter plot'
  formatX?: (v: number) => string;
  formatY?: (v: number) => string;
  formatSize?: (v: number) => string;
  tickCount?: number;                   // default 4
  xDomain?: [number, number];
  yDomain?: [number, number];
  onPointClick?: (args: {
    series: ChartScatterSeries;
    point: ChartScatterPoint;
    seriesIndex: number;
    pointIndex: number;
  }) => void;
  legendPlacement?: 'right' | 'bottom';
}
```

## Behaviour

- Renders a `<div role="region">` wrapping an inline `<svg>`
  sized to the configured `width` x `height` (520 x 280 by
  default). The SVG carries `role="img"` and the same
  `aria-label` so screen readers can find both the region
  and the chart-as-image.
- Each series draws its points as filled `<path>` shapes;
  default shape is `circle`, with `square`, `triangle`, and
  `diamond` available via `series.shape`. Stroke is drawn
  only on the currently-hovered point to keep the visual
  noise low.
- Each point exposes `role="graphics-symbol"` and an ARIA
  label. The default label is
  `"<series.label>: x=<formattedX>, y=<formattedY>"`.
  Supplying `point.label` overrides that label so adopters
  can spell out a domain meaning (e.g. country name,
  customer id).
- Hovering a point shows a tooltip with the series label and
  formatted x / y values; if the point carries a `size`,
  the tooltip adds a `size=` row; if the point carries a
  `label`, the tooltip adds that label as a third row.
- `onPointClick` receives the full series + point pair plus
  their indices so adopters can dispatch drill-down events
  without rebuilding the lookup.
- `xDomain` / `yDomain` clamp the axis range manually; with
  no overrides the bounds expand by 1 unit when the data
  set is collapsed (single point, or all points share the
  same x or y) so the chart still renders meaningfully.
- `legendPlacement="bottom"` stacks the canvas above a
  wrapped legend so wide-aspect adopters get a horizontal
  legend strip.
- The mount animation (`motion-safe:animate-fade-in`) is
  applied per-series group and honours
  `prefers-reduced-motion`. `animate=false` removes the
  class entirely.

## Pure helpers (exported)

```ts
DEFAULT_CHART_SCATTER_WIDTH       = 520
DEFAULT_CHART_SCATTER_HEIGHT      = 280
DEFAULT_CHART_SCATTER_PADDING     = 36
DEFAULT_CHART_SCATTER_TICK_COUNT  = 4
DEFAULT_CHART_SCATTER_DOT_SIZE    = 6
DEFAULT_CHART_SCATTER_MIN_DOT_SIZE = 3
DEFAULT_CHART_SCATTER_MAX_DOT_SIZE = 18

getChartScatterBounds(series, xDomain?, yDomain?):
  ChartScatterBounds
getSizeScale(sizeMin, sizeMax, minPx, maxPx):
  (value: number | undefined) => number
findNearestScatterPoint(series, sx, sy, positions, maxDistance?):
  { seriesIndex; pointIndex; distance } | null
buildShapePath(shape, cx, cy, halfSize): string
```

- `getChartScatterBounds` falls back to `(0, 1)` when no
  finite x / y values are seen, expands a collapsed range
  by 1 unit, and honours explicit `xDomain` / `yDomain`
  overrides. Size bounds default to `(0, 0)` when no point
  has a size so the bubble scale degenerates to `minPx`.
- `getSizeScale` clamps inputs outside the source range to
  the dest range, and returns `minPx` for `undefined`,
  non-finite values, or a zero-length source range.
- `findNearestScatterPoint` does an O(N) scan returning the
  closest point within `maxDistance` (default 24 px). The
  component does not currently use this helper at runtime
  but exports it so adopters can wire keyboard navigation
  or marquee-select downstream.
- `buildShapePath` emits a closed SVG path for each shape;
  circle is encoded via two semi-circular arcs so the path
  closes cleanly in `<path>`.

## ARIA + data attributes

- Root: `<div role="region">` + `aria-label`.
- SVG: `role="img"` + same `aria-label`.
- Each point: `role="graphics-symbol"` + `tabIndex={0}` so
  keyboard focus opens the tooltip via `onFocus`.
- `data-section` on every node:
  - `chart-scatter` (root)
  - `chart-scatter-canvas` (wrapper)
  - `chart-scatter-svg`
  - `chart-scatter-axis-x` / `chart-scatter-axis-y`
  - `chart-scatter-grid-y` / `chart-scatter-grid-x`
  - `chart-scatter-tick-y` / `chart-scatter-tick-x`
  - `chart-scatter-axis-x-label` / `chart-scatter-axis-y-label`
  - `chart-scatter-series` (per series group)
  - `chart-scatter-point` (per point path)
  - `chart-scatter-tooltip` + `-label` / `-coords` / `-size`
    / `-point-label`
  - `chart-scatter-legend` + `-item` / `-glyph` / `-label`
- Series groups mirror `data-series-id`, `data-series-color`,
  `data-series-shape`; points mirror `data-series-id`,
  `data-point-index`, `data-hovered`.

- Component:

  - legend default-on, suppressed via `showLegend=false`
  - legend placement right (default) / bottom

## Pairs with existing primitives

- `<ChartBar>` (11.438): categorical bar plots. Both
  primitives share `getDefaultBarColor` for visual
  consistency.
- `<ChartLine>` (11.439): time-series line. Scatter shares
  `getLinearScale` / `getChartLineTicks` /
  `formatChartLineTick`.
- `<ChartPie>` (11.440): part-of-whole.
- `<ChartArea>` (11.441): stacked / overlaid area.
- `<ChartHeatmap>` (11.442): calendar / grid heatmap.
- `<ChartSparkline>` (11.443): inline cell-sized trend.

## Out of scope

- Linear / polynomial regression overlays (host adds a
  custom SVG layer with a fitted line if needed).
- Marquee / brush selection (the host wraps with a
  pointer-event layer and uses `findNearestScatterPoint` to
  resolve point hits).
- 3D / time-axis scatter (future primitive, not part of
  this routine).
- Async / streaming data (host owns the series array and
  re-renders on update).

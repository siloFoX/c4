# TODO 11.504 - chart-line-area-gradient primitive

Status: SHIPPED v1.11.522 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartLineAreaGradient>`, a pure-SVG multi-series line + area
chart whose defining feature is a **vertical linear gradient fill**
from the line stroke down to a configurable baseline. Each series
declares its own `<linearGradient>` in `<defs>` (anchored to the
inner-plot top + baseline pixel via `gradientUnits="userSpaceOnUse"`)
and the area polygon below the line is filled with `url(#...)`. The
canonical "marketing chart" look: line on top, soft gradient shading
below.

## API

```tsx
<ChartLineAreaGradient
  series={[
    {
      id: 'revenue',
      label: 'Revenue',
      data: [
        { x: 0, y: 100 }, { x: 1, y: 130 }, { x: 2, y: 110 },
        { x: 3, y: 160 }, { x: 4, y: 190 },
      ],
    },
  ]}
  baseline={0}
  gradientTopOpacity={0.5}
  gradientBottomOpacity={0}
/>
```

## Distinct from prior chart-* primitives

- `<ChartLine>` - line stroke only, no filled area.
- `<ChartArea>` - flat solid area below the line; no gradient.
- `<ChartLineConfidence>` (11.502) - polygon between two CURVES
  (yLower / yUpper); this primitive fills between the line and a
  flat baseline.
- `<ChartLineThreshold>` (11.503) - horizontal threshold rules with
  zone shading; no gradient under the line.
- `<ChartStackedArea>` - stacked solid areas; no per-series
  gradient.

## Layout

### Baseline

- `resolveLineAreaGradientBaseline(bounds, explicit?)` returns the
  effective baseline:
  - explicit finite -> verbatim.
  - otherwise -> `bounds.yMin` (the floor of the visible data), so
    the gradient anchors to the actual bottom of the data range.
- Adopters who want a hard floor (typically 0) pass `baseline={0}`.
- The baseline pixel is clamped to the inner plot area so the area
  path stays visually meaningful even when the baseline is outside
  the y range.

### Paths

- `buildLineAreaGradientLinePath(points)` - `M x y L x y ...` central
  line.
- `buildLineAreaGradientAreaPath(points, baselineY)` - closed
  polygon: `M p[0] L p[1] ... L p[N-1] L p[N-1].x baselineY L p[0].x
  baselineY Z`. Empty -> `''`. Single point -> degenerate
  zero-width sliver for completeness.

### Gradient

- One `<linearGradient>` per series under `<defs>`, with id
  `{useId}-{series.id}-grad`.
- `gradientUnits="userSpaceOnUse"` so the gradient is anchored to
  the actual pixel range:
  - `y1 = topY` (the inner plot top, `padding`).
  - `y2 = baselineY` (the clamped baseline pixel).
- Two stops:
  - 0%: `stop-color="{series.fillColor ?? color}"` with
    `stop-opacity="{gradientTopOpacity}"`.
  - 100%: same color, `stop-opacity="{gradientBottomOpacity}"`.
- `series.fillColor` is separate from `series.color` so adopters
  can use a contrast colour for the line vs the fill.

## Interactivity

- Hover or focus on a dot dims every other series stroke + area to
  ~30% so the focused series visually dominates.
- Per-dot tooltip shows series label, formatted x, formatted y.
- Per-dot `onPointClick({series, point})` callback.
- Legend toggles series visibility (controlled + uncontrolled).
  Each legend swatch is a tiny gradient block + horizontal line so
  the styling is visible at a glance.
- Mount fade via `motion-safe:animate-fade-in` (suppress via
  `animate=false`).

## Exported helpers

- `DEFAULT_CHART_LINE_AREA_GRADIENT_*` numeric / string defaults.
- `DEFAULT_CHART_LINE_AREA_GRADIENT_PALETTE` (10-colour palette).
- `getLineAreaGradientDefaultColor(index)` - palette wrap.
- `getLineAreaGradientFinitePoints(points)` - drops non-finite.
- `resolveLineAreaGradientBaseline(bounds, explicit?)` - canonical
  baseline resolution.
- `getLineAreaGradientBounds(series, hidden?, baseline?)` - per-axis
  bounds; expands y range to include explicit baseline.
- `getLineAreaGradientTicks(min, max, count?)` - evenly-spaced.
- `buildLineAreaGradientLinePath(points)` - line builder.
- `buildLineAreaGradientAreaPath(points, baselineY)` - closed area
  polygon.
- `computeLineAreaGradientLayout(input)` - per-series layout with
  `linePath` + `areaPath` + global `baseline` + `baselineY` + `topY`.
- `describeLineAreaGradientChart(series, hidden?, baseline?,
  formatValue?)` - ARIA summary.

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only
  `data-section="chart-line-area-gradient-aria-desc"` span.
- SVG `role="img"` with the same `aria-label`.
- Each line path is `role="graphics-symbol"` with `tabIndex={0}` and
  `aria-label="<series>: line with K points"`.
- Each dot is `role="graphics-symbol"` with `tabIndex={0}` and
  `aria-label="<series>: x=<x>, y=<y>"`.

## Data attributes

- Root: `data-section="chart-line-area-gradient"`,
  `data-series-count`, `data-visible-series-count`,
  `data-total-points`, `data-baseline`, `data-baseline-y`,
  `data-animate`.
- Gradient def: `data-section="chart-line-area-gradient-gradient-def"`,
  `data-series-id`.
- Series group: `data-section="chart-line-area-gradient-series-group"`,
  `data-series-id`, `data-series-index`, `data-series-color`,
  `data-series-fill-color`, `data-series-point-count`,
  `data-series-finite-count`, `data-hovered`.
- Area / line / dot: standard pattern with `data-series-id`.
- Baseline (when `showBaseline=true`): `-baseline` with
  `data-baseline-value`.
- Tooltip, legend, axes, grid: standard pattern.

## Out of scope

- Stacked gradient areas (each series independent in v1; use
  `<ChartStackedArea>` for stacked solid).
- Pattern fills (only `<linearGradient>`).
- Horizontal gradients (vertical only; the canonical "fade to
  baseline" pattern is vertical).
- Animated gradient transitions on data change (snap; mount fade
  only).
- Multi-stop gradients (one top + one bottom stop in v1; adopters
  who want a 3-stop fade compose a custom `<linearGradient>` and
  wire the `id` into the area path).
- Log-scale axes - linear only.

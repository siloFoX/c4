# 11.1099 -- UX FIX: C4 help drawer auto-opens over content (v1.11.1117)

## Scope

An onboarding panel auto-opened over page content on view load
(observed over Sessions at 768). The auto-opener is the Sessions
onboarding tour (SessionsTour, a full-screen `fixed inset-0` modal).

## Root cause

`useSessionsTour` read the first-run flag (`localStorage
'sessions-tour-v1'`) on mount and auto-opened the tour if unset, but
wrote the flag ONLY in `dismissTour`. So a user who saw the tour but
navigated away (or reloaded) WITHOUT explicitly dismissing it left the
flag unset, and the tour re-auto-opened over content on every later
visit. (The HelpDrawer / global OnboardingTour were not the offenders:
the drawer starts closed, and the global tour already persists its flag.)

## Change

- `web/src/lib/use-sessions-tour.ts`: write the first-run flag the
  moment the tour auto-opens (not only on dismiss), so it auto-opens AT
  MOST ONCE and never re-opens automatically on later visits. Added a
  `reopenTour()` that shows the tour again WITHOUT clearing the flag
  (so manual reopen does not re-arm the auto-open). localStorage
  failures stay swallowed (private mode still shows the tour that
  session).
- `web/src/components/SessionsView.tsx`: a visible manual "Tour"
  control (HelpCircle ghost button, `data-testid="sessions-tour-reopen"`,
  aria-label "Replay sessions tour") wired to `reopenTour`, so the tour
  is still reachable on demand after the one-time auto-open.
- `web/src/components/SessionsTour.tsx`: `data-section="sessions-tour"`
  on the modal for the e2e.

## Verification (mandatory visual)

`web/e2e/help-drawer-firstrun.spec.ts` (new):
- Flag cleared -> the tour auto-opens over Sessions (screenshot
  `help-drawer-firstrun-open.png`); after a RELOAD without dismissing
  it does NOT re-open (the flag was written on open) -- the core
  regression guard.
- Flag set -> the tour does NOT auto-open (screenshot
  `help-drawer-firstrun-closed.png`), and the manual "Tour" control
  reopens it on demand.

Result: `2 passed` headless. Unit suites: `use-sessions-tour` 8 passed,
`SessionsView` 39 passed (the new control + hook shape are compatible).

## Risk

- The tour now auto-opens at most once. Existing users who had not yet
  dismissed it will see it one final time (on the next visit it sets
  the flag and stops). The manual control preserves access. Other
  banners / the global onboarding tour are untouched.

  IntersectionObserver-based lazy image component
  with LQIP (low-quality image placeholder)
  blur-up, native `loading="lazy"` fallback,
  `decoding="async"`.

The existing `components/ui/image.tsx` (11.226)

LQIP blur-up workflow. The two coexist; a future

## Public surface

```tsx
<LazyImage
  src               // full-resolution URL (required)
  alt               // accessible name (required)
  lqip?             // base64 / URL of the low-quality placeholder
  srcSet?           // responsive sources (forwarded verbatim)
  sizes?            // responsive sizes (forwarded verbatim)
  width?            // number (px) or CSS length string
  height?           // number (px) or CSS length string
  rootMargin?       // IO lookahead, default '200px'
  decoding?         // 'async' | 'sync' | 'auto', default 'async'
  blurPx?           // wrapper filter: blur(<n>px), default 12
  transitionMs?     // fade duration, default 300
  className?        // wrapper class override
  imgClassName?     // <img> class override
  onLoad?           // forwarded callback
  onError?          // forwarded callback
  data-testid?      // wrapper testid
/>
```

## Behaviour

### LQIP paint

The wrapper carries `background-image:
url('<lqip>')` and `background-size: cover` so the
visual lands the moment the component mounts.
LQIP is typically a tiny base64-encoded JPEG
(<2KB) generated at build time. While `loaded` is
false the wrapper applies `filter:
blur(<blurPx>px)` so the LQIP looks soft and
intentional rather than pixelated; `blurPx=0`
disables the blur entirely and still treats the
LQIP as a colour placeholder.

### Lazy gating

The full-resolution `<img>` does not mount until
the wrapper intersects the viewport (with
`rootMargin: 200px` of lookahead, configurable).
When `IntersectionObserver` is unavailable the
hook short-circuits to native `loading="lazy"` --
the `<img>` ships immediately and the browser
defers below-the-fold images. The native
`loading="lazy"` attribute also rides along on
every render as defence in depth.

### Load transition

On the `<img>`'s `load` event:

- Wrapper filter -> `none`.
- Wrapper `background-color` -> `transparent` so
  the LQIP edges do not bleed through the
  foreground alpha.
- `<img>` opacity 0 -> 1 over `transitionMs`.

`data-state` on the wrapper reports `idle` /
`fetching` / `loaded` / `error` so CSS adopters
can style off the state.

### Error handling

On the `<img>`'s `error` event:

- `<img>` unmounts.
- LQIP stays visible (the wrapper retains its
  background).
- `data-state` -> `error`.
- `onError` callback fires.

## Tests

`web/src/lib/lazy-image.test.tsx` -- 15 cases:

1. LQIP paints as background-image immediately.
2. `<img>` is NOT rendered before intersection
   (data-state="idle").
3. After intersection -> `<img>` rendered with
   `loading="lazy"` + `decoding="async"`;
   data-state="fetching".
4. `load` event transitions data-state to
   `loaded` and drops the wrapper filter.
5. `error` event transitions data-state to
   `error` and unmounts the `<img>`.
6. `onLoad` / `onError` callbacks forwarded.
7. Native `loading="lazy"` fallback when
   `IntersectionObserver` is undefined.
8. `srcSet` + `sizes` forwarded verbatim.
9. `decoding="sync"` override.
10. Numeric width / height (`64` -> `64px`).
11. CSS-length width / height (`50%` / `auto`).
12. `rootMargin` reaches the observer.
13. `className` + `imgClassName` overrides.
14. `blurPx=0` disables blur.
15. `displayName` sanity.

15/15 pass under vitest 4.1.5. TypeScript clean
for `lazy-image.tsx`.

## Test harness

IntersectionObserver is replaced with a mock that
captures the callback + observed targets. The
helper `fireIntersect()` synthesises an
`isIntersecting: true` entry inside `act(...)`.
Pattern mirrors `components/ui/infinite-scroll.test.tsx`
(11.338) so future IO-driven primitives can
copy-paste the harness.

## Out of scope

- Per-page adoption. The primitive ships
  standalone; no adopter currently emits LQIP
  metadata.
- LQIP generation pipeline. Adopters supply the
  data URI (typically a build-time `sharp` /
  `vite-plugin-image-presets` step).
- AVIF / WebP `<picture>` source selection.
  `srcSet` + `sizes` cover the common path;
  `<picture>` adaptor is a follow-up.
- Unification with `components/ui/image.tsx`.
  Both primitives address slightly different
  surfaces today; merging the public surfaces
  is a follow-up once a real adopter needs
  both stories.
- Real-browser visual coverage (Playwright).
  Unit tests assert behaviour.

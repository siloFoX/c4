# 11.778 chart-line-stoch-signal (v1.11.796)

## Summary

Pure-SVG dual-panel React/TypeScript chart primitive with
George Lane's classic **Stochastic Oscillator (%K)** and its
SMA signal line (**%D**) beneath the close. Added as the
`<ChartLineStochSignal />` UI primitive in
`web/src/components/ui/`.

## Algorithm

For each bar with `kPeriod` (default 14) and `dPeriod`
(default 3):

```
HH[i] = max(high[j], j = i - kPeriod + 1..i)
LL[i] = min(low[j],  j = i - kPeriod + 1..i)
K[i]  = 100 * (close[i] - LL[i]) / (HH[i] - LL[i])
D[i]  = SMA(K, dPeriod)[i]
```

`%K` warmup is `kPeriod - 1`; `%D` adds another `dPeriod - 1`
bars on top. When `HH == LL` (singular: completely flat HL
window) `%K = null` and so does any `%D` window touching it.

A configurable `overboughtThreshold` (default 80) and
`oversoldThreshold` (default 20) classifies bars.

## Anchors

- **Close at high (close == HH > LL)**: numerator = HH - LL =
  denominator -> `%K = 100` **bit-exact** at every bar past
  the warmup. SMA of constant 100 is 100, so `%D = 100`
  bit-exact too. Verified across `(range, kPeriod) in
  {1, 2, 5, 10, 100} x {5, 7, 14, 21}` (20 combos).
- **Close at low (close == LL, HH > LL)**: numerator = 0 ->
  `%K = 0` and `%D = 0` **bit-exact**. Same 20 combos.
- **Close at midpoint (close = (HH + LL) / 2)** with dyadic
  range: `%K = 50` **bit-exact** -> `%D = 50`. Verified across
  ranges `{2, 4, 8, 16}` and kPeriod `{5, 7, 14, 21}`.
- **CONST_HL (high == low at every bar)**: HH = LL -> singular
  -> `%K = null` -> `%D = null` at every bar.
- **Bounded**: `0 <= %K <= 100` for any finite input.

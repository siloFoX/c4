# 11.312 -- Keyboard Shortcuts overlay enhancements (v1.11.330)

Enhances `web/src/components/KeyboardShortcutsModal.tsx`
with platform-aware keymap rendering and a
"Recently used" section pinned above the category
accordion. Component-scope only, no daemon-side change
and no `c4` CLI surface change.

## Why

Two pain points in the current shortcuts overlay:

1. **Keymap labels are hardcoded to PC convention.** A
   Mac operator opening the overlay sees `Ctrl+B`,
   `Ctrl+F`, `Alt+F4`. The keyboard listener accepts
   both Ctrl and Cmd on Mac (the binding is
   platform-agnostic), but the rendered Kbd label
   does not match the operator's expectation. Mac
   muscle memory says "Cmd+B".

2. **No recall surface for frequently-used
   shortcuts.** Every operator has 3-5 shortcuts they
   use daily; the overlay forces them to scan the
   full categorised list every time they need a
   reminder. The dispatch asks for a
   "Recently used" row so the highest-traffic
   shortcuts surface first.

Earlier patches (v1.11.250 chord shortcuts,
v1.11.259 search field, v1.11.290 category Accordion)
already covered the dispatch's grouping + search +
accordion requirements; patch 11.312 ships the two
remaining items.

## What changed

### `web/src/lib/shortcut-keymap.ts` (new file)

Platform-aware keymap formatter, pure string -> string
transform so callers can drive it from a memoised hook
or a static lookup map without component state.

- `detectPlatform(): 'mac' | 'other'`. Reads
  `navigator.platform` first, falls back to
  `navigator.userAgent`. SSR-safe: returns
  `'other'` when `navigator` is undefined.
- `formatKeymap(label, platform?)`. Rewrites canonical
  shortcut labels per platform:
  - `Ctrl -> Cmd`, `Alt -> Option` on Mac.
  - `Cmd -> Ctrl`, `Option -> Alt` on others.
  - Multi-modifier shortcuts (`Ctrl+Shift+P`) are
    tokenised on `+` and rewritten per token.
  - Shift, single-key labels (`?`, `Esc`), and chord
    shortcuts (`g h`) pass through untouched.
- `formatKeymapForCurrentPlatform(label)` --
  convenience wrapper using the live platform.

### `web/src/lib/shortcut-recently-used.ts` (new file)

localStorage-backed recently-used shortcut tracker.

- Storage: `localStorage[c4:shortcuts:recent]` as a
  JSON array of canonical shortcut labels.
- Capacity: capped at `MAX_ENTRIES = 5` items so the
  storage cost stays bounded.
- Ordering: most-recent first. Marking an existing
  label moves it to the front (dedup).
- Broadcast: each mark fires a
  `shortcut-recently-used-changed` CustomEvent so
  every modal instance picks up the change without
  a parent context provider.
- API:
  - `getRecentlyUsedShortcuts(): string[]`.
  - `markShortcutUsed(label)`.
  - `clearRecentlyUsedShortcuts()`.
  - `useRecentlyUsedShortcuts()` -- live hook.
  - `useMarkShortcutUsed()` -- stable mark callback.
- Robust against malformed storage: non-JSON,
  non-array, and non-string entries all return an
  empty list rather than throwing.

### `web/src/components/KeyboardShortcutsModal.tsx`

- Every `<Kbd>` now passes its label through
  `formatKeymapForCurrentPlatform()` so Mac operators
  see "Cmd+B" while Win/Linux operators see "Ctrl+B".
  Both the recent and category rows share the same
  transform.
- New "Recently used" section rendered above the
  category accordion when there is history AND the
  filter still matches at least one recent row.
  Tagged `data-shortcuts-section="recent"` for e2e
  selectors. The section participates in the search
  contract (filter applies to it just like to the
  category sections).

### i18n

- `shortcuts.section.recent = "Recently used"` added
  to en.json + ko.json (lockstep preserved). The ko
  entry temporarily mirrors the English text pending
  a translation.

### Adopted

- Storage + UI surface only this commit. Wiring
  `markShortcutUsed(label)` into the global key
  listener (so the list actually fills up as the
  operator works) is a deferred per-handler patch.
  Same pattern as the previous lib-only patches in
  this push.

## Notes

- The formatter only rewrites tokens it recognises.
  Single-key shortcuts (`?`, `Esc`, `Enter`) and
  chord shortcuts (`g h`, `g g`) pass through
  byte-identical so the existing 47 modal tests
  that assert on those labels stay green without
  changes.
- The recently-used storage is keyed by the
  canonical shortcut label (`'Ctrl+B'`,
  `'g h'`, `'?'`) -- the same key shape as the
  `SHORTCUT_ROWS` array in
  `KeyboardShortcutsModal.tsx`. This means lookup
  from the recent list to the canonical Row is a
  single `Map<string, Row>` step inside the
  component's `useMemo`.
- The `useRecentlyUsedShortcuts` hook subscribes
  to the `shortcut-recently-used-changed`
  CustomEvent so every modal instance (the global
  one, any preview pane that mounts the modal,
  etc) stays in sync. No context provider is
  needed because the event flows through
  `window.dispatchEvent`.
- The capacity of 5 was picked to match the WCAG
  short-term-memory heuristic: a list longer than
  5-7 items stops functioning as a "recall" surface
  and starts working as a "navigation" surface,
  which is what the category accordion already
  covers. If a future patch wants to expose a
  longer recent list, the `MAX_ENTRIES` constant
  is a one-line change.
- The malformed-storage guards (`parsed` is
  validated for `Array.isArray` and each entry is
  validated for `typeof === 'string'`) protect
  against the localStorage value being mutated by
  an extension, a future schema migration that
  doesn't backfill, or a tampered storage value
  from a different c4 version.

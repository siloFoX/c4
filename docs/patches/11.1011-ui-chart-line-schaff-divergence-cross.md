component that renders the close in the top panel and the Doug
Schaff Trend Cycle (STC) line in the bottom panel, marking

## STC pipeline (Schaff)

```
macd[i] = SMA(close, fastLength) - SMA(close, slowLength)
k1[i]   = 100 * (macd[i] - LLV(macd, cycle)) / (HHV - LLV);
          50 when HHV === LLV (degenerate fallback)
d1[i]   = EMA(k1, factor)
k2[i]   = 100 * (d1[i]  - LLV(d1, cycle))  / (HHV - LLV);
          50 when HHV === LLV (degenerate fallback)
stc[i]  = EMA(k2, factor)
```

Mirrors the canonical STC formulation already used by the
other STC primitives. Default tuning matches Schaff's original
proposal: `fastLength = 23`, `slowLength = 50`, `cycleLength
= 10`, `factor = 0.5`.

Warmup is `slowLength + 2 * (cycleLength - 1) - 1 = 67` for the
default tuning: SMA seeds at `slowLength - 1 = 49`, the first
stochastic stage adds `cycleLength - 1 = 9` bars, and the
second stochastic stage adds another 9. The EMA chain inherits
each prior valid index because it seeds at the first valid
input.

## Divergence regime model

5-state classifier exactly like the other divergence primitives:

```
priceUp = close[i] > close[i - divergenceWindow]
stcUp   = stc[i]   > stc[i - divergenceWindow]
state   = aligned-bullish    if priceUp && stcUp
        | aligned-bearish    if !priceUp && !stcUp
        | divergent-bullish  if !priceUp && stcUp   (price down, STC up)
        | divergent-bearish  if priceUp && !stcUp   (price up, STC down)
        | none               if either input is null
bullish-cross : prev != 'divergent-bullish' && cur == 'divergent-bullish'
bearish-cross : prev != 'divergent-bearish' && cur == 'divergent-bearish'
```

Crosses suppress when either `prev` or `cur` is `none`, so
warmup transitions never fire spuriously. Strict inequality
guarantees boundary equality (STC sitting at the midline 50)
falls into the aligned-bearish state instead of double-firing.

## Bit-exact anchors

| input              | stc (post-warmup)         | regime (post i=72)  | crosses |
| ------------------ | ------------------------- | ------------------- | ------- |
| CONST close = K    | 50 (degenerate cascade)   | aligned-bearish     | 0       |
| LINEAR UP, close=i | 50 (constant macd -> 50)  | divergent-bearish   | 0       |
| LINEAR DOWN, close=-i | 50 (constant macd -> 50) | aligned-bearish    | 0       |

CONST K verified across K = 0..1234 -- macd = K - K = 0 for
every valid bar; the rolling stochastic over a flat constant
hits the degenerate `HHV === LLV` branch and returns 50, so
k1, d1, k2, stc all collapse to 50.

LINEAR UP exercises the constant-macd case via algebra:
fastSma settles at `i - (fastLength - 1) / 2 = i - 11`,
slowSma at `i - (slowLength - 1) / 2 = i - 24.5`, so
`macd = (i - 11) - (i - 24.5) = +13.5` constant. The
stochastic of a flat constant macd is degenerate, so once
again the entire downstream chain collapses to 50. priceUp =
true, stcUp = false (50 === 50) -> regime `divergent-bearish`
for the full 48 valid samples on the 120-bar fixture. The
cross count stays at 0 because the regime is entered from
`none`.

LINEAR DOWN is the algebraic mirror: macd = -13.5 constant ->
stochastic degenerate -> stc = 50. priceUp = false, stcUp =
false -> regime `aligned-bearish`.

Note: the STC sitting at the midline 50 on pure linear input
is the canonical signature of a cycle oscillator -- there is
no cycle to detect in a non-oscillating linear trend, so the
indicator parks at its neutral reading. This is the same
behaviour exercised by the existing STC primitives'
bit-exact anchors.

## Risk

Zero risk -- net-new component, no modifications to existing
files except the barrel append and the four manifest bumps.
The SMA / stochastic / EMA helpers and the STC pipeline are
locally scoped and do not interfere with the equivalent
helpers in `chart-line-stc-cross-sig` or other STC
primitives. The component name uses "Schaff" (not "STC") to
keep the divergence module's exports distinct from the
existing STC family.

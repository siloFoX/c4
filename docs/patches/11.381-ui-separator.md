## Goal

Dispatch:

- Build `web/src/components/ui/separator.tsx`
  -- horizontal / vertical divider with
  optional label slot, decorative role,
  spacing variants.

`<Separator>` (11.165) already shipped most
of the contract:

| dispatch item | shipped previously |
| --- | --- |
| horizontal / vertical orientation | `orientation` prop |
| optional label slot | `label` (horizontal only) |
| decorative role | `decorative` (default `true` -> `role="none"`, `false` -> `role="separator"` + `aria-orientation`) |
| weight (thin / thick) | `weight` prop |

The single remaining gap was **spacing
variants**. This patch adds them along with
ergonomic data attributes.

## Spacing scale

```ts
type SeparatorSpacing = 'none' | 'sm' | 'md' | 'lg';
```

| spacing | horizontal | vertical |
| --- | --- | --- |
| `none` (default) | no margin (byte-identical) | no margin (byte-identical) |
| `sm` | `my-1` (4px) | `mx-1` (4px) |
| `md` | `my-2` (8px) | `mx-2` (8px) |
| `lg` | `my-4` (16px) | `mx-4` (16px) |

The label variant inherits the same `my-N`
(applied to the outer flex container that
carries the rule edges).

Default `none` keeps the legacy 11.165
layout byte-for-byte. Callers can now
write:

```tsx
<Separator spacing="md" />
```

instead of:

```tsx
<div className="my-2">
  <Separator />
</div>
```

## Data attributes

- `data-section="separator"` on every root.
- `data-orientation` mirrors orientation.
- `data-weight` mirrors weight.
- `data-spacing` mirrors spacing.
- Label variant: `data-section="separator-label"`
  on the inner `<span data-slot="label">`.

Downstream tests + storyshots can branch
without parsing class lists.

## API additions

```ts
interface SeparatorProps {
  // ... existing
  spacing?: 'none' | 'sm' | 'md' | 'lg';
}
```

## Pairs with existing primitives

- `<Divider>` -- N/A, this IS the divider.
- `<Card>` / `<Panel>` -- typical hosts of
  the separator (between header + body,
  between sections).
- `<List>` -- horizontal separators can
  replace per-item bottom borders for a
  cleaner row rhythm.

## Out of scope

- Per-page adoption (`sm` / `md` / `lg`
  rollout). All four spacing values are
  additive; every existing call site stays
  byte-identical at `none`.
- Per-orientation independent spacing
  (e.g., `mySm-mxLg`). The single scale
  applies to the cross-axis only.
- Dashed or dotted rule variants. Solid
  `border-border` line stays.
- Animated transitions. Static layout
  property.
- Label slot for vertical orientation.
  Vertical separators with cross-axis
  labels are awkward; the horizontal-only
  contract stays.

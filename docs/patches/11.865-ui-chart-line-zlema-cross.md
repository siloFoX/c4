## Summary

Added `<ChartLineZlemaCross />` -- a pure-SVG, dual-panel React/TS
primitive that overlays the close with Ehlers' Zero-Lag EMA in
the top panel and renders the `close - ZLEMA` deviation in the
bottom panel. Markers fire at every close-vs-ZLEMA crossover --
the canonical lag-corrected trend event.

Cross events:

- `up` cross   -> regime `trending-up`
- `down` cross -> regime `trending-down`

Defaults: `length = 14`, zlemaColor `#7c3aed` (violet),
deviationColor `#0ea5e9` (sky), bullish marker `#16a34a`,
bearish marker `#dc2626`, zero line `#475569`.

## Formula

```
lag         = floor((length - 1) / 2)
adjusted[i] = 2 * close[i] - close[i - lag]    (i >= lag)
ZLEMA[i]    = EMA(adjusted, length)
```

The `2x` term cancels the natural EMA lag of `(n - 1) / 2` for
a linear series, so ZLEMA tracks the close more tightly than a
plain EMA. Cross detection uses the standard prev-relation +
strict-inequality guard.

## Bit-exact anchor

- **CONST close = K**: `adjusted = 2K - K = K`. The EMA helper
  carries the SMA-seeded `min === max` precision fix, so
  ZLEMA = K bit-exactly once seeded. `close - ZLEMA = 0`,
  relation `equal` forever, regime `aligned`, zero crosses.
  Verified across multiple K and length tuples.

## Soft anchor

For LINEAR UP / DOWN series ZLEMA's design cancels the EMA lag
algebraically, so ZLEMA tracks close in steady state. Floating-
point intermediates can produce 1-ULP drift; the layout still
renders both lines tightly together but the cross detector may
flag spurious flips near the warmup boundary.

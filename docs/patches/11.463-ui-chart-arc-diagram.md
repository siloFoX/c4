## Summary

Add a new `<ChartArcDiagram>` UI primitive in
`web/src/components/ui/chart-arc-diagram.tsx` -- pure-SVG
arc diagram. Nodes sit on a horizontal baseline, each
rendered as a dot whose radius can be scaled by
`node.weight`. Edges paint as semicircular arcs above (or
below) the baseline between the two node xs; the radius
is half the horizontal distance and stroke width encodes
the edge weight. Hover a node to highlight every connected
edge + neighbour; hover an edge to focus that single
connection. Per-node and per-edge click handlers.

## API

```ts
interface ChartArcDiagramNode {
  id: string;
  label: string;
  weight?: number;        // sqrt-scales the node dot radius
  color?: string;
  group?: string;
}

interface ChartArcDiagramEdge {
  source: string;
  target: string;
  weight?: number;        // default 1 -> stroke width
  color?: string;
}

interface ChartArcDiagramProps {
  nodes: readonly ChartArcDiagramNode[];
  edges: readonly ChartArcDiagramEdge[];
  width?: number;                       // default 560
  height?: number;                      // default 240
  padding?: number;                     // default 36
  nodeRadius?: number;                  // default 5
  baselineY?: number;
  arcsBelowBaseline?: boolean;          // default false
  defaultNodeColor?: string;            // default '#2563eb'
  defaultEdgeColor?: string;            // default '#94a3b8'
  edgeOpacity?: number;                 // default 0.45
  showLabels?: boolean;                 // default true
  showTooltip?: boolean;                // default true
  highlightOnHover?: boolean;           // default true
  animate?: boolean;                    // default true
  className?: string;
  ariaLabel?: string;                   // default 'Arc diagram'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  onNodeClick?: (args: {
    node: ChartArcDiagramNode;
    index: number;
    degree: number;
  }) => void;
  onEdgeClick?: (args: {
    edge: ChartArcDiagramEdge;
    source: ChartArcDiagramNode;
    target: ChartArcDiagramNode;
  }) => void;
}
```

## Behaviour

- Renders a `<div role="region">` wrapping `<svg
  role="img">` carrying the same `aria-label`.
- Node layout: `getArcDiagramNodePositions` spreads
  nodes evenly across the inner width on a single
  horizontal line. One-node layouts snap the node to
  the centre.
- Baseline: a thin horizontal line at
  `computedBaselineY` (defaulting to leave room
  under the chart for labels). Adopters can override
  with the `baselineY` prop.
- Edges: `buildArcDiagramEdgePath` builds a
  semicircle of radius `|x2 - x1| / 2` between the
  two node xs. `arcsBelowBaseline=true` flips every
  arc to the lower half. Self-edges (source ==
  target) render as a small loop near the node.
- Stroke width = `max(1, weight + (1 if hovered))`.
  Edges with non-finite / non-positive weight fall
  back to `weight = 1`.
- Hover propagation: hovering a node sets
  `data-hovered` on it and `data-highlighted` on
  every neighbour + every touching edge. Hovering
  an edge highlights both endpoints. When
  `highlightOnHover=true`, non-highlighted elements
  fade (nodes to 0.35, edges to 0.08).
- Per-node tooltip: label + degree + optional group.
  Per-edge tooltip: `source - target` + weight.
- Mount animation:
  `motion-safe:animate-fade-in` per node group + per
  edge group. `animate=false` removes the class.
- Default ARIA description: `"Arc diagram with N
  nodes, M edges. Top: <label> degree <d>, ..."` --
  top-5 by degree. `ariaDescription` overrides;
  empty input produces `"No data"`.
- Edges that reference unknown source / target ids
  are silently dropped so a typo can't crash the
  diagram.

## Pure helpers (exported)

```ts
DEFAULT_CHART_ARC_DIAGRAM_WIDTH         = 560
DEFAULT_CHART_ARC_DIAGRAM_HEIGHT        = 240
DEFAULT_CHART_ARC_DIAGRAM_PADDING       = 36
DEFAULT_CHART_ARC_DIAGRAM_NODE_RADIUS   = 5
DEFAULT_CHART_ARC_DIAGRAM_EDGE_OPACITY  = 0.45
DEFAULT_CHART_ARC_DIAGRAM_NODE_COLOR    = '#2563eb'
DEFAULT_CHART_ARC_DIAGRAM_EDGE_COLOR    = '#94a3b8'

getArcDiagramNodePositions(nodes, innerWidth, originX): number[]
getArcDiagramNodeRadius(node, defaultRadius): number
computeArcDiagramNodeDegree(nodeId, edges): number
buildArcDiagramEdgePath(x1, x2, baselineY, above): string
describeArcDiagram(nodes, edges, formatValue?): string
```

- `getArcDiagramNodePositions` returns `originX +
  i * step` for each node with
  `step = innerWidth / (n - 1)`; one-node layouts
  return `[origin + innerWidth / 2]`; empty input
  returns `[]`.
- `getArcDiagramNodeRadius` scales by
  `sqrt(weight)`; floor 2 px; fallback for
  non-finite / non-positive weight.
- `computeArcDiagramNodeDegree` counts every edge
  incident on `nodeId` (both source and target).
- `buildArcDiagramEdgePath` emits a single
  `M ... A r r 0 0 (1|0) ...` arc; non-finite
  coords -> `""`; self-edges produce a small loop.
- `describeArcDiagram` produces the default ARIA
  summary; honours `formatValue`.

  `data-section="chart-arc-diagram-aria-desc"`.

## Tests (45 cases)

  one-node snap to centre, empty -> [].
- `getArcDiagramNodeRadius`: sqrt scaling,
  fallback for missing / invalid weight, floor 2 px.
- `computeArcDiagramNodeDegree`: incident-edge
  count, isolated node -> 0.
- `buildArcDiagramEdgePath`: non-finite -> "",
  semicircle above, semicircle below, x1 > x2
  ordered left-to-right, self-edge loop.
- `describeArcDiagram`: empty "No data",
  top-degree summary, formatter shaping.

  - baseline renders

  - node tooltip on hover (label + degree)
  - group row when node has group; absent otherwise

  - tooltip hides on mouse-leave; suppression

  - highlight connected nodes + edges on node hover

## Out of scope

- Vertical orientation -- horizontal-only v1; host
  rotates with custom transform when needed.
- Drag-to-reorder nodes along the baseline -- host
  re-renders with a new node order.
- Curved (non-semicircular) edges -- semicircle
  only v1; host can supply a custom SVG layer for
  alternative shapes.
- Animated transitions on data change (snap v1;
  mount fade only).

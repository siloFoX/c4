## Summary

New `ChartLineMesaSine` UI primitive in
`web/src/components/ui/chart-line-mesa-sine.tsx` -- a pure-SVG two-panel
line chart with an **Ehlers MESA Sine Wave** panel plotting the cycle
phase sine and the lead sine. The top panel plots the price; the bottom
panel plots the sine and lead sine inside a fixed [-1, 1] band.

## The computation

John Ehlers' MESA Sine Wave plots the dominant cycle phase as a pair of
sine lines. `ChartLineMesaSinePoint` carries `{x, value}`.

- `computeLineMesaSineSma(values, period)` -- the simple moving average
  used to detrend the price, null in the warm-up window.
- `computeLineMesaSinePair(phase)` -- the sine pair for a phase angle:
  `sine = sin(phase)` and `leadSine = sin(phase + 45 degrees)`.
- `computeLineMesaSine(values, period)` -- detrends the price
  (`cycle = price - SMA`), pairs the cycle with a quarter-period-delayed
  quadrature (a 90-degree phase shift), takes the phase as
  `atan2(quadrature, in-phase)`, and plots the sine and lead sine of
  that phase. When the cycle amplitude is below `MESA_SINE_EPSILON` the
  phase, sine and lead sine are left null.
- `classifyLineMesaSineZone(sine, leadSine)` -- marks a bar **up** when
  the lead sine is above the sine, **down** when below, **flat** when
  level.

## Worked fixture

The phase computation runs through `atan2` and `sin`, so it is
transcendental, but the indicator has clean exact anchors.

The fixture `[8,16,24,32,40,32,24,16]` is chosen so its period-4 moving
average is an exact integer: SMA `[-,-,-,20,28,32,32,28]`, giving the
exact integer cycle `[-,-,-,12,12,0,-8,-12]` (asserted with `toEqual`).
With a quarter delay of one bar the quadrature pair lands the phase on
the clean quadrature angles -- `atan2(12,12) = pi/4`,
`atan2(12,0) = pi/2`, `atan2(0,-8) = pi` -- so the sine and lead sine
hit `sqrt(1/2)`, `1` and the lead values 45 degrees ahead. A constant
series has no cycle: the detrended cycle is exactly zero, the amplitude
falls below the epsilon floor, and the sine and lead sine drop out
entirely. The `computeLineMesaSinePair` helper is checked against its
defining identities, including that the lead sine of a phase equals the
sine of that phase advanced 45 degrees, and both lines stay within the
unit band.

## API

`data` (a `ChartLineMesaSinePoint[]`), `period` (default 20),
`pricePanelRatio` / `gap`, sizing/format/a11y props, `strokeWidth` /
`dotRadius`, the colour props, the toggle flags (`showAxis` / `showGrid`
/ `showDots` default false / `showSine` / `showLeadSine` /
`showZeroLine` / `showMarkers` / `showTooltip` / `showConfigBadge` /
`showLegend` / `animate`), controlled + uncontrolled `hiddenSeries`
(keys `price` / `sine` / `leadSine`), `formatValue` / `formatX`,
`onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineMesaSineFinitePoints`,
`normalizeLineMesaSinePeriod` (coerces the period to an integer of at
least 4), `computeLineMesaSineSma`, `computeLineMesaSinePair`,
`computeLineMesaSine`, `classifyLineMesaSineZone`, `runLineMesaSine`,
`computeLineMesaSineLayout`, `describeLineMesaSineChart`, plus the
`DEFAULT_CHART_LINE_MESA_SINE_*` constants, `MESA_SINE_LEAD`,
`MESA_SINE_EPSILON` and the `ChartLineMesaSineZone` /
`ChartLineMesaSinePair` / `ChartLineMesaSineComputed` types. Every
exported name carries a `LineMesaSine` infix so the `export *` barrel
line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen
`data-section="chart-line-mesa-sine-aria-desc"` description span; the
price line, the sine line, the lead sine line and every dot / marker
are `role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-period`, `data-sine-final`,
`data-lead-sine-final`, `data-up-count`, `data-down-count`,
`data-flat-count`, `data-total-points` and `data-animate`.

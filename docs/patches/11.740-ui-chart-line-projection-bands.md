# Patch 11.740 -- UI: chart-line-projection-bands primitive

Released as `v1.11.758` on 2026-05-26.

## Scope

Add `<ChartLineProjectionBands>` -- a pure-SVG single-panel chart
with Mel Widner Projection Band envelope overlays drawn from the
extremes of the linear-regression-projected highs and lows across
the lookback window.

## Definition

Inputs:

- `data: ChartLineProjectionBandsPoint[]` where
  `ChartLineProjectionBandsPoint = { x; high; low; close }`. Bars
  with `high < low` are dropped.
- `period: number` -- the lookback. Default `14`. Normalized to an
  integer `>= 2`.

Algorithm (per bar `i >= period - 1`):

```
slope_h = OLS slope of high[i-period+1..i] against j = 0..period-1
slope_l = OLS slope of low[i-period+1..i]  against j = 0..period-1
```

For each `k = 0..period - 1` (so `j = i - period + 1 + k`):

```
distance     = i - j = period - 1 - k
hPrime[k]    = high[j] + slope_h * distance
lPrime[k]    = low[j]  + slope_l * distance
```

```
upperBand[i] = max(hPrime[k])
lowerBand[i] = min(lPrime[k])
```

The first `period - 1` bars are null on both bands.

## Bit-exact algebraic anchor

The clean anchor is the **linear-ramp identity**. If
`high = low = c + j` for all bars (a perfect integer ramp):

- OLS on `[c + (i - period + 1), ..., c + i]` against `j =
  0..period - 1`: `sumXX * period - sumX^2 = denom`. The numerator
  reduces by integer cancellation to `denom * 1` (see the
  `chart-line-tsf` patch doc for the same argument applied to the
  TSF endpoint). So `slope_h = slope_l = 1` exactly.
- Each projected high is
  `high[j] + slope * (i - j) = (c + j) + 1 * (i - j) = c + i`
  exactly. Every `k` produces the same integer `c + i`, so
  `upperBand[i] = lowerBand[i] = c + i` bit-exact.
- The band width `upper - lower = 0` exactly.

Symmetrically for `FALLING` (`high = low = c - j`, here `c = 19`),
`slope = -1` exactly, and every projected high reduces to
`c - i` bit-exact.

Constant-series identity (`high = low = K`): `slope = 0` exactly,
projected high / low are all `K`, so `upper = lower = K` bit-exact.

Translation invariance: shifting `high / low / close` by an integer
`K` shifts the LSQ numerator by terms that all cancel, so the slope
is unchanged. Each projected value shifts by exactly `K`, so the
bands shift by exactly `K`.

The slope helper passes the standard integer fixtures bit-exact:

- `[0, 1, 2, 3]` -> `slope = 1`
- `[10, 12, 14, 16]` -> `slope = 2`
- `[3, 2, 1, 0]` -> `slope = -1`
- constant -> `slope = 0`

## Anchor: upper >= current high, lower <= current low

For the latest bar `k = period - 1` the distance is zero:

- `hPrime[period - 1] = high[i] + slope * 0 = high[i]`.
- The max over `k` therefore satisfies `upper >= high[i]`.

Symmetrically `lower <= low[i]`. This holds for every series, not
just integer fixtures.

## Component

`forwardRef<HTMLDivElement, ChartLineProjectionBandsProps>` with
displayName. Renders:

- A region-role outer `<div data-section="chart-line-projection-bands">`.
- An `<svg role="img">` with an `aria-describedby` description.
- Price line as a `<path>` with
  `data-section="chart-line-projection-bands-price-path"`.
- Two stub segments per defined bar
  (`data-section="chart-line-projection-bands-segment"`) coloured
  by `data-series-id="upper"` / `"lower"`.
- Zone-coloured markers
  (`data-section="chart-line-projection-bands-marker"`,
  `data-zone` attribute).
- Legend toggle for `price` / `upper` / `lower`, plus `showUpper`,
  `showLower`, and `onPointClick`.
- ARS design-system tokens for colour and motion (`motion-safe:
  animate-fade-in`).

  FALLING `upper = lower = 19 - i` bit-exact.

  `1000` for integer fixtures.

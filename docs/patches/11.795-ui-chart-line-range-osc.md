## Summary

`ChartLineRangeOsc` is a pure-SVG dual-panel React primitive that
plots the close on the top panel and a Range Oscillator on the
bottom panel. The oscillator subtracts a long-term EMA of true
range from a short-term EMA:

```
TR[i]      = max(high - low, |high - prevClose|, |low - prevClose|)
shortEMA[i] = EMA(TR, shortLength)[i]
longEMA[i]  = EMA(TR, longLength)[i]
rangeOsc[i] = shortEMA[i] - longEMA[i]
```

Defaults: `shortLength = 14`, `longLength = 28`. Bars before
`i = longLength - 1` are warmup nulls so the long-term EMA has
seen at least `longLength` samples in its history. `-0` from EMA
near-equality is normalized to `+0`.

A reading above zero (short > long) signals expanding volatility;
below zero signals contracting volatility.

## Bit-exact anchors

- **CONST_FLAT** (`H = L = C = K`): every TR is zero, both EMAs
  collapse to zero (`0 / 0 = ` not invoked because EMA seeds at
  the first finite value), and `rangeOsc = 0` bit-exact past
  warmup.
- **CONST_BAR with dyadic TR** (`H - L` a power of 2): every TR
  is the constant dyadic value. The EMA-of-constant identity
  `alpha*K + (1 - alpha)*K = K` holds bit-exact in IEEE 754 for
  dyadic K (the multiplications carry no rounding), so both
  short and long EMAs equal TR exactly, and the difference is
  zero bit-exact.

For non-dyadic TR the short and long EMAs drift independently by
up to 1 ULP, and the difference is close to but not exactly zero;
that case is tested with `toBeCloseTo(0, 9)`.

## Zone classifier

The oscillator reading is bucketed by ratio to the abs-max value
observed in the current series:

- `expanding` -- rangeOsc >= 0.5 * absMax (sharp uptick)
- `above` -- 0 < rangeOsc < 0.5 * absMax
- `at` -- rangeOsc == 0
- `below` -- -0.5 * absMax < rangeOsc < 0
- `contracting` -- rangeOsc <= -0.5 * absMax (sharp downtick)
- `none` -- rangeOsc is null (warmup)

When `absMax` is zero (e.g. CONST_FLAT or CONST_BAR dyadic) the
classifier falls back to above / below by sign so the buckets
remain defined for non-zero readings (otherwise it correctly
returns `at`).

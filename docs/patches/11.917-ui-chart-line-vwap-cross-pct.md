# Patch 11.917 -- UI: chart-line-vwap-cross-pct primitive

## Summary

Added `<ChartLineVwapCrossPct />` -- pure-SVG dual-panel
React/TS primitive that renders the close + rolling VWAP line
(top panel) with bullish / bearish arrow overlays at every
close vs VWAP cross trigger, and the percent-normalized
`(close - vwap) / vwap` deviation oscillator (bottom panel)
with cross dot markers and a dashed zero baseline. Percent-
change variant of the VWAP family that scales every signal to
the VWAP for cross-instrument comparable volume-weighted bias
magnitude.

## Algorithm

```
tp[i]    = close[i]                                  (typical price)
vol[i]   = close[i]                                  (volume proxy)
vwap[i]  = sum(tp * vol, length) / sum(vol, length)
         = sum(close^2, length) / sum(close, length)
pct[i]   = vwap > 0 ? ((close - vwap) / vwap) * 100 : 0
bullish : pct crosses up   (prev <= 0, cur > 0)
bearish : pct crosses down (prev >= 0, cur < 0)
```

Default `length = 20` (rolling VWAP window). Regime classifier
`bullish` (pct > 0), `bearish` (pct < 0), `neutral` (pct ===
0), `none` (pct null). Divide-by-zero guard returns 0 when
VWAP is non-positive.

## Bit-exact anchor

CONST close = K (K > 0): VWAP = K every bar via the `min ===
max` short-circuit; pct = (K - K) / K * 100 = 0. Regime
`neutral`, cross count = 0. CONST K = 0 falls through the
divide-by-zero guard to pct = 0, also regime `neutral`.
Verified across K = 0, 1, 5, 17, 100, 1234.

LINEAR UP with step 1 yields pct > 0 (close above lagging
VWAP) -> regime `bullish`.

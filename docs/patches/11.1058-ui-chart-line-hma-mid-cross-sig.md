# 11.1058 -- UI: chart-line-hma-mid-cross-sig primitive (v1.11.1076)

## Scope

Add a pure-SVG dual-panel `ChartLineHmaMidCrossSig`
component that renders the close in the top panel and the
Hull Moving Average (HMA) plus its smoothed SMA signal
line in the bottom panel, marking bullish / bearish HMA-
over-signal crossover trigger events with bias coloring
derived from the HMA slope at the trigger bar.

Alan Hull's 2005 Hull Moving Average is engineered to
virtually eliminate lag while preserving smoothness. The
formula `2 * WMA(close, period/2) - WMA(close, period)`
projects the trend forward by the WMA's centroid lag, and
the outer WMA over `round(sqrt(period))` bars smooths the
projection. The net effect is a near-zero-lag MA that
follows the trend tightly but without the noise of a pure
unsmoothed projection.

Sibling to chart-line-hma-divergence-cross v1.11.1052
(same HMA pipeline; the divergence-cross sibling compares
HMA direction vs price direction, this cross-sig compares
HMA vs its SMA signal line).

## HMA cross-sig pipeline

```
half        = floor(period / 2)
sqrtPeriod  = round(sqrt(period))
wmaHalf[i]  = WMA(close, half)
wmaFull[i]  = WMA(close, period)
raw[i]      = 2 * wmaHalf[i] - wmaFull[i]
hma[i]      = WMA(raw, sqrtPeriod)
signal[i]   = SMA(hma, signalLength)

bullish     : prev hma <= prev signal &&
              cur hma > cur signal
bearish     : prev hma >= prev signal &&
              cur hma < cur signal
regime      : 'bullish' when hma >= signal
              'bearish' when hma <  signal
              'none'    when either is null
bias        : hma[i] vs hma[i-1] -> up / down / flat /
              none
```

Default `period = 14`, `signalLength = 3` -- canonical
Hull tuning paired with the family-conventional short SMA
signal. For period=14: half=7, sqrtPeriod=round(sqrt(14))
=4.

Warmup is `period + sqrtPeriod + signalLength - 3 = 18`
for the default tuning: wmaHalf seeds at i = half - 1 = 6,
wmaFull seeds at i = period - 1 = 13, raw valid from i =
13, HMA valid from i = 13 + sqrtPeriod - 1 = 16, signal
valid from i = 16 + signalLength - 1 = 18.

## Bit-exact anchors

All anchors use close-only input.

- **CONST band** `close = K`: WMA on a constant is
  exactly the constant. wmaHalf = wmaFull = K. raw = 2 *
  K - K = K. hma = K. signal = SMA(K, 3) = K. hma ===
  signal -> regime `bullish` (>=) for every valid bar.
  0 crosses. Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `close = i`: WMA centroid lag on linear
  input is `(n - 1) / 3`. wmaHalf = WMA(close, 7) = i -
  (7 - 1) / 3 = i - 2. wmaFull = WMA(close, 14) = i -
  13 / 3. raw = 2 * (i - 2) - (i - 13/3) = i - 4 + 13/3
  = i + 1/3. hma = WMA(raw, 4) = raw - (4 - 1) / 3 =
  raw - 1 = i - 2/3. The +1/3 lift on `raw` and the -1
  from the outer WMA cancel net 2/3 of lag -- the
  canonical near-zero-lag Hull property. signal = SMA
  (hma, 3) = hma - 1 = i - 5/3. hma - signal = 1
  (constant). regime `bullish` (>=). 0 crosses. The
  constant +1 offset is the 1-bar SMA lag on a unit-
  slope line.
- **LINEAR DOWN** `close = -i`: mirror -> hma = -i +
  2/3. signal = -i + 5/3. hma - signal = -1. regime
  `bearish` for every valid bar. 0 crosses.

The HMA lag on LINEAR is just 2/3 of a bar, much less
than SMA (lag = (n-1)/2 = 6.5), EMA (same), or WMA
(lag = (n-1)/3 = 13/3 ~ 4.33). This near-zero-lag
property is the canonical Hull MA design feature.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 3-state regime classifier follows the cross-sig family
  convention (bullish on `===`, bearish on `<`).
- HMA's near-zero-lag property comes at the cost of some
  responsiveness to noise -- the outer WMA over sqrt
  (period) smooths but does not eliminate it. On steady
  LINEAR input the HMA produces a constant slope and the
  +1/-1 offset against the SMA signal yields zero
  crosses -- correct canonical behaviour.
- Hull's original formula uses `round(sqrt(period))` for
  the outer WMA length. Some libraries use `floor` or
  `ceil` -- this primitive uses `round` per the original
  Hull definition.
- WMA's `(n-1)/3` centroid lag on linear input is the
  canonical formula. Some libraries use slightly
  different weighting -- this primitive uses the
  standard linearly-decreasing weight `(n, n-1, ..., 1)`
  normalized by `n*(n+1)/2`.
- Sibling to chart-line-hma-divergence-cross v1.11.1052
  (same HMA pipeline, different signal); together they
  bracket the Hull MA family in the mid-cross-sig and
  divergence-cross conventions.
- Complementary to the sibling SMA / EMA / Bollinger /
  Keltner / Donchian / CCI / RSI / Ichimoku /
  Supertrend / Williams / DPO / TRIX mid-cross-sig
  primitives -- HMA adds a near-zero-lag fast-response
  variant to the family.

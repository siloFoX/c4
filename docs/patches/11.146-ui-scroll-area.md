# 11.146 -- UI: ScrollArea primitive (v1.11.164)

New `<ScrollArea>` primitive in `web/src/components/ui/scroll-area.tsx`.

## Shape

```tsx
<ScrollArea
  axis="y" | "x" | "both"           // default "y"
  maxHeight={256 | "50vh"}
  height={120 | "100%"}
  className="..."                    // merged via cn()
  ref={ref}                          // forwarded to the scroll container
  {...HTMLAttributes}                // role, aria-*, onScroll, etc.
>
  ...children
</ScrollArea>
```

Renders a `<div>` with `.c4-scroll` (new utility class in `web/src/index.css`)
plus axis-mapped `overflow-*` Tailwind classes. The `.c4-scroll` rule themes
both Firefox (`scrollbar-width: thin` + `scrollbar-color`) and WebKit
(`::-webkit-scrollbar*`) so the thumb starts transparent and fades in on
`:hover` / `:focus-within`. Track is transparent; thumb uses
`hsl(var(--muted-foreground) / 0.4)` (rises to `/ 0.6` on direct thumb hover)
so dark/light mode resolves automatically.

`maxHeight` / `height` accept numbers (treated as px) or any CSS string. Caller
inline `style` is merged after the size props so callers can still override.

## Tests

`web/src/components/ui/scroll-area.test.tsx` -- 13 vitest cases covering
children render, maxHeight numeric + string, height numeric, default axis=y,
axis=x and axis=both class wiring, ref forwarding, className merging, default
classes present, inline style merge, HTML attribute pass-through, and
unset-size-prop behaviour.

Note: `node_modules` is not installed in this worktree so vitest was not
executed in-tree; the suite uses the same `@testing-library/react` + `vitest`
setup as the surrounding UI primitives.

## Adoption sites (4)

1. `web/src/components/layout/Sidebar.tsx` -- inner nav list wrapper. Replaces
   `<div class="... overflow-y-auto ...">` around the workers/hierarchy list.
2. `web/src/components/ChatMessageLog.tsx` -- chat message log container.
   Replaces `<div ref={scrollRef} ... overflow-y-auto>` while keeping
   `onScroll`, `role="log"`, `aria-live`, and `aria-label` via spread.
3. `web/src/pages/Queue.tsx` -- the Automation > Queue table wrapper. Uses
   `axis="x"` (horizontal-only) for the wide queue table.
4. `web/src/components/SpecialistsAuditPanel.tsx` -- audit log viewer
   (`max-h-64 overflow-y-auto`). **Alternative substitution**: the task
   specified the Health log viewer, but `pages/Health.tsx` has no scrollable
   log container at present, so the audit log viewer (an explicitly-listed
   fallback target in the task) was used in its place.

No other UI primitive was modified. `docs/autonomous-queue-v10.md` and
`package-lock.json` are untouched.

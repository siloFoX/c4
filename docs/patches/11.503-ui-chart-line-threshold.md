# TODO 11.503 - chart-line-threshold primitive

Status: SHIPPED v1.11.521 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartLineThreshold>`, a pure-SVG multi-series line chart with
one or more **horizontal threshold reference lines** plus optional
**zone shading** above or below each threshold. The canonical SLO /
SLA / alert-threshold pattern: render a metric over time, drop a
horizontal rule at the threshold value, shade the "warning" or
"danger" zone, and surface which thresholds the hovered point has
crossed in the tooltip.

## API

```tsx
<ChartLineThreshold
  series={[
    {
      id: 'p95',
      label: 'p95 latency',
      data: [
        { x: 0, y: 30 }, { x: 1, y: 55 }, { x: 2, y: 75 },
        { x: 3, y: 92 }, { x: 4, y: 110 },
      ],
    },
  ]}
  thresholds={[
    { id: 'warn', label: 'Warning', value: 80, zone: 'above', color: '#f59e0b' },
    { id: 'crit', label: 'Critical', value: 100, zone: 'above', color: '#dc2626' },
  ]}
/>
```

## Distinct from prior chart-* primitives

- `<ChartLine>` - no threshold concept; reference lines must be
  composed externally.
- `<ChartLineConfidence>` (11.502) - per-point upper/lower bounds
  drawn as a filled polygon between curves; chart-line-threshold's
  zone is a fixed horizontal band defined by a single value.
- `<ChartLineZoom>` (11.501) / `<ChartLineMulti>` (11.500) -
  interactions (brush, crosshair); no thresholds.
- `<ChartLineDashed>` (11.499) - dash patterns; thresholds use a
  dashed reference line by default but support custom dashArray.
- `<ChartLineStep>` (11.497) / `<ChartLineSmooth>` (11.498) -
  interpolation variants; no thresholds.

## Threshold contract

### Threshold spec

`ChartLineThresholdSpec = { id, label, value, color?, zone?,
zoneColor?, dashArray? }`. Each threshold:

- `value` (required, must be finite): the y position of the
  horizontal reference line.
- `zone`: `'above'` shades the band from the threshold up to the
  top of the plot, `'below'` shades from the threshold down to the
  bottom, `'none'` (default) renders no zone shading.
- `color` (default `DEFAULT_CHART_LINE_THRESHOLD_DEFAULT_COLOR =
  '#dc2626'`): used for both the reference line stroke and the zone
  fill (unless overridden by `zoneColor`).
- `dashArray` (default `'6 4'`): SVG dasharray for the reference
  line so it visually contrasts with the series strokes.

### Layout

- `getLineThresholdBounds(series, thresholds, hidden?)` expands the
  y range to include every finite threshold value so reference
  lines are always visible.
- `getLineThresholdFiniteThresholds(thresholds)` drops thresholds
  with non-string ids or non-finite values.
- Each threshold computes `py` (its pixel y position) and
  `inRange` (true when value is within the active y bounds; the
  reference line + zone are skipped when out of range).
- Zone rectangle: `'above'` -> `{x: padding, y: padding, width:
  innerWidth, height: py - padding}`; `'below'` -> `{x: padding, y:
  py, width: innerWidth, height: padding + innerHeight - py}`.
- Per-point `thresholdState[]` records `isAbove` / `isBelow` /
  `isInZone` per threshold so tests + adopters can ask "did this
  point cross a threshold?".

### Render order (back-to-front)

1. Zone fills (`data-section="chart-line-threshold-zone"`).
2. Grid lines.
3. Axes + tick labels.
4. Series paths + dots.
5. Threshold reference lines + labels
   (`data-section="chart-line-threshold-reference"`).

Putting the reference line on top of the series stroke is
deliberate: at the threshold, the reference line wins visual
priority so the crossing point is unambiguous.

## Interactivity

- Hover or focus on a dot dims every other series stroke + dot to
  ~30% and opens a tooltip with series label, formatted x, formatted
  y, and (when the dot has crossed any zone-shaded threshold from
  below) a `above:` row listing each crossed threshold.
- Per-dot `data-in-zone="true"` exposes the in-zone state to tests
  and adopters who want to colour-code in-zone points.
- Per-dot `onPointClick({series, point})` callback.
- Threshold reference lines are focusable
  (`role="graphics-symbol"`, `tabIndex={0}`) so keyboard users can
  inspect each threshold via the screen reader-friendly aria-label.
- Legend toggles series visibility AND shows one
  `data-section="chart-line-threshold-legend-threshold"` item per
  threshold so the threshold dash pattern + colour are part of the
  legend even though thresholds are not toggleable.
- Mount fade via `motion-safe:animate-fade-in` (suppress via
  `animate=false`).

## Exported helpers

- `DEFAULT_CHART_LINE_THRESHOLD_*` numeric / string defaults.
- `DEFAULT_CHART_LINE_THRESHOLD_PALETTE` (10-colour palette).
- `getLineThresholdDefaultColor(index)` - palette wrap.
- `getLineThresholdFinitePoints(points)` - drops non-finite.
- `getLineThresholdFiniteThresholds(thresholds)` - drops thresholds
  with non-string ids / non-finite values.
- `getLineThresholdBounds(series, thresholds, hidden?)` - per-axis
  bounds with threshold expansion.
- `getLineThresholdTicks(min, max, count?)` - evenly-spaced ticks.
- `buildLineThresholdPath(points)` - `M x y L x y ...` line builder.
- `classifyLineThresholdPoint(pointY, threshold)` - returns
  `{ isAbove, isBelow, isInZone }`. A point exactly on the
  threshold is neither above nor below.
- `computeLineThresholdLayout(input)` - per-series layout + per-
  threshold pixel positions + per-point `thresholdState[]`.
- `describeLineThresholdChart(series, thresholds, hidden?,
  formatValue?)` - ARIA summary listing each threshold's value and
  zone.

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only
  `data-section="chart-line-threshold-aria-desc"` span.
- SVG `role="img"` with the same `aria-label`.
- Each line path is `role="graphics-symbol"` with `tabIndex={0}`.
- Each dot is `role="graphics-symbol"` with `tabIndex={0}`.
- Each threshold reference line is `role="graphics-symbol"` with
  `tabIndex={0}` and `aria-label="Threshold <label> at <value>,
  <zone> zone"` (zone omitted when `zone='none'`).

## Data attributes

- Root: `data-section="chart-line-threshold"`, `data-series-count`,
  `data-visible-series-count`, `data-total-points`,
  `data-threshold-count`, `data-animate`.
- Series group + path + dot: standard pattern.
- Dot: extra `data-in-zone="true|false"` flag indicating whether
  the point falls inside ANY zone-shaded threshold.
- Zone: `data-section="chart-line-threshold-zone"`,
  `data-threshold-id`, `data-threshold-zone`, `data-zone-y`,
  `data-zone-height`.
- Reference: `data-section="chart-line-threshold-reference"`,
  `data-threshold-id`, `data-threshold-index`, `data-threshold-value`,
  `data-threshold-zone`, `data-threshold-color`, `data-threshold-py`.
- Tooltip: `-tooltip-crossed` row appears only when the point is
  above any threshold.
- Legend: standard series items PLUS
  `data-section="chart-line-threshold-legend-threshold"` per
  threshold showing the dash pattern in the swatch.

## Out of scope

- Slanted / per-x thresholds (e.g. linearly-decreasing limits) -
  v1 supports horizontal thresholds only; for slanted limits compose
  a second series as the limit line.
- Animated zone fills - snap; mount fade only.
- Min/Max zones (two thresholds defining a band) - host adds two
  thresholds (one above, one below) and adopts the visual layering.
- Log-scale axes - linear only.
- Per-segment classification (e.g. mark which line segments cross a
  threshold) - per-point only in v1.

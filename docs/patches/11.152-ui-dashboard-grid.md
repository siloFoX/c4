# 11.152 -- UI: DashboardGrid layout primitive

Released in v1.11.170.

New `<DashboardGrid>` + `<DashboardGrid.Item>` primitive at
`web/src/components/ui/dashboard-grid.tsx`. Responsive 12-column CSS
grid container plus an Item child whose column span is configurable
per breakpoint.

## Props

`DashboardGrid`:
- `gap?: 'sm' | 'md' | 'lg'` (default `'md'`) -- maps to
  `gap-2` / `gap-4` / `gap-6`
- `className?: string`
- `children: ReactNode`
- forwardRef to the underlying `<div>`

`DashboardGrid.Item` (also exported as `DashboardGridItem`):
- `span?: 1..12 | 'full'` (default `'full'`)
- `smSpan?: 1..12 | 'full'`
- `mdSpan?: 1..12 | 'full'`
- `lgSpan?: 1..12 | 'full'`
- `className?: string`
- `children: ReactNode`
- forwardRef to the underlying `<div>`

Span values map to static `col-span-N` / `sm:col-span-N` /
`md:col-span-N` / `lg:col-span-N` Tailwind classes via a switch on
the literal value, so the JIT content scanner can pick them up
without runtime string interpolation.

## Adoption sites (2)

1. `web/src/pages/Health.tsx` -- the hero metrics row containing
   three `StatCard`s (Uptime trend / Workers / Queue trend) was
   wrapped in a `DashboardGrid gap="sm"` with each StatCard inside
   a `DashboardGrid.Item span="full" smSpan={6} lgSpan={4}`,
   replacing the previous `grid grid-cols-1 sm:grid-cols-2
   lg:grid-cols-3` div. The `data-stat-card-trends` data attribute
   is preserved on the new grid container.
2. `web/src/pages/Auto.tsx` -- the live-queue + timeline
   two-column panel below `HeroStats` was wrapped in a
   `DashboardGrid gap="md"` with `LiveQueueSection` in a
   `lgSpan={8}` item and `TimelineSection` in a `lgSpan={4}`
   item, replacing the previous `grid grid-cols-1 xl:grid-cols-3`
   + `xl:col-span-2` pair. The breakpoint shifted from `xl` to
   `lg` to match the primitive's responsive scale.

## Tests

`web/src/components/ui/dashboard-grid.test.tsx` covers 12 cases
(children render, default gap, gap=sm/lg, span=6, span='full',
default span 'full', smSpan/mdSpan/lgSpan individually, combined
breakpoints, className merges on both grid and item, and
`DashboardGrid.Item === DashboardGridItem` identity).

`npx vitest run web/src/components/ui/dashboard-grid.test.tsx` was
not exercised in this worktree because `web/node_modules` is not
populated -- run from the main checkout.

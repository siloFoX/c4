# 11.1039 -- UI: chart-line-mfi-mid-cross-sig primitive (v1.11.1057)

## Scope

Add a pure-SVG dual-panel `ChartLineMfiMidCrossSig`
component that renders the close in the top panel and the
Money Flow Index (MFI) plus its smoothed SMA signal line in
the bottom panel, marking bullish (MFI crosses up through
signal -- volume-weighted momentum trend trigger up) /
bearish (MFI crosses down through signal -- volume-weighted
momentum trend trigger down) MFI-over-signal crossover
trigger events with bias coloring derived from the MFI
slope at the trigger bar.

MFI is the volume-weighted RSI: it produces a momentum
reading bounded in `[0, 100]` and uses typical-price *
volume magnitude as the money-flow quantity. The conventional
centerline is 50 -- a neutral level above which momentum
reads bullish and below which bearish. This primitive
renders the 50-line as a dashed reference and runs the
canonical cross-sig pattern on MFI vs its SMA signal.

## MFI centerline-over-signal pipeline

```
typical[i]   = (high[i] + low[i] + close[i]) / 3
rawMf[i]     = |typical[i] * volume[i]|
posMf[i]     = rawMf[i] when typical[i] > typical[i-1]
                else 0
negMf[i]     = rawMf[i] when typical[i] < typical[i-1]
                else 0
sumPos[i]    = sum(posMf, period)
sumNeg[i]    = sum(negMf, period)
mfi[i]       = sumPos = sumNeg = 0   -> 50
               sumNeg = 0, sumPos>0  -> 100
               sumPos = 0, sumNeg>0  -> 0
               otherwise: 100 - 100 / (1 + sumPos/sumNeg)
signal[i]    = SMA(mfi, signalLength)

bullish      : prev mfi <= prev signal && cur mfi > cur signal
bearish      : prev mfi >= prev signal && cur mfi < cur signal
regime       : 'bullish' when mfi >= signal
               'bearish' when mfi <  signal
               'none'    when either is null
bias         : mfi[i] vs mfi[i-1] -> up / down / flat / none
```

Default `period = 14` (canonical MFI lookback) and
`signalLength = 3`. Warmup is `period + signalLength - 1
= 16`: posMf / negMf seed at `i >= 1`, the rolling sum
first fills at `i = period = 14`, then the signal SMA
adds another `signalLength - 1 = 2` bars.

The `|typical * volume|` magnitude (rather than the raw
signed product) is the textbook MFI convention: it makes
the indicator well-defined even for synthetic test data
with negative typical prices, which would otherwise turn
the money-flow contribution negative and break the bounded
`[0, 100]` range.

## Bit-exact anchors

All anchors use HLCV input with constant volume `V = 1`.

- **CONST band** `high = K + 1`, `low = K - 1`, `close
  = K`, `volume = 1`: typical = K constant. No directional
  change between bars -> posMf = negMf = 0 for every i >=
  1. Both sums zero -> MFI defaults to 50 (neutral).
  signal = SMA(50, 3) = 50. MFI === signal -> regime
  `bullish` (>=). 0 crosses. Verified across K in {0, 1,
  50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close
  = i`, `volume = 1`: typical = i monotone up. Every i
  >= 1 contributes only to posMf. sumNeg = 0 -> MFI =
  100. signal = 100. MFI === signal -> regime `bullish`.
  0 crosses.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`, `close
  = -i`, `volume = 1`: typical = -i monotone down. Every
  i >= 1 contributes only to negMf. sumPos = 0 -> MFI =
  0. signal = 0. MFI === signal -> regime `bullish` (>=).
  0 crosses.
- **MIXED DIRECTION** alternating typical 100 / 102 (close
  / high / low all shifted by the same constant): MFI
  stays in the open interval `(0, 100)` for every valid
  bar -- the alternating pattern produces non-zero sumPos
  and sumNeg, so neither zero-guard branch fires and the
  general `100 - 100 / (1 + sumPos / sumNeg)` formula is
  exercised.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- Strict-inequality bullish vs `>=` regime classifier
  means the bullish boundary (MFI === signal) settles
  into the bullish band -- matches every other
  chart-line-*-cross-sig sibling.
- Three guard branches (sumPos = sumNeg = 0 -> 50; sumNeg
  = 0 -> 100; sumPos = 0 -> 0) prevent division by zero
  in the canonical `100 - 100 / (1 + sumPos / sumNeg)`
  formula. The neutral-50 fallback on a fully-flat window
  is the standard convention in StockCharts /
  TradingView / TA-Lib.
- Uses `|typical * volume|` magnitude for the money-flow
  quantity so the indicator remains well-defined under
  synthetic test inputs with negative typical prices.
- Oscillator panel is hard-locked to `[0, 100]` so the
  centerline reference at 50 always renders at the same
  fractional height -- the bounded-MFI invariant lets
  the layout pin the y-range without scanning the data.
- Negative volume is filtered at the input boundary --
  the typical accounting convention treats volume as
  non-negative.
- Uses the same OBV / VWAP family ink colour
  (`#0d9488`, teal) since MFI is the third volume-aware
  primitive in the family -- visual grouping aids dashboard
  composition.

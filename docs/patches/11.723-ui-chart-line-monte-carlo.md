## Summary

`web/src/components/ui/chart-line-monte-carlo.tsx` ships
`<ChartLineMonteCarlo>`, a pure-SVG single-panel chart with a **Monte
Carlo forecast fan** overlay from bootstrapped return samples. The
historical price line sits on the left of the panel, the forecast fan
extends to the right past a dashed divider; a median line carries the
center of the fan and the band between the lower / upper quantiles is
shaded.

## Computation

The forecast is a non-parametric bootstrap. Given a historical price
series, the bar-to-bar returns are pooled. For each of `simulations`
sample paths and each of `horizon` future steps, one return is drawn
uniformly **with replacement** from the pool and accumulated onto the
last historical price:

```
paths[s][0] = lastPrice  for all s
for step = 1..horizon:
  for s = 1..simulations:
    paths[s][step] = paths[s][step-1] + sample(returns, rng)
```

The per-step quantile fan is collected from the simulated paths:
`lower` at `quantileLow`, `median` at 0.5, `upper` at `quantileHigh`,
plus the arithmetic mean across the simulations.

The PRNG is a deterministic **Mulberry32** seeded from `seed`, so two
runs with the same input return the same fan bit-for-bit.

## Worked anchors

The bootstrap is intrinsically stochastic, so a generic input cannot
be expected to give a bit-exact fan. The implementation has three
algebraically exact anchors that survive the random sampling:

- **Zero-variance pool** -- `computeLineMonteCarloForecast(100,
  [0, 0, 0, 0], opts)`. Every bootstrapped sample is 0, every path
  stays at 100, so `lower = median = upper = 100` at every horizon
  step. Asserted with `toBe(100)` per step, for any seed.
- **Single-value pool** -- a pool with one unique value `k` (e.g.
  `[1, 1, 1, 1]` or `[-1, -1, -1, -1]`). Every bootstrapped sample
  is `k`, every path at step `i` equals `lastPrice + i * k`. The
  fan width is zero. Asserted with `toBe(100 + step.step)` and
  `toBe(100 - step.step)`.
- **Determinism** -- `computeLineMonteCarloForecast(100,
  [1, -1, 2, -2], {seed: 42, ...})` is `toEqual` itself on a
  re-run. Mulberry32 is fully deterministic from the seed.
- **Different seeds** -- different seeds on a variable pool produce
  different fans (`not.toEqual`).
- **Quantile order** -- `lower <= median <= upper` at every step
  by construction of the floor-quantile rule on the sorted-ascending
  snapshot.

The `computeLineMonteCarloReturns([10, 11, 12, 14])` returns
`[1, 1, 2]` exact; a constant input returns `[0, 0, 0]` exact; a
single bar yields `[]`. The Mulberry32 contract is tested directly:
1000 outputs all in `[0, 1)`, deterministic across two instances of
the same seed, distinct across different seeds.

## Distinct from neighbouring primitives

- vs `<ChartLineBayesian>` -- the Bayesian Trend is a posterior over
  THREE hypothesised parametric models, producing three calibrated
  probabilities that sum to one. The Monte Carlo Fan is a NON-
  parametric bootstrap of the empirical return distribution,
  producing a quantile band over the projected path.
- vs `<ChartLineRsi>` / `<ChartLineCmo>` / `<ChartLineAroonOsc>` --
  those are deterministic indicators over the existing series; the
  Monte Carlo Fan projects FORWARD past the last bar.
- vs `<ChartLineGauss>` / `<ChartLineButterworth>` -- those are
  deterministic smoothers over the existing series; the Monte Carlo
  Fan is a forecast.
- vs `<ChartLineErgodic>` -- the Ergodic Oscillator is a triple-
  smoothed momentum ratio over the existing series; the Monte Carlo
  Fan extrapolates.

## API

`data` -- a single `ChartLineMonteCarloPoint[]` (`{x, value}`).
`horizon` (default 20, coerced to an integer >= 1). `simulations`
(default 200, coerced to an integer >= 1). `seed` (default 1,
coerced to a non-negative integer). `quantileLow` / `quantileHigh`
(default 0.1 / 0.9, coerced to strictly inside `(0, 1)`). Standard
sizing / colour / a11y props plus `strokeWidth`, `dotRadius`,
`tickCount`, `fanOpacity` (the shaded band's alpha). Toggle flags:
`showAxis`, `showGrid`, `showDots` (default false), `showFan`,
`showMedian`, `showBridge`, `showMarkers`, `showTooltip`,
`showConfigBadge`, `showLegend`, `animate`. Controlled and
uncontrolled visibility via `hiddenSeries`
(`'history' | 'median' | 'fan'`). Callbacks:
`onPointClick({step})`, `onSeriesToggle({seriesId, hidden})`.
Format functions: `formatValue`, `formatX`. ARIA: `ariaLabel`,
`ariaDescription`.

## Pure helpers

- `DEFAULT_CHART_LINE_MONTE_CARLO_*` -- defaults.
- `getLineMonteCarloFinitePoints` -- keep only points with a finite
  x and value.
- `normalizeLineMonteCarloHorizon(horizon, fallback)` -- integer
  >= 1; else fallback.
- `normalizeLineMonteCarloSimulations(simulations, fallback)` --
  integer >= 1; else fallback.
- `normalizeLineMonteCarloSeed(seed, fallback)` -- non-negative
  integer; else fallback.
- `normalizeLineMonteCarloQuantile(q, fallback)` -- finite in `(0,
  1)`; else fallback.
- `computeLineMonteCarloReturns(values)` -- bar-to-bar differences,
  skipping bars that point at a non-finite value.
- `computeLineMonteCarloQuantile(sortedAsc, q)` -- floor-quantile on
  a sorted-ascending array.
- `createLineMonteCarloRng(seed)` -- a deterministic Mulberry32
  PRNG returning a `() => number` in `[0, 1)`.
- `computeLineMonteCarloForecast(lastPrice, returns, options?,
  lastX?)` -- run the bootstrap, return one
  `{step, x, lower, median, upper, mean}` per horizon step.
- `runLineMonteCarlo(points, options?)` --
  `{history, returns, forecast, seed, horizon, simulations,
  quantileLow, quantileHigh, lastPrice, ok}`.
- `computeLineMonteCarloLayout({data, horizon, ...})` -- single
  panel with the history line, dashed forecast divider, fan band,
  median line, dashed upper / lower lines, one marker per horizon
  step on the median, and a bridge segment from the last history
  point to the first median point.
- `describeLineMonteCarloChart(data, options?)` -- ARIA summary
  mentioning the indicator, the horizon, the simulations, the
  bootstrap-with-replacement procedure and the final median.

## a11y / data hooks

Root `<div role="region">` with `aria-label` + `aria-describedby`; the
SVG is `role="img"` with the same label; an sr-only
`data-section="chart-line-monte-carlo-aria-desc"` span carries the
description. The history line, every history dot, the median path
and every marker carry `role="graphics-symbol"` and `tabIndex={0}`.
Every node carries a `data-section`. Root mirrors `data-horizon`,
`data-simulations`, `data-seed`, `data-quantile-low`,
`data-quantile-high`, `data-last-price`, `data-forecast-steps`,
`data-history-points`, `data-animate`, `data-empty`. Markers expose
`data-step`, `data-median`, `data-lower`, `data-upper`.

# 11.534 -- web/src/components/ui/chart-line-control.tsx

Pure-SVG **Statistical Process Control (SPC)** chart. Each
series renders against three horizontal reference rules: the
**center line (CL)** at the series mean, an **upper control
limit (UCL)** at `mean + kSigma * sigma`, and a **lower control
limit (LCL)** at `mean - kSigma * sigma`. Default `kSigma = 3`
(the canonical Western Electric Rule 1 threshold). A faint
in-control band fills the area between UCL and LCL; points
inside the band render in the in-control colour while points
outside render in the out-of-control colour (red by default) at
a slightly larger radius. A control badge in the top-left calls
out the total count of out-of-control points with an alert / ok
state.

## Distinct from other reference / outlier primitives

- `<ChartLineConfidence>` (11.502): per-point upper / lower
  bounds supplied EXTERNALLY (e.g. from a model). Control chart
  DERIVES the limits statistically from the series itself.
- `<ChartLineThreshold>` (11.503): arbitrary user-defined
  horizontal thresholds + zones. Control chart's thresholds are
  the standardized SPC mean +- k*sigma rules.
- `<ChartLineBaseline>` (11.510): single flat horizontal
  reference. Control chart has THREE coordinated rules driven by
  the same statistics.
- `<ChartLinePercentile>` (11.514): distribution percentile
  bands per x bucket (requires multi-sample input `{x,
  values[]}`). Control chart takes single-sample `{x, y}` input
  and applies global statistics.
- `<ChartLineMinMax>` (11.517): extrema markers + connecting
  rules. Control chart's lines are STATISTICAL limits, not
  observed extrema.
- `<ChartLineTarget>` (11.518): vs a flat user-supplied target
  with over / under shading. Control chart has both upper AND
  lower limits derived from data.
- `<ChartLineForecast>` (11.519): time-axis split with a
  prediction interval. Control chart applies the same rules to
  the whole series.
- `<ChartLineAnomaly>` (11.524): per-point z-score outlier
  highlighting with optional rolling window. Control chart adds
  the canonical CL / UCL / LCL **rules** (the lines themselves
  are the centrepiece) and emphasizes the **count** of
  out-of-control points as an alert metric.

The canonical SPC chart used in manufacturing quality control,
SRE error-budget tracking, and any context where the question is
"is the process in statistical control?".

## Public surface

- `<ChartLineControl series={...} />`.
- `series: ChartLineControlSeries[]` -- `{id, label, data,
  color?, kSigma?, centerLine?, sigma?}`. Per-series overrides
  (kSigma, centerLine, sigma) always beat chart-level defaults.
  `centerLine` overrides the data-derived mean (target chart);
  `sigma` overrides the data-derived population standard
  deviation (when an external estimate is available).
- `kSigma` (default `3`) -- limit width in multiples of sigma.
- `centerColor` / `limitColor` / `outColor` / `inColor` --
  reference + per-point colours.
- `fillOpacity` (default `0.08`) -- in-control band fill.
- `limitDashArray` (default `'6 4'`) -- UCL / LCL dash pattern;
  CL renders solid.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showControlBadge`, `showInControlFill`,
  `animate`.
- Controlled + uncontrolled visibility via `hiddenSeries` /
  `defaultHiddenSeries` / `onHiddenSeriesChange`.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX` / `formatSigma`.
- `onPointClick({series, point})`.

## Pure helpers

- `getLineControlDefaultColor(index)` -- 10-color palette cycle;
  NaN / negative -> color 0.
- `getLineControlFinitePoints(points)` -- finite-only filter.
- `normaliseLineControlKSigma(value)` -- clamps to > 0;
  non-finite / non-positive -> default 3.
- `computeLineControlStats(points, {kSigma?, centerLine?,
  sigma?})` -- canonical SPC statistics helper. Returns
  `{mean, sigma, ucl, lcl, count, ok, kSigma}`. Empty / null
  -> `ok: false` with zeros. Sigma is the **population**
  standard deviation (divide by N) so a single point yields
  sigma=0 (and UCL=LCL=mean). `centerLine` / `sigma` overrides
  beat the data-derived values; data-derived sigma is still
  computed from the actual values even when `centerLine` is
  overridden.
- `classifyLineControlState(value, stats)` -- `'in' | 'above' |
  'below'`. Non-finite value -> `'in'`. Not-ok stats -> `'in'`.
  Boundary values are treated as `'in'` (only STRICT excursions
  flag).
- `computeLineControlLayout({series, ...})` -- returns
  `{series, xTicks, yTicks, xMin, xMax, yMin, yMax, innerWidth,
  innerHeight, totalPoints, visibleSeriesCount}`. Y range is
  expanded to cover both the data envelope and the UCL / LCL
  limits so the rules are always visible. Per-series carries
  `{id, label, color, stats, points, path, centerPy, uclPy,
  lclPy, outOfControlCount, aboveCount, belowCount,
  finiteCount, totalCount}`. Per-point carries `{index, x, y,
  px, py, state, outOfControl, deviation}` where `deviation` is
  the z-score `(y - mean) / sigma` (0 when sigma=0).
- `describeLineControlChart(series, hidden?, kSigma?,
  formatValue?)` -- builds the screen-reader description with
  per-series CL / sigma / UCL / LCL / out count summary.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- CL + UCL + LCL lines are `role="graphics-symbol"` with
  descriptive aria-label (including the sigma multiple).
- Process path + every dot is `role="graphics-symbol"` +
  `tabIndex={0}`. Dot aria-label includes the state (in / above
  / below).
- Every node carries `data-section="chart-line-control-..."` for
  test selectors.
- Series groups expose `data-series-id`, `data-series-color`,
  `data-series-mean`, `data-series-sigma`, `data-series-ucl`,
  `data-series-lcl`, `data-series-k-sigma`,
  `data-series-out-of-control-count`, `data-series-above-count`,
  `data-series-below-count`, `data-series-finite-count`.
- Limits expose `data-kind` (`ucl` | `lcl`) + `data-value`; CL
  expose `data-kind="cl"` + `data-value`.
- Dots expose `data-state`, `data-out-of-control`,
  `data-deviation`.
- Root mirrors `data-series-count`,
  `data-visible-series-count`, `data-total-points`,
  `data-k-sigma`, `data-out-of-control-count`, `data-animate`.
- Badge mirrors the same out-of-control count + alert / ok
  state.

## Tooltip

Hovering a dot shows the series label in its colour, x, bold y,
a `CL: <mean>` row, and a coloured `out of control (<state>)` /
`in control` row with the point's z-score in sigma units.

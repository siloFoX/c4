# 11.1042 -- UI: chart-line-kama-divergence-cross primitive (v1.11.1060)

## Scope

Add a pure-SVG dual-panel `ChartLineKamaDivergenceCross`
component that renders the close in the top panel and the
Kaufman Adaptive Moving Average (KAMA) in the bottom panel,
flagging price-vs-KAMA direction-disagreement events as
volatility-adaptive trend reversal warnings with bias
coloring derived from the KAMA slope at the trigger bar.
Uses the canonical 5-state divergence regime model
(aligned-bullish / aligned-bearish / divergent-bullish /
divergent-bearish / none).

KAMA is Perry Kaufman's 1995 adaptive-smoothing moving
average -- its smoothing constant adapts each bar based on
the efficiency ratio (signal / noise over the lookback
window). Trending markets get fast smoothing (close to
fast^2 = (2/3)^2 = 4/9); ranging markets get slow smoothing
(close to slow^2 = (2/31)^2 ~= 0.004). This gives KAMA the
unique property among MA-divergence-cross siblings: it
auto-tunes to market volatility, so divergent events fire
only when sub-linear price patterns are strong enough to
overcome the adaptive lag.

Sixth and final member of the MA divergence-cross family
(SMA, EMA, HMA, WMA, TEMA, KAMA).

## KAMA divergence pipeline

```
change[i]   = |close[i] - close[i - period]|
volat[i]    = sum(|close[j] - close[j-1]|, j = i - period
                  + 1 .. i)
er[i]       = change[i] / volat[i]  (0 when volat = 0)
fastSc      = 2 / (fastLength + 1)
slowSc      = 2 / (slowLength + 1)
sc[i]       = (er[i] * (fastSc - slowSc) + slowSc) ^ 2
kama[period - 1] = SMA(close[0..period-1], period)
kama[i]     = kama[i-1] + sc[i] * (close[i] - kama[i-1])

priceUp     : close[i] > close[i-1]
priceDown   : close[i] < close[i-1]
kamaUp      : kama[i]  > kama[i-1]
kamaDown    : kama[i]  < kama[i-1]

regime ->
  'aligned-bullish'   when priceUp   && kamaUp
  'aligned-bearish'   when priceDown && kamaDown
  'divergent-bullish' when priceDown && kamaUp
  'divergent-bearish' when priceUp   && kamaDown
  'none'              when either side flat or null

bullish (divergence trigger up) :
  prev regime !== 'divergent-bullish' &&
  cur regime === 'divergent-bullish'
bearish (divergence trigger down) :
  prev regime !== 'divergent-bearish' &&
  cur regime === 'divergent-bearish'

bias        : kama[i] vs kama[i-1] -> up / down / flat /
              none
```

Default `period = 10`, `fastLength = 2`, `slowLength = 30`
-- Perry Kaufman's canonical 1995 tuning. The fast/slow
endpoints give `fastSc^2 = 4/9` for ER = 1 (pure trend) and
`slowSc^2 ~= 0.0042` for ER = 0 (pure noise).

Warmup is `period - 1 = 9` for the default tuning: KAMA is
seeded at `i = period - 1` with `SMA(close, period)`, then
the adaptive recurrence runs from `i = period`. Divergence
detection requires the previous KAMA + close, so the first
regime classification lands at `i = period = 10`.

The KAMA recurrence on LINEAR input converges *
asymptotically* to its steady-state value -- the SMA seed
puts the initial value (period-1)/2 below the steady-state
on LINEAR UP, and convergence to within 1e-9 takes roughly
50 bars at the default tuning. The divergence detector
classifies regime correctly from `i >= period` regardless
of convergence state, because KAMA is monotone in the
input direction even before convergence.

## Bit-exact anchors

- **CONST close = K**: SMA seed = K. With flat close,
  `change` and `volat` are both 0 -> ER = 0 (zero-volat
  guard). sc = slowSc^2. KAMA recurrence: `K + slowSc^2 *
  (K - K) = K` for every i. Both close and KAMA are flat
  -> regime `none` for every bar. 0 divergence crosses.
  Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP close = i**: change = |close[i] - close[i -
  period]| = period. volat = sum of per-bar |close[j] -
  close[j-1]| = period. ER = period / period = 1. sc =
  fastSc^2 = `(2 / 3)^2 = 4 / 9`. Steady-state lag is
  `1/sc - 1 = 9/4 - 1 = 5/4 = 1.25`, so KAMA converges
  asymptotically to `i - 1.25`. By the late tail
  (`i >= 70` for a 100-bar series), convergence error is
  below 1e-9 -- verified bit-exact via `toBeCloseTo(_, 9)`.
  KAMA monotonically rises from `i = period` onwards;
  close[i] - close[i-1] = +1 (up); KAMA[i] - KAMA[i-1] >
  0 (up). Both up -> regime `aligned-bullish`. 0
  divergence crosses.
- **LINEAR DOWN close = -i**: mirror -> KAMA monotonically
  falls; asymptote KAMA = `-i + 1.25`. Both close and
  KAMA falling -> regime `aligned-bearish`. 0 divergence
  crosses.

The `i = period - 1` seed bar (KAMA exists but prev KAMA
is null) classifies as `regime = 'none'` because the
regime classifier requires both sides to be non-null --
this is the only "none" sample in the LINEAR anchors.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 5-state regime classifier follows the
  chart-line-*-divergence-cross family convention; the
  `bullish` / `bearish` cross *kinds* still follow the
  sibling cross-sig convention (line direction).
- KAMA recurrence is asymptotic, not bit-exact at the seed
  -- the LINEAR anchors verify the asymptote at the
  series tail (i >= 70 in a 100-bar series gives
  convergence error below 1e-9). The regime
  classification is correct from `i >= period` onwards
  regardless of convergence state, because KAMA is
  monotone in the input direction even during
  convergence.
- Zero-volatility guard (`er = 0` when `volat = 0`)
  prevents NaN from the `0 / 0` division on flat-close
  windows. The matching `sc = slowSc^2` from `er = 0`
  gives the slowest-possible smoothing (4/961 ~= 0.4%),
  matching the textbook Kaufman convention.
- Cross detection fires only on transition INTO a
  divergent state (not while staying in one), preventing
  double-fire on consecutive bars in the same regime.
- Among the MA divergence-cross siblings, KAMA is the
  only adaptive variant -- the others (SMA, EMA, HMA,
  WMA, TEMA) use fixed weighting schemes. KAMA's
  efficiency-ratio feedback means it can react faster
  than EMA in trends and slower than EMA in ranges,
  giving the divergence detector context-aware
  sensitivity.

# Patch 11.840 -- ChartLineSmaCross primitive (v1.11.858)

## Summary

Add `<ChartLineSmaCross />`, a pure-SVG single-panel chart that
overlays a fast SMA and a slow SMA on a price line, with scatter
markers placed at every detected fast-over-slow crossover event
(the classic golden / death cross):

```
fastSma[i] = mean(close[i - fastLength + 1 .. i])
slowSma[i] = mean(close[i - slowLength + 1 .. i])
up cross   = prev fastSma <= prev slowSma  AND  curr fastSma > curr slowSma
down cross = prev fastSma >= prev slowSma  AND  curr fastSma < curr slowSma
```

- The SMA helper carries the `min === max` window short-circuit so
  constant-window slices return their exact value without 1-ULP
  drift.
- `null` propagates during SMA warmup.

## Defaults

| Knob              | Value     |
|-------------------|-----------|
| fastLength        | 10        |
| slowLength        | 50        |
| priceColor        | `#2563eb` |
| fastColor         | `#f59e0b` (amber) |
| slowColor         | `#1d4ed8` (deep blue) |
| bullishColor      | `#16a34a` (green) for up-cross markers |
| bearishColor      | `#dc2626` (red) for down-cross markers |
| markerRadius      | 5         |

Defaults match the canonical 10/50 SMA pair used by retail
chartists for golden/death-cross signals.

## Bit-exact anchors

- `CONST close = K`: `fastSma = slowSma = K` for every post-warmup
  bar; the relation never strictly flips so `upCrossCount =
  downCrossCount = 0`. Verified across `(K, fast, slow)` sweeps.
- `LINEAR UP close = i + 1`: SMA of arithmetic progression gives
  `fastSma = (i + 1) - (fastLength - 1) / 2` and
  `slowSma = (i + 1) - (slowLength - 1) / 2`. With
  `slowLength > fastLength` the fast line is always above the slow
  line, so `upCrossCount = downCrossCount = 0` for every
  `(fastLength, slowLength)` pair tested
  (`(2, 4)`, `(5, 10)`, `(10, 50)`).
- `STEP close = K1 for i < N, K2 for i >= N` (`K1 < K2`): the fast
  line lifts off `K1` sooner than the slow line, producing exactly
  one up-cross at a deterministic index. For `K1 = 1, K2 = 5,
  fastLength = 2, slowLength = 4, N = 4`, the up-cross fires at
  `i = 4`. The symmetric `STEP K1 = 5, K2 = 1` produces exactly one
  down-cross.

## Surface

Single-panel primitive surface:

- `ChartLineSmaCrossPoint = { x, close }`
- `runLineSmaCross(data, options?) -> Run` with
  `ok = series.length >= max(fastLength, slowLength)`.
- `computeLineSmaCrossLayout({ data, width?, height?, padding?, fastLength?, slowLength? }) -> Layout`.
  Single panel; y-axis auto-fits across `close / fast / slow`.
- `describeLineSmaCrossChart(data, opts?) -> string` for the ARIA
  description.
- `<ChartLineSmaCross />` `forwardRef<HTMLDivElement>` with three
  legend toggles (`price`, `fast`, `slow`), config badge with live
  cross counts, scatter markers at each crossover event, and a
  tooltip surfacing `close / fast / slow / relation / cross`. Live
  cross counts also exposed as `data-up-cross-count` /
  `data-down-cross-count` attributes for selector-driven assertions.

## Summary

Add `web/src/components/ui/chart-line-stoch-rsi-mid-cross-sig.tsx`
-- a pure-SVG dual-panel React/TypeScript chart primitive that
renders the close in the top panel and a close-only smoothed
Stochastic RSI signal line (SMA of stochastic-of-RSI) in the
bottom panel, marking discrete midline (default `50`) crossover
events. This is the midline-signal-line variant of the Stochastic
RSI family: it tracks when the signal line (SMA of stochRSI)
crosses the midline 50, flagging the canonical bull / bear regime
transitions for the double-smoothed momentum gauge.

The double smoothing (Wilder RSI followed by stochastic
normalization and a final SMA) suppresses the raw-stochRSI
whipsaws that plague the unfiltered variant while preserving
the responsiveness of the underlying momentum signal.

## Math anchor (bit-exact)

```
rsi[i]      = WilderRSI(close, length)
ll[i]       = min(rsi[i-n+1..i])
hh[i]       = max(rsi[i-n+1..i])
stochRsi[i] = 100 * (rsi - ll) / (hh - ll); 50 when hh==ll
sigK[i]     = SMA(stochRsi, kSmoothing)
bullish     : prev sigK <= 50 && cur sigK >  50
bearish     : prev sigK >= 50 && cur sigK <  50
```

- **CONST close = K**: deltas = 0 -> avgGain == avgLoss == 0 ->
  RSI degenerate = 50. Rolling window of constant 50 RSI -> hh ==
  ll, stochRsi = 50 (degenerate midline). SMA(50, 3) = 50 via
  the SMA `min === max` short-circuit. 50 == 50 -> regime
  `bullish` (sigK >= 50). 0 crosses. Verified across K = 0..1234.
- **LINEAR UP close = i**: deltas = +1 -> avgLoss = 0 -> RSI =
  100. Rolling window of constant 100 RSI -> hh == ll, stochRsi
  = 50. SMA(50, 3) = 50. regime `bullish`. 0 crosses.
- **LINEAR DOWN close = -i**: deltas = -1 -> avgGain = 0 -> RSI =
  0. Rolling window of constant 0 RSI -> hh == ll, stochRsi = 50.
  SMA(50, 3) = 50. regime `bullish`. 0 crosses.

Strict inequality on `cur` (`> T` for bullish, `< T` for bearish)
means boundary equality (sigK == 50) never double-fires a cross.

- Wrapping `<div role="region" aria-label="Stochastic RSI Midline
  Signal Cross chart" aria-describedby="...">` with the sr-only
  description copy carrying the length / kSmoothing / threshold
  parameters plus a "smoothed momentum centerline trigger" framing.

  tabIndex={0}>` for keyboard focus.

  readers can step through events.

  Enter, or Space.

# 11.1098 -- UX FIX: status banner contrast (v1.11.1116)

## Scope

The AutonomousStatusBanner reviewer-attention text and pending count
rendered as near-illegible washed text on the light-pink background
across all views.

## Root cause

The banner uses the shared `NotificationBanner` `critical` variant,
which is `bg-destructive/10 ... text-destructive-foreground`. On the
light theme `--destructive-foreground` is `0 0% 98%` (near-white) and
`bg-destructive/10` composites to a very light pink (~96% L), so
near-white text on near-white pink is ~1:1 -- far below WCAG AA. (The
`text-*-foreground` tokens are meant for SOLID destructive/warning
backgrounds, not the 10% tint.)

## Change (src/components/AutonomousStatusBanner.tsx)

Scoped to the banner (no change to the shared primitive). Each of the
three banners it renders now passes:

- `className="text-foreground"` -- overrides the washed
  `text-*-foreground` with the theme's primary text token, which is
  near-black on light and near-white on dark, so the text meets WCAG AA
  on the tint in BOTH themes (measured ~15:1). `text-destructive` was
  considered but is only ~3:1 on the light tint (fails AA).
- a colored `icon` (`AlertOctagon text-destructive` for critical,
  `AlertTriangle text-warning` for warn) so the alert/warning cue
  survives even though the text is now neutral-dark; the red/amber
  border + tint remain.
- the count is bolded (escalation `pendingEscalations`, halted
  `consecutiveHalts`) via a `<span className="font-bold">` in a
  ReactNode description.
- `data-section="autonomous-status-banner"` for the e2e.

## Verification (mandatory)

`web/e2e/banner-contrast.spec.ts` (new) stubs pending escalations so the
critical banner renders, then computes the WCAG contrast ratio of the
count text vs the banner's composited background (the semi-transparent
`destructive/10` over the first opaque ancestor) and asserts it is
>= 4.5 (AA), and that the count's font-weight is bold (>= 600), at
1440 / 768 / 375. Result: `3 passed`. The 1440 screenshot confirms a
red icon + dark readable "Reviewer attention required" + bold "3"
pending escalations on the light-pink background.

## Risk

- Scoped to AutonomousStatusBanner; the shared NotificationBanner
  primitive is unchanged (other banners keep their existing styling).
  The text is now theme-aware foreground, guaranteed AA in light + dark.

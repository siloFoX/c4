# Patch 11.954 - UI: chart-line-bb-percent-zero-cross primitive

Version: 1.11.972

## Summary

Add `web/src/components/ui/chart-line-bb-percent-zero-cross.tsx` -- a
pure-SVG dual panel line chart primitive that visualizes price alongside
John Bollinger's %B oscillator, with explicit cross markers and overlay
arrows at every %B 0.5-midline crossover. Bollinger %B normalises the
close to its position within fixed-multiple Bollinger Bands centered on
the rolling SMA: 0.5 = close on the middle band, 1 = close on the upper,
0 = close on the lower, > 1 / < 0 = close has exited the bands. Midline
(0.5) crossovers mark position-within-bands centerline regime transition
events -- the moment the close moves from below to above (or vice versa)
the rolling mean.

## Pipeline

1. Rolling SMA, population stdev, and bands:
   - `middle_i = SMA(close, length)` (CONST short-circuit when window is
     constant).
   - `stdev_i  = sqrt(sum((c - mean)^2) / length)`.
   - `upper_i  = middle + mult * stdev`.
   - `lower_i  = middle - mult * stdev`.
2. Percent B:
   - `range = upper - lower = 2 * mult * stdev`.
   - `range == 0` -> `pctB = 0.5` (CONST neutral fallback).
   - else -> `pctB = (close - lower) / range`.
3. `classifyLineBbPercentZeroCrossRegime(pctB, T)`: null -> none,
   pctB >= T -> bullish, pctB < T -> bearish.
4. `detectLineBbPercentZeroCrossCrosses(series, pctB, T)`: strict
   `prev <= T && cur > T` for bullish, strict `prev >= T && cur < T`
   for bearish. Boundary equality does not fire.

## Bit-exact anchors

| Pattern                  | %B (after warmup)         | Regime  |
|--------------------------|---------------------------|---------|
| CONST close = K          | 0.5 (range = 0 fallback)  | bullish |
| LINEAR UP close = i      | `0.5 + ((N-1)/2) / (2*mult*stdev)` | bullish |
| LINEAR DOWN close = -i   | `0.5 - ((N-1)/2) / (2*mult*stdev)` | bearish |

CONST anchor tested bit-exactly across K in `{0, 1, 42, 100, 1234}`.
LINEAR anchors tested with the analytical steady-state offset
`((N-1)/2) / (2 * mult * sqrt((N^2-1)/12))` -- for length=5 / mult=2
this is `sqrt(2) / 4 ~ 0.3536`, so LINEAR UP gives `pctB = 0.8536`
constant and LINEAR DOWN gives `pctB = 0.1464`.

## Layout

* Default dimensions: 720 x 460.
* Two panels: price panel on top (~55% of plot area), %B panel below.
* %B panel auto-fits to the observed `[min, max]` range with 5% padding
  on each end, always including both the `[0, 1]` band envelope and the
  threshold in the displayed range (so the standard band-band lines
  remain visible even when %B spills outside).
* Threshold band rendered as a horizontal rule at `y = thresholdY` in
  the %B panel.
* Cross markers drawn as circles on the %B panel at `(cx, cyOsc)` and
  as arrows on the price panel at `(cx, cyPrice)`.
* SVG path strings use `.toFixed(2)` for deterministic snapshot tests.

## Accessibility

* Wrapping `<div role="region" aria-label="BB Percent B Zero Cross
  chart">`.
* `<svg role="img">` with an aria-described-by linked `<desc>` sr-only
  text.
* Each cross marker: `role="graphics-symbol"` + `tabIndex={0}` +
  descriptive `aria-label` ("bullish cross at x=...").
* Legend toggles: `<button role="switch" aria-pressed="...">`.
* Motion-safe fade-in via `motion-safe:animate-fade-in` (opt-out with
  `animate={false}`).

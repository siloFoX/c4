# 11.532 -- web/src/components/ui/chart-line-streak.tsx

Pure-SVG line chart that highlights **consecutive same-direction
runs** ("streaks") in a series. The base line is rendered as a
dimmed gray reference; each detected streak overlays a thicker
coloured stroke segment on top of it -- green for up runs, red for
down runs, slate for flat runs. A run badge in the top-left calls
out the longest streak across the chart with direction icon
(`↑` / `↓` / `→`), length, and label.

## Distinct from other direction / outlier primitives

- `<ChartLineVelocity>` (11.527): per-point directional **arrow
  markers** based on tangent / sign. Streak is run-based -- a
  contiguous SEGMENT, not isolated arrows.
- `<ChartLineAcceleration>` (11.528): second derivative magnitudes
  surfaced per-point. Streak counts runs of same-sign first
  difference.
- `<ChartLineAnomaly>` (11.524): z-score outlier highlighting --
  rare DEVIATING points. Streak is the opposite -- consecutive
  AGREEING points.
- `<ChartLineMinMax>` (11.517): single global min/max marker.
  Streak surfaces every multi-step run.
- `<ChartLineTrend>` (11.512): one global linear regression rule.
  Streak is per-run, not global.
- `<ChartLinePeriodCompare>` (11.531): two periods, one metric.
  Streak is a single series with directional emphasis.

The canonical "5 consecutive winning days" / "3 quarters in a row
of decline" / "win streak / loss streak" visualisation used in
sports, trading, and operational monitoring.

## Public surface

- `<ChartLineStreak series={...} />`.
- `series: ChartLineStreakSeries[]` -- `{id, label, data, color?}`.
- `minLength` (default `2`) -- minimum run length to keep.
- `flatEpsilon` (default `0`) -- equality band; steps within +-eps
  classified as flat.
- `upColor` / `downColor` / `flatColor` -- per-direction overlay
  + dot colors.
- `baseColor` -- dimmed base line color.
- `highlightWidth` / `baseOpacity` / `highlightOpacity` -- visual
  emphasis controls.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showRunBadge`, `animate`.
- Controlled + uncontrolled visibility via `hiddenSeries` /
  `defaultHiddenSeries` / `onHiddenSeriesChange`.
- Standard sizing + format + a11y props.
- `onPointClick({series, point})`.

## Pure helpers

- `getLineStreakDefaultColor(index)` -- 10-color palette cycle;
  NaN / negative -> color 0.
- `getLineStreakFinitePoints(points)` -- finite-only filter.
- `normaliseLineStreakMinLength(value)` -- clamps to >= 2; floors
  fractional; non-finite -> default 2.
- `normaliseLineStreakFlatEpsilon(value)` -- non-finite / negative
  -> 0.
- `classifyLineStreakStep(delta, epsilon?)` -- single-step
  classifier; non-finite -> `'flat'`.
- `computeLineStreaks(points, {minLength?, flatEpsilon?})` --
  scans **sorted-by-x** finite points and returns the array of
  runs `{startIndex, endIndex, length, direction, startX, endX,
  startY, endY, delta}`. Boundary point belongs to both adjacent
  runs (the peak/trough is rendered in both overlays). Runs
  shorter than `minLength` are dropped. Sorts ascending before
  scanning.
- `computeLineStreakLongest(runs)` -- `{length, direction}` of
  the longest run; `{0, 'flat'}` when empty.
- `computeLineStreakLayout({series, ...})` -- returns
  `{series, xTicks, yTicks, xMin, xMax, yMin, yMax, innerWidth,
  innerHeight, totalPoints, visibleSeriesCount}`. Per-series
  carries `{id, label, color, points, path, runs, finiteCount,
  totalCount, longestRunLength, longestRunDirection, upRunCount,
  downRunCount, flatRunCount}`. Per-point carries `{index, x, y,
  px, py, direction, inStreak, streakIndex}`. Per-run carries
  `{startIndex, endIndex, length, direction, color, path,
  startX, endX, startY, endY, delta}`.
- `describeLineStreakChart(series, hidden?, minLength?,
  flatEpsilon?)` -- builds the screen-reader description with
  per-series longest streak summary.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Base path + every run overlay + every dot is
  `role="graphics-symbol"` + `tabIndex={0}`. Run aria-label
  includes the direction + length. Dot aria-label includes "in
  <direction> streak" when applicable.
- Every node carries `data-section="chart-line-streak-..."` for
  test selectors.
- Series groups expose `data-series-id`, `data-series-color`,
  `data-series-finite-count`, `data-series-run-count`,
  `data-series-longest-length`, `data-series-longest-direction`,
  `data-series-up-run-count`, `data-series-down-run-count`,
  `data-series-flat-run-count`.
- Base path has `data-kind="base"`.
- Run paths expose `data-direction`, `data-length`,
  `data-start-index`, `data-end-index`, `data-delta`.
- Dots expose `data-direction`, `data-in-streak`,
  `data-streak-index`.
- Root mirrors `data-series-count`,
  `data-visible-series-count`, `data-total-points`,
  `data-min-length`, `data-longest-length`,
  `data-longest-direction`, `data-animate`.
- Badge mirrors the same direction + length + series-id.

## Tooltip

Hovering a dot shows the series label in its colour, x, bold y,
and (when the point is in a streak) a coloured `<direction>
streak` row.

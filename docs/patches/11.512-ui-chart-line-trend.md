# 11.512 -- web/src/components/ui/chart-line-trend.tsx

Pure-SVG multi-series line chart with a **linear regression trend
line overlay** drawn on top of each series. The trend rule is the
ordinary least-squares best fit `y = m*x + b` of the series's finite
samples, drawn from the smallest finite x to the largest finite x and
clipped to the active x range. The legend optionally shows
`(slope / R^2)` next to each series so adopters can see the trend at
a glance.

## Distinct from other line primitives

- `<ChartLineMulti>` (11.500), `<ChartLineSmooth>` (11.498),
  `<ChartLineDashed>` (11.499), `<ChartLineStep>` (11.497), and
  `<ChartLineMarker>` (11.506) all render raw lines without computing
  a regression.
- `<ChartLineBaseline>` (11.510) and `<ChartLineThreshold>` (11.503)
  draw constant horizontal references; the trend line is **sloped**
  per series.
- `<ChartLineComparison>` (11.511) compares two curves; trend is one
  computed regression per series.
- `<ChartLineConfidence>` (11.502) is a per-point upper/lower band, not
  a regression.

## Public surface

- `<ChartLineTrend series={...} />` -- one or many series, each
  `{id, label, data, color?, trendColor?, trendDashArray?, hideTrend?}`.
- `trendColor` defaults to `color` (and then the 10-color palette).
- `trendDashArray` defaults to `'6 4'`; the chart-level `trendDashArray`
  prop is the fallback when a series does not specify its own.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showTrend`, `showTrendStats`, `animate`.
- Numbers: `width`, `height`, `padding`, `tickCount`, `strokeWidth`,
  `trendStrokeWidth`, `dotRadius`, `lineOpacity`, `trendOpacity`.
- Bounds: `xMin/xMax/yMin/yMax` (swapped if inverted; zero-range
  expanded +/- 0.5).
- `formatValue` / `formatX` for axis + tooltip; `formatSlope` /
  `formatR2` for legend stats.
- `xLabel` / `yLabel` axis-label slots.
- Controlled + uncontrolled visibility via `hiddenSeries` /
  `defaultHiddenSeries` / `onHiddenSeriesChange`.
- Callbacks: `onPointClick({series, point})`, `onSeriesToggle({series,
  hidden})`.

## Pure helpers

- `getLineTrendDefaultColor(index)` -- 10-color palette cycle.
- `getLineTrendFinitePoints(points)` -- finite-only filter.
- `getLineTrendXRange(points)` -- `{min, max}` x range or `null` when
  empty / all non-finite.
- `computeLineTrendRegression(points)` -- returns `{slope, intercept,
  r2, sampleCount, ok, meanX, meanY}`. Uses the canonical least-squares
  formula. `ok=false` when n<2 or `sxx===0` (all x equal). When all y
  values are equal, `slope=0`, `intercept=meanY`, `r2=1`. r2 clamps to
  [0,1].
- `predictLineTrendY(x, regression)` -- evaluates `slope*x + intercept`
  with a graceful intercept fallback on non-finite x.
- `computeLineTrendLayout({series, hiddenSeries?, width, height,
  padding, ...})` -- returns `{series, xTicks, yTicks, xMin, xMax,
  yMin, yMax, innerWidth, innerHeight, totalPoints,
  visibleSeriesCount}`. Each layout series carries
  `{id, label, index (stable original index), color, trendColor,
  trendDashArray, hideTrend, points, path, trendPath, trendStartX,
  trendStartY, trendEndX, trendEndY, regression, finiteCount,
  totalCount}`. `trendPath=''` when regression is not ok.
- `describeLineTrendChart(series, hidden?, formatValue?)` --
  builds the screen-reader description.

## Regression formula

OLS slope/intercept via mean-centered moments:

```
meanX = Σx / n,  meanY = Σy / n
sxx   = Σ (x - meanX)^2
sxy   = Σ (x - meanX) * (y - meanY)
slope     = sxy / sxx       (ok=false if sxx == 0)
intercept = meanY - slope * meanX
```

`R^2 = 1 - SS_res / SS_tot` where
`SS_tot = Σ(y - meanY)^2` and `SS_res = Σ(y - (slope*x + intercept))^2`.
When `SS_tot = 0`, `R^2 = 1` (flat data matches the flat fit) or `0`
(no fit exists). Final r2 is clamped to `[0,1]` to defend against
floating-point overshoot.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Per-series line, trend, and per-dot are `role="graphics-symbol"`
  with `tabIndex={0}` and per-shape aria-label including the slope /
  R^2 for the trend line.
- Every node carries `data-section="chart-line-trend-..."` for test
  selectors.
- Series groups expose `data-series-id`, `data-series-color`,
  `data-series-slope`, `data-series-intercept`, `data-series-r2`,
  `data-series-regression-ok`, `data-series-point-count`,
  `data-series-finite-count`.

## Tooltip

Hovering a dot shows the series label, `x:`, `y:`, and a `trend:`
line with the regression-predicted y and signed residual.

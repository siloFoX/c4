## Summary

Add a new `<ChartTreeRadial>` UI primitive in
`web/src/components/ui/chart-tree-radial.tsx` --
pure-SVG **radial tree** layout, the polar variant of
`<ChartDendrogram>` (11.486). The root sits at the
centre, leaves radiate out at the perimeter, and each
level of the hierarchy lives at a fixed radius. Three
link styles connect parents to children: `curve`
(default; smooth cubic bezier in polar space), `elbow`
(arc along the parent's radius then radial line to
the child), or `line` (straight line from parent to
child). Leaf labels rotate tangent to the perimeter
so they remain readable around the full circle;
labels on the bottom half are flipped to keep their
text upright.

## Behaviour

- Renders `<div role="region">` wrapping
  `<svg role="img">` with the same `aria-label`.
- Layout (`computeTreeRadialLayout`):
  - Walk the tree depth-first to assign each leaf a
    unique `leafIndex` (0..N-1).
  - Each leaf is placed at angle `startAngle +
    leafIndex * (2pi / leafCount)` and radius
    `outerRadius`.
  - Each internal node sits at the *mean* of its
    children's angles, at radius `depth * depthStep`
    where `depthStep = outerRadius / maxDepth`.
  - The root therefore lives at radius 0 (the centre).
  - `outerRadius` is derived from
    `min(innerW, innerH) / 2 - leafLabelReserve`.
- Link paths (`buildTreeRadialLinkPath`):
  - `curve` (default): cubic bezier with both
    control points midway between the source and
    target angles, projected at the source / target
    radii respectively (the canonical D3 radial
    cluster style).
  - `elbow`: arc along the source's radius from
    source angle to target angle, then a straight
    radial line out to the target. Falls back to
    `line` when the source radius is 0.
  - `line`: straight `M..L` between the cartesian
    points.
- Per-node `color` always wins; otherwise leaves use
  `leafColor` and internals use `internalColor`.
- Leaf labels (`showLeafLabels`, default true):
  - `rotateLeafLabels=true` (default): label is
    rotated to the leaf's tangent angle. Labels on
    the bottom half of the circle (angle in
    `(pi/2, 3pi/2)`) are flipped 180 deg so their
    text remains upright. Anchor flips to `end`
    accordingly.
  - `rotateLeafLabels=false`: label is placed
    absolutely outside the leaf in the radial
    direction with text-anchor based on the sign of
    `cos(angle)`.
- Internal labels (`showInternalLabels`, default
  false) render above the internal node centroid.
- Hover/focus on a node opens a tooltip with the
  label, depth, and either `leaf` (for leaves) or
  `<leafCount> leaves` (for internals).
- Hover/focus on a link opens a link tooltip showing
  `<source> -> <target>` when no node is also
  hovered.
- Mount animation:
  `motion-safe:animate-fade-in` per node + link.
  `animate=false` removes the class.
- Default ARIA description: `"Radial tree with N
  nodes, L leaves, depth D."` Null root -> `"No
  data"`.

## ARIA + data attributes

- Root: `<div role="region">` + `aria-label`.
- SVG: `role="img"` + same `aria-label`.
- sr-only span:
  `data-section="chart-tree-radial-aria-desc"`.
- Each node circle: `role="graphics-symbol"` +
  `tabIndex={0}`. aria-label = `"<label>"` (leaves)
  / `"<label> (<leafCount> leaves)"` (internals).
- Each link path: `role="graphics-symbol"` +
  `tabIndex={0}`. aria-label = `"Link <source> to
  <target>"`.
- `data-section` on every node:
  chart-tree-radial / -canvas / -aria-desc / -svg /
  -links / -link / -nodes / -node / -node-circle /
  -node-label / -tooltip / -tooltip-label /
  -tooltip-depth / -tooltip-leaves /
  -link-tooltip / -link-tooltip-label.
- Root mirrors `data-node-count`,
  `data-link-count`, `data-leaf-count`,
  `data-max-depth`, `data-link-style`,
  `data-animate`.
- Node groups mirror `data-node-id`,
  `data-node-depth`, `data-node-parent`,
  `data-node-is-leaf`, `data-node-leaf-index`,
  `data-node-leaf-count`, `data-node-child-count`,
  `data-node-color`, `data-node-angle`,
  `data-node-radius`, `data-hovered`.
- Link paths mirror `data-link-index`,
  `data-link-source`, `data-link-target`,
  `data-hovered`.

## Out of scope

- Edge weights / branch lengths (uniform `depth *
  step` only; host pre-aligns leaves if a true
  hclust-style branch length matters).
- Spiral or fish-eye distortions (canonical polar
  layout only).
- Click-to-collapse subtrees (v1 is read-only).
- Animated transitions on data / linkStyle change
  (snap v1; mount fade only).

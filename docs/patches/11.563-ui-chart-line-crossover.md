# 11.563 -- web/src/components/ui/chart-line-crossover.tsx

Pure-SVG line chart that draws two series -- a **primary**
and a **reference** -- and marks every point where they
intersect. A rising intersection (the primary crossing
**above** the reference) is a **golden cross**; a falling
intersection (the primary crossing **below**) is a **death
cross**. This is the classic two-moving-average crossover
chart from technical analysis.

## Crossover detection

1. **Align.** `alignLineCrossoverSamples(primary, reference)`
   keeps only the x values present in **both** series,
   sorted ascending. Each aligned sample carries
   `diff = primaryY - referenceY` -- the signal whose zero
   crossings are the crossover points.
2. **Detect.** `detectLineCrossovers(samples)` walks
   consecutive aligned samples. A crossover sits between
   sample `i-1` and `i` when their diffs have **strictly
   opposite signs** (`d0 * d1 < 0`).
3. **Interpolate.** The crossing is the zero of the linearly
   interpolated diff: `t = d0 / (d0 - d1)` (always in the
   open interval `(0, 1)` for a strict sign change),
   `x = x0 + t * (x1 - x0)`,
   `y = primaryY0 + t * (primaryY1 - primaryY0)`. Because
   both series are equal at the crossing, either series
   yields the same `y`.
4. **Classify.** `kind = 'golden'` when the diff rises
   through zero (`d1 > 0`), `'death'` when it falls
   (`d1 < 0`).

A sample sitting **exactly** on zero does not trigger a
crossover -- a strict sign change between two consecutive
aligned samples is required, which keeps the detector
deterministic and free of double-counting.

The component's own fixture demonstrates the whole pipeline:
primary `[10, 8, 6, 10, 14]`, reference `[6, 9, 9, 8, 9]` at
x `0..4` gives diffs `[4, -1, -3, 2, 5]`. Two crossovers
result -- a **death cross** at `x = 0.8` (`t = 4/(4-(-1)) =
0.8`) and a **golden cross** at `x = 2.6` (`t = -3/(-3-2) =
0.6`), both at `y = 8.4`.

## Distinct from neighbouring primitives

- `chart-line-multi` draws many series but detects nothing
  -- it has no concept of an intersection event.
- `chart-line-comparison` / `chart-line-correlation` relate
  two series in aggregate (deltas, correlation coefficient);
  ChartLineCrossover localises the **discrete events** where
  the ordering of the two series flips.
- `chart-line-moving-avg` overlays a single smoothed line.
  Crossover is the natural consumer of *two* such lines.

All exported helpers carry a `LineCrossover` infix so the
`export *` barrel line cannot collide with the existing
`chart-line-correlation` / `chart-line-comparison`.

## Public surface

- `<ChartLineCrossover primary={...} reference={...} />` --
  two explicit `ChartLineCrossoverSeries` props
  (`{id, label, data, color?}`), since a crossover chart is
  inherently a two-series comparison.
- `goldenColor` / `deathColor` -- marker colours, decoupled
  from the line colours.
- `primaryColor` / `referenceColor` -- line colour defaults.
- `markerSize` -- triangle marker half-extent.
- Toggle flags: `showAxis`, `showGrid`, `showDots` (default
  **false** -- the markers are the focus), `showLegend`,
  `showTooltip`, `showConfigBadge`, `showMarkers`,
  `showMarkerGuides`, `animate`.
- Controlled + uncontrolled visibility (`hiddenSeries` /
  `defaultHiddenSeries` / `onHiddenSeriesChange`). Hiding
  either series suppresses every crossover marker -- a
  crossover needs both lines.
- Standard sizing + format + a11y props. `formatValue` /
  `formatX`.
- `onCrossoverClick({crossover})` / `onPointClick({series,
  point})` / `onSeriesToggle({seriesId, hidden})`.

## Pure helpers

- `getLineCrossoverDefaultColor(index)`.
- `getLineCrossoverFinitePoints(points)`.
- `alignLineCrossoverSamples(primary, reference)` -- the
  shared-x alignment with `diff`.
- `detectLineCrossovers(samples)` -- the strict-sign-change
  detector with linear interpolation.
- `runLineCrossover(primary, reference)` -- aligns + detects;
  reports `goldenCount` / `deathCount` / `crossoverCount` /
  `alignedCount`.
- `computeLineCrossoverLayout({...})` -- projects both line
  paths and the crossover markers; markers carry
  `baselinePy` for the dashed guide down to the x-axis.
- `describeLineCrossoverChart(primary, reference, options?)`.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Each line path + every crossover marker + every dot
  `role="graphics-symbol"` + `tabIndex={0}`. The marker
  aria-label states the cross kind, x and value.
- Every node carries `data-section="chart-line-crossover-..."`.
- Root mirrors `data-golden-count`, `data-death-count`,
  `data-crossover-count`, `data-aligned-count`,
  `data-total-points`, `data-visible-series-count`,
  `data-animate`.
- Series groups expose `data-series-kind`
  (`primary` / `reference`), `data-series-id`,
  `data-series-finite-count`.
- Markers expose `data-kind` (`golden` / `death`),
  `data-crossover-index`, `data-segment-index`, `data-x`,
  `data-y`. Marker shape + guide also carry `data-kind`.

## Tooltip / badge

Hovering a marker shows the cross kind (coloured), x, y, and
the `fromDiff -> toDiff` transition. Hovering a dot (when
`showDots`) shows the series label, x, y. Config badge:
`XO gold=<n> death=<n> aligned=<n>`. Legend has two toggle
items plus a `<golden> golden / <death> death` tally.

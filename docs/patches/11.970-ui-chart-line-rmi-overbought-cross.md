## Summary

Adds the `ChartLineRmiOverboughtCross` primitive
(`web/src/components/ui/chart-line-rmi-overbought-cross.tsx`): a
pure SVG dual-panel chart that visualizes Relative Momentum Index
(RMI) crossings through the canonical overbought line 70. Mirror
of the existing `chart-line-rmi-zero-cross` (11.948) and
`chart-line-rmi-mid-cross` (11.967): identical RSI-with-lookback
Wilder pipeline, only the default threshold changes (70 instead of
50 baseline) and the framing flips from "centerline regime
transition" to "overbought trigger events".

## Pipeline

Same RMI pipeline shared with the rmi-zero-cross / rmi-mid-cross
siblings:

1. `mom[i] = close[i] - close[i - lookback]` (i >= lookback)
2. `gain[i] = max(0, mom)`, `loss[i] = max(0, -mom)`
3. `avgGain = Wilder(gain, length)`,
   `avgLoss = Wilder(loss, length)` -- SMA-seeded at
   `lookback + length - 1`, then
   `prev = (prev * (n - 1) + value) / n`.
4. RMI from the smoothed ratio with explicit fallbacks:
   - `avgLoss == 0 && avgGain == 0` -> RMI = 50 (neutral).
   - `avgLoss == 0 && avgGain > 0`  -> RMI = 100.
   - `avgGain == 0`                  -> RMI = 0.
   - else                            -> `100 - 100 / (1 + g / l)`.
5. Regime classifier (strict `>=`):
   - `RMI >= 70` -> bullish (in overbought zone)
   - `RMI <  70` -> bearish (below overbought)
6. Cross detector emits a `bullish` cross when
   `RMI[i-1] <= 70 && RMI[i] > 70` and a `bearish` cross when
   `RMI[i-1] >= 70 && RMI[i] < 70`. Strict inequality on the
   current side means the boundary value 70 does not double-fire.

## Defaults

- `length = 14`
- `lookback = 5` (Altman's canonical settings)
- `threshold = 70` (canonical overbought line)
- Warmup boundary: bar `length + lookback - 1 = 18`, so the first
  18 samples (40-bar run) are null and the remaining 22 are
  populated.

## Bit-exact anchors

- CONST `close = K` for all `K in {0, 1, 42, 100, 1234}` ->
  `rmi[i] === 50` literal for every `i >= 18`. Regime classifies
  as bearish because 50 < 70 -- the exact mirror of the mid-cross
  variant where 50 sat on the threshold and read bullish, and
  shares the same RMI value as the zero-cross variant.
- LINEAR UP `close = i` -> `rmi[i] === 100` constant from i = 18
  onward (no losses, avgLoss = 0, avgGain > 0). All bullish
  (100 >= 70), but 0 crosses because RMI jumps directly from null
  to 100 with no `prev <= threshold && cur > threshold` transition.
- LINEAR DOWN `close = -i` -> `rmi[i] === 0` constant (no gains,
  avgGain = 0). All bearish (0 < 70), 0 crosses.
- CONST K full-run counters under defaults, 40 bars:
  `noneCount = 18`, `bullishCount = 0`, `bearishCount = 22`,
  `crosses.length = 0`.
- `pricePath` and `rmiPath` are byte-identical across repeated
  layout calls, with `.toFixed(2)` precision and `M ... L ...`
  command shapes. The rmi path uses a single `M` for post-warmup
  data.
- `thresholdY` at 70 sits ABOVE the panel midpoint in SVG y-space
  (i.e., a smaller y value).
- Cross detector boundary: strict inequality (`> T`, `< T`).

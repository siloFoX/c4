No existing navigation-menu primitive in
the codebase. This patch ships one from
scratch.

## How NavigationMenu differs from Menubar

| concern | `<Menubar>` (11.387) | `<NavigationMenu>` (this patch) |
| --- | --- | --- |
| role | `role="menubar"` | `<nav>` element |
| trigger element | `<button role="menuitem">` always | `<button>` for sub-menu items, `<a>` for direct links |
| dropdown content | flat list of items + separators | sections (single-column list) or sections (multi-column mega) or custom content slot |
| target surface | application menu (File / Edit / View) | site nav (Products / Solutions / Pricing) |
| top-level navigation links | rare (items fire `onSelect`) | first-class (`href` on top-level items) |

## API

```ts
interface NavigationMenuLink {
  id: string;
  label: ReactNode;
  href: string;
  description?: ReactNode;
  icon?: ReactNode;
  external?: boolean;
}

interface NavigationMenuSection {
  id: string;
  heading?: ReactNode;
  links: NavigationMenuLink[];
}

interface NavigationMenuItem {
  id: string;
  label: ReactNode;
  href?: string;                          // direct top-level link
  sections?: NavigationMenuSection[];     // dropdown / mega-menu
  content?: ReactNode;                    // custom panel slot
  panelMinWidth?: number | string;        // default 12rem
  disabled?: boolean;
}
```

```tsx
<NavigationMenu
  ariaLabel="Main"
  items={[
    {
      id: 'products',
      label: 'Products',
      sections: [
        { id: 'core', heading: 'Core', links: [...] },
        { id: 'add', heading: 'Add-ons', links: [...] },
      ],
    },
    {
      id: 'community',
      label: 'Community',
      content: <CustomPanel />,
    },
    { id: 'pricing', label: 'Pricing', href: '/pricing' },
  ]}
/>
```

## Three sub-menu shapes

The item resolves to one of three render
paths:

1. **`href` only** (no sections / content)
   -> anchor trigger that follows the
   href on click.
2. **`sections`** -> dropdown panel:
   - 1 section: single-column list
     (`data-section="nav-menu-panel-list"`).
   - 2+ sections: multi-column mega panel
     (`data-section="nav-menu-mega"` +
     `data-is-mega="true"`).
3. **`content`** -> custom ReactNode slot
   (`data-section="nav-menu-panel-content"`).

`external: true` on a link sets
`target="_blank" rel="noopener noreferrer"`
and renders an ExternalLink glyph next to
the label.

## Viewport-aware positioning

```ts
export function computeNavPanelOffset(
  triggerLeft: number,
  panelWidth: number,
  viewportWidth: number,
): number;
```

Pure helper exported alongside the
component. Returns the horizontal shift
needed to keep the panel inside the
viewport. When the panel would overflow
the right edge, the shift is the negative
overflow amount, capped so the panel's
leading edge never moves past the viewport
left edge.

The component applies this offset via
inline `style.left` on the open panel and
re-computes on `window.resize`.

## Keyboard contract

| key | location | effect |
| --- | --- | --- |
| ArrowRight / ArrowLeft | on trigger | move focus between triggers (wrap, skip disabled) |
| ArrowDown / Enter / Space | on sub-menu trigger | open dropdown |
| Home / End | on trigger | jump to first / last enabled |
| Escape | anywhere | close dropdown + restore focus to trigger |

Direct-link triggers (no submenu) skip the
ArrowDown / Enter / Space handler -- the
browser default handles Enter on the
anchor.

## Behaviour

- **Single open panel at a time.**
- **Roving tabindex.** Only the focused
  (or first enabled) trigger has
  `tabindex=0`.
- **Hover-swap.** Once any panel is open,
  hovering a sibling trigger swaps panels.
  Hover with no panel open does NOT
  auto-open (preserves click affordance).
- **Click-outside dismiss.**
- **Disabled triggers** drop out of the
  ArrowLeft / Right cycle.

## ARIA

- Root: `<nav>` + `aria-label`.
- Sub-menu trigger: `<button>` with
  `aria-haspopup="menu"` +
  `aria-expanded` + `aria-controls`.
- Direct-link trigger: `<a>` with
  `aria-disabled` when applicable.
- Panel: `role="menu"` +
  `aria-orientation="vertical"` +
  `aria-labelledby=<trigger-id>`.

## Data attributes

| attr | location |
| --- | --- |
| `data-section="nav-menu"` | root nav |
| `data-section="nav-menu-item"` + `data-nav-menu-item=<id>` + `data-nav-menu-open` | per top-level li |
| `data-section="nav-menu-trigger"` + `data-nav-menu-trigger=<id>` | trigger element |
| `data-section="nav-menu-panel"` + `data-nav-menu-panel=<id>` | open dropdown panel |
| `data-section="nav-menu-mega"` (multi-section) or `"nav-menu-panel-list"` (single) | panel body |
| `data-is-mega="true"` | mega panel only |
| `data-section="nav-menu-panel-content"` | custom content slot |
| `data-section="nav-menu-section"` + `data-nav-menu-section=<id>` | section group |
| `data-section="nav-menu-section-heading"` | section heading row |
| `data-section="nav-menu-link"` + `data-nav-menu-link=<id>` | link anchor |
| `data-section="nav-menu-chevron"` | trigger chevron svg |

## Pairs with existing primitives

- `<Menubar>` (11.387) -- application
  menu strip. Use for desktop-style
  menus.
- `<DropdownMenu>` (11.362) -- single
  trigger + rich item types.
- `<HoverCard>` (11.385) -- pure preview
  popover; use for non-actionable
  previews vs nav links.

## Out of scope

- Per-page adoption of NavigationMenu.
- Mobile / hamburger sheet variant.
  Desktop-first; mobile fallback belongs
  in a separate primitive or
  `<Sheet>`-anchored composition.
- Portal-rendered dropdown panels. The
  current dropdown is absolute-positioned
  inside the trigger's container.
- Multi-row mega-menus with embedded
  forms / heroes. The `content` slot
  already covers fully custom layouts.
- Focus inside the panel after open.
  Currently uses native tab order;
  programmatic focus-on-open belongs in
  a separate accessibility patch.
- Sub-sub-menus (nested dropdowns inside
  a panel section). Use `<ContextMenu>`
  for nested menus.

# 11.1049 -- UI: chart-line-ichimoku-mid-cross-sig primitive (v1.11.1067)

## Scope

Add a pure-SVG dual-panel `ChartLineIchimokuMidCrossSig`
component that renders the close in the top panel and the
Ichimoku kumo midline (geometric centre of the cloud) plus
its smoothed SMA signal line in the bottom panel, marking
bullish / bearish midline-over-signal crossover trigger
events with bias coloring derived from the kumo midline
slope at the trigger bar.

Goichi Hosoda's 1969 Ichimoku Kinko Hyo ("one-glance
equilibrium chart") cloud is constructed from rolling
Donchian midpoints over three lookback windows (9 / 26 /
52). The kumo (cloud) is bounded by senkouA and senkouB;
its midline is the geometric centre of the cloud and
represents the equilibrium price implied by the cloud's
two spans. This primitive watches the kumo midline
against its own SMA-smoothed signal line to detect cloud
centerline trend trigger events.

## Ichimoku kumo midline pipeline

```
tenkan[i]    = (HH(high, tenkanPeriod) +
                LL(low,  tenkanPeriod)) / 2
kijun[i]     = (HH(high, kijunPeriod) +
                LL(low,  kijunPeriod)) / 2
senkouA[i]   = (tenkan[i] + kijun[i]) / 2
senkouB[i]   = (HH(high, senkouBPeriod) +
                LL(low,  senkouBPeriod)) / 2
kumoMid[i]   = (senkouA[i] + senkouB[i]) / 2
signal[i]    = SMA(kumoMid, signalLength)

bullish      : prev kumoMid <= prev signal &&
               cur kumoMid > cur signal
bearish      : prev kumoMid >= prev signal &&
               cur kumoMid < cur signal
regime       : 'bullish' when kumoMid >= signal
               'bearish' when kumoMid <  signal
               'none'    when either is null
bias         : kumoMid[i] vs kumoMid[i-1] -> up / down /
               flat / none
```

Default `tenkanPeriod = 9`, `kijunPeriod = 26`,
`senkouBPeriod = 52`, `signalLength = 3` -- canonical
Hosoda tuning paired with the family-conventional short
SMA signal.

Unlike the canonical Ichimoku visualisation, this
primitive uses **current-bar reads** (no 26-bar forward
displacement) so the current direction of the cloud
centerline can be compared with the current SMA signal
line without time-shifted alignment ambiguity. This is
the same convention adopted by the sibling
`chart-line-ichimoku-divergence-cross` primitive
(v1.11.1061).

Warmup is `senkouBPeriod + signalLength - 2 = 53` for
the default tuning: senkouB seeds at i = senkouBPeriod -
1 = 51, kumoMid is valid from i = 51, then the signal
SMA needs `signalLength - 1 = 2` more bars.

## Bit-exact anchors

All anchors use HL input.

- **CONST band** `high = K + 1`, `low = K - 1`: HH =
  K + 1, LL = K - 1 over every window -> tenkan =
  kijun = senkouB = K, senkouA = K, kumoMid = K. signal
  = SMA(K, 3) = K. kumoMid === signal -> regime
  `bullish` (>=) for every valid bar. 0 crosses.
  Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`: HH(high,
  n) at i = i + 1 (latest), LL(low, n) at i = i - n
  (oldest). tenkan = (i + 1 + i - 9) / 2 = i - 4.
  kijun = (i + 1 + i - 26) / 2 = i - 12.5. senkouA =
  (tenkan + kijun) / 2 = i - 8.25. senkouB = (i + 1 +
  i - 52) / 2 = i - 25.5. kumoMid = (senkouA +
  senkouB) / 2 = i - 16.875. SMA(kumoMid, 3) at i =
  ((i - 2 - 16.875) + (i - 1 - 16.875) + (i - 16.875))
  / 3 = i - 17.875. kumoMid - signal = +1 (constant).
  regime `bullish` for every valid bar. 0 crosses (no
  transition into bearish).
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`:
  mirror -> kumoMid = -i + 16.875. signal = -i +
  17.875. kumoMid - signal = -1 (constant). regime
  `bearish` for every valid bar. 0 crosses (no
  transition into bullish).

The +/- 1 offset between kumoMid and signal on LINEAR
input is the 1-bar SMA lag for a unit-slope line:
SMA(slope * i, 3) = slope * (i - 1).

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 3-state regime classifier follows the cross-sig family
  convention (bullish on `===`, bearish on `<`); the
  sibling `chart-line-ichimoku-divergence-cross`
  (v1.11.1061) uses a 5-state divergence model paired
  with price.
- Current-bar reads (no 26-bar forward displacement)
  deliberately deviate from the canonical Ichimoku
  visualisation. This is consistent with the divergence-
  cross sibling and is the correct convention for
  cross-sig detection -- a time-shifted cloud would
  introduce a 26-bar phase lag between kumoMid and
  signal that has nothing to do with price action.
- On steady LINEAR input kumoMid produces a constant
  +/- 1 offset against signal and zero crosses, which is
  correct canonical behaviour -- real-world price data
  with trend-direction changes produces typical kumo
  centerline cross patterns.
- The kumo midline aggregates three lookback windows
  (9 / 26 / 52). Warmup is dominated by the largest
  (senkouBPeriod = 52); shorter inputs return a `none`
  regime everywhere and `ok = false`.
- Complementary to the sibling SMA / EMA / Bollinger /
  Keltner / Donchian / CCI / RSI mid-cross-sig
  primitives -- Ichimoku adds a multi-timeframe Donchian
  composite to the cross-sig family, distinct from the
  single-MA siblings.

# 11.1065 -- UI: chart-line-adx-pos-cross primitive (v1.11.1083)

## Scope

Add a pure-SVG dual-panel `ChartLineAdxPosCross` component
that renders the close in the top panel and the Average
Directional Index +DI directional indicator in the bottom
panel, marking bullish (+DI crosses up through zero --
uptrend strength trigger up; directional pressure emerging
from indecision) +DI-vs-zero crossover trigger events with
bias coloring derived from the +DI slope at the trigger
bar.

J. Welles Wilder Jr's 1978 ADX +DI directional indicator
measures the strength of bullish price pressure: the
smoothed positive directional movement (DM) normalised by
smoothed true range (TR). Convention bounds it to
`[0, 100]`. This primitive flags transitions from `+DI ===
0` (no uptrend pressure -- indecision) to `+DI > 0`
(uptrend strength emerging) as the canonical "uptrend-
emerging" trigger.

## +DI zero-cross pipeline

```
upMove[i]    = high[i] - high[i-1]
downMove[i]  = low[i-1] - low[i]
+DM[i]       = (upMove > downMove && upMove > 0)
               ? upMove : 0
tr[i]        = max(high - low,
                   |high - close[i-1]|,
                   |low  - close[i-1]|)
smPlus[i]    = SMA(+DM, period)
smTr[i]      = SMA(tr,  period)
plusDI[i]    = (smTr > 0) ? 100 * smPlus / smTr : 0

bullish (uptrend-emerging trigger) :
  prev plusDI <= 0 && cur plusDI > 0
bearish (uptrend-fading trigger) :
  prev plusDI >= 0 && cur plusDI < 0
  (effectively never fires since plusDI >= 0 always)

regime       : 'bullish' when plusDI > 0
               'bearish' when plusDI < 0 (never)
               'none'    when plusDI === 0 or null
bias         : plusDI[i] vs plusDI[i-1] -> up / down /
               flat / none
```

Default `period = 14` -- canonical Wilder ADX tuning.

Since +DI is canonically bounded to `[0, 100]`, the
bearish (downward zero-cross) case never fires in
practice; the regime classifier and detector retain
symmetric shape with the other cross-sig family members
but the bearish path is effectively dead code on real
inputs. This is documented in the source as a
deliberate family-shape symmetry rather than an
intended signal.

## Sibling family

The ADX signal family has three primitives:

- **chart-line-adx-pos-neg-divergence v1.11.1075** --
  compares +DI direction vs -DI direction
- **chart-line-adx-trend-cross v1.11.1047** -- ADX itself
  vs the 20/25 threshold (trend emerging confirmation)
- **chart-line-adx-pos-cross v1.11.1083** (this
  primitive) -- +DI alone vs zero (uptrend strength
  emerging)

Each surfaces a different ADX signal niche. The +DI
zero-cross is the simplest of the three -- it watches a
single directional indicator's transition from zero
(indecision) to positive (uptrend pressure emerging).

Uses SMA-based smoothing (matching the adx-* family
convention) rather than Wilder's exponential smoothing,
for bit-exact integer DI values on linear input.

Warmup is `period + 1 = 15` for the default tuning: DM
and TR seed at i = 1, SMA window-fills at i = period =
14, and cross detection needs the previous bar's +DI,
so the first potential cross lands at i = period + 1 =
15.

## Bit-exact anchors

All anchors use HLC input.

- **CONST band** `high = K + 1`, `low = K - 1`, `close
  = K`: upMove = 0, downMove = 0 -> +DM = 0 (upMove > 0
  check fails). TR = 2. smPlus = 0, smTr = 2. plusDI =
  100 * 0 / 2 = 0. plusDI stays at 0 -> no transition
  above 0. 0 crosses. regime `none` (plusDI === 0).
  Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close
  = i`: upMove = +1, downMove = -1. +DM = 1. TR = 2.
  smPlus = 1, smTr = 2. plusDI = 50 (constant from i =
  period). prev plusDI at i = period - 1 is null
  (warmup), so cross detector skips i = period. From i
  = period + 1 onwards plusDI === 50 (constant), no
  transition. 0 crosses. regime `bullish` (plusDI > 0).
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`,
  `close = -i`: upMove = -1, downMove = +1. +DM = 0
  (upMove > 0 fails). plusDI = 0 (constant). 0 crosses.
  regime `none`.

All three steady-state anchors produce zero crosses
because +DI is constant (either 0 or 50). Real-world
crosses fire when +DI transitions from exactly 0 to
positive -- typically the first bar of an uptrend
emerging from a flat/range market. The synthetic
detect-function test verifies this with a hand-built
`[0, 0, 15, 30]` sequence.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 3-state regime classifier with `+DI === 0 -> none`
  semantics. Differs from most cross-sig family
  members which treat `=== signal -> bullish` (`>=`).
  Here `=== 0` is treated as "no signal" because zero
  DI literally means no directional movement, not a
  bullish-zone reading.
- Asymmetric usage: in practice the bearish cross
  never fires (+DI cannot go below 0). The bearish
  path in the detector is retained for family-shape
  symmetry but is effectively dead code. Documented in
  the source and accepted as a deliberate trade-off
  for consistent API shape with the broader cross-sig
  family.
- SMA-based smoothing (not Wilder) for bit-exact
  testability. Documented as a family-level tuning
  choice.
- Zero-guard on `smTr > 0` prevents NaN on CONST
  input -- returns 0. Matches pandas-ta and ta-lib
  behaviour.
- On steady LINEAR input +DI saturates at either 0
  (no bullish movement) or 50 (pure bullish movement)
  with flat slope -- no zero-crosses fire. Real-world
  crosses fire on the first bar where +DI rises above
  exactly 0, which corresponds to the first bar of an
  uptrend emerging from a flat range.
- Companion to the v1.11.1075 ADX +/-DI divergence
  primitive (compares +DI vs -DI directions) and the
  v1.11.1047 ADX trend-cross primitive (ADX vs
  20/25). This primitive is the simplest member --
  just +DI vs zero.

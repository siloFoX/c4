# 11.805 UI: chart-line-coppock-signal primitive (v1.11.823)

## Summary
Pure-SVG dual-panel React/TS primitive that plots the **Coppock Curve**
signal beneath the close. Coppock is the linear-weighted moving average
of the sum of long and short rate-of-change values over the lookback,
classically used to flag long-term momentum shifts at zero-line crosses.

## Formula
```
ROC(L)[i]   = (close[i] - close[i - L]) / close[i - L] * 100
sumROC[i]   = ROC(longROC)[i] + ROC(shortROC)[i]
coppock[i]  = WMA(sumROC, wmaPeriod)[i]
WMA weights = [1, 2, ..., wmaPeriod] / (wmaPeriod * (wmaPeriod + 1) / 2)
```
`coppock[i]` is `null` during warmup
(`i < max(longROC, shortROC) + wmaPeriod - 1`), whenever any close in the
window is non-finite, or when a denominator close is exactly zero.

## Defaults
- `longROC = 14`
- `shortROC = 11`
- `wmaPeriod = 10`
- `width = 720`, `height = 460`, `panelGap = 12`

## Zones and crosses
- `positive`: `coppock > 0`
- `negative`: `coppock < 0`
- `zero`: `coppock == 0`
- `none`: warmup / non-finite

Cross detection (`detectLineCoppockSignalCrosses`) walks the coppock
sequence and flags `'up'` when the previous defined value was `<= 0`
and the current is `> 0`, `'down'` for the mirror. Markers fire only
on cross bars.

## Bit-exact testability anchors
- **CONST close** (`close == K`, `K != 0`): `ROC = 0` at every valid
  bar, `sumROC = 0`, `WMA(0) = 0` -> `coppock = 0` (bit-exact).
  Verified across `K in {1, 5, 100, -3}` and `(longROC, shortROC,
  wmaPeriod)` combinations. `-0 -> +0` normalization applied so
  negative-K rounding through `(0 / -K) * 100` returns `+0`.
- **ZERO close** (`close == 0`): denominator is zero -> ROC null ->
  coppock null everywhere.
- **GEOMETRIC close = 2^k**: for every `L`,
  `ROC(L)[i] = (2^L - 1) * 100` is a constant positive integer that is
  exact in IEEE 754 within `Number.MAX_SAFE_INTEGER`. With defaults
  (14, 11, 10): `sumROC = (16383 + 2047) * 100 = 1843000`, WMA of
  constants is the constant -> `coppock = 1843000` (bit-exact).
  Verified for both the canonical preset and a small (4, 2, 3)
  preset which yields `1500 + 300 = 1800`.

## Design notes
- WMA uses the standard linear weights `[1..n] / (n*(n+1)/2)`. WMA of a
  constant is the constant exactly, which is what makes the bit-exact
  GEOMETRIC anchor work.
- `-0 -> +0` normalization in the ROC helper: `(0 / -K) * 100` produces
  `-0` in IEEE 754; the helper substitutes `+0` so CONST tests remain
  `Object.is`-bit-exact regardless of K's sign.
- Power axis is **auto-scaled with zero always in range** -- if all
  observed coppocks are positive (or all negative), the zero line still
  appears as a clear reference.
- Default colors: coppock line `#9333ea` (purple), bullish crosses
  `#16a34a` (green), bearish crosses `#dc2626` (red), zero line
  `#475569` dashed. Distinct from peak/trough/trend-power palettes.
- `noUncheckedIndexedAccess` satisfied via explicit `?? null` and
  defensive guards before access.

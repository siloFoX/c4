# 11.1031 -- UI: chart-line-bollinger-mid-cross-sig primitive (v1.11.1049)

## Scope

Add a pure-SVG dual-panel `ChartLineBollingerMidCrossSig`
component that renders the close in the top panel and the
Bollinger Band middle line (`SMA(close, period)`, the
Bollinger centerline) plus its smoothed SMA signal line in
the bottom panel, marking bullish (middle crosses up through
signal -- centerline trend trigger up) / bearish (middle
crosses down through signal -- centerline trend trigger
down) middle-over-signal crossover trigger events with bias
coloring derived from the middle slope at the trigger bar.

This is the Bollinger-tuned sibling of
`chart-line-sma-cross-sig` (v1.11.1042) -- same close-only
SMA-over-SMA-signal pipeline, but with `period = 20` (the
canonical Bollinger lookback, vs SMA's 14) so the centerline
matches Bollinger's standard configuration.

## Bollinger middle-over-signal pipeline

```
middle[i] = SMA(close, period)        ; i >= period - 1
signal[i] = SMA(middle, signalLength) ; i >= period +
                                        signalLength - 2
regime    = middle >= signal -> bullish
            middle <  signal -> bearish
            null            -> none
bias      = middle[i] vs middle[i-1] -> up / down / flat / none
bullish   : prev middle <= prev signal AND
            cur middle > cur signal
            (centerline trend trigger up)
bearish   : prev middle >= prev signal AND
            cur middle < cur signal
            (centerline trend trigger down)
```

Default tuning `period = 20`, `signalLength = 3` matches the
canonical Bollinger Band middle line lookback. The middle
centroid lag is `(period - 1) / 2 = 9.5` for the default
period -- precisely Bollinger's centerline lag. The signal
SMA adds another `(signalLength - 1) / 2 = 1` bar of lag, so
the steady-state `middle - signal` separation on linear
input is exactly `1`, matching the entire chart-line-*-
cross-sig family identity.

Warmup is `period + signalLength - 2 = 21` for the default
tuning: middle seeds at `period - 1 = 19`, then the signal
SMA adds `signalLength - 1 = 2` bars.

## Bit-exact anchors

- **CONST close = K**: every middle window averages K ->
  middle = K, signal = K, middle === signal -> regime
  `bullish` (>=) for every valid bar. 0 crosses. Verified
  across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP close = i**: SMA over linear input settles at
  `i - (period - 1) / 2 = i - 9.5`. The signal SMA adds
  `(signalLength - 1) / 2 = 1` bar of lag, giving signal =
  `i - 9.5 - 1 = i - 10.5`. middle - signal = `+1` exact
  for every valid bar -> regime `bullish`, 0 crosses.
- **LINEAR DOWN close = -i**: mirror -> middle = `-i +
  9.5`, signal = `-i + 10.5`. middle - signal = `-1` ->
  regime `bearish`, 0 crosses.

The clean `+1 / -1` separation under linear input -- a
direct consequence of the centroid-lag arithmetic `(period
- 1)/2 - (period - 1)/2 - (signalLength - 1)/2` collapsing
to `-(signalLength - 1)/2 = -1` -- is the identity bit-exact
tests rely on, and matches the SMA, WMA, and HMA cross-sig
siblings.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- Strict-inequality bullish vs `>=` regime classifier
  means the bullish boundary (middle === signal) settles
  into the bullish band, matching the SMA / WMA / HMA /
  EMA / TEMA / DEMA / KAMA cross-sig siblings.
- SMA helper uses `min === max` short-circuit for CONST
  bit-exactness so the FP rounding cannot drift K to
  K +/- epsilon.
- Bollinger centerline = SMA(close, 20) is the canonical
  lookback for upper / lower band derivation; the
  cross-sig primitive surfaces just the centerline
  trigger pattern. The full upper / lower band overlay is
  out of scope for this primitive (the
  `chart-line-bollinger-bands` overlay primitive can
  layer on top via composition).

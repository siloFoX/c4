# 11.539 -- web/src/components/ui/chart-line-ewma.tsx

Pure-SVG **exponentially-weighted moving average (EWMA)**
overlay. Each series runs the canonical recursive EWMA
formula forward and renders three coordinated elements on the
same axes: the dimmed raw observations line, the smoothed EWMA
line in the series color, and (when enabled) per-point
residual sticks linking raw to smoothed colored by sign
(green when raw > ewma, red when raw < ewma). A half-life
badge in the top-left calls out the canonical interpretation
of the strongest series' alpha: `ln(2) / -ln(1 - alpha)` is
the number of samples back that hold half the smoothing
weight.

## Recursive EWMA formula

For each step k from initial state `x_0`:

- `ewma[k] = alpha * raw[k] + (1 - alpha) * ewma[k-1]`

Where `alpha` in `(0, 1]` is the smoothing factor. Larger
alpha = more responsive (closer to raw); smaller alpha = more
smoothing. The half-life characterizes how fast old samples
decay in weight.

## Alternative parameterization

`alpha = 2 / (span + 1)` (pandas convention). The component
accepts either `alpha` or `span` -- when both are present
`alpha` wins. The layout records the effective `span = 2/alpha
- 1` for tooltips and aria labels.

## Distinct from related smoothers / filters

- `<ChartLineMovingAvg>` (11.513): SMA (simple equal-weight
  window). EWMA gives **exponentially decreasing** weights to
  older samples and has no fixed window.
- `<ChartLineBollinger>` (11.537): rolling SMA + envelope.
  EWMA is recursive and uses ALL past samples (with
  exponential decay).
- `<ChartLineKalman>` (11.538): recursive Bayesian filter with
  separate process and measurement noise variances.
  EWMA is **deterministic** with a single parameter -- no
  uncertainty model, no posterior covariance.
- `<ChartLineControl>` (11.534): SPC global mean +/- k*sigma.
  EWMA is a smoother, not a control rule (though EWMA control
  charts exist as a separate primitive opportunity).
- `<ChartLineTrend>` (11.512): one global linear regression
  line per series. EWMA is per-point, recursive.

The canonical recursive smoother used in pandas
(`Series.ewm`), monitoring (RRDtool's exponential averaging,
many Prometheus recording rules), and any context where the
question is "what is the running level of this signal with a
specific time-scale of memory?".

## Public surface

- `<ChartLineEwma series={...} />`.
- `series: ChartLineEwmaSeries[]` -- `{id, label, data,
  color?, alpha?, span?, initialEstimate?, ewmaDashArray?}`.
  Per-series alpha / span / initialEstimate / dash always
  beats chart-level.
- `alpha` (default `0.3`) -- smoothing factor in `(0, 1]`.
- `span` -- alternative parameterization
  (`alpha = 2 / (span + 1)`). Used only when `alpha` is not
  explicit.
- `initialEstimate` x_0 -- starting estimate. Defaults to the
  first observation.
- `rawColor` -- raw observation line color.
- `residualPosColor` / `residualNegColor` -- residual stick
  + dot tinting when enabled.
- `ewmaDashArray` -- chart-level dash for the EWMA line
  (default solid).
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showHalfLifeBadge`, `showRaw`,
  `showResidualSticks`, `animate`.
- Controlled + uncontrolled visibility via `hiddenSeries` /
  `defaultHiddenSeries` / `onHiddenSeriesChange`.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX` / `formatAlpha`.
- `onPointClick({series, point})`.

## Pure helpers

- `getLineEwmaDefaultColor(index)` -- 10-color palette cycle.
- `getLineEwmaFinitePoints(points)` -- finite-only filter.
- `normaliseLineEwmaAlpha(value)` -- clamps to `(0, 1]`;
  non-finite -> default 0.3.
- `lineEwmaSpanToAlpha(span)` -- `2 / (span + 1)`;
  non-positive / non-finite -> default alpha.
- `lineEwmaAlphaToSpan(alpha)` -- `2 / alpha - 1`; degenerate
  (alpha <= 0 or alpha >= 1) -> Infinity.
- `lineEwmaHalfLife(alpha)` -- `ln(2) / -ln(1 - alpha)`;
  alpha = 0 -> Infinity (no decay); alpha = 1 -> 0
  (immediate replacement).
- `classifyLineEwmaResidual(residual)` -- `'positive' |
  'negative' | 'zero'`; non-finite -> zero; exact 0 -> zero.
- `resolveLineEwmaAlpha(options?)` -- explicit `alpha` beats
  `span`; falls back to default when neither provided.
- `runLineEwma(points, options?)` -- canonical EWMA pass.
  Sorts ascending; drops non-finite; returns per-point
  `{index, x, raw, ewma, residual, residualSign}`. The first
  sample's residual is 0 when `initialEstimate` falls back to
  the first observation. Empty / null -> [].
- `computeLineEwmaLayout({series, ...})` -- returns `{series,
  xTicks, yTicks, xMin, xMax, yMin, yMax, innerWidth,
  innerHeight, totalPoints, visibleSeriesCount}`. Y range
  covers both raw and ewma. Per-series carries `{id, label,
  color, alpha, effectiveSpan, halfLife, initialEstimate,
  ewmaDashArray, points, rawPath, ewmaPath, residualSegments,
  finiteCount, totalCount, finalEwma, rmseResidual,
  positiveResidualCount, negativeResidualCount,
  zeroResidualCount, maxAbsResidual}`. Per-point carries
  projected `{..., px, rawPy, ewmaPy}`.
- `describeLineEwmaChart(series, options?)` -- screen-reader
  description with per-series alpha / half-life / final EWMA.

## Behaviour notes

- **alpha = 1** -> no smoothing (ewma == raw at every step).
- **alpha -> 0** -> infinite memory (ewma approaches the
  initial estimate, never updates).
- The first sample's residual is 0 when `initialEstimate`
  defaults to the first observation -- this is intentional
  so the smoother doesn't pretend to know more than it
  does at t=0.
- Span and alpha are interchangeable: passing `span=9` is
  identical to passing `alpha=0.2` (within float precision).

## Tooltip

Hovering a dot shows the series label in its colour, x, a
`raw:` row, a bold `ewma:` row, and a coloured `residual:` row
tinted by sign with the sign explicitly prefixed (`+` /
`-`).

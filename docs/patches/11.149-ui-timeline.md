# Patch 11.149 - UI: Timeline primitive

Web release: **v1.11.167**.

## New primitive

`web/src/components/ui/timeline.tsx` exports:

- `Timeline` - container `<ol>` with props `{ items: TimelineItem[]; groupByDay?: boolean; className?: string }`.
- `TimelineItem` (type) - `{ id; timestamp: string|Date; title; description?; icon?; tone? }`.
- `TimelineTone` (type) - `'neutral' | 'primary' | 'success' | 'warning' | 'danger'`.

Each `<li>` renders a tone-coloured dot (ARPS `bg-*` token), an optional icon, a localised timestamp, the title, and an optional description. A vertical connector line is drawn on every row except the last in its visible group. When `groupByDay` is true, items are bucketed by `toISOString().slice(0, 10)` and rendered under a `<h3>` day header per group, sorted ascending within each bucket.

Re-exported from `web/src/components/ui/index.ts`.

## Adoption sites (3 of 3)

1. **`web/src/pages/Auto.tsx` - autonomous dispatch timeline.** Full substitution. The inline `<ol>` previously rendered by `TimelineSection` is now produced by `<Timeline>`; the event-type to tone map (`dispatch`/`success`/`halt`/`dispatch-error`) preserves the original colour story while the lucide icon rides into the dot.
2. **`web/src/components/SessionsListSection.tsx` - sessions recent activity (subset).** The interactive group list is unfriendly to substitute fully (each session is a focusable `<button>` carrying selection state), so the Timeline is rendered **alongside** it at the top of the pane, showing the five most recently updated sessions as a read-only preview.
3. **`web/src/components/SpecialistsAuditPanel.tsx` - notifications log proxy.** No standalone Notifications page exists in this branch; the spec authorises the Audit log as a fallback. The audit `<ul>` is fully substituted by `<Timeline>`, with the prior action -> colour mapping translated into Timeline tones (`add`->success, `remove`->danger, `import`/`score-applied`/`tags-updated`->primary, `prompt-revised`/`score-reset`->warning).

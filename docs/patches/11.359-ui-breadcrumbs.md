## `lib/use-hash-route.ts`

```ts
const { hash, navigate } = useHashRoute();
const handler = makeBreadcrumbNavigator(navigate);
```

| export | purpose |
| --- | --- |
| `useHashRoute()` | returns `{ hash, navigate }` and re-renders on every `hashchange` event |
| `parseHashParams(hash)` | pure helper: splits `#feature=workers&worker=auto-w42` into a `Record<string, string>` (URL-decoded) |
| `makeBreadcrumbNavigator(navigate)` | factory that returns `(target) => (event) => void`; the handler preventDefaults + calls `navigate(target)` |

Behaviour:

- `useHashRoute()` is SSR-safe (every
  `window` access lives inside `useEffect` or
  behind a `typeof window` guard).
- `navigate(target)` normalises a leading `#`
  and emits a `hashchange` event even when
  the target equals the current hash (so the
  Breadcrumbs onClick handler still flips
  the route).
- `parseHashParams` skips malformed segments
  (no key / no `=`) and accepts strings with
  or without a leading `#`.
- `makeBreadcrumbNavigator` survives events
  without `preventDefault` so adopters can
  call the handler from synthetic test events.

## Adoption

### `HistoryDetailPane.tsx`

The "History" root crumb now carries an
`href="#feature=history"` + an `onClick` built
from `makeBreadcrumbNavigator`:

```tsx
const { navigate } = useHashRoute();
const breadcrumbNav = makeBreadcrumbNavigator(navigate);
// ...
<Breadcrumbs
  items={[
    {
      id: 'history',
      label: t('history.sidebar.title'),
      href: '#feature=history',
      onClick: breadcrumbNav('#feature=history'),
    },
    { id: 'worker', label: detail.name },
  ]}
/>
```

Middle-click + screen-reader behaviour still
work via the real `href`; primary-click
navigates via the hash without a full page
reload.

### `Snapshots.tsx`

Adds a new Breadcrumbs above the table:
Workers > Snapshots. The Workers crumb is
hash-routed via the helper; the Snapshots
crumb is the current page (no link).
`data-testid="snapshots-breadcrumbs"` for the
e2e selector.

## Tests

`use-hash-route.test.ts` -- 15 cases:

- `parseHashParams`: empty -> {}, single
  key=value, multiple separated by &,
  URL-decoded values, skips no-key /
  no-= segments, missing leading `#`.
- `useHashRoute`: initial hash on mount,
  updates on hashchange, navigate writes
  to location.hash with normalisation,
  accepts pre-#'d target, removes listener
  on unmount.
- `makeBreadcrumbNavigator`: returns a
  function that calls navigate(target),
  preventDefaults the event, survives
  events without preventDefault.

15/15 pass under vitest 4.1.5. TypeScript
clean for the new lib + the two adopted
files.

## Pairs with existing primitives

- `components/ui/breadcrumbs.tsx` (11.283)
  unchanged. The new lib slots into the
  existing `items[].onClick` contract.
- `App.tsx` hash routing remains the
  app-level pattern; the new lib shares the
  same hash format
  (`#feature=<name>`).
- `lib/scroll-restore.ts` (11.348) -- the
  popstate-based scroll restore composes
  cleanly with hash navigation since
  hashchange does NOT fire popstate.

## Out of scope

- React Router integration. No Router in
  the c4 app today; the `navigate` shape
  matches `useNavigate()` closely enough
  that a future migration can swap them
  with a small adapter.
- Per-page Breadcrumbs adoption beyond the
  two cited surfaces. The legacy
  `PageHeader.breadcrumbs[]` shape is the
  alternate path.
- Replace-vs-push history semantics.
- Per-page hash-route tests at
  HistoryDetailPane / Snapshots; the lib
  has 15 dedicated cases.

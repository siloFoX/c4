# 11.569 -- web/src/components/ui/chart-line-lag-plot.tsx

Pure-SVG line chart with a **lag-k scatter** side panel. The
left panel shows the raw time series; the right panel is a
lag plot -- each value `y[t]` scattered against the value
`k` steps earlier, `y[t-k]`. A tight cluster along the
diagonal signals strong lag-k serial dependence; a
structureless cloud signals randomness.

## The lag analysis

`runLineLagPlot(points, lag)`:

1. The series is sorted by x.
2. `normalizeLineLagPlotLag` coerces the lag to a positive
   integer -- a non-finite or sub-1 lag falls back to 1, a
   fractional lag is floored.
3. For every `t` from `k` to `n - 1` a pair is formed:
   `laggedValue = y[t-k]` (the scatter x), `currentValue =
   y[t]` (the scatter y). There are `n - k` pairs.
4. The **lag-k autocorrelation** is the Pearson correlation
   between the lagged column and the current column.

Each pair also carries an `aboveDiagonal` flag
(`currentValue > laggedValue`) -- the points above the
identity line are the steps where the series rose.

The component fixtures make the correlation exact: the
perfectly linear series `[0, 2, 4, 6, 8, 10]` has every
lag-1 pair on the line `current = lagged + 2`, so the lag-1
correlation is **1**; the strictly alternating series
`[0, 10, 0, 10, ...]` flips every step, so its lag-1
correlation is **-1**.

## Distinct from `chart-line-autocorrelation`

`chart-line-autocorrelation` draws the **correlogram** -- a
bar chart of the autocorrelation `r_k` across *many* lags
`k = 0, 1, 2, ...`. ChartLineLagPlot draws the **actual
scatter cloud** at *one* chosen lag `k`. The correlogram
reduces each lag to a single number; the lag plot keeps
every `(y[t-k], y[t])` pair so the shape of the dependence
-- linear, curved, clustered, outlier-driven -- is visible.
They are complementary: the correlogram picks the
interesting lag, the lag plot inspects it.

All exported helpers carry a `LineLagPlot` infix so the
`export *` barrel line cannot collide.

## Public surface

- `<ChartLineLagPlot data={...} />` -- a single
  `ChartLineLagPlotPoint[]` (`{x, value}`).
- `lag` -- the lag k (default 1; coerced to a positive
  integer).
- `linePanelRatio` -- the fraction of the width given to the
  series panel (default 0.55; clamped to [0.2, 0.8]).
- `gap` -- the pixel gap between the two panels.
- Toggle flags: `showAxis`, `showGrid`, `showSeriesDots`,
  `showIdentityLine`, `showPanelLabels`, `showTooltip`,
  `showConfigBadge`, `showFooter`, `animate`.
- Standard sizing + format + a11y props. `formatValue` /
  `formatX`.
- `onSampleClick({point})` (a series dot) /
  `onPairClick({pair})` (a scatter dot).

## Pure helpers

- `getLineLagPlotFinitePoints(points)`.
- `normalizeLineLagPlotLag(lag)`.
- `runLineLagPlot(points, lag?)` -- the lag pairing + the
  lag-k autocorrelation; `ok` requires at least 2 finite
  samples and at least one pair.
- `computeLineLagPlotLayout({...})` -- splits the canvas
  into the series and scatter panels and projects both; the
  scatter panel shares one value domain across both axes so
  the identity line is a true `y = x` diagonal.
- `describeLineLagPlotChart(data, options?)`.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- The series line, every series dot and every scatter dot
  `role="graphics-symbol"` + `tabIndex={0}`.
- Every node carries `data-section="chart-line-lag-plot-..."`.
- Root mirrors `data-lag`, `data-pair-count`,
  `data-correlation`, `data-animate`.
- Axis / tick / grid / panel-label nodes carry
  `data-panel="series"|"scatter"`. Scatter dots expose
  `data-t`, `data-lagged-value`, `data-current-value`,
  `data-above-diagonal`.

## Tooltip / badge / footer

Hovering a series dot shows x + value; hovering a scatter
dot shows the pair index `t`, the lagged value, the current
value and the delta. Config badge: `LAG k=<lag>
r=<correlation> n=<pairCount>`. The footer reports
`lag / pairs / r` and notes the `y = x` identity diagonal.

## Summary

Add a new `<ChartPictogram>` UI primitive in
`web/src/components/ui/chart-pictogram.tsx` -- pure-SVG
pictogram (isotype) chart that uses repeated icons to
represent quantities. Each row paints `value / unitValue`
icons in a left-to-right grid, with the final icon
partially filled when the value is fractional (clipped
via `clipPath` to encode the fractional portion).
Distinct from every other chart-* primitive: this is
the canonical "Otto Neurath isotype" presentation where
the **count of glyphs** -- not a continuous bar or area
-- carries the quantitative meaning.

## API

```ts
type ChartPictogramIconShape =
  | 'circle' | 'square' | 'rounded'
  | 'star' | 'triangle' | 'hexagon' | 'person';

interface ChartPictogramRow {
  id: string;
  label: string;
  value: number;
  color?: string;
  icon?: ChartPictogramIconShape | string;
}

interface ChartPictogramProps {
  rows: readonly ChartPictogramRow[];
  unitValue?: number;            // default 1
  width?: number;                // default 560
  height?: number;               // default 320
  padding?: number;              // default 32
  iconSize?: number;             // default 18
  iconGap?: number;              // default 4
  rowGap?: number;               // default 14
  labelWidth?: number;           // default 100
  defaultIcon?: ChartPictogramIconShape | string;  // default 'circle'
  emptyOpacity?: number;         // default 0.18
  showLabels?: boolean;          // default true
  showCounts?: boolean;          // default true
  showTooltip?: boolean;         // default true
  showLegend?: boolean;          // default false
  animate?: boolean;             // default true
  className?: string;
  ariaLabel?: string;            // default 'Pictogram chart'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  formatCount?: (count: number, value: number) => string;
  onRowClick?: (args: {
    row: ChartPictogramRow;
    layout: ChartPictogramLayoutRow;
  }) => void;
  style?: CSSProperties;
}
```

## Behaviour

- Renders `<div role="region">` wrapping
  `<svg role="img">` with the same `aria-label`.
- Each row paints `ceil(value / unitValue)` icons in a
  left-to-right grid. The `full` icons (`floor(value /
  unitValue)`) are painted at fill=1; the last icon
  (if `value` isn't a clean multiple of `unitValue`)
  is painted at the fractional fill via a per-row
  `<clipPath>` so partial values are visually
  meaningful.
- Empty (background) icons paint behind every glyph at
  `emptyOpacity` so each slot is visible even when
  empty -- e.g., showing 7-of-10 lights filled, not
  just 7 lights floating.
- Icons-per-row is auto-computed from the available
  inner width minus the label gutter and divided by
  `iconSize + iconGap`. Long rows wrap automatically
  into additional visual rows.
- Built-in icon shapes: `circle` (default), `square`,
  `rounded`, `star`, `triangle`, `hexagon`, `person`.
  A custom SVG path (starting with `M`) can be passed
  in place of a shape name; the path is used verbatim.
- Optional row label (left-anchored at the label
  gutter) and per-row count badge (right of the
  glyph block) describe each row.
- Hover/focus on a row sets the hovered id, dims the
  other rows to opacity 0.4, and opens a tooltip with
  label, formatted value, and the icon count.
- Mount animation:
  `motion-safe:animate-fade-in` per row group.
  `animate=false` removes the class.
- Default ARIA description: `"Pictogram chart with N
  rows, total <total> across K icons (each icon =
  <unitValue>)."` -- empty / all-zero -> `"No data"`.

```ts
DEFAULT_CHART_PICTOGRAM_WIDTH          = 560

getPictogramIconCount(value, unitValue): { full, fractional, total }
buildPictogramIconPath(shape, size): string
computePictogramLayout(input): ChartPictogramLayoutRow[]
describePictogramChart(rows, unitValue, formatValue?): string
```

- `getPictogramIconCount` clamps non-positive /
  non-finite to zero.
- `buildPictogramIconPath` accepts custom SVG path
  strings (starting with `M`) and returns them
  verbatim, otherwise emits a closed shape sized to
  `size`.
- `computePictogramLayout` returns row entries of
  shape `{ id, label, index, value, color, icon,
  totalIcons, fullIcons, fractionalIcon, iconsPerRow,
  visualRows, rowY, rowHeight, icons }` where each
  icon is `{ index, x, y, fill }`.

## ARIA + data attributes

- Root: `<div role="region">` + `aria-label`.
- SVG: `role="img"` + same `aria-label`.
- sr-only span:
  `data-section="chart-pictogram-aria-desc"`.
- Per-row icon group: `role="graphics-symbol"` +
  `tabIndex={0}`. aria-label =
  `"<label>: <value> (<count> icons)"`.
- `data-section` on every node:
  chart-pictogram / -canvas / -aria-desc / -svg /
  -rows / -row / -row-label / -icons / -icon /
  -icon-empty / -icon-fill / -count / -tooltip /
  -tooltip-label / -tooltip-value / -tooltip-icons /
  -legend / -legend-item / -legend-row /
  -legend-swatch / -legend-label.
- Root mirrors `data-row-count`,
  `data-visible-row-count`, `data-icon-count`,
  `data-unit-value`, `data-total-value`,
  `data-default-icon`, `data-animate`.
- Row groups mirror `data-row-id`,
  `data-row-index`, `data-row-value`,
  `data-row-color`, `data-row-icon-count`,
  `data-row-full-icons`, `data-row-fractional-icon`,
  `data-row-visual-rows`, `data-hovered`.
- Icon groups mirror `data-icon-index`,
  `data-icon-fill` (0..1).

## Out of scope

- 2D bin-packing (icons of different sizes per row)
  -- v1 uses a uniform grid.
- Auto-scale icon size based on canvas + value range
  -- v1 honours the explicit `iconSize` prop; host
  controls density.
- Animated reveal one-icon-at-a-time -- snap v1;
  mount fade only.
- Image / sprite icons via `<image>` -- v1 supports
  built-in shape names and custom SVG path strings.

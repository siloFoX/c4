RSI formula across a momentum lookback.

## Indicator math

The Relative Momentum Index (Roger Altman) generalises the RSI.
The ordinary RSI measures each bar's gain or loss against the
*immediately prior* close. The RMI measures it against the close
`momentumPeriod` bars earlier -- a longer momentum lookback --
then applies the same RSI smoothing and formula.

`computeLineRmiChange(closes, momentumPeriod)` is the momentum
change `change[i] = close[i] - close[i - momentumPeriod]`.

`computeLineRmi(closes, period, momentumPeriod)` turns that
change into gains and losses (`gain = max(change, 0)`,
`loss = max(-change, 0)`) and computes

```
RMI[i] = 100 * avgGain / (avgGain + avgLoss)
```

where `avgGain` and `avgLoss` are `period`-bar simple moving
averages of the gains and losses. A window with no movement at
all reads 50. With `momentumPeriod` 1 the RMI is exactly the
ordinary RSI.

### Worked fixture

The RMI is a ratio of summed gains and losses -- no
transcendentals -- so with the right closes it is exact (asserted
with `toEqual`).

The fixture closes `[40, 44, 48, 52, 56, 60, 48, 52, 40, 44, 48,
52, 40, 60]` are chosen so every 2-bar momentum change is exactly
`+/- 8`. With `period` 4 each RMI window then has gain+loss total
`4 * 8 = 32`, so `RMI = 100 * gainSum / 32` lands on the exact
dyadics `{0, 25, 50, 75, 100}`:

```
change [., ., 8, 8, 8, 8, -8, -8, -8, -8, 8, 8, -8, 8]
rmi    [., ., ., ., ., 100, 75, 50, 25, 0, 25, 50, 50, 75]
```

Against an overbought level of 70 and an oversold level of 30
the bars classify as overbought / oversold / neutral 3 each.

## API

- `getLineRmiFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineRmiPeriod(period, fallback)` -- coerce to a
  positive integer.
- `computeLineRmiChange(closes, momentumPeriod)` -- the momentum
  change against the close `momentumPeriod` bars ago.
- `computeLineRmi(closes, period, momentumPeriod)` -- the RMI
  series.
- `runLineRmi(points, options?)` -- sort finite points by `x`,
  run the pipeline, return `{ series, period, momentumPeriod,
  overbought, oversold, rmi, samples, rmiFinal, overboughtCount,
  oversoldCount, neutralCount, ok }`. Each sample is classified
  `overbought` / `oversold` / `neutral` / `none`. `ok` is false
  for fewer than two points.
- `computeLineRmiLayout(options)` -- stack the price panel above
  an RMI panel on a fixed 0-100 scale; project the price path,
  the RMI path, one marker per defined bar, and the overbought
  and oversold reference lines.
- `describeLineRmiChart(data, options?)` -- ARIA description.

## Component

`ChartLineRmi` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, two stacked panels
(price on top, RMI below) with their own axis ticks and grid,
panel labels, a config badge showing the period and the momentum
lookback, a three-series legend (Price / RMI / Levels) with
toggle, a hover/focus tooltip exposing the RMI and the zone, and
a `motion-safe:animate-fade-in` entrance. It consumes
`{ x, value }` points. The RMI panel is fixed to a 0-100 scale,
draws dashed overbought and oversold reference lines, and colours
each RMI marker by its zone (red overbought, green oversold,
violet neutral). Every node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-rsi** -- that primitive measures each bar
  against the prior close (a 1-bar momentum). The RMI
  parameterises the momentum lookback, so it is the RSI
  generalised across a multi-bar change.
- **vs chart-line-tdi** -- the TDI is a composite that layers a
  signal line and a band on the RSI. The RMI is a single
  oscillator line with overbought / oversold levels.
- **vs the momentum oscillators (dpo / cmo / tsi)** -- those use
  their own normalisations. The RMI keeps the exact RSI ratio
  formula and only widens the change window.

# 11.815 UI: chart-line-fractal-bb primitive (v1.11.833)

## Summary
Pure-SVG **single-panel** React/TS primitive that overlays the close
with a Bollinger Band envelope **anchored to fractal pivots** rather
than to `mean +/- sigma * stdDev`. Each bar reseeds upper/lower from
the highest and lowest fractal pivots confirmed within the lookback
window; the mean is the midpoint of those pivots.

## Formula
```
fractalHigh[k] = high[k] iff for r in [1, pivotRadius]:
                    high[k] > high[k - r] && high[k] > high[k + r]
                  else null
fractalLow[k]  = low[k]  iff for r in [1, pivotRadius]:
                    low[k] < low[k - r] && low[k] < low[k + r]
                  else null
window[i]      = [max(0, i - length + 1), i - pivotRadius]
upper[i]       = max(fractalHigh[k] for k in window)
lower[i]       = min(fractalLow[k]  for k in window)
mean[i]        = (upper[i] + lower[i]) / 2
```
The window upper bound is `i - pivotRadius` because pivots in the
most recent `pivotRadius` bars are not yet confirmed. Bars before
`length + pivotRadius - 1`, or with no confirmed pivots in the window,
yield `null` for the band.

## Defaults
- `length = 20`
- `pivotRadius = 1` (3-bar fractal)
- `width = 720`, `height = 400`, `padding = 44`

## Zones
- `above-upper`: `close > upper`
- `below-lower`: `close < lower`
- `at-mid`: `close == mean`
- `in-band`: between (and not at mean)
- `none`: warmup / no confirmed pivots

## Bit-exact testability anchors
- **CONST high=low=close=K**: strict-inequality fractal test always
  fails (no left/right neighbor is strictly less or greater) -> no
  pivots -> `upper = lower = mean = null` everywhere. Verified across
  `K in {0, 1, 5, 100, -3}` and `length in {3, 5, 7, 10}`.
- **Constructed peak fixture**
  `highs = [10, 11, 12, 11, 10, 8, 9, 13, 12, 11, 10]`,
  `lows = [9, 10, 11, 10, 9, 7, 8, 12, 11, 10, 9]`,
  `pivotRadius = 1`:
  - bar 2 is a fractal high with value `12`.
  - bar 7 is a fractal high with value `13`.
  - bar 5 is a fractal low with value `7`.
  At bar 8 with `length = 5` the window is `[4, 7]`:
  `upper = 13`, `lower = 7`, `mean = 10` (all integer-exact).
  At bar 6 with `length = 5` the window is `[2, 5]`:
  `upper = 12`, `lower = 7`, `mean = 9.5` (dyadic-exact).
- **`pivotRadius = 2`** detects 5-bar fractals: explicit fixture
  `highs = [10, 11, 12, 13, 12, 11, 10]` -> bar 3 is a fractal high
  with value `13`.

## Design notes
- The fractal-anchored approach makes the band react to **structural
  pivots** rather than rolling volatility -- it stays put until a
  newer pivot enters or an older one falls out of the lookback
  window. This produces a "blocky" envelope that reseeds at discrete
  event boundaries.
- `pivotRadius` controls the strictness of the local-max/min test:
  `1` accepts the lightest 3-bar pivots; higher values demand more
  confirmation but reduce false signals.
- `windowEnd = i - pivotRadius` enforces that pivots are only
  considered once they're **confirmed** (we have `pivotRadius` future
  bars to validate them). This avoids look-ahead.
- Markers render on the actual pivot bars (anchored at the pivot
  value, not the close). High markers in red, low markers in green.
- Default colors: band `#0891b2` (cyan), fractal-high `#dc2626`,
  fractal-low `#16a34a`, mean `#475569` dashed.
- `noUncheckedIndexedAccess` satisfied via explicit `?? null` /
  defensive null guards before access.

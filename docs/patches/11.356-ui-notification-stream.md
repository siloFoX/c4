## Goal

Dispatch:

- Build `web/src/lib/notification-stream.ts` --
  SSE subscriber for `/api/notifications` with
  toast queue (max 5 visible, auto-dismiss with
  hover-pause, click-to-action).

`useToast` / `Toast` already ship the imperative
host pattern (host calls `showToast(message,
kind)`). This patch ships the server-pushed
counterpart: a hook that subscribes to an SSE
endpoint, parses the JSON frames, and exposes a
bounded queue with auto-dismiss, hover-pause, and
click-to-action handlers.

## Wire shape

```json
{
  "id": "notif-1",
  "kind": "success" | "error" | "info" | "warning",
  "message": "Worker auto-w42 finished",
  "title": "Optional title",
  "ts": 1747584000000,
  "durationMs": 5000,
  "sticky": false,
  "action": {
    "label": "Open",
    "href": "/?view=workers",
    "payload": { ... }
  }
}
```

Parser tolerances:

- `type` is accepted as a synonym for `kind`.
- Unrecognised `kind` -> `'info'`.
- Missing `id` -> stable
  `generateFallbackId()`.
- Missing / non-numeric `ts` -> `Date.now()`.
- Action requires `label`; otherwise dropped.
- Malformed JSON returns `null` and the
  subscriber silently drops the frame.

## Hook API

```ts
const stream = useNotificationStream({
  url = '/api/notifications',
  maxVisible = 5,
  autoDismissMs = 5000,
  onMessage?,    // (notification) => void
  onAction?,     // (notification) => void
  onError?,      // (event) => void
  eventSourceFactory?,  // (url) => EventSourceLike (tests)
  enabled = true,
});
```

Returned:

| key | purpose |
| --- | --- |
| `toasts` | visible queue, FIFO, cap `maxVisible` |
| `connected` | true when EventSource is open |
| `paused` | hover-pause flag |
| `dismiss(id)` | remove one + clear its timer |
| `dismissAll()` | clear queue |
| `enqueue(notification)` | host-side insert; dedups by id |
| `setPaused(b)` | freeze every active timer |
| `fireAction(id)` | calls `onAction` then dismisses |

### Hover-pause semantics

`setPaused(true)` clears every active
`setTimeout` and records the remaining duration
per entry (`durationMs - elapsed`). `setPaused(false)`
schedules a fresh `setTimeout` per entry using
the recorded remainder, so a 1000ms timer paused
at 400ms fires 600ms after resume.

### Queue invariants

- `maxVisible` cap; older entries shift out FIFO
  on overflow. The dropped entries' timers are
  cleared first.
- `enqueue` dedups by id: a same-id entry
  replaces in place and resets the timer. Lets
  the server update a notification (progress
  message) without piling rows.
- `sticky: true` skips the auto-dismiss timer
  (equivalent to `durationMs: 0`).
- `durationMs` per entry wins over
  `autoDismissMs`.

### Lifecycle

- SSR-safe: every `EventSource` / `window`
  access runs inside `useEffect`.
- Closes the EventSource + clears every timer
  on unmount.
- `enabled: false` skips the SSE wiring but
  keeps the queue working for imperative
  `enqueue(...)` calls -- useful for tests +
  optional notification surfaces.

## Tests

`notification-stream.test.ts` -- 26 cases:

- `parseNotificationPayload`: minimal payload,
  `type` aliases `kind`, unrecognised kind ->
  info, action label + href, drops action with
  no label, null on malformed JSON, null when
  message missing, explicit ts honoured,
  sticky + durationMs.
- `useNotificationStream`: opens EventSource +
  connected flag flips; error flips connected +
  fires onError; enqueues on JSON message;
  silently drops malformed messages; queue cap
  at maxVisible (oldest shift out); dedup by
  id; auto-dismiss after autoDismissMs;
  per-entry durationMs override; sticky never
  dismiss; hover-pause freezes + resume keeps
  remaining duration; dismiss(id) removes one;
  dismissAll() empties; fireAction(id) calls
  onAction + dismisses; fireAction with unknown
  id no-ops; closes EventSource on unmount;
  enabled=false skips SSE but keeps enqueue;
  default autoDismissMs is 5000.

## Out of scope

- **Daemon-side `/api/notifications` SSE
  endpoint.** The client subscriber + queue
  are in place; the daemon route ships
  separately.
- **`<NotificationPortal>` component** that
  renders the queue as stacked toasts at the
  edge of the viewport. The hook is ready; a
  future patch wires the visual surface.
- **App.tsx adoption.** Per-surface adoption
  pairs better with each page's own
  notification routing.
- **Cross-tab dedup.** Each tab maintains its
  own EventSource connection; a fired
  notification reaches every open tab.
- **Reconnect logic on transient errors.** The
  hook flips `connected: false` on `onerror`
  and leaves the EventSource open (the browser
  auto-reconnects). Adopters that need explicit
  reconnect can layer it on top of `onError`.
- **Rate limit / drop policy** beyond the
  visible cap. A future patch can add a global
  drop counter when the server bursts faster
  than the user can dismiss.

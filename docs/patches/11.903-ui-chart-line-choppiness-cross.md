# Patch 11.903 -- UI: chart-line-choppiness-cross primitive

## Summary

Added `<ChartLineChoppinessCross />` -- pure-SVG dual-panel
React/TS primitive that renders the close (top panel) with
bullish (trending) / bearish (ranging) arrow overlays at every
Choppiness Index threshold cross, and a close-only Choppiness
Index line in the bottom panel on a fixed 0-100 oscillator with
38.2 / 50 / 61.8 reference bands. Threshold-cross variant that
separates trending vs ranging market regime transitions from
the base Choppiness Index line readings.

## Algorithm

```
TR[i]      = i == 0 ? 0 : |close[i] - close[i-1]|
sumTR[i]   = sum(TR, n) over the last n bars
range[i]   = max(close, n) - min(close, n) over last n bars
ci[i]      = range > 0
              ? 100 * log10(sumTR / range) / log10(n)
              : 50  (neutral fallback when range collapses)
bullish   : prev >= 38.2 && cur < 38.2  (trending start)
bearish   : prev <= 61.8 && cur > 61.8  (ranging start)
```

Defaults `length = 14` (canonical Choppiness Index window),
`trendThreshold = 38.2`, `chopThreshold = 61.8` (the Fibonacci
levels from the canonical study). Regime classifier `bullish`
(CI < 38.2, trending), `bearish` (CI > 61.8, ranging),
`neutral` (between), `none` (null).

## Bit-exact anchor

CONST close = K: TR = 0 every bar -> sumTR = 0, range = 0 ->
formula degenerates to 0/0 so the primitive applies the
explicit neutral fallback `ci = 50` on every settled bar.
38.2 < 50 < 61.8 -> regime is `neutral`, cross count = 0.
Verified across K = 0, 1, 5, 17, 100, 1234.

LINEAR UP also has a stable anchor: monotonic close means
sumTR = (n-1) and range = (n-1), so the log ratio is
log10(1) = 0 leading to CI = 0, which is well below 38.2 and
classifies as `bullish` (trending).

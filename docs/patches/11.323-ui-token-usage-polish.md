# 11.323 -- UI: TokenUsage page polish

- TODO id: 11.323

- Date: 2026-05-18
- Scope:
  - `web/src/pages/TokenUsage.tsx`
  - `web/src/pages/TokenUsage.test.tsx`

## Goal

Polish `web/src/pages/TokenUsage.tsx` to surface the same
breakdown / feedback affordances as the rest of the page
hero set:

- A DataList breakdown of the headline totals + per-day
  / per-worker averages so the operator has a single
  place to scan the numbers.
- A `ScrollArea` wrapper around the per-task table
  instead of an inline `<div overflow-y-auto>`.
- A `Sparkline` cell per per-task row showing the I/O
  split.
- An `ErrorState` with a Retry button on the error path.
- An "All time" period option so the operator can see
  every recorded day in one view.

Component-scope only. No daemon endpoint or CLI surface
change. The Sparkline header summary above the per-day
bars (v1.11.279) is preserved unchanged.

## Behaviour

### Period selector

`PeriodOption` now reads `'1' | '7' | '30' | '90' | 'all'`
(was `1 | 7 | 30 | 90` as numbers). The SegmentedControl
registers a fifth option ("All time"). A new
`PERIOD_TO_DAYS` map maps each key to a day count:

```ts
const PERIOD_TO_DAYS: Record<PeriodOption, number> = {
  '1': 1,
  '7': 7,
  '30': 30,
  '90': 90,
  all: 36500,
};
```

`all` -> 36500 days (~100 years) so `dateRange(days)`
produces a window that includes every realistic recorded
day. The breakdown hook signature stays unchanged.

### Breakdown DataList

A new Panel sits directly under the headline Total panel
and wraps a `DataList` with the following rows:

- `Total` -> `formatNumber(data.total)`
- `Input` -> `formatNumber(data.totalInput)` (only when
  `totalInput != null`)
- `Output` -> `formatNumber(data.totalOutput)` (only when
  `totalOutput != null`)
- `Avg / day` -> mean of `perDay[].total` (or `-` when
  `perDay` is empty)
- `Avg / worker` -> mean of `perWorker[].total` (or `-`
  when `perWorker` is empty)
- `Period` -> `Last <N> day[s]` or `All time` depending
  on the active period

Wrapper carries `data-testid="token-usage-breakdown-panel"`
for direct scoping in tests.

### Per-task Sparkline column

The per-task table picks up a new `io` column titled
`I/O`. Its render uses the row's `input` + `output` as a
2-point Sparkline (`variant="info"`, `size="sm"`,
`width=48`). Rows with `input === 0 && output === 0`
fall back to the `-` placeholder rather than rendering
a degenerate sparkline.

The column is registered in `ColumnPicker` so the
operator can hide it via the existing column-picker
modal. Default visibility includes the new column.

### ScrollArea

Replaces the prior inline
`<div className="max-h-64 overflow-y-auto">` wrapper with
`<ScrollArea axis="y" className="max-h-64"
data-testid="token-usage-per-task-scrollarea">`. Same
visual height cap, but picks up the primitive's
consistent scrollbar styling + e2e selector contract.

### ErrorState

`ErrorPanel` is replaced by the canonical `ErrorState`
primitive on the error path:

- Title: `"Could not load /api/token-usage"`.
- Description: `"The daemon returned an error. Retry to
  re-fetch."`
- `onRetry`: wired to the hook's `refresh` callback.
- `data-testid="token-usage-error-state"`.

`role="alert"` is preserved by the primitive so existing
`getByRole('alert')` assertions still hold.

## Tests

`web/src/pages/TokenUsage.test.tsx` is updated:

- Three text-presence cases (`Total`, `Input`, `Output`)
  switched from `getByText` to `getAllByText` because
  those values now appear in both the headline panel AND
  the breakdown DataList.
- One dash-placeholder case scoped via
  `within(scrollArea)` to avoid the new DataList's avg-
  rows that may also render `-` when breakdowns are
  empty.
- Six new cases:
  1. "All time" period tab exists;
  2. clicking the "All time" tab flips its
     `aria-selected`;
  3. ErrorState shows a Retry button that fires the
     hook's `refresh` callback;
  4. breakdown DataList renders avg-per-day,
     avg-per-worker, and Period rows;
  5. per-task table wraps in a `ScrollArea`;
  6. per-task table includes a `Sparkline` cell per row.

39/39 cases pass against vitest 4.1.5 + jsdom 29.1.1.

## Adoption sites

The TokenUsage page is the only adopter. No type changes
to the `useTokenUsage` / `useTokenUsageBreakdowns` hooks;
the page maps the new `PeriodOption` string state to the
numeric `days` value before calling the breakdown hook.

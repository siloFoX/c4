## Summary

Add a new `<ChartSpiderWeb>` UI primitive in
`web/src/components/ui/chart-spider-web.tsx` -- pure-SVG
**circular hub-spoke** layout for hub-spoke node
visualization. One hub node anchors the centre; every
remaining node spreads evenly on a perimeter circle.
Hub-spoke edges paint as radial spokes from the hub
outward; spoke-to-spoke edges paint as straight lines
across the chart. Optional `groupRings=true` stacks
spokes on **per-group concentric rings** so nodes in
the same `group` share a radius. Distinct from
`<ChartNetwork>` (11.454, force-directed),
`<ChartChord>` (11.462, chord around a ring),
`<ChartArcDiagram>` (11.463, baseline + arcs),
`<ChartRadar>` (11.445, multi-axis polygon), and
`<ChartRadarSpider>` (11.464, spider variant): this
primitive is specifically the **hub-centric topology**
display where a single node dominates and others orbit.

## API

```ts
interface ChartSpiderWebNode {
  id: string;
  label: string;
  group?: string;
  weight?: number;
  color?: string;
}

interface ChartSpiderWebEdge {
  source: string;
  target: string;
  weight?: number;
  color?: string;
}

interface ChartSpiderWebProps {
  nodes: readonly ChartSpiderWebNode[];
  edges?: readonly ChartSpiderWebEdge[];
  hubId?: string;
  width?: number;             // default 420
  height?: number;            // default 420
  padding?: number;           // default 32
  hubRadius?: number;         // default 12
  spokeRadius?: number;       // default 8 (base)
  startAngle?: number;        // default -pi/2
  hubColor?: string;          // default '#2563eb'
  spokeColor?: string;        // default '#0891b2'
  edgeColor?: string;         // default '#94a3b8'
  ringColor?: string;         // default '#e2e8f0'
  showRings?: boolean;        // default false
  showLabels?: boolean;       // default true
  showTooltip?: boolean;      // default true
  highlightOnHover?: boolean; // default true
  groupRings?: boolean;       // default false
  animate?: boolean;          // default true
  className?: string;
  ariaLabel?: string;         // default 'Spider web chart'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  onNodeClick?: (args: {
    node: ChartSpiderWebNode;
    layout: ChartSpiderWebLayoutNode;
  }) => void;
  onEdgeClick?: (args: {
    edge: ChartSpiderWebEdge;
    layout: ChartSpiderWebLayoutEdge;
  }) => void;
  style?: CSSProperties;
}
```

## Behaviour

- Renders `<div role="region">` wrapping
  `<svg role="img">` with the same `aria-label`.
- Hub selection: `hubId` prop names the centre node;
  if absent or not in `nodes`, the first node in the
  array is treated as the hub.
- Layout: `computeSpiderWebLayout` places the hub at
  `(cx, cy)` and arranges remaining nodes evenly
  around a circle starting at `startAngle` (default
  -pi/2 so the first spoke sits at 12 o'clock).
  Per-node radius is mirrored on the `data-node-r`
  + scaled by `sqrt(weight)`.
- `groupRings=true`: spokes are partitioned by their
  `group` field (in first-seen order). Each group
  gets its own concentric ring; the ring radius
  steps evenly from `outerRadius / N_groups` to
  `outerRadius`. Within a group, members are evenly
  spaced around the ring.
- Edges: hub-spoke edges flagged `isHubSpoke=true`;
  spoke-to-spoke edges flagged false. Both render as
  straight lines whose stroke width scales with
  `edge.weight` (clamped to [0.8, 4] px). Edges
  referencing unknown ids are silently dropped.
- Hover/focus on a node: dims every other unrelated
  node to opacity 0.3 and every unrelated edge to
  opacity 0.18 (when `highlightOnHover=true`).
  Tooltip surfaces label, degree, and group.
- Hover/focus on an edge: keeps the edge fully
  rendered while dimming siblings only if a node is
  also hovered; tooltip surfaces source-target +
  weight.
- Optional concentric dashed rings (`showRings=true`)
  paint at the per-group radii to make the
  hub-spoke geometry obvious.
- Mount animation:
  `motion-safe:animate-fade-in` per node + edge.
  `animate=false` removes the class.
- Default ARIA description: `"Spider web chart
  centered on <hub.label> with N spoke nodes, M
  edges; hub has K connections."` Empty / hub-only
  -> `"No data"`.

```ts
DEFAULT_CHART_SPIDER_WEB_WIDTH         = 420
DEFAULT_CHART_SPIDER_WEB_HEIGHT        = 420
DEFAULT_CHART_SPIDER_WEB_PADDING       = 32
DEFAULT_CHART_SPIDER_WEB_HUB_RADIUS    = 12
DEFAULT_CHART_SPIDER_WEB_SPOKE_RADIUS  = 8

DEFAULT_CHART_SPIDER_WEB_HUB_COLOR     = '#2563eb'
DEFAULT_CHART_SPIDER_WEB_SPOKE_COLOR   = '#0891b2'
DEFAULT_CHART_SPIDER_WEB_EDGE_COLOR    = '#94a3b8'
DEFAULT_CHART_SPIDER_WEB_RING_COLOR    = '#e2e8f0'

polarToCartesian(cx, cy, radius, angle): {x, y}
getSpiderWebNodeWeight(node): number
getSpiderWebGroups(nodes, hubId): string[]
getSpiderWebDegree(nodeId, edges): number
computeSpiderWebLayout(input): {
  hub, spokes, edges, rings: { groups, radii }, outerRadius
}
describeSpiderWebChart(nodes, edges, hubId?): string
```

- `getSpiderWebNodeWeight` returns the `weight`
  field for positive finite values; everything else
  defaults to 1.
- `getSpiderWebGroups` returns unique groups in
  first-seen order, excluding the hub node;
  ungrouped nodes appear as `""`.
- `computeSpiderWebLayout` returns the hub node
  separately (or `null` for empty input / non-
  positive radius), an array of spoke nodes, an
  array of edges projected to coordinates, and the
  ring radii (one per group).

## ARIA + data attributes

- Root: `<div role="region">` + `aria-label`.
- SVG: `role="img"` + same `aria-label`.
- sr-only span:
  `data-section="chart-spider-web-aria-desc"`.
- Each node circle: `role="graphics-symbol"` +
  `tabIndex={0}`. aria-label = `"<label>: degree
  <degree>"` (hub gets `"Hub <label>"` prefix).
- Each edge line: `role="graphics-symbol"` +
  `tabIndex={0}`. aria-label = `"<src> to <tgt>:
  weight <weight>"`.
- `data-section` on every node:
  chart-spider-web / -canvas / -aria-desc / -svg /
  -rings / -ring / -edges / -edge / -nodes / -node /
  -node-circle / -node-label / -tooltip /
  -tooltip-label / -tooltip-degree / -tooltip-group
  / -edge-tooltip / -edge-tooltip-label /
  -edge-tooltip-weight.
- Root mirrors `data-node-count`,
  `data-spoke-count`, `data-edge-count`,
  `data-hub-id`, `data-group-rings`, `data-animate`.
- Node groups mirror `data-node-id`,
  `data-node-index`, `data-node-group`,
  `data-node-color`, `data-node-degree`,
  `data-node-weight`, `data-node-is-hub`,
  `data-hovered`.
- Edge lines mirror `data-edge-index`,
  `data-edge-source`, `data-edge-target`,
  `data-edge-weight`, `data-edge-hub-spoke`,
  `data-hovered`.

## Out of scope

- Multi-hub / star-of-stars topologies (host
  composes multiple instances).
- Edge bundling for dense webs (host pre-bundles).
- Animated transitions on data changes (snap v1;
  mount fade only).
- Pan / zoom (host wraps in its own viewport).
- Drag-to-reorder spokes around the perimeter
  (host re-renders with the new node order).

## Summary

New `ChartLineRvi` UI primitive at
`web/src/components/ui/chart-line-rvi.tsx` -- a pure-SVG line chart
with a Relative Vigor Index oscillator panel and its signal line. The
top panel renders the close line with a shaded high-low band behind
it; the bottom panel renders the RVI line and a dashed signal line on
a symmetric y-axis with a dashed zero centerline and sign-coloured
markers.

This primitive consumes full OHLC points -- `{ x, open, high, low,
close }` -- since the RVI relates the close-open move to the high-low
range.

## Indicator math

John Ehlers' Relative Vigor Index, with an SMA period (default 10):

```
co      = close - open
hl      = high - low
SWMA(s) = (s[i] + 2*s[i-1] + 2*s[i-2] + s[i-3]) / 6     (fixed 1/2/2/1 average)
numerator   = SWMA(co)
denominator = SWMA(hl)
RVI    = SMA(numerator, period) / SMA(denominator, period)
signal = SWMA(RVI)
```

`computeLineRviSwma(src)` is the fixed 4-bar symmetric weighted
average; indices before the fourth bar -- or any window touching a
null -- read null. `computeLineRviSma(src, period)` is a simple moving
average that tolerates leading nulls. A zero denominator reads RVI 0.

The RVI rises when price closes far above where it opened relative to
the bar's range, and falls when it closes far below; a cross of the
RVI and its signal line marks a shift in vigor.

### Worked fixture

`open` 10, `high` 30, `low` 6 throughout; `close` 16 (bars 0-5) or 22
(bars 6-9); `period = 2`:

```
co          = [6,6,6,6,6,6,12,12,12,12]
hl          = [24,24,24,24,24,24,24,24,24,24]
numerator   = SWMA(co) = [.,.,.,6,6,6,7,9,11,12]
denominator = SWMA(hl) = [.,.,.,24,24,24,24,24,24,24]
rvi    = SMA(num,2)/SMA(den,2) = [.,.,.,.,0.25,0.25,6.5/24,8/24,10/24,11.5/24]
signal = SWMA(rvi) = [.,.,.,.,.,.,.,39/144,0.3125,0.375]
```

`rviMin = 0.25`, six readings above zero and none below.

## API

- `getLineRviFinitePoints(points)` -- drop points with a non-finite
  `x`, `open`, `high`, `low` or `close`.
- `normalizeLineRviPeriod(period, fallback)` -- positive integer;
  non-finite / sub-1 falls back, fractional floors.
- `computeLineRviSwma(src)` -- the fixed 1/2/2/1 symmetric weighted
  average.
- `computeLineRviSma(src, period)` -- the null-tolerant SMA.
- `computeLineRvi(opens, highs, lows, closes, period)` -- returns
  `{ co, hl, numerator, denominator, rvi, signal }`.
- `runLineRvi(points, options?)` -- sort finite points by `x`, compute
  the series, return `{ series, period, co, hl, numerator,
  denominator, rvi, signal, samples, rviFinal, signalFinal, rviMin,
  rviMax, positiveCount, negativeCount, ok }`. Each sample carries a
  `positive` / `negative` / `zero` sign from the RVI. `ok` is false
  for fewer than two points.
- `computeLineRviLayout(options)` -- stack a price panel above an RVI
  panel with a symmetric y-bound. Returns panels, ticks, the close
  path, the high-low band path, the RVI path, the signal path,
  sign-coloured markers, and the zero centerline `y`.
- `describeLineRviChart(data, options?)` -- ARIA description.

## Component

`ChartLineRvi` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge (SMA period), a three-series legend (Close / RVI / Signal) with
toggle, a shaded high-low band, the close line, a dashed zero
centerline, the RVI line, a dashed signal line, markers coloured by
sign, a hover/focus tooltip exposing the open, high, low, close, RVI,
signal, and sign, and a `motion-safe:animate-fade-in` entrance. Every
node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-stochastic / chart-line-williams-r** -- those
  oscillators measure where the *close sits* within the high-low
  range. The RVI instead compares the close-open *move* to the range:
  it asks how far price travelled from open to close, not where the
  close landed. The RVI also runs everything through the 1/2/2/1
  symmetric weighted average and carries a signal line.
- **vs chart-line-mass-index / chart-line-emv** -- those also use
  high-low data, but the Mass Index sums an EMA ratio and the EMV
  divides the midpoint move by volume. The RVI is the only chart-line
  primitive built on the close-open span, and the only one consuming
  full OHLC `{ x, open, high, low, close }`.

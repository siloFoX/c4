## Summary

New `ChartLineCoppock` UI primitive at
`web/src/components/ui/chart-line-coppock.tsx` -- a pure-SVG line chart
with a Coppock Curve long-term momentum panel. The value line occupies
the top panel; the bottom panel renders the single Coppock Curve line
on a symmetric y-axis with a dashed zero centerline and sign-coloured
markers.

## Indicator math

The Coppock Curve, devised by economist Edwin Coppock, is a long-term
momentum oscillator:

```
Coppock = WMA(wmaPeriod, ROC(roc1Period) + ROC(roc2Period))
```

with defaults `roc1Period = 14`, `roc2Period = 11`, `wmaPeriod = 10`.

`computeLineCoppockRoc(values, period)` is the percentage rate of
change over the lookback:

```
ROC[i] = base === 0 ? 0 : 100 * (values[i] - base) / base
base   = values[i - period]
```

Indices before the lookback read `null`; a zero base reads `0`. A
negative-zero result (a zero numerator over a negative base) is
normalised to positive zero.

`computeLineCoppockWma(src, period)` is a linearly weighted moving
average: across the `period`-value window the oldest value carries
weight `1` and the newest weight `period`, divided by the triangular
weight sum `period * (period + 1) / 2`. Any window touching a `null`
reads `null`, so the curve only begins once both ROC legs and a full
WMA window are defined.

`computeLineCoppock` sums the two ROC series bar by bar (`null` where
either leg is undefined) and smooths the running total with the WMA.
The curve oscillates around zero; a cross up through the zero line is
the classic long-term momentum buy signal.

### Worked fixture

`values = [100, 100, 100, 100, 100, 130, 160]`, `roc1Period = 3`,
`roc2Period = 2`, `wmaPeriod = 2`:

```
roc1    = [.,    .,    .,    0,  0,  30,  60]
roc2    = [.,    .,    0,    0,  0,  30,  60]
rocSum  = [.,    .,    .,    0,  0,  60, 120]
coppock = [.,    .,    .,    .,  0,  40, 100]
```

`coppockFinal = 100`, `coppockMin = 0`, `coppockMax = 100`,
`positiveCount = 2`, `negativeCount = 0`.

## API

- `getLineCoppockFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineCoppockPeriod(period, fallback)` -- positive integer;
  non-finite / sub-1 falls back, fractional floors.
- `computeLineCoppockRoc(values, period)` -- the percentage
  rate-of-change series.
- `computeLineCoppockWma(src, period)` -- the linearly weighted moving
  average series.
- `computeLineCoppock(values, roc1Period, roc2Period, wmaPeriod)` --
  the full Coppock series.
- `runLineCoppock(points, options?)` -- sort finite points by `x`,
  compute the Coppock series, return `{ series, roc1Period,
  roc2Period, wmaPeriod, coppock, samples, coppockFinal, coppockMin,
  coppockMax, positiveCount, negativeCount, ok }`. Each sample carries
  a `positive` / `negative` / `zero` sign. `ok` is false for fewer
  than two points.
- `computeLineCoppockLayout(options)` -- stack a value panel above a
  Coppock panel with a symmetric y-bound. Returns panels, ticks, the
  value path, the Coppock path, sign-coloured markers, and the zero
  centerline `y`.
- `describeLineCoppockChart(data, options?)` -- ARIA description.

## Component

`ChartLineCoppock` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge (ROC and WMA periods), a two-series legend (Value / Coppock)
with toggle, a dashed zero centerline, the Coppock line, markers
coloured by sign (green above zero, red below), a hover/focus tooltip
exposing the value, Coppock reading, and sign, and a
`motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs chart-line-rate** -- chart-line-rate plots a single raw ROC
  line straight off one lookback. The Coppock Curve sums *two* ROC
  legs of different lookbacks and then smooths that total with a
  linearly weighted moving average; the smoothing and the dual-leg
  sum are exactly what make it a long-term signal rather than a noisy
  bar-to-bar reading.
- **vs chart-line-trix** -- TRIX is the rate of change of a *triple*
  exponentially smoothed average. The Coppock Curve never triple-
  smooths and uses a *weighted* (linear-weight) moving average, not an
  EMA, applied to a *sum of two ROCs*. The two oscillators share no
  computation step.
- **vs chart-line-momentum / chart-line-cmo / chart-line-tsi** --
  those are single-stage momentum measures; the Coppock Curve is
  uniquely the WMA-smoothed sum of two ROC legs.

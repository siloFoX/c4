## Summary

`ChartLinePivotCamarilla` is a pure-SVG single-panel React primitive
that overlays Camarilla pivot levels on the close. For each bar `i`
the levels are derived from the **prior** bar's H, L, and C:

```
pivot[i] = (H + L + C) / 3
R1[i]    = C + (H - L) * 1.1 / 12
R2[i]    = C + (H - L) * 1.1 / 6
R3[i]    = C + (H - L) * 1.1 / 4
R4[i]    = C + (H - L) * 1.1 / 2
S1[i]    = C - (H - L) * 1.1 / 12
S2[i]    = C - (H - L) * 1.1 / 6
S3[i]    = C - (H - L) * 1.1 / 4
S4[i]    = C - (H - L) * 1.1 / 2
```

Bar 0 has no prior bar so all levels are `null`.

The component renders ten series on the same Y-axis: the close, the
pivot, the four resistance lines (R1..R4, dashed), and the four
support lines (S1..S4, dashed). Each can be hidden independently via
the legend; `showResistance` and `showSupport` group-toggle the
respective bundles.

## Bit-exact anchor

For **CONST_FLAT** (`H = L = C = K`): the prior bar's range is zero,
so every R/S offset vanishes. `pivot = (K + K + K) / 3 = K` (the
divide-by-3 is FP-safe because the integer summands collapse). Every
R_n and S_n offset multiplies by `(H - L) = 0` and therefore equals
zero exactly, so:

```
pivot = R1 = R2 = R3 = R4 = S1 = S2 = S3 = S4 = K
```

bit-exact past bar 0. The integration sweep verifies this across
`K` in `{0, 1, 5, 100, -3, 7, -50}`.

For non-flat constant-bar inputs the levels are only FP-soft (1.1 and
the divisors do not reduce to dyadics), so those checks use
`toBeCloseTo` to 9 decimal places.

## Zone classifier

- `breakout-up` -- close >= R4
- `above-pivot` -- pivot < close < R4
- `at-pivot` -- close == pivot
- `below-pivot` -- S4 < close < pivot
- `breakout-down` -- close <= S4
- `none` -- any of pivot / R4 / S4 is null

Singular fallback: when `R4 == pivot == S4` (zero-width stack from
CONST_FLAT), the classifier collapses to a pivot comparison so a
flat bar reads `at-pivot` instead of triggering both breakout
buckets.

## Out of scope

- No new dependencies.
- No canvas, no third-party chart libs.
- This primitive ships the standard Camarilla equation
  (H/L/C from prior bar). Daily-anchored Camarilla bands (where
  the prior bar is the prior day instead of the prior intraday
  bar) require external time bucketing and are not in scope.

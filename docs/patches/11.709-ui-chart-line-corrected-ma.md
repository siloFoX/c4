## Summary

New `ChartLineCorrectedMa` UI primitive in
`web/src/components/ui/chart-line-corrected-ma.tsx` -- a pure-SVG
single-panel line chart with a **Corrected Moving Average** overlay that
iterates until the average sits inside its variance band. The price line
carries the corrected line and a shaded variance band.

## The computation

The Corrected Moving Average smooths the price and then "corrects" the
result toward the simple moving average until it lands inside the
variance band. `ChartLineCorrectedMaPoint` carries `{x, value}`.

- `computeLineCorrectedMaSma(values, period)` -- the simple moving
  average over the window, null in the warm-up.
- `computeLineCorrectedMaVariance(values, period)` -- the population
  variance over the same window (the mean squared deviation).
- `correctLineCorrectedMa(prevCma, sma, variance)` -- corrects one bar:
  starting from `prevCma`, it halves the distance to `sma` each
  iteration until `(value - sma)^2 <= variance` (the variance band, one
  standard deviation around the average), returning `{cma, iterations}`.
  A non-positive variance collapses the band to the average itself.
- `computeLineCorrectedMa(values, period)` -- runs the pipeline bar by
  bar, seeding the corrected line from the average on its first defined
  bar and carrying it forward.
- `classifyLineCorrectedMaTrend(cma, prevCma)` -- marks a bar **up** /
  **down** / **flat** by the slope of the corrected line.

## Worked fixture

The general correction iterates a transcendental count of times, but the
indicator has clean exact anchors.

A series **periodic with the window length** -- `[10,20,30,40]` repeated,
period 4 -- makes every window a rotation of the same set, so the moving
average is a constant 25 and the variance a constant 125. The corrected
line is seeded at 25 and each later bar finds the prior corrected value
(25) already exactly at the band centre, so it pins to 25 with **zero**
corrections (`cma` and `iterations` both asserted with `toEqual`). A
constant series has zero variance, collapsing the band to the average,
and is reproduced exactly. The `correctLineCorrectedMa` helper is checked
directly: a value already inside the band is returned unchanged with
zero iterations; a far value (200 against an average of 5) is proven to
iterate and land inside the band; the result never moves past the
average. The moving average and variance are verified against exact
integer fixtures.

## API

`data` (a `ChartLineCorrectedMaPoint[]`), `period` (default 20),
sizing/format/a11y props, `strokeWidth` / `dotRadius`, the colour props,
the toggle flags (`showAxis` / `showGrid` / `showDots` default false /
`showCma` / `showBand` / `showMarkers` / `showTooltip` /

uncontrolled `hiddenSeries` (keys `price` / `cma` / `band`),
`formatValue` / `formatX`, `onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineCorrectedMaFinitePoints`,
`normalizeLineCorrectedMaPeriod` (coerces the period to an integer of at
least 2), `computeLineCorrectedMaSma`, `computeLineCorrectedMaVariance`,
`correctLineCorrectedMa`, `computeLineCorrectedMa`,
`classifyLineCorrectedMaTrend`, `runLineCorrectedMa`,
`computeLineCorrectedMaLayout`, `describeLineCorrectedMaChart`, plus the
`DEFAULT_CHART_LINE_CORRECTED_MA_*` constants,
`CHART_LINE_CORRECTED_MA_MAX_ITERATIONS` and the
`ChartLineCorrectedMaTrend` / `ChartLineCorrectedMaCorrection` /
`ChartLineCorrectedMaComputed` types. Every exported name carries a
`LineCorrectedMa` infix so the `export *` barrel line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen
`data-section="chart-line-corrected-ma-aria-desc"` description span; the
price line, the corrected line and every dot / marker are
`role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-period`, `data-cma-final`,
`data-up-count`, `data-down-count`, `data-flat-count`,
`data-total-points` and `data-animate`.

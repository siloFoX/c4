# TODO 11.501 - chart-line-zoom primitive

Status: SHIPPED v1.11.519 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartLineZoom>`, a pure-SVG multi-series line chart with
**brush-to-zoom** interaction. Drag horizontally across the inner
plot to define an x range; on release, the chart zooms into the
selected window. A "Reset zoom" button appears at the top-right while
zoomed; double-clicking the plot also resets. Series strokes are
clipped to the inner area via `<clipPath>` so partial points outside
the visible window crop cleanly.

## API

```tsx
<ChartLineZoom
  series={[
    {
      id: 'cpu',
      label: 'CPU',
      data: [{x: 0, y: 12}, {x: 25, y: 32}, {x: 50, y: 28}, {x: 75, y: 45}, {x: 100, y: 30}],
    },
  ]}
  onZoomChange={(range) => console.log('zoom:', range)}
/>
```

## Distinct from prior chart-* primitives

- `<ChartLine>` - static range; no brush interaction.
- `<ChartLineMulti>` (11.500) - synchronised crosshair tooltip;
  hover only.
- `<ChartLineStep>` (11.497) - step interpolation; no brush.
- `<ChartLineSmooth>` (11.498) - Bezier / monotone curves; no brush.
- `<ChartLineDashed>` (11.499) - dash patterns; no brush.

The brush-to-zoom primitive owns the canonical drag-select pattern
adopters reach for in time-series dashboards (Grafana / Datadog
range zoom).

## Brush contract

### Lifecycle

1. **Pointer down** on the overlay rect (left mouse button or any
   pointer) starts a brush. Internal state captures `startPx =
   pointerLocalPx(clientX)` clamped to the inner plot bounds.
   `setPointerCapture` is called so the brush keeps tracking even if
   the cursor leaves the chart.
2. **Pointer move** updates `endPx`; the brush rect renders between
   `min(startPx, endPx)` and `max(...)` so left-to-right and
   right-to-left drags both work.
3. **Pointer up** commits the brush if
   `isLineZoomBrushValid(startPx, endPx, minBrushWidth)` is true
   (i.e. the drag distance >= `minBrushWidth` pixels, default 4).
   Narrower drags are treated as a click and discarded.
4. **Pointer cancel** aborts the in-flight brush without committing.

### Commit

The committed range goes through `clampLineZoomRange(range, fullXMin,
fullXMax)`:

- Normalises reversed inputs (`xMin > xMax` -> swap).
- Clamps both endpoints to the full data bounds.
- Returns `null` for degenerate ranges (xMin >= xMax after clamping).

A valid range goes into internal state (uncontrolled) or fires
`onZoomChange(range)` (controlled). `onZoomChange(null)` fires on
reset.

### Reset

- The "Reset zoom" button (top-right of the canvas) is rendered when
  `data-zoom-active="true"` and `showResetButton !== false`. Click
  -> `applyZoom(null)`.
- Double-click on the overlay also resets when a zoom is active.

## Layout

`computeLineZoomLayout(input)` accepts an optional `zoom` range:

- If `zoom` is null / non-finite / degenerate, the layout uses the
  full data range. `zoomActive` is false.
- If `zoom` is valid, it overrides `xMin` / `xMax`. y-bounds are
  unchanged so vertical context stays steady while the user pans
  through x.
- `fullXMin` / `fullXMax` are always exposed so consumers can detect
  when a zoom is active and compute relative ratios.

Per-series paths use `buildLineZoomPath` (simple `M x y L x y ...`).
Each series's `<g>` is wrapped by `clip-path: url(#clipId)` so the
line strokes cleanly at the edge of the inner plot area.

## Exported helpers

- `DEFAULT_CHART_LINE_ZOOM_*` numeric / string defaults.
- `DEFAULT_CHART_LINE_ZOOM_PALETTE` (10-colour palette).
- `getLineZoomDefaultColor(index)` - palette wrap.
- `getLineZoomFinitePoints(points)` - drops non-finite.
- `getLineZoomBounds(series, hidden?)` - per-axis bounds.
- `getLineZoomTicks(min, max, count?)` - evenly-spaced ticks.
- `buildLineZoomPath(points)` - simple `M x y L x y ...` builder.
- `clampLineZoomRange(range, fullXMin, fullXMax)` - normalise +
  clamp + reject degenerate.
- `isLineZoomBrushValid(startPx, endPx, minBrushWidth?)` -
  pixel-width threshold check.
- `computeLineZoomLayout(input)` - canonical layout returning
  per-series points + paths plus `xMin`, `xMax`, `fullXMin`,
  `fullXMax`, `zoomActive`.
- `describeLineZoomChart(series, hidden?, zoom?, formatValue?)`
  - ARIA summary mentions the brush-to-zoom hint when no zoom is
  active, or the zoomed range when one is.

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only
  `data-section="chart-line-zoom-aria-desc"` span.
- SVG `role="img"` with the same `aria-label`.
- Each path is `role="graphics-symbol"` with `tabIndex={0}` and
  `aria-label="<series>: line with K points"`.

## Data attributes

- Root: `data-section="chart-line-zoom"`, `data-series-count`,
  `data-visible-series-count`, `data-total-points`,
  `data-zoom-active`, `data-zoom-min`, `data-zoom-max`,
  `data-full-x-min`, `data-full-x-max`, `data-brushing`,
  `data-animate`.
- Series group + path + dot: standard pattern with `data-series-id`,
  `data-series-index`, `data-series-color`, `data-point-index`,
  `data-x`, `data-y`.
- Brush rect: `data-section="chart-line-zoom-brush-rect"`,
  `data-brush-start`, `data-brush-end`.
- Overlay: `data-section="chart-line-zoom-overlay"` - transparent
  pointer-capture rect.
- Reset button: `data-section="chart-line-zoom-reset"`.
- Tooltip, legend, axes, grid: standard pattern with `data-axis="x"`
  or `"y"` on ticks / grid lines.

## Out of scope

- Y-axis brush (only x); the conventional dashboard pattern brushes
  on the time axis.
- Wheel zoom / pinch zoom - explicit drag-select v1.
- Pan-while-zoomed (drag the existing zoom box) - reset + re-brush.
- Synchronised crosshair tooltip while zoomed - use
  `<ChartLineMulti>` for that pattern.
- Animated zoom transitions - snap; mount fade only.
- Log-scale axes - linear only.

# Patch 11.923 -- UI: chart-line-stoch-overbought-cross primitive

## Summary

Added `<ChartLineStochOverboughtCross />` -- pure-SVG dual-panel
React/TS primitive that renders the close (top panel) with
bullish (entry) / bearish (exit) arrow overlays at every
Stochastic %K threshold cross trigger, and the close-only %K
line (bottom panel) on a fixed 0-100 oscillator with 50 / 80
reference bands. Single-threshold cross variant of the
Stochastic family that flags the discrete %K level-80 entry
and exit events distinct from the canonical %K / %D crossover.

## Algorithm

```
range[i] = max(close[i-n+1..i]) - min(close[i-n+1..i])
rawK[i]  = range > 0
            ? ((close - lowestClose) / range) * 100
            : 50
k[i]     = SMA(rawK, kSmoothing)
bullish : prev <= threshold && cur > threshold  (entry)
bearish : prev >= threshold && cur < threshold  (exit)
```

Defaults `length = 14` (canonical Stochastic window),
`kSmoothing = 3` (slow %K smoothing), `threshold = 80`
(overbought level). Regime classifier `bullish` (k >=
threshold, overbought), `neutral` (k < threshold), `none`
(k null).

## Bit-exact anchor

CONST close = K: range = 0 every bar -> rawK = 50 neutral
fallback. SMA of 50s = 50 via the `min === max` short-circuit.
50 < 80, so regime is `neutral` and the threshold is never
crossed. cross count = 0. Verified across K = 0, 1, 5, 17,
100, 1234.

LINEAR UP yields %K = 100 every settled bar (close at top of
rolling range) -> regime `bullish`. LINEAR DOWN yields %K = 0
(close at bottom of range) -> regime `neutral` (below
threshold).

## Summary

`web/src/components/ui/chart-line-butterworth.tsx` ships
`<ChartLineButterworth>`, a pure-SVG single-panel chart with the
**Ehlers Butterworth Filter** overlay -- a maximally-flat low-pass
IIR with selectable 2- or 3-pole order. The price line and the filter
line share one panel; the filter line is segment-coloured by its own
slope.

## Computation

The filter is a recursive difference equation. For the 2-pole order
(the default):

```
a   = exp(-sqrt(2) * pi / period)
b   = 2 * a * cos(sqrt(2) * pi / period)
c2  = b
c3  = -a^2
c1  = (1 - b + a^2) / 4
Filt[i] = c1 * (P[i] + 2*P[i-1] + P[i-2])
        + c2 * Filt[i-1] + c3 * Filt[i-2]
```

For the 3-pole order:

```
a   = exp(-pi / period)
b   = 2 * a * cos(1.738 * pi / period)
c   = a^2
c2  = b + c
c3  = -(c + b * c)
c4  = c^2
c1  = (1 - c2 - c3 - c4) / 8
Filt[i] = c1 * (P[i] + 3*P[i-1] + 3*P[i-2] + P[i-3])
        + c2 * Filt[i-1] + c3 * Filt[i-2] + c4 * Filt[i-3]
```

The first `poles` bars are seeded at the input value so the filter is
defined from bar 0; the recursion starts at index `poles`.

The defining algebraic invariant: the sum of the input-window
coefficients plus the feedback coefficients is exactly one. For
2-pole `4*c1 + c2 + c3 = 1` (by construction of `c1`); for 3-pole
`8*c1 + c2 + c3 + c4 = 1`. This is **DC gain unity**: a constant
input is preserved exactly.

## Worked anchors

The Butterworth coefficients are transcendental in the period, so a
non-zero constant cannot be expected to propagate with bit-exact
arithmetic. The fixture splits two ways:

- **CONST_ZERO** -- `[0 x 10]`. IEEE 754 guarantees that any finite
  `c * 0 = 0`, so every recursion step on a zero input produces a
  bit-exact zero output regardless of the coefficient values. The
  output is `[0, 0, ..., 0]` exactly. Asserted across `period in
  {3, 4, 10, 20}` and `poles in {2, 3}` -- 8 combinations, all
  `toEqual`.
- **CONST_FLAT** -- `[5 x 10]`. The DC-invariance still holds but
  rounding noise from the trigonometric coefficients accumulates
  through the recursion; each output is within `1e-10` of 5. Same
  8 combinations.
- **Analytic coefficient sum** -- `4*c1 + c2 + c3` is asserted close
  to 1 within `1e-10` for 2-pole; `8*c1 + c2 + c3 + c4` for 3-pole.
- **Seeding** -- the first 2 outputs equal the first 2 inputs for
  2-pole; the first 3 outputs equal the first 3 inputs for 3-pole.
- **RISING** (length 20, `[1..20]`) -- the 2-pole filter is strictly
  rising after the warm-up (`out[i] > out[i-1]` for `i >= 2`). For
  3-pole there is a brief transient where the seeded outputs are
  ahead of the steady-state ramp, so only the end-to-end trend is
  asserted (`out[end] > out[0]`).

## Distinct from neighbouring primitives

- vs `<ChartLineMovingAvg>` / a plain SMA -- the SMA is a single
  flat-weighted FIR with no recursion; the Butterworth is an IIR
  whose poles are placed on a Butterworth template (maximally flat
  passband) for a steeper roll-off and a flatter passband.
- vs `<ChartLineGauss>` -- the Ehlers Gaussian Filter is an N-pole
  CASCADE of one-pole EMAs whose alpha is tuned so the cumulative
  frequency response approximates a Gaussian; the Butterworth is a
  DIRECT-FORM IIR built from a single high-order transfer function.
  The Butterworth has a flatter passband; the Gaussian has a longer
  but smoother step response.
- vs `<ChartLineEma>` -- the EMA is a one-pole IIR; the Butterworth
  is the two- or three-pole generalisation on a Butterworth template
  (rather than a single-pole).
- vs `<ChartLineCoral>` / Tillson T3 -- the Coral injects LEAD via
  negative-weighted recombination of a six-pole cascade; the
  Butterworth is the pure low-pass cousin with no lead correction.

## API

`data` -- a single `ChartLineButterworthPoint[]` (`{x, value}`).
`period` (default 15, coerced to an integer >= 2). `poles` (default
2, coerced to 2 or 3 only). Standard sizing / colour / a11y props
plus `strokeWidth`, `dotRadius`, `tickCount`. Toggle flags:
`showAxis`, `showGrid`, `showDots` (default false), `showFilter`,
`showMarkers`, `showTooltip`, `showConfigBadge`, `showLegend`,
`animate`. Controlled and uncontrolled visibility via `hiddenSeries`
(`'price' | 'filter'`). Callbacks: `onPointClick({point})`,
`onSeriesToggle({seriesId, hidden})`. Format functions: `formatValue`,
`formatX`. ARIA: `ariaLabel`, `ariaDescription`.

## Pure helpers

- `DEFAULT_CHART_LINE_BUTTERWORTH_*` -- defaults (period 15, poles 2,
  etc.).
- `getLineButterworthFinitePoints` -- keep only points with a finite
  x and value.
- `normalizeLineButterworthPeriod(period, fallback)` -- integer >= 2;
  else fallback.
- `normalizeLineButterworthPoles(poles, fallback)` -- 2 or 3 only;
  else fallback.
- `computeLineButterworthCoefficients(period, poles)` -- the Ehlers
  coefficient triple (or quadruple). For 2-pole, `c4 = 0`. For 3-pole,
  all four are non-zero. The DC gain is exactly one.
- `computeLineButterworth(values, period, poles)` -- the difference
  equation. Non-array input returns `[]`. The first `poles` outputs
  equal the first `poles` inputs.
- `classifyLineButterworthSlope(filter, prevFilter)` --
  `'up' | 'down' | 'flat' | 'none'` by slope.
- `runLineButterworth(points, options?)` --
  `{series, period, poles, coefficients, filter, samples,
  filterFinal, upCount, downCount, flatCount, ok}`.
- `computeLineButterworthLayout({data, period, poles, ...})` --
  single panel with the price line, one segment between each pair of
  bars (slope-coloured), one marker per bar, and a value domain that
  spans both the price and the filter. Degenerate canvas or fewer
  than 2 points -> `ok = false`.
- `describeLineButterworthChart(data, options?)` -- ARIA summary
  mentioning the indicator, the pole order, the period, the DC
  passthrough property and the slope counts.

## a11y / data hooks

Root `<div role="region">` with `aria-label` + `aria-describedby`; the
SVG is `role="img"` with the same label; an sr-only
`data-section="chart-line-butterworth-aria-desc"` span carries the
description. The price line, every price dot and every marker carry
`role="graphics-symbol"` and `tabIndex={0}`. Every node carries a
`data-section`. Root mirrors `data-period`, `data-poles`,
`data-filter-final`, `data-up-count`, `data-down-count`,
`data-flat-count`, `data-total-points`, `data-animate`, `data-empty`.
Segments and markers expose `data-slope`; markers also expose
`data-filter`.

# 11.577 -- web/src/components/ui/chart-line-rsi.tsx

Pure-SVG line chart with a **Relative Strength Index (RSI)**
oscillator sub-panel. The price line sits in the top panel;
the bottom panel renders the RSI line bounded to 0-100, with
shaded **overbought** and **oversold** zones.

## The RSI computation

`computeLineRsi(values, period)` is Welles Wilder's RSI:

1. Period-over-period changes are split into **gains**
   (`max(delta, 0)`) and **losses** (`max(-delta, 0)`).
2. The average gain and average loss are **Wilder-smoothed**
   -- the first average is the simple mean of the first
   `period` changes, then `avg = (avg * (period - 1) +
   change) / period`.
3. `RSI = 100 - 100 / (1 + avgGain / avgLoss)`, bounded to
   [0, 100].

Three edge cases are handled deterministically: an
all-gain window reads **100**, an all-loss window reads
**0**, and a flat window reads **50**. Indices before the
window is full are `null`.

`runLineRsi` classifies each reading into a **zone**:
`overbought` at or above the overbought threshold (default
70), `oversold` at or below the oversold threshold (default
30), `neutral` between.

The component fixture (period 2) rises 10 -> 30 then falls
hard to 5: the RSI is `[null, null, 100, 33.333, 25]`, which
visits the overbought, neutral and oversold zones in turn --
one overbought reading, one oversold.

## Distinct from neighbouring primitives

RSI is a bounded 0-100 momentum oscillator built from the
ratio of Wilder-smoothed gains to losses, with shaded
overbought / oversold zones. `chart-line-macd` is an
unbounded momentum study (a difference of EMAs);
`chart-line-momentum` is a derivative oscillator;
`chart-line-zscore` is a rolling standardised score. None
compute the gain/loss ratio, the 0-100 bound or the
overbought/oversold zoning.

All exported helpers carry a `LineRsi` infix so the
`export *` barrel line cannot collide.

## Public surface

- `<ChartLineRsi data={...} />` -- a single
  `ChartLineRsiPoint[]` (`{x, value}`).
- `period` -- the RSI period (default 14).
- `overbought` / `oversold` -- the zone thresholds (defaults
  70 / 30; clamped to [0, 100]).
- `pricePanelRatio` -- the fraction of the height given to
  the price panel (default 0.58; clamped to [0.3, 0.8]).
- `gap` -- the pixel gap between the stacked panels.
- Toggle flags: `showAxis`, `showGrid`, `showDots` (default
  false), `showRsi`, `showZones`, `showTooltip`,
  `showConfigBadge`, `showLegend`, `animate`.
- Controlled + uncontrolled visibility (`hiddenSeries` keys
  `price` / `rsi` / `zones`).
- Standard sizing + format + a11y props. `formatValue` /
  `formatX`.
- `onPointClick({point})` / `onSeriesToggle({seriesId,
  hidden})`.

## Pure helpers

- `getLineRsiFinitePoints(points)`.
- `normalizeLineRsiPeriod(period, fallback)`.
- `computeLineRsi(values, period)` -- the Wilder RSI series.
- `runLineRsi(points, options?)` -- the RSI plus the
  per-sample zone classification.
- `computeLineRsiLayout({...})` -- stacks the price panel
  above the RSI panel; the RSI panel y axis is fixed 0-100.
- `describeLineRsiChart(data, options?)`.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- The price line, every price dot and every RSI marker
  `role="graphics-symbol"` + `tabIndex={0}`.
- Every node carries `data-section="chart-line-rsi-..."`.
- Root mirrors `data-period`, `data-overbought-count`,
  `data-oversold-count`, `data-total-points`, `data-animate`.
- Axis / tick / grid / panel-label nodes carry
  `data-panel="price"|"rsi"`. RSI markers expose `data-rsi`
  and `data-zone`; threshold lines carry `data-kind`.

## Tooltip / badge / legend

Hovering a price dot or an RSI marker shows x, price, the
RSI value and the zone. Config badge: `RSI p=<period>
ob=<overbought> os=<oversold>`. The legend has three toggle
items (Price / RSI / Zones).

# 11.513 -- web/src/components/ui/chart-line-moving-avg.tsx

Pure-SVG multi-series line chart with a **simple moving average
overlay** drawn on top of each series. Every series gets a raw line
(dimmed to opacity 0.55 by default so the smoothed line reads as the
emphasized signal) plus a thicker SMA rule whose path breaks at every
index where the SMA is not defined.

## Distinct from other line + reference primitives

- `<ChartLineTrend>` (11.512): one **global** linear regression rule
  per series (a straight line). SMA tracks local structure.
- `<ChartLineSmooth>` (11.498): visual curve interpolation
  (Catmull-Rom / monotone). It does not average -- it just smooths the
  path.
- `<ChartLineBaseline>` (11.510), `<ChartLineThreshold>` (11.503):
  horizontal references.
- `<ChartLineConfidence>` (11.502): per-point bounds.
- `<ChartLineMulti>` (11.500), `<ChartLineDashed>` (11.499),
  `<ChartLineMarker>` (11.506): raw lines, no derived overlay.

The moving average overlay is **always** an arithmetic mean of a
sliding window of finite y values, and is always rendered as a
separate path with its own colour + dash. Adopters can switch
modes (`trailing` / `centered` / `edge`) per series.

## Public surface

- `<ChartLineMovingAvg series={...} window={5} mode='trailing' />`.
- Per-series `{id, label, data, color?, maColor?, maDashArray?,
  window?, mode?, hideMa?}`.
- Per-series window/mode/colour/dash always beats chart-level
  defaults.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showMa`, `showMaInTooltip`, `animate`.
- Bounds overrides; tick count; per-line + per-MA stroke widths;
  separate `lineOpacity` (default 0.55) and `maOpacity` (default 1).
- Controlled + uncontrolled visibility via `hiddenSeries` /
  `defaultHiddenSeries` / `onHiddenSeriesChange`.
- Callbacks: `onPointClick({series, point})`, `onSeriesToggle({series,
  hidden})`.

## Pure helpers

- `getLineMovingAvgDefaultColor(index)` -- 10-color palette cycle.
- `getLineMovingAvgFinitePoints(points)` -- finite-only filter.
- `normaliseLineMovingAvgWindow(window)` -- positive integer >= 1.
  Non-finite, non-numeric, <1, and string inputs all clamp to 1
  (passthrough).
- `computeSimpleMovingAverage(values, window, mode?)` -- canonical
  SMA helper. Returns a same-length `(number | null)[]`. Non-finite
  entries are dropped from the moving sum; an output index is `null`
  only when the window cannot be filled (per mode) or every sample in
  the window is non-finite.
- `buildLineMovingAvgPath(points)` -- builds a path that emits one
  contiguous `M ... L ...` segment per run of consecutive non-null
  values, separated by spaces. Single-point runs emit only the `M`
  (so they still paint with `stroke-linecap='round'`).
- `computeLineMovingAvgLayout({series, hiddenSeries?, defaultWindow?,
  defaultMode?, ...})` -- returns `{series, xTicks, yTicks, xMin,
  xMax, yMin, yMax, innerWidth, innerHeight, totalPoints,
  visibleSeriesCount}`. Each layout series carries
  `{id, label, index (stable original index), color, maColor,
  maDashArray, window, mode, hideMa, points, path, maPath,
  maValidCount, finiteCount, totalCount}`. Per-point
  `{index, x, y, px, py, ma, maPy}`. y bounds are expanded to include
  the MA values so an SMA that overshoots / undershoots remains
  visible.
- `describeLineMovingAvgChart(series, hidden?, defaultWindow?)` --
  builds the screen-reader description with per-series window.

## SMA modes

- **trailing** (default): `out[i] = mean(values[i - W + 1 .. i])` for
  `i >= W - 1`; otherwise `null`.
- **centered**: `out[i] = mean(values[i - half .. i + (W - 1 - half)])`
  where `half = floor((W - 1) / 2)`. Only valid when both ends fit
  inside the array. Even windows are biased toward leading samples to
  match standard centered-SMA conventions.
- **edge**: same as trailing but the leading positions fill the
  available samples (partial-window average) so the smoothed line is
  drawn over every index. Useful when the leading nulls would
  otherwise be misread.

`window <= 0`, non-finite, or non-numeric window arguments normalise to
`1` (passthrough). Window strictly larger than the input length produces
all-`null` output under `trailing` and `centered`.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Per-series raw line, MA line, and per-dot are
  `role="graphics-symbol"` with `tabIndex={0}`. MA aria-label includes
  the window size and valid-sample count.
- Every node carries `data-section="chart-line-moving-avg-..."` for
  test selectors.
- Series groups expose `data-series-id`, `data-series-color`,
  `data-series-ma-color`, `data-series-window`, `data-series-mode`,
  `data-series-point-count`, `data-series-finite-count`,
  `data-series-ma-valid-count`.

## Tooltip

Hovering a dot shows the series label, `x:`, `y:`, and an `SMA(W):`
row with the moving-average value at that index (or `n/a` when the
SMA is not defined). Hide via `showMaInTooltip=false`.

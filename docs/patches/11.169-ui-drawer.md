## Component contract

```ts
interface DrawerProps {
  open: boolean;
  onOpenChange: (open: boolean) => void;
  side?: 'left' | 'right'; // default 'right'
  width?: number | string;  // default '320px'
  title?: ReactNode;        // rendered as <h2>, drives aria-labelledby
  description?: ReactNode;  // subtitle, drives aria-describedby
  showCloseButton?: boolean;     // default true
  closeOnBackdropClick?: boolean; // default true
  closeOnEsc?: boolean;          // default true
  className?: string;
  children: ReactNode;
}
```

Renders via `createPortal` to `document.body` when `open`. Panel uses
`role='dialog'` + `aria-modal='true'`. Focus is trapped inside the panel
via the shared `useFocusTrap` hook (v1.11.185) with
`restoreFocusOnUnmount: true`, and Escape routes through the hook's
`onEscape` slot so `closeOnEsc=false` cleanly disables the keybind. The
backdrop is a fixed `inset-0 bg-black/40` layer; clicks bubble to the
backdrop handler unless `e.stopPropagation` short-circuits them from the
panel. `forwardRef` exposes the panel element (not the portal wrapper).

## Adoption

1. **Health page** -- new "Open filters" `IconButton` (Filter icon) in the
   PageFrame actions row alongside the existing Refresh button. Opens a
   right-side Drawer titled "Filters" with description "status, severity,
   since" and an `<Alert variant='info' title='Coming soon'>` placeholder
   body. Real filter wiring is deferred.

2. **Auto page** (workers/autonomous dashboard) -- same approach. The
   Filter IconButton sits next to the existing Refresh button in the
   PageFrame actions row; Drawer description "worker name, status,
   target".

The goal of this patch is the Drawer integration surface. The actual
filter inputs land in a follow-up bundle.

## Test note

`web/node_modules` is not installed in this worktree, so
`npx vitest run web/src/components/ui/drawer.test.tsx` was not executed
locally. The test file covers 18 cases: render gating, dialog role,
aria-modal, side='left' / side='right', title + aria-labelledby,
description + aria-describedby, close button click, showCloseButton=false,
backdrop click + closeOnBackdropClick gating, Escape + closeOnEsc gating,
initial focus into panel, restore focus on close, width prop inline
style, className merge, forwardRef, displayName.

## Goal

Dispatch:

- Build `web/src/components/ui/progress.tsx` +
  `spinner.tsx` -- determinate progress bar
  (with label), indeterminate spinner sizes,
  circular progress variant, ARIA progressbar
  role.

Existing primitives already cover most of the
contract:

- `Progress` / `ProgressBar` (11.256 /
  v1.11.274) -- determinate linear bar with
  label, indeterminate stripe, variants
  (default/success/warning/destructive/info),
  sizes (sm/md/lg), `role="progressbar"` with
  aria-valuemin/now/max + `aria-label` (added
  in 11.327 / v1.11.345).
- `Spinner` (11.292 / v1.11.310) -- 4
  indeterminate sizes (xs/sm/md/lg), 2 tones
  (neutral/accent), reduced-motion text
  fallback, `role="status"`.

The single gap is the **circular progress
variant**. This patch ships `CircularProgress`
as a sibling export from the same
`progress.tsx` file so callers stay on the
canonical import path.

## API

```tsx
<CircularProgress
  value={42}
  max={100}
  size="md"             // 'xs' | 'sm' | 'md' | 'lg'
  variant="default"     // shares the linear bar's palette
  showPercent           // render the rounded percent inside the ring
  labelText="Saving"    // free-form label wins over showPercent
  ariaLabel="Upload"    // explicit accessible name
/>

<CircularProgress indeterminate size="sm" />
```

### Size geometry

| size | box (px) | stroke (px) | label class |
| --- | --- | --- | --- |
| `xs` | 24 | 2 | `text-[9px]` |
| `sm` | 32 | 3 | `text-[10px]` |
| `md` | 48 | 4 | `text-xs` |
| `lg` | 64 | 5 | `text-sm` |

Radius is `box/2 - stroke/2 - 1` so the stroke
stays inside the box and the rounded `linecap`
does not clip against the edge. Circumference
is `2 * PI * radius`.

### Determinate path

- `strokeDasharray = circumference`
- `strokeDashoffset = (1 - pct) * circumference`
- arc rotated `-90deg` so the stroke starts at
  12 o'clock.
- `transition-[stroke-dashoffset]` smooths
  percent updates.

### Indeterminate path

- `strokeDashoffset = circumference * 0.75` so
  a fixed quarter-arc is visible.
- `animate-spin` on the svg rotates the arc at
  the standard 1s linear cadence.
- `aria-valuenow` is omitted; the rest of the
  ARIA contract stays.

### Variant -> stroke token

| variant | class |
| --- | --- |
| `default` | `stroke-primary` |
| `success` | `stroke-success` |
| `warning` | `stroke-warning` |
| `destructive` | `stroke-destructive` |
| `info` | `stroke-info` |

Matches the linear `Progress` palette token by
token so callers can swap shapes without
retuning colours.

### Label resolution

- `labelText` wins over `showPercent` (string
  also doubles as `aria-label`).
- `showPercent` -> rounded integer percent
  (`"42%"`).
- `indeterminate` + `showPercent` -> no label
  rendered (no value to surface).
- Default: no label inside the ring (icon-sized).

### ARIA

```
role="progressbar"
aria-valuemin="0"
aria-valuemax="<max>"
aria-valuenow="<value>"   (omitted when indeterminate)
aria-label="<ariaLabel|labelText|'Progress'>"
```

Same precedence rule as the linear `Progress`
(11.327).

## Tests

`progress.test.tsx` extended with **+24 cases**
for `CircularProgress`:

- role="progressbar" + aria-valuemin/now/max
  attrs (3 cases).
- valuenow omitted when indeterminate.
- custom max forwards into aria-valuemax.
- default ariaLabel falls back to "Progress".
- string labelText is the accessible name.
- explicit ariaLabel wins over labelText.
- default size="md" -> 48px outer box +
  data-size attr.
- size="xs" / "lg" map to 24px / 64px.
- determinate dashoffset reflects percent
  (25% -> 0.75 of circumference).
- determinate at 100% -> zero dashoffset.
- clamps value above max to 100%.
- clamps negative value to 0%.
- indeterminate adds `animate-spin` + quarter-arc
  dashoffset.
- determinate omits `animate-spin`.
- variant -> stroke class (default / success /
  info / warning / destructive: 5 cases).
- showPercent renders rounded percent.
- labelText wins over showPercent.
- omitted label when showPercent + labelText
  both absent.
- indeterminate + showPercent -> no label.
- data-indeterminate attr toggles.
- data-section="circular-progress" on the root.
- caller className passes through.
- stable displayName.
- rounds 33.7 -> "34%".

## Pairs with existing primitives

- `<Progress>` / `<ProgressBar>` (11.256) --
  linear bar, unchanged.
- `<Spinner>` (11.292) -- indeterminate ring,
  4 sizes, reduced-motion fallback. Unchanged.
- `<UndoToast>` / `<Toast>` -- can host a
  `<CircularProgress size="xs">` next to status
  copy.

## Out of scope

- Per-page adoption swap. The new component is
  additive; no existing call sites rewired.
- Animated "shimmer" indeterminate gradient.
  The quarter-arc + spin is the canonical
  shadcn rhythm.
- Variant-specific stroke widths. All sizes use
  the same per-size stroke regardless of
  variant.
- Per-variant track colour. The track stays at
  20% `currentColor` opacity so the variant arc
  pops against a neutral ring.
- `prefers-reduced-motion` opt-out for the
  indeterminate spin. Callers who need a
  static affordance can pass a determinate
  `value` or use `<Spinner textOnly>` for a
  pure-text loader; the dedicated reduced-motion
  text fallback belongs on `<Spinner>`.

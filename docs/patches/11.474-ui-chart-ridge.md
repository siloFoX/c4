## Summary

Add a new `<ChartRidge>` UI primitive in
`web/src/components/ui/chart-ridge.tsx` -- pure-SVG
ridgeline (joy plot) chart stacking multiple Gaussian-KDE
density curves vertically with configurable overlap.
Each input series produces a smoothed density curve; the
curves are arranged top-to-bottom with optional overlap
so neighbouring rows visually overlap to highlight
distribution shape differences. Distinct from
`<ChartViolin>` (11.456) -- violin paints a mirrored
density per category as side-by-side vertical "violins",
whereas ridge stacks one-sided densities vertically for
multi-series visual comparison along a shared x axis.

## Behaviour

- Renders `<div role="region">` wrapping
  `<svg role="img">` with the same `aria-label`.
- Density estimation: Gaussian KDE per series.
  Bandwidth defaults to Silverman's rule
  (`1.06 * std * n^-0.2`; falls back to 1 when std
  is 0 or n < 2); explicit `bandwidth` overrides.
  `samples` evenly-spaced points across the
  rendered x range.
- Layout:
  - `rawStride = innerH / N` so all rows fit;
  - `rowHeight = rawStride * (1 + overlap)`, so
    overlap > 0 makes each curve taller than its
    own stride, producing the overlapping
    appearance of a ridgeline plot.
  - `baselineY` walks down by `rawStride`; the
    curve paints from baseline upward (`baselineY
    - normalised * rowHeight`).
- Normalisation:
  - default (`globalPeak=false`): each row
    normalises to its own peak density, so every
    ridge fills its rowHeight regardless of
    absolute density. Good for shape comparisons.
  - `globalPeak=true`: all rows share the maximum
    peak so taller curves carry larger areas. Good
    for absolute-density comparisons.
- x range: auto-derived via `getRidgeBounds` (with
  4% padding on each side); collapsed range
  expands by +/-0.5. Explicit `xMin` / `xMax`
  prop overrides.
- Hover / focus on a row sets the hovered id and
  dims every other row to opacity 0.25. Tooltip
  shows label + finite-value count + peak density
  (formatted).
- Rows are painted in reverse-z order
  (last series first, so series 0 sits visually on
  top) but their data attributes preserve original
  index for tests + callbacks.
- Mount animation:
  `motion-safe:animate-fade-in` per row group.
  `animate=false` removes the class.
- Default ARIA description: `"Ridgeline chart with
  N series, K values total. x range <xMin> to
  <xMax>."` Empty / all-non-finite returns
  `"No data"`; `ariaDescription` overrides.

## Out of scope

- Adaptive bandwidth per row -- v1 honours a
  single shared bandwidth via Silverman's rule
  (override via `bandwidth` prop) so rows are
  visually comparable.
- Animated transitions on data / overlap change
  -- snap v1; mount fade only.
- Reorderable rows via drag -- host owns
  ordering; v1 paints in array order.
- Color encoding by a third numeric dimension
  per row -- v1 uses a single color per series.

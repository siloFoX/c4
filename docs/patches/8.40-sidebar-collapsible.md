# 8.40 — Workers sidebar: collapsible (icon-rail) + Ctrl+B

Branch: `dgx-spark/sidebar-collapsible`

## What ships

A desktop icon-rail mode for the Workers sidebar so power users can
reclaim the ~17rem the worker list takes up. Mobile show/hide is
unchanged — the existing hamburger still toggles the full sidebar in
overlay form.

- `Sidebar` gains `collapsed` + `onToggleCollapsed` props. When
  collapsed, the aside shrinks to `md:w-14` (3.5rem), the worker list
  / hierarchy tree are hidden, and the List / Tree mode toggle moves
  to a stacked icon-only column under the logo.
- A new collapse handle (lucide `PanelLeftClose` / `PanelLeftOpen`)
  lives in the sidebar header next to the logo. Tooltip + aria-label
  flip with state; `aria-keyshortcuts="Control+B"` advertises the
  keyboard binding.
- Keyboard shortcut: **Ctrl+B / Cmd+B** toggles. Skipped when the
  active focus is on `<input>` / `<textarea>` / contenteditable so
  typing isn't hijacked. On mobile widths it falls back to toggling
  `sidebarOpen` (full hide) since collapsed icon-rail doesn't make
  sense in overlay mode.
- Persistence: new `c4.sidebar.collapsed` localStorage key (stored as
  `'1'` / `'0'` for forward-compat with shell readers). Cross-tab
  `storage` event re-syncs.

## Surface area

### `web/src/lib/preferences.ts`

- Adds `SIDEBAR_COLLAPSED_KEY = 'c4.sidebar.collapsed'` and
  `DEFAULT_SIDEBAR_COLLAPSED = false`.
- Exports `readSidebarCollapsed()` / `writeSidebarCollapsed(value)`.
- `resetPreferences()` now wipes the collapsed key too.

### `web/src/components/layout/Sidebar.tsx`

- New `collapsed` + `onToggleCollapsed` optional props.
- Collapsed mode hides the worker list, narrows to `md:w-14`, swaps
  the inline List/Tree pill for a stacked icon-only tab column, and
  centres the logo.
- Ships the collapse handle as an `IconButton` with tooltip,
  `aria-pressed={collapsed}`, `aria-keyshortcuts="Control+B"`.
- `transition-[width] duration-200 ease-out` + `data-collapsed` data
  attribute for E2E selectors.

### `web/src/App.tsx`

- New `sidebarCollapsed` state initialised from
  `readSidebarCollapsed()`, persisted via `writeSidebarCollapsed` on
  change, hydrated from cross-tab `storage` events.
- New global `keydown` listener binding **Ctrl+B / Cmd+B** to toggle
  collapsed (desktop) or `sidebarOpen` (mobile). Skips when focus is
  on a text-entry surface.
- Settings reset additionally restores `DEFAULT_SIDEBAR_COLLAPSED`.

### `web/src/components/KeyboardShortcutsModal.tsx`

- Adds `{ keys: 'Ctrl+B', descriptionKey: 'shortcuts.toggleSidebar' }`
  to `SHORTCUT_ROWS`.

### i18n

- `shortcuts.toggleSidebar` added to en + ko bundles.

## Test plan (manual)

1. `c4 daemon start && cd web && npm run dev`
2. Workers tab. Sidebar shows full width with the worker list.
3. Hover the new collapse icon (next to the logo): tooltip says
   `Collapse sidebar (Ctrl+B)`. Click → sidebar shrinks to icon-only
   width, list disappears, List/Tree icons stack under the logo. The
   collapse icon flips to `Expand sidebar (Ctrl+B)`.
4. Reload — sidebar comes back collapsed. Reset Preferences (or
   manually `localStorage.removeItem('c4.sidebar.collapsed')`) and
   reload — sidebar comes back expanded.
5. Press **Ctrl+B** (or Cmd+B) anywhere outside an input/textarea —
   toggles. Inside an input/textarea — does nothing.
6. Resize the window below 768px — collapse icon disappears (mobile
   uses the existing hamburger). Ctrl+B at mobile widths toggles the
   full sidebar overlay instead.
7. (review fix) Set `localStorage['c4.sidebar.collapsed'] = '1'`,
   reload at desktop width → icon rail. Resize below 768px → the
   sidebar restores to the full mobile layout (workers list visible)
   even though the persisted flag is still `'1'`. Resize back to
   desktop → icon rail returns. The `useEffectiveCollapsed` hook
   inside Sidebar.tsx watches the breakpoint and gates the
   collapsed-mode rendering on desktop only.
8. Open `?` cheat sheet — `Ctrl+B` row visible with the
   "Collapse / expand the workers sidebar" copy.

## Not in scope (follow-ups)

- Width-drag resize handle (per-user width between 200–500 px).
- Per-tab secondary panel collapse (`c4.panel.<tab>.collapsed`) for
  Features / WorkerDetail tree etc. — the preference key naming above
  is forward-compatible.
- Mobile swipe-from-left to open overlay (gesture lib).
- `prefers-reduced-motion` honoring (currently always animates the
  width transition).

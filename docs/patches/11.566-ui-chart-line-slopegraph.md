# 11.566 -- web/src/components/ui/chart-line-slopegraph.tsx

Pure-SVG **slopegraph** (Tufte's slope graph) for the
`chart-line` family. Each series is drawn as a single
straight line connecting its value at a **start** time
endpoint (left column) to its value at an **end** time
endpoint (right column); the slope of the line shows whether
the series rose or fell, and how steeply, between the two
points.

## Time-series input + endpoint extraction

Where the existing `chart-slope` (`<ChartSlope>`) takes
pre-computed `{before, after}` SCALAR pairs,
ChartLineSlopegraph is the time-series-native slopegraph: it
takes full `{x, value}[]` series and **extracts** the two
time endpoints itself.

- `resolveLineSlopegraphEndpoints(series, startX?, endX?)` --
  when `startX` / `endX` are both supplied they are used
  verbatim; otherwise they default to the minimum and
  maximum x present across every series.
- `runLineSlopegraph(series, startX?, endX?)` -- for every
  series the value at `startX` and at `endX` is looked up by
  EXACT x match. A series present at both endpoints is
  `complete` and gets a slope; one missing an endpoint is
  flagged incomplete and gets no slope line. Per series it
  reports `delta = end - start`, `direction` (up / down /
  flat) and `pctChange = delta / |start|` (NaN when the
  start value is 0 -- an undefined percentage).

Because the input is a full series, the component also draws
the optional **intermediate trajectory** -- a faint ghost
polyline through every point of the series, behind the bold
slope line -- so the reader sees both the net change and the
path taken. This is the `chart-line`-family touch that a
pure before/after slopegraph cannot offer.

## Distinct from `chart-slope`

- **Input shape.** `chart-slope` consumes `ChartSlopeItem
  {before, after}` -- the caller must already have reduced
  each series to two numbers. ChartLineSlopegraph consumes
  time series and does the endpoint reduction internally,
  with a configurable `startX` / `endX` window.
- **Trajectory overlay.** Only ChartLineSlopegraph can draw
  the intermediate path -- it has the full series.
- **Default colouring.** `chart-slope` colours by direction.
  ChartLineSlopegraph defaults to per-series palette colours
  (`colorMode='series'`, the `chart-line`-family
  convention); `colorMode='direction'` is available to
  recover the up/down/flat colouring.

All exported helpers carry a `LineSlopegraph` infix so the
`export *` barrel line cannot collide.

## Public surface

- `<ChartLineSlopegraph series={...} />` -- a
  `ChartLineSlopegraphSeries[]` (`{id, label, data, color?}`
  where each datum is `{x, value}`).
- `startX` / `endX` -- explicit endpoint x-values (default:
  global min / max x).
- `colorMode` -- `'series'` (default) or `'direction'`.
- Toggle flags: `showAxis`, `showGrid`, `showEndpointDots`,
  `showEndpointValues`, `showTrajectory` (default true),
  `showColumnGuides`, `showLegend`, `showTooltip`,
  `showConfigBadge`, `animate`.
- Controlled + uncontrolled visibility (`hiddenSeries` /
  `defaultHiddenSeries` / `onHiddenSeriesChange`).
- Standard sizing + format + a11y props. `formatValue` /
  `formatX`.
- `onSeriesClick({series})` / `onSeriesToggle({seriesId,
  hidden})`.

## Pure helpers

- `getLineSlopegraphDefaultColor(index)`.
- `getLineSlopegraphFinitePoints(points)`.
- `getLineSlopegraphDirection(startValue, endValue)`.
- `resolveLineSlopegraphEndpoints(series, startX?, endX?)`.
- `runLineSlopegraph(series, startX?, endX?)` -- the
  two-endpoint reduction; reports rising / falling / flat /
  complete counts.
- `computeLineSlopegraphLayout({...})` -- projects each
  series' slope line, endpoint nodes and trajectory.
- `describeLineSlopegraphChart(series, options?)`.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Each slope line + every endpoint dot
  `role="graphics-symbol"` + `tabIndex={0}`.
- Every node carries `data-section="chart-line-slopegraph-..."`.
- Root mirrors `data-series-count`,
  `data-visible-series-count`, `data-complete-count`,
  `data-rising-count`, `data-falling-count`,
  `data-flat-count`, `data-start-x`, `data-end-x`,
  `data-animate`.
- Series groups expose `data-direction`, `data-complete`,
  `data-delta`. Endpoint dots, column guides and column
  labels carry `data-edge="start"|"end"`.

## Tooltip / badge

Hovering a slope line or an endpoint dot shows the series
label, the start and end values, the bold delta with its
direction, and the percentage change. Config badge:
`SLOPE up=<n> down=<m> flat=<k>`. Legend has one toggle item
per series plus a `<n> up / <m> down / <k> flat` tally.

# 11.779 chart-line-cycle-amplitude (v1.11.797)

## Summary

Pure-SVG dual-panel React/TypeScript chart primitive with a
**Cycle Amplitude** oscillator panel beneath the close. The
amplitude is the peak-to-trough span of the detrended close
(close minus rolling SMA) over a lookback window. Added as the
`<ChartLineCycleAmplitude />` UI primitive in
`web/src/components/ui/`.

## Algorithm

For each bar with `lookback` (default 30) and `smoothLength`
(default 4):

```
detrended[i] = close[i] - SMA(close, smoothLength)[i]
amplitude[i] = max(detrended[i - lookback + 1..i])
             - min(detrended[i - lookback + 1..i])
```

The amplitude is non-negative by construction. Bars before
`i = lookback + smoothLength - 1` are `null` (warmup).

## Anchors

- **CONST_FLAT (close == K)**: SMA = K -> detrended = 0
  everywhere -> max - min = 0 - 0 = `0` **bit-exact** at every
  bar past the warmup. Verified across 72 combos
  `(K, lookback, smoothLength) in {0, 1, 5, 10, 100, -3} x
  {10, 20, 30, 40} x {2, 4, 6}`.
- **RISING_BY_S (close[i] = c0 + S * i)**: SMA settles to
  `close - S * (n - 1) / 2`, so detrended is a constant lag
  past `smoothLength - 1`. `max - min = const - const = 0`
  **bit-exact**. Verified across 20 combos
  `(S, lookback) in {1, 2, 5, 10, 0.5} x {10, 20, 30, 40}`.
- **FALLING_BY_S**: symmetric -> amplitude = 0.
- **Non-negative**: amplitude >= 0 for any finite input
  (`max >= min`).
- **Translation invariance**: shifting close by `C` does not
  change the amplitude (both SMA and detrender are linear).
- **Sinusoid (amplitude A)**: produces a positive amplitude
  bounded by `2 * A` (peak-to-trough) past the warmup.

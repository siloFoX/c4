## Summary

New `ChartLineReflex` UI primitive at
`web/src/components/ui/chart-line-reflex.tsx` -- a pure-SVG
two-panel chart with an Ehlers Reflex indicator detecting cycle
turning points from a Super Smoother.

## Indicator math

The Ehlers Reflex reads cycle turning points off a Super
Smoother in three stages.

- `computeLineReflexSmoother` -- the Ehlers Super Smoother of the
  price, a two-pole low-pass filter
  (`computeLineReflexSmootherCoefficients`). The Reflex runs the
  smoother at half the period; with
  `arg = 1.414*PI / (0.5*period)` the coefficients are
  `a1 = exp(-arg)`, `b1 = 2*a1*cos(arg)`, `c2 = b1`,
  `c3 = -(a1*a1)`, `c1 = 1 - c2 - c3`, and
  `filt[i] = c1*(price[i]+price[i-1])/2 + c2*filt[i-1] +
  c3*filt[i-2]`.

- `computeLineReflexSum` -- the Reflex sum. A trendline runs from
  `filt[i]` back to `filt[i-period]` with
  `slope = (filt[i-period] - filt[i]) / period`, and:

```
sum[i] = (1/period)
       * SUM[count=1..period] (filt[i] + count*slope - filt[i-count])
```

  Each term is the trendline value `count` bars back minus the
  actual filtered value there. Because the trendline is
  subtracted, the Reflex has **no trend response** -- a perfectly
  linear filter returns zero. It reacts only to the cyclic swings
  away from the line. Null until the lookback window is full.

- `computeLineReflex` -- the sum is normalized by the square root
  of an adaptively accumulated mean square:

```
ms[i]     = 0.04 * sum[i]^2 + 0.96 * ms[i-1]
reflex[i] = ms[i] != 0 ? sum[i] / sqrt(ms[i]) : 0
```

  The result is a zero-centred oscillator with no trend lag: it
  crosses zero at cycle turning points, spiking positive and
  negative as the price swings.

### Worked fixture

The Super Smoother coefficients come from `exp` and `cos` and
are irrational, so the pipeline values are pinned by the
recursions themselves plus exact anchors. The Reflex sum is
exact-testable on its own. Fed the linear ramp
`[0,1,2,3,4,5,6,7,8]` with `period 4`, the trendline equals the
ramp, so every deviation term is exactly zero and
`computeLineReflexSum` returns `[null,null,null,null, 0, 0, 0,
0, 0]` -- the defining "no trend response" property. Fed a
curved filter `[0,2,4,2,0]` (`period 4`) the deviation is the
exact non-zero value -2. An all-zero price series stays at zero
through the whole pipeline.

## API

- `getLineReflexFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineReflexPeriod(period, fallback)` -- coerce to an
  integer of at least 2.
- `computeLineReflexSmootherCoefficients(period)` -- the
  `{ a1, b1, c1, c2, c3 }` Super Smoother coefficients.
- `computeLineReflexSmoother(values, period)` -- the Super
  Smoother of the price.
- `computeLineReflexSum(filt, period)` -- the Reflex sum, null
  through the warm-up.
- `computeLineReflex(values, period)` -- the
  `{ filt, sum, ms, reflex }` series.
- `runLineReflex(points, options?)` -- sort finite points by
  `x`, run the pipeline, return `{ series, period, filt, sum, ms,
  reflex, samples, reflexFinal, reflexMin, reflexMax,
  positiveCount, negativeCount, ok }`. Each sample is classified
  `positive` / `negative` / `zero` by the sign of the Reflex.
  `ok` is false for fewer than two points.
- `computeLineReflexLayout(options)` -- stack the price panel
  above a zero-centred Reflex panel; project the price and Reflex
  paths plus markers and the zero line.
- `describeLineReflexChart(data, options?)` -- ARIA description.

## Component

`ChartLineReflex` is a `forwardRef` region (`role="region"`)
with an ARIA-linked description, a `role="img"` SVG, two stacked
panels (price on top, Reflex below) with their own axis ticks
and grid, panel labels, a config badge showing the period, a
two-series legend (Price / Reflex) with toggle, a hover/focus
tooltip exposing the value, filter, sum, Reflex and sign, and a
`motion-safe:animate-fade-in` entrance. It consumes
`{ x, value }` points. The Reflex panel carries a dashed zero
line and sign-coloured markers (green positive, red negative,
grey zero). Every node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-trendflex** -- the Trendflex sums the
  smoother's displacement directly, so it carries the full trend
  response (a linear ramp gives a constant non-zero sum). The
  Reflex subtracts a trendline first, so it has no trend lag (a
  linear ramp gives zero) and crosses zero at cycle turning
  points.
- **vs chart-line-roofing** -- the Roofing filter is a band-pass
  built from a high-pass and a smoother. The Reflex is a single
  Super Smoother whose deviation from a trendline is measured and
  RMS-normalized.
- **vs the band primitives (stddev / envelope)** -- those draw a
  band around the price. The Reflex is a zero-centred oscillator
  on its own sub-panel.

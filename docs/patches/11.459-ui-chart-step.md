## Summary

Add a new `<ChartStep>` UI primitive in
`web/src/components/ui/chart-step.tsx` -- pure-SVG step
chart (staircase plot). Adjacent points are connected by
right-angle segments, never interpolated. Three step
variants are supported: `after` (classic staircase,
default; horizontal first then vertical), `before` (jump
y at the previous x, then hold horizontally), and `middle`
(jump at the midpoint between samples). Hover surfaces a
dot indicator at the closest sample + tooltip with x label
and y value. Optional area fill below the step line.
Per-point click handler.

## API

```ts
type ChartStepType = 'before' | 'after' | 'middle';

interface ChartStepPoint {
  x: number | string;
  y: number;
}

interface ChartStepSeries {
  id: string;
  label: string;
  data: readonly ChartStepPoint[];
  color?: string;
}

interface ChartStepProps {
  series: readonly ChartStepSeries[];
  width?: number;                  // default 560
  height?: number;                 // default 280
  padding?: number;                // default 36
  stepType?: ChartStepType;        // default 'after'
  showDots?: boolean;              // default true
  showGrid?: boolean;              // default true
  showAxisTicks?: boolean;         // default true
  showLabels?: boolean;            // default true
  showLegend?: boolean;            // default true
  showTooltip?: boolean;           // default true
  showArea?: boolean;              // default false
  areaOpacity?: number;            // default 0.18
  animate?: boolean;               // default true
  className?: string;
  ariaLabel?: string;              // default 'Step chart'
  ariaDescription?: string;
  formatX?: (v: number | string) => string;
  formatY?: (v: number) => string;
  tickCount?: number;              // default 5
  strokeWidth?: number;            // default 1.75
  legendPlacement?: 'right' | 'bottom';   // default 'right'
  onPointClick?: (args: {
    series: ChartStepSeries;
    point: ChartStepPoint;
    index: number;
    seriesIndex: number;
  }) => void;
}
```

## Behaviour

- Renders a `<div role="region">` wrapping `<svg
  role="img">` carrying the same `aria-label`.
- X axis: `getStepXLabels` collects the unique ordered
  set of `x` values across every series (first
  occurrence wins). The chart distributes them evenly
  across the inner width so missing values in any one
  series still align with the others.
- Y axis: `getStepYBounds` computes the min/max
  across all series; collapsed range expands `+/- 0.5`;
  empty / all-non-finite falls back to `(0, 1)`.
- Step path: `buildStepPath` walks the series in
  order and emits `L` segments per the chosen step
  variant. All three variants are pure functions and
  exported.
- Optional area fill: `buildStepAreaPath` mirrors the
  step path then drops to the baseline (chart bottom)
  and closes with `Z`. `showArea=true` enables; the
  opacity defaults to 0.18.
- Dots: when `showDots=true` (default), each data
  point gets a small filled circle with a white
  border. Hover / focus enlarges the dot to 4 px and
  opens the tooltip; `onPointClick` fires with the
  full payload.
- Tooltip: hover opens a tooltip with the series
  label, x label, and y value. `showTooltip=false`
  suppresses; `formatX` and `formatY` shape both the
  tooltip and the axis labels.
- Legend: right-aligned by default. `showLegend=false`
  suppresses; `legendPlacement="bottom"` switches to
  a wrap-flex layout under the canvas.
- Mount animation:
  `motion-safe:animate-fade-in` per series group on
  mount. `animate=false` removes the class.
- Default ARIA description: `"Step chart with N
  series. <label> from <first> to <last>, ..."` --
  `ariaDescription` overrides; empty input produces
  `"No data"`.

## Pure helpers (exported)

```ts
DEFAULT_CHART_STEP_WIDTH        = 560
DEFAULT_CHART_STEP_HEIGHT       = 280
DEFAULT_CHART_STEP_PADDING      = 36
DEFAULT_CHART_STEP_TICK_COUNT   = 5
DEFAULT_CHART_STEP_STROKE_WIDTH = 1.75
DEFAULT_CHART_STEP_AREA_OPACITY = 0.18
DEFAULT_CHART_STEP_TYPE         = 'after'

getStepYBounds(series): { min, max }
getStepXLabels(series): (number | string)[]
buildStepPath(points, stepType): string
buildStepAreaPath(points, stepType, baselineY): string
findStepHitIndex(positions, screenX, maxDistance?): number
getStepTicks(min, max, count?): number[]
describeStepChart(series, formatY?): string
```

- `getStepXLabels` preserves the first-seen order of
  each unique x label across series; numeric and
  string x values are both supported.
- `buildStepPath` covers all three step variants:
  - `after`: `L x_(n+1), y_n  L x_(n+1), y_(n+1)`
  - `before`: `L x_n, y_(n+1) L x_(n+1), y_(n+1)`
  - `middle`: `L mid, y_n  L mid, y_(n+1)
    L x_(n+1), y_(n+1)`
- `buildStepAreaPath` re-uses `buildStepPath` and
  closes the polygon to the baseline.
- `findStepHitIndex` returns the closest sample
  (by horizontal distance only); returns `-1` when
  out of `maxDistance`.
- `describeStepChart` summarises first / last y per
  series; empty series report `"empty"`.

## Out of scope

- Linear / monotone interpolation between samples
  (use `<ChartLine>` for those).
- Stacked step series (single-series-per-track v1;
  host composes multiple instances when needed).
- Brushing / zoom (host wraps with custom layer).
- Animated transitions on data change (snap v1;
  mount fade only).

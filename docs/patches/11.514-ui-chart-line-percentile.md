# 11.514 -- web/src/components/ui/chart-line-percentile.tsx

Pure-SVG multi-series line chart showing **percentile bands over
time**. Each series's input is grouped by x-bucket
(`{x, values: number[]}` per bucket); at each x the chart computes
the lower (default p25), mid (default p50), and upper (default p75)
percentiles of the bucket's finite values. The result is rendered as
a shaded band between lower and upper, with a solid median line down
the middle and optional min/max dashed whiskers.

## Distinct from other line + band primitives

- `<ChartLineConfidence>` (11.502): per-point upper/lower bounds
  supplied **externally** as fields on the series. Percentile derives
  them from a sample group at each x.
- `<ChartLineMovingAvg>` (11.513): smooths a single y per x via a
  trailing/centered window. Percentile assumes multiple samples *at*
  each x.
- `<ChartLineTrend>` (11.512): single global linear regression line.
- `<ChartLineBaseline>` (11.510) / `<ChartLineThreshold>` (11.503):
  horizontal references.
- `<ChartLineComparison>` (11.511): two-curve comparison.

The percentile band is the canonical "distribution-over-time" view for
latency p25/p50/p75 dashboards, A/B test cohorts, etc.

## Public surface

- `<ChartLinePercentile series={...} percentiles={[25,50,75]} />`.
- Per-series `{id, label, data: {x, values[]}[], color?, bandColor?}`.
- `percentiles` defaults to `[25, 50, 75]`; out-of-range values clamp,
  non-finite slots fall back to defaults, the triple sorts ascending.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showBand`,
  `showWhiskers`, `showLegend`, `showTooltip`, `animate`.
- Numbers: `width`, `height`, `padding`, `tickCount`, `strokeWidth`,
  `whiskerStrokeWidth`, `dotRadius`, `bandOpacity`, `whiskerOpacity`,
  `whiskerDashArray`.
- Bounds: `xMin/xMax/yMin/yMax` (swapped if inverted; zero-range
  expanded +/- 0.5). When `showWhiskers=false`, the y range tightens
  to the lower/upper percentile envelope (it does NOT span min/max);
  with `showWhiskers=true` it spans min/max.
- Controlled + uncontrolled visibility via `hiddenSeries` /
  `defaultHiddenSeries` / `onHiddenSeriesChange`.
- `formatValue` / `formatX` for axis + tooltip; `xLabel` / `yLabel`
  axis-label slots.
- Callbacks: `onBucketClick({series, bucket})` and
  `onSeriesToggle({series, hidden})`.

## Pure helpers

- `getLinePercentileDefaultColor(index)` -- 10-color palette cycle.
- `sortFiniteLinePercentileValues(values)` -- ascending finite-only
  array; drops non-finite and non-array.
- `computePercentileFromSorted(sorted, p)` -- linearly-interpolated
  percentile (R type 7); `rank = (p/100) * (n-1)`, then linear
  interpolation between order statistics. Clamps p to [0, 100]; empty
  returns `NaN`; n=1 returns the lone value.
- `computeLinePercentile(values, p)` -- same, but sorts first.
- `normaliseLinePercentiles([l, m, u])` -- clamps to [0, 100],
  defaults non-finite slots, sorts ascending.
- `buildLinePercentileBucket(sample, percentiles, index)` -- returns
  `{index, x, count, min, max, lower, mid, upper}` or `null` when the
  x is non-finite or every value is non-finite.
- `computeLinePercentileLayout({series, percentiles?, hiddenSeries?,
  showWhiskers?, width, height, padding, ...})` -- returns
  `{series, xTicks, yTicks, xMin, xMax, yMin, yMax, innerWidth,
  innerHeight, totalBuckets, visibleSeriesCount, percentiles}`. Each
  layout series carries
  `{id, label, index (stable original), color, bandColor, buckets
  (sorted ascending by x), midPath, bandPath, minPath, maxPath,
  finiteCount, totalCount}`. Each layout bucket carries the raw
  bucket fields plus pixel-space `{px, pyMin, pyMax, pyLower, pyMid,
  pyUpper}`.
- `describeLinePercentileChart(series, percentiles?, hidden?,
  formatValue?)` -- builds the screen-reader description.

## Percentile formula (R type 7)

```
rank = (percentile / 100) * (n - 1)
lo   = floor(rank); hi = ceil(rank)
out  = sorted[lo] + (rank - lo) * (sorted[hi] - sorted[lo])
```

This matches the convention used by NumPy (`linear`), R (default), and
most percentile dashboards. `p0` returns the min and `p100` returns
the max exactly.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Per-series band, mid, min/max whiskers, and per-bucket dot are
  `role="graphics-symbol"` (dot + mid also `tabIndex={0}`).
- Every node carries `data-section="chart-line-percentile-..."` for
  test selectors.
- Series groups expose `data-series-id`, `data-series-color`,
  `data-series-band-color`, `data-series-bucket-count`,
  `data-series-finite-count`.
- Dots expose `data-bucket-index`, `data-x`, `data-mid`,
  `data-lower`, `data-upper`, `data-min`, `data-max`, `data-count`.

## Tooltip

Hovering a dot shows the series label, `x: <fmtX> (n=count)`, the mid
percentile (bold), the lower and upper percentiles, and optionally a
min/max range row when `showWhiskers=true`.

## Summary

New `ChartLineInstantaneousTrend` UI primitive in
`web/src/components/ui/chart-line-instantaneous-trend.tsx` -- a pure-SVG
single-panel line chart with an **Ehlers Instantaneous Trendline**
overlay from a low-lag recursive smoother. The price line, the
Instantaneous Trendline and its trigger share one panel.

## The computation

John Ehlers' Instantaneous Trendline is a low-lag trend follower: it
tracks the trend of the price with far less lag than a moving average.
`ChartLineInstantaneousTrendPoint` carries `{x, value}`.

- `computeLineInstantaneousTrendCoefficients(alpha)` -- turns a single
  smoothing constant `alpha` into the five filter weights
  `c1 = alpha - alpha^2/4`, `c2 = alpha^2/2`,
  `c3 = alpha - 0.75*alpha^2`, `c4 = 2*(1 - alpha)`,
  `c5 = (1 - alpha)^2`.
- `computeLineInstantaneousTrend(values, alpha)` -- runs the recursive
  low-lag formula
  `ITrend[i] = c1*price[i] + c2*price[i-1] - c3*price[i-2] + c4*ITrend[i-1] - c5*ITrend[i-2]`.
  The first bar seeds with the price, the second with the midpoint of
  the first two prices, bars two through six use the short averaging
  seed `(price + 2*price[-1] + price[-2]) / 4`, and from the seventh bar
  the recursion takes over. The trigger is the two-bar extrapolation
  `2*ITrend - ITrend[-2]`.
- `classifyLineInstantaneousTrendZone(itrend, trigger)` -- marks a bar
  **up** when the trigger leads the trendline, **down** when it lags,
  **flat** when they match.

## Worked fixture

The recursive region (`alpha`-dependent) is transcendental, but the
seven-bar warm-up is exact and `alpha`-independent.

The warm-up seed `(price[i] + 2*price[i-1] + price[i-2]) / 4` is a
weighted average with no `alpha`, so a six-bar integer fixture chosen so
every weighted sum divides by four computes to exact integers. The
fixture `[40,44,48,44,40,36]` -- a rise then a fall -- gives the
trendline `[40,42,44,46,44,40]` (asserted with `toEqual`), the trigger
`[null,null,48,50,44,34]`, and zones `[none,none,up,up,flat,down]` --
the warm-up region exercises every zone, and the result holds for any
`alpha`. The coefficients are checked against their defining identities
and against the constant-preserving property: the price-side weights
`c1 + c2 - c3` collapse to `alpha^2` and the total
`(c1 + c2 - c3) + (c4 - c5)` sums to one. A ten-bar constant series
confirms the recursive region reproduces the constant.

## API

`data` (a `ChartLineInstantaneousTrendPoint[]`), `alpha` (default 0.07),
sizing/format/a11y props, `strokeWidth` / `dotRadius`, the colour props,
the toggle flags (`showAxis` / `showGrid` / `showDots` default false /
`showItrend` / `showTrigger` / `showMarkers` / `showTooltip` /
`showConfigBadge` / `showLegend` / `animate`), controlled +
uncontrolled `hiddenSeries` (keys `price` / `itrend` / `trigger`),
`formatValue` / `formatX`, `onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineInstantaneousTrendFinitePoints`,
`normalizeLineInstantaneousTrendAlpha` (coerces alpha into the open unit
interval), `computeLineInstantaneousTrendCoefficients`,
`computeLineInstantaneousTrend`, `classifyLineInstantaneousTrendZone`,
`runLineInstantaneousTrend`, `computeLineInstantaneousTrendLayout`,
`describeLineInstantaneousTrendChart`, plus the
`DEFAULT_CHART_LINE_INSTANTANEOUS_TREND_*` constants,
`CHART_LINE_INSTANTANEOUS_TREND_WARMUP` and the
`ChartLineInstantaneousTrendZone` /
`ChartLineInstantaneousTrendCoefficients` types. Every exported name
carries a `LineInstantaneousTrend` infix so the `export *` barrel line
cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen
`data-section="chart-line-instantaneous-trend-aria-desc"` description
span; the price line, the trendline, the trigger line and every dot /
marker are `role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-alpha`, `data-itrend-final`,
`data-up-count`, `data-down-count`, `data-flat-count`,
`data-total-points` and `data-animate`.

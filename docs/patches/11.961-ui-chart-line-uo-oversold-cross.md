# Patch 11.961 - UI: chart-line-uo-oversold-cross primitive

Version: 1.11.979

## Summary

Add `web/src/components/ui/chart-line-uo-oversold-cross.tsx` -- a
pure-SVG dual panel line chart primitive that visualizes price
alongside Larry Williams' Ultimate Oscillator (UO) panel, with explicit
cross markers and overlay arrows at every UO crossing of the canonical
30 oversold line. Bullish (= exit, crossing UP through 30 = leaving
oversold) and bearish (= entry, crossing DOWN through 30 = entering
oversold). Mirror of the overbought-cross primitive with threshold=30.

## Pipeline

Identical to `chart-line-uo-overbought-cross`. Threshold default
flipped to 30.

## Bit-exact anchors

| Pattern                  | UO (after warmup)    | Regime  |
|--------------------------|----------------------|---------|
| CONST close = K          | 50 (0 / 0 -> 0.5)    | bullish |
| LINEAR UP (close = i)    | 100                  | bullish |
| LINEAR DOWN (close = -i) | 0                    | bearish |

The CONST anchor is tested across K in `{0, 1, 42, 100, 1234}` -- all
must produce `uo === 50` bit-exactly (regime bullish since 50 >= 30,
opposite side from the overbought variant). LINEAR DOWN saturates at
0 (bearish, below 30).

Cross events fire during transients:
* decline-then-rise: uo transitions from 0 (inside oversold) through
  30 going up -> bullish exit through 30.
* rise-then-decline: uo transitions from 100 (outside) through 30
  going down -> bearish entry through 30.

## Layout

* Default dimensions: 720 x 460.
* Two panels: price panel on top (~55% of plot area), UO panel below.
* UO panel uses a fixed `[0, 100]` range so the threshold band sits at
  the 30-line (closer to oscBottom than midpoint).
* Cross markers drawn as circles on the UO panel at `(cx, cyOsc)` and
  as arrows on the price panel at `(cx, cyPrice)`.
* SVG path strings use `.toFixed(2)` for deterministic snapshot tests.

## Accessibility

* Wrapping `<div role="region" aria-label="UO Oversold Cross chart">`.
* `<svg role="img">` with an aria-described-by linked `<desc>` sr-only
  text.
* Each cross marker: `role="graphics-symbol"` + `tabIndex={0}` +
  descriptive `aria-label` ("bullish cross at x=...").
* Legend toggles: `<button role="switch" aria-pressed="...">`.
* Motion-safe fade-in via `motion-safe:animate-fade-in` (opt-out with
  `animate={false}`).

# TODO 11.499 - chart-line-dashed primitive

Status: SHIPPED v1.11.517 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartLineDashed>`, a pure-SVG line chart with multiple named
dash patterns and first-class per-segment styling. Five named patterns
ship out of the box (`solid`, `dashed`, `dotted`, `dashDot`, `longDash`),
plus raw `dashArray` overrides at both series and per-point levels.
Consecutive segments sharing the same effective dash pattern merge into
a single `<path>` so the DOM stays small even for hundreds of points;
a dash transition opens a new `<path>` at the transition point.

## API

```tsx
<ChartLineDashed
  series={[
    {
      id: 'sales',
      label: 'Sales',
      dashStyle: 'solid',
      data: [
        { x: 0, y: 100 },
        { x: 1, y: 120 },
        { x: 2, y: 140 },
        { x: 3, y: 130, segmentStyle: 'dashed' },
        { x: 4, y: 150, segmentStyle: 'dashed' },
        { x: 5, y: 165, segmentStyle: 'dashed' },
      ],
    },
  ]}
/>
```

The example renders one solid segment for points 0..2 and one dashed
segment for points 2..5. The transition point (`x = 2`) belongs to
both segments so the line stays visually continuous.

## Dash patterns

| name      | dasharray     | description                |
| --------- | ------------- | -------------------------- |
| `solid`   | `''`          | no dasharray attribute     |
| `dashed`  | `'8 4'`       | classic medium dash        |
| `dotted`  | `'2 4'`       | small dots                 |
| `dashDot` | `'8 4 2 4'`   | dash followed by dot       |
| `longDash`| `'14 6'`      | long, infrequent dashes    |

`LINE_DASHED_PATTERNS` is exported so adopters can read or extend it.
Use a raw `dashArray` string when none of the named patterns fits.

## Dash resolution

`resolveSegmentDashArray(point, seriesDashStyle, seriesDashArray?)`
implements the resolution order (first match wins):

1. `point.segmentDashArray` (raw dasharray per point)
2. `LINE_DASHED_PATTERNS[point.segmentStyle]` (named per point)
3. `seriesDashArray` (raw, series-level)
4. `LINE_DASHED_PATTERNS[seriesDashStyle]` (named, series-level)

The segment "ending at point `i`" reads the style from `points[i]`, so
to transition mid-series an adopter marks the first point of the new
style with `segmentStyle` (or `segmentDashArray`).

## Segment building

`buildLineDashedSegments(points, seriesDashStyle, seriesDashArray?)`:

1. Walks pairs of consecutive points.
2. Each segment's dash array is resolved against the segment's
   **end** point.
3. Consecutive segments sharing the same dash array merge into one
   sub-path `M start L next L next ...`.
4. A dash transition closes the current sub-path at the transition
   point and opens a new sub-path starting at that point (so the line
   stays visually continuous).
5. Returns an array of
   `{ index, startIndex, endIndex, dashArray, styleName, path }`.

Edge cases:
- Empty input or single point -> `[]` (no segments to render).
- Two points -> exactly one segment.
- N points all same dash -> one segment.
- N points alternating dash -> N-1 segments.

## Distinct from prior chart-* primitives

- `<ChartLine>` - linear stroke without any dasharray management.
- `<ChartLineStep>` (11.497) - step interpolation; dashed-line uses
  straight L segments.
- `<ChartLineSmooth>` (11.498) - Bezier / monotone curves; dashed-line
  is linear segments only.
- `<ChartArea>` - filled area under a line; dashed-line strokes only.
- `<ChartScatter>` - 2D points without connecting line.

## Layout

- Value range: `getLineDashedBounds` auto-derives
  `(xMin, xMax, yMin, yMax)`; collapsed axes expand `+/- 0.5`; empty
  -> unit square; explicit per-axis props override.
- Each series is a `<g>` containing one `<path>` per merged segment
  plus one `<circle>` per finite point (when `showDots=true`).
- `data-section="chart-line-dashed-path"` carries `data-segment-style`
  (`'solid' | 'dashed' | 'dotted' | 'dashDot' | 'longDash' | 'custom'`)
  and `data-segment-dash-array` so tests / adopters can identify the
  pattern programmatically.

## Interactivity

- Hover or focus on a dot opens a tooltip showing series label, `x`
  (via `formatX`), and `y` (via `formatValue`).
- Hover or focus dims every other dot to opacity 0.3 and every other
  series stroke to 0.3 (the hovered series stays at full opacity).
- Per-dot `onPointClick({series, point})` callback.
- Legend buttons toggle series visibility (controlled + uncontrolled).
  Each legend swatch is a tiny `<svg>` line stroked with the series's
  default dash array so the pattern is visible at a glance.
- Mount fade via `motion-safe:animate-fade-in` (suppress via
  `animate=false`).

## Props

| prop                  | type                                | default                |
| --------------------- | ----------------------------------- | ---------------------- |
| `series`              | `readonly ChartLineDashedSeries[]`  | (required)             |
| `xMin` / `xMax` / `yMin` / `yMax` | `number?`               | auto                   |
| `width` / `height`    | `number`                            | 560 / 320              |
| `padding`             | `number`                            | `40`                   |
| `tickCount`           | `number`                            | `5`                    |
| `strokeWidth`         | `number`                            | `1.75`                 |
| `dotRadius`           | `number`                            | `3`                    |
| `lineOpacity`         | `number`                            | `1`                    |
| `gridColor` / `axisColor` | `string`                        | tokens                 |
| `hiddenSeries` / `defaultHiddenSeries` / `onHiddenSeriesChange` | controlled visibility | -- |
| `showAxis` / `showGrid` / `showDots` / `showLegend` / `showTooltip` | `boolean` | `true` |
| `animate`             | `boolean`                           | `true`                 |
| `formatValue` / `formatX` | functions                       | identity               |
| `xLabel` / `yLabel`   | `string?`                           | -                      |
| `onPointClick` / `onSeriesToggle` | callbacks               | -                      |
| `ariaLabel`           | `string`                            | `'Dashed line chart'`  |
| `ariaDescription`     | `string?`                           | auto                   |
| `className` / `style` | -                                   | -                      |

`ChartLineDashedSeries` carries `dashStyle?` (named pattern) and
`dashArray?` (raw override) at the series level.
`ChartLineDashedPoint` carries `segmentStyle?` and `segmentDashArray?`
at the per-point level (applies to the segment ENDING at that point).

## Exported helpers

- `DEFAULT_CHART_LINE_DASHED_*` numeric / string defaults.
- `DEFAULT_CHART_LINE_DASHED_PALETTE` (10-colour palette).
- `LINE_DASHED_PATTERNS` (named pattern dictionary).
- `getLineDashedDefaultColor(index)` - palette wrap.
- `getLineDashedStrokeDashArray(name)` - named -> raw dash string.
- `getLineDashedFinitePoints(points)` - drops points with non-finite
  `x` or `y`.
- `getLineDashedBounds(series, hidden?)` - per-axis bounds.
- `getLineDashedTicks(min, max, count?)` - evenly-spaced ticks.
- `resolveSegmentDashArray(point, seriesDashStyle, seriesDashArray?)`
  - returns `{ dashArray, styleName }` per the resolution order.
- `buildLineDashedSegments(points, seriesDashStyle, seriesDashArray?)`
  - canonical segment builder.
- `computeLineDashedLayout(input)` - per-series layout: pixel points
  + merged segments + tick positions.
- `describeLineDashedChart(series, hidden?, formatValue?)` - ARIA
  summary `"Dashed line chart with N series and K points. x range
  <xMin> to <xMax>, y range <yMin> to <yMax>."`.

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only
  `data-section="chart-line-dashed-aria-desc"` span.
- SVG `role="img"` with the same `aria-label`.
- Each segment path is `role="graphics-symbol"` with `tabIndex={0}`
  and `aria-label="<series> <style> segment <startIndex> to
  <endIndex>"`.
- Each dot is `role="graphics-symbol"` with `tabIndex={0}` and
  `aria-label="<series>: x=<x>, y=<y>"`.

## Data attributes

- Root: `data-section="chart-line-dashed"`, `data-series-count`,
  `data-visible-series-count`, `data-total-points`,
  `data-total-segments`, `data-animate`.
- Series group: `data-section="chart-line-dashed-series-group"`,
  `data-series-id`, `data-series-index`, `data-series-color`,
  `data-series-dash-style`, `data-series-default-dash-array`,
  `data-series-segment-count`, `data-series-point-count`,
  `data-series-finite-count`, `data-hovered`.
- Path: `data-section="chart-line-dashed-path"`, `data-series-id`,
  `data-segment-index`, `data-segment-start`, `data-segment-end`,
  `data-segment-style`, `data-segment-dash-array`.
- Dot: `data-section="chart-line-dashed-dot"`, `data-series-id`,
  `data-point-index`, `data-x`, `data-y`, `data-hovered`.
- Tooltip, legend, axes, grid: standard pattern with `data-axis="x"`
  or `"y"` on ticks / grid lines.

## Out of scope

- Per-point opacity / colour transitions (one colour per series).
- Curved segments (use `<ChartLineSmooth>` if curves needed; this
  primitive paints straight L segments only).
- Animated dash offset (no `stroke-dashoffset` animation in v1).
- Pattern fills (`pattern` element) - only `stroke-dasharray`.
- Log-scale axes - linear only.

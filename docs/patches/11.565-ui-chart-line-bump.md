# 11.565 -- web/src/components/ui/chart-line-bump.tsx

Pure-SVG **bump chart** (a.k.a. rank chart). Where an
ordinary multi-series line chart plots raw values, a bump
chart plots **rank**: at every x value the series are sorted
by value and assigned positions 1..N, and each series is
drawn as a line connecting its rank position from column to
column. Lines "bump" up and down as series overtake one
another -- the y-axis IS rank, inverted so rank 1 sits at the
top.

## Ranking

`computeLineBumpColumns(series, rankOrder)`:

1. The union of x values across all series defines the
   **columns** (sorted ascending).
2. Within each column, the series present there are sorted
   by value -- `desc` (default) puts the highest value at
   rank 1, `asc` puts the lowest at rank 1.
3. Each series in the column gets an **ordinal** rank 1..k
   -- every series gets a distinct rank, even on a tie, so
   the bump chart always has a distinct y position per
   series. Ties are broken deterministically by series
   declaration order.

A series missing at some x is simply absent from that
column; its trajectory connects the rank nodes it does have.

`runLineBump(series, rankOrder)` walks the columns and
collects each series' rank trajectory (`nodes` of
`{x, value, rank}`), plus `bestRank` / `worstRank` /
`meanRank` per series and the chart-wide `maxRank` (the
largest column size).

The component fixture is a symmetric carousel: three series,
three columns, every series visiting ranks 1, 2 and 3 in
turn -- A's rank trajectory is `[3, 1, 2]`, B's is
`[2, 3, 1]`, C's is `[1, 2, 3]`, all with mean rank 2.

## Distinct from neighbouring primitives

- `chart-line-multi` plots raw **values** on a value axis;
  the bump chart plots **rank** on a rank axis. The same
  data produces a completely different shape -- the bump
  chart's y position depends only on the *ordering*, not the
  magnitudes.
- `chart-slope` / `chart-line-slope` connect exactly **two**
  columns; the bump chart tracks rank across an arbitrary
  number of columns.
- `chart-line-comparison` / `chart-line-period-compare`
  compare *values* between series or periods; neither
  transforms the data into ordinal ranks.

All exported helpers carry a `LineBump` infix so the
`export *` barrel line cannot collide.

## Public surface

- `<ChartLineBump series={...} />` -- a
  `ChartLineBumpSeries[]` (`{id, label, data, color?}` where
  each datum is `{x, value}`).
- `rankOrder` -- `'desc'` (default; highest value = rank 1)
  or `'asc'`.
- `curved` -- default **true**; the connectors are smooth
  cubic-bezier S-curves (the classic bump shape). `false`
  draws straight segments.
- Toggle flags: `showAxis`, `showGrid`, `showNodes` (default
  true), `showLegend`, `showTooltip`, `showConfigBadge`,
  `showRankLabels` (default false -- the rank number drawn
  inside each node), `animate`.
- Controlled + uncontrolled visibility (`hiddenSeries` /
  `defaultHiddenSeries` / `onHiddenSeriesChange`). **Ranking
  is global**: hiding a series only hides its line, it does
  not re-rank the others.
- Standard sizing + format + a11y props. `formatValue` /
  `formatX`.
- `onNodeClick({series, node})` / `onSeriesToggle({seriesId,
  hidden})`.

## Pure helpers

- `getLineBumpDefaultColor(index)`.
- `getLineBumpFinitePoints(points)`.
- `computeLineBumpColumns(series, rankOrder?)` -- the
  per-column ranking.
- `runLineBump(series, rankOrder?)` -- per-series rank
  trajectories + best/worst/mean rank + `maxRank`.
- `computeLineBumpLayout({...})` -- projects each series'
  rank trajectory; rank 1 at the top, `maxRank` at the
  bottom; builds the curved or straight path.
- `describeLineBumpChart(series, options?)`.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Each series path + every rank node `role="graphics-symbol"`
  + `tabIndex={0}`. The node aria-label states the rank, the
  chart-wide max rank and the value.
- Every node carries `data-section="chart-line-bump-..."`.
- Root mirrors `data-series-count`,
  `data-visible-series-count`, `data-column-count`,
  `data-max-rank`, `data-animate`.
- Series groups expose `data-series-best-rank`,
  `data-series-worst-rank`, `data-series-node-count`.
- Rank nodes expose `data-rank`, `data-x`, `data-value`.
- Axis ticks carry `data-axis="x"|"rank"`.

## Tooltip / badge

Hovering a node shows the series label, x, `rank: <r> of
<maxRank>`, and the value. Config badge: `BUMP series=<n>
cols=<m>`. Legend has one toggle item per series (with the
series' best rank) plus a `<m> columns` note.

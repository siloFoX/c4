# 11.170 - UI motion utility hooks + helpers (v1.11.188)

Centralises animation primitives behind a `useReducedMotion` hook and a
`motion` preset table so every component reaches for the same Tailwind
classes (via `tailwindcss-animate`) and the same `prefers-reduced-motion`
behaviour.

## Helper API

### `useReducedMotion()` -- `web/src/hooks/use-reduced-motion.ts`

```ts
import { useReducedMotion } from '../hooks/use-reduced-motion';

const reduced = useReducedMotion(); // boolean
```

- On mount reads `window.matchMedia('(prefers-reduced-motion: reduce)').matches`.
- Subscribes via `addEventListener('change', ...)` (with `addListener`
  fallback for older browsers) and updates state when the OS pref flips.
- SSR-safe: returns `false` when `window` / `window.matchMedia` is
  unavailable.

### `motion` + `motionClass` -- `web/src/lib/motion.ts`

```ts
export const motion = {
  fadeIn:        'animate-in fade-in duration-200',
  fadeOut:       'animate-out fade-out duration-150',
  slideInRight:  'animate-in slide-in-from-right duration-200',
  slideOutRight: 'animate-out slide-out-to-right duration-150',
  slideInLeft:   'animate-in slide-in-from-left duration-200',
  slideOutLeft:  'animate-out slide-out-to-left duration-150',
  scaleIn:       'animate-in zoom-in-95 duration-150',
  scaleOut:      'animate-out zoom-out-95 duration-100',
} as const;

export type MotionKey = keyof typeof motion;
export function motionClass(
  key: MotionKey,
  reducedMotion: boolean,
  fallback?: string,
): string;
```

- `motionClass(key, false)` returns the preset class string.
- `motionClass(key, true, fallback)` returns the `fallback`.
- `motionClass(key, true)` returns `''` (no class).

Helpers return raw strings -- callers compose with `cn()` as usual. The
preset values are pre-baked Tailwind utility chains from
`tailwindcss-animate`, which is already enabled in `web/tailwind.config.js`.

## Adoption

| Site | Where | What changed |
| --- | --- | --- |
| `Toast.tsx` | `<Card>` className | `motionClass('fadeIn' \| 'fadeOut', reducedMotion)` toggled by the leaving flag; inline-style swipe transforms preserved so swipe-to-dismiss still works. |
| `ui/dialog.tsx` | backdrop + panel | `motionClass('fadeIn', reducedMotion)` on the backdrop, `motionClass('scaleIn', reducedMotion)` on the panel. |
| `ui/popover.tsx` | panel | `motionClass('fadeIn', reducedMotion)` on the panel className. |
| `PageTransition.tsx` | reduced-motion path | Local `prefersReducedMotion()` helper removed; the route-change effect now reads `useReducedMotion()` through a ref so the existing `[routeKey]` dependency is preserved. |

All four sites pass their existing test suites unchanged (72/72).

## Tests

- `web/src/hooks/use-reduced-motion.test.tsx` (5 cases): default match,
  active match, `change` event update, SSR-safe fallback, legacy
  `addListener` path.
- `web/src/lib/motion.test.ts` (8 cases): preset key shape, `animate-in`
  / `animate-out` invariants, `fade-*` primitives, `duration-*`
  presence, `motionClass` returns the preset / fallback / `''`, and
  ignores the fallback when motion is allowed.

## Notes

- `cn()` not required by the helper API (raw string output).
- No new dependencies. `tailwindcss-animate` was already loaded in the
  Tailwind config and supplies the underlying utilities.
- `PageTransition` keeps its CSS-transition based crossfade -- the
  `motion` presets are one-shot animate-in/out classes and are not a
  natural fit for the two-phase commit, so only the `useReducedMotion`
  refactor lands there.

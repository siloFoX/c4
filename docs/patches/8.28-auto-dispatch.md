# 8.28 — Autonomous TODO dispatch loop

Ships on `c4/auto-dispatch`. Resolves the reviewer dependency in the
TODO workflow: before this patch the manager worker would sit idle
after completing a task until a reviewer (human or separate Claude
Code session) manually nudged the next todo into it. The 2026-04-20
session stalled for hours because the reviewer forgot to dispatch the
next row after an idle transition.

## What shipped

### New module — `src/auto-dispatcher.js`

Pure-node module, no node-pty / no manager dep so tests drive the full
tick loop without spawning real workers. Public surface:

- `parseTodos(markdown)` — GFM table row parser. Handles plain ids
  (`8.28`), strikethrough ids (`~~7.8~~`), bolded status markers
  (`**done**`), and the `todo` / `done` / `pending` / `open` / `wip`
  status vocabulary. Returns
  `{id, title, status, detail, priority, dependencies, unsafe}[]`.
  Deduplicates on id; rejects unknown statuses; safe for null/empty
  input.
- `detectPriority(text)` — explicit-tag detector. `[urgent]`,
  `(urgent)`, `urgent:`, `[긴급]`, `긴급:` promote to `urgent`; `[halt]`,
  `halt:`, `halt-related` promote to `halt`. Narrative mentions of
  "urgent" inside long detail strings stay `normal` so a row describing
  the priority scheme does not accidentally promote itself.
- `detectUnsafe(text)` — flat-list match against compound shell
  connectors (`&&`, `||`, unescaped `;`) and the destructive patterns
  the spec calls out (`rm -rf`, `sudo`, `git push --force`, `shutdown`,
  `reboot`, `chmod -R 777`, fork-bomb). Returns the first match name or
  `null`. Reused by `pickNext` and by `tick` as a defense-in-depth
  gate.
- `extractDependencies(detail)` — pulls ids from `depends on N`,
  `blocked by N`, `needs N`, and `requires N` phrases so a row that
  should wait on another todo is not dispatched early.
- `sortByPriority(todos)` — `urgent > halt > normal`, tie-break by
  numeric major.minor id ascending.
- `pickNext(todos, {completedIds, skipUnsafe})` — walks the sorted
  list, skips `done` rows, skips rows whose dependencies are unmet,
  and (by default) returns unsafe rows so the caller can pause + log.
  Pass `skipUnsafe: true` to filter them instead.
- `buildDispatchPrompt(todo, {repoRoot, branch})` — assembles the
  sendTask prompt the manager sees: header, id, title, priority, repo
  root, branch, dependencies, detail, and a CLAUDE.md-style rules
  footer (`single-command only`, `git -C`, ASCII).

### New class — `AutoDispatcher`

Pluggable side-effects: `dispatch(todo)`, `idleCheck()`,
`completedIds()`, `notifier(event, payload)`, `clock`, `reader`. Every
injection is test-friendly. State surface:

- `enabled`, `paused`, `pauseReason`, `consecutiveHalts`,
  `circuitThreshold`, `throttleMs`, `managerName`, `todoPath`,
  `lastDispatchAt` (ISO), `lastDispatchId`, `lastError`, `dispatchLog`
  (ring buffer capped at 50).
- `tick()` (async) — the single entry point used by the interval timer
  *and* by `POST /autonomous/tick`. Returns one of
  `{dispatched, priority, result}` or
  `{skipped: 'disabled' | 'paused' | 'throttled' | 'manager-busy' |
    'todo-read-failed' | 'no-eligible-todo' | 'unsafe-pattern' |
    'no-dispatch-fn' | 'dispatch-failed' | 'idle-check-failed', ...}`.
  Never throws.
- `pause(reason)` / `resume()` — manual gate. `resume()` also zeroes
  the halt counter. Both are idempotent.
- `recordHalt(todoId, reason)` / `recordSuccess(todoId)` — the daemon
  calls these when the manager's outcome is observable. Three halts in
  a row auto-pause with
  `pauseReason = "circuit-breaker: N consecutive halts"`.
- `start()` / `stop()` — setInterval wrapper. Timer is `unref()`-ed so
  SIGTERM can exit cleanly.
- `reload(options)` — in-place update of throttle / threshold /
  managerName / todoPath so `/config/reload` does not require a daemon
  restart.

Safety rails: the throttle window (default 5 min) runs regardless of
`idleCheck`, so a flapping manager cannot trip a rapid-fire dispatch
storm. Reader errors surface as `skipped: 'todo-read-failed'` with the
error text in `lastError`. Dispatch callback errors surface the same
way via `skipped: 'dispatch-failed'`.

### Daemon wiring — `src/daemon.js`

- Require (`require('./auto-dispatcher')`) alongside the other module
  requires at the top.
- Module-scope `autoDispatcher` variable plus `_buildAutoDispatcher()`,
  `_startAutoDispatcher()`, `_stopAutoDispatcher()` helpers so the
  routes, `/config/reload`, and the signal handlers all share the same
  instance.
- `_buildAutoDispatcher()` returns `null` when `config.autonomous.mode
  !== true`, keeping the feature opt-in and backwards-compatible.
- `notifier` bridges `auto_dispatch_sent` → `safeEmit('task_start',
  {source: 'auto-dispatch', todo, priority, ...})` and
  `auto_dispatch_paused` → `safeEmit('halt_detected', ...)`. This keeps
  the 8.15 event vocabulary at its existing 10 types; the `source`
  discriminator is how Slack consumers filter.
- `idleCheck` reads `manager.list()`, finds the manager worker, and
  blocks dispatch unless `status === 'idle'` and intervention is not
  `approval_pending`.
- `dispatch` calls `manager.autoStart(prompt, {name})` if the manager
  is missing or exited, else `manager.sendTask(name, prompt,
  {autoMode: true})`.
- `manager.on('sse')` listener: errors with `escalation=true` call
  `recordHalt`, `task_complete` events call `recordSuccess`. Only the
  manager worker's events feed the circuit breaker.
- Lifecycle: `server.listen` fires `_startAutoDispatcher()` alongside
  `_startScheduleTick()`; SIGINT/SIGTERM call `_stopAutoDispatcher()`
  before `closeAll`. `/config/reload` preserves the prior instance's
  pause state across a rebuild so a live edit of webhook URLs does not
  wipe an operator's pause.

### HTTP routes

- `GET /autonomous/status` — `{enabled, paused, pauseReason,
  consecutiveHalts, circuitThreshold, throttleMs, managerName,
  todoPath, lastDispatchAt, lastDispatchId, lastError, recent[]}`.
  Returns a shallow `{enabled: false, reason: '...'}` stub when
  `autonomous.mode` is off.
- `POST /autonomous/pause` — body `{reason?}`. Same payload shape as
  status.
- `POST /autonomous/resume` — clears pause + halt counter.
- `POST /autonomous/tick` — manual kick, returns the tick result
  shape.

### CLI — `src/cli.js`

`c4 autonomous <status|pause|resume|tick> [reason]`. Pretty-prints
the status on `status` and `pause|resume`, prints the dispatch/skip
outcome on `tick`, and exits nonzero on `{error}` responses. Added to
the top-level help line too.

### Config — `config.example.json`

```
"autonomous": {
  "mode": false,
  "throttleMs": 300000,
  "circuitThreshold": 3,
  "managerName": "c4-mgr-auto",
  "todoPath": ""
}
```

`mode: false` keeps the feature off by default (backwards compat).
`todoPath` empty string falls back to `<repoRoot>/TODO.md`.

## Tests — `tests/auto-dispatch.test.js`

54 assertions across 12 sections (plain-node `assert` + buffered
console.log runner, matches the task-queue.test.js style):

- `parseTodos`: GFM row, strikethrough id, non-table-line skip, bolded
  `**done**` marker, unknown-status rejection, non-string input,
  dedup-by-id, real TODO.md snippet.
- `detectPriority`: `[urgent]` / `(urgent)` / `urgent:` / `[긴급]` /
  `긴급:` promotion, narrative "urgent" stays normal, `[halt]` and
  `halt-related` → halt, default normal.
- `detectUnsafe`: every pattern (`&&`, `||`, `;`, `rm -rf`, `sudo`,
  `git push --force`, `shutdown`, `reboot`, `chmod -R 777`), plus the
  "plain English" negative case.
- `extractDependencies`: `depends on`, `blocked by`, empty case.
- `sortByPriority + pickNext`: priority ordering, tie-break by id,
  done-row skip, unsafe-row pass-through vs `skipUnsafe`, unresolved
  dependency wait, completion override, done-list union, null on
  empty/all-done.
- `comparators`: priorityRank ordering + compareById major/minor.
- `AutoDispatcher core`: dispatch first eligible, disabled/paused
  short-circuit, throttle window + elapse, manager-busy gate, reader
  error surface, unsafe-pattern auto-pause, no-eligible-todo path.
- `circuit breaker`: 3 halts auto-pause with `circuit-breaker:` reason,
  `recordSuccess` resets counter, `resume` clears state.
- `notifier hooks`: `auto_dispatch_sent` on dispatch,
  `auto_dispatch_paused` on circuit trip.
- `status contract`: getStatus surface + reload().
- `buildDispatchPrompt`: id/title/priority/rules header, dependencies
  line only when non-empty.
- `smoke`: the real `TODO.md` parses and includes 8.28.

Full suite: 106 pass / 5 pre-existing failures (bcryptjs module-not-
found in `auth.js`, unrelated to this change). The new test file adds
no regressions.

## Not in scope for this patch

- Auto-merge of completed autonomous tasks. The spec lists this under
  (5), but the existing FULL AUTONOMY flow on the manager worker
  already handles post-task merge via `c4 merge` when npm test +
  merge-core pass. Hooking a daemon-side second-guess on top would
  duplicate that gate; deferred as a follow-up if the manager side
  turns out to be insufficient.
- Web UI toggle. The CLI + HTTP routes are in. The UI toggle lands as
  a follow-up UI-only change; the daemon contract is stable.
- 8.29 reviewer oversight is a separate todo.

## Rollback

`config.autonomous.mode` → `false` + `POST /config/reload` disables
the loop without restarting the daemon. No persisted state to clean.

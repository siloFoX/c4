## Summary

Add a new `<ChartHistogram>` UI primitive in
`web/src/components/ui/chart-histogram.tsx` -- pure-SVG
histogram with configurable bin count, count/density
y-axis modes, an optional Gaussian-KDE density-curve
overlay, and per-bar tooltips. Accepts a flat
`number[]` of raw values; auto-bin count defaults to
Sturges' rule. Distinct from `<ChartBar>` (11.438)
which paints categorical bars and from
`<ChartBoxplot>` (11.455) which summarises
distributions instead of plotting the full bin
frequencies.

## Behaviour

- Bin count: explicit `binCount` honored verbatim (>=
  1); otherwise Sturges' rule `ceil(log2(n) + 1)` from
  the finite-value count.
- Bin range: explicit `xMin` / `xMax` clamp the
  rendered range; values outside the range are
  dropped. Otherwise the range is the
  finite-min..finite-max of the input. Collapsed
  range expands by +/-0.5.
- Non-finite values are dropped before binning. The
  last bin is closed on the right edge so the
  maximum value still counts.
- y-axis modes:
  - `count`: bar height proportional to `bin.count`
    over the visible peak.
  - `density`: bar height proportional to
    `bin.density = count / (totalFinite * step)` so
    the rendered area sums to 1 across all bins when
    the range covers every value.
- Density curve: when `showDensityCurve=true`, a
  Gaussian-KDE overlay paints over the bars.
  Bandwidth defaults to Silverman's rule of thumb;
  `bandwidth` prop overrides. The curve is scaled
  back to the bar y-axis so it visually fits the
  histogram regardless of `yMode` (in `count` mode
  the kernel density is multiplied by
  `totalFinite * binWidth` to match the bar height).
- Hover/focus on a bar opens a tooltip with the
  bin range, count, and density (each formatted).
- Mount animation:
  `motion-safe:animate-fade-in` per bar group.
  `animate=false` removes the class.
- Default ARIA description: `"Histogram of N values
  across M bins, range <min> to <max>. Peak bin
  <peakStart> to <peakEnd> with K values."` --
  empty / all-non-finite returns `"No data"`;
  `ariaDescription` overrides.

```ts
getHistogramSturgesBinCount(n): number
getHistogramBounds(values): { xMin, xMax, yMax }
computeHistogramBins({values, binCount?, xMin?, xMax?}): ChartHistogramBin[]
silvermanBandwidth(values): number
computeKernelDensity({values, xMin, xMax, samples?, bandwidth?}):
  { x, density }[]
getHistogramTicks(min, max, count?): number[]
computeHistogramLayout(input): ChartHistogramLayoutBar[]
describeHistogram(values, bins, formatValue?): string
```

- `computeHistogramBins` returns bins with
  `{index, start, end, count, density}`; the last
  bin's `end` equals the requested `xMax` so the
  maximum value falls inside the last bin.
- `computeKernelDensity` returns samples evenly
  spaced across `[xMin, xMax]`; non-positive
  bandwidth or `xMin == xMax` -> `[]`.
- `silvermanBandwidth` returns `1.06 * std * n^-0.2`;
  falls back to `1` when `std <= 0` or `n < 2`.

## Out of scope

- Adaptive / unequal-width bins (Bayesian Blocks,
  Freedman-Diaconis with variable width) -- v1
  uses equal-width bins.
- Multi-series / overlay histograms -- host
  composes by stacking instances.
- Pre-binned input (skip the bin pass and accept
  `bins[]` directly) -- v1 always re-bins; host
  can pass already-binned data as raw values
  concentrated at bin midpoints if needed.
- Cumulative / step / ridge histogram variants --
  v1 is frequency / density only.
- Live streaming updates with incremental binning
  -- v1 re-runs the bin pass per render.

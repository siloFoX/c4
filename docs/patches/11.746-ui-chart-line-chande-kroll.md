# Patch 11.746 -- UI: chart-line-chande-kroll primitive

Released as `v1.11.764` on 2026-05-26.

## Scope

Add `<ChartLineChandeKroll>` -- a pure-SVG single-panel chart with
the **Chande-Kroll Stop** trailing-stop overlay (long stop below
the close, short stop above the close).

## Definition

Inputs:

- `data: ChartLineChandeKrollPoint[]` where
  `ChartLineChandeKrollPoint = { x; high; low; close }`. Bars
  with `high < low` are dropped.
- `atrPeriod: number` -- the lookback for the ATR. Default `10`.
  Normalized to an integer `>= 1`.
- `stopPeriod: number` -- the lookback for the rolling extreme.
  Default `9`. Normalized to an integer `>= 1`.
- `multiplier: number` -- the ATR multiplier. Default `1`.
  Normalized to a non-negative finite.

Algorithm (per bar `i`):

```
TR[i]         = max(high[i] - low[i],
                    abs(high[i] - close[i - 1]),
                    abs(low[i]  - close[i - 1]))     (i = 0: high - low)
ATR[i]        = SMA(TR, atrPeriod) at bar i

stopHigh[i]   = high[i] - multiplier * ATR[i]
stopLow[i]    = low[i]  + multiplier * ATR[i]

longStop[i]   = max(stopHigh over [i - stopPeriod + 1, i])
shortStop[i]  = min(stopLow  over [i - stopPeriod + 1, i])
```

The first `atrPeriod + stopPeriod - 2` bars are null on both
stops.

## Bit-exact algebraic anchors

**CONST_FLAT** (`high == low == close == K`):

- For every bar, `high - low = 0`, and `abs(high - prevClose) =
  abs(K - K) = 0`, same for low. So `TR == 0` exactly at every
  bar.
- `ATR = SMA(0, ...) = 0` bit-exact.
- `stopHigh = K - mult * 0 = K`, `stopLow = K + mult * 0 = K`,
  both bit-exact.
- The rolling extremes carry `K`, so `longStop = shortStop = K`
  bit-exact.

**STEADY_RANGE** (`high == 11, low == 9, close == 10`):

- Bar 0: TR = `high - low = 2`.
- Bar i > 0: prev close = 10, so `abs(11 - 10) = 1` and `abs(9 -
  10) = 1`; `high - low = 2` dominates. `TR == 2` exactly at every
  bar.
- `ATR = SMA(2, ...) = 2` bit-exact.
- With `multiplier = 1`: `stopHigh = 11 - 2 = 9`, `stopLow = 9 + 2
  = 11`, both constant.
- Rolling extremes carry the constants: `longStop = 9`, `shortStop
  = 11` bit-exact.

**Wider variant** (`high == 14, low == 6, close == 10`): `TR ==
8`, `ATR == 8`, `longStop = 14 - 8 = 6`, `shortStop = 6 + 8 = 14`
bit-exact.

**Multiplier scaling**:

- `multiplier = 0` -> stops collapse to high and low exactly:
  `longStop = high`, `shortStop = low`.
- `multiplier = 2` on STEADY_RANGE -> `longStop = 11 - 4 = 7`,
  `shortStop = 9 + 4 = 13` bit-exact.

**TR worked anchor**: prev close `= 5`, high `= 10`, low `= 8`.
- `hl = 2, hc = abs(10 - 5) = 5, lc = abs(8 - 5) = 3`. `TR =
  max(2, 5, 3) = 5` bit-exact.

**Translation invariance**: shifting every OHLC field by `+k`
leaves every TR unchanged (differences are invariant) and shifts
both stops by exactly `+k` bit-exact.

## Component

`forwardRef<HTMLDivElement, ChartLineChandeKrollProps>` with
displayName. Renders:

- A region-role outer `<div data-section="chart-line-chande-kroll">`.
- An `<svg role="img">` with an `aria-describedby` description.
- A close `<path>`, optional close dots, a long-stop `<path>`,
  a short-stop `<path>`, two stub segments per defined bar
  (`data-section="chart-line-chande-kroll-segment"`,
  `data-series-id="long" | "short"`), and one zone-coloured
  marker per defined bar.
- Legend toggle for `price` / `long` / `short`, plus `showLong`
  / `showShort` props and an `onPointClick` handler.
- ARS design-system tokens for colour and motion (`motion-safe:
  animate-fade-in`).

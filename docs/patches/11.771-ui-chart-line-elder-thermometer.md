# 11.771 chart-line-elder-thermometer (v1.11.789)

## Summary

Pure-SVG dual-panel React/TypeScript chart primitive with
Alexander Elder's **Market Thermometer** oscillator panel
beneath the close. The thermometer is the larger of the
high-gap and low-gap magnitudes relative to the prior bar.
Added as the `<ChartLineElderThermometer />` UI primitive in
`web/src/components/ui/`.

## Algorithm

For each bar with prior bar `(priorHigh, priorLow)`:

```
highGap[i] = |high[i] - priorHigh|
lowGap[i]  = |low[i]  - priorLow|
thermo[i]  = max(highGap[i], lowGap[i])
```

The seed bar (`i = 0`) has no prior reference and is `null`.
The thermometer is non-negative by construction. A
configurable `hotThreshold` separates "hot" reversal bars from
"cold" continuation bars.

## Anchors

- **Seed bar (`i = 0`)**: `null` (no prior reference).
- **CONST_HL (high and low constant)**: both gaps are zero ->
  `thermo = 0` **bit-exact** at every bar past the seed.
- **Rising-by-S (high[i] = h0 + S*i, low[i] = l0 + S*i)**:
  highGap = lowGap = |S|, `thermo = S` bit-exact. Verified
  across `S in {0.25, 0.5, 1, 2, 5, 10, 100}`.
- **Falling-by-S**: same `thermo = S` (absolute gap; the
  sign flips but `|.|` cancels it).
- **Antisymmetry**: rising and falling at the same step
  yield identical thermometer readings bit-exact.
- **Asymmetric gap**: worked anchor
  bar0 `(h=10, l=8)`, bar1 `(h=15, l=8.5)`,
  bar2 `(h=15.5, l=2)`:
  thermo[1] = max(5, 0.5) = 5; thermo[2] = max(0.5, 6.5) = 6.5
  bit-exact.
- **Translation invariance**: shifting both high and low by
  the same constant `C` does not change the thermometer.
- **Non-negative**: thermo >= 0 for any finite input.

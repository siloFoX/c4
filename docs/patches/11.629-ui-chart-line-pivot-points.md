## Summary

New `ChartLinePivotPoints` UI primitive at
`web/src/components/ui/chart-line-pivot-points.tsx` -- a pure-SVG
line chart with classic pivot point levels rendering the pivot
plus three support and three resistance lines.

## Indicator math

Classic ("floor trader" / standard) pivot points are derived
once from the **prior period's** high, low and close:

```
P  = (H + L + C) / 3                 (the pivot)
R1 = 2P - L          S1 = 2P - H
R2 = P + (H - L)     S2 = P - (H - L)
R3 = H + 2(P - L)    S3 = L - 2(H - P)
```

The pivot is the mean of the prior high/low/close; R1/S1 reflect
the low/high across the pivot; R2/S2 offset the pivot by the
prior range; R3/S3 extend furthest. `computeLinePivotPoints`
returns `null` when any of high / low / close is not a finite
number. The seven levels are constants for the whole chart --
they are not a per-bar transform of the price series.

### Worked fixture

`prior = { high: 20, low: 10, close: 18 }`:

```
P  = (20 + 10 + 18) / 3 = 16    range = H - L = 10
R1 = 2*16 - 10 = 22             S1 = 2*16 - 20 = 12
R2 = 16 + 10   = 26             S2 = 16 - 10   = 6
R3 = 20 + 2(16-10) = 32         S3 = 10 - 2(20-16) = 2
```

All seven land on integers, ordered
`S3 2 < S2 6 < S1 12 < P 16 < R1 22 < R2 26 < R3 32`, so the
tests assert them with `toEqual`. The price fixture
`[12, 14, 16, 19, 22, 17, 13]` straddles the pivot 16 --
3 bars above, 3 below, 1 on.

## API

- `getLinePivotPointsFinitePoints(points)` -- drop price points
  with a non-finite `x` or `value`.
- `computeLinePivotPoints(input)` -- the seven levels from a
  `{ high, low, close }` prior period; `null` when any field is
  not finite.
- `getLinePivotPointsLevelList(levels)` -- the seven levels as an
  ordered list (R3, R2, R1, pivot, S1, S2, S3), each with a
  stable `id`, a `label`, the `value`, and a `kind`
  (`resistance` / `pivot` / `support`).
- `runLinePivotPoints(points, options?)` -- sort finite points
  by `x`, compute the levels from `options.prior`, return
  `{ series, levels, levelList, samples, aboveCount, belowCount,
  ok }`. Each sample is classified `above` / `below` / `on` the
  pivot. `ok` is false for fewer than two points or a
  missing/invalid prior period.
- `computeLinePivotPointsLayout(options)` -- project the price
  path plus one horizontal line per level. The y-domain spans
  both the price series and all seven levels.
- `describeLinePivotPointsChart(data, options?)` -- ARIA
  description.

## Component

`ChartLinePivotPoints` is a `forwardRef` region (`role="region"`)
with an ARIA-linked description, a `role="img"` SVG, axis ticks
and grid, a config badge, a four-series legend (Price / Pivot /
Resistance / Support) with toggle, a hover/focus tooltip on the
price dots, and a `motion-safe:animate-fade-in` entrance. It
consumes `{ x, value }` price points plus a `{ high, low, close }`
prior period. The pivot line is solid; resistance and support
lines are dashed and each carries its label and value at the
right edge. Every node carries a `data-section` attribute.

## Distinctness

- **vs the moving-average overlays (chart-line-smma /
  chart-line-lsma / ...)** -- those derive a continuous line by
  averaging or regressing a sliding window of the price series.
  The pivot levels are seven *static horizontal lines* computed
  once from a *separate prior-period HLC*, not a per-bar
  transform of the plotted series.
- **vs chart-line-fractal** -- the fractal primitive detects
  swing pivots *within* the price series and marks discrete
  points. Pivot points are precomputed support/resistance price
  levels that do not depend on the in-period price action.
- The component takes a `prior` `{ high, low, close }` input
  rather than a smoothing `period` or a `wing`.

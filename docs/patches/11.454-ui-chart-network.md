## Summary

Add a new `<ChartNetwork>` UI primitive in
`web/src/components/ui/chart-network.tsx` -- pure-SVG
force-directed network graph. Nodes are placed by a
deterministic physics simulation (Coulomb-style repulsion
+ spring along edges + center gravity). The simulation
runs `iterations` steps on mount or data change. Edges
paint as thin lines between nodes; thickness scales with
edge weight. Drag a node to reposition; the pinned node
stays put while the layout re-runs around it. Mouse-wheel
zooms; drag the background to pan. Built-in zoom controls
(+, -, 1:1).

## Behaviour

- Renders a `<div role="region">` wrapping `<svg
  role="img">` carrying the same `aria-label`.
- Initial layout: `seedNetworkPositions` places each
  node on a golden-angle spiral centered on the canvas.
  Nodes carrying their own `(x, y)` keep that position.
  `fixed=true` pins the node during simulation.
- Force simulation: `stepForceSimulation` applies
  repulsion (Coulomb-style `k / r^2`), spring
  attraction along edges (`(dist - rest) * strength`),
  center gravity (`(cx - x) * gravity`), and a velocity
  damping coefficient. Identical positions get a small
  jitter so the simulation never produces NaN.
- `iterations` controls how many steps run. The
  simulation is deterministic given the same inputs --
  running twice produces the same coordinates.
- Drag: pressing on a node sets `data-hovered` and
  starts a drag; moving the mouse updates the node's
  position in world coordinates (zoom + pan are
  inverted). Releasing the mouse stops the drag.
  `enableDrag=false` disables this.
- Pan + zoom: pressing on the SVG background starts a
  pan that updates the viewport translation. Mouse
  wheel zooms in / out (clamped to `[0.25, 4]`).
  `enablePan=false` and `enableZoom=false` disable
  the respective interactions.
- Zoom controls: a small `+ / - / 1:1` button cluster
  in the top-right corner. `showZoomControls=false`
  removes the cluster.
- Tooltip: hovering a node opens a tooltip with the
  label + degree + optional group; hovering an edge
  (without an active node hover) opens an edge tooltip
  with `source - target` + weight.
- Default ARIA description: `"Network graph with N
  nodes, M edges. Top: <label> degree <d>, ..."` --
  top-5 by degree. `ariaDescription` overrides; empty
  data produces `"No data"`.
- Mount animation: `motion-safe:animate-fade-in` per
  node group on mount. `animate=false` removes the
  class.

## Pure helpers (exported)

```ts
DEFAULT_CHART_NETWORK_WIDTH           = 600
DEFAULT_CHART_NETWORK_HEIGHT          = 400
DEFAULT_CHART_NETWORK_ITERATIONS      = 80
DEFAULT_CHART_NETWORK_REPULSION       = 800
DEFAULT_CHART_NETWORK_SPRING_LENGTH   = 50
DEFAULT_CHART_NETWORK_SPRING_STRENGTH = 0.05
DEFAULT_CHART_NETWORK_GRAVITY         = 0.005
DEFAULT_CHART_NETWORK_DAMPING         = 0.85
DEFAULT_CHART_NETWORK_NODE_RADIUS     = 8
DEFAULT_CHART_NETWORK_NODE_COLOR      = '#2563eb'
DEFAULT_CHART_NETWORK_EDGE_COLOR      = '#94a3b8'

seedNetworkPositions(nodes, width, height):
  NetworkSimulationNode[]
stepForceSimulation(nodes, edges, params):
  NetworkSimulationNode[]
runForceLayout(nodes, edges, params, iterations):
  NetworkSimulationNode[]
getNetworkBounds(nodes):
  { minX, minY, maxX, maxY }
computeNodeDegree(nodeId, edges): number
findNodeAtPoint(nodes, x, y, radius): string | null
getNodeRadius(node, defaultRadius): number
describeNetworkChart(nodes, edges, formatValue?): string
```

- `seedNetworkPositions` is the deterministic seed.
  Nodes with explicit `(x, y)` keep their position;
  others land on a golden-angle spiral. `fixed` is
  preserved.
- `stepForceSimulation` is a pure function returning
  a new node array. Pinned nodes do not move; their
  velocity is reset to zero each step. Inputs are
  not mutated.
- `runForceLayout` is the deterministic full-layout
  helper: `iterations=0` returns the seed unchanged.
  Identical inputs produce identical outputs.
- `getNetworkBounds` returns the bbox; non-finite
  coords are ignored; empty input falls back to
  `{0, 0, 1, 1}`.
- `computeNodeDegree` counts incident edges (both
  directions); isolated nodes return 0.
- `findNodeAtPoint` returns the closest node within
  `radius` of `(x, y)`, or `null` if nothing is in
  range. At `radius=0` the closest node by distance
  is returned (centermost) so single-pixel clicks
  still pick a node.
- `getNodeRadius` scales the default radius by
  `sqrt(weight)` (floor 2 px); missing / invalid
  weight returns the default.
- `describeNetworkChart` produces the default ARIA
  summary: top-5 by degree.

## Tests (58 cases)

- `seedNetworkPositions`: one entry per node,
  preserves explicit (x, y), mirrors `fixed`,
  places near canvas center.
- `stepForceSimulation`: moves free nodes, pinned
  stationary, no input mutation, jitters identical
  positions to avoid NaN.
- `runForceLayout`: returns per-node positions,
  deterministic for same inputs, iterations=0
  returns seed unchanged.
- `getNetworkBounds`: min/max, empty fallback,
  ignores non-finite.
- `computeNodeDegree`: counts incident edges,
  isolated = 0.
- `findNodeAtPoint`: closest within radius, null
  when out of range, centermost at radius=0.
- `getNodeRadius`: sqrt scaling, fallback for
  missing / invalid weight, floor at 2 px.
- `describeNetworkChart`: empty -> "No data",
  top-degree summary, formatter shaping.

## Out of scope

- Continuous (animated) simulation that runs every
  frame -- v1 uses a fixed `iterations` pass on
  mount / data change.
- Curved / arc-shaped edges (host wraps with custom
  SVG layer when needed).
- Multi-edge collapse (parallel edges paint
  on top of each other; host de-dupes before
  passing).
- Force-graph clustering / community detection
  (host computes groups outside the primitive and
  feeds `group` per node).
- Keyboard pan / zoom (mouse-driven only for v1).

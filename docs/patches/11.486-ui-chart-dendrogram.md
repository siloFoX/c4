## Summary

Add a new `<ChartDendrogram>` UI primitive in
`web/src/components/ui/chart-dendrogram.tsx` -- pure-SVG
**dendrogram** (tree) chart with right-angle (elbow)
links between parent and child nodes and leaf labels.
Four orientations: `right` (root left, leaves right;
the default), `down` (root top, leaves bottom), `left`
(root right, leaves left), `up` (root bottom, leaves
top). Distinct from the other hierarchical primitives:
`<ChartSunburst>` (11.483) is radial banded,
`<ChartIcicle>` (11.484) is rectangular banded,
`<ChartCirclePacking>` (11.485) packs nested circles,
`<ChartTreemap>` / `<ChartTreemapSquarified>` nest
rectangles -- this primitive is the classic
**node-link tree** used for taxonomies, clustering,
and file-tree visualisations.

## API

```ts
type ChartDendrogramOrientation = 'right' | 'down' | 'left' | 'up';

interface ChartDendrogramNode {
  id: string;
  label: string;
  color?: string;
  children?: readonly ChartDendrogramNode[];
}

interface ChartDendrogramProps {
  root: ChartDendrogramNode | null;
  width?: number;            // default 560
  height?: number;           // default 360
  padding?: number;          // default 24
  orientation?: ChartDendrogramOrientation;  // default 'right'
  labelGap?: number;         // default 6
  leafLabelReserve?: number; // default 80 (px reserved for leaf labels)
  nodeRadius?: number;       // default 3
  leafRadius?: number;       // default 4
  leafColor?: string;        // default '#2563eb'
  internalColor?: string;    // default '#64748b'
  linkColor?: string;        // default '#94a3b8'
  showLeafLabels?: boolean;  // default true
  showInternalLabels?: boolean; // default false
  showNodes?: boolean;       // default true
  showTooltip?: boolean;     // default true
  animate?: boolean;         // default true
  className?: string;
  ariaLabel?: string;        // default 'Dendrogram'
  ariaDescription?: string;
  formatLabel?: (label: string, node: ChartDendrogramLayoutNode) => string;
  onNodeClick?: (args: { node: ChartDendrogramLayoutNode }) => void;
  onLinkClick?: (args: { link: ChartDendrogramLayoutLink }) => void;
  style?: CSSProperties;
}
```

## Behaviour

- Renders `<div role="region">` wrapping
  `<svg role="img">` with the same `aria-label`.
- Layout (`computeDendrogramLayout`):
  - Walk the input tree depth-first to assign each
    leaf a unique `leafIndex` (0..N-1 in DFS
    order).
  - Each leaf is placed at the leaf-axis offset
    `leafIndex * leafStep` and at the deepest
    depth-axis offset (with `leafLabelReserve` px
    reserved on the leaf side for labels).
  - Each internal node sits at the centroid of its
    children along the leaf axis and at its own
    depth offset along the depth axis.
  - Right / left orientations use x for depth and
    y for leaves; down / up orientations use y for
    depth and x for leaves.
  - Links use `buildDendrogramElbowPath` to draw a
    two-segment right-angle path from parent to
    child. For horizontal orientations the elbow is
    at `(targetDepth, sourceLeafCoord)`; for
    vertical orientations the elbow is at
    `(sourceLeafCoord, targetDepth)`.
- Single-leaf trees place the leaf at the canvas
  midpoint along the leaf axis.
- Per-node `color` always wins; otherwise leaves use
  `leafColor` and internals use `internalColor`.
- Leaf labels render anchored away from the leaf
  axis (`start` for `right`, `end` for `left`,
  `middle` for `down`/`up`). Internal labels are
  hidden by default; `showInternalLabels=true`
  enables them at the node centroid with a small
  upward offset.
- Hover/focus on a node opens a tooltip with the
  label, depth, and either `leaf` (for leaves) or
  `<leafCount> leaves` (for internal nodes).
- Hover/focus on a link opens a link tooltip
  showing `<source> -> <target>` (only when no
  node is also hovered).
- Mount animation:
  `motion-safe:animate-fade-in` per node + link.
  `animate=false` removes the class.
- Default ARIA description: `"Dendrogram
  (<orientation>) with N nodes, L leaves, depth
  D."` Null root -> `"No data"`.

```ts
DEFAULT_CHART_DENDROGRAM_WIDTH         = 560

getDendrogramDefaultColor(isLeaf, leafColor, internalColor): string
flattenDendrogramHierarchy(root): ChartDendrogramFlatNode[]
getDendrogramLeaves(root): ChartDendrogramFlatNode[]
getDendrogramMaxDepth(flat): number
buildDendrogramElbowPath(source, target, orientation): string
computeDendrogramLayout(input): {
  nodes, links, flat, leafCount, maxDepth
}
describeDendrogramChart(root, orientation): string
```

- `flattenDendrogramHierarchy` returns one entry per
  node with `{ id, label, depth, parentId, path,
  isLeaf, color? }`.
- `getDendrogramLeaves` is the filter
  `(n) => n.isLeaf` over the flatten output.
- `computeDendrogramLayout` returns the placed
  nodes, the elbow links (one per parent-child
  pair), the flatten output, the leaf count, and
  the observed max depth.

## ARIA + data attributes

- Root: `<div role="region">` + `aria-label`.
- SVG: `role="img"` + same `aria-label`.
- sr-only span:
  `data-section="chart-dendrogram-aria-desc"`.
- Each node circle: `role="graphics-symbol"` +
  `tabIndex={0}`. aria-label = `"<label>"` for
  leaves; `"<label> (<leafCount> leaves)"` for
  internals.
- Each link path: `role="graphics-symbol"` +
  `tabIndex={0}`. aria-label = `"Link <source> to
  <target>"`.
- `data-section` on every node:
  chart-dendrogram / -canvas / -aria-desc / -svg /
  -links / -link / -nodes / -node / -node-circle /
  -node-label / -tooltip / -tooltip-label /
  -tooltip-depth / -tooltip-leaves /
  -link-tooltip / -link-tooltip-label.
- Root mirrors `data-node-count`,
  `data-link-count`, `data-leaf-count`,
  `data-max-depth`, `data-orientation`,
  `data-animate`.
- Node groups mirror `data-node-id`,
  `data-node-depth`, `data-node-parent`,
  `data-node-is-leaf`, `data-node-leaf-index`,
  `data-node-leaf-count`, `data-node-child-count`,
  `data-node-color`, `data-hovered`.
- Link paths mirror `data-link-index`,
  `data-link-source`, `data-link-target`,
  `data-hovered`.

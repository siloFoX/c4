## Summary

`web/src/components/ui/chart-line-pivot-demark.tsx` ships
`<ChartLinePivotDemark>`, a pure-SVG single-panel chart with
**DeMark pivot** level overlays from the prior bar's
`(open, high, low, close)` and direction. The close line shares one
panel with three step-function level lines (R1 / PP / S1); each
defined bar carries the three levels as bar-wide horizontal stub
segments projected from the prior bar.

## Computation

The DeMark X seed varies with the prior bar's direction:

```
direction:
  bullish if close >  open  ->  X = 2H + L + C
  bearish if close <  open  ->  X =  H + 2L + C
  neutral if close == open  ->  X =  H +  L + 2C

PP = X / 4
R1 = X / 2 - L
S1 = X / 2 - H
```

The first bar has no prior and is left null on every level.

## Worked anchors

The DeMark formula is integer-rational when the prior bar is
integer. The implementation has four exact-style anchors.

- **Constant-series anchor** -- every prior bar `O=H=L=C=5`.
  Direction is neutral (`C == O`), `X = 5 + 5 + 2*5 = 20`,
  `PP = 5`, `R1 = 10 - 5 = 5`, `S1 = 10 - 5 = 5`. Every level
  reads 5 bit-exact.
- **Bullish anchor** -- prev `O=10, H=14, L=8, C=12`. `C > O` so
  direction = bullish, `X = 2*14 + 8 + 12 = 48`. `PP = 48/4 = 12`,
  `R1 = 48/2 - 8 = 24 - 8 = 16`, `S1 = 48/2 - 14 = 24 - 14 = 10`.
  All bit-exact integer.
- **Bearish anchor** -- prev `O=12, H=14, L=8, C=10`. `C < O` so
  direction = bearish, `X = 14 + 2*8 + 10 = 40`. `PP = 10`,
  `R1 = 20 - 8 = 12`, `S1 = 20 - 14 = 6`. All bit-exact integer.
- **Neutral anchor** -- prev `O=10, H=14, L=8, C=10`. `C == O` so
  direction = neutral, `X = 14 + 8 + 2*10 = 42`. `PP = 42/4 = 10.5`
  (1/2 is dyadic in IEEE 754), `R1 = 21 - 8 = 13`, `S1 = 21 - 14 = 7`.
  All bit-exact.

The strict order `R1 > PP > S1` for any non-degenerate prior is
tested directly.

## Distinct from neighbouring primitives

- vs `<ChartLinePivotFibonacci>` -- the Fibonacci pivot uses a
  fixed `pp = (H+L+C)/3` plus three Fibonacci-ratio offsets of the
  range; the DeMark pivot's X seed is DIRECTION-DEPENDENT, doubling
  whichever extreme matches the direction so the projection
  naturally tilts toward the prior bar's bias.
- vs Camarilla / Woodie / Classic pivots -- those use fixed
  coefficients independent of direction; the DeMark variant picks
  its X formula from the close-vs-open relationship.
- vs `<ChartLineVolatilityStop>` -- the Volatility Stop is a
  DIRECTIONAL trailing stop that flips on a cross; the DeMark
  pivot is a static set of horizontal levels per bar projected
  from the prior.

## API

`data` -- a single `ChartLinePivotDemarkPoint[]`
(`{x, open, high, low, close}`). Standard sizing / colour / a11y
props plus `strokeWidth`, `dotRadius`, `tickCount`. Toggle flags:
`showAxis`, `showGrid`, `showDots` (default false), `showPp`,
`showR1`, `showS1`, `showMarkers`, `showTooltip`, `showConfigBadge`,
`showLegend`, `animate`. Controlled and uncontrolled visibility via
`hiddenSeries` (`'price' | 'pp' | 'r1' | 's1'`). Callbacks:
`onPointClick({point})`, `onSeriesToggle({seriesId, hidden})`.
Format functions: `formatValue`, `formatX`. ARIA: `ariaLabel`,
`ariaDescription`.

## Pure helpers

- `DEFAULT_CHART_LINE_PIVOT_DEMARK_*` -- defaults.
- `getLinePivotDemarkFinitePoints` -- keep only points with finite
  OHLC and `high >= low`.
- `computeLinePivotDemarkDirection(bar)` --
  `'bullish' | 'bearish' | 'neutral'` from `close vs open`.
- `computeLinePivotDemarkX(bar)` -- `{direction, x}` for a single
  bar.
- `computeLinePivotDemarkLevels(prev)` -- the three levels for a
  bar given the prior; nulls when prev is null or has non-finite
  fields.
- `computeLinePivotDemark(bars)` -- per-bar levels; bar 0 is all
  nulls.
- `classifyLinePivotDemarkZone(close, levels)` -- `'above-r1' |
  'pp-to-r1' | 's1-to-pp' | 'below-s1' | 'none'`.
- `runLinePivotDemark(points)` --
  `{series, levels, samples, ppFinal, aboveCount, belowCount,
  betweenCount, ok}`.
- `computeLinePivotDemarkLayout({data, ...})` -- single panel with
  the close line, three level stub segments per defined bar (one
  segment per level per bar), one marker per defined-zone bar
  coloured by zone band.
- `describeLinePivotDemarkChart(data)` -- ARIA summary mentioning
  the direction-dependent X formula, the pivot point, the warm-up
  first bar, and the count of bars above / below / between.

## a11y / data hooks

Root `<div role="region">` with `aria-label` + `aria-describedby`; the
SVG is `role="img"` with the same label; an sr-only
`data-section="chart-line-pivot-demark-aria-desc"` span carries the
description. The close line, every price dot and every marker carry
`role="graphics-symbol"` and `tabIndex={0}`. Every node carries a
`data-section`. Root mirrors `data-pp-final`, `data-above-count`,
`data-below-count`, `data-between-count`, `data-total-points`,
`data-animate`, `data-empty`. Segments expose `data-series-id` and
`data-value`; markers expose `data-zone` and `data-close`.

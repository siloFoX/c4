# 11.1045 -- UI: chart-line-williams-divergence-cross primitive (v1.11.1063)

## Scope

Add a pure-SVG dual-panel `ChartLineWilliamsDivergenceCross`
component that renders the close in the top panel and the
Williams Percent R (%R) in the bottom panel, flagging
price-vs-%R direction-disagreement events as momentum
reversal warnings with bias coloring derived from the %R
slope at the trigger bar. Uses the canonical 5-state
divergence regime model (aligned-bullish / aligned-bearish
/ divergent-bullish / divergent-bearish / none).

Larry Williams's Percent R (1973) is the bounded momentum
oscillator that locates the current close within the
recent high-to-low range, scaled to `[-100, 0]`. Counter
to RSI / Stochastic convention, **-100 is oversold** and
**0 is overbought** -- so a rising %R means the close is
moving toward the period's high (more bullish), and a
falling %R means it is moving toward the period's low
(more bearish). The canonical reversal-warning signal is
price-vs-%R divergence: price still grinding higher while
%R rolls over (bearish), or price grinding lower while %R
turns up (bullish).

## Williams %R divergence pipeline

```
HH(period)   = max(high[i - period + 1 .. i])
LL(period)   = min(low[i - period + 1 .. i])
pctR[i]      = (HH - close[i]) / (HH - LL) * -100
              ; null when HH == LL (degenerate range)

priceUp     : close[i] > close[i-1]
priceDown   : close[i] < close[i-1]
wUp         : pctR[i] > pctR[i-1]
wDown       : pctR[i] < pctR[i-1]

regime ->
  'aligned-bullish'   when priceUp   && wUp
  'aligned-bearish'   when priceDown && wDown
  'divergent-bullish' when priceDown && wUp
  'divergent-bearish' when priceUp   && wDown
  'none'              when either side flat or null

bullish (divergence trigger up) :
  prev regime !== 'divergent-bullish' &&
  cur regime === 'divergent-bullish'
bearish (divergence trigger down) :
  prev regime !== 'divergent-bearish' &&
  cur regime === 'divergent-bearish'

bias        : pctR[i] vs pctR[i-1] -> up / down / flat /
              none
```

Default `period = 14` -- Larry Williams's 1973 standard
tuning. Warmup is `period - 1 = 13`. Divergence detection
requires the previous %R + close, so the first regime
classification lands at i = 14.

## Bit-exact anchors

All anchors use HLC input.

- **CONST band** `high = K + 1`, `low = K - 1`, `close
  = K`: HH = K + 1, LL = K - 1 -> range = 2. HH - close
  = 1. %R = 1 / 2 * -100 = `-50` (mid-range). Both close
  and %R are flat -> regime `none` for every bar. 0
  divergence crosses. Verified across K in {0, 1, 50,
  200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close
  = i`: For a window `[i - period + 1, i]` with high[j]
  = j + 1 and low[j] = j - 1, HH = high[i] = i + 1 and
  LL = low[i - period + 1] = i - period. HH - close = 1.
  HH - LL = period + 1. %R = `1 / (period + 1) * -100 =
  -100 / 15 ~= -6.667` (constant, near-overbought). %R
  is flat across the uptrend -> regime `none` (price up
  but %R flat). 0 divergence crosses. The canonical
  Williams %R behaviour: in a steady linear uptrend, the
  oscillator pins at the overbought boundary rather than
  tracking direction.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`,
  `close = -i`: mirror -> HH = -i + period, LL = -i - 1,
  HH - close = period, HH - LL = period + 1. %R =
  `-100 * 14 / 15 ~= -93.333` (constant, near-oversold).
  %R flat -> regime `none`. 0 divergence crosses.

The "flat %R on LINEAR input" property is the textbook
Williams %R behaviour and explains why this oscillator is
specifically a *divergence* detector rather than a
*trend-confirmation* signal: in a clean trend, %R sits at
its boundary; divergence appears only when the trend's
momentum decays (price still moves, but the close-to-
range ratio normalises away from the boundary).

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 5-state regime classifier follows the
  chart-line-*-divergence-cross family convention; the
  `bullish` / `bearish` cross *kinds* still follow the
  sibling cross-sig convention (line direction).
- Williams %R sign convention is inverted vs RSI /
  Stochastic: **-100 = oversold**, **0 = overbought**.
  Documentation makes this explicit so readers do not
  mistake the sign.
- Zero-range guard returns null when HH == LL (e.g.
  pure-CONST input with no high/low difference) --
  prevents `0 / 0` NaN from breaking the divergence
  detector.
- Oscillator panel is hard-locked to `[-100, 0]` so the
  overbought / oversold threshold lines always render at
  the same fractional heights -- the bounded-%R
  invariant lets the layout pin the y-range without
  scanning the data.
- HLC input boundary filter requires `high >= low` --
  malformed bars are dropped at the input boundary.
- On steady LINEAR input %R pins at its boundary and
  produces zero divergence crosses, which is correct
  Williams behaviour -- the divergence detector is
  specifically designed to fire on momentum decay, not
  trend confirmation. Real-world data with curvature or
  noise produces the expected divergent regime
  transitions.

# 11.536 -- web/src/components/ui/chart-line-pareto.tsx

Pure-SVG **Pareto chart in line form** -- categorical inputs
ranked by value descending, rendered as a primary value line
against the LEFT axis with a cumulative-percentage overlay
against the RIGHT axis. A dashed horizontal reference at the
configurable threshold (default 80% -- the canonical 80-20
rule) calls out the threshold, and a dashed vertical
"crossover" line marks the rank where the cumulative
percentage first reaches the threshold. Vital-few items
(those at or before the crossover) are tinted green and the
rectangle behind them is shaded; trivial-many items are
tinted gray. A vital-few badge in the top-left tells
operators "N vital few drive 80%".

## Distinct from related primitives

- `<ChartPareto>` (catalog earlier): the BAR-form of the
  Pareto analysis. This primitive is the LINE-form -- values
  rendered as a connected line so trends within the ranked
  series are visible, and the cumulative overlay is rendered
  as a separate line on a dual axis.
- `<ChartLineCumulative>` (11.520): simple cumulative SUM
  rising over time. Pareto's cumulative is RANKED-descending
  + NORMALISED to 100%, with the canonical 80% threshold
  reference + vital-few crossover.
- `<ChartLineTarget>` (11.518): vs an arbitrary user target.
  Pareto's reference is the standardized 80% (or
  user-overridable) rule with built-in vital-few
  identification.
- `<ChartLineThreshold>` (11.503): arbitrary user-defined
  horizontal thresholds + zones. Pareto's threshold is the
  canonical Pareto rule with semantic vital-few /
  trivial-many tinting.
- `<ChartLineBaseline>` (11.510): single flat horizontal
  reference. Pareto has TWO coordinated references (the 80%
  rule + the vertical crossover) plus the dual cumulative
  overlay.
- `<ChartLineSlope>` (catalog earlier): connects two
  categorical points. Pareto handles N ranked categories
  with cumulative tracking.

The canonical "80-20 rule" / "vital few vs trivial many"
visualisation used in root-cause analysis, defect
prioritisation, inventory ABC analysis, etc.

## Public surface

- `<ChartLinePareto data={...} />`.
- `data: ChartLineParetoItem[]` -- `{category, value}` with
  `value >= 0`. Items with non-finite or negative values are
  dropped. Items missing a string category are dropped.
- `threshold` (default `80`) -- cumulative-percent cutoff
  for the vital-few classification, clamped to `[0, 100]`.
- `valueColor` / `cumulativeColor` / `thresholdColor` /
  `vitalColor` / `trivialColor` -- per-element colors.
- `thresholdDashArray` (default `'6 4'`) /
  `crossoverDashArray` (default `'4 3'`).
- `fillOpacity` (default `0.12`) -- vital-few region shading.
- Toggle flags: `showAxis`, `showGrid`, `showDots`,
  `showLegend`, `showTooltip`, `showVitalFewBadge`,
  `showVitalFewFill`, `showCrossover`, `showThresholdLine`,
  `animate`.
- Standard sizing + format + a11y props.
- `formatValue` / `formatPercent` / `formatCategory`.
- `xLabel` / `yLabel` / `percentLabel` (right axis) /
  `valueLabel` / `cumulativeLabel`.
- `onPointClick({point})`.

## Pure helpers

- `getLineParetoFiniteItems(items)` -- filters to finite
  non-negative values with a string category.
- `normaliseLineParetoThreshold(value)` -- clamps to
  `[0, 100]`; non-finite -> default 80.
- `rankLineParetoItems(items, threshold?)` -- canonical
  Pareto ranking helper. Sorts by value descending; assigns
  `rank` (0-based); computes `cumulativeValue`,
  `cumulativePercent` (0-100), `share` (per-item percent),
  and `stratum` (`'vital' | 'trivial'`). The vital-few
  classification is: items whose cumulative percent
  (inclusive) is `<= threshold` are vital; the FIRST item
  that crosses the threshold is also vital (its predecessor
  was still below the threshold), so removing it would drop
  the cumulative below the threshold; subsequent items are
  trivial. `threshold <= 0` -> all trivial. `originalIndex`
  is preserved from the input array so callers can map
  rankings back to source rows.
- `findLineParetoCrossover(ranked, threshold?)` -- returns
  `{rank, cumulativePercent}` of the first item whose
  cumulative percent reaches the threshold. Empty -> `{-1,
  0}`. `threshold <= 0` -> rank 0. Threshold never reached
  (always possible with floating-point drift) -> the last
  rank.
- `computeLineParetoLayout({data, ...})` -- returns
  `{ok, ..., ranked, valuePath, cumulativePath,
  vitalFillPath, thresholdPy, crossoverRank, crossoverPx,
  vitalFewCount, trivialManyCount, finiteCount, totalCount,
  totalValue, threshold}`. The y axis spans `[0, max value]`
  on the LEFT and `[0, 100]%` on the RIGHT, sharing the same
  inner panel.
- `describeLineParetoChart(data, options?)` -- builds the
  screen-reader description (item count, total value, vital
  few count, threshold).

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Value path + cumulative path + threshold line + every
  value dot is `role="graphics-symbol"` + `tabIndex={0}`.
  Cumulative dots are decorative
  (`role="graphics-symbol"` + `aria-hidden="true"`) to keep
  the focus order clean. Dot aria-label includes the
  category, rank, value, cumulative percent, and stratum.
- Crossover vertical line is `role="graphics-symbol"` with
  aria-label including the rank.
- Every node carries `data-section="chart-line-pareto-..."`
  for test selectors.
- Value path has `data-kind="value"`; cumulative path
  `data-kind="cumulative"`.
- Value dot has `data-kind="value"`; cumulative dot
  `data-kind="cumulative"`.
- Dots expose `data-rank`, `data-category`, `data-value`,
  `data-cumulative-percent`, `data-share`, `data-stratum`.
- Root mirrors `data-total-points`,
  `data-vital-few-count`, `data-trivial-many-count`,
  `data-crossover-rank`, `data-threshold`,
  `data-total-value`, `data-animate`.
- Badge mirrors the same vital-few count + crossover rank +
  threshold.

## Tooltip

Hovering a value dot shows the category in value-color, a
`rank: <n>` row, a bold `value: <v> (share <p>%)` row, a
`cumulative: <p>%` row, and a coloured `vital few` /
`trivial many` row tinted by stratum.

## Summary

New `ChartLineChoppiness` UI primitive at
`web/src/components/ui/chart-line-choppiness.tsx` -- a pure-SVG line
chart with a Choppiness Index panel rating whether the market trends
or ranges. The close price line occupies the top panel; the bottom
panel renders the Choppiness Index on a fixed 0 to 100 scale with
dashed choppy / trending threshold lines and shaded zones.

## Indicator math

E.W. Dreiss's Choppiness Index:

```
TR[i]  = max(high - low, |high - prevClose|, |low - prevClose|)
CHOP[i] = 100 * log10( sum(TR over window) / (maxHigh - minLow) ) / log10(period)
```

The True Range captures the distance the price travelled in each
bar -- the greatest of the bar's own high-low span, the gap up from
the prior close, and the gap down from the prior close (the first
bar, with no prior close, uses the bare high-low span). Over a
trailing window of `period` bars the summed True Range is the total
path travelled; the window's high-low extent is the net ground
covered. Their ratio is at least 1 (the path is never shorter than
the extent) and at most `period`, so `log10(ratio) / log10(period)`
rescales it onto 0..1 and the `* 100` onto 0..100.

- A strong **trend** travels little more than its net extent, so the
  ratio is near 1, `log10(1) = 0`, and the index sits near **0**.
- A **choppy** range retraces constantly, so the summed range dwarfs
  the extent, the ratio approaches `period`, and the index climbs
  toward **100**.

The index is defined from index `period - 1` onward. A zero-extent
window reads 0, the result is clamped to 0..100, and a period below
2 leaves the index undefined.

### Worked fixture

Every bar has a high-low range of 10 and no gaps, so each True Range
is exactly 10. Bars 0-3 climb a clean staircase; bars 4-6 range
inside `[30, 40]`:

```
period 4, CHOP = 100 * log10(sumTR/extent) / log10(4)
CHOP[3] window 0..3: sumTR 40, extent 40,  ratio 1   -> 0       (trend)
CHOP[4] window 1..4: sumTR 40, extent 30,  ratio 4/3 -> ~20.752
CHOP[5] window 2..5: sumTR 40, extent 20,  ratio 2   -> 50
CHOP[6] window 3..6: sumTR 40, extent 10,  ratio 4   -> 100     (chop)
```

The True Ranges and the ratios are exact integers/rationals; the
`log10` step makes the intermediate index values inexact, so tests
assert them with `toBeCloseTo` and keep the exact `ratio 1 -> 0`
case as `toBe`.

## API

- `getLineChoppinessFinitePoints(points)` -- drop points with a
  non-finite `x`, `high`, `low` or `close`.
- `normalizeLineChoppinessPeriod(period, fallback)` -- coerce to a
  positive integer; non-finite or sub-1 falls back, fractional
  floors.
- `computeLineChoppinessTrueRange(highs, lows, closes)` -- the True
  Range series.
- `computeLineChoppiness(highs, lows, closes, period)` -- returns
  `{ trueRange, chop }`; `chop` is null through the warm-up.
- `runLineChoppiness(points, options?)` -- sort finite points by
  `x`, compute the series, return `{ series, period,
  choppyThreshold, trendingThreshold, trueRange, chop, samples,
  chopFinal, choppyCount, trendingCount, ok }`. Each sample is
  tagged `choppy` / `trending` / `neutral`. `ok` is false for fewer
  than two points.
- `computeLineChoppinessLayout(options)` -- stack a price panel
  above a Choppiness panel on a fixed 0..100 scale. Returns panels,
  ticks, the price path, the Choppiness path, state-coloured
  markers, the two threshold lines, and the two shaded zones.
- `describeLineChoppinessChart(data, options?)` -- ARIA description.

## Component

`ChartLineChoppiness` is a `forwardRef` region (`role="region"`)
with an ARIA-linked description, a `role="img"` SVG, panel labels, a
config badge, a two-series legend (Price / CHOP) with toggle, the
two dashed threshold lines, the Choppiness line, markers coloured by
state (choppy red, trending green), a hover/focus tooltip exposing
the high, low, close, true range, Choppiness Index and state, and a
`motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs chart-line-adx** -- the ADX rates trend *strength* from
  directional movement (the +DI / -DI spread). The Choppiness Index
  makes **no directional judgement at all** -- it only measures how
  much the price *retraced* relative to the ground it covered, so a
  strong uptrend and a strong downtrend both read near 0 and only
  sideways chop reads high.
- **vs chart-line-atr** -- the ATR is the *smoothed True Range*, an
  absolute volatility measure with no upper bound. The Choppiness
  Index uses the True Range only as an input, dividing its windowed
  *sum* by the window extent to produce a *bounded 0..100 ratio*
  about trend-versus-range, not volatility.
- **vs chart-line-rsi / chart-line-cmo** -- those are momentum
  oscillators built from up/down price changes. The Choppiness Index
  is built from True Range and the high-low extent and says nothing
  about momentum direction.

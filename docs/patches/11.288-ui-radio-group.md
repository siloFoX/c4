# 11.288 -- RadioGroup primitive (v1.11.306)

New `RadioGroup` primitive at
`web/src/components/ui/radio-group.tsx`. Component-scope only, no
daemon or CLI surface change.

## Why

The existing Radio primitive is a single `<input type="radio">`
wrapper. Three pain points across the current operator
surfaces:

1. SettingsView ships a local `ChoiceGroup` component that
   renders the same radiogroup pattern inline (theme picker /
   density picker / detail-mode picker). It does not implement
   arrow-key cycling, Home/End jumps, or Space/Enter
   commit-on-focus, so keyboard users have to tab through every
   row to reach the last option.
2. Profiles role picker (planned editor surface) wants a
   keyboard-navigable single-select with per-row descriptions
   ("Operator: full access", "Reviewer: read-only"). Building
   that into a one-off component would duplicate the SettingsView
   pattern.
3. The Radio primitive itself has no group semantics --
   `role="radiogroup"` is missing, so screen readers announce
   each row as a standalone input rather than "option 2 of 3".

Patch 11.288 ships the canonical RadioGroup primitive so all
three surfaces (and future ones) reach for one keyboard +
ARIA + descriptions contract.

## What changed

### `web/src/components/ui/radio-group.tsx` (new file)

- `RadioGroup<V extends string>` component with props
  `{ value, onChange, items, ariaLabel?, orientation?,
  showDescription?, className?, 'data-testid'? }`.
- Per-item shape
  `RadioGroupItem<V>: { value, label, description?, icon?, disabled? }`.
- `orientation: 'horizontal' | 'vertical'` (default vertical).
  Drives the `flex flex-wrap` vs `flex flex-col` container
  class AND the `useFocusCycle` `orientation` argument so
  ArrowLeft/Right cycles horizontal groups while
  ArrowUp/Down cycles vertical groups.
- `useFocusCycle` integration:
  - `containerRef` -> the radiogroup div.
  - `itemSelector` -> `[role=radio]:not([disabled])` so
    disabled rows are skipped automatically.
  - `wrap: true` so cycling at the ends loops back.
  - `onSelect(el)` -> reads `data-radio-value` and calls the
    parent `onChange`, so arrow-key nav commits the value as
    it moves (matches the WAI-ARIA radiogroup contract).
- Custom `onItemKeyDown` wraps the hook so Space / Enter on
  the focused row commits the value (the hook itself only
  handles arrows + Home/End).
- Roving tabindex policy:
  - If an item is active, only that item is tabbable.
  - If no item is active yet (uncontrolled-style empty
    value), the FIRST non-disabled item takes the tab stop
    so the keyboard user can enter the group.
- `aria-describedby` plumbing: each item with a
  `description` slot gets a unique `<id>` so SR users hear
  the secondary context after the primary label. Items
  without a description carry no `aria-describedby`
  attribute at all.
- `showDescription` prop (default true) lets dense rows hide
  the visual description while keeping the SR plumbing
  alive.
- Data-attribute selectors for e2e:
  - `data-section="radio-group"` on the container +
    `data-orientation="horizontal|vertical"`.
  - `data-section="radio-group-item"` per row.
  - `data-section="radio-group-description"` per
    description.
  - `data-radio-value="<value>"` per row.
  - `data-active="true|false"` per row.

### Adopted

- Primitive only this commit. SettingsView still mounts the
  local `ChoiceGroup` component. Migrating Settings theme +
  density pickers + the planned Profiles role picker to the
  new primitive is tracked as a follow-up so the migration
  can land with its own SettingsView test refresh.
- Re-exported from `web/src/components/ui/index.ts`.

## Deferred (dispatch follow-ups)

- Settings density picker: same story.

  surface that does not exist today.

## Tests

- `web/src/components/ui/radio-group.test.tsx` -- 19
  vitest cases:
  - Container role + ariaLabel.
  - One radio per item.
  - `aria-checked` on active vs inactive.
  - Roving tabindex (only the active radio is tabbable).
  - Click commits the value.
  - Click on the already-active row is a no-op.
  - Disabled item is unclickable.
  - Default `orientation="vertical"` + ArrowDown cycles
    forward.
  - `orientation="horizontal"` + ArrowRight cycles forward.
  - Home / End boundary jumps.
  - Space commits the focused row.
  - Enter commits the focused row.
  - Description renders + plumbs `aria-describedby`.
  - Items without description have no `aria-describedby`.
  - `showDescription={false}` hides the description text.
  - `data-section` + `data-orientation` + `data-active`

## Notes

- The primitive ships with no built-in label slot above the
  radiogroup; callers wrap with `<FormField>` (TODO 11.285)
  to get a labelled wrapper + helperText / warning / error
  state.
- The description plumbing via `aria-describedby` matches
  the contract `<Tooltip>` + `<Dialog>` use, so SR users
  hear the same chain regardless of which primitive carries
  the secondary copy.

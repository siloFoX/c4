## Summary

New `ChartLineCoral` UI primitive in
`web/src/components/ui/chart-line-coral.tsx` -- a pure-SVG single-panel
line chart with a **Coral Trend** overlay from a six-pole Tillson
smoothing of price. The close line and the Coral line share one panel.

## The computation

The Coral Trend indicator is a six-pole Tillson smoothing of the price. It
is a close-only indicator, so `ChartLineCoralPoint` carries `{x, value}`.

- `computeLineCoralCoefficients(cd)` -- the four Tillson coefficients for
  the constant `cd`: `c1 = -cd^3`, `c2 = 3(cd^2 + cd^3)`,
  `c3 = -3(2cd^2 + cd + cd^3)`, `c4 = 1 + 3cd + cd^3 + 3cd^2`. They
  recombine the third through sixth smoothing poles and sum to exactly
  one.
- `computeLineCoral(values, period, cd)` -- runs the price through six
  cascaded exponential filters (pole alpha derived from the period) and
  recombines poles three to six with the Tillson coefficients. The poles
  are seeded from the first value, so the Coral line is defined from the
  first bar.
- `classifyLineCoralTrend(coral, prevCoral)` -- marks each bar **up**,
  **down** or **flat** by the slope of the Coral line.

The Tillson coefficients give the higher poles a small negative weight,
which adds a touch of lead -- the Coral lags less than a plain moving
average of the same length.

## Worked fixture

The six-pole cascade is transcendental in general, but two anchors are
exact.

`cd = 0.5` makes `cd`, `cd^2` and `cd^3` exact dyadic values, so the
Tillson coefficients are exact -- `{c1: -0.125, c2: 1.125, c3: -3.375,
c4: 3.375}`, asserted with `toEqual`. They sum to exactly `1`.

Because the coefficients sum to one, the Coral of a constant series is
that constant: with the poles written in the incremental form
`pole + alpha * (input - pole)`, a constant input keeps every pole exactly
at the seed, and the coefficient recombination collapses to the constant.
`computeLineCoral([8,8,8,8,8], 5, 0.5)` is `[8,8,8,8,8]`, asserted with
`toEqual`. A rising series gives a rising Coral, a falling series a
falling Coral, and the trend classification is tested directly as a pure
function.

## API

`data` (a `ChartLineCoralPoint[]`), `period` (default 21), `cd` (default
0.4), sizing/format/a11y props, `strokeWidth` / `dotRadius`, the colour
props, the toggle flags (`showAxis` / `showGrid` / `showDots` default
false / `showCoral` / `showMarkers` / `showTooltip` / `showConfigBadge` /
`showLegend` / `animate`), controlled + uncontrolled `hiddenSeries` (keys
`price` / `coral`), `formatValue` / `formatX`, `onPointClick` /
`onSeriesToggle`.

Exported helpers: `getLineCoralFinitePoints`, `normalizeLineCoralPeriod`,
`computeLineCoralCoefficients`, `computeLineCoral`,
`classifyLineCoralTrend`, `runLineCoral`, `computeLineCoralLayout`,
`describeLineCoralChart`, plus the `DEFAULT_CHART_LINE_CORAL_*` constants
and the `ChartLineCoralTrend` type. Every exported name carries a
`LineCoral` infix so the `export *` barrel line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen `data-section="chart-line-coral-aria-desc"`
description span; the price line and every marker are
`role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-period`, `data-cd`,
`data-coral-final`, `data-up-count`, `data-down-count`,
`data-flat-count`, `data-total-points` and `data-animate`.

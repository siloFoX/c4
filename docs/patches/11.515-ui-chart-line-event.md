# 11.515 -- web/src/components/ui/chart-line-event.tsx

Pure-SVG multi-series line chart with **event markers**. A dedicated
horizontal track sits above the plot area; each event renders as a
kind-driven icon (release/incident/maintenance/milestone/info/custom)
on the track, with a faint dashed vertical rule dropping from the icon
through the plot to anchor the event in the time series. Each event
icon is independently focusable and clickable, with a tooltip showing
title + kind + optional description + optional timestamp label.

## Distinct from `<ChartLineAnnotated>` (11.509)

The annotated primitive paints **vertical reference lines with text
labels** inline with the plot. `<ChartLineEvent>` introduces:

- **An event track above the plot** (`trackHeight=22` by default)
  reserved exclusively for event icons.
- **Kind-driven icons** -- six canonical kinds with default colours +
  shapes (release/circle/green, incident/triangle/red,
  maintenance/square/amber, milestone/star/purple, info/diamond/cyan,
  custom/cross/slate). Per-event `icon` and `color` overrides always
  win.
- **A kind legend** that toggles all events of a kind on/off
  (controlled + uncontrolled via `hiddenKinds` /
  `defaultHiddenKinds` / `onHiddenKindsChange` /
  `onKindToggle({kind, hidden})`).
- **Interactive icons** -- each icon is `role="graphics-symbol"` +
  `tabIndex={0}` with a hover/focus tooltip; `onEventClick({event})`
  drives drill-downs.
- **Out-of-range guarding** -- events with x outside the active
  domain are emitted to the layout with `inRange=false` and skipped
  from rendering (no visual overflow).

Both primitives are useful; `<ChartLineEvent>` is the canonical
release-tracker / incident-overlay surface for an SRE / DevOps
dashboard. `<ChartLineAnnotated>` remains the right pick for ad-hoc
labelled vertical references without the kind/track concept.

## Public surface

- `<ChartLineEvent series={...} events={...} />`.
- `events: ChartLineEventEvent[]` -- `{id, x, title, description?,
  kind?, color?, icon?, timestampLabel?}`. `kind` defaults to
  `'custom'`; unknown kinds fall back to custom def too.
- `iconRadius`, `trackHeight`, `trackGap`, `trackBgColor`,
  `ruleDashArray`, `ruleOpacity` tune the track and rules.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showKindLegend`, `showTooltip`, `showEventTrack`, `showEventRule`,
  `showTrackBg`, `animate`.
- Standard sizing + format + a11y props (`width`, `height`, `padding`,
  `tickCount`, `strokeWidth`, `dotRadius`, `lineOpacity`,
  `formatValue`, `formatX`, `xLabel`, `yLabel`).
- Controlled + uncontrolled `hiddenSeries`, `hiddenKinds`.
- Callbacks: `onPointClick({series, point})`,
  `onEventClick({event})`, `onKindToggle({kind, hidden})`,
  `onHiddenSeriesChange`, `onHiddenKindsChange`.

## Pure helpers

- `getLineEventDefaultColor(index)` -- 10-color palette cycle.
- `getLineEventFinitePoints(points)` -- finite-only filter.
- `getLineEventKindDef(kind)` -- canonical
  `{color, icon, label}` per kind, falling back to `custom` for
  unknown / missing.
- `getLineEventValidEvents(events)` -- drops malformed events (null,
  non-finite x, missing id/title); preserves input order.
- `buildLineEventIconPath(icon, cx, cy, r)` -- returns an SVG `d`
  string for the requested icon. `cross` emits two sub-paths
  (`M..L..M..L`); `star` emits a 10-vertex polygon (5 outer + 5
  inner). Non-finite inputs or `r <= 0` return `''`.
- `computeLineEventLayout({series, events?, hiddenSeries?,
  hiddenKinds?, ...})` -- returns `{series, events, xTicks, yTicks,
  xMin, xMax, yMin, yMax, innerWidth, innerHeight, trackTop,
  trackBottom, trackCenterY, totalPoints, visibleSeriesCount,
  visibleEventCount}`. The plot area is shifted down by
  `trackHeight + trackGap` when `showEventTrack=true`.
- `describeLineEventChart(series, events?, hiddenSeries?,
  hiddenKinds?)` -- builds the screen-reader description.

## Canonical kinds

| Kind | Color | Icon | Label |
| --- | --- | --- | --- |
| release | `#16a34a` | circle | Release |
| incident | `#dc2626` | triangle | Incident |
| maintenance | `#f59e0b` | square | Maintenance |
| milestone | `#9333ea` | star | Milestone |
| info | `#0891b2` | diamond | Info |
| custom | `#64748b` | cross | Custom |

Adopters can still override `color` and `icon` per event.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Per-line path + per-dot + per-event icon are
  `role="graphics-symbol"` with `tabIndex={0}`. Event icon aria-label:
  `"Event <title> (<kind>) at x <x>"`.
- Every node carries `data-section="chart-line-event-..."` for test
  selectors.
- Event icons expose `data-event-id`, `data-event-kind`,
  `data-event-icon`, `data-event-color`, `data-event-x`,
  `data-event-hovered`.
- Root mirrors `data-event-count`, `data-visible-event-count`.

## Tooltip

- Hovering an icon shows the event tooltip: title (bold), `<kind
  label> - x <fmtX>` (with optional `(timestampLabel)` suffix), and a
  description row when present.
- Hovering a dot shows a small point tooltip with the series label
  + x + y (event tooltip takes precedence when both are hovered).
- `showTooltip=false` suppresses both.

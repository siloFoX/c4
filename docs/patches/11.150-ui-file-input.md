# Patch 11.150 - UI: FileInput dropzone primitive

Web release: **v1.11.168**.

## New primitive

`web/src/components/ui/file-input.tsx` exports the `FileInput` component
(forwardRef to the visible dropzone `<div>`, not the hidden `<input
type="file">`). Props:

- `label?: ReactNode` -- rendered above the dropzone via `<Label
  htmlFor={inputId}>` and `aria-labelledby` on the dropzone.
- `hint?: ReactNode` -- helper text below, wired via `aria-describedby`.
- `error?: ReactNode` -- destructive-toned alert below, wired via
  `aria-describedby`; also flips the dropzone border to
  `border-destructive`.
- `accept?: string` -- forwarded to the hidden `<input>` and used to
  validate dropped files. Comma-separated MIME (`application/json`),
  prefix wildcards (`image/*`), and dotted extensions (`.json`) all
  match.
- `multiple?: boolean` (default `false`) -- dropping >1 file when
  `multiple=false` is rejected via `onError`.
- `maxSize?: number` -- per-file byte cap; oversize files trigger
  `onError`.
- `onFiles?: (files: File[]) => void` -- fired with valid files (drop
  or native picker change).
- `onError?: (msg: string, file?: File) => void` -- fired with the
  first validation failure.
- `className?: string` -- merged onto the dropzone via `cn(...)`.
- `disabled?: boolean` -- blocks click/keyboard/drop; native input is
  disabled too.

Behaviour:

- Click anywhere on the dropzone -- including the inline `Browse`
  button -- opens the picker by `.click()`-ing the hidden input.
- Keyboard `Enter` / `Space` on the focused dropzone opens the picker.
- Drag enter / leave events maintain a nesting counter so child
  elements don't toggle `data-active=false` mid-drag; on `drop` the
  counter resets and the highlight clears.
- Validation order: count -> accept -> maxSize. First failure short-
  circuits and calls `onError`.

Re-exported from `web/src/components/ui/index.ts`.

## Adoption sites (2 of 2)

Neither page had a pre-existing import flow, so both received a
placeholder block that wires `onFiles` to `console.log` and `onError`
to a local error-slot state. The intent is real adoption surface, not
a stub: the components mount, accept drops, and gate validation.

1. **`web/src/pages/Templates.tsx` -- `ImportTemplateForm`.** Inserted
   above the pagination row. Accepts `application/json`, `.json`,
   `.yaml`, `.yml`; 1 MiB cap. `onFiles` logs the file name/size to
   the console under the `[templates]` tag. `onError` populates the
   error slot.
2. **`web/src/pages/Config.tsx` -- `ImportConfigSection`.** Rendered
   as a fresh `<Panel>` below the existing config viewer panel.
   Accepts `application/json`, `.json`; 1 MiB cap. Same console-log /
   error-slot pattern under the `[config]` tag.

## Tests

`web/src/components/ui/file-input.test.tsx` covers 14 vitest cases
including: label+hint render, error slot, click opens picker, drag-
and-drop valid file fires `onFiles`, maxSize triggers `onError`,
multiple=false rejects multi-drop, accept rejects mismatch + accepts
matching MIME, disabled ignores drops, aria-describedby contains both
ids, className merge, forwardRef points at the dropzone div,
drag-enter / drag-leave toggles `data-active`, Enter key opens picker,
stable displayName.

`npx vitest run web/src/components/ui/file-input.test.tsx` was
**skipped** in the worktree (`web/node_modules` not installed); the
test file is committed and runs against a populated install.

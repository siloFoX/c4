## Summary

New `ChartLineGmma` UI primitive at
`web/src/components/ui/chart-line-gmma.tsx` -- a pure-SVG line chart
with a Guppy Multiple Moving Average ribbon of short and long EMA
groups.

## Indicator math

Daryl Guppy's Multiple Moving Average. The price is run through two
groups of exponential moving averages:

- a **short / trader group** of short-period EMAs (Guppy default
  `3, 5, 8, 10, 12, 15`);
- a **long / investor group** of long-period EMAs (Guppy default
  `30, 35, 40, 45, 50, 60`).

The result is a *ribbon* of EMA lines. The signal is the
**relationship between the two ribbons**, not any single line. For
each bar, with `shortMin/shortMax` and `longMin/longMax` the
envelopes of the two groups:

- `shortMin > longMax` -- the short ribbon holds entirely above the
  long ribbon: **bullish**.
- `shortMax < longMin` -- the short ribbon holds entirely below:
  **bearish**.
- otherwise the ribbons overlap: **crossing** (a trend change).
- if any EMA in either group is still in warm-up: **forming**.

Each EMA is the standard SMA-seeded EMA, defined from index
`period - 1`, so a bar is `forming` until the longest period's EMA
is defined.

### Worked fixture

`values` is an 18-point linear ramp (`value[i] = 10 + 2i`), short
periods `[1, 3]`, long periods `[7, 15]`. The EMA of a ramp with
step `d` settles exactly to `value - d*(period-1)/2`, so with step 2
each `EMA-k = value - (k-1)`:

```
short: EMA-1 = value,      EMA-3 = value - 2
long:  EMA-7 = value - 6,  EMA-15 = value - 14
```

The short ribbon sits entirely above the long ribbon, so once all
four EMAs are defined (from index 14) the state is `bullish`. Every
EMA value is exact, so the tests assert bands and EMA values with
`toBe` / `toEqual`. A descending ramp yields `bearish` and a flat
series yields `crossing`.

## API

- `getLineGmmaFinitePoints(points)` -- drop points with a non-finite
  `x` or `value`.
- `normalizeLineGmmaPeriods(periods, fallback)` -- coerce a period
  list to ascending, deduped positive integers; an empty result or
  non-array falls back.
- `computeLineGmmaEma(src, period)` -- a single SMA-seeded EMA
  series.
- `computeLineGmma(values, shortPeriods, longPeriods)` -- returns
  `{ short, long }`, one EMA series per period in each group.
- `runLineGmma(points, options?)` -- sort finite points by `x`,
  compute every EMA, return `{ series, shortPeriods, longPeriods,
  short, long, samples, bullishCount, bearishCount, crossingCount,
  ok }`. Each sample carries the two ribbon bands and the
  `bullish` / `bearish` / `crossing` / `forming` state. `ok` is
  false for fewer than two points.
- `computeLineGmmaLayout(options)` -- project the price and every
  ribbon line onto one shared panel. Returns the panel, ticks, the
  price path, price dots, and `shortLines` / `longLines` (each a
  `{ period, path }`).
- `describeLineGmmaChart(data, options?)` -- ARIA description.

## Component

`ChartLineGmma` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, axis ticks and grid, a
config badge calling out the group sizes, the short ribbon in one
colour and the long ribbon in another, a three-series legend
(Price / Short / Long) with toggle, a hover/focus tooltip exposing
the two ribbon bands and the state, and a
`motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute; each ribbon line carries `data-period`.

## Distinctness

- **vs every single-line moving average** (chart-line-ema,
  chart-line-dema, chart-line-tema, chart-line-wma,
  chart-line-trima, chart-line-t3, chart-line-vidya, ...) -- those
  plot **one** average line and read the trend from where the price
  sits relative to it. The GMMA plots a **ribbon of twelve EMAs in
  two groups** and reads the trend from the **separation between the
  groups** -- it is the only moving-average primitive whose signal
  is a group-versus-group relationship rather than a single line.

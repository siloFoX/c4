# 11.164 -- UI Image primitive

## Why

Three product surfaces (Avatar today, future Workspace / Wiki thumbnail
tiles, and meeting attachment previews) all need the same loading
behaviour: a muted placeholder that pulses while the bytes arrive,
fade-in once `onLoad` fires, and a graceful fallback on `onError`
instead of a broken-image glyph. Wiring this from scratch each time has
already produced three slightly different fade durations and two
different error-state colours. The Image primitive consolidates the
behaviour in one component so subsequent adopters get the same look for
free.

The lazy-load path (default on) also closes a real perf gap: long
thumbnail lists (Workspaces, future Wiki gallery) currently issue a
network request for every avatar in the DOM even when scrolled out of
view.

## What changed

### `web/src/components/ui/image.tsx`

`forwardRef<HTMLDivElement, ImageProps>`. Props:

- `src: string` -- required.
- `alt: string` -- required for a11y; empty string allowed for purely
  decorative usage so callers must opt in explicitly.
- `aspect?: 'square' | '4/3' | '16/9' | 'auto'` -- default `'auto'`.
  Maps to `aspect-square` / `aspect-[4/3]` / `aspect-video` Tailwind
  classes; `'auto'` leaves the wrapper at its intrinsic size.
- `width? / height?: string | number` -- inline-style overrides; numeric
  values get a `px` suffix.
- `placeholderColor?: string` -- pre-load wrapper background. Anything
  starting with `bg-` is forwarded as a class; otherwise it becomes
  `style.backgroundColor`. Default is the `bg-muted` token used by the
  rest of the design system.
- `fallback?: ReactNode` -- custom error slot. Wins over
  `fallbackInitials`.
- `fallbackInitials?: string` -- centred initials in a muted tile when
  `fallback` is omitted.
- `lazy?: boolean` -- default `true`. When `true` the underlying
  `<img>` is not rendered (and therefore not requested) until the
  wrapper enters the viewport, observed via `IntersectionObserver` with
  `rootMargin: '50px'`. Observer disconnects after the first
  intersection. `lazy=false` renders the `<img>` immediately, matching
  the eager semantics callers expect for above-the-fold imagery.
- `rounded?: 'sm' | 'md' | 'lg' | 'full' | 'none'` -- default `'md'`.
- `className?: string` -- merged onto the outer wrapper.

Behaviour:

- Outer wrapper is a `<div data-ui="image">` with `relative
  overflow-hidden`, the aspect class, the rounded class, the
  placeholder class, and `animate-pulse` while the image is in-view but
  not yet loaded.
- Inner `<img>` only mounts after `inView` is true. It starts at
  `opacity-0` and transitions to `opacity-100` on the `load` event,
  giving the fade-in. `onError` flips an `error` state that swaps the
  body for the configured fallback ladder
  (`fallback` -> `fallbackInitials` -> `ImageOff` lucide icon).
- `data-image-state` exposes `loading | loaded | error` on the wrapper
  for tests + future skin overrides.
- `useEffect` resets `loaded` / `error` when `src` changes so reusing
  the same wrapper for a different image starts the cycle over.

### `web/src/components/ui/image.test.tsx`

15 RTL cases under vitest. `IntersectionObserver` is stubbed via
`vi.stubGlobal` with a fake whose constructor stores the callback so
the test can call it manually -- this lets us assert both the "no img
rendered before observation" and "img receives src after observation"
states deterministically. Other cases cover the three aspect classes,
eager mode, the `load` opacity transition, all three error-fallback
branches, alt forwarding, the `rounded` prop, `forwardRef` plumbing,
className merge, and the pulse-while-loading lifecycle.

### Avatar adoption (`web/src/components/ui/avatar.tsx`)

When `src` is provided, the Avatar now renders an `<Image>` inside the
existing rounded-full span instead of its own one-off `<img>` + error
useState. `lazy={false}` preserves the eager-load semantics callers
expect (avatars are typically already in view above the fold).
`fallbackInitials={avatarInitials(name)}` reuses the existing
initials-fallback path through the Image, so an avatar whose URL 404s
now renders the same initials chip it always has -- just via the
shared Image fallback ladder. The initials-only path (no src) is
untouched. The outer span stays role-less when src is present so
`getByRole('img', { name })` keeps resolving to the inner `<img>` and
the existing avatar.test.tsx assertions stay green.

## Future adoption

- Workspaces / Wiki thumbnail tiles (`lazy=true` + `aspect='16/9'` for
  the gallery card grid, `fallbackInitials` from the workspace name).
- Meeting attachment previews (`aspect='4/3'`, custom `fallback`
  rendering the existing file-type chip for non-image attachments).
- Account menu profile picture (drop the bespoke `<img>` once the
  fade-in lands behind a feature flag).

## Out of scope

- Native `<img loading="lazy">` attribute. The IntersectionObserver
  approach is intentional: it lets us pulse the placeholder + run the
  fade transition, which the native attribute cannot coordinate with.
- Progressive/blurhash placeholders. The current placeholder is a flat
  muted tile -- adopters that need a real low-res preview can pass
  their own `placeholderColor` for now and a follow-up patch can wire
  a `blurDataURL` prop.

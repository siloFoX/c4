## Summary

Add a new `<ChartBubble>` UI primitive in
`web/src/components/ui/chart-bubble.tsx` -- pure-SVG
bubble chart. Each datum carries three numeric dimensions:
`x` (x-axis position), `y` (y-axis position), and `size`
(drives bubble radius via sqrt-area scaling). Optional
per-point `category` drives palette-based color encoding
with a focusable `<button aria-pressed>` legend per
category that toggles visibility. Distinct from
`<ChartScatter>` (11.444) in that bubbles have a 3rd
encoded variable (size) instead of uniform dot radius,
and distinct from `<ChartDotPlot>` (11.465) in that the
x axis is numeric here rather than categorical.

## API

```ts
interface ChartBubblePoint {
  id: string;
  label?: string;
  x: number;
  y: number;
  size: number;
  category?: string;
  color?: string;
}

interface ChartBubbleProps {
  data: readonly ChartBubblePoint[];
  width?: number;             // default 560
  height?: number;            // default 360
  padding?: number;           // default 40
  minRadius?: number;         // default 4
  maxRadius?: number;         // default 28
  tickCount?: number;         // default 5
  xMin?: number;              // (controlled bound override)
  xMax?: number;
  yMin?: number;
  yMax?: number;
  hiddenCategories?: readonly string[];
  defaultHiddenCategories?: readonly string[];
  onHiddenCategoriesChange?: (hidden: string[]) => void;
  showLegend?: boolean;       // default true (when categories present)
  showTooltip?: boolean;      // default true
  showAxisTicks?: boolean;    // default true
  showGrid?: boolean;         // default true
  showXLabel?: boolean;       // default true
  showYLabel?: boolean;       // default true
  xLabel?: string;
  yLabel?: string;
  animate?: boolean;          // default true
  className?: string;
  ariaLabel?: string;         // default 'Bubble chart'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  formatSize?: (v: number) => string;
  legendPlacement?: 'right' | 'bottom'; // default 'bottom'
  onBubbleClick?: (args: {
    point: ChartBubblePoint;
    layout: ChartBubbleLayoutPoint;
  }) => void;
  onCategoryToggle?: (args: { category: string; hidden: boolean }) => void;
  style?: CSSProperties;
}
```

## Behaviour

- Renders `<div role="region">` wrapping
  `<svg role="img">` with the same `aria-label`.
- Layout: `computeBubbleLayout` walks every point,
  drops non-finite-x / non-finite-y entries,
  skips entries whose category or id is in the
  `hidden` set, then projects `(x, y)` into the
  inner canvas. SVG-y-down convention: `yMax` paints
  at the top (cy=padY) and `yMin` at the bottom
  (cy=padY+innerH).
- Size encoding: `getBubbleRadius` does sqrt-area
  interpolation between `minRadius` and `maxRadius`.
  Collapsed size range falls back to `(min+max)/2`.
  Non-finite / negative size -> 0 -> `minRadius`.
- Bounds: auto-derived via `getBubbleBounds`; can be
  overridden with `xMin` / `xMax` / `yMin` / `yMax`
  props. Collapsed range expands by +/- 0.5.
- Category color: when present, follows palette order
  (`getBubbleCategoryColor`). Per-point `color`
  override beats palette. Uncategorized points use
  the default color (#2563eb).
- Hover / focus on a bubble sets the hovered id and
  opens a tooltip (label / category / x / y / size,
  all formatted).
- Legend (when categories are present): focusable
  `<button aria-pressed>` per category. Controlled
  via `hiddenCategories` + `onHiddenCategoriesChange`,
  or uncontrolled via `defaultHiddenCategories`.
  `onCategoryToggle({category, hidden})` fires on
  click.
- Mount animation:
  `motion-safe:animate-fade-in` per bubble.
  `animate=false` removes the class.
- Default ARIA description: `"Bubble chart with N
  bubbles[ across M categories]. x range <xMin> to
  <xMax>; y range <yMin> to <yMax>."` Empty / all-
  invalid data returns `"No data"`; `ariaDescription`
  overrides.

## ARIA + data attributes

- Root: `<div role="region">` + `aria-label`.
- SVG: `role="img"` + same `aria-label`.
- sr-only span:
  `data-section="chart-bubble-aria-desc"`.
- Each bubble circle: `role="graphics-symbol"` +
  `tabIndex={0}`. aria-label = `"<label>: x <x>, y
  <y>, size <size>"`.
- Legend button: `<button aria-pressed>` per category.
- `data-section` on every node (chart-bubble /
  -canvas / -aria-desc / -svg / -grid / -grid-line /
  -axes / -axis / -ticks / -tick / -tick-label /
  -x-label / -y-label / -bubbles / -point / -circle
  / -tooltip / -tooltip-label / -tooltip-category /
  -tooltip-x / -tooltip-y / -tooltip-size /
  -legend / -legend-item / -legend-button /
  -legend-swatch / -legend-label).
- Root mirrors `data-bubble-count`,
  `data-visible-count`, `data-category-count`,
  `data-animate`.

  `data-point-size`, `data-point-color`,
  `data-point-category`, `data-point-r`,
  `data-hovered`.

  `data-category-hidden`.

## Tests (65 cases)

- Constants exported.
- `getBubbleDefaultColor`: palette entry, modulo
  wrap, invalid -> color 0.
- `getBubbleBounds`: empty -> (0..1, 0..1, 0..0);
  multi-point min/max; collapsed x/y expansion;
  non-finite x/y ignored.
- `getBubbleRadius`: min at 0; max at upper bound;
  sqrt interpolation; collapsed range fallback;
  non-finite / negative -> 0.
- `getBubbleCategories`: unique non-empty in first-
  seen order; empty / undefined skipped.
- `getBubbleCategoryColor`: palette lookup;
  unknown / null -> default.
- `getBubbleTicks`: count evenly-spaced; collapsed
  -> [min]; count clamp.
- `computeBubbleLayout`: one entry per visible
  valid point; skips hidden categories + ids; skips
  non-finite x/y; cx maps xMin/xMax to padX /
  padX+innerW; cy maps yMax/yMin to padY /
  padY+innerH (SVG y-down); radius sqrt-scaled;
  default color when no category; per-point color
  override beats palette; non-positive inner -> [].
- `describeBubbleChart`: empty / all-invalid ->
  "No data"; count + ranges + categories;
  formatValue honored.

## Out of scope

- Pan / zoom on the canvas (host owns viewport).
- Animated reflow on data change (snap v1; mount
  fade only).
- Quadrant overlays / mean lines / regression lines
  (host wraps with custom SVG layer using exported
  scale helpers).
- Density / hexbin variant for very large data sets
  (v1 paints one circle per point; host pre-bins
  if needed).

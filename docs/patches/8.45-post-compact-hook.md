# 8.45 - Auto-compact detection + rule auto-reinject (post-compact hook)

## Root cause

Claude Code auto-compacts a worker's conversation once context usage
crosses its internal threshold. Compaction replaces the prior turns
with a short summary so the assistant keeps going without hitting the
hard context limit. The side effect, in c4 managers, is that the
halt-prevention rules delivered at session start (no compound
commands, `git -C` not `cd X && git`, no `sleep`-based polling,
approval protocol, merge criteria, anti-spawn discipline) are pruned
out of the summary the model keeps in memory. The next few actions
therefore look like the failure modes 5.28 and 7.26 were supposed to
eliminate: an operator approving `rm -rf`, a worker piping `cd ... &&
git reset --hard origin/main`, or a manager spawning a duplicate
worker because it "forgot" one already existed.

The existing `PostCompact` hook in worker `.claude/settings.json`
only re-displays `CLAUDE.md` and pings the daemon so `compactEvent()`
increments a counter. The counter triggers manager rotation once a
threshold is reached but does nothing for the 0-to-N-1 compactions
before that. It also never re-delivers the rules themselves to the
worker's running session - nothing currently exists to make the
model re-ingest them.

## Design

A dedicated pure-logic module, `src/post-compact-hook.js`, owns the
lifecycle:

- `detectCompactEvent(chunk, state)` runs on every PTY chunk emitted
  to the worker. It matches a short, conservative list of Claude
  Code surface strings (`Context compacted`, `Compacting
  conversation`, `/compact complete`, etc.) and debounces repeat
  matches per worker (default 60s) so a marker that survives into
  the scrollback tail does not re-fire.
- `resolveWorkerType(worker)` picks the rule tier: `manager` when
  `_autoWorker` is set or `tier === 'manager'`, `attached` for
  session-attach imports, `worker` otherwise.
- `getRuleTemplate(type, { templateDir })` loads
  `<templateDir>/<type>-post-compact.md`. Unknown roles fall back to
  the `worker` template so unusual tiers still receive something.
- `injectRules(manager, workerName, opts)` composes a banner + the
  template body and hands it to `manager.send(name, payload, false)`.
  The manager reference is passed in rather than imported so tests
  can stub `send` and `workers.get`.
- `detectDrift(command)` + `updateDriftWindow(state, command)` arm a
  rolling audit of the first `driftWindow` (default 3) Bash commands
  that follow a compact. Forbidden patterns (`&&`, `||`, `|`, `;`
  followed by a word, `cd ... git`, `sleep`, `for`, `while`) match.

### Rule templates

Three markdown files ship under `docs/rules/`:

- `manager-post-compact.md` - halt-prevention rules, approval
  protocol, merge criteria, anti-spawn checklist.
- `worker-post-compact.md` - halt-prevention rules, task discipline,
  merge prep.
- `attached-post-compact.md` - short form for attached sessions that
  do not own a worktree.

Each template ends with an explicit ack sentence so the daemon can
tell whether the re-injection landed.

### Wiring into `pty-manager`

1. `src/pty-manager.js` requires `./post-compact-hook` once at the
   top. Each worker gains a `_postCompactState` struct tracking
   `lastFiredAt`, a drift window, and an ack deadline.
2. The PTY `onData` handler calls `_handlePostCompactChunk(worker,
   chunk)` right after `screen.write(data)`. That runs the debounced
   regex scan and, if it fires, enqueues an async injection.
3. `compactEvent(name)` (the existing curl endpoint called by the
   PostCompact hook in worker settings.json) also calls
   `_triggerPostCompactInjection(name, 'hook')`. Both sources share
   the same injection path so the re-injection happens whether the
   hook fired via PTY tail or via the worker's settings.json curl.
4. `_triggerPostCompactInjection` loads config, calls
   `postCompactHook.injectRules`, stamps `lastInjectionAt`, arms the
   drift window via `armDriftWindow`, and schedules a
   `verifyTimeoutMs` timer. When the timer elapses without seeing
   `rules received` in subsequent output, a warning snapshot lands
   and `notifications.pushAll` fires.
5. The `isPermissionPrompt` branch in `onData` calls
   `_inspectPostCompactDrift(worker, bashCommand, proc, screenText)`
   when the prompt is a Bash command. If the worker is inside its
   drift window and the command matches a forbidden pattern, the
   hook writes the deny keystrokes, flips the worker to
   `critical_deny`, and fires a second (stronger) re-injection.

### Config knobs (`config.example.json`)

```json
"postCompactHook": {
  "enabled": true,
  "templateDir": "docs/rules",
  "verifyTimeoutMs": 10000,
  "debounceMs": 60000,
  "driftWindow": 3
}
```

All knobs have safe defaults baked into the module so operators who
do not update `config.json` still get the feature enabled with the
documented behavior.

## Tests

`tests/post-compact-hook.test.js` adds 40 node:test assertions:

- Regex coverage for every marker the module claims to handle plus
  negative cases (empty, unrelated text, Buffer input).
- Debounce before/after the window.
- Worker-type resolution across `_autoWorker`, `tier`, `kind`.
- Template resolution, worker-fallback path, and default-dir lookup.
- Banner/payload format.
- `injectRules` wiring against a stubbed manager (send args, snapshot
  side-effect, missing worker, missing template, attached tier).
- Drift detection for each pattern family.
- Drift window lifecycle (arm, observe, close after N, accumulate
  matches).

Full suite: 112 passed, 0 failed.

## Operational notes

- When Claude Code introduces a new compact-completion surface
  string, add the regex to `COMPACT_MARKERS`. Keep the list small -
  false positives would trigger redundant rule injections.
- The drift window is intentionally narrow (3 commands). A wider
  window risks blocking legitimate compound pipelines in profiles
  that allow them. The manager template explicitly forbids those
  patterns so the detector is a safety net, not the main enforcement.
- The ack-timeout path only warns; it never kills the worker. If the
  operator wants auto-close behavior, layer it on top of the warning
  surface (`[C4 POST-COMPACT WARN] ... did not ack rules`).

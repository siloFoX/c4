## Summary

`web/src/components/ui/chart-line-rocket-rsi.tsx` ships
`<ChartLineRocketRsi>`, a pure-SVG two-panel chart with a **Rocket
RSI** panel applying the Fisher Transform to a double-smoothed RSI to
sharpen turning points. The top panel plots the price; the bottom
panel plots the rocket reading with a horizontal zero line and
zone-coloured markers (green bull bias, red bear bias).

## Computation

The pipeline:

```
1. Standard RSI of close-to-close changes over `rsiPeriod`.
   A flat window (no gains, no losses) yields RSI = 50.
2. SMA-smooth the RSI with length `smooth1`.
3. SMA-smooth that with length `smooth2`.
4. Normalize to [-1, 1]: x = RSI / 50 - 1.
5. Clamp x to [-CLAMP, CLAMP] so the Fisher Transform stays finite.
6. Fisher Transform: rocket = 0.5 * ln( (1 + x) / (1 - x) ).
```

The Fisher Transform stretches values near the boundaries of the
input range -- a price stuck at an extreme RSI keeps producing
extreme rocket-RSI readings, so reversals stand out sharply.

## Worked anchors

- **Fisher(0) = 0 bit-exact** -- the defining algebraic anchor.
  `(1 + 0) / (1 - 0) = 1`, `ln(1) = 0`, `0.5 * 0 = 0`. Asserted
  with `toBe(0)`.
- **Constant-series anchor** -- a constant series has every
  bar-to-bar change equal to zero, so `sumGain = sumLoss = 0` and
  the standard RSI by convention returns 50. Two SMAs of a
  constant 50 stay at 50 bit-exact. Normalize gives `x = 50/50 - 1
  = 0`. Fisher(0) = 0. So the Rocket RSI is exactly 0 at every
  defined bar of a constant series. Asserted with `toBe(0)` for
  every defined bar.
- **Fisher(0.5) = 0.5 * ln(3)** -- the worked transcendental
  anchor. `(1 + 0.5) / (1 - 0.5) = 1.5 / 0.5 = 3`, `0.5 * ln(3)`.
  Asserted with `toBeCloseTo(0.5 * Math.log(3), 12)`.
- **Fisher antisymmetry within ULP** -- `Fisher(-x)` is the
  mathematical negative of `Fisher(x)` but `ln(1/y)` and `-ln(y)`
  agree only to ULP in IEEE 754, so the test uses
  `toBeCloseTo(_, 12)`.
- **Fisher clamp** -- inputs above 1 saturate at the clamp; inputs
  below -1 saturate at -clamp. The clamp parameter controls the
  saturation magnitude.
- **Rising / Falling antisymmetry** -- a strictly rising series
  gives RSI = 100 at every defined bar -> Fisher of clamped +1; a
  strictly falling series gives RSI = 0 -> Fisher of clamped -1.
  The two are antisymmetric within ULP.
- **RSI shape on the three classic regimes** -- constant -> 50,
  rising -> 100, falling -> 0 at every defined bar (the standard
  RSI by-convention behaviour).
- **SMA bit-exact identities** -- the SMA of a constant series is
  the constant bit-exact; the SMA of `[1..5]` window 3 is exactly
  `[null, null, 2, 3, 4]`; a null in the window propagates to the
  output.

## Distinct from neighbouring primitives

- vs `<ChartLineRsi>` -- the standard RSI is bounded in `[0, 100]`
  and changes hands at 30 / 70; the Rocket RSI is a Fisher-
  Transformed signed reading whose extreme values stretch under
  the transform, so turning points stand out sharper.
- vs `<ChartLineBayesian>` -- the Bayesian Trend yields three
  calibrated probabilities; the Rocket RSI is a single signed
  reading. The Bayesian Trend is probabilistic; the Rocket RSI is
  deterministic.
- vs `<ChartLineCmo>` -- the CMO is
  `100 * (sumUp - sumDown) / (sumUp + sumDown)`, a signed
  unsigned-share with no transform; the Rocket RSI is the
  Fisher-stretched cousin of the standard unsigned share.
- vs `<ChartLineMahalanobis>` -- the Mahalanobis Distance is an
  unsigned multivariate z-score; the Rocket RSI is a signed
  univariate transform.

## API

`data` -- a single `ChartLineRocketRsiPoint[]` (`{x, value}`).
`rsiPeriod` (default 8, integer >= 1). `smooth1` / `smooth2`
(defaults 5, integer >= 1). `clamp` (default 0.999, finite in
`(0, 1)`). Standard sizing / colour / a11y props plus
`pricePanelRatio` (default 0.55, clamped to `[0.3, 0.8]`), `gap`,
`strokeWidth`, `dotRadius`, `tickCount`. Toggle flags: `showAxis`,
`showGrid`, `showDots` (default false), `showRocket`, `showZeroLine`,
`showMarkers`, `showTooltip`, `showConfigBadge`, `showLegend`,
`animate`. Controlled and uncontrolled visibility via
`hiddenSeries` (`'price' | 'rocket'`). Callbacks:
`onPointClick({point})`, `onSeriesToggle({seriesId, hidden})`.
Format functions: `formatValue`, `formatX`. ARIA: `ariaLabel`,
`ariaDescription`.

## Pure helpers

- `DEFAULT_CHART_LINE_ROCKET_RSI_*` -- defaults.
- `getLineRocketRsiFinitePoints` -- keep only points with a finite
  x and value.
- `normalizeLineRocketRsiInteger(value, fallback, min = 1)` --
  integer >= min; else fallback.
- `normalizeLineRocketRsiClamp(value, fallback)` -- finite in
  `(0, 1)`; else fallback.
- `computeLineRocketRsiRSI(values, period)` -- standard RSI; flat
  window -> 50; warm-up bars null.
- `computeLineRocketRsiSmooth(values, window)` -- SMA over a window
  with null-propagation.
- `computeLineRocketRsiFisher(x, clamp?)` -- Fisher Transform with
  clamping; `Fisher(0) === 0` bit-exact.
- `classifyLineRocketRsiZone(value)` -- `'up' | 'down' | 'flat' |
  'none'` by sign.
- `computeLineRocketRsi(values, options?)` --
  `{rsi, smoothed, rocketRsi}`.
- `runLineRocketRsi(points, options?)` --
  `{series, rsiPeriod, smooth1, smooth2, clamp, rsi, rocketRsi,
  samples, rocketFinal, upCount, downCount, flatCount, ok}`.
- `computeLineRocketRsiLayout({data, rsiPeriod, smooth1, smooth2,
  clamp, ...})` -- two panels with the price line on top and the
  rocket line below in a symmetric domain `[-bound, +bound]`, one
  marker per defined-rocket bar coloured by zone, and `zeroY` for
  the zero line.
- `describeLineRocketRsiChart(data, options?)` -- ARIA summary
  mentioning the Fisher Transform, the double smoothing, the
  lookback periods and the zone counts.

## a11y / data hooks

Root `<div role="region">` with `aria-label` + `aria-describedby`; the
SVG is `role="img"` with the same label; an sr-only
`data-section="chart-line-rocket-rsi-aria-desc"` span carries the
description. The price line, every price dot, the rocket line and
every marker carry `role="graphics-symbol"` and `tabIndex={0}`.
Every node carries a `data-section`. Root mirrors `data-rsi-period`,
`data-smooth1`, `data-smooth2`, `data-clamp`, `data-rocket-final`,
`data-up-count`, `data-down-count`, `data-flat-count`,
`data-total-points`, `data-animate`, `data-empty`. Grid / axis /
tick / panel-label nodes carry `data-panel="price"|"rocket"`;
markers expose `data-zone` and `data-rocket-rsi`.

# 11.156 -- UI: Skeleton composable shape variants (v1.11.174)

## Summary

Polish the existing `<Skeleton>` primitive in
`web/src/components/ui/skeleton.tsx` by adding six small composable
shape exports that loading states can assemble structurally instead of
reaching for ad-hoc inline `animate-pulse rounded-md bg-muted/60`
blocks. Base `<Skeleton>` is preserved verbatim; the new shapes share
its `animate-pulse bg-muted` base classes.

## New exports

- `TextLine` -- `{ width = '100%'; height = '0.875em'; className? }`,
  horizontal pulse bar sized for a single line of text. `forwardRef`.
- `Rect` -- `{ width?; height?; rounded = 'md'; className? }`,
  rounded `sm | md | lg | xl | full`. `forwardRef`.
- `Circle` -- `{ size = '2.5rem'; className? }`, rounded-full square.
- `AvatarShape` -- `{ size = 'md'; className? }`, rounded-full sized to
  `sm = 1.5rem` / `md = 2.5rem` / `lg = 3.5rem` (Tailwind class hints
  `h-6 w-6` / `h-10 w-10` / `h-14 w-14`). Renamed from the spec's
  `Avatar` to avoid the name clash with the existing `<Avatar>`
  primitive in `web/src/components/ui/avatar.tsx`.
- `StatCardShape` -- pre-built composition of three `Rect`s (label /
  number / delta) sized to mirror the `StatCard` primitive layout.
- `TableRowShape` -- `{ columns = 5; className? }`, renders one row of
  N `Rect`s with widths cycling `60% / 100% / 45% / 70% / 30%`.

The base `Skeleton` component, its `SkeletonVariant` union, and the
`v1.11.135` legacy variant aliases are unchanged.

## Adoption sites

1. `web/src/pages/Health.tsx` -- the `loading && !data` branch
   previously rendered `<LoadingSkeleton rows={3} />` from `PageFrame`.
   Replaced with the actual page structure: a 3-column
   `<DashboardGrid>` of `<StatCardShape>`s plus a 6-tile
   `<Panel>` grid each holding a 2-column `<TableRowShape>` (mirrors
   the post-load `Stat` panels).
2. `web/src/pages/Auto.tsx` -- the queue section's `loading` branch
   previously rendered `<BlockSkeleton rows={6} />`. Replaced with a
   stack of six `<TableRowShape columns={5}>` rows wrapped in the same
   `role="status"` container.
3. `web/src/pages/Auto.tsx` -- the active-workers section's `loading`
   branch previously rendered three `h-28 w-56 animate-pulse` blocks.
   Replaced with three card-shaped containers each composed of one
   `<AvatarShape size="md">` + three `<TextLine>`s of varying widths
   (mirrors the post-load WorkerCard layout: avatar + name + status
   + branch).

The originally listed `WorkerList`/`Workers` target has no loading
state in the current codebase (skeleton block never existed there).
Per the task's "Do not introduce new loading states" rule, the
adoption substitutes the Auto page's existing workers-loading slot.

## Tests

`web/src/components/ui/skeleton.test.tsx` extended (old `Skeleton`
cases preserved verbatim) with 23 new cases across the six shapes:
TextLine default/custom width, className merge, `forwardRef` exposes
DOM node; Rect default `rounded-md`, `rounded='full'`, string +
number width/height, className merge, `forwardRef`; Circle w/h
matches size prop (string + number), className merge; AvatarShape
sm/md/lg class assertions + `data-avatar-size` attribute, className
merge; StatCardShape 3-Rect children + className merge; TableRowShape
default 5 columns, custom 3 columns, className merge, width-pattern
cycling over 7 columns.

## Out of scope

- Base `Skeleton` component, its variant union, and the legacy
  `LoadingSkeleton` block in `pages/PageFrame.tsx` are intentionally
  untouched.
- ARPS design-system surface tokens (`--surface-2` / `--surface-panel`)
  remain not wired into this bundle; the shapes stay on the project's
  existing `bg-muted` Tailwind class. Reference:
  `arps-design-system-v1/`.
- Auto page's local `BlockSkeleton` helper is left in place (used by
  other sections like the dispatch timeline); only the structural
  shapes inside the queue + workers slots were swapped.

# 11.176 - UI: DatePicker primitive

Version: v1.11.194
Branch: c4/auto-ui-date-picker

## Scope

- New primitive `web/src/components/ui/date-picker.tsx` exporting
  `DatePicker` (single mode) and `DateRangePicker` (two-month range mode).
- Native `Date` only, no external date dependency. Reuses the existing
  `<Popover>` primitive for the calendar surface.
- Date helpers in `web/src/lib/date-format.ts`
  (`toISODate` / `parseISODate` / `isSameDay` / `addDays` /
  `addMonths` / `addYears` / `startOfMonth` / `endOfMonth`).

## Behaviour highlights

- Read-only trigger button (forwarded ref) labelled with the
  formatted `YYYY-MM-DD` value or placeholder, plus a Calendar icon
  from `lucide-react`.
- Calendar header: prev/next month (ChevronLeft / ChevronRight), prev/next
  year (`<<` / `>>`), centered month + year label.
- Month grid: 7 columns, weekday header (Su..Sa), today ring,
  selected fill (`bg-primary`), inter-range tint for range mode,
  outside-month dimming, `min`/`max` cells rendered disabled +
  `aria-disabled` + `pointer-events-none`.
- Keyboard navigation (when popover is open):
  - ArrowLeft / ArrowRight: -1 / +1 day
  - ArrowUp / ArrowDown: -7 / +7 days
  - PageUp / PageDown: -1 / +1 month (Shift = year)
  - Home / End: first / last day of the focused week
  - Enter: select the focused day (close popover)
  - Escape: close popover (delegated to the Popover primitive)
- Range mode: clicking selects `from` then `to`; out-of-order pairs
  are swapped before being emitted.

## Adoption (2 sites)

1. `web/src/components/HistoryView.tsx` - filter row replaced the prior
   pair of `<Input type='date'>` for `sinceDay` / `untilDay` with a
   single `<DateRangePicker>`. State stays as `YYYY-MM-DD` strings so
   the `useHistorySummary` hook contract is unchanged. New i18n key
   `history.filter.range.label` for `en` and `ko`.
2. `web/src/pages/Health.tsx` - the Filters Drawer gains a placeholder
   `Since` `<DatePicker>` alongside the previously placed
   `<NumberInput>` poll-timeout control. Local state; not yet threaded
   into `useHealth`.

# 11.578 -- web/src/components/ui/chart-line-stochastic.tsx

Pure-SVG line chart with a **stochastic oscillator**
sub-panel. The price line sits in the top panel; the bottom
panel plots the **%K** and **%D** lines, bounded to 0-100,
with shaded overbought and oversold zones.

## The stochastic computation

`computeLineStochasticK(values, period)` is George Lane's
%K: for each period it measures where the current value
sits within its trailing high-low range,
`%K = 100 * (value - lowestLow) / (highestHigh - lowestLow)`
over a window of `period` values. A flat window (zero
range) reads 50; the middle of a range reads 50; the top
reads 100, the bottom 0. Indices before the window is full
are `null`.

`computeLineStochasticD(kValues, dPeriod)` is the **%D**
signal line -- a simple moving average of %K over `dPeriod`
values, defined only where the full %K window is itself
defined.

`runLineStochastic` classifies each reading into a **zone**
from %K: `overbought` at or above the overbought threshold
(default 80), `oversold` at or below the oversold threshold
(default 20), `neutral` between.

The component fixture (period 3, %D period 2) makes the math
exact: %K is `[null, null, 100, 0, 0, 100]`, %D is
`[null, null, null, 50, 0, 50]` -- two overbought readings,
two oversold.

## Distinct from neighbouring primitives

Both the stochastic and `chart-line-rsi` are 0-100
oscillators with overbought/oversold zones, but the
**computation is different**: RSI is a gain/loss ratio,
whereas the stochastic is a **range-position** measure --
where the close sits between its recent low and high. The
stochastic also plots **two** lines (%K and its %D signal
SMA), where RSI plots one. `chart-line-macd` is an unbounded
EMA-difference study; `chart-line-momentum` is a derivative
oscillator. None compute the trailing range position.

All exported helpers carry a `LineStochastic` infix so the
`export *` barrel line cannot collide.

## Public surface

- `<ChartLineStochastic data={...} />` -- a single
  `ChartLineStochasticPoint[]` (`{x, value}`).
- `period` / `dPeriod` -- the %K lookback and the %D
  smoothing period (defaults 14 / 3).
- `overbought` / `oversold` -- the zone thresholds (defaults
  80 / 20; clamped to [0, 100]).
- `pricePanelRatio` -- the fraction of the height given to
  the price panel (default 0.58; clamped to [0.3, 0.8]).
- `gap` -- the pixel gap between the stacked panels.
- Toggle flags: `showAxis`, `showGrid`, `showDots` (default
  false), `showK`, `showD`, `showZones`, `showTooltip`,
  `showConfigBadge`, `showLegend`, `animate`.
- Controlled + uncontrolled visibility (`hiddenSeries` keys
  `price` / `k` / `d` / `zones`).
- Standard sizing + format + a11y props. `formatValue` /
  `formatX`.
- `onPointClick({point})` / `onSeriesToggle({seriesId,
  hidden})`.

## Pure helpers

- `getLineStochasticFinitePoints(points)`.

- `computeLineStochasticK(values, period)`.
- `computeLineStochasticD(kValues, dPeriod)`.
- `runLineStochastic(points, options?)`.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- The price line, every price dot and every %K marker
  `role="graphics-symbol"` + `tabIndex={0}`.
- Every node carries `data-section="chart-line-stochastic-..."`.
- Root mirrors `data-period`, `data-d-period`,
  `data-overbought-count`, `data-oversold-count`,
  `data-total-points`, `data-animate`.
- Axis / tick / grid / panel-label nodes carry
  `data-panel="price"|"stoch"`. %K markers expose `data-k`
  and `data-zone`; threshold lines carry `data-kind`.

## Tooltip / badge / legend

Hovering a price dot or a %K marker shows x, price, %K, %D
and the zone. Config badge: `STOCH k=<period> d=<dPeriod>
ob=<overbought> os=<oversold>`. The legend has four toggle
items (Price / %K / %D / Zones).

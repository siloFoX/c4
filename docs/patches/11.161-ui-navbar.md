# Patch 11.161 -- UI Navbar primitive

Version: v1.11.179 (web)
Branch: c4/auto-ui-navbar

## Summary

Introduce a reusable semantic `<Navbar>` primitive in
`web/src/components/ui/navbar.tsx` and adopt it as the wrapper
for the existing global `AppHeader`. The primitive keeps the
component scope strict -- no design-system token changes, no
other layout components touched.

- `web/src/components/ui/navbar.tsx` (new) -- `<Navbar>` primitive
- `web/src/components/ui/navbar.test.tsx` (new) -- 15 RTL cases

- `web/src/components/layout/AppHeader.tsx` -- wraps the

## API

```ts
interface NavbarProps {
  brand?: ReactNode;
  center?: ReactNode;
  actions?: ReactNode;
  sticky?: boolean; // default false
  variant?: 'plain' | 'bordered' | 'elevated'; // default 'plain'
  className?: string;       // outer header
  innerClassName?: string;  // inner nav container
}
```

Layout:

- Outer `<header role='banner'>` with width + variant classes
- Inner `<nav role='navigation' aria-label='Primary'>` with
  `grid grid-cols-[auto_1fr_auto]` so the center slot is
  predictably centred regardless of brand / actions width
- `sticky=true` adds `sticky top-0 z-30 bg-background/95
  backdrop-blur supports-[backdrop-filter]:bg-background/60`
- `variant='bordered'` adds `border-b border-border`;
  `variant='elevated'` adds `shadow-sm`; `variant='plain'`
  adds neither
- `forwardRef` to the outer `<header>` element

## AppHeader substitution

The minimal substitution keeps every existing IconButton,
Tooltip, AccountMenu, TopTabs, hamburger toggle and badge wire
intact. The original outer `<header className='flex
items-center justify-between ... bg-card'>` is replaced with
`<Navbar variant='bordered' className='rounded-none bg-card'
brand={...} center={<TopTabs ... />} actions={...} />`. The
`bg-card` class continues to land on the outer `<header>` so
the existing `AppHeader` snapshot/test assertions on
`toHaveClass('bg-card')` still pass.

## Tests

`navbar.test.tsx` covers (15 cases):

1. renders brand slot content
2. renders center slot content
3. renders actions slot content
4. outer element exposes `role='banner'`
5. inner element exposes `role='navigation'`
6. `sticky=true` applies the sticky / top-0 / z-30 /
   backdrop-blur classes

8. `variant='bordered'` applies `border-b border-border`
9. `variant='elevated'` applies `shadow-sm` and omits
   `border-b`
10. `variant='plain'` applies neither border nor shadow
11. `className` merges on the outer header
12. `innerClassName` merges on the inner nav container
13. `forwardRef` resolves to the outer `<header>` element
14. `displayName` is `'Navbar'`
15. brand + center + actions render together in one tree

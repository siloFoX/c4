## Summary

Add a new `<LogStream>` UI primitive in
`web/src/components/ui/log-stream.tsx` -- auto-scrolling log
viewer. Each entry rendered with a level chip, a time stamp,
optional component / source tag, and a message body that is
ANSI-color-parsed + search-highlighted. Level filters,
free-text search, a time-range pair, and pause / resume
controls let adopters triage live streams without leaving the
panel.

## API

```ts
type LogLevel = 'trace'|'debug'|'info'|'warn'|'error'|'fatal';

interface LogEntry {
  id: string;
  timestamp: string | number | Date;
  level: LogLevel;
  message: string;
  component?: string;
  source?: string;
}

interface LogTimeRange {
  start?: string | number | Date;
  end?: string | number | Date;
}

interface LogStreamProps {
  entries: LogEntry[];
  levels?: readonly LogLevel[];        // default: all 6
  enabledLevels?: readonly LogLevel[]; // controlled
  defaultEnabledLevels?: readonly LogLevel[];
  onEnabledLevelsChange?: (l: LogLevel[]) => void;

  query?: string;                      // controlled
  defaultQuery?: string;
  onQueryChange?: (q: string) => void;

  isPaused?: boolean;                  // controlled
  defaultPaused?: boolean;
  onPausedChange?: (p: boolean) => void;

  timeRange?: LogTimeRange;            // controlled
  defaultTimeRange?: LogTimeRange;
  onTimeRangeChange?: (r: LogTimeRange) => void;

  autoScroll?: boolean;                // default true
  maxHeight?: number | string;         // default 320
  ariaLabel?: string;                  // default 'Log stream'
  emptyState?: ReactNode;              // default 'No log entries'
  showSearch?: boolean;                // default true
  showTimeRange?: boolean;             // default true
  showClear?: boolean;                 // default false
  onClear?: () => void;
  className?: string;
}
```

## Behaviour

- The list scroll container auto-scrolls to the bottom
  whenever the filtered entry count changes -- unless
  `isPaused` is true. We track the last scrolled length so
  adopters that re-render the array do not retrigger scroll
  on a no-op rerender.
- Level chips are `role="switch"` with `aria-checked`
  mirroring the enabled state; clicking toggles the level in
  the enabled set + fires `onEnabledLevelsChange` with the
  ordered subset (canonical order = `LOG_LEVELS`).
- Search input filters on message + component + source
  (case-insensitive substring match). Matches are wrapped in
  `<mark>` via `highlightLogMatches`.
- Time-range start + end are `<input type="datetime-local">`
  fields that emit string values to `onTimeRangeChange`.
  Clearing a field deletes the corresponding key.
- Pause button swaps aria-label between "Pause stream" /
  "Resume stream" and the icon between Pause / Play.
- Clear button is opt-in via `showClear`; only renders when
  an `onClear` callback is supplied.
- Each entry renders the time (HH:MM:SS.cs), level chip,
  optional component tag, ANSI-color-rendered message body,
  and optional trailing source tag.

## Pure helpers (exported)

```ts
LOG_LEVELS = ['trace','debug','info','warn','error','fatal']
getLogLevelRank(level): number
getLogLevelClass(level): string
formatLogTimestamp(value): string
inLogTimeRange(ts, range?): boolean
filterLogEntries(entries, { enabledLevels?, query?, timeRange? }): LogEntry[]
parseAnsi(input): AnsiSegment[]
highlightLogMatches(text, query?): ReactNode
```

- `parseAnsi` consumes the basic SGR codes most servers
  emit:
  - 30..37 / 40..47 -> fg / bg standard 8-color palette.
  - 90..97 / 100..107 -> bright variants.
  - 1 / 2 / 3 / 4 -> bold / dim / italic / underline.
  - 22 / 23 / 24 -> attribute resets.
  - 39 / 49 -> fg / bg colour resets.
  - 0 (or bare `ESC[m`) -> full reset.
  Unknown codes are no-ops so the parser never throws on
  unexpected input.
- `filterLogEntries` combines the three filters (level set /
  query / time range); whitespace-only query is treated as
  empty.
- `highlightLogMatches` escapes regex specials in the query
  so adopters can search for `$10` or `a.b*` literally.

## Tests (54 cases)

- `LOG_LEVELS` order + `getLogLevelRank` indexing.
- `getLogLevelClass` returns per-level class.
- `formatLogTimestamp` HH:MM:SS.cs shape; empty for falsy /
  unparseable; accepts Date / number / string.
- `inLogTimeRange`: undefined, start bound, end bound, both
  bounds.
- `filterLogEntries`: no options copy; level filter; query
  filter against message / component / source;
  whitespace-only query empty; level + query + time range
  combined.
- `parseAnsi`: plain text, red fg, bold + green composition,
  background colours, italic + underline + dim, unknown
  codes, bare ESC[m reset, empty input.
- `highlightLogMatches`: empty query passthrough,
  case-insensitive wrap, regex-special escape.
- Component: default + custom `ariaLabel`; one row per
  filtered entry; per-entry `data-level`.
- Level toggle deactivates + fires `onChange`.
- Search input narrows rows; highlights query.
- Pause toggles aria-label + `data-paused`; controlled
  `isPaused` pins state.
- Controlled `query` pins the search field.
- Time range start / end filter; clearing a bound emits
  `onTimeRangeChange` with the deleted key.
- Per-entry component + source tags render when supplied.
- Empty entries renders the empty state; custom `emptyState`
  slot wins.
- ANSI-colored message renders one segment per code.
- Clear button fires `onClear` when `showClear`; hidden
  otherwise.
- Root data attrs mirror state.
- `showSearch=false` / `showTimeRange=false` opt-outs.
- `maxHeight` applies as inline style.
- `forwardRef` to the list scroll container.
- Stable `displayName`.
- Auto-scroll: scrollTop bumps to scrollHeight when new
  entries arrive (unpaused); does NOT bump when paused.

## Out of scope

- Server-side log subscription (host owns fetch / SSE /
  WebSocket and passes the resulting `entries` array).
- Virtualisation for very long streams (host wraps in
  `<VirtualizedList>` if the entry count exceeds a few
  thousand).
- Color-256 / true-color ANSI parsing (only the standard
  16-color SGR palette is parsed; truecolor sequences parse
  as resets).
- Persisting filter state across reloads (host owns
  persistence via the controlled props).

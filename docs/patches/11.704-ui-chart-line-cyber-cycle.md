## Summary

New `ChartLineCyberCycle` UI primitive in
`web/src/components/ui/chart-line-cyber-cycle.tsx` -- a pure-SVG two-panel
line chart with an **Ehlers Cyber Cycle** oscillator panel from a
smoothed second difference of price. The top panel plots the price; the
bottom panel plots the Cyber Cycle and its trigger.

## The computation

John Ehlers' Cyber Cycle extracts the cyclic swing of the market. It is a
close-only indicator, so `ChartLineCyberCyclePoint` carries `{x, value}`.

- `computeLineCyberCycleSmooth(values)` -- the four-tap Ehlers smoother
  `(value + 2*value[-1] + 2*value[-2] + value[-3]) / 6`, defined from the
  fourth bar.
- `computeLineCyberCycle(values, alpha)` -- feeds the **second
  difference** of the smoothed price (`smooth[i] - 2*smooth[i-1] +
  smooth[i-2]`) into a two-pole resonant filter
  `cycle = (1 - 0.5a)^2 * secondDiff + 2(1-a)*cycle[-1] -
  (1-a)^2*cycle[-2]`, whose feedback turns the impulse into an
  oscillation around zero. While the smoother warms up (the first seven
  bars) the cycle uses the raw price second difference, quartered. The
  trigger is the cycle delayed one bar.
- `classifyLineCyberCycleZone(cycle)` -- marks a bar **up** above zero,
  **down** below, **flat** at zero.

## Worked fixture

The resonant filter and the smoother's `/6` make the cycle transcendental
past the warm-up, but two anchors are exact.

A **short series** (six bars, all inside the smoother warm-up) computes
the cycle as the raw price second difference quartered --
`(value - 2*value[-1] + value[-2]) / 4` -- pure integer arithmetic. The
fixture `[40,48,60,56,72,64]` gives `[0,0,1,-4,5,-6]`, asserted with
`toEqual`, and the trigger lags it to `[null,0,0,1,-4,5]`.

A **constant series** leaves every second difference -- raw and smoothed
-- at zero, so the cycle is exactly 0 across both the warm-up and the
resonant path; the test asserts every cycle reading is 0. The smoother is
checked with `[6,12,18,24,30,36]` -> `[null,null,null,15,21,27]`, and the
zone classifier is tested directly.

## API

`data` (a `ChartLineCyberCyclePoint[]`), `alpha` (default 0.07),
`pricePanelRatio` / `gap`, sizing/format/a11y props, `strokeWidth` /
`dotRadius`, the colour props, the toggle flags (`showAxis` / `showGrid` /
`showDots` default false / `showCycle` / `showTrigger` / `showZeroLine` /
`showMarkers` / `showTooltip` / `showConfigBadge` / `showLegend` /
`animate`), controlled + uncontrolled `hiddenSeries` (keys `price` /
`cycle` / `trigger`), `formatValue` / `formatX`, `onPointClick` /
`onSeriesToggle`.

Exported helpers: `getLineCyberCycleFinitePoints`,
`normalizeLineCyberCycleAlpha` (coerces alpha into the open unit
interval), `computeLineCyberCycleSmooth`, `computeLineCyberCycle`,
`classifyLineCyberCycleZone`, `runLineCyberCycle`,
`computeLineCyberCycleLayout`, `describeLineCyberCycleChart`, plus the
`DEFAULT_CHART_LINE_CYBER_CYCLE_*` constants,
`CHART_LINE_CYBER_CYCLE_WARMUP` and the `ChartLineCyberCycleZone` type.
Every exported name carries a `LineCyberCycle` infix so the `export *`
barrel line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen
`data-section="chart-line-cyber-cycle-aria-desc"` description span; the
price line, the cycle line, the trigger line and every dot / marker are
`role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-alpha`, `data-cycle-final`,
`data-up-count`, `data-down-count`, `data-flat-count`,
`data-total-points` and `data-animate`.

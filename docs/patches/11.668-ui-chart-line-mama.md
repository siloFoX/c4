## Indicator math

John Ehlers' MESA Adaptive Moving Average (MAMA) is an
exponential moving average whose smoothing factor `alpha` is not
fixed -- it adapts to how fast the price cycle is turning.

`computeLineMamaPhase(values, period)` measures the phase. The
price is detrended -- the `period`-bar moving average subtracted
-- to expose its cycle. The cycle value and its one-bar change
are treated as the in-phase and quadrature components of a
phasor, so the instantaneous phase is

```
phase = atan2(cycle - cycle[-1], cycle)
```

`computeLineMama(values, period, slowLimit, fastLimit)` turns the
phase rate of change into the adaptive alpha and runs two
recursions:

```
alpha = slowLimit + (fastLimit - slowLimit) * |dPhase| / pi
MAMA  = alpha * price + (1 - alpha) * MAMA[-1]
FAMA  = 0.5 * alpha * MAMA + (1 - 0.5 * alpha) * FAMA[-1]
```

A fast-advancing phase pushes alpha toward `fastLimit` so the
MAMA hugs the price; a slow phase pulls it toward `slowLimit` so
it smooths. The FAMA -- the Following Adaptive Moving Average --
trails the MAMA at half its alpha, and a MAMA above its FAMA is
read as bullish. Both are seeded with the first price.

### Exactness

The phase uses `atan2`, so the MAMA is transcendental and the
varying-series tests assert with `toBeCloseTo`. Three properties
are exact or near-exact: the MAMA and FAMA are seeded at the
first price (`toBe`); a flat price detrends to a zero cycle whose
phase is exactly 0, so the MAMA and FAMA stay on the price; and
`alpha` is always inside `[slowLimit, fastLimit]`. The two
recursions are verified bar by bar against the exposed `alpha`
series.

## API

- `getLineMamaFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineMamaPeriod(period, fallback)` -- coerce to a
  positive integer.
- `normalizeLineMamaLimit(limit, fallback)` -- coerce an alpha
  limit into `(0, 1)`.
- `computeLineMamaPhase(values, period)` -- the cycle phase per
  bar.
- `computeLineMama(values, period, slowLimit, fastLimit)` -- the
  `{ mama, fama, alpha }` series.
- `runLineMama(points, options?)` -- sort finite points by `x`,
  run the pipeline, return `{ series, period, slowLimit,
  fastLimit, mama, fama, alpha, samples, mamaFinal, famaFinal,
  bullishCount, bearishCount, neutralCount, ok }`. Each sample
  is classified `bullish` / `bearish` / `neutral` by the MAMA
  versus its FAMA. `ok` is false for fewer than two points.
- `computeLineMamaLayout(options)` -- project the price, MAMA
  and FAMA as three overlaid lines in a single panel.
- `describeLineMamaChart(data, options?)` -- ARIA description.

## Component

`ChartLineMama` is a `forwardRef` region (`role="region"`) with
an ARIA-linked description, a `role="img"` SVG, a single panel
with axis ticks and a grid, a config badge showing the period, a
three-series legend (Price / MAMA / FAMA) with toggle, one marker
per MAMA bar, a hover/focus tooltip exposing the MAMA, the FAMA
and the cross, and a `motion-safe:animate-fade-in` entrance. It
consumes `{ x, value }` points. The price, MAMA and FAMA are
drawn as three overlaid lines sharing one y-domain; the FAMA is
dashed. Every node carries a `data-section` attribute.

## Distinctness

- **vs the fixed-weight moving averages (ema / wma / hma)** --
  those use a constant smoothing weight. The MAMA changes its own
  alpha bar by bar from the measured cycle.
- **vs chart-line-kama** -- Kaufman's adaptive MA scales alpha by
  an efficiency ratio of price travel. The MAMA scales it by the
  rate of the cycle phase.
- **vs chart-line-frama** -- the fractal adaptive MA uses the
  fractal dimension of the price. The MAMA pairs its average with
  a FAMA follower for a crossover signal.

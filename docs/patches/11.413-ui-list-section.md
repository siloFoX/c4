## API

```ts
interface ListSectionItem {
  id: string | number;
  content: ReactNode;
  ariaLabel?: string;
}

interface ListSectionGroup {
  id: string;
  label: ReactNode;
  items: ListSectionItem[];
  defaultCollapsed?: boolean;
  count?: number;          // overrides items.length
  description?: ReactNode;
  badge?: ReactNode;       // custom slot
}

interface ListSectionProps {
  groups: ListSectionGroup[];
  collapsedGroups?: string[];
  defaultCollapsedGroups?: string[];
  onCollapsedGroupsChange?:
    (ids: string[]) => void;
  collapsible?: boolean;    // default true
  showBadges?: boolean;     // default true
  stickyHeaders?: boolean;  // default true
  ariaLabel?: string;
  className?: string;
  renderItem?:
    (item, group) => ReactNode;
  emptyLabel?: ReactNode;
    // default '(no items)'
}
```

## Behaviour

### Header

Each group renders a `<button>`
header with:

- Chevron icon (right when
  collapsed, down when expanded)
- Label
- Count badge (default), custom
  badge slot, or nothing when
  `showBadges` is false
- Optional description text on
  the right

Click toggles collapse; Enter /
Space also toggle. The header
carries `aria-expanded` mirroring
the collapsed state, and
`aria-controls` pointing to the
body id.

### Sticky headers

When `stickyHeaders` is true
(default) each header gets
`sticky top-0 z-10` so it pins
to the top of the scroll
container while the group body
scrolls past. When false, the
header scrolls normally with
the body.

### Collapsing

`collapsible` (default true)
enables the toggle. When false:

- Header has the chevron
  hidden.
- Header is `disabled`.
- Clicking is a no-op.
- `aria-expanded` is omitted.

### Controlled state

`collapsedGroups` is the
controlled set;
`defaultCollapsedGroups` +
per-group `defaultCollapsed` is
the uncontrolled seed (union of
both). `onCollapsedGroupsChange
(ids)` fires on every toggle in
both modes.

### Count badge

Small pill rendered next to the
label.

| supplied | result |
| --- | --- |
| `group.count` (number) | clamped to >=0 + floored |
| no `count`, items array | `items.length` |
| `group.badge` | replaces the pill |
| `showBadges={false}` | hides all badges |

The pill carries
`aria-label="N items"` so SR
users hear the count.

### Per-group empty state

An empty `items` array renders
the `emptyLabel` inside the
group body rather than
collapsing the group.

### Empty groups array

When `groups.length === 0`, the
root renders the `emptyLabel`
and carries `data-empty="true"`.
The role=list is preserved so
AT users still get the section
label.

## Pure helpers (exported)

```ts
export function isGroupCollapsed(
  groupId: string,
  collapsedSet: Set<string>,
): boolean;

export function getGroupCount(
  group: ListSectionGroup,
): number;

export function totalItemCount(
  groups: ListSectionGroup[],
): number;

export function visibleItemCount(
  groups: ListSectionGroup[],
  collapsedSet: Set<string>,
): number;
```

`getGroupCount`:
- explicit `count` (clamped,
  floored) wins
- else falls back to
  `items.length`

`visibleItemCount`:
- sums groups not in the
  collapsed set

## ARIA + data attributes

Root (`<div role="list">`,
`aria-label`):

- `data-section="list-section"`
- `data-empty` ('true' /
  'false')
- `data-group-count`
- `data-total-items`
- `data-sticky-headers`
- `data-collapsible`

Per group (`<section
role="region">`):

- `aria-labelledby` ->
  header id
- `data-section="list-section-group"`
- `data-group-id`
- `data-collapsed`
- `data-count`

Header (`<button>`):

- `data-section="list-section-header"`
- `data-group-id`
- `aria-expanded` (when
  collapsible)
- `aria-controls` -> body id

Sub-nodes:

- chevron span:
  `data-section="list-section-chevron"`
- label span:
  `data-section="list-section-label"`
- count pill:
  `data-section="list-section-count"`
  + `aria-label="N items"`
- custom badge slot:
  `data-section="list-section-badge-slot"`
- description span:
  `data-section="list-section-description"`

Body (`<ul role="list">`):

- `data-section="list-section-body"`
- `data-group-id`
- empty-state li:
  `data-section="list-section-empty-group"`

Item (`<li
role="listitem">`):

- `data-section="list-section-item"`
- `data-item-id`
- `data-group-id`
- `aria-label` (per-item)

## Pairs with existing primitives

- `<ListVirtualizer>` (11.401)
  -- virtualized list variant.
  ListSection is the small-to-
  mid list with declared
  groups; ListVirtualizer is
  the windowed long list.
- `<TreeView>` (11.397) --
  nested-hierarchy variant.
  ListSection is the flat
  variant (one level of
  grouping).
- `<CommandBar>` (11.395) --
  often paired with
  ListSection when the host
  wants multi-selection
  across groups.
- ThemeCustomizer (11.394) --
  the list reads token-based
  classes so it auto-themes.

## Out of scope (deferred)

- Nested groups (groups
  within groups). Single-
  level only for v1; TreeView
  covers recursive trees.
- Drag-and-drop reorder of
  items between groups.
  Adopters can layer via the
  `renderItem` slot.
- Per-group filter / search.
  Caller pre-filters the
  `items` arrays.
- Virtualized item rendering
  per group. Pair with
  `<ListVirtualizer>` for
  that.
- Sortable group order
  (drag-the-header).
  Source-array order is
  canonical.
- Multi-select range (Shift+
  click). Host manages
  selection state.

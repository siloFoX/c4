## Summary

New `ChartLineTsi` UI primitive at
`web/src/components/ui/chart-line-tsi.tsx` -- a pure-SVG line chart
with a True Strength Index double-smoothed momentum oscillator panel.
The value line occupies the top panel; the bottom panel renders the
TSI line and its signal line on a fixed -100 to +100 scale with a
dashed zero centerline and +25/-25 reference lines.

## Indicator math

`computeLineTsiMomentum(values)` is the per-period price momentum
`value[i] - value[i-1]` and its absolute value (index 0 null).

`computeLineTsiEma(src, period)` is an exponential moving average
that tolerates leading nulls: it seeds at the first defined value and
rolls forward with `k = 2/(period+1)`.

`computeLineTsi(values, longPeriod, shortPeriod)` is William Blau's
True Strength Index:

```
momentum    double-smoothed: EMA(longPeriod) then EMA(shortPeriod)
absMomentum double-smoothed the same way
TSI = 100 * doubleSmoothedMomentum / doubleSmoothedAbsMomentum
```

Because the absolute momentum dominates the signed momentum, the
ratio is bounded, so TSI stays within -100 to +100. A flat series
reads 0 without dividing by zero.

### Worked fixture

`values = [10, 14, 12, 18, 16, 22]`, `long = short = 3` (so every EMA
factor is `k = 0.5`):

```
momentum    = [., 4, -2, 6, -2, 6]
ema1        = [., 4, 1,   3.5,  0.75,  3.375]
ema2        = [., 4, 2.5, 3,    1.875, 2.625]   (double-smoothed momentum)
ema2abs     = [., 4, 3.5, 4,    3.625, 4.125]   (double-smoothed |momentum|)
tsi         = [., 100, 71.4286, 75, 51.7241, 63.6364]
```

A strictly rising series reads +100, a strictly falling series -100.

## API

- `getLineTsiFinitePoints(points)` -- drop points with a non-finite
  `x` or `value`.
- `normalizeLineTsiPeriod(period, fallback)` -- positive integer;
  non-finite / sub-1 falls back, fractional floors.
- `computeLineTsiMomentum(values)` -- the `{ momentum, absMomentum }`
  pair.
- `computeLineTsiEma(src, period)` -- a leading-null-tolerant EMA.
- `computeLineTsi(values, longPeriod, shortPeriod)` -- the TSI series.
- `runLineTsi(points, options?)` -- sort finite points by `x`, compute
  the TSI and a signal EMA, return `{ series, longPeriod, shortPeriod,
  signalPeriod, tsi, signal, samples, tsiFinal, signalFinal, tsiMin,
  tsiMax, ok }`. `ok` is false for fewer than two points.
- `computeLineTsiLayout(options)` -- stack a value panel above a fixed
  -100 to +100 TSI panel. Returns panels, ticks, the value path, TSI
  path, signal path, markers, the zero centerline, and the two level
  positions.
- `describeLineTsiChart(data, options?)` -- ARIA description string.

## Component

`ChartLineTsi` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge (long + short periods), a three-series legend (Value / TSI /
Signal) with toggle, a dashed zero centerline, dashed +25/-25 level
lines, the TSI line, a dashed signal line, markers on the TSI line, a
hover tooltip exposing the value, TSI, and signal, and a
`motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs chart-line-trix** -- TRIX triple-smooths the *price* itself and
  then takes a rate of change, producing an *unbounded* oscillator.
  TSI double-smooths the *momentum* and normalizes by the
  double-smoothed *absolute* momentum: that normalization step is what
  bounds TSI to -100..+100, a property TRIX does not have. TRIX is a
  smoothed ROC; TSI is a smoothed, normalized momentum ratio.

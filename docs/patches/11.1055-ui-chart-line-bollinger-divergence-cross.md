# 11.1055 -- UI: chart-line-bollinger-divergence-cross primitive (v1.11.1073)

## Scope

Add a pure-SVG dual-panel
`ChartLineBollingerDivergenceCross` component that renders
the close in the top panel and the Bollinger Band envelope
(SMA midline plus upper / lower sigma bands) in the bottom
panel, marking price-vs-band direction disagreement events
as volatility-reversal warnings with bias coloring derived
from the Bollinger midline slope at the trigger bar.

John Bollinger's 1980s Bollinger Bands surround a Simple
Moving Average midline with bands at `+/- numStdev *
stdev`. The bands widen when volatility (stdev of close)
expands and contract when volatility shrinks. The midline
is the SMA centerline of the volatility envelope. This
primitive compares the midline slope against price
direction to surface volatility-reversal warnings.

## Bollinger divergence-cross pipeline

```
midline[i]   = SMA(close, period)
variance[i]  = sum((close[j] - midline[i])^2,
                    j in [i - period + 1, i]) / period
stdev[i]     = sqrt(variance[i])
upperBand[i] = midline[i] + numStdev * stdev[i]
lowerBand[i] = midline[i] - numStdev * stdev[i]

priceUp     : close[i] > close[i-1]
priceDown   : close[i] < close[i-1]
midlineUp   : midline[i] > midline[i-1]
midlineDown : midline[i] < midline[i-1]

regime ->
  'aligned-bullish'   when priceUp   && midlineUp
  'aligned-bearish'   when priceDown && midlineDown
  'divergent-bullish' when priceDown && midlineUp
                       (selling pressure but the SMA
                        midline still trending up --
                        bullish volatility-reversal
                        warning)
  'divergent-bearish' when priceUp   && midlineDown
                       (rally but the SMA midline trending
                        down -- bearish volatility-reversal
                        warning)
  'none'              when either side flat or null

bullish (divergence trigger up) :
  prev regime !== 'divergent-bullish' &&
  cur regime === 'divergent-bullish'
bearish (divergence trigger down) :
  prev regime !== 'divergent-bearish' &&
  cur regime === 'divergent-bearish'

bias        : midline[i] vs midline[i-1] -> up / down /
              flat / none
```

Default `period = 20`, `numStdev = 2` -- canonical
Bollinger 1980s tuning.

Uses **population standard deviation** (divide by N)
matching Bollinger's original formulation. Some libraries
use sample stdev (divide by N - 1) for unbiased estimation
-- this primitive uses the canonical population variant.

Sibling to chart-line-keltner-divergence-cross v1.11.1072
(same midline-plus-volatility-bands shape; Keltner uses
EMA + ATR, Bollinger uses SMA + stdev).

Warmup is `period = 20` for the default tuning: SMA and
stdev seed at i = period - 1 = 19, and direction
detection requires the previous bar's midline, so the
first regime classification lands at i = period = 20.

## Bit-exact anchors

All anchors use HLC input.

- **CONST band** `high = K + 1`, `low = K - 1`, `close
  = K`: midline = K (constant). variance = 0 (all
  deviations are 0, short-circuited via min === max).
  stdev = 0. bands = K (midline +/- 0). Both price and
  midline are flat -> regime `none` for every valid bar.
  0 crosses. Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close
  = i`: midline = i - 9.5 (centroid of window). Deviations
  from midline: -9.5, -8.5, ..., 9.5. Sum of squares = 665.
  variance = 665 / 20 = 33.25. stdev = sqrt(33.25) =
  sqrt(133) / 2 ~= 5.766. bands = (i - 9.5) +/- 2 *
  sqrt(133) / 2 = (i - 9.5) +/- sqrt(133). priceUp +
  midlineUp -> aligned-bullish for every valid bar. 0
  crosses.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`,
  `close = -i`: mirror -> midline = -i + 9.5, stdev =
  sqrt(133) / 2 (same dispersion). priceDown + midlineDown
  -> aligned-bearish. 0 crosses.

The constant stdev = sqrt(133)/2 on LINEAR input is
canonical: the dispersion of a window of consecutive
integers around its centroid is constant regardless of
where the window sits along the integer line. This is the
key distinguishing feature from the Keltner sibling -- the
Keltner ATR also lands at a constant on LINEAR (=2 for
this anchor pattern), but ATR is the true-range based
volatility measure while Bollinger stdev is the dispersion
based one.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 5-state regime classifier follows the divergence-cross
  family convention.
- Population stdev (divide by N) matches Bollinger's
  original. Some libraries use sample stdev (divide by
  N - 1) for unbiased estimation -- documented as a
  deliberate choice.
- Min === max short-circuit on stdev computation: when
  all values in the window are equal (CONST input), stdev
  is exactly 0 -- bypasses floating-point sqrt to avoid
  NaN-of-negative-tiny-value drift.
- The Bollinger midline is the SMA centerline; it lags
  more than the Keltner EMA midline on trending data but
  is more rigid (less responsive to abrupt moves). On
  steady LINEAR input the midline produces a constant
  slope and zero divergence crosses, which is correct
  canonical behaviour.
- HLC input boundary filter requires `high >= low` --
  malformed bars are dropped at the input boundary. Note
  that Bollinger Bands use only close (not HLC), but the
  primitive accepts HLC for family consistency with
  Keltner; high / low are validated and surfaced in the
  point sample even though only close drives the
  computation.
- Complementary to chart-line-keltner-divergence-cross
  v1.11.1072 -- same shape, different volatility measure
  (stdev vs ATR), different midline (SMA vs EMA).
  Together they bracket the modern volatility-envelope
  family.

## Summary

`web/src/components/ui/chart-line-confluence.tsx` ships
`<ChartLineConfluence>`, a pure-SVG two-panel chart with a
**Confluence** panel summing aligned signals from multiple lookback
oscillators into a single integer score. The top panel plots the
price; the bottom panel plots the score with a horizontal zero line
and dashed `+/-threshold` reference lines.

## Computation

For each lookback `L_k`, the per-bar signal is the sign of the
bar's value minus the value `L_k` bars ago:

```
signal_k(i) = +1   if value[i] > value[i - L_k]
             -1   if value[i] < value[i - L_k]
              0   if value[i] = value[i - L_k]
              null if i < L_k (the lookback has not filled)
                   or if either operand is non-finite

score(i) = sum over k of signal_k(i)   in [-N, +N]
```

The score is null until **every** lookback has filled. The threshold
partitions the score into three zones: `bullish` (score above
threshold), `bearish` (score below the negative threshold), `neutral`
(within the band).

## Worked anchors

Pure integer arithmetic, no transcendentals -- every anchor is bit-
exact for integer fixtures.

- **Rising series** -- `[10..34]`, lookbacks `[3, 5, 10]`. At every
  bar `i >= 10`, `value[i] > value[i - L_k]` for every `L_k`, so
  every signal is `+1` and the score is `3`. Asserted with
  `toBe(3)` for every defined bar.
- **Falling series** -- `[30..6]`, same lookbacks. Mirror: every
  signal is `-1`, score is `-3` at every defined bar. Asserted
  with `toBe(-3)`.
- **Constant series** -- `[5 x 25]`. Every bar-to-bar difference is
  zero, so every signal is `0`, the score is `0` at every defined
  bar. Asserted with `toBe(0)`.
- **Five-lookback set** -- `[2, 3, 5, 8, 13]` on the rising series.
  The score on every defined bar is `5` (every signal `+1`).
  Asserted with `toBe(5)`.
- **Score is an integer in `[-N, +N]`** -- on a wave fixture, every
  defined score is `Number.isInteger` and within the bound.

The warm-up rule (the score is null at every bar before the largest
lookback has filled) and the non-finite propagation rule (a non-
finite operand at bar i null-ifies that bar and any future bar that
looks back to it) are tested directly.

## Distinct from neighbouring primitives

- vs single-lookback momentum primitives (`<ChartLineRoc>`,
  `<ChartLineCmo>`, `<ChartLineRsi>`) -- those each measure one
  lookback at a time; the Confluence aggregates the signs from
  multiple lookbacks into a single score so a unanimous signal
  reads as a clear `+/-N` and a mixed signal lands inside the band.
- vs `<ChartLineBayesian>` -- the Bayesian Trend produces three
  calibrated probabilities under a parametric model; the Confluence
  is a deterministic vote over deterministic lookback signs.
- vs `<ChartLineMonteCarlo>` -- the Monte Carlo Fan is a forward-
  looking probabilistic forecast; the Confluence is a backward-
  looking deterministic alignment metric.

## API

`data` -- a single `ChartLineConfluencePoint[]` (`{x, value}`).
`lookbacks` (default `[5, 10, 20]`, sorted positive integers; sub-1
or non-finite entries dropped; empty falls back to the default).
`threshold` (default `floor(N / 2)`, non-negative finite). Standard
sizing / colour / a11y props plus `pricePanelRatio` (default 0.55,
clamped to `[0.3, 0.8]`), `gap`, `strokeWidth`, `dotRadius`,
`tickCount`. Toggle flags: `showAxis`, `showGrid`, `showDots`
(default false), `showScore`, `showZeroLine`, `showThresholdLines`,
`showMarkers`, `showTooltip`, `showConfigBadge`, `showLegend`,
`animate`. Controlled and uncontrolled visibility via
`hiddenSeries` (`'price' | 'score'`). Callbacks:
`onPointClick({point})`, `onSeriesToggle({seriesId, hidden})`.
Format functions: `formatValue`, `formatX`. ARIA: `ariaLabel`,
`ariaDescription`.

## Pure helpers

- `DEFAULT_CHART_LINE_CONFLUENCE_*` -- defaults.
- `getLineConfluenceFinitePoints` -- keep only points with a finite
  x and value.
- `normalizeLineConfluenceLookbacks(lookbacks, fallback)` -- sort,
  floor, drop sub-1 entries; fallback for empty / non-array.
- `normalizeLineConfluenceThreshold(threshold, lookbackCount)` --
  non-negative finite; defaults to `floor(lookbackCount / 2)`.
- `computeLineConfluenceSignal(values, lookback)` -- per-bar signal
  in `{-1, 0, +1, null}`.
- `computeLineConfluence(values, lookbacks)` --
  `{signals: Array<Array<-1|0|1|null>>, score: Array<number|null>}`;
  the score is null until every lookback has filled.
- `classifyLineConfluenceZone(score, threshold)` --
  `'bullish' | 'bearish' | 'neutral' | 'none'`.
- `runLineConfluence(points, options?)` --
  `{series, lookbacks, threshold, signals, score, samples,
  scoreFinal, bullishCount, bearishCount, neutralCount, ok}`.
- `computeLineConfluenceLayout({data, lookbacks, threshold, ...})`
  -- two panels with the price line on top and the score line below
  in a symmetric domain, one marker per defined-score bar coloured
  by zone, plus `zeroY`, `upperThresholdY`, `lowerThresholdY` for
  the reference lines.
- `describeLineConfluenceChart(data, options?)` -- ARIA summary
  mentioning the lookbacks, the threshold, the score range and the
  zone counts.

## a11y / data hooks

Root `<div role="region">` with `aria-label` + `aria-describedby`; the
SVG is `role="img"` with the same label; an sr-only
`data-section="chart-line-confluence-aria-desc"` span carries the
description. The price line, every price dot, the score line and
every marker carry `role="graphics-symbol"` and `tabIndex={0}`.
Every node carries a `data-section`. Root mirrors `data-lookbacks`,
`data-threshold`, `data-score-final`, `data-bullish-count`,
`data-bearish-count`, `data-neutral-count`, `data-total-points`,
`data-animate`, `data-empty`. Grid / axis / panel-label nodes carry
`data-panel="price"|"score"`. Threshold lines carry
`data-direction="upper"|"lower"`. Markers expose `data-zone` and
`data-score`.

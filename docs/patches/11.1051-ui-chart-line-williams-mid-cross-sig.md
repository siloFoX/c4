# 11.1051 -- UI: chart-line-williams-mid-cross-sig primitive (v1.11.1069)

## Scope

Add a pure-SVG dual-panel `ChartLineWilliamsMidCrossSig`
component that renders the close in the top panel and the
Williams Percent Range (Williams %R) plus its smoothed SMA
signal line in the bottom panel, marking bullish / bearish
Williams-over-signal crossover trigger events with bias
coloring derived from the Williams %R slope at the trigger
bar.

Larry Williams' 1973 Williams %R is the canonical bounded
momentum oscillator: it locates the current close within
the rolling [HH, LL] high-low range as a negative
percentage. `> -20` is the conventional overbought zone,
`< -80` is oversold, and `-50` is the natural momentum
centerline. This primitive watches the Williams %R vs its
own SMA-smoothed signal line to detect centerline momentum
trigger events.

## Williams %R pipeline

```
hh[i]        = max(high[j], j in [i-period+1..i])
ll[i]        = min(low[j],  j in [i-period+1..i])
pctR[i]      = (hh[i] === ll[i])
               ? -50  (flat-window guard -> midline)
               : -100 * (hh[i] - close[i]) /
                       (hh[i] - ll[i])
signal[i]    = SMA(pctR, signalLength)

bullish      : prev pctR <= prev signal &&
               cur pctR > cur signal
bearish      : prev pctR >= prev signal &&
               cur pctR < cur signal
regime       : 'bullish' when pctR >= signal
               'bearish' when pctR <  signal
               'none'    when either is null
bias         : pctR[i] vs pctR[i-1] -> up / down / flat /
               none
```

Default `period = 14`, `signalLength = 3` -- canonical
Williams tuning paired with the family-conventional short
SMA signal.

The flat-window guard `hh === ll -> -50` prevents 0/0
NaN on input where the rolling high-low range collapses
to a single value. The centerline is the natural fallback,
matching pandas-ta and other reference implementations.

Warmup is `period + signalLength - 2 = 15` for the
default tuning: pctR seeds at i = period - 1 = 13, then
the signal SMA needs `signalLength - 1 = 2` more bars.

## Bit-exact anchors

All anchors use HLC input.

- **CONST band** `high = K + 1`, `low = K - 1`, `close
  = K`: hh = K + 1, ll = K - 1. range = 2. pctR =
  -100 * (K + 1 - K) / 2 = -100 * 1 / 2 = -50 (exactly
  the midline). signal = SMA(-50, 3) = -50. pctR ===
  signal -> regime `bullish` (>=). 0 crosses. Verified
  across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close
  = i`: hh = i + 1 (latest high). ll = i - 14 (oldest
  low at j = i - 13: (i - 13) - 1 = i - 14). pctR =
  -100 * (i + 1 - i) / (i + 1 - (i - 14)) = -100 * 1
  / 15 = -20 / 3 ~= -6.67 (deep into overbought zone).
  signal = -20 / 3 (constant). pctR === signal ->
  regime `bullish` (>=). 0 crosses.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`,
  `close = -i`: hh = -i + 14 (oldest high at j =
  i - 13: -(i - 13) + 1 = -i + 14). ll = -i - 1
  (latest low). pctR = -100 * (-i + 14 - (-i)) /
  (-i + 14 - (-i - 1)) = -100 * 14 / 15 = -280 / 3
  ~= -93.33 (deep into oversold zone). signal =
  -280 / 3 (constant). pctR === signal -> regime
  `bullish` (>=) for the same `===` reason. 0
  crosses.

Williams %R has the elegant property that the CONST
anchor lands *exactly on the midline* (-50), making it
the simplest visual confirmation that the centerline
reference is rendering correctly.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 3-state regime classifier follows the cross-sig family
  convention (bullish on `===`, bearish on `<`); the
  sibling `chart-line-williams-divergence-cross`
  (v1.11.1063) uses a 5-state divergence model paired
  with price.
- Flat-window guard on `hh === ll` returns the
  centerline value of -50 instead of 0/0 NaN. This
  matches pandas-ta and other reference implementations.
- On steady LINEAR input Williams %R saturates near the
  zone extremes (-6.67 overbought, -93.33 oversold) and
  produces zero crosses, which is correct canonical
  behaviour -- real-world price data with trend-
  direction changes produces typical Williams cross
  patterns.
- Williams %R is hard-bounded to `[-100, 0]` so the
  layout panel can be locked, providing stable threshold
  reference rendering. Differs from CCI (unbounded
  mean-reversion) and Ichimoku (bounded to data range)
  -- this primitive locks the panel.
- Complementary to the sibling SMA / EMA / Bollinger /
  Keltner / Donchian / CCI / RSI / Ichimoku /
  Supertrend mid-cross-sig primitives -- Williams %R
  adds another canonical bounded momentum oscillator
  alongside RSI; the two differ in scale (`[-100, 0]`
  vs `[0, 100]`) and centerline (`-50` vs `50`) but
  share the bounded-oscillator-vs-SMA-signal pattern.

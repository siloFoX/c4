# Patch 11.916 -- UI: chart-line-vwap-cross-sig primitive

## Summary

Added `<ChartLineVwapCrossSig />` -- pure-SVG dual-panel
React/TS primitive that renders the close (top panel) with
bullish / bearish arrow overlays at every rolling VWAP vs
signal cross trigger, and the close-only rolling Volume
Weighted Average Price (VWAP) line + EMA-smoothed signal line
(bottom panel) with cross dot markers. Signal-cross variant
of the VWAP family that flags volume-weighted session bias
trigger events distinct from a static session VWAP anchor.

## Algorithm

```
tp[i]     = close[i]                                  (typical price)
vol[i]    = close[i]                                  (volume proxy)
vwap[i]   = sum(tp * vol, length) / sum(vol, length)
          = sum(close^2, length) / sum(close, length)
signal[i] = EMA(vwap, signalLength)
bullish  : (vwap - signal) crosses up   (prev <= 0, cur > 0)
bearish  : (vwap - signal) crosses down (prev >= 0, cur < 0)
```

Defaults `length = 20` (rolling VWAP window), `signalLength
= 9`. Regime classifier `bullish` (vwap > signal), `bearish`
(vwap < signal), `neutral` (vwap === signal), `none` (either
null). Divide-by-zero guard returns null when the volume sum
is non-positive.

## Bit-exact anchor

CONST close = K (K > 0): every typical price = K and every
weight = K, so VWAP = (K * K * n) / (K * n) = K every bar.
The `min === max` short-circuit also yields VWAP = K exactly.
signal = EMA(K) = K via the same precision fix. vwap ===
signal -> regime `neutral`, cross count = 0. K = 0 yields the
divide-by-zero guard at every bar (vwap = signal = null) which
the regime classifier maps to `none`. Verified across K = 0,
1, 5, 17, 100, 1234.

LINEAR UP with step 1 has a clean anchor too: each rolling
window has VWAP > SMA so the EMA signal of VWAP lags below
VWAP -> vwap > signal, regime `bullish`.

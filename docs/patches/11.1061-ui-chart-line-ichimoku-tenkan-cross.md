# 11.1061 -- UI: chart-line-ichimoku-tenkan-cross primitive (v1.11.1079)

## Scope

Add a pure-SVG dual-panel `ChartLineIchimokuTenkanCross`
component that renders the close in the top panel and the
Ichimoku tenkan-sen, kijun-sen, and kumo cloud (senkouA /
senkouB) in the bottom panel, marking bullish / bearish
TK-cross events with **cloud-color bias** at the trigger
bar.

The TK cross (tenkan over kijun) is the canonical
Ichimoku entry signal: when the fast conversion line
crosses the slower base line, it suggests a shift in the
short-term equilibrium. The cloud color (senkouA vs
senkouB at the trigger bar) provides directional
confirmation -- a bullish cross with bullish cloud is
canonical "strong bull"; a bullish cross with bearish
cloud is "weak bull" / counter-trend.

## TK-cross pipeline

```
tenkan[i]    = (HH(high, tenkanPeriod) +
                LL(low,  tenkanPeriod)) / 2
kijun[i]     = (HH(high, kijunPeriod) +
                LL(low,  kijunPeriod)) / 2
senkouA[i]   = (tenkan[i] + kijun[i]) / 2
senkouB[i]   = (HH(high, senkouBPeriod) +
                LL(low,  senkouBPeriod)) / 2

bullish (TK cross up) :
  prev tenkan <= prev kijun &&
  cur tenkan > cur kijun
bearish (TK cross down) :
  prev tenkan >= prev kijun &&
  cur tenkan < cur kijun

regime       : 'bullish' when tenkan >= kijun
               'bearish' when tenkan <  kijun
               'none'    when either is null

bias (cloud color at trigger bar) :
  'bullish' when senkouA >  senkouB (green cloud)
  'bearish' when senkouA <  senkouB (red cloud)
  'flat'    when senkouA === senkouB (flat cloud)
  'none'    when either is null (cloud not yet valid)
```

Default `tenkanPeriod = 9`, `kijunPeriod = 26`,
`senkouBPeriod = 52` -- canonical Goichi Hosoda (1969)
Ichimoku Kinko Hyo tuning.

The **cloud-color bias is the distinguishing feature**:
unlike the other cross-sig family members which colour
markers by indicator slope (up/down/flat), this primitive
uses cloud-color confirmation (bullish/bearish/flat
cloud), which is more idiomatic for Ichimoku. A bullish
TK cross inside a bullish cloud is the canonical "strong
bull" signal; the same cross inside a bearish cloud is
counter-trend.

## Sibling family

The Ichimoku signal family has three primitives:

- **chart-line-ichimoku-mid-cross-sig v1.11.1067** --
  kumo midline vs SMA signal cross (mid-cross-sig family
  convention)
- **chart-line-ichimoku-divergence-cross v1.11.1061**
  -- kumo midline direction vs price direction
  (divergence-cross family convention)
- **chart-line-ichimoku-tenkan-cross v1.11.1079** (this
  primitive) -- canonical TK cross with cloud-color
  bias (the historically primary Ichimoku trading
  signal)

Each surfaces a different signal niche. The TK-cross
variant is the most canonical Ichimoku signal as
described in Hosoda's original work.

Uses current-bar reads (no 26-bar forward displacement)
matching the sibling convention -- the senkouA/senkouB
values at bar `i` are computed from the high/low data up
to bar `i`, with no time-shift.

Warmup is `kijunPeriod = 26` for TK cross detection
(both tenkan and kijun valid at i and i-1). Cloud-color
bias additionally requires senkouB valid, which seeds at
i = senkouBPeriod - 1 = 51. Crosses that fire before i =
51 carry `bias = 'none'`.

## Bit-exact anchors

All anchors use HL input.

- **CONST band** `high = K + 1`, `low = K - 1`: HH = K
  + 1, LL = K - 1 for every window. tenkan = kijun = K
  (constant). tenkan === kijun every bar -> no cross.
  senkouA = K, senkouB = K -> cloud flat. regime
  `bullish` (via >=). 0 crosses. Verified across K in
  {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`: HH(n) =
  i + 1, LL(n) = i - n. tenkan = (i + 1 + i - 9) / 2
  = i - 4. kijun = (i + 1 + i - 26) / 2 = i - 12.5.
  **tenkan - kijun = +8.5 (constant spread, tenkan
  always above kijun) -> no cross**. senkouA =
  (tenkan + kijun) / 2 = i - 8.25. senkouB = (i + 1 +
  i - 52) / 2 = i - 25.5. **senkouA - senkouB = +17.25
  -> bullish cloud** (would colour any cross marker
  bullish, but no crosses fire here). 0 crosses.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`:
  mirror -> tenkan = -i + 4, kijun = -i + 12.5,
  **tenkan - kijun = -8.5 (tenkan always below kijun)
  -> no cross**. senkouA = -i + 8.25, senkouB = -i +
  25.5 -> **senkouA - senkouB = -17.25 -> bearish
  cloud**. 0 crosses.

The constant +/- 8.5 tenkan-kijun spread on LINEAR is
the difference of the two Donchian centroids: (9-1)/2 -
(26-1)/2 = 4 - 12.5 = -8.5 (kijun is 8.5 bars lagged
relative to tenkan on linear input). The +/- 17.25
senkouA-senkouB spread reflects the difference between
the tenkan+kijun average centroid and the 52-bar
Donchian centroid.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 3-state regime classifier (bullish/bearish/none)
  follows the cross-sig family convention.
- **Cloud-color bias deviates from the slope-bias used
  by other cross-sig members**. This is intentional --
  cloud-color confirmation is the canonical Ichimoku
  trading rule for evaluating cross quality. The
  `bullish`/`bearish`/`flat`/`none` values for bias
  reflect cloud color, not slope direction.
- Current-bar reads (no 26-bar forward displacement)
  match the sibling Ichimoku primitives. Many trading
  platforms display Ichimoku with senkouA/senkouB
  forward-projected by 26 bars; this primitive uses
  the un-shifted form for direct cross-with-cloud
  alignment at the current bar.
- On steady LINEAR input the tenkan-kijun spread is
  constant (+/- 8.5) so 0 crosses fire -- correct
  canonical behaviour. Crosses appear when price has
  curvature that lets tenkan momentarily reverse its
  position relative to kijun.
- Cloud-color confirmation requires senkouB valid (i
  >= 51 for default tuning). Earlier crosses carry
  `bias = 'none'` rather than being suppressed.

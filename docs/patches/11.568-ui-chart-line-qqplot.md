# 11.568 -- web/src/components/ui/chart-line-qqplot.tsx

Pure-SVG line chart with a **quantile-quantile (Q-Q) plot**
side panel. The left panel shows the raw time series; the
right panel is a normal probability plot -- the sample
quantiles charted against the quantiles of a standard normal
distribution. If the sample is normally distributed the
Q-Q points fall on a straight line; departures from that
line reveal skew, heavy tails and other non-normality.

## The Q-Q analysis

- `normalInverseCDF(p)` is the probit / quantile function --
  the inverse of the standard normal CDF -- via Peter
  Acklam's rational approximation (relative error below
  ~1.15e-9 across the open interval). `p` outside [0, 1]
  yields NaN; `p` of exactly 0 / 1 yields -Infinity /
  +Infinity. It is exactly 0 at `p = 0.5`.
- `computeLineQQPlotStats(values)` reports the mean, the
  variance and the **sample** standard deviation (the
  unbiased `n - 1` denominator).
- `runLineQQPlot(points)` sorts the sample ascending and,
  for each order statistic at rank `i` of `n`, computes the
  plotting position `p = (i - 0.5) / n`, the theoretical
  normal quantile `z = normalInverseCDF(p)`, and pairs it
  with the observed value. The **reference line** is
  `sample = mean + stddev * z` -- what a perfectly normal
  sample would follow. It also reports the Pearson
  **correlation** between the theoretical and sample
  quantiles: a value near 1.0 means the sample is close to
  normal (this is the statistic behind the Filliben
  normality test).

The component fixture is the symmetric sample
`[-2, -1, 0, 1, 2]` -- mean 0, sample variance `10/4 = 2.5`.
Its five plotting positions are `[.1, .3, .5, .7, .9]`, the
median quantile maps to theoretical `z = 0` exactly, and the
Q-Q correlation lands above 0.99.

## Distinct from `chart-line-distribution`

`chart-line-distribution` summarises a series with a
**histogram / density** side panel -- it shows the *shape*
of the distribution. ChartLineQQPlot answers a different
question: *is this sample normal, and how does it deviate?*
It plots **quantile against theoretical quantile** with a
straight-line reference, so the eye reads non-normality
directly off the curvature. The defining transform is the
probit (`normalInverseCDF`), which a histogram never needs.

All exported helpers carry a `LineQQPlot` infix (or the
clearly-namespaced `normalInverseCDF`) so the `export *`
barrel line cannot collide.

## Public surface

- `<ChartLineQQPlot data={...} />` -- a single
  `ChartLineQQPlotPoint[]` (`{x, value}`).
- `linePanelRatio` -- the fraction of the width given to the
  series panel (default 0.55; clamped to [0.2, 0.8]).
- `gap` -- the pixel gap between the two panels.
- Toggle flags: `showAxis`, `showGrid`, `showSeriesDots`,
  `showReferenceLine`, `showPanelLabels`, `showTooltip`,
  `showConfigBadge`, `showFooter`, `animate`.
- Standard sizing + format + a11y props. `formatValue` /
  `formatX`.
- `onSampleClick({point})` (a series dot) /
  `onQuantileClick({quantile})` (a Q-Q dot).

## Pure helpers

- `getLineQQPlotFinitePoints(points)`.
- `normalInverseCDF(p)` -- the probit function.
- `computeLineQQPlotStats(values)` -- mean / variance /
  sample stddev / count.
- `runLineQQPlot(points)` -- the full Q-Q analysis;
  `ok` requires at least 2 finite samples.
- `computeLineQQPlotLayout({...})` -- splits the canvas into
  the series and Q-Q panels and projects both.
- `describeLineQQPlotChart(data, options?)`.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- The series line, every series dot and every Q-Q dot
  `role="graphics-symbol"` + `tabIndex={0}`.
- Every node carries `data-section="chart-line-qqplot-..."`.
- Root mirrors `data-count`, `data-mean`, `data-stddev`,
  `data-correlation`, `data-animate`.
- Axis / tick / grid / panel-label nodes carry
  `data-panel="series"|"qq"`. Q-Q dots expose `data-rank`,
  `data-theoretical`, `data-sample`.

## Tooltip / badge / footer

Hovering a series dot shows x + value; hovering a Q-Q dot
shows the quantile rank, the theoretical z, the sample
value and the residual from the reference line. Config
badge: `QQ r=<correlation> n=<count>`. The footer reports
`n / mean / sd` and the reference-line equation
`sample = <intercept> + <slope> z`.

- Defaults, `getLineQQPlotFinitePoints`.
- `normalInverseCDF`: **exactly 0 at p=0.5**, **matches
  known quantiles (1.959964 at .975, -1.281552 at .1)**,
  **antisymmetric about p=0.5**, infinities at the boundary,
  out-of-range / non-finite p -> NaN.

  stddev (n-1)**, empty -> count 0, a single value has a

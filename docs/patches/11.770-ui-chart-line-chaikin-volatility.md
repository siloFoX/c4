# 11.770 chart-line-chaikin-volatility (v1.11.788)

## Summary

Pure-SVG dual-panel React/TypeScript chart primitive with a
**Chaikin Volatility (CV)** oscillator panel beneath the close.
CV is the percentage rate of change of the EMA of the
high-minus-low range. Added as the
`<ChartLineChaikinVolatility />` UI primitive in
`web/src/components/ui/`.

## Algorithm

For each bar with lookback length `n` (default 10):

```
range[i] = high[i] - low[i]
ema[i]   = EMA(range, n)
CV[i]    = 100 * (ema[i] - ema[i - n]) / ema[i - n]
```

The EMA uses `alpha = 2 / (n + 1)` and seeds at the first finite
value. Bars before `i = 2 * n` are `null` (warmup: needs n bars
of seed plus n bars of lookback). When `ema[i - n] == 0`
(singular: flat HL) the bar is `null`.

## Anchors

- **CONST_HL (high - low == K, K != 0)**: range is constant ->
  both EMAs (current and lookback) carry the same drift, so the
  ratio `ema[i] / ema[i - n]` is exactly 1 in IEEE 754,
  yielding `CV = 100 * (1 - 1) = 0` **bit-exact**. Verified
  across `(range, length) in {1, 2, 5, 10, 100} x {3, 5, 10,
  14, 20}` (25 combos).
- **FLAT_HL (range == 0)**: ema = 0 every bar -> denominator
  zero -> `null` at every bar.
- **EMA helper**: zeros stay zero bit-exact, dyadic constant 2
  stays 2 bit-exact.
- **Expanding range**: range doubles partway through the series
  -> CV becomes positive in the second half.
- **Contracting range**: range halves partway through -> CV
  becomes negative in the second half.
- **Translation invariance**: shifting `(high, low)` by the
  same constant leaves CV unchanged within numerical tolerance.

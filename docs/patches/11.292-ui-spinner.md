# 11.292 -- Spinner primitive (v1.11.310)

New `Spinner` primitive at `web/src/components/ui/spinner.tsx`.
Component-scope only, no daemon or CLI surface change. Fresh
companion to the existing `web/src/components/Spinner.tsx`
(which keeps driving the loading-motion-integration suite).

## Why

Three motivations:

1. The dispatch asked for `web/src/components/ui/spinner.tsx` --
   the canonical ui/ primitive path. The existing
   `components/Spinner.tsx` was authored before the ui/
   primitive layer crystallised; new adopters reaching for a
   "loading spinner" expect to find it under ui/.
2. The existing Spinner ships SVG ring chrome + a 4-tone
   palette (primary / muted / inverse / destructive). New
   adopters want a smaller surface: 2 tones (neutral / accent
   from the dispatch), 4 sizes (xs through lg), and a CSS-only
   ring so the per-frame DOM stays tiny.
3. Reduced-motion users see a static ring on the legacy
   Spinner. The new primitive falls back to a visible text
   label ("Loading...") under reduced motion so a
   motion-sensitive operator gets the same affordance as a
   `textOnly` caller -- no surprise animation, no surprise
   static ring that could be mistaken for a focus indicator.

## What changed

### `web/src/components/ui/spinner.tsx` (new file)

- `Spinner` component + new exported types
  `SpinnerSize = 'xs' | 'sm' | 'md' | 'lg'` and
  `SpinnerTone = 'neutral' | 'accent'`.
- Props:
  - `size?: SpinnerSize` (default `'md'`). Footprints
    12px / 16px / 20px / 24px; border width scales
    1.5 / 2 / 2 / 3 px.
  - `tone?: SpinnerTone` (default `'neutral'`). Neutral uses
    `muted-foreground/40` ring + label; accent uses
    `primary` ring + label.
  - `label?: string` (default `"Loading"`). Aria-label on the
    wrapper. Also rendered as the visible text fallback
    under reduced motion or `textOnly`.
  - `textOnly?: boolean` (default false). Hides the ring AND
    surfaces the label as the visible affordance.
- The ring is a CSS border trick (top transparent, sides
  toned) rotated via Tailwind's `animate-spin`. Zero per-
  frame DOM; the GPU composites the animation off the main
  thread.
- Reduced-motion gate via `useReducedMotion()`. When the
  operator sets `prefers-reduced-motion: reduce`, the
  primitive drops the spinning ring AND falls back to the
  text label. Exposed as `data-reduced-motion="true|false"`
  + `data-render="ring|text"` on the wrapper.
- ARIA:
  - `role="status"` + `aria-live="polite"` on the wrapper.
  - `aria-hidden="true"` on the inner ring so SR users hear
    only the wrapper's `aria-label` (no doubled
    announcement).
- Data-attribute selectors:
  - `data-section="spinner"` on the wrapper.
  - `data-section="spinner-ring"` on the animated ring.
  - `data-section="spinner-text"` on the text fallback.
  - `data-size` + `data-tone` + `data-reduced-motion` +
    `data-render` on the wrapper for e2e + theming.

### Adopted

- Primitive only this commit. Re-exported from
  `web/src/components/ui/index.ts`.

## Deferred (dispatch follow-ups)

- Button loading state: the Button primitive carries its
  own ad-hoc `<Loader2 className="animate-spin">` mount.
  Swapping to `<Spinner size="sm" />` is a per-page
  follow-up so the reduced-motion text fallback flows
  through.
- Page transition loader: the existing RouteProgressBar
  primitive is a different visual (a thin top-of-page
  bar). A separate evaluation is needed.
- Modal save in-progress: dispatch sites (Snapshots delete,
  AttachModal busy state) currently render an inline
  Loader2 glyph. Adoption is straightforward per-page once
  the buttons opt into the new primitive.

## Notes

- The two Spinner components (`components/Spinner.tsx` and
  `components/ui/spinner.tsx`) coexist on purpose. The
  legacy module is tied into the cross-primitive
  loading-motion-integration tests; the new module is the
  canonical ui/ primitive for fresh adopters. A future
  consolidation can route both behind one
  `getLoadingMotionClass()` call without touching call
  sites.
- `useReducedMotion()` is plumbed at the same depth as the
  Drawer + Tooltip + Switch primitives so a future cross-
  primitive visual-regression sweep can target every
  motion-gated surface via the shared
  `data-reduced-motion` selector.

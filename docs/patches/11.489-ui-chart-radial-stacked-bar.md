## Summary

Add a new `<ChartRadialStackedBar>` UI primitive in
`web/src/components/ui/chart-radial-stacked-bar.tsx` --
pure-SVG **radial stacked bar** chart. Each category is
a concentric ring around the centre; within each ring,
the configured series stack **angularly** so the full
2pi of the ring is divided proportionally by each
series's value within that category. Distinct from
`<ChartRadialBar>` (11.452) (one arc per series, value
encodes arc length), `<ChartPolarArea>` (11.478)
(equal-angle wedges, radius encodes value), and
`<ChartStackedBar>` (11.468) (linear stacked bars):
this primitive is the rings-of-stacks variant where
each ring shows the categorical breakdown across the
visible series.

  onHiddenSeriesChange?: (hidden: string[]) => void;

## Behaviour

- Renders `<div role="region">` wrapping
  `<svg role="img">` with the same `aria-label`.
- Layout (`computeRadialStackedBarLayout`):
  - One ring per category. Rings stack outward from
    `innerRadius` to `outerRadius`, with `ringGap`
    pixels between consecutive rings.
  - `ringThickness = (outerRadius - innerRadius - gaps)
    / N` where N is the category count.
  - Within each ring, the visible series fill the
    full 2pi minus per-segment `padAngle`s; each
    segment occupies an arc proportional to
    `value / categoryTotal`.
  - Non-positive / non-finite values produce no
    segment (skipped from the ring entirely).
  - Hidden series (controlled or uncontrolled) drop
    their segments everywhere; the remaining series
    reproportion to fill the ring.
- Default `startAngle = -pi/2` so the first series
  in each ring starts at 12 o'clock.
- Per-series `color` always wins; otherwise the
  10-color palette maps each `seriesIndex` to its
  default colour, so the same series uses the same
  colour across every ring.
- Center disc shows the grand total of all visible
  segments; an optional `centerLabel` (small caption
  above the value) is rendered when supplied.
  `showCenterTotal=false` removes the centre group.
- Ring labels (`showRingLabels=true`, default) render
  the ring's category label above the ring's
  midradius for non-empty rings.
- Hover/focus on a segment sets the hovered key
  (`categoryId::seriesId`) and opens a tooltip with
  the category + series labels, the formatted
  value, and the share within the category.
- Mount animation:
  `motion-safe:animate-fade-in` per segment group.
  `animate=false` removes the class.
- Default ARIA description: `"Radial stacked bar
  chart with N rings and V visible series. Total
  <grand>."` Empty input / all-hidden / zero grand
  total -> `"No data"`.

## Pairs with existing primitives

- `<ChartRadialBar>` (11.452) -- concentric rings
  where each ring shows a *single* series value as
  an arc (radial-equivalent of a bar chart).
- `<ChartPolarArea>` (11.478) -- equal-angle wedges
  with radius encoding value (Nightingale rose).
- `<ChartStackedBar>` (11.468) / `<ChartGroupedBar>`
  (11.467) -- linear stacked / grouped siblings.
- `<ChartDonut>` (11.469) -- single-ring stacked
  donut alternative for one category.

## Out of scope

- 100%-rescaled rings (every ring sized to span
  2pi already; no additional rescale needed).
- Mixed orientation per ring (every ring shares
  start angle + pad angle).
- Animated transitions on data / hidden-series
  changes (snap v1; mount fade only).
- Per-series ordering different per ring (host
  re-orders the input array if needed).

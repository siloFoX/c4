## Summary

New `ChartLineRmta` UI primitive in `web/src/components/ui/chart-line-rmta.tsx`
-- a pure-SVG single-panel line chart with a **Recursive Moving Trend
Average** overlay folding price into a recursive EMA-like trend.

## The computation

The RMTA seeds from the first price and recurses with two corrections
per bar -- an EMA pull toward the current price plus a momentum lead.
`ChartLineRmtaPoint` carries `{x, value}`.

- `computeLineRmta(values, alpha, beta)`:
  - `rmta[0] = price[0]`.
  - `rmta[i] = rmta[i-1] + alpha * (price[i] - rmta[i-1]) + beta * (price[i] - price[i-1])`.
  - A non-finite value carries the prior RMTA forward.
- `classifyLineRmtaTrend(rmta, prevRmta)` -- marks a bar **up** when the
  RMTA rose, **down** when it fell, **flat** when unchanged, **none**
  for the first bar or for null inputs.

## Worked fixture

The recurrence has a clean exact identity when `alpha + beta = 1`.

Let `eta[i] = rmta[i] - price[i]`. Subtracting `price[i]` from both
sides of the recursion gives `eta[i] = (1 - alpha) * eta[i-1]`. Since
`eta[0] = rmta[0] - price[0] = 0`, induction gives `eta[i] = 0` for
every bar. So **whenever alpha + beta equals one, the RMTA tracks the
price exactly**, for any series; the recurrence reduces to
`rmta[i] = price[i-1] + (price[i] - price[i-1]) = price[i]` and that
subtraction-then-addition is FP-exact for integer prices.

The test asserts `toEqual(price)` with `alpha = beta = 0.5` on the
integer fixture `[10,30,25,45,40,55,50,65]`, and again with
`alpha = 0.25, beta = 0.75` to show the identity holds for asymmetric
weights summing to one. A constant series stays at the constant for any
weights (the EMA pull is zero and the price change is zero). A rising
series with `alpha + beta < 1` is checked structurally: the RMTA grows
monotonically and sits strictly between the prior RMTA and the current
price -- it lags the rise but tracks the trend.

## API

`data` (a `ChartLineRmtaPoint[]`), `alpha` (default 0.1), `beta`
(default 0.5), sizing/format/a11y props, `strokeWidth` / `dotRadius`,
the colour props, the toggle flags (`showAxis` / `showGrid` / `showDots`
default false / `showRmta` / `showMarkers` / `showTooltip` /
`showConfigBadge` / `showLegend` / `animate`), controlled +
uncontrolled `hiddenSeries` (keys `price` / `rmta`), `formatValue` /
`formatX`, `onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineRmtaFinitePoints`, `normalizeLineRmtaWeight`
(coerces a weight to a finite number in the open interval (0, 2)),
`computeLineRmta`, `classifyLineRmtaTrend`, `runLineRmta`,
`computeLineRmtaLayout`, `describeLineRmtaChart`, plus the
`DEFAULT_CHART_LINE_RMTA_*` constants and the `ChartLineRmtaTrend` type.
Every exported name carries a `LineRmta` infix so the `export *` barrel
line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen `data-section="chart-line-rmta-aria-desc"`
description span; the price line, the RMTA line and every dot / marker
are `role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-alpha`, `data-beta`,
`data-rmta-final`, `data-up-count`, `data-down-count`,
`data-flat-count`, `data-total-points` and `data-animate`.

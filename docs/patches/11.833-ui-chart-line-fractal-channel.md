# Patch 11.833 -- ChartLineFractalChannel primitive (v1.11.851)

## Summary

Add `<ChartLineFractalChannel />`, a pure-SVG single-panel chart that
overlays a Fractal Channel envelope on a price line. The channel
tracks the highest and lowest Bill Williams 5-bar fractal pivot
values inside a fixed-bar lookback window:

```
isUpperFractal[i] = high[i] is strict max of high[i - 2 .. i + 2]
isLowerFractal[i] = low[i]  is strict min of low [i - 2 .. i + 2]
                    (confirmed `fractalLookback` bars later)

upper[i] = max(confirmed upper-fractal high
               for indices in [max(0, i - length + 1) .. i])
lower[i] = min(confirmed lower-fractal low
               for indices in [max(0, i - length + 1) .. i])
```

- Channel is piece-wise constant between fractal events.
- Channel value drops to `null` when an old fractal scrolls off the
  lookback window without a new one having confirmed.
- `-0` is normalized to `+0` on every arithmetic result.

Distinct from the Fractal Keltner primitive (11.832), which uses a
single most-recent pivot anchor plus an ATR-derived half-width. This
primitive ignores ATR entirely and lets the pivot levels themselves
define the band.

## Defaults

| Knob              | Value     |
|-------------------|-----------|
| length            | 50        |
| fractalLookback   | 2         |
| priceColor        | `#2563eb` |
| upperColor        | `#7c2d12` (dark amber) |
| lowerColor        | `#14532d` (dark green) |
| upperFractalColor | `#ef4444` |
| lowerFractalColor | `#22c55e` |

Length 50 is loose enough that monthly daily data will normally have
multiple fractals inside the window; shorter periods can be passed
to tighten the channel.

## Bit-exact anchors

- `CONST high = low = K`: the strict-max / strict-min tests fail on
  flat sequences, so no fractals confirm and `upper = lower = null`
  everywhere. Verified across `(K, length)` sweeps.
- `LINEAR UP / LINEAR DOWN`: monotonic sequences also yield no
  fractals; the channel stays `null` throughout.
- `SAWTOOTH` with peaks at fixed `peakK` and troughs at fixed
  `troughL`, period 4 (peak / mid / trough / mid / peak / ...): every
  peak is a strict upper fractal and every trough is a strict lower
  fractal. Once the lookback window contains at least one confirmed
  pivot of each side, `upper = peakK` and `lower = troughL` bit-exact
  (max / min of identical IEEE 754 doubles is exact). Verified
  across `(peakK, troughL, length)` sweeps.

## Surface

Single-panel primitive surface:

- `ChartLineFractalChannelPoint = { x, high, low, close }`
- `runLineFractalChannel(data, options?) -> Run` with
  `ok = series.length >= 2 * fractalLookback + 1`.
- `computeLineFractalChannelLayout({ data, width?, height?, padding?, length?, fractalLookback? }) -> Layout`.
  Single panel; y-axis auto-fits across `high/low/upper/lower`.
- `describeLineFractalChannelChart(data, opts?) -> string` for the
  ARIA description.
- `<ChartLineFractalChannel />` `forwardRef<HTMLDivElement>` with
  three legend toggles (`price`, `upper`, `lower`), config badge,
  fractal-pivot scatter markers (upper red, lower green), and a
  tooltip surfacing `close / upper / lower / width / zone / cross`.

# 11.1073 -- UI: chart-line-cmo-zero-cross-sig primitive (v1.11.1091)

## Scope

Add a pure-SVG dual-panel `ChartLineCmoZeroCrossSig`
component that renders the close in the top panel and the
Tushar Chande Momentum Oscillator (CMO) along with its
smoothed SMA signal line in the bottom panel, marking
bullish (CMO crosses up through signal -- centerline
momentum trigger; CMO pulling above its smoothed mean) /
bearish (CMO crosses down through signal -- centerline
momentum trigger; CMO pulling below its smoothed mean)
CMO-vs-signal crossover trigger events with bias coloring
derived from the (CMO - signal) slope at the trigger bar.

The CMO was introduced by Tushar Chande in 1994 as a
zero-centered alternative to the RSI: it normalises the
up-minus-down momentum sum by the up-plus-down total, so
the result lives in [-100, +100] with zero as the
centerline. The CMO-vs-signal cross is the canonical
centerline momentum trigger -- when the centerline-
oscillating CMO pulls above (or below) its smoothed mean,
recent momentum balance has tilted up (or down).

## CMO zero cross-sig pipeline

```
For each i = 1..n-1:
  diff[i] = close[i] - close[i-1]
  up[i]   = max(diff[i], 0)
  dn[i]   = max(-diff[i], 0)

For each i >= period:
  Su[i]   = sum(up[i-period+1..i])
  Sd[i]   = sum(dn[i-period+1..i])
  CMO[i]  = (Su[i] === 0 && Sd[i] === 0)
              ? 0
              : ((Su[i] - Sd[i]) / (Su[i] + Sd[i])) * 100

signal[i] = SMA(CMO, signalLength)[i]

bullish (CMO-cross up) :
  prev CMO <= prev signal && cur CMO > cur signal
bearish (CMO-cross down) :
  prev CMO >= prev signal && cur CMO < cur signal

regime : 'bullish' when CMO >= signal
         'bearish' when CMO <  signal
         'none'    when either is null
bias   : (CMO - signal)[i] vs prev -> up/down/flat/none
```

Default `period = 14`, `signalLength = 9` -- canonical
Chande + cross-sig family signal length pairing.

## Bit-exact anchors

All anchors use single-close input.

- **CONST** `close = K`: All diff[i] = 0 -> up = dn =
  0 -> Su = Sd = 0. Divide-by-zero guard returns CMO
  = 0 (constant). signal = SMA(0) = 0. CMO === 0 ===
  signal every bar -> regime `bullish` (via >=). 0
  crosses. Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `close = i`: All diff[i] = +1 ->
  up[i] = 1, dn[i] = 0. Su = period, Sd = 0. CMO =
  (period - 0) / (period + 0) * 100 = **100**
  (constant from i = period). signal = SMA(100) = 100.
  CMO === signal = 100. regime `bullish`. 0 crosses.
- **LINEAR DOWN** `close = -i`: All diff[i] = -1 ->
  up = 0, dn = 1. Su = 0, Sd = period. CMO = (0 -
  period) / (0 + period) * 100 = **-100** (constant).
  signal = -100. CMO === signal. regime `bullish` (via
  ===, family convention). 0 crosses.
- **PEAK** (linear up then linear down): CMO climbs to
  100 in the first half, then falls toward -100 in the
  second half. The signal SMA lags behind. At the
  reversal, CMO falls below signal -> bearish cross
  fires. At least one cross is produced.

The bit-exact bound at +/-100 is a unique CMO signature:
the normalised numerator-over-denominator ratio
guarantees CMO never exceeds the [-100, +100] band, so
the chart panel can be hard-locked to this range with
no auto-scaling.

The family convention `bullish on ===` (via `>=`)
produces a slightly asymmetric label on LINEAR DOWN:
the price is falling but the cross-sig regime is
"bullish" because CMO === signal. This is the deliberate
family convention -- the regime measures the indicator-
vs-signal relationship, not the price direction.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 3-state regime classifier follows the cross-sig
  family convention (bullish on `===` via `>=`).
- Divide-by-zero guard returns 0 when Su + Sd = 0
  (matches pandas-ta and ta-lib conventions). Verified
  by the CONST anchor which produces CMO = 0 throughout
  without throwing.
- The +/-100 bound is mathematically guaranteed by the
  normalisation, so the panel can be hard-locked. This
  is a unique CMO property distinct from the family's
  unbounded oscillators (Momentum, raw differences).
- Family convention `bullish on ===` produces
  asymmetric LINEAR DOWN labelling (price falls but
  regime bullish). Documented as the family standard.
- SMA-based signal smoothing (not EMA) for bit-exact
  testability. The PEAK anchor verifies that the
  signal lags behind CMO at trend reversals, producing
  the canonical bearish cross.
- Eighth-or-so member of the cross-sig family
  (depending on count); pairs naturally with the
  zero-centered oscillator cousins (Momentum, ROC,
  TRIX, DPO mid-cross-sig variants).

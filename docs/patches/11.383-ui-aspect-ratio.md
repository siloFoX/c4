## What changed

### `forceFallback` prop

```ts
interface AspectRatioProps {
  // ... existing
  forceFallback?: boolean;
}
```

Default `false`. When `true`, the wrapper
sets `padding-bottom: <h/w>%` instead of
`aspect-ratio`. The content slot stays
absolutely positioned inside as before.

### Runtime feature detection

`supportsAspectRatio()` is exported as a
helper. Reads `'aspectRatio' in
document.documentElement.style` once per
process (memoized). When unsupported, the
default path silently falls back to
`padding-bottom`. SSR returns `true` so
the modern path renders on the server
without hydration mismatch.

`__resetAspectRatioSupportCache()` is
exported as a test hook to clear the
memoized value.

### `ratioToPaddingBottom()` helper

```ts
export function ratioToPaddingBottom(
  input: AspectRatioPreset | number | undefined,
): string;
```

Returns the percent string (e.g.
`'56.25%'` for `'16:9'`). Adopters use it
to inline the trick in non-AspectRatio
surfaces.

### `data-fallback` attr

`data-fallback="true"|"false"` mirrors
which path rendered. Downstream tests +
storyshots can assert the resolved
behaviour without parsing inline styles.

## Padding-bottom math

```
padding-bottom = (height / width) * 100%
```

| ratio | padding-bottom |
| --- | --- |
| `16:9` | `56.25%` |
| `4:3` | `75%` |
| `1:1` | `100%` |
| `21:9` | `42.857%` |
| `9:16` | `177.778%` |
| numeric `2` | `50%` |
| numeric `4` | `25%` |

Values clamped to 3 decimals for
stability across browsers.

## Out of scope

- Per-page adoption of `forceFallback`.
  Additive; every existing call site
  stays byte-identical on the modern
  path.
- Container queries for ratio
  responsiveness. Aspect-ratio is a
  width-only contract; per-breakpoint
  ratios belong in a separate
  `ResponsiveAspectRatio` primitive.
- CSS `@supports` feature detection in
  the stylesheet. Runtime detection +
  memoization is cheaper than per-
  surface CSS branching and keeps the
  inline-style story stable under SSR.
- Auto-flush of the feature-detection
  cache on document visibility change.
  Tests that need to re-probe use the
  `__resetAspectRatioSupportCache()`
  hook.
- More preset ratios (e.g., 5:4, 3:2).
  The shipped six cover the common
  cases; numeric `ratio={n}` covers
  edge cases.

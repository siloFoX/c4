## Summary

New `ChartLinePvt` UI primitive at
`web/src/components/ui/chart-line-pvt.tsx` -- a pure-SVG line chart
with a Price Volume Trend cumulative line. The top panel renders the
close price line; the bottom panel renders the PVT line on its own
data-driven y-axis with a dashed zero reference line and markers
coloured by money flow direction.

## Indicator math

The Price Volume Trend:

```
roc[i] = (close[i] - close[i-1]) / close[i-1]
pvc[i] = volume[i] * roc[i]
PVT[i] = PVT[i-1] + pvc[i]    with PVT[0] = 0
```

`computeLinePvt` scales each bar's volume by its percentage price
change into a price-volume contribution, and the PVT is the running
cumulative total of that contribution. Index 0 has no prior close so
its contribution is `0`; a zero prior close reads a zero rate of
change. Like the ADL the PVT has **no warm-up** -- it is defined from
the first bar.

### Worked fixture

Closes are powers-of-two ratios so the percentage change is exact:

```
close  = [8, 12, 6, 6, 12]    volume = [100, 20, 40, 100, 30]
roc    = [0, 0.5, -0.5, 0, 1]
pvc    = volume * roc = [0, 10, -20, 0, 30]
pvt    = cumulative   = [0, 10, -10, -10, 20]
```

`pvtFinal = 20`, `pvtMin = -10`, `pvtMax = 20`, two rising bars and
one falling.

## API

- `getLinePvtFinitePoints(points)` -- drop points with a non-finite
  `x`, `close` or `volume`.
- `computeLinePvt(closes, volumes)` -- returns `{ roc, pvc, pvt }`.
- `runLinePvt(points)` -- sort finite points by `x`, compute the
  series, return `{ series, roc, pvc, pvt, samples, pvtFinal,
  pvtMin, pvtMax, risingCount, fallingCount, ok }`. Each sample
  carries a `rising` / `falling` / `flat` flow. `ok` is false for
  fewer than two points. The PVT takes no period parameter.
- `computeLinePvtLayout(options)` -- stack a price panel above a PVT
  panel with a data-driven y-domain. Returns panels, ticks, the
  price path, the PVT path, flow-coloured markers, and the zero
  reference line `y` plus a `zeroInRange` flag.
- `describeLinePvtChart(data)` -- ARIA description.

## Component

`ChartLinePvt` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge, a two-series legend (Price / PVT) with toggle, a dashed zero
reference line, the PVT line, markers coloured by flow (rising green,
falling red), a hover/focus tooltip exposing the close, volume,
contribution, PVT, and flow, and a `motion-safe:animate-fade-in`
entrance. Every node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-obv** -- On-Balance Volume adds or subtracts the
  *whole* volume by the close's up/down direction. The PVT adds only
  the *fraction* of volume proportional to the size of the
  percentage price move, so a 1% move with a given volume shifts the
  line a tenth as far as a 10% move would.
- **vs chart-line-adl** -- the ADL weights volume by where the close
  sits within the bar's high-low range. The PVT weights it by the
  close-to-close *percentage change*; it needs no high or low.
- **vs chart-line-force-index** -- the Force Index multiplies the
  *absolute* price change by volume and EMA-smooths it. The PVT uses
  the *percentage* change and is a raw cumulative total.

# 11.776 chart-line-trend-quality (v1.11.794)

## Summary

Pure-SVG dual-panel React/TypeScript chart primitive with a
**Trend Quality Index (TQI)** oscillator panel beneath the
close. The TQI is the Kaufman Efficiency Ratio scaled to 100:
the signed net price change over the total path length across
a lookback window. Added as the `<ChartLineTrendQuality />`
UI primitive in `web/src/components/ui/`.

## Algorithm

For each bar with lookback `n` (default 14):

```
netChange[i] = close[i] - close[i - n]
sumAbs[i]    = sum_{j = i - n + 1..i} |close[j] - close[j - 1]|
TQI[i]       = 100 * netChange[i] / sumAbs[i]
```

The TQI is bounded in `[-100, +100]` by the triangle
inequality (`|netChange| <= sumAbs` always). A straight rising
trend yields `+100`; a straight falling trend yields `-100`;
chop with no net change yields values near zero. Bars before
`i = lookback` are `null` (warmup). When the path length sum
is zero (singular: completely flat close) the bar is `null`.

A configurable `strongThreshold` (default 70) tags strong-up /
strong-down zones; readings between are "weak".

## Anchors

- **RISING_BY_S (close[i] = c0 + S * i, S > 0)**:
  netChange = S * lookback, each step contributes |S| = S,
  sumAbs = lookback * S, so `TQI = +100` **bit-exact** at
  every bar past the warmup. Verified across
  `(S, lookback) in {1, 2, 5, 10, 100, 0.5, 0.25} x
  {3, 5, 7, 14, 20}` (35 combos).
- **FALLING_BY_S**: same lookback magnitude, opposite sign ->
  `TQI = -100` **bit-exact**. Verified across the same 35
  combos.
- **CONST_FLAT (close == K)**: `netChange = 0` and `sumAbs =
  0` -> singular -> `TQI = null` at every bar.
- **V-shape (rise then fall to same level)**: `netChange = 0`
  and `sumAbs > 0` -> `TQI = 0` bit-exact at the apex.
- **Worked anchor**: `close = [5, 10, 5]`, lookback 2 ->
  `netChange = 0`, `sumAbs = 10`, `TQI = 0` bit-exact.
- **Bounded**: `|TQI| <= 100` for any finite input.

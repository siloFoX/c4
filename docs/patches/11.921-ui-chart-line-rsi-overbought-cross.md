# Patch 11.921 -- UI: chart-line-rsi-overbought-cross primitive

## Summary

Added `<ChartLineRsiOverboughtCross />` -- pure-SVG dual-panel
React/TS primitive that renders the close (top panel) with
bullish (entry) / bearish (exit) arrow overlays at every RSI
overbought threshold cross trigger, and the close-only RSI
line (bottom panel) on a fixed 0-100 oscillator with 50 / 70
reference bands. Single-threshold cross variant of the RSI
family that flags the discrete level-70 entry and exit events
distinct from RSI regime classification.

## Algorithm

```
delta[i]   = close[i] - close[i-1]
gain[i]    = max(delta, 0)
loss[i]    = max(-delta, 0)
avgGain[i] = Wilder smoothing of gain over length
avgLoss[i] = Wilder smoothing of loss over length
rsi[i]     = avgGain + avgLoss == 0
              ? 50  (neutral fallback)
              : avgLoss == 0
                ? 100
                : 100 - 100 / (1 + avgGain / avgLoss)
bullish   : prev <= threshold && cur > threshold  (entry)
bearish   : prev >= threshold && cur < threshold  (exit)
```

Defaults `length = 14` (canonical RSI window), `threshold = 70`
(overbought level). Regime classifier `bullish` (RSI >=
threshold, overbought), `neutral` (RSI < threshold, normal),
`none` (RSI null).

## Bit-exact anchor

CONST close = K: every delta = 0 -> gain = loss = 0 ->
avgGain = avgLoss = 0 -> RSI uses the neutral fallback 50.
50 < 70 so regime is `neutral` and the threshold is never
crossed. cross count = 0. Verified across K = 0, 1, 5, 17,
100, 1234.

LINEAR UP yields RSI = 100 every settled bar (avgLoss = 0,
short-circuit branch). 100 >= 70 -> regime `bullish`. LINEAR
DOWN yields RSI = 0 (avgGain = 0) -> regime `neutral` (below
threshold).

  arrows at threshold cross bars.
- Bottom: RSI line + cross dot markers + 50 / threshold
  reference bands.
- Tooltip lists x / close / RSI / regime / entries / exits /
  crosses.

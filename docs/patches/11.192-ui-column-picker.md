# 11.192 -- UI: ColumnPicker primitive (v1.11.210)

## Summary

New `<ColumnPicker>` primitive that lets users hide / show columns in a
table or list. Opens a `<Popover>` panel from a small trigger button
(lucide `Columns3` icon), lists one `<Checkbox>` per column, and offers
a `Reset` footer button that restores every column to visible. Optional
`storageKey` persists the visible-id list to `localStorage` and seeds
from it on mount.

## Props

```ts
interface ColumnPickerProps {
  columns: Array<{ id: string; label: ReactNode; alwaysVisible?: boolean }>;
  value: string[];
  onChange: (visible: string[]) => void;
  storageKey?: string;
  buttonLabel?: ReactNode; // default 'Columns'
  className?: string;
  ariaLabel?: string;
}
```

## Behavior notes

- `alwaysVisible` columns are rendered disabled in the popover and
  appear force-checked; their id is ignored when toggling.
- `Reset` emits `onChange(columns.map(c => c.id))` (all visible) and,
  when `storageKey` is set, mirrors the write to `localStorage`.
- The seed-from-`localStorage` effect runs once per mount (guarded by a
  `seededRef`) so a stale stored list never clobbers a parent that has
  already updated `value` after the first render.
- `forwardRef` to the trigger button. Because the existing `<Popover>`
  clones its trigger and overrides the ref via `cloneElement`, the
  primitive instead tags the trigger with a `useId`-derived
  `data-column-picker-trigger` attribute and resolves the DOM node in a
  `useEffect`.
- The trigger has `aria-label` only when an explicit `ariaLabel` prop
  is provided; otherwise its accessible name is the visible
  `buttonLabel` text (so `getByRole('button', { name: /columns/i })`
  works without an override).

# 11.209 -- UI: list virtualizer hook

Standalone `useListVirtualizer` hook for the web UI -- a small,
dependency-free, fixed-row windowed-list primitive. Component-scope
only: no daemon / no API surface. Replaces the v1.11.197 `VirtualList`
primitive at the `HistoryView` sidebar callsite, and is published as
the new go-to import for any future long-list adopter that wants the
windowing math without committing to a wrapping component.

## Files

- `web/src/hooks/use-list-virtualizer.ts` (new, ~120 lines) -- the
  hook itself, plus `VirtualItem`, `UseListVirtualizerOptions`, and
  `UseListVirtualizerResult` exported types.
- `web/src/hooks/use-list-virtualizer.test.tsx` (new, 9 cases) --
  dedicated unit coverage.
- `web/src/components/HistoryView.tsx` -- sidebar summary list
  migrated off `VirtualList` onto the hook through a new
  `SidebarVirtualizedList` extracted at the bottom of the file.

## API

```ts
interface UseListVirtualizerOptions {
  itemCount: number;
  itemHeight: number;            // fixed row height in v1
  overscan?: number;             // default 3
  containerRef?: RefObject<HTMLElement | null>;
}

interface VirtualItem { index: number; top: number; }

interface UseListVirtualizerResult {
  items: VirtualItem[];
  totalHeight: number;           // itemCount * itemHeight
  offsetY: number;               // startIndex * itemHeight
  containerProps: {
    onScroll: (e: UIEvent<HTMLElement>) => void;
    style: CSSProperties;        // overflowY: 'auto', position: 'relative'
  };
}

function useListVirtualizer(
  opts: UseListVirtualizerOptions,
): UseListVirtualizerResult;
```

## Behaviour

- **Visible-range math**: `rawStart = floor(scrollTop / itemHeight)`.
  `visibleSpan = ceil(viewport / itemHeight)`. The window spans
  `[max(0, rawStart - overscan), min(itemCount, rawStart + visibleSpan
  + overscan))`. `offsetY = startIndex * itemHeight` -- the caller
  applies it as `transform: translateY(offsetY px)` on the inner
  block so absolute positioning per row is unnecessary.
- **Tail clamp**: a runaway `scrollTop` past the end (e.g. the user
  page-downs into nowhere, or a row delete shrinks the list while
  scroll position survives) collapses the window onto the tail
  `[max(0, itemCount - (visibleSpan + 2*overscan)), itemCount)` so
  callers always see a sensible last-page render instead of an empty
  band.
- **Viewport source**: the first scroll event captures
  `event.currentTarget.clientHeight` and stores it in `useState`.
  Before that first event, the initial viewport falls back to
  `containerRef.current.clientHeight` (when the caller provides a
  ref), then to `window.innerHeight` (when `window` is defined),
  then to `0`. There is no `ResizeObserver` in v1; container resize
  refreshes on the next scroll.
- **SSR-safe**: every `window` access is guarded by `typeof window !==
  'undefined'`. Under SSR (`window` undefined) the hook collapses to
  a 0 viewport and returns an overscan-sized initial band rather
  than crashing.
- **Fixed itemHeight**: v1 does not measure rows. `itemHeight` is a
  scalar; mismatched DOM heights produce visual misalignment, which
  is the documented price of opting into virtualization here.

## HistoryView integration shape

The sidebar workers list (previously a v1.11.197 `VirtualList`
adoption) now reads from a small `SidebarVirtualizedList` component
at the bottom of `web/src/components/HistoryView.tsx`. The component
owns the hook call and the outer + inner DOM shape so the orchestrator
function (`HistoryView`) stays readable:

```tsx
function SidebarVirtualizedList({ filtered, ... }) {
  const { items, totalHeight, offsetY, containerProps } =
    useListVirtualizer({
      itemCount: filtered.length,
      itemHeight: SIDEBAR_ITEM_HEIGHT, // 64
    });
  return (
    <div
      role="list"
      aria-label={t('history.sidebar.title')}
      className="pr-1 overflow-auto"
      style={{ ...containerProps.style, height: '60vh' }}
      onScroll={containerProps.onScroll}
    >
      <div style={{ height: totalHeight, position: 'relative' }}>
        <div style={{ transform: `translateY(${offsetY}px)` }}>
          {items.map(({ index }) => {
            const w = filtered[index];
            // ... existing button row, byte-equivalent to prior render
          })}
        </div>
      </div>
    </div>
  );
}
```

`itemHeight = 64` matches the existing badge row + meta row +
`space-y-1` gap. The outer `60vh` height is preserved from the prior
`VirtualList height="60vh"` prop so the visual surface does not
shift. ARIA wiring (`role="list"` + `role="listitem"` + the
`aria-label` driven by the localized `history.sidebar.title`) is
preserved verbatim. Bulk-select + click + context-menu handlers are
all unchanged.

`HistoryDetailPane`'s past-tasks list is intentionally **not**
migrated in this patch -- those rows are variable-height (commit
list, status badge, optional branch chip) and v1's fixed-row contract
would visually misalign them. A future patch can either (a) split
the rows into a fixed-shape variant or (b) extend the hook with
dynamic measurement before wiring it through.

## Test coverage

9 vitest + RTL cases in `web/src/hooks/use-list-virtualizer.test.tsx`:

1. Returns a top-anchored window of items at scrollTop=0 (default
   render, viewport from `window.innerHeight` mocked to 600).
2. Updates the visible window on a scroll event (mocked
   `currentTarget.scrollTop` + `clientHeight`).
3. Honours custom `overscan` above + below the visible band.
4. `totalHeight = itemCount * itemHeight`.
5. `offsetY` moves with `startIndex` as the user scrolls (`overscan=0`
   so the bare math is asserted, no buffer noise).
6. `itemCount=0` returns `[]` items + `totalHeight=0`.
7. Clamps the visible window to the last items when scrolled past
   the end (`scrollTop=9999` against a 30-row list collapses onto
   the tail; no item index over-runs `itemCount`).
8. Uses `containerRef.current.clientHeight` as the initial viewport
   when provided (`clientHeight=320`, `itemHeight=40`, `overscan=0`
   -> exactly 8 rows render).
9. Falls back to a 0 viewport when `window.innerHeight` is
   unavailable (SSR-safe path -- the hook returns the overscan-sized
   initial band without throwing).

## What it is

Pure-SVG line chart with a **1D Rauch-Tung-Striebel (RTS)
smoother** and a shaded uncertainty band. Where
`<ChartLineKalman>` runs only the *forward filter*,
`<ChartLineKalmanSmoother>` runs the forward filter **and**
the RTS backward pass, and draws all three lines on shared
axes -- raw observations, the causal forward-filter estimate
(dashed), and the RTS smoothed estimate (solid) -- so the
difference between filtering and smoothing is visible.

## Math

The forward **Kalman filter** is causal: at step k it
estimates the state from observations `z_1 .. z_k` only --
`x_k | z_1..z_k`. It is the online estimate.

The **RTS smoother** adds a backward pass that refines every
estimate using **all** observations `z_1 .. z_N`, so each
smoothed estimate is `x_k | z_1..z_N`. Backward recursion for
the random-walk model (run from `k = N-2` down to 0; the
final point's smoothed estimate equals the filter estimate):

```
C_k        = P_filt_k / P_pred_{k+1}
x_smooth_k = x_filt_k + C_k * (x_smooth_{k+1} - x_pred_{k+1})
P_smooth_k = P_filt_k + C_k^2 * (P_smooth_{k+1} - P_pred_{k+1})
```

`x_pred_{k+1}` / `P_pred_{k+1}` are the one-step predictions
the forward filter recorded going into step k+1. The forward
pass is reused directly: `runRtsSmoother` calls
`runLineKalmanFilter` (imported from `./chart-line-kalman`)
and then layers the backward recursion on top -- the smoother
*is* "the filter plus a backward pass", and the code says so.

### Key property

`P_smooth_k <= P_filt_k` for every k -- the smoother is at
least as confident as the filter everywhere, strictly
tighter except at the final point. So the smoothed
uncertainty band is **narrower** than the filter band, and
the badge reports the mean variance reduction the backward
pass buys.

### Verified properties (by test)

- With **Q = 0** (constant-state model) the smoother
  collapses every estimate onto the final filter estimate
  and every variance onto the final (smallest) filter
  variance. For the fixture `[10, 12, 8, 10]` the forward
  filter estimates are `[10, 32/3, 10, 10]`; the smoothed
  estimates are all `10` and the smoothed variances all
  `0.2`.
- The final point's smoothed estimate and variance equal the
  filter's, for any Q.
- The smoother never increases the variance over the filter.
- A single point: the smoother equals the filter.
- The smoother gain is 0 at the final point.

## Distinct from `<ChartLineKalman>` (11.538)

- `<ChartLineKalman>` runs the **forward filter only** --
  causal, online, `x_k | z_1..z_k`. Each estimate is "best
  so far".
- `<ChartLineKalmanSmoother>` runs the filter **plus the RTS
  backward pass** -- non-causal, offline, `x_k | z_1..z_N`.
  Each estimate is "best given everything", and is strictly
  lower-variance.

The smoother chart additionally shows the forward-filter
estimate as a dashed line for direct comparison, fills the
*smoothed* uncertainty band as the primary band, and
optionally overlays the filter band edges (`showFilterBand`)
so the band-narrowing is visible.

## Public surface

- `<ChartLineKalmanSmoother series={...} />`.
- `series: ChartLineKalmanSmootherSeries[]` --
  `{id, label, data, color?, processNoise?, measurementNoise?,
  initialEstimate?, initialVariance?, kSigma?}`. Per-series
  Q / R / x_0 / P_0 / k always beat chart-level.
- `processNoise` Q (default 0.01), `measurementNoise` R
  (default 1), `initialVariance` P_0 (default 1), `kSigma`
  (default 2).
- `obsColor`, `filterColor`, `bandOpacity`, `obsOpacity`.
- Toggle flags: `showAxis`, `showGrid`, `showDots`,
  `showLegend`, `showTooltip`, `showConfigBadge`,
  `showObservations`, `showFilter`, `showSmoothed`,
  `showBand`, `showFilterBand`, `animate`.
- Controlled + uncontrolled visibility.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX` / `formatCoefficient`.
- `onPointClick({series, point})`.

## Pure helpers

- `getLineKalmanSmootherDefaultColor(index)`.
- `getLineKalmanSmootherFinitePoints(points)`.
- `runRtsSmoother(points, options?)` -- forward filter
  (via `runLineKalmanFilter`) plus the RTS backward pass;
  returns per-point `{index, x, observation, filterEstimate,
  filterVariance, smoothedEstimate, smoothedVariance,
  smootherGain, filterUpper, filterLower, smoothedUpper,
  smoothedLower}`.
- `computeLineKalmanSmootherLayout({...})` -- projects the
  observation / filter / smoothed paths and the smoothed band
  + filter band edges; per-series carries the mean filter /
  smoothed variance and the variance-reduction percentage.
- `describeLineKalmanSmootherChart(series, options?)`.

## Tooltip / badge

Hovering a dot shows the series label, x, observation,
filter estimate, bold smoothed estimate, filter variance and
smoothed variance. Config badge: `RTS Q=<q> R=<r>
-<reduction>% var`.

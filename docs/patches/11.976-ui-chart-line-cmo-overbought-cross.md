## Summary

Adds the `ChartLineCmoOverboughtCross` primitive
(`web/src/components/ui/chart-line-cmo-overbought-cross.tsx`): a
pure SVG dual-panel chart that visualizes Chande Momentum
Oscillator (CMO) crossings through the canonical overbought line
50 for momentum overbought trigger entry / exit events. Joins the
existing `chart-line-cmo-zero-cross` (TODO 11.946) and
`chart-line-cmo-mid-cross` (TODO 11.966) variants -- same CMO
pipeline, threshold flipped from 0 / midline to the +50 overbought
level.

## Pipeline

Same close-only CMO pipeline shared with the zero / mid siblings:

1. `delta[i] = close[i] - close[i-1]`
2. `gain[i] = max(delta, 0)`, `loss[i] = max(-delta, 0)`
3. `sumGain = sum(gain, length)`, `sumLoss = sum(loss, length)`
4. `cmo[i] = (sumGain - sumLoss) / (sumGain + sumLoss) * 100`
   - When `sumGain + sumLoss == 0` (no movement in the window),
     short circuit to `cmo[i] = 0` (bit-exact +0, NOT -0;
     verified via `Object.is`).
5. Regime classifier (strict `>=`):
   - `cmo >= 50` -> bullish (in overbought zone)
   - `cmo <  50` -> bearish (below overbought)
6. Cross detector emits a `bullish` cross when
   `cmo[i-1] <= 50 && cmo[i] > 50` (entry) and a `bearish` cross
   when `cmo[i-1] >= 50 && cmo[i] < 50` (exit). Strict
   inequality on the current side ensures the boundary value 50
   does not double-fire.

## Defaults

- `length = 14` (canonical CMO window)
- `threshold = 50` (canonical overbought line)
- Warmup boundary: bar `length = 14`, so the first 14 samples
  (30-bar run) are null and the remaining 16 are populated.

## Bit-exact anchors

- CONST `close = K` for all `K in {0, 1, 42, 100, 1234}` ->
  `cmo[i] === 0` literal (NOT `-0`) for every `i >= 14`. Regime
  classifies as bearish because 0 < 50 -- the exact mirror of the
  mid-cross variant (where 0 sat on threshold 0 and read bullish)
  and consistent with the zero-cross variant's neutral CMO=0
  fallback at the threshold-50 framing.
- LINEAR UP `close = i` -> `cmo[i] === 100` constant. All
  bullish (100 >= 50), 0 crosses (jumps directly from null to
  100, no `prev <= T && cur > T` transient).
- LINEAR DOWN `close = -i` -> `cmo[i] === -100` constant. All
  bearish, 0 crosses.
- Balanced alternation `close = i % 2 ? 11 : 10` -> `cmo[i] === 0`
  (sumGain == sumLoss).
- CONST K full-run counters under defaults, 30 bars:
  `noneCount = 14`, `bullishCount = 0`, `bearishCount = 16`,
  `crosses.length = 0`.
- `pricePath` and `cmoPath` are byte-identical across repeated
  layout calls, with `.toFixed(2)` precision and `M ... L ...`
  command shapes. The cmo path uses a single `M` for post-warmup
  data.
- `thresholdY` at 50 sits ABOVE the panel midpoint of `[-100,
  100]` (smaller y in SVG space).
- Cross detector boundary: strict inequality (`> T`, `< T`).
- Decline-then-rise transient yields a confirmed bullish
  overbought-entry cross (the canonical use case).

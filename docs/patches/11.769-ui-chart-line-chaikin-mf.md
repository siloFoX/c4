# 11.769 chart-line-chaikin-mf (v1.11.787)

## Summary

Pure-SVG dual-panel React/TypeScript chart primitive with a
**Chaikin Money Flow (CMF)** oscillator panel beneath the close.
The CMF is the sum of money flow volume divided by the sum of
volume over a lookback window. Added as the
`<ChartLineChaikinMf />` UI primitive in `web/src/components/ui/`.

## Algorithm

For each bar with lookback `n` (default 20):

```
MFM[i] = ((close - low) - (high - close)) / (high - low)
       = (2 * close - high - low) / (high - low)
MFV[i] = MFM[i] * volume[i]
CMF[i] = sum(MFV[i - n + 1..i]) / sum(volume[i - n + 1..i])
```

When `high == low` the MFM is treated as zero (no money flow
contribution from that bar). Bars before `i = n - 1` are `null`
(warmup). When the volume sum over the window is zero, the
result is `null` (singular).

## Anchors

- **Close at high (close = high, high > low)** with constant
  positive volume: MFM = 1 at every bar -> MFV = volume,
  numerator = sum(volume), denominator = sum(volume),
  `CMF = +1` **bit-exact**.
- **Close at low**: MFM = -1, `CMF = -1` bit-exact.
- **Close at midpoint** (`close = (high + low) / 2`): MFM = 0,
  numerator = 0, `CMF = 0` bit-exact.
- **Flat bar (high == low)** with positive volume: MFM = 0,
  `CMF = 0` bit-exact.
- **ZERO_VOLUME** window: denominator = 0 -> `null`.
- **Mixed**: equal counts of close-at-high and close-at-low
  bars at the same volume cancel to `CMF = 0` bit-exact.
- **Worked dyadic MFM anchors**:
  `(h, l, c) = (10, 2, 8) -> MFM = 0.5` bit-exact,
  `(10, 2, 4) -> MFM = -0.5` bit-exact.
- **Bounded**: CMF is always in `[-1, 1]` for finite, mixed
  fixtures.
- **CMF = +1 verified across** `(range, volume, length) in
  {1, 2, 5, 10, 100} x {1, 50, 100, 1000} x {5, 14, 20, 30}`
  (80 combos).

## Summary

cycle with a highpass subtraction.

## Indicator math

John Ehlers' Decycler rests on the idea that a price series is
the sum of a slow trend and a fast cycle:
`price = trend + cycle`. If a highpass filter extracts the cycle,
then subtracting it leaves the trend:
`decycler = price - highpass`.

`computeLineDecyclerAlpha(period)` is the Ehlers one-pole
highpass coefficient for a cutoff `period`:

```
alpha = (cos(w) + sin(w) - 1) / cos(w),   w = 2*pi / period
```

clamped to the stable range `[0, 1]`. The lone degenerate integer
period -- 4, where `cos(w)` is zero -- returns 1.

`computeLineDecyclerHighpass(values, period)` runs the one-pole
highpass recursion

```
HP[i] = (1 - alpha/2) * (v[i] - v[i-1]) + (1 - alpha) * HP[i-1]
```

seeded with `HP[0] = 0`. `computeLineDecycler(values, period)`
subtracts it: `decycler[i] = v[i] - HP[i]`.

### Exactness

The highpass recursion uses `cos` and `sin` of `2*pi/period`, so
the decycler is transcendental in general and the varying-series
tests assert with `toBeCloseTo`. Three properties are exact and
asserted with `toEqual` / `toBe`:

- the highpass is seeded at zero, so `decycler[0] = price[0]`
  exactly for any input;
- a flat price has no bar-to-bar change, so the highpass stays
  zero and the decycler equals the price exactly;
- `decycler[i] = price[i] - HP[i]` is an exact identity.

Two further checks pin the transcendental filter: the recursion
relationship is verified bar by bar against
`computeLineDecyclerAlpha`, and a long constant-slope ramp is
checked against the closed-form steady state
`HP* = (1 - alpha/2) * slope / alpha`.

## API

- `getLineDecyclerFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineDecyclerPeriod(period, fallback)` -- coerce to an
  integer of at least 2.
- `computeLineDecyclerAlpha(period)` -- the highpass coefficient.
- `computeLineDecyclerHighpass(values, period)` -- the one-pole
  highpass series.
- `computeLineDecycler(values, period)` -- the decycler series
  (`price - highpass`).
- `runLineDecycler(points, options?)` -- sort finite points by
  `x`, run the filter, return `{ series, period, alpha,
  highpass, decycler, samples, decyclerFinal, risingCount,
  fallingCount, flatCount, ok }`. Each sample is classified
  `rising` / `falling` / `flat`. `ok` is false for fewer than
  two points.
- `computeLineDecyclerLayout(options)` -- project the price and
  decycler as two overlaid lines in a single panel.
- `describeLineDecyclerChart(data, options?)` -- ARIA
  description.

## Component

`ChartLineDecycler` is a `forwardRef` region (`role="region"`)
with an ARIA-linked description, a `role="img"` SVG, a single
panel with axis ticks and a grid, a config badge showing the
period, a two-series legend (Price / Decycler) with toggle, one
marker per decycler bar, a hover/focus tooltip exposing the
decycler value and the trend, and a `motion-safe:animate-fade-in`
entrance. It consumes `{ x, value }` points. The price and the
decycler are drawn as two overlaid lines sharing one y-domain.
Every node carries a `data-section` attribute.

## Distinctness

- **vs the moving-average overlays (vwma / kama / hma)** -- those
  smooth the price with a lagging average. The decycler is the
  price minus a highpass, so it tracks the price closely without
  the lag.
- **vs chart-line-roofing / chart-line-supersmoother** -- those
  keep a filtered cycle band. The decycler does the opposite: it
  removes the cycle and keeps the trend.
- **vs chart-line-detrend / chart-line-dpo** -- those plot the
  cycle (the price minus the trend). The decycler plots the
  trend (the price minus the cycle).

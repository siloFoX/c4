# 11.290 -- NativeSelect primitive (v1.11.308)

New `NativeSelect` primitive co-located with the existing
`Select` combobox in `web/src/components/ui/select.tsx`. No
daemon-side change and no `c4` CLI surface change. The existing
`Select` is preserved byte-for-byte.

## Why

The existing `Select` is a rich custom combobox (portal-style
listbox, type-ahead, focus trap, ARIA combobox + listbox markup,
keyboard cycle). It is the right choice when the option list is
medium-large and the operator wants type-ahead. It is overkill
for the simple "pick one of 3-7 fixed options" surface where:

- The native platform dropdown is already accessible.
- Mobile users prefer the platform-native picker (iOS wheel,
  Android dropdown).
- The custom-combobox bundle weight + ARIA surface is more
  cost than the surface needs.

Patch 11.290 ships a thin styled wrapper around `<select>` for
those simple surfaces, sharing the same module + import path as
the existing `Select` so the export site stays single.

## What changed

### `web/src/components/ui/select.tsx`

- New `NativeSelect` forwardRef component + new exported
  types `NativeSelectSize = 'sm' | 'md'` and
  `NativeSelectOption = { value, label, disabled? }`.
- Props:
  - `options`, `value`, `onChange(next)`.
  - `size?: 'sm' | 'md'` (default `'md'`).
  - `error?: boolean` -- flips `aria-invalid` AND swaps the
    border palette.
  - `icon?: ReactNode` -- optional leading icon rendered in
    an absolutely-positioned span so the native select
    stays pixel-perfect.
  - `placeholder?: string` -- renders as a hidden + disabled
    first `<option>` (native selects do not show
    placeholder text natively).
  - All standard `<select>` HTML attributes pass through.
- Visual chrome: relative wrapper span owns the leading icon
  (when set) + the chevron; the `<select>` itself is the
  focusable element. The chevron uses `lucide-react`'s
  `ChevronDown` to match Combobox / Tabs / Drawer.
- Data-attribute selectors for e2e:
  - `data-section="native-select-root"` on the wrapper.
  - `data-section="native-select"` on the select.
  - `data-section="native-select-icon"` on the leading icon
    span (only when `icon` is set).
  - `data-section="native-select-chevron"` on the chevron.
  - `data-size="sm|md"` + `data-error="true|false"` on both
    the root and the select.

### Adopted

## Notes

- Co-located with `Select` in the same file because the two
  primitives share the same conceptual surface ("pick one
  value from a list"). Callers import either via the same
  module path.
- The placeholder is intentionally a disabled + hidden first
  `<option value="">` rather than a real placeholder
  attribute (which `<select>` does not support natively).
  Callers passing `placeholder="..."` get a sensible empty
  initial state without a separate label slot.
- The leading-icon offset (`pl-7` / `pl-9`) is applied to
  the select itself so the platform-native arrow position
  on Windows / macOS stays aligned with the chevron we
  render on top.

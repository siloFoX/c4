# 11.1035 -- UI: chart-line-ema-divergence-cross primitive (v1.11.1053)

## Scope

Add a pure-SVG dual-panel `ChartLineEmaDivergenceCross`
component that renders the close in the top panel and the
Exponential Moving Average (EMA) in the bottom panel,
flagging price-vs-EMA direction-disagreement events as
trend reversal warnings with bias coloring derived from the
EMA slope at the trigger bar. Uses the canonical 5-state
divergence regime model (aligned-bullish / aligned-bearish
/ divergent-bullish / divergent-bearish / none) shared by
the rest of the chart-line-*-divergence-cross family.

This is the SMA-seeded-EMA sibling of
`chart-line-hma-divergence-cross` (v1.11.1052, HMA-based)
-- same 5-state regime classifier and transition-into-
divergent detection, but with a plain SMA-seeded EMA
centerline instead of HMA's composite construction.

## EMA divergence pipeline

```
ema[i]      = SMA-seeded EMA(close, period)
              (seed at index period - 1 with SMA(close,
               period), then ema[i] = prev*(1-alpha) +
               value*alpha; alpha = 2/(period+1))

priceUp     : close[i] > close[i-1]
priceDown   : close[i] < close[i-1]
emaUp       : ema[i]   > ema[i-1]
emaDown     : ema[i]   < ema[i-1]

regime ->
  'aligned-bullish'   when priceUp   && emaUp
  'aligned-bearish'   when priceDown && emaDown
  'divergent-bullish' when priceDown && emaUp
  'divergent-bearish' when priceUp   && emaDown
  'none'              when either side flat or null

bullish (divergence trigger up) :
  prev regime !== 'divergent-bullish' &&
  cur regime === 'divergent-bullish'
bearish (divergence trigger down) :
  prev regime !== 'divergent-bearish' &&
  cur regime === 'divergent-bearish'

bias        : ema[i] vs ema[i-1] -> up / down / flat / none
```

Warmup is `period - 1 = 13` for the default tuning -- much
shorter than the HMA sibling's `period + sqrtPeriod - 2 =
16` since EMA has no outer-window smoothing. Divergence
detection requires the previous EMA + close, so the first
regime classification lands at i = 14 (one bar past
warmup).

## Bit-exact anchors

- **CONST close = K**: SMA seed = K -> EMA stays at K (the
  recurrence `K * (1 - alpha) + K * alpha = K`). Both close
  and EMA are flat -> regime `none` for every bar. 0
  divergence crosses. Verified across K in {0, 1, 50, 200,
  1234}.
- **LINEAR UP close = i**: SMA seed at i = 13 is `(0 + 1 +
  ... + 13) / 14 = 91 / 14 = 6.5 = 13 - 6.5`. EMA[14] =
  `6.5 * (13 / 15) + 14 * (2 / 15) = (84.5 + 28) / 15 = 7.5
  = 14 - 6.5`. By induction, EMA = `i - 6.5` for every i
  >= 13. EMA[i] - EMA[i-1] = +1 (up); close[i] -
  close[i-1] = +1 (up). Both up -> regime
  `aligned-bullish`. 0 divergence crosses.
- **LINEAR DOWN close = -i**: mirror -> EMA = `-i + 6.5`.
  Both close and EMA falling -> regime `aligned-bearish`.
  0 divergence crosses.

The clean `i - 6.5` lag on LINEAR input is the same SMA
centroid identity that the chart-line-keltner-mid-cross-sig
sibling uses -- a direct consequence of the SMA-seeded EMA
construction.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 5-state regime classifier follows the
  chart-line-*-divergence-cross family convention; the
  `bullish` / `bearish` cross *kinds* still follow the
  sibling cross-sig convention (line direction).
- EMA helper uses SMA seeding (not first-value seeding)
  so the steady-state lag on linear input is exactly
  `(period - 1) / 2` from the seed bar onwards. This
  matches the SMA centroid identity bit-exact and lets the
  LINEAR anchors verify EMA without an asymptotic
  convergence tolerance.
- Cross detection fires only on transition INTO a
  divergent state (not while staying in one), preventing
  double-fire on consecutive bars in the same divergent
  regime.

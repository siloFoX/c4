# 11.1071 -- UI: chart-line-roc-divergence-cross primitive (v1.11.1089)

## Scope

Add a pure-SVG dual-panel `ChartLineRocDivergenceCross`
component that renders the close in the top panel and the
Rate Of Change (ROC) oscillator in the bottom panel,
marking bullish (price down while ROC up -- bullish
momentum-percentage reversal warning) / bearish (price up
while ROC down -- bearish momentum-percentage reversal
warning) price-vs-ROC direction disagreement (divergence)
trigger events with bias coloring derived from the ROC
slope at the divergence-entry bar.

ROC is the percentage version of the Momentum oscillator:
rather than the absolute price change, ROC measures the
percentage change over the lookback period. This
normalisation makes ROC comparable across instruments at
different price levels and surfaces *relative* (not
absolute) momentum reversal warnings -- the canonical
divergence signal for percentage-momentum traders.

## ROC divergence-cross pipeline

```
ROC[i] = (close[i] - close[i - period]) /
         close[i - period] * 100
         (null when close[i - period] === 0)

priceUp     = close[i] > close[i-1]
priceDown   = close[i] < close[i-1]
rocUp       = ROC[i]   > ROC[i-1]
rocDown     = ROC[i]   < ROC[i-1]

regime :
  'aligned-bullish'   when priceUp   && rocUp
  'aligned-bearish'   when priceDown && rocDown
  'divergent-bullish' when priceDown && rocUp
                      (bullish momentum percentage
                       reversal warning)
  'divergent-bearish' when priceUp   && rocDown
                      (bearish momentum percentage
                       reversal warning)
  'none'              otherwise

bullish cross :
  prev regime !== 'divergent-bullish' &&
  cur regime === 'divergent-bullish'
bearish cross :
  prev regime !== 'divergent-bearish' &&
  cur regime === 'divergent-bearish'

bias : ROC[i] vs ROC[i-1] -> up/down/flat/none
```

Default `period = 12` -- canonical ROC tuning (common
across most TA references). Distinct from the sibling
momentum-divergence-cross v1.11.1088 which defaults to
period=10 to mirror its raw Momentum sibling's tuning.

- **chart-line-roc-divergence-cross v1.11.1089** (this
  primitive) -- price vs ROC percentage oscillator

Distinct from sibling momentum-divergence-cross
v1.11.1088: same divergence detector, different
underlying oscillator. ROC normalises by the base price
so it is more reactive on instruments at different price
levels and more sensitive to *relative* (not absolute)
momentum changes. The bit-exact anchors shift accordingly:
LINEAR UP gives divergent-bearish under ROC
(decelerating percentage growth) where under Momentum it
gives `none` (constant momentum, flat).

Warmup is `period = 12` for the default tuning. Cross
detection needs the previous bar's regime, so the first
potential cross lands at i = period + 1 = 13.

## Bit-exact anchors

All anchors use single-close input.

- **CONST** `close = K` (K != 0): ROC[i] = (K - K)/K *
  100 = 0 (constant from i = period). Price delta = 0
  (flat), ROC delta = 0 (flat). regime `none`
  throughout. 0 crosses. Verified across K in {1, 50,
  200, 1234}.
- **CONST K = 0**: close[i-period] = 0, division by
  zero -> ROC = null throughout. regime `none`. 0
  crosses. Verified safely (no NaN, no throw).
- **LINEAR UP** `close = i + 1` (positive
  monotonically increasing): ROC[i] = period * 100 /
  (i - period + 1) -- DECAYS as i grows (same dollar
  delta but larger base). priceUp = true (constant
  +1). rocDown = true (ROC declining). regime
  `divergent-bearish` throughout post-warmup. At
  least one bearish cross fires when regime first
  enters divergent-bearish.
- **LINEAR DOWN** `close = n - i` (positive
  monotonically decreasing): ROC[i] = -period * 100 /
  (n - i + period) -- becomes more negative as i grows.
  priceDown = true. rocDown = true. regime
  `aligned-bearish`. 0 crosses (no entry into
  divergent state).
- **DECELERATING DECLINE** `close = 100 - sqrt(i+1)`
  (positive monotonically decreasing at decreasing
  rate): price still falls each bar (priceDown) but
  ROC magnitude shrinks -- rocUp (ROC rising toward
  zero from below). regime `divergent-bullish`. At
  least one bullish cross fires.

The percentage normalisation of ROC produces
QUALITATIVELY DIFFERENT bit-exact behaviour from the
Momentum sibling. Under Momentum, LINEAR UP gives
constant M (flat momentum, regime none); under ROC, the
same input gives DECAYING percentage (divergent-bearish,
fires crosses). This is documented as the family's
canonical reactivity distinction: ROC is "always
divergent against linear price growth" while Momentum
treats linear price growth as flat-momentum.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 5-state regime classifier follows the divergence-
  cross family convention exactly.
- Divide-by-zero guard: when close[i-period] === 0,
  ROC[i] returns null (not NaN or Infinity). Verified
  by the CONST K=0 anchor which runs without throwing.
- Strict inequalities (priceUp = `>`, etc.) -- exact
  ties produce regime `none` rather than ambiguous
  state. Matches the family convention.
- The percentage normalisation of ROC means that even
  LINEAR price input produces a non-constant
  oscillator. This is documented behavior: ROC's
  reactivity to *relative* growth is a feature
  (instruments at different prices are directly
  comparable) and a cost (more crosses fire on linear-
  trend input than under Momentum). The LINEAR UP
  anchor verifies this: it fires bearish crosses,
  unlike the Momentum sibling which fires zero
  crosses on the same input.
- Distinct from chart-line-roc-mid-cross-sig
  v1.11.1081 (ROC vs zero level cross -- absolute
  momentum sign change) and chart-line-roc (the raw
  ROC oscillator). This primitive specifically
  measures price-vs-ROC DIRECTION disagreement.
- The zero-visibility guarantee in layout ensures the
  ROC sign is always readable even on a series that
  never crosses zero.
- Pairs naturally with chart-line-momentum-divergence-
  cross v1.11.1088 for a percentage-vs-absolute
  divergence comparison.

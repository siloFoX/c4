# 11.544 -- web/src/components/ui/chart-line-fft-window.tsx

Pure-SVG **windowed-FFT** chart: applies a tapering window
(Rectangular / Hann / Hamming / Blackman, toggleable at
runtime) to the time series BEFORE running the DFT. Pairs a
time-domain panel (left) with a frequency-spectrum panel
(right), and overlays a dashed window envelope on the time
panel so operators can SEE the taper being applied. Distinct
from `<ChartLineFft>` (11.540) which used no window
(implicit rectangular with full spectral leakage).

## Why apply a window?

A finite-length DFT implicitly treats the signal as
periodic. When the signal doesn't end exactly at the same
phase it started, the discontinuity at the wraparound
splatters energy across all frequency bins -- **spectral
leakage**. A tapering window forces the signal to zero at
both ends so the wraparound is seamless, trading slightly
broader main lobes for dramatically lower side lobes.

## Window formulas (verified by test)

For a window of length N indexed n = 0..N-1:

- **Rectangular**: `w[n] = 1`
  -- coherent gain 1.0, processing gain 1.0
- **Hann**: `w[n] = 0.5 * (1 - cos(2*pi*n/(N-1)))`
  -- endpoints 0, center 1, coherent gain ~0.5
- **Hamming**: `w[n] = 0.54 - 0.46 * cos(2*pi*n/(N-1))`
  -- endpoints 0.08, coherent gain ~0.54
- **Blackman**: `w[n] = 0.42 - 0.5*cos(2*pi*n/(N-1)) +
  0.08*cos(4*pi*n/(N-1))`
  -- endpoints 0, center 1, coherent gain ~0.42

Reported per spectrum:
- **Coherent gain** = `(sum w[n]) / N` -- the amplitude
  scale factor (multiply un-windowed amplitude by 1 /
  coherent gain to recover the original tone amplitude).
- **Processing gain** = `(sum w[n]^2) / N` -- the noise
  power scale factor.

## Distinct from related primitives

- `<ChartLineFft>` (11.540): no window -- this is the
  windowed equivalent with the four canonical window
  options, a window-mode toggle, and per-bin gain
  reporting.
- `<ChartLineDecompose>` (11.543): time-domain seasonal
  decomposition. FFT-window is frequency-domain.
- `<ChartLineMomentum>` (11.533): sub-panel oscillator with
  DERIVATIVE -- not a spectrum.
- All other time-domain smoothers (MovingAvg, Ewma, Kalman,
  Bollinger, Control) are time-only.

## Pure helpers

- `LINE_FFT_WINDOW_MODES` -- `['rectangular', 'hann',
  'hamming', 'blackman']`.
- `getLineFftWindowDefaultColor` / `getLineFftWindowFinitePoints`.
- `normaliseLineFftWindowPanelRatio` -- clamps `[0.1, 0.9]`.
- `normaliseLineFftWindowMode` -- identity for valid;
  default `'hann'`.
- `computeLineFftWindowCoefficients(N, mode)` -- canonical
  window-coefficient generator. Returns length-N array of
  weights. N <= 0 / non-finite -> []. N = 1 returns `[1]`.
- `applyLineFftWindowToValues(values, coefficients)` --
  element-wise multiply with the input. Non-finite slots
  emit 0. Uses the shorter of the two arrays.
- `computeLineFftWindowDft(values)` -- canonical real-input
  half-spectrum DFT. Returns `{real, imag}[]` of length
  `floor(N/2) + 1`. Non-finite values skipped from the sum.
- `computeLineFftWindowSpectrum(points, {windowMode?,
  detrend?, excludeDc?})` -- canonical pipeline: sort
  ascending, drop non-finite, detrend (default true),
  compute window coefficients, apply window, run DFT,
  compute magnitudes / normalised magnitudes / dominant
  bin (skipping DC by default), and report coherent gain
  + processing gain.
- `findLineFftWindowDominantBin(bins, excludeDc?)` -- pick
  the max-magnitude bin (skipping DC by default).
- `computeLineFftWindowLayout` -- splits canvas into time
  + spectrum panels, projects raw + windowed + window
  envelope paths in the time panel, and bin bars in the
  spectrum panel.
- `describeLineFftWindowChart` -- screen-reader summary
  including window mode + coherent gain + dominant
  frequency / period.

## Tooltip

Hovering a time dot shows the series label in its colour,
x, raw value, bold windowed value, and the window
coefficient at that index. Hovering a spectrum bin shows
label + bin number, frequency, period, bold magnitude,
the window mode + coherent gain, and (when dominant) a
coloured "dominant" row.

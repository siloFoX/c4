## Summary

two-panel chart with a Negative Volume Index panel tracking
cumulative change on lower-volume bars.

## Indicator math

Paul Dysart's Negative Volume Index is a cumulative index built
on a single rule: it moves only on the quiet bars. The premise
is that informed money trades on low-volume days, so the NVI
isolates what the price does on those days.

`computeLineNviIndex(closes, volumes, base)` seeds the index at
`base` and, for each later bar:

```
NVI[i] = NVI[i-1] * close[i] / close[i-1]   if volume[i] < volume[i-1]
NVI[i] = NVI[i-1]                           otherwise
```

So on a lower-volume bar the index compounds the price's percent
change; on a higher-volume bar (or an unchanged-volume bar) it
holds flat. An NVI above its starting base means the price has
gained ground on the quiet bars.

### Exactness

The index is a chain of ratio multiplications, so it is exact
when the closes are powers of two -- every ratio
`close[i]/close[i-1]` is then dyadic, and the index, seeded at
1000, stays a dyadic multiple of 1000. The fixture closes
`[128, 256, 256, 128, 128, 256, 64, 128, 64, 256]` paired with
the volumes `[50, 30, 60, 20, 70, 40, 30, 80, 25, 50]` give the
exact index `[1000, 2000, 2000, 1000, 1000, 2000, 500, 500, 250,
250]`, asserted with `toEqual`.

## API

- `getLineNviFinitePoints(points)` -- drop points with a
  non-finite `x`, `value` or `volume`, or a negative `volume`.
- `normalizeLineNviBase(base)` -- coerce the base to a positive
  number, falling back to 1000.
- `computeLineNviIndex(closes, volumes, base)` -- the cumulative
  index.
- `runLineNvi(points, options?)` -- sort finite points by `x`,
  run the pipeline, return `{ series, base, nvi, samples,
  nviFinal, upCount, downCount, flatCount, ok }`. Each sample is
  classified `up` / `down` / `flat` by the index movement. `ok`
  is false for fewer than two points.
- `computeLineNviLayout(options)` -- stack the price panel above
  an NVI panel; project the price path, the NVI path, one marker
  per bar, and the base reference line.
- `describeLineNviChart(data, options?)` -- ARIA description.

## Component

`ChartLineNvi` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, two stacked panels
(price on top, NVI below) with their own axis ticks and grid,
panel labels, a config badge showing the base, a two-series
legend (Price / NVI) with toggle, a hover/focus tooltip exposing
the volume, the NVI and the movement state, and a
`motion-safe:animate-fade-in` entrance. It consumes
`{ x, value, volume }` points. The NVI panel draws the index line
over a dashed base reference line, with each marker coloured by
its movement -- green up, red down, indigo flat. Every node
carries a `data-section` attribute.

## Distinctness

- **vs chart-line-pvi** -- the Positive Volume Index is the
  mirror: it moves on the loud (rising-volume) bars. The NVI
  moves on the quiet (falling-volume) bars.
- **vs chart-line-obv / chart-line-vzo** -- those accumulate
  volume itself. The NVI ignores volume size entirely and only
  uses it as a gate to decide which bars compound the price
  change.
- **vs chart-line-cumulative** -- a cumulative return runs on
  every bar. The NVI runs on a selected subset of bars and
  freezes on the rest.

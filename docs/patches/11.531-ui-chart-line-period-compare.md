# 11.531 -- web/src/components/ui/chart-line-period-compare.tsx

Pure-SVG **period-over-period comparison** line chart that overlays
two series representing the **same metric across different time
periods** (current vs prior) on a shared relative x-axis. The
current series renders solid; the prior series renders dashed and
slightly faded so the two read as primary signal + reference. A
coloured change badge in the top-left corner surfaces the total
period-over-period percent change with direction (up / down / flat).

## Distinct from other two-series primitives

- `<ChartLineComparison>` (11.511): two **different** curves with
  difference shading between them. Period-compare is the SAME
  metric across two periods.
- `<ChartLineCorrelation>` (11.529): dual-axis for two **unrelated**
  metrics. Period-compare uses a SINGLE shared y axis.
- `<ChartLineForecast>` (11.519): historical/forecast split on the
  time axis. Period-compare overlays two periods.
- `<ChartLineTarget>` (11.518): vs a flat target. Period-compare's
  reference is another curve.
- `<ChartLineMulti>` (11.500): multi-series + crosshair (no
  semantic "current vs prior" distinction).
- `<ChartLineResidual>` (11.525): observed minus predicted (model
  diagnostic). Period-compare is a business comparison.

The canonical "this week vs last week" / "Q4 vs Q3" / "YoY change"
visualisation used in monitoring dashboards, sales reports, and any
case where the question is "are we doing better or worse than last
time".

## Public surface

- `<ChartLinePeriodCompare current={...} prior={...} />`.
- `current` / `prior`: `{id, label, data, color?}` -- each can be
  `null` for a single-period view.
- `flatEpsilon` (default `0`) widens the "flat" equality band for
  the direction classification.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showChangeBadge`, `animate`.
- `priorDashArray` (`'4 3'`), `priorOpacity` (`0.6`),
  `upColor` / `downColor` / `flatColor` for the badge + per-point
  tooltip tinting.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX` / `formatPercent`.
- `onPointClick({series, point})`.

## Pure helpers

- `getLinePeriodCompareDefaultColor(index)` -- 10-color palette
  cycle.
- `getLinePeriodCompareFinitePoints(points)` -- finite-only filter.
- `pairLinePeriodCompareByX(current, prior)` -- pairs the two
  series by **exact x match**. Returns one entry per shared x value
  where both periods have a finite sample, sorted ascending by x.
  `currentIndex` / `priorIndex` reference ORIGINAL input arrays.
  `delta = current - prior`; `percentChange = delta / |prior|` with
  `prior === 0` collapsing to `0` to avoid division by zero.
- `classifyLinePeriodCompareDirection(percentChange, epsilon?)` --
  `'up' | 'down' | 'flat'`. Non-finite -> `'flat'`. Non-finite /
  negative epsilon falls back to default.
- `computeLinePeriodCompareTotals(current, prior, flatEpsilon?)` --
  returns `{currentTotal, priorTotal, totalDelta,
  totalPercentChange, ok, direction, pairCount, currentCount,
  priorCount}`. `totalPercentChange = totalDelta / |priorTotal|`
  with priorTotal=0 collapsing to 0. `ok=false` when both series
  empty.
- `computeLinePeriodCompareLayout({current, prior, ...})` --
  returns `{current, prior, totals, xTicks, yTicks, xMin, xMax,
  yMin, yMax, innerWidth, innerHeight, totalPoints}`. Each layout
  series carries `{id, label, role ('current' | 'prior'), color,
  points, path, finiteCount, totalCount}`. Per-point on the
  CURRENT series additionally carries `{delta, percentChange,
  direction}` derived from the prior-period pair; per-point on the
  PRIOR series leaves these `null` / `'flat'`.
- `describeLinePeriodCompareChart(current, prior, formatValue?,
  formatPercent?)` -- builds the screen-reader description.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Per-line path + per-dot are `role="graphics-symbol"` +
  `tabIndex={0}`. Dot aria-label includes the period role.
- Every node carries `data-section="chart-line-period-compare-..."`
  for test selectors.
- Series groups expose `data-series-role` (`'current' |
  'prior'`), `data-series-color`, `data-series-finite-count`.
- Dots expose `data-series-role`, `data-delta`,
  `data-percent-change`, `data-direction`.
- Root mirrors `data-pair-count`, `data-current-total`,
  `data-prior-total`, `data-total-delta`,
  `data-total-percent-change`, `data-direction`, `data-totals-ok`.
- Badge mirrors the same direction + percent + delta + pair-count.

## Tooltip

Hovering a dot shows the series label in its colour with a
`(role)` suffix, x, bold y, and (for CURRENT dots only) a
coloured `vs prior: +/-<delta> (<percent>)` row tinted by
direction. Prior-period dots omit the delta row since they ARE
the reference.

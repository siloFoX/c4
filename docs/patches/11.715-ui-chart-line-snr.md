## Summary

New `ChartLineSnr` UI primitive in `web/src/components/ui/chart-line-snr.tsx`
-- a pure-SVG two-panel line chart with a **Signal to Noise Ratio**
panel from the decibel ratio of bar range to its rolling average. The
top panel plots the bar midpoint; the bottom panel plots the SNR.

## The computation

The Signal to Noise Ratio reads how loud the current bar's range is
against the recent "noise floor" of the same range averaged across the
lookback. `ChartLineSnrPoint` carries `{x, high, low}`.

- `computeLineSnrRange(bars)` -- the bar range `high - low` per bar;
  null for a non-finite bar.
- `computeLineSnrSma(values, period)` -- the simple moving average of
  the range over the lookback, null in the warm-up.
- `computeLineSnrDb(value, reference)` -- the decibel formula
  `10 * log10(value / reference)`, returning null when either input is
  non-positive or non-finite.
- `computeLineSnr(bars, options)` -- chains the three: the bar range,
  its rolling average, and the SNR as the dB ratio of the two.
- `classifyLineSnrZone(snr)` -- marks a bar **up** when the SNR is
  positive (loud), **down** when negative (quiet), **flat** at zero
  (the noise floor), **none** for null.

## Worked fixture

The whole pipeline is pure arithmetic except for the `log10`. The dB
formula has an exact anchor at unity: `Math.log10(1)` is exactly 0, so
`computeLineSnrDb(1, 1)` reads exactly 0 (asserted with `toBe`).

A **constant-range fixture** (high 60, low 40, period 3) has every range
equal to 20 and every rolling average equal to 20, so `range / avgRange`
is exactly 1 and the SNR is exactly 0 across every defined bar
(asserted with `toEqual`). A **rising-range** fixture (high climbs from
10 to 17, low fixed at 8) gives a current range larger than the trailing
average, so the SNR is strictly positive at every defined bar; a
**falling-range** fixture mirrors it negative. The dB helper is checked
against textbook values (`10:1 -> 10 dB`, `100:1 -> 20 dB`, `1:10 -> -10
dB`, all `toBeCloseTo`) and against the null-on-zero rule. A
zero-range series (where the average range is zero) is checked to
return null instead of dividing by zero.

## API

`data` (a `ChartLineSnrPoint[]`), `period` (default 14),
`pricePanelRatio` / `gap`, sizing/format/a11y props, `strokeWidth` /
`dotRadius`, the colour props, the toggle flags (`showAxis` / `showGrid`
/ `showDots` default false / `showSnr` / `showZeroLine` / `showMarkers`
/ `showTooltip` / `showConfigBadge` / `showLegend` / `animate`),
controlled + uncontrolled `hiddenSeries` (keys `price` / `snr`),
`formatValue` / `formatX`, `onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineSnrFinitePoints`, `normalizeLineSnrPeriod`
(coerces the period to an integer of at least 1), `computeLineSnrDb`,
`computeLineSnrRange`, `computeLineSnrSma`, `classifyLineSnrZone`,
`computeLineSnr`, `runLineSnr`, `computeLineSnrLayout`,
`describeLineSnrChart`, plus the `DEFAULT_CHART_LINE_SNR_*` constants
and the `ChartLineSnrZone` / `ChartLineSnrComputed` types. Every
exported name carries a `LineSnr` infix so the `export *` barrel line
cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen `data-section="chart-line-snr-aria-desc"`
description span; the midpoint line, the SNR line and every dot /
marker are `role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-period`, `data-snr-final`,
`data-up-count`, `data-down-count`, `data-flat-count`,
`data-total-points` and `data-animate`.

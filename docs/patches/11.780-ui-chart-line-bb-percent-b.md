# 11.780 chart-line-bb-percent-b (v1.11.798)

## Summary

Pure-SVG dual-panel React/TypeScript chart primitive with John
Bollinger's **%B** oscillator beneath the close. %B locates the
close within the Bollinger Band envelope. Added as the
`<ChartLineBbPercentB />` UI primitive in
`web/src/components/ui/`.

## Algorithm

For each bar with `length` (default 20) and `numStd` (default 2):

```
middle[i] = SMA(close, length)[i]
var[i]    = sum (close[j] - middle[i])^2 / length
std[i]    = sqrt(var[i])
upper[i]  = middle[i] + numStd * std[i]
lower[i]  = middle[i] - numStd * std[i]
%B[i]     = (close[i] - lower[i]) / (upper[i] - lower[i])
```

Bars before `i = length - 1` are `null` (warmup). When `std == 0`
(singular: completely flat window) `%B` is `null`. The
implementation uses population standard deviation (divide by
length) as originally defined by Bollinger.

## Anchors

- **Worked anchor `[10, 12]`, `length = 2, numStd = 1`**:
  middle = 11, variance = 1, std = 1, upper = 12, lower = 10.
  close = 12 = upper -> `%B = (12 - 10) / (12 - 10) = 1`
  **bit-exact**.
- **Worked anchor `[12, 10]`, `length = 2, numStd = 1`**:
  middle = 11, std = 1, upper = 12, lower = 10. close = 10 =
  lower -> `%B = 0 / 2 = 0` **bit-exact**.
- **Alternating anchor `[10, 12, 10, 12, ...]`**: `%B` cycles
  through `1, 0, 1, 0, ...` bit-exact.
- **CONST_FLAT (close == K)**: std = 0 -> upper = lower ->
  `%B = null` at every bar. Verified across `K in {0, 1, 5, 10,
  100, -3}`.
- **Translation invariance**: shifting close by `C` does not
  change `%B` (within numerical tolerance).

# 11.533 -- web/src/components/ui/chart-line-momentum.tsx

Pure-SVG line chart with a **momentum oscillator** in a dedicated
sub-panel below the main price line. Momentum at index `i` is
defined as `value[i] - value[i - period]` (the classic technical
analysis momentum indicator) -- the first `<period>` indices are
null because the look-back window is not yet filled. The
sub-panel y axis is centered on zero when both signs are present,
with a dashed zero reference line spanning the chart, and the
momentum line under the zero line is shaded red while the part
above is shaded green.

## Distinct from other derivative / smoothing primitives

- `<ChartLineRate>` (11.521): per-segment first derivative
  (`(y[i+1] - y[i]) / (x[i+1] - x[i])`) -- depends on the x step.
  Momentum is a fixed-lag difference in y space only.
- `<ChartLineAcceleration>` (11.528): second derivative. Momentum
  is a single-stage diff.
- `<ChartLineVelocity>` (11.527): per-point directional arrow
  markers (tangent / sign). Momentum is a continuous line in a
  separate panel.
- `<ChartLineMovingAvg>` (11.513): trailing/centered/edge SMA --
  smooths the same series. Momentum produces a separate derived
  signal.
- `<ChartLineTrend>` (11.512): one global linear regression rule
  per series. Momentum is per-point.
- `<ChartLineCumulative>` (11.520): running cumulative sum, also a
  derived signal but ACCUMULATES rather than DIFFERENCES.

The canonical "momentum oscillator" view used in
finance/trading dashboards to spot bullish (positive momentum) vs
bearish (negative momentum) phases and divergences from the
underlying price line.

## Public surface

- `<ChartLineMomentum series={...} />`.
- `series: ChartLineMomentumSeries[]` -- `{id, label, data,
  color?, period?}`. Per-series `period` always beats the
  chart-level `period`.
- `period` (default `10`) -- lag for the momentum calculation.
- `subHeightRatio` (default `0.32`) -- fraction of the inner
  drawable area allocated to the momentum sub-panel; clamped to
  `[0.1, 0.9]`.
- `positiveColor` / `negativeColor` / `zeroColor` -- per-sign
  fill + tooltip tinting.
- `fillOpacity` (default `0.2`) -- momentum-vs-zero shading
  opacity.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showSignBadge`, `showMomentumFill`, `animate`.
- Controlled + uncontrolled visibility via `hiddenSeries` /
  `defaultHiddenSeries` / `onHiddenSeriesChange`.
- `formatValue` / `formatMomentum` / `formatX`.
- `momentumLabel` (default `'Momentum'`).
- `onPointClick({series, point})`.

## Pure helpers

- `getLineMomentumDefaultColor(index)` -- 10-color palette cycle;
  NaN / negative -> color 0.
- `getLineMomentumFinitePoints(points)` -- finite-only filter.
- `normaliseLineMomentumPeriod(value)` -- clamps to >= 1; floors
  fractional; non-finite -> default 10.
- `normaliseLineMomentumSubHeightRatio(value)` -- clamps to
  `[0.1, 0.9]`; non-finite -> default 0.32.
- `classifyLineMomentumSign(value)` -- `'positive' | 'negative' |
  'zero'`; null / non-finite -> zero; exact 0 -> zero.
- `computeLineMomentumValues(points, period?)` -- canonical
  momentum helper; sorts ascending, drops non-finite, returns a
  `(number | null)[]` of the same length as sorted-finite input
  where the first `<period>` entries are null and subsequent
  entries are `y[i] - y[i - period]`. Period >= length yields all
  null. Empty/null input -> [].
- `buildLineMomentumSamples(points, period?)` -- combines the
  above two helpers into per-point `{index, x, y, momentum,
  sign}` records.
- `computeLineMomentumLayout({series, ...})` -- returns
  `{series, xTicks, yTicks, momentumTicks, xMin, xMax, yMin,
  yMax, momentumMin, momentumMax, innerWidth, mainHeight,
  subHeight, subTop, totalPoints, visibleSeriesCount}`. The
  momentum range is **centered on zero** when both signs are
  present so the zero reference line lands in the middle of the
  sub-panel. Per-series carries `{id, label, color, period,
  points, path, momentumPath, positivePath, negativePath,
  finiteCount, totalCount, momentumValidCount, positiveCount,
  negativeCount, zeroCount, latestMomentum, latestSign}`.
  Per-point carries `{index, x, y, px, py, momentum, momentumPy,
  sign}` -- momentumPy is null whenever momentum is null.
- `describeLineMomentumChart(series, hidden?, period?,
  formatMomentum?)` -- builds the screen-reader description
  including per-series period and latest momentum (n/a when
  unfilled).

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Main path + momentum path + every dot is
  `role="graphics-symbol"` + `tabIndex={0}`. Momentum aria-label
  includes the period; dot aria-label includes the momentum value
  + sign when available, n/a when unfilled.
- Every node carries `data-section="chart-line-momentum-..."` for
  test selectors.
- Series groups expose `data-series-id`, `data-series-color`,
  `data-series-period`, `data-series-finite-count`,
  `data-series-momentum-valid-count`, `data-series-positive-count`,
  `data-series-negative-count`, `data-series-zero-count`,
  `data-series-latest-sign`.
- Main path has `data-kind="main"`; momentum path
  `data-kind="momentum"`; momentum fills `data-sign` (`positive` |
  `negative`).
- Dots expose `data-momentum`, `data-sign`.
- Root mirrors `data-series-count`, `data-visible-series-count`,
  `data-total-points`, `data-period`, `data-dominant-sign`,
  `data-animate`.
- Badge mirrors the same sign + momentum + series-id.

## Tooltip

Hovering a dot shows the series label in its colour, x, bold y,
and a coloured `momentum(<period>): <value> (<sign>)` row tinted
by sign. When momentum is unfilled (early indices below the
period) the row shows `n/a`.

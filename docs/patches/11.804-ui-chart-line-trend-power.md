# 11.804 UI: chart-line-trend-power primitive (v1.11.822)

## Summary
Pure-SVG dual-panel React/TS primitive that plots **rolling OLS regression
trend strength** beneath the close. Trend Power is the squared regression
slope scaled by the inverse of the residual variance over the lookback,
producing a signal-to-noise reading that flags strong-trend, weak-trend,
or flat regimes.

## Formula
```
meanX        = (length - 1) / 2
meanY[i]     = mean(close[i - length + 1 .. i])
slope[i]     = sum((k - meanX) * (close[k] - meanY[i]))
              / sum((k - meanX)^2)
intercept[i] = meanY[i] - slope[i] * meanX
resVar[i]    = sum((close[k] - (slope[i]*k + intercept[i]))^2) / length
power[i]     = slope[i]^2 / max(resVar[i], MIN_RESIDUAL_VARIANCE)
```
`power[i]` is `null` during warmup (`i < length - 1`) and whenever any
close in the window is non-finite.

`MIN_RESIDUAL_VARIANCE = 2^-50` is the dyadic floor on the denominator -
a perfectly linear fit otherwise yields `0/0` Infinity; clamping at a
power-of-two keeps the IEEE 754 reciprocal exact.

## Defaults
- `length = 14`
- `strongThreshold = 1` (power >= 1 -> strong-trend zone)
- `width = 720`
- `height = 460`
- `panelGap = 12`

## Zones
- `strong-trend`: `power >= strongThreshold`
- `weak-trend`: `0 < power < strongThreshold`
- `flat`: `power == 0`
- `none`: warmup or non-finite

Markers fire only on `strong-trend` bars.

## Bit-exact testability anchors
- **CONST close** (`close == K`): `slope = 0`, `residualVariance = 0`,
  numerator = 0 -> `power = 0 / MIN_RESIDUAL_VARIANCE = 0` (bit-exact).
  Verified across `K in {0, 1, 5, 100, -3}` and `length in {3, 4, 7, 10}`.
- **LINEAR close = a*k** (length=4, a=1): OLS recovers `slope = 1`,
  `intercept = 0`, residuals = 0 -> `residualVariance = 0` (exact),
  denominator clamped to `2^-50` -> `power = 1 / 2^-50 = 2^50`
  (bit-exact). Verified `power = Math.pow(2, 50)`.
- **LINEAR close = 2*k** (length=4): same path -> `power = 4 * 2^50 =
  2^52` (bit-exact).
- **LINEAR close = k** (length=8): regression still exact -> `power =
  2^50` (bit-exact).

## Design notes
- Power axis is **0-based with auto max**: `powerMax = max(observedPower,
  strongThreshold * 1.25)`. This keeps the threshold line visible even
  when the data is all-flat.
- Threshold line dashed `#475569`, drawn separately from gridlines so it
  reads as a reference level.
- Markers fire only on `strong-trend` bars, not every defined bar - the
  power panel can otherwise be dense for noisy data.
- Default colors: power line / strong marker `#ea580c` (orange), weak
  `#f59e0b` (amber), flat `#94a3b8`, threshold `#475569`. Distinct
  palette from peak-detector (red) and trough-detector (green).
- `noUncheckedIndexedAccess` satisfied via explicit `?? null` and
  defensive guards on every window access in
  `fitLineTrendPowerWindow`.

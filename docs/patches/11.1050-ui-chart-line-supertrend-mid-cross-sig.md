# 11.1050 -- UI: chart-line-supertrend-mid-cross-sig primitive (v1.11.1068)

## Scope

Add a pure-SVG dual-panel `ChartLineSupertrendMidCrossSig`
component that renders the close in the top panel and the
Supertrend trailing-stop line (the active trend-aware
centerline of price) plus its smoothed SMA signal line in
the bottom panel, marking bullish / bearish supertrend-
over-signal crossover trigger events with bias coloring
derived from the supertrend slope at the trigger bar.

Olivier Seban's Supertrend is a volatility-aware trailing
stop indicator: it brackets price by an ATR-scaled band
around HL2 and follows the lower band in an uptrend (or
upper band in a downtrend) as a dynamic stop. This
primitive treats the active supertrend line as the "mid
level" -- the trend-aware centerline of price -- and
watches it against its own SMA-smoothed signal line to
detect trend stop centerline trigger events.

## Supertrend pipeline

```
tr[i]         = max(high[i] - low[i],
                    |high[i] - close[i-1]|,
                    |low[i]  - close[i-1]|)
atr[i]        = SMA(tr, period)
hl2[i]        = (high[i] + low[i]) / 2
upperBand[i]  = hl2[i] + multiplier * atr[i]
lowerBand[i]  = hl2[i] - multiplier * atr[i]

finalUpper[i] = (upperBand[i] < finalUpper[i-1] ||
                 close[i-1] > finalUpper[i-1])
                ? upperBand[i]
                : finalUpper[i-1]
finalLower[i] = (lowerBand[i] > finalLower[i-1] ||
                 close[i-1] < finalLower[i-1])
                ? lowerBand[i]
                : finalLower[i-1]

Initial trend seeded by sign(close[period] -
close[period-1]) -- positive or zero means uptrend,
negative means downtrend. This keeps canonical LINEAR
anchors flip-free.

trend[i]      = uptrend if close > prev finalUpper
                downtrend if close < prev finalLower
                prev trend otherwise
supertrend[i] = uptrend ? finalLower[i] : finalUpper[i]
signal[i]     = SMA(supertrend, signalLength)

bullish       : prev supertrend <= prev signal &&
                cur supertrend > cur signal
bearish       : prev supertrend >= prev signal &&
                cur supertrend < cur signal
regime        : 'bullish' when supertrend >= signal
                'bearish' when supertrend <  signal
                'none'    when either is null
bias          : supertrend[i] vs supertrend[i-1] -> up /
                down / flat / none
```

Default `period = 10`, `multiplier = 3`, `signalLength
= 3` -- canonical Seban tuning paired with the family-
conventional short SMA signal.

This primitive uses **SMA-based ATR** (matching the
`chart-line-atr-breakout-cross` v1.11.1048 and
`chart-line-atr-divergence-cross` v1.11.1064 siblings)
rather than Wilder-style ATR (used by
`chart-line-supertrend-divergence-cross` v1.11.1062). SMA
gives bit-exact integer ATR values on linear input with
no convergence transient. The two ATR variants in the
codebase are a documented family-level tuning choice.

Warmup is `period + signalLength - 1 = 12` for the
default tuning: ATR seeds at i = period = 10, supertrend
is valid from i = period = 10, then the signal SMA needs
`signalLength - 1 = 2` more bars.

## Bit-exact anchors

All anchors use HLC input.

- **CONST band** `high = K + 1`, `low = K - 1`, `close
  = K`: tr = 2, atr = 2. hl2 = K. upperBand = K + 6,
  lowerBand = K - 6. close[period] - close[period-1] = 0
  -> init uptrend. supertrend = finalLower = K - 6
  (constant). signal = SMA(K - 6, 3) = K - 6.
  supertrend === signal -> regime `bullish` (>=). 0
  crosses. Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close
  = i`: tr = 2, atr = 2. hl2 = i. upperBand = i + 6,
  lowerBand = i - 6. close[period] - close[period-1] =
  +1 -> init uptrend. close = i is always > previous
  finalLower = i - 7 -> stays uptrend. supertrend =
  finalLower = i - 6 (slope +1). signal = SMA
  (supertrend, 3) = ((i - 2 - 6) + (i - 1 - 6) + (i -
  6)) / 3 = i - 7. supertrend - signal = +1 (constant).
  regime `bullish` for every valid bar. 0 crosses (no
  transition into bearish).
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`,
  `close = -i`: tr = 2, atr = 2. hl2 = -i. upperBand
  = -i + 6, lowerBand = -i - 6. close[period] -
  close[period-1] = -1 -> init downtrend. close = -i is
  always < previous finalUpper = -i + 7 -> stays
  downtrend. supertrend = finalUpper = -i + 6.
  signal = -i + 7. supertrend - signal = -1 (constant).
  regime `bearish` for every valid bar. 0 crosses (no
  transition into bullish).

The +/- 1 offset between supertrend and signal on LINEAR
input is the 1-bar SMA lag for a unit-slope line: SMA
(slope * i, 3) = slope * (i - 1). Mirrors the Ichimoku
sibling v1.11.1067 behaviour.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 3-state regime classifier follows the cross-sig family
  convention (bullish on `===`, bearish on `<`); the
  sibling `chart-line-supertrend-divergence-cross`
  (v1.11.1062) uses a 5-state divergence model paired
  with price.
- Initial trend detection via sign(close[period] -
  close[period-1]) keeps LINEAR anchors flip-free. The
  more common "always start uptrend" convention used by
  some libraries would produce a single mid-data trend
  flip on LINEAR DOWN; this primitive avoids that
  ambiguity.
- The supertrend line itself can change abruptly on
  trend flips (closes ratchet finalUpper / finalLower);
  the SMA signal smooths this somewhat but the cross-sig
  remains responsive to volatility regime changes. This
  is the canonical Supertrend behaviour.
- On steady LINEAR input supertrend produces a constant
  +/- 1 offset against signal and zero crosses, which is
  correct canonical behaviour -- real-world price data
  with trend-direction changes produces typical trend-
  stop centerline cross patterns.
- Complementary to the sibling SMA / EMA / Bollinger /
  Keltner / Donchian / CCI / RSI / Ichimoku mid-cross-
  sig primitives -- Supertrend adds a volatility-aware
  ratcheting trailing stop to the cross-sig family,
  distinct from the simple-MA, Donchian-composite, and
  pure-momentum siblings.

# Patch 11.748 -- UI: chart-line-kase-peak primitive

Released as `v1.11.766` on 2026-05-26.

## Scope

Add `<ChartLineKasePeak>` -- a pure-SVG two-panel chart with a
Cynthia Kase **Peak Oscillator** panel reading the difference of
two volatility-scaled momentum series at the two cycle lengths.

## Definition

Inputs:

- `data: ChartLineKasePeakPoint[]` where
  `ChartLineKasePeakPoint = { x; close }`. Bars with non-finite
  `x` / `close` are dropped.
- `fastLength: number` -- the fast-cycle lookback. Default `5`.
  Normalized to an integer `>= 1`.
- `slowLength: number` -- the slow-cycle lookback. Default `30`.
  Normalized to an integer `>= 1`.
- `threshold: number` -- the `+/-` peak-zone boundary. Default
  `1`. Normalized to a positive finite.

Algorithm (per bar `i >= L`):

```
mom(L, i)     = close[i] - close[i - L]
volAvg(L, i)  = mean over [i - L + 1, i] of abs(close[j] - close[j - 1])
peakOut(L, i) = mom(L, i) / volAvg(L, i)

KP[i] = peakOut(fastLength, i) - peakOut(slowLength, i)
```

A bar with a zero `volAvg` on either leg is null. The first
`max(fast, slow)` bars are null on the KP.

## Bit-exact algebraic anchors

**CONST_FLAT** (`close == K`): every delta is zero, so `volAvg = 0`
and the bar is null on every cycle.

**Monotone rising** (`close[j+1] - close[j] = +1`): for any cycle
`L`,

- `mom(L, i) = (i + 10) - (i - L + 10) = L`,
- `volAvg(L, i) = mean of |+1, +1, ..., +1| = 1`,
- `peakOut(L, i) = L / 1 = L` bit-exact.

So with `fast = 2`, `slow = 4`, `KP = 2 - 4 = -2` bit-exact at
every defined bar.

**Monotone falling**: every delta is `-1`, so `mom = -L`, `volAvg
= 1`, `peakOut(L) = -L`. With same lengths, `KP = -2 - (-4) = +2`
bit-exact.

**Antisymmetry**: reversing the closes (RISING vs FALLING) flips
every `mom` sign while leaving every `volAvg` unchanged, so
`peakOut(L)` flips sign and `KP` flips sign. `KP(RISING) =
-KP(FALLING)` bit-exact.

**Translation invariance**: shifting every close by `+k` leaves
every `close[j] - close[j - 1]` and every `close[i] - close[i -
L]` unchanged, so `peakOut` and `KP` are unchanged bit-exact.

**Scale invariance**: multiplying every close by `s` scales both
`mom` and `volAvg` by `s`, so the ratio is unchanged bit-exact.

**`fast == slow`**: both `peakOut` values are identical, so `KP =
0` bit-exact at every defined bar.

## Component

`forwardRef<HTMLDivElement, ChartLineKasePeakProps>` with
displayName. Renders:

- A region-role outer `<div data-section="chart-line-kase-peak">`.
- An `<svg role="img">` with an `aria-describedby` description.
- Top panel: close line, optional dots.
- Bottom panel: KP line, optional dashed fast / slow `peakOut`
  overlays, zone-coloured markers (peak-bull / bull / flat /
  bear / peak-bear / none), zero line, dashed `+/- threshold`
  reference lines.
- Legend toggle for `price` / `kp` / `fast` / `slow`, plus
  `showKp` / `showFast` / `showSlow` props and an `onPointClick`
  handler.
- ARS design-system tokens for colour and motion (`motion-safe:
  animate-fade-in`).

## Acceptance

- CONST_FLAT identity: KP null at every bar.
- Monotone rising: `peakOut(L) = L` -> KP = `fast - slow`
  bit-exact.
- Monotone falling: KP = `slow - fast` bit-exact.
- Antisymmetry: `KP(RISING) = -KP(FALLING)` bit-exact.
- Translation invariance: `+1000` leaves KP unchanged.
- Scale invariance: `*100` leaves KP unchanged.
- `fast == slow`: KP = 0 bit-exact.
- Layout / component / ARIA / data-section hooks all covered.

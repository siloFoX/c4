## Summary

Add a new `<ChartRadarSpider>` UI primitive in
`web/src/components/ui/chart-radar-spider.tsx` -- pure-SVG
spider / radar variant that is **distinct from
`<ChartRadar>` (11.445)**. Where `<ChartRadar>` renders one
filled polygon per series (colour-by-series),
`<ChartRadarSpider>` **decomposes each series's polygon
into per-axis wedges**: each wedge is the triangle from
chart centre through the data point at axis N to the data
point at axis N+1, coloured by axis. The result emphasises
the per-axis contribution rather than the per-series
silhouette -- ideal for highlighting which dimensions
dominate a single profile. Optional outline polygon
connects the per-axis points to retain the silhouette
shape on top of the wedges.

## API

```ts
interface ChartRadarSpiderAxis {
  id: string;
  label: string;
  max?: number;
  color?: string;            // per-axis colour override
}

interface ChartRadarSpiderSeries {
  id: string;
  label: string;
  data: readonly number[];
  outlineColor?: string;
}

interface ChartRadarSpiderProps {
  axes: readonly ChartRadarSpiderAxis[];
  series: readonly ChartRadarSpiderSeries[];
  size?: number;                       // default 320
  padding?: number;                    // default 48
  levels?: number;                     // default 4
  maxValue?: number;
  showGrid?: boolean;                  // default true
  showAxisLabels?: boolean;            // default true
  showLegend?: boolean;                // default true
  showTooltip?: boolean;               // default true
  showPoints?: boolean;                // default true
  showOutline?: boolean;               // default true
  animate?: boolean;                   // default true
  wedgeOpacity?: number;               // default 0.28
  outlineOpacity?: number;             // default 0.85
  className?: string;
  ariaLabel?: string;                  // default 'Spider chart'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  legendPlacement?: 'right' | 'bottom';
  onWedgeClick?: (args: {
    series: ChartRadarSpiderSeries;
    axis: ChartRadarSpiderAxis;
    seriesIndex: number;
    axisIndex: number;
    value: number;
  }) => void;
}
```

## Behaviour

- Renders a `<div role="region">` wrapping `<svg
  role="img">` carrying the same `aria-label`.
- Layout: axes are placed around a circle starting at
  12 o'clock and proceeding clockwise. The chart
  radius derives from `size / 2 - padding`.
- Per-axis colour: each axis owns its own colour. By
  default the palette cycles a distinct 8-colour set
  (`DEFAULT_CHART_RADAR_SPIDER_AXIS_PALETTE`) which
  is intentionally different from the per-series
  palette in `<ChartRadar>` so the two primitives are
  visually distinguishable. `axis.color` overrides
  per axis.
- Wedges: for every series, one triangle (`centre,
  point_N, point_(N+1) mod N`) per axis. Wedge fill =
  axis colour at `wedgeOpacity`; stroke = axis colour
  at higher opacity. Hover lifts the opacity and
  thickens the stroke.
- Outline polygon: closes the per-axis points back to
  form the canonical radar silhouette on top of the
  wedges. Useful to retain the "shape" reading while
  the wedges still encode axis contribution. Opt-out
  via `showOutline=false`.
- Per-axis data points: small filled circles at each
  data point. Opt-out via `showPoints=false`.
- Grid + spokes: level polygons drawn at evenly-spaced
  ratios; spokes render in the matching axis colour
  for a strong per-axis identity. `showGrid=false`
  suppresses both.
- Legend: one entry per axis (per-axis colour
  encoding). Right-aligned by default; bottom layout
  available.
- Tooltip: hover or focus a wedge to surface series
  label + axis label + value. Empty values display as
  `'?'`.
- `onWedgeClick` receives the full payload (`series`,
  `axis`, both indices, value).
- Mount animation:
  `motion-safe:animate-fade-in` per series group.
  `animate=false` removes the class.
- Default ARIA description: `"Spider chart with N
  axes and S series. <label>: <axis> <v>, ..."` --
  empty input returns `"No data"`;
  `ariaDescription` overrides.

## Pure helpers (exported)

```ts
DEFAULT_CHART_RADAR_SPIDER_SIZE             = 320
DEFAULT_CHART_RADAR_SPIDER_PADDING          = 48
DEFAULT_CHART_RADAR_SPIDER_LEVELS           = 4
DEFAULT_CHART_RADAR_SPIDER_WEDGE_OPACITY    = 0.28
DEFAULT_CHART_RADAR_SPIDER_OUTLINE_OPACITY  = 0.85
DEFAULT_CHART_RADAR_SPIDER_AXIS_PALETTE     = 8 colours

getDefaultRadarSpiderAxisColor(index): string
getRadarSpiderAngle(axisIndex, total): number   // degrees
getRadarSpiderMax(axes, series, override?): number
getRadarSpiderRatio(value, max): number
getRadarSpiderPoint(axisIndex, totalAxes, ratio,
                    cx, cy, radius): { x, y }
buildRadarSpiderWedgePath(cx, cy, pa, pb): string
buildRadarSpiderGridPolygon(axisCount, ratio,
                            cx, cy, radius): string
describeRadarSpiderChart(axes, series, formatValue?): string
```

- `getDefaultRadarSpiderAxisColor` cycles the palette
  via modulo; negative indices return palette[0].
- `getRadarSpiderAngle` returns -90 for axis 0 and
  spaces the rest clockwise; total=0 short-circuits
  to -90.
- `getRadarSpiderMax` honours an override (positive +
  finite) first, then per-axis max, then the largest
  finite datum; falls back to 1.
- `getRadarSpiderRatio` clamps `value / max` into
  `[0, 1]`; non-finite / non-positive collapse to 0.
- `getRadarSpiderPoint` reuses `polarToCartesian`
  from `<ChartRadialBar>` for cartesian mapping;
  clamps ratio outside `[0, 1]`.
- `buildRadarSpiderWedgePath` emits a closed triangle
  (`M centre L pa L pb Z`); non-finite -> "".
- `buildRadarSpiderGridPolygon` emits an SVG
  `<polygon points>` string at the supplied ratio.
- `describeRadarSpiderChart` produces the default
  ARIA summary.

## ARIA + data attributes

- SVG: `role="img"` + same `aria-label`.

  `data-section="chart-radar-spider-aria-desc"`.

  `tabIndex={0}`. aria-label =
  `"<series.label> / <axis.label>: <value>"`.

  - `chart-radar-spider-wedge` (per-(series, axis)

- Wedges mirror `data-series-id`, `data-axis-id`,
  `data-axis-index`, `data-wedge-color`,
  `data-hovered`.

## Tests (56 cases)

- `getRadarSpiderAngle`: axis 0 at -90, clockwise
  spacing, total=0 short-circuit.
- `getDefaultRadarSpiderAxisColor`: index 0,
  modulo wrap, negative -> palette[0].
- `getRadarSpiderMax`: override (positive /
  non-positive), data max, per-axis max wins,
  empty / non-finite fallback to 1.
- `getRadarSpiderRatio`: linear, over-max clamp,
  non-finite / non-positive -> 0.
- `getRadarSpiderPoint`: ratio=0 at centre, axis 0
  full ratio above, axis 1 full ratio right, ratio
  clamping.
- `buildRadarSpiderWedgePath`: closed triangle,
  non-finite -> "".
- `buildRadarSpiderGridPolygon`: one point per
  axis, zero-axes -> "".

  summary, formatter shaping.

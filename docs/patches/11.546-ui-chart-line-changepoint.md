# 11.546 -- web/src/components/ui/chart-line-changepoint.tsx

Pure-SVG line chart that DETECTS structural change points via a
simple **variance-shift heuristic** and renders dashed vertical
markers at each detected boundary. Between markers, the chart
shades each "regime" with a different segment color (cycled
from a 6-color palette) so the regime boundaries are visible at
a glance. An optional segment-mean overlay draws the per-
segment mean as a dashed horizontal rule within the segment.

## Algorithm

For a series `y[0..N-1]` (sorted ascending by x, non-finite
dropped):

1. **Score every candidate index** `i` in `[minSegment,
   N - minSegment]`:
   - left segment = `y[0..i)`
   - right segment = `y[i..N)`
   - score = `|log(var(right) / var(left))|`
     (variance floored at `1e-9` to avoid log(0))
2. **Threshold**: keep only indices where `score >=
   threshold` (default 0.5).
3. **Non-maximum suppression**: walk the surviving candidates
   in descending score order and keep one only if no
   already-kept detection is within `suppressionWindow`
   indices.
4. **Segments**: split the series into `K+1` segments at the
   `K` detected indices. Each segment carries
   `{startIndex, endIndex, startX, endX, count, mean,
   variance, stdDev}`.

The change point at index `i` marks the FIRST sample of the new
segment (left-edge convention).

## Distinct from related primitives

- `<ChartLineAnomaly>` (11.524): per-point z-score outlier
  flag. Changepoint detects PERSISTENT regime shifts, not
  isolated outliers.
- `<ChartLineControl>` (11.534): global SPC mean +/- k*sigma
  reference rules. Changepoint splits the series into
  multiple regimes, each with its own mean / variance.
- `<ChartLineEvent>` (11.515): external event markers
  supplied by the caller. Changepoint markers are DETECTED
  from the data itself.
- `<ChartLineStreak>` (11.527): consecutive same-direction
  RUNS. Changepoint marks any regime shift (mean / variance),
  not direction-specific.
- `<ChartLineDecompose>` (11.543): periodic trend + seasonal
  split. Changepoint marks ABRUPT structural breaks with no
  periodicity assumption.

The canonical "find me the regime shifts in this time series"
primitive used in monitoring, financial regime detection, EEG
state changes, and any context where the question is "where
did the behavior change?".

## Public surface

- `<ChartLineChangepoint series={...} />`.
- `series: ChartLineChangepointSeries[]` -- `{id, label,
  data, color?, minSegment?, threshold?, suppressionWindow?}`.
  Per-series detector overrides always beat chart-level.
- `minSegment` (default `5`; clamped to `>= 2`) -- minimum
  number of samples each side of a candidate.
- `threshold` (default `0.5`; allows 0; non-finite -> default)
  -- minimum log-variance-ratio score to keep.
- `suppressionWindow` (default `3`; clamped to `>= 1`) --
  non-max suppression radius in samples.
- `markerColor` (default red), `segmentColors` (6-color
  palette), `markerDashArray` (default `'4 3'`),
  `segmentOpacity` (default `0.06`).
- Toggle flags: `showAxis`, `showGrid`, `showDots`,
  `showLegend`, `showTooltip`, `showCountBadge`,
  `showMarkers`, `showSegmentShading`, `showSegmentMeans`,
  `animate`.
- Controlled + uncontrolled visibility via `hiddenSeries` /
  `defaultHiddenSeries` / `onHiddenSeriesChange`.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX` / `formatScore`.
- `onPointClick({series, point})` /
  `onMarkerClick({series, detection})` /
  `onSeriesToggle({series, hidden})`.

## Pure helpers

- `getLineChangepointDefaultColor(index)` -- 10-color
  palette cycle; NaN / negative -> color 0.
- `getLineChangepointSegmentColor(index, palette?)` -- cycles
  the segment palette.
- `getLineChangepointFinitePoints(points)` -- finite-only
  filter.
- `normaliseLineChangepointMinSegment(value)` -- clamps to
  `>= 2`; floors fractional; non-finite -> default 5.
- `normaliseLineChangepointThreshold(value)` -- non-finite /
  negative -> default 0.5; zero allowed.
- `normaliseLineChangepointSuppressionWindow(value)` --
  clamps to `>= 1`; non-finite -> default 3.
- `computeLineChangepointScores(values, {minSegment?})` --
  per-index variance-shift score. Returns null at indices
  where the window is not yet filled. Non-array -> []. Input
  shorter than `2*minSegment` -> all null. Variance floored
  at `1e-9` to avoid log(0).
- `detectLineChangepoints(points, {minSegment?, threshold?,
  suppressionWindow?})` -- canonical detector pipeline.
  Sorts ascending; drops non-finite; computes scores;
  applies threshold; applies non-max suppression; builds
  segments from cut points. Returns
  `{ok, samples, scores, detections, segments, minSegment,
  threshold, suppressionWindow}`. Each detection carries
  `{index, x, leftMean, rightMean, leftVariance,
  rightVariance, score, meanShift}`. Each sample carries
  `{index, x, y, segmentIndex}`. Detections are sorted
  ascending by index.
- `computeLineChangepointLayout({series, ...})` -- projects
  per-series path + detection markers + segment shading
  rects + segment-mean overlay positions. Returns standard
  layout shape with `series, panel, xTicks, yTicks, ...,
  totalPoints, totalDetections, visibleSeriesCount`.
- `describeLineChangepointChart(series, options?)` --
  screen-reader description.

  `data-animate`.
- Badge mirrors the same detection count + threshold.

## Tooltip

Hovering a marker shows "<label> changepoint", x, bold
score, mean shift (with `left -> right` values), and
variance shift. Hovering a dot shows label, x, bold y, and
the segment number with `(mean <m>, sigma <s>)`.

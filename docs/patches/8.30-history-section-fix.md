# TODO 8.30 - HistoryView scribe section transition fix

**Status:** done
**Branch:** `c4/history-fix`
**Scope:** Fix the HistoryView sidebar/main-pane transition so clicking
a worker while the scribe viewer is open actually swaps the main pane
to the worker detail. Until 8.30 `showScribe` stayed `true` after a
worker selection, so scribe content kept rendering and the newly
fetched detail was invisible.

## Reproduction (pre-fix)

1. Open the Web UI History tab.
2. Click the sidebar `Scribe` button - the main pane renders the long
   `docs/session-context.md` viewer.
3. Click any worker in the sidebar list.
4. Observed: scribe content stays visible. The new detail pane does not
   appear. User must first press `Close` on the scribe viewer.

## Root cause

`web/src/components/HistoryView.tsx` drove the main area with two
independent flags:

```tsx
{showScribe ? <ScribeCard /> : detail ? <HistoryDetailPane ... /> : <Placeholder />}
```

`openScribe()` set `showScribe = true` but the worker-list `onClick`
only set `selected`. `showScribe` was never cleared on selection, so
the ternary kept the first branch. Likewise, the sidebar highlight
stayed on whichever worker was `selected`, giving no visual clue that
the active section was still `scribe`.

## Fix

1. **Single discriminator.** A derived
   `const activeSection: 'scribe' | 'detail' | 'placeholder'` is
   computed from `showScribe` and `detail` once per render. This is the
   source of truth for what the main pane is currently showing.
2. **`selectWorker` helper.** The sidebar list `onClick` funnels
   through `selectWorker(name)` which calls `setShowScribe(false)`
   before `setSelected(name)`. Scribe closes whether or not the worker
   was already selected, so re-clicking the same row while scribe is
   open still flips the pane.
3. **Key-on-main remount.** `<main key={activeSection} ...>` forces
   React to unmount and remount the entire content subtree on a
   section change. The ternary branches already rendered distinct
   subtrees, but the `Card` in both the scribe and placeholder branches
   was the same React type - the key ensures scroll position, in-flight
   focus, and any future local state do not leak across sections.
4. **Visual active state.** The `Scribe` button now picks
   `variant = showScribe ? 'default' : 'secondary'` and carries
   `aria-pressed={showScribe}` + `data-active`. The sidebar
   `isSelected` predicate is narrowed to
   `!showScribe && selected === w.name` so a worker row loses the
   `ring-1 ring-ring` highlight while scribe is the active section.
   Each worker button also carries `aria-pressed={isSelected}` for
   screen readers.

## Why not just clear `showScribe` in a `useEffect`?

An effect on `[selected]` would clear scribe *after* the selection
commit, which (a) still renders one frame of scribe before the swap
and (b) would not clear scribe when the user re-clicks the currently
selected worker (no state change, no effect). The `selectWorker`
helper makes the intent explicit at the callsite and fires in the
same commit.

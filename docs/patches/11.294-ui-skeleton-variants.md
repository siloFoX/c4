# 11.294 -- ChipSkeleton + dispatch aliases (v1.11.312)

Component-scope-only enhancement of the existing `Skeleton`
primitive family. No daemon-side change and no `c4` CLI surface
change. Adds the new `ChipSkeleton` shape + dispatch-friendly
aliases for `AvatarShape` / `TableRowShape`.

## Why

The dispatch asked for four semantic skeleton variants:
`TextLine`, `AvatarSkeleton`, `ChipSkeleton`, `TableRowSkeleton`.

The existing primitive family already covers three of them:
- `TextLine` -- shipped at v1.11.174.
- `AvatarShape` -- the same idea as `AvatarSkeleton`.
- `TableRowShape` -- the same idea as `TableRowSkeleton`.

Two adjustments make the surface match the dispatch vocabulary:

1. The missing `ChipSkeleton` shape is a rounded-full pill
   placeholder for chip / tag / badge rows during loading
   (Templates filter chip rows, Workers metric chips,
   Profiles role chips). Inline `<Rect rounded="full">` works
   but does not match the chip silhouette by default; a
   dedicated component nails the height / width ratio.
2. `AvatarShape` + `TableRowShape` are renamed via aliases to
   `AvatarSkeleton` + `TableRowSkeleton` so callers reach for
   the consistent `*Skeleton` vocabulary the dispatch named.
   Existing `AvatarShape` / `TableRowShape` imports continue to
   work; the new aliases are additive.

## What changed

### `web/src/components/ui/skeleton.tsx`

- New exported type `ChipSkeletonSize = 'sm' | 'md' | 'lg'`.
- New `ChipSkeleton` component. Props:
  - `size?: 'sm' | 'md' | 'lg'` (default `'md'`). Per-size
    height + width:
    - `sm`: 14px / 48px.
    - `md`: 18px / 64px.
    - `lg`: 24px / 80px.
  - `width?: string | number` -- overrides the per-size
    default. Number coerces to px; string passes through.
  - `className?` + all standard `HTMLDivElement` attrs.
- Shimmer routed through the shared `loading-motion` contract
  so the pulse rhythm matches every other Skeleton variant
  AND stops entirely under `prefers-reduced-motion: reduce`.
- ARIA: `role="status"` + `aria-hidden="true"` (the visible
  pill is decorative; SR users hear "Loading" via the
  surrounding region).
- Data-attribute selectors:
  - `data-section="chip-skeleton"`.
  - `data-skeleton-shape="chip"`.
  - `data-chip-skeleton-size="sm|md|lg"`.
- New exported aliases:
  - `AvatarSkeleton` = `AvatarShape` (same component reference).
  - `TableRowSkeleton` = `TableRowShape` (same component
    reference).

### Adopted

- Primitive enhancement only this commit. Existing Skeleton
  call sites (LoadingSkeleton in PageFrame, Templates filter
  loaders, etc.) inherit the new shape options; threading
  `<ChipSkeleton>` into the per-page loading states is a
  follow-up.

## Deferred (dispatch follow-ups)

- HistoryView loading: thread `<ChipSkeleton>` into the
  status / branch / commit-hash chips that load before the
  HistoryDetailPane mounts.
- Workers loading: thread `<AvatarSkeleton>` next to the
  worker-row skeletons so the loading state previews the
  Avatar tile.
- Sessions loading: same story for the SessionsListSection
  rows.

## Notes

- The ChipSkeleton dimensions are intentionally a rough
  match for the Chip primitive (not pixel-perfect). Callers
  preview a chip row without specifying the exact label
  length.
- A pre-existing exactOptionalPropertyTypes TS error in
  `TableRowShape` (line 331, where
  `width={TABLE_ROW_WIDTHS[i % len]}` returns
  `string | undefined`) is unchanged by this commit. Tracked
  as a follow-up cleanup; the runtime behaviour is correct
  (Rect accepts undefined width gracefully).

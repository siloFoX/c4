- **`pageLabelFormatter`** -- i18n hook
  for the per-page `aria-label`.
- **`disabled`** -- whole-pagination
  disable (matching the table-loading
  use case).
- **`buildPaginationItems` export** --
  pure helper for tests + alternate
  hosts that want their own page-list
  chrome.

## What changed

### `boundaryCount`

Default `1`. Controls how many always-
visible pages render at each boundary
(leading / trailing).

| total | page | boundary=1 (default) | boundary=2 | boundary=3 (small total) |
| --- | --- | --- | --- | --- |
| 10 | 5 | `1, ..., 4, 5, 6, ..., 10` | `1, 2, ..., 4, 5, 6, ..., 9, 10` | `1, 2, 3, ..., 5, ..., 8, 9, 10` |
| 7 | 3 | `1, 2, 3, 4, ..., 7` | `1, 2, 3, 4, ..., 6, 7` | `1, 2, 3, 4, 5, 6, 7` (no ellipsis) |

### `pageLabelFormatter`

```ts
pageLabelFormatter?: (page: number) => string;
```

When supplied, the resolver wraps the
formatter in a try/catch. If the formatter
throws or returns an empty string, the
legacy `"Page <n>"` default is used --
assistive tech always gets a label.

### `disabled`

```ts
disabled?: boolean;     // default false
```

Disables every interactive control:
- page number buttons
- prev / next
- first / last (when `showFirstLast`)
- jump-to-page input + Go button
- jump-form submission (no-op)

Mirrors on `data-disabled="true|false"` on
the root nav for CSS hooks.

### `buildPaginationItems` (pure helper)

```ts
export function buildPaginationItems(
  page: number,
  totalPages: number,
  siblingCount: number,
  boundaryCount?: number,
): Array<number | '...'>;
```

Returns the rendered page list. Exported
for tests + adopters that need to render
their own chrome with the same ellipsis
math.

## Tests

`pagination.test.tsx` -- **+17 new cases
(43 total)**:

`buildPaginationItems` (6):
- empty / 1-total returns `[1]`.
- default `boundaryCount=1` produces
  `[1, ..., page-1, page, page+1, ..., N]`.
- `boundaryCount=2` shows two pages on
  each side.
- `boundaryCount=3` with small total
  renders consecutive pages (no
  ellipsis).
- current page near start collapses
  trailing ellipsis only.
- current page near end collapses
  leading ellipsis only.

Component (11):
- `boundaryCount=2` renders 4 anchor
  pages (1+2+19+20 for a 20-page list).
- `pageLabelFormatter` overrides the
  aria-label.
- formatter that throws falls back to
  `"Page N"`.
- formatter that returns empty falls
  back.
- `disabled=true` disables every page
  button + suppresses onPageChange.
- `disabled=true` disables prev / next.
- `disabled=true` disables jump input +
  Go button.
- `disabled=true` suppresses jump form
  submission.
- `data-disabled` attr mirrors disabled.
- default `disabled=false` keeps
  controls active.
- ellipsis spans carry
  `data-section="pagination-ellipsis"`.

## Pairs with existing primitives

- `<DataTable>` (11.355) -- non-virtual
  table; pair with Pagination for
  paginated lists.
- `<VirtualTable>` (11.375) -- virtual
  table for 10k+ rows; do NOT pair with
  Pagination (the virtualizer is the
  paging strategy).

## Out of scope

- Per-page adoption of new props.
  Additive; every existing call site
  stays byte-identical at the legacy
  defaults.
- Page-size selector
  ("10 / 25 / 50 per page"). Belongs
  in a `<PageSizeSelect>` primitive
  paired with Pagination by the caller.
- Total-items count display
  ("Showing 21-30 of 250"). Belongs
  in a `<PaginationSummary>` sibling.
- Cursor-based pagination contract.
  Current primitive is offset / index
  only.
- Compact mobile variant. The existing
  primitive wraps when short on
  horizontal space.

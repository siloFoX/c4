# 11.1053 -- UI: chart-line-trix-mid-cross-sig primitive (v1.11.1071)

## Scope

Add a pure-SVG dual-panel `ChartLineTrixMidCrossSig`
component that renders the close in the top panel and the
TRIX (triple-smoothed exponential rate-of-change) plus its
smoothed SMA signal line in the bottom panel, marking
bullish / bearish TRIX-over-signal crossover trigger
events with bias coloring derived from the TRIX slope at
the trigger bar.

Jack Hutson's 1980 TRIX is the canonical triple-
exponential smoothed rate-of-change oscillator: by
chaining three EMAs and taking the change of the third,
it filters out short-term noise and pure-trend lag while
preserving turning-point detection. `0` is the natural
centerline; positive TRIX means the triple-smoothed close
is accelerating up (bullish momentum), negative means
decelerating (bearish momentum).

## TRIX pipeline

```
ema1[i]      = EMA(close, period)   (SMA-seeded)
ema2[i]      = EMA(ema1,  period)
ema3[i]      = EMA(ema2,  period)
trix[i]      = ema3[i] - ema3[i - 1]   (i >= 3p - 2)
signal[i]    = SMA(trix, signalLength)

bullish      : prev trix <= prev signal &&
               cur trix > cur signal
bearish      : prev trix >= prev signal &&
               cur trix < cur signal
regime       : 'bullish' when trix >= signal
               'bearish' when trix <  signal
               'none'    when either is null
bias         : trix[i] vs trix[i-1] -> up / down / flat /
               none
```

Default `period = 15`, `signalLength = 3` -- canonical
Hutson tuning paired with the family-conventional short
SMA signal.

This primitive uses the **absolute-slope variant** of TRIX
(`ema3[i] - ema3[i-1]`, in the same units as price) rather
than Hutson's percentage form (`100 * (ema3[i] - ema3[i-1])
/ ema3[i-1]`). The two variants share the centerline-at-0
semantics; the absolute-slope form:

1. gives bit-exact integer anchors on LINEAR input (the
   percentage form is non-bit-exact because the
   denominator `ema3[i-1]` grows with i),
2. avoids the 0/0 NaN edge case when EMA3 transits
   through 0 (e.g. close=0 CONST band),
3. mirrors the TEMA divergence sibling v1.11.1059's
   zero-lag identity convention.

This is a documented family-level tuning choice. Consumers
who prefer the percentage form can wrap with `100 * trix /
prevEma3`.

The internal EMA helper (`applyLineTrixMidCrossSigEma`)
implements a **null-aware seed**: when invoked on the
output of another EMA (chained EMAs have leading nulls),
it finds the first `length`-window of consecutive non-null
inputs and seeds the SMA there. This makes
`EMA(EMA(EMA(x)))` work cleanly without explicit padding.

Warmup is `3 * (period - 1) + signalLength = 45` for the
default tuning: EMA1 seeds at i = period - 1 = 14, EMA2
at i = 2 * (period - 1) = 28, EMA3 at i = 3 * (period -
1) = 42, TRIX (needs EMA3[i - 1]) at i = 43, then the
signal SMA needs `signalLength - 1 = 2` more bars.

## Bit-exact anchors

All anchors use close-only input.

- **CONST band** `close = K`: SMA-seeded EMA on a
  constant input is exactly the constant. ema1 = ema2 =
  ema3 = K for every valid bar. trix = K - K = 0
  (exactly the centerline). signal = SMA(0, 3) = 0.
  trix === signal -> regime `bullish` (>=) for every
  valid bar. 0 crosses. Verified across K in {0, 1, 50,
  200, 1234}. The CONST anchor lands *exactly* on the
  centerline -- canonical TRIX behaviour on a flat trend.
- **LINEAR UP** `close = i`: SMA(close[0..14], 15) = 7,
  so ema1[14] = 7. Recurrence on linear: ema1[i] =
  alpha * i + (1 - alpha) * (i - 1 - 7) = i - 7
  (exact). Similarly ema2[i] = ema1[i] - 7 = i - 14
  and ema3[i] = ema2[i] - 7 = i - 21. trix[i] =
  ema3[i] - ema3[i - 1] = (i - 21) - (i - 1 - 21) =
  +1 (constant slope). signal = SMA(+1, 3) = +1.
  trix === signal -> regime `bullish` (>=). 0 crosses.
  The constant +1 reflects the unit slope of the LINEAR
  input -- canonical TRIX behaviour: the rate-of-change
  of a triple-smoothed line is exactly the original
  line's slope.
- **LINEAR DOWN** `close = -i`: mirror -> trix = -1
  (constant). signal = -1. trix === signal -> regime
  `bullish` (>=) for the same `===` reason. 0 crosses.

The 3 * 7 = 21 cumulative SMA centroid lag (period=15
gives centroid offset 7 per EMA, applied 3x) on LINEAR
input is the canonical triple-EMA lag.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 3-state regime classifier follows the cross-sig family
  convention (bullish on `===`, bearish on `<`).
- Absolute-slope variant of TRIX differs from Hutson's
  original percentage form. Documented as a deliberate
  family-level choice for bit-exact testability and
  NaN-free behaviour. The signal-line cross semantics
  are identical between the two variants (both share
  the centerline at 0).
- Null-aware EMA seed lets chained EMAs work without
  explicit padding. This matches how pandas-ta and
  ta-lib handle chained smoothing.
- TRIX is unbounded -- the layout panel auto-ranges but
  clamps to keep the centerline (0) visible.
- Large warmup (45 bars for default period=15): TRIX is
  the deepest-smoothed oscillator in the family. Short-
  input data returns ok=false.
- Complementary to the sibling SMA / EMA / Bollinger /
  Keltner / Donchian / CCI / RSI / Ichimoku /
  Supertrend / Williams / DPO mid-cross-sig primitives
  -- TRIX adds a triple-smoothed momentum-rate oscillator
  alongside the simple-MA, bounded-momentum, mean-
  reversion, and true-detrending siblings. Sibling to
  the TEMA divergence-cross primitive v1.11.1059 (same
  chained-EMA family, different output: TRIX is slope-
  of-EMA3, TEMA is 3*EMA1 - 3*EMA2 + EMA3).

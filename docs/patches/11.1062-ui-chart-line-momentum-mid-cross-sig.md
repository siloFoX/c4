# 11.1062 -- UI: chart-line-momentum-mid-cross-sig primitive (v1.11.1080)

## Scope

Add a pure-SVG dual-panel `ChartLineMomentumMidCrossSig`
component that renders the close in the top panel and the
Momentum oscillator plus its smoothed SMA signal line in
the bottom panel, marking bullish / bearish Momentum-
over-signal crossover trigger events with bias coloring
derived from the Momentum slope at the trigger bar.

The Momentum oscillator is the simplest and oldest of all
momentum indicators: the raw difference between the
current close and the close `period` bars ago. Unlike the
rate-of-change (ROC) variant, Momentum is in price units
rather than a percentage. `0` is the natural centerline.

## Momentum cross-sig pipeline

```
momentum[i]  = close[i] - close[i - period]
signal[i]    = SMA(momentum, signalLength)

bullish      : prev momentum <= prev signal &&
               cur momentum > cur signal
bearish      : prev momentum >= prev signal &&
               cur momentum < cur signal
regime       : 'bullish' when momentum >= signal
               'bearish' when momentum <  signal
               'none'    when either is null
bias         : momentum[i] vs momentum[i-1] -> up / down /
               flat / none
```

Default `period = 10`, `signalLength = 3` -- canonical
Momentum tuning paired with the family-conventional short
SMA signal.

Warmup is `period + signalLength - 1 = 12` for the
default tuning: momentum is valid from i = period = 10,
then the signal SMA needs `signalLength - 1 = 2` more
bars.

## Bit-exact anchors

All anchors use close-only input.

- **CONST band** `close = K`: momentum = K - K = 0
  (exactly the centerline). signal = SMA(0, 3) = 0.
  momentum === signal -> regime `bullish` (>=) for every
  valid bar. 0 crosses. Verified across K in {0, 1, 50,
  200, 1234}.
- **LINEAR UP** `close = i`: momentum = i - (i - period)
  = period = +10 (constant -- exactly the slope per bar
  times the lookback). signal = SMA(+10, 3) = +10.
  momentum === signal -> regime `bullish` (>=). 0
  crosses. The constant +10 is the simplest possible
  momentum reading: a steady +1/bar slope over a 10-bar
  lookback yields a constant momentum of +10.
- **LINEAR DOWN** `close = -i`: momentum = -i - (-i +
  period) = -period = -10 (mirror). signal = -10.
  momentum === signal -> regime `bullish` (>=) for the
  same `===` reason. 0 crosses.

The clean integer LINEAR anchors (+/- period exactly) are
a defining feature of Momentum -- no centroid lag, no
smoothing offset, just the raw slope-over-lookback. This
is the simplest member of the mid-cross-sig family.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 3-state regime classifier follows the cross-sig family
  convention (bullish on `===`, bearish on `<`).
- Momentum is unbounded -- the layout panel auto-ranges
  but clamps to keep the centerline (0) visible
  regardless of data range. Differs from RSI / Williams
  %R (hard-bounded panels) and matches CCI / TRIX /
  DPO (unbounded with centerline guard).
- The simple `close - close[i-period]` formulation is
  the original Momentum definition. Some libraries
  refer to "Momentum (100 base)" which scales by 100 *
  close[i] / close[i-period] -- this primitive uses the
  raw subtraction form per the canonical definition.
- On steady LINEAR input momentum saturates at `+/-
  period` and the signal === momentum, so 0 crosses
  fire. Real-world price data with reversals produces
  typical momentum cross patterns.
- Sibling to the ROC mid-cross-sig family member (rate-
  of-change percentage variant); together they bracket
  the simplest-momentum centerline family. The other
  mid-cross-sig siblings (SMA / EMA / Bollinger /
  Keltner / Donchian / CCI / RSI / Ichimoku /
  Supertrend / Williams / DPO / TRIX / HMA) use
  different underlying smoothing or transformations.

# Patch 11.843 -- ChartLineDonchianCross primitive (v1.11.861)

## Summary

Add `<ChartLineDonchianCross />`, a pure-SVG single-panel chart that
overlays the Donchian channel on a price line, with scatter markers
at every event where the close newly breaks above the upper band or
below the lower band:

```
upper[i] = max(high[i - length .. i - 1])
lower[i] = min(low[i  - length .. i - 1])
upperCross = curr close > curr upper  AND  prev close <= prev upper
lowerCross = curr close < curr lower  AND  prev close >= prev lower
```

- The lookback window deliberately excludes the current bar so that
  a fresh new high in `close[i]` can register as an "above the
  band" event. The strict-inequality + prev-state guard ensures the
  cross fires only on the bar that newly enters the breakout zone
  (not every subsequent bar inside it).
- Warmup: `i < length` -> `null` bands.

## Defaults

| Knob              | Value     |
|-------------------|-----------|
| length            | 20        |
| priceColor        | `#2563eb` |
| upperColor        | `#06b6d4` (cyan) |
| lowerColor        | `#0e7490` (dark teal) |
| bullishColor      | `#16a34a` (green) for upper-cross markers |
| bearishColor      | `#dc2626` (red) for lower-cross markers |
| markerRadius      | 5         |

Length 20 matches the canonical Donchian channel (Richard Donchian's
turtle trading setup); cyan/teal distinguishes the band overlay from
the prior sma-cross / ema-cross primitives.

## Bit-exact anchors

- `CONST high = low = close = K`: `upper = lower = K` for every
  post-warmup bar. Strict inequality `close > upper` and `close <
  lower` both fail (equal), so `upCrossCount = downCrossCount = 0`.
  Verified across `(K, length)` sweeps.
- `LINEAR UP high = low = close = i + 1`: `upper[i] = i` and
  `close[i] = i + 1`, so `close > upper` is true post-warmup. But
  `close[i-1] > upper[i-1]` was already true, so no NEW crosses
  fire. Zero crosses across `length in {2, 4, 7}`.
- `LINEAR DOWN high = low = close = N - i`: symmetric -> zero
  crosses.
- `STEP close = K1 for i < N, K2 for i >= N` with `K1 != K2` and
  `N >= length`: between `i = length` and `i = N - 1` close == upper,
  so no cross. At `i = N`, `upper = K1` (max of K1's), `close = K2`;
  if `K2 > K1` an upper-cross fires, if `K2 < K1` a lower-cross
  fires. At `i = N + 1` the lookback window already includes the
  step bar so `upper = K2` -> no second cross. Exactly one cross at
  exactly `i = N` across `(K1, K2, length)` sweeps including
  `(1, 5)`, `(5, 1)`, `(10, 100)`, `(100, 10)`.

## Surface

Single-panel primitive surface:

- `ChartLineDonchianCrossPoint = { x, high, low, close }`
- `runLineDonchianCross(data, options?) -> Run` with
  `ok = series.length > length`.
- `computeLineDonchianCrossLayout({ data, width?, height?, padding?, length? }) -> Layout`.
  Single panel; y-axis auto-fits across `close / high / low / upper /
  lower`.
- `describeLineDonchianCrossChart(data, opts?) -> string` for the
  ARIA description.
- `<ChartLineDonchianCross />` `forwardRef<HTMLDivElement>` with
  three legend toggles (`price`, `upper`, `lower`), config badge
  with live cross counts, scatter markers at each crossover event,
  and a tooltip surfacing `close / high / low / upper / lower /
  relation / cross`. Live cross counts also exposed as
  `data-up-cross-count` / `data-down-cross-count` attributes for
  selector-driven assertions.

# TODO 11.510 - chart-line-baseline primitive

Status: SHIPPED v1.11.528 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartLineBaseline>`, a pure-SVG multi-series line chart with
**an adjustable horizontal baseline reference** and per-point
classification of whether each y value sits **above, below, or
equal to** the baseline. Each dot can be coloured by direction
(green above, red below, neutral equal) and the tooltip surfaces
the signed delta from baseline. The canonical "metric vs target"
pattern for performance dashboards (delta from 0, delta from
budget, delta from SLO).

## API

```tsx
<ChartLineBaseline
  series={[
    {
      id: 'sales',
      label: 'Sales',
      data: [
        { x: 0, y: 80 }, { x: 1, y: 120 }, { x: 2, y: 100 },
        { x: 3, y: 150 }, { x: 4, y: 90 },
      ],
    },
  ]}
  baseline={100}
  baselineLabel="target"
/>
```

## Distinct from prior chart-* primitives

- `<ChartLineAreaGradient>` (11.504) - baseline as area floor with
  gradient fill below; chart-line-baseline uses the baseline as a
  *reference line* with per-point classification.
- `<ChartLineThreshold>` (11.503) - multiple horizontal thresholds
  + zone shading; chart-line-baseline is one reference baseline
  with above/below dot colouring.
- `<ChartLineAnnotated>` (11.509) - vertical event annotations.
- `<ChartLineDiscontinuous>` (11.508) - gap markers.
- All other line variants focus on different dimensions.

## Baseline contract

### Classification

`classifyLineBaselinePoint(y, baseline, epsilon?)` returns
`{ delta, direction }`:

- `delta = y - baseline` (signed).
- `direction = 'above'` when `delta > epsilon`.
- `direction = 'below'` when `delta < -epsilon`.
- `direction = 'equal'` otherwise (within the epsilon noise band).
- Non-finite y or baseline -> `{ delta: 0, direction: 'equal' }`.

`epsilon` (default 0) widens the equality band so adopters who want
"essentially equal" can ignore tiny noise.

### Dot colour

`pickLineBaselineDotColor(direction, fallback, above, below, equal)`
returns the appropriate colour. Empty direction colour -> falls back
to series colour.

### Layout

`computeLineBaselineLayout` records per-series `aboveCount`,
`belowCount`, `equalCount` plus per-point `direction` and `delta`.
The baseline y is also exposed as `baseline` + `baselineY` (pixel),
clamped to inside the inner plot so the rule stays visually
meaningful even when the baseline is outside the active y range.

`getLineBaselineBounds` expands the y range to include the baseline
so the rule is always visible without an explicit `yMin` / `yMax`.

## Interactivity

- Hover or focus on a dot dims other series strokes to ~30%; opens
  a tooltip showing series label, formatted x, formatted y, and a
  coloured **delta row** describing `+/- delta vs baseline`.
- Per-dot `onPointClick({series, point})` callback.
- Legend includes "above" / "below" swatches when
  `colorDotsByDirection=true` (default).
- Mount fade via `motion-safe:animate-fade-in` (suppress via
  `animate=false`).

## Exported helpers

- `DEFAULT_CHART_LINE_BASELINE_*` numeric / string defaults.
- `DEFAULT_CHART_LINE_BASELINE_PALETTE` (10-colour palette).
- `getLineBaselineDefaultColor(index)` - palette wrap.
- `getLineBaselineFinitePoints(points)` - drops non-finite.
- `classifyLineBaselinePoint(y, baseline, epsilon?)` - canonical
  point-vs-baseline classifier.
- `pickLineBaselineDotColor(direction, fallback, above, below,
  equal)` - colour picker per direction.
- `getLineBaselineBounds(series, hidden?, baseline?)` - per-axis
  bounds with baseline expansion.
- `getLineBaselineTicks(min, max, count?)` - evenly-spaced ticks.
- `buildLineBaselinePath(points)` - `M x y L x y ...` line builder.
- `computeLineBaselineLayout(input)` - per-series layout with
  per-point direction + delta + per-series direction counts +
  baselineY clamped to plot.
- `describeLineBaselineChart(series, baseline, hidden?,
  formatValue?)` - ARIA summary with above/below/equal breakdown.

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only
  `data-section="chart-line-baseline-aria-desc"` span.
- SVG `role="img"` with the same `aria-label`.
- Baseline line is `role="graphics-symbol"` with `tabIndex={0}` and
  `aria-label="Baseline at <value>: <label>"`.
- Each line path + dot is `role="graphics-symbol"` with
  `tabIndex={0}`.
- Each dot's aria-label includes direction + delta:
  `"<series>: x=<x>, y=<y>, above/below baseline by <delta>"`.

## Data attributes

- Root: `data-section="chart-line-baseline"`, `data-baseline`,
  `data-baseline-y`, `data-above-count`, `data-below-count`,
  `data-equal-count`, `data-series-count`, `data-total-points`,
  `data-animate`.
- Series group: `data-series-above-count`, `data-series-below-count`,
  `data-series-equal-count` (plus the standard data-series-* attrs).
- Dot: `data-section="chart-line-baseline-dot"`, `data-series-id`,
  `data-point-index`, `data-x`, `data-y`, `data-delta`,
  `data-direction`, `data-dot-color`, `data-hovered`.
- Baseline reference: `data-section="chart-line-baseline-reference-line"`,
  `data-baseline-value`, `data-baseline-y`.
- Tooltip: `-tooltip-delta` row appears only when
  `showDeltaInTooltip=true`.
- Legend: standard series items PLUS
  `data-section="chart-line-baseline-legend-direction"` for the
  above/below swatches (only when `colorDotsByDirection=true`).

## Out of scope

- Per-series baseline (the baseline is chart-level; adopters who
  want a per-series target compose multiple instances).
- Slanted baseline (linearly-changing target). v1 is horizontal
  only; use `<ChartLineSegment>` if a slanted reference series is
  needed.
- Animated baseline drag (the baseline is a prop, not a draggable
  handle; adopters wire their own slider).
- Log-scale axes - linear only.
- Filled deviation area between line and baseline - use
  `<ChartLineAreaGradient>` with `baseline=value` for that pattern.

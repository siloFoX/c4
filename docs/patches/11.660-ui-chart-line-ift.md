## Summary

New `ChartLineIft` UI primitive at
`web/src/components/ui/chart-line-ift.tsx` -- a pure-SVG
two-panel chart with an Inverse Fisher Transform panel
compressing an oscillator into a sharp bounded signal.

## Indicator math

John Ehlers' Inverse Fisher Transform squashes any real input
into the open interval `(-1, 1)`:

```
IFT(x) = (exp(2x) - 1) / (exp(2x) + 1)
```

This is the hyperbolic tangent. Large positive inputs saturate
near `+1`, large negative inputs near `-1`, with a fast
transition through zero -- so a soft oscillator becomes a sharp,
bounded signal.

`computeLineIftTransform(x)` evaluates the transform in the
overflow-safe form -- it divides through by `exp` of the
larger-magnitude exponent so a large positive input saturates at
`+1` cleanly rather than producing `Infinity / Infinity = NaN`.

`computeLineIftOscillator(closes, period, scale)` builds the
input oscillator -- the scaled deviation of the close from its
`period`-bar moving average, `osc[i] = scale * (close[i] -
sma[i])`. `computeLineIft(closes, period, scale)` compresses that
oscillator through the transform.

### Exactness

The transform uses `exp`, so the IFT is transcendental in
general and the tests assert with `toBeCloseTo` -- in particular
`computeLineIftTransform(x)` is checked against `Math.tanh(x)`.
Two anchors are exact: `IFT(0) = 0` exactly (`exp(0) = 1`, so
`(1-1)/(1+1) = 0`), and therefore the IFT is exactly zero
wherever the oscillator is zero -- which happens whenever the
price sits exactly on its moving average.

The fixture closes `[20, 24, 20, 28, 20, 12, 24, 20, 20, 28]`
with `period` 2 and `scale` 0.5 make the oscillator an exact
dyadic -- `osc[i] = (close[i] - close[i-1]) / 4` -- so the
oscillator array `[., 1, -1, 2, -2, -2, 3, -1, 0, 2]` is asserted
with `toEqual`, and the bar where it is zero produces an exact
`IFT = 0`.

## API

- `getLineIftFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineIftPeriod(period, fallback)` -- coerce to a
  positive integer.
- `computeLineIftTransform(x)` -- the Inverse Fisher Transform of
  a scalar.
- `computeLineIftSma(values, period)` -- the moving average of a
  nullable series.
- `computeLineIftOscillator(closes, period, scale)` -- the
  scaled deviation of the close from its moving average.
- `computeLineIft(closes, period, scale)` -- the oscillator
  compressed through the transform.
- `runLineIft(points, options?)` -- sort finite points by `x`,
  run the pipeline, return `{ series, period, scale, threshold,
  oscillator, ift, samples, iftFinal, bullishCount,
  bearishCount, neutralCount, ok }`. Each sample is classified
  `bullish` / `bearish` / `neutral` / `none`. `ok` is false for
  fewer than two points.
- `computeLineIftLayout(options)` -- stack the price panel above
  an IFT panel on a fixed -1 to +1 scale; project the price
  path, the IFT path, one marker per defined bar, and the zero
  and trigger reference lines.
- `describeLineIftChart(data, options?)` -- ARIA description.

## Component

`ChartLineIft` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, two stacked panels
(price on top, IFT below) with their own axis ticks and grid,
panel labels, a config badge showing the period and the scale, a
three-series legend (Price / IFT / Levels) with toggle, a
hover/focus tooltip exposing the oscillator, the IFT and the
zone, and a `motion-safe:animate-fade-in` entrance. It consumes
`{ x, value }` points. The IFT panel is fixed to a -1 to +1
scale, draws dashed zero and trigger reference lines, and colours
each IFT marker by its zone. Every node carries a `data-section`
attribute.

## Distinctness

- **vs the oscillator panels (rmi / tdi / cmo)** -- those derive
  a new oscillator from the price. The IFT does not derive a
  signal; it is a non-linear compressor that sharpens an input
  oscillator toward the bounds.
- **vs chart-line-fisher** -- the Fisher Transform expands the
  tails of a bounded input (it is `0.5 * ln((1+x)/(1-x))`). The
  Inverse Fisher Transform is its inverse: it compresses an
  unbounded input back into bounds.
- **vs chart-line-zscore** -- a z-score is an unbounded linear
  normalisation. The IFT is a bounded non-linear one.

## Summary

New `ChartLineAroon` UI primitive at
`web/src/components/ui/chart-line-aroon.tsx` -- a pure-SVG line chart
with an Aroon panel plotting the Aroon-up and Aroon-down trend-age
lines. The value line occupies the top panel; the bottom panel renders
the two 0-100 Aroon lines with dashed reference levels at the
conventional weak/strong trend bands. Both panels share one x-axis.

## Indicator math

`computeLineAroon(values, period)` is Tushar Chande's Aroon. For each
index from `period` onward the window of `period + 1` values ending at
that index is scanned:

```
Aroon-up   = 100 * (period - periodsSinceHigh) / period
Aroon-down = 100 * (period - periodsSinceLow)  / period
```

`periodsSinceHigh` is the offset (0..period) of the window's maximum
from the current bar; `periodsSinceLow` likewise from the minimum. A
reading of 100 means the extreme is the current bar (a fresh high or
low); 0 means the extreme is the oldest bar in the window. On ties the
most recent occurrence wins, so a flat series reads both lines at 100.
Indices before the window fills read `null`.

The Aroon oscillator is `Aroon-up - Aroon-down`, ranging -100 to +100.

### Worked fixture

`values = [10, 14, 11, 12, 18, 9, 20, 8]`, `period = 4`:

```
i=4 window [10,14,11,12,18]: high@4 off 0 -> up 100; low@0 off 4 -> down 0
i=5 window [14,11,12,18,9] : high@4 off 1 -> up 75;  low@5 off 0 -> down 100
i=6 window [11,12,18,9,20] : high@6 off 0 -> up 100; low@5 off 1 -> down 75
i=7 window [12,18,9,20,8]  : high@6 off 1 -> up 75;  low@7 off 0 -> down 100

aroonUp    = [.,.,.,.,100, 75, 100, 75]
aroonDown  = [.,.,.,.,0,   100, 75, 100]
oscillator = [.,.,.,.,100, -25, 25, -25]
aroonUpFinal = 75, aroonDownFinal = 100, oscillatorFinal = -25
```

## API

- `getLineAroonFinitePoints(points)` -- drop points with a non-finite
  `x` or `value`.
- `normalizeLineAroonPeriod(period, fallback)` -- coerce a period to a
  positive integer; non-finite or sub-1 falls back, fractional floors.
- `computeLineAroon(values, period)` -- the
  `{ aroonUp, aroonDown }` series.
- `runLineAroon(points, options?)` -- sort finite points by `x`,
  compute the Aroon series plus the oscillator, return `{ series,
  period, aroonUp, aroonDown, oscillator, samples, aroonUpFinal,
  aroonDownFinal, oscillatorFinal, ok }`. `ok` is false for fewer than
  two points.
- `computeLineAroonLayout(options)` -- stack a value panel above a
  fixed 0-100 Aroon panel sharing one x-axis. Returns panels, ticks,
  the value path, Aroon-up / Aroon-down paths, markers, and the two
  projected reference level lines.
- `describeLineAroonChart(data, options?)` -- ARIA description string.

## Component

`ChartLineAroon` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge (period + final oscillator), a three-series legend (Value /
Aroon-up / Aroon-down) with toggle, two dashed reference level lines,
markers on the Aroon-up line, a hover tooltip exposing the value,
Aroon-up, Aroon-down, and oscillator, and a
`motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs chart-line-adx** -- the ADX measures trend *strength* from the
  magnitude of smoothed directional movement. Aroon measures trend
  *age* -- the count of periods since the window's extreme, a purely
  time-based reading that does not look at how far price moved.
- **vs chart-line-stochastic** -- the stochastic %K measures *where*
  the value sits within the high-low range (a position). Aroon
  measures *how long ago* the high or low occurred (an age). Both are
  0-100 and both involve window extremes, but one is a magnitude
  position and the other is an elapsed-time count.

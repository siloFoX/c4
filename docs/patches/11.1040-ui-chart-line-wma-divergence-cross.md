# 11.1040 -- UI: chart-line-wma-divergence-cross primitive (v1.11.1058)

## Scope

Add a pure-SVG dual-panel `ChartLineWmaDivergenceCross`
component that renders the close in the top panel and the
Weighted Moving Average (WMA) in the bottom panel, flagging
price-vs-WMA direction-disagreement events as recent-emphasis
trend reversal warnings with bias coloring derived from the
WMA slope at the trigger bar. Uses the canonical 5-state
divergence regime model (aligned-bullish / aligned-bearish /
divergent-bullish / divergent-bearish / none).

This is the linearly-weighted sibling of
`chart-line-sma-divergence-cross` (v1.11.1054, uniform-weight),
`chart-line-ema-divergence-cross` (v1.11.1053, exponential
weight), and `chart-line-hma-divergence-cross` (v1.11.1052,
Hull composite). The four MA-divergence-cross primitives
share the same 5-state classifier and transition-into-
divergent detector, but WMA's `(period - 1) / 3` centroid
lag is the fastest non-Hull variant -- roughly two-thirds
the lag of SMA / EMA -- making it the speed-tier sibling
without HMA's outer-window smoothing.

## WMA divergence pipeline

```
wma[i]      = WMA(close, period)
              ; weight 1 on the oldest bar through weight
              `period` on the newest, normalised by the
              triangular weight sum period * (period + 1) / 2

priceUp     : close[i] > close[i-1]
priceDown   : close[i] < close[i-1]
wmaUp       : wma[i]   > wma[i-1]
wmaDown     : wma[i]   < wma[i-1]

regime ->
  'aligned-bullish'   when priceUp   && wmaUp
  'aligned-bearish'   when priceDown && wmaDown
  'divergent-bullish' when priceDown && wmaUp
  'divergent-bearish' when priceUp   && wmaDown
  'none'              when either side flat or null

bullish (divergence trigger up) :
  prev regime !== 'divergent-bullish' &&
  cur regime === 'divergent-bullish'
bearish (divergence trigger down) :
  prev regime !== 'divergent-bearish' &&
  cur regime === 'divergent-bearish'

bias        : wma[i] vs wma[i-1] -> up / down / flat / none
```

Default `period = 14`. WMA's steady-state lag on linear
input is `(period - 1) / 3 = 13/3 ~= 4.333` -- roughly
two-thirds the SMA / EMA lag of `(period - 1) / 2 = 6.5`
for the same period -- so divergent events fire on price
patterns that the slower siblings would smooth over. The
trade-off: WMA is more sensitive to single-bar noise than
SMA / EMA, so the false-positive rate is higher.

Warmup is `period - 1 = 13` for the default tuning,
identical to the SMA / EMA siblings. Divergence detection
requires the previous WMA + close, so the first regime
classification lands at i = 14.

## Bit-exact anchors

- **CONST close = K**: every WMA window weighted-averages
  K -> WMA = K. Both close and WMA are flat -> regime
  `none` for every bar. 0 divergence crosses. Verified
  across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP close = i**: WMA over linear input settles
  at `i - (period - 1) / 3 = i - 13/3`. WMA[i] -
  WMA[i-1] = +1 (up); close[i] - close[i-1] = +1 (up).
  Both up -> regime `aligned-bullish`. 0 divergence
  crosses.
- **LINEAR DOWN close = -i**: mirror -> WMA = `-i + 13/3`.
  Both close and WMA falling -> regime `aligned-bearish`.
  0 divergence crosses.

The clean `i - 13/3` lag on LINEAR input is the textbook
WMA centroid identity -- a direct consequence of the
triangular-weight normalisation: the weighted-average
index of a window `[i - period + 1, i]` with weights
`[1, 2, ..., period]` is `i - (period - 1) / 3`.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 5-state regime classifier follows the
  chart-line-*-divergence-cross family convention; the
  `bullish` / `bearish` cross *kinds* still follow the
  sibling cross-sig convention (line direction).
- WMA centroid lag `(period - 1) / 3` is shorter than SMA
  / EMA `(period - 1) / 2`, so this primitive surfaces
  divergence patterns earlier but with more false
  positives. It is complementary to the SMA / EMA / HMA
  siblings, not a strict replacement -- use the slower
  centerlines for confirmation and WMA for early
  warning.
- Cross detection fires only on transition INTO a
  divergent state (not while staying in one), preventing
  double-fire on consecutive bars in the same regime.
- WMA's triangular-weight construction uses the canonical
  closed-form normaliser `period * (period + 1) / 2`
  rather than recursive accumulation, matching the
  chart-line-wma-cross-sig sibling for bit-exact ink
  consistency when both primitives ship in the same
  dashboard.

# TODO 8.20 part B — Web UI pages covering CLI features

**Status:** done
**Branch:** `c4/ui-cli-coverage`
**Scope:** Wrap the CLI feature surface that was CLI-only in 1.11.x with
a first-class Web UI, so operators can drive C4 from a browser without
dropping to `c4 <cmd>` for everyday flows. Coordinated with the parallel
`ui-settings` and `jsonl-view` workers — this patch does not touch the
Settings / Dashboard / Command Palette area that `ui-settings` owns, nor
the Sessions tab that `jsonl-view` owns.

## Summary

A new **Features** top-tab mounts a dedicated sidebar layout with pages
grouped into five categories (Operations / Cost / Automation / Config /
Diagnostics). Pages are lazy-loaded through a registry so the main
bundle stays small; the current build ships them as 12 code-split
chunks, each in the 2–6 KB gzipped range.

Route pattern: `#/feature/<id>`. Selection is persisted to
`localStorage` (`c4.features.selected`) and reflected in the URL hash so
deep links work.

## Worker-detail actions

- **Status-to-Slack** is added as a new `StatusMessageCard` sibling on

  `{ worker, message }`. Oncall handoffs no longer require a terminal
  round-trip.

## New daemon endpoint

`POST /batch` accepts one of two shapes:

```json
{ "tasks": ["first task", "second task"], "namePrefix": "batch" }
{ "task": "same task", "count": 5, "branch": "feature" }
```

Optional flags: `branch`, `profile`, `autoMode`, `target`, `namePrefix`.

The route sits behind the existing `rbac.ACTIONS.WORKER_TASK` gate and
rejects missing input with HTTP 400. Each task is dispatched through
`manager.sendTask(name, task, opts)` and per-item outcomes are returned
as `{ ok, fail, total, results: [{ name, ok, error? }] }` so the UI can
render a results table without fanning out N `/task` calls itself.

## Shared helpers

- `web/src/lib/format.js` + `.d.ts` — formatNumber / formatBytes /
  formatDuration / formatRelativeTime / formatTimestamp / dateRange /
  dateRangeLabel. Plain JS + JSDoc types so tests can import directly
  (Node 20 has no TS strip).
- `web/src/lib/fuzzyFilter.js` + `.d.ts` — substring-ranking filter
  used by Templates / Profiles / Validation.
- `web/src/lib/markdown.tsx` — minimal markdown renderer covering the
  subset the daemon emits (ATX headings, fenced code, lists, blockquote,
  inline code, bold, italic, links). Keeps Plan + Morning rendering
  without pulling in `react-markdown` as a new runtime dep.

## Tests

`tests/ui-cli-coverage.test.js` — 56 assertions across 15 suites:

- Pure behaviour tests for `format.formatNumber / formatBytes /
  formatDuration / formatRelativeTime / dateRange / dateRangeLabel` and
  `fuzzyFilter.fuzzyScore / fuzzyFilter` including empty / NaN / prefix
  ranking / combined filter cases.
- Source-wiring tests for the new `POST /batch` route — RBAC gate,
  tasks[]-or-count body shape, namePrefix default, HTTP 400 on missing
  input, sendTask dispatch, per-item result shape.
- Source-wiring for Features tab (`TopTabs` entry, `App.tsx` mount),
  the page registry (every id + category ordering + lazy-loader count),
  Batch / Plan / TokenUsage component wiring (per spec — component
  tests for at least these three), and the new ControlPanel
  `StatusMessageCard`.

Full suite: **101 / 101 pass**. Web build clean (`tsc --noEmit && vite
build`).

## Unfinished stubs

The following actions toast "not implemented yet" and are tracked as
sub-TODOs in `TODO.md`:

- **Templates** — add / edit / remove (GET `/templates` exists; POST/PUT/
  DELETE do not).
- **Profiles** — add / edit / remove (same story as Templates).
- **Health** — event-loop lag and loaded-modules fields. The
  `/health` route today only returns the basic heartbeat, so the UI
  renders `-` for those two stats and calls out the gap.

All three are contained: flip the daemon endpoints on and the UI
already has the buttons wired to `notImplemented()` toasts — the
replacement is a one-line swap.

# 11.1066 -- UI: chart-line-adx-neg-cross primitive (v1.11.1084)

## Scope

Add a pure-SVG dual-panel `ChartLineAdxNegCross` component
that renders the close in the top panel and the Average
Directional Index -DI directional indicator in the bottom
panel, marking bearish (-DI crosses up through zero --
downtrend strength trigger; bearish directional pressure
emerging from indecision) -DI-vs-zero crossover trigger
events with bias coloring derived from the -DI slope at
the trigger bar.

Mirror of the chart-line-adx-pos-cross v1.11.1083 sibling
but watches -DI. Critically, the **semantics are inverted**:
when -DI rises, downtrend pressure is increasing, which is
**bearish** for price. The detector therefore maps:

- `kind: 'bearish'` = -DI crosses up through 0 (downtrend
  emerging, price-bearish)
- `kind: 'bullish'` = -DI crosses down through 0
  (downtrend fading; never fires since -DI ∈ [0, 100])

This deliberately deviates from the indicator-perspective
convention used elsewhere in the cross-sig family (where
"bullish" always = up-cross of the indicator). The
trade-off: trader-facing semantics remain consistent
(every "bullish" cross in this primitive is price-bullish)
at the cost of detector-shape asymmetry with the +DI
sibling. Documented in source.

## -DI zero-cross pipeline

```
upMove[i]    = high[i] - high[i-1]
downMove[i]  = low[i-1] - low[i]
-DM[i]       = (downMove > upMove && downMove > 0)
               ? downMove : 0
tr[i]        = max(high - low,
                   |high - close[i-1]|,
                   |low  - close[i-1]|)
smMinus[i]   = SMA(-DM, period)
smTr[i]      = SMA(tr,  period)
minusDI[i]   = (smTr > 0) ? 100 * smMinus / smTr : 0

bearish (downtrend-emerging trigger) :
  prev minusDI <= 0 && cur minusDI > 0
bullish (downtrend-fading trigger) :
  prev minusDI >= 0 && cur minusDI < 0
  (effectively never fires since minusDI >= 0 always)

regime       : 'bearish' when minusDI > 0
               'bullish' when minusDI < 0 (never)
               'none'    when minusDI === 0 or null
bias         : minusDI[i] vs minusDI[i-1] -> up / down /
               flat / none
```

Default `period = 14` -- canonical Wilder ADX tuning.

## Sibling family

The ADX signal family now has four primitives:

- **chart-line-adx-trend-cross v1.11.1047** -- ADX
  itself vs the 20/25 threshold (trend strength
  emerging)
- **chart-line-adx-pos-neg-divergence v1.11.1075** --
  compares +DI direction vs -DI direction
- **chart-line-adx-pos-cross v1.11.1083** -- +DI alone
  vs zero (uptrend strength emerging)
- **chart-line-adx-neg-cross v1.11.1084** (this
  primitive) -- -DI alone vs zero (downtrend strength
  emerging)

The pos/neg pair complete the simple-zero-cross variants
of the ADX family. Each surfaces a different signal niche.

Uses SMA-based smoothing (matching the adx-* family
convention) rather than Wilder's exponential smoothing,
for bit-exact integer DI values on linear input.

Warmup is `period + 1 = 15` for the default tuning: DM
and TR seed at i = 1, SMA window-fills at i = period =
14, and cross detection needs the previous bar's -DI,
so the first potential cross lands at i = period + 1 =
15.

## Bit-exact anchors

All anchors use HLC input.

- **CONST band** `high = K + 1`, `low = K - 1`, `close
  = K`: upMove = 0, downMove = 0 -> -DM = 0 (downMove >
  0 check fails). TR = 2. smMinus = 0, smTr = 2.
  minusDI = 100 * 0 / 2 = 0. minusDI stays at 0 -> no
  transition above 0. 0 crosses. regime `none` (minusDI
  === 0). Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close
  = i`: upMove = +1, downMove = -1. -DM = 0 (downMove
  > 0 fails). minusDI = 0 (constant). 0 crosses.
  regime `none`.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`,
  `close = -i`: upMove = -1, downMove = +1. -DM = 1.
  TR = 2. smMinus = 1, smTr = 2. minusDI = 50 (constant
  from i = period). prev minusDI at i = period - 1 is
  null (warmup), so cross detector skips i = period.
  From i = period + 1 onwards minusDI === 50
  (constant), no transition. 0 crosses. regime
  `bearish` (minusDI > 0; downtrend pressure active).

All three steady-state anchors produce zero crosses
because -DI is constant (either 0 or 50). Real-world
crosses fire when -DI transitions from exactly 0 to
positive -- typically the first bar of a downtrend
emerging from a flat/range market. The synthetic
detect-function test verifies this with a hand-built
`[0, 0, 15, 30]` sequence -- and crucially verifies the
trigger is labeled `bearish` (the inverted semantics).

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- **Inverted semantics**: this primitive deliberately
  swaps the cross-detection labels so that "bearish" =
  -DI rising (downtrend emerging, price-bearish) and
  "bullish" = -DI falling (downtrend fading, never
  fires). Maintains trader-facing semantic consistency
  with the +DI sibling at the cost of detector-shape
  asymmetry. Documented in source and in the test
  suite (`detectLineAdxNegCrossCrosses (inverted
  semantics)`).
- 3-state regime classifier with `-DI === 0 -> none`
  semantics (matching +DI sibling): zero literally
  means no movement, not bullish/bearish-zone.
- Asymmetric usage: in practice the bullish cross
  never fires (-DI cannot go below 0). The bullish
  path in the detector is retained for family-shape
  symmetry but is effectively dead code on real
  inputs. Documented as a deliberate trade-off.
- SMA-based smoothing (not Wilder) for bit-exact
  testability. Documented as a family-level tuning
  choice.
- On steady LINEAR input -DI saturates at either 0
  (no bearish movement) or 50 (pure bearish movement)
  with flat slope -- no zero-crosses fire. Real-world
  crosses fire on the first bar where -DI rises above
  exactly 0, which corresponds to the first bar of a
  downtrend emerging from a flat range.
- Completes the simple-zero-cross pair with
  chart-line-adx-pos-cross v1.11.1083. Together they
  provide independent uptrend/downtrend emergence
  detection alongside the trend-strength (adx-trend-
  cross) and direction-divergence (adx-pos-neg-
  divergence) variants.

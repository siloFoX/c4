# 11.808 UI: chart-line-coppock-trigger primitive (v1.11.826)

## Summary
Pure-SVG dual-panel React/TS primitive that plots the **Coppock Curve**
with emphasis on zero-line crossings as actionable triggers. Trigger
variant of `<ChartLineCoppockSignal />` (11.805) -- same math, different
visual emphasis: hard bullish/bearish fill split at the zero line,
triangle markers on every cross (upward for bullish, downward for
bearish), and an explicit `triggers: Array<Event>` list on the run for
downstream consumption.

## Formula
Identical to coppock-signal (11.805):
```
ROC(L)[i]   = (close[i] - close[i - L]) / close[i - L] * 100
sumROC[i]   = ROC(longROC)[i] + ROC(shortROC)[i]
coppock[i]  = WMA(sumROC, wmaPeriod)[i]
```

## Defaults
- `longROC = 14`, `shortROC = 11`, `wmaPeriod = 10`
- `triangleSize = 6`, `width = 720`, `height = 460`, `panelGap = 12`

## Visual differences vs coppock-signal
- **Hard bullish/bearish split**: a closed polygon traces the coppock
  curve and closes along the zero baseline; its fill uses a
  user-space `linearGradient` with two stop pairs at the
  `zeroSplitOffset` so the upper half tints bullish-green and the
  lower half tints bearish-red.
- **Triangle markers**: cross events render as SVG triangle paths
  (upward = bullish, downward = bearish) instead of circles. The path
  `d` string is precomputed in the layout for testability.
- **Explicit triggers list**: `run.triggers` is an array of
  `{ index, x, close, coppock, kind }` events, suitable for chaining
  into an alert table or follow-up callback. `bullishTriggerCount` /
  `bearishTriggerCount` are surfaced as data attributes.
- **`onTriggerClick` callback**: marker clicks fire both
  `onPointClick` and `onTriggerClick` with the event payload.

## Bit-exact testability anchors
- **CONST close** (`close == K`, `K != 0`): coppock = 0 -> no triggers
  fire. Run yields `bullishTriggerCount = 0`, `bearishTriggerCount =
  0`, `triggers = []` across `K in {1, 5, 100, -3}` and `(long, short,
  wma)` sweep.
- **GEOMETRIC close = 2^k**: coppock = 1843000 with defaults (proven
  in 11.805); signal stays positive after warmup -> zero triggers.
- `detectLineCoppockTriggerEvents` directly tested with synthetic
  sequences (`[null, -1, 1] -> 'bullish'`, `[null, 1, -1] ->
  'bearish'`, `[null, 0, 1] -> 'bullish'`, etc.).

## Design notes
- The hard-split fill is implemented via `linearGradient` with two
  pairs of stops at the same `offset` (the zero-line position
  normalized to [0, 1] within the coppock panel). This produces a
  clean color change at the zero line without per-bar clip paths.
- Triangle paths use absolute pixel coordinates (`.toFixed(2)`) so the
  rendered output is deterministic and snapshot-friendly.
- Triggers fire on the first bar of the new sign; `prev <= 0 -> v > 0`
  is bullish, `prev >= 0 -> v < 0` is bearish (handles zero-to-sign
  transitions explicitly).
- `noUncheckedIndexedAccess` satisfied via explicit `?? null` /
  `?? 'none'` and defensive guards.

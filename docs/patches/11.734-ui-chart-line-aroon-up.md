## Summary

`web/src/components/ui/chart-line-aroon-up.tsx` ships
`<ChartLineAroonUp>`, a pure-SVG two-panel chart with an **Aroon Up**
panel from the distance to the most recent high scaled to the
lookback window. The top panel plots the bar high; the bottom panel
plots the Aroon Up reading in a fixed `[0, 100]` band with dashed
horizontal reference lines at the high and low thresholds.

## Computation

For each bar `i`, the window of `period + 1` bars `[i - period .. i]`
is scanned for the highest high. The most-recent bar wins on a tie
(strict `>`). The Aroon Up scales the bars-since-the-high to a
percent:

```
barsSinceHigh = i - argmax(high over [i - period .. i])
aroonUp       = 100 * (period - barsSinceHigh) / period
```

A new high pushes Aroon Up to 100; a high made `period` bars ago
drops it to 0. The reading is bounded to `[0, 100]`. Conventional
reference lines sit at 70 (strong uptrend) and 30 (weak uptrend).

## Worked anchors

The pipeline is pure rational arithmetic, no transcendentals --
integer fixtures compute to exact integer values. At period 4 every
reading is a multiple of 25.

- **Rising series** -- highs `[1..10]`. At every bar `i >= 4`, the
  highest high is the CURRENT bar (highs are strictly increasing),
  so `barsSinceHigh = 0` and `aroonUp = 100`. Asserted with
  `toBe(100)` for every defined bar.
- **Falling series** -- highs `[10..1]`. At every defined bar the
  highest is the OLDEST bar in the window, so `barsSinceHigh =
  period` and `aroonUp = 0`. Asserted with `toBe(0)`.
- **Constant series** -- highs `[5 x 10]`. Every bar ties; the
  most-recent-tie rule (strict `>` keeps the earliest scan position
  set to the current bar) pins `barsSinceHigh = 0` -> `aroonUp =
  100`. Asserted with `toBe(100)`.
- **Worked peak fixture** -- highs `[1, 2, 3, 4, 5, 6, 5, 4, 3, 2]`
  at period 4. The high peaks at bar 5; the window scan from
  `i = 5` onward locks the high at bar 5 while `i` advances. Step
  by step: `bar 4 -> aroonUp=100`, `bar 5 -> 100`, `bar 6 ->
  barsSince=1 -> 75`, `bar 7 -> 50`, `bar 8 -> 25`, `bar 9 -> 0`.
  Asserted with `toEqual([null x4, 100, 100, 75, 50, 25, 0])`.
- **`[0, 100]` bound** -- every defined value on a wave fixture is
  in the bound.

## Distinct from neighbouring primitives

- vs `<ChartLineAroonOsc>` -- the Aroon Oscillator is `Aroon Up -
  Aroon Down` in `[-100, 100]`; this primitive isolates the Aroon
  Up half in `[0, 100]` with its own threshold band.
- vs `<ChartLineRsi>` -- the RSI is `100 - 100 / (1 + sumUp /
  sumDown)` over the bar-to-bar price changes; Aroon Up measures
  the RECENCY of the highest high.
- vs `<ChartLineStochastic>` -- the Stochastic ranks the current
  close inside the lookback's range; Aroon Up ranks the TIME of
  the high.

## API

`data` -- a single `ChartLineAroonUpPoint[]` (`{x, high}`). `period`
(default 25, integer >= 1). `thresholdHigh` (default 70, in
`[0, 100]`). `thresholdLow` (default 30, in `[0, 100]`). Standard
sizing / colour / a11y props plus `pricePanelRatio` (default 0.55,
clamped to `[0.3, 0.8]`), `gap`, `strokeWidth`, `dotRadius`,
`tickCount`. Toggle flags: `showAxis`, `showGrid`, `showDots`
(default false), `showAroonUp`, `showThresholdLines`, `showMarkers`,
`showTooltip`, `showConfigBadge`, `showLegend`, `animate`.
Controlled and uncontrolled visibility via `hiddenSeries`
(`'price' | 'aroonUp'`). Callbacks: `onPointClick({point})`,
`onSeriesToggle({seriesId, hidden})`. Format functions:
`formatValue`, `formatX`. ARIA: `ariaLabel`, `ariaDescription`.

## Pure helpers

- `DEFAULT_CHART_LINE_AROON_UP_*` -- defaults (period 25,
  thresholds 70 / 30, etc.).
- `CHART_LINE_AROON_UP_MIN/MAX` -- 0 and 100 (the panel bounds).
- `getLineAroonUpFinitePoints` -- keep only points with finite x
  and high.
- `normalizeLineAroonUpPeriod(period, fallback)` -- integer >= 1;
  else fallback.
- `normalizeLineAroonUpThreshold(threshold, fallback)` -- finite
  in `[0, 100]`; else fallback.
- `computeLineAroonUp(bars, period)` -- per-bar Aroon Up; warm-up
  null; the most-recent bar wins on a tie.
- `classifyLineAroonUpZone(value, thresholdHigh, thresholdLow)` --
  `'strong' | 'mid' | 'weak' | 'none'`.
- `runLineAroonUp(points, options?)` --
  `{series, period, thresholdHigh, thresholdLow, aroonUp, samples,
  aroonUpFinal, strongCount, midCount, weakCount, ok}`.
- `computeLineAroonUpLayout({data, period, thresholdHigh,
  thresholdLow, ...})` -- two panels with the high line on top,
  the Aroon Up line below in a fixed `[-5, 105]` band, one marker
  per defined-aroon bar coloured by zone, plus `thresholdHighY`
  and `thresholdLowY` for the reference lines.
- `describeLineAroonUpChart(data, options?)` -- ARIA summary
  mentioning the period, the threshold band, the new-high rule and
  the zone counts.

## a11y / data hooks

Root `<div role="region">` with `aria-label` + `aria-describedby`; the
SVG is `role="img"` with the same label; an sr-only
`data-section="chart-line-aroon-up-aria-desc"` span carries the
description. The high line, every price dot, the Aroon Up line and
every marker carry `role="graphics-symbol"` and `tabIndex={0}`.
Every node carries a `data-section`. Root mirrors `data-period`,
`data-threshold-high`, `data-threshold-low`, `data-aroon-up-final`,
`data-strong-count`, `data-mid-count`, `data-weak-count`,
`data-total-points`, `data-animate`, `data-empty`. Grid / axis /
panel-label nodes carry `data-panel="price"|"aroon"`. Threshold
lines carry `data-direction="high"|"low"`. Markers expose
`data-zone` and `data-aroon-up`.

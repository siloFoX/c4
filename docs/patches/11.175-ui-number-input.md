# 11.175 - UI NumberInput primitive (v1.11.193)

New `<NumberInput>` primitive plus two adoption sites.

## Files

- `web/src/components/ui/number-input.tsx` - new primitive.
- `web/src/components/ui/number-input.test.tsx` - vitest + RTL + userEvent suite (>= 12 cases).
- `web/src/components/ui/index.ts` - re-exports `NumberInput`.
- `web/src/pages/Risk.tsx` - swaps the `windowHours` `<Input type='number'>`
  for `<NumberInput unit='h' size='sm'>`. Test updated to find the input via
  `getByLabelText('window:')` since the inner element is now
  `type='text' inputmode='numeric'` (no `role='spinbutton'`).
- `web/src/pages/Health.tsx` - new Settings sub-panel with a placeholder
  poll-timeout `NumberInput` (min 500, max 60000, step 500, unit `ms`,
  size `sm`) bound to local state. Not yet wired into `useHealth` -
  follow-up item.

## Adoption sites

1. **Risk page - recent-denials window** (`web/src/pages/Risk.tsx`,
   inside the *Recent denials* panel). The previous `<Input type='number'
   min={1} max={720}>` is replaced by the new primitive; clamp logic stays
   in the page handler (`Math.max(1, Math.min(720, next ?? 24))`).

2. **Health page - poll-timeout placeholder**
   (`web/src/pages/Health.tsx`, new *Settings* sub-panel above the
   dashboard rating row). Demonstrates adoption since the page had no
   pre-existing numeric setting. State (`pollTimeoutMs`) is local; a
   follow-up will thread it into `useHealth`.

## Behavior summary

- Controlled `value: number | undefined`; `undefined` = empty input.
- Steppers clamp into `[min, max]` and disable at the bounds.
- ArrowUp / ArrowDown step from inside the input.
- Stepping from `undefined` starts at `min ?? 0`.
- Blur clamps to `[min, max]` and applies `toFixed(precision)` when set.
- Empty input fires `onChange(undefined)`.
- Optional `unit` renders as an inline suffix span (aria-hidden).
- `size`: `'sm'` (h-7) or `'md'` (h-10, default).
- `forwardRef` exposes the underlying `<input>` element.

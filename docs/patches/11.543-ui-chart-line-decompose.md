# 11.543 -- web/src/components/ui/chart-line-decompose.tsx

Pure-SVG **additive seasonal decomposition** chart. Takes one
time series and splits it into three coordinated components:

- **Trend**: centered moving average of window=period
  (handles even periods with the canonical 2xN half-weight
  formula).
- **Seasonal**: per-phase average of the detrended values,
  then centered so the seasonal pattern has zero mean
  (additive convention).
- **Residual**: observed - trend - seasonal.

These are rendered as a stacked four-panel chart (Observed /
Trend / Seasonal / Residual) sharing the x-axis, with
per-panel y-scales and dashed zero reference lines on the
seasonal + residual panels. The seasonal and residual y ranges
are pinned symmetric around zero when both signs are present
so the reference line lands in the middle of the panel.

## Algorithm

For points `(x_i, y_i)` sorted ascending by x and period `p`:

1. `trend[i] = centered MA of y over window p`
   - odd p: `mean(y[i-half..i+half])` where `half = floor(p/2)`
   - even p: `(0.5*y[i-half] + y[i-half+1] + ... + y[i+half-1] + 0.5*y[i+half]) / p`
   - edges (where the window is not yet filled) -> null.
2. `detrended[i] = y[i] - trend[i]` (null where trend is
   null).
3. `seasonalPattern[phase]` = mean of detrended values where
   `i % p === phase`, then subtract the mean of the pattern
   so it has zero mean.
4. `seasonal[i] = seasonalPattern[i % p]`.
5. `residual[i] = y[i] - trend[i] - seasonal[i]` where trend
   is valid, else null.

This matches pandas' `seasonal_decompose(model='additive',
period=p)` output (centered moving-average convention).

## Distinct from related primitives

- `<ChartLineMovingAvg>` (11.513): SMA only -- decompose
  extracts THE trend component but ALSO produces seasonal +
  residual splits.
- `<ChartLineResidual>` (11.525): observed minus EXTERNALLY-
  supplied predicted. Decompose derives the predicted (trend
  + seasonal) internally from the same input series.
- `<ChartLineFft>` (11.540): frequency-domain transform.
  Decompose stays in time domain and extracts ONE
  pre-specified period (vs FFT's full spectrum). Use FFT to
  IDENTIFY the dominant period, then decompose to SPLIT the
  series around that period.
- `<ChartLineBollinger>` (11.537): rolling SMA + envelope.
  Decompose is a 4-panel split, not an envelope.
- `<ChartLineKalman>` (11.538): recursive Bayesian filter.
  Decompose is a deterministic windowed decomposition.

The canonical statsmodels / pandas `seasonal_decompose` 4-pane
output -- used for visual diagnosis of seasonality vs trend
vs noise in any periodic time series (sales, weather, web
traffic, etc.).

## Public surface

- `<ChartLineDecompose data={...} />`.
- `data: ChartLineDecomposePoint[]` -- `{x, y}`.
- `period` (default `12`; clamped to `>= 2`) -- season length
  in samples (e.g. 12 for monthly with yearly seasonality).
- `observedColor` / `trendColor` / `seasonalColor` /
  `residualColor` -- per-panel line color.
- Toggle flags: `showAxis`, `showGrid`, `showDots` (default
  false; the dense paths are usually enough), `showLegend`,
  `showTooltip`, `showPeriodBadge`, `showResidualZero`,
  `animate`.
- Controlled + uncontrolled component visibility via
  `hiddenComponents` / `defaultHiddenComponents` /
  `onHiddenComponentsChange` -- toggle individual panels.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX`.
- `onPointClick({panel, point, sample})` -- fires on dot
  hover-click; payload includes the full per-sample
  decomposition.

## Pure helpers

- `LINE_DECOMPOSE_COMPONENT_KINDS` -- `['observed', 'trend',
  'seasonal', 'residual']`.
- `getLineDecomposeFinitePoints(points)` -- finite-only
  filter.
- `normaliseLineDecomposePeriod(value)` -- clamps to `>= 2`;
  floors fractional; non-finite -> default 12.
- `computeCenteredMovingAverage(values, period)` -- canonical
  centered MA. Handles odd and even periods (even uses the
  half-weight endpoints formula). Edges are null. Non-finite
  values skipped from the window sum.
- `computeLineDecomposeSeasonalPattern(detrended, period)` --
  returns length-`period` array of phase means with the
  overall mean subtracted (centers the pattern). Skips null
  entries.
- `computeLineDecomposition(points, period)` -- canonical
  decomposition. Sorts ascending; drops non-finite; returns
  `{samples, period, trendValidCount, residualValidCount,
  seasonalPattern, ok}`. Each sample carries `{index, x,
  observed, trend, seasonal, residual, phase}`. Reconstructs
  `observed = trend + seasonal + residual` exactly where
  trend is valid (verified by test).
- `computeLineDecomposeLayout({data, ...})` -- stacks the
  enabled panels evenly with `gap` separators; returns
  `{ok, width, height, panels, panelMap, xTicks, xMin, xMax,
  innerWidth, innerHeight, totalPoints, finiteCount, period,
  decomposition}`. Each panel projects its own y range
  (seasonal / residual symmetric around 0 when both signs
  present). Per-panel `points[i].py` is null where the
  decomposition is null (trend edges).
- `describeLineDecomposeChart(data, options?)` -- screen-
  reader description.

## Tooltip

Hovering a per-panel dot shows the panel label in its colour,
x, and the full breakdown: `obs:`, `trend:` (n/a at edges),
`seasonal:`, `residual:` (n/a at edges), and `phase:` (the
index within the period).

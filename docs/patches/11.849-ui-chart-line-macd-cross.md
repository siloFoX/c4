## Summary

Added `<ChartLineMacdCross />` -- a pure-SVG, dual-panel React/TS
primitive that renders a close line in the top panel and a
MACD-vs-signal crossover oscillator in the bottom panel. Scatter
markers fire at every signal cross and are tinted by the zero-line
bias (whether MACD is positive, negative, or exactly zero at the
cross bar). Defaults: classic MACD `fastLength = 12`,
`slowLength = 26`, `signalLength = 9`; macdColor `#0ea5e9`
(sky), signalColor `#f97316` (orange), bullish marker `#16a34a`,
bearish marker `#dc2626`, neutral marker `#475569`. Zero line
drawn for visual reference; the oscillator y-axis is clamped to
include zero.

## Formula

```
ema1   = EMA(close, fastLength)
ema2   = EMA(close, slowLength)
macd   = ema1 - ema2
signal = EMA(macd, signalLength)
```

Cross events fire on the bar where the strict inequality between
`macd` and `signal` newly flips (prev-relation guard +
strict-inequality on the current side keeps repeated samples from
double-firing). The bias for a cross is:

- `bullish` when `macd > 0` at the cross bar
- `bearish` when `macd < 0` at the cross bar
- `neutral` when `macd === 0` at the cross bar

## Bit-exact anchors

The implementation reuses the SMA-seeded EMA helper with the
`min === max` precision fix from 11.831 TRIX, plus the
`next = v === smoothed ? v : alpha*v + (1-alpha)*prev` CONST short-
circuit, so degenerate inputs land bit-exactly:

- **CONST close = K**: every EMA equals K, so `macd = 0` everywhere
  after warmup, then `signal = EMA(0, signalLength) = 0`. The
  relation is always `equal` so zero crosses fire and the bias is
  `neutral` for every eligible bar.
- **LINEAR UP / DOWN**: both fast and slow EMAs eventually settle
  to the same lag from the line, so `macd` becomes a constant
  offset and `signal = EMA(constant)` converges back to the same
  constant. Once they agree, the relation is `equal` and zero
  crosses fire from that point onward. The integration test
  verifies this near the tail of a 120-bar linear series.

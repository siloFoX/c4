# 11.567 -- web/src/components/ui/chart-line-heatline.tsx

Pure-SVG **heatline**: a single-series line chart whose
stroke is colour-graded by value along its length. Instead
of one solid stroke colour, the line is split into per-pair
segments and each segment is painted with a colour sampled
from a value->colour scale, so the line's hue encodes
magnitude alongside its y-position -- peaks read hot, troughs
read cool.

## Colour grading

The line is rendered as N solid-colour segments (one per
consecutive pair of points), not a single SVG gradient --
this keeps the rendering deterministic and works for any
line shape.

- `parseHexColor(hex)` parses a 6-digit `#rrggbb` string into
  RGB channels.
- `interpolateLineHeatlineColor(scale, t)` samples a
  multi-stop colour scale at normalized position `t`. The
  stops are spread evenly across [0, 1]; `t` selects a pair
  of surrounding stops and the RGB channels are linearly
  interpolated. `t` is clamped to [0, 1]; a non-finite `t`
  falls back to 0.
- `normalizeLineHeatlineValue(value, min, max)` maps a value
  to `t`. A zero-width domain collapses every point to the
  scale midpoint (0.5).
- `runLineHeatline(points, options?)` grades every point by
  its value and every **segment** by the midpoint value of
  its two endpoints, so the colour of a segment reflects the
  values it spans.

The colour `domain` defaults to the data's value range but
can be pinned via `domainMin` / `domainMax` so the same
scale is comparable across charts; an inverted domain is
swapped.

The component fixture makes the grading exact: values
`[0, 25, 50, 75, 100]` over the default 5-stop scale give
point `t` values `[0, .25, .5, .75, 1]` -- each landing
directly on a scale stop -- and segment midpoint `t` values
`[.125, .375, .625, .875]`.

## Distinct from neighbouring primitives

- `chart-line-area-gradient` applies a gradient **fill** to
  the area beneath the line (a vertical area gradient). The
  heatline grades the **stroke itself**, segment by segment,
  along the line's length.
- `chart-line` / `chart-line-marker` draw a single solid
  stroke colour.
- `chart-heatmap` / `chart-calendar-heatmap` are 2D grid
  heatmaps -- a different geometry entirely.

All exported helpers carry a `LineHeatline` infix (and the
shared `parseHexColor`) so the `export *` barrel line cannot
collide.

## Public surface

- `<ChartLineHeatline data={...} />` -- a single
  `ChartLineHeatlinePoint[]` (`{x, value}`); the heatline is
  a single-series transformation.
- `colorScale` -- a `>= 1`-stop array of `#rrggbb` strings
  (default: a 5-stop deep-blue -> teal -> green -> amber ->
  red ramp).
- `domainMin` / `domainMax` -- pin the colour domain
  (default: the data's value range).
- Toggle flags: `showAxis`, `showGrid`, `showDots` (default
  true), `showScale` (the colour legend), `showTooltip`,
  `showConfigBadge`, `animate`.
- Standard sizing + format + a11y props. `formatValue` /
  `formatX`.
- `onPointClick({point})`.

## Pure helpers

- `getLineHeatlineFinitePoints(points)`.
- `parseHexColor(hex)`.
- `interpolateLineHeatlineColor(scale, t)`.
- `normalizeLineHeatlineValue(value, min, max)`.
- `runLineHeatline(points, options?)` -- per-point + per-
  segment colour grading; reports the value range and the
  resolved colour domain.
- `computeLineHeatlineLayout({...})` -- projects the points
  and builds the per-segment paths.
- `describeLineHeatlineChart(data, options?)`.

## Tooltip / scale legend

Hovering a dot shows a colour swatch, x, the bold value, and
the level (`t` as a percentage). Config badge:
`HEAT lo=<domainMin> hi=<domainMax>`. The colour scale
legend below the chart shows one swatch per scale stop
flanked by the domain min / max labels.

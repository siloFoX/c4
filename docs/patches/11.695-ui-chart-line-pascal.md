## Summary

New `ChartLinePascal` UI primitive in
`web/src/components/ui/chart-line-pascal.tsx` -- a pure-SVG single-panel
line chart with a **Pascal Triangle Moving Average** (PMA) overlay
weighting the lookback window by binomial coefficients. The price line and
the PMA line share one panel.

## The computation

The Pascal Triangle Moving Average weights its lookback window by the
binomial coefficients of a Pascal's triangle row: for a window of `period`
values the weights are `C(period - 1, k)`, k = 0..period-1. Those
coefficients form a symmetric bell that peaks in the middle of the window
and approaches a Gaussian as the window grows. It is a close-only
indicator, so `ChartLinePascalPoint` carries `{x, value}`.

- `computeLinePascalWeights(period)` -- row `period - 1` of Pascal's
  triangle, built by the additive Pascal recurrence
  (`row[k] = prev[k-1] + prev[k]`), so every weight is an exact integer.
- `computeLinePascal(values, period)` -- the PMA: the binomial-weighted
  average of each full window, `sum(weight[k] * value[k]) / 2^(period-1)`.
  The row sums to `2^(period-1)`, a power of two, so dividing by it is
  exact. A window with a non-finite value yields null; a period of 1 is
  the identity.

Each defined bar is classified by where the PMA sits relative to the
price: **above**, **below** or **equal**.

## Worked fixture

The Pascal MA is the rare moving average that is fully exact: integer
binomial weights and a power-of-two normalizer mean dividing the weighted
sum by `2^(period-1)` introduces no rounding. The whole PMA series is
asserted with `toEqual`.

The fixture is an 8-bar series read with period 3. The period-3 weights
are `[1,2,1]` and the normalizer is 4, so the PMA is
`(v[i-2] + 2*v[i-1] + v[i]) / 4`. The values `[16,16,16,32,8,24,40,8]` are
picked so every weighted sum is a multiple of 4 -- the PMA is
`[null,null,16,20,22,18,24,28]`. A separate test checks a dyadic fraction
(`(10 + 2*10 + 11)/4 = 10.25`) to show the power-of-two division is exact
for non-integer results too. The weight rows are checked directly:
`[1]`, `[1,1]`, `[1,2,1]`, `[1,3,3,1]`, `[1,4,6,4,1]`; the rows are
symmetric and sum to a power of two. A period of 1 is the identity, and a
constant series stays at its constant level.

## API

`data` (a `ChartLinePascalPoint[]`), `period` (default 5, coerced to an
integer >= 1), sizing/format/a11y props, `strokeWidth` / `dotRadius`, the
colour props, the toggle flags (`showAxis` / `showGrid` / `showDots`
default false / `showPma` / `showMarkers` / `showTooltip` /
`showConfigBadge` / `showLegend` / `animate`), controlled + uncontrolled
`hiddenSeries` (keys `price` / `pma`), `formatValue` / `formatX`,
`onPointClick` / `onSeriesToggle`.

Exported helpers: `getLinePascalFinitePoints`, `normalizeLinePascalPeriod`,
`computeLinePascalWeights`, `computeLinePascal`, `classifyLinePascalZone`,
`runLinePascal`, `computeLinePascalLayout`, `describeLinePascalChart`, plus
the `DEFAULT_CHART_LINE_PASCAL_*` constants and the `ChartLinePascalZone`
type. Every exported name carries a `LinePascal` infix so the `export *`
barrel line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen `data-section="chart-line-pascal-aria-desc"`
description span; the price line, the PMA line and every dot / marker are
`role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-period`, `data-pma-final`,
`data-above-count`, `data-below-count`, `data-equal-count`,
`data-total-points` and `data-animate`.

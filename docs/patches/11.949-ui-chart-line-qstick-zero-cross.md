# Patch 11.949 - UI: chart-line-qstick-zero-cross primitive

Version: 1.11.967

## Summary

Add `web/src/components/ui/chart-line-qstick-zero-cross.tsx` -- a pure-SVG
dual panel line chart primitive that visualizes price alongside Tushar
Chande's QStick panel, with explicit cross markers and overlay arrows at
every QStick zero crossover. QStick is the simple moving average of the
candle body (`close - open`) over a fixed window. A positive QStick means
bullish bodies are dominating the window; a negative QStick means bearish
bodies dominate. Zero crossovers mark the baseline transition from one
body-momentum regime to the other -- a sharper signal than a single
candle close because it filters short-term body noise.

## Input shape

This primitive is the first in the chart-line-* family to require
**OHLC-style input** -- specifically `{x, open, close}` -- because the
indicator depends on intra-bar body sign rather than close-to-close
change. The point shape is:

```typescript
{ x: number, open: number, close: number }
```

`getLineQstickZeroCrossFinitePoints` drops any point whose `x`, `open`,
or `close` is non-finite.

## Pipeline

1. Body per bar: `body_i = close_i - open_i`.
2. SMA over the last `length` bodies (CONST short-circuit when
   `min === max` in the window). Returns null for `i < length - 1`.
3. `classifyLineQstickZeroCrossRegime(qstick, T)`: null -> none,
   qstick >= T -> bullish, qstick < T -> bearish.
4. `detectLineQstickZeroCrossCrosses(series, qstick, T)`: strict
   `prev <= T && cur > T` for bullish, strict `prev >= T && cur < T`
   for bearish. Boundary equality does not fire.

## Bit-exact anchors

| Pattern                                | QStick (after warmup) | Regime  |
|----------------------------------------|-----------------------|---------|
| CONST open = close = K                 | 0                     | bullish |
| CONST body = +B (open = i, close = i+B)| +B                    | bullish if B > 0, bearish if B < 0 |
| CONST body = -B                        | -B                    | bearish |

CONST anchor tested bit-exactly across K in `{0, 1, 42, 100, 1234}`.
Sign-flipping bodies (first half bearish, second half bullish) fire
a bullish QStick zero cross; the reverse pattern fires a bearish cross.

## Layout

* Default dimensions: 720 x 460.
* Two panels: price panel on top (~55% of plot area), QStick panel
  below.
* QStick panel auto-fits to the observed `[min, max]` range with 10%
  padding on each end, always including the threshold in the displayed
  range.
* Threshold band rendered as a horizontal rule at `y = thresholdY` in
  the QStick panel.
* Cross markers drawn as circles on the QStick panel at `(cx, cyOsc)`
  and as arrows on the price panel at `(cx, cyPrice)`.
* SVG path strings use `.toFixed(2)` for deterministic snapshot tests.

## Accessibility

* Wrapping `<div role="region" aria-label="QStick Zero Cross chart">`.
* `<svg role="img">` with an aria-described-by linked `<desc>` sr-only
  text.
* Each cross marker: `role="graphics-symbol"` + `tabIndex={0}` +
  descriptive `aria-label` ("bullish cross at x=...").
* Legend toggles: `<button role="switch" aria-pressed="...">`.
* Motion-safe fade-in via `motion-safe:animate-fade-in` (opt-out with
  `animate={false}`).

## Tests

  close), null / undefined guards.

  fallback on non-finite.

  identity.

  `{0, 1, 42, 100, 1234}`, CONST body +/-2 -> qstick +/-2, null warmup
  before index `length - 1`, custom length=5, alternating bodies sum to
  zero, body array reports close-open exactly.

  behaviour.

  gaps, boundary-not-crossed, no-cross stays.

# 11.1043 -- UI: chart-line-ichimoku-divergence-cross primitive (v1.11.1061)

## Scope

Add a pure-SVG dual-panel `ChartLineIchimokuDivergenceCross`
component that renders the close in the top panel and the
Ichimoku Cloud (kumo) midpoint line in the bottom panel,
flagging price-vs-kumo direction-disagreement events as
cloud reversal warnings with bias coloring derived from the
kumo slope at the trigger bar. Uses the canonical 5-state
divergence regime model (aligned-bullish / aligned-bearish
/ divergent-bullish / divergent-bearish / none).

The kumo midpoint condenses the full Ichimoku cloud
(Senkou Span A + Senkou Span B) into a single
direction-indicating line: it tracks the long-range mean
of the rolling high-low range, so direction changes
correspond to the cloud "flipping" colour in the full
Ichimoku visualisation.

## Ichimoku kumo divergence pipeline

```
tenkan[i]   = (HH(high, 9)  + LL(low, 9))  / 2
kijun[i]    = (HH(high, 26) + LL(low, 26)) / 2
senkouA[i]  = (tenkan + kijun) / 2
senkouB[i]  = (HH(high, 52) + LL(low, 52)) / 2
midpoint[i] = (senkouA + senkouB) / 2

priceUp     : close[i] > close[i-1]
priceDown   : close[i] < close[i-1]
kumoUp      : midpoint[i] > midpoint[i-1]
kumoDown    : midpoint[i] < midpoint[i-1]

regime ->
  'aligned-bullish'   when priceUp   && kumoUp
  'aligned-bearish'   when priceDown && kumoDown
  'divergent-bullish' when priceDown && kumoUp
  'divergent-bearish' when priceUp   && kumoDown
  'none'              when either side flat or null

bullish (divergence trigger up) :
  prev regime !== 'divergent-bullish' &&
  cur regime === 'divergent-bullish'
bearish (divergence trigger down) :
  prev regime !== 'divergent-bearish' &&
  cur regime === 'divergent-bearish'

bias        : midpoint[i] vs midpoint[i-1] -> up / down /
              flat / none
```

Default `tenkanLength = 9`, `kijunLength = 26`,
`senkouBLength = 52` -- Goichi Hosoda's original 1968
Ichimoku Kinko Hyo tuning. The triple-window structure
gives the kumo midpoint a deep lag of `(52 - 1) / 4 +
(26 - 1) / 4 + ... = 16.875` bars on linear input, making
it slower than any single-MA divergence-cross sibling but
gives a higher-confidence "cloud-flip" signal: the kumo
midpoint only reverses when the full long-range range
shifts.

This primitive uses current-bar reads (no 26-bar forward
displacement). The canonical Ichimoku formulation
forward-shifts Senkou A and Senkou B by 26 bars, but for
divergence detection we want the kumo's *current*
direction, not its future-projected position -- so we
align the spans to the current bar.

Warmup is `max(tenkanLength, kijunLength, senkouBLength) -
1 = 51` for the default tuning. Divergence detection
requires the previous midpoint + close, so the first regime
classification lands at i = 52.

## Bit-exact anchors

All anchors use HLC input.

- **CONST band** `high = K + 1`, `low = K - 1`, `close
  = K`: every HH window yields K + 1, every LL window
  yields K - 1 -> tenkan = kijun = senkouA = senkouB = K.
  midpoint = K. Both close and midpoint are flat ->
  regime `none`. 0 divergence crosses. Verified across
  K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close
  = i`: For a window `[i - L + 1, i]` with high[j] = j +
  1 and low[j] = j - 1, HH = high[i] = i + 1 and LL =
  low[i - L + 1] = i - L. tenkan window L = 9 -> midpoint
  = (i + 1 + i - 9) / 2 = i - 4. kijun L = 26 -> midpoint
  = i - 12.5. senkouA = (tenkan + kijun) / 2 = i - 8.25.
  senkouB L = 52 -> midpoint = i - 25.5. midpoint =
  (senkouA + senkouB) / 2 = (i - 8.25 + i - 25.5) / 2 =
  `i - 16.875`. close[i] - close[i-1] = +1 (up);
  midpoint[i] - midpoint[i-1] = +1 (up). Both up ->
  regime `aligned-bullish`. 0 divergence crosses.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`,
  `close = -i`: mirror -> midpoint = `-i + 16.875`. Both
  close and midpoint falling -> regime `aligned-bearish`.
  0 divergence crosses.

The clean `i - 16.875` lag is a direct consequence of the
rolling-range midpoint identity: for linear input, the
midpoint of `[HH, LL]` over a window of length L is `i -
(L - 1) / 2 + 1/2 - 1/2 = i - (L - 1) / 2 + 1`... actually
the computation is `(i + 1 + i - L) / 2 = i - (L - 1) /
2`. So tenkan lag = (9-1)/2 = 4, kijun lag = (26-1)/2 =
12.5, senkouA lag = 8.25, senkouB lag = (52-1)/2 = 25.5,
midpoint lag = (8.25 + 25.5) / 2 = 16.875. The lag value
is purely a function of the three window lengths.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 5-state regime classifier follows the
  chart-line-*-divergence-cross family convention; the
  `bullish` / `bearish` cross *kinds* still follow the
  sibling cross-sig convention (line direction).
- Warmup is `senkouBLength - 1 = 51` -- much longer than
  the MA divergence-cross siblings (SMA/EMA/WMA/HMA have
  warmup 13; KAMA has 9; TEMA has 39). Callers feeding
  short data series may want to use shorter tunings, but
  the canonical 9/26/52 is preserved for cross-tool
  compatibility.
- Cross detection fires only on transition INTO a
  divergent state (not while staying in one), preventing
  double-fire on consecutive bars in the same regime.
- Uses current-bar reads rather than the canonical
  26-bar forward displacement -- documented as a
  deliberate divergence-detection adaptation, since
  forward displacement would compare price[i] to a
  future-projected kumo[i] computed from price[i-26]'s
  range, which conflates time-shift with direction.
- HLC input boundary filter requires `high >= low` --
  malformed bars are dropped at the input boundary.
- Sixth divergence-cross primitive in the chart-line
  family but the only one using rolling HH/LL midpoint
  rather than a moving-average centerline. Complementary
  to the MA siblings (SMA/EMA/WMA/HMA/TEMA/KAMA): a
  stock that diverges from its Ichimoku cloud but not
  its MA suggests range-driven sentiment shift; a stock
  that diverges from its MA but not its Ichimoku
  suggests trend-driven mean reversion.

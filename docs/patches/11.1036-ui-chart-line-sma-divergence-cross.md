# 11.1036 -- UI: chart-line-sma-divergence-cross primitive (v1.11.1054)

## Scope

Add a pure-SVG dual-panel `ChartLineSmaDivergenceCross`
component that renders the close in the top panel and the
Simple Moving Average (SMA) in the bottom panel, flagging
price-vs-SMA direction-disagreement events as trend reversal
warnings with bias coloring derived from the SMA slope at
the trigger bar. Uses the canonical 5-state divergence
regime model (aligned-bullish / aligned-bearish /
divergent-bullish / divergent-bearish / none) shared by the
rest of the chart-line-*-divergence-cross family.

This is the uniform-weight sibling of
`chart-line-ema-divergence-cross` (v1.11.1053, EMA-based) and
`chart-line-hma-divergence-cross` (v1.11.1052, HMA-based) --
all three share the same 5-state regime classifier and
transition-into-divergent detection, but SMA uses a plain
uniform-weight rolling average instead of EMA's
recency-weighted decay or HMA's composite construction.

## SMA divergence pipeline

```
sma[i]      = SMA(close, period)

priceUp     : close[i] > close[i-1]
priceDown   : close[i] < close[i-1]
smaUp       : sma[i]   > sma[i-1]
smaDown     : sma[i]   < sma[i-1]

regime ->
  'aligned-bullish'   when priceUp   && smaUp
  'aligned-bearish'   when priceDown && smaDown
  'divergent-bullish' when priceDown && smaUp
  'divergent-bearish' when priceUp   && smaDown
  'none'              when either side flat or null

bullish (divergence trigger up) :
  prev regime !== 'divergent-bullish' &&
  cur regime === 'divergent-bullish'
bearish (divergence trigger down) :
  prev regime !== 'divergent-bearish' &&
  cur regime === 'divergent-bearish'

bias        : sma[i] vs sma[i-1] -> up / down / flat / none
```

Warmup is `period - 1 = 13` for the default tuning -- same
as the EMA sibling. Divergence detection requires the
previous SMA + close, so the first regime classification
lands at i = 14 (one bar past warmup).

## Bit-exact anchors

- **CONST close = K**: every SMA window averages K -> SMA
  = K. Both close and SMA are flat -> regime `none` for
  every bar. 0 divergence crosses. Verified across K in
  {0, 1, 50, 200, 1234}.
- **LINEAR UP close = i**: SMA over linear input settles
  at `i - (period - 1) / 2 = i - 6.5`. SMA[i] - SMA[i-1]
  = +1 (up); close[i] - close[i-1] = +1 (up). Both up
  -> regime `aligned-bullish`. 0 divergence crosses.
- **LINEAR DOWN close = -i**: mirror -> SMA = `-i + 6.5`.
  Both close and SMA falling -> regime `aligned-bearish`.
  0 divergence crosses.

The clean `i - 6.5` lag on LINEAR input is the textbook
SMA centroid identity -- a direct consequence of averaging
uniform-weight bars in a centered window.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 5-state regime classifier follows the
  chart-line-*-divergence-cross family convention; the
  `bullish` / `bearish` cross *kinds* still follow the
  sibling cross-sig convention (line direction).
- SMA helper uses `min === max` short-circuit so CONST
  input gives bit-exact K (no FP-drift to K +/- eps).
- Cross detection fires only on transition INTO a
  divergent state (not while staying in one), preventing
  double-fire on consecutive bars in the same divergent
  regime.
- Among the chart-line-*-divergence-cross siblings, SMA is
  the slowest-reacting centerline (uniform weights vs
  EMA's exponential decay vs HMA's zero-lag composite),
  so it surfaces only the most persistent divergence
  patterns -- complementary to the faster siblings rather
  than redundant with them.

# TODO 11.500 - chart-line-multi primitive

Status: SHIPPED v1.11.518 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartLineMulti>`, a pure-SVG multi-series line chart whose
defining feature is a **synchronised crosshair** that reads the y
value of every visible series at the hovered x position. A
transparent overlay rectangle captures pointer events across the
inner plot area; on move, the overlay snaps the crosshair to the
nearest x in the union of all visible series' x values, draws a
vertical dashed rule, highlights one marker per series, and opens a
single tooltip listing the snapped x value plus one row per series
(swatch + label + value).

## API

```tsx
<ChartLineMulti
  series={[
    {
      id: 'cpu',
      label: 'CPU',
      data: [
        { x: 0, y: 12 }, { x: 1, y: 32 }, { x: 2, y: 28 }, { x: 3, y: 45 },
      ],
    },
    {
      id: 'mem',
      label: 'Memory',
      data: [
        { x: 0, y: 60 }, { x: 1, y: 65 }, { x: 2, y: 70 }, { x: 3, y: 68 },
      ],
    },
  ]}
  onCrosshairChange={(info) => {
    if (info) console.log(`At x=${info.x}:`, info.points);
  }}
/>
```

## Distinct from prior chart-* primitives

- `<ChartLine>` - one series at a time (or basic multi without
  synchronised tooltip); each dot owns its own tooltip.
- `<ChartLineStep>` (11.497) - H+V segments; per-dot tooltip.
- `<ChartLineSmooth>` (11.498) - Bezier / monotone curves; per-dot
  tooltip.
- `<ChartLineDashed>` (11.499) - dash patterns; per-dot tooltip.

`<ChartLineMulti>` is purpose-built around the synchronised-crosshair
interaction pattern that is the canonical way to read multi-series
time-series dashboards (Grafana, Datadog, CloudWatch all use this
form).

## Crosshair contract

### Snap algorithm

1. Build `xUnion = collectLineMultiXValues(series, hidden)` once per
   render: sorted unique x values across every visible series.
2. On pointer move, convert the pointer's client x to data space:
   `xData = xMin + ((pointerLocalX - padding) / innerWidth) * xRange`.
3. `findNearestXIndex(xUnion, xData)` returns the closest index in
   `xUnion` (binary search; ties break toward the lower index).
4. The snapped x lives at `xUnion[idx]`; the SVG-space pixel is
   `xUnionPx[idx]`.
5. For each visible series, `findNearestPointInSeries(series.points,
   snappedX)` returns the closest point in that series (so series
   with sparse / missing samples still contribute a row to the
   tooltip).

### Visuals

- Vertical dashed line (`data-section="chart-line-multi-crosshair-line"`)
  from `padding` to `padding + innerHeight` at the snapped x.
- One emphasised marker per visible series
  (`data-section="chart-line-multi-crosshair-marker"`); the marker is
  a `dotRadius + 2` circle with a white border so it pops against
  the series stroke.
- Per-dot `data-crosshair-hit="true"` on the underlying series dots
  whose index matches a crosshair-hit point.
- Single tooltip with the snapped `x:` header, then one row per
  visible series (swatch + label + value at that x).

### Callback

`onCrosshairChange(info | null)` fires on every snap change and on
pointer-leave (with `null`). The payload is
`{ x, px, xIndex, points: [{series, point}] }`.

## Exported helpers

- `DEFAULT_CHART_LINE_MULTI_*` numeric / string defaults.
- `DEFAULT_CHART_LINE_MULTI_PALETTE` (10-colour palette).
- `getLineMultiDefaultColor(index)` - palette wrap.
- `getLineMultiFinitePoints(points)` - drops non-finite.
- `getLineMultiBounds(series, hidden?)` - per-axis bounds.
- `getLineMultiTicks(min, max, count?)` - evenly-spaced ticks.
- `collectLineMultiXValues(series, hidden?)` - sorted unique union of
  x values across every visible series (drops non-finite).
- `findNearestXIndex(sortedXs, target)` - binary search returning the
  index of the closest x; ties to the lower index; empty / non-finite
  -> `-1`.
- `findNearestPointInSeries(layoutPoints, targetX)` - linear scan
  returning the closest point in a series; ties to the lower index;
  empty / non-finite -> `null`.
- `buildLineMultiPath(points)` - simple `M x y L x y ...` builder.
- `computeLineMultiLayout(input)` - per-series layout + `xUnion`
  (data values) + `xUnionPx` (pixel positions) so the crosshair
  doesn't recompute them per pointer event.
- `describeLineMultiChart(series, hidden?, formatValue?)` - ARIA
  summary mentioning the synchronised crosshair.

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only
  `data-section="chart-line-multi-aria-desc"` span.
- SVG `role="img"` with the same `aria-label`.
- Each path is `role="graphics-symbol"` with `tabIndex={0}` and
  `aria-label="<series>: line with K points"`.

## Data attributes

- Root: `data-section="chart-line-multi"`, `data-series-count`,
  `data-visible-series-count`, `data-total-points`,
  `data-x-union-count`, `data-crosshair-active`,
  `data-crosshair-index`, `data-animate`.
- Series group: `data-section="chart-line-multi-series-group"`,
  `data-series-id`, `data-series-index`, `data-series-color`,
  `data-series-point-count`, `data-series-finite-count`.
- Path: `data-section="chart-line-multi-path"`, `data-series-id`.
- Dot: `data-section="chart-line-multi-dot"`, `data-series-id`,
  `data-point-index`, `data-x`, `data-y`, `data-crosshair-hit`.
- Crosshair line + markers: `data-section="chart-line-multi-crosshair-line"`
  with `data-x-index` / `data-x-value`;
  `data-section="chart-line-multi-crosshair-marker"` per series.
- Overlay: `data-section="chart-line-multi-overlay"` - transparent
  pointer-capture rect spanning the inner plot area.
- Tooltip: `-tooltip`, `-tooltip-x`, `-tooltip-list`, `-tooltip-row`,
  `-tooltip-swatch`, `-tooltip-label`, `-tooltip-value`.
- Legend, axes, grid: standard pattern with `data-axis="x"` or
  `"y"` on ticks / grid lines.

## Out of scope

- Linear y interpolation between samples for series with sparse x
  values - v1 snaps to the nearest data point in each series.
- Drag-to-zoom / brush selection - hover only.
- Vertical-only or horizontal-only crosshair toggle - this primitive
  always paints a vertical line (matches multi-series time-series
  convention).
- Animated dash on the crosshair line (snap; static dasharray only).
- Log-scale axes - linear only.

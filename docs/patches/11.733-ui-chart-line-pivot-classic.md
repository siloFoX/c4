## Summary

`web/src/components/ui/chart-line-pivot-classic.tsx` ships
`<ChartLinePivotClassic>`, a pure-SVG single-panel chart with the
**Classic Floor Pivot** overlays for the pivot point, S1, S2, R1,
and R2 lines. The close line shares one panel with five step-
function level lines (R2 / R1 / PP / S1 / S2); each defined bar
carries the five levels as bar-wide horizontal stub segments
projected from the prior bar.

## Computation

From the PRIOR bar's `(high, low, close)`:

```
range = high - low
PP    = (high + low + close) / 3
R1    = 2 * PP - low
S1    = 2 * PP - high
R2    = PP + range
S2    = PP - range
```

The first bar has no prior reference and is left null on every level.

## Worked anchors

The Classic formula is integer-rational when `H + L + C` is a
multiple of 3. The implementation has three exact-style anchors.

- **Constant-series anchor** -- prev `H = L = C = 5`. `range = 0`,
  `PP = 5`. Every R/S level is 5 bit-exact.
- **Worked anchor A** -- prev `H = 12, L = 8, C = 10`. Sum = 30,
  `PP = 30 / 3 = 10`. `range = 4`. `R1 = 2*10 - 8 = 12`,
  `S1 = 2*10 - 12 = 8`, `R2 = 10 + 4 = 14`, `S2 = 10 - 4 = 6`.
  All bit-exact integer.
- **Worked anchor B** -- prev `H = 15, L = 6, C = 9`. Sum = 30,
  `PP = 30 / 3 = 10`. `range = 9`. `R1 = 2*10 - 6 = 14`,
  `S1 = 2*10 - 15 = 5`, `R2 = 10 + 9 = 19`, `S2 = 10 - 9 = 1`.
  All bit-exact integer.

The strict order `R2 > R1 > PP > S1 > S2` is tested directly. Two
algebraic identities are bit-exact for integer fixtures:

- `R2 + S2 = 2 * PP` (the R2/S2 pair is PP-symmetric).
- `R1 + S1 = 4 * PP - H - L` (the R1/S1 identity from `2*PP - L`
  and `2*PP - H`).

Plus `R2 - PP = range` and `PP - S2 = range`.

A direct Classic-vs-Woodie sanity check is included: for an
asymmetric prev `(H=10, L=8, C=11)` where Classic gives `PP = 29/3
~= 9.67` and Woodie would give `PP = (10+8+22)/4 = 10`, the Classic
formula matches `29/3` to 12 places, distinguishing it from Woodie.

## Distinct from neighbouring primitives

- vs `<ChartLinePivotWoodie>` -- Woodie doubles the close in PP:
  `PP_w = (H + L + 2C) / 4`. Classic weighs it once: `PP_c =
  (H + L + C) / 3`. For an asymmetric `H=10, L=8, C=11`, Classic
  gives `~9.67` and Woodie gives `10`. Same R/S formulas
  otherwise.
- vs `<ChartLinePivotFibonacci>` -- Fibonacci uses the same
  `PP = (H+L+C)/3` but its R/S levels offset by Fibonacci ratios
  of the range (0.382 / 0.618 / 1.0) instead of the R1/S1 lever-
  arm formula.
- vs `<ChartLinePivotDemark>` -- DeMark's X seed is DIRECTION-
  DEPENDENT (bullish doubles H, bearish doubles L, neutral
  doubles C); Classic is direction-agnostic with all OHL terms
  weighted equally in PP.

## API

`data` -- a single `ChartLinePivotClassicPoint[]`
(`{x, high, low, close}`). Standard sizing / colour / a11y props
plus `strokeWidth`, `dotRadius`, `tickCount`. Toggle flags:
`showAxis`, `showGrid`, `showDots` (default false), `showPp`,
`showLevel1`, `showLevel2`, `showMarkers`, `showTooltip`,
`showConfigBadge`, `showLegend`, `animate`. Controlled and
uncontrolled visibility via `hiddenSeries`
(`'price' | 'pp' | 'r1' | 's1' | 'r2' | 's2'`). Callbacks:
`onPointClick({point})`, `onSeriesToggle({seriesId, hidden})`.
Format functions: `formatValue`, `formatX`. ARIA: `ariaLabel`,
`ariaDescription`.

## Pure helpers

- `DEFAULT_CHART_LINE_PIVOT_CLASSIC_*` -- defaults.
- `getLinePivotClassicFinitePoints` -- keep only points with finite
  x / high / low / close and `high >= low`.
- `computeLinePivotClassicLevels(prev)` -- the five levels for a
  bar given the prior; nulls when prev is null or non-finite.
- `computeLinePivotClassic(bars)` -- per-bar levels; bar 0 is all
  nulls.
- `classifyLinePivotClassicZone(close, levels)` -- `'above-r2' |
  'r1-to-r2' | 'pp-to-r1' | 's1-to-pp' | 's2-to-s1' | 'below-s2'
  | 'none'`.
- `runLinePivotClassic(points)` --
  `{series, levels, samples, ppFinal, aboveCount, belowCount,
  betweenCount, ok}`.
- `computeLinePivotClassicLayout({data, ...})` -- single panel
  with the close line, five level stub segments per defined bar,
  one marker per defined-zone bar coloured by zone band.
- `describeLinePivotClassicChart(data)` -- ARIA summary
  mentioning the pivot formula and the warm-up first bar.

## a11y / data hooks

Root `<div role="region">` with `aria-label` + `aria-describedby`; the
SVG is `role="img"` with the same label; an sr-only
`data-section="chart-line-pivot-classic-aria-desc"` span carries the
description. The close line, every price dot and every marker carry
`role="graphics-symbol"` and `tabIndex={0}`. Every node carries a
`data-section`. Root mirrors `data-pp-final`, `data-above-count`,
`data-below-count`, `data-between-count`, `data-total-points`,
`data-animate`, `data-empty`. Segments expose `data-series-id` and
`data-value`; markers expose `data-zone` and `data-close`.

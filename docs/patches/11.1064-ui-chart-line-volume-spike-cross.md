# 11.1064 -- UI: chart-line-volume-spike-cross primitive (v1.11.1082)

## Scope

Add a pure-SVG dual-panel `ChartLineVolumeSpikeCross`
component that renders the close in the top panel and
volume bars + a spike-threshold line (multiplier *
rolling SMA of volume) in the bottom panel, marking
bullish (volume crosses up through threshold -- abnormal
volume surge entry trigger, breakout confirmation) /
bearish (volume crosses down through threshold -- spike
ends, volume returns to normal) volume-vs-threshold
crossover trigger events with bias coloring derived from
the volume slope at the trigger bar.

The volume spike pattern is one of the oldest breakout-
confirmation filters: when current volume exceeds its
rolling average by a configured multiplier, the move is
considered "confirmed by participation". A `2x` spike on
a 20-bar rolling window is the canonical "abnormal
volume" threshold used by Wyckoff / Joe Granville
volume-pattern analysts.

## Volume spike-cross pipeline

```
avgVolume[i] = SMA(volume, period)
threshold[i] = multiplier * avgVolume[i]

bullish (spike-entry trigger) :
  prev volume <= prev threshold &&
  cur volume > cur threshold
bearish (spike-exit trigger) :
  prev volume >= prev threshold &&
  cur volume < cur threshold
regime       : 'bullish' when volume >= threshold
               'bearish' when volume <  threshold
               'none'    when either is null
bias         : volume[i] vs volume[i-1] -> up / down /
               flat / none
```

Default `period = 20`, `multiplier = 2` -- canonical
Wyckoff/Granville volume-spike tuning.

**New input type**: this is the first family member that
accepts a `volume` field alongside `close`. The input
point shape is `{ x, close, volume }`. Volume must be
non-negative; the input filter drops any point with
negative volume.

Unlike most cross-sig family members which feature
symmetric thresholds around 0, this primitive uses an
asymmetric threshold (always positive, always above
avgVolume for multiplier > 1). Steady-state volume sits
*below* the threshold -- the bullish "spike entry"
trigger fires only on extraordinary volume readings.

Warmup is `period + 1 = 21` for the default tuning:
avgVolume valid from i = period - 1 = 19, and cross
detection requires the previous bar's volume +
threshold, so the first potential cross lands at i =
period = 20.

## Bit-exact anchors

All anchors use close + volume input.

- **CONST volume** `volume = V`: avgVolume = V,
  threshold = multiplier * V. For V > 0: threshold =
  2V > V -> regime `bearish` (below threshold). For
  V = 0: threshold = 0, volume = 0 >= 0 -> regime
  `bullish`. 0 crosses. Verified across V in {0, 1, 50,
  200, 1234}.
- **LINEAR UP volume** `volume = i + 100`: avgVolume =
  i + 100 - 9.5 = i + 90.5 (centroid lag). threshold =
  2 * (i + 90.5) = 2i + 181. volume = i + 100.
  threshold - volume = i + 81. Always positive and
  growing, so volume stays below threshold forever ->
  regime `bearish`. 0 crosses.
- **LINEAR DOWN volume** `volume = 1000 - i`:
  avgVolume = 1000 - i + 9.5 = 1009.5 - i. threshold =
  2 * (1009.5 - i). volume = 1000 - i. threshold -
  volume = 1019 - i. Stays positive for i < 1019, so
  volume below threshold for the entire useful range.
  0 crosses.

All steady-state inputs produce zero crosses because
volume (regardless of trend direction) sits below the
multiplier-scaled threshold. Crosses only fire on volume
spikes that exceed the configured multiple of the
rolling average -- this is the defining behaviour of the
indicator. The test suite includes a synthetic spike (5x
above baseline) verifying that a bullish cross fires on
real-world-shaped data.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- First family member with a **volume input**. The
  point shape `{ x, close, volume }` extends the
  close-only convention used by other primitives. The
  finite-points filter rejects negative volume per
  spec (volume cannot be negative).
- Asymmetric threshold (always above avgVolume for
  multiplier > 1) means CONST/LINEAR steady-state
  produces regime `bearish` throughout. The bullish
  regime appears only during volume spikes. This is
  intentional and matches the canonical spike-detection
  semantics.
- Volume bars in the bottom panel render as opaque
  rectangles (vs the line-based primitives in other
  family members). This is the conventional volume
  representation; threshold is overlaid as a dashed
  line.
- The `multiplier = 2` default is one common choice;
  Wyckoff analysts often use 1.5-3x depending on
  instrument volatility and timeframe. The primitive
  accepts any positive multiplier.
- 0 crosses on steady-state input is canonical
  behaviour; the test suite explicitly verifies a
  synthetic 5x spike fires a bullish cross to confirm
  the detector works in real-world shapes.
- Sibling to the broader cross-sig family (SMA, EMA,
  Bollinger, Keltner, Donchian, CCI, RSI, Ichimoku,
  Supertrend, Williams, DPO, TRIX, HMA, Momentum,
  ROC) -- this primitive adds volume-based confirmation
  alongside the price-based cross-sig members.

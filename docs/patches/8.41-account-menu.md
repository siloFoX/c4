# 8.41 — Account menu (sidebar bottom + header avatar)

Branch: `dgx-spark/account-menu`

## What ships

A claude.ai / VS Code / Linear-style account menu. The standalone
`Sign out` IconButton in `AppHeader` is gone; in its place:

- An avatar + name + role-badge + chevron row pinned to the bottom of
  the existing `Sidebar` (visible on the Workers tab — the primary
  surface).
- A compact icon-only avatar trigger in `AppHeader` that opens the
  same dropdown (fallback for tabs without a sidebar — Sessions, Chat,
  History, Workflows, Settings, Features).

Both triggers open the same dropdown menu:

| Item | Action |
|------|--------|
| Profile | Disabled (`soon` hint). Reserved for the multi-account / avatar follow-up. |
| Preferences | `setTopView('settings')` |
| Keyboard shortcuts | Dispatches `HELP_EVENT_OPEN_SHORTCUTS` (existing global handler) |
| Help center | Dispatches `HELP_EVENT_OPEN_DRAWER` (existing global handler) |
| Sign out | Calls the host-supplied `onLogout` (delegates to `lib/api.logout`). |

## Surface area

### `web/src/components/ui/dropdown-menu.tsx` (new)

Hand-rolled dropdown / popover primitive matching the rest of the UI
layer (no new third-party dep — no radix-ui dropdown):

- `<DropdownMenu trigger={...} items={...} placement="top|bottom"
  ariaLabel header />`
- Auto-wires `aria-haspopup="menu"` / `aria-expanded` /
  `aria-controls` on the trigger via `cloneElement`.
- Click-outside dismiss (`mousedown` listener on `document`).
- Escape dismiss + ArrowUp / ArrowDown roving focus that skips
  disabled rows.
- `variant: 'default' | 'danger'` per item — danger gets the red
  destructive token treatment.
- Items expose `key / label / icon / hint / disabled / onSelect`.
- Optional `header` slot rendered above the items (the AccountMenu
  uses it to mirror the user identity inside the popover).

### `web/src/components/AccountMenu.tsx` (new)

- Reads `getAuthUser()` / `getAuthRole()` from `lib/api.ts` (cached
  from the `/auth/login` response — see below).
- Re-syncs on `AUTH_EVENT` (401-driven logout) **and** the cross-tab
  `storage` event so logging out in another tab wipes the avatar.
- `initialsFor(user)` derives a 2-char avatar fallback (e.g.
  `admin` → `AD`, `silo.fox` → `SF`).
- `roleBadgeClass(role)` maps `admin / manager / viewer` to
  destructive / primary / muted token-backed classes; unknown roles
  fall back to neutral secondary so an undefined role never paints
  itself as admin.
- Exports `ACCOUNT_LABEL_*` literal constants so source-grep tests
  pin the canonical copy and a future i18n pass has clean keys.
- `collapsed` prop renders icon-only (used by the AppHeader compact
  trigger).

### `web/src/lib/api.ts`

- Adds `c4.authUser` + `c4.authRole` localStorage keys.
- `LoginResponse` gains `role?: string | null`.
- `login()` calls a new private `setAuthUser(user, role)` after a
  successful `/auth/login`.
- `clearToken()` now wipes user + role keys too, so logout / 401 fully
  resets the cached identity.
- `getAuthUser()` / `getAuthRole()` exported for the AccountMenu
  consumer.

### `web/src/components/layout/Sidebar.tsx`

- Re-laid as a flex column: scrollable worker list grows, a
  bottom-pinned `AccountMenu` sits in a `border-t` strip styled
  `bg-card/50`.
- New optional props: `onLogout` / `onOpenPreferences`. When omitted
  (e.g. test mounts), the bottom row simply doesn't render.

### `web/src/components/layout/AppHeader.tsx`

- `LogOut` icon import + standalone `IconButton` removed.
- Compact `<AccountMenu collapsed onLogout={...} onOpenPreferences=
  {...} />` rendered in its place (hidden on mobile via `md:block`,
  since the mobile flow opens the sidebar via the hamburger).
- `onOpenPreferences` added as an optional prop and wired by
  `App.tsx` to `setTopView('settings')`.

### `web/src/App.tsx`

- Passes `onLogout={handleLogout}` + `onOpenPreferences={() =>
  setTopView('settings')}` to both `Sidebar` and `AppHeader`.

## Tests

`tests/account-menu.test.js` — 33 source-grep assertions across 7
suites covering:

1. DropdownMenu primitive contract (exports, placement, aria-* wiring,
   role="menu" / role="menuitem", Escape / click-outside / item
   activation, ArrowUp/Down skipping disabled, danger variant).
2. UI primitive index re-exports `dropdown-menu`.
3. `lib/api.ts` user+role caching contract.
4. AccountMenu component (exported labels, auth event wiring, help
   event dispatch, danger variant on Sign out, role badge mapping,
   placement="top").
5. Sidebar mounts AccountMenu only when `onLogout` is wired and
   forwards the optional props.
6. AppHeader removed `LogOut` import + mounts `<AccountMenu collapsed
   />` + retains help/shortcut/locale controls.
7. App.tsx wires `onOpenPreferences` and `onLogout` correctly.

Full test run: 116 / 117 pass (the lone failure is the pre-existing
`file-transfer` test PR #24 fixes — not introduced by this patch).

## Test plan (manual)

1. `c4 daemon start && cd web && npm run dev`
2. Log in. The header avatar shows initials of the username; the
   sidebar bottom shows full `avatar + name + role badge + chevron`.
3. Click sidebar bottom row → dropdown opens above the avatar with
   Profile (disabled, "soon"), Preferences, Keyboard shortcuts (`?`),
   Help center, Sign out (red).
4. Click Preferences → top view flips to Settings.
5. Click Keyboard shortcuts → existing shortcuts modal opens.
6. Click Help center → existing help drawer slides in.
7. Click Sign out → logout flow runs and the login screen renders.
8. Log in again, switch to Sessions tab. The sidebar isn't rendered
   on Sessions, but the compact avatar in the header still opens the
   same dropdown.
9. Open a second tab, log out from one — the other tab's avatar /
   name / role clears within a couple of seconds (storage event).

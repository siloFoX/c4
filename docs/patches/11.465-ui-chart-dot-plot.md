## Summary

Add a new `<ChartDotPlot>` UI primitive in
`web/src/components/ui/chart-dot-plot.tsx` -- pure-SVG dot
plot for categorical-numeric data. Each category gets a
vertical slot; every data point in that category renders
as a circle at the value's y position. Optional
deterministic jitter spreads overlapping dots horizontally
(seeded so the layout is reproducible across renders and
snapshot tests). Optional median tick draws a short
horizontal segment at each category's median. Hover a dot
to surface its category + value; per-dot click handler
fires with the full payload.

## API

```ts
interface ChartDotPlotCategory {
  id: string;
  label: string;
  values: readonly number[];
  color?: string;
}

interface ChartDotPlotProps {
  categories: readonly ChartDotPlotCategory[];
  width?: number;                  // default 520
  height?: number;                 // default 320
  padding?: number;                // default 36
  dotRadius?: number;              // default 3.5
  jitter?: number;                 // 0..1 of slotWidth (default 0.35)
  jitterSeed?: number;             // default 1
  showMedianTick?: boolean;        // default true
  medianTickWidth?: number;        // 0..1 of slotWidth (default 0.65)
  showLabels?: boolean;            // default true
  showAxisTicks?: boolean;         // default true
  showTooltip?: boolean;           // default true
  showLegend?: boolean;            // default false
  animate?: boolean;               // default true
  className?: string;
  ariaLabel?: string;              // default 'Dot plot'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  defaultDotColor?: string;        // default '#2563eb'
  dotOpacity?: number;             // default 0.7
  tickCount?: number;              // default 5
  legendPlacement?: 'right' | 'bottom';
  onDotClick?: (args: {
    category: ChartDotPlotCategory;
    categoryIndex: number;
    value: number;
    valueIndex: number;
  }) => void;
}
```

## Behaviour

- Renders a `<div role="region">` wrapping `<svg
  role="img">` carrying the same `aria-label`.
- Vertical scale: `getDotPlotBounds` returns chart-
  wide y min / max across every category; non-finite
  values are dropped; collapsed range expands `+/- 0.5`;
  empty data falls back to `(0, 1)`.
- Slot layout: each category occupies an equal slice
  of the inner width; the slot centre x is
  `padding + ci * slotWidth + slotWidth / 2`.
- Jitter: `getDotPlotJitterOffset(seed, idx, jitter,
  slotWidth)` is a deterministic mulberry32-style
  PRNG. The offset is bounded to `+/- (jitter *
  slotWidth / 2)`. Identical inputs always produce
  the same offset so renders are reproducible.
- Median tick: when `showMedianTick=true` (default),
  a short horizontal line draws across each category
  at its median y (computed via the shared
  `getBoxplotQuantile` from `<ChartBoxplot>`).
  `medianTickWidth` controls the width as a fraction
  of `slotWidth`.
- Hover: enlarges the dot to `dotRadius + 1`,
  swaps the stroke to dark, and opens a tooltip
  with the category label + formatted value.
- Per-dot click: fires `onDotClick` with the full
  payload (`category`, `categoryIndex`, `value`,
  `valueIndex`).
- Legend: off by default (dot plots are usually
  small + tightly labelled). Toggle on with
  `showLegend=true`; right-aligned by default,
  `legendPlacement="bottom"` switches layout.
- Mount animation:
  `motion-safe:animate-fade-in` per category group.
  `animate=false` removes the class.
- Default ARIA description: `"Dot plot with N
  categories. <label> n=<count>, median <m>,
  ..."` -- empty input returns `"No data"`;
  `ariaDescription` overrides.

```ts
DEFAULT_CHART_DOT_PLOT_WIDTH              = 520
DEFAULT_CHART_DOT_PLOT_HEIGHT             = 320
DEFAULT_CHART_DOT_PLOT_PADDING            = 36
DEFAULT_CHART_DOT_PLOT_DOT_RADIUS         = 3.5
DEFAULT_CHART_DOT_PLOT_JITTER             = 0.35
DEFAULT_CHART_DOT_PLOT_JITTER_SEED        = 1
DEFAULT_CHART_DOT_PLOT_DOT_COLOR          = '#2563eb'
DEFAULT_CHART_DOT_PLOT_DOT_OPACITY        = 0.7
DEFAULT_CHART_DOT_PLOT_MEDIAN_TICK_WIDTH  = 0.65
DEFAULT_CHART_DOT_PLOT_TICK_COUNT         = 5

getDotPlotBounds(categories): { min, max }
getDotPlotCategoryMedian(values): number | null
getDotPlotPseudoRandom(seed, index): number
getDotPlotJitterOffset(seed, index, jitter, slotWidth): number
getDotPlotTicks(min, max, count?): number[]
describeDotPlot(categories, formatValue?): string
```

- `getDotPlotBounds` ignores non-finite values;
  empty / all-non-finite falls back to `(0, 1)`;
  collapsed range expands `+/- 0.5`.
- `getDotPlotCategoryMedian` returns the canonical
  median (linear interpolation between adjacent
  values via the shared `getBoxplotQuantile`);
  returns `null` when the input has no finite
  values.
- `getDotPlotPseudoRandom` is a deterministic
  mulberry32-style PRNG: same `(seed, index)`
  always returns the same value in `[0, 1)`.
  Non-finite seeds are treated as 0.
- `getDotPlotJitterOffset` returns 0 for zero
  jitter, zero slot width, or non-finite inputs.
  Otherwise it returns a deterministic offset
  bounded to `+/- (jitter * slotWidth / 2)`.
- `getDotPlotTicks` evenly-spaced inclusive
  endpoints; collapsed -> `[min]`; min count 2.
- `describeDotPlot` produces the default ARIA
  summary including n and median per category.

## Out of scope

- Beeswarm / force-based collision resolution
  (jitter-only v1; host can pre-compute
  layout offsets and pass them through a custom
  wrapper).
- Horizontal orientation (vertical-only v1).
- Grouped dots within a category (host splits into
  multiple categories or composes side-by-side).
- Animated transitions on data change (snap v1;
  mount fade only).

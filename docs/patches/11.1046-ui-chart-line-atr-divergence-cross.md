# 11.1046 -- UI: chart-line-atr-divergence-cross primitive (v1.11.1064)

## Scope

Add a pure-SVG dual-panel `ChartLineAtrDivergenceCross`
component that renders the close in the top panel and the
Average True Range (ATR) in the bottom panel, flagging
price-vs-ATR direction-disagreement events as volatility-
regime reversal warnings with bias coloring derived from
the ATR slope at the trigger bar. Uses the canonical
5-state divergence regime model (aligned-bullish /
aligned-bearish / divergent-bullish / divergent-bearish /
none).

Wilder's 1978 Average True Range is the rolling average of
the true-range bar magnitude. Unlike directional indicators,
ATR is always non-negative and direction-agnostic -- rising
ATR means expanding volatility regardless of trend, falling
ATR means contracting volatility. Divergence between price
direction and ATR direction surfaces volatility-regime
reversals: a fading rally on contracting range, or a
capitulation on expanding range.

## ATR divergence pipeline

```
tr[i]        = max(high[i] - low[i],
                   |high[i] - close[i-1]|,
                   |low[i]  - close[i-1]|)
atr[i]       = SMA(tr, period)

priceUp     : close[i] > close[i-1]
priceDown   : close[i] < close[i-1]
atrUp       : atr[i]   > atr[i-1]
atrDown     : atr[i]   < atr[i-1]

regime ->
  'aligned-bullish'   when priceUp   && atrUp
                       (price rising on expanding
                        volatility -- aligned breakout)
  'aligned-bearish'   when priceDown && atrDown
                       (price falling on contracting
                        volatility -- aligned drift)
  'divergent-bullish' when priceDown && atrUp
                       (price falling but ATR rising --
                        bullish volatility-regime reversal
                        warning; selling pressure releases
                        on expanding range)
  'divergent-bearish' when priceUp   && atrDown
                       (price rising but ATR falling --
                        bearish volatility-regime reversal
                        warning; rally fades on contracting
                        range)
  'none'              when either side flat or null

bullish (divergence trigger up) :
  prev regime !== 'divergent-bullish' &&
  cur regime === 'divergent-bullish'
bearish (divergence trigger down) :
  prev regime !== 'divergent-bearish' &&
  cur regime === 'divergent-bearish'

bias        : atr[i] vs atr[i-1] -> up / down / flat / none
```

Default `period = 14` -- Wilder's 1978 standard tuning.

This primitive uses **SMA-based ATR** (matching
`chart-line-atr-breakout-cross` v1.11.1048) rather than
Wilder-style smoothing (matching
`chart-line-supertrend-divergence-cross` v1.11.1062). The
trade-off:

- **SMA ATR**: exact integer values on linear input with
  no convergence transient. Used by trader-friendly modern
  variants (pyalgotrade, tulip-indicators, ta-lib's
  `ATR_SMA`).
- **Wilder ATR**: asymptotic convergence; trends a bit
  smoother in noisy data. Used by canonical 1978 Wilder
  and Olivier Seban supertrend.

SMA was chosen here for bit-exact testability and
consistency with the breakout sibling. The two ATR
variants are documented as a deliberate family-level
tuning decision rather than an oversight.

Warmup is `period = 14` for the default tuning: TR seeds
at i = 1, ATR first valid at i = period = 14. Divergence
detection requires the previous ATR + close, so the first
regime classification lands at i = period + 1 = 15.

## Bit-exact anchors

All anchors use HLC input.

- **CONST band** `high = K + 1`, `low = K - 1`, `close
  = K`: range = 2, |high - prevClose| = 1, |low -
  prevClose| = 1 -> tr = max(2, 1, 1) = 2. atr = 2
  (constant). Both close and ATR are flat -> regime
  `none` for every bar. 0 divergence crosses. Verified
  across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close
  = i`: range = 2, |high - prevClose| = 2, |low -
  prevClose| = 0 -> tr = max(2, 2, 0) = 2. atr = 2
  constant. Price rises at +1 per bar but ATR is flat
  -> regime `none` (volatility is stable through the
  uptrend). 0 divergence crosses. This is canonical ATR
  behaviour -- in a steady linear uptrend, the per-bar
  range is constant, so ATR pins at the range magnitude.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`,
  `close = -i`: range = 2, |high - prevClose| = 0, |low
  - prevClose| = 2 -> tr = max(2, 0, 2) = 2. atr = 2
  constant. Mirror -> regime `none`. 0 divergence
  crosses.

The "flat ATR on LINEAR input" property mirrors the
Williams %R sibling (`chart-line-williams-divergence-cross`
v1.11.1063): both are direction-agnostic momentum /
volatility oscillators that pin at a constant value on
steady linear input. Divergence detection fires only on
curvature or noise -- the steady-state anchors verify the
underlying pipeline rather than producing divergence
events.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 5-state regime classifier follows the
  chart-line-*-divergence-cross family convention; the
  `bullish` / `bearish` cross *kinds* still follow the
  sibling cross-sig convention (line direction).
- ATR uses SMA-based smoothing (not Wilder) for bit-exact
  integer test values. Different from
  `chart-line-supertrend-divergence-cross` (which uses
  Wilder per Olivier Seban's canonical formulation).
  Documentation makes the variant choice explicit.
- ATR is non-negative -- the layout clamps `oscMin` to 0
  to prevent the padded oscillator range from drifting
  below zero (a no-op for typical inputs, but a guard
  against the `oscRawMin === oscRawMax` padding pulling
  the floor negative).
- On steady LINEAR input ATR pins at a constant value and
  produces zero divergence crosses, which is correct
  canonical behaviour -- the divergence detector is
  designed to fire on volatility-regime changes, not
  trend confirmation. Real-world data with volatility
  expansion / contraction produces the expected divergent
  regime transitions.
- HLC input boundary filter requires `high >= low` --
  malformed bars are dropped at the input boundary.
- Complementary to `chart-line-atr-breakout-cross`
  v1.11.1048 (same ATR pipeline; different signal --
  breakout cross watches ATR vs threshold, divergence
  cross watches ATR direction vs price direction).
  Dashboards can ship both for orthogonal volatility
  signals.

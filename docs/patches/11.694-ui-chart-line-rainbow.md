## Summary

New `ChartLineRainbow` UI primitive in
`web/src/components/ui/chart-line-rainbow.tsx` -- a pure-SVG single-panel
line chart with a **Rainbow Moving Average** overlay of recursively
smoothed average bands. The price line and the rainbow fan share one
panel.

## The computation

The Rainbow Moving Average is a fan of recursively smoothed bands: the
first band is a simple moving average of the price, and every next band is
a simple moving average of the band before it. Each successive band is
smoother and lags more, so plotted together they fan out like a rainbow.
It is a close-only indicator, so `ChartLineRainbowPoint` carries
`{x, value}`.

- `computeLineRainbowSma(arr, period)` -- the windowed simple moving
  average that drives the recursion; a window touching a null slot yields
  null.
- `computeLineRainbowBands(values, period, bandCount)` -- the recursive
  fan: band 0 is `SMA(prices)`, band k is `SMA(band k-1)`. Returns one
  array per band.
- `classifyLineRainbowZone(value, low, high)` -- where the price sits
  relative to the full rainbow envelope: **above**, **below**, **inside**
  or **none** (until every band is defined).
- `getLineRainbowBandColor(index, bandCount)` -- a red-to-violet hue ramp
  so the bands plot as a rainbow.

## Worked fixture

A simple moving average of an arithmetic progression over an odd window is
its centre value, so a 3-period SMA of a linear ramp is the ramp lagged
one bar -- and every recursive band lags one more bar than the band before
it. The fixture is a 10-bar linear ramp `[10,20,...,100]` read with period
3 and 3 bands, so every band lands on an exact integer (each /3 average is
of three consecutive multiples of 10):

```
band 0  [.,.,20,30,40,50,60,70,80,90]
band 1  [.,.,.,.,30,40,50,60,70,80]
band 2  [.,.,.,.,.,.,40,50,60,70]
```

All three band arrays are asserted with `toEqual`, and the recursion
itself is checked -- band 1 equals `SMA(band 0)`, band 2 equals
`SMA(band 1)`. The ramp rises faster than the bands lag, so once the full
rainbow is defined the price runs above it: zones are `none` through the
warm-up then `above` (4 bars above, 0 below, 0 inside). A constant series
collapses every band to the same level, so the price runs inside the
rainbow.

## API

`data` (a `ChartLineRainbowPoint[]`), `period` (default 2), `bandCount`
(default 6), both coerced to an integer >= 1, sizing/format/a11y props,
`strokeWidth` / `bandStrokeWidth` / `dotRadius`, the colour props, the
toggle flags (`showAxis` / `showGrid` / `showRainbow` / `showMarkers` /
`showTooltip` / `showConfigBadge` / `showLegend` / `animate`), controlled +
uncontrolled `hiddenSeries` (keys `price` / `rainbow`), `formatValue` /
`formatX`, `onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineRainbowFinitePoints`, `normalizeLineRainbowInt`,
`computeLineRainbowSma`, `computeLineRainbowBands`,
`classifyLineRainbowZone`, `getLineRainbowBandColor`, `runLineRainbow`,
`computeLineRainbowLayout`, `describeLineRainbowChart`, plus the
`DEFAULT_CHART_LINE_RAINBOW_*` constants and the `ChartLineRainbowZone`
type. Every exported name carries a `LineRainbow` infix so the `export *`
barrel line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen `data-section="chart-line-rainbow-aria-desc"`
description span; the price line and every marker are
`role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-period`, `data-band-count`,
`data-above-count`, `data-below-count`, `data-inside-count`,
`data-total-points` and `data-animate`.

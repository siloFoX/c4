## Summary

New `ChartLineAdl` UI primitive at
`web/src/components/ui/chart-line-adl.tsx` -- a pure-SVG line chart
with an Accumulation Distribution Line cumulative volume-flow panel.
The top panel renders the close price line; the bottom panel renders
the ADL line on its own data-driven y-axis with a dashed zero
reference line and markers coloured by money flow direction.

## Indicator math

Marc Chaikin's Accumulation/Distribution Line:

```
MFM[i] = ((close - low) - (high - close)) / (high - low)   in [-1, +1]
MFV[i] = MFM[i] * volume[i]
ADL[i] = ADL[i-1] + MFV[i]    with ADL[0] = MFV[0]
```

`computeLineAdlMfm` is the Money Flow Multiplier -- where the close
sits within the bar's high-low range: a close at the high reads `+1`,
at the low `-1`, at the midpoint `0`; a zero-range bar reads `0`.
`computeLineAdl` scales each multiplier by volume into Money Flow
Volume and accumulates it into a running cumulative total. Unlike the
moving-average primitives the ADL has **no warm-up** -- it is defined
from the first bar.

### Worked fixture

`high = 10`, `low = 0` throughout; `close` at high / low / mid gives
a multiplier of `+1 / -1 / 0`:

```
close   = [10, 0, 5, 10, 0]    volume = [100, 100, 100, 200, 50]
mfm     = [1, -1, 0, 1, -1]
mfv     = mfm * volume = [100, -100, 0, 200, -50]
adl     = cumulative   = [100, 0, 0, 200, 150]
```

`adlFinal = 150`, `adlMin = 0`, `adlMax = 200`, two accumulation bars
and two distribution bars.

## API

- `getLineAdlFinitePoints(points)` -- drop points with a non-finite
  `x`, `high`, `low`, `close` or `volume`.
- `computeLineAdlMfm(highs, lows, closes)` -- the Money Flow
  Multiplier series.
- `computeLineAdl(highs, lows, closes, volumes)` -- returns
  `{ mfm, mfv, adl }`.
- `runLineAdl(points)` -- sort finite points by `x`, compute the
  series, return `{ series, mfm, mfv, adl, samples, adlFinal,
  adlMin, adlMax, accumulationCount, distributionCount, ok }`. Each
  sample carries an `accumulation` / `distribution` / `neutral` flow.
  `ok` is false for fewer than two points. The ADL takes no period
  parameter.
- `computeLineAdlLayout(options)` -- stack a price panel above an ADL
  panel with a data-driven y-domain. Returns panels, ticks, the price
  path, the ADL path, flow-coloured markers, and the zero reference
  line `y` plus a `zeroInRange` flag.
- `describeLineAdlChart(data)` -- ARIA description.

## Component

`ChartLineAdl` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge, a two-series legend (Price / ADL) with toggle, a dashed zero
reference line, the ADL line, markers coloured by flow (accumulation
green, distribution red), a hover/focus tooltip exposing the high,
low, close, volume, money flow volume, ADL, and flow, and a
`motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs chart-line-obv** -- On-Balance Volume adds or subtracts the
  *whole* volume by the close's up/down direction. The ADL weights
  the volume by the *Money Flow Multiplier*: a close near the high
  accumulates more than a close near the low even on identical
  volume, and a bar that closes mid-range contributes near zero
  regardless of how large its volume is.
- **vs chart-line-mfi / chart-line-emv** -- the Money Flow Index is a
  *bounded 0-100* oscillator and the Ease of Movement *divides* by
  volume. The ADL is an *unbounded cumulative* line that *multiplies*
  the multiplier by volume.
- **no warm-up** -- unlike the moving-average overlays, the ADL is
  defined from the first bar (it seeds with the first money flow
  volume), so every sample has an ADL reading.

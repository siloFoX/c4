## API

```ts
type ChangelogEntryKind =
  | 'feat' | 'fix' | 'chore' | 'docs'
  | 'refactor' | 'test' | 'perf' | 'breaking';

interface ChangelogEntry {
  kind: ChangelogEntryKind;
  message: string;
  scope?: string;
  prNumber?: number;
  prHref?: string;
}

interface ChangelogVersion {
  version: string;
  date?: string;
  title?: string;
  entries: ChangelogEntry[];
  body?: string;          // markdown
  diffHref?: string;      // URL to the diff for this version
}

interface ChangelogViewerProps {
  versions: ChangelogVersion[];
  defaultExpandedVersions?: readonly string[];
  expandedVersions?: readonly string[];  // controlled
  onExpandedChange?: (versions: string[]) => void;
  ariaLabel?: string;                    // default 'Changelog'
  className?: string;
  renderMarkdown?: (text: string) => ReactNode;
  showDiffLinks?: boolean;               // default true
  emptyState?: ReactNode;                // default 'No release history'
  expandFirstByDefault?: boolean;        // default true
}
```

## Behaviour

- Versions render newest-first via
  `sortVersionsDescending` (date parsed via `Date.parse`;
  ties fall back to lexicographic version order so `1.0.2`
  sorts before `1.0.1`).
- Each version renders as a `<article>` with a clickable
  header that toggles a body region; the chevron icon swaps
  between collapsed + expanded.
- Initial expansion:
  - `expandedVersions` (when supplied) is the controlled
    truth.
  - `defaultExpandedVersions` seeds the uncontrolled state
    (overrides `expandFirstByDefault`).
  - Otherwise `expandFirstByDefault=true` (default) opens
    the newest version on mount.
- Body groups entries by kind in the canonical order
  `breaking > feat > fix > perf > refactor > docs > test >
  chore`; empty kind buckets do not render their section.
- Per-entry row: kind badge + optional scope + message +
  optional PR link. `prHref` swaps the PR pill from a
  `<span>` to an `<a>`.
- Diff link slot: renders an anchor with
  `aria-label="Diff for <version>"` when both
  `showDiffLinks` (default true) and `version.diffHref` are
  present.
- Markdown body: when `renderMarkdown` is supplied the host
  renderer takes over. Without it the body falls back to a
  `<pre>` block so the raw text remains readable; the wrapper
  is marked
  `data-section="changelog-viewer-markdown-fallback"` so
  adopters can target it.

## Pure helpers (exported)

```ts
CHANGELOG_ENTRY_KINDS = [
  'breaking', 'feat', 'fix', 'perf',
  'refactor', 'docs', 'test', 'chore',
]
CHANGELOG_ENTRY_LABELS = { breaking: 'Breaking', feat: 'Features', ... }
groupEntriesByKind(entries): Record<kind, ChangelogEntry[]>
formatVersionTitle(version, date?): string
sortVersionsDescending(versions): ChangelogVersion[]
```

- `groupEntriesByKind` is a pure partition; unknown kinds
  fall into the `chore` bucket so adopters can pass free-form
  data without losing entries.
- `formatVersionTitle` returns `vX.Y.Z` and appends
  ` - <date>` when supplied.
- `sortVersionsDescending` is non-mutating; copies the input
  array before sorting.

## ARIA + data attributes

- Region: `role="region"` + `aria-label`.
- Toggle button per version: `aria-expanded` mirrors state,
  `aria-controls` points to the body region id.
- Body region: `aria-labelledby` points to the toggle id.
- Diff link: `aria-label="Diff for <version>"`.
- `data-section` on every node:
  - `changelog-viewer` (root region)
  - `changelog-viewer-version` (per-version article)
  - `changelog-viewer-header`
  - `changelog-viewer-toggle`

  - `changelog-viewer-diff`
  - `changelog-viewer-body`

  mirrors `data-kind`. Each entry mirrors `data-entry-kind`.

## Tests (44 cases)

- All three pure helpers: `groupEntriesByKind` partitions +
  unknown-kind fallback into `chore`; `formatVersionTitle`
  with / without date; `sortVersionsDescending` date sort +
  lexicographic tiebreak + non-mutation.
- Constants -- kind list + labels.
- Default + custom `ariaLabel`.
- One article per version; versions render newest-first.
- Version label includes `v` prefix + date.
- Version title slot renders.
- `expandFirstByDefault` default and false override.
- `defaultExpandedVersions` seeds the open set (overrides
  `expandFirstByDefault`).
- Toggle button flips expand state; `aria-expanded` mirrors.
- `onExpandedChange` fires with the new list.
- Controlled `expandedVersions` pins the rendered state
  through rerender.
- Expanded body renders one group per non-empty kind in the
  canonical order; empty kinds skipped.
- Per-entry kind badge with `data-entry-kind`; scope
  renders; PR number renders as link (with prHref) or plain
  span (without).
- Diff link renders when `diffHref` + `showDiffLinks`
  (default); hidden when `showDiffLinks=false`; hidden when
  no `diffHref`.
- Markdown body falls back to `<pre>` when no
  `renderMarkdown`; `renderMarkdown` is invoked with the body
  string when supplied; `renderMarkdown` is NOT invoked when
  the version has no body.
- Empty `versions` renders the empty state (default +
  custom).
- Root `data-version-count` reflects the count.
- Body `aria-labelledby` + toggle `aria-controls` are
  reciprocal.
- Stable `displayName` + `forwardRef` to the region.
- Per-entry messages render inside the expanded body.

## Pairs with existing primitives

- `<AnnouncementBanner>` (11.427) for the most-recent
  release callout.
- `<CodeBlock>` for inline code references inside the body
  when the host wires a markdown renderer.
- `<Accordion>` underlies the expand/collapse pattern; the
  changelog viewer bakes in the per-version chevron + ARIA
  wiring so adopters do not have to glue it.

## Out of scope

- Built-in markdown engine (host wires `renderMarkdown` --
  the primitive stays renderer-agnostic so projects can
  choose `marked` / `markdown-it` / `mdast` / etc).
- Filtering by kind / date range (host filters `versions`
  before passing in).
- Pagination / virtualisation for very long histories (host
  wraps with `<InfiniteScroll>` or `<VirtualizedList>`).
- Live release feed (host owns the data source; the
  primitive renders a static array snapshot).

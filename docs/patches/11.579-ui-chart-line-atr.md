## Summary

New `ChartLineAtr` UI primitive at
`web/src/components/ui/chart-line-atr.tsx` -- a pure-SVG line chart
with an Average True Range (ATR) volatility sub-panel. The price line
occupies the top panel; the bottom panel plots the ATR as a standalone
volatility study, with the raw per-period true range drawn behind it as
faint bottom-anchored bars. Both panels share one x-axis.

## Indicator math

### True range

`computeLineAtrTrueRanges(values)` returns the per-period true range.
For a single-value series the true range is the absolute period-over-
period change:

```
trueRange[i] = |values[i] - values[i-1]|   for i >= 1
trueRange[0] = null                        (no prior value)
```

### Average True Range (Welles Wilder)

`computeLineAtr(values, period)` is Wilder's ATR. The first ATR, at
index `period`, seeds with the simple mean of the first `period` true
ranges; each later value uses Wilder smoothing:

```
atr[period] = mean(trueRange[1..period])
atr[i]      = (atr[i-1] * (period - 1) + trueRange[i]) / period
```

Indices before the window fills read `null`. The ATR is non-negative
throughout, since every true range is an absolute value.

### Worked fixture

`values = [10, 20, 30, 20, 40, 30]`, `period = 2`:

```
true range = [null, 10, 10, 10, 20, 10]
atr[2] = mean(10, 10)        = 10
atr[3] = (10*1 + 10) / 2     = 10
atr[4] = (10*1 + 20) / 2     = 15
atr[5] = (15*1 + 10) / 2     = 12.5
atr        = [null, null, 10, 10, 15, 12.5]
atrFinal = 12.5   atrMax = 15
```

## API

- `getLineAtrFinitePoints(points)` -- drop points with a non-finite `x`
  or `value`.
- `normalizeLineAtrPeriod(period, fallback)` -- coerce a period to a
  positive integer; non-finite or sub-1 falls back, fractional floors.
- `computeLineAtrTrueRanges(values)` -- per-period true range series.
- `computeLineAtr(values, period)` -- Wilder ATR series.
- `runLineAtr(points, options?)` -- sort finite points by `x`, compute
  true range + ATR, return `{ series, period, trueRange, atr, samples,
  atrFinal, atrMax, ok }`. `ok` is false for fewer than two points.
- `computeLineAtrLayout(options)` -- stack a price panel above an ATR
  panel sharing one x-axis. The ATR panel y-range floors at 0 and spans
  the larger of the peak ATR and peak true range so the faint
  true-range bars and the ATR line share one scale. Returns panels,
  ticks, the price path, ATR path, ATR markers, and true-range bars.
- `describeLineAtrChart(data, options?)` -- ARIA description string.

## Component

`ChartLineAtr` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge (period + peak ATR), a three-series legend (Price / ATR / True
Range) with toggle, a hover tooltip exposing the price, true range, and
ATR for the hovered period, and a `motion-safe:animate-fade-in`
entrance. Every node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-keltner** -- Keltner uses the ATR only to *scale a
  band envelope* around an EMA centre overlaid on the price.
  chart-line-atr displays the ATR *itself* as a volatility line in a
  dedicated sub-panel -- the volatility study, not a derived band.
- **vs chart-line-bollinger** -- Bollinger bands measure dispersion via
  the rolling standard deviation; ATR measures the typical absolute
  move. Different statistic, different presentation (band overlay vs
  sub-panel line).
- **vs chart-line-macd / chart-line-rsi / chart-line-stochastic** --
  those sub-panels plot momentum / range-position oscillators;
  chart-line-atr plots a non-negative volatility magnitude.

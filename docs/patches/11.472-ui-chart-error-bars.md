## Summary

Add a new `<ChartErrorBars>` UI primitive in
`web/src/components/ui/chart-error-bars.tsx` -- pure-SVG
bar / scatter chart with vertical and (in scatter mode)
horizontal error-bar overlays. Each datum carries a
required `y` value and optional `lower`/`upper` (or a
single symmetric `error`) for the y-axis confidence
interval; in scatter mode it can also carry an `x` value
plus optional `xLower`/`xUpper`/`xError` for the x-axis
CI. Error bars render as T-shaped whiskers: a stem
connecting `lower` to `upper` plus a cap at each end.
Mode toggles between bar (categorical-x with vertical
T whiskers above/below the bar) and scatter (numeric-x,
numeric-y with whiskers in either or both axes).

## API

```ts
type ChartErrorBarsMode = 'bar' | 'scatter';

interface ChartErrorBarsPoint {
  id: string;
  label?: string;
  x?: number;
  y: number;
  category?: string;
  lower?: number;
  upper?: number;
  error?: number;
  xLower?: number;
  xUpper?: number;
  xError?: number;
  color?: string;
}

interface ChartErrorBarsProps {
  data: readonly ChartErrorBarsPoint[];
  mode?: ChartErrorBarsMode;       // default 'bar'
  width?: number;                  // default 560
  height?: number;                 // default 320
  padding?: number;                // default 40
  tickCount?: number;              // default 5
  pointRadius?: number;            // default 4 (scatter dot radius)
  whiskerLength?: number;          // default 6 (cap width)
  barGap?: number;                 // default 12
  xMin?: number;                   // controlled bound overrides
  xMax?: number;
  yMin?: number;
  yMax?: number;
  showTooltip?: boolean;           // default true
  showAxisTicks?: boolean;         // default true
  showGrid?: boolean;              // default true
  showCategoryLabels?: boolean;    // default true (bar mode)
  showXErrorBars?: boolean;        // default true (scatter mode)
  showYErrorBars?: boolean;        // default true
  animate?: boolean;               // default true
  className?: string;
  ariaLabel?: string;              // default 'Error bar chart'
  ariaDescription?: string;
  primaryColor?: string;           // default '#2563eb'
  errorColor?: string;             // default '#1e40af'
  formatValue?: (v: number) => string;
  formatCategory?: (label: string, index: number) => string;
  onPointClick?: (args: {
    point: ChartErrorBarsPoint;

  }) => void;
  style?: CSSProperties;
}
```

## Behaviour

- Renders `<div role="region">` wrapping
  `<svg role="img">` with the same `aria-label`.
- CI resolution (per axis):
  - if both `lower` + `upper` are finite numbers, they
    are used verbatim;
  - else if `error` is a finite, non-negative number,
    `lower = value - error` and `upper = value + error`
    (symmetric).
  - if both forms are absent, the CI is `[value,
    value]` and `hasError = false`;
  - if the resulting `lower > upper`, they swap so the
    range is always non-degenerate;
  - non-finite y / negative error skip CI.
- Bar mode (default):
  - categorical x; bar slot per data index across the
    inner canvas with `barGap` between slots.
  - y axis baseline is always 0; bars grow from the
    baseline towards `value`. `getErrorBarBounds` pulls
    `yMin` down to 0 when all values are positive.
  - vertical whisker centered on the bar with a cap at
    each CI endpoint.
- Scatter mode:
  - numeric x + numeric y; auto-bounds expand to
    cover x CI ranges as well.
  - vertical whisker for y CI; horizontal whisker for
    x CI (both render only if the corresponding CI is
    present and not suppressed).
- Bounds: auto-derived; can be overridden with
  `xMin` / `xMax` / `yMin` / `yMax`. Collapsed x/y
  range expands by +/-0.5.
- Hover or focus on a mark (bar or scatter point)
  sets the hovered id; the tooltip surfaces label,
  the value (or `(x, y)` in scatter mode), and the
  formatted y/x CI ranges when present.
- Mount animation:
  `motion-safe:animate-fade-in` per mark group.
  `animate=false` removes the class.
- Default ARIA description: `"Error bar chart
  (<mode>) with N points, M with confidence intervals.
  y range <yMin> to <yMax>."` -- empty / all-non-finite
  returns `"No data"`; `ariaDescription` overrides.

## ARIA + data attributes

- Root: `<div role="region">` + `aria-label`.
- SVG: `role="img"` + same `aria-label`.
- Description: sr-only span +
  `data-section="chart-error-bars-aria-desc"`.
- Each mark (bar or scatter point):
  `role="graphics-symbol"` + `tabIndex={0}`.
  aria-label includes the value and any CI ranges.
- `data-section` on every node:
  chart-error-bars / -canvas / -aria-desc / -svg /
  -grid / -grid-line / -axes / -axis / -ticks / -tick /
  -tick-label / -categories / -category-label /

  -tooltip-y-range / -tooltip-x-range.

## Pairs with existing primitives

- `<ChartBar>` (11.438) -- single-series bar
  sibling without CI.
- `<ChartScatter>` (11.444) -- two-axis dot
  sibling without CI.
- `<ChartBoxplot>` (11.455) -- distributional
  alternative for full quartile summaries.
- `<ChartBullet>` (11.442) -- target-vs-actual
  alternative.

## Out of scope

- Asymmetric per-axis percentile bands (5/25/75/95) --
  v1 is a single CI per axis; use `<ChartBoxplot>`
  for full distributional summaries.
- Stacked / grouped error bars within one slot --
  host stacks `<ChartErrorBars>` instances or uses
  `<ChartGroupedBar>` + a custom whisker layer.
- Animated reflow on data change (snap v1; mount
  fade only).
- Tukey / Studentized bars or per-cap variants
  (square cap, double cap, etc.) -- v1 is a single
  cap style.

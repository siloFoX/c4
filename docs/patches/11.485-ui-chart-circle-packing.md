## Summary

Add a new `<ChartCirclePacking>` UI primitive in
`web/src/components/ui/chart-circle-packing.tsx` --
pure-SVG **hierarchical circle packing**. Every node
in the input hierarchy becomes a circle; parent
circles enclose their children; child radii are
proportional to `sqrt(value)` so circle *area* tracks
the value (the standard area-honest encoding). Distinct
from `<ChartTreemap>` (11.444) /
`<ChartTreemapSquarified>` (11.479) (rectangular
nesting), `<ChartSunburst>` (11.483) (radial banded),
and `<ChartIcicle>` (11.484) (rectangular banded) --
this primitive is the **packed-circles** layout that
emphasises hierarchy + relative size at a glance.

## Behaviour

- Renders `<div role="region">` wrapping
  `<svg role="img">` with the same `aria-label`.
- Value resolution
  (`getCirclePackingNodeValue`): leaves use their
  explicit `value`; non-leaf nodes prefer the sum of
  their children's values, falling back to the
  explicit value only when child totals collapse to
  0. Childless without value -> 0.
- Layout (`computeCirclePackingLayout`):
  - Root circle is placed at `(cx, cy)` with the
    outer radius.
  - For each parent, children are packed via the
    `packCirclesFrontChain` algorithm (Wang et al.
    2006 style):
    1. Place first two children tangent to each
       other.
    2. Place the third tangent to the first two.
    3. For each subsequent child, walk the
       circular "front" of boundary circles and
       try every adjacent pair `(a, b)`; place the
       new circle tangent to both at the position
       closest to the centroid; reject if it
       intersects any other front circle; otherwise
       insert into the front between `a` and `b`.
  - The packed cluster's minimum enclosing circle
    is computed
    (`getCirclePackingEnclosingCircle`), then the
    whole cluster is translated + scaled so it
    sits inside the parent's circle with
    `childPadding` clearance on the boundary.
- Non-positive / non-finite values are dropped from
  the layout. Non-overlap is enforced by the
  packing algorithm (siblings non-overlapping
  within ~1 px tolerance, every child fits inside
  its parent within ~1 px tolerance).
- Labels: leaves (and the focused root) get labels
  by default; setting `showLeafLabelsOnly=false`
  labels every visible circle. Labels paint at the
  centroid; font size scales with the radius
  (clamped to [9, 13] px).
- Hover/focus on a circle opens a tooltip with the
  full breadcrumb path, formatted value, and share
  against the root.
- Mount animation:
  `motion-safe:animate-fade-in` per circle.
  `animate=false` removes the class.
- Default ARIA description: `"Circle packing chart
  with N nodes across L levels, total <total>."`
  Null root / zero total -> `"No data"`.

```ts
getCirclePackingEnclosingCircle(circles): { x, y, r }
computeCirclePackingLayout(input): { circles, flat, rootValue }
describeCirclePackingChart(root, formatValue?): string
```

- `packCirclesFrontChain` accepts a flat array of
  radii and returns one `{ x, y, r }` per input
  (zero placeholder for non-positive / non-finite
  inputs). Sibling circles are mutually non-
  overlapping within numeric tolerance.
- `getCirclePackingEnclosingCircle` returns a
  bounding circle that encloses every input
  circle.

  + parentId + path correct; null -> [].
- `packCirclesFrontChain`:
  - empty -> [];
  - single radius -> at origin;
  - two radii -> tangent placement (`d = r0 + r1`);
  - three equal radii -> mutually non-overlapping;
  - many circles -> all non-overlapping (within
    epsilon);
  - non-positive / non-finite -> zero-radius
    placeholders.
- `getCirclePackingEnclosingCircle`:
  - empty -> zero;
  - single -> same as input;
  - encloses every input circle.
- `computeCirclePackingLayout`:
  - null / non-positive radius -> empty;
  - 7 circles for sample tree;
  - root sits at centre with outer radius;
  - **every child fits inside its parent** (within
    ~1 px tolerance);
  - **siblings do not overlap** (within ~1 px
    tolerance);
  - isLeaf=true for leaves, false for parents;
  - share = value / rootValue;
  - per-node color override beats palette.

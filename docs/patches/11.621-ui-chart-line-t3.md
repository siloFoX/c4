## Summary

New `ChartLineT3` UI primitive at
`web/src/components/ui/chart-line-t3.tsx` -- a pure-SVG line chart
with a Tillson T3 Moving Average overlay using a cascaded smoothing
factor.

## Indicator math

Tim Tillson's T3 Moving Average:

```
ema1 = EMA(value, period)
ema2 = EMA(ema1, period)   ... through ema6
c1 = -v^3
c2 = 3v^2 + 3v^3
c3 = -6v^2 - 3v - 3v^3
c4 = 1 + 3v + 3v^2 + v^3
T3 = c1*ema6 + c2*ema5 + c3*ema4 + c4*ema3
```

The price is run through a **cascade of six exponential moving
averages**, and four of them (`ema3..ema6`) are blended with four
coefficients that are cubic polynomials in the **volume factor** `v`
-- the cascaded smoothing factor. The coefficients always sum to 1,
so the T3 of a constant series is that constant. The volume factor
dials how responsive the average is: at `v = 0` the blend collapses
to a plain triple-nested EMA (`c1 = c2 = c3 = 0`, `c4 = 1`), while a
higher factor sharpens the response.

The EMA helper is the project's null-tolerant, SMA-seeded EMA.
EMA1 is defined from index `period - 1`; each later nesting needs
another `period` defined values, so EMA6 -- and therefore the
**T3 is defined from index `6 * period - 6` onward**.

### Worked fixture

`values` is a linear ramp of 15 points (`value[i] = 3 + 3i`),
`period 3`, `vfactor 0.5`. With period 3 the EMA multiplier is
exactly `0.5` and with `v = 0.5` the four coefficients are exact
dyadic numbers:

```
c1 = -0.125, c2 = 1.125, c3 = -3.375, c4 = 3.375   (sum = 1)
```

so the six-EMA cascade and the blend stay bit-exact. Each EMA stage
lags the ramp by 3, so `ema-k = value - 3k`, and the T3 settles to
`value - 4.5` from index `6*3-6 = 12` onward:

```
t3 = [.,.,.,.,.,.,.,.,.,.,.,., 34.5, 37.5, 40.5]
```

Every value is exact, so the tests assert the series with `toBe`.

## API

- `getLineT3FinitePoints(points)` -- drop points with a non-finite
  `x` or `value`.
- `normalizeLineT3Period(period, fallback)` -- coerce to a positive
  integer; non-finite or sub-1 falls back, fractional floors.
- `normalizeLineT3Vfactor(vfactor, fallback)` -- coerce the volume
  factor to `[0, 1]`; non-finite falls back, out-of-range clamps.
- `lineT3Coefficients(vfactor)` -- the four cubic coefficients
  `{ c1, c2, c3, c4 }`; they sum to 1.
- `computeLineT3Ema(src, period)` -- the null-tolerant SMA-seeded
  EMA series.
- `computeLineT3(values, period, vfactor)` -- returns
  `{ ema1, ema2, ema3, ema4, ema5, ema6, t3 }`.
- `runLineT3(points, options?)` -- sort finite points by `x`,
  compute the series, return `{ series, period, vfactor, ema1..ema6,
  t3, samples, t3Final, t3Min, t3Max, aboveCount, belowCount, ok }`.
  Each sample carries the price's `above` / `below` / `on` position
  versus the T3. `ok` is false for fewer than two points.
- `computeLineT3Layout(options)` -- project the price and T3 onto
  one shared panel. Returns the panel, ticks, the price path, the
  T3 path, and T3 markers.
- `describeLineT3Chart(data, options?)` -- ARIA description.

## Component

`ChartLineT3` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, axis ticks and grid, a
config badge calling out the period and the volume factor, a
two-series legend (Price / T3) with toggle, a hover/focus tooltip
exposing the value, T3 and position, and a
`motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs chart-line-dema / chart-line-tema** -- the DEMA combines two
  EMAs (`2*EMA - EMA(EMA)`) and the TEMA three
  (`3*EMA - 3*EMA(EMA) + EMA^3`) with *fixed integer* coefficients.
  The T3 uses a *six-EMA cascade* and *four volume-factor*
  coefficients -- a generalised, tunable smoother. At `vfactor = 0`
  the T3 collapses to a plain triple-nested EMA.
- **vs chart-line-zlema** -- the ZLEMA removes lag by feeding a
  de-lagged input through one EMA. The T3 leaves the input
  untouched and tunes lag with the volume-factor blend of a deeper
  EMA cascade.
- **vs chart-line-kama / chart-line-vidya** -- those *adapt* their
  smoothing bar to bar from market conditions. The T3 applies a
  *fixed* cascade and a fixed volume factor.

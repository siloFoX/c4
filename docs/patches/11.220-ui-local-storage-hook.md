# 11.220 -- useLocalStorage hook + imperative helpers

Component-scope-only addition. No daemon-side change and no `c4` CLI
surface change.

## Why

The web bundle had grown a long tail of near-identical
`localStorage`-touching helpers, each with its own
`typeof window === 'undefined'` guard, its own `try { JSON.parse }`
fallback, and its own quota-error swallow. Some of them also
re-implemented cross-tab sync by hand (the feature-flags / keyboard
bindings / ab-variant / health-layout dispatch contracts), but the
rest -- command palette recents, command palette favorites, column
picker visibility, grid-debug overlay flag, locale persistence,
onboarding tour seen flag -- did not. Every new feature that wanted
"persist this across reloads" copy-pasted the same five lines.

Patch 11.220 collapses that into one well-tested primitive and
swaps the simple JSON-payload sites onto it. Sites that already
maintain their own dispatch contract (feature-flags,
keyboard-bindings, ab-variant, use-health-layout) are deliberately
left alone -- their custom event names are part of the public
contract that other modules subscribe to.

## Surface

### Hook

```ts
useLocalStorage<T>(key: string, defaultValue: T):
  [value: T, setValue: (v: T) => void, remove: () => void]
```

- Reads the initial value through a lazy `useState` initializer so
  the first render is the only render that touches `localStorage`.
- Subscribes to the browser-native `storage` event so other tabs
  writing the same key update this tab live.
- `setValue` and `remove` re-dispatch a synthetic `StorageEvent` on
  `window` so two `useLocalStorage` calls for the same key in the
  same tab also stay in sync (the browser does not fire `storage`
  for the originating window).
- Malformed JSON falls back to `defaultValue` instead of throwing.
- A `key` change between renders re-seeds state from the new key's
  current payload (or `defaultValue` if absent).
- SSR-safe: `typeof window` guards every browser touch.

### Imperative helpers

```ts
getLocalStorage<T>(key: string, defaultValue: T): T
setLocalStorage<T>(key: string, value: T): void
removeLocalStorage(key: string): void
```

Same JSON contract as the hook. `setLocalStorage` /
`removeLocalStorage` dispatch the synthetic `StorageEvent` so any
mounted hooks for that key wake up immediately. Quota errors and
JSON-stringify errors are swallowed silently -- the call site is
free to assume "best effort, never throws".

## Storage format

Every value is `JSON.stringify`-d on write and `JSON.parse`-d on
read. That keeps the on-disk payload self-describing (numbers come
back as numbers, booleans as booleans, arrays as arrays) and is
forward-compatible with structured values. Sites that historically
stored a raw string (e.g. the `c4.locale` key, the
`c4:grid-debug:visible` `'1'` / `'0'` flag) keep their bespoke
inline `getItem` / `setItem` for now -- swapping them onto
`useLocalStorage` would change the on-disk encoding from `1` to
`"1"` and break their existing test seeds. They are tracked as
follow-ups, not part of 11.220.

## Adoption inventory (this patch)

Six inline `localStorage.{get,set,remove}Item` callsites swapped
across two files. All of them already used JSON-encoded arrays so
the on-disk payload is byte-identical:

| File                                       | Helpers swapped                                            | Keys                                              |
| ------------------------------------------ | ---------------------------------------------------------- | ------------------------------------------------- |
| `web/src/components/CommandPalette.tsx`    | `loadRecent`, `saveRecent`, `loadFavorites`, `saveFavorites` | `cmdk:recent`, `c4:cmdk:favorites`               |
| `web/src/components/ui/column-picker.tsx`  | `readFromStorage`, `writeToStorage`                          | caller-provided (`storageKey` prop)              |

The existing `CommandPalette.test.tsx` and `column-picker.test.tsx`
cases continue to pass against the new helpers because the storage
payload format is unchanged.

## Sites intentionally left alone

These keep their bespoke `localStorage` access because they already
publish a custom event contract that other modules subscribe to,
or because the on-disk encoding is not JSON:

- `web/src/lib/feature-flags.ts` -- `c4:feature-flags-changed`
  CustomEvent bus is part of the public flag-change contract.
- `web/src/lib/keyboard-bindings.ts` -- `c4:keyboard-bindings-changed`
  CustomEvent fan-out.
- `web/src/lib/ab-variant.ts` -- bucketing helper with its own
  deterministic hash + change event.
- `web/src/lib/use-health-layout.ts` -- has its own dispatch
  contract used by the health pages.
- `web/src/components/dev/GridDebugOverlay.tsx` -- raw `'1'` /
  `'0'` payload, would break existing tests if we wrapped it in
  JSON.
- `web/src/lib/i18n.ts` -- raw locale code (`'en'` / `'ko'`)
  payload, similar reason.

## Test coverage

`web/src/hooks/use-local-storage.test.tsx` -- 11 cases:

1. `useLocalStorage` returns `defaultValue` when storage is empty.
2. Returns the parsed JSON value when storage is pre-seeded.
3. `setValue` updates state and writes JSON to storage.
4. `remove()` clears the key and restores `defaultValue`.
5. Malformed JSON falls back to `defaultValue` (hook + helper).
6. A real cross-tab `storage` event for the same key updates state.
7. A `storage` event for a different key is ignored.
8. Two hooks for the same key stay in sync within one tab
   (covers the synthetic-event re-dispatch contract).
9. A `key` change between renders re-seeds from the new key.
10. `getLocalStorage` / `setLocalStorage` / `removeLocalStorage`
    round-trip and notify mounted hooks.
11. SSR-safe smoke: with `globalThis.window` undefined the helpers
    return `defaultValue` / no-op without throwing.

## Migration notes (future patches)

- `dev/GridDebugOverlay.tsx` and `lib/i18n.ts` could move onto the
  hook in a follow-up that also migrates their stored payload from
  raw strings to JSON (one-time read-and-rewrite shim).
- The four "already has a custom event" sites can keep their
  custom event names but could share the JSON-parse / quota-swallow
  plumbing if the hook learns an "event channel override" option.
  Out of scope for 11.220.

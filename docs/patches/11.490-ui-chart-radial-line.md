## Summary

Add a new `<ChartRadialLine>` UI primitive in
`web/src/components/ui/chart-radial-line.tsx` -- pure-SVG
**radial line / spiral plot** for cyclic / periodic
time-series data. Two modes:

1. **cyclic** (default): the line is laid out on polar
   coordinates with the angle encoding **position
   within a cycle** (0..2pi over `cycleLength`
   samples) and the radius encoding value. The
   resulting line closes back to the starting point so
   the cycle is visually obvious. Ideal for
   hour-of-day / day-of-week / month-of-year
   periodicity.
2. **spiral**: the angle accumulates continuously over
   multiple cycles (so cycle 1 sample 0 is at
   `startAngle + 2pi`, cycle 2 sample 0 at
   `startAngle + 4pi`, etc.) and the radius walks
   outward over the full sample range while still
   modulating with the value. Ideal for showing both
   the within-cycle phase and the long-term trend
   across many cycles.

Distinct from `<ChartRadialBar>` (11.452) (concentric
arc bars), `<ChartPolarArea>` (11.478) (equal-angle
wedges), `<ChartRadar>` (11.445) (multi-axis polygon
for categorical dimensions), and
`<ChartRadialStackedBar>` (11.489) (rings of stacks):
this primitive is the polar line variant for
periodic data.

## Behaviour

- Renders `<div role="region">` wrapping
  `<svg role="img">` with the same `aria-label`.
- Layout (`computeRadialLineLayout`):
  - `radiusForValue`:
    - `cyclic`: `innerRadius + (value/maxValue) *
      (outerRadius - innerRadius)`.
    - `spiral`: weighted combination -- 40% from
      value ratio + 60% from sample position along
      the full sample range. The result is that even
      a flat input still spirals outward with the
      sample index, while values modulate the
      radius locally for shape.
  - `angleFor`:
    - `cyclic`: `startAngle + (positionInCycle /
      cycleLength) * 2pi` (wraps every cycle).
    - `spiral`: `startAngle + (sampleIndex /
      cycleLength) * 2pi` (accumulates).
  - Non-finite values break the painted line
    (path emits a new `M`) and the offending point
    collapses to the inner radius.
  - `closeCyclic=true` (default, cyclic mode only)
    appends a line back to the first finite point
    and a closing `Z` so the cycle is visually
    sealed; that closed string is also exposed as
    `fillPath` when `showFill=true`.
- Axis spokes + axis labels: one per
  `cycleLength` step around the perimeter. Labels
  default to the sample index; supply `axisLabels`
  (length = cycleLength) for human-readable text
  (e.g. hour-of-day, month name); `formatAxis`
  rewrites further.
- Rings: one concentric dashed ring per non-zero
  tick value from `getRadialLineTicks(maxValue,
  tickCount)`.
- Hover/focus on a point sets the hovered key
  (`seriesId::sampleIndex`) and opens a tooltip
  with the series label, position (axis label or
  `position N`, plus `cycle K` in spiral mode),
  and formatted value.
- Mount animation:
  `motion-safe:animate-fade-in` per series group.
  `animate=false` removes the class.
- Default ARIA description: `"Radial line chart
  (<mode>) with N series, cycle length L, K
  cycles of data, peak value <peak>."` Empty /
  all-hidden / non-positive cycle -> `"No data"`.

## Summary

New `ChartLineVwma` UI primitive at
`web/src/components/ui/chart-line-vwma.tsx` -- a pure-SVG line
chart with a Volume Weighted Moving Average overlay weighting
each price by its bar volume.

## Indicator math

The Volume Weighted Moving Average (VWMA) is a moving average
where each price is weighted by its bar's traded volume.

`computeLineVwma(values, volumes, period)` -- for each bar the
trailing window of `period` bars is averaged:

```
VWMA[i] = sum(price[j] * volume[j]) / sum(volume[j])
```

over the window. A high-volume bar pulls the average toward its
price more strongly than a low-volume bar, so the VWMA tracks
where the trading actually happened. A window whose total volume
is zero has an undefined `0/0` average and is null, as are bars
before the window is full.

This is the first chart primitive to consume a per-bar `volume`
field: `ChartLineVwmaPoint` is `{ x, value, volume }`.

### Worked fixture

The fixture (`period 2`) is hand-tuned so every window divides
cleanly:

```
prices  [10, 20, 40, 10, 16,  4, 34,  7]
volumes [ 1,  3,  1,  2,  2,  2,  1,  2]
vwma    [ .,17.5, 25, 20, 13, 10, 14, 16]
```

The first window `[10, 20]` with volumes `[1, 3]` gives
`(10*1 + 20*3) / (1+3) = 70/4 = 17.5` -- pulled toward 20 by its
higher volume, **not** the simple average 15. When the volumes
are equal the VWMA reduces exactly to the simple moving average.

## API

- `getLineVwmaFinitePoints(points)` -- drop points with a
  non-finite `x` / `value` / `volume` or a negative `volume`.
- `normalizeLineVwmaPeriod(period, fallback)` -- coerce to a
  positive integer.
- `computeLineVwma(values, volumes, period)` -- the
  volume-weighted moving average series; null through the
  warm-up and for zero-volume windows.
- `runLineVwma(points, options?)` -- sort finite points by `x`,
  compute the VWMA, return `{ series, period, vwma, samples,
  vwmaFinal, vwmaMin, vwmaMax, aboveCount, belowCount, ok }`.
  Each sample is classified `above` / `below` / `on` the VWMA.
  `ok` is false for fewer than two points.
- `computeLineVwmaLayout(options)` -- project the price and VWMA
  paths onto one shared panel with a y-domain spanning both.
- `describeLineVwmaChart(data, options?)` -- ARIA description.

## Component

`ChartLineVwma` is a `forwardRef` region (`role="region"`) with
an ARIA-linked description, a `role="img"` SVG, axis ticks and
grid, a config badge showing the period, a two-series legend
(Price / VWMA) with toggle, a hover/focus tooltip exposing the
value, bar volume, VWMA and price position, and a
`motion-safe:animate-fade-in` entrance. It consumes
`{ x, value, volume }` points. The price line and the VWMA line
share one panel; the VWMA carries per-bar markers. Every node
carries a `data-section` attribute.

## Distinctness

- **vs a simple moving average** -- the SMA weights every bar in
  the window equally. The VWMA weights each bar by its traded
  volume, so it tracks where the volume concentrated. When all
  volumes are equal the VWMA reduces exactly to the SMA.
- **vs the EMA / WMA / smoothing overlays** -- those weight bars
  by recency (exponential or linear). The VWMA weights by
  volume, an independent per-bar input.
- **vs the band primitives (envelope / stddev)** -- those draw a
  band; the VWMA is a single overlay line.
- It is the first primitive in the chart-line family to require
  a `volume` field on each data point.

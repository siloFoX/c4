  source: string;

## Block grammar

- ATX headings 1..6 (`# ...` ...
  `###### ...`)
- Fenced code blocks (` ``` ` /
  ` ```lang `) -- routed through
  `<CodeBlock>` (11.379) for
  syntax highlighting + copy
  button + line numbers.
- Blockquote (lines starting with
  `> `, joined verbatim).
- Horizontal rule (`---`, `***`,
  `___`).
- Unordered list (`-` / `*` /
  `+`).
- Ordered list (`1. ` / `2. `).
- Checkbox list (`- [ ]` /
  `- [x]`; `[X]` allowed).
- Paragraph (everything else;
  blank line separates).

## Inline grammar

- `` `inline code` ``
- `**bold**`
- `*italic*` / `_italic_`
- `‾‾strikethrough‾‾`
- `[label](url)`
- bare `http://` / `https://`
  auto-link

## URL sanitization

`safeUrl(url): string | null` is
the canonical filter:

| input | result |
| --- | --- |
| empty / whitespace | null |
| `http://...` | passthrough |
| `https://...` | passthrough |
| `mailto:...` | passthrough |
| `tel:...` | passthrough |
| `/abs/path` | passthrough |
| `./local` / `../sibling` | passthrough |
| `#anchor` | passthrough |
| no-scheme relative | passthrough |
| `javascript:` | null |
| `data:` | null |
| `vbscript:` | null |
| `file:` | null |
| any other scheme | null (safer fallback) |

When a `[label](url)` link
sanitizes to `null`:

- The anchor is NOT rendered.
- The label remains as plain
  text -- the operator still sees
  the words, just without a
  clickable trap.

Bare-URL autolinks omit entirely
when blocked.

Override via the `sanitizeUrl`
prop to add app-specific schemes
(e.g. `myapp://`).

## Anchor click

- `linkTarget` (default `_blank`)
  selects the `target` attr.
- `rel="noopener noreferrer"`
  auto-applied when target is
  `_blank`; omitted otherwise.
- `onAnchorClick(href, event)`
  fires with the SANITIZED href.
  The raw URL is preserved on
  `data-href-raw` for tests +
  analytics.

## Checkbox lists

When `enableCheckboxes` is true
(default), a contiguous run of
`- [ ]` / `- [x]` items renders
as a dedicated checkbox list:

- Outer container:
  `data-section="markdown-checkbox-list"`.
- Each item:
  `data-section="markdown-checkbox-item"`
  + `data-checkbox-index`
  (flat index across the entire
  document) + `data-checked`.
- The `<input type="checkbox">`
  is read-only when no
  `onCheckboxChange` is supplied.
- `onCheckboxChange(index,
  checked)` fires on toggle.

Pass `enableCheckboxes={false}` to
collapse checkbox lists down to
plain unordered lists with
`data-checkbox-disabled="true"`
on the wrapper.

## Code-block integration

Fenced code blocks delegate to
the existing `<CodeBlock>`
primitive (11.379). The
`<MarkdownRenderer>` wraps each
fence in:

```tsx
<CodeBlock
  data-section="markdown-code-block"
  code={text}
  language={fenceLanguage || props.language}
/>
```

so the renderer inherits the
highlighter, copy button, line
numbers, and theme of the
underlying primitive without
duplicating logic.

## Pairs with existing primitives

- `<CodeBlock>` (11.379) --
  fenced code blocks render via
  this primitive; markdown
  renderer inherits its
  highlighter + copy + theme.
- `lib/markdown.tsx` (legacy
  feature-page renderer) --
  pre-dates the design system,
  used by Plan + Morning pages.
  The new primitive is the
  canonical surface for fresh
  adopters.
- ThemeCustomizer (11.394) --
  the renderer reads token-based
  Tailwind classes so it
  auto-themes.

## Out of scope (deferred)

- Tables. GFM tables belong to a
  follow-on; pipe-table parsing
  is non-trivial.
- HTML pass-through. The
  primitive intentionally
  sanitizes; raw HTML is denied
  by omission.
- Footnotes / definitions /
  reference-style links. Out of
  scope for v1.
- Image rendering. `<img>` adds
  CORS + referrer-policy
  concerns; follow-on.
- Math / LaTeX. Out of scope.
- Smart-typography (curly
  quotes, em-dash). Out of scope.
- Adoption sweep through
  existing pages. Legacy
  `lib/markdown.tsx` continues to
  drive Plan + Morning until a
  separate migration dispatch.

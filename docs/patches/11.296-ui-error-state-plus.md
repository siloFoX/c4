# 11.296 -- ErrorState icon + reportLink + details (v1.11.314)

Component-scope-only enhancement of the existing `ErrorState`
primitive. No daemon-side change and no `c4` CLI surface change.
The default surface (no icon override, no reportLink, no
showDetails) is preserved byte-for-byte modulo the new
data-section attribute set.

## Why

The existing ErrorState shipped with three slots: title,
description, message (single-line `error.message` text), and
one action (retry button). Three pain points across the recent
error surfaces:

1. Per-domain error illustrations (network down, permission
   denied, missing resource) were handled by inlining a custom
   icon as the first child of `description`, which leaked into
   the title row layout.
2. Operator feedback flows ("file an issue", "report this
   error") had no canonical affordance -- pages were rendering
   an ad-hoc `<a>` next to the retry button with their own
   styling.
3. Debug-mode error stacks (the canonical
   `error.stack` string from a caught exception) were either
   omitted entirely OR rendered as plain text inside
   `error.message`, which is verbose and unscannable.

Patch 11.296 puts all three affordances inside the primitive
so every error surface reaches for the same vocabulary.

## What changed

### `web/src/components/ui/error-state.tsx`

- New `icon?: ReactNode` prop. Replaces the default
  `AlertTriangle` glyph with a caller-supplied illustration.
  When omitted, the canonical AlertTriangle stays put.
- New exported type
  `ErrorStateReportLink = { label: string; href: string }`.
- New `reportLink?: ErrorStateReportLink` prop. Renders below
  the retry button as a quieter `<a>` styled in
  muted-foreground (matches the EmptyState helpLink
  convention). External hrefs (`startsWith('http')`) get
  `target="_blank"` + `rel="noreferrer noopener"`.
- New `showDetails?: boolean` prop (default false). When true
  AND the `error` prop is an Error instance with a `.stack`
  string, the surface mounts a `<details>` disclosure with a
  "Stack trace" summary and the formatted stack inside a
  `<pre>` block. The inline `error.message` rendering is
  suppressed in this mode so the surface does not
  double-print. Strings or Error instances without `.stack`
  fall back to the inline rendering.
- New helper `errorStack(error)` returns the trimmed stack
  string or null. Mirrors the existing `errorText(error)`
  shape so future tests can match on either field
  consistently.
- Data-attribute selectors added across every slot:
  - `data-section="error-state"` on the root.
  - `data-section="error-state-icon"` on the icon wrapper.
  - `data-section="error-state-text"` on the
    title+description+message block.
  - `data-section="error-state-title"` on the title span.
  - `data-section="error-state-description"` on the
    description span.
  - `data-section="error-state-message"` on the inline error
    message (when shown).
  - `data-section="error-state-details"` on the `<details>`.
  - `data-section="error-state-details-summary"` on the
    summary.
  - `data-section="error-state-stack"` on the `<pre>` stack.
  - `data-section="error-state-retry"` on the retry button.
  - `data-section="error-state-report-link"` on the report
    anchor.
- `ErrorState.displayName = 'ErrorState'` for devtools.

### Adopted

- Primitive enhancement only this commit. The existing
  ErrorState call sites (HistoryView error, Workers error,
  Sessions error, Validation error, etc.) inherit the new
  attributes without any caller change.

## Deferred (dispatch follow-ups)

- Thread `reportLink` into HistoryView / Workers / Sessions
  error states pointing at the canonical issue tracker URL.
- Thread `showDetails` into developer-facing error surfaces
  (Debug page, Risk page) where the stack trace aids triage.

  vitest cases:

  - reportLink renders with label + relative href.

## Notes

- The `<details>` disclosure starts collapsed by default
  (browser-native behaviour). Callers can pass `open` via
  the standard HTML attribute on the disclosure if a flow
  needs the trace pre-expanded.
- External-link detection on `reportLink.href` matches the
  EmptyState `helpLink` convention (`startsWith('http')`)
  for consistency across the two state surfaces.

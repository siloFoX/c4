# TODO 8.25 - ChatView past-history backfill

**Status:** done
**Branch:** `c4/chat-backfill`
**Scope:** Make `web/src/components/ChatView.tsx` load past conversation
on mount before attaching the SSE live stream, so a fresh page load or
a worker swap no longer shows "No messages yet" when the scrollback
buffer or the Claude session JSONL already has hundreds of turns.

## Summary

Before this patch, `ChatView` only subscribed to `/api/watch` SSE. The
stream carries *new* PTY bytes only - there is no replay - so a user
opening the Chat tab mid-conversation saw an empty log until the next
write. 8.25 closes that gap:

1. On mount (and on every `workerName` change) ChatView runs a
   backfill pass *before* rendering the live stream:
   1. Prefer `/api/sessions?workerName=<name>` - the daemon resolves
      the worker's current session id via `manager.getSessionId`,
      parses the JSONL through the 8.18 `session-parser`, and returns
      the full `Conversation`. ChatView maps `user`, `assistant`,
      and `tool_use` turns into chat bubbles (`thinking`,
      `tool_result`, and `system` collapse into the dedicated
      ConversationView tab; Chat is a trimmed conversation surface).
   2. Fall back to `/api/scrollback?name=<name>&lines=2000` with a
      naive `> ` user-prompt splitter when no session JSONL resolves
      yet (new worker / LOST / `--resume` miss). Best effort; the
      ConversationView tab remains the source of truth.
2. Once the backfill lands, the existing SSE effect connects and
   streams live bubbles on top of the history.
3. **Dedup**: every rendered bubble's trimmed text is tracked in
   `seenTextsRef`. If an SSE chunk's text matches a text already in
   the backfill (the tail of an in-flight assistant message that
   landed in JSONL *and* on the PTY stream) the live bubble is
   dropped. `seenIdsRef` mirrors the JSONL turn ids so infinite
   scroll reloads don't double-count either.
4. **Loading skeleton** + **"Loaded N past messages"** badge.
5. **Infinite scroll**: in scrollback mode, scrolling to the top
   or clicking the inline *Load older* button bumps the `lines`
   parameter by 2000 up to a 10000 cap and rehydrates the history.
   In session mode the JSONL already holds the whole conversation,
   so the control is hidden.
6. **Worker change**: the backfill effect depends on `workerName` and
   resets every state slot (history, live messages, dedup sets,
   scrollback cursor, buffers, timers) at the top. A closure-scoped
   `cancelled` flag short-circuits stale fetches when a fast worker
   swap races a slow backfill.

## API contract addition: `/api/sessions?workerName=<name>`

### New shape

```http
GET /api/sessions?workerName=worker-1
```

```jsonc
{
  "sessionId": "abc123-def",
  "conversation": {
    "sessionId": "abc123-def",
    "projectPath": "/root/c4",
    "createdAt": "2026-04-20T09:00:00.000Z",
    "updatedAt": "2026-04-20T09:42:11.000Z",
    "model": "claude-opus-4-7",
    "totalInputTokens": 1842,
    "totalOutputTokens": 931,
    "turns": [ /* 8.18 Turn shape */ ],
    "warnings": []
  },
  "workerName": "worker-1"
}
```

When the worker has no resolvable JSONL yet, the endpoint returns a
null-shaped body at HTTP 200 so the client can decide to fall back
to `/api/scrollback` rather than getting a 404:

```jsonc
{ "sessionId": null, "conversation": null, "workerName": "worker-1" }
```

### Back-compat

Calling `/api/sessions` without `workerName` returns the legacy list
shape untouched:

```jsonc
{ "rootDir": "/root/.claude/projects", "sessions": [...], "groups": [...], "total": 42 }
```

`SessionsView` and the 8.18 session-list consumers are therefore
unaffected.

### Where it lives

`src/daemon.js` - `route === '/sessions'` handler. The workerName
branch runs before the legacy list branch so query-string order does
not matter. Resolution flow:

```
workerName -> manager.getSessionId(name)
           -> sessionParser.listSessions(rootDir).find(sessionId)
           -> sessionParser.parseJsonl(match.path)
```

## UI details

- **Skeleton**: three animated `bg-muted` pulse bars inside the log
  pane while `backfillLoading` is true, plus a spinner + "Loading
  past messages..." label.
- **Loaded badge**: secondary badge in the header when
  `backfillCount > 0`, with a `title` tooltip that distinguishes
  session JSONL vs scrollback source.
- **Past tag**: each backfilled bubble gets a tiny "past" marker
  and a faint opacity reduction so users can tell at a glance which
  messages came from history vs the live stream.
- **Jump to latest**: unchanged - still shows when the user is
  scrolled up and `autoScroll` is false.
- **Load older**: rendered at the top of the message list only in
  scrollback mode while `hasOlder` is true. Auto-fires when the user
  scrolls to the top (within 8 px), with an explicit button fallback
  for keyboards / assistive tech.

## Decisions / deviations

- **No 404 on unresolved worker.** A 404 would force the client to
  treat it as an error state. The plan requires a *fallback* to
  scrollback, so returning a null-shaped 200 body keeps the control
  flow in the component.
- **Chat view filters `thinking` / `tool_result` / `system`.** The
  Chat tab is the trimmed conversation surface; the full mixed-role
  view already lives at the ConversationView tab wired by 8.18.
  Including thinking and tool_result here would double the noise.
- **Scrollback fallback is intentionally naive.** The heuristic
  `^> ` prompt marker only recovers coarse user/worker boundaries.
  When the session JSONL exists (the normal path), the fallback
  never runs - we optimize for the common case.
- **Infinite scroll caps at 10000 lines.** `w.screen.scrollback`
  has a finite `maxScrollback`; beyond 10k the tail is gone anyway.
  Document the cap in the constant name so future tuning is one
  line.
- **Dedup key is trimmed text, not hash.** With a 300-message cap
  the Set stays under a few hundred entries - hashing adds cost
  without meaningful collision resistance.

## Known limitations

- If the JSONL and the PTY stream *disagree* on how an assistant
  message was whitespace-formatted (e.g. trailing newline only on
  one side), the dedup key misses and the bubble doubles. Acceptable
  edge case; documenting here rather than papering over with a
  fuzzy matcher.
- The scrollback fallback does not re-chunk turn boundaries exactly
  as the JSONL would. Operators who need exact chunking should
  prefer a live worker (which resolves to JSONL) over a LOST /
  fresh-create view.

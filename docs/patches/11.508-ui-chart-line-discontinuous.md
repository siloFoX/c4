# TODO 11.508 - chart-line-discontinuous primitive

Status: SHIPPED v1.11.526 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartLineDiscontinuous>`, a pure-SVG multi-series line chart
that **explicitly breaks the line at `null` / `undefined` data
points**. Unlike every other line primitive (which drops non-finite
points silently), this one treats `null` / `undefined` as a gap
marker: the surrounding line segments stay distinct and a dashed
gap-marker rule under the x axis brackets each gap. The canonical
"data outage" pattern for monitoring dashboards.

## API

```tsx
<ChartLineDiscontinuous
  series={[
    {
      id: 'uptime',
      label: 'Uptime',
      data: [
        { x: 0, y: 99 },
        { x: 1, y: 100 },
        { x: 2, y: null },     // gap marker
        { x: 3, y: null },     // gap marker
        { x: 4, y: 98 },
        { x: 5, y: 100 },
      ],
    },
  ]}
/>
```

## Distinct from prior chart-* primitives

- `<ChartLine>` and other variants drop non-finite y values silently.
  This primitive distinguishes `null` / `undefined` (an explicit gap)
  from `NaN` (a dropped point) and surfaces the gap visually.
- `<ChartLineSegment>` (11.507) - threshold-coloured line; no gap
  semantics.
- `<ChartLineMarker>` (11.506) - per-point marker shape.
- `<ChartLineAreaStacked>` (11.505) - stacked layers.
- `<ChartLineAreaGradient>` (11.504) - gradient fill.
- `<ChartLineThreshold>` (11.503), `<ChartLineConfidence>` (11.502),
  `<ChartLineZoom>` (11.501), `<ChartLineMulti>` (11.500),
  `<ChartLineDashed>` (11.499), `<ChartLineSmooth>` (11.498),
  `<ChartLineStep>` (11.497).

## Gap contract

### Point classification

`isFiniteDiscontinuousPoint(point)` returns true only when `x` is
finite AND `y` is a finite number. `null` / `undefined` y both
register as gaps.

`isLineDiscontinuousGap(point)` returns true for `y === null` or
`y === undefined`. `NaN` is NOT a gap -- it is a dropped point
(consistent with other line primitives). This distinction lets
adopters use `NaN` for "no data ever recorded" and `null` for
"recorded but the metric was unavailable".

### Run / gap building

`buildLineDiscontinuousRuns(points, connectGaps)` walks an ordered
sequence of layout points (with optional `isGap: true` markers) and
returns:

- `runs`: contiguous arrays of finite-y points. Each run owns an
  `M ... L ...` path. Single-finite-point runs render as a lone `M`.
- `gaps`: one record per gap that sits between two bracketing finite
  points. Exposes `startIndex` / `endIndex` (original-data indices),
  `startX` / `endX` (pixel positions), and `width`. Leading and
  trailing gaps (no bracketing finite point on one side) record
  no `gaps[]` entry.

`connectGaps=true` collapses the runs into a single run that spans
across the gaps; the gap records are still preserved so adopters
who want to visually mark "the data was nulled here even though we
connected through it" can do so.

## Layout

- `getLineDiscontinuousBounds` includes gap-marker x positions in
  the x range so the dashed gap rule below the axis is always
  visible.
- Per-series `gapCount` + `runs.length` recorded as data attrs.
- Dots render only at finite-y points.

## Interactivity

- Hover or focus on a dot dims every other series to ~30%; tooltip
  shows series label, formatted x, formatted y.
- Per-dot `onPointClick({series, point})` callback.
- Legend toggles series visibility (controlled + uncontrolled).
- Per-gap dashed rule renders below the x axis between bracketing
  finite points so adopters can see exactly where the gap lives;
  suppressed by `showGapMarkers=false` or by `series.connectGaps=true`.
- Mount fade via `motion-safe:animate-fade-in` (suppress via
  `animate=false`).

`number | null | undefined`.

## Exported helpers

- `DEFAULT_CHART_LINE_DISCONTINUOUS_*` numeric / string defaults.
- `DEFAULT_CHART_LINE_DISCONTINUOUS_PALETTE` (10-colour palette).
- `getLineDiscontinuousDefaultColor(index)` - palette wrap.
- `isFiniteDiscontinuousPoint(point)` - classifies a point as
  renderable.
- `isLineDiscontinuousGap(point)` - classifies a point as a gap
  marker (null / undefined y).
- `getLineDiscontinuousFinitePoints(points)` - returns `{point,
  originalIndex}` pairs for finite points only.
- `getLineDiscontinuousGapCount(points)` - count of null / undefined
  markers.
- `getLineDiscontinuousBounds(series, hidden?)` - per-axis bounds.
- `getLineDiscontinuousTicks(min, max, count?)` - evenly-spaced.
- `buildLineDiscontinuousRuns(points, connectGaps)` - canonical
  run/gap builder.
- `computeLineDiscontinuousLayout(input)` - per-series layout with
  runs + gaps + per-point data.
- `describeLineDiscontinuousChart(series, hidden?, formatValue?)`
  - ARIA summary mentioning total points + gap marker count.

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only
  `data-section="chart-line-discontinuous-aria-desc"` span.
- SVG `role="img"` with the same `aria-label`.
- Each run path is `role="graphics-symbol"` with `tabIndex={0}` and
  `aria-label="<series> run <startIndex> to <endIndex> (N points)"`.
- Each dot is `role="graphics-symbol"` with `tabIndex={0}` and a
  point-level label.

## Data attributes

- Root: `data-section="chart-line-discontinuous"`,
  `data-series-count`, `data-visible-series-count`,
  `data-total-points`, `data-total-gaps`, `data-total-runs`,

## Out of scope

- Dashed line through gaps (use `connectGaps=true` + a custom
  `<ChartLineDashed>` overlay for that pattern).
- Inferred null points from sample-rate detection (caller passes
  explicit nulls).
- Animated reveal at gap edges (snap; mount fade only).
- Log-scale axes - linear only.
- Per-gap onClick handler - the dashed marker is decorative; adopters
  who need callbacks should derive them from the `gaps[]` layout
  output.

# 11.1034 -- UI: chart-line-hma-divergence-cross primitive (v1.11.1052)

## Scope

Add a pure-SVG dual-panel `ChartLineHmaDivergenceCross`
component that renders the close in the top panel and the
Hull Moving Average (HMA) in the bottom panel, flagging
price-vs-HMA direction-disagreement events as fast trend
reversal warnings with bias coloring derived from the HMA
slope at the trigger bar. Uses the canonical 5-state
divergence regime model (aligned-bullish / aligned-bearish
/ divergent-bullish / divergent-bearish / none) shared by
the rest of the chart-line-*-divergence-cross family.

## HMA divergence pipeline

```
wmaHalf[i]  = WMA(close, halfPeriod)  ; halfPeriod = period / 2
wmaFull[i]  = WMA(close, period)
inner[i]    = 2 * wmaHalf[i] - wmaFull[i]
sqrtPeriod  = round(sqrt(period))
hma[i]      = WMA(inner, sqrtPeriod)

priceUp     : close[i] > close[i-1]
priceDown   : close[i] < close[i-1]
hmaUp       : hma[i]   > hma[i-1]
hmaDown     : hma[i]   < hma[i-1]

regime ->
  'aligned-bullish'   when priceUp   && hmaUp
  'aligned-bearish'   when priceDown && hmaDown
  'divergent-bullish' when priceDown && hmaUp
  'divergent-bearish' when priceUp   && hmaDown
  'none'              when either side flat or null

bullish (divergence trigger up) :
  prev regime !== 'divergent-bullish' &&
  cur regime === 'divergent-bullish'
bearish (divergence trigger down) :
  prev regime !== 'divergent-bearish' &&
  cur regime === 'divergent-bearish'

bias        : hma[i] vs hma[i-1] -> up / down / flat / none
```

HMA is Alan Hull's near-zero-lag composite of WMAs. Because
WMA centroid lag is `(length - 1) / 3`, the recipe `2 *
WMA(period/2) - WMA(period)` cancels out the bulk of the
WMA lag, then the outer `WMA(inner, sqrt(period))` smooths
the residual jitter. The result is a moving-average line
that reacts roughly twice as fast to recent price moves as
a simple WMA of the same length, with comparable noise
suppression.

Warmup is `period + sqrtPeriod - 2 = 16` for the default
tuning (period = 14, sqrtPeriod = 4): WMA(close, 14) seeds
at i = 13, then WMA(inner, 4) adds another `sqrtPeriod - 1
= 3` bars, putting the first valid HMA at i = 16.
Divergence detection requires the previous HMA + close, so
the first regime classification lands at i = 17.

## Bit-exact anchors

- **CONST close = K**: WMA(close, 7) = K, WMA(close, 14) =
  K, inner = 2K - K = K, HMA = K. Both close and HMA are
  flat -> regime `none` from `i = warmup + 1`. 0 divergence
  crosses. Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP close = i**: WMA(close, 7) = i - (7 - 1) / 3
  = i - 2. WMA(close, 14) = i - 13 / 3. inner = 2 * (i - 2)
  - (i - 13/3) = i + 1/3. HMA = WMA(inner, 4) = (i + 1/3)
  - (4 - 1) / 3 = i - 2/3. HMA[i] - HMA[i-1] = +1 (up);
  close[i] - close[i-1] = +1 (up). Both up -> regime
  `aligned-bullish`. 0 divergence crosses.
- **LINEAR DOWN close = -i**: mirror -> HMA = -i + 2/3.
  Both close and HMA falling -> regime `aligned-bearish`.
  0 divergence crosses.

The clean `i - 2/3` lag on LINEAR input is a direct
consequence of the WMA-centroid arithmetic chained through
the Hull recipe -- a fixed property of the HMA construction,
not a sample-by-sample artifact.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 5-state regime classifier follows the
  chart-line-*-divergence-cross family convention; the
  `bullish` / `bearish` cross *kinds* still follow the
  sibling cross-sig convention (line direction).
- HMA `sqrtPeriod` uses `round(sqrt(period))` (the
  canonical Alan Hull formulation). For period = 9 this
  gives 3, for period = 14 it gives 4, for period = 25 it
  gives 5. `Math.max(2, ...)` clamps to a minimum window
  of 2 to keep the WMA helper well-defined for small
  periods.
- The 5-state regime makes the `flat` case (`priceUp ==
  hmaUp == false`, but neither down) explicit -- it always
  classifies as `none`, never silently leaking into one of
  the aligned states.

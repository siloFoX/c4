# Patch 11.953 - UI: chart-line-coppock-zero-cross primitive

Version: 1.11.971

## Summary

Add `web/src/components/ui/chart-line-coppock-zero-cross.tsx` -- a pure-SVG
dual panel line chart primitive that visualizes price alongside Edwin
Coppock's Curve, with explicit cross markers and overlay arrows at every
Coppock zero crossover. Coppock is the weighted moving average of the
sum of two percentage rate-of-change windows (a slow / fast pair) over
the close. Originally designed for monthly equity-index trend
confirmation, zero crossovers mark long-term momentum baseline regime
transition events -- a slower, lower-noise trigger than a single
short-window momentum indicator.

## Pipeline

1. Two ROC windows over close:
   - `roc_s_i = (close[i] - close[i - short]) / close[i - short] * 100`
   - `roc_l_i = (close[i] - close[i - long])  / close[i - long]  * 100`
   - Each is null for `i < N` or when the divisor close is 0.
2. Sum the two ROCs: `sum_i = roc_s_i + roc_l_i`.
3. `applyLineCoppockZeroCrossWma(sum, period)`: weighted moving average
   with weights `1, 2, ..., period` (most recent weight = period),
   denominator `period * (period + 1) / 2`. CONST short-circuit when
   the window is constant.
4. `classifyLineCoppockZeroCrossRegime(coppock, T)`: null -> none,
   coppock >= T -> bullish, coppock < T -> bearish.
5. `detectLineCoppockZeroCrossCrosses(series, coppock, T)`: strict
   `prev <= T && cur > T` for bullish, strict `prev >= T && cur < T`
   for bearish. Boundary equality does not fire.

## Bit-exact anchors

| Pattern                          | Coppock (after warmup)              | Regime  |
|----------------------------------|-------------------------------------|---------|
| CONST close = K (K > 0)          | 0                                   | bullish |
| GEOMETRIC UP close = K * r^i (r>1)  | `((r^short - 1) + (r^long - 1)) * 100` | bullish |
| GEOMETRIC DOWN close = K * r^i (r<1)| same formula, < 0                | bearish |

CONST anchor tested bit-exactly across K in `{1, 42, 100, 1234}` (K
must be positive for ROC divisor). GEOMETRIC anchors tested with r in
{1.01, 0.99} where the ROC ratio collapses to `r^N - 1` constant, so
the sum is constant and WMA of a constant is bit-exact that constant.

## Layout

* Default dimensions: 720 x 460.
* Two panels: price panel on top (~55% of plot area), Coppock panel
  below.
* Coppock panel auto-fits to the observed `[min, max]` range with 10%
  padding on each end, always including the threshold in the displayed
  range.
* Threshold band rendered as a horizontal rule at `y = thresholdY` in
  the Coppock panel.
* Cross markers drawn as circles on the Coppock panel at `(cx, cyOsc)`
  and as arrows on the price panel at `(cx, cyPrice)`.
* SVG path strings use `.toFixed(2)` for deterministic snapshot tests.

## Accessibility

* Wrapping `<div role="region" aria-label="Coppock Zero Cross chart">`.
* `<svg role="img">` with an aria-described-by linked `<desc>` sr-only
  text.
* Each cross marker: `role="graphics-symbol"` + `tabIndex={0}` +
  descriptive `aria-label` ("bullish cross at x=...").
* Legend toggles: `<button role="switch" aria-pressed="...">`.
* Motion-safe fade-in via `motion-safe:animate-fade-in` (opt-out with
  `animate={false}`).

advancing issues to advancing volume.

## Indicator math

Richard Arms' Index (TRIN -- the TRading INdex) is a market
breadth indicator. Each bar carries four market-wide counts:
the advancing issues, the declining issues, the advancing
volume and the declining volume. The TRIN is a ratio of ratios:

```
TRIN = (advIssues / decIssues) / (advVolume / decVolume)
```

`computeLineTrinValue(advIssues, decIssues, advVolume,
decVolume)` evaluates it in the cross-multiplied form
`(advIssues * decVolume) / (decIssues * advVolume)`. A zero in
the denominator -- no declining issues or no advancing volume --
returns null.

A TRIN of 1 means volume is spread in proportion to the issue
counts. A TRIN below 1 is bullish: volume is concentrating in
the advancing stocks. A TRIN above 1 is bearish. The chart
classifies each bar against a neutral band of half-width
`threshold` around 1: bullish below `1 - threshold`, bearish
above `1 + threshold`, neutral in between.

### Exactness

The TRIN is a ratio of integer products, so it is exact when the
breadth counts are integers and the division lands cleanly. The
fixture counts give the exact TRIN series `[1, 0.5, 2, 1, 0.5,
1.5, 0.5, 2, 1, 4]`, asserted with `toEqual`.

## API

- `getLineTrinFinitePoints(points)` -- drop points with a
  non-finite field, or a negative breadth count.
- `normalizeLineTrinThreshold(threshold, fallback)` -- coerce
  the neutral-band half-width into `(0, 1)`.
- `computeLineTrinValue(advIssues, decIssues, advVolume,
  decVolume)` -- the TRIN for one bar.
- `computeLineTrin(points)` -- the TRIN for each point of a
  series.
- `runLineTrin(points, options?)` -- sort finite points by `x`,
  run the pipeline, return `{ series, threshold, trin, samples,
  trinFinal, bullishCount, bearishCount, neutralCount, ok }`.
  Each sample is classified `bullish` / `bearish` / `neutral` /
  `none`. `ok` is false for fewer than two points.
- `computeLineTrinLayout(options)` -- stack the market-index
  panel above a TRIN panel; project the price path, the TRIN
  path, one marker per bar, and the balance and band reference
  lines.
- `describeLineTrinChart(data, options?)` -- ARIA description.

## Component

`ChartLineTrin` is a `forwardRef` region (`role="region"`) with
an ARIA-linked description, a `role="img"` SVG, two stacked
panels (market index on top, TRIN below) with their own axis
ticks and grid, panel labels, a config badge showing the band
threshold, a three-series legend (Price / TRIN / Levels) with
toggle, a hover/focus tooltip exposing the TRIN and the zone, and
a `motion-safe:animate-fade-in` entrance. It consumes
`{ x, value, advIssues, decIssues, advVolume, decVolume }`
points. The TRIN panel draws the index line over dashed balance
(1.0) and band reference lines, with each marker coloured by its
zone. Every node carries a `data-section` attribute.

## Distinctness

- **vs the single-series oscillators (rmi / cmo / pzo)** --
  those derive their value from one price series. The TRIN is a
  market-breadth indicator fed by four advance/decline counts
  per bar.
- **vs chart-line-vzo / chart-line-obv** -- those use a single
  instrument's volume. The TRIN compares market-wide advancing
  volume against declining volume.
- **vs an advance/decline line** -- that accumulates a running
  net breadth total. The TRIN is an instantaneous per-bar ratio
  with no accumulation.

## Summary

Added `<ChartLineHmaPct />` -- a pure-SVG, dual-panel React/TS
primitive that overlays the close with the Hull Moving Average
in the top panel and renders the HMA percent deviation in the
bottom panel:

```
wma1[i]   = WMA(close, floor(length / 2))
wma2[i]   = WMA(close, length)
raw[i]    = 2 * wma1[i] - wma2[i]
HMA[i]    = WMA(raw, floor(sqrt(length)))
hmaPct[i] = HMA[i] === 0 ? null : (close[i] - HMA[i]) / HMA[i] * 100
```

Regime classifier: `above` (hmaPct > 0), `below` (< 0), `at`
(= 0), `none` (null). Defaults: `length = 9` (classic Hull
period), hmaColor `#7c3aed` (violet), pctColor `#0ea5e9` (sky),
zero line `#475569`. The WMA helper carries a `min === max`
window-constant precision fix so constant inputs lock onto their
values exactly.

## Bit-exact anchor

- **CONST close = K (K > 0)**: every WMA in the chain collapses
  to K via the precision fix. `raw = 2K - K = K`, `HMA = K`,
  `close - HMA = 0`, `hmaPct = 0 / K * 100 = 0`. Verified across
  multiple `K` and `length`.
- **CONST close = K = 0**: same as above except the divide-by-
  zero guard leaves `hmaPct = null` for every bar (regime
  `none`).

## Algebraic soft anchor

The Hull formula's defining property is that it perfectly tracks
a linear series. For LINEAR UP `close = i + 1` (and LINEAR DOWN
mirror) the algebra works out to `HMA = close`, so `hmaPct = 0`
exactly in math. In IEEE 754 the intermediate `1/3` rationals
introduce a small 1-ULP drift; the test asserts
`abs(hmaPct) < 1e-10` rather than bit-exact zero.

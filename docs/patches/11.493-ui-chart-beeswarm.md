# TODO 11.493 - chart-beeswarm primitive

Status: SHIPPED v1.11.511 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartBeeswarm>`, a pure-SVG beeswarm plot. Each data point is a
small circle placed along a value axis; a force-directed greedy packer
deterministically displaces overlapping points in the orthogonal
direction so that no two circles overlap. Multiple groups stack into
parallel lanes. Distinct from `<ChartStrip>` (11.492) which uses
random/seeded jitter and tolerates overlaps.

## API

```tsx
<ChartBeeswarm
  groups={[
    { id: 'a', label: 'Control', values: [12, 14, 15, 15, 17, 18, 18, 21] },
    { id: 'b', label: 'Treatment', values: [10, 11, 13, 14, 14, 16, 17] },
  ]}
  orientation="horizontal"
  radius={4}
  gap={1}
  valueLabel="ms"
  onPointClick={({ group, point }) => console.log(group.id, point.value)}
/>
```

## Packing algorithm

`packBeeswarmGroup(items, laneCenter, radius, gap)` implements the
classic D3 / Sergio Pegoraro non-overlapping placement:

1. Sort `items` by `(value, index)` so a deterministic order drives the
   packer.
2. Set `spacing = 2 * radius + gap` (minimum allowed center-to-center
   distance).
3. For each new item in sorted order:
   - Candidate set starts at `{ laneCenter }`.
   - For every previously placed point `p` whose `|along - p.along| <
     spacing`, add `p.across +/- sqrt(spacing^2 - dx^2)` to the
     candidate set (those are the two y-coordinates that put the new
     point exactly tangent to `p`'s collision disk).
   - Sort candidates by `|candidate - laneCenter|` ascending.
   - Pick the first candidate that does not collide with any placed
     point (i.e. `dx^2 + dy^2 >= spacing^2 - 1e-9` for every placed
     point).
4. Return placed points sorted by original index ascending.

Output is fully deterministic - identical input always produces
identical layout. No randomness, no animation reshuffle.

## Distinct from prior chart-* primitives

- `<ChartStrip>` (11.492) - 1D distribution with random/seeded jitter
  that tolerates overlaps. Beeswarm is the non-overlapping sibling.
- `<ChartScatter>` - 2D scatter; both axes encode values.
- `<ChartBubble>` (11.470) - 2D scatter with circle size encoding.
- `<ChartDotPlot>` - Wilkinson dot plot (stacked at fixed bins).
- `<ChartBoxplot>` (11.455) - five-number summary.
- `<ChartHistogram>` (11.473) - binned counts.
- `<ChartViolin>` - KDE-shaped density envelope.
- `<ChartRidge>` (11.474) - stacked density curves.

## Exported helpers

- `DEFAULT_CHART_BEESWARM_*` numeric / string defaults.
- `DEFAULT_CHART_BEESWARM_PALETTE` (10-colour palette).
- `getBeeswarmDefaultColor(index)` - palette index with modulo wrap.
- `getBeeswarmFiniteValues(values)` - drops non-finite values.
- `getBeeswarmBounds(groups, hidden?)` - min/max across visible finite
  values; empty -> `(0..1)`, collapsed range expands +/- 0.5.
- `getBeeswarmTicks(min, max, count?)` - evenly-spaced ticks.
- `packBeeswarmGroup(items, laneCenter, radius, gap)` - canonical
  non-overlapping placer. Items are `{ index, value, along }` where
  `along` is the value-axis pixel coordinate; output has the same
  shape plus `across` (the orthogonal coordinate). Output is
  deterministic and sorted by `index`.
- `computeBeeswarmLayout(input)` - canonical layout returning
  `{ groups, ticks, valueMin, valueMax, orientation, innerWidth,
  innerHeight, laneHeight, radius, gap }`; empty for degenerate
  canvas / empty groups / all-hidden.
- `describeBeeswarmChart(groups, hidden?, formatValue?)` - ARIA
  summary "Beeswarm plot of N values across M groups, range <min> to
  <max>.".

## Layout

- Value axis: range auto-derived from finite visible values (collapsed
  range expands +/- 0.5; empty -> `(0..1)`). `valueMin` / `valueMax`
  clamp out-of-range points to the nearest endpoint (still rendered
  at the boundary, never dropped).
- Lanes: total inner height (or width, for vertical) is divided into N
  equal lanes separated by `laneGap`.
- `horizontal` maps the value axis to x; `vertical` maps it to y
  (bottom -> top).

## Interactivity

- Hover or focus on a circle dims every other circle to opacity 0.3
  and opens a tooltip with the group label + formatted value.
- Click on a circle fires `onPointClick({group, point})`.
- Legend buttons toggle group visibility (uncontrolled by default;
  bind `hiddenGroups` + `onHiddenGroupsChange` for controlled).
- Mount fade via `motion-safe:animate-fade-in` (suppressed via
  `animate=false`).

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only `data-section="chart-beeswarm-aria-desc"`
  span carrying the auto summary.
- SVG `role="img"` with the same `aria-label`.
- Each circle mark is `role="graphics-symbol"` with `tabIndex={0}` and
  `aria-label="<group label>: <formatted value>"`.

- Tooltip / legend / axes / grid / lane labels: standard set mirroring
  the other chart-* primitives.

## Tests

bounds + 3 ticks + 8 pack invariants + 11 layout invariants + 4
describe + 32 component) all pass under vitest 4.1.5. TypeScript clean
for touched files.

## Out of scope

- Force-directed simulation with velocity / collisions per frame
  (D3-force style); v1 is one-shot greedy placement.
- Pre-aggregated input (e.g. `{value, count}` tuples); v1 takes one
  mark per raw value.
- Log-scale value axis (linear only).
- External selection by id (hover / focus only).
- Continuous category axis (lane is per-group, not a numeric axis).

## Summary

New `ChartLineHma` UI primitive at
`web/src/components/ui/chart-line-hma.tsx` -- a pure-SVG line chart
with a Hull Moving Average overlay that reduces lag while staying
smooth. Unlike the recent two-panel indicator primitives, this is a
**single-panel** chart: the HMA line is drawn directly over the price
line on shared axes.

## Indicator math

Alan Hull's Hull Moving Average, with a period (default 16):

```
halfPeriod = floor(period / 2)
sqrtPeriod = floor(sqrt(period))
raw = 2 * WMA(values, halfPeriod) - WMA(values, period)
HMA = WMA(raw, sqrtPeriod)
```

`computeLineHmaWma(src, period)` is a linearly weighted moving
average: the oldest value in the window carries weight 1, the newest
weight `period`, divided by the triangular weight sum. It tolerates
leading `null` placeholders, so the WMA of the raw line (which has a
null prefix) is well-defined.

The raw line `2 * WMA(half) - WMA(full)` over-shoots the trend to
cancel the lag of a plain moving average; smoothing it with a short
`sqrt`-length WMA removes the resulting noise. On a perfectly linear
trend the HMA tracks the price with zero lag.

### Worked fixture

`values = [0, 3, 6, 9, 12, 15, 18]` (a step-3 linear ramp),
`period = 4` (halfPeriod 2, sqrtPeriod 2):

```
wmaHalf = WMA(values, 2) = [null,2,5,8,11,14,17]
wmaFull = WMA(values, 4) = [null,null,null,6,9,12,15]
raw     = 2*wmaHalf - wmaFull = [null,null,null,10,13,16,19]
hma     = WMA(raw, 2) = [null,null,null,null,12,15,18]
```

The HMA reproduces the value exactly (`12 / 15 / 18`) -- the zero-lag
property on a linear trend.

## API

- `getLineHmaFinitePoints(points)` -- drop points with a non-finite
  `x` or `value`.
- `normalizeLineHmaPeriod(period, fallback)` -- positive integer;
  non-finite / sub-1 falls back, fractional floors.
- `computeLineHmaWma(src, period)` -- the null-tolerant linearly
  weighted moving average.
- `computeLineHma(values, period)` -- returns
  `{ wmaHalf, wmaFull, raw, hma }`.
- `runLineHma(points, options?)` -- sort finite points by `x`, compute
  the HMA, return `{ series, period, halfPeriod, sqrtPeriod, wmaHalf,
  wmaFull, raw, hma, samples, hmaFinal, hmaMin, hmaMax, aboveCount,
  belowCount, ok }`. Each sample carries an `above` / `below` / `on`
  position. `ok` is false for fewer than two points.
- `computeLineHmaLayout(options)` -- fit one panel whose y-domain
  spans both the price and the HMA. Returns the panel, ticks, the
  price path, the HMA path, the price dots, and the HMA markers.
- `describeLineHmaChart(data, options?)` -- ARIA description.

## Component

`ChartLineHma` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, a single panel with axis
ticks and grid, a config badge (period), a two-series legend (Price /
HMA) with toggle, the price line, the HMA line drawn over it, markers
on the HMA line, optional price dots, a hover/focus tooltip exposing
the value, HMA, and position, and a `motion-safe:animate-fade-in`
entrance. Every node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-ewma / chart-line-moving-avg** -- a single EMA or
  SMA always lags the trend. The HMA nests *three* weighted moving
  averages (a half, a full, and a square-root length) in the
  `2*WMA(half) - WMA(full)` arrangement specifically to cancel that
  lag; no single moving average does this.
- **vs chart-line-coppock / chart-line-kst** -- those apply a WMA to
  *rate-of-change* legs in a sub-panel oscillator. The HMA applies
  the WMA to the *price itself* and is a single-panel price overlay,
  not an oscillator.
- **single-panel** -- unlike the recent two-panel indicator
  primitives, the HMA shares one set of axes with the price.

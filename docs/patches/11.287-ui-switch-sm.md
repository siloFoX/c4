# 11.287 -- Switch sm size + motion-safe thumb (v1.11.305)

Component-scope-only enhancement of the existing `Switch`
primitive. No daemon-side change and no `c4` CLI surface change.
The current `md` size (h-5 w-9) is preserved byte-for-byte; new
affordances opt in via the `size` prop and the new
data-attribute selector set.

## Why

Two recent pain points:

1. The FeatureFlags override table renders 12-20 toggle rows on
   a single screen. At the existing 20-pixel md height plus the
   row padding, the table felt loud -- the toggles dominated
   the visual weight instead of being a quiet "off / on"
   indicator. A 16-pixel sm size lets the toggles read as
   inline glyphs rather than primary controls.
2. The thumb transition was unconditional `transition-transform`.
   Operators with `prefers-reduced-motion: reduce` saw the
   thumb slide every time they toggled a setting, despite the
   rest of the UI primitives (Tooltip, Dialog, Drawer, Toast)
   already gating their slide animations.

## What changed

### `web/src/components/ui/switch.tsx`

- New exported type `SwitchSize = 'sm' | 'md'`.
- New `size?: SwitchSize` prop (default `'md'`). `md` keeps
  the existing 20x36 px track + 16x16 px thumb byte-for-byte.
  `sm` lands at 16x28 px track + 12x12 px thumb.
- Per-size translate constants for the thumb on/off positions
  (`THUMB_TX_OFF`, `THUMB_TX_ON`) so the thumb visually centres
  in each end of the track regardless of size.
- Motion-safe thumb -- the `transition-transform` class is
  applied ONLY when `useReducedMotion()` returns false.
- New data-attribute selectors:
  - `data-section="switch"` on the button.
  - `data-section="switch-thumb"` on the moving span.
  - `data-section="switch-row"` on the label-mode wrapper.
  - `data-section="switch-label"` on the `<label>`.
  - `data-size="sm|md"` on the button.
  - `data-checked="true|false"` on the button.
  - `data-reduced-motion="true|false"` on the button.

### Adopted

- Primitive enhancement only this commit. Existing Switch
  call sites (Batch, TokenUsage, Risk, FeatureFlags,
  DesignSystem) inherit the new defaults without any caller
  change.

## Deferred (dispatch follow-ups)

- Settings toggles size adoption: the current Settings page
  uses ChoiceGroup buttons rather than Switch, so the size
  prop has nowhere to land there. A follow-up will refactor
  if a true Switch row makes sense.
- FeatureFlags overrides size adoption: the size prop is
  now available; threading `size="sm"` through the override
  table mount points is a per-page follow-up.

## Notes

- The thumb diameter is intentionally smaller than the track
  height (sm: 12/16 = 75%, md: 16/20 = 80%) so the thumb
  visually centres with room to breathe on either end -- this
  avoids the "filled track" look that defeats the point of a
  toggle.
- `data-reduced-motion` is plumbed at the same depth as the
  Drawer + Tooltip primitives so a future
  cross-primitive visual-regression sweep can target every
  motion-gated surface via one selector.

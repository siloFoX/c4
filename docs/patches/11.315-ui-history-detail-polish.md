# 11.315 -- HistoryDetailPane polish (v1.11.333)

Polishes `web/src/components/HistoryDetailPane.tsx` --
adds three Tabs (Task / Output / Metrics), swaps the
ad-hoc date strings for TimeAgo, and renders commit
hashes via the canonical CopyButton primitive (which
wraps a Tooltip + clipboard + "Copied!" pulse).
Component-scope only, no daemon-side change and no `c4`
CLI surface change.

## Why

HistoryDetailPane is the right-pane detail surface
inside the History view: it shows a worker's recent
dispatches, the raw scrollback, and the commit history.
The previous shape was a single scrollable column with
two stacked sections (Past tasks + Scrollback) and
ad-hoc date formatting.

Three pain points motivated the polish:

1. **No way to focus on the scrollback.** The
   scrollback section sat below the past-tasks list
   and scrolled the operator past every record before
   landing on the live output. Operators frequently
   wanted "just the output", not the history.

2. **Date strings were locale-blind.** The inline
   `formatDate(iso)` helper rendered the ISO string
   minus 'T' (e.g. "2026-05-12 03:04:05"). That's
   neither relative ("5 min ago") nor locale-aware --
   the operator had to mentally subtract from "now"
   to gauge freshness.

3. **Commit hashes were copy-clumsy.** The hash
   rendered as `<code>{hash}</code>` -- text that the
   operator had to triple-click and Cmd+C to grab.
   The canonical CopyButton primitive (v1.11.255)
   already exists; HistoryDetailPane was the last
   commit-hash surface that hadn't adopted it.

Patch 11.315 fixes all three. Previous patches already
delivered the breadcrumbs (v1.11.301) and the
status-overlay Avatar (v1.11.300); this patch
completes the dispatch with Tabs, TimeAgo, and
CopyButton.

## What changed

### `web/src/components/HistoryDetailPane.tsx`

- **Tabs wrap the body.** A `<Tabs>` (v1.11.299) with
  three items splits the body:
  - `task` (default-active): the past-tasks list
    with pagination. Wrapped in
    `<section data-section="history-detail-task">`.
  - `output`: the raw scrollback `<pre>`. Wrapped in
    `<section data-section="history-detail-output">`.
  - `metrics`: a new record breakdown rendered via
    DataList. Wrapped in
    `<section data-section="history-detail-metrics">`.
  Tab order keeps Task first so the first-glance
  surface stays byte-identical (Task tab content was
  previously the first thing the operator saw).
- **TimeAgo for record timestamps.** The `startedAt`
  and `completedAt` strings now render through
  `<TimeAgo value={iso} variant="short" />`. The
  short variant emits a compact "5m" / "2h" /
  "1d" string; the absolute timestamp lands on
  `title=` for hover; the `<time dateTime=...>`
  element carries the ISO timestamp for e2e
  selectors. The completedAt span carries a
  leading "->" glyph (preserving the original
  arrow convention) inside a sibling span with
  `aria-hidden="true"`.
- **CopyButton for commit hashes.** Each commit row
  now renders a `<CopyButton value={hash} variant="icon+label">`
  with the short hash (`hash.slice(0, 7)`) as the
  visible label. Clicking the icon copies the full
  SHA to the clipboard; the Tooltip + "Copied!"
  pulse + clipboard write all come from the
  CopyButton primitive. The per-row `data-testid`
  is `history-record-commit-<short>` so e2e tests
  can target a specific commit.
- **Metrics tab.** New per-detail breakdown rendered
  via DataList:
  - Total records
  - Completed / merged (status includes "ok" /
    "complete" / "merged")
  - Errors / failures (status includes "error" /
    "fail")
  - Pending / busy (status includes "pending" /
    "busy")
  - Other / unknown (anything else)
  - Commits recorded (sum across all records'
    `commits[]`)

### Removed

- The inline `formatDate(iso)` helper -- its only
  callers were the two timestamp spans that now flow
  through TimeAgo.

### Data attributes

- `data-section="history-detail-body"` on the
  CardContent.
- `data-section="history-detail-task" | "history-detail-output" | "history-detail-metrics"` on
  the three tab section bodies.
- `data-section="history-record-started"` on the
  TimeAgo wrapper for the start time.
- `data-section="history-record-completed"` on the
  TimeAgo wrapper for the completion time.
- `data-section="history-record-commits"` on the
  commits sublist.

## Notes

- The Tabs primitive ships with role=tablist + arrow-
  key navigation + roving tabindex via the v1.11.320
  useRovingTabindex hook. Switching tabs preserves
  the pagination state (recordsPage) so a Metrics ->
  Task round-trip keeps the operator on the right
  page.
- The TimeAgo `short` variant emits compact strings
  ("5m", "2h", "1d") that fit inside the small
  metadata row without truncating. The absolute
  timestamp on hover is the canonical "what time
  was that?" affordance.
- The CopyButton's "Copied!" pulse is gated by
  useReducedMotion so operators with
  `prefers-reduced-motion: reduce` see a static
  glyph flip instead of an animated pulse.
- The Metrics tab renders zero counts gracefully:
  every category lands as a 0 in the DataList when
  no records match, so the section reads
  consistently across empty / non-empty histories.
- The 7-character short-hash slice matches the
  convention used by `git log --oneline` and the
  rest of the c4 web UI. Operators who want the
  full hash get it via the Copy click; the visible
  label stays compact.

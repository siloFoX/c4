## Summary

Adds the `ChartLineChoppinessMidCross` primitive
(`web/src/components/ui/chart-line-choppiness-mid-cross.tsx`): a
pure SVG dual-panel chart that visualizes the Choppiness Index
crossings through the 50 midline for trending versus ranging
regime baseline transitions. Companion to
`chart-line-choppiness-cross`, but with a single-threshold midline
contract instead of the dual Fibonacci 38.2 / 61.8 framing.
Together the two variants give operators a choice of "soft
midpoint split" (this one) or "hard trending / ranging buckets"
(the dual-threshold cross).

## Pipeline

Choppiness Index uses the close-only true range and the rolling
high-low range over a fixed window:

1. `tr[i] = |close[i] - close[i-1]|` (tr[0] = 0 by convention)
2. `sumTR[i] = sum(tr, length)` over the last `length` bars
3. `range[i] = max(close, length) - min(close, length)` over the
   same window
4. `ci[i] = range > 0 && sumTR > 0`
   `  ? 100 * log10(sumTR / range) / log10(length)`
   `  : 50  (zero-flow neutral fallback)`
5. Regime classifier (Choppiness-specific inversion of the
   "bullish above" convention -- LOW CI = strong trend = the
   actionable / momentum-friendly regime):
   - `ci <  50` -> bullish (trending bias)
   - `ci >= 50` -> bearish (ranging bias)
6. Cross detector:
   - `bullish` (trending start) when `ci[i-1] >= 50 && ci[i] < 50`
   - `bearish` (ranging start) when `ci[i-1] <= 50 && ci[i] > 50`
   - Strict inequalities on the current side means the boundary
     value 50 never double-fires.

## Defaults

- `length = 14` (canonical Choppiness Index window)
- `threshold = 50` (midline / centerline)
- Warmup boundary: bar `length - 1 = 13`, so the first 13 samples
  (30-bar run) are null and the remaining 17 are populated.

## Bit-exact anchors

- CONST `close = K` for all `K in {0, 1, 42, 100, 1234}` ->
  TR = 0 every bar -> sumTR = 0, range = 0 -> formula degenerates
  to the explicit neutral fallback `ci = 50` on every settled
  bar. 50 sits on the threshold and classifies as bearish (ci >=
  threshold), so the CONST run is fully bearish (ranging side)
  with 0 crosses.
- LINEAR UP `close = i` (length = 14): TR = 1 for every i >= 1
  with tr[0] = 0. At the seed bar i = 13, the window includes
  tr[0] = 0 so sumTR = 13 and range = 13 -> ratio = 1 ->
  `ci[13] = 0` exactly (bit-exact +0, NOT -0; verified via
  `Object.is`). From i = 14 onward, tr[0] drops out, sumTR = 14
  and range = 13 -> ci = `100 * log10(14/13) / log10(14)` ~
  2.808. Both values land well below 50, so the run is fully
  bullish (trending side) with 0 crosses (ci jumps from null
  directly to 0 then to ~2.808 without a `prev <= T && cur < T`
  transient).
- LINEAR DOWN `close = -i`: same shape by symmetry.
- Saw-tooth (alternating bodies, e.g. 10, 11, 10, 11, ...) -> for
  i >= length, sumTR = length and range = 1 -> ci = 100 exactly
  (the ceiling). The canonical "fully ranging" anchor.
- CONST K full-run counters under defaults, 30 bars:
  `noneCount = 13`, `bullishCount = 0`, `bearishCount = 17`,
  `crosses.length = 0`.
- `pricePath` and `ciPath` are byte-identical across repeated
  layout calls, with `.toFixed(2)` precision and `M ... L ...`
  command shapes. The ci path uses a single `M` for post-warmup
  data.
- `thresholdY` at 50 sits exactly at the panel midpoint
  ((oscTop + oscBottom) / 2) -- the canonical centerline.
- Cross detector boundary: strict inequality on the cur side.

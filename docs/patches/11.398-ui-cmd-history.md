## Behaviour

### Search

Case-insensitive substring match
against `command`, `output`, and
`cwd`. Whitespace trimmed. Empty
query returns everything.
Controlled-or-uncontrolled via
`query` / `defaultQuery` +
`onQueryChange`.

### Selection

Single-id model. Click selects;
ArrowDown / ArrowUp / Home / End
move the selection.
Controlled-or-uncontrolled via
`selectedId` / `defaultSelectedId` +
`onSelectedIdChange`.

### Counter

`<filtered> / <total>` shown next to
the search box. Always visible.

### Exit codes

Per-row badge with `data-status="ok"`
(exitCode === 0) or
`data-status="fail"` (any other
value). Hidden when
`showExitCodes={false}` or when the
entry has no `exitCode`.

### Timestamps

`HH:MM:SS` derived from
`entry.timestamp` (ms epoch). Hidden
when `showTimestamps={false}` or
when `timestamp` is missing.

### Copy buttons

| label | target | shown when |
| --- | --- | --- |
| Cmd | `command` | always |
| Out | `output` | `entry.output` is non-empty |
| All | `both` | `entry.output` is non-empty |

Click writes the canonical
serialization via
`navigator.clipboard.writeText` and
fires `onCopy(entry, target)`. The
callback fires even when clipboard
is unavailable so tests + non-https
hosts get a reliable side-channel.

Serialization (via
`formatCmdHistoryClipboard`):

- `command` -> `entry.command`
- `output` -> `entry.output ?? ''`
- `both` -> `entry.command +
  '\n---\n' + entry.output` (the
  `---` separator is omitted when
  output is empty)

### Scroll restoration

Opt-in via `storageKey`. The latest
`scrollTop` writes to
`localStorage[cmd-history-scroll:
<key>]` on every scroll, and
restores on mount via
`useLayoutEffect` (no flash).
Read/write errors swallowed.

### renderCommand

Optional override slot for the
default command text -- pages can
hyperlink, decorate, or replace the
inline label without losing the
copy / select wiring.

## ANSI parser

`parseAnsi(text)` walks the input
once with a `\x1b\[([0-9;]*)m`
pattern, applying SGR codes to a
rolling style and emitting one
segment per text span. Empty input
returns `[]`.

Supported SGR codes:

| code | effect |
| --- | --- |
| 0 | reset all attributes |
| 1 | bold |
| 2 | dim |
| 3 | italic |
| 4 | underline |
| 7 | inverse |
| 9 | strikethrough |
| 22 | bold + dim off |
| 23 | italic off |
| 24 | underline off |
| 27 | inverse off |
| 29 | strikethrough off |
| 30-37 | fg color (4-bit) |
| 39 | default fg |
| 40-47 | bg color (4-bit) |
| 49 | default bg |
| 90-97 | bright fg |
| 100-107 | bright bg |

Unsupported codes (including 256-
color `38;5;N` and truecolor
`38;2;R;G;B`) are ignored silently
-- the next text segment continues
under the rolling style.

`ansiStyleToClassName(style)` maps
to Tailwind utility classes (`text-
red-500`, `bg-emerald-500/30`,
`font-bold`, `opacity-60`, `italic`,
`underline`, `line-through`,
`invert`).

## Out of scope (deferred)

- 24-bit truecolor (`38;2;R;G;B` /
  `48;2;R;G;B`). The 4-bit + bright
  surface covers canonical bash /
  git / npm output.
- 256-color palette
  (`38;5;N` / `48;5;N`).
- OSC sequences (hyperlinks,
  cursor moves, title sets). Out
  of scope; these are not "color"
  payloads.
- Streaming append (rows added
  while watching). The host hands
  in the snapshot each render.
- Inline replay (re-execute the
  command). Belongs in the host.
- Multi-select. Single-id model
  for v1.
- Output search highlighting. The
  query filters rows; in-row match
  highlighting is a follow-on.

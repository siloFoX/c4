# 11.1087 -- UX FIX: skeleton loading states for perceived speed (v1.11.1105)

## Scope

The functional audit found loading states are ALREADY adequate on
Sessions, Meetings, Specialists, Wiki, Autonomous, and Settings (each
shows a skeleton or Loading indicator during fetch) -- those were left
untouched. This todo fixes only the two real gaps.

## Gap 1: ChartLineGallery lazy tiles

`ChartLineGallery` lazy-loads 41 chart-line primitives, each behind a
per-tile `Suspense`. The fallback was a static "Loading chart..." line,
so on first visit the grid read as 41 frozen labels while the chunks
fetched. The fallback is now an animated `Skeleton` (variant `rect`,
filling the 200px chart area), so each tile signals progress while its
chunk mounts. `data-section="chart-gallery-tile-skeleton"`.

## Gap 2: Features Workers hero stat tiles

`pages/Workers.tsx` (`HeroBody` / `useWorkerCounts`) polls `/api/list`
and starts every count at 0, so the Busy / Idle / Lost stat tiles
showed a literal `0` during the first fetch -- reading as real data and
making the panel look frozen. `useWorkerCounts` now returns a `loading`
flag (true until the first poll settles, success OR failure); while
loading the hero renders skeleton stat tiles
(`data-section="workers-hero-count-skeleton"`, label visible, count is a
Skeleton) instead of the zero-value blocks. The frame matches
`CountBlock` so the swap to real counts does not shift layout.

## Design system

Both use the existing `Skeleton` primitive (`components/ui/skeleton`,
`animate-pulse bg-muted`), which is the project's token-backed loading
shape -- no new colours or ad-hoc placeholders.

2. Throttles the lazy `chart-line-*` module requests (3s), opens the

Result: `2 passed`. Screenshots confirmed visually: Workers shows
skeleton bars under BUSY/IDLE/LOST; the gallery shows animated skeleton
blocks in every tile's chart area. The `Workers` unit suite was updated
(count tiles now render after load; added a skeleton-while-loading
test) -- `12 passed`.

## Risk

- `useWorkerCounts` return shape changed from `WorkerCounts` to
  `{ counts, loading }`; the only consumer is `HeroBody` (same file),
  updated in lockstep, and the unit suite covers both the loading and
  loaded paths.
- The gallery change is a fallback swap only; the chart-render path is
  unchanged.
- No change to the six views the audit flagged as already adequate.

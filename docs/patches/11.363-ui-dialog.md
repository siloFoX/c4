## `size` prop

```ts
type DialogSize = 'sm' | 'md' | 'lg' | 'full';
```

| size | max-width / clamp | typical use |
| --- | --- | --- |
| `sm` | `max-w-sm` | confirmations, single-question prompts |
| `md` | `max-w-lg` | default; legacy width preserved |
| `lg` | `max-w-3xl` | multi-field edit forms |
| `full` | `max-w-[min(96vw,1280px)] max-h-[95vh] h-[95vh] overflow-y-auto` | full-screen edit surfaces with internal scroll |

`md` is the default so every existing call site
stays byte-identical. The panel gains
`data-size="<size>"` for e2e selectors.

`className` overrides compose AFTER the size
class so adopters can layer custom widths when
the canonical four do not fit.

## Tests

`dialog.test.tsx` -- 6 new cases on top of the
existing 27:

- Default size = `md`; `max-w-lg` present;
  `data-size="md"`.
- `size="sm"` -> `max-w-sm`.
- `size="lg"` -> `max-w-3xl`.
- `size="full"` -> viewport-fill clamps
  (`96vw` + `max-h-[95vh]` + `h-[95vh]` +
  `overflow-y-auto`).
- `className` override composes after the
  size classes.
- `data-size` attribute is always present
  across all four sizes.

33/33 pass under vitest 4.1.5. TypeScript
clean.

## Pairs with existing primitives

- `useFocusTrap` -- unchanged.
- `getPortalRoot('dialog-root')` -- unchanged.
- Variants (`default` / `destructive` /
  `confirmation`) remain orthogonal to size;
  combine freely.
- `data-size` attribute joins `data-variant`
  so e2e selectors can target the visual axis
  they care about.

## Out of scope

- Per-page adoption (Snapshots restore /
  delete, Templates import, Profiles delete
  etc). The new size prop is additive; existing
  call sites pick a size when they need it.
- An `xs` size below `sm`.
- Responsive size cascades (small viewport
  collapses `lg` to `md`). Tailwind's
  `max-w-*` self-clamps to the viewport width
  already.
- Animation differences per size. The
  `motionClass('scaleIn')` transition stays
  identical.

## Goal

Dispatch:

- Build `web/src/components/ui/badge.tsx` +
  `chip.tsx` -- status badge (success / warning
  / error / info), removable chip with delete
  handler, count badge.

Existing primitives cover the three shapes:

- `<Badge>` (11.229 / v1.11.247) -- status
  variants with leading signal icons. No
  `size` scale though.
- `<Chip>` (existing) -- removable chip with
  `onDismiss` + `dismissLabel`. Tones, variants,
  sizes already shipped.
- `<BadgeCounter>` (11.278 / v1.11.296) -- count
  / dot badge with tones, sizes, pulse, overflow.

This patch rounds out the contract:

1. Adds a `size: 'sm' | 'md' | 'lg'` scale to
   `Badge` so callers can pick a density. `md`
   is the legacy default and stays
   byte-identical (`px-2.5 py-0.5 text-xs`).
2. Re-exports `BadgeCounter` as `CountBadge`
   from `badge.tsx` so the dispatch-style name
   resolves at the canonical import path.

## API

```tsx
import { Badge, CountBadge } from './components/ui/badge';

<Badge variant="success">ready</Badge>          // md (default)
<Badge variant="success" size="sm">ready</Badge>
<Badge variant="warning" size="lg">overdue</Badge>

<CountBadge count={3} tone="danger" />
<CountBadge count={250} max={99} />             // -> "99+"
<CountBadge count={0} showZero />               // -> "0"
```

## Size geometry

| size | padding | text class |
| --- | --- | --- |
| `sm` | `px-2 py-0.5` | `text-[10px]` |
| `md` (default) | `px-2.5 py-0.5` | `text-xs` |
| `lg` | `px-3 py-1` | `text-sm` |

`data-size` attribute is exposed on the
rendered `<span>` so downstream tests /
storyshots can assert the resolved size
without parsing the class list.

## CountBadge alias

```ts
export {
  BadgeCounter as CountBadge,
  type BadgeCounterProps as CountBadgeProps,
  type BadgeCounterTone as CountBadgeTone,
  type BadgeCounterSize as CountBadgeSize,
  type BadgeCounterVariant as CountBadgeVariant,
} from './badge-counter';
```

Both names resolve to the same component.
`BadgeCounterProps` keep their full surface
(numeric overflow `max`, `showZero`, tones,
sizes, pulse, sr-label, dot variant).

## Pairs with existing primitives

- `<Chip>` -- interactive removable token.
  Unchanged.
- `<StatusDot>` (`status-dot.tsx`) -- pure
  signal dot, no text. Pairs with `Badge`
  when a row needs a colour + label.
- `<StatusPill>` (`status-pill.tsx`) -- text
  pill without the icon affordance.

## Out of scope

- Per-page adoption swap (Badge -> sized
  Badge). The new size prop is additive;
  every existing call site stays
  byte-identical at `md`.
- Replacing `Chip` with `Badge` (or vice
  versa). The two surfaces serve different
  semantics: Badge = read-only status pill;
  Chip = interactive removable token. Both
  stay.
- A new `xs` Badge size. The 10px text floor
  in `sm` already hits the readability
  threshold.
- Removing `data-size` from the rendered DOM.
  The attribute is cheap and lets downstream
  tests / storyshots assert without parsing
  classes.

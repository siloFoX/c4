## Summary

`web/src/components/ui/chart-line-vsa.tsx` ships `<ChartLineVsa>`, a
pure-SVG two-panel chart with a **Volume Spread Analysis** panel
highlighting effort-versus-result divergences from bar range against
volume. The top panel plots the bar midpoint; the bottom panel plots
the VSA index with a horizontal zero line and dashed `+/-threshold`
reference lines that classify each defined bar by zone.

## Computation

Each bar carries a SPREAD (`high - low`) and a VOLUME. For a rolling
window of `period` bars:

```
spreadMean = mean(spread)   over the window
volumeMean = mean(volume)   over the window
effort     = volume / volumeMean   for the current bar
result     = spread / spreadMean   for the current bar
vsa        = effort - result
```

A bar where the effort and the result agree gets `vsa ~ 0`. A bar
where the volume punches above its mean while the spread sits at or
below its mean reads `vsa > 0` -- the **no-demand** pattern (a lot
of effort produces no result). The mirror reads `vsa < 0` -- the
**ease-of-move** pattern (a big move on light volume). A zero-mean
spread or zero-mean volume window is degenerate (the ratios are
undefined); those bars are left null.

## Worked anchors

The arithmetic is integer-rational when the spread and volume sums
divide cleanly. The implementation has three exact-style anchors.

- **Equal-all anchor** -- spreads `[4 x 5]`, volumes `[4 x 5]`,
  period 5. `spreadMean = 4`, `volumeMean = 4`, both means exact.
  At bar 4 (the only defined bar): `effort = 4/4 = 1`,
  `result = 4/4 = 1`, `VSA = 0`. Asserted with `toBe(0)`.
- **Ease-of-move anchor** -- spreads `[2, 2, 2, 2, 12]`, volumes
  `[4, 4, 4, 4, 4]`, period 5. `spreadMean = (2+2+2+2+12)/5 = 20/5
  = 4` (exact). `volumeMean = 4` (exact). At bar 4: `spread = 12`,
  `volume = 4`. `effort = 4/4 = 1` (exact), `result = 12/4 = 3`
  (exact), `VSA = 1 - 3 = -2`. Asserted with `toBe(-2)`. Zone
  classification at threshold 0.5: `-2 < -0.5` -> `ease-of-move`.
- **No-demand anchor** -- spreads `[4 x 5]`, volumes
  `[2, 2, 2, 2, 12]`, period 5. `spreadMean = 4`, `volumeMean = 4`.
  At bar 4: `spread = 4`, `volume = 12`. `effort = 12/4 = 3`,
  `result = 4/4 = 1`, `VSA = 3 - 1 = 2`. Asserted with `toBe(2)`.
  Zone classification at threshold 0.5: `2 > 0.5` -> `no-demand`.

The degenerate-window cases are also exact: a window with every
bar at `high == low` gives `spreadMean = 0`, leaving every VSA null;
a window with every volume zero gives `volumeMean = 0`, same null
result.

## Distinct from neighbouring primitives

- vs `<ChartLineVroc>` -- the VROC is the percent change of volume
  over a lookback; the VSA cross-checks volume AGAINST the bar
  spread, surfacing the relationship between effort and result.
- vs `<ChartLineVfi>` -- the VFI is a signed signed-volume cumulative
  flow; the VSA is a per-bar ratio comparing two normalised series.
- vs `<ChartLineUpDownVolumeRatio>` -- the up/down ratio measures the
  share of volume on up vs down bars; the VSA does not care about
  direction, only about whether the volume produces a spread
  response.
- vs `<ChartLineMahalanobis>` over `[volume, spread]` -- the
  Mahalanobis distance computes a covariance-aware multivariate
  z-score; the VSA is the cleaner ratio-based effort-minus-result
  divergence that classifies by sign.

## API

`data` -- a single `ChartLineVsaPoint[]` (`{x, high, low, volume}`).
`period` (default 14, integer >= 2). `threshold` (default 0.5,
strictly positive finite). Standard sizing / colour / a11y props
plus `pricePanelRatio` (default 0.55, clamped to `[0.3, 0.8]`),
`gap`, `strokeWidth`, `dotRadius`, `tickCount`. Toggle flags:
`showAxis`, `showGrid`, `showDots` (default false), `showVsa`,
`showZeroLine`, `showThresholdLines`, `showMarkers`, `showTooltip`,
`showConfigBadge`, `showLegend`, `animate`. Controlled and
uncontrolled visibility via `hiddenSeries` (`'price' | 'vsa'`).
Callbacks: `onPointClick({point})`,
`onSeriesToggle({seriesId, hidden})`. Format functions:
`formatValue`, `formatX`. ARIA: `ariaLabel`, `ariaDescription`.

## Pure helpers

- `DEFAULT_CHART_LINE_VSA_*` -- defaults.
- `getLineVsaFinitePoints` -- keep only points with finite
  `x` / `high` / `low` / `volume`, `high >= low`, and `volume >= 0`.
- `normalizeLineVsaPeriod(period, fallback)` -- integer >= 2;
  else fallback.
- `normalizeLineVsaThreshold(threshold, fallback)` -- finite > 0;
  else fallback.
- `computeLineVsa(bars, period)` --
  `{effort, result, vsa}` per bar; warm-up null; degenerate window
  null.
- `classifyLineVsaZone(vsa, threshold)` --
  `'agreement' | 'no-demand' | 'ease-of-move' | 'none'`; the
  at-threshold boundary is treated as agreement (strict inequality).
- `runLineVsa(points, options?)` --
  `{series, period, threshold, vsa, samples, vsaFinal,
  agreementCount, noDemandCount, easeOfMoveCount, ok}`.
- `computeLineVsaLayout({data, period, threshold, ...})` -- two
  panels with the midpoint line on top, the VSA line below in a
  symmetric domain, one marker per defined-VSA bar coloured by
  zone, plus `zeroY`, `upperThresholdY`, `lowerThresholdY` for the
  reference lines.
- `describeLineVsaChart(data, options?)` -- ARIA summary mentioning
  the indicator, the effort / result decomposition, the
  no-demand / ease-of-move / agreement zones, and the final
  reading.

## a11y / data hooks

Root `<div role="region">` with `aria-label` + `aria-describedby`; the
SVG is `role="img"` with the same label; an sr-only
`data-section="chart-line-vsa-aria-desc"` span carries the
description. The midpoint line, every price dot, the VSA line and
every marker carry `role="graphics-symbol"` and `tabIndex={0}`.
Every node carries a `data-section`. Root mirrors `data-period`,
`data-threshold`, `data-vsa-final`, `data-agreement-count`,
`data-no-demand-count`, `data-ease-of-move-count`,
`data-total-points`, `data-animate`, `data-empty`. Grid / axis /
tick / panel-label nodes carry `data-panel="price"|"vsa"`. Threshold
lines carry `data-direction="upper"|"lower"`. Markers expose
`data-zone` and `data-vsa`.

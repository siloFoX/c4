## API

```ts
type EnvKind =
  | 'production' | 'staging' | 'development'
  | 'preview' | 'test' | 'local' | 'unknown';

interface EnvBadgeOption {
  env: string;
  label?: string;
  description?: ReactNode;
}

interface EnvBadgeProps {
  env: string;
  label?: string;                     // overrides resolved label
  description?: ReactNode;
  options?: readonly EnvBadgeOption[];
  onSwitch?: (env: string) => void;
  size?: 'sm' | 'md' | 'lg';          // default 'md'
  ariaLabel?: string;
  icon?: ReactNode;                   // overrides default per-kind icon
  showLabel?: boolean;                // default true
  className?: string;
}

interface EnvBannerProps extends EnvBadgeProps {
  bannerClassName?: string;
  align?: 'left' | 'center' | 'right'; // default 'left'
}
```

## Behaviour

- Renders a chip with the resolved env label, a per-kind
  icon, and the per-kind color class.
- Becomes interactive when both `options` and `onSwitch` are
  supplied: the chip becomes a `<button>` with
  `aria-haspopup="listbox"` + `aria-expanded`; clicking
  opens a listbox of the configured environments.
- Listbox options carry `role="option"` + `aria-selected`
  for the current env. Clicking fires `onSwitch(env)` and
  closes the popover.
- Escape closes the open popover. Mousedown outside both
  the trigger and the popover closes it.
- `description` slot renders inline after the chip
  (non-interactive layout only; interactive flows surface
  descriptions inside the per-option rows).
- `EnvBadge.Banner` wraps the chip in a full-width
  `role="banner"` strip with the per-kind background class
  and an optional inline description. Hosts drop this into
  the app chrome / top navigation area.

## Pure helpers (exported)

```ts
ENV_KINDS = ['production','staging','development','preview','test','local','unknown']
getEnvKind(env): EnvKind
getEnvLabel(env): string
getEnvBadgeClass(env): string
getEnvBannerClass(env): string
isProductionEnv(env): boolean
```

- `getEnvKind` aliases the common spellings:
  - production: `production` / `prod` / `live` / `release`
  - staging: `staging` / `stage` / `uat` / `qa`
  - development: `dev` / `development`
  - preview: `preview` / `pr`
  - test: `test` / `testing` / `ci`
  - local: `local` / `localhost`
  Trims + case-insensitive; unknown spellings map to
  `'unknown'`.
- `getEnvLabel` returns `'Production'` / `'Staging'` / ... /
  `'Environment'` for unknown.
- `getEnvBadgeClass` / `getEnvBannerClass` return the
  Tailwind class for the resolved kind.
- `isProductionEnv` is a convenience for callers that want
  to gate destructive UI on prod.

## ARIA + data attributes

- Root region: `role="status"` + `aria-label` (default:
  `"<Label> environment"`).
- Interactive trigger: `aria-haspopup="listbox"` +
  `aria-expanded`.
- Popover: `role="listbox"` + `aria-label`.
- Per-option: `role="option"` + `aria-selected`.
- Banner: `role="banner"` + `aria-label` (default:
  `"<Label> environment banner"`).
- `data-section` on every node:
  - `env-badge` (root region)
  - `env-badge-icon`
  - `env-badge-label`
  - `env-badge-trigger`
  - `env-badge-description`
  - `env-badge-popover`
  - `env-badge-option`
  - `env-badge-option-icon`
  - `env-badge-option-label`
  - `env-badge-option-description`
  - `env-banner` (banner root)
  - `env-banner-description`
- Root mirrors `data-env` (raw prop), `data-env-kind`
  (resolved), `data-size`, `data-interactive`, `data-open`.
  Per-option mirrors `data-env`, `data-env-kind`,
  `data-current`. Banner mirrors `data-env`,
  `data-env-kind`, `data-align`.

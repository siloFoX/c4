# 11.151 -- UI: Fieldset primitive

Released in v1.11.169.

New `<Fieldset>` primitive at `web/src/components/ui/fieldset.tsx`.
Semantic `<fieldset>` + `<legend>` wrapper for grouping related form
controls.

## Props

- `legend?: ReactNode` -- rendered inside `<legend>`
- `description?: ReactNode` -- small muted `<p>` below the legend
- `collapsible?: boolean` -- legend becomes a toggle button (chevron + aria-expanded)
- `defaultOpen?: boolean` (default `true`)
- `open?: boolean` -- controlled mode
- `onOpenChange?: (open: boolean) => void`
- `disabled?: boolean` -- forwards to `<fieldset disabled>`
- `className?: string`
- `children: ReactNode`

When `collapsible` is true the content stays in the DOM and is hidden
via the native `hidden` attribute so form state is preserved across
toggles. `prefers-reduced-motion` drops the chevron rotation
animation.

## Adoption sites (3)

1. `web/src/pages/Profiles.tsx` -- the expanded permission row now
   wraps the allow + deny `PatternList` grid in a Fieldset legend
   ("Allow / Deny").
2. `web/src/components/RiskRuleCatalogPanel.tsx` -- the filter input
   and the per-severity rule list inside the rule-catalog panel are
   grouped under a Fieldset legend ("Filter rule catalog"), the
   group used as the Risk classifier filter group target.
3. `web/src/components/NewChatModal.tsx` -- the model + agent grid
   is wrapped in a Fieldset legend ("Routing"). The Profiles edit
   form alt (per task) -- NewChatModal is the only place in the
   sessions flow with a related-controls group of selects, used as
   the third adoption site because Health has no settings form and
   Profiles/Config sections did not have a clean alternative.

# Patch 11.900 -- UI: chart-line-stoch-cross-sig primitive

## Summary

Added `<ChartLineStochCrossSig />` -- pure-SVG dual-panel React
/TS primitive that renders the close (top panel) with bullish /
bearish arrow overlays at every Stochastic %K vs signal cross
trigger, and a close-only Stochastic %K alongside its EMA-
smoothed signal in the bottom panel with cross dot markers.
Signal-cross variant of the Stochastic family that separates
the EMA-trigger events from the canonical %K over %D crossings
published by the base Stochastic primitives.

## Algorithm

```
range[i]   = highestClose[i-n+1..i] - lowestClose[i-n+1..i]
rawK[i]    = range === 0
              ? 50
              : ((close[i] - lowestClose) / range) * 100
k[i]       = SMA(rawK, kSmoothing)
signal[i]  = EMA(k, signalLength)
bullish   : (k - signal) crosses up   (prev <= 0, cur > 0)
bearish   : (k - signal) crosses down (prev >= 0, cur < 0)
```

Defaults `length = 14`, `kSmoothing = 3`, `signalLength = 3`
(canonical slow Stochastic). Regime classifier `bullish` (k >
signal), `bearish` (k < signal), `neutral` (k === signal),
`none` (either null).

## Bit-exact anchor

CONST close = K: range = 0 every bar -> rawK = 50 neutral
fallback. SMA of 50s = 50, signal EMA of 50s = 50 via the
`min === max` precision short-circuit. k === signal on every
settled bar, regime `neutral`, cross count = 0. Verified
across K = 0, 1, 5, 17, 100, 1234.

## Visual

- Two stacked panels with the osc panel pinned to 0-100.
- Top: close line + bullish/bearish polygon arrows at cross
  bars.
- Bottom: %K + signal lines + cross dot markers + 20/50/80
  reference bands (overbought / mid / oversold).
- Tooltip lists x / close / %K / signal / regime / counts /
  crosses.
- Config badge + togglable legend (close / %K / signal).
- Cross marker is `role="graphics-symbol"` with `tabIndex={0}`.

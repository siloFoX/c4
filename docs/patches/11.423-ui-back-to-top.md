## API

```ts
interface BackToTopProps {
  threshold?: number;                // px, default 200
  scrollBehavior?: 'smooth' | 'auto'; // default 'smooth'
  scrollTarget?: HTMLElement | Window | null;
  ariaLabel?: string;                // default 'Back to top'
  position?: 'bottom-right' | 'bottom-left' | 'bottom-center';
  icon?: ReactNode;                  // default ArrowUp
  showLabel?: boolean;               // default false
  label?: ReactNode;                 // default 'Back to top'
  className?: string;
  onClick?: () => void;
  initiallyVisible?: boolean;        // SSR / test override
  visible?: boolean;                 // fully controlled
  hideOnVisible?: boolean;           // default true
}
```

## Behaviour

- Renders a `<button>` fixed to one of three corner positions
  (`bottom-right`, `bottom-left`, `bottom-center`).
- Subscribes to the watched target's `scroll` event with
  `{ passive: true }` and flips visibility once the position
  crosses `threshold`. The listener detaches on unmount.
- The initial visibility is computed from `initiallyVisible`
  (default `false`) and then upgraded to `true` on mount **only
  when the current scroll position already meets the
  threshold**. This preserves an explicit `initiallyVisible=
  true` override on a fresh page (scrollY=0) while still
  revealing the button when the user navigates to a deep-link
  / restored scroll position past the threshold.
- Click handler fires `onClick`, calls
  `scrollTargetToTop(target, scrollBehavior)`, and (when
  uncontrolled + `hideOnVisible`) optimistically hides the
  button. The next scroll event re-syncs once smooth-scroll
  lands at zero.
- Controlled `visible` always wins; clicks still call the
  scroll target + `onClick` but do not flip local state.

## Pure helpers (exported)

```ts
DEFAULT_BACK_TO_TOP_THRESHOLD = 200
DEFAULT_BACK_TO_TOP_POSITION = 'bottom-right'
DEFAULT_BACK_TO_TOP_LABEL = 'Back to top'
shouldShowBackToTop(scrollY, threshold?): boolean
getScrollTop(target): number
scrollTargetToTop(target, behavior?): void
```

- `shouldShowBackToTop` is inclusive at the threshold (a 200px
  scroll with a 200px threshold counts as "show"), negative
  thresholds collapse to 0, and NaN inputs return `false`.
- `getScrollTop` accepts `window`, `HTMLElement`, or null /
  undefined; reads `scrollY` for `window` and `scrollTop` for
  elements; falls back to
  `document.documentElement.scrollTop` when `window.scrollY` is
  missing.
- `scrollTargetToTop` uses `window.scrollTo` /
  `element.scrollTo`; falls back to setting `element.scrollTop
  = 0` when `scrollTo` is missing (jsdom-safe).

## ARIA + data attributes

- Button: `aria-label` (default `"Back to top"`);
  `aria-hidden="true"` when invisible (the control is removed
  from the accessibility tree until needed); `tabIndex=-1` when
  hidden, `0` when visible.
- `data-section` on every node:
  - `back-to-top` (root button)
  - `back-to-top-icon` (icon slot)
  - `back-to-top-label` (label slot when `showLabel=true`)
- Button mirrors `data-visible` (`'true'`/`'false'`) and
  `data-position`.

## Tests (46 cases)

- All three pure helpers across below / at / above threshold
  inclusivity, default threshold, negative threshold, NaN
  guards, window vs. element scrollTop reads, and the
  jsdom-safe `scrollTop` fallback.
- Default + custom `ariaLabel`.
- Initially hidden when `scrollY < threshold`.
- `initiallyVisible` flag forces visible on mount.
- Controlled `visible=true` / `visible=false` overrides.
- Scroll event flips visibility when crossing the threshold.
- `aria-hidden` absent when visible, `'true'` when hidden.
- `tabIndex` swaps -1/0 with visibility.
- Click calls `window.scrollTo({ top: 0, behavior })` with the
  configured scrollBehavior.
- `onClick` fires before scroll.
- Uncontrolled click optimistically hides the button;
  `hideOnVisible=false` blocks the hide; controlled visible
  never flips internally.
- Default icon is rendered; custom `icon` prop replaces it;
  `showLabel=true` renders the label slot (hidden by default).
- Position prop reflects on `data-position`; default is
  `bottom-right`.
- Custom `scrollTarget` HTMLElement wires the scroll listener
  on the element + the click calls `element.scrollTo`.
- Stable `displayName` + `forwardRef` to the button.
- `data-section` markers present on root and icon slot.
- `threshold=0` makes the button visible from scrollY=0.

## Pairs with existing primitives

- `<ScrollSpy>` (11.422) for intra-document section navigation
  -- a back-to-top button often sits alongside.
- `<Tooltip>` to label the icon-only variant on hover for
  sighted users.
- `<Toast>` undo-row pattern for transient floating
  affordances at the same bottom anchor.

## Out of scope

- Auto-hide after N seconds of inactivity (host wraps with a
  hover/idle timer).
- Animated scroll progress ring around the button (host
  composes with `<Progress>` if needed).
- Multi-target (the primitive watches a single target; host
  renders multiple `<BackToTop>` for nested scroll panes).
- Keyboard global shortcut (host wires a shortcut handler ->
  `ref.current?.click()`).

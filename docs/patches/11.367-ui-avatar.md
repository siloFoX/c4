Existing primitives cover most of the

  11.282 / v1.11.300) -- image with `<Image>`-

## API

```tsx
import { Avatar, AvatarGroup } from './components/ui/avatar';

<Avatar name="Alice" size="xl" />                                  // 48px hero tile
<Avatar src="/u.png" alt="Alice" size="xl" status="online" />     // hero + status dot
<AvatarGroup items={[{ name: 'Alice' }, ...]} max={5} size="xl" />
```

## Geometry

### Avatar size scale

| size | tile | text | status dot |
| --- | --- | --- | --- |
| `xs` | `h-5 w-5` | `text-[10px]` | `h-1.5 w-1.5` |
| `sm` | `h-6 w-6` | `text-xs` | `h-2 w-2` |
| `md` (default) | `h-8 w-8` | `text-sm` | `h-2.5 w-2.5` |
| `lg` | `h-10 w-10` | `text-base` | `h-3 w-3` |
| `xl` (new) | `h-12 w-12` | `text-lg` | `h-3.5 w-3.5` |

The status-dot diameter scales 1.5/2/2.5/3/3.5
keeping the ~1/3-tile rhythm consistent across
the matrix.

### AvatarGroup size scale

| size | adjacent overlap | overflow chip |
| --- | --- | --- |
| `xs` | `-ml-1.5` | `h-5 w-5 text-[9px]` |
| `sm` | `-ml-2` | `h-6 w-6 text-[10px]` |
| `md` | `-ml-3` | `h-8 w-8 text-xs` |
| `lg` | `-ml-4` | `h-10 w-10 text-sm` |
| `xl` (new) | `-ml-5` | `h-12 w-12 text-base` |

`-ml-5` (negative 20px) pulls the next tile
left so the visible crescent on adjacent
avatars matches the rhythm at smaller tiers.

## Why xl exists

The four pre-existing tiers (xs/sm/md/lg)
covered row cells (xs), inline headers (sm),
page panels (md), and modal headers (lg). The
hero / profile-header surface (account
sidebar, "welcome" cards, oncall takeover
banner) needs an even larger tile. `xl` =
48px tile + `text-lg` initials fills that
slot.

## Pairs with existing primitives

- `<Image>` -- the fallback-initials path
  inside Avatar. Unchanged.
- `<StatusDot>` -- shares the palette token
  with the Avatar status overlay.
- `<AvatarGroup>` -- now matches the same
  size scale.

## Out of scope

- Per-page adoption of the new `xl` tier. The
  prop is additive; every existing call site
  stays byte-identical at its current size.
- A new `2xl` tier. The 48px tile already
  hits the "hero card / profile header"
  density; larger surfaces should use a
  full-bleed `<Image>` instead.
- A `size="auto"` mode that scales to
  container width. The fixed-tile contract is
  intentional for predictable stacking.
- Reflowing the overlap formula to a
  percentage-based offset. Tailwind `-ml-N`
  stays consistent with the rest of the
  primitive library.

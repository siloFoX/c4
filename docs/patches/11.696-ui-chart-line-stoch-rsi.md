## Summary

New `ChartLineStochRsi` UI primitive in
`web/src/components/ui/chart-line-stoch-rsi.tsx` -- a pure-SVG two-panel
line chart with a **Stochastic RSI** panel applying the stochastic
oscillator formula to the RSI series. The top panel plots the price; the
bottom panel plots the Stochastic RSI.

## The computation

The Stochastic RSI applies the stochastic oscillator formula to the RSI
series rather than to the price: it measures where the current RSI sits
within its own recent high-low range, on a 0..100 scale. It is an
"indicator of an indicator" and is far more sensitive than the plain RSI.
It is a close-only indicator, so `ChartLineStochRsiPoint` carries
`{x, value}`.

- `computeLineStochRsiRsi(closes, rsiPeriod)` -- Wilder's RSI of the close
  series.
- `computeLineStochRsiStoch(values, stochPeriod)` -- the stochastic
  oscillator formula applied to an array:
  `100 * (value - lowest) / (highest - lowest)` over the trailing window.
  A window touching a non-finite slot, or a flat window (highest equals
  lowest), yields null.
- `computeLineStochRsi(closes, rsiPeriod, stochPeriod)` -- composes the
  two: the stochastic of the RSI.

Each defined bar is classified against the thresholds: **overbought**
above the upper, **oversold** below the lower, **neutral** between.

## Worked fixture

The RSI is transcendental for a mixed series, but the pieces have exact
anchors:

- **The stochastic formula** is exactly testable. The fixture
  `[20,60,40,80,0,25]` with a 3-slot window gives `[null,null,50,100,0,
  31.25]` -- asserted with `toEqual`. The newest value topping the window
  reads 100, the newest at the window low reads 0.
- **The RSI** has monotone anchors: a strictly rising series has no
  losses so the RSI pins at exactly 100; a strictly falling one pins at 0.
- **The composed pipeline**: a strictly rising price pins the RSI at 100,
  a flat RSI window has no stochastic, so the Stochastic RSI is all null
  for a trending series -- an exact end-to-end anchor.

The run / component use a choppy fixture and are checked structurally:
every Stochastic RSI reading is null or within 0..100, every zone is
valid, the counts are self-consistent.

## API

`data` (a `ChartLineStochRsiPoint[]`), `rsiPeriod` (default 14),
`stochPeriod` (default 14), `upperThreshold` (default 80), `lowerThreshold`
(default 20), `pricePanelRatio` / `gap`, sizing/format/a11y props,
`strokeWidth` / `dotRadius`, the colour props, the toggle flags (`showAxis`
/ `showGrid` / `showDots` default false / `showStoch` / `showThresholds` /
`showMarkers` / `showTooltip` / `showConfigBadge` / `showLegend` /
`animate`), controlled + uncontrolled `hiddenSeries` (keys `price` /
`stoch`), `formatValue` / `formatX`, `onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineStochRsiFinitePoints`,
`normalizeLineStochRsiPeriod`, `computeLineStochRsiRsi`,
`computeLineStochRsiStoch`, `computeLineStochRsi`,
`classifyLineStochRsiZone`, `runLineStochRsi`, `computeLineStochRsiLayout`,
`describeLineStochRsiChart`, plus the `DEFAULT_CHART_LINE_STOCH_RSI_*`
constants and the `ChartLineStochRsiZone` type. Every exported name
carries a `LineStochRsi` infix so the `export *` barrel line cannot
collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen `data-section="chart-line-stoch-rsi-aria-desc"`
description span; the price line, the Stochastic RSI line and every dot /
marker are `role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-rsi-period`, `data-stoch-period`,
`data-upper-threshold`, `data-lower-threshold`, `data-stoch-rsi-final`,
`data-overbought-count`, `data-oversold-count`, `data-neutral-count`,
`data-total-points` and `data-animate`.

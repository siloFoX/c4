# 11.554 -- web/src/components/ui/chart-line-cusum.tsx

Pure-SVG two-panel line chart with a **CUSUM (cumulative
sum) drift detector** and **reset-on-trigger markers**. The
main panel (top) renders the raw series plus a dashed target
reference line; the CUSUM sub-panel (bottom) renders the two
cumulative sums with their decision thresholds; drift
triggers are marked on both panels.

## Math

The two-sided **tabular CUSUM** is the canonical sequential
change-detection method. For a target mean `T` and a
reference allowance `k` it tracks two non-negative running
sums:

```
deviation_t = y_t - T
C+_t = max(0, C+_{t-1} + deviation_t - allowance)
C-_t = max(0, C-_{t-1} - deviation_t - allowance)
```

- `C+` accumulates persistent **upward** drift.
- `C-` accumulates persistent **downward** drift.
- Both are floored at 0 by the `max`, so a brief excursion
  that reverts decays back toward 0 rather than persisting.

An alarm fires when either sum crosses the **decision
interval** `h`. The slack and threshold are expressed in
sigma units, so the actual values are:

```
allowance        = slack     * sigma
decisionInterval = threshold * sigma
```

with defaults `slack = 0.5` and `threshold = 5` (the
textbook k = 0.5 sigma, h = 5 sigma tuning, which catches a
1-sigma sustained shift quickly with a low false-alarm
rate).

### Reset-on-trigger

When a sum crosses its decision interval the chart records
the crossing value (so the triggering spike is visible) and
then **resets that sum to 0**, so detection of the next
shift starts fresh. This is what makes CUSUM a *repeated*
detector rather than a one-shot one: a process that drifts,
is corrected, and drifts again produces a separate trigger
each time. A test verifies the reset directly -- after a
single huge spike triggers, the very next sample's `C+` is
back at 0.

### Verified properties (by test)

- A series constant at the target produces `C+ = C- = 0`
  throughout and zero triggers.
- An upward shift accumulates `C+` step by step
  (`3 -> 6 -> 9 -> 12`) and triggers when it crosses
  `h = 10`.
- A downward shift accumulates `C-` symmetrically.
- After a trigger the sum resets to 0 (reset-on-trigger).
- A `threshold` of 0 disables triggering.
- Both sums are always non-negative.

## Distinct from related primitives

- `<ChartLineControl>` (11.534): a Shewhart SPC chart with
  CL / UCL / LCL at `mean +- k*sigma`. The control chart is
  **memoryless** -- each point is judged independently, so
  it only catches *large* single excursions. CUSUM has
  **memory** -- it accumulates small deviations, so a small
  *persistent* shift is caught far sooner. CUSUM is the
  standard answer to "the control chart is too slow to
  notice a 1-sigma drift."
- `<ChartLineChangepoint>` (11.546): detects *variance*
  shifts via a windowed variance-ratio score. CUSUM detects
  *mean* shifts via the cumulative sum, and signals
  sequentially rather than scoring every index.
- `<ChartLineCumulative>` (11.520): a plain running sum of
  the raw values. CUSUM is the running sum of
  *deviations from a target, minus a slack*, floored at 0,
  with reset -- a detector, not an accumulator.
- `<ChartLineAnomaly>` (11.524): per-point z-score outlier
  flags. Each point is independent; CUSUM is sequential and
  cumulative.
- `<ChartLineControl>` and `<ChartLineAnomaly>` answer "is
  *this point* unusual?"; CUSUM answers "has the *process
  mean* shifted, and when?".

## Public surface

- `<ChartLineCusum series={...} />`.
- `series: ChartLineCusumSeries[]` --
  `{id, label, data, color?, target?, sigma?}`. Per-series
  `target` / `sigma` overrides beat chart-level.
- `target` -- expected mean (defaults to the series mean).
- `sigma` -- scale for slack + threshold (defaults to the
  population standard deviation).
- `slack` (default 0.5) -- reference allowance in sigma
  units; clamped to >= 0.
- `threshold` (default 5) -- decision interval in sigma
  units; clamped to >= 0 (0 disables triggering).
- `subHeightRatio` (default 0.4; clamped to `[0.1, 0.9]`).
- `targetColor`, `thresholdColor`, `triggerColor`.
- Toggle flags: `showAxis`, `showGrid`, `showDots`,
  `showLegend`, `showTooltip`, `showConfigBadge`,
  `showTargetLine`, `showThresholds`, `showTriggers`,
  `showTriggerLines`, `animate`.
- Controlled + uncontrolled visibility.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX` / `formatCusum`.
- `onPointClick({series, point})`,
  `onTriggerClick({series, trigger})`.

- `computeLineCusumLayout({...})` -- projects the main +
  sub panels, raw + CUSUM paths, target line, thresholds,
  and the trigger list.
- `describeLineCusumChart(series, options?)`.

## Layout

Two stacked SVG panels sharing the x axis:
- **Main panel** (top, ~60%): the raw series line + a
  dashed horizontal target reference line.
- **CUSUM sub-panel** (bottom, ~40%): `C+` plotted upward
  from a centred zero line, `C-` plotted downward (negated),
  dashed decision-interval thresholds at `+h` and `-h`. The
  sub-panel y range is symmetric around zero.

Drift triggers render as diamond markers on the sub-panel
(at the crossing CUSUM value) plus a dashed vertical
trigger line through the main panel at the drift x.

## Tooltip

Hovering a dot shows the series label, x, raw value, the
deviation from target (with `+`/`-` prefix), bold `CUSUM+`,
`CUSUM-`, and -- when the sample triggered -- a coloured
`drift trigger (<side>)` row.

## Config badge

Top-left badge shows
`CUSUM T=<target> k=<slack> h=<threshold> drift=<count>`.

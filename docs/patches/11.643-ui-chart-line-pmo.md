## Summary

New `ChartLinePmo` UI primitive at
`web/src/components/ui/chart-line-pmo.tsx` -- a pure-SVG
two-panel chart with a DecisionPoint Price Momentum Oscillator
overlay using a double-smoothed rate of change.

## Indicator math

The DecisionPoint Price Momentum Oscillator (PMO) double-smooths
a rate of change.

- `computeLinePmoRoc` -- the 1-period rate of change as a
  percentage:

```
roc = 100 * (price[i] / price[i-1] - 1)
```

  The first bar has no prior price, so it is null; a zero prior
  price (division by zero) is also null.

- `computeLinePmoCustomEma` -- the DecisionPoint custom
  exponential moving average with smoothing constant
  `2 / period` (`computeLinePmoSmoothingConstant`). Leading nulls
  pass through; the average seeds at the first defined value and
  runs `ema[i] = k*input[i] + (1-k)*ema[i-1]`.

- `computeLinePmo` -- the full pipeline:

```
roc         = 100 * (price / prevPrice - 1)
smoothedRoc = customEMA(roc, smooth1Period) * 10
pmo         = customEMA(smoothedRoc, smooth2Period)
signal      = customEMA(pmo, signalPeriod)
```

  The ROC carries the raw momentum; two custom EMAs strip its
  noise; the `* 10` rescales it; a signal line smooths the PMO
  once more. The PMO oscillates around zero. Defaults are the
  classic 35 / 20 / 10.

### Worked fixture

`smooth1Period 4`, `smooth2Period 4`, `signalPeriod 4`, price
series `[64, 32, 48, 72, 108, 162]`. The ratios are 0.5 and 1.5
(ROC -50 / +50) and every custom EMA constant is `2/4 = 0.5`, so
each step lands on an exact number:

```
prices       [64,  32,  48,   72,  108,   162]
roc          [ .,  -50,  50,   50,   50,    50]
ema1         [ .,  -50,   0,   25, 37.5, 43.75]
smoothedRoc  [ ., -500,   0,  250,  375, 437.5]
pmo          [ ., -500,-250,    0,187.5, 312.5]
signal       [ ., -500,-375,-187.5,   0,156.25]
```

The PMO runs negative on 2 bars, positive on 2 and crosses zero
once.

## API

- `getLinePmoFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLinePmoPeriod(period, fallback)` -- coerce to an
  integer of at least 2 (the custom EMA constant `2/period` stays
  in `(0, 1]`).
- `computeLinePmoSmoothingConstant(period, fallback?)` -- the
  `2 / period` smoothing constant.
- `computeLinePmoRoc(values)` -- the 1-period percentage rate of
  change, null at the first bar and for a zero prior price.
- `computeLinePmoCustomEma(values, period)` -- the custom EMA
  over a nullable input.
- `computeLinePmo(values, smooth1Period, smooth2Period,
  signalPeriod)` -- the `{ roc, smoothedRoc, pmo, signal }`
  series.
- `runLinePmo(points, options?)` -- sort finite points by `x`,
  run the pipeline, return `{ series, smooth1Period,
  smooth2Period, signalPeriod, roc, smoothedRoc, pmo, signal,
  samples, pmoFinal, signalFinal, pmoMin, pmoMax, positiveCount,
  negativeCount, ok }`. Each sample is classified `positive` /
  `negative` / `zero` by the sign of the PMO. `ok` is false for
  fewer than two points.
- `computeLinePmoLayout(options)` -- stack the price panel above
  a zero-centred PMO panel; project the price, PMO and signal
  paths plus PMO markers and the zero line.
- `describeLinePmoChart(data, options?)` -- ARIA description.

## Component

`ChartLinePmo` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, two stacked panels
(price on top, PMO below) with their own axis ticks and grid,
panel labels, a config badge showing the smoothing and signal
periods, a three-series legend (Price / PMO / Signal) with
toggle, a hover/focus tooltip exposing the value, ROC, PMO,
signal and sign, and a `motion-safe:animate-fade-in` entrance. It
consumes `{ x, value }` points. The PMO panel carries a dashed
zero line, the PMO line with sign-coloured markers, and the
signal line as a dashed overlay. Every node carries a
`data-section` attribute.

## Distinctness

- **vs chart-line-stddev / chart-line-envelope** -- those draw a
  band around the price on the price panel. The PMO is a
  momentum oscillator on its own zero-centred sub-panel, not a
  band.
- **vs chart-line-roofing** -- the Roofing filter is a band-pass
  in the frequency sense. The PMO is a double-smoothed rate of
  change: a momentum measure, not a frequency filter.
- **vs chart-line-bandwidth** -- bandwidth is a single non-
  negative scalar anchored at zero. The PMO swings positive and
  negative around zero and carries a second (signal) line.

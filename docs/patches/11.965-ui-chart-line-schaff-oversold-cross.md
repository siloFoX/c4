## Summary

Adds the `ChartLineSchaffOversoldCross` primitive
(`web/src/components/ui/chart-line-schaff-oversold-cross.tsx`): a pure
SVG dual-panel chart that visualizes Schaff Trend Cycle (STC) crossings
through a configurable oversold threshold (default 25). The top panel
plots the underlying price series; the bottom oscillator panel plots
the STC line (fixed 0..100 range) with a horizontal threshold guide at
25, shaded oversold band, cross markers, and overlay arrows on the
price panel that mark the bar where the cycle exited (bullish, crossed
up through 25) or re-entered (bearish, crossed down through 25) the
oversold zone.

This is the mirror complement of `chart-line-schaff-overbought-cross`
(11.964): identical pipeline, opposite threshold semantics. For the
canonical STC pipeline, see also `chart-line-stc`,
`chart-line-schaff-zero-cross`. Together with mid-cross and
overbought/oversold variants for RSI / Stoch / Stoch-RSI / Williams %R
/ CCI / MFI / UO / TSI / Schaff, this completes the threshold-crossing
family for the Schaff trio.

## Pipeline

Schaff Trend Cycle is a double-stochastic of a MACD line, with each
stochastic stage running through a 0.5-weighted recursive smoother
(half-EMA seeded at 50). The pipeline:

1. `macd = ema(close, fast) - ema(close, slow)` (Schaff canonical fast
   = 23, slow = 50)
2. Stage-1 stochastic over the cycle window on `macd`:
   `k1[i] = 100 * (macd[i] - min(macd, cycle)) / (max - min)`,
   then `d1[i] = 0.5 * d1[i-1] + 0.5 * k1[i]` (seed 50).
3. Stage-2 stochastic over the cycle window on `d1`:
   `k2[i] = 100 * (d1[i] - min(d1, cycle)) / (max - min)`,
   then `stc[i] = 0.5 * stc[i-1] + 0.5 * k2[i]` (seed 50).
4. Regime classifier:
   - `stc >= threshold` -> bullish (out of oversold)
   - `stc <  threshold` -> bearish (in oversold zone)
5. Cross detector emits a `bullish` cross at `i` when
   `stc[i-1] <= T && stc[i] > T` and a `bearish` cross when
   `stc[i-1] >= T && stc[i] < T`.

When the input is constant or strictly linear, MACD steady-state is
constant, so both stochastic ranges collapse and the half-EMA stays at
its seed value (50). With the default threshold = 25, this means
`stc = 50 >= 25` and the regime is reported as bullish over the whole
warmup-cleared range -- the exact mirror of the overbought variant
(where 50 < 75 -> bearish).

## Defaults

- `cycle = 10`, `fast = 23`, `slow = 50`, `threshold = 25`
- Warmup boundary: `slow + 2 * (cycle - 1) - 1` -- with defaults that
  is bar 67; for `SMALL = {cycle: 3, fast: 5, slow: 10}` it is bar 13.

## Bit-exact anchors

The tests pin the following determinism contracts:

- CONST `close = K` for all `K in {0, 1, 42, 100, 1234}` ->
  `stc[i] === 50` (literal) for every `i >= 13` under SMALL.
- LINEAR UP and LINEAR DOWN (slope == 1 or -1) -> `stc[i]` is within
  `1e-10` of 50 over the same range (MACD steady-state).
- CONST K full-run counters under SMALL:
  `noneCount = 13`, `bullishCount = 27`, `bearishCount = 0`,
  `crosses.length = 0`.
- `pricePath` and `stcPath` are byte-identical across repeated layout
  calls, with `.toFixed(2)` precision and `M ... L ...` command shapes.
- Cross detector boundary: strict inequality (`> T`, `< T`), so
  equality with the threshold does not fire.

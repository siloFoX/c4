## Summary

Adds the `ChartLineAwesomeOverboughtCross` primitive
(`web/src/components/ui/chart-line-awesome-overbought-cross.tsx`):
a pure SVG dual-panel chart that visualizes Bill Williams' Awesome
Oscillator (AO -- 5-over-34 SMA difference) crossings through a
configurable positive upper band for momentum surge entry trigger
events. Companion to `chart-line-awesome-zero-cross` (TODO
11.940): shares the same close-only AO pipeline (fast SMA - slow
SMA with the SMA min === max short-circuit), only the default
threshold flips from 0 to +0.5 -- a configurable positive band
analysts tune to match the volatility scale of the underlying
close series.

## Pipeline

Same close-only AO pipeline as the zero-cross variant:

1. `fastSma = SMA(close, fastLength)` (with CONST short-circuit
   via `winMin === winMax`)
2. `slowSma = SMA(close, slowLength)` (same short-circuit)
3. `ao[i] = fastSma[i] - slowSma[i]`
4. Regime classifier (strict `>=`):
   - `ao >= 0.5` -> bullish (in overbought zone)
   - `ao <  0.5` -> bearish (below overbought)
5. Cross detector emits a `bullish` cross when
   `ao[i-1] <= 0.5 && ao[i] > 0.5` (momentum surge entry) and a
   `bearish` cross when `ao[i-1] >= 0.5 && ao[i] < 0.5` (surge
   exit). Strict inequality on the current side ensures the
   boundary value 0.5 does not double-fire.

## Defaults

- `fastLength = 5` (canonical AO fast window)
- `slowLength = 34` (canonical AO slow window)
- `threshold = 0.5` (configurable positive upper band -- the
  canonical "alligator awake" level)
- Warmup boundary: bar `max(fast, slow) - 1 = 33`, so for a
  50-bar run the first 33 samples are null and the remaining 17
  are populated.

## Bit-exact anchors

- CONST `close = K` for all `K in {0, 1, 42, 100, 1234}` ->
  `ao[i] === 0` literal (NOT `-0`) for every `i >= 33` via the
  SMA short-circuit. Regime classifies as bearish because 0 <
  0.5 -- opposite of the zero-cross variant where 0 sat on the
  threshold 0 and read bullish.
- LINEAR UP `close = i` -> ao = 14.5 constant after warmup
  (steady-state SMA lag: fast=(5-1)/2=2, slow=(34-1)/2=16.5,
  ao = -2 + 16.5 = 14.5). 14.5 >> 0.5 -> all bullish, 0
  crosses (ao jumps from null directly to 14.5).
- LINEAR DOWN `close = -i` -> ao = -14.5 constant, all bearish,
  0 crosses.
- CONST K full-run counters under defaults, 50 bars:
  `noneCount = 33`, `bullishCount = 0`, `bearishCount = 17`,
  `crosses.length = 0`.
- `pricePath` and `aoPath` are byte-identical across repeated
  layout calls, with `.toFixed(2)` precision and `M ... L ...`
  command shapes. The ao path uses a single `M` for post-warmup
  data.
- Cross detector boundary: strict inequality (`> T`, `< T`).
- Layout auto-fit: when ao collapses to 0 (CONST case), the
  panel expands UP to include threshold = 0.5 with 10% padding
  both sides -> oscMin = -0.05, oscMax = 0.55.

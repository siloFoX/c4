# 11.180 -- UI: FileTree primitive (v1.11.198)

## Summary

New `<FileTree>` primitive in `web/src/components/ui/file-tree.tsx`. Collapsible
folder/file tree implementing the WAI-ARIA tree pattern.

## API

```ts
export interface FileTreeNode {
  id: string;
  name: string;
  type: 'file' | 'folder';
  children?: FileTreeNode[];
}

export interface FileTreeProps {
  nodes: FileTreeNode[];
  defaultExpanded?: string[];
  selectedId?: string;
  onSelect?: (id: string, node: FileTreeNode) => void;
  className?: string;
  ariaLabel?: string;
}
```

## Behavior

- Folders render `ChevronRight` (rotated on open) + `Folder` / `FolderOpen` icon.
  Files render a placeholder spacer + `File` icon.
- Clicking a folder toggles its expanded state. Clicking a file calls
  `onSelect(id, node)`. Folder clicks never fire `onSelect`.
- Keyboard navigation (focus inside the tree):
  - `ArrowDown` / `ArrowUp`: move through the flattened visible list.
  - `ArrowRight` on a collapsed folder expands it; on an expanded folder
    moves focus to its first child; no-op on files.
  - `ArrowLeft` on an expanded folder collapses it; otherwise moves focus
    to the parent folder (no-op at the root).
  - `Enter` calls `onSelect(id, node)` on the focused node.
  - `Home` / `End` jump to the first / last visible node.
- Roving tabindex: a single item is `tabIndex=0`; the rest are `-1`. Outer
  `ul` carries `role='tree'`; each `li` carries `role='treeitem'` with
  `aria-level`, `aria-expanded` (folders only), `aria-selected`.
- Internal expanded state is uncontrolled, seeded from `defaultExpanded`.

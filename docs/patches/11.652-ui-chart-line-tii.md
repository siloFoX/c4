share of deviations above a moving average.

## Indicator math

The Trend Intensity Index (M.H. Pee) measures the strength of a
trend from how the close deviates from a moving average.

- `computeLineTiiDeviations(closes, period)` -- the per-bar
  deviation of the close from its `period`-bar moving average,
  `deviation[i] = close[i] - SMA(close, period)[i]`, null
  through the moving-average warm-up.

- `computeLineTii(closes, period)` -- the Trend Intensity Index.
  Over the trailing `period` window the deviations are split:
  the positive deviations are summed into `SDPos` and the
  negative deviations are summed in magnitude into `SDNeg`:

```
TII[i] = 100 * SDPos / (SDPos + SDNeg)
```

  the percentage share of the positive deviation sum. The TII
  runs from 0 to 100: a reading above 50 marks an uptrend
  (positive deviations dominate), below 50 a downtrend. A window
  with no deviation at all (a flat market) is null, as are bars
  before the window is full -- the warm-up is `2 * period - 2`
  bars, since the deviation itself warms up before the sum
  window can fill.

### Worked fixture

The TII is a ratio of summed deviations -- no transcendentals --
so the whole pipeline is exact. The fixture (`period 2`) is
hand-tuned so the deviations are
`[., 2, 6, -2, -6, 4, -1, 1, 3, -5]` and every TII window total
divides cleanly:

```
tii  [., ., 100, 75, 0, 40, 80, 50, 100, 37.5]
```

For example `TII = 100 * 6 / (6 + 2) = 75`. A series whose close
runs above its moving average on every bar reads 100; one below
on every bar reads 0; a flat market reads null.

## API

- `getLineTiiFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineTiiPeriod(period, fallback)` -- coerce to an
  integer of at least 2.
- `computeLineTiiDeviations(closes, period)` -- the per-bar
  deviation from the moving average.
- `computeLineTii(closes, period)` -- the Trend Intensity Index
  series, null through the warm-up and for flat markets.
- `runLineTii(points, options?)` -- sort finite points by `x`,
  run the pipeline, return `{ series, period, deviations, tii,
  samples, tiiFinal, tiiMin, tiiMax, upCount, downCount, ok }`.
  Each sample is classified `up` / `down` / `neutral`. `ok` is
  false for fewer than two points.
- `computeLineTiiLayout(options)` -- stack the price panel above
  a TII panel with a fixed `[0, 100]` y-domain; project the
  price and TII paths plus markers and the 50 reference line.
- `describeLineTiiChart(data, options?)` -- ARIA description.

## Component

`ChartLineTii` is a `forwardRef` region (`role="region"`) with
an ARIA-linked description, a `role="img"` SVG, two stacked
panels (price on top, TII below) with their own axis ticks and
grid, panel labels, a config badge showing the period, a
two-series legend (Price / TII) with toggle, a hover/focus
tooltip exposing the value, deviation, TII and trend, and a
`motion-safe:animate-fade-in` entrance. It consumes
`{ x, value }` points. The TII panel has a fixed `[0, 100]`
y-domain with a dashed 50 reference line and trend-coloured
markers (green up, red down, grey neutral). Every node carries a
`data-section` attribute.

## Distinctness

- **vs the zero-centred deviation oscillators (cog / pgo)** --
  those report the signed deviation from a reference. The TII
  reports the *share* of the deviation magnitude that is
  positive -- a 0 to 100 gauge, not a signed oscillator.
- **vs chart-line-hurst / chart-line-entropy** -- those are
  fixed-domain gauges too, but the Hurst exponent measures
  long-term memory and the entropy measures disorder. The TII
  measures directional bias: how much of the recent deviation
  sits above the moving average.
- **vs chart-line-qstick** -- the Qstick averages the
  close-minus-open spread; the TII averages the share of
  close-minus-MA deviations that are positive.

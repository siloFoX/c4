# 11.1033 -- UI: chart-line-donchian-mid-cross-sig primitive (v1.11.1051)

## Scope

Add a pure-SVG dual-panel `ChartLineDonchianMidCrossSig`
component that renders the close in the top panel and the
Donchian Channel middle line (`(max(high, period) +
min(low, period)) / 2`) plus its smoothed SMA signal line
in the bottom panel, marking bullish (middle crosses up
through signal -- centerline range trend trigger up) /
bearish (middle crosses down through signal -- centerline
range trend trigger down) middle-over-signal crossover
trigger events with bias coloring derived from the middle
slope at the trigger bar.

This is the HLC-range sibling of
`chart-line-bollinger-mid-cross-sig` (v1.11.1049, SMA
centerline) and `chart-line-keltner-mid-cross-sig`
(v1.11.1050, EMA centerline). All three share the same
`period = 20` lookback and the same `(period - 1) / 2 =
9.5` centroid-lag identity on linear input, but the
Donchian middle tracks the *range* midpoint
(`(max(high) + min(low)) / 2`) rather than a moving average
of close. That makes it sensitive to extreme highs / lows
in the lookback window, which other moving-average
centerlines smooth away.

## Donchian middle-over-signal pipeline

```
upper[i]  = max(high[i-period+1..i])   ; i >= period - 1
lower[i]  = min(low[i-period+1..i])    ; i >= period - 1
middle[i] = (upper[i] + lower[i]) / 2
signal[i] = SMA(middle, signalLength)  ; i >= period +
                                         signalLength - 2
regime    = middle >= signal -> bullish
            middle <  signal -> bearish
            null            -> none
bias      = middle[i] vs middle[i-1] -> up / down / flat / none
bullish   : prev middle <= prev signal AND
            cur middle > cur signal
            (centerline range trend trigger up)
bearish   : prev middle >= prev signal AND
            cur middle < cur signal
            (centerline range trend trigger down)
```

Default tuning `period = 20`, `signalLength = 3` matches
the canonical Donchian Channel lookback. The middle's lag
on LINEAR HLC input solves to `(period - 1) / 2 = 9.5` --
identical to SMA / EMA centerlines, despite using the
range midpoint rather than a moving average. The signal
SMA adds `(signalLength - 1) / 2 = 1` bar of lag, giving
the steady-state `middle - signal` separation of exactly
`1`, matching the sibling cross-sig family identity.

Warmup is `period + signalLength - 2 = 21` for the default
tuning, identical to the Bollinger / Keltner siblings.

## Bit-exact anchors

All anchors use HLC input.

- **CONST band** `high = K + 1`, `low = K - 1`, `close
  = K`: upper = K + 1, lower = K - 1 -> middle = K. signal
  = K. middle === signal -> regime `bullish` (>=). 0
  crosses. Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close = i`:
  the rolling window of length 20 spans indices `[i - 19,
  i]`. The max of `high[k] = k + 1` over the window is
  `high[i] = i + 1` (the latest bar). The min of `low[k]
  = k - 1` over the window is `low[i - 19] = i - 20` (the
  oldest bar). So middle = `((i + 1) + (i - 20)) / 2 =
  (2i - 19) / 2 = i - 9.5`. signal = SMA(middle, 3) adds
  `(signalLength - 1) / 2 = 1` bar of lag, giving signal
  = `i - 10.5`. middle - signal = `+1` -> regime
  `bullish`. 0 crosses.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`, `close
  = -i`: max(high) over `[i - 19, i]` is `high[i - 19] =
  -(i - 19) + 1 = -i + 20`. min(low) over `[i - 19, i]`
  is `low[i] = -i - 1`. middle = `((-i + 20) + (-i - 1))
  / 2 = (-2i + 19) / 2 = -i + 9.5`. signal = `-i + 10.5`.
  middle - signal = `-1` -> regime `bearish`. 0 crosses.

The clean `+1 / -1` separation under linear input is the
same identity as the SMA / EMA siblings, despite the
fundamentally different computation (range midpoint vs
moving average). This is a consequence of the centroid
arithmetic: for any monotone input, the highest-high and
lowest-low over a window of length `period` are
symmetric around the window center, so their midpoint
equals the SMA exactly.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- Strict-inequality bullish vs `>=` regime classifier
  means the bullish boundary (middle === signal) settles
  into the bullish band, matching every other
  chart-line-*-cross-sig sibling.
- Rolling-max / rolling-min helpers use linear scans
  inside the window (not deque-based O(1) amortised) --
  acceptable for typical chart sizes (period = 20, total
  bars in the hundreds). Can be swapped for a deque
  implementation if pushed to multi-thousand-bar charts.
- Donchian middle = `(max(high) + min(low)) / 2` is the
  midpoint of the range, not a moving average. The
  primitive surfaces just the centerline trigger pattern
  -- the full upper / lower channel overlay is exposed
  on the run object (`upperValues`, `lowerValues`) but
  not drawn by default; a composing layout primitive
  can render them on top.
- Uses a distinct middle ink colour (`#0891b2`, cyan-700)
  so it does not collide with Bollinger's purple
  (`#7c3aed`) or Keltner's teal (`#0d9488`) when all
  three primitives ship in the same dashboard.

# 11.1027 -- UI: chart-line-vortex-neg-cross primitive (v1.11.1045)

## Scope

Add a pure-SVG dual-panel `ChartLineVortexNegCross`
component that renders the close in the top panel and the
Vortex Indicator negative line (VI-) plus its smoothed SMA
signal line in the bottom panel, marking VI- crosses up
through signal -- downtrend confirmation (downward pressure
mounting) / VI- crosses down through signal -- downtrend
confirmation lost (downward pressure releasing) VI-
over-signal crossover trigger events with bias coloring
derived from the VI- slope at the trigger bar.

This is the mirror of `chart-line-vortex-pos-cross`
(v1.11.1044) -- same HLC pipeline but with the negative
vortex movement numerator.

## Vortex--over-signal pipeline

```
VM+[i]    = |high[i] - low[i-1]|          ; i >= 1
VM-[i]    = |low[i]  - high[i-1]|
TR[i]     = max(high[i] - low[i],
                |high[i] - close[i-1]|,
                |low[i]  - close[i-1]|)
VI-[i]    = sum(VM-[i-period+1..i])       ; i >= period
            / sum(TR[i-period+1..i])
signal[i] = SMA(VI-, signalLength)        ; i >= period + signalLength - 1
regime    = VI- >= signal -> bullish
            VI- <  signal -> bearish
            null          -> none
bias      = VI-[i] vs VI-[i-1] -> up / down / flat / none
bullish   : prev VI- <= prev signal AND cur VI- > cur signal
            (downtrend confirmation -- VI- rising through
            signal means downward pressure is mounting)
bearish   : prev VI- >= prev signal AND cur VI- < cur signal
            (downtrend confirmation lost -- VI- falling
            through signal means downward pressure is
            releasing)
```

Cross kind labels (`bullish`/`bearish`) refer to the line
crossing direction (VI- up through / down through signal);
the semantic meaning is downtrend mounting / releasing.
This convention matches every other `chart-line-*-cross-sig`
primitive (the cross kind classifies the line, not the
underlying market mood).

The dual-line VI+ / VI- view lives in
`chart-line-vortex-divergence-cross` and the single VI+ trigger
lives in `chart-line-vortex-pos-cross` (v1.11.1044).

Warmup is `period + signalLength - 1 = 16`.

## Bit-exact anchors

All anchors use HLC input. The VI- values are mirrors of the
VI+ anchors from the 11.1026 sibling.

- **CONST band** `high = K + 1`, `low = K - 1`, `close =
  K`: VM- = 2, VM+ = 2, TR = 2 from i >= 1, so VI- = 1 from
  `i = period`, signal = 1 from `i = period + signalLength
  - 1`. VI- === signal -> regime `bullish` (>=). 0 crosses.
  Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close = i`:
  VM- = |(i-1) - i| = 1, VM+ = 3, TR = 2. VI- = 14/28 =
  0.5, signal = 0.5. VI- === signal -> regime `bullish`,
  0 crosses (downward pressure is low and steady).
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`, `close
  = -i`: VM- = |(-i-1) - (-(i-1)+1)| = 3, VM+ = 1, TR = 2.
  VI- = 42/28 = 1.5, signal = 1.5. VI- === signal -> regime
  `bullish` (>=), 0 crosses (downward pressure is high and
  steady).

The clean half-integer separation (CONST 1, LINEAR UP 0.5,
LINEAR DOWN 1.5) under linear HLC inputs is a direct
consequence of the rolling-sum identity: VM- and TR each
settle to per-bar constants, so VI- is a constant from `i =
period` and the SMA signal locks to that same constant from
`i = period + signalLength - 1`.

Note the regime stays bullish (>=) across all three anchors
because in steady-state VI- === signal -- the bullish
boundary settles into the bullish band by convention,
matching the entire `chart-line-*-cross-sig` family.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no chart
  lib dependency.
- Strict-inequality bullish vs `>=` regime classifier means
  the boundary (VI- === signal) settles into the bullish
  band, matching every other `chart-line-*-cross-sig`
  sibling.
- High < low input rows are filtered at the boundary so the
  TR / VM math never receives a negative range.
- Zero-TR rolling window is short-circuited to null so a
  degenerate window cannot produce Infinity.
- Uses a distinct VI- ink colour (`#db2777`) so it does not
  collide with the VI+ ink colour (`#0ea5e9`) when both
  primitives ship in the same dashboard.

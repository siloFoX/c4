## Goal

Dispatch:

- Build `web/src/components/ui/command-palette.tsx`
  -- Cmd+K / Ctrl+K modal with fuzzy search
  across routes, recent actions, settings.
- Integrate with shortcut registry.

The `CommandPalette` component shipped in 11.277
(v1.11.295). It already supports:

- Cmd+K / Ctrl+K open via
  `useCommandPaletteShortcut`.
- Fuzzy search (`fuzzyScore`) against label +
  keywords + group.
- Persisted recents
  (`c4:command-palette:recents`).
- Group rendering + arrow / Home / End / Enter
  keyboard navigation.

What it lacked was integration with the canonical
keyboard-shortcut registry (`SHORTCUT_ROWS` in
`KeyboardShortcutsModal.tsx`). This patch ships
the integration helper, the unit harness, and the
App.tsx adoption.

## Module API

`web/src/lib/palette-commands.ts` exposes four
pure builders:

### `buildRouteCommands(routes, group?)`

Maps a thin `RouteCommandSpec[]` to `Command[]`.
Default group `"Navigate"`. Forwards optional
`shortcut` + `keywords`; omits absent props so
the `Command` shape stays clean.

### `buildShortcutCommands({ onSelect, category?, group?, rows? })`

Consumes `SHORTCUT_ROWS` (or a custom rows
override) and emits one `Command` per row.

| field | source |
| --- | --- |
| `id` | `shortcut.<category>.<keys>` |
| `label` | `t(row.descriptionKey)` (locale-aware) |
| `group` | option, default `"Shortcuts"` |
| `shortcut` | `formatKeymapForCurrentPlatform(row.keys)` (Cmd vs Ctrl) |
| `keywords` | `[raw keys, category, description, display]` -- fuzzy search hits every angle |
| `action` | `() => onSelect(row)` |

The default action calls back to the host with
the matched row. Hosts wire
`openShortcutsModal` so the binding lands in
context.

Why not fire the shortcut directly? Synthetic
`Ctrl+F` dispatch is not safe (the chord can hit
disparate handlers that gate on `event.isTrusted`
or `event.metaKey` semantics). The lookup-then-modal
flow keeps the operator in control.

### `buildSettingsCommands(entries, group?)`

Parallel helper. Default group `"Settings"`.
Forwards keywords.

### `mergePaletteCommands(...lists)`

Concatenates `Command[]` lists, drops duplicate
`id`s (first occurrence wins), preserves
ordering across lists.

## App.tsx adoption

```tsx
const shortcutCommands = useMemo<Command[]>(
  () => buildShortcutCommands({
    onSelect: () => openShortcutsModal(),
  }),
  [],
);
const mergedCommands = useMemo<Command[]>(
  () => mergePaletteCommands(paletteCommands, shortcutCommands),
  [paletteCommands, shortcutCommands],
);

<CommandPalette ... commands={mergedCommands} />
```

The existing navigation + app + settings commands
keep their group order; the shortcut rows render
under a "Shortcuts" group below them. Selecting a
shortcut closes the palette and opens the
keyboard shortcuts modal via the
`openShortcutsModal` global from `HelpUIRoot`.

## Tests

`web/src/lib/palette-commands.test.ts` -- 15
cases:

- `buildRouteCommands`: default + custom group,
  shortcut + keywords forwarding, omits absent
  props, action identity preserved.
- `buildShortcutCommands`: emits one per
  `SHORTCUT_ROWS` row by default; every command
  has a shortcut chip + non-empty keywords;
  category filter; rows override; onSelect
  receives the matched row; custom group label.
- `buildSettingsCommands`: default group label;
  keywords forwarding.
- `mergePaletteCommands`: concat + dedup (first
  wins); empty arg returns `[]`; ordering
  preserved across lists.

15/15 pass under vitest 4.1.5. TypeScript clean
for `palette-commands.ts` and `App.tsx`.

## Out of scope

- Direct keystroke dispatch from a shortcut
  command (`Cmd+F` as a synthetic event). Treats
  the shortcuts dimension as a learn / lookup
  surface today.
- Per-page scoped palettes. The helpers support
  arbitrary `Command[]` composition; a future
  patch can wire palettes inside Settings /
  WorkflowEditor / SpecialistsView.
- Localisation of the shortcut-chip text itself
  (we render the formatted binding, not the
  translation of "Ctrl"). The shortcut
  descriptions flow through i18n keys today;
  modifier labels follow the platform-aware
  rewrite.

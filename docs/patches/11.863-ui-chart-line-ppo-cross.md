## Summary

Added `<ChartLinePpoCross />` -- a pure-SVG, dual-panel React/TS
primitive that renders the close on top and the Percentage Price
Oscillator plus its signal-line EMA on the bottom. PPO normalises
the MACD difference by the slow EMA so the resulting oscillator
is independent of price magnitude and comparable across
instruments. Markers fire at every PPO-vs-signal crossover --
the canonical momentum regime shift.

Cross events:

- `up` cross   -> regime `accelerating-up`
- `down` cross -> regime `accelerating-down`

Defaults: `fastLength = 12`, `slowLength = 26`, `signalLength = 9`
(classic MACD/PPO parameters), ppoColor `#0ea5e9` (sky),
signalColor `#f97316` (orange), bullish marker `#16a34a`,
bearish marker `#dc2626`, neutral marker `#475569`, zero line
`#475569`.

## Formula

```
ema1[i]   = EMA(close, fastLength)
ema2[i]   = EMA(close, slowLength)
PPO[i]    = ema2[i] === 0 ? null
                          : (ema1[i] - ema2[i]) / ema2[i] * 100
signal[i] = EMA(PPO, signalLength)
```

Cross detection uses the standard prev-relation + strict-
inequality guard. The tooltip surfaces the zero-line bias
(`bullish` if PPO > 0, `bearish` if PPO < 0, `neutral` if = 0)
alongside the cross direction.

## Bit-exact anchor

- **CONST close = K (K > 0)**: every EMA collapses to K via the
  SMA-seeded `min === max` precision fix. PPO = (K - K) / K *
  100 = 0, signal = EMA(0) = 0. Relation `equal` forever,
  zero crosses. Verified across multiple K and `(fastLength,
  slowLength, signalLength)` tuples.
- **CONST close = K = 0**: divide-by-zero guard leaves PPO and
  signal `null` forever.

## Soft anchor

For LINEAR UP / DOWN inputs PPO and signal share the same sign
once defined (positive for LINEAR UP, negative for LINEAR DOWN)
with the signal trailing PPO in the same direction. They never
cross back the other way -- zero crosses across multiple length
tuples.

# Patch 11.846 -- ChartLineTemaPct primitive (v1.11.864)

## Summary

Add `<ChartLineTemaPct />`, a pure-SVG dual-panel line chart with a
Triple-EMA Percent-Change oscillator beneath a price line and the
underlying TEMA overlay on the top panel. Fourth and final "moving
average percent change" variant in this batch (after 11.836 ema-pct,
11.837 sma-pct, 11.845 wma-pct):

```
ema1[i]    = EMA(close, length)[i]
ema2[i]    = EMA(ema1,  length)[i]
ema3[i]    = EMA(ema2,  length)[i]
tema[i]    = 3 * ema1[i] - 3 * ema2[i] + ema3[i]
temaPct[i] = tema[i] === 0 ? null
           : (close[i] - tema[i]) / tema[i] * 100
```

TEMA cancels the EMA lag at linear-trend order, giving a closer
track to current price than a single EMA at the same length.
`temaPct[i]` is `null` during the triple-EMA warmup
(`i < 3 * (length - 1)`) and whenever `tema[i]` is zero.

## Defaults

| Knob              | Value     |
|-------------------|-----------|
| length            | 14        |
| bullishThreshold  | 0         |
| bearishThreshold  | 0         |
| temaColor         | `#0d9488` (dark teal) |
| pctColor          | `#db2777` (pink)      |

Dark teal + pink distinguishes the TEMA panel from the prior pct
primitives (emerald + purple for SMA, orange + violet for EMA,
lime + magenta-purple for WMA).

## EMA helper precision

The EMA helper carries the same `min === max` seed precision fix
used in the prior MACD / TRIX / ema-pct primitives. For constant
inputs the seed installs the exact value and the CONST short-circuit
preserves it through all three smoothing passes, so the TEMA
combination `3*K - 3*K + K = K` is bit-exact.

## Bit-exact anchors

- `CONST close = K, K != 0`: every EMA pass collapses to `K`, so
  `tema = 3K - 3K + K = K`, `delta = K - K = 0`, `temaPct = 0`
  bit-exact post-warmup. Verified across `(K, length)` sweeps.
- `CONST close = 0`: every EMA pass is `0`, so `tema = 0` ->
  divide-by-zero guard returns `null`.

## Surface

Standard primitive surface mirroring 11.836 / 11.837 / 11.845:

- `ChartLineTemaPctPoint = { x, close }`
- `runLineTemaPct(data, options?) -> Run` with
  `ok = series.length >= 3 * (length - 1) + 1`.
- `computeLineTemaPctLayout({ data, width?, height?, padding?, panelGap?, length?, bullishThreshold?, bearishThreshold? }) -> Layout`.
  Top panel renders close + TEMA overlay; bottom panel renders the
  temaPct oscillator.
- `describeLineTemaPctChart(data, opts?) -> string` for the ARIA
  description.
- `<ChartLineTemaPct />` `forwardRef<HTMLDivElement>` with three
  legend toggles (`price`, `tema`, `pct`), config badge, optional
  threshold band + zero-line overlay, markers on cross events, and
  a tooltip surfacing
  `close / ema1 / ema2 / ema3 / tema / temaPct / zone / cross`.

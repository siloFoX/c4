## Summary

cascade.

## Indicator math

John Ehlers' Laguerre filter is built from a four-stage cascade
of L stages. Seeded with the first price, each bar advances:

```
L0 = (1 - gamma) * price + gamma * L0_prev
L1 = -gamma * L0 + L0_prev + gamma * L1_prev
L2 = -gamma * L1 + L1_prev + gamma * L2_prev
L3 = -gamma * L2 + L2_prev + gamma * L3_prev
Filter = (L0 + 2*L1 + 2*L2 + L3) / 6
```

Each stage damps the one before it; chaining four gives the
filter its smooth, low-lag shape from a single parameter. The
`gamma` damping factor lives in `[0, 1)` -- a higher gamma damps
more (smoother, more lag). The filter is defined from index 0
with no warm-up (the seed makes `Filter[0] = price[0]`); a flat
series holds at its constant.

### Worked fixture

`gamma 0.5`, price series `[10, 18, 8, 16, 6]`. With gamma a
half every cascade step is an exact half, so the stages

```
L0 = [10, 14, 11, 13.5,  9.75]
L1 = [10,  8, 12.5, 10.5, 13.875]
L2 = [10, 11, 7.25, 10.875, 9]
L3 = [10, 9.5, 12.125, 7.875, 10.3125]
```

blend exactly into the filter
`[10, 10.25, 10.4375, 10.6875, 10.96875]`. The price oscillates
wildly (10, 18, 8, 16, 6) but the filter rises smoothly -- the
cascade is the noise filter. 2 bars above, 2 below, 1 on.

## API

- `getLineLaguerreFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineLaguerreGamma(gamma, fallback)` -- coerce gamma
  to the valid damping range `[0, 1)`; anything outside it, or a
  non-finite value, falls back.
- `computeLineLaguerreStages(values, gamma)` -- the four cascade
  stage series `{ l0, l1, l2, l3 }`.
- `computeLineLaguerre(values, gamma)` -- the filter line, the
  `(L0 + 2L1 + 2L2 + L3)/6` blend of the stages.
- `runLineLaguerre(points, options?)` -- sort finite points by
  `x`, compute the filter, return `{ series, gamma, filter,
  samples, filterFinal, filterMin, filterMax, aboveCount,
  belowCount, ok }`. Each sample is classified `above` /
  `below` / `on` the filter. `ok` is false for fewer than two
  points.
- `computeLineLaguerreLayout(options)` -- project the price path
  and the filter path on one shared panel.
- `describeLineLaguerreChart(data, options?)` -- ARIA
  description.

## Component

`ChartLineLaguerre` is a `forwardRef` region (`role="region"`)
with an ARIA-linked description, a `role="img"` SVG, axis ticks
and grid, a config badge, a two-series legend (Price / Laguerre)
with toggle, a hover/focus tooltip exposing the value, the filter
reading and the position, and a `motion-safe:animate-fade-in`
entrance. It consumes `{ x, value }` points. Every node carries
a `data-section` attribute.

## Distinctness

- **vs the windowed moving averages (chart-line-smma /
  chart-line-wma)** -- those average a fixed look-back window.
  The Laguerre filter has no window: it chains four damped
  recursive stages governed by one gamma.
- **vs chart-line-mcginley** -- the McGinley Dynamic is a single
  recursive step with an adaptive divisor. The Laguerre filter
  is a fixed four-stage cascade; its smoothness comes from the
  depth of the cascade, not from adapting the weight.
- **vs chart-line-t3** -- the T3 cascades EMAs with a volume
  factor. The Laguerre cascade uses Laguerre-polynomial stages
  and blends them by the fixed `(1, 2, 2, 1)/6` weights.

## Summary

Added `<ChartLineMacdCrossSig />` -- a pure-SVG, dual-panel
React/TS primitive that renders the close in the top panel and
overlays the MACD with its signal line in the bottom panel,
marking only the bullish / bearish cross trigger events. This is
the trigger-focused variant of the MACD family: 11.870 published
the single-panel close-scaled MACD percent oscillator, while
this primitive isolates the actionable crossovers and also paints
them as arrow overlays on the price panel for direct charting-
overlay use:

```
ema_fast[i] = EMA(close, fastLength)
ema_slow[i] = EMA(close, slowLength)
MACD[i]     = ema_fast[i] - ema_slow[i]
signal[i]   = EMA(MACD, signalLength)
bullish    : (MACD - signal) crosses up    (prev <= 0, cur > 0)
bearish    : (MACD - signal) crosses down  (prev >= 0, cur < 0)
```

Defaults: `fastLength = 12`, `slowLength = 26`,
`signalLength = 9` (canonical MACD). Regime classifier:
`bullish` (MACD > signal), `bearish` (MACD < signal),
`neutral` (MACD === signal), `none` (either side null).

## Bit-exact anchor

- **CONST close = K (any K including 0)**: `ema_fast = ema_slow
  = K` via the SMA-seeded `min === max` precision fix in
  `applyLineMacdCrossSigEma`. `MACD = K - K = 0` every bar after
  warmup. Signal EMA of zeros stays at 0. So `MACD === signal`
  everywhere -> regime `neutral`, cross count = 0. Verified for
  K = 0 and K > 0 across multiple fast / slow / signal tuples
  (warmup = `slow - 1 + signal - 1`).

## Difference vs chart-line-macd-cross-pct (11.870)

| primitive | bottom panel | overlays | use-case |
| --- | --- | --- | --- |
| `chart-line-macd-cross-pct` (11.870) | single MACD%% oscillator | none | percent of close magnitude |
| `chart-line-macd-cross-sig` (11.876) | MACD + signal + crosses | bullish/bearish arrows on price | actionable trigger events |

11.870 publishes the close-scaled MACD%% as a single line for
cross-instrument comparison of magnitude. 11.876 adds the EMA
signal line and emits the bullish / bearish crossovers as
discrete trigger events with dual rendering (oscillator dots and
price-panel arrow overlays).

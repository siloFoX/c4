## Behaviour

### Debounced onChange

Typing reflects to the local
input value synchronously so the
field feels responsive; the
external `onChange` callback is
debounced by `debounceMs`
(default 200).

| `debounceMs` | behaviour |
| --- | --- |
| <= 0 / non-finite | immediate |
| positive number | delayed |

Pending debounce timers are
cleared on unmount, on the
clear button click, on Enter
(when onSubmit fires), and on
selecting a recent entry --
those paths emit immediately so
the host always sees the
authoritative latest value.

### Clear button

Renders when `showClearButton !==
false` AND value is non-empty
AND not disabled / readOnly.
Click clears the value, cancels
the pending debounce, fires
`onChange('')` immediately, and
refocuses the input. The clear
button's `aria-label` is
`clearLabel` (default "Clear
search").

### Shortcut hint

When `shortcut` is provided AND
value is empty AND input is not
focused, a `<kbd>` badge appears
on the right edge of the input.
The badge is decorative -- the
host binds the actual hotkey
(documented as out of scope).

### Recent searches dropdown

When `recentSearches` is
non-empty AND the input is
focused AND not disabled /
readOnly:

- Dropdown opens below the
  input via absolute
  positioning.
- Entries filter against the
  current value via the exported
  pure helper
  `filterRecentSearches`
  (case-insensitive substring,
  query trimmed, duplicates
  collapsed, empties dropped,
  capped at `maxRecent`).
- Keyboard nav: ArrowDown +
  ArrowUp cycle (wrap at the
  ends).
- Escape closes the dropdown
  without selecting.
- Enter on a highlighted entry
  selects it: fires both
  `onSelectRecent(entry)` AND
  `onChange(entry)` (immediate,
  bypassing debounce).

### Submit

Enter without a highlighted
recent fires
`onSubmit(query)` AND flushes
the debounce so the host sees
the authoritative latest value.

### Controlled / uncontrolled

- Controlled: pass `value`.
  Caller owns state; defaults
  ignored.
- Uncontrolled: pass
  `defaultValue` for seeding.
  Local state updates on
  typing + clear + recent
  selection.

Both modes fire onChange / on
SubmitRecent / onSubmit so the
host can persist either way.

### Disabled + readOnly

- `disabled` -> input disabled,
  clear button hidden, dropdown
  closed, root carries
  `data-disabled="true"`.
- `readOnly` -> input read-only,
  clear button hidden, dropdown
  closed, root carries
  `data-read-only="true"`.

### autoFocus

Forwarded to the native input
attribute. The hook does not
manage focus; the browser's
default autofocus path applies.

## Pure helper (exported)

```ts
export function filterRecentSearches(
  recents: ReadonlyArray<string>,
  query: string,
  max: number,
): string[];
```

- empty input -> `[]`
- empty query -> all unique
  non-empty entries up to `max`
- non-empty query -> case-
  insensitive substring match
  with the trimmed query
- duplicates collapsed (first
  occurrence kept)
- empty / whitespace-only
  entries dropped
- result capped at `max`

## ARIA + data attributes

Root (wrapper):

- `data-section="search-input"`
- `data-has-value`
  ('true' / 'false')
- `data-focused`
  ('true' / 'false')
- `data-disabled`
  ('true' / 'false')
- `data-read-only`
  ('true' / 'false')

Input:

- `role="searchbox"`
- `aria-label`
- `aria-autocomplete="list"`
  when recents exist
- `aria-expanded` when dropdown
  open
- `data-section="search-input-input"`

Inner nodes:

- `data-section="search-input-icon"`
  (search icon)
- `data-section="search-input-clear"`
  + `aria-label`
- `data-section="search-input-shortcut"`
  (`<kbd>`)
- `data-section="search-input-recent"`
  -- `<ul role="listbox"
  aria-label="Recent searches">`
- `data-section="search-input-recent-item"`
  + `role="option"` +
  `aria-selected` +
  `data-active` +
  `data-recent-index`

## Pairs with existing primitives

- `<Select>` / `<MultiSelect>`
  -- their internal search
  inputs filter option lists;
  SearchInput is the page-
  level primitive.
- `<Kbd>` -- richer keyboard
  combination rendering for
  shortcut overlays.
- ThemeCustomizer (11.394) --
  the input reads token-based
  Tailwind classes so it
  auto-themes.

## API

```ts
interface ImageGalleryItem {
  src: string;
  alt: string;
  thumb?: string;        // alternate thumbnail src
  caption?: ReactNode;
  width?: number;
  height?: number;
}

interface ImageGalleryProps {
  items: ImageGalleryItem[];
  layout?: 'grid' | 'masonry';       // default 'grid'
  columns?: number;                  // default 3
  gap?: number;                      // px, default 8
  defaultIndex?: number | null;
  openIndex?: number | null;
  onOpenChange?: (i: number | null) => void;
  className?: string;
  ariaLabel?: string;                // default 'Image gallery'
  lazy?: boolean;                    // default true
  onSelect?: (i: number) => void;
  swipeThreshold?: number;           // px, default 50
  closeOnBackdropClick?: boolean;    // default true
  containerId?: string;              // portal root id
  wrap?: boolean;                    // default true
}
```

## Behaviour

- Grid layout: CSS grid with
  `grid-template-columns: repeat(N, minmax(0, 1fr))`.
- Masonry layout: CSS multi-column (`column-count: N`) with
  `break-inside: avoid` per item.
- Thumbs render as buttons; clicking opens the lightbox at that
  index. Per-item `thumb` overrides the thumbnail src while the
  lightbox still uses the full `src`.
- Lightbox renders through a portal (`app-portal-root` by
  default; override via `containerId`).
- Body scroll lock while the lightbox is open (toggles
  `document.body.style.overflow` with restore on close).
- Auto-focus the lightbox root on open so keyboard nav works
  without a manual tab.
- Keyboard nav on the lightbox:
  - Arrow Right: next
  - Arrow Left: prev
  - Home: jump to 0
  - End: jump to last
  - Escape: close
- Pointer / touch swipe: pointer-down at clientX1, pointer-up at
  clientX2.
  - `clientX2 - clientX1 <= -threshold` -> next.
  - `clientX2 - clientX1 >= threshold` -> prev.
  - Otherwise no-op.
- Wrap by default: next-from-last -> 0; prev-from-first -> last.
  `wrap={false}` disables wrap and visually disables the boundary
  navigation button (`disabled` attribute + opacity).
- `closeOnBackdropClick` (default true) closes only when the click
  target equals the backdrop (not a child element).

## Pure helpers (exported)

```ts
DEFAULT_GALLERY_COLUMNS = 3
DEFAULT_GALLERY_GAP = 8
DEFAULT_SWIPE_THRESHOLD = 50
clampGalleryIndex(index, total): number
nextGalleryIndex(current, total, wrap?): number
prevGalleryIndex(current, total, wrap?): number
isSwipeLeft(dx, threshold?): boolean
isSwipeRight(dx, threshold?): boolean
```

- `clampGalleryIndex` returns 0 for empty totals, clamps below 0
  and above `total - 1`, NaN -> 0, and floors fractional inputs.
- `nextGalleryIndex` / `prevGalleryIndex` wrap by default; pass
  `wrap=false` to clamp at the boundary.
- `isSwipeLeft` / `isSwipeRight` are inclusive at the threshold
  (a 50px swipe with a 50px threshold counts).

## ARIA + data attributes

- Region: `role="region"` + `aria-label`.
- Thumb button: `aria-label="Open image N: ALT"`,
  `data-section="image-gallery-item"`, `data-index`.
- Lightbox: `role="dialog"` + `aria-modal="true"` +
  `aria-label="<gallery label> lightbox"` + `tabIndex=-1`,
  `data-section="image-gallery-lightbox"`, `data-active-index`.
- Region root: `data-section="image-gallery"`,
  `data-layout`, `data-columns`, `data-lazy`,
  `data-item-count`.
- Lightbox sub-nodes:
  - `data-section="image-gallery-close"` (close button)
  - `data-section="image-gallery-prev"` (prev nav)
  - `data-section="image-gallery-next"` (next nav)
  - `data-section="image-gallery-figure"` (figure wrapper)
  - `data-section="image-gallery-full"` (full-size image)
  - `data-section="image-gallery-caption"` (caption, when supplied)
  - `data-section="image-gallery-counter"` (N / total)

## Pairs with existing primitives

- `<ComparisonSlider>` (11.417) for before / after image pairs.
- `<DiffEditor>` (11.399) for text rather than image diffs.
- `<Dialog>` underlies the modal shell pattern; the lightbox is a
  portal + `role=dialog` without the Dialog primitive's full
  feature set.

## Out of scope

- Pinch / wheel zoom inside the lightbox (host can wrap).
- Drag-to-reorder (gallery is read-only here).
- Server-side rendering of the lightbox (the portal lazy-mounts
  after the first effect per the canonical `<Portal>` contract).
- Virtualisation for very large galleries (host can wrap with
  `<VirtualizedList>` for thousands of thumbs).

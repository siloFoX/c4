## Summary

New `ChartLineErgodic` UI primitive in
`web/src/components/ui/chart-line-ergodic.tsx` -- a pure-SVG two-panel
line chart with an **Ergodic Oscillator** panel from a triple-smoothed
momentum ratio with a signal line. The top panel plots the price; the
bottom panel plots the oscillator and its signal.

## The computation

William Blau's Ergodic Oscillator is a triple-smoothed momentum ratio.
`ChartLineErgodicPoint` carries `{x, value}`.

- `computeLineErgodicMomentum(values)` -- the one-bar price change; the
  first bar has no prior, so it is null.
- `computeLineErgodicEma(values, period)` -- an exponential moving
  average over a series that may carry leading nulls; the nulls pass
  through and the average seeds from the first finite value with
  `alpha = 2 / (period + 1)`.
- `computeLineErgodic(values, options)` -- triple-smooths the momentum
  through three cascaded exponential averages (`num`) and the absolute
  momentum through the same three (`den`), takes `ergodic = 100 *
  num / den`, then smooths the oscillator into the signal line. Because
  every exponential average is a weighted mean with positive weights,
  `|num| <= den`, so the oscillator is bounded to [-100, 100].
- `classifyLineErgodicZone(ergodic, signal)` -- marks a bar **up** when
  the oscillator leads its signal, **down** when it lags, **flat** when
  they match.

## Worked fixture

The triple cascade is transcendental for a general series, but the
indicator has clean period-independent anchors.

A **constant series** has zero momentum, so the numerator and
denominator are both zero and the oscillator reads exactly 0. A **linear
ramp** has a constant momentum `b`: the momentum series fed to the
cascade is constant from its first finite bar, and an exponential
average of a constant series is exactly that constant (the
`alpha*(b - b)` term is exactly zero), so the triple cascade preserves
`b` in both the numerator and the denominator. The oscillator is
therefore `100 * b / |b|` -- exactly +100 for an ascending ramp and
-100 for a descending ramp -- on every bar past the first, for any
choice of smoothing periods. The tests assert these with `toEqual`,
along with the exact dyadic exponential average (period 3 gives
`alpha = 0.5`, so `[4,8,16]` smooths to `[4,6,11]`).

## API

`data` (a `ChartLineErgodicPoint[]`), `firstPeriod` (default 12),
`secondPeriod` (default 6), `thirdPeriod` (default 3), `signalPeriod`
(default 3), `pricePanelRatio` / `gap`, sizing/format/a11y props,
`strokeWidth` / `dotRadius`, the colour props, the toggle flags
(`showAxis` / `showGrid` / `showDots` default false / `showErgodic` /
`showSignal` / `showZeroLine` / `showMarkers` / `showTooltip` /
`showConfigBadge` / `showLegend` / `animate`), controlled +
uncontrolled `hiddenSeries` (keys `price` / `ergodic` / `signal`),
`formatValue` / `formatX`, `onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineErgodicFinitePoints`,
`normalizeLineErgodicPeriod` (coerces a period to an integer of at least
1), `computeLineErgodicMomentum`, `computeLineErgodicEma`,
`computeLineErgodic`, `classifyLineErgodicZone`, `runLineErgodic`,
`computeLineErgodicLayout`, `describeLineErgodicChart`, plus the
`DEFAULT_CHART_LINE_ERGODIC_*` constants, `CHART_LINE_ERGODIC_BOUND` and
the `ChartLineErgodicZone` / `ChartLineErgodicComputed` types. Every
exported name carries a `LineErgodic` infix so the `export *` barrel
line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen
`data-section="chart-line-ergodic-aria-desc"` description span; the
price line, the oscillator line, the signal line and every dot / marker
are `role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-first-period`,
`data-second-period`, `data-third-period`, `data-signal-period`,
`data-ergodic-final`, `data-up-count`, `data-down-count`,
`data-flat-count`, `data-total-points` and `data-animate`.

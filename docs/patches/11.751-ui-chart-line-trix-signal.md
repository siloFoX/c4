# Patch 11.751 -- UI: chart-line-trix-signal primitive

Released as `v1.11.769` on 2026-05-26.

## Scope

Add `<ChartLineTrixSignal>` -- a pure-SVG two-panel chart with a
Jack Hutson **TRIX + signal** panel. The TRIX is the bar-over-bar
rate of change of a triple EMA of `log(close)`, scaled by `10000`
(basis points of the log-rate).

## Definition

Inputs:

- `data: ChartLineTrixSignalPoint[]` where
  `ChartLineTrixSignalPoint = { x; close }`. Bars with non-finite
  or non-positive close are dropped.
- `period: number` -- the EMA length used for each smoothing
  pass. Default `14`. Normalized to an integer `>= 2`.
- `signalPeriod: number` -- the EMA length for the signal line.
  Default `9`. Normalized to an integer `>= 2`.
- `threshold: number` -- the `+/-` strong-zone boundary. Default
  `50`. Normalized to a positive finite.

Algorithm:

```
ema1[i] = EMA(log(close), period)
ema2[i] = EMA(ema1, period)
ema3[i] = EMA(ema2, period)
TRIX[i] = 10000 * (ema3[i] - ema3[i - 1])
Signal[i] = EMA(TRIX, signalPeriod)
```

EMAs seed at the first finite input (`EMA[0] = input[0]`) with
`alpha = 2 / (length + 1)`. The first bar is null on the TRIX
(no prior `ema3`); the signal needs at least one defined TRIX.

## Bit-exact algebraic anchor

**EMA-of-constant lemma**: for constant input `c`, `EMA[0] = c`
and `EMA[i] = EMA[i - 1] + alpha * (c - EMA[i - 1]) = EMA[i - 1]`.
So the EMA stays at `c` exactly at every bar.

**CONST_FLAT** (`close == K > 0`): `log(K)` is a constant `L`.
The cascade is `ema1 = ema2 = ema3 = L` at every bar (by the
EMA-of-constant lemma applied three times). So:

- `TRIX[i] = 10000 * (L - L) = 0` bit-exact at every defined
  bar.
- `Signal[i] = EMA(0) = 0` bit-exact at every defined bar.

**CONST_FLAT_HIGH** (`close == 1000`): same identity. `log(1000)
= 6.907...` is a constant `L`; cascade collapses; `TRIX = 0`,
`Signal = 0` bit-exact.

**Scale invariance**: `log(s * c) = log(s) + log(c)`. The
additive constant `log(s)` propagates unchanged through the EMA
cascade (linearity), so `ema3(scaled) = ema3(original) +
log(s)`. The bar-over-bar difference erases the additive
constant: `TRIX(scaled) = TRIX(original)` to floating-point
precision. Tests use `toBeCloseTo(_, 10)`.

## Component

`forwardRef<HTMLDivElement, ChartLineTrixSignalProps>` with
displayName. Renders:

- A region-role outer `<div data-section="chart-line-trix-signal">`.
- An `<svg role="img">` with an `aria-describedby` description.
- Top panel: close line, optional dots.
- Bottom panel: TRIX line, dashed signal line, zone-coloured
  markers (strong-bull / bull / flat / bear / strong-bear /
  none), zero reference line, dashed `+/- threshold` reference
  lines.
- Legend toggle for `price` / `trix` / `signal`, plus `showTrix`
  / `showSignal` props and an `onPointClick` handler.
- ARS design-system tokens for colour and motion (`motion-safe:
  animate-fade-in`).

# 11.760 chart-line-ehlers-supersmoother (v1.11.778)

## Summary

Pure-SVG single-panel React/TypeScript chart primitive with an
**Ehlers SuperSmoother** overlay applying the two-pole Butterworth
recursive low-pass filter to the close. Added as the
`<ChartLineEhlersSupersmoother />` UI primitive in
`web/src/components/ui/`.

## Algorithm

For a period `p`:

```
arg = sqrt(2) * PI / p
a   = exp(-arg)
c2  = 2 * a * cos(arg)
c3  = -a * a
c1  = 1 - c2 - c3
```

`c1` is computed by subtraction from 1 so that the floating-point
identity `c1 + c2 + c3 = 1` is held as tightly as IEEE 754 allows
(within `1 ULP` of `1` across the periods 2..100). The recurrence is:

```
ss[i] = c1 * (close[i] + close[i - 1]) / 2
        + c2 * ss[i - 1] + c3 * ss[i - 2]
```

Seeds: `ss[0] = close[0]`, `ss[1] = close[1]`. A non-finite close
nulls the bar and resets the recurrence (a fresh seed kicks in on
the next finite close).

## Anchors

The defining stationary anchor:

- **Constant close passes through unchanged**. For `close == K` at
  every bar, the seeds give `ss[0] = ss[1] = K`. The recurrence then
  becomes `ss[i] = c1 * K + c2 * K + c3 * K = K * (c1 + c2 + c3)`,
  which equals `K` to within IEEE 754 rounding. The output is
  numerically equal to `K` at every bar for any period.

- **Bit-exact at `K = 0`**. Multiplication by zero is exact, so
  `ss = 0` at every bar regardless of period.

- **Translation invariance**. Shifting the close by a constant `C`
  shifts the SuperSmoother by `C` (within rounding).

- **Recurrence reproduction**. On a worked fixture
  `[1, 2, 4, 7, 11, 16]` with `period = 10`, the implementation
  reproduces the hand-computed recurrence step by step.

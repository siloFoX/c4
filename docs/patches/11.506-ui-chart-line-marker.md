# TODO 11.506 - chart-line-marker primitive

Status: SHIPPED v1.11.524 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartLineMarker>`, a pure-SVG multi-series line chart with
**configurable per-point markers**. Four built-in shapes ship out of
the box (`'circle'`, `'square'`, `'triangle'`, `'diamond'`) plus
`'none'` to omit the marker entirely. Series declare a default
shape via `series.marker`; individual points override the series
default via `point.marker`. All shapes render as closed `<path>`
elements so they fill, stroke, and animate uniformly.

## API

```tsx
<ChartLineMarker
  series={[
    {
      id: 'cpu',
      label: 'CPU',
      marker: 'circle',
      data: [{x:0,y:12}, {x:1,y:18}, {x:2,y:30, marker:'diamond'}, {x:3,y:24}],
    },
    {
      id: 'mem',
      label: 'Memory',
      marker: 'triangle',
      data: [{x:0,y:60}, {x:1,y:65}, {x:2,y:70}, {x:3,y:68}],
    },
  ]}
/>
```

## Distinct from prior chart-* primitives

- `<ChartLine>` and other variants render markers as a fixed circle.
  This primitive parametrises the marker shape per series and per
  point.
- `<ChartLineAreaStacked>` (11.505) - stacked layers; no marker
  control.
- `<ChartLineAreaGradient>` (11.504) - gradient fill.
- `<ChartLineThreshold>` (11.503) - threshold rules.
- `<ChartLineConfidence>` (11.502) - polygon band.
- `<ChartLineMulti>` (11.500) - synchronised crosshair.
- `<ChartLineZoom>` (11.501) - brush-to-zoom.
- `<ChartLineDashed>` (11.499) - dash patterns on the stroke.
- `<ChartLineSmooth>` (11.498) - curve interpolation.
- `<ChartLineStep>` (11.497) - step interpolation.

## Marker contract

### Resolution order

`resolveLineMarkerShape(point, seriesDefault)`:

1. `point.marker` if defined (per-point override).
2. `seriesDefault` (series-level default).
3. `'circle'` as final fallback.

`'none'` is honoured verbatim at any layer; the marker `<path>` is
suppressed entirely.

### Shape paths

`buildLineMarkerShapePath(shape, cx, cy, size)` returns a closed SVG
path string for the marker centered at `(cx, cy)` with half-extent
`size`:

| shape      | path components                                    |
| ---------- | -------------------------------------------------- |
| `circle`   | `M cx-r cy A r r 0 1 0 cx+r cy A r r 0 1 0 cx-r cy Z` |
| `square`   | 4-vertex axis-aligned square                       |
| `triangle` | equilateral pointing up; apex at `(cx, cy - size)` |
| `diamond`  | 4 cardinal-point rhombus                           |
| `none`     | empty string                                       |

Non-positive `size`, non-finite center, or `'none'` -> empty string.
The four real shapes always produce distinct path strings (asserted
in tests).

## Layout

`computeLineMarkerLayout(input)` returns per-series points with:

- `shape` resolved per point.
- `markerPath` precomputed (empty for `'none'`).
- `defaultShape` recorded at the series level.

Bounds, ticks, line path, and hidden filter follow the standard
line-chart pattern.

## Interactivity

- Hover or focus on a marker dims every other series stroke +
  marker to ~30%; the focused series stays at full opacity.
- Per-marker tooltip shows series label, formatted x, formatted y,
  and a `marker:` row identifying the shape.
- Per-marker `onPointClick({series, point})` callback.
- Legend buttons toggle series visibility; each legend swatch shows
  a horizontal line plus the series's default marker shape so the
  combination is unmistakable.
- Mount fade via `motion-safe:animate-fade-in` (suppress via
  `animate=false`).

`markerFill?: string` (separate from line stroke color).
`ChartLineMarkerPoint` accepts a `marker?` override.

## Exported helpers

- `DEFAULT_CHART_LINE_MARKER_*` numeric / string defaults.
- `DEFAULT_CHART_LINE_MARKER_PALETTE` (10-colour palette).
- `LINE_MARKER_SHAPES` (canonical shape array).
- `getLineMarkerDefaultColor(index)` - palette wrap.
- `getLineMarkerFinitePoints(points)` - drops non-finite.
- `getLineMarkerBounds(series, hidden?)` - per-axis bounds.
- `getLineMarkerTicks(min, max, count?)` - evenly-spaced ticks.
- `buildLineMarkerLinePath(points)` - `M x y L x y ...`.
- `resolveLineMarkerShape(point, seriesDefault)` - canonical
  resolution per the order above.
- `buildLineMarkerShapePath(shape, cx, cy, size)` - canonical SVG
  path builder for the four shapes.
- `computeLineMarkerLayout(input)` - per-series layout with per-point
  shape + markerPath.
- `describeLineMarkerChart(series, hidden?, formatValue?)` - ARIA
  summary listing each series's shape.

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only
  `data-section="chart-line-marker-aria-desc"` span.
- SVG `role="img"` with the same `aria-label`.
- Each line path is `role="graphics-symbol"` with `tabIndex={0}`.
- Each marker `<path>` is `role="graphics-symbol"` with
  `tabIndex={0}` and `aria-label="<series>: x=<x>, y=<y> (<shape>)"`.

## Data attributes

- Root: `data-section="chart-line-marker"`, `data-series-count`,
  `data-visible-series-count`, `data-total-points`,
  `data-marker-size`, `data-animate`.
- Series group: `data-series-id`, `data-series-index`,
  `data-series-color`, `data-series-marker` (default shape),
  `data-series-point-count`, `data-series-finite-count`,
  `data-hovered`.
- Marker: `data-section="chart-line-marker-marker"`,
  `data-series-id`, `data-point-index`, `data-marker-shape`,
  `data-x`, `data-y`, `data-hovered`.
- Tooltip: includes `-tooltip-shape` row identifying the shape of
  the hovered marker.
- Legend: each item exposes `data-series-marker` so the swatch
  shape can be queried.

## Out of scope

- Image / sprite markers (only built-in vector shapes + `'none'` v1).
- Per-series stroke dash on the connecting line - use
  `<ChartLineDashed>` for that.
- Curved line between markers - use `<ChartLineSmooth>` or
  `<ChartLineStep>` for non-linear segments.
- Per-shape size override (single `markerSize` for the whole chart;
  adopters who want size variation pass a custom path).
- Filled area under the line - use `<ChartLineAreaGradient>` or
  `<ChartLineAreaStacked>`.
- Log-scale axes - linear only.

## Summary

New `ChartLineVolumeOsc` UI primitive in
`web/src/components/ui/chart-line-volume-osc.tsx` -- a pure-SVG two-panel
chart with a **Volume Oscillator** panel from the percent spread of a fast
and a slow volume moving average. The top panel plots the raw volume; the
bottom panel plots the oscillator around a zero baseline.

## The computation

The Volume Oscillator gauges the momentum of trading volume. It works on a
volume series, so `ChartLineVolumeOscPoint` carries `{x, volume}`.

- `computeLineVolumeOscSma(volumes, period)` -- the simple moving average of
  the volume over `period` bars; null through the warm-up window.
- `computeLineVolumeOsc(volumes, fastPeriod, slowPeriod)` -- the Volume
  Oscillator itself: `100 * (fastMA - slowMA) / slowMA`, the percent spread
  between a fast and a slow volume average. Defined once both averages exist
  and the slow average is non-zero (a zero slow average yields null rather
  than a divide-by-zero).

A positive reading marks **expanding** volume (the fast average leads the
slow); a negative reading marks **contracting** volume; zero is **flat**.

## Worked fixture

The oscillator is `100 * (fast - slow) / slow`. Choosing a volume series
that is periodic with period equal to the slow window makes every slow
window hold one full cycle, so the slow SMA is a constant -- and a constant
slow average that divides 100 cleanly makes every reading an exact integer.

The fixture uses a volume series periodic with period 4 -- `[40,80,20,60]`
repeated -- with fast period 2 and slow period 4. Every 4-bar window sums to
200, so the slow SMA is the constant 50; the oscillator collapses to
`2 * (fast - 50)`:

```
volume   [40, 80, 20, 60, 40, 80, 20, 60, 40, 80, 20, 60]
fastSMA  [ -, 60, 50, 40, 50, 60, 50, 40, 50, 60, 50, 40]
slowSMA  [ -,  -,  -, 50, 50, 50, 50, 50, 50, 50, 50, 50]
osc      [ -,  -,  -,-20,  0, 20,  0,-20,  0, 20,  0,-20]
zone     [none x3, contracting, flat, expanding, flat,
          contracting, flat, expanding, flat, contracting]
```

2 expanding, 3 contracting, 4 flat; final reading -20. The all-up and
all-down legs exercise every zone, and every oscillator value is an exact
integer asserted with `toEqual`.

## API

`data` (a `ChartLineVolumeOscPoint[]`), `fastPeriod` (default 5),
`slowPeriod` (default 10), `pricePanelRatio` (default 0.55, clamped to
[0.3, 0.8]), `gap`, sizing/format/a11y props, `strokeWidth` / `dotRadius`,
the colour props, the toggle flags (`showAxis` / `showGrid` / `showDots`
default false / `showZeroLine` / `showMarkers` / `showTooltip` /
`showConfigBadge` / `showLegend` / `animate`), controlled + uncontrolled
`hiddenSeries` (keys `volume` / `osc`), `formatValue` / `formatX`,
`onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineVolumeOscFinitePoints`,
`normalizeLineVolumeOscPeriod`, `computeLineVolumeOscSma`,
`computeLineVolumeOsc`, `runLineVolumeOsc`, `computeLineVolumeOscLayout`,
`describeLineVolumeOscChart`, plus the `DEFAULT_CHART_LINE_VOLUME_OSC_*`
constants and the `ChartLineVolumeOscZone` type. Every exported name
carries a `LineVolumeOsc`/`VolumeOsc` infix so the `export *` barrel line
cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen `data-section="chart-line-volume-osc-aria-desc"`
description span; the volume line, the oscillator line and every dot /
marker are `role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-fast-period`, `data-slow-period`,
`data-osc-final`, `data-expanding-count`, `data-contracting-count`,
`data-flat-count`, `data-total-points` and `data-animate`.

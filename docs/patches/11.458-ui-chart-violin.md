## Summary

Add a new `<ChartViolin>` UI primitive in
`web/src/components/ui/chart-violin.tsx` -- pure-SVG
violin plot. Each series renders as a violin silhouette
mirrored around a vertical axis; the width at every
y-position is proportional to the local kernel density
estimate (Gaussian kernel + Silverman bandwidth by
default). A median dot marks the centre; an optional
inner mini-boxplot overlay (IQR box + min/max whisker
line) overlays on the same axis. Hover tooltip surfaces
median, IQR, range, and sample count. Per-violin click
handler.

## API

```ts
interface ChartViolinSeries {
  id: string;
  label: string;
  data: readonly number[];
  color?: string;
}

interface ChartViolinProps {
  series: readonly ChartViolinSeries[];
  width?: number;                  // default 560
  height?: number;                 // default 320
  padding?: number;                // default 40
  violinGap?: number;              // default 16
  bandwidth?: number;              // default: Silverman
  resolution?: number;             // default 64
  showMedian?: boolean;            // default true
  showInnerBox?: boolean;          // default true
  showLabels?: boolean;            // default true
  showAxisTicks?: boolean;         // default true
  showTooltip?: boolean;           // default true
  animate?: boolean;               // default true
  className?: string;
  ariaLabel?: string;              // default 'Violin plot'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  defaultViolinColor?: string;     // default '#2563eb'
  tickCount?: number;              // default 5
  onViolinClick?: (args: {
    series: ChartViolinSeries;
    index: number;
    stats: ViolinStats;
  }) => void;
}

interface ViolinStats {
  median: number;
  q1: number;
  q3: number;
  min: number;
  max: number;
  mean: number;
  count: number;
  bandwidth: number;
  density: ViolinDensityPoint[];
  maxDensity: number;
}

interface ViolinDensityPoint {
  y: number;
  density: number;
}
```

## Behaviour

- Renders a `<div role="region">` wrapping `<svg
  role="img">` carrying the same `aria-label`.
- KDE: `computeKDE` evaluates a Gaussian kernel at each
  y in the evaluation grid. Bandwidth defaults to
  Silverman's rule of thumb (`h = 1.06 * sigma *
  n^(-1/5)`); the `bandwidth` prop overrides it.
- Stats: `getViolinStats` returns the median, Q1, Q3,
  min, max, mean, count, bandwidth, and the full
  density array. Non-finite values are dropped.
- Violin geometry: each series occupies a vertical
  slot; the violin is centred horizontally. Half-width
  is 50% of the slot. Density is normalised per-series
  against its own peak so each violin uses the full
  width regardless of absolute sample count.
- Mirror silhouette: `buildViolinPath` walks the
  density top-to-bottom along the right edge, then
  mirrors back bottom-to-top along the left edge, and
  closes the path with `Z`.
- Inner mini-boxplot: when `showInnerBox=true`
  (default), a thin vertical line spans min-to-max
  and a small filled IQR rectangle overlays at the
  centre. Helps adopters read the summary stats
  inside the density silhouette.
- Median dot: small white-filled circle with dark
  stroke at the median y. `showMedian=false`
  suppresses.
- Hover / focus: thickens the silhouette stroke +
  lightens the fill, and opens a tooltip with median,
  IQR, range, and sample count. `onViolinClick`
  receives the full `ViolinStats` payload.
- Mount animation: `motion-safe:animate-fade-in` per
  series group. `animate=false` removes the class.
- Default ARIA description: `"Violin plot with N
  series. <label> median <m> n=<count>, ..."` --
  `ariaDescription` overrides; empty input produces
  `"No data"`.

## Pure helpers (exported)

```ts
DEFAULT_CHART_VIOLIN_WIDTH       = 560
DEFAULT_CHART_VIOLIN_HEIGHT      = 320
DEFAULT_CHART_VIOLIN_PADDING     = 40
DEFAULT_CHART_VIOLIN_GAP         = 16
DEFAULT_CHART_VIOLIN_RESOLUTION  = 64
DEFAULT_CHART_VIOLIN_TICK_COUNT  = 5
DEFAULT_CHART_VIOLIN_COLOR       = '#2563eb'

gaussianKernel(u): number
getViolinStdDev(data): number
silvermanBandwidth(data): number
computeKDE(data, evalPoints, bandwidth): number[]
getViolinEvalPoints(min, max, resolution?): number[]
getViolinStats(data, options?): ViolinStats
getViolinBounds(series): { min, max }
getViolinTicks(min, max, count?): number[]
buildViolinPath(density, centerX, halfWidth, maxDensity,
                yFor): string
describeViolinChart(series, formatValue?): string
// also re-exports getBoxplotStats from <ChartBoxplot> for
// host composition convenience.
```

- `gaussianKernel` is the standard normal density;
  peaks at u=0; symmetric; returns 0 for non-finite
  input.
- `getViolinStdDev` is the population std dev; drops
  non-finite values; empty -> 0.
- `silvermanBandwidth` returns 1 for empty / single-
  point / zero-sigma data so the KDE never divides by
  zero.
- `computeKDE` is a pure Gaussian-KDE pass; returns
  zeros for empty data; clamps non-positive bandwidth
  to 1.
- `getViolinEvalPoints` returns evenly-spaced y values
  inclusive of endpoints; collapsed range -> [min];
  non-finite -> [].
- `getViolinStats` is the single-shot stats helper;
  honours optional bandwidth + resolution + rangeMin
  + rangeMax options.
- `getViolinBounds`, `getViolinTicks` mirror the
  standard chart-tick helpers.
- `buildViolinPath` builds a closed mirror polygon
  using a caller-supplied `yFor` projection.
- `describeViolinChart` produces the default ARIA
  summary.

## Tests (63 cases)

- `gaussianKernel`: peaks at 0, symmetric, decays,
  non-finite -> 0.
- `getViolinStdDev`: known value, empty -> 0, drops
  non-finite.
- `silvermanBandwidth`: positive for normal data,
  fallback 1 for empty / single / zero-sigma.
- `computeKDE`: length-match, empty data -> zeros,
  peak near data centre, non-positive bandwidth
  fallback.
- `getViolinEvalPoints`: evenly-spaced, collapsed
  -> [min], non-finite -> [], min resolution 2.
- `getViolinStats`: empty zero record, summary
  (median/IQR/range/count), positive density for
  non-degenerate data, bandwidth option affects peak,
  rangeMin / rangeMax options shape eval points.
- `getViolinBounds`: across series, empty fallback,
  collapsed expansion.
- `getViolinTicks`: evenly-spaced, collapsed -> [min],
  default count.
- `buildViolinPath`: empty -> "", closed mirror
  polygon, zero maxDensity collapses to centre.
- `describeViolinChart`: empty "No data", per-series
  summary, formatter shaping.

  `getBoxplotStats`) so host code can compose both

## Out of scope

- Split / asymmetric violins (e.g., density by
  category overlaid on the same axis) -- host can
  wrap two `<ChartViolin>` instances side by side.
- Horizontal orientation -- vertical-only v1.
- Custom kernel choice (Epanechnikov, triangular) --
  Gaussian-only v1; host can pre-process data.
- Animated transitions on data change (snap v1;
  mount fade only).

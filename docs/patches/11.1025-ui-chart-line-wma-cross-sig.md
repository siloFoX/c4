that renders the close in the top panel and the Weighted

## WMA-over-signal pipeline

```
wma[i]     = WMA(close, period)        ; weights 1..period
                                         normalised by L*(L+1)/2
signal[i]  = SMA(wma, signalLength)    ; smoothed signal line
regime     = wma >= signal -> bullish
             wma <  signal -> bearish
             null          -> none
bias       = wma[i] vs wma[i-1] -> up / down / flat / none
bullish    : prev wma <= prev signal AND cur wma > cur signal
bearish    : prev wma >= prev signal AND cur wma < cur signal
```

The default tuning `period = 14`, `signalLength = 3` gives a
WMA centroid lag of `(period - 1) / 3 ~= 4.333` (vs SMA's
`(period - 1) / 2 = 6.5` for the same period), so the
trigger reacts noticeably faster to fresh moves while the
SMA-of-WMA signal line still suppresses single-bar jitter.

Warmup is `period + signalLength - 2 = 15` (WMA seeds at
`period - 1 = 13`, then SMA(signalLength) adds another
`signalLength - 1 = 2` bars).

## Bit-exact anchors

- **CONST close = K**: every WMA window averages K with
  triangular weights -> wma = K, signal = K, wma === signal
  -> regime `bullish` (>=) for every valid bar. 0 crosses.
  Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP close = i**: WMA over linear input settles at
  `i - (period - 1) / 3 = i - 13/3 ~= i - 4.333`. The signal
  SMA adds `(signalLength - 1) / 2 = 1` bar of lag, giving
  `signal = i - 13/3 - 1 = i - 16/3`. wma - signal = `+1`
  exact for every valid bar -> regime `bullish`, 0 crosses.
- **LINEAR DOWN close = -i**: mirror -> wma = `-i + 13/3`,
  signal = `-i + 16/3`. wma - signal = `-1` -> regime
  `bearish`, 0 crosses.

The clean `+1 / -1` separation under linear input -- a
direct consequence of the centroid-lag arithmetic
`(period - 1)/3 - (period - 1)/3 - (signalLength - 1)/2`
collapsing to `-(signalLength - 1)/2 = -1` -- is the
identity bit-exact tests rely on.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no chart
  lib dependency.
- Strict-inequality bullish vs `>=` regime classifier means
  the bullish boundary (wma == signal) settles into the
  bullish band, matching the SMA cross-sig sibling.
- Window logic returns null when any input is null, so
  warmup is correctly suppressed.

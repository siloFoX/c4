# Patch 11.171 -- UI: StatusDot primitive

Released as **v1.11.189** (web/package.json bump 1.11.188 -> 1.11.189).

## Primitive

`web/src/components/ui/status-dot.tsx` (~80 lines).

Props:

- `variant?: 'online' | 'busy' | 'away' | 'offline' | 'unknown'` (default `'unknown'`) -- drives dot color
- `size?: 'sm' | 'md' | 'lg'` (default `'md'`) -- 6 / 8 / 10 px
- `pulse?: boolean` (default `false`) -- subtle `animate-ping` halo behind the dot in the same color at 75% opacity
- `label?: ReactNode` -- optional inline text next to the dot
- `className?: string`

Color mapping:

| variant   | class                |
|-----------|----------------------|
| online    | `bg-green-500`       |
| busy      | `bg-yellow-500`      |
| away      | `bg-orange-500`      |
| offline   | `bg-muted-foreground`|
| unknown   | `bg-muted`           |

Layout: inline-flex row -- `<span>` wrapper carrying `role='status'`, an inner `relative inline-flex` containing the dot (`rounded-full` + size class + color class) and (when `pulse`) an `absolute` ping span at 75% opacity, then the optional `label` slot. `aria-label='Status: <variant>'` is auto-applied when no `label` is provided; consumers passing a `label` (or an explicit `aria-label`) suppress the auto label so the visible text owns the accessible name.

Tests: `web/src/components/ui/status-dot.test.tsx` (vitest + RTL, 13 cases) -- role=status default, each variant color, sm/md/lg size class, pulse on/off, label rendering, aria-label fallback + suppression, className merge.

Exported from `web/src/components/ui/index.ts` (export-only addition).

## Adoption (3 sites)

1. **`web/src/components/WorkerList.tsx`** -- per-worker row. New `mapStatusDotVariant(w)` mirrors the existing `mapWorkerStatusToBadgeVariant` rules: intervention -> `away`, busy -> `busy`, idle -> `online`, else `unknown`. The dot renders between the `<Avatar>` and the worker name with `size='sm'` and `pulse` when the worker is busy. The existing status `<Badge>` on the right is kept; the dot adds an at-a-glance liveness signal in the left cluster.
2. **`web/src/components/SessionsListSection.tsx`** -- per-session row inside the project group. Sessions do not carry an explicit live/closed status field, so we derive one from `session.updatedAt` recency: <5 min -> `online`, <1 h -> `away`, else `offline`; missing `updatedAt` -> `unknown`. The dot is placed in the `ListItem` title slot ahead of the short session id with `size='sm'`.
3. **`web/src/pages/Health.tsx`** -- top status row. The dot renders beside the existing healthy / degraded `<Badge>` and maps the page's `ok` flag (computed from `data.ok !== false && !error`) to `online` / `offline`, with `pulse` when healthy.

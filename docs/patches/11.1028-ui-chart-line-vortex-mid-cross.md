# 11.1028 -- UI: chart-line-vortex-mid-cross primitive (v1.11.1046)

## Scope

Add a pure-SVG dual-panel `ChartLineVortexMidCross`
component that renders the close in the top panel and both
Vortex Indicator lines (VI+ in cyan, VI- in pink) in the
bottom panel, marking VI+ crosses up through VI- (trend
reversal to uptrend) / VI+ crosses down through VI- (trend
reversal to downtrend) direct mid-cross trigger events with
bias coloring derived from the VI+ slope at the trigger bar.

This is the canonical Vortex Indicator trade signal -- VI+
crossing VI- directly, without an intermediate SMA signal
smoothing line. Unlike `chart-line-vortex-pos-cross`
(v1.11.1044, VI+ vs SMA(VI+)) and
`chart-line-vortex-neg-cross` (v1.11.1045, VI- vs SMA(VI-)),
this primitive collapses the two lines into one panel and
detects the dual-line crossing directly.

## Vortex mid-cross pipeline

```
VM+[i]   = |high[i] - low[i-1]|          ; i >= 1
VM-[i]   = |low[i]  - high[i-1]|
TR[i]    = max(high[i] - low[i],
               |high[i] - close[i-1]|,
               |low[i]  - close[i-1]|)
VI+[i]   = sum(VM+[i-period+1..i])       ; i >= period
           / sum(TR[i-period+1..i])
VI-[i]   = sum(VM-[i-period+1..i])
           / sum(TR[i-period+1..i])
regime   = VI+ >= VI- -> bullish (uptrend)
           VI+ <  VI- -> bearish (downtrend)
           null       -> none
bias     = VI+[i] vs VI+[i-1] -> up / down / flat / none
bullish  : prev VI+ <= prev VI- AND cur VI+ > cur VI-
           (trend reversal to uptrend)
bearish  : prev VI+ >= prev VI- AND cur VI+ < cur VI-
           (trend reversal to downtrend)
```

No signal SMA -- VI+ and VI- are compared directly. Warmup
is therefore `period = 14` (no extra signal-length bars), 2
fewer than the `period + signalLength - 1 = 16` warmup of
the single-line cross-sig siblings.

## Bit-exact anchors

All anchors use HLC input.

- **CONST band** `high = K + 1`, `low = K - 1`, `close =
  K`: VM+ = 2, VM- = 2, TR = 2 -> VI+ = VI- = 1 from `i =
  period`. VI+ === VI- -> regime `bullish` (>=). 0 crosses.
  Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close = i`:
  VM+ = 3, VM- = 1, TR = 2 -> VI+ = 1.5, VI- = 0.5. VI+ -
  VI- = `+1` exact -> regime `bullish`. 0 crosses.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`, `close
  = -i`: VM+ = 1, VM- = 3, TR = 2 -> VI+ = 0.5, VI- = 1.5.
  VI+ - VI- = `-1` exact -> regime `bearish`. 0 crosses.

The clean integer separation under linear input is the
direct, sign-mirrored counterpart of the two single-line
siblings: LINEAR UP forces VI+ above VI- by exactly 1 (the
steady-state bullish anchor), LINEAR DOWN forces VI+ below
VI- by exactly 1 (the steady-state bearish anchor). This
is the first primitive in the Vortex trio that surfaces a
genuinely bearish steady-state regime anchor, since the
single-line siblings collapse their regime to bullish on
all three anchors (their signal SMA always tracks the
indicator).

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no chart
  lib dependency.
- Strict-inequality bullish vs `>=` regime classifier means
  the boundary (VI+ === VI-) settles into the bullish band,
  matching every other `chart-line-*-cross` sibling.
- High < low input rows are filtered at the boundary so the
  TR / VM math never receives a negative range.
- Zero-TR rolling window is short-circuited to null so a
  degenerate window cannot produce Infinity for either VI+
  or VI-.
- Both VI+ and VI- use the *same* `sumTr` divisor (a single
  rolling-sum loop computes all three sums), so the two
  lines stay numerically coherent under floating-point
  drift (no chance of VI+ using a slightly different TR
  denominator than VI-).

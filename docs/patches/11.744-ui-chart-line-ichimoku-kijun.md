# Patch 11.744 -- UI: chart-line-ichimoku-kijun primitive

Released as `v1.11.762` on 2026-05-26.

## Scope

Add `<ChartLineIchimokuKijun>` -- a pure-SVG single-panel chart
with an Ichimoku Kinko Hyo **Kijun-sen** base-line overlay drawn
from the midpoint of the highest high and the lowest low across
the lookback. Same computational shape as the v1.11.761
Tenkan-sen primitive, but with a longer default base lookback
(`26`) and a `kijun` series-id / colour / data-section namespace.

## Definition

Inputs:

- `data: ChartLineIchimokuKijunPoint[]` where
  `ChartLineIchimokuKijunPoint = { x; high; low; close }`. Bars
  with `high < low` are dropped.
- `period: number` -- the base-line lookback. Default `26`.
  Normalized to an integer `>= 1`.

Algorithm (per bar `i >= period - 1`):

```
HH        = max(high over [i - period + 1, i])
LL        = min(low  over [i - period + 1, i])
kijun[i]  = (HH + LL) / 2
```

The first `period - 1` bars are null. A non-finite high or low
anywhere in the window also nulls the bar.

## Bit-exact algebraic anchors

The same anchors as the Tenkan-sen primitive hold (the formula is
identical, only the default lookback differs):

- **CONST_FLAT** (`high == low == K`) -> `kijun = K` bit-exact at
  every defined bar.
- **RISING integer ramp** (`high == low == i + 10`) period 4 ->
  `kijun = i + 8.5` bit-exact dyadic.
- **FALLING integer ramp** (`high == low == 19 - i`) period 4 ->
  `kijun = 20.5 - i` bit-exact dyadic.
- **Period 2** RISING -> `kijun = i + 9.5` bit-exact dyadic.
- **Period 1**: `kijun = high[i]` bit-exact on the RISING fixture.
- **Default period 26 warm-up**: 25 leading nulls on the long
  CONST_FLAT fixture, then `kijun = K` bit-exact from bar 25 on.
- **Translation invariance**: shifting every high / low / close
  by an integer `k` shifts `HH` and `LL` by exactly `k`, so the
  midpoint shifts by exactly `k`.

## Component

`forwardRef<HTMLDivElement, ChartLineIchimokuKijunProps>` with
displayName. Renders:

- A region-role outer `<div data-section="chart-line-ichimoku-kijun">`.
- An `<svg role="img">` with an `aria-describedby` description.
- A close `<path>`, optional close dots, a Kijun-sen `<path>`,
  and one marker per defined bar at the close position with
  `data-zone = 'above-kijun' | 'at-kijun' | 'below-kijun'`.
- Legend toggle for `price` / `kijun`, plus a `showKijun` prop
  and an `onPointClick` handler.
- ARS design-system tokens for colour and motion (`motion-safe:
  animate-fade-in`).

## Summary

New `ChartLineCfo` UI primitive at
`web/src/components/ui/chart-line-cfo.tsx` -- a pure-SVG
two-panel chart with a Chande Forecast Oscillator panel of the
percent gap between price and its linear regression forecast.

## Indicator math

Tushar Chande's Forecast Oscillator measures how far the price
has strayed from the value a linear regression predicts for it.

`computeLineCfoForecast(closes, period)` fits a least-squares
line `y = a + b*t` through the trailing `period` closes (with
`t = 0..period-1`) and returns its endpoint value `a + b*(period
- 1)` -- the regression's forecast for the current bar.

`computeLineCfo(closes, period)` is the percent gap:

```
CFO[i] = 100 * (close[i] - forecast[i]) / close[i]
```

A positive CFO means the price is running above the forecast, a
negative CFO below it; the oscillator crosses zero whenever the
price meets its regression line.

### Exactness

The regression is rational, but its slope divides by
`period * sumT2 - sumT^2` (20 for `period` 4), which is not a
power of two, so the forecast is generally not exactly
representable and the non-linear-fixture tests assert with
`toBeCloseTo`. One case is exact: a perfectly linear price. The
regression fits a linear series with no residual, so the
forecast equals the price exactly and the CFO is exactly zero --
asserted with `toEqual` on the linear fixture
`[10, 20, 30, 40, 50, 60]`.

The non-linear fixture uses closes that are all powers of two, so
each CFO ratio is dyadic; the hand-computed values
`[., ., ., 0, 15, -40, 10, 5, -50, 20, -20, 10, -20]` are matched
with `toBeCloseTo`, and the flat-window bar is an exact 0.

## API

- `getLineCfoFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineCfoPeriod(period, fallback)` -- coerce to an
  integer of at least 2.
- `computeLineCfoForecast(closes, period)` -- the linear
  regression endpoint forecast.
- `computeLineCfo(closes, period)` -- the percent-gap series.
- `runLineCfo(points, options?)` -- sort finite points by `x`,
  run the pipeline, return `{ series, period, threshold,
  forecast, cfo, samples, cfoFinal, aboveCount, belowCount,
  neutralCount, ok }`. Each sample is classified `above` /
  `below` / `neutral` / `none`. `ok` is false for fewer than two
  points.
- `computeLineCfoLayout(options)` -- stack the price panel
  (price plus the forecast line) above a CFO panel; project the
  price path, the forecast path, the CFO path, one marker per
  defined bar, and the zero and trigger reference lines.
- `describeLineCfoChart(data, options?)` -- ARIA description.

## Component

`ChartLineCfo` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, two stacked panels
(price on top, CFO below) with their own axis ticks and grid,
panel labels, a config badge showing the period, a four-series
legend (Price / Forecast / CFO / Levels) with toggle, a
hover/focus tooltip exposing the forecast, the CFO and the zone,
and a `motion-safe:animate-fade-in` entrance. It consumes
`{ x, value }` points. The price panel overlays the linear
regression forecast as a dashed line; the CFO panel draws the
oscillator over dashed zero and trigger reference lines, with
each marker coloured by its zone. Every node carries a
`data-section` attribute.

## Distinctness

- **vs the deviation oscillators (cog / dpo)** -- those measure
  the price against a trailing moving average. The CFO measures
  it against a forward-fitted regression line, so it reads the
  residual of a trend forecast.
- **vs chart-line-theilsen / chart-line-lsma** -- those plot the
  regression line itself. The CFO plots the percent gap between
  the price and that line as a zero-centred oscillator.
- **vs chart-line-residual** -- a residual is the raw price
  minus a fit. The CFO normalises that residual by the price, so
  it is a percentage comparable across price levels.

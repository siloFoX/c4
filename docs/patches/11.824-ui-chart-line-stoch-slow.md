# Patch 11.824 — ChartLineStochSlow primitive (v1.11.842)

## Summary

Add `<ChartLineStochSlow />`, a pure-SVG dual-panel line chart that renders a
Slow Stochastic Oscillator (SMA-smoothed %K) panel beneath a price line.
Slow %K is the simple moving average of the raw fast %K across a configurable
smoothing window, dampening the oscillator's whipsaw versus the unsmoothed
Stoch Fast primitive (11.820 / v1.11.838).

## Formula

```
highestHigh[i] = max(high[i - length + 1 .. i])
lowestLow[i]   = min(low[i  - length + 1 .. i])
fastK[i]       = (close[i] - lowestLow[i]) / (highestHigh[i] - lowestLow[i]) * 100
                 (null when highestHigh - lowestLow <= 0)
slowK[i]       = SMA(fastK, smoothLength)[i]
```

Both rolling extrema and the smoothing SMA short-circuit `null` warmup
slots so the warmup region of `slowK` is exactly `length + smoothLength - 2`
indices wide.

## Defaults

| Knob            | Value     |
|-----------------|-----------|
| length          | 14        |
| smoothLength    | 3         |
| overbought      | 80        |
| oversold        | 20        |
| stochColor      | `#7c3aed` |

Color is a distinct purple from the Stoch Fast primitive's indigo (`#4f46e5`)
so the two oscillators stay visually disambiguated when stacked.

## Bit-exact anchors

The integration test sweeps `(K, length, smoothLength)` and asserts:

- `CONST h=l=close=K` &rarr; flat lookback window, `fastK = null`,
  `slowK = null` post-warmup. The SMA short-circuit propagates `null`
  through the smoothing pass.
- `LINEAR UP h=l=close=k+1` &rarr; `fastK = 100` every defined bar
  (close hits highest-high), `slowK = SMA(100) = 100` bit-exact.
- `LINEAR DOWN h=l=close=N-k` &rarr; `fastK = 0` every defined bar
  (close hits lowest-low), `slowK = SMA(0) = 0` bit-exact.

These three anchors stay exact under IEEE 754 because the divisions
collapse to `(c - l) / (h - l)` with `c == h` or `c == l`, and the
subsequent SMA sums and divides only the constants `100` or `0`.

# 11.1014 -- UI: chart-line-stc-divergence-cross primitive (v1.11.1032)

## Scope

Add a pure-SVG dual-panel `ChartLineStcDivergenceCross`
component -- the **bias-coloured** variant of the Schaff Trend
Cycle divergence detector. Sibling of
`chart-line-schaff-divergence-cross` (11.1011): same underlying
STC pipeline, same 5-state divergence regime model, but every
sample and every cross marker additionally carries a `bias`
field (`up` / `down` / `flat` / `none`) derived from the STC
slope at the trigger bar. The bias is used to colour the cross
markers green/red/grey so the chart distinguishes between a
divergence cross that fires while STC is still rising vs.
falling vs. parked.

## STC pipeline (Schaff) + bias

```
macd[i] = SMA(close, fastLength) - SMA(close, slowLength)
k1[i]   = 100 * (macd[i] - LLV(macd, cycle)) / (HHV - LLV);
          50 when HHV === LLV (degenerate fallback)
d1[i]   = EMA(k1, factor)
k2[i]   = 100 * (d1[i]  - LLV(d1, cycle))  / (HHV - LLV);
          50 when HHV === LLV (degenerate fallback)
stc[i]  = EMA(k2, factor)

bias[i]
  up    : stc[i] > stc[i-1]
  down  : stc[i] < stc[i-1]
  flat  : stc[i] === stc[i-1]
  none  : either side null
```

Mirrors the canonical STC formulation already used by the
other STC primitives. Default tuning matches Schaff's original
proposal: `fastLength = 23`, `slowLength = 50`, `cycleLength
= 10`, `factor = 0.5`. The bias classifier matches
`chart-line-stc-overbought-cross`'s `classifyLineStc*Bias`.

Warmup is `slowLength + 2 * (cycleLength - 1) - 1 = 67` (same
as `chart-line-schaff-divergence-cross`).

## Divergence regime model + bias coloring

5-state classifier exactly like the other divergence primitives,
plus per-sample / per-cross bias:

```
priceUp = close[i] > close[i - divergenceWindow]
stcUp   = stc[i]   > stc[i - divergenceWindow]
state   = aligned-bullish    if priceUp && stcUp
        | aligned-bearish    if !priceUp && !stcUp
        | divergent-bullish  if !priceUp && stcUp
        | divergent-bearish  if priceUp && !stcUp
        | none               if either input is null

cross fill   = upColor       if bias === 'up'
             | downColor     if bias === 'down'
             | flatColor     if bias === 'flat'
             | bullish/bearishColor if bias === 'none' (fallback)
```

Crosses suppress when either `prev` or `cur` regime is `none`,
so warmup transitions never fire spuriously. The bias on each
cross marker is computed from `stc[i]` vs `stc[i-1]` at the
cross bar (not from the divergence window).

## Bit-exact anchors

| input              | stc (post-warmup)         | bias post-warmup | regime (post i=72) | crosses |
| ------------------ | ------------------------- | ---------------- | ------------------ | ------- |
| CONST close = K    | 50 (degenerate cascade)   | flat             | aligned-bearish    | 0       |
| LINEAR UP, close=i | 50 (constant macd -> 50)  | flat             | divergent-bearish  | 0       |
| LINEAR DOWN, close=-i | 50 (constant macd -> 50) | flat            | aligned-bearish    | 0       |

CONST K verified across K = 0..1234. STC is constant 50 from
i = 67 onwards because the inner stochastic of a flat macd
returns 50 via the degenerate fallback. The bias of `stc[i]`
vs `stc[i-1]` is therefore `flat` for every post-warmup
sample (the first valid sample at i=67 has bias `none`
because i=66 is null). For 120 bars: noneCount=72 (regime),
flatBiasCount=52 (sample i=68..119), regime samples i=72..119
all aligned-bearish (CONST/DOWN) or divergent-bearish (UP).

LINEAR UP exercises the constant-macd case via algebra:
fastSma settles at `i - 11`, slowSma at `i - 24.5`, so
`macd = +13.5` constant. The stochastic of a flat constant
hits the degenerate branch and returns 50. priceUp = true,
stcUp = false -> regime `divergent-bearish`. bias = flat
because `stc[i] === stc[i-1] === 50`.

LINEAR DOWN is the algebraic mirror: macd = -13.5 -> stc =
50. priceUp = false, stcUp = false -> regime
`aligned-bearish`, bias = flat.

The bias coloring is exercised in dedicated unit tests for
`detectLineStcDivergenceCrossCrosses`, which checks that
crosses entering from rising STC are tagged `up`, from
falling STC are tagged `down`, and from flat STC are tagged
`flat`.

## Relationship to `chart-line-schaff-divergence-cross`

This primitive is the **bias-coloured** companion of the
plain `chart-line-schaff-divergence-cross` (11.1011). Both
share the same STC pipeline and divergence regime model.
Differences:

| feature                  | schaff-divergence-cross | stc-divergence-cross    |
| ------------------------ | ----------------------- | ----------------------- |
| 5-state regime           | yes                     | yes                     |
| cross detection          | regime transition       | regime transition       |
| per-sample bias          | no                      | yes (up/down/flat/none) |
| per-cross bias           | no                      | yes                     |
| marker fill              | bullishColor/bearishColor | bias-coloured           |
| stc path coloring        | single color            | single color            |
| `upBiasCount` etc        | no                      | yes (data attrs)        |
| tooltip "bias" field     | no                      | yes                     |

Both ship in the public barrel and can coexist; users who
just want the divergence signal pick the Schaff variant, and
users who want the slope-aware coloring pick the STC variant.

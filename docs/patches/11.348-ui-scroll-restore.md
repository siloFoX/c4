## Goal

Dispatch:

- Build `web/src/lib/scroll-restore.ts` --
  per-route scroll position save/restore via
  sessionStorage, integrated with React Router.
  Restore on back/forward, reset on forward
  navigation.

The c4 app does NOT use React Router today --
navigation is driven by a `topView` enum + a
per-route render switch in `App.tsx`. The module
ships as a router-agnostic primitive: callers pass
a `routeKey` string and a `navigationType`
discriminator. A future React Router adoption can
plug `useLocation().pathname` + `useNavigationType()`
into the same surface.

## Module API

```ts
saveScrollPosition(routeKey, position): boolean
getScrollPosition(routeKey): number | null
clearScrollPosition(routeKey): boolean
clearAllScrollPositions(): number  // returns count
shouldRestoreScroll(navigationType): boolean

useWindowScrollRestore({
  routeKey,
  navigationType?,   // default 'forward'
  targetRef?,        // default = window
  debounceMs?,       // default 100
});
```

- `routeKey` -- bare strings get the
  `c4:scroll-restore:` prefix; colon-bearing
  strings are treated as fully-qualified.
- `navigationType` -- `'forward'` clears + resets
  to 0; `'pop'` / `'replace'` restore (fall back
  to 0 when no saved value).
- `targetRef` -- pass a ref for a container; omit
  to drive window-level scroll.
- `debounceMs` -- coalesces rapid scroll events
  before writing to sessionStorage.

The hook is SSR-safe: every browser-only access is
gated behind `typeof window`.

## App.tsx adoption

```tsx
const popRef = useRef(false);
useEffect(() => {
  const onPop = () => { popRef.current = true; };
  window.addEventListener('popstate', onPop);
  return () => window.removeEventListener('popstate', onPop);
}, []);
const navigationType = popRef.current ? 'pop' : 'forward';
useEffect(() => { popRef.current = false; }, [topView]);
useWindowScrollRestore({
  routeKey: `topView:${topView}`,
  navigationType,
});
```

The popstate listener flips a ref; the next
topView change reads the ref to decide
restore-vs-reset, then clears it. Sidebar clicks
(which do not fire popstate) hit the `forward`
path and reset.

## Tests

19 cases in `scroll-restore.test.ts`:

- save / get / clear round-trip; null on cold key;
  rounds non-integers; rejects negative / NaN /
  Infinity; prefix handling for bare vs
  colon-bearing keys.
- `clearAllScrollPositions` removes only prefixed
  entries.
- `shouldRestoreScroll` truth table.
- `useWindowScrollRestore` element mode: forward
  resets, forward clears, pop restores, pop with
  no saved -> 0, save outgoing on routeKey
  change, debounced write, unmount flush.
- `useWindowScrollRestore` window mode: calls
  `window.scrollTo` on pop when saved value
  exists.

for `scroll-restore.ts` and `App.tsx`.

## Out of scope

- Migration of the existing
  `hooks/use-scroll-restoration.ts` (container-
  scoped) into the new module. They cover
  different surfaces -- container vs window /
  per-route -- and the old hook still has
  in-tree callers (HistoryView sidebar).
- React Router integration. Hook's
  `navigationType` arg lets a future adoption
  pass `useNavigationType()` from
  `react-router-dom` directly.
- Per-page scope hooks (Snapshots table scroll,
  Workers grid scroll, etc.). The `targetRef`
  option supports it; adoption is a follow-up.
- Cross-tab sync. sessionStorage scope is
  intentional -- each tab keeps its own
  scroll state.

# 11.518 -- web/src/components/ui/chart-line-target.tsx

Pure-SVG multi-series line chart with **a target reference line + over/under
shading** between the data and the target. Each per-segment gap is
painted with the `overColor` when the line is above the target and
the `underColor` when below; segments that straddle the target are
split exactly at the crossing point. Per-series goal-tracking stats
(over count, under count, peak gaps, percent over target) drive a
legend progress badge and the auto-generated ARIA description.

## Distinct from other horizontal-reference primitives

- `<ChartLineThreshold>` (11.503): thresholds + **zone shading above
  / below the threshold across the whole chart**. The chart-line-target
  primitive shades the **gap between the line and the target**, not
  the absolute regions of the canvas.
- `<ChartLineBaseline>` (11.510): per-point above/below/equal
  classification, but **no area shading**. chart-line-target adds the
  signed area visualisation + goal-tracking stats.
- `<ChartLineBand>` (11.516): chart-wide rectangle between two fixed
  y-values. chart-line-target is a single goal line.
- `<ChartLineComparison>` (11.511): difference shading between **two
  curves**. chart-line-target shades between **one curve and a flat
  target**.
- `<ChartLineMinMax>` (11.517): markers at the data-derived extremes.
  chart-line-target is about a user-supplied goal.

The canonical "did we beat the goal" / "SLO compliance" / "sales
target progress" visualisation.

## Public surface

- `<ChartLineTarget series={...} target={100} targetLabel="Target" />`.
- `target` (default `0`) is the single horizontal goal. `epsilon`
  (default `0`) widens the "at" equality band for noise tolerance.
- Per-series `{id, label, data, color?, overColor?, underColor?}`.
- Per-series `overColor` / `underColor` overrides win over the
  chart-level `overColor` / `underColor` props.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showOverUnderShading`, `showTargetLine`,
  `showTargetLabel`, `showProgress`, `animate`.
- `targetDashArray` (`'6 4'`), `targetColor` (`'#0f172a'`),
  `targetStrokeWidth` (`2`), `fillOpacity` (`0.18`).
- Standard sizing + format + a11y props.
- `formatValue`, `formatX`, `formatDelta` (signed delta formatter),
  `formatPercent`.
- Controlled + uncontrolled `hiddenSeries`.
- Callbacks: `onPointClick({series, point})`,
  `onRegionClick({series, region})`.

## Pure helpers

- `getLineTargetDefaultColor(index)` -- 10-color palette cycle.
- `getLineTargetFinitePoints(points)` -- finite-only filter.
- `classifyLineTargetPoint(y, target, epsilon?)` -- returns
  `{delta, direction}` where `direction` is `'over' | 'under' | 'at'`.
  Non-finite `y` returns `{delta: 0, direction: 'at'}`. Non-finite
  `target` falls back to `0`. `epsilon` widens the equality band.
- `findLineTargetCrossing(x1, y1, x2, y2, target)` -- returns the x
  at which the segment crosses the horizontal `target` line via the
  absolute-delta proportion, or `null` when no crossing exists,
  one endpoint sits on the target, or inputs are non-finite /
  degenerate.
- `computeLineTargetStats(points, target, epsilon?)` -- returns
  `{overCount, underCount, atCount, maxOverGap, maxUnderGap,
  totalArea, finiteCount, percentOver, percentUnder, percentAt}`.
  Drops non-finite samples silently. `totalArea` approximates the
  signed area as `sum( (|delta_i| + |delta_{i+1}|) / 2 * |dx| )`.
- `computeLineTargetLayout({series, target?, epsilon?, hiddenSeries?,
  showOverUnderShading?, overColor?, underColor?, width, height,
  padding, tickCount?, xMin?, xMax?, yMin?, yMax?})` -- returns
  `{series, xTicks, yTicks, xMin, xMax, yMin, yMax, target, targetY,
  innerWidth, innerHeight, totalPoints, visibleSeriesCount}`.
  Each layout series carries `{id, label, index (stable original),
  color, overColor, underColor, points, path, regions, stats,
  finiteCount, totalCount}`. Each region is a closed `M ... L ... L
  ... L ... Z` polygon from the segment endpoints down to the target
  line. Regions are emitted from points sorted by x ascending; the
  series `path` follows the original point order so adopters that
  feed pre-sorted data keep their order.
- `describeLineTargetChart(series, target, hidden?, formatValue?)` --
  builds the screen-reader description ("Line chart vs target T across
  N series. A: K/Total over (P%), peak over O, peak under U.").

## Region build algorithm

Per consecutive pair `(x1, y1) -> (x2, y2)` in the **x-sorted finite
subset**:

1. If `crossing` is `null`:
   - When both endpoints sit exactly on the target, skip (no area).
   - Otherwise emit a single polygon
     `(x1, y1) -> (x2, y2) -> (x2, target) -> (x1, target) -> Z`,
     coloured by whichever endpoint deviates from target (or `over`
     if both are equal due to rounding).
2. If `crossing` is non-null:
   - Compute `crossing = x1 + (x2 - x1) * |y1-target| /
     (|y1-target| + |y2-target|)`.
   - Emit two polygons, one for each side of the target, with
     opposite directions.

`x1 == x2`, zero-total delta, exact-zero endpoint deltas, and
non-finite values all yield no crossing.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Target reference line is `role="graphics-symbol"` with aria-label
  `"<targetLabel>: <value>"`.
- Per-line path + per-dot are `role="graphics-symbol"` +
  `tabIndex={0}`; dot aria-label includes the direction + delta.
- Every node carries `data-section="chart-line-target-..."` for test
  selectors.
- Series groups expose `data-series-over-count`,
  `data-series-under-count`, `data-series-at-count`,
  `data-series-max-over-gap`, `data-series-max-under-gap`,
  `data-series-percent-over`, `data-series-total-area`,
  `data-series-color`, `data-series-finite-count`.
- Dots expose `data-direction` (`'over' | 'under' | 'at'`),
  `data-delta`.
- Regions expose `data-region-is-over`, `data-region-start-x`,
  `data-region-end-x`, `data-region-fill-color`, `data-region-area`.
- Root mirrors `data-target`, `data-target-y`.

## Tooltip

Hovering a dot shows the series label, x, y, and a coloured
`over|under|at <targetLabel.toLowerCase()>: +/-<delta>` row using
`formatDelta` (sign-aware).

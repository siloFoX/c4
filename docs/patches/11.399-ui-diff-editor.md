## Composition

The editor reuses the diff engine
from `<SnapshotDiff>` (11.396):

- `computeLineDiff(before, after)`
- `findDiffChangeBlocks(lines)`
- `groupDiffHunks(lines,
  contextLines)`

That keeps the two primitives
strictly in sync; any future
upgrade to the engine (e.g.,
Myers / Patience) automatically
threads through both.

## Behaviour

### Hunk navigation

| input | action |
| --- | --- |
| Prev button | previous hunk; wraps |
| Next button | next hunk; wraps |
| `n` / `Alt+ArrowDown` | next |
| `p` / `Alt+ArrowUp` | prev |
| Counter | `<idx+1> / <N>` or `0 / 0` |

`activeHunkIndex` controlled or
uncontrolled. Buttons disable when
`changeBlocks.length === 0`.

### Accept / Reject per hunk

Two buttons attach to the LAST
line of every change block (or
the only line if the block is one
row). Click:

1. Records the decision in the
   internal Map (uncontrolled) or
   passes through to the
   `onDecisionsChange` callback
   (controlled).
2. Fires `onAcceptHunk(range)` /
   `onRejectHunk(range)`.
3. Re-renders the row with the
   new `data-decision` attribute
   AND visual treatment
   (rejected: `line-through` +
   `opacity-60`).

The pure helper `applyHunkDecisions`
projects the decision Map onto the
`after` string. Decision matrix:

| line type | decision | included? |
| --- | --- | --- |
| equal | (any) | yes verbatim |
| add | accept (default) | yes |
| add | reject | no |
| remove | accept (default) | no |
| remove | reject | yes (restore before) |

Lines outside any block always
pass through.

### Inline edit

`readOnly={false}` (default) ->
header shows an `Edit` button.
Click toggles into edit mode:

- The diff body is replaced by a
  `<textarea>` bound to `after`.
- `aria-label="After content"`,
  `data-section="diff-editor-textarea"`.
- Typing fires
  `onAfterChange(next)`.
- The diff recomputes on every
  re-render of `before` / `after`.
- The button label flips to
  `Done`. Click again to return
  to read mode.

`readOnly={true}` -> the Edit
button is suppressed; Accept /
Reject buttons are also hidden;
`data-read-only="true"`.

### Folding

Long unchanged regions collapse
to a one-line click-to-expand
row, same contract as
`<SnapshotDiff>`:

- `data-section="diff-editor-fold"`
- `data-fold-id`
- `data-fold-count`
- inner button with aria-label
  "Expand <N> unchanged lines"

`defaultExpandedFolds={true}`
mounts everything open.

### Syntax highlighting

The editor itself does not bundle
a highlighter. Pass `renderText:
(text, language) => ReactNode`
to plug in Prism / Highlight.js /
Shiki. Default render is the raw
text in a monospace pre-wrap cell.

## Out of scope (deferred)

- Side-by-side mode in the
  editor. Unified is canonical
  for the edit surface; side-by-
  side belongs to the viewer
  variant.
- Inline word-level diff. Line-
  level only for v1.
- Save / Discard buttons. The
  host owns the after string;
  submit UX is host responsibility.
- Three-way merge. Two-way only.
- Conflict markers
  (<<<<<<<<<). Out of scope;
  belongs to a merge-specific
  primitive.
- Imperative scroll-to-hunk API.
  Host can compute the row via
  `data-block-index` + scroll
  into view itself.

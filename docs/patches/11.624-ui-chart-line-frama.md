## Summary

New `ChartLineFrama` UI primitive at
`web/src/components/ui/chart-line-frama.tsx` -- a pure-SVG line chart
with a Fractal Adaptive Moving Average overlay whose smoothing
tracks the fractal dimension of price.

## Indicator math

John Ehlers' Fractal Adaptive Moving Average. For each trailing
window of `period` bars (the period must be even):

```
half = period / 2
N1   = (range of the first half)  / half
N2   = (range of the second half) / half
N3   = (range of the whole window) / period
D    = (log(N1 + N2) - log(N3)) / log(2)        clamped to [1, 2]
alpha = exp(-4.6 * (D - 1))                     clamped to [0.01, 1]
FRAMA[i] = alpha*value[i] + (1 - alpha)*FRAMA[i-1]
```

`D` is the **fractal dimension** of the price -- a roughness
measure. It runs near 1 for a smooth trend (the two halves stack
into the whole) and near 2 for a jagged choppy market (each half
fills the whole range). The dimension drives the adaptive smoothing
constant: a low dimension makes alpha near 1 so the average tracks
the price closely; a high dimension makes alpha near 0.01 so the
average slows almost to a halt. A flat window reads dimension 1.

The FRAMA is defined from index `period - 1` onward; it is seeded
there with that bar's price and is recursive from index `period`.

### Worked fixture

`values` is a linear ramp of 8 points (`value[i] = 10 + 2i`),
`period 4`. A ramp is maximally smooth, so its fractal dimension
clamps to exactly 1, the adaptive alpha is `exp(0) = 1`, and the
FRAMA tracks the price exactly:

```
dimension = [.,.,., 1, 1, 1, 1, 1]
alpha     = [.,.,., 1, 1, 1, 1, 1]
frama     = value = [.,.,., 16, 18, 20, 22, 24]
```

Every value is exact, so the tests assert the series with
`toEqual`. A square-wave fixture fills each half-window with the
full range, giving a fractal dimension of 2 (a `log(8)/log(2)`
rounding artifact makes it `1.9999999999999998`, so the tests
assert it with `toBeCloseTo`) and an adaptive alpha of `exp(-4.6)`,
which barely moves the average.

## API

- `getLineFramaFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineFramaPeriod(period, fallback)` -- coerce to an even
  positive integer of at least 2; non-finite or sub-2 falls back,
  fractional floors, odd rounds down to even.
- `computeLineFramaDimension(values, period)` -- the fractal
  dimension series, clamped to [1, 2].
- `computeLineFrama(values, period)` -- returns `{ dimension, alpha,
  frama }`.
- `runLineFrama(points, options?)` -- sort finite points by `x`,
  compute the series, return `{ series, period, dimension, alpha,
  frama, samples, framaFinal, framaMin, framaMax, aboveCount,
  belowCount, ok }`. Each sample carries the price's `above` /
  `below` / `on` position versus the FRAMA. `ok` is false for fewer
  than two points.
- `computeLineFramaLayout(options)` -- project the price and FRAMA
  onto one shared panel. Returns the panel, ticks, the price path,
  the FRAMA path, and FRAMA markers.
- `describeLineFramaChart(data, options?)` -- ARIA description.

## Component

`ChartLineFrama` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, axis ticks and grid, a
config badge, a two-series legend (Price / FRAMA) with toggle, a
hover/focus tooltip exposing the value, FRAMA, fractal dimension and
position, and a `motion-safe:animate-fade-in` entrance. Every node
carries a `data-section` attribute.

## Distinctness

- **vs chart-line-kama** -- both are adaptive recursive averages.
  The KAMA scales its smoothing by an *efficiency ratio* (net change
  over the path travelled). The FRAMA scales by the *fractal
  dimension* -- a roughness measure read from comparing the range of
  two half-windows to the whole.
- **vs chart-line-vidya** -- the VIDYA scales its smoothing by the
  *absolute Chande Momentum Oscillator*. The FRAMA scales by the
  fractal dimension. Three adaptive averages, three different
  market-state drivers.
- **vs chart-line-ema / dema / tema / wma / trima / t3** -- those
  use a *fixed* smoothing scheme. The FRAMA's smoothing varies bar
  to bar with the measured fractal dimension.

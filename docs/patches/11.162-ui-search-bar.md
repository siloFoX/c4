# 11.162 -- UI SearchBar primitive

New `<SearchBar>` UI primitive in `web/src/components/ui/search-bar.tsx`.
Shipped in `web/package.json` v1.11.180.

## Shape

- Controlled (`value` + `onChange`) or uncontrolled (`defaultValue`).
- `onChange(value)` fires on every keystroke; `onDebouncedChange(value)`
  fires once after `debounceMs` (default 200ms). Initial emit is skipped
  when `defaultValue === currentValue` and no user input has occurred.
- Wrapper is `role='search'`. Native `<input type='search'>` so browsers
  suppress autosuggest, with `aria-label` forwarded via `ariaLabel`.
- Leading lucide-react `Search` icon (left, absolutely positioned).
  Trailing lucide-react `X` clear button (right, absolutely positioned)
  appears when `clearable !== false` and `value` is non-empty.
- Controlled clear does NOT mutate internally - just calls
  `onChange('')` + `onClear()`. Uncontrolled clear also clears the
  internal state.
- `forwardRef` to the underlying input so callers can focus
  programmatically.
- Tokens match the existing `<Input>` primitive (`h-10` / `min-h-[44px]`
  on `md`, `h-8` on `sm`, same border / focus ring / placeholder colors).

## Adoption sites (v1.11.180)

1. `web/src/pages/Templates.tsx` -- top filter row (templates list).
2. `web/src/pages/Profiles.tsx`  -- top filter row (profiles list).
3. `web/src/pages/Validation.tsx` -- top filter row (workers list).

Each call site forwards the existing controlled state (the filter
string and its setter) into `SearchBar` and keeps the same i18n
placeholder + aria-label keys. The previous `<Input>` + `Tooltip`
wrappers remain; only the inner control was swapped.

## Tests

`web/src/components/ui/search-bar.test.tsx` covers 17 cases with
vitest + RTL: search icon, type=search, placeholder, per-keystroke
onChange, debounced single emit (vi.useFakeTimers), clear-button
visibility on (non-)empty / clearable=false, controlled vs
uncontrolled clear semantics, disabled (input + clear button),
forwardRef, ariaLabel, size sm/md classes, className /
inputClassName merging, initial-emit skip, stable displayName.

Adoption sites still pass their existing test suites (Templates,
Profiles, Validation) since they assert via `getByLabelText` /
`getByPlaceholderText`, both of which SearchBar honours.

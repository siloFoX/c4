## Summary

New `ChartLineWae` UI primitive in
`web/src/components/ui/chart-line-wae.tsx` -- a pure-SVG two-panel line
chart with a **Waddah Attar Explosion** panel from the MACD delta against
a Bollinger band width. The top panel plots the price; the bottom panel
plots the signed momentum bracketed by the explosion band.

## The computation

The Waddah Attar Explosion pits a MACD-derived momentum against a
Bollinger Band width "explosion line". It is a close-only indicator, so
`ChartLineWaePoint` carries `{x, value}`.

- `computeLineWaeMacd(values, fastPeriod, slowPeriod)` -- the MACD line:
  the fast EMA of the closes minus the slow EMA.
- `computeLineWaeStdev(values, period)` -- the rolling population standard
  deviation; `computeLineWaeBollWidth(values, period, mult)` scales it into
  the Bollinger Band width (`2 * mult * stddev`).
- `computeLineWae(values, options)` -- composes the pipeline: the WAE
  momentum is the change in the MACD scaled by a sensitivity factor (the
  signed push), the explosion line is the Bollinger Band width.
- `classifyLineWaeZone(trend, explosion)` -- a bar is **explosive-up** or
  **explosive-down** when the momentum magnitude tops the explosion line
  (the sign gives the direction), else **quiet**.

## Worked fixture

Two conditions make the pipeline exact. The fast EMA period 3 has alpha
`2/4 = 0.5` and the slow EMA period 7 has alpha `2/8 = 0.25` -- both
dyadic -- so the MACD stays exact. The Bollinger width is read at period
2, where the population standard deviation collapses to half the absolute
close change, also exact.

The fixture has closes stepping `16 -> 32`. With sensitivity 4 the whole
pipeline is exact: the MACD line `[0,0,0,0,4,5,4.75,4.0625]`, the momentum
`[.,0,0,0,16,4,-1,-2.75]` and the explosion line `[.,0,0,0,16,0,0,0]` are
all asserted with `toEqual`. The zone classification is also tested
directly as a pure function (explosive-up / explosive-down / quiet / none,
including the equality boundary). A constant series gives a zero MACD,
zero momentum and zero explosion, so it is wholly quiet.

## API

`data` (a `ChartLineWaePoint[]`), `fastPeriod` (default 20), `slowPeriod`
(default 40), `bbPeriod` (default 20), `bbMult` (default 2), `sensitivity`
(default 150), `pricePanelRatio` / `gap`, sizing/format/a11y props,
`strokeWidth` / `dotRadius`, the colour props, the toggle flags (`showAxis`
/ `showGrid` / `showDots` default false / `showMomentum` / `showExplosion`
/ `showZeroLine` / `showMarkers` / `showTooltip` / `showConfigBadge` /
`showLegend` / `animate`), controlled + uncontrolled `hiddenSeries` (keys
`price` / `momentum` / `explosion`), `formatValue` / `formatX`,
`onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineWaeFinitePoints`, `normalizeLineWaePeriod`,
`computeLineWaeEma`, `computeLineWaeMacd`, `computeLineWaeStdev`,
`computeLineWaeBollWidth`, `classifyLineWaeZone`, `computeLineWae`,
`runLineWae`, `computeLineWaeLayout`, `describeLineWaeChart`, plus the
`DEFAULT_CHART_LINE_WAE_*` constants and the `ChartLineWaeZone` type.
Every exported name carries a `LineWae` infix so the `export *` barrel
line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen `data-section="chart-line-wae-aria-desc"`
description span; the price line, the momentum line, the explosion line
and every dot / marker are `role="graphics-symbol" tabIndex={0}`. Every
node carries a `data-section`; the root mirrors `data-fast-period`,
`data-slow-period`, `data-bb-period`, `data-bb-mult`, `data-sensitivity`,
`data-trend-final`, `data-explosive-up-count`,
`data-explosive-down-count`, `data-quiet-count`, `data-total-points` and
`data-animate`.

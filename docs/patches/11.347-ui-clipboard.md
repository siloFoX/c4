## `copyWithToast`

```ts
copyWithToast(text, {
  label,             // 'commit hash', 'worker name', ...
  showToast,         // (message, tone) => unknown -- typically useToast().showToast
  successMessage?,   // override the default "Copied <label>"
  errorMessage?,     // override the default "Failed to copy <label>"
  onCopy?,           // (text) => void
  onError?,          // (err) => void
}): Promise<{ ok, error, toasted }>
```

Defaults:

- success: `Copied <label>` / `Copied` when no
  label.
- error: `Failed to copy <label>` / `Failed to
  copy`.
- `toasted` flag tells the caller whether the
  pipeline actually fired -- useful for tests
  and for callers that want to confirm the toast
  surface.

## `useCopyShortcut`

```ts
useCopyShortcut({
  value,         // string written on Cmd+C
  enabled?,      // default true; pass false to disable
  label?,        // forwarded to copyWithToast
  showToast?,    // forwarded to copyWithToast
  onCopy?,
  onError?,
});
```

Listener lives on `window`. Skip conditions:

- `enabled === false`.
- The active target is an INPUT / TEXTAREA /
  SELECT / BUTTON / contenteditable surface --
  the browser's native Cmd+C handles those.
- The operator currently has a non-empty text
  selection -- the native copy wins so partial
  range selection still works.
- The modifiers are wrong (Cmd+Shift+C and
  Cmd+Alt+C pass through; only plain Cmd+C /
  Ctrl+C fires).

Refs let the listener stay stable while reading
the latest `value` / `label` / callbacks on each
press. `preventDefault` fires only when the
shortcut is actually about to copy.

## Adoption

### CopyButton

Old: handleClick called
`copyTextToClipboardWithError` and ran its own
if/else over the toast plumbing. New: routes
through `copyWithToast` so the success / failure
message templates live in one file.

### HistoryView

```tsx
const { toast, showToast, dismissToast } = useToast();
useCopyShortcut({
  value: selected ?? '',
  enabled: !!selected,
  label: 'worker name',
  showToast,
});
// ...
{toast ? <Toast message={toast.message} type={toast.type} onDismiss={() => dismissToast()} /> : null}
```

The Toast portal is mounted at the page root so
the toast lands above both the sidebar and the
detail pane regardless of which one has focus.

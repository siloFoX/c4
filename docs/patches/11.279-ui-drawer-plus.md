# 11.279 -- Drawer primitive: top/bottom + motion-safe (v1.11.297)

Component-scope-only enhancement of the existing `Drawer` primitive.
No daemon-side change and no `c4` CLI surface change. The
`left` / `right` behaviour is preserved byte-for-byte; new
variants opt in via the `side` prop.

## Why

The existing Drawer only docked to left or right. Mobile detail
panes and toolbar overlays want top / bottom anchoring (Material
"bottom sheet" pattern). Patch 11.279 adds the missing two sides
and brings the motion contract in line with the rest of the UI
primitives (Dialog, Tooltip, Popover, Toast) by routing the slide
through `useReducedMotion`.

## What changed

### `web/src/components/ui/drawer.tsx`

- `side="top" | "bottom"` added. Each variant docks the panel
  along the matching edge with its own divider border
  (`top` -> `border-b`, `bottom` -> `border-t`).
- New `height?: number | string` prop for top / bottom drawers
  (analogous to `width` for left / right). Defaults to `"50%"`
  when omitted. Numbers coerce to px; strings pass through any
  CSS length. Ignored on `left` / `right` drawers.
- Reduced-motion gate: slide transition + backdrop fade-in only
  apply when `useReducedMotion()` returns false. Reduced-motion
  users get an instant open / close.
- `data-section="drawer"` on the panel +
  `data-section="drawer-backdrop"` on the backdrop +
  `data-reduced-motion="true|false"` on the panel.
- `maxWidth: 100%` / `maxHeight: 100%` clamping so an oversized
  number / length still fits the viewport.

### Adopted

- `web/src/pages/DesignSystem.tsx` -- new `<DrawerDemo>` with four
  trigger buttons (left / right / top / bottom). Top / bottom
  variants pass `height="40%"` to exercise the height-prop
  affordance.

### Deferred

- HistoryView filter drawer: the existing always-visible
  `<aside>` filter sidebar spans ~270 lines of stateful filters.
  A clean mobile-drawer adoption would require duplicating the
  filter-content rendering or refactoring the aside into a
  reusable component first.
- Sessions detail drawer: `SessionsRightPane` already adopts
  `DetailPanel` (a `Drawer` wrapper) on desktop. A bottom-
  anchored mobile drawer is the next iteration.

### Tests

- 13 new vitest cases (side="top" / "bottom" anchor + border,
  height prop as number / string / default, height ignored on
  left / right, data-section="drawer" + "drawer-backdrop"
  selectors, data-reduced-motion baseline, transition-transform
  dropped under reduced motion, backdrop animate-in dropped
  under reduced motion). 29 / 29 (16 original + 13 new) green.
- 99 / 99 across drawer + detail-panel + HelpDrawer +
  DesignSystem suites.

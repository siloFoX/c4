# 11.299 -- AspectRatio primitive (v1.11.317)

New `AspectRatio` primitive at
`web/src/components/ui/aspect-ratio.tsx`. Component-scope only,
no daemon-side change and no `c4` CLI surface change.

## Why

Two pain points across the recent media surfaces:

1. Health metric chart containers had no enforced ratio --
   chart libraries (Recharts, etc.) inherit their size from the
   parent. Without a ratio guarantee the chart space appeared
   only after the first render, producing visible Cumulative
   Layout Shift (CLS) as the page settled.
2. Snapshots image previews loaded an `<img>` with no
   placeholder reservation. The bitmap landing pushed the
   surrounding rows down, breaking the operator's reading
   position.

Patch 11.299 ships the canonical aspect-ratio primitive so any
media surface can opt into a CLS-safe layout with one prop.

## What changed

### `web/src/components/ui/aspect-ratio.tsx` (new file)

- `AspectRatio` forwardRef component + new exported types
  `AspectRatioPreset = '16:9' | '9:16' | '4:3' | '3:4' | '21:9' | '1:1'`.
- Props:
  - `ratio?: AspectRatioPreset | number` (default `'16:9'`).
    Preset string OR a numeric value (`width / height`).
    Invalid numeric (NaN, negative, zero) falls back to the
    default.
  - `children: ReactNode` -- single child positioned
    absolutely inside the ratio box.
  - All standard `HTMLDivElement` attrs pass through.
- Uses the modern CSS `aspect-ratio` property (the wrapper
  itself stays in the document flow with its computed height
  derived from the available width). The content slot is
  `absolute inset-0` so chart / image / iframe children fill
  the box.
- Data-attribute selectors:
  - `data-section="aspect-ratio"` on the wrapper.
  - `data-section="aspect-ratio-content"` on the content
    slot.
  - `data-ratio="<preset|number>"` carrying the raw input
    so e2e tests can target a specific ratio without
    parsing the inline style.
- Re-exported from `web/src/components/ui/index.ts`.

### Adopted

- Primitive only this commit. The dispatch sites (Health
  metric chart containers + Snapshots image previews)
  inherit the new primitive; threading
  `<AspectRatio ratio="16:9">` (or `"4:3"` for thumbnails)
  into the per-page mount points is a follow-up.

## Notes

- Modern browsers (Chromium 88+, Firefox 89+, Safari 15+)
  support `aspect-ratio` natively; the c4 web bundle does
  not need a polyfill.
- The absolute-positioned content slot is what lets a
  chart library size itself against the wrapper without
  re-implementing the CLS-safe contract -- the consumer
  just renders into the slot and inherits the reserved
  space.
- The `data-ratio` attribute carries the raw input value
  (`"16:9"`, `"3"`, etc) rather than the resolved CSS
  string so e2e tests can target a known preset without
  guessing the normalisation rules.

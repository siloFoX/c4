## Summary

`web/src/components/ui/chart-line-bayesian.tsx` ships
`<ChartLineBayesian>`, a pure-SVG two-panel chart with a **Bayesian
Trend** probability panel computed from the posterior over three
trend hypotheses given the rolling-window of observed returns. The
top panel is the price; the bottom panel carries three probability
curves in a fixed `[0, 1]` band with a dashed midline at `0.5`.

## Computation

Three hypotheses about the rolling-window return generator are kept:

```
H_up   : returns drawn from N(+delta, sigma^2)
H_flat : returns drawn from N(    0, sigma^2)
H_down : returns drawn from N(-delta, sigma^2)
```

For each bar the lookback of `window` returns ending at the bar is
formed, the log-likelihood of each hypothesis is summed, and the
posterior probabilities `(pUp, pFlat, pDown)` are obtained by a
softmax over the three log-likelihoods with equal priors. The
normal-density constants cancel in the softmax so the implementation
keeps only the `-0.5 * ((r - mu) / sigma)^2` term:

```
l_h = sum over r in window of -0.5 * ((r - mu_h) / sigma)^2
m   = max(l_up, l_flat, l_down)
p_h = exp(l_h - m) / sum_h' exp(l_h' - m)
```

The first `window` bars are null (the lookback has not yet filled).
After that, the classifier marks each bar `up` if `pUp` leads,
`down` if `pDown` leads, otherwise `flat` (ties favour flat).

## Worked anchors

The exp in the softmax is transcendental, so the posterior cannot in
general be bit-exact. The implementation has two exact invariants:

- **Simplex invariant** -- `pUp + pFlat + pDown = 1` within ULP
  (`toBeCloseTo(1, 12)`) at every defined bar. The softmax denominator
  may introduce ULP noise on the re-summed probabilities, but the
  underlying division `s/s = 1.0` is exact in IEEE 754; only the
  re-addition `p_up + p_flat + p_down` after dividing each numerator
  can drift by an ULP.
- **Symmetry anchor** -- for a constant series, every return is zero;
  `l_up = sum of -0.5 * ((0 - delta) / sigma)^2` and `l_down = sum of
  -0.5 * ((0 + delta) / sigma)^2`. The squared term is the same bit
  pattern for both signs, so `l_up === l_down` bit-exact, and
  therefore `pUp === pDown` bit-exact. Asserted with `toBe`.
- **Mirror anchor** -- a window of returns `[r1, r2, ..., rn]` and
  its sign-flipped mirror `[-r1, -r2, ..., -rn]` swap the up and
  down log-likelihoods bit-by-bit while leaving the flat one alone;
  the implementation returns `a.pUp === b.pDown`, `a.pDown === b.pUp`,
  `a.pFlat === b.pFlat`. Asserted with `toBe`.
- **Concentration property** -- with all returns equal to `+delta`,
  a longer lookback adds more identical positive log-likelihood
  contributions per bar, concentrating the posterior more on `H_up`.
  Asserted as `posterior(window=10).pUp > posterior(window=3).pUp`.
- **Ordering anchors** -- RISING `[10..21]` makes every return
  `+1 = +delta`, so `pUp > pFlat > pDown` at every defined bar.
  FALLING `[21..10]` mirrors to `pDown > pFlat > pUp`. CONST_FLAT
  `[5 x 12]` makes every return `0`, so `pFlat > pUp = pDown` at
  every defined bar.

## Distinct from neighbouring primitives

- vs `<ChartLineRsi>` -- the RSI is a fixed `100 -
  100 / (1 + sumUp / sumDown)` over the bar-to-bar price changes;
  the Bayesian Trend is a probabilistic classifier over three
  parametrised hypotheses with a tunable drift and noise.
- vs `<ChartLineAroonOsc>` -- the Aroon Osc measures the recency of
  the highest high vs the lowest low (a TIME-only signal); the
  Bayesian Trend measures the LIKELIHOOD-weighted evidence over the
  observed returns (an AMPLITUDE-aware signal).
- vs `<ChartLineErgodic>` -- the Ergodic Oscillator is a triple-
  smoothed momentum ratio (deterministic + frequentist); the
  Bayesian Trend is a posterior over a generative model
  (probabilistic + Bayesian).
- vs `<ChartLineCmo>` -- the CMO is `100 * (sumUp - sumDown) /
  (sumUp + sumDown)`, a signed unsigned-share; the Bayesian Trend
  produces three calibrated probabilities that sum to one.

## API

`data` -- a single `ChartLineBayesianPoint[]` (`{x, value}`).
`window` (default 10, coerced to an integer >= 2). `delta` (default
0.01, coerced to a strictly positive finite). `sigma` (default 0.02,
coerced to a strictly positive finite). Standard sizing / colour /
a11y props plus `pricePanelRatio` (default 0.55, clamped to
`[0.3, 0.8]`), `gap`, `strokeWidth`, `dotRadius`, `tickCount`.
Toggle flags: `showAxis`, `showGrid`, `showDots` (default false),
`showProb`, `showMidline`, `showMarkers`, `showTooltip`,
`showConfigBadge`, `showLegend`, `animate`. Controlled and
uncontrolled visibility via `hiddenSeries`
(`'price' | 'pUp' | 'pFlat' | 'pDown'`). Callbacks:
`onPointClick({point})`, `onSeriesToggle({seriesId, hidden})`.
Format functions: `formatValue`, `formatX`. ARIA: `ariaLabel`,
`ariaDescription`.

## Pure helpers

- `DEFAULT_CHART_LINE_BAYESIAN_*` -- defaults.
- `CHART_LINE_BAYESIAN_MIDLINE` -- 0.5 (the probability midline).
- `getLineBayesianFinitePoints` -- keep only points with a finite x
  and value.
- `normalizeLineBayesianWindow(window, fallback)` -- integer >= 2;
  else fallback.
- `normalizeLineBayesianDelta(delta, fallback)` -- strictly positive
  finite; else fallback.
- `normalizeLineBayesianSigma(sigma, fallback)` -- strictly positive
  finite; else fallback.
- `computeLineBayesianReturns(values)` -- bar-to-bar returns with
  null at the first bar and at any bar pointing at a non-finite
  value.
- `computeLineBayesianPosteriorWindow(returns, delta, sigma)` -- the
  posterior over the three hypotheses for a single window.
- `computeLineBayesianPosterior(values, window, delta, sigma)` --
  the per-bar rolling posterior. The first `window` bars are null.
- `classifyLineBayesianZone(posterior)` --
  `'up' | 'flat' | 'down' | 'none'`; ties favour flat over up over
  down.
- `runLineBayesian(points, options?)` --
  `{series, window, delta, sigma, posterior, samples, pUpFinal,
  pFlatFinal, pDownFinal, upCount, flatCount, downCount, ok}`.
- `computeLineBayesianLayout({data, window, delta, sigma, ...})` --
  two panels with the price line on top, the three probability
  curves in a `[0, 1]` band below, one marker per defined-posterior
  bar coloured by the argmax zone, and `midlineY` for the 0.5 line.
- `describeLineBayesianChart(data, options?)` -- ARIA summary
  mentioning the indicator, the lookback window, the three
  hypotheses and the final posterior.

## a11y / data hooks

Root `<div role="region">` with `aria-label` + `aria-describedby`; the
SVG is `role="img"` with the same label; an sr-only
`data-section="chart-line-bayesian-aria-desc"` span carries the
description. The price line, every price dot, every probability line
and every marker carry `role="graphics-symbol"` and `tabIndex={0}`.
Every node carries a `data-section`. Root mirrors `data-window`,
`data-delta`, `data-sigma`, `data-p-up-final`, `data-p-flat-final`,
`data-p-down-final`, `data-up-count`, `data-flat-count`,
`data-down-count`, `data-total-points`, `data-animate`,
`data-empty`. Grid / axis / tick / panel-label nodes carry
`data-panel="price"|"prob"`; markers expose `data-zone`, `data-p-up`,
`data-p-flat`, `data-p-down`.

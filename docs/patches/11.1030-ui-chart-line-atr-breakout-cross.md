# 11.1030 -- UI: chart-line-atr-breakout-cross primitive (v1.11.1048)

## Scope

Add a pure-SVG dual-panel `ChartLineAtrBreakoutCross`
component that renders the close in the top panel and the
Average True Range (ATR), its long-window SMA baseline, and
a derived breakout threshold line (`baseline * multiplier`)
in the bottom panel, marking ATR crosses up through
threshold (volatility expansion breakout -- range expansion
entry trigger) / ATR crosses down through threshold
(volatility compressing back to baseline) events with bias
coloring derived from the ATR slope at the trigger bar.

## ATR breakout pipeline

```
TR[i]       = max(high[i] - low[i],
                  |high[i] - close[i-1]|,
                  |low[i]  - close[i-1]|)
ATR[i]      = SMA(TR, period)        ; i >= period
baseline[i] = SMA(ATR, baselineLen)  ; i >= period +
                                       baselineLen - 1
threshold[i] = baseline[i] * multiplier
regime    = 'compressed' when ATR < baseline
            'neutral'    when baseline <= ATR < threshold
            'expanded'   when ATR >= threshold
            'none'       when any input is null
bias      = ATR[i] vs ATR[i-1] -> up / down / flat / none
bullish   : prev ATR <= prev threshold AND
            cur ATR > cur threshold
            (volatility expansion breakout)
bearish   : prev ATR >= prev threshold AND
            cur ATR < cur threshold
            (volatility compressing back to baseline)
```

ATR uses a simple SMA (not the original Wilder running
smoothing) so the steady-state value of constant TR exactly
equals TR -- no convergence transient. This matches the
trader-friendly modern ATR variant used by pyalgotrade,
tulip-indicators, and ta-lib's `ATR_SMA` mode, and lets the
bit-exact CONST/LINEAR anchors hit clean integer ATR values
from the very first valid bar at `i = period`.

Warmup is `period + baselineLength - 1 = 33` for the default
tuning: TR seeds at `i >= 1`, ATR fills the SMA window at `i
= period = 14`, then baseline = SMA(ATR, baselineLength)
needs another `baselineLength - 1 = 19` bars.

## Bit-exact anchors

All anchors use HLC input.

- **CONST band** `high = K + 1`, `low = K - 1`, `close = K`:
  range = 2, |high-prevClose| = 1, |low-prevClose| = 1 ->
  TR = max(2, 1, 1) = 2 from `i >= 1` -> ATR = 2 from `i =
  period` -> baseline = 2, threshold = 3 from `i = period +
  baselineLength - 1`. ATR === baseline -> the `>=`
  boundary settles into the `neutral` band. ATR < threshold
  -> never `expanded`. 0 crosses. Verified across K in
  {0, 1, 50, 200, 1234}.
- **WIDE CONST band** `high = K + 2`, `low = K - 2`,
  `close = K`: range = 4, |high-prevClose| = 2,
  |low-prevClose| = 2 -> TR = max(4, 2, 2) = 4. ATR = 4,
  baseline = 4, threshold = 6. Same regime as the narrow
  band but at a 2x ATR magnitude, exercising the bit-exact
  ATR / baseline / threshold pipeline at a second
  magnitude. Validates the `threshold === baseline *
  multiplier` derived-channel invariant.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close = i`:
  range = 2, |high-prevClose| = |i+1 - (i-1)| = 2,
  |low-prevClose| = |i-1 - (i-1)| = 0 -> TR = max(2, 2, 0)
  = 2. Same as CONST.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`,
  `close = -i`: range = 2, |high-prevClose| = 0,
  |low-prevClose| = 2 -> TR = max(2, 0, 2) = 2. Mirror of
  LINEAR UP.

Note that LINEAR inputs do not increase TR -- the per-bar
range stays constant because the bands shift in lockstep
with the close. Real volatility expansion would need
inputs where the band *widens* over time. The bit-exact
anchors here verify the bottom-of-the-range steady state;
the detector function tests directly verify breakout
detection with synthetic `(atr, threshold)` arrays.

  matching the chart-line-adx-trend-cross sibling.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 3-zone regime classifier (`compressed` / `neutral` /
  `expanded`) follows the chart-line-adx-trend-cross
  convention (semantic zone names rather than
  bullish/bearish), since ATR is a non-directional
  volatility indicator. The `bullish` / `bearish` cross
  *kinds* still follow the sibling convention (line
  crossing up / down through threshold).
- Uses simple SMA for both ATR and baseline -- no Wilder
  smoothing transient, so the bit-exact CONST/LINEAR
  anchors hit clean integer values from the first valid
  bar.
- Multiplier validates as `> 0` (zero would collapse
  threshold = baseline, breaking the breakout semantics).

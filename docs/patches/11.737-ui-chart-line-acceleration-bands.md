## Summary

`web/src/components/ui/chart-line-acceleration-bands.tsx` ships
`<ChartLineAccelerationBands>`, a pure-SVG single-panel chart with
Price Headley's **Acceleration Bands** envelope overlays from a
simple moving average scaled by the high-low range factor. The close
line shares one panel with three step-function level lines (upper /
middle / lower); each defined bar carries the three bands as
bar-wide horizontal stub segments.

## Computation

Per bar the raw upper / lower factors scale the high and the low by
the relative range:

```
ratio       = (high - low) / (high + low)
upperFactor = high * (1 + 2 * ratio)
lowerFactor = low  * (1 - 2 * ratio)
```

The bands are then simple moving averages of length `period`:

```
upper  = SMA(upperFactor, period)
middle = SMA(close,       period)
lower  = SMA(lowerFactor, period)
```

The first `period - 1` bars are null on every band. A degenerate
bar with `H + L <= 0` or non-finite OHLC yields a null factor that
null-ifies the SMA until a full window of valid factors fills.

## Worked anchors

The factor formula is integer-rational when the ratio is a dyadic
fraction. The implementation has three exact-style anchors.

- **Constant-series anchor** -- `H = L = C = 5`. The ratio is
  `0 / 10 = 0`; the upper and lower factors collapse to `H = L = 5`;
  the SMA of the constant 5 is 5. Every band reads 5 bit-exact.
- **Worked anchor A** -- `H = 10, L = 6, C = 8` per bar.
  `ratio = 4 / 16 = 0.25` (dyadic, exact in IEEE 754).
  `upperFactor = 10 * 1.5 = 15`, `lowerFactor = 6 * 0.5 = 3`,
  `middle = SMA(close) = 8`. All bit-exact integer.
- **Worked anchor B** -- `H = 12, L = 4, C = 8` per bar.
  `ratio = 8 / 16 = 0.5`. `upperFactor = 12 * 2 = 24`,
  `lowerFactor = 4 * 0 = 0`, `middle = 8`. All bit-exact integer.

The `upper >= middle >= lower` order on a varied wave is tested
directly.

## Distinct from neighbouring primitives

- vs `<ChartLineBollinger>` -- Bollinger Bands offset the middle
  SMA by a standard-deviation multiple; Acceleration Bands offset
  the bands by a HIGH-LOW range factor scaled per bar. Headley's
  bands widen on volatile bars without a rolling std-dev pass.
- vs `<ChartLineKeltner>` -- Keltner Channels offset an EMA by an
  ATR multiple; Acceleration Bands use a per-bar high-low ratio
  factor and a flat SMA backbone.
- vs `<ChartLinePivotClassic>` -- pivot bands project from a
  SINGLE PRIOR bar's HLC; Acceleration Bands project from a
  ROLLING SMA of factors over `period`.

## API

`data` -- a single `ChartLineAccelerationBandsPoint[]`
(`{x, high, low, close}`). `period` (default 20, integer >= 1).
Standard sizing / colour / a11y props plus `strokeWidth`,
`dotRadius`, `tickCount`. Toggle flags: `showAxis`, `showGrid`,
`showDots` (default false), `showUpper`, `showMiddle`, `showLower`,
`showMarkers`, `showTooltip`, `showConfigBadge`, `showLegend`,
`animate`. Controlled and uncontrolled visibility via `hiddenSeries`
(`'price' | 'upper' | 'middle' | 'lower'`). Callbacks:
`onPointClick({point})`, `onSeriesToggle({seriesId, hidden})`.
Format functions: `formatValue`, `formatX`. ARIA: `ariaLabel`,
`ariaDescription`.

## Pure helpers

- `DEFAULT_CHART_LINE_ACCELERATION_BANDS_*` -- defaults.
- `getLineAccelerationBandsFinitePoints` -- keep only points with
  finite OHLC and `high >= low`.
- `normalizeLineAccelerationBandsPeriod(period, fallback)` --
  integer >= 1; else fallback.
- `computeLineAccelerationBandsFactors(bar)` --
  `{upperFactor, lowerFactor}`; null on null bar / non-finite OHLC
  / `H + L <= 0`. Zero range collapses both factors to `H = L`.
- `computeLineAccelerationBandsSma(values, period)` -- SMA of a
  nullable series; warm-up null; null inside the window null-ifies
  the bar.
- `computeLineAccelerationBands(bars, period)` -- per-bar
  `{upper, middle, lower}`; warm-up null.
- `classifyLineAccelerationBandsZone(close, levels)` --
  `'above-upper' | 'middle-to-upper' | 'lower-to-middle' |
  'below-lower' | 'none'`.
- `runLineAccelerationBands(points, options?)` --
  `{series, period, levels, samples, middleFinal, aboveCount,
  betweenCount, belowCount, ok}`.
- `computeLineAccelerationBandsLayout({data, period, ...})` --
  single panel with the close line, three level stub segments per
  defined bar, one marker per defined-zone bar coloured by zone
  band.
- `describeLineAccelerationBandsChart(data, options?)` -- ARIA
  summary mentioning the band formulas, the period, the warm-up
  and the zone counts.

## a11y / data hooks

Root `<div role="region">` with `aria-label` + `aria-describedby`; the
SVG is `role="img"` with the same label; an sr-only
`data-section="chart-line-acceleration-bands-aria-desc"` span
carries the description. The close line, every price dot and every
marker carry `role="graphics-symbol"` and `tabIndex={0}`. Every node
carries a `data-section`. Root mirrors `data-period`,
`data-middle-final`, `data-above-count`, `data-between-count`,
`data-below-count`, `data-total-points`, `data-animate`,
`data-empty`. Segments expose `data-series-id` and `data-value`;
markers expose `data-zone` and `data-close`.

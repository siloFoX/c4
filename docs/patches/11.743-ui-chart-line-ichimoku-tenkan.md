# Patch 11.743 -- UI: chart-line-ichimoku-tenkan primitive

Released as `v1.11.761` on 2026-05-26.

## Scope

Add `<ChartLineIchimokuTenkan>` -- a pure-SVG single-panel chart
with an Ichimoku Kinko Hyo **Tenkan-sen** conversion-line overlay
drawn from the midpoint of the highest high and the lowest low
across the lookback.

## Definition

Inputs:

- `data: ChartLineIchimokuTenkanPoint[]` where
  `ChartLineIchimokuTenkanPoint = { x; high; low; close }`. Bars
  with `high < low` are dropped.
- `period: number` -- the conversion-line lookback. Default `9`.
  Normalized to an integer `>= 1`.

Algorithm (per bar `i >= period - 1`):

```
HH        = max(high over [i - period + 1, i])
LL        = min(low  over [i - period + 1, i])
tenkan[i] = (HH + LL) / 2
```

The first `period - 1` bars are null. A non-finite high or low
anywhere in the window also nulls the bar.

## Bit-exact algebraic anchors

**CONST_FLAT** (`high == low == K`): `HH == LL == K`, so
`tenkan = (K + K) / 2 = K` bit-exact.

**RISING integer ramp** (`high == low == i + 10`) period 4: window
`[i - 3, i]` carries the consecutive values `i + 7, i + 8, i + 9,
i + 10`. So `HH = i + 10`, `LL = i + 7`, and

```
tenkan = (HH + LL) / 2 = ((i + 10) + (i + 7)) / 2
       = (2 * i + 17) / 2 = i + 8.5
```

bit-exact dyadic (integer sum divided by 2, no rounding loss for
finite IEEE 754 doubles).

**FALLING integer ramp** (`high == low == 19 - i`) period 4:

```
HH = 19 - (i - 3) = 22 - i
LL = 19 - i
tenkan = ((22 - i) + (19 - i)) / 2 = (41 - 2i) / 2 = 20.5 - i
```

bit-exact dyadic.

**Period 2** on RISING: window `[i - 1, i]` carries `i + 9, i +
10`, so `tenkan = (2i + 19) / 2 = i + 9.5` bit-exact dyadic.

**Period 1**: window collapses to the bar itself. `tenkan =
(high[i] + low[i]) / 2`, which on the RISING fixture is `i + 10`
bit-exact integer (high == low).

**Translation invariance**: shifting every high / low / close by an
integer `k` shifts `HH` and `LL` by exactly `k`, so the midpoint
shifts by exactly `k` bit-exact.

## Component

`forwardRef<HTMLDivElement, ChartLineIchimokuTenkanProps>` with
displayName. Renders:

- A region-role outer `<div data-section="chart-line-ichimoku-tenkan">`.
- An `<svg role="img">` with an `aria-describedby` description.
- A close `<path>`, optional close dots, a Tenkan-sen `<path>`,
  and one marker per defined bar at the close position with
  `data-zone = 'above-tenkan' | 'at-tenkan' | 'below-tenkan'`.
- Legend toggle for `price` / `tenkan`, plus a `showTenkan` prop
  and an `onPointClick` handler.
- ARS design-system tokens for colour and motion (`motion-safe:
  animate-fade-in`).

## Acceptance

- CONST_FLAT identity: `tenkan = K` bit-exact at every defined
  bar.
- RISING ramp period 4: `tenkan = i + 8.5` bit-exact dyadic.
- FALLING ramp period 4: `tenkan = 20.5 - i` bit-exact dyadic.
- Period 2 RISING: `tenkan = i + 9.5` bit-exact dyadic.
- Period 1: `tenkan = high[i]` bit-exact on the RISING fixture.
- Translation invariance: `+1000` shifts tenkan by exactly 1000.
- Non-finite high/low in the window nulls the bar.
- Layout / component / ARIA / data-section hooks all covered.

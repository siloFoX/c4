# Patch 11.183 - UI: Grid Debug Overlay (v1.11.201)

Dev-only visual debug overlay that draws a 12-column grid aligned to the
app's `max-w-7xl mx-auto` container, plus a Tailwind breakpoint pill and
a viewport/scroll pill. Useful for verifying that page layouts snap to
the design grid and for sanity-checking responsive behaviour while
working in `web/`.

## Files

- `web/src/components/dev/GridDebugOverlay.tsx` - component (about 130
  lines).
- `web/src/components/dev/GridDebugOverlay.test.tsx` - vitest + RTL
  tests (7 cases).
- `web/src/App.tsx` - mounts `<GridDebugOverlay />` once, below the
  `<AutonomousStatusBanner />`. The component self-gates so production
  builds render `null`.

## Usage

1. Run the dev server (`npm --prefix web run dev`).
2. Press `Cmd+Shift+G` on Mac or `Ctrl+Shift+G` elsewhere to toggle the
   overlay. Visibility persists across reloads via
   `localStorage['c4:grid-debug:visible']`.
3. The overlay sits behind modals (`z-[60]`) but above page content,
   uses `pointer-events-none` so it never blocks clicks, and is marked
   `aria-hidden='true'` (decorative).

The breakpoint pill in the top-right corner shows the active Tailwind
breakpoint based on `window.innerWidth`:

| width  | label |
|--------|-------|
| < 640  | xs    |
| >= 640 | sm    |
| >= 768 | md    |
| >= 1024| lg    |
| >= 1280| xl    |
| >= 1536| 2xl   |

The pill in the bottom-left corner shows the current viewport
dimensions and scroll position as `<W>x<H> @ y=<scrollY>`.

## Production gating

The component checks `import.meta.env.PROD` first; if that is `true`
it returns `null` immediately. As a fallback for non-Vite environments
it also checks `process.env.NODE_ENV !== 'development'`. The keydown
listener is only attached when the component is in dev mode, so a
production bundle never installs a global hotkey.

## Why a separate `dev/` folder?

Other dev tools (storybook-style sandboxes, debug helpers) can land
alongside this overlay without polluting `components/ui/` or any
feature folder. The folder name doubles as the natural lint filter
for stripping dev-only modules from bundle analysis when needed.

## Tests

```
env -C /root/c4-worktree-auto-w167/web /root/c4/web/node_modules/.bin/vitest run src/components/dev/GridDebugOverlay.test.tsx
```

Cases:

1. `PROD=true` -> renders `null`.
2. Dev + `localStorage='1'` -> overlay present with all 12 columns,
   `aria-hidden='true'`, `pointer-events-none`.
3. Empty localStorage starts hidden; `Cmd+Shift+G` toggles on and off
   and writes the persisted flag.
4. `Ctrl+Shift+G` also toggles (non-Mac path).
5. Visibility persists across remount via localStorage.
6. Breakpoint label changes with `window.innerWidth` (xs / md / 2xl).
7. Viewport pill includes `WxH`.

# 11.196 -- UI: print stylesheet (v1.11.214)

A component-scoped `@media print` stylesheet for the web UI plus the
class hooks needed for the top-level chrome to opt out of print.

## Files

- `web/src/styles/print.css` (new, ~150 lines). Single `@media print`
  rule plus a top-level `.print-only { display: none }` default so the
  utility class stays hidden on screen.
- `web/src/main.tsx`. Imports `./styles/print.css` immediately after
  `./index.css` so the cascade lands after Tailwind base.
- `web/src/components/layout/AppHeader.tsx`. Outer `<Navbar>` adopts
  `data-print-hide` and the `no-print` className. Spreads via the
  existing `...rest` pass-through on `Navbar`.
- `web/src/components/layout/Sidebar.tsx`. The root `<aside>` adopts
  `data-print-hide`.
- `web/src/components/Toast.tsx`. The portal-rendered toast root adopts
  `data-print-hide` and merges the `no-print` className into the
  existing `pointer-events-auto select-none` list.
- `web/src/styles/print.test.ts` (new, ~80 lines). vitest cases that
  read `print.css` as a raw string and assert the contract.

## Print rules

- Hides nav, sidebar, toast, command palette, drawer, tooltip, popover,
  and any node opting in via `[data-print-hide]` or `.no-print`.
- Forces white background and black text on `html`, `body`, `main`,
  `#root`, `.app-main`, `.page-frame`.
- Bumps base typography to 12pt / line-height 1.4 / Georgia serif and
  drops link decoration to black underline.
- Appends `(href)` to non-hash, non-empty links via
  `a[href]:not([href^="#"]):not([href=""])::after` with `content: " ("
  attr(href) ")"`. Skips `javascript:` and `mailto:`.
- Marks `section`, `.panel`, `.card`, `figure`, `pre`, `blockquote`,
  `table`, `tr`, `img` with `break-inside: avoid`; `h1`-`h6` with
  `break-after: avoid`.
- Promotes `thead` to `table-header-group` (and `tfoot` to
  `table-footer-group`) for repeating table headers across pages.
- Sets `force-color-adjust: exact` on `.badge` / `.chip` / `.keep-color`
  while everything else inherits the default economy/grayscale path.
- Exposes utility classes `.print-only` (block in print, hidden on
  screen), `.screen-only` (hidden in print), `.print-page-break-before`
  / `.print-page-break-after`, and `.no-print`.
- `@page { margin: 1.5cm }`.

## Tests

8 vitest cases in `web/src/styles/print.test.ts` read `print.css` via
`fs.readFileSync` and assert: `@media print` declared; nav / sidebar /
toast hidden via `display: none`; `.print-only` and `.screen-only`
utility classes; page-break rules (`break-inside: avoid`,
`break-after: avoid`, `.print-page-break-before`, `break-before: page`);
serif 12pt typography; `thead { display: table-header-group }`; link
href appended via `::after` with `attr(href)`; white background and
black text on body.

The test suite was not executed inside the worktree because
`web/node_modules` is not provisioned in this build environment. The
canonical command is documented below and works from the main repo
copy.

```bash
env -C /root/c4-worktree-auto-w180/web \
  /root/c4/web/node_modules/.bin/vitest run src/styles/print.test.ts
```

## Out of scope

- No global typography rebrand. Print typography is gated under
  `@media print` only; the screen cascade is untouched.
- No paginated dashboard. Page-level layouts (e.g. `PageFrame`,
  `HistoryView`, `Sessions`) keep their existing wrappers; the rules
  match the generic `section / .panel / .card` selectors and
  break-inside avoidance already implied by those wrappers.
- No `docs/autonomous-queue-v10.md` touch -- the queue is owned by the
  manager.

## API

```ts
type StatusIndicatorKind =

interface StatusIndicatorProps {
```

## Behaviour

### Kinds

Six canonical colour tokens, each
mapping to a dot bg / ping / label
text class set:

| kind | colour |
| --- | --- |
| success | emerald |
| warning | amber |
| error | rose |
| info | sky |
| neutral | zinc / muted-foreground |
| pending | violet |

The full map is exported as
`STATUS_INDICATOR_KIND_CLASS` so
adopters can reuse the same tokens
in custom layouts.

### Size

| size | dot dim | label text | gap |
| --- | --- | --- | --- |
| sm | h-1.5 w-1.5 | text-[11px] | gap-1 |
| md (default) | h-2 w-2 | text-xs | gap-1.5 |
| lg | h-2.5 w-2.5 | text-sm | gap-2 |

### Pulse animation

| `pulse` | effect |
| --- | --- |
| `false` (default) | no ping overlay |
| `true` | ping overlay 60% opacity, `motion-safe:animate-ping` |
| `'subtle'` | ping overlay 30% opacity, `motion-safe:animate-ping` |

`prefers-reduced-motion: reduce`
drops the ping animation but keeps
the static colour cue (no flicker
for sensitive users).

### Label slot

When supplied, renders next to the
dot with the kind's text colour
token. When omitted, only the dot
renders.

### Decorative mode

Default the indicator carries
`role="status"` +
`aria-live="polite"` +
`aria-atomic="true"` so screen
readers announce kind transitions
without operator focus.

`decorative={true}` drops all
three (no role, no aria-live, no
aria-label). Use when the
indicator sits next to a text
node that already announces the
state.

### hideDot

When `true`, the dot wrapper is
omitted entirely -- only the
label renders. Useful when the
colour is redundant but the
kind semantics still matter
(e.g., in a dense table cell).

### ARIA label resolution

```ts
export function getStatusIndicatorAriaLabel(
  ariaLabel: string | undefined,
  kind: StatusIndicatorKind,
  label: ReactNode,
): string | undefined;
```

| priority | source |
| --- | --- |
| 1 | explicit `ariaLabel` prop |
| 2 | string `label` (non-empty after trim) |
| 3 | fallback `"Status: <kind>"` |

Exported so adopters can mirror
the same fallback chain in
custom hosts.

## Pairs with existing primitives

- `<Badge>` -- discrete chip
  variant for labelled state.
  StatusIndicator is the
  dot-only / dot-plus-label
  variant for inline
  affordances.
- `<Toast>` (11.280) --
  full-screen polite live
  region. StatusIndicator is
  the inline equivalent.
- `<Spinner>` --
  indeterminate progress.
  StatusIndicator is the
  discrete state variant.
- ThemeCustomizer (11.394) --
  the indicator reads
  Tailwind colour classes
  directly so it auto-themes.

## Out of scope (deferred)

- Auto-clear after a timeout
  (flash "saved" for 3s then
  revert). Host owns the
  timer.
- Multi-state animation
  (flipping between two kinds
  with a transition). Snap to
  the new kind for v1; CSS
  transitions belong in a
  follow-on motion layer.
- Tooltip slot on hover.
  Host wraps with the
  existing `<Tooltip>`
  primitive.
- Per-kind icon glyph (check,
  x, exclamation). Adopters
  who want an icon can wrap
  with the existing icon
  primitives; a built-in icon
  slot is a follow-on.
- Adoption sweep through
  existing status-line
  affordances. Existing
  inline `<span class="bg-
  emerald-500 ...">` dots
  ship until a migration
  dispatch.

## Summary

Add a new `<ChartBullet>` UI primitive in
`web/src/components/ui/chart-bullet.tsx` -- pure-SVG bullet
chart in the Stephen Few sense. Visualises a single primary
measure against a target plus a stack of qualitative range
bands (e.g., low / medium / high). Horizontal or vertical
orientation. Carries `role="progressbar"` so assistive tech
describes the measure as a percentage of the chart max.
Numeric axis ticks render under (or beside) the track for
scale context. Optional label + sub-label header echoes
the current value alongside the target.

## API

```ts
interface ChartBulletRange {
  max: number;          // upper bound of this band
  label?: string;
  color?: string;
}

interface ChartBulletProps {
  value: number;                   // actual measure
  target?: number;                 // comparison marker
  ranges?: readonly ChartBulletRange[];
  max?: number;                    // chart max override
  label?: string;
  subLabel?: string;
  width?: number;                  // default 360
  height?: number;                 // default 48
  orientation?: 'horizontal' | 'vertical';
  measureColor?: string;           // default '#1f2937'
  targetColor?: string;            // default '#ef4444'
  showAxisTicks?: boolean;         // default true
  tickCount?: number;              // default 5
  formatValue?: (v: number) => string;
  ariaLabel?: string;
  className?: string;
  measureThickness?: number;       // 0..1 of bar height (default 0.4)
  animate?: boolean;               // default true
}
```

## Behaviour

- Renders a `<div role="progressbar">` with the standard
  ARIA progressbar fields populated from `value` / `max`:
  `aria-valuenow`, `aria-valuemin=0`, `aria-valuemax=max`,
  and an `aria-valuetext` that reads
  `"<formattedValue> of <formattedTarget> target"` (falls
  back to just `<formattedValue>` when `target` is
  undefined).
- The SVG carries `aria-hidden="true"` since the root
  region already owns the accessible semantics.
- Qualitative ranges paint largest-to-smallest so smaller
  bands layer on top of larger bands. Missing `ranges` -> 3
  default tiers (Low / Medium / High at 33% / 66% / 100% of
  the chart max). Custom ranges keep their `label` /
  `color` and fall back to the default neutral palette
  when a colour is missing.
- The measure bar is drawn at `measureThickness` (0..1) of
  the bar height, vertically centred. Horizontal: bar
  grows left -> right. Vertical: bar grows bottom -> top.
- The target marker draws perpendicular to the measure
  bar: a thick vertical line on the horizontal layout, a
  thick horizontal line on the vertical layout. Hidden
  when `target` is undefined or non-finite.
- Tick labels: `tickCount` evenly-spaced labels under the
  horizontal track (or to the right of the vertical
  track). `showAxisTicks=false` removes them. Ticks honour
  `formatValue` so percentages, currency, or any custom
  format renders consistently across the chart.
- Optional header: when either `label` or `subLabel` is
  supplied, the primitive renders a small flex row with
  the label (and optional sub-label below it) on the left
  and the formatted value (plus `/ <target>`) on the
  right. Missing labels collapse the header entirely.
- `motion-safe:animate-fade-in` is applied to the SVG so
  the chart mounts with a fade respecting
  `prefers-reduced-motion`. `animate=false` removes the
  class.

## Pure helpers (exported)

```ts
DEFAULT_CHART_BULLET_WIDTH              = 360
DEFAULT_CHART_BULLET_HEIGHT             = 48
DEFAULT_CHART_BULLET_MEASURE_COLOR      = '#1f2937'
DEFAULT_CHART_BULLET_TARGET_COLOR       = '#ef4444'
DEFAULT_CHART_BULLET_TICK_COUNT         = 5
DEFAULT_CHART_BULLET_MEASURE_THICKNESS  = 0.4
DEFAULT_CHART_BULLET_RANGE_COLORS = [
  '#e5e7eb', '#cbd5e1', '#94a3b8'
]

getBulletMax(value, target?, ranges?, override?): number
getDefaultBulletRanges(max): ChartBulletRange[]
getBulletRatio(value, max): number
getBulletTicks(max, count?): number[]
sortBulletRangesAscending(ranges): ChartBulletRange[]
```

- `getBulletMax` honours `override` first; otherwise picks
  the largest of `value`, `target`, and any range max.
  Falls back to 1 when nothing finite is present so the
  chart still renders.
- `getDefaultBulletRanges` returns 3 tiers at 33% / 66% /
  100% of the chart max, each pre-labelled (Low / Medium /
  High) and pre-coloured.
- `getBulletRatio` clamps `value / max` into `[0, 1]` and
  returns 0 for non-finite / negative / zero-max inputs so
  the painter never produces NaN coordinates.
- `getBulletTicks` returns `count` evenly-spaced ticks
  including endpoints; collapses to `[0]` when max is
  non-positive; minimum count is 2.
- `sortBulletRangesAscending` returns a non-mutating
  ascending copy of the band list.

## ARIA + data attributes

- Root: `<div role="progressbar">` with
  `aria-label` / `aria-valuenow` / `aria-valuemin=0` /
  `aria-valuemax` / `aria-valuetext`.
- SVG: `aria-hidden="true"` (decorative; the root owns
  the semantics).
- `data-section` on every node:
  - `chart-bullet` (root)
  - `chart-bullet-header` (when label/subLabel present)
  - `chart-bullet-label`
  - `chart-bullet-sublabel`
  - `chart-bullet-value`
  - `chart-bullet-target-text`
  - `chart-bullet-svg`
  - `chart-bullet-range` (one per band)
  - `chart-bullet-measure`
  - `chart-bullet-target`
  - `chart-bullet-tick` (one per tick)
- Root mirrors `data-orientation`, `data-animate`,
  `data-value`, `data-target`, `data-max`, `data-ratio`.
- Range `<rect>`s mirror `data-range-label`,
  `data-range-max`, `data-range-ratio`. Measure mirrors
  `data-value` / `data-ratio`. Target line mirrors
  `data-target` / `data-target-ratio`. Tick text mirrors
  `data-tick-value`.

  - measure bar renders with data-value + data-ratio

  - target marker absent when target undefined

  - axis ticks render by default + endpoint values

## Out of scope

- Stacked / grouped bullet variants (host stacks
  multiple instances vertically for KPI dashboards).
- Hover tooltips on the measure / target (host wraps
  with `<Tooltip>` when needed).
- Animated value transitions on data change (snap-only
  for v1; `motion-safe:animate-fade-in` covers mount only).
- Negative / signed values (host normalises into
  `[0, max]` before passing the value).

# 8.33 - Web UI feature docs and intuition improvements

**Status:** done
**Branch:** `c4/ui-docs`
**Scope:** Layer documentation, tooltips, help center, onboarding, and
confirmable destructive actions on top of the 8.20B CLI-coverage pages
so operators can navigate the feature set without dropping to `c4
<cmd>`. Strictly additive over 8.20B - no page removed, no endpoint
renamed, no existing test file modified.

## Summary

Every page under the Features tab now opens with a shared
`PageDescriptionBanner` that carries a 1-2 line summary, the matching
`c4` CLI command, a collapsible "When to use" list and a collapsible
concrete example. Every action button, filter input and checkbox is
wrapped in a new `Tooltip` primitive so hover (and focus for keyboard
users) reveals what the control does. Empty-state panels pull localized
copy from a new i18n bundle (English + Korean). A global help drawer is
reachable from the header or the `h` shortcut, a keyboard shortcut
cheat sheet is reachable from `?`, and first-visit users see a
dismissable 4-step onboarding tour. The destructive Cleanup flow
replaces `window.confirm` with a shared `ConfirmDialog` that renders the
concrete list of branches / worktrees / directories about to be removed
so the user sees what is about to happen before they commit.

### Global keyboard shortcuts
Registered by `HelpUIRoot`:
- `?` / `Shift+/` - open the shortcut cheat sheet
- `h` / `H` - open the help drawer
- `Esc` - close any modal or drawer

All shortcuts skip when focus is inside `<input>` / `<textarea>` /
`contenteditable` so prose typing cannot hijack them.

## Per-page changes

Each of the 12 CLI-coverage pages now:

1. Mounts `<PageDescriptionBanner>` as the first child of `PageFrame`
   with `summaryKey`, `cliKey`, `exampleKey`, `useCasesKey` bound to
   the page's i18n namespace (e.g. `scribe.summary`).
2. Wires `onOpenHelp={openHelpDrawer}` on the banner so the Learn more
   button opens the help drawer.
3. Wraps every action button / input / checkbox in `<Tooltip
   label={t('<page>.tooltip.<name>')}>` so hover explains each control.
4. Calls `useLocale()` so the page re-renders when the user flips
   language.
5. Pulls its empty-state copy from `t('<page>.empty')`.

## Onboarding tour

`OnboardingTour` is mounted once inside `HelpUIRoot`. It auto-opens on
the first visit (tracked by `c4.onboardingTour.v1` in localStorage),
ships four steps (welcome, categories, banner anatomy, help +
shortcuts), and can be replayed via the exported `startOnboardingTour()`
helper so a future Settings entry can re-trigger it without a custom
storage dance.

## Destructive action preview

`ConfirmDialog` is a shared primitive accepting a `preview` React node,
a `busy` flag, `onConfirm` / `onCancel` handlers, and an optional
`destructive` flag (default `true`). Cleanup now uses it to render
three preview groups (branches, worktrees, directories) with counts so
the operator sees the concrete items about to be removed before
committing. `window.confirm` is removed from Cleanup. Rollback / close
paths in `ControlPanel` are not touched in this patch - they keep
their existing confirm dialogs since they live outside the CLI-coverage
page inventory; wiring them to `ConfirmDialog` is left as a follow-up
(see `TODO.md` row 8.33).

## Help drawer

`HelpDrawer` is a right-side slide-out that:
- Carries a global intro, feature-navigation hint, CLI mapping hint, and
  shortcut hint from the `help.*` i18n namespace.
- Renders one card per feature from the registry, each showing title,
  CLI equivalent, summary, use cases, example.
- Scrolls the active feature card into view on open (derived from the
  `#/feature/<id>` hash maintained by `FeatureView`).
- Searchable - a live query filters across title / summary / CLI /
  example / use cases.

## Tests

`tests/ui-docs.test.js` (new) - 100 assertions across 27 suites:

1. **i18n bundle integrity** - both `en.json` and `ko.json` parse as
   flat string maps, share an identical key set, carry every `<page>.summary
   / cli / example / useCases` key each page consumes, carry the
   required `help.*` / `tour.*` / `shortcuts.*` / `common.*` keys, and
   use the `|` delimiter for `useCases` values so `tList` can split them.
2. **Component contracts** - `PageDescriptionBanner`, `HelpDrawer`,
   `OnboardingTour`, `KeyboardShortcutsModal`, `ConfirmDialog`,
   `Tooltip`, `HelpUIRoot`, and the `lib/i18n.ts` loader expose the
   symbols the suite locks as their public surface.
3. **Per-page wiring** - each of the 12 CLI-coverage pages imports
   `PageDescriptionBanner`, mounts it with `summaryKey=<base>.summary`,
   hooks `onOpenHelp` to `openHelpDrawer`, wraps at least one control
   with a localized `<Tooltip>`, and subscribes to locale changes via
   `useLocale()`.
4. **Cleanup + Batch + Auto specifics** - Cleanup no longer uses
   `window.confirm`, renders `<ConfirmDialog>` with per-group previews;
   Batch defines `prefillExample` using `batch.example` and
   `batch.exampleMulti`; Auto surfaces the three scenario entries.

Regression guard - the full suite goes from **110 / 110** passing to
**111 / 111** passing with this patch.

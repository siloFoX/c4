## Indicator math

The TTM Squeeze (John Carter) detects periods of low volatility
-- a "coil" -- that often precede a sharp move. It compares two
volatility bands drawn around the same middle:

- the **Bollinger Bands** -- `mid +/- bbMult * stddev(close)`,
- the **Keltner Channels** -- `mid +/- kcMult * ATR`,

where `mid` is the `period`-bar moving average of the close. The
squeeze is **ON** when the Bollinger Bands sit fully inside the
Keltner Channels (`bbUpper < kcUpper` and `bbLower > kcLower`)
and **OFF** when they expand outside.

`computeLineSqueezeSma(values, period)` is the moving average.
`computeLineSqueezeStd(values, period)` is the rolling population
standard deviation. `computeLineSqueezeRange(closes)` is the
close-only true range -- the absolute bar-to-bar change
`abs(close[i] - close[i-1])` -- and its moving average is the
ATR.

`computeLineSqueeze(closes, period, bbMult, kcMult)` assembles
the bands and also returns the `compression`:

```
compression[i] = kcMult * atr[i] - bbMult * stddev[i]
```

Because both bands share the same middle, `compression` is
positive exactly when the Bollinger Bands sit inside the Keltner
Channels -- it is the squeeze in a single scalar.

### Worked fixture

With `period` 2 the population standard deviation of a window is
exactly `abs(a - b) / 2`, so with integer closes the whole
pipeline -- both bands and the compression -- lands on exact
integers (asserted with `toEqual`).

The fixture closes `[50, 54, 58, 78, 70, 78, 82, 106, 98, 102]`
give:

```
bbUpper [., 56, 60, 88, 82, 82, 84, 118, 110, 104]
bbLower [., 48, 52, 48, 66, 66, 76, 70, 94, 96]
kcUpper [., ., 62, 86, 95, 86, 89, 115, 126, 109]
kcLower [., ., 50, 50, 53, 62, 71, 73, 78, 91]
comp    [., ., 2, -2, 13, 4, 5, -3, 16, 5]
state   [none, none, on, off, on, on, on, off, on, on]
```

The squeeze is on for 6 bars, off for 2, and the longest
unbroken squeeze runs 3 bars.

## API

- `getLineSqueezeFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineSqueezePeriod(period, fallback)` -- coerce to a
  positive integer.
- `computeLineSqueezeSma(values, period)` -- the moving average
  of a nullable series.
- `computeLineSqueezeStd(values, period)` -- the rolling
  population standard deviation.
- `computeLineSqueezeRange(closes)` -- the close-only true range.
- `computeLineSqueeze(closes, period, bbMult, kcMult)` -- the
  `{ mid, bbUpper, bbLower, kcUpper, kcLower, compression }`
  series.
- `runLineSqueeze(points, options?)` -- sort finite points by
  `x`, run the pipeline, return `{ series, period, bbMult,
  kcMult, mid, bbUpper, bbLower, kcUpper, kcLower, compression,
  samples, squeezeOnCount, squeezeOffCount, longestSqueeze,
  finalState, ok }`. Each sample is classified `on` / `off` /
  `none`. `ok` is false for fewer than two points.
- `computeLineSqueezeLayout(options)` -- stack the price panel
  above a squeeze panel; project the price path, the Bollinger
  Band area, the Keltner Channel paths, the compression path,
  one squeeze dot per classified sample and the zero line.
- `describeLineSqueezeChart(data, options?)` -- ARIA description.

## Component

`ChartLineSqueeze` is a `forwardRef` region (`role="region"`)
with an ARIA-linked description, a `role="img"` SVG, two stacked
panels (price on top, squeeze below) with their own axis ticks
and grid, panel labels, a config badge showing the period and
the two multipliers, a four-series legend (Price / Bollinger /
Keltner / Squeeze) with toggle, a hover/focus tooltip exposing
the compression and the squeeze state, and a
`motion-safe:animate-fade-in` entrance. It consumes `{ x, value }`
points. The price panel draws the Bollinger Bands as a shaded
area and the Keltner Channels as two dashed channel lines, so the
squeeze reads directly as the blue area sitting inside the orange
lines; the squeeze panel draws the compression line over a dashed
zero line plus a row of squeeze dots coloured by state. Every
node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-bollinger / chart-line-keltner** -- those draw
  a single volatility band. The squeeze co-plots both bands and
  flags their containment as an explicit on/off state.
- **vs chart-line-bandwidth** -- that measures one band's width.
  The squeeze compares two bands' widths against each other.
- **vs the oscillator panels (tdi / gator / ravi)** -- those
  reduce price to a derived line. The squeeze panel is a
  state-flag strip plus a compression line.

# 11.325 -- UI: AppShell consolidation

- TODO id: 11.325

## Goal

Introduce a reusable application-shell primitive that
codifies the header / sidebar / main / footer layout
across the c4 web app. The dispatch lists:

- Consistent sidebar / main / footer regions.
- ScrollArea around long surfaces.
- Responsive collapse with Drawer for mobile.
- Tooltip on collapsed nav items.
- Focus-visible rings everywhere.

The existing App.tsx composition wires AppHeader +
Sidebar + FeatureView together by hand. This patch
delivers the primitive that future patches can adopt
instead of re-deriving the same layout rhythm. The
existing App.tsx layout is left untouched in this patch;
no live screen changes ship.

## Public API

```ts
export interface AppShellNavItem {
  id: string;
  label: string;
  icon?: ReactNode;
  href?: string;

export interface AppShellProps {
  header: ReactNode;
  nav?: AppShellNavItem[];
  sidebarTitle?: ReactNode;
  sidebarFooter?: ReactNode;
  sidebarCollapsed?: boolean;
  onSidebarCollapsedChange?: (collapsed: boolean) => void;
  mobileNavOpen?: boolean;
  onMobileNavOpenChange?: (open: boolean) => void;
  footer?: ReactNode;
  children: ReactNode;
  className?: string;
  headerClassName?: string;
  sidebarClassName?: string;
  mainClassName?: string;
  footerClassName?: string;
}

export const APP_SHELL_FOCUS_RING: string;
```

## Behaviour

### Regions

The primitive renders four semantic landmarks:

| landmark | role | data-section | when |
| --- | --- | --- | --- |
| `<header>` | banner | `app-shell-header` | always |
| `<aside>` | sidebar | `app-shell-sidebar` | when `nav` is provided |
| `<main>` | main | `app-shell-main` | always |
| `<footer>` | contentinfo | `app-shell-footer` | when `footer` is provided |

The body wraps the aside + main in a flex row
(`data-section="app-shell-body"`) so the persistent
sidebar sits to the left of the scrollable main region.

### ScrollArea

Both the sidebar nav list and the main content are
wrapped in vertical `ScrollArea` primitives so long
content scrolls inside its own shell while the header,
sidebar chrome, and footer stay fixed in the viewport.

| ScrollArea | testid |
| --- | --- |
| sidebar | `app-shell-sidebar-scroll` |
| main | `app-shell-main-scroll` |

### Collapse axis (desktop)

`sidebarCollapsed` shrinks the rail to `3.5rem` and
hides the nav labels. Each label moves into a Tooltip
that fires on hover / focus, so the operator can still
identify each icon. The `<aside>` carries
`data-collapsed="true|false"` for e2e + styling.

A collapse toggle button (testid
`app-shell-sidebar-collapse-toggle`) renders at the
bottom of the aside ONLY when
`onSidebarCollapsedChange` is wired. Hosts that do not
care about collapse simply omit the prop and the toggle
disappears.

### Mobile Drawer

On viewports below `md` the persistent aside is hidden
(`hidden md:flex`). A hamburger button (testid
`app-shell-mobile-nav-toggle`) sits at the left of the
header and toggles a left-anchored Drawer (`width=80vw`)
containing the same nav items. Tapping a nav item
auto-closes the drawer so the new route is visible
without an extra tap.

`mobileNavOpen` / `onMobileNavOpenChange` are
controlled. When omitted the primitive owns the state
internally.

### Focus-visible rings

Every interactive surface inside the shell applies the
shared focus-visible ring class:

```ts
export const APP_SHELL_FOCUS_RING =
  'focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-primary focus-visible:ring-offset-2 focus-visible:ring-offset-background';
```

The constant is exported so other layout primitives can
adopt the same contract without re-deriving the class
string. Tab navigation inside the shell always shows a
visible ring.

## NavItem rendering

`NavItemButton` is an internal helper that renders each
`AppShellNavItem` as either a `<button>` (default) or
an `<a>` (when `href` is set). The element carries:

- `data-testid="app-shell-nav-<id>"`
- `data-active="true|false"`
- `aria-current="page"` when active
- `aria-label="<label>"` only when the label is hidden
  (collapsed)
- The shared focus-visible ring class

Disabled items prevent the click handler from firing
(both via `disabled` on `<button>` and via an explicit
guard inside the click handler for the `<a>` form).

When collapsed, the entire nav item is wrapped in a
`Tooltip` with the label as the body.

## Tests

`web/src/components/layout/AppShell.test.tsx` is new.
Twenty cases:

1. renders header / main landmarks;
2. skips footer landmark when omitted;
3. renders footer landmark when provided;
4. skips aside when `nav` is omitted;
5. renders aside + nav items when `nav` is provided;
6. marks the active nav item with `aria-current` +
   `data-active`;
7. fires nav onClick when clicked;
8. skips onClick when disabled;
9. hides nav labels + adds aria-label when collapsed;
10. aside carries `data-collapsed="true"` when
    collapsed;
11. aside carries `data-collapsed="false"` when
    expanded;
12. collapse toggle only renders when
    `onSidebarCollapsedChange` is wired;
13. collapse toggle flips the controlled value;
14. mobile nav toggle only renders when `nav` is
    provided;
15. mobile nav toggle opens the Drawer via the
    controlled callback;
16. sidebar title renders only when expanded;
17. sidebar footer renders when provided;
18. sidebar + main wrap in ScrollArea primitives;
19. exports the focus-visible ring class;
20. nav item renders as an `<a>` when `href` is set.

20/20 cases pass against vitest 4.1.5 + jsdom 29.1.1.

## Adoption

No active adopters in this patch. The primitive is
ready to be threaded into App.tsx (replacing the
AppHeader + Sidebar + FeatureView composition) in a
future patch. Doing so will involve:

- Mapping the existing `topView` switch to an
  `AppShellNavItem[]` array.
- Threading `sidebarMode` + `sidebarCollapsed` through
  the new primitive's collapse axis.
- Migrating the Drawer-style filter / preferences
  affordance to `mobileNavOpen`.

That migration is left out of scope here because it
would touch the SettingsView / FeatureView surface and
should land with its own test coverage.

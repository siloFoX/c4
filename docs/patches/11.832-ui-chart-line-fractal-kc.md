# Patch 11.832 -- ChartLineFractalKc primitive (v1.11.850)

## Summary

Add `<ChartLineFractalKc />`, a pure-SVG single-panel chart that
overlays a fractal-anchored Keltner Channel envelope on a price
line. The channel middle reseeds off the most recently confirmed
Bill Williams fractal pivots; Wilder's ATR provides the channel
half-width:

```
isUpperFractal[i] = high[i] is strict max of high[i - 2 .. i + 2]
isLowerFractal[i] = low[i]  is strict min of low [i - 2 .. i + 2]
                    (confirmed at index i + lookback)

lastUF[i] = high of last confirmed upper fractal at index <= i
lastLF[i] = low  of last confirmed lower fractal at index <= i

trueRange[i] = max(h - l, |h - prevC|, |l - prevC|);  TR[0] = h - l
atr[i]       = Wilder(trueRange, atrLength)[i]

fractalMid[i] = both fractals available -> (lastUF + lastLF) / 2
              | only one available      -> that one
              | neither                 -> close[i]
upper[i]      = fractalMid[i] + multiplier * atr[i]
lower[i]      = fractalMid[i] - multiplier * atr[i]
```

- Warmup: `i < atrLength - 1` -> bands `null`.
- The mid always exists (closes fallback when no fractal yet).
- `-0` normalized to `+0` on every arithmetic result.

Lookback 2 matches the canonical Bill Williams 5-bar pattern;
multiplier 2 matches Chester Keltner's original spec.

## Wilder helper precision fix

The Wilder smoother now seeds with the exact input value when the
seed window is constant (`min === max`) instead of `sum / length`.
This avoids 1-ULP drift when the constant is non-dyadic and the
length is not a power of 2. The CONST short-circuit
`next = v === smoothed ? v : (smoothed * (n - 1) + v) / n` then
preserves the constant exactly because for `v === smoothed`,
`(v * (n - 1) + v) / n = v * n / n = v` algebraically and bit-exact
when `v` is integer-representable.

## Bit-exact anchors

- `CONST high = low = close = K`: TR = 0 everywhere; ATR = 0; no
  fractals trigger (degenerate window has equal highs/lows so the
  strict-max test fails); `fractalMid = close = K`;
  `upper = lower = mid = K` bit-exact post-warmup. Verified across
  `(K, atrLength, multiplier)` sweeps.
- `CONSTANT-SPREAD high = K + D, low = K, close = K + D/2`: TR =
  `D` from `i = 0` (`|h - prevC| = D/2 <= D` and `|l - prevC| = D/2
  <= D`); Wilder ATR = `D` bit-exact; no fractals trigger (highs
  and lows are flat); `fractalMid = close = K + D/2`;
  `upper = K + D/2 + multiplier*D`, `lower = K + D/2 - multiplier*D`
  bit-exact. Verified across `(D, baseLow, atrLength, multiplier)`
  sweeps.

The LINEAR anchor that worked for other primitives is NOT bit-exact
here because TR[0] = h - l = 0 in `h = l = close = i + 1` data,
which breaks the constant-TR-from-index-0 invariant the SMA seed
needs. CONSTANT-SPREAD restores that invariant.

## Surface

Single-panel primitive surface:

- `ChartLineFractalKcPoint = { x, high, low, close }`
- `runLineFractalKc(data, options?) -> Run` with
  `ok = series.length >= atrLength + fractalLookback`.
- `computeLineFractalKcLayout({ data, width?, height?, padding?, atrLength?, fractalLookback?, multiplier? }) -> Layout`.
  Single panel; y-axis auto-fits across `high/low/upper/lower`.
- `describeLineFractalKcChart(data, opts?) -> string` for the ARIA
  description.
- `<ChartLineFractalKc />` `forwardRef<HTMLDivElement>` with four
  legend toggles (`price`, `upper`, `mid`, `lower`), config badge,
  dashed mid line, fractal-pivot scatter markers (upper red, lower
  green), and a tooltip surfacing
  `close / upper / mid / lower / atr / lastUF / lastLF / zone /
  cross`.

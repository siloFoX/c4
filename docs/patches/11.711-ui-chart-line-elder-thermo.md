## Summary

New `ChartLineElderThermo` UI primitive in
`web/src/components/ui/chart-line-elder-thermo.tsx` -- a pure-SVG
two-panel line chart with an **Elder Market Thermometer** panel from the
larger of the high and low excursions per bar. The top panel plots the
bar midpoint; the bottom panel plots the thermometer and its moving
average.

## The computation

Alexander Elder's Market Thermometer reads market "temperature" from how
far each bar reaches beyond the prior bar. `ChartLineElderThermoPoint`
carries `{x, high, low}`.

- `computeLineElderThermometer(bars)` -- for each bar past the first, the
  larger of the absolute change in the high and the absolute change in
  the low: `max(|high - high[-1]|, |low - low[-1]|)`. The first bar has
  no prior, so it is null.
- `computeLineElderThermoEma(values, period)` -- an exponential moving
  average over a series that may carry leading nulls; the nulls pass
  through and the average seeds from the first finite value.
- `computeLineElderThermo(bars, options)` -- pairs the thermometer with
  its moving average.
- `classifyLineElderThermoZone(thermometer, thermoMa, hotFactor)` --
  marks a bar **calm** when the thermometer is below its average (or at
  rest), **warm** when it is above the average, **hot** once it crosses
  the average times the hot factor.

## Worked fixture

The thermometer is pure max / abs / difference arithmetic, so it is
exactly computable; only the moving average is transcendental, and a
dyadic alpha makes that exact too.

A **flat market** (constant high and low) has no excursion: every
thermometer reading is exactly 0 and the market is wholly calm. A
**steady ramp** -- both bands rising by a fixed step -- has a constant
excursion, so the thermometer holds a constant value and an exponential
average of that constant is exactly the constant. A varying fixture
`high [20,24,19,28,22,30,25,33]`, `low [10,14,12,8,15,11,18,14]`
computes to the exact integer thermometer `[-,4,5,9,7,8,7,8]` (asserted
with `toEqual`), and with a period-3 average (`alpha = 0.5`) the moving
average is the exact dyadic series `[-,4,4.5,6.75,6.875,7.4375,...]`.
The zone classifier is checked directly for calm, warm and hot.

## API

`data` (a `ChartLineElderThermoPoint[]`), `period` (default 22),
`hotFactor` (default 2), `pricePanelRatio` / `gap`, sizing/format/a11y
props, `strokeWidth` / `dotRadius`, the colour props, the toggle flags
(`showAxis` / `showGrid` / `showDots` default false / `showThermo` /
`showThermoMa` / `showMarkers` / `showTooltip` / `showConfigBadge` /
`showLegend` / `animate`), controlled + uncontrolled `hiddenSeries`
(keys `price` / `thermo` / `thermoMa`), `formatValue` / `formatX`,
`onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineElderThermoFinitePoints`,
`normalizeLineElderThermoPeriod` (coerces the period to an integer of at
least 1), `normalizeLineElderThermoHotFactor` (coerces the hot factor to
a number above 1), `computeLineElderThermometer`,
`computeLineElderThermoEma`, `classifyLineElderThermoZone`,
`computeLineElderThermo`, `runLineElderThermo`,
`computeLineElderThermoLayout`, `describeLineElderThermoChart`, plus the
`DEFAULT_CHART_LINE_ELDER_THERMO_*` constants and the
`ChartLineElderThermoZone` / `ChartLineElderThermoComputed` types. Every
exported name carries a `LineElderThermo` infix so the `export *` barrel
line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen
`data-section="chart-line-elder-thermo-aria-desc"` description span; the
midpoint line, the thermometer line, the moving-average line and every
dot / marker are `role="graphics-symbol" tabIndex={0}`. Every node
carries a `data-section`; the root mirrors `data-period`,
`data-hot-factor`, `data-thermo-final`, `data-calm-count`,
`data-warm-count`, `data-hot-count`, `data-total-points` and
`data-animate`.

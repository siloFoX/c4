# 11.1019 -- UI: chart-line-hma-cross-sig primitive (v1.11.1037)

## Scope

Add a pure-SVG dual-panel `ChartLineHmaCrossSig` component
that renders the close in the top panel and the Alan Hull
Moving Average (HMA) plus its smoothed SMA signal line in the
bottom panel, marking bullish (HMA crosses up through signal,
fast smoothed trend trigger up) / bearish (HMA crosses down
through signal, fast smoothed trend trigger down) HMA-over-

from the HMA slope at the trigger bar.

## HMA pipeline (Hull)

```
wma_half  = WMA(close, floor(period / 2))
wma_full  = WMA(close, period)
raw[i]    = 2 * wma_half - wma_full
hma[i]    = WMA(raw, floor(sqrt(period)))
signal[i] = SMA(hma, signalLength)
bullish   : prev hma <= prev signal && cur hma > cur signal
bearish   : prev hma >= prev signal && cur hma < cur signal
regime    : bullish (hma >= signal), bearish (hma < signal)
bias      : up/down/flat/none from hma[i] vs hma[i-1]
```

The double-WMA cancels lag (raw leads price by half the
window), and the sqrt-length final WMA restores smoothness.
Bias-coloured cross markers mirror the convention used by
`chart-line-macd-zero-cross-sig` /
`chart-line-stc-divergence-cross`.

Warmup is `period + floor(sqrt(period)) + signalLength - 3 =
20` for the default tuning: WMA full-period seeds at `period
- 1 = 15`, then the sqrt(period)-bar WMA over raw adds
`floor(sqrt(period)) - 1 = 3` bars, then the signal SMA adds
another `signalLength - 1 = 2` bars.

Defaults: `period = 16` (Hull's recommended HMA window),
`signalLength = 3` (signal SMA window).

## Bit-exact anchors

| input              | hma (post-warmup) | signal (post-warmup) | hma - signal | regime  | crosses |
| ------------------ | ----------------- | -------------------- | ------------ | ------- | ------- |
| CONST close = K    | K                 | K                    | 0            | bullish | 0       |
| LINEAR UP, close=i | i - 2/3           | i - 5/3              | +1           | bullish | 0       |
| LINEAR DOWN, close=-i | -i + 2/3       | -i + 5/3             | -1           | bearish | 0       |

CONST K verified across K = 0..1234 -- both WMA layers
collapse to K, so raw = K and hma = K. signal = SMA(K, 3) =
K. hma === signal -> regime `bullish` (inclusive `>=`
boundary). bias `flat` from second valid sample.

LINEAR UP exercises the WMA lag algebra: for LINEAR close=i,
WMA(close, L) settles at `i - (L-1)/3`. The HMA pipeline
yields `hma = i - 2/3` and the SMA signal lags hma by
`(signalLength - 1)/2 = 1`, giving signal = `i - 5/3`.
hma - signal = `+1` constant. regime `bullish`.

LINEAR DOWN is the algebraic mirror: WMA(-close, L) = `-i +
(L-1)/3`. hma = `-i + 2/3`, signal = `-i + 5/3`. hma -
signal = `-1` constant. regime `bearish`.

The hma-signal cross detector is exercised in dedicated unit
tests for `detectLineHmaCrossSigCrosses`, which checks that
hma rising through signal fires `bullish`, falling through
signal fires `bearish`, and the trigger does not double-fire
on consecutive bars above the signal.

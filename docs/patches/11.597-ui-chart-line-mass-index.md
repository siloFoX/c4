## Summary

New `ChartLineMassIndex` UI primitive at
`web/src/components/ui/chart-line-mass-index.tsx` -- a pure-SVG line
chart with a Mass Index panel detecting reversals from
range-expansion bulges. The top panel renders the high and low lines
with a shaded band between them; the bottom panel renders the Mass
Index line with dashed setup and trigger threshold lines and markers
coloured by reversal phase.

Unlike the other `chart-line-*` primitives, this one consumes
`{ x, high, low }` points -- the Mass Index is intrinsically a
high-low range study.

## Indicator math

Donald Dorsey's Mass Index, with an EMA period (default 9) and a sum
period (default 25):

```
range[i]    = high[i] - low[i]
single      = EMA(range, emaPeriod)
double      = EMA(single, emaPeriod)
ratio[i]    = single[i] / double[i]
massIndex[i] = sum of ratio over the last sumPeriod periods
```

`computeLineMassIndexEma(src, period)` is an EMA that tolerates
leading `null` placeholders (the single-EMA series has them): it
collects the defined values in order, seeds with the simple mean of
the first `period` of them, then folds each later defined value in at
weight `2 / (period + 1)`.

A widening high-low range pushes the single EMA above its own smoothed
double EMA, so the ratio climbs above one and the running sum bulges.
That bulge is the reversal signal.

### Reversal-bulge state machine

`runLineMassIndex` walks the Mass Index series:

- a reading at or above the **setup** threshold (default 27) arms a
  bulge -- the bar is tagged `bulge`;
- a later reading below the **trigger** threshold (default 26.5),
  while a bulge is armed, fires a reversal -- the bar is tagged
  `signal` and `bulgeCount` increments;
- every other reading is `calm`.

### Worked fixture

`low = 0` throughout, `high` (= range) `[6,6,6,6,6,18,18,18,18,18]`,
`emaPeriod = 2`, `sumPeriod = 3`:

```
single = [.,6,6,6,6,14,50/3,158/9,482/27,1454/81]      (EMA mult 2/3)
double = [.,.,6,6,6,34/3,134/9,50/3,1414/81,4322/243]
ratio  = [.,.,1,1,1,21/17,75/67,79/75,723/707,2181/2161]
mass   = [.,.,.,.,3,55/17,3.354697,3.408030,3.195367,3.085219]
```

With `setup = 3.4` / `trigger = 3.3`: the Mass Index crosses above
3.4 at index 7 (`bulge`), then drops below 3.3 at index 8 (`signal`)
-- one reversal bulge.

## API

- `getLineMassIndexFinitePoints(points)` -- drop points with a
  non-finite `x`, `high` or `low`.
- `normalizeLineMassIndexPeriod(period, fallback)` -- positive
  integer; non-finite / sub-1 falls back, fractional floors.
- `computeLineMassIndexEma(src, period)` -- the null-tolerant EMA.
- `computeLineMassIndex(highs, lows, emaPeriod, sumPeriod)` -- returns
  `{ range, single, double, ratio, massIndex }`.
- `runLineMassIndex(points, options?)` -- sort finite points by `x`,
  compute the series, walk the bulge state machine, return
  `{ series, emaPeriod, sumPeriod, setupThreshold, triggerThreshold,
  range, single, double, ratio, massIndex, samples, massIndexFinal,
  massIndexMin, massIndexMax, bulgeCount, ok }`. Each sample carries a
  `calm` / `bulge` / `signal` phase. `ok` is false for fewer than two
  points.
- `computeLineMassIndexLayout(options)` -- stack a range panel above a
  Mass Index panel; fit the Mass Index y-domain around both threshold
  lines. Returns panels, ticks, the high/low/band paths, the Mass
  Index path, phase-coloured markers, and the threshold-line `y`
  positions.
- `describeLineMassIndexChart(data, options?)` -- ARIA description.

## Component

`ChartLineMassIndex` is a `forwardRef` region (`role="region"`) with
an ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge (EMA and sum periods), a three-series legend (High / Low / Mass
Index) with toggle, a shaded high-low band, the Mass Index line,
dashed setup and trigger threshold lines, markers coloured by phase
(`bulge` red, `signal` green, `calm` purple), a hover/focus tooltip
exposing the high, low, range, Mass Index, and phase, and a
`motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs every other chart-line primitive** -- the Mass Index is the
  only one that consumes a `{ x, high, low }` point. It is computed
  entirely from the high-low *range*, not from a single value series.
- **vs chart-line-bollinger / chart-line-keltner / chart-line-atr** --
  those measure volatility as a band width or an average true range.
  The Mass Index instead takes the *ratio of a single to a double EMA*
  of the range and *sums* it over a window: it is a reversal detector
  built on the curvature of range smoothing, not a volatility band or
  an average. The setup/trigger bulge state machine is unique to it.

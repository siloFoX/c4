## Summary

New `ChartLineRenko` UI primitive at
`web/src/components/ui/chart-line-renko.tsx` -- a pure-SVG chart
rendering a Renko brick transform that advances only on fixed
price moves.

## Indicator math

A Renko chart discards time and sub-brick noise: it advances only
when the price moves a full `brickSize`. The transform walks the
price series from the first value, tracking the last brick
boundary `level`:

```
while price >= level + brickSize:  emit up brick   [level, level+brickSize], level += brickSize
while price <= level - brickSize:  emit down brick  [level, level-brickSize], level -= brickSize
```

A move smaller than one brick emits nothing; a large move emits
several bricks in one step. Each brick records its ordinal
`index`, the `sourceIndex` of the price that triggered it, a
`direction`, and its `open` / `close` levels (up brick:
`open < close`; down brick: `open > close`).

### Worked fixture

`brickSize 2`, price series `[10, 11, 14, 13, 9, 10, 16]`:

```
->14  emits two up bricks    (10-12, 12-14)
->9   emits two down bricks  (14-12, 12-10)
->16  emits three up bricks  (10-12, 12-14, 14-16)
```

The 11, 13 and 10 wiggles never move a full brick, so they emit
nothing. Seven bricks total -- 5 up, 2 down -- spanning the price
range 10 to 16, with a last brick close of 16.

## API

- `getLineRenkoFinitePoints(points)` -- drop price points with a
  non-finite `x` or `value`.
- `normalizeLineRenkoBrickSize(size, fallback)` -- coerce the
  brick size to a positive finite number; zero, negative or
  non-finite falls back.
- `computeLineRenkoBricks(values, brickSize)` -- the brick series
  `{ index, sourceIndex, direction, open, close }`.
- `runLineRenko(points, options?)` -- sort finite points by `x`,
  run the transform, return `{ series, brickSize, bricks,
  upCount, downCount, priceMin, priceMax, lastClose, ok }`. `ok`
  is false when no brick forms.
- `computeLineRenkoLayout(options)` -- lay the bricks out one per
  column and project each to a rect, plus a trend path through
  the brick closes.
- `describeLineRenkoChart(data, options?)` -- ARIA description.

## Component

`ChartLineRenko` is a `forwardRef` region (`role="region"`) with
an ARIA-linked description, a `role="img"` SVG, axis ticks and
grid, a config badge showing the brick size, a three-series
legend (Up brick / Down brick / Trend) with toggle, a
hover/focus tooltip exposing the brick index, direction, open and
close, and a `motion-safe:animate-fade-in` entrance. It consumes
`{ x, value }` points plus a `brickSize`. Each brick is an SVG
rect -- up bricks green, down bricks red -- laid out one per
column, with a trend line connecting the brick closes. Every node
carries a `data-section` attribute.

## Distinctness

- **vs the time-based line charts (chart-line-smma /
  chart-line-heikin-ashi / ...)** -- those plot one mark per time
  bar. The Renko x-axis is a *brick ordinal*, not time: a brick
  is added only on a full price move, so a flat stretch of the
  price series produces no bricks at all.
- **vs chart-line-heikin-ashi** -- Heikin-Ashi smooths every bar
  into a bar of the same cadence. Renko re-samples the series
  onto a fixed price grid and drops the time axis entirely.
- **vs chart-line-fractal** -- fractals mark discrete pivots on
  the original series; Renko rebuilds the whole series as a row
  of equal-size bricks.

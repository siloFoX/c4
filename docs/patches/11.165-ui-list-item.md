# Patch 11.165 -- UI: ListItem primitive (v1.11.183)

New `<ListItem>` row primitive consolidating the "leading icon + title + description + trailing action(s)" pattern that recurred across the sidebar / page list views. Adds:

- `web/src/components/ui/list-item.tsx` -- component + types.
- `web/src/components/ui/list-item.test.tsx` -- RTL coverage (17 cases).
- `web/src/components/ui/index.ts` -- re-export.

## Component contract

Props:

- `leading?: ReactNode` -- leading icon / avatar slot (fixed shrink-0 column).
- `title: ReactNode` -- required; rendered with truncate + font-medium on the foreground.
- `description?: ReactNode` -- 2-line clamp by default; line-clamp-1 when `descriptionLines=1`.
- `trailing?: ReactNode` -- right-aligned action(s).
- `onClick?: () => void` -- when set (and no href) the whole row renders as a native `<button>` so Enter/Space toggle it.
- `active?: boolean` -- selected visuals: `bg-muted ring-1 ring-primary/30` + `aria-current="true"` (interactive variants).
- `disabled?: boolean` -- `opacity-50 + pointer-events-none + aria-disabled`. `href` is dropped when disabled so anchors do not navigate.
- `href?: string` -- renders as `<a>` instead of `<button>` / `<div>`.
- `size?: 'sm' | 'md'` (default `md`) -- sm reduces padding to `px-2 py-1.5` with `text-xs` title / `text-[11px]` description.
- `descriptionLines?: 1 | 2` (default `2`).
- `className?: string` -- merged into the root via `cn()`.

Element selection rules:

| onClick | href | element            | implicit role |
| ------- | ---- | ------------------ | ------------- |
| -       | yes  | `<a href>`         | `link`        |
| yes     | -    | `<button type=button>` | `button`  |
| -       | -    | `<div role=listitem>` | `listitem` |

`forwardRef` is wired through to whichever element renders (typed as the union `HTMLAnchorElement | HTMLButtonElement | HTMLDivElement`).

## Adoption sites

1. **SessionsListSection per-session row** -- the inner `<button>` that selects a session was replaced with `<ListItem onClick active title=<shortId> description=<snippet + relative time> trailing=<turnCount Badge> />`. `aria-current` is preserved on the active row (existing test relies on it).
2. **Templates per-template row** (`pages/Templates.tsx`) -- the row inside each `Panel` becomes a non-interactive `<ListItem>` (no onClick) so the inner Edit / Remove buttons remain valid interactive elements in the trailing slot. Title slot now carries the name + source/model/effort/profile Chips.
3. **Profiles per-profile row** (`pages/Profiles.tsx`) -- the outer collapsed-header `<button>` becomes an interactive `<ListItem onClick=toggle active=isOpen aria-expanded=isOpen>`. The expanded Fieldset + Edit/Remove action row stays outside the ListItem (Fieldset and its buttons would otherwise be nested under a button).

WorkerList was considered the original third target but its per-worker `Card` carries the failure-hint banner, intervention sub-row, and branch line below the header -- those sub-blocks would have to be jammed into the description slot, which defeats the row contract. The per-worker Card therefore stays as-is; Templates is the substitute per the task's substitution clause.

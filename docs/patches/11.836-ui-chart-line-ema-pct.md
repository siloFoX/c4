# Patch 11.836 -- ChartLineEmaPct primitive (v1.11.854)

## Summary

Add `<ChartLineEmaPct />`, a pure-SVG dual-panel line chart with an
EMA Percent-Change oscillator beneath a price line that also
overlays the underlying EMA on the top panel:

```
ema[i]    = EMA(close, length)[i]
emaPct[i] = ema[i] === 0 ? null : (close[i] - ema[i]) / ema[i] * 100
```

- Warmup: `i < length - 1` -> `null`.
- `null` whenever `ema[i]` is zero (divide-by-zero guard).
- `-0` normalized to `+0` on every arithmetic result.

## Defaults

| Knob              | Value     |
|-------------------|-----------|
| length            | 14        |
| bullishThreshold  | 0         |
| bearishThreshold  | 0         |
| emaColor          | `#f97316` (orange) |
| pctColor          | `#8b5cf6` (violet) |

The orange EMA overlay matches typical EMA-band conventions on
price panels; the violet oscillator distinguishes the
percent-change panel from prior percent-style oscillators (the pink
`#ec4899` of `chart-line-momentum-pct` shipped one patch earlier).

## EMA seed precision

The SMA-seeded EMA helper carries the `min === max` seed fix added
in patches 11.831 (TRIX) and 11.832 (fractal KC). When all values
in the seed window are identical the EMA is seeded with that exact
value instead of `sum / length`, avoiding 1-ULP drift when `length`
is not a power of 2 and the constant is non-dyadic. The CONST
short-circuit `next = v === ema ? v : alpha*v + (1-alpha)*ema` then
preserves the constant exactly for the recursive smoothing.

## Bit-exact anchors

- `CONST close = K, K != 0`: EMA collapses to `K` (min/max seed) and
  the recursion short-circuit keeps it there; the numerator
  `close - ema = 0`, so `emaPct = 0` bit-exact post-warmup. Verified
  across `K in {1, 5, 100, -3}` and `length in {2, 4, 7, 14}`.
- `CONST close = 0`: `ema = 0`, divide-by-zero guard -> `null`.
- `LINEAR UP close = i + 1 at i = L - 1`: SMA seed
  `ema = (1 + 2 + ... + L) / L = (L + 1) / 2`, current
  `close = L`, so
  `emaPct = (L - (L+1)/2) / ((L+1)/2) * 100 = (L-1)/(L+1) * 100`.
  For `L in {3, 7, 15}` the constant is dyadic and bit-exact:
  `L=3 -> 50`, `L=7 -> 75`, `L=15 -> 87.5`. Each tested explicitly
  per L.

## Surface

Standard primitive surface:

- `ChartLineEmaPctPoint = { x, close }`
- `runLineEmaPct(data, options?) -> Run` with
  `ok = series.length >= length`.
- `computeLineEmaPctLayout({ data, width?, height?, padding?, panelGap?, length?, bullishThreshold?, bearishThreshold? }) -> Layout`.
  Top panel renders close + EMA overlay; bottom panel renders the
  emaPct oscillator. Bottom y-axis auto-fits but always includes 0.
- `describeLineEmaPctChart(data, opts?) -> string` for the ARIA
  description.
- `<ChartLineEmaPct />` `forwardRef<HTMLDivElement>` with three
  legend toggles (`price`, `ema`, `pct`), config badge, optional
  threshold band + zero-line overlay, markers on cross events, and
  a tooltip surfacing `close / ema / delta / emaPct / zone / cross`.

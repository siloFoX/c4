## Summary

Add a new `<ChartTreemap>` UI primitive in
`web/src/components/ui/chart-treemap.tsx` -- pure-SVG
hierarchical treemap with a squarified layout (Bruls,
Huijsen, van Wijk 1999). Each node is drawn as a rectangle
whose area is proportional to its value; the squarified
algorithm keeps aspect ratios close to 1 so labels stay
legible. Click-to-drill descends into the hovered node; the
drill path is fully controllable and emits `onDrillChange`
so adopters can wire breadcrumbs or back buttons. Default
colour-by-value gradient highlights outliers.

## API

```ts
interface ChartTreemapNode {
  id: string;
  label: string;
  value?: number;
  children?: readonly ChartTreemapNode[];
  color?: string;
}

interface ChartTreemapProps {
  data: ChartTreemapNode;
  width?: number;                // default 480
  height?: number;               // default 320
  padding?: number;              // default 2
  showLabels?: boolean;          // default true
  showValues?: boolean;          // default true
  showTooltip?: boolean;         // default true
  animate?: boolean;             // default true
  className?: string;
  ariaLabel?: string;            // default 'Treemap'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  gradient?: { from?: string; to?: string };
  onNodeClick?: (args: {
    node: ChartTreemapNode;
    path: string[];
    value: number;
    rect: TreemapRect;
  }) => void;
  drillPath?: readonly string[];
  defaultDrillPath?: readonly string[];
  onDrillChange?: (path: string[]) => void;
}
```

## Behaviour

- Renders a `<div role="region">` with the canonical
  `aria-label` plus a `<svg role="img">` carrying the same
  label so screen readers find both the region and the
  chart-as-image.
- Layout: at every drill level, the primitive calls
  `squarifyTreemap(items, rect)` to partition the chart
  area into rectangles whose areas are proportional to
  each child's value. The squarified algorithm picks the
  shorter side of the remaining rect and packs items into
  a row until adding another item would worsen the worst
  aspect ratio.
- Drilling: clicking a child that has its own children
  pushes the child's id onto the drill path; the chart
  re-renders showing the child's children. Leaves do not
  drill (no children to descend into). `drillPath` makes
  the path fully controllable; `defaultDrillPath` is the
  uncontrolled initial state. Every drill emits
  `onDrillChange(path)` regardless of controlled state so
  the host can keep its breadcrumbs in sync.
- Colour: per-child colour interpolates between
  `gradient.from` and `gradient.to` based on the child's
  position within the current drill level's value range.
  Per-node `color` overrides this interpolation. Defaults
  match the funnel primitive (`#bfdbfe` -> `#1d4ed8`) for
  visual consistency.
- Labels & values: rendered in-cell as white text, only
  when the cell exceeds 40 px wide and 18 px tall so
  cramped cells stay readable.
- Tooltip: hover shows label + formatted value plus a
  drill hint when the node has children.
- `motion-safe:animate-fade-in` per cell group on mount;
  `animate=false` removes the class.
- Default ARIA description auto-summary describes the
  current drill level: `"Treemap of <label> with N
  children: <child> <value>, ..."` for parents;
  `"<label> (leaf): <value>"` for leaves. `ariaDescription`
  overrides.

## Pure helpers (exported)

```ts
DEFAULT_CHART_TREEMAP_WIDTH          = 480
DEFAULT_CHART_TREEMAP_HEIGHT         = 320
DEFAULT_CHART_TREEMAP_PADDING        = 2
DEFAULT_CHART_TREEMAP_GRADIENT_FROM  = '#bfdbfe'
DEFAULT_CHART_TREEMAP_GRADIENT_TO    = '#1d4ed8'

getNodeValue(node): number
findNodeByPath(root, path): { node, resolved }
worstAspectRatio(areas, length): number
squarifyTreemap(items, rect): { item, rect }[]
getTreemapColor(value, min, max, from, to): string
describeTreemap(root, path, formatValue?): string
```

- `getNodeValue` returns the explicit `value` for a leaf,
  the sum of children otherwise. Parents declaring their
  own `value` win over the children sum (lets adopters
  collapse subtrees). Non-positive / non-finite values
  collapse to 0.
- `findNodeByPath` descends the path; unknown ids stop
  the walk and the helper returns the last resolved
  ancestor plus the path tokens that were resolved -- the
  component uses this to keep `drillPath` defensible
  against stale paths.
- `worstAspectRatio` implements the squarified
  worst-ratio formula. Returns `+Infinity` for empty
  input, zero length, or non-positive minimum so the
  squarify loop can compare safely.
- `squarifyTreemap` is the squarified treemap layout.
  Drops items with non-positive value, partitions the
  rect proportionally, and returns one rect per item.
- `getTreemapColor` interpolates between `from` and `to`
  using `(value - min) / (max - min)`; non-finite inputs
  fall back to `from`; collapsed range returns `to`.
- `describeTreemap` produces the default ARIA summary
  for the current drill level.

## ARIA + data attributes

- Root: `<div role="region">` + `aria-label`.
- SVG: `role="img"` + same `aria-label`.
- sr-only span: `data-section="chart-treemap-aria-desc"`
  carries the auto-summary or `ariaDescription` override.
- Each cell `<rect>`: `role="graphics-symbol"` +
  `tabIndex={0}` so keyboard focus opens the tooltip.
- `data-section` on every node:
  - `chart-treemap` (root)
  - `chart-treemap-aria-desc`
  - `chart-treemap-svg`
  - `chart-treemap-cell` (per-child group)
  - `chart-treemap-rect`
  - `chart-treemap-label`
  - `chart-treemap-value`
  - `chart-treemap-tooltip` + `-label` / `-value` /
    `-drill-hint`
- Root mirrors `data-node-id` (current drill node),
  `data-depth`, `data-child-count`, `data-animate`.
  Cells mirror `data-node-id`, `data-value`, `data-color`,
  `data-hovered`, `data-has-children`.

## Tests (56 cases)

  non-positive value, missing value.
- `findNodeByPath`: empty path returns root, single
  descent, deep walk, last-resolved on stale path, root
  fallback for unknown id.
- `worstAspectRatio`: +Infinity for empty / zero
  length, +Infinity for non-positive min, finite for
  valid inputs.
- `squarifyTreemap`: empty items, zero-area rect, drops
  non-positive values, single item fills rect, total
  cell area equals rect area, cells fit inside bounds,
  areas proportional to values.

## Out of scope

- Animated reflow on data change (snap-only for v1;
  `motion-safe:animate-fade-in` covers mount only).
- Multi-level breadcrumb UI (host renders a separate
  `<Breadcrumb>` and feeds `drillPath`).
- Search / filter inside the chart (host filters the data
  tree before passing it to the primitive).
- Negative / signed values (host normalises before
  passing values; the primitive drops non-positive
  values during layout).

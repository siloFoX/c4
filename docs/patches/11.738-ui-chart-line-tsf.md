# Patch 11.738 -- UI: chart-line-tsf primitive

Released as `v1.11.756` on 2026-05-26.

## Scope

Add `<ChartLineTsf>` -- a pure-SVG single-panel chart with a **Time
Series Forecast** overlay. The TSF is the ordinary-least-squares
linear regression endpoint extrapolated one bar past the lookback
window.

## Definition

Inputs:

- `data: ChartLineTsfPoint[]` where `ChartLineTsfPoint = { x: number;
  value: number }`. The `value` field is the close.
- `period: number` -- the lookback. Default `9`. Normalized to an
  integer `>= 2`.

Algorithm (per bar `i >= period - 1`):

1. Take the most recent `period` closes ending at bar `i`:
   `closes[i - period + 1 .. i]`. Re-index as `j = 0 .. period - 1`.
2. Solve the OLS slope / intercept of `value` against `j`:

   ```
   denom     = n * sumXX - sumX * sumX
   slope     = (n * sumXY - sumX * sumY) / denom
   intercept = (sumY - slope * sumX) / n
   ```

3. Project the regression line one bar forward (to `j = period`):

   ```
   TSF[i] = intercept + slope * period
   ```

If `n < 2` or `denom == 0` (degenerate X-variance, which never
happens for `period >= 2` since the X values are a contiguous integer
sequence), the bar's TSF is `null`. The first `period - 1` bars are
always null.

## Bit-exact algebraic anchors

The reason this primitive is testable is the linear-ramp identity. If
the most recent `period` closes are a perfect integer ramp `close =
c + j` (`j = 0 .. period - 1`):

- `sumX = period * (period - 1) / 2`, `sumXX = period * (period - 1)
  * (2 * period - 1) / 6`, `sumY = period * c + sumX`, `sumXY = c *
  sumX + sumXX`.
- `numerator = period * sumXY - sumX * sumY = period * (c * sumX +
  sumXX) - sumX * (period * c + sumX) = period * sumXX - sumX ^ 2 =
  denom`. So `slope = 1` exactly (integer/integer cancellation).
- `intercept = (sumY - slope * sumX) / period = (period * c + sumX -
  sumX) / period = c` exactly.
- `TSF[i] = c + 1 * period = c + period` -- which is `close[i + 1]`
  exactly for the next bar of the same ramp.

For period 4 on `[10, 11, 12, ..., 19]`:

```
TSF = [null, null, null, 14, 15, 16, 17, 18, 19, 20]
```

The mirror identity holds for a falling ramp (`slope = -1`,
`intercept = c`, `TSF[i] = close[i + 1]` exactly).

Constant-series identity (`close = K`): `sumX, sumXX` unchanged, but
`sumY = period * K` and `sumXY = K * sumX`. `numerator = period * K *
sumX - sumX * period * K = 0`. So `slope = 0` exactly, `intercept =
(period * K - 0) / period = K`. `TSF[i] = K + 0 * period = K` --
bit-exact at every defined bar via IEEE 754 `c * 0 = 0`.

Translation invariance: shifting `close` by `k` shifts `intercept`
by `k` and leaves `slope` unchanged. So the TSF shifts by exactly
`k` -- bit-exact for integer `k`.

## Component

`forwardRef<HTMLDivElement, ChartLineTsfProps>` with displayName.
Renders:

- A region-role outer `<div data-section="chart-line-tsf">`.
- An `<svg role="img">` with an `aria-describedby` description and a
  `<title>` / `<desc>` pair.
- Price line as a `<path>` with `data-section="chart-line-tsf-price-path"`.
- One `<circle>` per bar (`data-section="chart-line-tsf-price-dot"`).
- The TSF rendered as one segment per consecutive pair of
  defined-TSF bars (`data-section="chart-line-tsf-segment"`),
  colour-coded by the slope sign (`data-trend = 'up' | 'down' |
  'flat'`).
- One marker per defined-TSF bar (`data-section="chart-line-tsf-marker"`),
  also `data-trend`-attributed.
- Legend toggle (`data-section="chart-line-tsf-legend-item"`) for
  the `tsf` series, plus a `showTsf` prop and an `onPointClick`
  handler.
- ARS design-system tokens for colour and motion (`motion-safe:
  animate-fade-in`).

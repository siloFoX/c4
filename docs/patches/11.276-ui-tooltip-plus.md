# 11.276 -- Tooltip primitive enhancements (v1.11.294)

Component-scope-only enhancement. No daemon-side change and no `c4`
CLI surface change. Existing Tooltip API is preserved byte-for-byte;
new affordances opt in via props.

## Why

Operators chaining hovers between adjacent tooltipped icons saw the
tooltip flicker because the prior single `delayMs` value applied to
both show AND hide directions. Splitting the knobs (`showDelay` long
enough to gate noise, `hideDelay` long enough to bridge hops) keeps
the tooltip stable while scanning a row of related controls. The
`arrow` glyph addresses a visibility regression in dense layouts:
when two tooltips overlap, the operator could not tell which
trigger each body was anchored to without the chevron pointer.

## What changed

### `web/src/components/ui/tooltip.tsx`

- `showDelay?: number` / `hideDelay?: number` props. Override
  `delayMs` per direction. Back-compat: when only `delayMs` is set,
  both directions inherit it (and `hideDelay` falls back to 0,
  matching the prior behaviour).
- `arrow?: boolean` prop. When true, renders a small CSS-only
  rotated-square triangle docked at the popover edge that points
  toward the trigger. Per-placement positioning (`top` -> bottom
  edge, `bottom` -> top edge, etc). `data-tooltip-arrow="true"`
  selector for e2e.
- `data-section="tooltip"` + `data-placement=<placement>` +
  `data-arrow="true|false"` + `data-visible="true|false"` on the
  wrapper root.
- Unmount-safe timer cleanup via useEffect so a delayed show /
  hide does not fire against an unmounted component.

### Adopted

- `web/src/components/HistoryView.tsx` -- sort-direction Tooltip
  picked up `arrow showDelay={120} hideDelay={80}`.
- `web/src/pages/TokenUsage.tsx` -- range-button Tooltip picked up
  `arrow showDelay={150} hideDelay={80}`.
- `web/src/components/HelpTip.tsx` -- canonical Settings field hint
  wrapper picked up `arrow showDelay={150} hideDelay={100}`.

### Tests

- `web/src/components/ui/tooltip.test.tsx`: 8 new vitest cases
  (data-section + data-placement on wrapper, data-visible flips on
  open, arrow not rendered by default, arrow renders when `arrow`
  true, `showDelay` overrides delayMs for hover-in, `hideDelay`
  defers hover-out, `hideDelay=0` closes synchronously,
  unmount-while-delayed-show does not throw). 26 / 26 green.

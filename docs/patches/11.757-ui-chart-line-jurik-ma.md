# Patch 11.757 -- UI: chart-line-jurik-ma primitive

Released as `v1.11.775` on 2026-05-26.

## Scope

Add `<ChartLineJurikMa>` -- a pure-SVG single-panel chart with a
Jurik moving-average overlay (simplified open-source
formulation). The JMA cascades three coupled recurrences with
an adaptive lag-reduction parameterisation.

## Definition

Inputs:

- `data: ChartLineJurikMaPoint[]` where `ChartLineJurikMaPoint
  = { x; close }`. Bars with non-finite fields are dropped.
- `length: number` -- the JMA lookback. Default `14`.
  Normalized to an integer `>= 2`.
- `phase: number` -- the phase adjustment. Default `0`.
  Normalized to a finite in `[-100, 100]`.
- `power: number` -- the power exponent. Default `2`. Normalized
  to a finite in `[1, 4]`.

Algorithm:

```
beta       = 0.45 * (length - 1) / (0.45 * (length - 1) + 2)
alpha      = beta ^ power
phaseRatio = phase / 100 + 1.5

For each bar i >= 1:
  e0[i] = (1 - alpha) * close + alpha * e0[i - 1]
  e1[i] = (close - e0) * (1 - beta) + beta * e1[i - 1]
  e2[i] = (e0 + phaseRatio * e1 - jma[i - 1])
          * (1 - alpha)^2 + alpha^2 * e2[i - 1]
  jma[i] = jma[i - 1] + e2

Seeds at i = 0: e0[0] = close[0], e1[0] = 0, e2[0] = 0,
                jma[0] = close[0].
```

Non-finite close nulls the bar and resets the recurrence (the
next finite close re-seeds the chain).

## Bit-exact algebraic anchor

**Stationary fixed point**: with the seeds we use, a constant
close is a fixed point of every recurrence:

- `e0[i] = (1 - alpha) * K + alpha * K = K`. So `e0` stays at
  `K`.
- `e1[i] = (K - K) * (1 - beta) + beta * 0 = 0`. So `e1` stays
  at `0`.
- `e2[i] = (K + phaseRatio * 0 - K) * (1 - alpha)^2 + alpha^2 *
  0 = 0`. So `e2` stays at `0`.
- `jma[i] = jma[i - 1] + 0 = K`.

Therefore **`CONST_FLAT (close == K) -> jma = K`** bit-exact at
every bar, for any `(length, phase, power)` triple. Tests verify
this across multiple triples: `(2, -100, 1)`, `(50, 0, 2)`,
`(200, 100, 4)`, `(7, 50, 3)`.

**Translation invariance**: shifting every close by `+k` shifts
every variable by `+k` (each recurrence is affine in the close
and the prior state). So `jma(close + k) = jma(close) + k` to
within floating-point rounding. Tests use `toBeCloseTo(_, 10)`.

## Summary

New `ChartLineWavetrend` UI primitive in
`web/src/components/ui/chart-line-wavetrend.tsx` -- a pure-SVG two-panel
line chart with a **WaveTrend Oscillator** panel from the smoothed
deviation of price off its average. The top panel plots the price; the
bottom panel plots WT1 and its signal line WT2.

## The computation

The WaveTrend Oscillator measures the smoothed deviation of the price off
its own moving average. It is a close-only indicator, so
`ChartLineWavetrendPoint` carries `{x, value}`.

- `computeLineWavetrendEma(values, period)` -- the EMA, in the incremental
  form `prev + alpha*(v - prev)` so a constant series stays exactly
  constant; `computeLineWavetrendSma(values, period)` -- the SMA.
- `computeLineWavetrend(values, options)` -- the pipeline: the EMA
  baseline `esa`, the EMA of the absolute deviation from it `d`, the
  channel index `ci = (price - esa) / (0.015 * d)` (guarded to 0 when the
  average deviation is zero), the EMA of `ci` into **WT1** and a
  four-period SMA of WT1 into the signal line **WT2**.
- `classifyLineWavetrendZone(wt1, upper, lower)` -- marks a bar
  **overbought** above the upper threshold, **oversold** below the lower,
  **neutral** between.

## Worked fixture

The EMA cascade and the `0.015` divisor make the WaveTrend transcendental
for a varying series, but a constant series collapses to exact zero. With
a flat price the EMA baseline equals the price, so the absolute deviation
is zero, the average deviation is zero, the channel index is guarded to 0,
and both WT1 and WT2 are exactly 0. The test asserts every WT1 reading is
0 and every defined WT2 reading is 0 for a constant series.

The EMA helper is checked with a dyadic alpha (`[4,8,16]` -> `[4,6,11]`
for period 3), the SMA helper with `[10,20,30,40]` -> `[null,15,25,35]`,
and WT2 is verified to be the four-period SMA of WT1. The zone classifier
is tested directly. The run / component use a swinging fixture and are
checked structurally.

## API

`data` (a `ChartLineWavetrendPoint[]`), `channelLength` (default 10),
`averageLength` (default 21), `upperThreshold` (default 60),
`lowerThreshold` (default -60), `pricePanelRatio` / `gap`,
sizing/format/a11y props, `strokeWidth` / `dotRadius`, the colour props,
the toggle flags (`showAxis` / `showGrid` / `showDots` default false /
`showWt1` / `showWt2` / `showZeroLine` / `showThresholds` / `showMarkers`
/ `showTooltip` / `showConfigBadge` / `showLegend` / `animate`),
controlled + uncontrolled `hiddenSeries` (keys `price` / `wt1` / `wt2`),
`formatValue` / `formatX`, `onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineWavetrendFinitePoints`,
`normalizeLineWavetrendPeriod`, `computeLineWavetrendEma`,
`computeLineWavetrendSma`, `computeLineWavetrend`,
`classifyLineWavetrendZone`, `runLineWavetrend`,
`computeLineWavetrendLayout`, `describeLineWavetrendChart`, plus the
`DEFAULT_CHART_LINE_WAVETREND_*` constants,
`CHART_LINE_WAVETREND_SIGNAL_PERIOD` and the `ChartLineWavetrendZone`
type. Every exported name carries a `LineWavetrend` infix so the
`export *` barrel line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen
`data-section="chart-line-wavetrend-aria-desc"` description span; the
price line, WT1, WT2 and every dot / marker are
`role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-channel-length`,
`data-average-length`, `data-upper-threshold`, `data-lower-threshold`,
`data-wt1-final`, `data-overbought-count`, `data-oversold-count`,
`data-neutral-count`, `data-total-points` and `data-animate`.

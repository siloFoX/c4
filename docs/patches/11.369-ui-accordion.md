## Goal

Dispatch:

- Build `web/src/components/ui/accordion.tsx`
  -- single / multiple expanded mode,
  keyboard navigation, smooth expand /
  collapse animation with reduced-motion
  respect, ARIA disclosure pattern.

Existing primitive (11.272 / v1.11.290)
already covers the full contract:

| dispatch item | shipped at |
| --- | --- |
| single / multiple expanded mode | `mode: 'single' \| 'multi'` |
| keyboard navigation | ArrowUp / ArrowDown / Home / End (Enter / Space via native button) |
| smooth expand / collapse animation | `transition-all duration-200` on the panel + chevron rotation |
| reduced-motion respect | `useReducedMotion()` -- transition class drops |
| ARIA disclosure pattern | `aria-expanded` / `aria-controls` on trigger; `role="region"` + `aria-labelledby` on panel |
| controlled / uncontrolled | `openIds` / `defaultOpenIds` / `onOpenIdsChange` |
| roving tabindex | active trigger `tabindex=0`, rest `-1` |
| `disabled` items | per-item flag, skipped in focus cycle |

This patch ships three additive
ergonomics around the existing core. None of
the new props change the default render.

## API

```tsx
import { Accordion } from './components/ui/accordion';

// Legacy (byte-identical)
<Accordion items={ITEMS} mode="single" />

// Custom heading level (drop to h4 inside a panel that already has h3)
<Accordion items={ITEMS} headerLevel="h4" />

// Lazy panel render -- defer expensive widgets until first open
<Accordion items={ITEMS} renderMode="lazy" />

// Always-one-open accordion (single mode)
<Accordion items={ITEMS} mode="single" collapsible={false} />
```

## New props

### `headerLevel?: 'h2' | 'h3' | 'h4' | 'h5' | 'h6'`

Heading tag wrapping the trigger row. Default
`'h3'` matches the legacy markup
byte-for-byte. Use lower levels (h4-h6) when
the accordion nests under existing page
headings; use h2 for a hero FAQ at the top of
a page.

The tag is rendered via `React.createElement`
without a type-cast: the `HeaderTag` local is
typed as `'h2' | 'h3' | 'h4' | 'h5' | 'h6'` so
JSX accepts it directly.

### `renderMode?: 'always' | 'lazy'`

Panel content render strategy.

- `'always'` (default, legacy byte-identical):
  every panel mounts on first render; closed
  panels are `hidden` so React/effects inside
  still fire. Cheap for small surfaces;
  matches the existing test suite.
- `'lazy'`: a panel does not mount its
  `content` until the first time the item
  opens. Once mounted it stays mounted (so
  re-closing does NOT unmount). Useful when a
  panel hosts an expensive widget (chart,
  virtualized list, fetch-on-mount form).

A `defaultOpen` flag (or `defaultOpenIds`)
seeds the per-item "ever opened" set on first
render, so a `defaultOpen` item in lazy mode
still mounts on the first paint.

### `collapsible?: boolean`

Single mode only. When `false`, clicking the
active item is a no-op so the accordion
always has exactly one panel open. Default
`true` preserves the legacy "click-to-toggle-
closed" behaviour.

Multi mode ignores the flag (multi-open
accordions can always close every panel).

## Data attributes

| attr | values | notes |
| --- | --- | --- |
| `data-header-level` (root) | `h2`-`h6` | mirrors `headerLevel` |
| `data-collapsible` (root) | `'true'` / `'false'` | mirrors `collapsible` |
| `data-render-mode` (root) | `'always'` / `'lazy'` | mirrors `renderMode` |
| `data-accordion-item-mounted` (item) | `'true'` / `'false'` | flips to `'true'` once the panel has rendered its content |

The `*-mounted` attr is useful for downstream
tests/storyshots that want to assert lazy
mount without parsing the children tree.

## Pairs with existing primitives

- `<Collapsible>` / `<CollapsibleGroup>`
  (v1.11.177) -- lower-level disclosure
  primitives. Reach for those when the
  surface is a single stand-alone disclosure
  with no sibling items. The Accordion is
  the right pick for a list of related
  sections.
- `<Tabs>` -- horizontal/vertical
  selection of a SINGLE pane. Accordion is
  for multiple panes that can read in any
  order; Tabs is for one-of-N navigation.

## Out of scope

- Per-page swap of legacy accordions to the
  new ergonomics. All three new props are
  additive; every existing call site stays
  byte-identical at the default values.
- Animated content height measurement
  (replacing `max-h-[1000px]`). The 1000px
  ceiling stays for now -- the height-measure
  story belongs in a separate patch with its
  own perf budget.
- A `vertical` orientation. Accordions are
  always vertical by convention; the
  horizontal equivalent is `<Tabs
  orientation="horizontal">`.
- A `collapsible="all"` mode in `single`
  that hides ALL panels at startup. The
  current default already supports that
  (no `defaultOpenIds` -> nothing open
  until the user clicks).

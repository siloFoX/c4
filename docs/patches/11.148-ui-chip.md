# 11.148 -- UI Chip primitive

`web/src/components/ui/chip.tsx` -- inline tag with optional leading
icon, optional dismiss X (aria-label="Remove" by default,
keyboard-focusable), forwardRef.

Knobs:

- `variant`: `subtle` (default) | `solid` | `outline`
- `tone`: `neutral` (default) | `primary` | `success` | `warning` | `danger`
- `size`: `sm` (default, ~11px) | `md` (xs)
- `icon?: ReactNode` -- rendered before children with an aria-hidden wrapper.
- `onDismiss?: () => void` -- when provided, renders a small X button.
- `dismissLabel?: string` -- aria-label for the dismiss button (default `Remove`).

Tone classes pull from the existing ARPS Tailwind palette
(`bg-muted` / `bg-primary/15` / `bg-success/15` / etc.) via
`class-variance-authority` compound variants. `cn()` merges caller
className with the variant output.

## Adoption

1. `web/src/components/SpecialistsTagEditor.tsx` -- read-only `#tag`
   list switched from `<Badge variant="info">` to `<Chip tone="primary">`.
2. `web/src/pages/Templates.tsx` -- per-template `source` / `model` /
   `effort` / `profile` tags switched from `<Badge variant="outline">`
   to `<Chip variant="outline">`.
3. `web/src/components/RiskCheckResult.tsx` -- level / suggestedAction
   / wouldDeny / denyList badge row switched to `<Chip>` with local
   `LEVEL_CHIP_TONE` / `ACTION_CHIP_TONE` maps (critical/high collapse
   onto `danger`, medium -> `warning`, low -> `success`; allow/review/deny
   map to success/warning/danger). The privileged badge inside the
   static-intent rollup keeps `<Badge variant="destructive">` -- not in
   scope for the badge-row swap.

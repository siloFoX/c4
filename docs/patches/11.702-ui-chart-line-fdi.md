## Summary

New `ChartLineFdi` UI primitive in
`web/src/components/ui/chart-line-fdi.tsx` -- a pure-SVG two-panel line
chart with a **Fractal Dimension Index** panel scoring path roughness
over the lookback window. The top panel plots the price; the bottom panel
plots the FDI.

## The computation

The Fractal Dimension Index scores how rough the price path is. It is a
close-only indicator, so `ChartLineFdiPoint` carries `{x, value}`.

- `computeLineFdiValue(window)` -- scores one window: it normalizes the
  window into a unit square (x over `[0,1]`, price over `[0,1]` of the
  window range), measures the length `L` of the price curve and returns
  `1 + ln(L) / ln(N - 1)`, clamped to `[1, 2]`. A flat window is exactly
  1; a window of two or fewer points is a straight segment, also 1.
- `computeLineFdi(values, period)` -- rolls the score over the close
  series.
- `classifyLineFdiZone(fdi, upper, lower)` -- marks a bar **trending**
  below the lower threshold (a smooth path), **choppy** above the upper
  (a rough path), **neutral** between.

A smooth, straight path has a curve length near the unit diagonal, so the
FDI sits near 1; a jagged, space-filling path has a much longer curve, so
the FDI climbs toward 2.

## Worked fixture

The FDI involves a square root and a logarithm and is transcendental in
general, but two anchors are clean.

A **flat window** has a zero price range, so every normalized step is
zero, the curve length is exactly 1 (the sum of the equal x-steps), and
`ln(1) = 0` -- so the FDI is exactly 1. `computeLineFdiValue([8,8,8,8,8])`
is `1`, asserted with `toBe`.

A **straight ramp at period 5** has a curve length of `sqrt(2)` and
`ln(N-1) = ln(4) = 2*ln(2)`, so the FDI is
`1 + ln(sqrt(2))/ln(4) = 1.25` -- `computeLineFdiValue([10,20,30,40,50])`
is checked with `toBeCloseTo(1.25)`. A maximally jagged window clamps to
2, a jagged window scores above a smooth ramp, and the zone classifier is
tested directly.

## API

`data` (a `ChartLineFdiPoint[]`), `period` (default 30), `upperThreshold`
(default 1.6), `lowerThreshold` (default 1.4), `pricePanelRatio` / `gap`,
sizing/format/a11y props, `strokeWidth` / `dotRadius`, the colour props,
the toggle flags (`showAxis` / `showGrid` / `showDots` default false /
`showFdi` / `showThresholds` / `showMarkers` / `showTooltip` /
`showConfigBadge` / `showLegend` / `animate`), controlled + uncontrolled
`hiddenSeries` (keys `price` / `fdi`), `formatValue` / `formatX`,
`onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineFdiFinitePoints`, `normalizeLineFdiPeriod`
(coerces to an integer >= 2), `computeLineFdiValue`, `computeLineFdi`,
`classifyLineFdiZone`, `runLineFdi`, `computeLineFdiLayout`,
`describeLineFdiChart`, plus the `DEFAULT_CHART_LINE_FDI_*` constants and
the `ChartLineFdiZone` type. Every exported name carries a `LineFdi`
infix so the `export *` barrel line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen `data-section="chart-line-fdi-aria-desc"`
description span; the price line, the FDI line and every dot / marker are
`role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-period`, `data-upper-threshold`,
`data-lower-threshold`, `data-fdi-final`, `data-trending-count`,
`data-choppy-count`, `data-neutral-count`, `data-total-points` and
`data-animate`.

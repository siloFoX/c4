## Summary

New `ChartLineRwi` UI primitive at
`web/src/components/ui/chart-line-rwi.tsx` -- a pure-SVG line chart
with a Random Walk Index panel plotting the RWI-high and RWI-low
trend-strength lines. The close price line occupies the top panel;
the bottom panel renders the two RWI lines on a data-driven y-axis
with a dashed threshold reference line.

## Indicator math

E. Michael Poulos's Random Walk Index. For each bar `i` and each
lookback `k` from 2 to `period`:

```
TR[i]   = max(high - low, |high - prevClose|, |low - prevClose|)
atr(k)  = mean of the True Range over the k bars ending at i
RWI-high = max over k of  (high[i] - low[i-k])  / (atr(k) * sqrt(k))
RWI-low  = max over k of  (high[i-k] - low[i])  / (atr(k) * sqrt(k))
```

The numerator is the actual high-low displacement over the
lookback. The denominator is what a random walk of the same
volatility would be expected to cover -- the average True Range
scaled by `sqrt(k)`, because a random walk's expected travel grows
as the square root of elapsed time. Their ratio is the Random Walk
Index: a value **above 1** means the price has travelled further
than random noise would explain, i.e. a real trend; **at or below
1** the move is within random-walk expectations.

Both lines are clamped to be non-negative (a negative displacement
reads 0) and are defined from index `period` onward (all lookbacks
`2..period` need `period` bars of history). A zero-volatility window
reads 0 rather than dividing by zero, and a period below 2 leaves
the index undefined.

### Worked fixture

Seven bars, each with a high-low range of 4 and no gaps, so every
True Range is exactly 4 and the ATR of any window is 4. The price
climbs a clean staircase to a peak at bar 3 then descends:

```
period 3, atr = 4 throughout, lookbacks k = 2, 3
RWI-high = [.,.,., 2.5/sqrt3, 1.5/sqrt3, 0.5/sqrt3, 0]
         ~= [.,.,., 1.443376, 0.866025,  0.288675,  0]
RWI-low  = [.,.,., 0, 1/sqrt2,  sqrt2,    2.5/sqrt3]
         ~= [.,.,., 0, 0.707107, 1.414214, 1.443376]
```

The True Ranges, displacements and ATRs are exact integers; the
`sqrt(k)` factor makes the RWI values irrational, so tests assert
them with `toBeCloseTo` and keep the exact zeros as `toBe`.

## API

- `getLineRwiFinitePoints(points)` -- drop points with a non-finite
  `x`, `high`, `low` or `close`.
- `normalizeLineRwiPeriod(period, fallback)` -- coerce to a positive
  integer; non-finite or sub-1 falls back, fractional floors.
- `computeLineRwiTrueRange(highs, lows, closes)` -- the True Range
  series.
- `computeLineRwi(highs, lows, closes, period)` -- returns
  `{ trueRange, rwiHigh, rwiLow }`; the RWI lines are null through
  the warm-up.
- `runLineRwi(points, options?)` -- sort finite points by `x`,
  compute the series, return `{ series, period, trendThreshold,
  trueRange, rwiHigh, rwiLow, samples, rwiHighFinal, rwiLowFinal,
  uptrendCount, downtrendCount, ok }`. Each sample is tagged
  `uptrend` / `downtrend` / `ranging`. `ok` is false for fewer than
  two points.
- `computeLineRwiLayout(options)` -- stack a price panel above an
  RWI panel with a zero-anchored y-domain. Returns panels, ticks,
  the price path, the two RWI paths, per-line markers, and the
  threshold line.
- `describeLineRwiChart(data, options?)` -- ARIA description.

## Component

`ChartLineRwi` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge, a three-series legend (Price / RWI-H / RWI-L) with toggle,
the dashed threshold line, the RWI-high and RWI-low lines with
per-line markers, a hover/focus tooltip exposing the close, both RWI
readings and the state, and a `motion-safe:animate-fade-in`
entrance. Every node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-adx** -- the ADX rates trend *strength* from
  smoothed directional movement (the +DI / -DI spread). The RWI
  benchmarks the price's *displacement* against a random walk of
  equal volatility -- it asks whether the move is statistically
  larger than noise, not how persistent the direction has been, and
  splits the answer into an explicit up line and down line.
- **vs chart-line-choppiness** -- the Choppiness Index divides a
  single windowed sum of True Range by the window extent for a
  bounded 0..100 trend-vs-range rating. The RWI takes the *maximum
  over many lookbacks* of a displacement-to-`sqrt(k)`-ATR ratio,
  producing an unbounded pair of lines whose dominant member also
  signals trend *direction*.
- **vs chart-line-kama** -- the KAMA uses an efficiency ratio to
  *adapt a moving average*. The RWI produces no average at all -- it
  is a direct statistical-significance score for the move itself.

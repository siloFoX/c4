## Goal

Dispatch:

- Build
  `web/src/components/ui/timeline-step.tsx`
  -- vertical step indicator with
  completed / current / pending
  states, connector lines, custom
  icon slot.

`<Stepper>` (11.252 / 11.376)
already ships the full multi-step
composer that walks a
`StepperStep[]` array. This patch
adds the building block as a
standalone primitive so hosts can
assemble custom timeline layouts
(audit log entries, release-cut
milestones, etc.) without
adopting Stepper's full surface.

## API

```ts
type TimelineStepState =
  | 'completed'
  | 'current'
  | 'pending'
  | 'error';

type TimelineStepSize = 'sm' | 'md' | 'lg';
type TimelineStepConnectorState =
  | 'completed'
  | 'pending';

interface TimelineStepProps {
  state?: TimelineStepState;
    // default 'pending'
  icon?: ReactNode;
    // overrides default icon
  label?: ReactNode;
  description?: ReactNode;
  meta?: ReactNode;
    // right-rail timestamp slot
  showConnectorBefore?: boolean;
    // default false
  showConnectorAfter?: boolean;
    // default false
  connectorState?:
    TimelineStepConnectorState;
    // overrides per-side derivation
  size?: TimelineStepSize;
    // default 'md'
  ariaLabel?: string;
  className?: string;
  // ...HTMLLIElement attributes
}
```

## Behaviour

### States

| state | circle | default icon | label colour |
| --- | --- | --- | --- |
| completed | emerald + check | Check | foreground |
| current | primary + ring | (none) | foreground semibold |
| pending | bg + border | (none) | muted-foreground |
| error | rose + alert | AlertCircle | rose |

### Sizes

| size | circle | icon | label | gap |
| --- | --- | --- | --- | --- |
| sm | h-5 w-5 | h-2.5 w-2.5 | text-xs | gap-2 |
| md (default) | h-6 w-6 | h-3 w-3 | text-sm | gap-3 |
| lg | h-8 w-8 | h-4 w-4 | text-base | gap-4 |

### Connector lines

`showConnectorBefore` /
`showConnectorAfter` flags
control the vertical rail
segments above / below the
circle. When false on a side,
a same-height spacer renders
so the circle stays vertically
centred regardless of label
length.

Per-side state derivation
(when `connectorState` is not
supplied):

| step state | before | after |
| --- | --- | --- |
| completed | completed | completed |
| current | completed | pending |
| error | completed | pending |
| pending | pending | pending |

`connectorState` prop
overrides both sides
uniformly when the host needs
explicit control (e.g., a
disconnected step at the
middle of a flow).

### Custom icon slot

`icon` prop replaces the
default. Default icons:

- completed -> `<Check>`
- error -> `<AlertCircle>`
- current / pending -> none

When an icon is provided
(custom or default), the
inner `<span>` carries
`data-icon-kind="custom"` (or
`data-icon-kind="default-
<state>"`) so tests can tell
which path rendered.

### Right-rail content

`label` + `description` +
`meta` slots render in a
column to the right of the
rail. The entire content
block is omitted when all
three slots are undefined --
useful for icon-only timeline
strips.

### ARIA

- `<li role="listitem">` so
  the step fits cleanly into a
  parent `<ol>` or `<ul>`.
- `aria-current="step"` when
  `state === 'current'`. Omits
  on other states.
- `ariaLabel` prop adds an
  explicit label for the step
  (use when label/description
  are JSX nodes rather than
  strings).

## Pure helpers (exported)

```ts
export const TIMELINE_STEP_STATE_CLASS: Record<
  TimelineStepState,
  { circle: string; icon: string; label: string; description: string }
>;

export function isTimelineStepReachable(
  state: TimelineStepState,
): boolean;

export function getTimelineStepDefaultIcon(
  state: TimelineStepState,
): ReactNode;

export function getTimelineStepAriaCurrent(
  state: TimelineStepState,
): 'step' | undefined;
```

- `isTimelineStepReachable`:
  true for completed +
  current; false for pending +
  error. Used by hosts that
  gate click handlers on
  reachable steps.
- `getTimelineStepDefaultIcon`:
  Check / AlertCircle / null
  per state.
- `getTimelineStepAriaCurrent`:
  'step' only for current.

## Pairs with existing primitives

- `<Stepper>` (11.252 /
  11.376) -- full multi-step
  composer. TimelineStep is
  the per-row primitive
  inside or outside Stepper.
- `<Timeline>` -- log /
  activity timeline (existing
  layout). TimelineStep is
  the wizard-flavoured row
  variant.
- `<StatusIndicator>` (11.414)
  -- inline state dot. The
  circle on TimelineStep
  serves the same purpose
  but locks into a
  prescribed wizard layout.
- ThemeCustomizer (11.394) --
  the step reads token-based
  classes so it auto-themes.

## Out of scope (deferred)

- Horizontal orientation.
  Dispatched contract is
  vertical; `<Stepper>` covers
  the horizontal variant.
- Click handler on the step
  (jump-to-step). Adopters
  wrap with their own button /
  anchor; the primitive stays
  presentational.
- Animated connector fill
  ("progress bar" between
  steps). Snap to new state
  for v1; CSS transitions
  belong in a follow-on motion
  layer.
- Tooltip slot on the circle.
  Host wraps with the
  existing `<Tooltip>`
  primitive.
- Per-cell drag-to-reorder.
  Wizard order is canonical.

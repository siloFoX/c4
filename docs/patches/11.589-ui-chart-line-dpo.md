## Summary

New `ChartLineDpo` UI primitive at
`web/src/components/ui/chart-line-dpo.tsx` -- a pure-SVG line chart
with a Detrended Price Oscillator panel removing the long-term trend
to expose cycles. The value line occupies the top panel; the bottom
panel renders the DPO on a symmetric y-axis with a dashed zero
centerline and sign-coloured markers. Both panels share one x-axis.

## Indicator math

`computeLineDpoShift(period)` is the DPO displacement,
`floor(period / 2) + 1`.

`computeLineDpo(values, period)` is the Detrended Price Oscillator.
For each index the simple moving average over `period` values is
subtracted from the price `shift` bars earlier:

```
shift  = floor(period / 2) + 1
SMA[i] = mean(values[i-period+1 .. i])
DPO[i] = value[i - shift] - SMA[i]
```

The displaced subtraction removes the longer trend and leaves the
shorter cycle oscillating around zero. Indices before both windows
are available read `null`; a flat series reads 0 throughout.

### Worked fixture

`values = [20, 12, 16, 16, 8, 8]`, `period = 4` (so `shift = 3`):

```
i=3 SMA mean(20,12,16,16) = 16 -> value[0]=20 - 16 = 4
i=4 SMA mean(12,16,16,8)  = 13 -> value[1]=12 - 13 = -1
i=5 SMA mean(16,16,8,8)   = 12 -> value[2]=16 - 12 = 4

dpo = [.,.,.,4, -1, 4]
```

`positiveCount = 2`, `negativeCount = 1`, `dpoFinal = 4`.

## API

- `getLineDpoFinitePoints(points)` -- drop points with a non-finite
  `x` or `value`.
- `normalizeLineDpoPeriod(period, fallback)` -- coerce a period to a
  positive integer; non-finite or sub-1 falls back, fractional floors.
- `computeLineDpoShift(period)` -- the displacement `floor(p/2) + 1`.
- `computeLineDpo(values, period)` -- the DPO series.
- `runLineDpo(points, options?)` -- sort finite points by `x`, compute
  the DPO series, classify each reading's sign, return `{ series,
  period, shift, dpo, samples, dpoFinal, dpoMin, dpoMax,
  positiveCount, negativeCount, ok }`. `ok` is false for fewer than
  two points.
- `computeLineDpoLayout(options)` -- stack a value panel above a DPO
  panel with a symmetric y-bound. Returns panels, ticks, the value
  path, DPO path, sign-tagged markers, and the zero centerline `y`.
- `describeLineDpoChart(data, options?)` -- ARIA description string.

## Component

`ChartLineDpo` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge (period + displacement shift), a two-series legend (Value /
DPO) with toggle, a dashed zero centerline, the DPO line,
sign-coloured markers (positive green / negative red), a hover
tooltip exposing the value, DPO, and sign, and a
`motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs chart-line-trix / chart-line-macd** -- those oscillators build
  on exponential moving averages (a triple-smoothed EMA rate of
  change; the difference of two EMAs). DPO subtracts a single *simple*
  moving average and, critically, *displaces* it backward by half the
  period. That displacement is the defining trait: it aligns the
  average with the cycle's natural lag so the trend cancels cleanly,
  which a non-displaced residual or moving-average difference does not
  do. DPO is explicitly *not* a momentum indicator -- it is a
  cycle-isolation tool.

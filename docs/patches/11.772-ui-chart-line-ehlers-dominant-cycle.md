# 11.772 chart-line-ehlers-dominant-cycle (v1.11.790)

## Summary

Pure-SVG dual-panel React/TypeScript chart primitive with an
**Ehlers Dominant Cycle** period panel beneath the close. The
dominant cycle is derived from the Hilbert transform phase
difference of the detrended close. Added as the
`<ChartLineEhlersDominantCycle />` UI primitive in
`web/src/components/ui/`.

## Algorithm

For each bar:

```
detrender[i] = 0.0962 * close[i] + 0.5769 * close[i - 2]
             - 0.5769 * close[i - 4] - 0.0962 * close[i - 6]
I[i]         = detrender[i - 3]                      (in-phase)
Q[i]         = 0.0962 * detrender[i] + 0.5769 * detrender[i - 2]
             - 0.5769 * detrender[i - 4] - 0.0962 * detrender[i - 6]
phase[i]     = atan2(Q[i], I[i])                     (radians)
dPhase[i]    = unwrap(phase[i - 1] - phase[i])       (in (0, 2*pi])
period[i]    = 2 * pi / dPhase[i]
period[i]    clamped to [minPeriod, maxPeriod]       (defaults 6, 50)
```

The 6-tap Hilbert FIR uses Ehlers' canonical coefficients
`(0.0962, 0, 0.5769, 0, -0.5769, 0, -0.0962)`. Bars before
`i = 10` are `null` (warmup). When both `|I|` and `|Q|` are
below `PHASE_EPSILON = 1e-10` the phase is undefined and the
bar yields `maxPeriod` (treated as singular: no oscillation).

## Anchors

- **CONST_FLAT (close == K)**: the FIR collapses to within a
  few ULPs of zero (bit-exact only at K = 0). Both `|I|` and
  `|Q|` fall under the epsilon, so the phase is undefined and
  the period clamps to `maxPeriod` bit-exact at every bar past
  the warmup. Verified across `(K, maxPeriod) in {0, 1, 5, 10,
  100, -3} x {25, 40, 50, 100}` (24 combos).
- **K = 0 (bit-exact)**: the FIR yields exactly 0 in IEEE 754.
- **Sinusoid of period P**: produces a finite in-band period
  reading after the warmup window.
- **Period bounds**: `minPeriod <= period <= maxPeriod` for any
  finite input.
- **Invalid bounds**: when the user supplies `maxPeriod <=
  minPeriod`, the implementation bumps `maxPeriod` to
  `minPeriod + 1`.

## Goal

Polish `web/src/pages/Notifications.tsx` to bring the
unified lifecycle feed up to the rest-of-app rhythm:

- Tabs for kind filtering with count chips.
- BadgeCounter for the unread count next to the
  Mark-all-read button.
- ScrollArea cap on the Timeline so long feeds stay
  inside the viewport.
- Success Toast on the Mark-all-read action.
- TimeAgo timestamps already shipped in v1.11.289 and
  are preserved.

Component-scope only. No daemon endpoint or CLI surface
change. The v1.11.262 Clear-all + UndoToast flow is left
unchanged.

## Filter strip

The prior hand-rolled `<button><Chip>` per-filter cluster
is replaced by a single `Tabs` primitive instance. Six
options:

| value | label |
| --- | --- |
| `all` | All |
| `dispatch` | Dispatch |
| `complete` | Complete |
| `halt` | Halt |
| `escalation` | Escalation |
| `system` | System |

Each label renders inline with a count chip
(`data-testid="notifications-tab-count-<key>"`) carrying
the per-kind total derived from `items`.

Compat: the outer wrapper keeps `role="group"` and
`aria-label="Filter notifications by type"`, and each
label span keeps `data-filter="<key>"`, so the existing
test queries (`getByRole('group', { name: ... })` +
`querySelector('[data-filter="halt"]')`) still match. Click
events from the inner span bubble up to the tab `<button>`
which the Tabs primitive owns.

## BadgeCounter

A `BadgeCounter` chip sits between the Mark-all-read
button and the rest of the action cluster:

```tsx
{totalUnread > 0 ? (
  <BadgeCounter
    count={totalUnread}
    tone="accent"
    size="sm"
    srLabel={`${totalUnread} unread notification[s]`}
    data-testid="notifications-unread-counter"
  />
) : null}
```

Hidden when there are zero unread items.

## ScrollArea

The Timeline render is wrapped in:

```tsx
<ScrollArea
  axis="y"
  data-testid="notifications-scrollarea"
  className="max-h-[60vh]"
>
  <Timeline items={timelineItems} groupByDay />
</ScrollArea>
```

`max-h-[60vh]` caps the feed at 60% of the viewport so a
50+ item Timeline does not push the toolbar + filter
strip below the fold. Below the cap the Timeline
scrolls inside its own shell.

## Success Toast on Mark-all-read

`handleMarkAllRead` snapshots `totalUnread` before
flipping `readAt`, then fires:

```ts
showToast(
  `Marked ${unreadBefore} notification[s] as read.`,
  'success',
);
```

Skips the toast when the count was zero. The Toast slot
is the standard fixed-position cluster used by Snapshots
/ Queue / Profiles / Templates.

## Tests

`web/src/pages/Notifications.test.tsx` is updated to add
five cases on top of the existing seventeen. New cases:

1. BadgeCounter chip shows the unread count;
2. BadgeCounter chip hides when there are no unread
   items;
3. Mark-all-read fires a success toast with the marked
   count;
4. per-kind count chips render correct totals;
5. timeline wraps in a `ScrollArea`.

22/22 cases pass against vitest 4.1.5 + jsdom 29.1.1.

## Adoption sites

The Notifications page is the only adopter. The Tabs +
BadgeCounter + ScrollArea + Toast primitives come from
the existing ui barrel and were already adopted by
Snapshots / Queue / Templates / Profiles in earlier
patches.

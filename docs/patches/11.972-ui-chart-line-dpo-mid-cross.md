## Summary

Adds the `ChartLineDpoMidCross` primitive
(`web/src/components/ui/chart-line-dpo-mid-cross.tsx`): a pure SVG
dual-panel chart that visualizes Detrended Price Oscillator (DPO)
crossings through the 0 midline for detrended momentum centerline
regime transitions with bias coloring. Companion to
`chart-line-dpo-zero-cross` (TODO 11.953): shares the same DPO
pipeline (`close[i - shift] - SMA(close, length)` with
`shift = floor(length/2) + 1`) and defaults (length = 20,

regime transition" to "centerline regime transitions with bias

(`chart-line-rsi-mid-cross`, `chart-line-stoch-mid-cross`,
`chart-line-cci-mid-cross`, `chart-line-mfi-mid-cross`,

`chart-line-rmi-mid-cross`).

## Pipeline

DPO removes the trend component from price by subtracting a
displaced SMA:

1. `shift = floor(length / 2) + 1`
2. `smaN = SMA(close, length)` (with CONST short-circuit via
   `winMin === winMax`)
3. `dpo[i] = close[i - shift] - smaN[i]` (for i >= shift,
   smaN[i] != null, and close[i - shift] != null)
4. Regime classifier (strict `>=`):
   - `dpo >= 0` -> bullish
   - `dpo <  0` -> bearish
5. Cross detector emits a `bullish` cross when
   `dpo[i-1] <= 0 && dpo[i] > 0` and a `bearish` cross when
   `dpo[i-1] >= 0 && dpo[i] < 0`. Strict inequality on the current
   side ensures the boundary value 0 does not double-fire.

## Defaults

- `length = 20` (canonical DPO window)
- `threshold = 0` (midline / centerline)
- Derived `shift = 11` (floor(20/2) + 1)
- Warmup boundary: bar `length - 1 = 19`, so the first 19 samples
  (40-bar run) are null and the remaining 21 are populated.

## Bit-exact anchors

- CONST `close = K` for all `K in {0, 1, 42, 100, 1234}` ->
  `dpo[i] === 0` literal (NOT `-0`; verified via `Object.is`) for
  every `i >= 19`. Regime classifies as bullish because 0 sits on
  the threshold (`dpo >= threshold`).
- LINEAR UP `close = i` -> `dpo[i] === -1.5` constant from i = 19
  onward (`(i - 11) - (i - 9.5) = -1.5`). The classic DPO
  "look-back inversion": shifted close is further back in time
  than the SMA centroid, so an upward trend yields a NEGATIVE DPO.
- LINEAR DOWN `close = -i` -> `dpo[i] === +1.5` constant -- same
  inversion in reverse, regime bullish.
- CONST K full-run counters under defaults, 40 bars:
  `noneCount = 19`, `bullishCount = 21`, `bearishCount = 0`,
  `crosses.length = 0`.
- LINEAR UP / LINEAR DOWN under defaults, 60 bars: all 41
  cleared samples land in the same regime (bearish for UP,
  bullish for DOWN), 0 crosses.
- `pricePath` and `dpoPath` are byte-identical across repeated
  layout calls, with `.toFixed(2)` precision and `M ... L ...`
  command shapes. The dpo path uses a single `M` for post-warmup
  data.
- Cross detector boundary: strict inequality (`> T`, `< T`).
- Layout auto-fit: when dpo collapses to a single value (CONST
  case, dpo = 0), the panel uses the `threshold +/- 1` fallback
  (oscMin = -1, oscMax = 1).
- Rise-then-decline trajectories cross UP through 0 (bearish ->
  bullish via the inversion), confirming the documented look-back
  semantics.

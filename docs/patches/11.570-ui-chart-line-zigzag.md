# 11.570 -- web/src/components/ui/chart-line-zigzag.tsx

Pure-SVG line chart with a **ZigZag** overlay. The ZigZag
indicator filters out every reversal smaller than a percent
threshold and connects only the **significant pivots** --
the swing highs and swing lows that survive the filter. The
raw series renders as a dimmed line beneath the bold ZigZag,
so the major trend structure stands out from the noise.

## The ZigZag filter

`runLineZigzag(points, thresholdPercent)`:

1. The series is sorted by x. The first point is always the
   starting pivot.
2. Scanning forward, a running high and a running low since
   the last confirmed pivot are tracked.
3. When price retraces from the running extreme by at least
   the percent threshold, that extreme is confirmed as a
   pivot -- a `peak` off a running high, a `trough` off a
   running low -- and the trend reverses. The scan rewinds
   to the confirmed pivot so the next swing's running
   extreme is built correctly.
4. The final point is appended as an `end` pivot unless it
   is already the last confirmed pivot.

A pivot takes the **extreme value of its swing**, not the
value at the confirmation point. Each pivot also carries
`moveFromPrev` -- the percent move from the previous pivot.

The component fixture `[100, 102, 110, 105, 95, 98, 120,
115, 90]` makes the filtering visible: at a **5%** threshold
all five pivots survive -- `start, peak(110), trough(95),
peak(120), end` -- but at **20%** the 110 peak is filtered
out, because the retracement from 110 to 95 is only 13.6%,
below the threshold (4 pivots). At a threshold no swing can
clear, only `start` and `end` remain.

## Distinct from neighbouring primitives

- `chart-line-changepoint` detects statistical changepoints
  (shifts in the mean). The ZigZag detects **swing pivots**
  by a percent-reversal rule -- a geometric filter, not a
  statistical test.
- `chart-line-min-max` draws a rolling min/max envelope; it
  never connects discrete pivots.
- `chart-line-trend` / `chart-line-segment` fit or split a
  line; neither filters reversals by magnitude.

The ZigZag's defining transform is the percent-reversal
pivot rule. All exported helpers carry a `LineZigzag` infix
so the `export *` barrel line cannot collide.

## Public surface

- `<ChartLineZigzag data={...} />` -- a single
  `ChartLineZigzagPoint[]` (`{x, value}`).
- `thresholdPercent` -- the reversal threshold as a percent
  (default 5; coerced to a non-negative number).
- Toggle flags: `showAxis`, `showGrid`, `showRawLine`,
  `showRawDots` (default false), `showZigzag`, `showPivots`,
  `showTooltip`, `showConfigBadge`, `showFooter`, `animate`.
- Standard sizing + format + a11y props. `formatValue` /
  `formatX`.
- `onPivotClick({pivot})`.

## Pure helpers

- `getLineZigzagFinitePoints(points)`.
- `normalizeLineZigzagThreshold(thresholdPercent)`.
- `runLineZigzag(points, thresholdPercent?)` -- the ZigZag
  filter; `ok` requires at least 2 finite samples.
- `computeLineZigzagLayout({...})` -- projects the raw line,
  the ZigZag overlay path and the pivot markers.
- `describeLineZigzagChart(data, options?)`.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- The ZigZag overlay path and every pivot marker
  `role="graphics-symbol"` + `tabIndex={0}`.
- Every node carries `data-section="chart-line-zigzag-..."`.
- Root mirrors `data-pivot-count`, `data-threshold-percent`,
  `data-total-points`, `data-animate`.
- The raw path carries `data-kind="raw"`, the overlay
  `data-kind="zigzag"`. Pivot markers expose `data-kind`
  (`start` / `peak` / `trough` / `end`), `data-x`,
  `data-value`, `data-move`.

## Tooltip / badge / footer

Hovering a pivot shows the pivot kind, x, value and the
percent move from the previous pivot. Config badge:
`ZZ thr=<percent>% pivots=<count>`. The footer reports the
threshold, the pivot count and the raw point count.

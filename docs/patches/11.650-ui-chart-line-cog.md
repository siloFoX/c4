panel from a weighted price sum.

## Indicator math

The Ehlers Center of Gravity (CoG) oscillator finds the
"center of gravity" -- the centroid -- of the prices in a
rolling window.

- `computeLineCogWeightedSum(window)` -- the weighted price sum
  of a window (ordered oldest first). Each bar is weighted by
  its age: the oldest bar carries the heaviest weight `m` and
  the most recent bar the lightest weight `1` (with `m` the
  window length):

```
numerator   = SUM[j] (m - j) * window[j]
denominator = SUM[j] window[j]
```

- `computeLineCog(values, period)` -- the Center of Gravity:

```
CoG[i] = -numerator / denominator + (period + 1) / 2
```

  the centroid `numerator / denominator` negated and offset by
  `(period + 1) / 2`. The offset is exactly the centroid of a
  flat window, so a flat series reads zero. A positive CoG marks
  recent prices outweighing older ones (a rising window), a
  negative CoG the reverse. A window whose prices sum to zero
  has an undefined centroid and is null, as are bars before the
  window is full.

### Worked fixture

The CoG is a ratio of two weighted sums plus a constant offset
-- no transcendentals, so the whole pipeline is exact. The
fixture is a period-4 periodic series `[4, 12, 20, 28]`: every
window sums to 64 (a power of two), so every CoG divides
cleanly. The weighted numerators cycle 120 / 168 / 184 / 168,
giving `CoG = -120/64 + 2.5 = 0.625`, `-168/64 + 2.5 = -0.125`,
`-184/64 + 2.5 = -0.375`. A flat series reads exactly zero.

## API

- `getLineCogFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineCogPeriod(period, fallback)` -- coerce to an
  integer of at least 2.
- `computeLineCogWeightedSum(window)` -- the
  `{ numerator, denominator }` weighted price sum of a window.
- `computeLineCog(values, period)` -- the Center of Gravity
  series, null through the warm-up and for zero-sum windows.
- `runLineCog(points, options?)` -- sort finite points by `x`,
  run the pipeline, return `{ series, period, cog, samples,
  cogFinal, cogMin, cogMax, positiveCount, negativeCount, ok }`.
  Each sample is classified `positive` / `negative` / `zero`.
  `ok` is false for fewer than two points.
- `computeLineCogLayout(options)` -- stack the price panel above
  a zero-centred CoG panel; project the price and CoG paths plus
  markers and the zero line.
- `describeLineCogChart(data, options?)` -- ARIA description.

## Component

`ChartLineCog` is a `forwardRef` region (`role="region"`) with
an ARIA-linked description, a `role="img"` SVG, two stacked
panels (price on top, CoG below) with their own axis ticks and
grid, panel labels, a config badge showing the period, a
two-series legend (Price / CoG) with toggle, a hover/focus
tooltip exposing the value, CoG and sign, and a
`motion-safe:animate-fade-in` entrance. It consumes
`{ x, value }` points. The CoG panel carries a dashed zero line
and sign-coloured markers (green positive, red negative, grey
zero). Every node carries a `data-section` attribute.

## Distinctness

- **vs the moving-average overlays (vwma / smma / ema)** --
  those average the price level. The CoG measures the centroid
  of the price window -- where its weight sits in time -- so it
  is a zero-centred oscillator on its own panel, not a price
  overlay.
- **vs the oscillator panels (roofing / trendflex / reflex)** --
  those derive their oscillator from a filter of the close. The
  CoG derives it from the time-position of the price mass: a
  weighted price sum normalized by the plain price sum.
- **vs chart-line-pmo** -- the PMO is a double-smoothed rate of
  change. The CoG is a single weighted-sum ratio with no
  smoothing.

# 11.301 -- FocusTrap primitive (v1.11.319)

New `FocusTrap` primitive at
`web/src/components/ui/focus-trap.tsx`. Component-scope only,
no daemon-side change and no `c4` CLI surface change.

## Why

The c4 web app already has the canonical focus-trap behaviour
implemented as a hook at
`web/src/hooks/use-focus-trap.ts`. The hook supports:

- First-focus: when the trap activates, focus moves to the
  explicit `initialFocusRef` if provided, else to the first
  focusable inside the container, else to the container
  itself (which is why the container needs `tabIndex=-1`).
- Tab wraparound: Tab from the last focusable returns to
  the first; Shift+Tab from the first jumps to the last.
- Escape handler slot: an optional `onEscape` callback fires
  on the Escape key with `stopPropagation` so outer Escape
  handlers do not double-fire.
- Restore-on-unmount: when the trap deactivates, focus
  returns to whatever element was focused before the trap
  mounted (toggleable via the `restoreFocusOnUnmount`
  option).

Three call sites currently wire the hook by hand:

```tsx
// dialog.tsx
const cardRef = useRef<HTMLDivElement>(null);
useFocusTrap(cardRef, { active: open, ...(closeOnEsc ? { onEscape: onClose } : {}) });
return <div ref={cardRef} tabIndex={-1}>...</div>;

// drawer.tsx + command-palette.tsx -- same shape
```

Each call site has to remember three things:
1. Create the ref.
2. Apply `tabIndex={-1}` to the container so the focus
   fallback works.
3. Conditionally spread the `onEscape` option to satisfy
   `exactOptionalPropertyTypes`.

Patch 11.301 ships a `<FocusTrap>` primitive that bundles
all three: ref creation, container element with the right
`tabIndex`, and the hook wiring. Callers go from "three
boilerplate steps" to a single declarative wrap.

## What changed

### `web/src/components/ui/focus-trap.tsx` (new file)

- `FocusTrap` forwardRef component. Internally:
  - Creates a `localRef` and calls `useFocusTrap` against
    it.
  - Renders a polymorphic container (`'div' | 'section' |
    'aside' | 'main'`, default `'div'`) with `tabIndex=-1`
    by default and the caller's children inside.
  - Spreads `data-section="focus-trap"` and
    `data-active="true|false"` onto the container for e2e
    + theming.
  - Forwards the ref to the underlying DOM node so callers
    that need to measure or portal the trap container can
    still reach it.
- Props (mirroring the hook plus a few ergonomic additions):
  - `children: ReactNode` -- required.
  - `active?: boolean` -- inert when false. Default true.
  - `initialFocusRef?: RefObject<HTMLElement>` -- explicit
    first-focus target.
  - `restoreFocusOnUnmount?: boolean` -- default true.
  - `onEscape?: () => void` -- Escape slot.
  - `as?: 'div' | 'section' | 'aside' | 'main'` --
    polymorphic tag override.
  - All standard `HTMLAttributes<HTMLElement>` pass through
    (role, aria-*, className, style, id).
  - `tabIndex` defaults to `-1` but caller can override.
- Re-exported from `web/src/components/ui/index.ts`.

### Adopted

- Primitive only this commit. The Dialog, Drawer, and
  CommandPalette call sites continue to use the
  `useFocusTrap` hook directly until the per-site adoption
  patches land (each will need its own test refresh because
  the wrapped DOM tree changes shape).

## Deferred (dispatch follow-ups)

- `web/src/components/ui/dialog.tsx` -- replace the
  `useFocusTrap(cardRef, ...)` + `tabIndex={-1}` pattern
  with `<FocusTrap active={open} onEscape={closeOnEsc ? onClose : undefined} role="dialog" aria-modal="true">`.
- `web/src/components/ui/drawer.tsx` -- same swap, with
  `role="dialog" aria-modal="true"`.
- `web/src/components/ui/command-palette.tsx` -- same swap
  plus the existing aria-* attrs.

## Tests

`web/src/components/ui/focus-trap.test.tsx` -- 19 vitest
cases:

- Renders a `<div>` with `data-section="focus-trap"` and
  `data-active="true"` by default.
- Focuses the first focusable child on activation.
- Focuses the explicit `initialFocusRef` target when
  provided.
- Falls back to focusing the container when there are no
  focusable children.
- Tab from the last focusable wraps to the first.
- Shift+Tab from the first focusable wraps to the last.
- `onEscape` fires on Escape key.
- No crash when `onEscape` is omitted and Escape is pressed.
- Restores focus to the previously focused element on
  unmount (default).
- Does NOT restore focus on unmount when
  `restoreFocusOnUnmount=false`.
- `active=false` sets `data-active="false"` and skips
  first-focus.
- Forwards ref to the underlying DOM node.
- Polymorphic `as="section"` renders a `<section>`.
- Polymorphic `as="aside"` renders an `<aside>`.
- Polymorphic `as="main"` renders a `<main>`.
- Forwards `className` + arbitrary HTML attributes
  (id, role, aria-labelledby) onto the container.
- Container has `tabIndex=-1` by default (focus fallback).
- Caller-supplied `tabIndex` overrides the default.
- Stable `displayName = 'FocusTrap'` for devtools.

All 19 green via
`vitest run --project unit src/components/ui/focus-trap.test.tsx`.

## Notes

- The primitive intentionally does NOT spread `tabIndex={-1}`
  unconditionally -- callers that want the container to be
  Tab-reachable (rare, but valid for nested traps) can set
  `tabIndex={0}` and the override survives.
- The hook already handles SSR safely (it bails when
  `typeof document === 'undefined'`), so the primitive does
  not need a separate SSR guard. The container is still
  rendered server-side; only the focus side-effects defer
  to the post-mount effect.
- The polymorphic tag choice is intentionally limited to
  `'div' | 'section' | 'aside' | 'main'` -- these cover the
  documented use cases (dialog content, drawer panel,
  command palette card) without exposing every intrinsic
  HTML tag. Adding more is a one-line addition if a future
  surface needs it.

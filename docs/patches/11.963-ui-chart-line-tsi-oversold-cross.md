# Patch 11.963 - UI: chart-line-tsi-oversold-cross primitive

Version: 1.11.981

## Summary

Add `web/src/components/ui/chart-line-tsi-oversold-cross.tsx` -- mirror
of the `chart-line-tsi-overbought-cross` primitive with the threshold
negated to track the canonical -25 oversold line. Bullish (= exit,
crossing UP through -25 = leaving oversold) and bearish (= entry,
crossing DOWN through -25 = entering oversold). Shares the bit-exact
TSI pipeline with the other TSI siblings.

## Pipeline

Identical to `chart-line-tsi-overbought-cross`. Threshold default
flipped to -25.

## Bit-exact anchors

| Pattern                  | TSI (after warmup) | Regime  |
|--------------------------|--------------------|---------|
| CONST close = K          | 0 (0/0 -> 0)       | bullish |
| LINEAR UP close = i      | 100                | bullish |
| LINEAR DOWN close = -i   | -100               | bearish |

CONST anchor tested bit-exactly across K in `{0, 1, 42, 100, 1234}`
(regime bullish since 0 >= -25). LINEAR DOWN reaches -100 (bearish,
below -25).

Cross events fire during transients:
* decline-then-rise: tsi transitions from -100 through -25 going up ->
  bullish exit.
* rise-then-decline: tsi transitions from 100 through -25 going down
  -> bearish entry.

## Layout

* Default dimensions: 720 x 460.
* Two panels: price panel on top (~55% of plot area), TSI panel below.
* TSI panel uses a fixed `[-100, 100]` range; threshold band sits
  below the midpoint (= 0) at the -25 line.
* Cross markers drawn as circles on the TSI panel at `(cx, cyOsc)`
  and as arrows on the price panel at `(cx, cyPrice)`.
* SVG path strings use `.toFixed(2)` for deterministic snapshot tests.

## Accessibility

* Wrapping `<div role="region" aria-label="TSI Oversold Cross chart">`.
* `<svg role="img">` with an aria-described-by linked `<desc>` sr-only
  text.
* Each cross marker: `role="graphics-symbol"` + `tabIndex={0}` +
  descriptive `aria-label` ("bullish cross at x=...").
* Legend toggles: `<button role="switch" aria-pressed="...">`.
* Motion-safe fade-in via `motion-safe:animate-fade-in` (opt-out with
  `animate={false}`).

# 11.216 -- UI: NumberFormat component (v1.11.234)

Component-scope-only addition. No daemon-side change and no `c4`
CLI surface change.

## Why

The web UI had ~17 inline `.toLocaleString()` callsites scattered
across feature pages, each rebuilding an implicit `Intl.NumberFormat`
on every render and silently producing `'NaN'` / `'Infinity'` for
non-finite inputs (token counters, byte / percent stat cards,
history counts). Wrapping these behind a single component plus a
helper gives us:

- One place to change locale resolution when the i18n surface grows
  past `'en' | 'ko'`.
- One place to enforce the em-dash fallback for non-finite numbers
  so stat cards stop leaking `'NaN'` into the UI.
- Cached `Intl.NumberFormat` instances per locale + style + digits +
  notation tuple so heavy lists (conversation token rows, meeting
  previews) stop re-allocating formatters per render.
- A consistent surface for compact notation (`1.5K` / `2.5M`) so we
  do not hand-roll suffix logic per call-site.

The `formatNumber()` helper covers the non-component callsites
(string-template values, `tFormat()` args) where mounting a `<span>`
would break the surrounding string concatenation -- same options
surface, same em-dash fallback.

## Surface

`web/src/components/NumberFormat.tsx` (~110 lines) exports:

```tsx
export type NumberStyle = 'decimal' | 'currency' | 'percent';

export interface NumberFormatOptions {
  style?: NumberStyle;            // default 'decimal'
  currency?: string;              // default 'USD' when style='currency'
  minimumFractionDigits?: number;
  maximumFractionDigits?: number;
  locale?: string;                // override; default useLocale()
  compact?: boolean;              // notation='compact'
}

export interface NumberFormatProps extends NumberFormatOptions {
  value: number;
  className?: string;
}

export function formatNumber(value: number, opts?: NumberFormatOptions): string;
export function NumberFormat(props: NumberFormatProps): JSX.Element;
```

- `value` non-finite (`NaN`, `Infinity`, `-Infinity`) -> em-dash.
- Component renders inline `<span>` with the formatted string and an
  optional `className`.
- Helper is pure -- safe to call from `lib/` (no React).

## Adoption

Five existing `.toLocaleString()` numeric callsites swapped to
`formatNumber()`:

- `web/src/lib/conversation-render.tsx` -- `formatTokens()`
  rewires its four `input` / `output` / `cacheRead` / `cacheCreate`
  parts.
- `web/src/components/MeetingsComposer.tsx` -- preview plan
  `estimatedTokens` interpolation.
- `web/src/components/ConversationView.tsx` -- conversation header
  turns count + input / output total tokens.

Date-formatting `.toLocaleString()` call-sites were deliberately
left alone -- those belong to the 11.210 date formatter track.

## Tests

`web/src/components/NumberFormat.test.tsx` -- 17 cases (vitest +
RTL):

- Default decimal grouping
- Currency style (USD)
- Percent style (multiplies by 100)
- Compact: 1500 -> 1.5K, 1500000 -> 1.5M
- minimumFractionDigits / maximumFractionDigits respected
- NaN / +Infinity / -Infinity render as em-dash
- Custom locale override (de-DE uses period for thousand separator)
- className forwarded to the rendered span
- formatNumber helper coverage (defaults, em-dash, compact, percent,
  locale override, EUR currency)

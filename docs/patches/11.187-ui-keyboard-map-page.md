# 11.187 -- UI: Keyboard Map page

## Why

`KeyboardShortcutsModal` exposes a 9-row compact reference for the
overlay invoked by `?` / `Shift+/`. Operators asked for a dedicated,
searchable surface that documents every binding wired into the web UI,
including overlay-internal keys (Escape closes Dialog/Drawer, Enter
activates the focused command palette item, Tab cycles focus) that
were never listed anywhere. The compact modal cannot grow without
becoming unscannable, so a full page lives separately.

## What

New `<KeyboardMap>` page at `web/src/pages/KeyboardMap.tsx`.

Layout (inside `<PageFrame>`):

- Header: `Keyboard Map` title + short description (i18n
  `keyboardMap.title` / `keyboardMap.description`).
- Top filter row: `<SearchBar size="md">` with both `onChange` and
  `onDebouncedChange` wired to the same `setFilter` so typing reacts
  immediately for tests and debounces in real input. Filter matches
  case-insensitively against the row's i18n description **or** the key
  combo string.
- Six grouped sections (`Navigation` / `Actions` / `View` / `Editing` /
  `Global` / `Custom`). Each section is a `<section
  data-keyboard-map-section="...">` with an `<h3>` heading; a
  `<Separator>` divides adjacent sections.
- Each row is a `<tr>` with a `<Kbd>` combo in the left column and the
  translated label in the right column.
- Empty state: `<EmptyState>` with the lucide `KeyboardOff` icon and
  `keyboardMap.empty.title` / `keyboardMap.empty.description` copy,
  rendered when the filter matches zero rows.

## Data source

The page imports `SHORTCUT_ROWS` + the `ShortcutCategory` type from
`web/src/components/KeyboardShortcutsModal.tsx` and concatenates an
`EXTRA_ROWS` list defined locally. Extras cover bindings the modal
omits:

- `Escape` -- close Dialog / Drawer (two rows, two descriptions).
- `Ctrl+K` -- open command palette.
- `Enter` -- activate focused command palette item.
- `Down` / `Up` -- move focus in a menu.
- `Tab` / `Shift+Tab` -- cycle next / previous item.
- `Ctrl+Enter` -- submit the active form.
- `Ctrl+]` -- detach from an attached worker session (matches
  `c4 attach` detach contract).
- `Ctrl+C` / `Ctrl+V` / `Ctrl+Z` / `Ctrl+Shift+Z` -- editing.

The merged list is exported as `ALL_ROWS` so future tests can lock the
total count without re-importing the modal constant.

## Nav integration

`web/src/pages/registry.ts` gains a new `keyboard-map` entry placed in
the `config` category with the lucide `Keyboard` icon, lazy-loaded via
`import('./KeyboardMap')`. en/ko bundles get matching `feature.keyboardMap.*`
keys + the per-row description keys (`keyboardMap.row.*`) and section
labels (`keyboardMap.section.*`).

## Tests

`web/src/pages/KeyboardMap.test.tsx` (vitest + RTL):

- renders the page title;
- renders at least one `<kbd>` element via the `Kbd` primitive;
- filters rows by label substring (`sidebar` keeps the sidebar row,
  drops `send chat`);
- filters rows by key combo (`Ctrl+]` keeps only the detach row);
- shows every section when the filter is empty (>= 4 sections);
- renders the empty state when no rows match;
- asserts the registry entry exists in the `config` category with the
  expected `labelKey`.

## Out of scope

- No mutation -- the page is read-only documentation.
- No deep-linking from the modal to the page (will land with the next
  patch once the modal grows a `View full map` link).
- The `Custom` section is reserved for user-defined bindings once a
  binding registry lands; today it renders empty (so the search filter
  naturally hides it).

# TODO 11.498 - chart-line-smooth primitive

Status: SHIPPED v1.11.516 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartLineSmooth>`, a pure-SVG smoothed line chart that supports
two distinct interpolation algorithms:

- **catmullRom** (default) - canonical Catmull-Rom-to-Bezier with a
  `tension` parameter clamped to `[0, 1]`. `tension=0` collapses to
  straight segments; `tension=0.5` (default) gives the standard
  Catmull-Rom weight; `tension=1` gives full smoothing.
- **monotone** - Fritsch-Carlson monotone cubic Hermite interpolation.
  Each segment's endpoint tangents are adjusted so the curve never
  overshoots adjacent y values. Use this for CDFs, probability data,
  or any series where visual overshoot would be misleading.

## API

```tsx
<ChartLineSmooth
  series={[
    {
      id: 'rps',
      label: 'Requests/s',
      data: [
        { x: 0, y: 12 },
        { x: 1, y: 80 },
        { x: 2, y: 45 },
        { x: 3, y: 90 },
      ],
    },
  ]}
  curve="monotone"
  xLabel="t (s)"
  yLabel="req/s"
/>
```

## Distinct from prior chart-* primitives

- `<ChartLine>` - linear segments between points; no curve
  interpolation.
- `<ChartLineStep>` (11.497) - horizontal / vertical step segments;
  no curve.
- `<ChartStream>` / `<ChartStreamgraph>` - stacked stream layers with
  per-stack curve; chart-line-smooth is non-stacked per-series lines.
- `<ChartArea>` - filled area below the curve; chart-line-smooth
  strokes only.
- `<ChartScatter>` - 2D points without connecting line.

## Layout

### Value range

- `getLineSmoothBounds` auto-derives `(xMin, xMax, yMin, yMax)` from
  every finite point across visible series; collapsed axes expand
  `+/- 0.5`; empty -> unit square `(0..1, 0..1)`.
- Explicit `xMin` / `xMax` / `yMin` / `yMax` props override per axis.

### Curve algorithms

The two exported builders both take pixel-space points and return
SVG path strings using `M ... C ... C ...` cubic Bezier commands. Empty
input -> empty path; single point -> a lone `M`.

#### `buildCatmullRomPath(points, tension)`

For each segment from `P[i]` to `P[i+1]` (with `P[i-1]` and `P[i+2]`
duplicated at the endpoints):

    CP1 = P[i]   + tension/3 * (P[i+1] - P[i-1])
    CP2 = P[i+1] - tension/3 * (P[i+2] - P[i])

`tension` is clamped to `[0, 1]`. `tension=0` makes `CP1 = P[i]` and
`CP2 = P[i+1]` so the curve collapses to a straight line. `tension=1`
gives full Catmull-Rom smoothing.

#### `buildMonotonePath(points)`

Fritsch-Carlson monotone cubic Hermite:

1. Secant slopes `d[i] = (y[i+1] - y[i]) / (x[i+1] - x[i])` (0 when
   `dx === 0` to avoid divide-by-zero on vertical segments).
2. Initial tangents `m[i] = (d[i-1] + d[i]) / 2` for interior; for the
   endpoints `m[0] = d[0]`, `m[N-1] = d[N-2]`.
3. Adjustment: if `d[i] === 0` force `m[i] = m[i+1] = 0`; else with
   `a = m[i] / d[i]`, `b = m[i+1] / d[i]`, if `a^2 + b^2 > 9` scale
   `(m[i], m[i+1])` by `tau = 3 / sqrt(a^2 + b^2)`.
4. Convert each Hermite segment to a cubic Bezier:

       CP1 = (x[i] + dx/3, y[i] + m[i] * dx / 3)
       CP2 = (x[i+1] - dx/3, y[i+1] - m[i+1] * dx / 3)

The monotone curve **requires** strictly increasing x for the
mathematical guarantee. `computeLineSmoothLayout` therefore sorts the
points by `px` before passing them to the monotone builder so adopter
data with out-of-order points still produces a valid path. The
catmullRom variant accepts any order and renders the points in input
sequence.

## Interactivity

- Hover or focus on a dot opens a tooltip showing series label, `x`
  (via `formatX`), and `y` (via `formatValue`).
- Hover or focus also dims every other series stroke to 0.3 and every
  other dot to 0.3 (the hovered series + dot stay at full opacity).
- Per-dot `onPointClick({series, point})` callback.
- Legend buttons toggle series visibility (controlled +
  uncontrolled).
- Mount fade via `motion-safe:animate-fade-in` (suppress via
  `animate=false`).

## Exported helpers

- `DEFAULT_CHART_LINE_SMOOTH_*` numeric / string defaults.
- `DEFAULT_CHART_LINE_SMOOTH_PALETTE` (10-colour palette).
- `getLineSmoothDefaultColor(index)` - palette wrap.
- `getLineSmoothFinitePoints(points)` - drops points with non-finite
  `x` or `y`.
- `getLineSmoothBounds(series, hidden?)` - per-axis bounds; collapsed
  expands `+/- 0.5`; empty -> unit square.
- `getLineSmoothTicks(min, max, count?)` - evenly-spaced ticks.
- `buildCatmullRomPath(points, tension?)` - canonical CR-to-Bezier
  builder; tension clamped to `[0, 1]`.
- `buildMonotonePath(points)` - canonical Fritsch-Carlson builder.
- `buildSmoothLinePath(points, curve, tension?)` - dispatcher.
- `computeLineSmoothLayout(input)` - per-series layout: pixel points
  + smooth path + tick positions. Monotone curve sorts points by px.
- `describeLineSmoothChart(series, hidden?, curve?, formatValue?)`
  - ARIA summary `"Smooth line chart (<curve>) with N series and K
  points. x range <xMin> to <xMax>, y range <yMin> to <yMax>."`.

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only
  `data-section="chart-line-smooth-aria-desc"` span.
- SVG `role="img"` with the same `aria-label`.
- Each path is `role="graphics-symbol"` with `tabIndex={0}` and
  `aria-label="<series>: <curve> smoothed line with K points"`.
- Each dot is `role="graphics-symbol"` with `tabIndex={0}` and
  `aria-label="<series>: x=<x>, y=<y>"`.

## Data attributes

- Root: `data-section="chart-line-smooth"`, `data-curve`,
  `data-tension`, `data-series-count`, `data-visible-series-count`,

- Tooltip, legend, axes, grid: standard pattern with `data-axis="x"`
  or `"y"` on ticks / grid lines.

## Out of scope

- Cardinal / basis / bundle / step-after / step-before
  interpolations - this primitive ships catmullRom + monotone; use
  `<ChartLineStep>` for step variants, `<ChartLine>` for straight
  segments.
- Filled area below the smoothed line - use `<ChartArea>` for area
  primitives.
- Log-scale axes - linear only.
- Animated transitions on data / curve / tension change (snap; mount
  fade only).
- Per-point marker shapes (circle only; v1).

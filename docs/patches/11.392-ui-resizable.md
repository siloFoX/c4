## Goal

Dispatch:

- Build `web/src/components/ui/resizable.tsx`
  -- composable resizable group with
  horizontal / vertical direction, multiple
  panels, min/max constraints per panel.

`<SplitPane>` / `<SplitView>` (11.391)
already handle the two-pane case. The new
`<Resizable>` primitive ships the N-pane
variant with per-panel constraints and a
canonical boundary-local resize contract.

## API

```ts
interface ResizablePanelConfig {
  id: string;
  content: ReactNode;
  defaultSize?: number;     // ratio 0..1, normalized on mount
  minSize?: number;         // default 0.1
  maxSize?: number;         // default 1
  ariaLabel?: string;
}

interface ResizableProps {
  panels: ResizablePanelConfig[];
  direction?: 'horizontal' | 'vertical';
  storageKey?: string;
  keyboardStep?: number;    // default 0.025
  ariaLabel?: string;
  handleAriaLabel?: (prev: string, next: string) => string;
  className?: string;
  onSizesChange?: (sizes: number[]) => void;
}
```

```tsx
<Resizable
  direction="horizontal"
  panels={[
    { id: 'sidebar', content: <Sidebar />, defaultSize: 0.2, minSize: 0.15, maxSize: 0.4 },
    { id: 'main',    content: <Main />,    defaultSize: 0.5, minSize: 0.3 },
    { id: 'aside',   content: <Aside />,   defaultSize: 0.3, minSize: 0.15 },
  ]}
  storageKey="workbench"
/>
```

## Behaviour

### Boundary-local drag

Dragging the handle between panel[i] and
panel[i+1] adjusts ONLY those two panels;
siblings stay put. This is the canonical
multi-pane resizer contract -- propagating
delta to siblings creates non-local layout
effects that operators find confusing.

### Per-panel clamping

During drag + keyboard nudge, each panel
respects its `[minSize, maxSize]`. Whichever
cap binds first stops the boundary:

- `dx > aMax - a` -> clamp to grow-cap on
  panel[i].
- `dx < aMin - a` -> clamp to shrink-cap.
- `dx > b - bMin` -> partner panel hits
  its min.
- `dx < b - bMax` -> partner panel hits
  its max.

### Normalization on mount

Initial `defaultSize` values are normalized
so the group sums to 1.0 (callers do not
need perfect arithmetic). NaN / negative
inputs fall back to per-panel `minSize`;
over-max entries clamp.

If the clamped + scaled array drifts off
1.0 by more than 1e-9, the trailing slack
distributes to the LAST panel (canonical
convention). If the trailing clamp also
fails, the helper falls back to equal
distribution across all panels.

### Persistence

Opt-in via `storageKey`. The size array
serializes as JSON. Reads validate:

- correct length (matching `panels.length`),
- every entry numeric + finite.

Invalid entries fall back to defaults. The
`storage` event listener re-syncs sibling
instances when another tab writes to the
same key.

### Pointer drag

`pointerdown` with the primary button
captures the start coordinate + sizes
snapshot. `pointermove` computes
`delta = (coord - startCoord) /
containerSize` and applies
`applyResizableDelta`. `pointerup` /
`pointercancel` ends the drag. Uses
`setPointerCapture` for cross-element
drag reliability.

### Keyboard

Direction-aware:

| direction | grow start | shrink start |
| --- | --- | --- |
| horizontal | ArrowRight | ArrowLeft |
| vertical | ArrowDown | ArrowUp |

Home pushes panel[i] to `minSize`; End to
`maxSize`. The partner panel absorbs the
delta as in pointer drag.

## ARIA

- Root: `<div role="group">` +
  `aria-orientation` + `aria-label`.
- Panel container: `<div role="group">` +
  `aria-label`.
- Handle: `<div role="separator">` +
  `aria-orientation` (vertical for
  horizontal layout, horizontal for
  vertical layout -- per the standard
  separator pattern) +
  `aria-valuemin` / `aria-valuenow` /
  `aria-valuemax` (percent integers).

## Pairs with existing primitives

- `<SplitView>` / `<SplitPane>` (11.391)
  -- exactly two panes. Use for binary
  splits; Resizable for three-or-more
  pane layouts.
- `<DrawerResize>` (11.390) -- single
  fixed-sided resizable sidebar.

## Out of scope

- Per-page adoption.
- Imperative size set / get methods.
  Controlled `onSizesChange` + `storageKey`
  cover the common cases.
- Collapse-on-double-click. Ambiguous in
  N-panel; the binary primitive
  `<SplitView>` covers the canonical
  collapse case.
- Animated resize transitions. Sizes
  update instantly; CSS transitions on
  `flex-basis` need a separate animation
  lifecycle.
- Drag-anywhere-in-panel resize. Handle
  is the canonical affordance.
- Pixel-mode sizing across the group.
  All math is ratio-based; pixel
  conversion belongs in a follow-on.

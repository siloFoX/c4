## Indicator

Nicolas Darvas's box theory. A Darvas box bounds a consolidation:
it forms from a new high, then locks once that high holds, and
breaks when the price escapes the range. It needs the high, low
and close, so `ChartLineDarvasPoint` carries `{x, high, low,
close}`.

- `computeLineDarvasBoxes(bars, confirm)` -- the box detector, a
  state machine. Walking the bars, a box top is the highest high
  of a formation; once `confirm` bars pass with no higher high,
  the top locks and the bottom is fixed at the lowest low
  recorded since the top bar. The box then runs as a fixed
  rectangle until a bar closes above the top (`broken-up`) or
  below the bottom (`broken-down`), at which point the next
  formation begins. A box still open at the end of the data is
  `active`.
- `runLineDarvas` -- sorts the finite bars by x, runs the
  detector, and returns per-bar samples assigned to their box and
  classified `inside` / `breakout-up` / `breakout-down` / `none`,
  the box count and the breakout counts; `ok` is false for fewer
  than two bars.
- `computeLineDarvasLayout` -- overlays the price line and one
  rectangle per box on a single panel, with a marker on each
  breakout bar.
- `describeLineDarvasChart` -- an off-screen sentence summary for
  assistive technology.

## Testability

The Darvas box detector uses only comparisons, max and min -- no
arithmetic -- so an integer OHLC fixture makes every box exact,
asserted with `toEqual`. The ten-bar fixture is hand-traced for a
confirmation count of two: box 0 sets its top at the opening bar,
holds two bars, then breaks upward; a lead-in bar then precedes
box 1, whose top is a fresh high that holds two bars before
breaking downward. Separate fixtures cover a box still active at
the end of the data and a run of straight new highs that never
confirms a box.

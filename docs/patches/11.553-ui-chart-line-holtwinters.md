# 11.553 -- web/src/components/ui/chart-line-holtwinters.tsx

Pure-SVG line chart with an **additive Holt-Winters
triple-exponential smoothing overlay**. Raw observations
render as a dimmed reference line; the Holt-Winters fitted
(one-step in-sample) value renders as a solid coloured
overlay; an optional dashed multi-step forecast extends a
full season beyond the data.

## Math

Holt-Winters extends simple exponential smoothing with
**three** coupled recursions -- one each for level, trend,
and seasonality. For a season length `m` the additive
variant updates as:

```
level_t  = alpha * (y_t - season_{t-m})
           + (1 - alpha) * (level_{t-1} + trend_{t-1})
trend_t  = beta  * (level_t - level_{t-1})
           + (1 - beta)  * trend_{t-1}
season_t = gamma * (y_t - level_t)
           + (1 - gamma) * season_{t-m}
```

The in-sample one-step fitted value is:

```
yhat_t = level_{t-1} + trend_{t-1} + season_{t-m}
```

and the multi-step forecast `h` periods beyond the data:

```
yhat_{N-1+h} = level + h * trend + season_{(N-1+h) mod m}
```

### Three smoothing factors

- `alpha` -- level smoothing (default 0.5).
- `beta`  -- trend smoothing (default 0.3).
- `gamma` -- seasonal smoothing (default 0.3).

All three live on `[0, 1]`.

### Initialisation (standard textbook scheme)

- `initialLevel` = mean of the first season `y_0 .. y_{m-1}`.
- `initialTrend` = the mean per-step slope between the first
  two seasons: `(1/m) * sum_{i=0}^{m-1} (y_{m+i} - y_i) / m`.
  When fewer than two full seasons are available the trend
  initialises to 0.
- `initialSeasonals[i]` = `y_i - initialLevel` for the first
  season. These sum to 0 by construction (additive).

The recursion runs for `t = m .. N-1`, so the first `m`
fitted values are null. The fit reports `ok = false` when
there are fewer than `m + 1` finite values.

### Verified properties (by test)

- A **purely seasonal series with no trend**
  (`[10,20,30,40]` repeated) is fitted **exactly** -- every
  fitted value equals the raw value, residuals 0, RMSE 0 --
  and the forecast reproduces the continuing pattern
  (`[10,20,30,40]`). This holds for any valid alpha/beta/gamma.
- A **constant series** is fitted exactly (level = mean,
  trend 0, seasonals 0).
- `initialSeasonals` sum to 0.
- `initialTrend` is 0 with fewer than two full seasons; with
  two full seasons of a linear ramp `0..7` (m = 4) it
  recovers the per-step slope exactly (1).

## Distinct from related primitives

- `<ChartLineEwma>` (11.539): **single** exponential
  smoothing -- one recursion (level only). Holt-Winters is
  **triple** exponential smoothing -- it additionally models
  trend and seasonality, which single ES cannot capture.
- `<ChartLineArima>` (11.552): an AR(1) autoregressive model
  fitted by least squares. Holt-Winters is an exponential
  smoothing method (recursively weighted averages), not a
  regression; and it explicitly decomposes seasonality.
- `<ChartLineDecompose>` (11.543): splits a series into
  trend + seasonal + residual *components* for inspection.
  Holt-Winters uses the same three concepts but as a
  *forecasting* model -- it produces fitted values and a
  forward forecast, not a static decomposition.
- `<ChartLineKalman>` (11.538): Bayesian state-space filter
  with explicit process/measurement noise. Holt-Winters is
  deterministic exponential smoothing with no noise model.
- `<ChartLineForecast>` (renders pre-computed forecast
  data): Holt-Winters fits the model itself and computes the
  forecast.
- `<ChartLineMovingAvg>` (11.513) / `<ChartLineLoess>`
  (11.549) / `<ChartLineSavgol>` (11.547): smoothers with no
  trend or seasonal model and no forecast.

The canonical seasonal forecasting method taught alongside
ARIMA -- the natural choice when a series has a clear
repeating cycle plus drift.

- `<ChartLineHoltWinters series={...} />`.
- `series: ChartLineHoltWintersSeries[]` --
  `{id, label, data, color?, alpha?, beta?, gamma?,
  seasonLength?}`. Per-series factor / season overrides
  always beat chart-level.

  `gamma` (default 0.3) -- clamped to `[0, 1]`.

- `forecastHorizon` (default 4; integer 0..366).
- `forecastColor`, `rawColor`, `residualPosColor`,
  `residualNegColor`.
- Toggle flags: `showAxis`, `showGrid`, `showDots`,
  `showLegend`, `showTooltip`, `showConfigBadge`, `showRaw`,
  `showForecast`, `showResidualSticks`, `animate`.
- Controlled + uncontrolled visibility.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX` / `formatCoefficient`.
- `onPointClick({series, point})`.

## Pure helpers

- `getLineHoltWintersDefaultColor(index)`.
- `getLineHoltWintersFinitePoints(points)`.
- `normaliseLineHoltWintersSmoothingFactor(value, fallback)`
  -- clamps to `[0, 1]`.
- `normaliseLineHoltWintersSeasonLength(value)` -- integer
  >= 2.
- `normaliseLineHoltWintersForecastHorizon(value)` --
  integer 0..366.
- `classifyLineHoltWintersResidualSign(residual)`.
- `fitLineHoltWinters(values, options?)` -- the additive
  Holt-Winters fit; returns `{ok, fitted, initialLevel,
  initialTrend, initialSeasonals, level, trend, seasonals,
  alpha, beta, gamma, seasonLength}`.
- `forecastLineHoltWinters(fit, horizon)` -- multi-step
  forecast values; `[]` when the fit failed or horizon <= 0.
- `runLineHoltWinters(points, options?)` -- canonical
  pipeline. Sorts ascending, drops non-finite, fits the
  model, computes residuals + RMSE + a forecast array.
- `computeLineHoltWintersLayout({...})` -- projects raw +
  fitted paths + residual segments + the forecast path.
- `describeLineHoltWintersChart(series, options?)`.

fitted value (or n/a for the first season), residual (with
`+`/`-` prefix), and the config
`a=<alpha>, b=<beta>, g=<gamma>, m=<season>`.

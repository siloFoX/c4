# 11.282 -- Avatar: xs size + status overlay (v1.11.300)

Component-scope-only enhancement of the existing Avatar primitive.
The current sm / md / lg sizes are preserved byte-for-byte; the
previous shape (initials inside a coloured circle, or `<img>`
inside a circle) is preserved when callers do not opt into the new
`status` prop.

## Why

`WorkerList` was rendering Avatar + StatusDot as two adjacent
glyphs. The pair takes too much horizontal room on narrow viewports
and forces the operator to mentally bind them together (which dot
goes with which avatar?). Patch 11.282 dock the status dot onto the
avatar tile itself so one glyph carries both identity AND presence.
The new `xs` size unblocks nav-row + activity-feed densities where
the 24 px `sm` was too tall.

## What changed

### `web/src/components/ui/avatar.tsx`

- `size="xs"` -- new 20x20 px (h-5 w-5) tile with 10 px text.
- `status?: StatusDotVariant` -- optional presence dot rendered
  in the bottom-right corner of the tile. Maps to the StatusDot
  palette (online -> success, busy -> warning, away -> chart-3,
  offline -> muted-foreground, unknown -> muted) so a future hue
  migration in one primitive does not desynchronise the other.
  The overlay is ringed with `bg-background` so it reads cleanly
  against any per-name fill colour beneath, and scales with the
  size class (h-1.5 for xs through h-3 for lg).
- SR label composition -- when `status` is set, aria-label reads
  `<name>, <variant>` (`"Alice, busy"`) so SR users hear both in
  one announcement.
- `data-section="avatar-root"` on the wrapper when the overlay
  mounts, `data-section="avatar-status"` on the dot itself +
  `data-status` attr, plus `data-section="avatar-initials"` on
  the initials fallback.
- exactOptionalPropertyTypes-safe spread for the underlying
  `<Image srcSet sizes>` prop forwarding.

### `web/src/components/ui/avatar-group.tsx`

- OVERLAP + CHIP_SIZE maps picked up the new `xs` token
  (`-ml-1.5` / `h-5 w-5 text-[9px]`). Forwarding to the inner
  Avatar uses exactOptionalPropertyTypes-safe conditional spread.

### Adopted

- `web/src/components/WorkerList.tsx` -- per-row Avatar now
  carries `status={dotVariant}`. The adjacent `<StatusDot>` is
  retained so busy workers still get the pulse motion signal
  (avatar overlay = at-rest indicator, StatusDot pulse =
  real-time activity).
- `web/src/components/SessionsAttachedSection.tsx` -- attached
  session rows wear `status="online"` so the presence is visible
  without reading the "attached" badge text.
- `web/src/components/HistoryDetailPane.tsx` -- worker identity
  header now leads with a md-size Avatar tile (online when
  `detail.alive`, offline otherwise) followed by the CardTitle.
  The DataList row underneath still carries the verbose `status`
  badge so closed / exited states stay readable.

### Tests

- 8 new vitest cases: size=xs tile classes, no overlay when
  status omitted, overlay renders for initials + image, palette
  mapping for busy / online / offline, overlay scales per size,
  SR label folds status into name for both variants. 29 / 29
  green.

# 11.1029 -- UI: chart-line-adx-trend-cross primitive (v1.11.1047)

## Scope

Add a pure-SVG dual-panel `ChartLineAdxTrendCross`
component that renders the close in the top panel and the
Average Directional Index (ADX) plus the two fixed
trend-strength thresholds (`lower = 20`, `upper = 25`) in
the bottom panel, marking ADX crosses up through a
threshold (trend emerging / trend confirmed) and ADX
crosses down through a threshold (trend weakening / trend
dissolved) with bias coloring derived from the ADX slope at
the trigger bar.

## ADX trend-cross pipeline

```
+DM[i]    = upMove > downMove && upMove > 0 ?
             upMove : 0       ; upMove = high[i]-high[i-1]
-DM[i]    = downMove > upMove && downMove > 0 ?
             downMove : 0     ; downMove = low[i-1]-low[i]
TR[i]     = max(high[i]-low[i],
                |high[i]-close[i-1]|,
                |low[i]-close[i-1]|)
+DMs[i]   = Wilder(+DM, period)   ; sum-init then running
-DMs[i]   = Wilder(-DM, period)
ATR[i]    = Wilder(TR,  period)
+DI[i]    = 100 * +DMs / ATR
-DI[i]    = 100 * -DMs / ATR
DX[i]     = 100 * |+DI - -DI| / (+DI + -DI)
            (0 when +DI + -DI = 0)
ADX[i]    = SMA-init Wilder smooth of DX, period
            ; first valid at i = 2*period - 1
regime    = 'weak'    when ADX < lower
            'forming' when lower <= ADX < upper
            'strong'  when ADX >= upper
            'none'    when ADX is null
bias      = ADX[i] vs ADX[i-1] -> up / down / flat / none
bullish   : prev ADX <= threshold AND cur ADX > threshold
            (trend emerging or trend confirmed)
bearish   : prev ADX >= threshold AND cur ADX < threshold
            (trend weakening or trend dissolved)
```

Each cross event carries an extra `threshold: 'lower' |
'upper'` field so callers can distinguish "trend emerging"
(through 20) from "trend confirmed" (through 25). Both
thresholds are checked independently per bar; if ADX leaps
from below 20 to above 25 in a single bar, two crosses
fire (one through each threshold).

Warmup is `2 * period - 1 = 27` for the default tuning,
reflecting Wilder's standard ADX initialisation: +DM, -DM,
TR each need `period` bars to seed their first smoothed
value (sum of the first 14 bars at `i = period`), then ADX
itself needs `period` DX values to seed its SMA-init
average (first valid at `i = 2*period - 1`).

## Bit-exact anchors

All anchors use HLC input.

- **CONST band** `high = K + 1`, `low = K - 1`, `close = K`:
  upMove = downMove = 0 everywhere, so +DM = -DM = 0. TR
  = 2 from i >= 1 -> ATR settles to 2*period = 28 but
  +DMs = -DMs = 0 -> +DI = -DI = 0 -> DX = 0 (via the
  `+DI + -DI = 0` zero-guard). ADX = 0 from `i = 2*period
  - 1`. ADX < lower=20 -> regime `weak`. 0 crosses.
  Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close = i`:
  upMove = 1, downMove = -1; upMove > downMove and > 0 so
  +DM = 1. downMove not > 0 so -DM = 0. TR = max(2, 2, 0)
  = 2. After Wilder convergence: +DMs = 14, -DMs = 0, ATR
  = 28 -> +DI = 50, -DI = 0 -> DX = 100. ADX = 100 from
  `i = 2*period - 1`. ADX >= upper=25 -> regime `strong`.
  0 crosses (first valid ADX is already above both
  thresholds; there is no prev valid ADX to compare).
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`, `close
  = -i`: mirror of LINEAR UP. upMove = -1, downMove = 1;
  downMove > upMove and > 0 so -DM = 1. +DM = 0. TR = 2.
  After Wilder convergence: +DI = 0, -DI = 50, DX = 100.
  ADX = 100. Regime `strong`. 0 crosses.

The clean integer steady-state values (0 for CONST, 100 for
both LINEAR anchors) make the warmup boundary easy to
verify -- ADX is `null` for `i < 27` and snaps to its
steady-state value at `i = 27` since DX is constant for `i
>= 14`.

  `data-strong-count`, `data-none-count`

- `data-lower-cross-count`, `data-upper-cross-count`
- `data-up-bias-count`, `data-down-bias-count`,
  `data-flat-bias-count`
- `data-cross-count`

The threshold-specific cross counters
(`data-lower-cross-count`, `data-upper-cross-count`) are
new for the ADX trend-cross family -- they let observers
distinguish "trend emerging" frequency (lower crosses)
from "trend confirmed" frequency (upper crosses).

## Accessibility

- `role="region"`, descriptive `aria-label`, narrative
  `describeLineAdxTrendCrossChart` description.
- Trigger markers are individually focusable
  (`tabIndex={0}`) with kind+threshold+bias `aria-label`.
- Hover targets respond to mouse and keyboard focus.
- Legend buttons are keyboard-toggleable via Enter /
  Space.
- Threshold lines use distinct dash patterns (`4 3` for
  lower, `6 3` for upper) so they remain distinguishable
  in high-contrast / colour-blind modes.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 3-zone regime classifier (`weak` / `forming` /
  `strong`) breaks the `bullish` / `bearish` / `none`
  convention shared by the rest of the
  `chart-line-*-cross` family -- intentional, since ADX
  is a non-directional trend-strength indicator (it does
  not say which direction the trend is going, only how
  strongly it is trending). The `bullish` / `bearish`
  cross *kinds* still follow the sibling convention (line
  crossing up / down).
- Zero-ATR rolling window is short-circuited to null so a
  degenerate window cannot produce Infinity.
- `+DI + -DI = 0` zero-guard returns DX = 0 (instead of
  NaN) so the CONST anchor stays bit-exact.
- Wilder smoothing uses sum-init for +DM / -DM / TR
  (matches the original 1978 Wilder formulation) but
  SMA-init for ADX (keeps it bounded in [0, 100] from
  the first valid sample).

# 11.571 -- web/src/components/ui/chart-line-fibonacci.tsx

Pure-SVG line chart that overlays **Fibonacci retracement
levels**. The chart detects the series' swing high and swing
low, then draws the canonical Fibonacci ratio lines between
them -- horizontal support/resistance levels at 0%, 23.6%,
38.2%, 50%, 61.8%, 78.6% and 100% of the swing range.

## The retracement grid

`runLineFibonacci(points, ratios)`:

1. The series is sorted by x. The **swing high** (max value)
   and **swing low** (min value) are detected.
2. The **trend** is `up` when the high occurs at or after
   the low, `down` otherwise.
3. The level at ratio `r` is measured from the swing END
   (the later extreme) back toward the swing START (the
   older one): `value = end - r * (end - start)`. So `r = 0`
   sits on the most recent extreme and `r = 1` on the older
   one -- the standard retracement grid. For an up swing
   that puts 0% at the high and 100% at the low; a down
   swing mirrors it.

A series whose values are all equal has no range and is not
`ok`. `normalizeLineFibonacciRatios` coerces a ratio set to
an ascending, de-duplicated list of finite values clamped
to [0, 1], falling back to the standard seven ratios.

The component fixture is an up swing -- low `0` at x=1, high
`100` at x=4 -- so the levels land on round values: `100,
76.4, 61.8, 50, 38.2, 21.4, 0`. The down-swing fixture
mirrors the grid: `0, 23.6, ..., 100`.

## Distinct from neighbouring primitives

- `chart-line-threshold` draws horizontal lines at
  **user-supplied fixed thresholds**. The Fibonacci levels
  are **derived** from the detected swing high/low at the
  fixed Fibonacci ratios -- the operator supplies no values.
- `chart-line-zigzag` filters swing pivots by a percent
  reversal; it never draws ratio levels.
- `chart-line-min-max` draws a rolling min/max envelope, not
  discrete horizontal levels.
- `chart-line-target` / `chart-line-baseline` mark a single
  reference value.

The defining transform is the swing detection plus the
Fibonacci ratio grid. All exported helpers carry a
`LineFibonacci` infix so the `export *` barrel line cannot
collide.

## Public surface

- `<ChartLineFibonacci data={...} />` -- a single
  `ChartLineFibonacciPoint[]` (`{x, value}`).
- `ratios` -- a custom ratio set (default: the standard
  seven; normalized to ascending unique [0,1] values).
- Toggle flags: `showAxis`, `showGrid`, `showDots` (default
  false), `showLevels`, `showLevelLabels`, `showAnchor` (the
  swing-low-to-swing-high diagonal), `showSwings`,
  `showTooltip`, `showConfigBadge`, `showFooter`, `animate`.
- Standard sizing + format + a11y props. `formatValue` /
  `formatX`.
- `onLevelClick({level})` / `onSampleClick({point})`.

## Pure helpers

- `getLineFibonacciFinitePoints(points)`.
- `normalizeLineFibonacciRatios(ratios)`.
- `runLineFibonacci(points, ratios?)` -- swing detection +
  the retracement levels; `ok` requires at least 2 finite
  samples with a non-zero range.
- `computeLineFibonacciLayout({...})` -- projects the line,
  the level lines, the swing markers and the anchor.
- `describeLineFibonacciChart(data, options?)`.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- The series line, every level and every swing marker
  `role="graphics-symbol"` + `tabIndex={0}`.
- Every node carries `data-section="chart-line-fibonacci-..."`.
- Root mirrors `data-trend`, `data-level-count`,
  `data-range`, `data-total-points`, `data-animate`.
- Level groups expose `data-ratio`, `data-value`. Swing
  markers expose `data-kind` (`high` / `low`), `data-value`.

## Tooltip / badge / footer

Hovering a level shows the ratio percent and the level
value; hovering a dot shows x + value. Config badge:
`FIB <trend> H=<high> L=<low>`. The footer reports the
trend, the swing range and the level count.

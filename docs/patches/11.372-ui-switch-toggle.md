## Goal

Dispatch:

- Build `web/src/components/ui/switch.tsx` +
  `toggle.tsx` -- animated switch with on/off
  states, ARIA switch role; toggle button
  with pressed state, icon support.

`<Switch>` (11.287 / v1.11.305) already
shipped the full contract:

| dispatch item | shipped at |
| --- | --- |
| animated switch | `transition-transform` on the thumb |
| on/off states | `checked` boolean + `onChange(next)` |
| ARIA switch role | `role="switch"` + `aria-checked` |
| reduced-motion respect | thumb transition drops via `useReducedMotion()` |
| size variants | `sm` (h-4 w-7) / `md` (h-5 w-9) |
| label slot | inline `<label>` row |

This patch ships the missing companion
`<Toggle>` primitive.

## Toggle API

```tsx
import { Toggle } from './components/ui/toggle';

// Controlled
<Toggle pressed={isBold} onPressedChange={setIsBold} aria-label="Bold">
  B
</Toggle>

// Uncontrolled
<Toggle defaultPressed aria-label="Italic">I</Toggle>

// Icon + label
<Toggle icon={<BoldIcon />} aria-label="Bold" variant="outline" size="lg">
  Bold
</Toggle>
```

### Props

| prop | type | default | notes |
| --- | --- | --- | --- |
| `pressed` | `boolean?` | undefined | controlled state |
| `defaultPressed` | `boolean?` | `false` | uncontrolled seed |
| `onPressedChange` | `(next: boolean) => void?` | undefined | fires on every commit |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | h-7/9/11 px-2/3/4 |
| `variant` | `'default' \| 'outline'` | `'default'` | pressed fill vs pressed border |
| `icon` | `ReactNode?` | undefined | leading slot, aria-hidden |
| `children` | `ReactNode?` | undefined | label slot |
| `disabled` | `boolean?` | false | click + keyboard no-op |
| `type` | `'button' \| 'submit' \| 'reset'` | `'button'` | inherited |

### Controlled-vs-uncontrolled contract

| Props | Mode |
| --- | --- |
| `pressed` + `onPressedChange` | controlled |
| `defaultPressed` only | uncontrolled, internal state |
| `defaultPressed` + `onPressedChange` | uncontrolled, observable |
| `pressed` + `defaultPressed` | controlled (`pressed` wins) |
| neither | uncontrolled, starts unpressed |

### Render

```
<button
  type="button"
  role="button"
  aria-pressed={pressed}
  data-section="toggle"
  data-pressed
  data-size
  data-variant
>
  <span data-section="toggle-icon" aria-hidden="true">{icon}</span>
  <span data-section="toggle-label">{children}</span>
</button>
```

Native `<button>` Space/Enter synthesis flips
the pressed state without extra keyboard
handling. The caller's `onClick` runs AFTER
`onPressedChange` so the side-effect ordering
is predictable.

### Variant -> palette

| variant | unpressed | pressed |
| --- | --- | --- |
| `default` | `bg-transparent text-foreground` + accent hover | `bg-accent text-accent-foreground` |
| `outline` | `border border-input bg-transparent` + accent hover | `border border-accent bg-accent/20 text-accent-foreground` |

### Size -> dimensions

| size | dimensions |
| --- | --- |
| `sm` | `h-7 px-2 text-xs` |
| `md` (default) | `h-9 px-3 text-sm` |
| `lg` | `h-11 px-4 text-base` |

## Pairs with existing primitives

- `<Switch>` -- "this setting is on/off".
  Unchanged.
- `<Button>` -- non-stateful action button.
- `<IconButton>` -- icon-only button without
  pressed state.
- `<Toolbar>` -- can host rows of Toggle for
  text-editor surfaces; the integration is
  additive (no Toolbar changes).

## Out of scope

- **ToggleGroup.** A radio-or-checkbox-style
  row of Toggle items belongs in a follow-on
  patch (need to settle the single-vs-multi
  mode API alongside MultiSelect).
- **Animated pressed transition** (e.g.,
  scale-on-press). The pressed state is
  visual-only via the variant palette; motion
  can be added on a future per-surface tuning
  patch.
- **Per-surface adoption** of Toggle (e.g.,
  swapping a manually-styled `<button
  aria-pressed>` to `<Toggle>`). The new
  primitive is additive; no call sites
  rewired.
- **Switch animation tuning.** Switch is
  unchanged from 11.287; its motion path
  already respects reduced-motion.

# TODO 11.496 - chart-tornado primitive

Status: SHIPPED v1.11.514 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartTornado>`, a pure-SVG tornado / butterfly chart for paired
two-value comparisons. Each category renders two bars that extend in
opposite directions from a central axis: the `left` value grows
leftward, the `right` value grows rightward. Optimised for showing
two magnitudes side-by-side across many categories (e.g.
male/female population pyramids, pros/cons, win/loss).

## Distinct from prior chart-* primitives

- `<ChartClevelandDot>` (11.495) - two dots + connector on a single
  shared value axis. Tornado paints two bars extending out from a
  central axis so both magnitudes are emphasised independently.
- `<ChartBar>` - one value per category in one direction. Tornado has
  two values per category in opposite directions.
- `<ChartStackedBar>` - sub-series stack within a single bar. Tornado
  has two independent bars per category.
- `<ChartLollipop>` (11.494) - one head + stick from a fixed baseline.
- `<ChartFunnel>` - progressive narrowing showing stages; tornado is
  paired horizontal bars (no flow / stage semantics).

## API

```tsx
<ChartTornado
  items={[
    { id: 'a', label: '20-29', left: 800, right: 820 },
    { id: 'b', label: '30-39', left: 1100, right: 1050 },
    { id: 'c', label: '40-49', left: 950, right: 980 },
  ]}
  leftLabel="Male"
  rightLabel="Female"
  scaleMode="shared"
  sortBy="sum"
  sortOrder="desc"
/>
```

## Layout

### Center axis

- `centerLine = padding + innerWidth/2` (horizontal) or
  `padding + innerHeight/2` (vertical).
- `centerGap` (default 4) is the empty pixel band on each side of the
  center line so the bars never touch the center axis.
- Left half spans `[padding, centerLine - centerGap]`; right half
  spans `[centerLine + centerGap, padding + innerWidth]`. The
  symmetric layout makes magnitudes visually comparable side by side.

### Scale modes

- `'shared'` (default): both sides use the same max
  (`max(all left, all right)`) so directly comparable magnitudes share
  one ruler. Best for same-unit comparisons (e.g. counts).
- `'independent'`: each side has its own max
  (`max(left side)` vs `max(right side)`). Use when units differ or
  one side is much larger.
- Explicit `valueMax` overrides both sides; useful for stable
  cross-chart comparison.

### Category axis

- N items share the inner length evenly (`bandWidth = (innerLength -
  bandGap*(N-1)) / N`). `bandGap` (default 4) is the gap between
  adjacent category rows.
- `barThickness` defaults to `0.7 * bandWidth` (min 2 px); explicit
  prop wins.
- `sortBy = 'left' | 'right' | 'sum' | 'diff' | 'absDiff'` plus
  `sortOrder = 'asc' | 'desc'` reorders items. Ties broken by
  original index. Default preserves input order.

### Bars

- Left bar: `width = (left / leftMax) * leftHalfWidth`, anchored at
  the right edge of the left half (grows toward `padding`).
- Right bar: `width = (right / rightMax) * rightHalfWidth`, anchored
  at `rightSideStart` (grows toward the inner right edge).
- Negative or non-finite values clamp to 0 width.
- Vertical orientation swaps the roles: bars extend up and down from a
  horizontal center axis; categories tile along the x axis.

## Interactivity

- Hover or focus on a bar dims every other bar to opacity 0.3 (the
  hovered side stays at `barOpacity`, the other side of the same item
  also dims).
- Tooltip shows the category label, both formatted values (the hovered
  side bolded), and the signed diff (right - left).
- Click on a bar fires `onItemClick({item, layout, side})` where
  `side` is `'left'` or `'right'`.
- Mount fade via `motion-safe:animate-fade-in` (suppressed via
  `animate=false`).

## Exported helpers

- `DEFAULT_CHART_TORNADO_*` numeric / string defaults.
- `getTornadoFiniteItems(items)` - drops items with non-finite left
  OR right or non-string ids; non-array -> `[]`.
- `getTornadoMaxValue(items, side, mode)` - max across both sides
  when shared; per-side max when independent. Negative values clamp
  to 0. Empty -> `1`.
- `getTornadoTicks(max, count?)` - evenly-spaced ticks from 0 to
  max; non-positive max -> fallback `[0, 1]`.
- `sortTornadoItems(items, sortBy?, order?)` - returns `{item,
  originalIndex}` pairs; preserves original index after sort.
- `computeTornadoLayout(input)` - canonical layout returning
  `{ items, leftTicks, rightTicks, leftMax, rightMax, orientation,
  scaleMode, innerWidth, innerHeight, bandWidth, barThickness,
  centerLine, leftSideStart, leftSideEnd, rightSideStart,
  rightSideEnd }`. Per-item layout exposes both bars (`leftX,
  leftY, leftWidth, leftHeight, rightX, rightY, rightWidth,
  rightHeight`), the diff, the sum, the resolved colours, and the
  row centre.
- `describeTornadoChart(items, leftLabel, rightLabel, formatValue?)`
  - ARIA summary "Tornado chart comparing <left> and <right> across
  N categor[y|ies]. Peak <left> <leftMax>, peak <right> <rightMax>.".

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only
  `data-section="chart-tornado-aria-desc"` span.
- SVG `role="img"` with the same `aria-label`.
- Each bar is `role="graphics-symbol"` with `tabIndex={0}` and
  `aria-label="<category>: <sideLabel> <value>"`.

## Data attributes

- Root: `data-section="chart-tornado"`, `data-orientation`,
  `data-scale-mode`, `data-category-count`, `data-finite-count`,
  `data-left-max`, `data-right-max`, `data-sort-by`,
  `data-sort-order`, `data-animate`.
- Item: `data-section="chart-tornado-item"`, `data-item-id`,
  `data-item-index`, `data-item-original-index`, `data-item-left`,
  `data-item-right`, `data-item-diff`, `data-item-sum`,
  `data-hovered`.
- Bar: `data-section="chart-tornado-bar"`, `data-item-id`,
  `data-side="left"` or `"right"`.
- Tooltip: `data-section="chart-tornado-tooltip"`, `data-item-id`,
  `data-side`, plus `-tooltip-label`, `-tooltip-left`,
  `-tooltip-right`, `-tooltip-diff` children.
- Center line, legend, axes, ticks, grid, value-text, category-text:
  standard pattern.

## Tests

65 vitest cases (1 constants + 3 finite items + 4 max value + 2 ticks
+ 7 sort + 13 layout + 5 describe + 30 component) all pass under
vitest 4.1.5. TypeScript clean for touched files.

## Out of scope

- Negative values painted below an internal baseline; v1 clamps
  negatives to 0 (host can normalise to positive magnitudes).
- Animated transitions on data / sort / scaleMode changes (snap; mount
  fade only).
- Log-scale value axis (linear only).
- More than two values per category (use parallel coordinates or
  faceted bar charts).
- Diverging palette interpolation across the difference (v1 uses two
  flat colours).

# 11.311 -- IconButton primitive enhancements (v1.11.329)

Enhances `web/src/components/ui/icon-button.tsx` with
tone and size presets plus a loading state mirroring
the Button primitive's contract. Component-scope only,
no daemon-side change and no `c4` CLI surface change.

## Why

IconButton has been the canonical "icon-only button"
primitive since v1.6, but it has been frozen at one
shape: 36x36 square, muted-foreground icon, accent
hover. The dispatch asks for three additions that map
directly onto recurring call-site needs:

1. **Danger tone** for delete-style affordances. Today
   the Workers row "delete worker" icon button is the
   same neutral muted-foreground colour as the
   "view details" icon button -- two destructive
   semantics rendering identically. Adopters that
   need the destructive tone have to either re-style
   the button via `className` (drift hazard) or
   reach for the regular Button with `size="icon"`
   and `variant="destructive"` (gives up the
   IconButton ergonomics).

2. **Sm/lg sizes** for compact toolbars and primary
   "command palette open" buttons. The current 36x36
   fits the row-action use case but reads small in
   the AppHeader and oversized in dense Workers
   toolbars.

3. **Loading state** for "Delete worker" / "Restart"
   / "Pin" icon buttons that fire an async network
   call. Today the call sites manage their own
   `pending` flag, render a spinner *next to* the
   IconButton, and remember to disable the click
   target. The Button primitive shipped its loading
   state in v1.11.326 (TODO 11.308); IconButton
   should mirror that contract.

Patch 11.311 ships all three additions.

## What changed

### `web/src/components/ui/icon-button.tsx`

New props:

- `tone?: 'neutral' | 'danger' | 'accent'` (default
  `'neutral'`).
  - `neutral` -- pre-existing baseline, now explicit.
    Muted-foreground icon -> accent-foreground on
    hover.
  - `danger` -- destructive palette. Red text +
    destructive focus ring + red-tinted hover.
  - `accent` -- primary-tinted palette. Primary text
    + primary-tinted hover.
- `size?: 'sm' | 'md' | 'lg'` (default `'md'`).
  Square footprints 32px / 36px / 40px. The 44x44
  minimum mobile touch target is preserved on every
  size via
  `min-h-[44px] min-w-[44px] sm:min-h-0 sm:min-w-0`.
- `loading?: boolean` + `loadingLabel?: string`
  (default `'Loading'`). Mirrors the Button
  primitive's loading contract:
  - Inline `<Spinner>` replaces the icon slot. The
    spinner size adapts: `xs` for `sm`, `sm` for
    `md`/`lg`.
  - Auto-disables the click target (`disabled ||
    loading`).
  - `aria-busy="true"` on the button.
  - `<VisuallyHidden>{loadingLabel}</VisuallyHidden>`
    so screen readers narrate the pending state.

Refactored:

- Per-prop `TONE_CLASSES` and `SIZE_CLASSES` records
  replace the single inline class blob so adopters
  can read the intent at the call site.
- `aria-label` remains mandatory at the TS interface
  level. Documented inline.

Data attributes:

- `data-section="icon-button"` on the button.
- `data-section="icon-button-icon"` on the icon
  slot (when not loading), with `aria-hidden="true"`.
- `data-section="icon-button-spinner"` on the
  spinner (when loading), with `aria-hidden="true"`.
- `data-tone`, `data-size`,
  `data-loading="true|false"` for e2e selectors.

### Adopted

- Primitive enhancement only this commit. The
  dispatch listed toolbar actions, row action menus,
  and modal close buttons as adoption sites.
  Threading `tone="danger"` /
  `size="sm" | "lg"` / `loading={pending}` at those
  call sites is a follow-up per-page patch so each
  migration lands with its own test refresh.

## Deferred (dispatch follow-ups)

Representative migration:

```tsx
// Before -- inline pending state for a row action
const [deleting, setDeleting] = useState(false);
<IconButton
  aria-label="Delete worker"
  icon={<Trash2 />}
  onClick={onDelete}
  disabled={deleting}
  className={deleting ? 'cursor-wait' : undefined}
/>;

// After -- using the new props
<IconButton
  aria-label="Delete worker"
  icon={<Trash2 />}
  onClick={onDelete}
  tone="danger"
  loading={deleting}
  loadingLabel="Deleting"
/>;
```

Per-page follow-ups: WorkerDetailHeader row actions
(`tone="danger"` on the delete icon button),
AppHeader toolbar (size="lg" for the primary
command palette opener), ConfirmDialog close icon
(no change needed, the default neutral tone is
correct).

## Notes

- The 44x44 mobile-touch-target rule remains
  enforced on every size preset (not just `md`).
  The `sm:min-h-0 sm:min-w-0` modifier removes the
  enforced minimum on the desktop breakpoint so
  dense toolbars still look right.
- The spinner size mapping (`xs` for `sm`,
  `sm` for `md`/`lg`) keeps the loading icon
  visually balanced inside the square footprint
  -- a `sm` spinner inside a `sm` IconButton
  would crop awkwardly.
- The `aria-label` is mandatory at the TS interface
  level (the prop is declared as
  `'aria-label': string`). This means an adopter
  who omits it gets a type error at compile time,
  not a runtime warning. The pre-existing
  enforcement is preserved in this patch and
  documented inline.
- The icon slot is wrapped in a span with
  `data-section="icon-button-icon"` and
  `aria-hidden="true"` so the icon does not
  contribute an accessible name on its own --
  the `aria-label` on the button is the single
  authoritative announcement.
- Danger tone uses `hover:bg-destructive/10`
  (10% tinted overlay) rather than the full
  `bg-destructive` so the hover state remains
  legible while still signalling intent.

## Summary

Adds the `ChartLineRmiMidCross` primitive
(`web/src/components/ui/chart-line-rmi-mid-cross.tsx`): a pure SVG
dual-panel chart that visualizes Relative Momentum Index (RMI)
centerline-cross events through the 50 midline with momentum bias
coloring. The top panel renders the close with bullish (up green) /
bearish (down red) chevron overlays at every RMI midline cross; the
bottom panel renders the RMI line on a fixed 0..100 oscillator with
the 50 reference band and discrete cross markers tinted by the
momentum bias direction.

Companion to `chart-line-rmi-zero-cross` (TODO 11.948): shares the
same RSI-with-lookback Wilder pipeline and defaults (length = 14,
lookback = 5, threshold = 50) but reframes the ARIA / describe text
from "baseline regime transition" to "centerline regime transition
with momentum bias coloring for trend confirmation" so it slots into
the broader mid-cross family
(`chart-line-rsi-mid-cross`, `chart-line-stoch-mid-cross`,
`chart-line-cci-mid-cross`, `chart-line-mfi-mid-cross`,
`chart-line-stoch-rsi-mid-cross`, `chart-line-williams-r-mid-cross`,
`chart-line-uo-mid-cross`, `chart-line-cmo-mid-cross`).

## Pipeline

RMI = RSI with a lookback period. Instead of comparing the current
close to the previous close, RMI compares to the close `lookback`
bars ago and Wilder-smooths the resulting up/down moves over a
`length` window:

1. `mom[i] = close[i] - close[i - lookback]` (i >= lookback)
2. `gain[i] = max(0, mom)`, `loss[i] = max(0, -mom)`
3. `avgGain = Wilder(gain, length)`, `avgLoss = Wilder(loss, length)`
   - SMA-seeded at `firstValidIdx + length - 1` (so the seed slot is
     `lookback + length - 1`), then recursive
     `prev = (prev * (n - 1) + value) / n`.
4. RMI from the smoothed ratio:
   - `avgLoss == 0 && avgGain == 0` -> RMI = 50 (neutral fallback,
     never -0).
   - `avgLoss == 0 && avgGain > 0`  -> RMI = 100.
   - `avgGain == 0`                  -> RMI = 0.
   - else                            -> `RMI = 100 - 100 / (1 + g / l)`.
5. Regime classifier (strict `>=`):
   - `RMI >= 50` -> bullish
   - `RMI <  50` -> bearish
6. Cross detector emits a `bullish` cross when
   `RMI[i-1] <= 50 && RMI[i] > 50` and a `bearish` cross when
   `RMI[i-1] >= 50 && RMI[i] < 50`. Strict inequality on the current
   side ensures the boundary value 50 does not double-fire.

## Defaults

- `length = 14`
- `lookback = 5` (Altman's canonical settings)
- `threshold = 50` (neutral centerline)
- Warmup boundary: bar `length + lookback - 1 = 18` under defaults,
  so the first 18 samples are null and the remaining 22 (out of 40
  bars) are populated.

## Bit-exact anchors

The tests pin the following determinism contracts:

- CONST `close = K` for all `K in {0, 1, 42, 100, 1234}` ->
  `rmi[i] === 50` literal for every `i >= 18`. Regime classifies as
  bullish because 50 sits on the threshold (`rmi >= threshold`).
- LINEAR UP `close = i` -> `rmi[i] === 100` constant from i = 18
  onward (no losses, avgLoss = 0, avgGain > 0).
- LINEAR DOWN `close = -i` -> `rmi[i] === 0` constant (no gains,
  avgGain = 0).
- Custom `{ length: 5, lookback: 2 }` -> warmup boundary at bar 6,
  LINEAR UP yields rmi = 100 thereafter.
- CONST K full-run counters under defaults, 40 bars:
  `noneCount = 18`, `bullishCount = 22`, `bearishCount = 0`,
  `crosses.length = 0`.
- `pricePath` and `rmiPath` are byte-identical across repeated layout
  calls, with `.toFixed(2)` precision and `M ... L ...` command
  shapes. The rmi path uses a single `M` (no null-gap re-starts) for
  post-warmup data because samples are all finite.
- `thresholdY` sits at the panel midpoint for threshold = 50.
- Cross detector boundary: strict inequality (`> T`, `< T`), so
  equality with the threshold does not fire.

# 11.147 -- UI Separator primitive

`web/src/components/ui/separator.tsx` (v1.11.165) adds a single divider
primitive so the app stops sprinkling `<hr>` and container-mounted
`border-t` / `border-b` utilities for what is conceptually a section
break.

## API

```tsx
<Separator />
<Separator orientation="vertical" />
<Separator weight="thick" />
<Separator label="OR" />
<Separator decorative={false} />    // role=separator + aria-orientation
```

- `orientation`: `'horizontal'` (default) | `'vertical'`. Vertical uses
  `self-stretch border-l` so the parent must be a flex/grid row.
- `weight`: `'thin'` (default, 1px) | `'thick'` (2px). Maps to
  `border-t` / `border-t-2` (or `border-l` / `border-l-2` on vertical).
- `label`: horizontal-only centered slot rendered between two
  `aria-hidden` line segments (`flex-1 border-t`).
- `decorative`: `true` (default) emits `role="none"`; `false` emits
  `role="separator"` plus `aria-orientation` for semantic breaks.
- `className`: merged via `cn` (tailwind-merge), so callers can layer
  spacing utilities (`my-4`, etc.).

Border color comes from the `border-border` Tailwind class which is
already wired to the ARPS muted palette token, so no token import is
needed at the call sites.

## Entry types

```ts
type DropdownMenuEntry =
  | DropdownMenuItem        // existing
  | DropdownMenuSeparator   // existing
  | DropdownMenuSection     // new
  | DropdownMenuCheckbox    // new
  | DropdownMenuRadio;      // new
```

### Section heading

```ts
{ key: 'workspace', kind: 'section', label: 'Workspace' }
```

Renders as a non-interactive row with
`role="presentation"` and
`data-section="dropdown-menu-section"`. Keyboard
nav + type-ahead skip it.

### Checkbox

```ts
{
  key: 'wifi',
  kind: 'checkbox',
  label: 'Wifi',
  checked: true,
  onCheckedChange: (next) => setWifi(next),
  closeOnChange?: boolean,
  disabled?: boolean,
  icon?: ReactNode,
  hint?: ReactNode,
  shortcut?: string,
  onPrefetch?: () => void,
}
```

- `role="menuitemcheckbox"` + `aria-checked`
  mirrors state.
- Click / Enter / Space calls
  `onCheckedChange(!checked)`.
- Default keeps the menu OPEN after toggle
  (filter-dropdown pattern). Set
  `closeOnChange: true` to mirror legacy
  item-dismiss.
- Leading slot reserved (`h-4 w-4`) for the
  check glyph so the labels line up across
  mixed-kind menus.

### Radio

```ts
{
  key: 'sort-name',
  kind: 'radio',
  label: 'Name',
  value: 'name',
  groupValue: 'name',           // currently selected
  onValueChange: (v) => setSort(v),
  closeOnChange?: boolean,
  ...
}
```

- `role="menuitemradio"` + `aria-checked` is
  `groupValue === value`.
- Click / Enter / Space fires
  `onValueChange(value)`.
- Use the same `groupValue` / `onValueChange`
  ref across multiple rows to model one
  exclusive group.

### Shortcut chip

`shortcut?: string` on every interactive item
type. Renders as a `<kbd>` chip in the right
slot
(`data-section="dropdown-menu-shortcut"`).
Shortcut takes precedence over `hint` so the
right column never doubles.

## Glyphs

Inline SVG (no new icon dep):

- Check (matches lucide `Check`): 12x12, 3px
  stroke, currentColor.
- Radio: 8x8 filled circle, currentColor.

Both are `aria-hidden`; the leading slot has
`h-4 w-4` so it stays the same width across
all entry kinds.

## Keyboard / type-ahead

The selector used by `useFocusCycle` is
`[role=menuitem]:not([aria-disabled=true])` --
it already covers ARIA-extended roles via
attribute fallback, so checkbox + radio rows
participate in the cycle automatically.

The type-ahead matcher now uses the
`isInteractive(entry)` guard so sections AND
separators are skipped while looking for a
label match.

## Tests

`dropdown-menu.test.tsx` -- 10 new cases:

- Section heading: renders heading row;
  type-ahead skips it and lands on the
  matching action below.
- Checkbox: renders `role="menuitemcheckbox"`
  + `aria-checked` + check glyph when
  checked.
- Click toggles via `onCheckedChange`; menu
  stays open by default.
- `closeOnChange=true` dismisses the menu
  after the toggle.
- Disabled checkbox does not toggle.
- Radio: renders `role="menuitemradio"` per
  row + `aria-checked` tied to `groupValue`.
- Click fires `onValueChange(value)`.
- Shortcut chip: renders `<kbd>` with the
  shortcut text.
- Shortcut wins over hint when both are set
  (no double right column).

47/47 pass under vitest 4.1.5. TypeScript clean.

## Pairs with existing primitives

- `useFocusCycle` -- unchanged; ARIA-role

  automatically.
- `lib/i18n.ts` -- existing `t()` continues to
  drive `ariaLabel` fallback.
- `<KeyboardShortcutsModal>` -- the shortcut
  chip class composes the canonical border +
  bg-muted style so the menu shortcuts

  documentation surface.

## Out of scope

- Per-page adoption (row actions, header user
  menu) beyond what existing Sidebar
  AccountMenu covers. The new entry kinds are
  additive; adopters opt in row-by-row.
- Submenu (`kind: 'submenu'`). A separate
  primitive.
- Indeterminate checkbox state. Adopters can
  render a custom glyph via `icon` +
  `checked: false`.
- Group-level radio scope. The current
  pattern relies on shared `groupValue` /
  `onValueChange` refs across siblings; a
  future `<RadioGroup>` wrapper could
  formalise the contract.

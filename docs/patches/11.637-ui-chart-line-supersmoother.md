## Summary

New `ChartLineSuperSmoother` UI primitive at

line chart with an Ehlers Super Smoother two-pole filter overlay.

## Indicator math

John Ehlers' Super Smoother is a two-pole low-pass filter. Its
coefficients come from `arg = 1.414 * PI / period`:

```
a1 = exp(-arg)
b1 = 2 * a1 * cos(arg)
c2 = b1
c3 = -(a1 * a1)
c1 = 1 - c2 - c3
```

`c1 + c2 + c3 = 1`, so the filter has unit gain at zero
frequency and passes a flat series straight through. The filter
itself blends the two most recent prices and feeds back the two
most recent filter values:

```
SS[i] = c1 * (price[i] + price[i-1]) / 2 + c2 * SS[i-1] + c3 * SS[i-2]
```

The two-pole feedback gives a steep frequency roll-off -- it
rejects bar-to-bar noise sharply while adding very little lag.
The first two bars seed straight from the price
(`SS[0] = price[0]`, `SS[1] = price[1]`), so the line is defined
from index 0 with no warm-up.

### Worked fixture

`period 10`, price series `[10, 10, 30, 8, 32, 6]`. The
coefficients come from `exp` and `cos`, so they are irrational
and the filter values are asserted against the recursion itself
rather than hard-coded. The price oscillates wildly while the
filter stays in a smooth 10..19 band, so bars 2-5 sit clearly
above or below it with a wide margin -- positions
`on, on, above, below, above, below`, 2 above and 2 below.

## API

- `getLineSuperSmootherFinitePoints(points)` -- drop points with
  a non-finite `x` or `value`.
- `normalizeLineSuperSmootherPeriod(period, fallback)` -- coerce
  to a positive integer; non-finite or sub-1 falls back,
  fractional floors.
- `computeLineSuperSmootherCoefficients(period)` -- the two-pole
  filter coefficients `{ a1, b1, c1, c2, c3 }`.
- `computeLineSuperSmoother(values, period)` -- the filter line,
  a plain `number[]` (no warm-up).
- `runLineSuperSmoother(points, options?)` -- sort finite points
  by `x`, compute the filter, return `{ series, period,
  smoother, samples, smootherFinal, smootherMin, smootherMax,
  aboveCount, belowCount, ok }`. Each sample is classified
  `above` / `below` / `on` the smoother. `ok` is false for fewer
  than two points.
- `computeLineSuperSmootherLayout(options)` -- project the price
  path and the smoother path on one shared panel.
- `describeLineSuperSmootherChart(data, options?)` -- ARIA
  description.

## Component

`ChartLineSuperSmoother` is a `forwardRef` region
(`role="region"`) with an ARIA-linked description, a
`role="img"` SVG, axis ticks and grid, a config badge, a
two-series legend (Price / Super Smoother) with toggle, a
hover/focus tooltip exposing the value, the smoother reading and
the position, and a `motion-safe:animate-fade-in` entrance. It
consumes `{ x, value }` points. Every node carries a
`data-section` attribute.

## Distinctness

- **vs the single-pole exponential averages (an EMA,
  chart-line-smma)** -- those are one-pole filters with a gentle
  6 dB/octave roll-off. The Super Smoother is a true *two-pole*
  filter, so it rejects high-frequency noise far more sharply
  for the same lag.
- **vs chart-line-laguerre** -- the Laguerre filter chains four
  Laguerre-polynomial stages governed by gamma. The Super
  Smoother is a fixed second-order recursion whose coefficients
  come directly from a cutoff period via `exp` and `cos`.
- **vs chart-line-mcginley** -- the McGinley Dynamic adapts its
  divisor every bar. The Super Smoother is a fixed linear
  filter -- its response is set entirely by the period.

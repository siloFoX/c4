# 11.188 -- Command palette v3 polish

Polish notes for `web/src/components/CommandPalette.tsx`.

## Scope

Component-only polish (no daemon / API changes). Builds on the v1.11.86
foundation (Cmd+K open, fuzzy filter, Recent section).

## Changes

### 1. Favorites

- New per-row Star toggle rendered to the right of the optional `<Kbd>`
  shortcut chip. Uses `lucide-react`'s `Star` icon (fills when active).
- Click handler stops propagation so the favorite toggle does not also
  activate the underlying command row.
- Persisted to localStorage under the key `c4:cmdk:favorites` (string
  array of command ids). Reads on every `open` flip so a star toggled in
  another tab is reflected the next time the palette mounts.
- A `Favorites` section is pinned at the top of the listbox whenever the
  query is empty and the seed has at least one resolvable id. Items
  that appear in `Favorites` are deduped out of `Recent` and the
  regular section grouping so the catalog stays scannable.

### 2. Action mode

- Typing `>` as the first character flips the palette into action mode.
  The leading `>` is stripped from the effective query passed to
  `filterCommands`, so `>r` matches `Reload` via the existing
  substring/acronym matcher.
- In action mode the listbox renders a single `Actions` section. Nav
  results, Favorites, and Recent are hidden until the leading `>` is
  removed.
- A `<Chip>` badge (`web/src/components/ui/chip.tsx`) sits to the right
  of the search input and reads `Nav` (subtle / neutral) or `Action`
  (solid / primary) based on the current mode. Marked with
  `data-command-mode` for tests.

### 3. Quick actions

| id                    | label             | shortcut | behaviour                                              |
| --------------------- | ----------------- | -------- | ------------------------------------------------------ |
| `action:quit`         | `Quit`            | `q`      | Calls the palette's `onClose` (placeholder for now).   |
| `action:reload`       | `Reload`          | `r`      | `window.location.reload()`.                            |
| `action:toggle-theme` | `Toggle theme`    | `t`      | Cycles `system -> light -> dark` via `useTheme`.       |
| `action:open-settings`| `Open Settings`   | `s`      | `ctx.navigateTopView('settings')`.                     |
| `action:show-shortcuts`| `Show shortcuts` | `?`      | Dispatches `c4:shortcuts-open` (existing HelpUIRoot). |

Each action lists a one-key hint via the existing `<Kbd>` chip so the
new rows visually align with the Queue tick shortcut already in nav
mode.

## Tests

Cases added in `web/src/components/CommandPalette.test.tsx`:

- Star toggle renders for every command row.
- Click on the star persists the id under `c4:cmdk:favorites`.
- Second click removes the id.
- `Favorites` is the top section when the storage seed is non-empty.
- Typing `>` flips to action mode and the listbox shows the quick
  action ids.
- Mode badge reads `Nav` by default and `Action` after typing `>`.
- Action mode `Reload` invokes the (mocked) `window.location.reload`.
- Action mode `Toggle theme` cycles the persisted `c4:theme` value.

## Out of scope

- The `Quit` action is a placeholder; a real shutdown wire-up is left
  for a follow-up.
- No changes to the underlying `commands.ts` catalog or its unit tests.
- No daemon-side API additions; favorites live entirely in browser
  localStorage.

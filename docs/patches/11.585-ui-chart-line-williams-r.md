## Summary

New `ChartLineWilliamsR` UI primitive at
`web/src/components/ui/chart-line-williams-r.tsx` -- a pure-SVG line
chart with a Williams %R momentum oscillator panel. The value line
occupies the top panel; the bottom panel renders the %R oscillator on
a 0 to -100 scale (0 at the top, -100 at the bottom) with dashed
-20/-80 level lines and shaded overbought/oversold zones. Both panels
share one x-axis.

## Indicator math

`computeLineWilliamsR(values, period)` is Larry Williams' Percent
Range. For each index from `period - 1` onward the window of `period`
values yields:

```
%R = -100 * (highestHigh - value) / (highestHigh - lowestLow)
```

The result runs 0 when the value sits at the window high down to -100
when it sits at the window low. A flat window (zero range) reads -50,
the neutral middle. Indices before the window fills read `null`.

### Worked fixture

`values = [10, 14, 12, 18, 8, 16]`, `period = 3`:

```
i=2 window [10,14,12]: hh 14, ll 10 -> -100*(14-12)/4  = -50
i=3 window [14,12,18]: hh 18, ll 12 -> -100*(18-18)/6  = 0
i=4 window [12,18,8] : hh 18, ll 8  -> -100*(18-8)/10  = -100
i=5 window [18,8,16] : hh 18, ll 8  -> -100*(18-16)/10 = -20

williamsR = [.,.,-50, 0, -100, -20]
```

`%R[3] = 0` (value at the window high) is overbought; `%R[4] = -100`
(value at the window low) is oversold; `williamsRFinal = -20`,
`overboughtCount = 1`, `oversoldCount = 1`.

## API

- `getLineWilliamsRFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineWilliamsRPeriod(period, fallback)` -- coerce a period
  to a positive integer; non-finite or sub-1 falls back, fractional
  floors.
- `computeLineWilliamsR(values, period)` -- the %R series.
- `runLineWilliamsR(points, options?)` -- sort finite points by `x`,
  compute the %R series, classify each reading as overbought /
  oversold / neutral, return `{ series, period, overbought, oversold,
  williamsR, samples, williamsRFinal, overboughtCount, oversoldCount,
  ok }`. `ok` is false for fewer than two points.
- `computeLineWilliamsRLayout(options)` -- stack a value panel above a
  fixed 0 to -100 %R panel. Returns panels, ticks, the value path, %R
  path, zone-tagged markers, the level positions, and the two shaded
  zone rectangles.
- `describeLineWilliamsRChart(data, options?)` -- ARIA description.

## Component

`ChartLineWilliamsR` is a `forwardRef` region (`role="region"`) with
an ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge (period + total extreme count), a two-series legend (Value / %R)
with toggle, dashed -20/-80 level lines, two shaded extreme zone
rectangles, zone-coloured %R markers, a hover tooltip exposing the
value, %R, and zone, and a `motion-safe:animate-fade-in` entrance.
Every node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-stochastic** -- although %R is algebraically
  `%K - 100`, the stochastic plots *two* bounded 0-100 lines (%K and a
  smoothed %D signal) and reads a high %K as overbought. Williams %R
  is a *single* line on a negative 0 to -100 scale with the
  orientation inverted -- a %R near 0 is overbought, near -100 is
  oversold -- and carries no signal line. The %R numerator is measured
  from the window *high* (`highestHigh - value`) rather than the
  window *low*, and the panel y-axis runs 0 at the top down to -100.

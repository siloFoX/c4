## Summary

Add a new `<DataExport>` UI primitive in
`web/src/components/ui/data-export.tsx` -- format selector
(CSV / JSON / XLSX), filename input, and a view-vs-all scope
toggle. The Export button fires
`onExport({ format, filename, scope, onProgress })`; hosts
stream progress back via the supplied `onProgress(0..1)`
callback or pin a `progress` prop for fully-controlled
behaviour. The primitive renders a progress bar + a disabled
button while the export is in flight.

## API

```ts
type ExportFormat = 'csv' | 'json' | 'xlsx';
type ExportScope = 'view' | 'all';

interface ExportInvocation {
  format: ExportFormat;
  filename: string;
  scope: ExportScope;
  onProgress?: (progress: number) => void;
}

interface DataExportProps {
  formats?: readonly ExportFormat[];   // default ['csv','json','xlsx']
  format?: ExportFormat;               // controlled
  defaultFormat?: ExportFormat;        // default 'csv'
  filename?: string;                   // controlled
  defaultFilename?: string;
  baseFilename?: string;               // default 'data'
  scope?: ExportScope;                 // controlled
  defaultScope?: ExportScope;          // default 'view'
  onFormatChange?: (format: ExportFormat) => void;
  onFilenameChange?: (filename: string) => void;
  onScopeChange?: (scope: ExportScope) => void;
  onExport: (i: ExportInvocation) => Promise<void> | void;
  progress?: number;                   // 0..1, fully-controlled
  isExporting?: boolean;               // controlled
  errorMessage?: ReactNode;
  ariaLabel?: string;                  // default 'Data export'
  scopeLabels?: { view?: ReactNode; all?: ReactNode };
  formatLabels?: Partial<Record<ExportFormat, string>>;
  disabled?: boolean;
  showScopeToggle?: boolean;           // default true
  className?: string;
}
```

## Behaviour

- Three controls in a single row: a `<select>` of formats, a
  sanitised `<input>` for the filename, and a radio-group
  toggle for the scope.
- Filename is sanitised on every keystroke
  (`sanitizeFilename` strips path separators, shell-reserved
  chars, ASCII control bytes; collapses whitespace and
  trims).
- Changing the format auto-rewrites the filename extension
  via `ensureFilenameHasExtension` -- but ONLY when the
  filename is uncontrolled. Hosts that pin the filename own
  the rename.
- Export button submits the form (Enter key in the input
  also submits). The button is disabled while a previous
  export is in flight and when the filename is empty.
- `onExport(invocation)` is awaited. While running, an
  internal `isExporting` is set to true (only when the host
  has not pinned the `isExporting` prop). The invocation's
  `onProgress` callback updates an internal mirror so the
  progress bar reflects host-reported progress even when
  uncontrolled.
- `progress` (0..1) is fully-controlled when supplied -- the
  host owns the number and the primitive renders it
  verbatim.
- `errorMessage` opt-in: renders an inline alert row above
  the action button.
- `showScopeToggle=false` hides the radio group for flows
  that always export the current view.

## Pure helpers (exported)

```ts
DEFAULT_EXPORT_FORMATS = ['csv', 'json', 'xlsx']
DEFAULT_EXPORT_FORMAT = 'csv'
DEFAULT_EXPORT_SCOPE = 'view'
DEFAULT_EXPORT_BASE_FILENAME = 'data'
EXPORT_FORMAT_LABELS = { csv: 'CSV', json: 'JSON', xlsx: 'XLSX' }
getFileExtensionForFormat(format): string
sanitizeFilename(filename): string
ensureFilenameHasExtension(filename, format): string
getDefaultExportFilename(format, scope, base?): string
formatProgressPercent(progress): string
```

- `sanitizeFilename` is conservative -- it removes every
  char that breaks a cross-platform write (path separators,
  shell meta, ASCII controls) and collapses whitespace.
- `ensureFilenameHasExtension` knows the three formats so
  flipping CSV -> JSON rewrites `orders.csv` ->
  `orders.json` instead of stacking `orders.csv.json`.
- `formatProgressPercent` clamps to `[0%, 100%]`, rounds to
  integer, and guards NaN.

- Export click fires `onExport` with the current selection;

## Pairs with existing primitives

- `<DataTable>` / `<VirtualizedList>` for the rows that
  feed the export.
- `<Progress>` underlies the inline progress-bar pattern;
  this primitive bakes its own progress chip into the
  export workflow.
- `<Toast>` for post-export success / failure
  notifications.

## Out of scope

- File generation itself (host owns CSV / JSON / XLSX
  serialisation).
- File-save dialog or download trigger (host owns whether
  to call `URL.createObjectURL` / push to S3 / etc).
- Schema selection / column-picker (host can wrap with
  `<ColumnPicker>` if needed).
- Resumable / chunked uploads (host implements the resume
  protocol).

## Summary

New `ChartLineVfi` UI primitive in `web/src/components/ui/chart-line-vfi.tsx`
-- a pure-SVG two-panel line chart with a **Markos Katsanos Volume Flow
Indicator** panel from cumulative signed volume relative to its mean.
The top panel plots the close; the bottom panel plots the VFI.

## The computation

Markos Katsanos's VFI reads the bias of volume across the lookback
window. `ChartLineVfiPoint` carries `{x, high, low, close, volume}`.

- `computeLineVfiTypical(bars)` -- the typical price per bar,
  `(high + low + close) / 3`.
- `computeLineVfiSignedVolume(bars)` -- signs each bar's volume by the
  direction of the typical change from the prior bar: positive when the
  typical rose, negative when it fell, zero when it held. The first bar
  has no prior, so it is null.
- `computeLineVfi(bars, options)` -- divides the cumulative signed
  volume over the lookback by the cumulative volume over the same
  window and scales to 100:
  `vfi = 100 * sum(signedVolume, period) / sum(volume, period)`. Because
  `|signedVolume| <= volume`, the ratio sits in [-1, 1], so the VFI is
  bounded to [-100, 100].
- `classifyLineVfiZone(vfi)` -- marks a bar **up** when the indicator is
  positive, **down** when negative, **flat** when zero.

## Worked fixture

The whole pipeline -- typical, signed volume, rolling sums, ratio -- is
pure rational arithmetic with no transcendentals, so an integer fixture
computes to an exact integer VFI.

A **strictly rising** typical (`close [10,12,14,16,18,20,22,24]`,
half-range `+/-1` so the typical equals the close) makes every bar an up
bar; the signed volume equals the volume, the sums match and the VFI
pins to exactly +100. A **strictly falling** typical mirrors it to -100.
A **flat typical** (constant close 50, half-range +/-2) gives every
signed volume exactly zero and the VFI exactly 0. An **alternating
fixture** -- closes `[10,12,11,13,11,14,12,15]`, volumes
`[100,200,100,200,100,200,100,200]` -- gives the exact integer VFI
series `[null,null,null,60,0,60,0,60]` (asserted with `toEqual`). The
zone classifier is checked directly for up, down, flat and none.

## API

`data` (a `ChartLineVfiPoint[]`), `period` (default 26),
`pricePanelRatio` / `gap`, sizing/format/a11y props, `strokeWidth` /
`dotRadius`, the colour props, the toggle flags (`showAxis` / `showGrid`
/ `showDots` default false / `showVfi` / `showZeroLine` / `showMarkers`
/ `showTooltip` / `showConfigBadge` / `showLegend` / `animate`),
controlled + uncontrolled `hiddenSeries` (keys `price` / `vfi`),
`formatValue` / `formatX`, `onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineVfiFinitePoints`, `normalizeLineVfiPeriod`
(coerces the period to an integer of at least 2), `computeLineVfiTypical`,
`computeLineVfiSignedVolume`, `classifyLineVfiZone`, `computeLineVfi`,
`runLineVfi`, `computeLineVfiLayout`, `describeLineVfiChart`, plus the
`DEFAULT_CHART_LINE_VFI_*` constants, `CHART_LINE_VFI_BOUND` and the
`ChartLineVfiZone` / `ChartLineVfiComputed` types. Every exported name
carries a `LineVfi` infix so the `export *` barrel line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen `data-section="chart-line-vfi-aria-desc"`
description span; the close line, the VFI line and every dot / marker
are `role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-period`, `data-vfi-final`,
`data-up-count`, `data-down-count`, `data-flat-count`,
`data-total-points` and `data-animate`.

# 11.1063 -- UI: chart-line-roc-mid-cross-sig primitive (v1.11.1081)

## Scope

Add a pure-SVG dual-panel `ChartLineRocMidCrossSig`
component that renders the close in the top panel and the
Rate of Change (ROC) oscillator plus its smoothed SMA
signal line in the bottom panel, marking bullish /
bearish ROC-over-signal crossover trigger events with
bias coloring derived from the ROC slope at the trigger
bar.

ROC is the percentage variant of the canonical Momentum
oscillator. Where Momentum gives the raw price change in
absolute units, ROC gives the relative change as a
percentage, making it comparable across instruments with
different price scales.

## ROC cross-sig pipeline

```
roc[i]       = (close[i - period] !== 0)
               ? 100 * (close[i] - close[i - period])
                     / close[i - period]
               : null  (zero-guard: indeterminate)
signal[i]    = SMA(roc, signalLength)

bullish      : prev roc <= prev signal &&
               cur roc > cur signal
bearish      : prev roc >= prev signal &&
               cur roc < cur signal
regime       : 'bullish' when roc >= signal
               'bearish' when roc <  signal
               'none'    when either is null
bias         : roc[i] vs roc[i-1] -> up / down / flat /
               none
```

Default `period = 10`, `signalLength = 3` -- matches the
Momentum sibling default.

The zero-guard on `close[i - period] === 0` returns
`null` per pandas-ta / TA-Lib convention: the percentage
change is undefined when the reference price is zero.
Regime stays `none` until valid ROC is available.

Warmup is `period + signalLength - 1 = 12` for the
default tuning: ROC is valid from i = period = 10
(assuming non-zero close[0]), then the signal SMA needs
`signalLength - 1 = 2` more bars.

## Bit-exact anchors

All anchors use close-only input.

- **CONST band** `close = K, K > 0`: ROC = 100 * (K - K)
  / K = 0 / K = 0 (exactly the centerline). signal = SMA
  (0, 3) = 0. ROC === signal -> regime `bullish` (>=)
  for every valid bar. 0 crosses. Verified across K in
  {1, 50, 200, 1234}.
- **CONST K = 0**: ROC = 0 / 0 -> `null` via zero-guard.
  regime `none` for every bar. 0 crosses. Documented
  limitation of ROC on zero reference prices.
- **LINEAR UP shifted** `close = i + 100`: close[i] -
  close[i - period] = period = 10. close[i - period] =
  i + 100 - 10 = i + 90. ROC = 1000 / (i + 90). Non-
  constant (decays from ~10 at i=10 toward 0 as i
  grows). Verified at specific i values:
  - i=10: ROC = 10
  - i=20: ROC = 1000/110 ~= 9.09
  - i=30: ROC = 1000/120 ~= 8.33
- **LINEAR DOWN shifted** `close = 1000 - i`: ROC =
  100 * (-10) / (1000 - i + 10). Negative, magnitude
  approaches 1 as i grows. Verified at i=10: ROC =
  100*(-10)/1000 = -1.
- **0 crosses on monotonic input**: ROC is monotonically
  decreasing on LINEAR UP (decay shape), signal lags so
  signal > roc -> regime bearish throughout. No
  transitions -> 0 crosses. Same on LINEAR DOWN.

Unlike the Momentum sibling (which produces clean
integer constant LINEAR anchors at +/- period), ROC does
**not** produce constant LINEAR anchors due to the
percentage normalisation. This is the canonical
limitation of ROC: it's not scale-invariant. For bit-
exact testability the primitive verifies (a) CONST cases
work exactly, (b) specific i values on LINEAR shifted,
(c) no crosses fire on monotonic input.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 3-state regime classifier follows the cross-sig family
  convention (bullish on `===`, bearish on `<`).
- ROC is unbounded -- the layout panel auto-ranges but
  clamps to keep the centerline (0) visible regardless
  of data range. Matches Momentum / DPO / TRIX / CCI
  (unbounded with centerline guard).
- Zero-guard returns null per pandas-ta / TA-Lib
  convention. Some libraries return 0 or NaN -- this
  primitive uses null for clean indeterminate semantics
  in the regime classifier.
- **Unlike all other mid-cross-sig family members, ROC
  does not produce bit-exact constant LINEAR anchors**
  due to its percentage normalisation. The Momentum
  sibling (v1.11.1080) provides the absolute-value form
  with clean integer anchors. ROC's decay-shape on
  LINEAR is the canonical limitation of percentage-
  normalised oscillators. Documented with specific-i
  anchors in tests.
- ROC can produce extreme values (~+/-100% over a
  10-bar window) on volatile data. Real-world dashboards
  often clamp ROC to a fixed range for display
  consistency; this primitive auto-ranges per data.
- Direct sibling to chart-line-momentum-mid-cross-sig
  v1.11.1080 (raw difference variant); together they
  bracket the simplest-momentum centerline family.
  Companion to the broader mid-cross-sig family (SMA,
  EMA, Bollinger, Keltner, Donchian, CCI, RSI,
  Ichimoku, Supertrend, Williams, DPO, TRIX, HMA).

# 11.564 -- web/src/components/ui/chart-line-winsorized.tsx

Pure-SVG line chart that overlays a **winsorized** copy of a
series against the raw line. Winsorization is a global
percentile clamp: any value below the lower-quantile bound is
clamped **up** to that bound, any value above the
upper-quantile bound is clamped **down** to it. Unlike
trimming (which discards outliers), winsorizing **replaces**
them with the boundary value, so the sample count is
preserved.

## Math

`computeLineWinsorizedQuantile(values, q)` is a
linear-interpolation quantile (the numpy `linear` / R type-7
convention): the position is `h = (n - 1) * q`, and the
result interpolates between the two surrounding order
statistics. Non-finite values are dropped; an empty input
yields NaN; `q` is clamped to `[0, 1]`.

`runLineWinsorized(points, lowerQuantile, upperQuantile)`:

1. Sort the finite points ascending by x; collect the y
   values.
2. `lowerBound = quantile(y, lowerQuantile)`,
   `upperBound = quantile(y, upperQuantile)`. The bounds are
   computed **once over the whole series** -- this is a
   global percentile clamp, not a rolling-window filter.
3. Per sample: `winsorized = clamp(y, lowerBound,
   upperBound)`. `clampedLow` when `y < lowerBound`,
   `clampedHigh` when `y > upperBound`, `delta = winsorized
   - y`.

The quantile arguments are clamped to `[0, 1]`; if the lower
quantile exceeds the upper, the pair is swapped so the band
is always well ordered. A value sitting exactly on a bound
is **not** clamped.

The component fixture makes the bounds exact: y sorted is
`[0, 10, 20, 30, 100]`; with quantiles `0.25 / 0.75` the
positions `h = 4 * 0.25 = 1` and `h = 4 * 0.75 = 3` land
directly on order statistics, giving `lowerBound = 10` and
`upperBound = 30`. The winsorized series is then
`[10, 10, 20, 30, 30]` -- the `0` is clamped up to `10`
(`delta +10`) and the `100` down to `30` (`delta -70`).

## Distinct from neighbouring primitives

- `chart-line-hampel` replaces outliers using a **rolling**
  window MAD test -- a *local* decision. Winsorize uses a
  single *global* percentile band.
- `chart-line-percentile` draws a percentile **band** as
  context but never transforms the series; ChartLineWinsorized
  produces a new clamped line.
- `chart-line-threshold` clips against **fixed user-supplied
  thresholds**; winsorize derives its bounds from the data's
  own quantiles and clamps **both** tails symmetrically in
  intent.
- `chart-line-savgol` / `chart-line-loess` smooth; they do
  not clamp.

All exported helpers carry a `LineWinsorized` infix so the
`export *` barrel line cannot collide.

## Public surface

- `<ChartLineWinsorized data={...} />` -- a single
  `ChartLineWinsorizedPoint[]` (`{x, y}`); winsorization is a
  single-series transformation.
- `lowerQuantile` / `upperQuantile` -- the tail quantiles
  (defaults `0.05` / `0.95`).
- `rawColor` / `winsorizedColor` / `boundColor` /
  `clampColor` -- the raw line, the clamped overlay, the
  bound lines/band, the clamped markers.
- `rawOpacity` -- the raw line is dashed + dimmed; the
  winsorized line is solid.
- Toggle flags: `showAxis`, `showGrid`, `showDots` (default
  **false**), `showLegend`, `showTooltip`, `showConfigBadge`,
  `showBounds`, `showBoundBand`, `showClampedMarkers`,
  `animate`.
- Controlled + uncontrolled visibility (`hiddenSeries` keys
  `raw` / `winsorized`). Hiding the winsorized series also
  drops its dots and clamped markers.
- Standard sizing + format + a11y props. `formatValue` /
  `formatX`.
- `onPointClick({sample})` / `onSeriesToggle({seriesId,
  hidden})`.

## Pure helpers

- `getLineWinsorizedDefaultColor(index)`.
- `getLineWinsorizedFinitePoints(points)`.
- `computeLineWinsorizedQuantile(values, q)` -- the
  linear-interpolation quantile.

  -- the full transformation; reports `lowerBound`,
  `upperBound`, `clampedCount` / `clampedLowCount` /
  `clampedHighCount`.
- `computeLineWinsorizedLayout({...})` -- projects the raw +
  winsorized paths and the bound lines.
- `describeLineWinsorizedChart(data, options?)`.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Raw path + winsorized path + every dot + every clamped
  marker `role="graphics-symbol"` + `tabIndex={0}`. The
  clamped-marker aria-label states the clamp direction, x,
  raw value and clamped value.
- Every node carries `data-section="chart-line-winsorized-..."`.
- Root mirrors `data-total-samples`, `data-clamped-count`,
  `data-clamped-low-count`, `data-clamped-high-count`,
  `data-lower-bound`, `data-upper-bound`, `data-animate`.
- Raw path `data-kind="raw"`; winsorized path
  `data-kind="winsorized"`. Bound lines carry
  `data-edge="upper"|"lower"`. Clamped markers carry
  `data-direction="low"|"high"`.

## Tooltip / badge

Hovering a dot or a clamped marker shows the sample number,
x, raw value, the bold winsorized value, and a clamp row
(`clamped low/high (delta +/-N)` or `not clamped`). Config
badge: `WIN lo=<lowerBound> hi=<upperBound> clamp=<count>`.
Legend has two toggle items (`Raw` / `Winsorized`) plus a
`<n> clamped (<low> low / <high> high)` tally.

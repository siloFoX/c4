# Patch 11.182 -- UI: ContextMenu primitive

Release: v1.11.200 (2026-05-14)

## Summary

New `<ContextMenu>` primitive at `web/src/components/ui/context-menu.tsx`
backing right-click triggered action menus across the web UI. The menu is
rendered through `createPortal` into `document.body` and positioned at the
captured `clientX` / `clientY`, then clamped against `window.innerWidth` /
`innerHeight` after measuring its own `getBoundingClientRect` so it never
spills off the viewport.

## API

```ts
export interface ContextMenuItem {
  id: string;
  label: ReactNode;
  icon?: ReactNode;
  disabled?: boolean;
  danger?: boolean;
  onSelect?: () => void;
  separator?: boolean;
}

export interface ContextMenuProps {
  trigger: ReactElement;
  items: ContextMenuItem[];
  ariaLabel?: string;
  className?: string;
}
```

`ContextMenu` `forwardRef`s to the trigger element (not the portal node).

## Behavior

- Wraps the trigger via `cloneElement` and attaches `onContextMenu` which
  calls `preventDefault()` and opens the menu at the event coordinates.
- ArrowDown / ArrowUp move the highlight, skipping `disabled` items and
  `separator` rows.
- Enter activates the highlighted item then closes (focus restored to the
  trigger).
- Escape closes through the existing `useFocusTrap({ onEscape })` hook.
- Tab cycles within the menu via `useFocusTrap` (focus trap also restores
  focus on unmount).
- Click on an enabled item calls `onSelect` then closes; disabled items are
  no-ops.
- `mousedown` capture outside the menu closes it.
- `role='menu'` on the container, `role='menuitem'` per item, `role='separator'`
  for separator rows.
- `aria-disabled='true'` is set on disabled items.
- `danger=true` tints the item label with `text-destructive`.

## Adoption (2 sites)

1. `web/src/components/WorkerList.tsx` -- wraps each row Card in a
   `ContextMenu` with Attach / Logs / Kill actions. Attach + Logs reuse
   `onSelect(w.name)`; Kill is a placeholder handler with `danger=true`
   pending the daemon kill endpoint.
2. `web/src/components/SessionsListSection.tsx` -- wraps each per-session
   `ListItem` with a ContextMenu of Open / Rename / Delete. Open reuses
   `onSelect(sessionId)`; Rename + Delete are placeholder handlers.

## Notes / non-goals

- No submenu / nested menu support.
- No type-ahead (DropdownMenu's `TYPEAHEAD_RESET_MS` loop is not ported
  here -- right-click menus are typically short).
- No animation.
- The trigger element must accept `onContextMenu` and (when ref-forwarding
  is desired) `ref` -- works with any host element and with `Card`,
  `ListItem`, etc. since those forward both.

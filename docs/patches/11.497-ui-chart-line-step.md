# TODO 11.497 - chart-line-step primitive

Status: SHIPPED v1.11.515 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartLineStep>`, a pure-SVG step-line chart that supports the
full family of step interpolation types: `'before'` (vh / step left),
`'after'` (hv / step right - default), and `'center'` (hvh / step at
midpoint). Each series renders as one path stroked with straight
horizontal + vertical segments. Distinct from the v1
`<ChartStep>` primitive (which paints a single hardcoded step type
suited to staircase-style cumulative data); chart-line-step exposes
the step type as a first-class prop and adds the `center` mode that
chart-step does not support.

## Step types

For each segment from `(prev.x, prev.y)` to `(curr.x, curr.y)`:

- **after** (`hv`, default): emit `H curr.x V curr.y`. The previous y
  value persists until the new x, then snaps to the new y. Use this
  when the y value at a sample applies *forward* in time until the
  next sample (canonical event-driven data).
- **before** (`vh`): emit `V curr.y H curr.x`. The new y value applies
  at the *previous* x. Use this when the y value reflects what was
  measured *before* the timestamp (canonical batch-end data).
- **center** (`hvh`): emit
  `H (prev.x + curr.x)/2 V curr.y H curr.x`. The y transition happens
  at the midpoint between the two x values. Use this when neither
  side better represents the transition (canonical midpoint-rule
  data).

## API

```tsx
<ChartLineStep
  series={[
    {
      id: 'cpu',
      label: 'CPU',
      data: [
        { x: 0, y: 12 },
        { x: 1, y: 80 },
        { x: 2, y: 45 },
        { x: 3, y: 90 },
      ],
    },
  ]}
  stepType="center"
  xLabel="t (s)"
  yLabel="%"
/>
```

## Distinct from prior chart-* primitives

- `<ChartStep>` - single hardcoded step interpolation (legacy);
  chart-line-step parametrises the step type.
- `<ChartLine>` - smooth/linear continuous line between points; no
  horizontal-vertical segmentation.
- `<ChartStream>` / `<ChartStreamgraph>` - stacked stream layers with
  curve interpolation; chart-line-step is non-stacked, per-series
  step lines.
- `<ChartArea>` - filled area below the line; chart-line-step strokes
  only the step path.
- `<ChartScatter>` - 2D points without connecting line.

## Layout

### Value range

- `getLineStepBounds` auto-derives `(xMin, xMax, yMin, yMax)` from
  every finite point across visible series; collapsed axes expand
  `+/- 0.5`; empty -> unit square `(0..1, 0..1)`.
- Explicit `xMin` / `xMax` / `yMin` / `yMax` props override per-axis.

### Step path

- `buildStepLinePath(points, stepType)` is the canonical exported
  builder. Inputs are *pixel-space* points; output is an SVG path
  string with `M` / `H` / `V` commands.
- Empty points -> empty path.
- Single point -> a lone `M` (degenerate).
- Numbers are formatted with 3 decimal places for stable strings.

### Dots

- One dot per finite point at the input `(x, y)` (not at corners).
- Hover or focus on a dot dims every other dot to opacity 0.3.

## Interactivity

- Hover or focus on a dot opens the tooltip showing series label, `x`
  (via `formatX`), and `y` (via `formatValue`).
- Hover or focus also dims unrelated series strokes to 0.3 (the
  hovered series stays at full opacity).
- Per-dot `onPointClick({series, point})` callback.
- Legend buttons toggle series visibility (controlled +
  uncontrolled).
- Mount fade via `motion-safe:animate-fade-in` (suppress via
  `animate=false`).

## Exported helpers

- `DEFAULT_CHART_LINE_STEP_*` numeric / string defaults.
- `DEFAULT_CHART_LINE_STEP_PALETTE` (10-colour palette).
- `getLineStepDefaultColor(index)` - palette wrap.
- `getLineStepFinitePoints(points)` - drops points with non-finite
  `x` or `y`.
- `getLineStepBounds(series, hidden?)` - per-axis bounds; collapsed
  expands `+/- 0.5`; empty -> unit square.
- `getLineStepTicks(min, max, count?)` - evenly-spaced ticks.
- `buildStepLinePath(points, stepType)` - canonical SVG path builder.
- `computeLineStepLayout(input)` - per-series layout: pixel points +
  step path + tick positions.
- `describeLineStepChart(series, hidden?, stepType?, formatValue?)`
  - ARIA summary `"Step line chart (<stepType>) with N series and K
  points. x range <xMin> to <xMax>, y range <yMin> to <yMax>."`.

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only
  `data-section="chart-line-step-aria-desc"` span.
- SVG `role="img"` with the same `aria-label`.
- Each path is `role="graphics-symbol"` with `tabIndex={0}` and
  `aria-label="<series>: step line with K points"`.
- Each dot is `role="graphics-symbol"` with `tabIndex={0}` and
  `aria-label="<series>: x=<x>, y=<y>"`.

## Data attributes

- Root: `data-section="chart-line-step"`, `data-step-type`,
  `data-series-count`, `data-visible-series-count`,
  `data-total-points`, `data-animate`.
- Series group: `data-section="chart-line-step-series-group"`,
  `data-series-id`, `data-series-index`, `data-series-color`,
  `data-series-point-count`, `data-series-finite-count`,
  `data-hovered`.
- Path: `data-section="chart-line-step-path"`, `data-series-id`.
- Dot: `data-section="chart-line-step-dot"`, `data-series-id`,
  `data-point-index`, `data-x`, `data-y`, `data-hovered`.
- Tooltip, legend, axes, grid: standard pattern with `data-axis="x"`
  or `"y"` on ticks / grid lines.

## Out of scope

- Smooth curve interpolation (cardinal / catmullRom / linear) - use
  `<ChartLine>` or `<ChartStreamgraph>`.
- Filled area below the step line - use `<ChartArea>` for the
  non-stepped variant or compose by overlaying a fill path.
- Log-scale axes - linear only.
- Animated transitions on data / stepType change (snap; mount fade
  only).
- Per-point marker shapes (circle only; v1).

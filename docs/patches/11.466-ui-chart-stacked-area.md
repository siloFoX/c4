## Summary

Add a new `<ChartStackedArea>` UI primitive in
`web/src/components/ui/chart-stacked-area.tsx` -- pure-SVG
stacked area chart with a toggleable absolute / percentage
mode and clickable legend interaction for hiding
individual series. Distinct from `<ChartArea>` (11.441) in
that the mode toggle and the legend-driven visibility
filter are first-class: a small `%`/`#` button switches
the y-axis between absolute totals and percentage
normalisation (each column sums to 1), and every legend
entry is a focusable button (`aria-pressed`) that
hides / shows its series. Brushes carry through to the
tooltip (per-bar percent and column total).

## Behaviour

- Renders a `<div role="region">` wrapping `<svg
  role="img">` carrying the same `aria-label`.
- Mode: controlled via `mode` + `onModeChange`; or
  uncontrolled via `defaultMode` (default
  `'absolute'`). A small button in the top-right
  (`%` when in absolute mode, `#` when in
  percentage) toggles between modes; suppressed via
  `showModeToggle=false`.
- Hidden series: controlled via `hiddenSeries` +
  `onHiddenSeriesChange`, or uncontrolled via
  `defaultHiddenSeries`. Every legend entry is a
  `<button aria-pressed>` -- clicking toggles the
  series's hidden state and fires `onSeriesClick`
  with the next state.
- Layout: `computeStackedAreaLayout` walks every
  series in declaration order; series 0 sits at the
  bottom of the stack, series 1 above it, and so on
  (the boundary invariant `upperY[i] ===
  lowerY[i + 1]` is verified by the test suite).
  Non-finite / non-positive values are dropped.
- Percentage mode: each column is normalised so the
  visible series sum to 1 at every x. Columns with
  total = 0 collapse to the baseline.
- Smooth interpolation: `smooth=true` uses a
  Catmull-Rom-to-cubic-Bezier pass on both upper and
  lower edges (matches `<ChartStream>` /
  `<ChartArea>`); falls back to straight polylines
  for < 3 points.
- Tooltip: hover opens a tooltip with the series
  label, raw value at that x, column total, optional
  percent of total (percentage mode only), and the
  optional x label.
- Mount animation:
  `motion-safe:animate-fade-in` per layer.
  `animate=false` removes the class.
- Default ARIA description: `"Stacked area chart
  (<mode>) with N visible layers, M samples.
  <label> sum <value>, ..."` -- empty input returns
  `"No data"`; `ariaDescription` overrides.

## Pure helpers (exported)

```ts
DEFAULT_CHART_STACKED_AREA_WIDTH       = 560
DEFAULT_CHART_STACKED_AREA_HEIGHT      = 280
DEFAULT_CHART_STACKED_AREA_PADDING     = 36
DEFAULT_CHART_STACKED_AREA_TICK_COUNT  = 5
DEFAULT_CHART_STACKED_AREA_MODE        = 'absolute'

getStackedAreaTotalAt(series, hidden, index): number
getStackedAreaMaxTotal(series, hidden): number
computeStackedAreaLayout(series, hidden, mode, innerW,
                          innerH, padX, padY):
  Array<{ series; hidden; points: StackedAreaPoint[] }>
buildStackedAreaBandPath(points, smooth): string
getStackedAreaTicks(max, mode, count?): number[]
describeStackedAreaChart(series, hidden, mode,
                          formatValue?): string
```

- `getStackedAreaTotalAt` clamps non-finite + non-
  positive values to 0; respects the `hidden` set.
- `getStackedAreaMaxTotal` falls back to 1 for empty /
  zero-length / all-non-finite data.
- `computeStackedAreaLayout` is the canonical layout
  pass. Hidden layers carry zero geometry so the
  layout array stays aligned with the input.
  Percentage mode normalises every column to 1.
- `buildStackedAreaBandPath` shares the polygon
  encoding with `<ChartStream>`: upper edge +
  reversed lower edge + close-to-Z.
- `getStackedAreaTicks` defaults to `[0..max]` in
  absolute mode and `[0..1]` in percentage mode; min
  count 2.
- `describeStackedAreaChart` produces the default
  ARIA summary including the active mode + visible
  count.

## Tests (59 cases)

- `getStackedAreaTotalAt`: visible sum, hidden
  skip, non-finite / non-positive clamp, empty 0.
- `getStackedAreaMaxTotal`: largest column total,
  respects hidden filter, empty / zero-length / all-
  non-finite fallback to 1.
- `computeStackedAreaLayout`: empty -> [], one
  entry per series with one point per sample, no
  overlap (upperY[i] == lowerY[i+1] invariant),
  percentage mode normalises columns to 1, hidden
  layers collapse to baseline.
- `buildStackedAreaBandPath`: empty -> "",
  single-point closes to Z, polyline (no C) when
  smooth=false, C commands when smooth=true with
  >= 3 points.
- `getStackedAreaTicks`: absolute ticks 0..max,
  percentage ticks 0..1, non-positive max -> [0],
  min count 2.
- `describeStackedAreaChart`: empty "No data", mode
  + summary, percentage mode reported, hidden
  excluded from visible count.

## Summary

New `ChartLineGator` UI primitive at

two-panel chart with a Gator Oscillator panel of two histograms
from the Alligator moving average jaws.

## Indicator math

The Gator Oscillator (Bill Williams) is built from the
Alligator, which is three moving averages of different speed:

- the **jaw** -- a slow moving average,
- the **teeth** -- a medium moving average,
- the **lips** -- a fast moving average.

`computeLineGatorSma(values, period)` is the simple moving
average used for each jaw.

`computeLineGator(values, jawPeriod, teethPeriod, lipsPeriod)`
computes the three averages and reduces them to two histograms:

```
upper[i] = abs(jaw[i] - teeth[i])     (the jaw-teeth gap)
lower[i] = -abs(teeth[i] - lips[i])   (the negated teeth-lips gap)
```

The upper histogram is non-negative, the lower non-positive.
When the three averages converge (the Alligator "sleeps") the
histograms shrink toward zero; when they spread (it "feeds")
they grow. Both histograms share the jaw warm-up, so they are
null until all three averages are defined.

### Worked fixture

The histograms are absolute gaps between simple moving averages
-- no transcendentals -- so the whole pipeline is exact. The
fixture (jaw 4, teeth 3, lips 2) uses closes that are all
multiples of 12, so every SMA is an integer and every histogram
value divides cleanly:

```
jaw    [., ., ., 30, 36, 36, 30, 24, 24, 30, 36, 36]
teeth  [., ., 24, 36, 40, 36, 24, 20, 24, 36, 40, 36]
lips   [., 18, 30, 42, 42, 30, 18, 18, 30, 42, 42, 30]
upper  [., ., ., 6, 4, 0, 6, 4, 0, 6, 4, 0]
lower  [., ., ., -6, -2, -6, -6, -2, -6, -6, -2, -6]
```

Each sample's phase compares its total gator spread
(`upper + abs(lower)`) to the previous bar's: a larger spread is
`feeding`, a smaller one `sleeping`, an equal one `steady`.

## API

- `getLineGatorFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineGatorPeriod(period, fallback)` -- coerce to a
  positive integer.
- `computeLineGatorSma(values, period)` -- the simple moving
  average.
- `computeLineGator(values, jawPeriod, teethPeriod, lipsPeriod)`
  -- the `{ jaw, teeth, lips, upper, lower }` series.
- `runLineGator(points, options?)` -- sort finite points by `x`,
  run the pipeline, return `{ series, jawPeriod, teethPeriod,
  lipsPeriod, upper, lower, samples, upperFinal, lowerFinal,
  upperMax, lowerMin, feedingCount, sleepingCount, ok }`. Each
  sample is classified `feeding` / `sleeping` / `steady`. `ok`
  is false for fewer than two points.
- `computeLineGatorLayout(options)` -- stack the price panel
  above a Gator panel; project the price path plus one upper and
  one lower histogram bar per defined sample, the bar width and
  the zero line.
- `describeLineGatorChart(data, options?)` -- ARIA description.

## Component

`ChartLineGator` is a `forwardRef` region (`role="region"`) with
an ARIA-linked description, a `role="img"` SVG, two stacked
panels (price on top, Gator below) with their own axis ticks and
grid, panel labels, a config badge showing the three periods, a
three-series legend (Price / Upper / Lower) with toggle, a
hover/focus tooltip exposing the upper, lower and phase, and a
`motion-safe:animate-fade-in` entrance. It consumes
`{ x, value }` points. The Gator panel renders the two
histograms as `<rect>` bars straddling a dashed zero line --
green upper bars above, red lower bars below. Every node carries
a `data-section` attribute.

## Distinctness

- **vs the line oscillators (roofing / trendflex / cog / ravi)**
  -- those draw a single oscillator line with circle markers.
  The Gator panel renders two histograms of bars, and it
  measures the convergence of three moving averages rather than
  one derived signal.
- **vs chart-line-ravi** -- the RAVI compares two moving
  averages as one signed line. The Gator compares three
  averages as two separate gap histograms.
- It is the first chart-line primitive whose sub-panel is a bar
  histogram rather than a line.

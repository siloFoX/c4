# 11.318 -- Templates page redesign (v1.11.336)

Refactors `web/src/pages/Templates.tsx` to use the Tabs
primitive for source grouping and add an inline Textarea
body editor inside the ImportTemplateForm.
Component-scope only, no daemon-side change and no `c4`
CLI surface change.

## Why

Most of the dispatch's "redesign" items already shipped
in earlier patches:

- v1.11.280: ListActionMenu for per-row actions
  (Edit / Duplicate / Archive / Remove).
- v1.11.281: FieldGroup wrapper around the import
  form.
- v1.11.283: RichText for template descriptions.
- v1.11.287: NumberInput for the max-tokens ceiling.
- v1.11.288: FileDrop for the import upload.

The two remaining dispatch items were:

1. **Tabs for type.** The flat list mixes built-in
   presets (web, api, plan, etc) with operator-defined
   customs. Source-grouping Tabs make the split a
   single click -- the same pattern just shipped on
   Profiles (patch 11.317).

2. **Textarea for body.** The page accepted import via
   FileDrop only -- an operator who wanted to paste a
   template body quickly had to first save it to a
   file. An inline body editor closes that loop.

Patch 11.318 fixes both.

## What changed

### `web/src/pages/Templates.tsx`

- **Source-grouping Tabs.** Wraps the row list in
  `<Tabs value={activeTab} onChange={setActiveTab}>`
  with three items: `all` (default-selected),
  `builtin`, `custom`. Each tab label carries a count
  chip (`All (3)` / `Built-in (1)` / `Custom (2)`)
  showing how many templates match the active fuzzy
  filter. Tabs filter the visible rows in addition to
  the existing fuzzy search filter, so the two
  compose.
- **Inline Textarea body editor.** Adds a new
  FormField inside the ImportTemplateForm with a
  Textarea (autoResize, max 16 rows, 20,000 char
  cap, character count visible). Lets the operator
  paste a template body without first saving it to a
  file. The textarea complements the FileDrop --
  the FileDrop import is still the primary path
  for batch imports; the body field is the quick
  single-template path. Captured locally until the
  daemon import endpoint accepts inline body.
- **Pagination wiring.** The pagination block now
  reads `tabFiltered.length` (the
  tab-and-search-filtered list) instead of the raw
  `filtered.length` so a Tab switch correctly reflows
  pagination state.

### Imports added

- `Tabs`, `TabsPanel`, `TabsItem` -- the source-grouping
  tab strip.
- `Textarea` -- the inline body editor.

### Data attributes

- `data-section="templates-list"` on the row UL.
- `data-testid="templates-tabs"` on the Tabs strip.
- `data-testid="templates-import-body"` on the new
  Textarea.

## Deferred (dispatch follow-ups)

- The dispatch listed FormField wrappers, FileDrop
  for import, and ActionMenu for row actions. Those
  were already delivered in earlier patches
  (v1.11.281/v1.11.283 for FormField + RichText,
  v1.11.288 for FileDrop, v1.11.280 for
  ListActionMenu) and are preserved here.
- Inline body POST: the daemon's import endpoint
  doesn't accept inline body yet. The Textarea
  value is captured locally; a follow-up patch
  will wire it through once the endpoint accepts
  the new field.

## Notes

- The Tabs pattern (source grouping + count chips +
  filter composition) is identical to the Profiles
  page redesign in patch 11.317. Same vocabulary so
  the operator's mental model carries between the
  two pages.
- The Textarea uses `autoResize maxRows={16}` which
  matches the v1.11.309 Textarea primitive's
  documented auto-grow contract. Large bodies cap
  out at 16 rows; the textarea becomes scrollable
  past that point.
- The 20,000 character cap is conservative -- most
  template bodies are 2-5 KB. The cap exists so a
  pasted runaway log doesn't blow out the page
  height; the operator can raise it if their
  templates legitimately exceed it.
- Pagination wiring change: the prior bug was that
  switching to a tab with fewer items did not reset
  the page to 1 when the new total fell below the
  current page. The `tabFiltered.length` swap
  routes the existing `useEffect` clamp through
  the right denominator.

# 11.173 -- Keyboard shortcut overlay polish

Scope: `web/src/components/KeyboardShortcutsModal.tsx` and its test file,
plus i18n bundle additions and a `web/package.json` version bump.

## What changed

1. **Categorized sections.** `SHORTCUT_ROWS` rows now carry a `category`
   field (`'navigation' | 'actions' | 'view'`). The modal renders three
   sections in a fixed order with localized headings, separated by the
   shared `<Separator>` primitive. Section assignments:

   - **Navigation:** `?`, `Shift+/`, `H`, `Ctrl+B`, `T`
   - **Actions:** `Enter`, `Shift+Enter`, `Ctrl+F`
   - **View:** `Esc`

   Each section is wrapped in `<section data-shortcuts-section="<cat>">`
   so tests and downstream tooling can locate the buckets without
   string-matching the localized headings.

2. **Search filter.** A `<SearchBar size="sm">` is rendered above the
   sections. It is controlled (`value` + `onChange`) and also wires
   `onDebouncedChange` to the same setter so callers get both the
   immediate and debounced (`debounceMs=200`) signal -- the modal uses
   the immediate one because filtering is cheap and operators expect
   the list to react per keystroke. The filter compares the trimmed
   lower-cased needle against both the row's key combo and its
   localized description label. An empty filter restores the full list;
   when no row matches, an inline `shortcuts.search.empty` message
   replaces the section block.

3. **Animated reveal.** No change needed in this component -- the
   shared `<Dialog>` already pipes `motionClass('fadeIn', ...)` to the
   backdrop and `motionClass('scaleIn', ...)` to the panel, both gated
   on `useReducedMotion()`. Documented here so future readers know the
   motion path is already covered.

4. **Global `?` hotkey.** Already wired in
   `web/src/lib/use-help-overlay-triggers.ts` (mounted from
   `HelpUIRoot`). The hook checks
   `target.tagName === 'INPUT' | 'TEXTAREA'`,
   `target.isContentEditable`, and `role="textbox"` and bails out
   without firing when any of those holds, so the `?` press inside a
   composer / search input does not open the modal. The polish task's
   "no-op when focused on an input" expectation is now locked in by
   tests in `KeyboardShortcutsModal.test.tsx` (the test renders a
   harness component that mounts the hook + modal together and drives
   `fireEvent.keyDown` on the input vs. on `window`).

## API stability

`KeyboardShortcutsModalProps` (`{ open, onClose }`) is unchanged.
`SHORTCUT_ROWS` is still exported and still has the same nine entries
with the same `keys` + `descriptionKey` values; the only addition is
the new `category` field. The new `ShortcutCategory` type is exported
for callers that want to enumerate buckets.

## i18n keys added

- `shortcuts.section.navigation` -- "Navigation" / "탐색"
- `shortcuts.section.actions` -- "Actions" / "동작"
- `shortcuts.section.view` -- "View" / "보기"
- `shortcuts.search.placeholder` -- input placeholder
- `shortcuts.search.ariaLabel` -- input aria-label
- `shortcuts.search.empty` -- no-results body copy

## Tests

`web/src/components/KeyboardShortcutsModal.test.tsx` was extended with
nine new cases on top of the existing suite:

- renders Navigation / Actions / View section headings
- groups every row under a `[data-shortcuts-section]` bucket
- renders a SearchBar with the localized aria-label
- filters by description label substring (`tour`)
- filters by key combo (`Ctrl+F`)
- empty filter restores the full row count
- empty-state message renders when nothing matches
- pressing `?` on `window` opens the modal
- pressing `?` while an input is focused does NOT open the modal

The pre-existing tests (open=false null, dialog scaffolding, row count,
kbd glyphs, descriptions, close paths via X / backdrop / Escape, locale
flip, rerender stability) were left intact -- the section refactor
preserves the row count and kbd count because each data row still
lives in a `<tr>` under `<tbody>` even though the table is now split
across three `<section>` blocks.

## Out of scope

- No changes to other UI primitives. `SearchBar`, `Dialog`,
  `Separator`, `Kbd`, `IconButton` consumed as-is.
- No `App.tsx` edits. The global `?` listener already lived in
  `HelpUIRoot` via `useHelpOverlayTriggers`; we did not move it.
- `package-lock.json` untouched.
- `docs/autonomous-queue-v10.md` untouched.

# TODO 11.494 - chart-lollipop primitive

Status: SHIPPED v1.11.512 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartLollipop>`, a pure-SVG lollipop chart. Each category gets
one thin "stick" (a line from the baseline to the value position) plus
one circle "head" at the value position. Optimised for sparse
categorical data where a bar would feel visually heavy. Both
orientations (vertical / horizontal) are supported, with optional
sorting and a configurable baseline.

## API

```tsx
<ChartLollipop
  items={[
    { id: 'a', label: 'Apples', value: 12 },
    { id: 'b', label: 'Berries', value: 25 },
    { id: 'c', label: 'Cherries', value: 18 },
  ]}
  orientation="vertical"
  baseline={0}
  sortOrder="desc"
  showValueLabels
  valueLabel="kg"
  onItemClick={({ item, layout }) => console.log(item.id, layout.headY)}
/>
```

## Distinct from prior chart-* primitives

- `<ChartBar>` - rectangle bars spanning a full bandWidth; lollipop
  replaces each bar with a 1-2 px line + circle marker.
- `<ChartDotPlot>` - stacked dots showing frequency; lollipop is one
  mark per category.
- `<ChartScatter>` / `<ChartBubble>` - 2D xy; lollipop is categorical.
- `<ChartStrip>` (11.492) / `<ChartBeeswarm>` (11.493) - distribution
  viewers (many points per group); lollipop is one value per category.
- `<ChartSlope>` (11.476) - two values per category, drawing a slope
  between them; lollipop is single-value per category.

## Layout

### Value axis

- Range auto-derived from the items plus the baseline so the stick is
  always visible (`min(values, baseline)`, `max(values, baseline)`).
- Collapsed range expands by `+/- 0.5`.
- `valueMin` / `valueMax` clamp the rendered range; out-of-range
  values are clamped at the boundary (drawn at the edge instead of
  being dropped).
- `vertical` (default): value axis is y (bottom -> top).
- `horizontal`: value axis is x (left -> right).

### Category axis

- N items share the inner length of the category axis evenly. Each
  category occupies `bandWidth = innerLength / N`.
- `sortOrder` reorders items by value ascending / descending; ties
  are broken by original index. Default preserves input order.

### Stick + head

- Stick: line from baseline to value position (vertical in `vertical`
  mode; horizontal in `horizontal` mode).
- Head: circle of radius `headRadius` (default 5) at the value
  position.
- `isAboveBaseline` is true when `value >= baseline` (drives value
  label placement when `showValueLabels=true`).

## Interactivity

- Hover or focus on a head dims every other lollipop to opacity 0.35
  and opens a tooltip with the category label + formatted value.
- Click on a head fires `onItemClick({item, layout})`.
- Mount fade via `motion-safe:animate-fade-in` (suppressed via
  `animate=false`).

## Exported helpers

- `DEFAULT_CHART_LOLLIPOP_*` numeric / string defaults.
- `DEFAULT_CHART_LOLLIPOP_PALETTE` (10-colour palette).
- `getLollipopDefaultColor(index)` - palette with modulo wrap.
- `getLollipopFiniteItems(items)` - drops items with non-finite
  values; non-array -> `[]`.
- `getLollipopBounds(items, baseline?)` - includes baseline so the
  stick is always visible; empty -> `(baseline +/- 0.5)`.
- `getLollipopTicks(min, max, count?)` - evenly-spaced ticks.
- `sortLollipopItems(items, order?)` - returns `{item,
  originalIndex}` pairs; preserves original index after sort.
- `computeLollipopLayout(input)` - canonical layout returning
  `{ items, ticks, valueMin, valueMax, baseline, orientation,
  innerWidth, innerHeight, bandWidth }`; empty when canvas is
  degenerate or no finite items.
- `describeLollipopChart(items, formatValue?)` - ARIA summary
  `"Lollipop chart with N categories. Range <min> (<minLabel>) to
  <max> (<maxLabel>)."`.

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only `data-section="chart-lollipop-aria-desc"`
  span.
- SVG `role="img"` with the same `aria-label`.
- Each head circle is `role="graphics-symbol"` with `tabIndex={0}`
  and `aria-label="<category label>: <formatted value>"`.

## Data attributes

- Root: `data-section="chart-lollipop"`, `data-orientation`,
  `data-baseline`, `data-category-count`, `data-finite-count`,
  `data-sort-order`, `data-animate`.
- Item: `data-section="chart-lollipop-item"`, `data-item-id`,
  `data-item-index`, `data-item-original-index`, `data-item-color`,
  `data-item-value`, `data-above-baseline`, `data-hovered`.
- Stick / head / value text / category text: each carries
  `data-item-id` so they remain queryable by category.
- Tooltip, baseline, axes, grid, ticks: standard pattern matching the
  other chart-* primitives.

## Tests

64 vitest cases (1 constants + 2 default colour + 3 finite items + 5
bounds + 3 ticks + 4 sort + 12 layout + 4 describe + 30 component) all
pass under vitest 4.1.5. TypeScript clean for touched files.

## Out of scope

- Dumbbell / range lollipop (two heads per category connected by a
  segment); future `<ChartDumbbell>` primitive.
- Multi-series grouped lollipops; one value per category in v1.
- Log-scale value axis (linear only).
- Continuous category axis (categories are discrete; numeric axis is
  the value axis).
- Animated transitions on value / sortOrder changes (snap; mount fade
  only).

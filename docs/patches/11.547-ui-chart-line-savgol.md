# 11.547 -- web/src/components/ui/chart-line-savgol.tsx

Pure-SVG line chart with a **Savitzky-Golay** polynomial
smoothing overlay. For each interior position, fits a
polynomial of degree `p` (default 2) to a sliding window of
`windowLength` samples (default 7, must be odd) by least
squares and uses the polynomial's value at the window center
as the smoothed estimate. The first and last `m = (W-1)/2`
samples have no smoothed value because the window cannot be
centered there.

## Why Savitzky-Golay over SMA?

A Simple Moving Average flattens local peaks because it
averages over a window assuming the underlying signal is
constant. Savitzky-Golay fits a polynomial locally, so it
preserves features that match the polynomial's complexity:

- Order 0 SG == SMA (constant fit).
- Order 1 SG preserves linear trends through the window.
- Order 2 SG preserves quadratic features (peaks, valleys).
- Order 3 SG additionally preserves cubic asymmetries.

Verified by test: a quadratic input is reproduced EXACTLY by
order >= 2 smoothing at every interior position.

## Distinct from related smoothers

- `<ChartLineMovingAvg>` (11.513): SMA -- equivalent to SG
  with polyOrder=0.
- `<ChartLineEwma>` (11.539): exponential decay weights --
  recursive single-parameter smoother.
- `<ChartLineKalman>` (11.538): recursive Bayesian filter
  with Q/R noise model.
- `<ChartLineBollinger>` (11.537): rolling SMA + envelope --
  no polynomial fit.
- `<ChartLineFftWindow>` (11.544): frequency-domain filtering
  via FFT. SG is time-domain polynomial fit.
- `<ChartLineDecompose>` (11.543): seasonal split with
  centered moving average -- different decomposition goal.

The canonical scipy `savgol_filter` / numpy reference smoother
used in spectroscopy, sensor signal processing, and any
context where peak shapes matter ("don't flatten my peaks").

## Algorithm

For windowLength `N = 2m+1` and polynomial order `p`:

1. Build symmetric `(p+1) x (p+1)` matrix `M[j][k] =
   sum_{i=-m..m} i^(j+k)`.
2. Solve `M alpha = e_0` (where `e_0 = [1, 0, ..., 0]`)
   via Gauss-Jordan elimination on the augmented matrix.
3. Smoothing kernel `c_i = sum_{j=0..p} alpha_j * (i - m)^j`
   for i = 0..N-1.
4. Apply the kernel: at each interior position `i` in
   `[m, N - m)`, `smoothed[i] = sum_k coeffs[k] * y[i - m + k]`.

The kernel is **symmetric** about the center and its
coefficients **sum to 1** (preserves DC). Both invariants
verified by test.

Verified canonical kernels:
- W=5, p=2: `[-3, 12, 17, 12, -3] / 35`
- W=7, p=2: `[-2, 3, 6, 7, 6, 3, -2] / 21`

## Public surface

- `<ChartLineSavgol series={...} />`.
- `series: ChartLineSavgolSeries[]` -- `{id, label, data,
  color?, windowLength?, polyOrder?}`. Per-series overrides
  always beat chart-level.
- `windowLength` (default 7; clamped to odd >= 3; even input
  rounded up; floored from fractional).
- `polyOrder` (default 2; clamped to >= 0; floored from
  fractional). Effective order capped at `windowLength - 1`.
- `rawColor` -- dimmed raw line color.
- `residualPosColor` / `residualNegColor` -- stick colors.
- Toggle flags: `showAxis`, `showGrid`, `showDots`,
  `showLegend`, `showTooltip`, `showConfigBadge`, `showRaw`,
  `showResidualSticks`, `animate`.
- Controlled + uncontrolled visibility.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX` / `formatCoefficient`.
- `onPointClick({series, point})`.

## Pure helpers

- `getLineSavgolDefaultColor(index)` -- 10-color palette
  cycle; NaN / negative -> color 0.
- `getLineSavgolFinitePoints(points)` -- finite-only filter.
- `normaliseLineSavgolWindow(value)` -- clamps to `>= 3`;
  floors fractional; rounds even up to next odd; non-finite
  -> default 7.
- `normaliseLineSavgolPolyOrder(value)` -- clamps to `>= 0`;
  floors fractional; non-finite -> default 2.
- `computeSavgolCoefficients(windowLength, polyOrder)` --
  returns the smoothing kernel array, or null when invalid
  (window even / non-positive / polyOrder >= window).
- `classifyLineSavgolResidualSign(residual)` -- `'positive'
  | 'negative' | 'zero'`.
- `applyLineSavgol(values, windowLength, polyOrder)` --
  smooths a numeric array. Returns same-length array with
  leading + trailing `m` nulls. Non-finite within a window
  produces null at that position. polyOrder is silently
  capped at `windowLength - 1` to avoid singularities.
- `runLineSavgol(points, options?)` -- canonical pipeline.
  Sorts ascending; drops non-finite; applies the smoother;
  returns `{samples, coefficients, windowLength, polyOrder}`
  where each sample carries `{index, x, raw, smoothed,
  residual, residualSign}`.
- `computeLineSavgolLayout({series, ...})` -- projects raw
  + smoothed paths + residual segments. Y range includes
  both raw and smoothed values.
- `describeLineSavgolChart(series, options?)` -- screen-
  reader description.

## Tooltip

Hovering a dot shows the series label, x, raw, bold
smoothed (or n/a at the edges), residual (with `+`/`-`
prefix), and the SG config `W=<w>, p=<p>`.

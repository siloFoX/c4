# 11.1068 -- UI: chart-line-sar-cross-sig primitive (v1.11.1086)

## Scope

Add a pure-SVG dual-panel `ChartLineSarCrossSig` component
that renders the close in the top panel and the Wilder
Parabolic SAR with a smoothed SMA signal line in the
bottom panel, marking bullish (SAR crosses up through
signal -- trailing stop confirms uptrend; SAR is pulling
above its smoothed mean) / bearish (SAR crosses down
through signal -- trailing stop confirms downtrend; SAR
is pulling below its smoothed mean) SAR-vs-signal
crossover trigger events with bias coloring derived from
the (SAR - signal) slope at the trigger bar.

The Wilder Parabolic SAR (Stop And Reverse) is a trailing
stop line that pulls upward in uptrends (SAR below price)
and downward in downtrends (SAR above price). Crosses of
SAR against its own SMA flag trailing stop confirmation
trigger events -- the SAR's direction is sustained
enough to pull above (or below) its smoothed mean.

## SAR cross-sig pipeline

```
nextSar  = sar + af * (ep - sar)              (Wilder)
clamp    = min(nextSar, values[i-1], values[i-2])
           in uptrend (cannot cross into recent lows)
         = max(nextSar, values[i-1], values[i-2])
           in downtrend (cannot cross into recent highs)
reverse  = if price pierces nextSar -> trend flips,
           SAR jumps to ep, ep := price, af := step
           (otherwise) if new extreme posted, ep updated
           and af incremented by step capped at maxStep
signal[i] = SMA(SAR, signalLength)

bullish (SAR-cross up) :
  prev sar <= prev signal && cur sar > cur signal
bearish (SAR-cross down) :
  prev sar >= prev signal && cur sar < cur signal

regime   : 'bullish' when sar >= signal
           'bearish' when sar <  signal
           'none'    when either is null
bias     : (sar - signal)[i] vs prev -> up/down/flat/none
```

Default `step = 0.02`, `maxStep = 0.2` -- canonical Wilder
SAR tuning. Default `signalLength = 9` -- matches the
cross-sig family convention (chart-line-supertrend-cross-
sig, others).

## Sibling family

SAR signal family:

- **chart-line-parabolic-sar** -- raw SAR with reversal
  markers (the source of the SAR computation reused
  here)
- **chart-line-psar-cross-pct v1.11.x** -- SAR vs close
  percent threshold
- **chart-line-supertrend-cross-sig v1.11.x** --
  analogous trailing-stop cross-sig (Supertrend instead
  of SAR, conceptual sibling)
- **chart-line-sar-cross-sig v1.11.1086** (this
  primitive) -- SAR vs SMA(SAR) trailing stop
  confirmation

The Wilder Parabolic SAR was published in J. Welles
Wilder Jr's 1978 *New Concepts in Technical Trading
Systems*, the same book that introduced the ADX, RSI,
and the Directional Movement system. SAR cross-sig is a
trailing-stop confirmation signal in the same family as
the supertrend-cross-sig.

Uses SMA-based smoothing rather than EMA for bit-exact
determinism on CONST input. Warmup is `signalLength = 9`
for the default tuning: SAR starts emitting at i = 0
(seeded from values[0]), and the SMA window fills at i
= signalLength - 1 = 8. The first potential cross lands
at i = signalLength = 9 (needs the previous bar's signal
value).

## Bit-exact anchors

All anchors use single-value input.

- **CONST** `value = K`: First move = 0, initial trend
  'up' (>=). SAR seeds at K. Every subsequent bar:
  nextSar = K + step*(K-K) = K. Never pierced (v === K
  === nextSar, the v < nextSar check fails). SAR stays
  at K forever. signal = SMA(K, 9) = K. SAR === signal
  every bar -> no cross. regime `bullish` (via >=). 0
  crosses, 0 reversals. Verified across K in {0, 1,
  50, 200, 1234}.
- **LINEAR UP** `value = i`: First move +1, trend 'up'.
  SAR rises but is clamped by min(values[i-1],
  values[i-2]) which prevents SAR from crossing into
  recent lows. SAR forms a slow-moving lower envelope.
  SMA(SAR) trails further behind. Steady-state: SAR >=
  signal throughout the post-warmup window. 0 crosses
  (no transition from below). 0 reversals (price never
  pierces SAR). regime `bullish`.
- **LINEAR DOWN** `value = -i`: First move -1, trend
  'down'. SAR falls clamped by max(values[i-1],
  values[i-2]) which prevents SAR from crossing into
  recent highs. SAR forms a slow-moving upper envelope.
  SMA(SAR) trails further behind. Steady-state: SAR <=
  signal throughout. 0 crosses (no transition from
  above). 0 reversals. regime `bearish`.

All three steady-state anchors produce zero crosses
because SAR and signal remain on the same side of each
other throughout. Real-world crosses fire when SAR's
direction reverses or accelerates -- typically at trend
reversals or strong moves.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 3-state regime classifier follows the cross-sig
  family convention (bullish on `===` via `>=`).
- SMA-based signal smoothing (matching siblings) for
  bit-exact testability. EMA would also work but lacks
  the clean SMA(K) = K identity used for CONST anchor.
- SAR algorithm is verbatim Wilder 1978 -- same code
  path as chart-line-parabolic-sar so reversal counts
  match across the family.
- The SAR cross-sig is a confirmation signal -- it
  fires AFTER a trend has been sustained, not at the
  start. Use chart-line-parabolic-sar for raw reversal
  triggers; use this primitive for confirmation
  filters.
- On steady LINEAR input SAR saturates at a lagged
  envelope (slow-moving min/max of recent values) and
  SMA(SAR) trails further behind. Steady-state has SAR
  >= signal (uptrend) or SAR <= signal (downtrend)
  throughout -- no crosses fire. Real crosses fire when
  SAR's direction or velocity changes.
- The SMA window's `winMin === winMax` short-circuit
  keeps CONST input numerically pinned at K (not
  approximately K from floating-point summation).
- Completes the SAR signal family with the canonical
  cross-sig variant. Pairs naturally with
  chart-line-supertrend-cross-sig as the conceptual
  trailing-stop confirmation primitives.

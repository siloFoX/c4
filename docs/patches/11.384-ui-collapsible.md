## Goal

Dispatch:

- Build `web/src/components/ui/collapsible.tsx`
  -- single open/close panel with smooth
  animation, trigger slot,
  controlled/uncontrolled, ARIA disclosure
  pattern (subset of accordion).

`<Collapsible>` (11.177) already covered
most of the contract:

| dispatch item | shipped previously |
| --- | --- |
| single open/close panel | yes |
| smooth animation | `transition-all duration-200` + `motion-reduce:transition-none` |
| controlled / uncontrolled | `open` + `onOpenChange` (controlled) / `defaultOpen` (uncontrolled) |
| ARIA disclosure pattern | `aria-expanded` / `aria-controls` / `role="region"` / `aria-labelledby` / `aria-hidden` / `hidden` |
| group wrapper | `<CollapsibleGroup>` (exclusive vs simultaneous) |

The single remaining gap was the **trigger
slot** -- the existing primitive only
exposed `title` + `description` props for
the trigger content. This patch adds a
custom-trigger escape hatch plus a
`hideChevron` toggle and ergonomic data
attributes.

## What changed

### `trigger` slot

```ts
interface CollapsibleProps {
  // ... existing
  trigger?: ReactNode;
}
```

When `trigger` is set, it replaces the
legacy title + description content
inside the trigger button. The chevron
stays as part of the button (the
open/closed visual cue is structural).
When unset, the legacy title/description
path renders byte-identical.

```tsx
<Collapsible
  title="ignored"
  trigger={
    <span className="flex items-center gap-2">
      <SettingsIcon />
      <span>Advanced settings</span>
      <Badge>3 changed</Badge>
    </span>
  }
>
  <p>body</p>
</Collapsible>
```

### `hideChevron` opt-out

```ts
interface CollapsibleProps {
  // ... existing
  hideChevron?: boolean;
}
```

When `true`, the leading ChevronRight
glyph is omitted. Useful for trigger
compositions that supply their own
open/closed visual.

### Data attributes

| attr | values | location |
| --- | --- | --- |
| `data-section="collapsible"` | static | root section |
| `data-open` | `'true'` / `'false'` | root section |
| `data-collapsible-open` | `'true'` / `'false'` | root section (retained from 11.177) |
| `data-section="collapsible-trigger"` | static | trigger button |
| `data-section="collapsible-trigger-body"` | static | inner trigger span |
| `data-section="collapsible-chevron"` | static | chevron svg (omitted when `hideChevron`) |
| `data-section="collapsible-title"` | static | title span (omitted when `trigger` supplied) |
| `data-section="collapsible-description"` | static | description span (omitted when `trigger` supplied or description absent) |
| `data-section="collapsible-panel"` | static | panel region |

Downstream tests + storyshots can branch
on attributes without parsing class
lists.

## Pairs with existing primitives

- `<Accordion>` (11.272) -- multi-panel
  surface that composes Collapsible-
  shaped rows with its own group state.
  Reach for that when the surface needs
  the full WAI-ARIA accordion contract
  (roving tabindex + ArrowUp/Down
  navigation); reach for Collapsible
  when the section is stand-alone or
  inside `<CollapsibleGroup>`.
- `<CollapsibleGroup>` (11.177) --
  unchanged. Hosts multiple
  `<Collapsible>` items with `exclusive
  vs simultaneous` open semantics.

## Out of scope

- Per-page adoption of `trigger`.
  Additive; every existing call site
  stays byte-identical at the legacy
  `title` / `description` path.
- Animated `max-height` with measured
  `scrollHeight`. The current
  `max-h-[1000px]` ceiling stays.
  Height-measure story belongs in a
  separate patch with its own perf
  budget (matches the `<Accordion>`
  11.369 out-of-scope note).
- Vertical-axis or RTL layout flip.
  Disclosure pattern is always vertical
  by convention.
- `renderTrigger?: ({ open, toggle })
  => ReactNode` callback form. The
  current `trigger?: ReactNode` slot
  covers the common case; the callback
  variant is a heavier API that pushes
  state into the caller without a clear
  win for the existing call sites.

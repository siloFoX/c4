## API

```ts
type ActionMenuItem = ActionMenuAction | ActionMenuSeparator;

interface ActionMenuAction {
  id: string;
  label: string;
  onClick: () => void;
  icon?: ReactNode;
  variant?: 'primary' | 'secondary' | 'destructive';
  disabled?: boolean;
  shortcut?: string;
  ariaLabel?: string;
}

interface ActionMenuSeparator {
  id: string;
  type: 'separator';
}

interface ActionMenuProps {
  actions: ActionMenuItem[];
  maxVisible?: number;          // default 3
  ariaLabel?: string;            // default 'Page actions'
  className?: string;
  overflowLabel?: string;        // default 'More'
  align?: 'start' | 'end';      // default 'end'
}
```

## Behaviour

### Partition

Pure helper:

```ts
export function partitionActionMenu(
  items: ActionMenuItem[],
  maxVisible: number,
): { visible: ActionMenuItem[]; overflow: ActionMenuItem[] };
```

- When the ACTION count (separators
  do not count) is `<= maxVisible`,
  all items go to `visible`.
- Otherwise the first `maxVisible`
  actions are `visible`; the rest
  go to `overflow`.
- Trailing separators in `visible`
  and leading separators in
  `overflow` are dropped so no
  orphan dividers appear at the
  split boundary.
- Empty input -> empty partitions.
- Non-finite / negative
  `maxVisible` coerces to 0
  (everything overflows).

### Overflow menu

- The More button toggles a popover
  that renders the overflow items
  as `role="menuitem"` rows.
- More button:
  `aria-haspopup="menu"` +
  `aria-expanded` mirroring the
  open state.
- Click an item -> fires its
  `onClick` AND closes the menu.
- Click outside the More button +
  popover -> closes.
- Escape -> closes + returns focus
  to the More button.
- Popover: `role="menu"` +
  `aria-label="<ariaLabel>
  overflow"`.

### Toolbar contract (WAI-ARIA)

- Root: `<div role="toolbar"
  aria-orientation="horizontal"
  aria-label>`.
- Single-tabstop pattern:
  exactly one VISIBLE action has
  `tabIndex=0` at a time; the
  rest carry `tabIndex=-1`.
- ArrowLeft / ArrowRight cycle
  through visible focusable
  actions (wrap at ends).
- Home / End jump to first /
  last visible focusable.
- Disabled actions + separators
  are skipped in the focus
  cycle.

### Variants

| variant | classes |
| --- | --- |
| `primary` | `border-primary bg-primary text-primary-foreground hover:bg-primary/90` |
| `secondary` (default) | `border-border bg-background text-foreground hover:bg-muted` |
| `destructive` | `border-destructive bg-destructive text-destructive-foreground hover:bg-destructive/90` |

The variant IS the contract --
callers cannot smuggle arbitrary
classes via the component (use
the `className` prop on the
wrapper for layout adjustments
only).

### Disabled

- `disabled: true` flips native
  `disabled` on the button.
- `data-disabled="true"`.
- Click handler stripped.
- Cursor / opacity reflect the
  state.
- Focus cycle skips disabled
  actions.

### Align

`align="end"` (default) right-
justifies the toolbar; `"start"`
left-justifies. Mirrors on
`data-align`.

## ARIA + data attributes

Root (`<div role="toolbar">`,
`aria-orientation="horizontal"`,
`aria-label`):

- `data-section="action-menu"`
- `data-align`
- `data-visible-count`
- `data-overflow-count`
- `data-overflow-open`
  ('true' / 'false')

Visible wrapper:

- `data-section="action-menu-visible"`

Visible action button:

- `data-section="action-menu-action"`
- `data-action-id`
- `data-action-variant`
- `data-disabled`
- inner: `data-section="action-
  menu-action-icon"` / `-label`
  / `-shortcut`

Inline separator:

- `role="separator"` +
  `aria-orientation="vertical"`
- `data-section="action-menu-separator"`

Overflow:

- `data-section="action-menu-overflow"`
- More button:
  `data-section="action-menu-more"`
  + `aria-haspopup="menu"` +
  `aria-expanded`
- Popover:
  `role="menu"` +
  `aria-label="<ariaLabel>
  overflow"` +
  `data-section="action-menu-
  overflow-popover"`
- Item:
  `data-section="action-menu-
  overflow-item"` +
  `role="menuitem"` +
  `data-action-id` +
  `data-action-variant` +
  `data-disabled`
- inner sub-nodes: icon /
  label / shortcut data-section
- Overflow separator:
  `data-section="action-menu-
  overflow-separator"`

## Out of scope (deferred)

- Per-action keyboard binding.
  The `shortcut` field renders
  as a decorative `<kbd>`; host
  binds the actual hotkey.
- Resize-aware auto maxVisible.
  Adopters compute their own
  cap from container width
  and pass via the prop;
  automatic measurement is a
  follow-on.
- Multi-row overflow (drawer
  variant). v1 ships popover
  only.
- Nested overflow sub-menus.
  Out of scope; popover items
  fire onClick or do nothing.
- Drag-to-reorder. Source-array
  order is canonical.

# 11.1052 -- UI: chart-line-dpo-mid-cross-sig primitive (v1.11.1070)

## Scope

Add a pure-SVG dual-panel `ChartLineDpoMidCrossSig`
component that renders the close in the top panel and the
Detrended Price Oscillator (DPO) plus its smoothed SMA
signal line in the bottom panel, marking bullish / bearish
DPO-over-signal crossover trigger events with bias
coloring derived from the DPO slope at the trigger bar.

The Detrended Price Oscillator removes long-term trend
components from price by subtracting a backward-shifted
SMA from the close. Unlike trend-following oscillators
(MACD, RSI, etc), DPO does not predict future direction
-- it merely identifies short-term cycle peaks and
troughs by stripping the trend bias. `0` is the natural
centerline; positive DPO means the close (shift periods
ago) was above the current rolling average (cycle high),
negative DPO means below (cycle low).

## DPO pipeline

```
shift        = floor(period / 2) + 1
sma[i]       = SMA(close, period)
dpo[i]       = close[i - shift] - sma[i]    (i >= shift)
signal[i]    = SMA(dpo, signalLength)

bullish      : prev dpo <= prev signal &&
               cur dpo > cur signal
bearish      : prev dpo >= prev signal &&
               cur dpo < cur signal
regime       : 'bullish' when dpo >= signal
               'bearish' when dpo <  signal
               'none'    when either is null
bias         : dpo[i] vs dpo[i-1] -> up / down / flat /
               none
```

Default `period = 20`, `signalLength = 3`, `shift = 11`
-- standard William Blau DPO tuning paired with the
family-conventional short SMA signal.

The `shift = floor(period/2) + 1` displacement is the
standard DPO convention introduced by William Blau -- it
aligns the SMA's centroid lag with the historical close
so the residual purely reflects deviation from trend, not
lag. For period=20, shift=11. For period=10, shift=6. The
shift makes DPO non-causal (uses past data, but past
relative to the SMA's centroid).

Warmup is `period + signalLength - 2 = 21` for the
default tuning: DPO is valid from i = period - 1 = 19
(the rolling SMA needs `period` bars and the shifted
close at i - 11 is available since i >= 19 > 11), then
the signal SMA needs `signalLength - 1 = 2` more bars.

## Bit-exact anchors

All anchors use close-only input.

- **CONST band** `close = K`: SMA(K, 20) = K. close[i -
  11] = K. dpo = K - K = 0 (exactly the centerline).
  signal = SMA(0, 3) = 0. dpo === signal -> regime
  `bullish` (>=) for every valid bar. 0 crosses.
  Verified across K in {0, 1, 50, 200, 1234}. The CONST
  anchor lands *exactly* on the centerline -- canonical
  detrended behaviour on a flat trend.
- **LINEAR UP** `close = i`: SMA(close, 20) at i =
  (i + (i-1) + ... + (i-19)) / 20 = i - 9.5.
  close[i - 11] = i - 11. dpo = (i - 11) - (i - 9.5) =
  -1.5 (constant). signal = SMA(-1.5, 3) = -1.5. dpo ===
  signal -> regime `bullish` (>=). 0 crosses. The
  constant -1.5 reflects that the close 11 periods ago
  was 1.5 units below the centroid of the current 20-bar
  SMA -- the canonical DPO residual on a linear trend.
- **LINEAR DOWN** `close = -i`: SMA(close, 20) at i =
  -i + 9.5. close[i - 11] = -i + 11. dpo = (-i + 11) -
  (-i + 9.5) = +1.5 (constant). signal = +1.5. dpo ===
  signal -> regime `bullish` (>=) for the same `===`
  reason. 0 crosses.

The non-zero `+/- 1.5` constant on LINEAR input is
canonical -- DPO does not eliminate slope-induced offset
on a perfectly linear trend (it only neutralises the SMA
lag effect on cyclical components). This is the
designed behaviour: DPO is meant to surface short-term
cycles, not zero out long-term trends.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 3-state regime classifier follows the cross-sig family
  convention (bullish on `===`, bearish on `<`).
- The `shift = floor(period/2) + 1` displacement is the
  standard DPO convention. Some libraries (TA-Lib's
  optional implementations, certain pandas-ta variants)
  use `floor(period/2)` without the `+ 1` adjustment.
  This primitive uses the `+ 1` form per the original
  Blau definition.
- DPO is unbounded -- the layout panel auto-ranges but
  clamps to keep the centerline (0) visible regardless
  of data range. Differs from RSI/Williams %R
  (hard-bounded panels).
- DPO does not zero out on linear trends -- it produces
  a constant `+/- (slope * shift_adjusted)` residual.
  This is designed behaviour: DPO is meant to surface
  short-term cycles, not detrend perfectly. The non-zero
  LINEAR anchors illustrate this canonical property.
- Complementary to the sibling SMA / EMA / Bollinger /
  Keltner / Donchian / CCI / RSI / Ichimoku /
  Supertrend / Williams mid-cross-sig primitives -- DPO
  adds a true-detrending-flavoured oscillator alongside
  the trend-following (price-against-MA), bounded-
  momentum (RSI, %R), and mean-reversion (CCI)
  siblings.

## Summary

New `ChartLineZlema` UI primitive at
`web/src/components/ui/chart-line-zlema.tsx` -- a pure-SVG line chart
with a Zero-Lag Exponential Moving Average overlay. Like
`chart-line-hma` it is a **single-panel** chart: the ZLEMA line is
drawn directly over the price line on shared axes.

## Indicator math

John Ehlers and Ric Way's Zero-Lag Exponential Moving Average, with
a period (default 14):

```
lag        = floor((period - 1) / 2)
deLagged[i] = 2 * value[i] - value[i - lag]
ZLEMA      = EMA(deLagged, period)
```

The de-lagged series adds each bar's own momentum over the `lag`
window back into the value, shifting the price forward to anticipate
the trend. `computeLineZlemaEma` then runs a standard EMA over that
de-lagged series -- the EMA's own lag cancels the forward shift, so
the average keeps pace with the price instead of trailing it. On a
linear trend the ZLEMA tracks the price with zero lag.

`computeLineZlemaEma(src, period)` is a null-tolerant, SMA-seeded EMA
(the de-lagged series has a `null` prefix for indices below `lag`).

### Worked fixture

`values = [0, 3, 6, 9, 12, 15, 18]` (a step-3 linear ramp),
`period = 3` (lag 1):

```
deLagged = 2*value[i] - value[i-1] = [null,6,9,12,15,18,21]
zlema    = EMA(deLagged, 3) = [null,null,null,9,12,15,18]
```

The ZLEMA reproduces the value exactly (`9 / 12 / 15 / 18`) -- the
zero-lag property on a linear trend.

## API

- `getLineZlemaFinitePoints(points)` -- drop points with a non-finite
  `x` or `value`.
- `normalizeLineZlemaPeriod(period, fallback)` -- positive integer;
  non-finite / sub-1 falls back, fractional floors.
- `computeLineZlemaEma(src, period)` -- the null-tolerant EMA.
- `computeLineZlema(values, period)` -- returns
  `{ lag, deLagged, zlema }`.
- `runLineZlema(points, options?)` -- sort finite points by `x`,
  compute the ZLEMA, return `{ series, period, lag, deLagged, zlema,
  samples, zlemaFinal, zlemaMin, zlemaMax, aboveCount, belowCount,
  ok }`. Each sample carries an `above` / `below` / `on` position.
  `ok` is false for fewer than two points.
- `computeLineZlemaLayout(options)` -- fit one panel whose y-domain
  spans both the price and the ZLEMA. Returns the panel, ticks, the
  price path, the ZLEMA path, the price dots, and the ZLEMA markers.
- `describeLineZlemaChart(data, options?)` -- ARIA description.

## Component

`ChartLineZlema` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, a single panel with axis
ticks and grid, a config badge (period and derived lag), a
two-series legend (Price / ZLEMA) with toggle, the price line, the
ZLEMA line drawn over it, markers on the ZLEMA line, optional price
dots, a hover/focus tooltip exposing the value, ZLEMA, and position,
and a `motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs chart-line-hma** -- both are zero-lag price overlays, but the
  HMA nests *three* weighted moving averages
  (`2*WMA(half) - WMA(full)` then a sqrt-length WMA). The ZLEMA
  cancels lag differently: it shifts the price forward by its own
  momentum (`2*value - value[lag]`) and then applies *one* EMA. A
  lag-shift plus one EMA, not nested WMAs.
- **vs chart-line-ewma** -- a plain EMA always trails the trend. The
  ZLEMA feeds the EMA a momentum-anticipated series specifically so
  the EMA's lag is cancelled.
- **single-panel** -- like the HMA and unlike the recent two-panel
  indicator primitives, the ZLEMA shares one set of axes with the
  price.

## Vortex+-over-signal pipeline

```
VM+[i]    = |high[i] - low[i-1]|          ; i >= 1
VM-[i]    = |low[i]  - high[i-1]|
TR[i]     = max(high[i] - low[i],
                |high[i] - close[i-1]|,
                |low[i]  - close[i-1]|)
VI+[i]    = sum(VM+[i-period+1..i])       ; i >= period
            / sum(TR[i-period+1..i])
signal[i] = SMA(VI+, signalLength)        ; i >= period + signalLength - 1
regime    = VI+ >= signal -> bullish
            VI+ <  signal -> bearish
            null          -> none
bias      = VI+[i] vs VI+[i-1] -> up / down / flat / none
bullish   : prev VI+ <= prev signal AND cur VI+ > cur signal
bearish   : prev VI+ >= prev signal AND cur VI+ < cur signal
```

Botes / Siepman (2010) VI+ traditionally pairs with VI- (the
dual-line view lives in
`chart-line-vortex-divergence-cross`); this primitive instead
focuses on the VI+ vs its own SMA signal smoothing cadence
for a single-line uptrend confirmation tracker.

Warmup is `period + signalLength - 1 = 16` (VM/TR seed at
`i >= 1`, the rolling sum across `period` bars first fills
at `i = 14`, then the SMA(signalLength=3) adds another 2
bars).

## Bit-exact anchors

All anchors use HLC input.

- **CONST band** `high = K + 1`, `low = K - 1`, `close =
  K`: every bar yields VM+ = 2, VM- = 2, TR = max(2, 1, 1)
  = 2. The rolling sums are 28/28 so VI+ = 1 from `i =
  period`, and signal = 1 from `i = period + signalLength
  - 1`. VI+ === signal -> regime `bullish` (>=) for every
  valid bar. 0 crosses. Verified across K in {0, 1, 50,
  200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close = i`:
  VM+ = |(i+1) - (i-2)| = 3, VM- = |(i-1) - i| = 1, TR =
  max(2, 2, 0) = 2. VI+ = 42/28 = 1.5, signal = 1.5. VI+
  === signal -> regime `bullish`, 0 crosses.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`, `close
  = -i`: VM+ = |(-i+1) - (-(i-1)-1)| = 1, VM- = |(-i-1) -
  (-(i-1)+1)| = 3, TR = max(2, 2, 2) = 2. VI+ = 14/28 =
  0.5, signal = 0.5. VI+ === signal -> regime `bullish`
  (>=), 0 crosses. The classifier settles bullish even on
  falling close because VI+ stabilises above 0 and the
  signal SMA tracks; this primitive is not the dual-line
  view -- it watches VI+ relative to its own smoothing.

The clean integer / half-integer separation under linear
input is a direct consequence of the rolling-sum identity
on linear HLC inputs: VM+ and TR each settle to constants,
so VI+ is a constant from `i = period` and the SMA signal
locks to that same constant from `i = period + signalLength
- 1`.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no chart
  lib dependency.
- Strict-inequality bullish vs `>=` regime classifier means
  the boundary (VI+ === signal) settles into the bullish
  band, matching every other `chart-line-*-cross-sig`
  sibling.
- High < low input rows are filtered at the boundary so the
  TR / VM math never receives a negative range.
- Zero-TR rolling window is short-circuited to null (the
  numerator's `sumTr === 0` guard) so a degenerate window
  cannot produce Infinity.

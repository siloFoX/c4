## Summary

New `ChartLineSineWeighted` UI primitive in
`web/src/components/ui/chart-line-sine-weighted.tsx` -- a pure-SVG
single-panel line chart with a **Sine Weighted Moving Average** (SWMA)
overlay weighting the lookback window by a half sine cycle. The price line
and the SWMA line share one panel.

## The computation

The SWMA weights its lookback window by a half sine cycle: the weight for
window slot k is `sin((k + 1) * pi / (period + 1))`, k = 0..period-1. The
weights climb from a small value at the oldest slot to a peak in the middle
of the window and taper back down at the newest slot. The weighting is
symmetric (`weight[k] = weight[period-1-k]`), so the SWMA emphasizes the
center of the window and de-emphasizes both the oldest and the newest
values. It is a close-only indicator, so `ChartLineSineWeightedPoint`
carries `{x, value}`.

- `computeLineSineWeightedWeights(period)` -- the raw half-sine weights.
- `computeLineSineWeighted(values, period)` -- the SWMA: the weighted
  average of each full window, `sum(weight[k] * value[k]) / sum(weight)`.
  Defined once a full window is available; a window with a non-finite
  value yields null.

Each defined bar is classified by where the SWMA sits relative to the
price: **above**, **below** or **equal**.

## Worked fixture

The sine weights are transcendental, but two cases collapse to exact
arithmetic:

- **Period 1**: the single weight is `sin(pi/2) = 1` exactly, so the SWMA
  is the identity -- `computeLineSineWeighted([5,10,15], 1)` is
  `[5,10,15]`, asserted with `toEqual`.
- **Period 2**: the two weights are `sin(pi/3)` and `sin(2*pi/3)` -- both
  `sqrt(3)/2`, so they are equal and the SWMA collapses to the simple
  two-bar average `(v[i-1] + v[i]) / 2`. The fixture `[10,20,14,30,22,40]`
  gives `[null,15,17,22,26,31]`, checked with `toBeCloseTo`.

The weight shape is checked structurally: every weight is positive, the
set is symmetric about the center, the middle slot carries the largest
weight, and the center slot of an odd window has a unit weight
(`sin(pi/2)`). The classification is a pure function tested directly
(above / below / equal / none).

## API

`data` (a `ChartLineSineWeightedPoint[]`), `period` (default 14, coerced to
an integer >= 1), sizing/format/a11y props, `strokeWidth` / `dotRadius`,
the colour props, the toggle flags (`showAxis` / `showGrid` / `showDots`
default false / `showSwma` / `showMarkers` / `showTooltip` /
`showConfigBadge` / `showLegend` / `animate`), controlled + uncontrolled
`hiddenSeries` (keys `price` / `swma`), `formatValue` / `formatX`,
`onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineSineWeightedFinitePoints`,
`normalizeLineSineWeightedPeriod`, `computeLineSineWeightedWeights`,
`computeLineSineWeighted`, `classifyLineSineWeightedZone`,
`runLineSineWeighted`, `computeLineSineWeightedLayout`,
`describeLineSineWeightedChart`, plus the
`DEFAULT_CHART_LINE_SINE_WEIGHTED_*` constants and the
`ChartLineSineWeightedZone` type. Every exported name carries a
`LineSineWeighted` infix so the `export *` barrel line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen
`data-section="chart-line-sine-weighted-aria-desc"` description span; the
price line, the SWMA line and every dot / marker are
`role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-period`, `data-swma-final`,
`data-above-count`, `data-below-count`, `data-equal-count`,
`data-total-points` and `data-animate`.

# 11.177 -- UI: Breadcrumbs primitive (v1.11.195)

## Summary

Add a `<Breadcrumbs>` UI primitive at `web/src/components/ui/breadcrumbs.tsx`
for semantic breadcrumb navigation. Renders a `nav aria-label="breadcrumb"`
wrapping an `<ol>` of items separated by a customisable token (default
lucide-react `ChevronRight`). The last item, or any item without an
`href`, is rendered as a non-link `<span>` carrying `aria-current="page"`.

When `maxItems` is set and `items.length > maxItems`, the list collapses
to `first + ellipsis + last (maxItems-2)` items so the visible count
matches `maxItems`. The ellipsis itself is a `<button type="button">`
wrapped in the existing `<Tooltip>` primitive; the tooltip body lists
the collapsed labels, and the button's `aria-label` mirrors the same
information for SR users.

## API

```tsx
<Breadcrumbs
  items={[
    { id: 'home', label: 'Home', href: '/' },
    { id: 'workers', label: 'Workers', href: '/workers' },
    { id: 'current', label: 'auto-w1' },
  ]}
  separator="/"            // optional; default <ChevronRight />
  maxItems={4}             // optional; collapses to first + ... + last (maxItems - 2)
  ellipsisLabel="Show collapsed breadcrumbs"
  className="text-xs"
/>
```

`BreadcrumbItem`:

```ts
interface BreadcrumbItem {
  id: string;       // stable React key + collapse identity
  label: ReactNode; // string preferred so the tooltip can list it
  href?: string;    // missing href -> rendered as current page span
}
```

The component is `forwardRef<HTMLElement>` so callers can attach refs to
the underlying `<nav>` (e.g. to programmatically scroll it into view).
Class merging uses the shared `cn()` helper.

## Adoption

`Panel` (`web/src/components/ui/panel.tsx`) previously rendered an
inline breadcrumb `<nav>` inside its header. It now delegates to the
new `<Breadcrumbs>` primitive. The `breadcrumbs` prop on Panel was
widened to `Array<PanelBreadcrumb | BreadcrumbItem> | ReactNode` so:

- Legacy callsites passing `{ label, href }` objects without an `id`
  still compile -- Panel auto-fills `id` via `crumb-<index>`.
- New callsites are encouraged to pass `BreadcrumbItem[]` with stable
  ids.
- Callers that need full control (custom separators, max-items,
  bespoke labels) can pass a pre-rendered `<Breadcrumbs ... />`
  element directly, and Panel will splat it into the header band
  without modification.

The only legacy callsite in the repo (`web/src/pages/Validation.tsx`)
passes `[{ label }]` arrays and continues to work without a code
change at that callsite.

## Tests

`web/src/components/ui/breadcrumbs.test.tsx` covers 11 cases: the nav
aria-label, the ol > li structure, anchors for items with href, last
item rendered as a current-page span, mid-list items without href
also render as current, default ChevronRight separator presence,
custom separator override, `maxItems=3` truncation shape (1 + ... + 2),
ellipsis `aria-label` exposing collapsed labels, className merge, and
ref forwarding to the nav. The existing `panel.test.tsx` breadcrumb
cases were updated for the new lowercase `aria-label="breadcrumb"`
and for the fact that separators are now rendered as `<li
aria-hidden="true">` siblings instead of inline spans.

## Out of scope

- No popover-based "expand collapsed" UI -- the tooltip is read-only;
  a future enhancement could swap it for a `<Popover>` listing the
  collapsed items as links.
- Right-to-left mirroring of the default ChevronRight separator is
  not handled -- callers in RTL contexts should pass an explicit
  `separator` (e.g. `<ChevronLeft />`).
- No automatic structured-data (`schema.org/BreadcrumbList`) markup.

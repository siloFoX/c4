# 11.542 -- web/src/components/ui/chart-line-rangefocus.tsx

Pure-SVG line chart implementing the canonical **focus +
context** pattern (also known as "overview + detail" or
"brush + zoom"). The canvas splits vertically into two
panels:

- **Detail** (top, large) -- renders only the points within
  the selected sub-range, projected to fill the panel.
- **Overview** (bottom, small) -- renders the entire data
  range dimmed, with a draggable **brush** rectangle
  marking the selected sub-range.

The brush has three pointer-driven affordances: drag the body
to move the window, drag either edge handle to resize, or
click anywhere on the overview to recenter the window on the
click point.

## Distinct from related primitives

- `<ChartLineZoom>` (catalog earlier): single-chart viewport
  zoom -- one panel where the visible x-range is mutated.
  Range-focus uses TWO panels with a brush on the overview.
- `<ChartLineResample>` (11.541): changes the underlying
  sample COUNT (LTTB / stride / mean / minmax / upsample).
  Range-focus keeps the full data and zooms an x-window.
- `<ChartLineForecast>` (11.519): semantic split of the time
  axis between historical and forecast. Range-focus is a UI
  affordance for navigation, not a semantic split.
- `<ChartLineFft>` (11.540): split panel pairing time-domain
  + frequency-domain. Range-focus's two panels are both
  time-domain views at different x-resolutions.

The canonical "navigate dense time-series" UI used in Grafana
(time-range bar), D3 brush examples, and any context where
the question is "drill into a sub-window without losing the
big picture".

## Pure helpers

- `getLineRangeFocusDefaultColor(index)` -- 10-color palette
  cycle.
- `getLineRangeFocusFinitePoints(points)` -- finite-only
  filter.
- `normaliseLineRangeFocusOverviewRatio(value)` -- clamps to
  `[0.1, 0.5]`; non-finite -> default 0.28.
- `clampLineRangeFocusRange(range, fullXMin, fullXMax,
  minWidth?)` -- enforces:
  - x0 <= x1 (swaps when reversed)
  - x0 >= fullXMin, x1 <= fullXMax
  - x1 - x0 >= minWidth (expanded around centre; pinned to
    bounds when centre would go past)
  - non-finite bounds -> defaults
- `computeLineRangeFocusLayout({series, range, ...})` --
  splits canvas; returns `{series, detailPanel,
  overviewPanel, brushRect, brushLeftHandle, brushRightHandle,
  detailXTicks, detailYTicks, overviewXTicks, fullXMin,
  fullXMax, detailXMin, detailXMax, fullYMin, fullYMax,
  detailYMin, detailYMax, range, totalPoints, totalInRange,
  visibleSeriesCount, ok}`. Per-series carries `{id, label,
  color, detailPoints, detailPath (in-range only),
  overviewPoints, overviewPath (full), inRangeCount,
  finiteCount, totalCount}`. Each detail point has
  `{index, x, y, px, py, inRange}`.
- `describeLineRangeFocusChart(series, options?)` -- screen-
  reader description with range + in-range count.

## Brush interaction model

The brush is a `<g>` with `role="slider"` and three child
rects:

- `data-section="chart-line-rangefocus-brush-rect"`
  (`data-kind="rect"`): the central body. Pointer-down
  begins a **move** drag.
- `data-section="chart-line-rangefocus-brush-handle"`
  (`data-kind="left"` / `"right"`): the two edge handles.
  Pointer-down begins an **edge resize** drag.
- A transparent overlay over the entire overview captures
  clicks outside the brush and recenters the window on the
  click x.

Drag uses native pointer events with capture; the
`pointermove` / `pointerup` listeners live on `window` so
drags continue when the cursor leaves the SVG. Delta-x in
pixels is converted to delta-x in data using the overview
panel's projection. Move drags pin against the full bounds
when the window would slide off either side. Edge drags
extend or shrink the corresponding end independently. Every
mutation goes through `clampLineRangeFocusRange` so the
final state is always valid.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Detail line path + every detail dot are
  `role="graphics-symbol"` + `tabIndex={0}`. Detail aria-
  label includes the in-range count; dot aria-label includes
  x and y.
- Overview path is `role="graphics-symbol"` (decorative;
  not focusable to keep tab order clean).
- Brush group is `role="slider"` with
  `aria-valuemin`/`aria-valuemax`/`aria-valuenow`/
  `aria-valuetext`.
- Every node carries `data-section="chart-line-rangefocus-..."`
  for test selectors.
- Series groups expose `data-series-id`, `data-series-color`,
  `data-series-in-range-count`, `data-series-finite-count`.
- Detail line has `data-kind="detail"`; overview line
  `data-kind="overview"`. Brush handles have `data-kind`
  (`'left'` | `'right'`).
- Root mirrors `data-series-count`,
  `data-visible-series-count`, `data-total-points`,
  `data-total-in-range`, `data-range-x0`, `data-range-x1`,
  `data-range-ratio`, `data-animate`.
- Badge mirrors the same range + width + total-in-range.

## Tooltip

Hovering a detail dot shows the series label in its colour,
x, and bold y. Tooltips are anchored to the detail panel
projection (not the overview).

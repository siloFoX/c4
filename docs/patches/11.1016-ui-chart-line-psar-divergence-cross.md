# 11.1016 -- UI: chart-line-psar-divergence-cross primitive (v1.11.1034)

## Scope

Add a pure-SVG dual-panel `ChartLinePsarDivergenceCross`
component that renders the close in the top panel and the
Wilder Parabolic SAR (Stop And Reverse) trailing-stop line in
the bottom panel, marking discrete price-versus-PSAR direction
disagreement events as bullish / bearish divergence cross
transitions. First divergence primitive built on a trailing-
stop indicator: PSAR is a directional trailing stop rather
than an oscillator, so its "divergence" reads track regime
shifts in the trend follow vs the price.

## PSAR pipeline (Wilder)

```
Init:   trend = up, SAR[0] = low[0], EP = high[0], AF = step.
Uptrend at i:
  next_sar = SAR + AF * (EP - SAR)
  if high[i] > EP: EP = high[i], AF = min(AF + step, max)
  if low[i]  < next_sar: flip to down,
                          next_sar = old EP, AF = step,
                          EP = low[i]
Downtrend at i:
  next_sar = SAR + AF * (EP - SAR)
  if low[i]  < EP: EP = low[i], AF = min(AF + step, max)
  if high[i] > next_sar: flip to up,
                          next_sar = old EP, AF = step,
                          EP = high[i]
```

Defaults: `step = 0.02`, `maxStep = 0.20` (Wilder's canonical
acceleration factor). The first `warmupLength = 15` bars of
SAR are nulled out so the initial trend-detection flip
transient (which makes SAR briefly non-monotonic until AF
saturates) does not contaminate the divergence regime
classifier.

## Divergence regime model

5-state classifier exactly like the other divergence primitives:

```
priceUp = close[i] > close[i - divergenceWindow]
psarUp  = sar[i]   > sar[i - divergenceWindow]
state   = aligned-bullish    if priceUp && psarUp
        | aligned-bearish    if !priceUp && !psarUp
        | divergent-bullish  if !priceUp && psarUp    (price down, SAR up)
        | divergent-bearish  if priceUp && !psarUp    (price up, SAR down)
        | none               if either input is null
```

Crosses suppress when either `prev` or `cur` is `none`, so
warmup transitions never fire spuriously.

## Bit-exact anchors

| input                                  | psar (post-warmup)  | regime (post i=20) | crosses |
| -------------------------------------- | ------------------- | ------------------ | ------- |
| CONST H=L=C=K                          | K (pinned)          | aligned-bearish    | 0       |
| LINEAR UP (h=i+1, l=i-1, c=i)          | rising              | aligned-bullish    | 0       |
| LINEAR DOWN (h=-i+1, l=-i-1, c=-i)     | falling             | aligned-bearish    | 0       |

CONST H=L=K verified across K = 0..1234 -- there is no
acceleration target (EP = K = SAR), so SAR remains pinned at
K. priceUp = false, psarUp = false -> regime
`aligned-bearish`.

LINEAR UP exercises the steady-state trailing case: AF
saturates at `maxStep = 0.20` after ~10 bars and SAR climbs
at roughly one unit per bar behind the rising EP. priceUp =
true, psarUp = true -> regime `aligned-bullish` -- the
trailing stop trends with the price, no divergence signal
expected.

LINEAR DOWN: the init uptrend triggers a flip on bar 1
(low[1] = -2 falls below next_sar = -0.96), kicking the SAR
into downtrend with SAR=1. From there the AF re-accelerates
and SAR descends monotonically to track above the falling EP.
priceUp = false, psarUp = false -> regime `aligned-bearish`.

The PSAR monotonicity is exercised in dedicated unit tests
for `applyLinePsarDivergenceCrossPsar` (CONST K, LINEAR UP
monotone-rising, LINEAR DOWN monotone-falling after warmup).

Note: the **aligned** anchor sense for PSAR is the inverse of
the **divergent** sense in oscillator primitives. The PSAR
follows price; aligned regimes are the natural state of a
healthy trend. Divergent regimes only emerge near a
PSAR flip (the indicator's actual reversal signal), and the
0-cross anchor count reflects that no flip occurs in pure
linear data after warmup.

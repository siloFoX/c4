## Summary

Adds the `ChartLineCmoOversoldCross` primitive
(`web/src/components/ui/chart-line-cmo-oversold-cross.tsx`): a pure
SVG dual-panel chart that visualizes Chande Momentum Oscillator
(CMO) crossings through the canonical oversold line -50 for
momentum oversold trigger entry / exit events. Mirror of
`chart-line-cmo-overbought-cross` (TODO 11.976): identical CMO
pipeline, threshold flipped from +50 to -50. Joins
`chart-line-cmo-zero-cross` (11.946) and
`chart-line-cmo-mid-cross` (11.966) to round out the canonical CMO
threshold layer (overbought + oversold + zero / midline).

## Pipeline

Same close-only CMO pipeline shared with the zero / mid /
overbought siblings:

1. `delta[i] = close[i] - close[i-1]`
2. `gain[i] = max(delta, 0)`, `loss[i] = max(-delta, 0)`
3. `sumGain = sum(gain, length)`, `sumLoss = sum(loss, length)`
4. `cmo[i] = (sumGain - sumLoss) / (sumGain + sumLoss) * 100`
   - When `sumGain + sumLoss == 0` (no movement in the window),
     short circuit to `cmo[i] = 0` (bit-exact +0, NOT -0;
     verified via `Object.is`).
5. Regime classifier (strict `>=`):
   - `cmo >= -50` -> bullish (above oversold zone)
   - `cmo <  -50` -> bearish (in oversold zone)
6. Cross detector emits a `bullish` cross when
   `cmo[i-1] <= -50 && cmo[i] > -50` (exit from oversold) and a
   `bearish` cross when `cmo[i-1] >= -50 && cmo[i] < -50` (entry
   into oversold). Strict inequality on the current side ensures
   the boundary value -50 does not double-fire.

## Defaults

- `length = 14` (canonical CMO window)
- `threshold = -50` (canonical oversold line)
- Warmup boundary: bar `length = 14`, so the first 14 samples
  (30-bar run) are null and the remaining 16 are populated.

## Bit-exact anchors

- CONST `close = K` for all `K in {0, 1, 42, 100, 1234}` ->
  `cmo[i] === 0` literal (NOT `-0`) for every `i >= 14`. Regime
  classifies as bullish because 0 >= -50 -- mirror of the
  overbought variant (where 0 < 50 gave bearish).
- LINEAR UP `close = i` -> `cmo[i] === 100` constant. All
  bullish (100 >= -50), 0 crosses.
- LINEAR DOWN `close = -i` -> `cmo[i] === -100` constant. All
  bearish (-100 < -50), 0 crosses (cmo jumps directly from null
  to -100, no `prev >= T && cur < T` transient).
- Balanced alternation `close = i % 2 ? 11 : 10` -> `cmo[i] === 0`
  -> bullish (above oversold).
- CONST K full-run counters under defaults, 30 bars:
  `noneCount = 14`, `bullishCount = 16`, `bearishCount = 0`,
  `crosses.length = 0`.
- `pricePath` and `cmoPath` are byte-identical across repeated
  layout calls, with `.toFixed(2)` precision and `M ... L ...`
  command shapes. The cmo path uses a single `M` for post-warmup
  data.
- `thresholdY` at -50 sits BELOW the panel midpoint of `[-100,
  100]` (larger y in SVG space) -- exact mirror of the
  overbought variant where +50 sat above.
- Cross detector boundary: strict inequality (`> T`, `< T`).
- Rise-then-decline transient yields a confirmed bearish
  oversold-entry cross (the canonical use case).

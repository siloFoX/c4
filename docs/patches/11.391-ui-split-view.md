## Goal

Dispatch:

- Build `web/src/components/ui/split-view.tsx`
  -- horizontal/vertical resizable panels
  with drag handle, collapse-on-double-click,
  percentage or pixel-based sizing.

`<SplitPane>` (11.274 / v1.11.292) already
shipped most of the contract:

| dispatch item | shipped previously |
| --- | --- |
| horizontal/vertical | `orientation: 'horizontal' \| 'vertical'` |
| drag handle | pointerdown/move/up + setPointerCapture |
| ARIA `role="separator"` | aria-orientation + aria-valuemin/now/max |
| keyboard nav | Arrow keys + Home / End + Enter |
| persistence | `storageKey` (localStorage) |
| cross-tab sync | `storage` event listener |
| snap zones | `collapseThreshold` / `expandThreshold` |

The remaining gaps were
**collapse-on-double-click** and
**pixel-based sizing**. This patch adds
both to `<SplitPane>` and ships a thin
`<SplitView>` alias at the dispatch-style
import path.

## What changed

### `collapseOnDoubleClick`

```ts
collapseOnDoubleClick?: boolean;     // default false
```

When true, double-clicking the divider:

- If the start pane is currently open
  (`ratio > 0`): collapses to ratio=0
  (skipSnap so the clamp does not stop
  the collapse from reaching 0).
- If the start pane is currently
  collapsed: restores the last open
  ratio. A `lastOpenRatioRef` tracks
  the most recent non-zero ratio.
  Falls back to `defaultRatio` when no
  prior open size exists.

Persistence: when `storageKey` is set,
the collapse + restore values write
back to localStorage just like keyboard
+ pointer commits.

### `defaultSizePx`

```ts
defaultSizePx?: number;
```

Alternative to `defaultRatio`. On mount,
the component:

1. Reads the container's
   `getBoundingClientRect()`.
2. Computes `ratio = defaultSizePx /
   container.<width or height>` (depends
   on `orientation`).
3. Clamps to `[0, 1]`.
4. Seeds the internal ratio with the
   computed value.

`storageKey` still wins -- when a valid
stored ratio is present, the px seed is
skipped. This keeps cross-mode adopters
compatible (callers who flip from
ratio to px mode see their stored
preference honored).

### `onSizeChange(pixels)`

```ts
onSizeChange?: (pixels: number) => void;
```

Fires alongside `onRatioChange(ratio)` on
every commit. Computes the px size from
the current container dimensions; skipped
on the initial mount race when the
container has not measured yet.

### `<SplitView>` alias

```tsx
import { SplitView } from './components/ui/split-view';
```

Re-exports `<SplitPane>` under the
dispatch-style name. Both names resolve
to the same component. Types
re-exported: `SplitViewProps`,
`SplitViewOrientation`.

### Data attributes

- `data-collapse-on-double-click="true|false"`
  on the divider (new).
- Legacy attrs unchanged.

## Tests

`split-pane.test.tsx` -- **27 legacy
cases unchanged**.

`split-view.test.tsx` -- **16 new cases**:

Alias (2):
- `SplitView === SplitPane` identity.
- Render-through.

collapseOnDoubleClick (6):
- Default false: dblclick no-op + attr
  mirrors.
- First dblclick collapses to ratio=0.
- Second dblclick restores last open
  ratio.
- Restores to `defaultRatio` when no
  prior open size (stored value=0).
- Collapse via dblclick persists to
  localStorage.
- ArrowRight after collapse clamps to
  `minRatio`.

defaultSizePx (5):
- Horizontal: px / container.width
  -> ratio.
- Vertical: px / container.height
  -> ratio.
- Stored ratio wins over `defaultSizePx`.
- Oversized px clamps to 100%.
- Omitted px falls back to
  `defaultRatio` (legacy byte-identical).

onSizeChange (3):
- Fires alongside `onRatioChange` with
  px size.
- Vertical uses container height.
- Omitted does not throw.

## Pairs with existing primitives

- `<DrawerResize>` (11.390) -- single
  fixed-sided resizable panel. Use for
  layout sidebars where one side is
  the "main" content and the other is
  the sidebar; SplitView for symmetric
  two-pane content.
- `<ScrollArea>` (11.382) -- composable
  scroll surface; pair inside either
  start/end pane when long content
  needs custom scroll chrome.

## Out of scope

- Per-page adoption swap (`SplitPane`
  -> `SplitView`). Alias re-exports the
  same component; no migration needed.
- Three-or-more pane composition. Two-
  pane only; multi-pane layouts belong
  in `<Resizable>` (TODO 11.392).
- Touch-gesture collapse (swipe to
  collapse). The divider responds to
  pointer events uniformly; per-device
  gesture tuning is a follow-on.
- Animated collapse transition. Ratio
  jumps instantly on dblclick; CSS
  transitions would need a separate
  animation lifecycle.
- Per-pane min size in px (vs ratio).
  Current min/max stay ratio-based; a
  pixel-mode min/max story is a
  follow-on with its own ResizeObserver
  considerations.

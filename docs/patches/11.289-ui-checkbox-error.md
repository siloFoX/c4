# 11.289 -- Checkbox size + error state + data selectors (v1.11.307)

Component-scope-only enhancement of the existing `Checkbox`
primitive. No daemon-side change and no `c4` CLI surface change.
The default surface (md size, no error) is preserved byte-for-byte;
new affordances opt in via the new `size` and `error` props.

## Why

Two pain points:

1. Templates filter sidebar renders a dense column of filter
   chip rows -- each row pairs a Checkbox with a chip label.
   At the existing 16-px md height the checkbox dominated the
   row visually; a 14-px sm size reads as an inline affordance
   rather than a primary control.
2. Form-style checkboxes (e.g., a "I agree" Profiles editor
   row, a Templates "save as default" toggle) wanted an
   `error` state when the operator submitted without a
   required tick. Callers were wrapping the checkbox in a
   destructive-tinted div, fragmenting the validation
   chrome across pages.

Patch 11.289 puts both affordances inside the primitive so
every consumer picks the same surface vocabulary.

## What changed

### `web/src/components/ui/checkbox.tsx`

- New exported type `CheckboxSize = 'sm' | 'md'`.
- New `size?: CheckboxSize` prop (default `'md'`). `md` keeps
  the existing h-4 w-4 footprint byte-for-byte. `sm` lands at
  h-3.5 w-3.5.
- New `error?: boolean` prop (default false). When true,
  flips `aria-invalid="true"` on the input AND swaps the
  border palette from `border-input` to `border-destructive`.
  When false, the attribute is absent (rather than `"false"`)
  so the accessibility tree stays clean.
- Data-attribute selectors:
  - `data-section="checkbox"` on the input.
  - `data-section="checkbox-row"` on the label-mode wrapper.
  - `data-section="checkbox-label"` on the label text span.
  - `data-size="sm|md"` on the input.
  - `data-error="true|false"` on the input (always emitted
    so CSS attribute-selectors can drive theming without a
    null-check).
  - `data-indeterminate="true|false"` on the input so e2e
    can target the mixed state without inspecting the
    DOM-only `indeterminate` property.

### Adopted

- Primitive enhancement only this commit. The existing
  Checkbox call sites (`ColumnPicker` + every page that
  mounts a bulk-select header) inherit the new defaults +
  data attributes without any caller change.

## Deferred (dispatch follow-ups)

## Notes

- `aria-invalid` is set conditionally via `error || undefined`
  so the attribute is absent (not `"false"`) when there is
  no error -- avoids cluttering the accessibility tree.
- `data-error` is always emitted as `"true"` or `"false"`
  so a CSS attribute-selector can drive theming without the
  null-check the `aria-invalid` attribute would require.

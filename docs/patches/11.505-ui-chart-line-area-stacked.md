# TODO 11.505 - chart-line-area-stacked primitive

Status: SHIPPED v1.11.523 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartLineAreaStacked>`, a pure-SVG stacked line + area chart
with two stacking modes:

- **`'absolute'`** (default): cumulative sum across layers; the y axis
  shows raw totals.
- **`'percent'`**: per-x normalisation; each layer is `raw / total[x]`
  so every column sums to 1.0; the y axis shows percentages.

Distinct from the v1 `<ChartStackedArea>` which paints only the
absolute mode and lacks the per-x normalisation that `'percent'`
provides. This primitive also strokes the top edge of each layer
(line on top of fill, like `<ChartLineAreaGradient>` but stacked) so
the layer boundaries stay visually crisp.

## API

```tsx
<ChartLineAreaStacked
  series={[
    { id: 'free', label: 'Free', data: [{x:0,y:5},{x:1,y:6},{x:2,y:8}] },
    { id: 'pro',  label: 'Pro',  data: [{x:0,y:2},{x:1,y:4},{x:2,y:7}] },
    { id: 'ent',  label: 'Ent',  data: [{x:0,y:1},{x:1,y:1},{x:2,y:2}] },
  ]}
  stackMode="percent"
/>
```

## Distinct from prior chart-* primitives

- `<ChartStackedArea>` - absolute stacking only; no percent mode; no
  per-layer top-line stroke.
- `<ChartLineAreaGradient>` (11.504) - single-series-per-instance
  area below the line with a vertical gradient; not stacked.
- `<ChartLineThreshold>` (11.503) - reference lines + zone shading;
  no stacking.
- `<ChartLineConfidence>` (11.502) - polygon between two curves per
  series; not cross-series stacking.
- `<ChartLineMulti>` (11.500) - independent layered lines (not
  stacked).

## Stacking contract

### x union

`collectLineAreaStackedXValues(series, hidden?)` returns the sorted
unique union of x values across every visible series. Each layer is
evaluated at every x in this union. Series that lack a sample at
some x contribute 0 at that x (zero-fill). Negative contributions
also clamp to 0.

### Mode resolution

For each `x` in the union:

1. `total[x] = sum(clamp(raw_i, 0))` across visible layers.
2. In **absolute** mode each layer's normalised value is `raw`
   (clamped to non-negative).
3. In **percent** mode each layer's normalised value is `raw /
   total[x]` (or 0 if `total[x] = 0`).
4. `yBottom = cumulative sum of layers below`; `yTop = yBottom +
   normalised`. Layers are stacked in the visible-series order
   (the layer at series index 0 sits at the bottom).

### y range

- Absolute: `yMin = 0`, `yMax = max(total[x])` so the entire stack
  fits.
- Percent: `yMin = 0`, `yMax = 1` so the chart fills the full
  vertical extent.
- Explicit `yMin` / `yMax` props override either mode.

### Paths

For each layer:

- `linePath` = `M p[0] L p[1] ... L p[n-1]` along the top edge.
- `areaPath` = top edge forward + bottom edge reversed + `Z`. This
  is the closed polygon between adjacent layer tops; rendered with
  `fill={series.color}` at `areaOpacity`.

## Interactivity

- Hover or focus on a dot dims every other series stroke + area to
  ~30%; the focused series stays at full opacity.
- Tooltip shows series label, formatted x, formatted y (raw), and a
  `share:` row with formatted percent + total at that x. In percent
  mode the share is the canonical chart value; in absolute mode it
  is a useful secondary read.
- Per-dot `onPointClick({series, point})` callback.
- Legend buttons toggle series visibility (controlled +
  uncontrolled).
- Mount fade via `motion-safe:animate-fade-in` (suppress via
  `animate=false`).

## Exported helpers

- `DEFAULT_CHART_LINE_AREA_STACKED_*` numeric / string defaults.
- `DEFAULT_CHART_LINE_AREA_STACKED_PALETTE` (10-colour palette).
- `getLineAreaStackedDefaultColor(index)` - palette wrap.
- `getLineAreaStackedFinitePoints(points)` - drops non-finite.
- `collectLineAreaStackedXValues(series, hidden?)` - sorted unique
  union of x values across visible series.
- `buildLineAreaStackedYLookup(series)` - `Map<x, y>` of finite
  samples for fast x->y resolution.
- `computeLineAreaStackedLayout(input)` - per-series stacked layout
  + x union + per-x totals + stackMode + tick positions.
- `describeLineAreaStackedChart(series, hidden?, stackMode?,
  formatValue?)` - ARIA summary mentioning stack mode + sample
  counts.

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only
  `data-section="chart-line-area-stacked-aria-desc"` span.
- SVG `role="img"` with the same `aria-label`.
- Each line path is `role="graphics-symbol"` with `tabIndex={0}`
  and `aria-label="<series>: stacked layer top with N samples"`.
- Each dot is `role="graphics-symbol"` with `tabIndex={0}` and
  `aria-label="<series>: x=<x>, y=<y>"` (with appended share in
  percent mode).

## Data attributes

- Root: `data-section="chart-line-area-stacked"`, `data-stack-mode`,
  `data-series-count`, `data-visible-series-count`,
  `data-total-points`, `data-x-union-count`, `data-animate`.
- Series group: `data-section="chart-line-area-stacked-series-group"`,
  `data-series-id`, `data-series-index`, `data-series-color`,
  `data-series-point-count`, `data-series-finite-count`,
  `data-hovered`.
- Area / line / dot: standard pattern with `data-series-id`.
- Dot: extra `data-y-raw`, `data-y-bottom`, `data-y-top`,
  `data-share` exposing the full stacked coordinates for tests.
- Tooltip, legend, axes, grid: standard pattern.

## Out of scope

- Curved layer tops (Bezier / monotone) - linear segments only.
- Wiggle / silhouette baselines (the streamgraph variant lives in
  `<ChartStreamgraph>`).
- Negative values stacked below baseline - v1 clamps negatives to
  0; host normalises if needed.
- Animated stack transitions on data / stackMode change - snap;
  mount fade only.
- Log-scale axes - linear only.

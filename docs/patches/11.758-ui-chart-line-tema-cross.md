# Patch 11.758 -- UI: chart-line-tema-cross primitive

Released as `v1.11.776` on 2026-05-26.

## Scope

Add `<ChartLineTemaCross>` -- a pure-SVG single-panel chart with a
Triple-EMA fast-over-slow **crossover** overlay marking up-cross
and down-cross events.

## Definition

Inputs:

- `data: ChartLineTemaCrossPoint[]` where
  `ChartLineTemaCrossPoint = { x; close }`. Bars with non-finite
  fields are dropped.
- `fastLength: number` -- the fast TEMA length. Default `8`.
  Normalized to an integer `>= 2`.
- `slowLength: number` -- the slow TEMA length. Default `21`.
  Normalized to an integer `>= 2`.

Algorithm (for each lookback `L`):

```
ema1 = EMA(close, L)
ema2 = EMA(ema1, L)
ema3 = EMA(ema2, L)
TEMA = 3 * ema1 - 3 * ema2 + ema3
```

Crosses (per bar `i >= 1`):

```
upCross   at i: fast[i-1] <= slow[i-1] && fast[i] > slow[i]
downCross at i: fast[i-1] >= slow[i-1] && fast[i] < slow[i]
```

## Bit-exact algebraic anchor

**CONST_FLAT** (`close == K`):

- `ema1[i] = K` (EMA-of-constant lemma).
- `ema2[i] = EMA(K, _) = K`.
- `ema3[i] = K`.
- `TEMA = 3 * K - 3 * K + K = K` -- the integer coefficients
  `(3, -3, 1)` cancel exactly in floating point (no rounding
  loss for integer arithmetic on a value times integer
  coefficients with subtraction).

So `TEMA = K` bit-exact at every bar, at any length. With `fast
== slow == K` the chart records zero crosses and tags every bar
`at`.

**Translation invariance**: shifting every close by `+k` shifts
every EMA by `+k` (EMAs are affine). `TEMA(close + k) = 3*(EMA1
+ k) - 3*(EMA2 + k) + (EMA3 + k) = TEMA(close) + (3 - 3 + 1) *
k = TEMA(close) + k`. Floating-point rounding -> tests use
`toBeCloseTo(_, 10)` for non-trivial closes.

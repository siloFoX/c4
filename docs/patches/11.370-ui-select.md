## API

### `<Select>` (single, additive)

```tsx
import { Select } from './components/ui/select';

<Select
  options={FRUITS}
  value={v}
  onChange={setV}
  ariaLabel="Fruit"
  // New (default false)
  clearable
  searchable
  searchPlaceholder="Type to filter..."
/>
```

New props (default values keep legacy
byte-identical behaviour):

| prop | default | effect |
| --- | --- | --- |
| `clearable` | `false` | when `value !== ''`, renders an inline X button on the trigger that calls `onChange('')` |
| `searchable` | `false` | popup gets a search input above the listbox that filters by case-insensitive label substring |
| `searchPlaceholder` | `'Search...'` | search input placeholder text |

`filterSelectOptions(options, query)` is
exported as a pure helper for callers that
need their own client-side filter (and
for tests).

### `<MultiSelect>` (new)

```tsx
import { MultiSelect } from './components/ui/select';

<MultiSelect
  options={FRUITS}
  values={selected}
  onChange={setSelected}
  ariaLabel="Fruits"
  searchable
  clearable
  maxLabelChips={3}
/>
```

- `values: string[]` + `onChange: (next:
  string[]) => void` -- array-based value
  shape.
- `Space` / `Enter` / click on an option
  toggles it in/out of the value set
  WITHOUT closing the popup.
- Listbox carries
  `aria-multiselectable="true"`; each option
  exposes `aria-selected` per membership.
- Each option renders a checkbox glyph
  before its label so the selection state is
  visible without parsing aria.
- Trigger label renders comma-joined labels
  up to `maxLabelChips` (default 3); above
  that it collapses to "<n> selected".
- `data-selected-count` mirrors
  `values.length` on the trigger for tests
  and storyshots.
- Same `clearable` + `searchable` +
  `searchPlaceholder` props as Select.

## Search filter algorithm

```ts
function filterSelectOptions(
  options: SelectOption[],
  query: string,
): SelectOption[] {
  const q = query.trim().toLowerCase();
  if (!q) return options;
  return options.filter((opt) =>
    opt.label.toLowerCase().includes(q),
  );
}
```

Case-insensitive substring match. Empty /
whitespace-only query passes the input
through unchanged.

## Keyboard contract

| key | Single `<Select>` | MultiSelect |
| --- | --- | --- |
| ArrowDown / ArrowUp | move highlight (wraps) | same |
| Home / End | jump to first/last enabled visible | same |
| Enter / Space | commit highlight + close | toggle highlight (no close) |
| Escape | close + restore focus | same |
| Tab | close (focus moves naturally) | same |
| printable | searchable: types into search; else: type-ahead | searchable: types into search; else: ignored |

When `searchable=true` the type-ahead matcher
is suppressed because the search input owns
printable keystrokes. Closing the menu resets
the query.

## Pairs with existing primitives

- `<NativeSelect>` -- platform `<select>`
  wrapper. Unchanged.
- `<Combobox>` (11.371 follow-on) -- can reuse
  `filterSelectOptions` for its filter
  story.
- `<VirtualizedList>` -- callers that need
  windowed listbox rendering compose
  manually for now.

## Out of scope

- **Inline virtualization** of the listbox.
  `<VirtualizedList>` covers windowed render
  and can be dropped into a custom popup.
  Inlining inside `<Select>` adds a
  scrollHeight measurement + ResizeObserver
  story -- deferred to its own patch with a
  perf budget.
- **Async `loadOptions(query)`**. Needs
  debounce + request cancellation + loading
  state. Deferred.
- **Tag chips on the MultiSelect trigger.**
  Comma-join + count fallback covers the
  baseline; per-chip removal belongs in a
  TagInput-like primitive.
- **Custom option renderer.** All options
  render as plain text labels. A renderer
  callback is straightforward to add but
  was not in scope.

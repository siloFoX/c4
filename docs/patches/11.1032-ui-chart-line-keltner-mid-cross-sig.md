# 11.1032 -- UI: chart-line-keltner-mid-cross-sig primitive (v1.11.1050)

## Scope

Add a pure-SVG dual-panel `ChartLineKeltnerMidCrossSig`
component that renders the close in the top panel and the
Keltner Channel middle line (`EMA(close, period)`) plus its
smoothed SMA signal line in the bottom panel, marking
bullish (middle crosses up through signal -- centerline
volatility trend trigger up) / bearish (middle crosses down
through signal -- centerline volatility trend trigger down)
middle-over-signal crossover trigger events with bias
coloring derived from the middle slope at the trigger bar.

This is the EMA sibling of
`chart-line-bollinger-mid-cross-sig` (v1.11.1049): both use
`period = 20`, but Keltner's middle is an EMA (vs
Bollinger's SMA), giving the centerline a slightly faster
response to recent bars while keeping the same steady-state
lag on linear input thanks to SMA-seed EMA initialisation.

## Keltner middle-over-signal pipeline

```
middle[i] = EMA(close, period)        ; i >= period - 1
            (SMA-seeded: middle[period-1] = SMA(close,
             period), then ema[i] = prev*(1-alpha) +
             close*alpha with alpha = 2/(period+1))
signal[i] = SMA(middle, signalLength) ; i >= period +
                                        signalLength - 2
regime    = middle >= signal -> bullish
            middle <  signal -> bearish
            null            -> none
bias      = middle[i] vs middle[i-1] -> up / down / flat / none
bullish   : prev middle <= prev signal AND
            cur middle > cur signal
            (centerline volatility trend trigger up)
bearish   : prev middle >= prev signal AND
            cur middle < cur signal
            (centerline volatility trend trigger down)
```

Default tuning `period = 20`, `signalLength = 3` matches the
canonical Keltner Channel middle line lookback. The EMA's
steady-state lag on linear input solves to `i - 1/alpha + 1
= i - (period - 1) / 2`, matching the SMA centroid lag. SMA
seeding (initialise EMA at index `period - 1` with `SMA(
close, period)`) makes that equality bit-exact from the
seed bar onwards -- no asymptotic convergence transient.

Warmup is `period + signalLength - 2 = 21` for the default
tuning, identical to the Bollinger sibling.

## Bit-exact anchors

- **CONST close = K**: SMA seed = K -> EMA stays at K
  (`K*(1-alpha) + K*alpha = K`). signal = K. middle ===
  signal -> regime `bullish` (>=) for every valid bar. 0
  crosses. Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP close = i**: SMA seed at i = period - 1 =
  `(0+1+...+19)/20 = 9.5 = 19 - 9.5`. The EMA recurrence
  on linear input from that seed gives EMA[20] = `9.5 *
  (19/21) + 20 * (2/21) = (9.5*19 + 40)/21 = 220.5/21 =
  10.5 = 20 - 9.5`. By induction, middle = `i - 9.5` for
  every i >= 19. signal = SMA(middle, 3) adds `(signalLength
  - 1) / 2 = 1` bar of lag, giving signal = `i - 10.5`.
  middle - signal = `+1` -> regime `bullish`. 0 crosses.
- **LINEAR DOWN close = -i**: mirror -> middle = `-i +
  9.5`, signal = `-i + 10.5`. middle - signal = `-1` ->
  regime `bearish`. 0 crosses.

The clean `+1 / -1` separation under linear input is
identical to the Bollinger sibling -- both have a `(period
- 1)/2 = 9.5` lag on linear input. The difference between
the two primitives shows up on volatile / non-linear
inputs, where EMA gives the trigger more weight on recent
bars.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- Strict-inequality bullish vs `>=` regime classifier
  means the bullish boundary (middle === signal) settles
  into the bullish band, matching every other
  chart-line-*-cross-sig sibling.
- EMA helper uses SMA seeding (not first-value seeding)
  so the steady-state lag on linear input is exactly
  `(period - 1) / 2` from the seed bar onwards. This
  matches the original Wilder / canonical Keltner
  convention.
- Keltner middle = EMA(close, 20) is the canonical
  centerline for the upper / lower ATR-band derivation;
  the cross-sig primitive surfaces just the centerline
  trigger pattern. The full upper / lower band overlay is
  out of scope for this primitive.
- Uses a distinct middle ink colour (`#0d9488`, teal) so
  it does not collide with Bollinger's purple (`#7c3aed`)
  when both primitives ship in the same dashboard.

## Summary

New `ChartLineGeometricMa` UI primitive in
`web/src/components/ui/chart-line-geometric-ma.tsx` -- a pure-SVG
single-panel line chart with a **Geometric Moving Average** (GMA) overlay
from the rolling geometric mean of prices. The price line and the GMA line
share one panel.

## The computation

The Geometric Moving Average is the rolling geometric mean: for each window
of `period` values it is the period-th root of their product. It is a
close-only indicator, so `ChartLineGeometricMaPoint` carries `{x, value}`.

- `computeLineGeometricMa(values, period)` -- the GMA, computed in log
  space as `exp(mean(ln(window)))` to avoid the overflow a direct product
  would risk on long windows. The geometric mean needs strictly positive
  inputs, so a window with a non-finite or non-positive value yields null.
  A period of 1 is the identity (the 1st root of a value is the value).

Each defined bar is classified by where the GMA sits relative to the
price: **above**, **below** or **equal**.

The geometric mean has a defining property: by the AM-GM inequality it
always sits **at or below** the arithmetic mean of the same window, with
equality only when every value is equal. It is pulled down by low values,
so it dampens upside spikes more than a simple moving average.

## Worked fixture

The geometric mean involves roots and is transcendental, but two cases
collapse to clean arithmetic:

- **Period 1**: the identity -- `computeLineGeometricMa([5,10,15], 1)` is
  `[5,10,15]`, asserted with `toEqual`.
- **Period 2**: the GMA is `sqrt(v[i-1] * v[i])`. The fixture
  `[4,9,4,25,4,49]` is built so every consecutive product is a perfect
  square, so the GMA lands on an exact integer -- `[null,6,6,10,10,14]`,
  checked with `toBeCloseTo`. The values never have an equal consecutive
  pair, so every defined bar is unambiguously above or below: 2 above, 3
  below.

Structural checks: the geometric mean of a geometric progression is its
middle term (`[1,2,4]` -> cube root of 8 = 2); a constant series stays at
its constant level; the GMA never exceeds the arithmetic mean of its
window (AM-GM); a window with a negative or zero value yields null. The
classification is a pure function tested directly.

## API

`data` (a `ChartLineGeometricMaPoint[]`), `period` (default 14, coerced to
an integer >= 1), sizing/format/a11y props, `strokeWidth` / `dotRadius`,
the colour props, the toggle flags (`showAxis` / `showGrid` / `showDots`
default false / `showGma` / `showMarkers` / `showTooltip` /
`showConfigBadge` / `showLegend` / `animate`), controlled + uncontrolled
`hiddenSeries` (keys `price` / `gma`), `formatValue` / `formatX`,
`onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineGeometricMaFinitePoints`,
`normalizeLineGeometricMaPeriod`, `computeLineGeometricMa`,
`classifyLineGeometricMaZone`, `runLineGeometricMa`,
`computeLineGeometricMaLayout`, `describeLineGeometricMaChart`, plus the
`DEFAULT_CHART_LINE_GEOMETRIC_MA_*` constants and the
`ChartLineGeometricMaZone` type. Every exported name carries a
`LineGeometricMa` infix so the `export *` barrel line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen `data-section="chart-line-geometric-ma-aria-desc"`
description span; the price line, the GMA line and every dot / marker are
`role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-period`, `data-gma-final`,
`data-above-count`, `data-below-count`, `data-equal-count`,
`data-total-points` and `data-animate`.

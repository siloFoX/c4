## Summary

Add a new `<ChartStackedBar>` UI primitive in
`web/src/components/ui/chart-stacked-bar.tsx` -- pure-SVG
stacked bar chart. Each category has one bar divided into
colored segments stacked per series. The chart orientation
toggles between vertical (stack grows upward from the
baseline) and horizontal (stack grows rightward from the
left axis); both controlled and uncontrolled. Optional
value labels paint the formatted value inside each segment
that is large enough to hold readable text. Distinct from
`<ChartGroupedBar>` (11.467) in that bars are stacked
within a category rather than placed side-by-side, and
distinct from `<ChartStackedArea>` (11.466) in that this
primitive uses discrete bars per category rather than
continuous areas across an x axis. Legend interaction:
every entry is a focusable `<button aria-pressed>` that
toggles its series's visibility.

## API

```ts
type ChartStackedBarOrientation = 'vertical' | 'horizontal';

interface ChartStackedBarSeries {
  id: string;
  label: string;
  data: readonly number[];
  color?: string;
}

interface ChartStackedBarProps {
  categories: readonly string[];
  series: readonly ChartStackedBarSeries[];
  width?: number;                      // default 560
  height?: number;                     // default 320
  padding?: number;                    // default 36
  barGap?: number;                     // default 8
  orientation?: ChartStackedBarOrientation;
  defaultOrientation?: ChartStackedBarOrientation;
  onOrientationChange?: (o: ChartStackedBarOrientation) => void;
  hiddenSeries?: readonly string[];
  defaultHiddenSeries?: readonly string[];
  onHiddenSeriesChange?: (hidden: string[]) => void;
  showOrientationToggle?: boolean;     // default true
  showLegend?: boolean;                // default true
  showValueLabels?: boolean;           // default false (in-segment)
  showAxisTicks?: boolean;             // default true
  showGrid?: boolean;                  // default true
  showLabels?: boolean;                // default true (category labels)
  showTooltip?: boolean;               // default true
  animate?: boolean;                   // default true
  className?: string;
  ariaLabel?: string;                  // default 'Stacked bar chart'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  tickCount?: number;                  // default 5
  legendPlacement?: 'right' | 'bottom'; // default 'bottom'
  onSegmentClick?: (args: {
    series: ChartStackedBarSeries;
    seriesIndex: number;
    category: string;
    categoryIndex: number;
    value: number;
  }) => void;
  onSeriesClick?: (args: {
    series: ChartStackedBarSeries;
    index: number;
    hidden: boolean;
  }) => void;
}
```

## Behaviour

- Renders a `<div role="region">` wrapping `<svg
  role="img">` carrying the same `aria-label`.
- Orientation: controlled via `orientation` +
  `onOrientationChange`, or uncontrolled via
  `defaultOrientation` (default `'vertical'`). A
  small button in the top-right corner toggles
  between the two; suppressed via
  `showOrientationToggle=false`.
- Layout: `computeStackedBarLayout` walks every
  category. In `vertical` mode each category gets a
  vertical bar that stacks series 0 at the bottom
  and later series upward; in `horizontal` mode the
  bar grows rightward with series 0 on the left.
  Zero-value segments are dropped from the layout
  to keep the SVG compact.
- Y / value scale: `getStackedBarMaxTotal` returns
  the largest category total across the visible
  series; falls back to 1 for empty / all-zero /
  all-non-finite data.
- Hidden series: controlled via `hiddenSeries` +
  `onHiddenSeriesChange`, or uncontrolled via
  `defaultHiddenSeries`. Every legend entry is a
  focusable `<button aria-pressed>`; clicking
  toggles the series's hidden state and fires
  `onSeriesClick` with the next state.
- Tooltip: hover opens a tooltip with the category,
  series label, formatted value, and the category
  total of the visible series.
- Value labels: when `showValueLabels=true`, a
  formatted-value caption renders inside each
  segment whose smaller dimension exceeds 18 px so
  cramped segments stay readable.
- Mount animation:
  `motion-safe:animate-fade-in` per segment.
  `animate=false` removes the class.
- Default ARIA description: `"Stacked bar chart
  (<orientation>) with N categories, M visible
  series. <label> sum <value>, ..."` -- empty
  input returns `"No data"`; `ariaDescription`
  overrides.

## Pure helpers (exported)

```ts
DEFAULT_CHART_STACKED_BAR_WIDTH        = 560
DEFAULT_CHART_STACKED_BAR_HEIGHT       = 320
DEFAULT_CHART_STACKED_BAR_PADDING      = 36
DEFAULT_CHART_STACKED_BAR_BAR_GAP      = 8
DEFAULT_CHART_STACKED_BAR_TICK_COUNT   = 5
DEFAULT_CHART_STACKED_BAR_ORIENTATION  = 'vertical'

getStackedBarCategoryTotal(series, hidden, categoryIndex): number
getStackedBarMaxTotal(categories, series, hidden): number
computeStackedBarLayout(categories, series, hidden,
                        orientation, innerW, innerH,
                        padX, padY, barGap): StackedBarSegment[]
getStackedBarTicks(max, count?): number[]
describeStackedBarChart(categories, series, hidden,
                        orientation, formatValue?): string
```

- `getStackedBarCategoryTotal` clamps non-finite +
  non-positive values to 0; respects the `hidden`
  set.
- `getStackedBarMaxTotal` falls back to 1 for
  empty / all-zero / all-non-finite data.
- `computeStackedBarLayout` is the canonical
  layout pass. Zero-value segments are skipped to
  keep the layout array compact. The vertical
  stack baseline-to-top invariant + the horizontal
  left-to-right invariant are both verified in the
  test suite.
- `getStackedBarTicks` evenly-spaced from 0 to
  `max`; non-positive max -> `[0]`; min count 2.
- `describeStackedBarChart` produces the default
  ARIA summary including the orientation +
  visible-count + per-series sums.

## Out of scope

- Percentage (100%) mode toggle -- absolute-only v1
  (use `<ChartStackedArea>` percentage mode when
  needed and discretise the x axis via the host).
- Negative values stacked below the baseline --
  non-positive values clamp to 0; host normalises
  to positive before passing.
- Drag-to-reorder series within the stack -- host
  owns the array.
- Animated reflow on data / orientation change
  (snap v1; mount fade only).

## Indicator math

A Williams Fractal is a swing pivot in the price series. For a
`wing` of `w` (the classic Williams fractal uses `w = 2`, a 5-bar
window), a bar at index `i` is:

```
up   fractal  if  value[i] > value[i-k]  AND  value[i] > value[i+k]
                  for every k in 1..w
down fractal  if  value[i] < value[i-k]  AND  value[i] < value[i+k]
                  for every k in 1..w
```

An up fractal is a local swing high (a potential reversal down);
a down fractal is a local swing low (a potential reversal up).
Comparisons are **strict**, so a tie with any neighbour
disqualifies the bar -- a flat or strictly monotonic series
carries no fractals. Because a fractal needs `w` bars of
confirmation on each side, the first `w` and last `w` bars can
never be fractals. `computeLineFractals` returns an array
parallel to the values carrying `'up'` / `'down'` / `null` per
bar; a bar can never be both (it cannot be strictly above and
strictly below the same neighbour).

### Worked fixture

`values = [5, 8, 20, 7, 3, 10, 22, 6, 9]`, `wing 2`:

```
i=2 (20) > {5, 8, 7, 3}      -> up
i=4 (3)  < {20, 7, 10, 22}   -> down
i=6 (22) > {3, 10, 6, 9}     -> up
fractals = [null, null, 'up', null, 'down', null, 'up', null, null]
```

Indices 0, 1, 7 and 8 are always null (the wing-2 warm-up edges).
upCount 2, downCount 1, lastUp 22, lastDown 3. A `wing` of 1
(a 3-bar window) on `[1, 5, 2, 4, 1]` yields
`[null, 'up', 'down', 'up', null]`.

## API

- `getLineFractalFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineFractalWing(wing, fallback)` -- coerce to a
  positive integer; non-finite or sub-1 falls back, fractional
  floors.
- `computeLineFractals(values, wing)` -- the per-bar fractal
  array (`'up'` / `'down'` / `null`).
- `runLineFractals(points, options?)` -- sort finite points by
  `x`, compute the fractal array, return `{ series, wing,
  fractals, signals, samples, upCount, downCount, lastUp,
  lastDown, ok }`. `signals` is the list of detected fractals
  (`{ index, x, value, type }`); `samples` is one row per bar
  tagged with its fractal type; `lastUp` / `lastDown` carry the
  value of the most recent up / down fractal. `ok` is false for
  fewer than two points.
- `computeLineFractalLayout(options)` -- project the price path
  and one marker per detected fractal.
- `describeLineFractalChart(data, options?)` -- ARIA description.

## Component

`ChartLineFractal` is a `forwardRef` region (`role="region"`)
with an ARIA-linked description, a `role="img"` SVG, axis ticks
and grid, a config badge, a three-series legend (Price / Up
fractal / Down fractal) with toggle, a hover/focus tooltip
exposing the value and fractal type, and a
`motion-safe:animate-fade-in` entrance. Up fractals draw as an
up-pointing triangle above the price line, down fractals as a
down-pointing triangle below it. Every node carries a
`data-section` attribute.

## Distinctness

- **vs the moving-average overlays (chart-line-smma /
  chart-line-lsma / chart-line-wma / ...)** -- those plot a
  continuous *smoothed line* derived by averaging or regressing
  a window of prices. The fractal primitive plots *discrete
  swing-pivot markers*: it detects local extrema, it does not
  average, and it takes a `wing` (bars per side) rather than a
  smoothing `period`.
- **vs an oscillator panel (chart-line-cmf /
  chart-line-disparity / ...)** -- those derive a continuous
  second series on a sub-panel. The fractal output is a sparse
  set of typed events overlaid directly on the price line.
- The two marker classes are mutually exclusive per bar -- a
  bar that is strictly above all its neighbours cannot also be
  strictly below them.

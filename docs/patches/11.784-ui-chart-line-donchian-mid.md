## Summary

`ChartLineDonchianMid` is a pure-SVG **single-panel** React primitive
that overlays the Donchian Channel midline on the close (a departure
from the dual-panel oscillator primitives shipped in the previous
dispatches). Definition:

```
upper[i]  = max(high, [i - length + 1, i])
lower[i]  = min(low,  [i - length + 1, i])
middle[i] = (upper[i] + lower[i]) / 2
```

Defaults: `length = 20`. Bars `0 .. length - 2` are warmup
(`middle = null`); the rolling max / min need `length` samples.

The component renders four series on the same Y-axis:

- the close line
- the Donchian midline overlay (primary; the dispatch's focus)
- the upper channel (dashed)
- the lower channel (dashed)
- an optional filled channel polygon between upper / lower

Each series can be hidden via the legend or via controlled
`hiddenSeries`. Markers respond to click + Enter + Space; tooltip
surfaces close + H/L + middle + upper/lower + zone on hover and focus.

## Bit-exact anchors

- **CONST_FLAT** (`high = low = close = K`): every window's max and
  min equal `K`, so `middle = K` and the midline coincides with the
  close. Bit-exact past warmup. The integration test sweeps several
  `K` values (including negatives).
- **CONST_BAR** (constant `high = H`, `low = L`, `close = (H + L)/2`):
  `upper = H`, `lower = L`, so `middle = (H + L) / 2 = close` exactly.
  The bar classifies as `at` post-warmup. The integration test
  sweeps `(H, L)` and varies `length` to confirm.

Additional anchors covered:

- CONST_BAR with `close = H` -> all post-warmup bars classify `above`.
- CONST_BAR with `close = L` -> all post-warmup bars classify `below`.
- Ramped highs / lows (`high[i] = i + 2`, `low[i] = i - 2`, length 5)
  yield `middle[i] = i - 2` for `i >= 4`, an integer arithmetic check.

- No new dependencies.
- No canvas, no third-party chart libs.

## `lib/settings-sections.ts`

Pure, no React, no DOM.

### Types

```ts
type SettingsGroup = 'account' | 'appearance' | 'shortcuts' | 'advanced';

interface SettingsSection {
  id: string;
  title: string;
  description?: string;
  group: SettingsGroup;
  keywords?: readonly string[];
  priority?: number;  // ascending within group
  badge?: string;
}
```

### Helpers

| function | purpose |
| --- | --- |
| `groupSections(sections)` | buckets by group, drops empty groups, sorts by `priority` |
| `normalizeSearchQuery(query)` | lower-case + whitespace split |

## `lib/use-debounced-save.ts`

```ts
useDebouncedSave({
  initialValue,
  onSave,         // (next) => Promise<T | void>
  debounceMs,     // default 300
  onBeforeSave?,
  onAfterSave?,
}) -> {
  value,          // optimistic, renders instantly
  status,         // 'idle' | 'pending' | 'saving' | 'error'
  error,
  commit(next),
  flush(),
  reset(next?),
}
```

### Lifecycle

1. `commit(next)` -- value updates immediately
   so the input renders the new state without
   waiting; status flips to `pending`; debounce
   timer scheduled.
2. Rapid commits coalesce -- only the latest
   value reaches the network.
3. Timer fires -> status `saving`, `onSave`
   runs.
4. **Success:** if `onSave` returns a value, it
   becomes the new committed state
   (server-authoritative; useful when the
   server normalises e.g. trims whitespace).
   Returning `void` keeps the optimistic value.
   Status -> `idle`.
5. **Failure:** value reverts to the last
   committed state. Status -> `error`. The next
   `commit()` clears the error and starts a
   fresh attempt.
6. `flush()` cancels the timer and runs the
   pending save immediately. Useful for the
   page's "Save now" button.
7. `reset(next?)` cancels + restores the
   optimistic + committed state.
8. Unmount cancels the pending timer so no
   save fires after the host tears down.

## Tests

`settings-sections.test.ts` -- 14 cases:

- `groupSections`: one entry per non-empty
  group, empty groups dropped, priority
  ascending, non-priority in source order,
  empty input, canonical labels.
- `normalizeSearchQuery`: empty / whitespace
  -> [], lowercase + split, collapse
  whitespace.
- `filterSections`: empty query -> full,
  id/title/group match,
  description/keywords match, AND semantics,
  case-insensitive.
- `filterAndGroupSections`: returns grouped
  shape, empty groups dropped after filter.
- `CANONICAL_SETTINGS_SECTIONS`: declared ids,
  every group covered, searchable by
  keywords.

`use-debounced-save.test.ts` -- 12 cases:

- Initial state shape.
- commit updates value + status -> pending.
- Coalesces rapid commits into one save.
- Status flow on success (idle -> pending ->
  saving -> idle).
- Revert + status error on failure.
- Server-authoritative value adoption when
  `onSave` returns a result.
- `flush()` fires pending save immediately.
- `reset()` restores last committed when no
  arg.
- `reset(next)` overrides both optimistic +
  committed.
- Next `commit()` after error clears error
  + restarts.
- `onBeforeSave` + `onAfterSave` fire.
- Unmount cancels pending timer (no save
  fires).

4.1.5. TypeScript clean.

## Pairs with existing primitives

- `lib/use-optimistic-mutation.ts` (11.336) --
  one-shot optimistic mutation (single submit,
  rollback on error). The new
  `useDebouncedSave` is the typed counterpart
  for live-editing surfaces (text field,
  toggle) where the host doesn't want to fire
  on every keystroke.
- `hooks/use-debounce.ts` (11.230) --
  `useDebouncedCallback` is the generic
  primitive; `useDebouncedSave` layers the
  optimistic UI contract on top.
- `lib/use-toast.ts` (11.137) -- error toasts
  are the natural surface for the `'error'`
  state. The host wires `status === 'error'`
  to a `showToast`.

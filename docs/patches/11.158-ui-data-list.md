# 11.158 -- UI: DataList semantic dl/dt/dd primitive (v1.11.176)

## Component

- `web/src/components/ui/data-list.tsx` -- semantic dl/dt/dd wrapper for
  key-value metadata blocks.
  - Exports: `DataList` (forwardRef to `HTMLDListElement`),
    `DataListItem` (type), `DataListOrientation` (type), `DataListProps`.
  - Item shape: `{ id, label, value, copyValue?, truncate? }`.
  - Orientations:
    - `horizontal` (default): fixed-width muted `dt` label + flex-1 `dd`
      value row.
    - `vertical`: small uppercase muted `dt` above foreground `dd`.
  - Copy chip: when `copyValue` is set, a small icon button is right-
    aligned inside `dd` (lucide Copy icon). On click,
    `navigator.clipboard.writeText(copyValue)` runs and the icon flips
    to lucide Check for 1.2s via local state. `aria-label='Copy <label>'`
    when label is a string, else `'Copy value'`.
  - Truncate: when `truncate` is true and value is a string, the dd
    span gets `truncate` class + `title` attribute and is wrapped in
    the existing Tooltip primitive for consistency; when value is a
    ReactNode (not a string), only the `truncate` class is applied.
- `web/src/components/ui/index.ts` -- adds `export * from './data-list'`.
- `web/src/components/ui/data-list.test.tsx` -- vitest + RTL coverage.

## Adoption sites (3)

1. `web/src/pages/Health.tsx` -- the stat grid (pid, uptime, started,
   workersTotal, active, idle, queueDepth, lostWorkers, eventLoopLag)
   now renders inside a single Panel as a horizontal DataList. The
   private `<Stat>` helper component is removed. The `statValue` test
   helper in `Health.test.tsx` resolves the dd via dt's next sibling
   when the label element is a dt.
2. `web/src/pages/Scribe.tsx` -- the status panel rows (running,
   lastScan, scans, sessions, contextPath) become DataList items. The
   private `<StatusRow>` helper is removed. `contextPath` carries
   `copyValue` so operators can copy the path with one click.
3. `web/src/components/HistoryDetailPane.tsx` -- the CardDescription
   row (status badge, branch, worktree) becomes a DataList with three
   items. Branch + worktree carry `copyValue` so the path is one
   click away. New i18n keys: `history.detail.label.status`,
   `history.detail.label.branch`, `history.detail.label.worktree`
   (en + ko bundles).

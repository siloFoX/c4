# 8.34 — Global scrollbar theme

## Why

User reported (2026-04-20): "모든 스크롤바가 디자인 적용 안 됐고 스크롤바 없어야 하는 곳에도 스크롤바 있어". After 8.9 (ARPS-style redesign) and 8.13 (auto-fit), scrollbar styling was still default browser chrome — looked out of place against the shadcn token palette.

## What changed

`web/src/index.css` gains a 51-line scrollbar style block:

- `::-webkit-scrollbar` — 8px width, transparent track
- `::-webkit-scrollbar-thumb` — rounded muted token, accent on hover
- Firefox `scrollbar-width: thin` + `scrollbar-color: hsl(var(--muted-foreground)/.4) transparent`
- Opt-in `.no-scrollbar` utility for snap carousels / mobile composer panels where chrome would dominate

Both light and dark theme tokens map cleanly via `--muted-foreground`.

## Scope

Step 1 of the 8.34 series. The remaining sub-items — overflow audit, viewport-fit refinement, hidden-scrollbar preference, puppeteer scrollbar audit — ship as follow-ups under TODO 8.34.

## Tests

CSS-only patch; no runtime test added. Browser smoke verified via `~/scripts/browser-automation/verify-global-scrollbar.js` (4/4 pass: index.css contains the selectors, Firefox scrollbar-width applied, themed pseudo-rules present in stylesheet at runtime, page renders without console errors).

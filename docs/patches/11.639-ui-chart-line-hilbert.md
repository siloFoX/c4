## Indicator math

The Hilbert Transform dominant cycle measures how many bars the
current price cycle takes. Three stages:

**Pre-smooth.** `computeLineHilbertSmooth` applies Ehlers'
4-3-2-1 weighted average `(4*p[i] + 3*p[i-1] + 2*p[i-2] +
p[i-3]) / 10`.

**Hilbert transform.** `computeLineHilbertTransform` is Ehlers'
seven-tap quadrature FIR
`0.0962*(v[i]-v[i-6]) + 0.5769*(v[i-2]-v[i-4])`. It shifts the
signal 90 degrees and has no DC response, so it both detrends
and produces the quadrature component. A constant series
transforms to zero exactly.

**Dominant cycle.** `computeLineHilbertCycle` smooths the price,
detrends it through the transform, and takes a second transform
for the quadrature. The in-phase signal is the detrender delayed
three bars (the filter's group delay). The instantaneous phase
`atan2(quadrature, in-phase)` rotates once per cycle, so the
per-bar phase change `deltaPhase` gives the dominant cycle
period `2*PI / deltaPhase`. The period is clamped to
`[minPeriod, maxPeriod]` and exponentially smoothed -- both
operations keep every reading inside the band. A flat series
reports the maximum period.

### Worked fixture

The phase uses `atan2`, so the period values are irrational --
they are asserted via the `[min, max]` clamp band (every value
provably inside it) and the exact-testable smoother and
transform recursions. A flat series reports `maxPeriod` for
every bar; the smoother holds a flat series at its constant and
the transform takes it to zero exactly.

## API

- `getLineHilbertFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineHilbertPeriod(period, fallback)` -- coerce a
  period bound to a positive integer.
- `computeLineHilbertSmooth(values)` -- the 4-3-2-1 weighted
  pre-smoother.
- `computeLineHilbertTransform(values)` -- the Hilbert transform
  quadrature filter.
- `computeLineHilbertCycle(values, minPeriod, maxPeriod)` -- the
  dominant cycle period series, clamped to the band.
- `runLineHilbert(points, options?)` -- sort finite points by
  `x`, run the pipeline, return `{ series, minPeriod, maxPeriod,
  smooth, period, samples, periodFinal, periodMin, periodMax,
  fastCount, slowCount, ok }`. Each sample is classified
  `fast` / `slow` / `mid` versus the band midpoint. `ok` is
  false for fewer than two points.
- `computeLineHilbertLayout(options)` -- stack a price panel
  above a cycle-period panel.
- `describeLineHilbertChart(data, options?)` -- ARIA
  description.

## Component

`ChartLineHilbert` is a `forwardRef` region (`role="region"`)
with an ARIA-linked description, a `role="img"` SVG, axis ticks
and grid for both panels, a config badge showing the period
band, a two-series legend (Price / Cycle period) with toggle, a
hover/focus tooltip exposing the value, the smoothed price, the
period and the cycle class, and a `motion-safe:animate-fade-in`
entrance. The top panel plots the raw price; the bottom panel
plots the dominant cycle period over the fixed
`[minPeriod, maxPeriod]` band, with a dashed midline at the
fast/slow boundary and class-coloured markers. Every node
carries a `data-section` attribute.

## Distinctness

- **vs the smoothing filters (chart-line-supersmoother,
  chart-line-roofing, chart-line-laguerre)** -- those produce a
  smoothed or filtered *price*. This primitive measures the
  *period* of the price cycle -- its output is a count of bars,
  not a price.
- **vs chart-line-roofing** -- the Roofing filter is a band-pass
  on the price. The Hilbert dominant cycle uses the transform's
  quadrature output to read the cycle length via the phase rate.
- **vs the oscillator panels** -- the cycle panel is not
  zero-centred; it spans the fixed `[minPeriod, maxPeriod]` band
  with a midline at the fast/slow boundary.

## Variant presets

```ts
type EmptyStateVariant =
  | 'empty-list'
  | 'no-results'
  | 'error'
  | 'loading-failed';

const VARIANT_DEFAULTS: Record<EmptyStateVariant, {
  illustration: EmptyStateIllustration;
  title: string;
  description: string;
}> = {
  'empty-list': {
    illustration: 'empty-queue',
    title: 'Nothing here yet',
    description: 'No items to show...',
  },
  'no-results': {
    illustration: 'search-empty',
    title: 'No matches',
    description: 'Your filter or search...',
  },
  error: {
    illustration: 'access-denied',
    title: 'Something went wrong',
    description: 'The request could not be completed...',
  },
  'loading-failed': {
    illustration: 'no-data',
    title: 'Could not load',
    description: 'We could not reach the server...',
  },
};
```

`EMPTY_STATE_VARIANTS` is exported as the
canonical list for adopters that need to
iterate.

### Override precedence

| explicit prop | overrides |
| --- | --- |
| `icon` | both `illustration` and the variant illustration |
| `illustration` | the variant illustration |
| `title` | the variant title |
| `description` | the variant description |

When a variant is set, the root carries
`data-empty-state-variant="<variant>"` so e2e
selectors can target the category without
knowing the title.

`title` becomes optional when `variant` is set;
otherwise the legacy contract still requires
it.

## Adoption

### `HistoryView.tsx`

The empty-state branch now picks `no-results`
when any filter is active and `empty-list`
otherwise:

```tsx
const filterActive =
  !!query || !!statusFilter || !!sinceDay || !!untilDay;
<EmptyState
  variant={filterActive ? 'no-results' : 'empty-list'}
  illustration="no-workers"
  title={t('history.empty')}
  description={
    filterActive
      ? 'No workers matched the current filter...'
      : 'No workers have completed tasks...'
  }
  secondaryAction={{
    label: 'Browse archived workers',
    href: '#feature=cleanup',
  }}
  data-testid="history-empty-state"
/>
```

The existing `data-testid="history-empty-state"`
survives so the existing test fixtures keep
working.

### `Templates.tsx`

Annotates the existing empty surface with
`variant="empty-list"`; the custom
`EmptyQueueIllustration` icon is preserved so
the visual stays byte-identical.

### `SpecialistsAuditPanel.tsx`

Swaps the inline `<div>` empty message for the
canonical `<EmptyState>`. Loading still renders
inline (EmptyState is the post-resolution
surface). The audit-window filter picks
`variant="no-results"` for non-`all` values;
otherwise `variant="empty-list"`. A new
`data-testid="audit-empty-state"` is added; the
loading branch gains `data-testid="audit-loading"`.

## Tests

`empty-state.test.tsx` -- 8 new cases:

- `empty-list` applies default illustration +
  title + description.
- `no-results` / `error` / `loading-failed`
  each set their own
  `data-empty-state-variant` + default title.
- Explicit `title` overrides the variant
  default.
- Explicit `description` overrides the variant
  default.
- Explicit `illustration` overrides the
  variant default.
- Explicit `icon` wins over the variant
  illustration.
- `data-empty-state-variant` is omitted when
  no variant is set.

41/41 EmptyState cases pass; HistoryView (39
existing) + Templates (44 existing) still
green. TypeScript clean for the touched
files; pre-existing strict-tuple / TS6133
warnings outside scope.

## Out of scope

- Every existing EmptyState call site
  migration. The variant prop is additive --
  legacy call sites with explicit
  illustration / title / description keep
  working byte-for-byte.
- i18n keys for the variant default copy.
  The defaults are English; adopters that
  need locale-aware copy keep passing
  explicit `title` / `description`.
- Light-mode token parity for the variant
  illustrations (already covered by the
  canonical illustrations).

  `'permission'` variant. The illustration
  token is available; adopters that need a
  fifth variant for the auth-related branch
  can pass `illustration="access-denied"`
  explicitly.

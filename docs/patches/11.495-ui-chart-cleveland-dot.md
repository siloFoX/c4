# TODO 11.495 - chart-cleveland-dot primitive

Status: SHIPPED v1.11.513 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartClevelandDot>`, a pure-SVG Cleveland dot plot (also known
as a dumbbell chart). Each category has two values rendered as a pair
of filled circles on the same row (horizontal) or column (vertical),
joined by a thin connector segment whose length and direction make the
gap immediately readable. Named after William Cleveland's preferred
visual form for paired-value comparisons.

## API

```tsx
<ChartClevelandDot
  items={[
    { id: 'a', label: 'Apples', before: 30, after: 50 },
    { id: 'b', label: 'Berries', before: 80, after: 40 },
    { id: 'c', label: 'Cherries', before: 20, after: 20 },
  ]}
  beforeLabel="2020"
  afterLabel="2025"
  sortBy="absDelta"
  sortOrder="desc"
  useDeltaColors
  valueLabel="kg per capita"
/>
```

## Distinct from prior chart-* primitives

- `<ChartLollipop>` (11.494) - one head + stick from a fixed baseline.
  Cleveland dot has two heads (independent values) connected by a
  segment between them, not from a baseline.
- `<ChartSlope>` (11.476) - two values per item drawn from a left axis
  to a right axis (visualises rank change). Cleveland dot puts both
  values on the SAME row/column on a single shared value axis.
- `<ChartBar>` - one value per category as a rectangle.
- `<ChartDotPlot>` - Wilkinson stacked dots showing frequency.
- `<ChartStrip>` (11.492) / `<ChartBeeswarm>` (11.493) - distribution
  viewers; Cleveland dot is two-point-per-category.
- `<ChartScatter>` / `<ChartBubble>` (11.470) - 2D xy; Cleveland dot
  is categorical with two values per category.

## Layout

### Value axis

- Range auto-derived from both endpoints across all finite items
  (`getClevelandDotBounds`).
- Collapsed range expands by `+/- 0.5`.
- `valueMin` / `valueMax` clamp the rendered range; out-of-range
  values clamp at the boundary (drawn at the edge, never dropped).
- `horizontal` (default): value axis = x, categories stacked
  vertically (Cleveland's preferred form for long category lists).
- `vertical`: value axis = y, categories side by side.

### Category axis

- N items share the inner length evenly. Each occupies
  `bandWidth = innerLength / N`.
- `sortBy = 'before' | 'after' | 'delta' | 'absDelta'` plus
  `sortOrder = 'asc' | 'desc'` reorders items. Ties break by
  original index. Default preserves input order.

### Connector + dots

- Two circles per item: `before` (left endpoint, default cyan) and
  `after` (right endpoint, default blue) of radius `dotRadius`
  (default 5).
- Connector: line from before-dot center to after-dot center,
  `connectorWidth` (default 1.5).
- When `useDeltaColors=true`: connector colour = `positiveColor` when
  `after > before`, `negativeColor` when `after < before`, otherwise
  `neutralColor`. When `useDeltaColors=false` (default) the connector
  is always `neutralColor` so the direction signal lives only in the
  dot positions.
- `epsilon` widens the flat band so tiny deltas register as flat.
- `isAboveBaseline` is implicit in `direction`.

## Interactivity

- Hover or focus on either dot dims every other category to opacity
  0.35 and opens a tooltip with the category label, both formatted
  values, the signed delta, and the direction symbol.
- Click on either dot fires `onItemClick({item, layout, series})`
  where `series` is `'before'` or `'after'`.
- Mount fade via `motion-safe:animate-fade-in` (suppressed via
  `animate=false`).

## Exported helpers

- `DEFAULT_CHART_CLEVELAND_DOT_*` numeric / string defaults including
  the five colour tokens (before, after, positive, negative, neutral).
- `getClevelandDotDirection(before, after, epsilon?)` -
  `'up' | 'down' | 'flat'`. Non-finite inputs -> `'flat'`. `epsilon`
  widens the flat band.
- `getClevelandDotFiniteItems(items)` - drops items with non-finite
  `before` OR `after`, or non-string `id`. Non-array -> `[]`.
- `getClevelandDotBounds(items)` - min/max across both endpoints;
  empty -> `(0..1)`; collapsed range expands `+/- 0.5`.
- `getClevelandDotTicks(min, max, count?)` - evenly-spaced ticks.
- `sortClevelandDotItems(items, sortBy?, order?)` - returns
  `{item, originalIndex}` pairs; preserves original index after sort.
- `computeClevelandDotLayout(input)` - canonical layout returning
  `{ items, ticks, valueMin, valueMax, orientation, innerWidth,
  innerHeight, bandWidth, upCount, downCount, flatCount }`. Per-item
  layout exposes both endpoint coordinates, the delta, the direction,
  the resolved connector colour, and the row centre. Empty when canvas
  is degenerate or no finite items.
- `describeClevelandDotChart(items, beforeLabel, afterLabel,
  formatValue?)` - ARIA summary "Cleveland dot plot comparing
  <before> to <after> across N categories: U up, D down, F flat.
  Value range <min> to <max>.".

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only
  `data-section="chart-cleveland-dot-aria-desc"` span.
- SVG `role="img"` with the same `aria-label`.
- Each dot is `role="graphics-symbol"` with `tabIndex={0}` and
  `aria-label="<category>: <beforeLabel> <before> -> <afterLabel>
  <after> (<direction>) (<series label>)"`.

## Data attributes

- Root: `data-section="chart-cleveland-dot"`, `data-orientation`,
  `data-category-count`, `data-finite-count`, `data-up-count`,
  `data-down-count`, `data-flat-count`, `data-use-delta-colors`,
  `data-sort-by`, `data-sort-order`, `data-animate`.
- Item: `data-section="chart-cleveland-dot-item"`, `data-item-id`,
  `data-item-index`, `data-item-original-index`,
  `data-item-direction`, `data-item-before`, `data-item-after`,
  `data-item-delta`, `data-connector-color`, `data-hovered`.
- Dot: `data-section="chart-cleveland-dot-dot"`, `data-item-id`,
  `data-series="before"` or `data-series="after"`.
- Connector: `data-section="chart-cleveland-dot-connector"`,
  `data-item-id`.
- Tooltip, legend, axes, grid, ticks, value-text, category-text:
  standard pattern.

## Tests

67 vitest cases (1 constants + 3 direction + 3 finite items + 3
bounds + 3 ticks + 6 sort + 14 layout + 5 describe + 32 component) all
pass under vitest 4.1.5. TypeScript clean for touched files.

## Out of scope

- More than two values per category (use parallel coordinates
  `<ChartParallel>`).
- Highlighted subsets via external selection (hover/focus only).
- Animated transitions on data / sort changes (snap; mount fade only).
- Log-scale value axis (linear only).
- Continuous category axis (categories are discrete).

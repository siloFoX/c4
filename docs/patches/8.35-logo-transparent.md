# 8.35 — Transparent logo background

## Why

User reported (2026-04-20): "아이콘의 누끼가 따진게 아니고 백그라운드 이미지가 있네". The C4 logo SVG carried a `#0D1B2A` opaque background `<path>` that wrapped the silhouette, so the logo painted itself a navy block instead of letting the host surface (sidebar / header / login modal / favicon) show through.

## What changed

`web/public/logo.svg` — single 60-byte deletion of the opaque background `<path>`. The C4 silhouette now renders against whatever surface hosts it.

Audit of related concerns:

- lucide-react icons were already stroke vectors (no background) — no work needed
- No external PNG raster icons in current code
- Both light + dark theme verified: silhouette stays visible against either
- Wrapper class audit (WorkerList / Sidebar / TopTabs / SettingsView IconButton variants) — already use `ghost` / transparent variants; no change needed

## Scope

Single asset edit. Closes the 8.35 row entirely.

## Tests

Asset-only patch; no runtime test added. Browser smoke verified the page still renders without errors (covered by `verify-final-smoke.js` 7-tab cycle).

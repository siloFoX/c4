# 11.516 -- web/src/components/ui/chart-line-band.tsx

Pure-SVG multi-series line chart with **horizontal reference bands**
between two y-values, spanning the entire x range. Each band is a
chart-wide rectangle anchored to `[yMin, yMax]` (not per-point) with a
kind-driven default colour (safe/warning/critical/info/neutral/custom)
and optional dashed top + bottom borders. Per-point band membership
is computed and surfaced via the tooltip + a `data-in-bands`
attribute on every dot.

## Distinct from other reference primitives

- `<ChartLineThreshold>` (11.503): horizontal threshold **lines** plus
  zone shading above / below each threshold. The chart-line-band
  primitive is **band-first** (the rectangle is the data),
  multi-band, with per-point membership classification.
- `<ChartLineBaseline>` (11.510): adjustable horizontal **baseline**
  with above / below / equal point classification. Band is a
  range, not a single line.
- `<ChartLineConfidence>` (11.502): per-point upper / lower bounds
  around ONE series.
- `<ChartLinePercentile>` (11.514): percentile bands derived from
  sample groups at each x. Band is a constant.
- `<ChartLineEvent>` (11.515): vertical event markers on a top track.
  Band runs horizontally.

The canonical "acceptable range" / "SLO target" / "tier zones" view
for an SRE / capacity-planning dashboard.

## Public surface

- `<ChartLineBand series={...} bands={...} />`.
- `bands: ChartLineBandSpec[]` -- `{id, label, yMin, yMax, color?,
  kind?, opacity?, borderDashArray?, showBorder?, labelPosition?}`.
- Six canonical kinds: safe / warning / critical / info / neutral /
  custom (each with a default colour). Per-spec `color` always wins.
- Label positions: `inside-top-left` (default), `inside-top-right`,
  `outside-right`, `outside-left`, `none`.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showBandLegend`, `showTooltip`, `showBandLabels`, `animate`.
- Standard sizing, format, a11y props.
- Controlled + uncontrolled `hiddenSeries` and `hiddenBands` (own
  legend below the series legend; clicking a band swatch toggles all
  events of that band).
- Callbacks: `onPointClick({series, point})`,
  `onBandClick({band})`, `onBandToggle({band, hidden})`.

## Pure helpers

- `getLineBandDefaultColor(index)` -- 10-color palette cycle.
- `getLineBandFinitePoints(points)` -- finite-only filter.
- `getLineBandKindDef(kind)` -- canonical
  `{color, label}` per kind, falling back to `custom` for unknown /
  missing.
- `normaliseLineBandSpec(spec)` -- validates and normalises a band
  spec. Drops null, missing id / label, or non-finite yMin / yMax.
  Swaps yMin / yMax when inverted. Defaults `kind` to `'custom'`,
  `color` to the kind def, `opacity` to the chart default (clamped to
  `[0, 1]`), `showBorder` to `true`, `labelPosition` to
  `'inside-top-left'`, `borderDashArray` to `'4 3'`.
- `isPointInLineBand(y, band)` -- returns `true` when `y` is finite
  and falls inclusively inside `[yMin, yMax]`.
- `getLineBandMembership(y, bands)` -- returns the ids of every band
  whose `[yMin, yMax]` contains `y`, in input order.
- `computeLineBandLayout({series, bands?, hiddenSeries?, hiddenBands?,
  ...})` -- returns `{series, bands, xTicks, yTicks, xMin, xMax, yMin,
  yMax, innerWidth, innerHeight, totalPoints, visibleSeriesCount,
  visibleBandCount}`. Y range expands to include band ranges.
  Per-point `inBands` records membership.
- `describeLineBandChart(series, bands?, hiddenSeries?, hiddenBands?,
  formatValue?)` -- builds the screen-reader description.

Per-spec `color` always wins.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Each band rect is `role="graphics-symbol"` + `tabIndex={0}` with
  aria-label `"Band <label> (<kind>): <yMin> to <yMax>"`.
- Per-line path + per-dot are `role="graphics-symbol"` +
  `tabIndex={0}`. Dot aria-label includes the in-band membership list
  when non-empty.
- Every node carries `data-section="chart-line-band-..."` for test
  selectors.
- Bands expose `data-band-id`, `data-band-kind`, `data-band-color`,
  `data-band-y-min`, `data-band-y-max`, `data-band-hovered`,
  `data-band-in-range`.
- Dots expose `data-in-bands` (comma-separated band ids),
  `data-band-count`.

## Tooltip

Hovering a dot shows the series label, x, y, and a `bands:` row with
the comma-joined labels of every band containing the point (or
`none` when the point is outside every band).

## Goal

Dispatch:

- Build `web/src/components/ui/tooltip.tsx`
  -- Radix-style tooltip with hover/focus
  triggers, placement (top/right/bottom/left),
  arrow, controlled/uncontrolled, ARIA
  describedby.

`<Tooltip>` already shipped in 11.276
(v1.11.294) with hover/focus triggers,
placements, arrow, ARIA-describedby, separate
showDelay/hideDelay, and a one-way `open`
prop. This patch lands the missing Radix-style
features: observable controlled mode,
`defaultOpen`, portal rendering, and touch
long-press.

## New props

| prop | default | purpose |
| --- | --- | --- |
| `onOpenChange(next)` | `undefined` | fires on every visibility flip |
| `defaultOpen` | `false` | uncontrolled initial value |
| `portal` | `false` | true / HTMLElement -> mount via React portal |
| `touchOpen` | `true` | enable touch long-press to show |
| `touchHoldMs` | `500` | long-press duration before show |

The legacy `open` prop is preserved as a
loose-control signal -- a useEffect mirrors it
into local state on every prop change. Local
flips (hover / focus / Escape / touch) still
update internal state and fire `onOpenChange`.
This preserves byte-identical behaviour for
existing call sites (Escape still closes when
`open` is set) while adding the observability
hook.

## Portal rendering

```tsx
<Tooltip label="Open settings" portal>
  <IconButton ... />
</Tooltip>
```

`portal={true}` mounts the body under the
`tooltip-root` portal node managed by
`lib/portal-root.ts` (auto-created on first
use). The body switches to `position: fixed`
instead of `absolute` so it sits at the
viewport level and escapes any
`overflow:hidden` ancestor. The trigger
wrapper gets `data-portal="true"`.

A caller-supplied HTMLElement target is used
verbatim (no portal root lookup):

```tsx
<Tooltip portal={document.getElementById('my-portal')}>
  ...
</Tooltip>
```

## Touch long-press

Touch input doesn't fire `mouseenter`, so
hover-only tooltips never showed on phones.
The new touch handlers:

- `touchstart` -> schedule `setTimeout(setOpen(true), touchHoldMs)`.
- `touchend` / `touchcancel` -> cancel the
  timer + call `hide()`.

A quick tap never shows the tooltip; only a
held touch does. `touchOpen={false}` disables
the handlers so a long-press passes through
to the host (e.g. a custom context menu or a
drag interaction).

## Tests

9 new cases in `tooltip.test.tsx` on top of
the existing 26:

- `onOpenChange` fires on hover transitions
  (true then false).
- `defaultOpen` makes the tooltip visible on
  mount.
- `open` prop sync writes the latest
  controlled value into local state on every
  change (rerender flip test).
- `portal={true}` renders the body in the
  portal root + flips `data-portal="true"`.
- Legacy inline render keeps
  `data-portal="false"`.
- Custom HTMLElement portal target works.
- Touch long-press shows after `touchHoldMs`.
- `touchend` before `touchHoldMs` cancels.
- `touchOpen={false}` disables touch.

35/35 pass under vitest 4.1.5. TypeScript
clean.

## Pairs with existing primitives

- `lib/portal-root.ts` --
  `getPortalRoot('tooltip-root')` is the
  canonical mount target.
- Existing `placement` / `arrow` / `showDelay`
  / `hideDelay` props unchanged.
- The 47-case `tooltip.snapshot.test.tsx`
  suite still renders the legacy inline
  shape so the snapshot contract is
  preserved (no `portal` adoption in the
  visual tests).

## Out of scope

- Collision detection / flip when the
  tooltip would overflow the viewport.
  Requires layout measurements that jsdom
  does not provide; a floating-ui-style
  follow-up can layer it.
- React 19 ref-as-prop migration. The
  trigger still uses cloneElement; a future
  patch can swap to React 19's new ref
  semantics once the project upgrades.
- Native Popover API integration. Browser
  support is uneven; a follow-up can offer
  it as an opt-in.
- Strict controlled mode where the host's
  `open` prop is the only source of truth
  (Escape would be a no-op without the
  host's cooperation). The current loose
  control matches the existing call-site
  contract; a strict mode would break the
  Escape test fixtures.

# 11.210 -- UI: relative-time component

Standalone `RelativeTime` component for the web UI -- a tiny,
dependency-free, SSR-safe `<time>` wrapper that renders a self-
refreshing relative timestamp ("3 minutes ago", "in 2 hours") with
the absolute locale string parked on the `title=` attribute for
hover disclosure. Component-scope only: no daemon endpoint, no API
surface, no i18n binding. Replaces five inline date-formatting
callsites where a relative label is more useful than an ISO / locale
dump.

## Files

- `web/src/components/RelativeTime.tsx` (new, ~120 lines) -- the

- `web/src/components/RelativeTime.test.tsx` (new, 17 cases) --

- `web/src/components/MeetingsRecapPanel.tsx` -- escalation

- `web/src/components/AttachModal.tsx` -- preview list `updatedAt`

  laid out when the timestamp is missing.
- `web/src/pages/ErrorReports.tsx` -- captured-error timestamp
  swap; the local `formatTimestamp` helper is retired in the same
  pass.
- `web/src/pages/Snapshots.tsx` -- snapshot `createdAt` cell swap.

## API

```ts
type RelativeTimeValue = Date | string | number;

interface RelativeTimeProps {
  value: RelativeTimeValue;
  now?: Date;                  // override for tests / frozen UIs
  absoluteOnHover?: boolean;   // default true -> title=<locale>
  className?: string;
}
```

## Behaviour

- **Label formula** (positive delta = past):
  - `< 30s` -- `just now`
  - `< 60s` -- `<N> seconds ago`
  - `< 1h`  -- `<N> minute(s) ago`
  - `< 1d`  -- `<N> hour(s) ago`
  - `< 30d` -- `<N> day(s) ago`
  - `< 1y`  -- `<N> month(s) ago`
  - else    -- `<N> year(s) ago`
- **Future tense** (negative delta) mirrors the same buckets:
  `in <N> seconds`, `in <N> minutes`, etc. The `< 30s` bucket
  collapses to `just now` in both directions so a freshly-arrived
  row doesn't flicker around zero.
- **Auto-refresh tick**: the delta picks the interval -- `1s` while
  the delta is under a minute, `1min` under an hour, `1h` under a
  day, `1d` otherwise. Each tick reschedules the next one so an
  entry that crosses a boundary doesn't keep firing at the wrong
  rate.
- **SSR safety**: the initial render uses the `now` prop (or
  `new Date()`) directly so server output matches the first client
  paint. The `setTimeout` install is gated behind `useEffect`, and
  passing an explicit `now` prop short-circuits the timer entirely
  -- test fixtures and frozen-frame stories get a deterministic
  label that never drifts.
- **DOM contract**: always renders `<time dateTime={iso}>` with the
  relative label inside. With `absoluteOnHover` (default), the
  `title=` attribute holds `Date#toLocaleString()` so hover surfaces
  the exact timestamp. Add a `className` to style the wrapper at
  the callsite.
- **Invalid input**: if `Date.parse` rejects the value, the
  component renders the raw input inside a `<time>` element with
  `data-relative-time="invalid"` -- no crash, no NaN string.

## Why now

The web UI has been growing a quiet drift of one-off date helpers
(`formatTimestamp`, `formatRelative`, `formatRelativeTime`) sprinkled
across `SessionsView`, `MeetingsView`, `ErrorReports`,
`lib/format.js`, and a few raw `new Date(x).toLocaleString()` calls.
Each helper handles a slightly different subset (null guards,
relative vs. absolute, i18n, no auto-refresh). 11.210 keeps the
existing string-returning helpers around for callers that still want
a string, and introduces a dedicated JSX primitive for the common
"render this timestamp inline and keep it fresh" case.

The component is intentionally narrow: timestamps only, no numeric
pretty-printing, no i18n binding. Adopters that need a translated
relative string keep the existing `tFormat('...relative.minutes',
{ n })` callsite; the v1 of `RelativeTime` is English-only and
trades i18n for SSR + tooltip + auto-refresh ergonomics.

## Scope discipline

- No `package.json` dependency additions (no `date-fns`, no `dayjs`,
  no `moment`). Built-in `Date` math only.
- No `package-lock.json` change.
- No daemon-side change.
- Callsites that format numbers (`toLocaleString()` on a token count,
  a turn count, etc.) are intentionally left untouched; the
  component is for timestamps.
- Callsites where the displayed string is a duration (elapsed
  meeting time, scribe lookback) keep their existing helper.

## Tests

`web/src/components/RelativeTime.test.tsx` covers:

1. `just now` for a delta under 30 seconds.
2. `N seconds ago` between 30s and 1 minute.
3. `N minutes ago` in the 1-59 minute range.
4. Singular `minute` form for exactly one minute.
5. `N hours ago` in the 1-23 hour range.
6. `N days ago` in the 1-29 day range.
7. `N months ago` once the delta crosses 30 days.
8. `N years ago` once the delta crosses 365 days.
9. Future tense `in N minutes` for negative deltas.
10. Future tense `in N hours` for a multi-hour negative delta.
11. The `dateTime` attribute carries the ISO string.
12. The `title` attribute carries `toLocaleString()` when
    `absoluteOnHover` is the default.
13. The `title` attribute is absent when `absoluteOnHover={false}`.
14. Accepts a `Date` instance as the `value` prop.
15. Accepts a numeric epoch as the `value` prop.
16. Invalid input renders the raw value as fallback.
17. Auto-refresh tick updates the displayed label after
    `vi.advanceTimersByTime`.
18. The scheduled timeout is cleared on unmount.

`vi.useFakeTimers()` + `vi.setSystemTime` pin the wall clock; the
`now` prop is used wherever the assertion needs to be deterministic
without freezing the auto-refresh path under test.

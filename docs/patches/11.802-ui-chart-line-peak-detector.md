# 11.802 UI: chart-line-peak-detector primitive (v1.11.820)

## Summary
Pure-SVG dual-panel React/TS primitive that marks **local maxima of `close`
above a rolling SMA+stddev threshold** across the lookback. Top 60% panel
plots the price line; bottom 40% panel plots a binary 0/1 peak indicator
and renders a marker on each strict local maximum.

## Formula
```
mean[i]      = SMA(close, length)[i]
stdDev[i]    = sqrt( (1/length) * sum_{k=i-length+1..i} (close[k] - mean[i])^2 )
threshold[i] = mean[i] + threshFactor * stdDev[i]
peak[i]      = (close[i] > close[i-1])
             AND (close[i] > close[i+1])
             AND (close[i] > threshold[i])
             ? 1 : 0
```
`peak[i]` is `null` at boundaries (`i == 0` or `i == n - 1`), during SMA
warmup (`i < length - 1`), and whenever any input is non-finite.

## Defaults
- `length = 14`
- `threshFactor = 1`
- `width = 760`
- `height = 280`
- `panelGap = 12`

## Bit-exact testability anchors
- **CONST close** (`close[i] == K` for every `i`): `stdDev = 0`,
  `threshold = K`, and the strict `> threshold` predicate fails -> every
  valid bar has `peak = 0`. Verified across `K in {0, 1, 5, 100, -3}` and
  every `(length, threshFactor)` combination.
- **ALTERNATING [0, 1, 0, 1, ...]** with `threshFactor = 0`: odd-index
  bars (`close = 1`) have neighbours `0`, `mean = 0.5`, `threshold = 0.5`,
  and `1 > 0.5` -> `peak = 1`. Even-index bars stay at 0.

## Design notes
- Strict `>` everywhere (so a flat plateau never reports a peak).
- Peak axis is fixed to `[0, 1]` regardless of data, so the indicator
  panel reads as a binary gate.
- Markers only render where `peak == 1` (zero-valued bars don't clutter
  the panel with empty dots).
- `noUncheckedIndexedAccess` is satisfied via explicit `?? null` and
  defensive guards before access.

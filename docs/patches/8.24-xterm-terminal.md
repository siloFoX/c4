# 8.24 + 8.27 -- xterm.js terminal in WorkerDetail

**Branch:** `c4/xterm-fix`
**Status:** done
**Date:** 2026-04-20

## Summary

The WorkerDetail "Screen" tab used to render the worker PTY stream
through an append-only `<pre>` with a hand-rolled `stripAnsi`. The
stripper removed every `ESC[...` CSI sequence, so Claude Code's
spinner / thinking-box / progress frames, each of which is emitted
with cursor-up + `ESC[2K` + re-paint, ended up concatenated line
after line. Any alt-screen TUI (the Claude Code prompt picker,
`htop`, `fzf`) turned into a wall of repeated frames.

8.24 swaps that `<pre>` for a real `@xterm/xterm` Terminal inside a
new `web/src/components/XtermView.tsx`. xterm parses the ANSI
control flow natively so in-place redraws land in the right cells
and the pane now reflects the current screen state, not the
history of it. Fold into the same patch: 8.27 (tab-switch was
required before the 120ms fit loop applied). The fix is the same
commit - `<XtermView>` stays mounted under
`tab === 'screen' ? 'block' : 'hidden'` so the ResizeObserver +
window resize pipeline never tears down, and a `useLayoutEffect`
re-fits the moment `visible` flips back to true.

## Reproduction

1. `npm --prefix web install && npm --prefix web run dev`, open a
   worker in the web UI, Terminal tab.
2. Ask the worker to run something that triggers the Claude Code
   spinner or thinking box.
3. Before 8.24: every render tick appended another copy of the
   spinner frame + box. The pre scrolled faster than the worker
   responded and you could not tell the current state.
4. After 8.24: spinner animates in place, alt-screen TUIs paint at
   the right grid positions, scroll stays stable.
5. Before 8.27: toggle the sidebar or resize the window with the
   Chat tab active, then switch to Terminal -- cols stayed stuck
   at the pre-toggle value until the next tab flip.
6. After 8.27: cols are always current because the observer stayed
   wired and `useLayoutEffect(visible)` re-fits on return.

## Implementation

### Dependencies

Added to `web/package.json`:

- `@xterm/xterm ^6.0.0`
- `@xterm/addon-fit ^0.11.0`
- `@xterm/addon-search ^0.16.0`
- `@xterm/addon-web-links ^0.12.0`

No other version pins. The deprecated plain `xterm` package was
checked and discarded in favour of the scoped `@xterm/xterm` v6

### web/src/components/XtermView.tsx (new)

- Mounts a `Terminal({ fontFamily: 'ui-monospace, ...', fontSize,
  scrollback: 5000, cursorStyle: 'block', convertEol: false,
  allowProposedApi: true, theme: buildXtermTheme() })` per worker
  and opens it into a `<div ref={containerRef}>`.
- Loads `FitAddon`, `SearchAddon`, `WebLinksAddon` via
  `term.loadAddon(...)` before `term.open(container)`.
- Subscribes to `/api/watch?name=<workerName>` via
  `eventSourceUrl(...)` + `new EventSource(url)` (same auth
  contract as ChatView). Each SSE `{ type: 'output', data: <b64> }`
  frame is base64-decoded and handed to `term.write(...)` without
  stripping. A source-grep test forbids `stripAnsi(` in the file
  so the regression cannot creep back.
- Theme is driven from shadcn CSS custom properties:
  `readShadcnColor('--background', '#ffffff')`,
  `readShadcnColor('--foreground', '#0a0a0a')`,
  `--muted-foreground`, `--primary`, `--accent`, `--destructive`.
  The tokens are stored as bare `H S% L%` triples in
  `web/src/index.css` so `readShadcnColor` wraps them in
  `hsl(...)`. A `MutationObserver` on
  `document.documentElement` with
  `attributeFilter: ['class']` re-runs `buildXtermTheme()`
  whenever the dark-mode class flips.
- Alt-screen tracking: `term.buffer.onBufferChange(onBufferChange)`
  sets an `altScreen` state flag to `term.buffer.active.type ===
  'alternate'`. xterm already freezes the scrollback while the alt
  buffer is live, so no bespoke auto-scroll guard is needed; the
  flag drives the in-panel status badge only.
- Fit pipeline: `FitAddon.fit()` is called through a 120ms
  `setTimeout` debounce (`FIT_DEBOUNCE_MS = 120`) shared across a
  `ResizeObserver(container)` and `window.addEventListener(
  'resize', ...)`. The measured `cols`/`rows` are clamped to the
  daemon PTY range (`MIN_COLS=20`, `MAX_COLS=400`, `MIN_ROWS=5`,
  `MAX_ROWS=200`, mirrors `src/pty-manager.js _clampResizeDims`)
  and POSTed to `/api/resize` with `lastResizeRef` dedupe so the
  daemon only sees real dim changes. `fit.fit()` runs inside a
  `try/catch` because it throws when the container is 0x0 (tab
  hidden on first mount); the ResizeObserver retries once the tab
  becomes visible.
- 8.27 visible re-fit: a `useLayoutEffect([visible])` calls
  `scheduleFit()` whenever the `visible` prop flips to true, so
  any window-resize that happened while the terminal tab was
  hidden gets reconciled the moment the tab comes back.
- `VITE_AUTOFIT_DEBUG` carried over from 8.22. When the env var is
  `'1'` / `'true'`, the fit callback emits one
  `console.debug('[autofit] cols=%d rows=%d -> POST /api/resize',
  cols, rows)` per recompute.
- Ctrl+F / Cmd+F (focus inside the terminal container) opens a
  small search overlay bound to `SearchAddon.findNext(query)` and
  `SearchAddon.findPrevious(query)`. Shift+Enter runs prev, Enter
  runs next, Escape closes.
- Clean mount/unmount: the `workerName` effect creates + disposes
  the Terminal; a separate `[fontSize]` effect writes
  `term.options.fontSize = fontSize` + reschedules a fit so
  scrollback is preserved across font changes.

### web/src/components/WorkerDetail.tsx (changed)

- Imports + renders `<XtermView workerName fontSize
  visible={tab === 'screen'} />` inside a wrapper that toggles
  `tab === 'screen' ? 'block' : 'hidden'` so the terminal stays
  mounted while the Scrollback tab is active. This is the 8.27
  keep-alive that kept the auto-fit loop wired across tab
  switches.
- Dropped: the ruler-based char-width measurement (`rulerRef`,
  `charW`, `inner`), the `autoFit` toggle, the manual `cols`
  input, `AUTOFIT_DEBUG` local, `recomputeFit`, `requestResize`,
  the `window.resize` + `ResizeObserver(pre)` effects, and the
  `serverDims` state. xterm + XtermView own measurement now. The
  font-size +/- control stays.
- Kept: the send-text / Enter / Merge / Close / Esc / Ctrl-C /
  Ctrl-D / Tab / arrow-key buttons, and the Scrollback tab which
  still renders a stripAnsi `<pre>` of the latest
  `/api/scrollback` response (grep-style historical view; we do
  not try to re-emit past frames into xterm).

## Rollback

The old `<pre>` + `stripAnsi` code path is in git history at the
commit before this patch. If xterm.js needs to be removed:

1. `git -C /path revert <commit>` on `c4/xterm-fix`;
2. `npm --prefix web uninstall @xterm/xterm @xterm/addon-fit
   @xterm/addon-search @xterm/addon-web-links`;
3. Re-run `node tests/run-all.js` and confirm the 8.22 P1 block
   re-targets `WorkerDetail.tsx`.

No server-side migration is required -- the daemon PTY stream is
unchanged.

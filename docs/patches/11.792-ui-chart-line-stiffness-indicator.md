## Summary

`ChartLineStiffnessIndicator` is a pure-SVG dual-panel React
primitive that plots the close on the top panel and the Markos
Katsanos Stiffness Indicator on the bottom panel. The indicator
counts how often the close sits at or above a noise floor below
the moving average:

```
middle[i]    = SMA(close, length)[i]
stdDev[i]    = sqrt(sum((close - middle)^2) / length)
               (population)
threshold[i] = middle[i] - stiffnessFactor * stdDev[i]
above[i]     = 1 if close[i] >= threshold[i] else 0
stiffness[i] = sum(above, [i - length + 1, i]) / length * 100
```

Defaults: `length = 20`, `stiffnessFactor = 0.2`. Bars before
`i = 2 * length - 2` are warmup nulls because the count window
needs `length` valid `above` samples, and `above` itself needs
`length` close samples for the SMA / stdDev.

## Bit-exact anchor

For **CONST close** (`close = K`): the SMA collapses to `K`,
every centered residual is zero so `stdDev = 0`, the threshold
equals `K` regardless of the stiffness factor, and the comparison
`close >= threshold` is always true. So every `above` is `1` and
`stiffness = length / length * 100 = 100` exactly past warmup.

The integration sweep verifies this across `K` in `{0, 1, 5, 100,
-3, 7, -50}`, `length` in `{3, 5, 7, 10}`, and
`stiffnessFactor` in `{0, 0.2, 0.5, 1}`. The factor independence
demonstrates that the volatility band cannot leak in when the
close is constant.

## Zone classifier

- `rigid` -- stiffness >= 75 (close holds above the band most of
  the time)
- `firm` -- 50 <= stiffness < 75
- `soft` -- 25 <= stiffness < 50
- `fluid` -- stiffness < 25 (close drops below the band often)
- `none` -- stiffness is null (warmup)

## Out of scope

- No new dependencies.
- No canvas, no third-party chart libs.
- This primitive uses a single unified `length` for the SMA / stdev
  window AND the count window. Katsanos's original prescription
  allows separate windows (e.g. SMA over 100 bars, count over 60).
  A two-length variant could ship in a future dispatch.

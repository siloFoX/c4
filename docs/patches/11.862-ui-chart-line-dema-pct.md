## Summary

Added `<ChartLineDemaPct />` -- a pure-SVG, dual-panel React/TS
primitive that overlays the close with the Double Exponential
Moving Average (DEMA) in the top panel and renders the DEMA
percent deviation in the bottom panel:

```
ema1[i]    = EMA(close, length)
ema2[i]    = EMA(ema1,  length)
DEMA[i]    = 2 * ema1[i] - ema2[i]
demaPct[i] = DEMA[i] === 0 ? null
                           : (close[i] - DEMA[i]) / DEMA[i] * 100
```

Regime classifier: `above` (demaPct > 0), `below` (< 0),
`at` (= 0), `none` (null). Defaults: `length = 14`, demaColor
`#7c3aed` (violet), pctColor `#0ea5e9` (sky), zero line `#475569`.
The EMA helper carries the SMA-seeded `min === max` precision
fix plus the `next = v === smoothed ? v : ...` short-circuit, so
constant inputs lock onto their values exactly.

## Bit-exact anchor

- **CONST close = K (K > 0)**: every EMA in the chain collapses
  to K via the precision fix. `DEMA = 2K - K = K`, deviation = 0,
  `demaPct = 0 / K * 100 = 0`. Verified across multiple K and
  length.
- **CONST close = K = 0**: same chain but the divide-by-zero
  guard leaves `demaPct = null` (regime `none`).

## Algebraic soft anchor

For a linear series the steady-state EMA lag is `(n - 1) / 2`
(alpha = 2 / (n + 1)), so `ema1 = close - lag`,
`ema2 = ema1 - lag = close - 2*lag`,
`DEMA = 2*ema1 - ema2 = close`. Floating-point intermediates
introduce a small 1-ULP drift; the test asserts the demaPct
magnitude stays inside a 1.0 tolerance once the chain has
sufficiently warmed (40 bars into a 60-bar linear series with
length = 5).

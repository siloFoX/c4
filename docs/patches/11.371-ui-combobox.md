## API

```tsx
import {
  Combobox,
  selectOptionsToComboboxOptions,
} from './components/ui/combobox';

// Share an options array with <Select>
const fruits = [
  { value: 'apple', label: 'Apple' },
  { value: 'banana', label: 'Banana' },
];

<Combobox
  options={selectOptionsToComboboxOptions(fruits)}
  value={picked}
  onChange={setPicked}
  ariaLabel="Pick a fruit"
  onQueryChange={(q) => fetchRemote(q)}
  debounceMs={300}
/>
```

## New surface

### `debounceMs?: number`

Default `0` (legacy fire-every-keystroke).

When `> 0`:
- each new keystroke clears the pending
  timer and starts a new one;
- the host's `onQueryChange` fires once,
  `debounceMs` ms after the LAST keystroke;
- the internal client-side filter (used
  when no `onQueryChange` is wired) is NOT
  debounced -- the dropdown updates
  immediately so the user sees responsive
  feedback regardless of `debounceMs`;
- the pending timer is cleared on unmount.

Semantics are trailing-edge only -- there is
no leading-edge call. This matches the
typical autocomplete pattern where the host
wants the FINAL typed value, not the first.

### Home / End

When the popup is open, jumps the highlight
to the first / last ENABLED option in the
visible list (skipping disabled rows). When
the popup is closed, both keys are no-ops at
the combobox level (so the parent surface's
Home/End behaviour is preserved).

### `selectOptionsToComboboxOptions(options)`

Pure adapter from `SelectOption` (or any
`{ value, label: string, disabled? }` shape)
to `ComboboxOption`:

```ts
function selectOptionsToComboboxOptions<V extends string = string>(
  options: ReadonlyArray<{
    value: V;
    label: string;
    disabled?: boolean;
  }>,
): ComboboxOption<V>[]
```

- Forwards `value` + `label` verbatim.
- Forwards `disabled` only when it is set
  on the input (so the output object does
  not carry a stray `disabled: undefined`
  key under `exactOptionalPropertyTypes:
  true`).
- Returns a fresh array; the caller can
  mutate without affecting the input.
- Preserves the type-parameter widening for
  `V`, so a `'red' | 'blue'` input keeps the
  literal-union type at the boundary.

## Tests

total)**:

Debounce + Home/End (7 cases):

- default debounceMs=0 fires onQueryChange
  synchronously per keystroke.
- debounceMs>0 collapses bursts of keystrokes
  into a single trailing call.
- debounce window resets on each new
  keystroke (trailing-only).
- debounceMs does NOT delay the internal

  enabled option.

## Out of scope

- **Multi-select mode.** `<MultiSelect>`
  (11.370) covers the array picker; Combobox
  stays single-value.
- **Async cancellation.** `debounceMs`
  collapses typed bursts but does not cancel
  in-flight host fetches. Hosts pair
  `onQueryChange` with their own
  `AbortController` story.
- **ReactNode label adapter.** The shipped
  adapter expects `label: string` (matching
  SelectOption); ReactNode labels belong in
  the native ComboboxOption shape.
- **Leading-edge debounce.** The contract is
  trailing-edge only -- matches the typical
  autocomplete pattern where the host wants
  the final typed value.

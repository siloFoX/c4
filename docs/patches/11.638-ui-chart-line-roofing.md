## Summary

New `ChartLineRoofing` UI primitive at
`web/src/components/ui/chart-line-roofing.tsx` -- a pure-SVG
two-panel chart with an Ehlers Roofing filter combining a
high-pass and a smoother.

## Indicator math

The Roofing filter chains two stages.

**Stage one -- high-pass.** A two-pole high-pass strips the
low-frequency trend ("wander"). With
`angle = 0.707 * 2 * PI / hpPeriod`:

```
alpha1 = (cos(angle) + sin(angle) - 1) / cos(angle)
HP[i]  = (1 - alpha1/2)^2 * (price[i] - 2*price[i-1] + price[i-2])
       + 2*(1 - alpha1) * HP[i-1] - (1 - alpha1)^2 * HP[i-2]
```

**Stage two -- smoother.** An Ehlers Super Smoother (two-pole
low-pass) strips the high-frequency noise. With
`arg = 1.414 * PI / ssPeriod`: `a1 = exp(-arg)`,
`b1 = 2*a1*cos(arg)`, `c2 = b1`, `c3 = -(a1*a1)`,
`c1 = 1 - c2 - c3`, then
`Filt[i] = c1*(in[i] + in[i-1])/2 + c2*Filt[i-1] + c3*Filt[i-2]`.

The Roofing filter is the smoother applied to the high-pass --
combining the two leaves only a band of cycle frequencies, an
oscillator centred on zero. The first two bars seed at zero, and
a flat price series stays at zero through both stages.

### Worked fixture

`hpPeriod 48`, `ssPeriod 10`, price series
`[10, 12, 11, 15, 13, 18, 14, 20]`. The coefficients come from
`cos`, `sin` and `exp`, so they are irrational and the filter
values are asserted against the recursions themselves (using the
exported `computeLineRoofingHighpassAlpha` and
`computeLineRoofingSmootherCoefficients`) rather than hard-coded.
A flat series stays at zero throughout.

## API

- `getLineRoofingFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineRoofingPeriod(period, fallback)` -- coerce to a
  positive integer.
- `computeLineRoofingHighpassAlpha(hpPeriod)` -- the high-pass
  alpha coefficient.
- `computeLineRoofingHighpass(values, hpPeriod)` -- stage one,
  the detrended high-pass series.
- `computeLineRoofingSmootherCoefficients(ssPeriod)` -- the
  Super Smoother coefficients `{ a1, b1, c1, c2, c3 }`.
- `computeLineRoofingSmoother(values, ssPeriod)` -- stage two,
  the Super Smoother of a series.
- `computeLineRoofing(values, hpPeriod, ssPeriod)` -- the full
  Roofing filter, `smoother(highpass(values))`.
- `runLineRoofing(points, options?)` -- sort finite points by
  `x`, run both stages, return `{ series, hpPeriod, ssPeriod,
  highpass, roofing, samples, roofingFinal, roofingMin,
  roofingMax, positiveCount, negativeCount, ok }`. Each sample is
  classified `positive` / `negative` / `zero`. `ok` is false for
  fewer than two points.
- `computeLineRoofingLayout(options)` -- stack a price panel
  above a roofing panel; project both lines plus a zero line.
- `describeLineRoofingChart(data, options?)` -- ARIA
  description.

## Component

`ChartLineRoofing` is a `forwardRef` region (`role="region"`)
with an ARIA-linked description, a `role="img"` SVG, axis ticks
and grid for both panels, a config badge showing the two
periods, a two-series legend (Price / Roofing filter) with
toggle, a hover/focus tooltip exposing the value, the high-pass
reading, the roofing reading and the sign, and a
`motion-safe:animate-fade-in` entrance. The top panel plots the
raw price; the bottom panel plots the Roofing filter with a
dashed zero centerline and sign-coloured markers. Every node
carries a `data-section` attribute.

## Distinctness

- **vs chart-line-supersmoother** -- the Super Smoother is a
  single low-pass stage that keeps the trend. The Roofing filter
  prepends a high-pass, so its output is *detrended* -- a
  band-pass oscillator, not a smoothed price.
- **vs the moving-average overlays** -- those produce a smoothed
  price that shares the price axis. The Roofing filter is
  zero-centred and lives on its own sub-panel.
- **vs chart-line-laguerre** -- the Laguerre filter is one
  low-pass cascade. The Roofing filter explicitly combines a
  high-pass and a low-pass to pass only a band.

# patch 11.153 -- UI HScroll horizontal-list primitive (v1.11.171)

## Primitive

`web/src/components/ui/h-scroll.tsx` -- single-row flex container that
horizontally scrolls its children. CSS scroll-snap (`snap-x snap-mandatory`
on the row, `snap-start` on children carrying `data-h-scroll-item`) is on
by default and can be disabled with `snap={false}`. Hover-revealed left /
right arrow buttons (`arrows` prop, default off) toggle their visibility
via `data-visible` based on `scrollLeft` and
`scrollLeft + clientWidth < scrollWidth`, recomputed on `scroll`,
`resize`, and ResizeObserver firings; each arrow click triggers
`scrollBy({ left: clientWidth * 0.8, behavior: 'smooth' })`. Gap prop
maps `sm | md | lg` to `gap-2 / gap-3 / gap-4`.

Imperative ref handle `HScrollHandle` exposes:

- `scrollToIndex(index, opts?)` -- calls `scrollIntoView({ behavior:
  'smooth', block: 'nearest', inline: 'center', ...opts })` on the
  `index`-th descendant marked `data-h-scroll-item`.
- `scrollToEl(el, opts?)` -- same shape on the provided element.

The forwarded ref is the imperative handle, not the underlying DOM div.

Exports added to `web/src/components/ui/index.ts` (`HScroll` value +
`HScrollHandle` type). No existing primitives modified.

## Adoption sites (3)

1. **`web/src/components/WorkerList.tsx`** -- worker-card badges row
   (unread snapshots + intervention reason). Previously
   `flex flex-wrap items-center gap-1.5`; now an `HScroll` (snap off,
   gap sm) so a narrow sidebar lane keeps badges on one line and
   horizontally scrolls instead of wrapping into multiple rows.
2. **`web/src/components/SpecialistsTagEditor.tsx`** -- tag chip
   display (the `#tag` Chips shown when the editor is closed). Was
   `flex flex-wrap`; now scrolls horizontally so the chip row stays a
   single line regardless of tag count. The empty-state branch stays
   on its own non-scrolling row.
3. **`web/src/components/HistoryDetailPane.tsx`** -- per-record
   metadata row inside the right-pane timeline (branch + started-at +
   completed-at). Was `flex flex-wrap`; now scrolls horizontally so the
   row never reflows even when the branch name is long.

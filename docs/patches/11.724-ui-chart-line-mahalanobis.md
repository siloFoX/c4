## Summary

`web/src/components/ui/chart-line-mahalanobis.tsx` ships
`<ChartLineMahalanobis>`, a pure-SVG two-panel chart with a
**Mahalanobis Distance** panel from the multivariate standardized
deviation of each bar's bivariate feature vector off the rolling
mean. The price line sits on top; the distance panel below carries
a dashed horizontal outlier threshold.

## Computation

Each bar carries a bivariate feature vector
`f = [value, return]` where `return = value[i] - value[i-1]`; the
first bar's return is null. For a rolling window of `window` features
ending at bar i, the sample mean vector `mu` and the 2x2 sample
covariance matrix `Sigma` (with the `1/(window - 1)` normalization)
are formed:

```
mu      = (1/W) * sum f
Sigma   = (1/(W-1)) * sum (f - mu) (f - mu)^T
det     = Sigma_xx * Sigma_yy - Sigma_xy^2
inv     = (1/det) * [[ Sigma_yy, -Sigma_xy],
                    [-Sigma_xy,  Sigma_xx]]
d(i)    = sqrt( (f[i] - mu)^T * inv * (f[i] - mu) )
```

If `|det| < CHART_LINE_MAHALANOBIS_DET_EPSILON` (a constant series,
a linear ramp -- the rolling features are collinear) the distance is
left null because the inverse is undefined.

The distance is **invariant under any affine reparameterisation** of
the feature vector. Two operationally important cases:

- **Translation** -- shifting every input value by a constant `k`
  shifts `mu_value` by `k` while leaving the return feature alone,
  so `f - mu` is unchanged and `d` is unchanged.
- **Scale** -- multiplying every input value by a constant `c > 0`
  scales `mu` by `c`, `Sigma` by `c^2`, `inv` by `1/c^2`, and
  `f - mu` by `c`; the `c^2` factors cancel in `d^2`, so `d` is
  unchanged.

## Worked anchors

- **Constant series** -- the bivariate features `[value, return]` are
  all `[k, 0]`. The sample covariance has zero variance in the return
  dimension (`Sigma_yy = 0`), so `det = 0` and `d` is null. Asserted
  with `toBeNull` per defined bar.
- **Linear ramp** -- the bivariate features are all `[value, slope]`
  with a constant slope. Both `Sigma_yy = 0` and `Sigma_xy = 0`, so
  `det = 0` and `d` is null per defined bar.
- **Symmetric three-point anchor** -- the window
  `[[1, 0], [0, 1], [-1, -1]]` has mean at the origin and sample
  covariance `(1/2) * [[2, 1], [1, 2]] = [[1, 0.5], [0.5, 1]]` (the
  `1/(N-1) = 1/2` normalization). `det = 1 - 0.25 = 0.75`,
  `inv = (1/0.75) * [[1, -0.5], [-0.5, 1]] = [[4/3, -2/3], [-2/3, 4/3]]`.
  For the feature `[1, 0]` the diff is `[1, 0]`, so
  `d^2 = 1 * 4/3 + 0 + 0 = 4/3`, asserted to 10 places with
  `toBeCloseTo(Math.sqrt(4/3), 10)`.
- **Zero-distance** -- a feature exactly at the rolling mean of a
  non-degenerate window gives `d = 0` (within ULP).
- **Translation invariance** -- `computeLineMahalanobisRolling` on
  `VARIED` and on `VARIED + 1000` are asserted equal within ULP at
  every defined bar (`toBeCloseTo(a, 10)`). Also tested for a
  negative shift `-50`. The result is not bit-exact because
  floating-point summation isn't order-invariant under translation:
  `(a+k) + (b+k) + ...` rounds differently than `(a+b+...) + n*k`.
- **Scale invariance** -- `VARIED` and `VARIED * 2` are asserted
  equal within ULP at every defined bar.

## Distinct from neighbouring primitives

- vs `<ChartLineBayesian>` -- the Bayesian Trend produces three
  calibrated probabilities under three parametric hypotheses; the
  Mahalanobis Distance is an unsigned multivariate z-score whose
  classifier is a single threshold.
- vs `<ChartLineRsi>` / `<ChartLineCmo>` -- those are univariate
  shape statistics over the bar-to-bar returns; the Mahalanobis
  Distance is a TRUE multivariate measure that accounts for the
  COVARIANCE between value and return (a feature pair that drifts
  together is treated as one-dimensional).
- vs `<ChartLineMonteCarlo>` -- the Monte Carlo Fan is a forward-
  looking probabilistic forecast; the Mahalanobis Distance is a
  backward-looking outlier metric over the existing series.
- vs `<ChartLineErgodic>` / `<ChartLineSnr>` -- those are momentum /
  noise statistics with a sign; the Mahalanobis Distance is a
  non-negative outlier metric (`>= 0` by construction since
  `inv` is positive-semi-definite when `det > 0`).

## API

`data` -- a single `ChartLineMahalanobisPoint[]` (`{x, value}`).
`window` (default 20, integer >= 3). `threshold` (default 2, strictly
positive finite). Standard sizing / colour / a11y props plus
`pricePanelRatio` (default 0.55, clamped to `[0.3, 0.8]`), `gap`,
`strokeWidth`, `dotRadius`, `tickCount`. Toggle flags: `showAxis`,
`showGrid`, `showDots` (default false), `showDistance`,
`showThreshold`, `showMarkers`, `showTooltip`, `showConfigBadge`,
`showLegend`, `animate`. Controlled and uncontrolled visibility via
`hiddenSeries` (`'price' | 'distance'`). Callbacks:
`onPointClick({point})`, `onSeriesToggle({seriesId, hidden})`.
Format functions: `formatValue`, `formatX`. ARIA: `ariaLabel`,
`ariaDescription`.

## Pure helpers

- `DEFAULT_CHART_LINE_MAHALANOBIS_*` -- defaults.
- `CHART_LINE_MAHALANOBIS_DET_EPSILON` -- the singular-det guard
  (1e-12).
- `getLineMahalanobisFinitePoints` -- keep only points with a finite
  x and value.
- `normalizeLineMahalanobisWindow(window, fallback)` -- integer >= 3;
  else fallback.
- `normalizeLineMahalanobisThreshold(threshold, fallback)` -- finite
  > 0; else fallback.
- `computeLineMahalanobisFeatures(values)` -- bivariate
  `[value, return]` features; the first bar's feature is null; a
  non-finite bar yields null.
- `computeLineMahalanobisStats(features)` --
  `{muValue, muRet, varValue, varRet, covValueRet, det}` from the
  sample mean and the `1/(N-1)` covariance.
- `computeLineMahalanobisDistance(feature, stats)` -- the scalar
  distance; null when `|det| < EPS`.
- `computeLineMahalanobisRolling(values, window)` -- the per-bar
  rolling distance; warm-up bars (i < window) are null.
- `classifyLineMahalanobisZone(distance, threshold)` --
  `'normal' | 'outlier' | 'none'`; the distance at the threshold
  is treated as an outlier.
- `runLineMahalanobis(points, options?)` --
  `{series, window, threshold, distance, samples, distanceFinal,
  normalCount, outlierCount, ok}`.
- `computeLineMahalanobisLayout({data, window, threshold, ...})` --
  two panels with the price line on top, the distance line below,
  one marker per defined-distance bar, a dashed threshold line, and
  `thresholdY` for the threshold line.
- `describeLineMahalanobisChart(data, options?)` -- ARIA summary
  mentioning the indicator, the bivariate feature pair, the rolling
  window, the singular-covariance behaviour and the final distance.

## a11y / data hooks

SVG is `role="img"` with the same label; an sr-only
`data-section="chart-line-mahalanobis-aria-desc"` span carries the
description. The price line, every price dot, the distance line and
every marker carry `role="graphics-symbol"` and `tabIndex={0}`. Every
node carries a `data-section`. Root mirrors `data-window`,
`data-threshold`, `data-distance-final`, `data-normal-count`,
`data-outlier-count`, `data-total-points`, `data-animate`,
`data-empty`. Grid / axis / tick / panel-label nodes carry
`data-panel="price"|"distance"`; markers expose `data-zone` and
`data-distance`.

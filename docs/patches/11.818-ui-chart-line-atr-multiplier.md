# 11.818 UI: chart-line-atr-multiplier primitive (v1.11.836)

## Summary
Pure-SVG dual-panel React/TS primitive plotting a **normalized
volatility ratio** beneath the close. Each bar divides the rolling
close-to-close range by the ATR over the lookback -- a unitless
measure of how much of typical bar-by-bar movement is being absorbed
into directional travel. High ratios indicate trending regimes; low
ratios indicate chop.

## Formula
```
closeRange[i] = max(close[i - length + 1 .. i])
                - min(close[i - length + 1 .. i])
tr[i]         = max(high - low, |high - prevClose|, |low - prevClose|)
atr[i]        = SMA(tr, atrLength)[i]
ratio[i]      = atr[i] > 0 ? closeRange[i] / atr[i] : null
```
`ratio[i]` is `null` during warmup or when the ATR is zero
(divide-by-zero guard).

## Defaults
- `length = 14`, `atrLength = 14`
- `highThreshold = 1.5`, `lowThreshold = 0.5`
- `width = 720`, `height = 460`, `panelGap = 12`

## Zones and crosses
- `high`: `ratio >= highThreshold` (trending)
- `normal`: in between
- `low`: `ratio < lowThreshold` (choppy)
- `flat`: `ratio == 0`
- `none`: warmup / non-finite
- `crossed`: `'up'` when prev `< highThreshold` and current `>=
  highThreshold`; `'down'` when prev `>= lowThreshold` and current `<
  lowThreshold`. Markers fire only on threshold transitions.

## Bit-exact testability anchors
- **CONST h=l=close=K**: `closeRange = 0`, `tr = 0`, `atr = 0` ->
  divide-by-zero guard returns `ratio = null` everywhere. Verified
  across `K in {0, 1, 5, 100, -3}` and `length in {3, 4, 7, 10}`.
- **LINEAR close=k+1, high=low=close**: `tr[0] = 0`, `tr[i >= 1] = 1`.
  After ATR warmup (i.e. once `tr[0]` falls out of the window),
  `atr = 1` exactly. Rolling `closeRange` over a unit-step sequence
  of `length` bars is exactly `length - 1`. So `ratio = (length - 1)
  / 1 = length - 1` (integer-exact). Verified for `length in {4, 5,
  7, 10}`.

## Design notes
- The LINEAR anchor exploits two properties: per-bar `tr` is exactly
  `1` for a unit-step monotonic close (after the first bar), and the
  rolling max-min of consecutive integers is exactly `length - 1`.
  Both are integer-exact in IEEE 754 within `MAX_SAFE_INTEGER`.
- Ratio panel y-axis is `0`-based and includes the `highThreshold *
  1.25` cushion so the high-zone reference stays visible even for
  all-low ratios.
- Threshold lines dashed `#9333ea` (purple), consistent with other
  oscillators in this push.
- Default colors: ratio line `#0e7490` (teal), bullish `#16a34a`,
  bearish `#dc2626`, thresholds `#9333ea`.
- `noUncheckedIndexedAccess` satisfied via explicit `?? null` and
  defensive guards.

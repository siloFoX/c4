## Summary

Add a new `<ChartWaterfall>` UI primitive in
`web/src/components/ui/chart-waterfall.tsx` -- pure-SVG
waterfall bar chart. Each `delta` bar shows a positive
or negative change relative to the running total; `total`
bars draw from zero to the cumulative running total
(starting balance, subtotals, final). Thin connector lines
link the top of each bar to the bottom of the next so
adopters can read the cumulative flow at a glance.
Positive deltas paint green, negative red, totals neutral
by default; all colours overridable. Per-bar tooltip shows
delta + running total; per-bar click handler.

## API

```ts
type ChartWaterfallBarType = 'delta' | 'total';

interface ChartWaterfallBar {
  id: string;
  label: string;
  value: number;             // signed delta or absolute total
  type?: ChartWaterfallBarType;  // default 'delta'
  color?: string;
}

interface ChartWaterfallProps {
  bars: readonly ChartWaterfallBar[];
  width?: number;                    // default 480
  height?: number;                   // default 280
  padding?: number;                  // default 36
  showLabels?: boolean;              // default true
  showValues?: boolean;              // default true
  showConnectors?: boolean;          // default true
  showTooltip?: boolean;             // default true
  showAxisTicks?: boolean;           // default true
  showZeroLine?: boolean;            // default true
  animate?: boolean;                 // default true
  className?: string;
  ariaLabel?: string;                // default 'Waterfall chart'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  positiveColor?: string;            // default '#16a34a'
  negativeColor?: string;            // default '#dc2626'
  totalColor?: string;               // default '#475569'
  tickCount?: number;                // default 5
  barGap?: number;                   // default 6
  onBarClick?: (args: {
    bar: ChartWaterfallBar;
    index: number;
    start: number;
    end: number;
  }) => void;
}
```

## Behaviour

- Renders a `<div role="region">` wrapping `<svg
  role="img">` carrying the same `aria-label`.
- `computeWaterfallSteps` walks the bar list and emits a
  per-bar `{ start, end, delta, isTotal, runningTotal }`
  record. Delta bars increment the running total by
  `value`; total bars draw from zero to `value` and
  reset the running total to that value so subsequent
  deltas stack on top.
- Vertical scale: `getWaterfallBounds` returns a
  chart-wide min / max that always includes zero. The
  scale auto-expands for collapsed (all-zero) data so
  the chart never collapses to a flat line.
- Bar geometry: bar width =
  `(innerWidth - barGap * (n - 1)) / n` (min 1 px).
  Vertical extent =
  `|yFor(end) - yFor(start)|` (min 1 px so a
  zero-delta bar still renders as a hairline).
- Colour: per-bar `color` wins; otherwise
  `total` -> `totalColor`, positive delta ->
  `positiveColor`, negative delta -> `negativeColor`,
  zero delta -> `totalColor` (neutral).
- Connectors: thin dashed line between the top of bar N
  and the start of bar N+1, anchored at the running
  total. Opt-out via `showConnectors=false`.
- Zero line: rendered only when the chart spans both
  signs (min < 0 < max). Opt-out via
  `showZeroLine=false`.
- Hover / focus: enlarges the bar (stroke + full
  opacity) and opens a tooltip with the bar label,
  signed delta (or absolute total), and running total
  (for delta bars only).
- `motion-safe:animate-fade-in` per bar group on mount.
  `animate=false` removes the class.
- Default ARIA description auto-summary lists every
  bar with its formatted delta + running total:
  `"Waterfall with N bars. <label> total <value>,
  <label> +<delta> -> <total>, ..."`.
  `ariaDescription` overrides.

## Pure helpers (exported)

```ts
DEFAULT_CHART_WATERFALL_WIDTH          = 480
DEFAULT_CHART_WATERFALL_HEIGHT         = 280
DEFAULT_CHART_WATERFALL_PADDING        = 36
DEFAULT_CHART_WATERFALL_TICK_COUNT     = 5
DEFAULT_CHART_WATERFALL_POSITIVE_COLOR = '#16a34a'
DEFAULT_CHART_WATERFALL_NEGATIVE_COLOR = '#dc2626'
DEFAULT_CHART_WATERFALL_TOTAL_COLOR    = '#475569'
DEFAULT_CHART_WATERFALL_BAR_GAP        = 6

computeWaterfallSteps(bars): WaterfallStep[]
getWaterfallBounds(steps): { min, max }
getWaterfallBarColor(step, positive, negative, total): string
getWaterfallTicks(min, max, count?): number[]
describeWaterfallChart(bars, formatValue?): string
```

- `computeWaterfallSteps` is the canonical layout pass.
  Delta values fall back to 0 when non-finite;
  total values fall back to the current running
  total when non-finite.
- `getWaterfallBounds` falls back to `(0, 1)` for
  empty input and expands a collapsed (min == max)
  range so the chart still renders.
- `getWaterfallBarColor` mirrors the component's
  colour-decision logic so adopters can paint their
  own SVG overlay using the same palette.
- `getWaterfallTicks` returns at least `count` evenly-
  spaced ticks (min 2); `[min]` for collapsed ranges.
- `describeWaterfallChart` produces the default ARIA
  summary; honours `formatValue`.

## ARIA + data attributes

- Root: `<div role="region">` + `aria-label`.
- SVG: `role="img"` + same `aria-label`.
- sr-only span:
  `data-section="chart-waterfall-aria-desc"` carries the
  auto-summary or `ariaDescription` override.
- Each bar rect: `role="graphics-symbol"` +
  `tabIndex={0}` so keyboard focus opens the tooltip.
- `data-section` on every node:
  - `chart-waterfall` (root)
  - `chart-waterfall-aria-desc`
  - `chart-waterfall-svg`
  - `chart-waterfall-axis-y`

  - `chart-waterfall-zero-line`
  - `chart-waterfall-connector`
  - `chart-waterfall-bar` (per-bar group)
  - `chart-waterfall-rect`
  - `chart-waterfall-label`
  - `chart-waterfall-value`
  - `chart-waterfall-tooltip` + `-tooltip-label` /

## Out of scope

- Stacked / grouped waterfall (host stacks instances).
- Animated value transitions on data change (snap v1;
  mount fade only).
- Horizontal waterfall orientation (vertical-bar v1;
  host wraps with custom transform when needed).

  represents them as additional delta bars).

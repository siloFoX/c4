# 11.1044 -- UI: chart-line-supertrend-divergence-cross primitive (v1.11.1062)

## Scope

Add a pure-SVG dual-panel `ChartLineSupertrendDivergenceCross`
component that renders the close in the top panel and the
Supertrend line in the bottom panel, flagging price-vs-
supertrend direction-disagreement events as trend-stop
reversal warnings with bias coloring derived from the
supertrend slope at the trigger bar. Uses the canonical
5-state divergence regime model (aligned-bullish /
aligned-bearish / divergent-bullish / divergent-bearish
/ none).

Supertrend is Olivier Seban's 2007 trailing-stop volatility
envelope: an ATR-based band system that "ratchets" -- the
supertrend line only moves in the direction of the trend
(up in an uptrend, down in a downtrend) and flips bands
when close breaches the active band. Among the
divergence-cross primitives, supertrend is uniquely
stair-step rather than smooth: it stays flat for many bars,
then snaps sharply when the trend flips.

## Supertrend divergence pipeline

```
hl2[i]       = (high[i] + low[i]) / 2
tr[i]        = max(high[i] - low[i],
                   |high[i] - close[i-1]|,
                   |low[i]  - close[i-1]|)
atr[i]       = Wilder(tr, period)
                ; first valid: SMA(tr, period)
                ; subsequent: (prevAtr * (period - 1) +
                  tr[i]) / period
upperBand[i] = hl2[i] + multiplier * atr[i]
lowerBand[i] = hl2[i] - multiplier * atr[i]
finalUpper[i] = upperBand[i] when
                upperBand[i] < finalUpper[i-1] OR
                close[i-1]   > finalUpper[i-1]
                else finalUpper[i-1]
finalLower[i] = lowerBand[i] when
                lowerBand[i] > finalLower[i-1] OR
                close[i-1]   < finalLower[i-1]
                else finalLower[i-1]
trend[i]     = up-trend  -> down-trend when
                close[i] < finalLower[i]
               down-trend -> up-trend when
                close[i] > finalUpper[i]
               otherwise unchanged
supertrend[i] = trend[i] === 'up'  -> finalLower[i]
                trend[i] === 'down' -> finalUpper[i]

priceUp     : close[i] > close[i-1]
priceDown   : close[i] < close[i-1]
stUp        : supertrend[i] > supertrend[i-1]
stDown      : supertrend[i] < supertrend[i-1]

regime ->
  'aligned-bullish'   when priceUp   && stUp
  'aligned-bearish'   when priceDown && stDown
  'divergent-bullish' when priceDown && stUp
  'divergent-bearish' when priceUp   && stDown
  'none'              when either side flat or null

bullish (divergence trigger up) :
  prev regime !== 'divergent-bullish' &&
  cur regime === 'divergent-bullish'
bearish (divergence trigger down) :
  prev regime !== 'divergent-bearish' &&
  cur regime === 'divergent-bearish'

bias        : supertrend[i] vs supertrend[i-1] -> up /
              down / flat / none
```

Default `period = 10`, `multiplier = 3` -- Olivier Seban's
canonical 2007 tuning. Initial trend direction at `i =
period` is detected from `sign(close[period] -
close[period - 1])`: positive -> uptrend (supertrend =
lowerBand), negative -> downtrend (supertrend = upperBand),
zero -> uptrend by convention.

Warmup is `period = 10` for the default tuning. Divergence
detection requires the previous supertrend + close, so
the first regime classification lands at i = period + 1
= 11.

## Bit-exact anchors

All anchors use HLC input.

- **CONST band** `high = K + 1`, `low = K - 1`, `close
  = K`: hl2 = K, range = 2, |high - prevClose| = 1, |low
  - prevClose| = 1 -> tr = max(2, 1, 1) = 2. atr = 2
  (from SMA + Wilder recurrence). multiplier * atr = 6.
  upperBand = K + 6, lowerBand = K - 6. Initial direction
  from close[period] - close[period - 1] = 0 -> uptrend.
  supertrend = lowerBand = K - 6 flat throughout. close
  = K stays between bands, never flips. Both close and
  supertrend flat -> regime `none`. 0 divergence crosses.
  Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close
  = i`: hl2 = i, range = 2, |high - prevClose| = 2, |low
  - prevClose| = 0 -> tr = max(2, 2, 0) = 2. atr = 2.
  bands = i +/- 6. Initial direction from close[period]
  - close[period - 1] = +1 -> uptrend. supertrend =
  lowerBand = i - 6, rising at +1 per bar. lowerBand[i]
  > finalLower[i-1] holds for every i, so finalLower
  follows lowerBand exactly (the ratchet never blocks).
  Both close and supertrend rising -> regime
  `aligned-bullish`. 0 divergence crosses.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`,
  `close = -i`: mirror -> hl2 = -i, tr = max(2, 0, 2) =
  2, atr = 2. Initial direction from close[period] -
  close[period - 1] = -1 -> downtrend. supertrend =
  upperBand = -i + 6, falling at -1 per bar. Both close
  and supertrend falling -> regime `aligned-bearish`.
  0 divergence crosses.

The sign-of-delta initial trend detection is what makes
the LINEAR anchors fire 0 crosses -- without it, both
LINEAR UP and LINEAR DOWN would experience a band flip
6 bars after warmup (when close first crosses the
initial band), producing a spurious divergent event. By
detecting initial direction from the close trajectory,
the supertrend starts on the correct side of close from
the first valid bar.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 5-state regime classifier follows the
  chart-line-*-divergence-cross family convention; the
  `bullish` / `bearish` cross *kinds* still follow the
  sibling cross-sig convention (line direction).
- Supertrend is stair-step rather than smooth -- in a
  trend, supertrend can stay flat for many bars before
  flipping. Such flat plateaus produce `bias = 'flat'`
  and `regime = 'none'`, not aligned regimes. This
  faithfully reflects the trailing-stop semantic: when
  the supertrend is flat, the trend has no fresh
  confirmation.
- Trend-flip events produce sharp opposite-direction
  snaps (e.g. from upperBand to lowerBand or vice
  versa). On the flip bar, supertrend changes direction
  opposite to price, which triggers a divergent regime.
  This is the intended trend-stop reversal warning --
  the supertrend has officially flipped, but price is
  still moving in the original direction.
- Initial direction detection uses
  `sign(close[period] - close[period - 1])`. Zero
  delta defaults to uptrend by convention -- which
  works correctly for the CONST anchor since the close
  is flat and supertrend stays at lowerBand =
  K - 6 forever (no flip is possible because close
  never breaches either band).
- ATR uses Wilder-style smoothing: first ATR is SMA of
  the first `period` TR values, then `(prevAtr *
  (period - 1) + tr[i]) / period` for subsequent bars.
  Different from the SMA-style ATR used in
  `chart-line-atr-breakout-cross` (v1.11.1048) -- this
  matches the canonical Seban supertrend formulation.
- HLC input boundary filter requires `high >= low` --
  malformed bars are dropped at the input boundary.

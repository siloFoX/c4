## Summary

Add a new `<ChartFunnel>` UI primitive in
`web/src/components/ui/chart-funnel.tsx` -- pure-SVG
conversion funnel. Each stage paints as a trapezoid whose
width is proportional to its value, narrowing toward the
bottom for the classic funnel silhouette. Drop-off labels
between consecutive stages quantify users lost; hover opens
a tooltip with the stage value, % of top stage, and % from
previous stage. Default colour gradient flows from a light
tint at the top to a darker brand colour at the bottom.

## API

```ts
interface ChartFunnelStage {
  id: string;
  label: string;
  value: number;
  color?: string;
}

interface ChartFunnelProps {
  stages: readonly ChartFunnelStage[];
  width?: number;              // default 360
  height?: number;             // default 320
  stageGap?: number;           // default 4
  showDropoff?: boolean;       // default true
  showValues?: boolean;        // default true
  showLabels?: boolean;        // default true
  showTooltip?: boolean;       // default true
  animate?: boolean;           // default true
  className?: string;
  ariaLabel?: string;          // default 'Funnel chart'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  formatPercent?: (v: number) => string;
  onStageClick?: (args: {
    stage: ChartFunnelStage;
    index: number;
    dropoff: number;
    conversion: number;
    ofTop: number;
  }) => void;
  gradient?: { from?: string; to?: string };
}
```

## Behaviour

- Renders a `<div role="region">` wrapping a `<svg
  role="img">`. Both carry the same `aria-label` so screen
  readers find both the region and the chart-as-image.
- Geometry: each stage's slice width is
  `(width - 16) * value / max` at its top. The bottom of
  the slice continues into the next stage's top width to
  produce the trapezoid -- the final stage tapers to
  half its top width by default so the visual still
  funnels.
- Default colour gradient: stage colour is interpolated
  between `gradient.from` (default `#bfdbfe`, a light blue
  tint) and `gradient.to` (default `#1d4ed8`, a deep brand
  blue) across stage indices. `stage.color` overrides the
  interpolated colour for any individual stage.
- Labels: stage label centred inside the trapezoid (white,
  size 12, bold). Stage value rendered just below the
  label. Both opt-out via `showLabels` / `showValues`.
- Drop-off: between consecutive stages the primitive
  renders a thin label above each stage past the first,
  reading `-<lost> (<drop%> drop)`. Hidden when
  `showDropoff=false` or when the drop is zero / negative.
- Hover / focus: the slice gets a stroke + full opacity
  and opens a tooltip with stage label + formatted value
  + % of top + % from previous (omitted on the first
  stage). `onStageClick` fires with the full payload.
- Mount animation: `motion-safe:animate-fade-in` per
  stage group. `animate=false` removes the class.
- Default ARIA description: `"Funnel with N stages.
  <stage> <value> (<ofTop%> of top). ..."` -- a custom
  `ariaDescription` overrides it; empty stages produce
  `"No data"`.

## Pure helpers (exported)

```ts
DEFAULT_CHART_FUNNEL_WIDTH         = 360
DEFAULT_CHART_FUNNEL_HEIGHT        = 320
DEFAULT_CHART_FUNNEL_STAGE_GAP     = 4
DEFAULT_CHART_FUNNEL_GRADIENT_FROM = '#bfdbfe'
DEFAULT_CHART_FUNNEL_GRADIENT_TO   = '#1d4ed8'

getFunnelMax(stages): number
getFunnelRatio(value, max): number
getStageDropoff(prevValue, currValue): number
getStageConversion(prevValue, currValue): number
getStageOfTop(topValue, value): number
interpolateColor(from, to, t): string
buildFunnelPath(topW, bottomW, top, height, cx): string
describeFunnelChart(stages, formatValue?, formatPercent?): string
```

- `getFunnelMax` returns the largest stage value; falls
  back to 1 when nothing finite is present so the chart
  still renders.
- `getFunnelRatio` clamps `value / max` into `[0, 1]`;
  non-finite, non-positive, or zero-max inputs collapse
  to 0.
- `getStageDropoff` returns `prev - curr` when both are
  positive; first stage (no prev) returns 0; `curr >
  prev` returns 0 (no negative drop-offs).
- `getStageConversion` returns `curr / prev` in `[0, 1]`;
  first stage returns 1 (100% retained); guards against
  zero / non-finite inputs returning 0.
- `getStageOfTop` returns `value / topValue` in
  `[0, 1]`; zero/non-finite top returns 0.
- `interpolateColor` linearly interpolates two hex
  colours (#rgb shorthand expanded). Parse failures fall
  back to the `from` colour so the gradient is always
  defined. `t` is clamped into `[0, 1]`.
- `buildFunnelPath` emits a closed quadrilateral SVG
  path (one trapezoid). Negative widths clamp to zero so
  the path stays valid.
- `describeFunnelChart` produces the default ARIA
  summary and honours `formatValue` + `formatPercent`.

## ARIA + data attributes

- Root: `<div role="region">` + `aria-label`.
- SVG: `role="img"` + same `aria-label`.
- sr-only span: `data-section="chart-funnel-aria-desc"`
  carries the auto-summary or `ariaDescription` override.
- Each slice path: `role="graphics-symbol"` +
  `tabIndex={0}` so keyboard focus opens the tooltip.
- `data-section` on every node:
  - `chart-funnel` (root)
  - `chart-funnel-aria-desc`
  - `chart-funnel-svg`
  - `chart-funnel-stage` (per-stage group)
  - `chart-funnel-slice` (per-stage trapezoid path)
  - `chart-funnel-label` / `chart-funnel-value`
  - `chart-funnel-dropoff`
  - `chart-funnel-tooltip` + `-label` / `-value` /
    `-of-top` / `-conversion`
- Root mirrors `data-stage-count`, `data-animate`.
  Stage groups mirror `data-stage-id`, `data-stage-index`,
  `data-stage-color`, `data-hovered`. Drop-off text
  mirrors `data-dropoff` + `data-conversion`.

## Tests (60 cases)

- `getFunnelMax`: max of values, empty + non-finite -> 1.
- `getFunnelRatio`: linear, over-max clamp, non-finite /
  non-positive guards.
- `getStageDropoff`: positive drop, first-stage zero,
  curr > prev guard, non-finite / non-positive prev,
  non-finite curr.
- `getStageConversion`: first-stage = 1, normal flow,
  over-1 clamp, non-positive guards.

  non-finite / non-positive value, over-top clamp.

  negative widths clamp.

  formatter shaping.

## Summary

pivot.

## Indicator math

A W.D. Gann fan is a set of nine straight rays projected from a
pivot point at fixed price-time ratios:

```
1x8, 1x4, 1x3, 1x2,  1x1,  2x1, 3x1, 4x1, 8x1
```

The notation `AxB` is A price units per B time units; `ratio =
priceUnits / timeUnits`. The central `1x1` ray is the 45-degree
reference of the fan. Each ray's slope is
`ratio * pricePerUnit` price per one x-unit, where
`pricePerUnit` is the price rise of the 1x1 ray over one time
unit. The slope is positive for an up fan (projected from a
swing low) and negative for a down fan (from a swing high). A
ray's value at a coordinate `x` is
`pivotValue + slope * (x - pivotX)`.

### Worked fixture

Pivot at index 0 (`x 0`, `value 10`), `pricePerUnit 3`, up fan.
The slopes are `ratio * 3`:

```
1x8 0.375  1x4 0.75  1x3 1  1x2 1.5  1x1 3
2x1 6  3x1 9  4x1 12  8x1 24
```

Over the span to the last bar (`x 8`) each ray's endpoint value
is `10 + slope * 8`:

```
1x8 13  1x4 16  1x3 18  1x2 22  1x1 34
2x1 58  3x1 82  4x1 106  8x1 202
```

The 1x1 ray's value at `x` is `10 + 3x`. The price fixture
`[10,14,13,18,25,24,31,30,40]` straddles it -- 4 bars above,
4 below, 1 on (the pivot itself always sits on the 1x1).

## API

- `GANN_RATIOS` -- the nine `{ id, label, priceUnits, timeUnits }`
  ratio specs in fan order.
- `getLineGannFinitePoints(points)` -- drop price points with a
  non-finite `x` or `value`.
- `normalizeLineGannPivotIndex(index, count)` -- clamp a pivot
  index to `[0, count-1]`; non-finite or negative falls back to
  0, past-the-end clamps to the last point, fractional floors.
- `computeLineGannRays(options?)` -- the nine rays for a
  `pricePerUnit` and `direction`. Each ray carries the `ratio`,
  the signed `slope`, and `isPrimary` (true only for `1x1`).
- `runLineGann(points, options?)` -- sort finite points by `x`,
  resolve the pivot from `options.pivotIndex`, build the rays,
  return `{ series, pivotIndex, pivot, pricePerUnit, direction,
  rays, samples, aboveCount, belowCount, ok }`. Each sample is
  classified `above` / `below` / `on` the 1x1 ray. `ok` is false
  for fewer than two points.
- `computeLineGannLayout(options)` -- project the price path
  plus one line per ray, from the pivot to the chart's right
  edge.
- `describeLineGannChart(data, options?)` -- ARIA description.

## Component

`ChartLineGann` is a `forwardRef` region (`role="region"`) with
an ARIA-linked description, a `role="img"` SVG, axis ticks and
grid, a config badge, a three-series legend (Price / Gann fan /
Pivot) with toggle, a hover/focus tooltip on the price dots, and
a `motion-safe:animate-fade-in` entrance. It consumes
`{ x, value }` points plus a `pivotIndex`, `pricePerUnit` and
`direction`. The 1x1 ray is drawn solid and emphasized; the
other eight rays are dashed; the pivot is a diamond marker. The
rays are clipped to the panel with an SVG `clipPath`, so a steep
ray leaving the panel does not stretch the price axis. Every
node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-pivot-points** -- pivot points are seven
  *horizontal* support/resistance levels. The Gann fan is a set
  of *sloped rays* radiating from one anchor point at fixed
  angular ratios.
- **vs the moving-average overlays** -- those derive a
  continuous line from a sliding window of the price series. The
  Gann rays are static straight lines determined entirely by the
  pivot, the price-per-unit scale and the fixed ratio set.
- **vs chart-line-fractal** -- the fractal markers are discrete
  per-bar swing detections; the Gann fan is a fixed geometric
  construction projected forward from a single chosen pivot.

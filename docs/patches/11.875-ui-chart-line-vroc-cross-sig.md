## Summary

Added `<ChartLineVrocCrossSig />` -- a pure-SVG, dual-panel
React/TS primitive that renders the volume series in the top
panel and overlays the Volume Rate of Change (VROC) with its
EMA-smoothed signal line in the bottom panel, marking bullish
and bearish crossover events. This separates the *trigger
events* from the *raw VROC line* that 11.869 published as a
single-line oscillator:

```
VROC[i]    = volume[i - length] === 0
               ? null
               : (volume[i] - volume[i - length]) / volume[i - length] * 100
signal[i]  = EMA(VROC, signalLength)
bullish    : (VROC - signal) crosses up   (prev <= 0, cur > 0)
bearish    : (VROC - signal) crosses down (prev >= 0, cur < 0)
```

Defaults: `length = 12`, `signalLength = 9`. Regime classifier:
`bullish` (VROC > signal), `bearish` (VROC < signal),
`neutral` (VROC === signal), `none` (either side null).

## Bit-exact anchor

- **CONST volume = V (V > 0)**: VROC = (V - V) / V * 100 = 0 for
  every bar after warmup. Signal EMA of zeros stays at 0. So
  VROC === signal everywhere -> regime `neutral`, cross count
  = 0. Verified across multiple V tuples in the integration
  block.
- **CONST V = 0**: divide-by-zero guard yields VROC = null,
  hence signal = null, regime `none`, cross count = 0.

## Difference vs chart-line-vroc-pct (11.869)

| primitive | bottom panel | use-case |
| --- | --- | --- |
| `chart-line-vroc-pct` (11.869) | single VROC line | raw oscillator level |
| `chart-line-vroc-cross-sig` (11.875) | VROC + EMA signal + crosses | trigger events on the VROC |

11.869 publishes the raw VROC magnitude as a single line
oscillator. 11.875 adds an EMA-smoothed signal line on top and
extracts the bullish / bearish crossover events so they can be
acted on as discrete trigger points.

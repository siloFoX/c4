## Summary

New `ChartLineUlcer` UI primitive at
`web/src/components/ui/chart-line-ulcer.tsx` -- a pure-SVG
two-panel chart with an Ulcer Index panel from the root mean
square of drawdowns off the running peak.

## Indicator math

Peter Martin's Ulcer Index measures downside risk -- not just the
depth of a fall but how long it drags on.

`computeLineUlcerDrawdown(closes)` tracks the running peak (the
highest close from the start up to each bar) and the percentage
drawdown of the close below it:

```
drawdown[i] = 100 * (runningPeak - close) / runningPeak
```

It is zero at a new running high and grows as the close falls
away from the peak.

`computeLineUlcer(closes, period)` is the Ulcer Index -- the root
mean square of those drawdowns over the trailing `period` bars:

```
UI[i] = sqrt( mean of drawdown[k]^2 over the window )
```

A single deep drawdown raises the index, but so does a shallow
drawdown that persists, because every bar in the window
contributes its square. The chart classifies each bar against a
`threshold`: an Ulcer Index above the threshold is `stress`, at
or below is `calm`.

### Exactness

The Ulcer Index uses `sqrt`, but a window whose mean squared
drawdown is a perfect square is exact. The fixture holds the
running peak at 100 -- so each drawdown is `100 - close` -- and is
arranged so every window's mean squared drawdown is a perfect
square (0, 9, 25, 16): the Ulcer series `[., ., ., 0, 3, 5, 5,
5, 4, 0, 3, 5, 5]` is asserted with `toEqual`.

## API

- `getLineUlcerFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineUlcerPeriod(period, fallback)` -- coerce to a
  positive integer.
- `computeLineUlcerDrawdown(closes)` -- the percentage drawdown
  off the running peak.
- `computeLineUlcer(closes, period)` -- the Ulcer Index series.
- `runLineUlcer(points, options?)` -- sort finite points by `x`,
  run the pipeline, return `{ series, period, threshold,
  drawdown, ulcer, samples, ulcerFinal, stressCount, calmCount,
  ok }`. Each sample is classified `stress` / `calm` / `none`.
  `ok` is false for fewer than two points.
- `computeLineUlcerLayout(options)` -- stack the price panel
  above an Ulcer panel on a non-negative scale; project the
  price path, the Ulcer path, one marker per defined bar, and
  the threshold reference line.
- `describeLineUlcerChart(data, options?)` -- ARIA description.

## Component

`ChartLineUlcer` is a `forwardRef` region (`role="region"`) with
an ARIA-linked description, a `role="img"` SVG, two stacked
panels (price on top, Ulcer below) with their own axis ticks and
grid, panel labels, a config badge showing the period, a
three-series legend (Price / Ulcer / Threshold) with toggle, a
hover/focus tooltip exposing the drawdown, the Ulcer Index and
the zone, and a `motion-safe:animate-fade-in` entrance. It
consumes `{ x, value }` points. The Ulcer panel is a non-negative
scale, draws a dashed threshold reference line, and colours each
marker by its zone -- red for stress, green for calm. Every node
carries a `data-section` attribute.

## Distinctness

- **vs chart-line-stddev** -- a rolling standard deviation
  measures dispersion both up and down. The Ulcer Index counts
  only downside, the drawdown from the running peak.
- **vs chart-line-vix-fix** -- the VIX Fix is a single bar's
  drop from its recent peak. The Ulcer Index is the root mean
  square over a window, so a long shallow decline scores higher
  than one sharp dip.
- **vs a maximum-drawdown reading** -- that reports only the
  worst point. The Ulcer Index integrates the whole drawdown
  history in the window.

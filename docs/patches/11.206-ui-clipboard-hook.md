# 11.206 -- UI: copy-to-clipboard hook

`web/v1.11.224`. Component scope only.

## Summary

A new `useCopyToClipboard` hook centralises the
`navigator.clipboard.writeText` + reset-timer + SSR fallback pattern
that several lib hooks were re-implementing inline. Three inline
callsites are migrated; one (`use-copy-pulse.ts`) keeps its public
API and now delegates internally.

## Public API

```ts
useCopyToClipboard(resetMs?: number /* default 1500 */): {
  copy(text: string): Promise<{ ok: boolean; error: Error | null }>;
  copied: boolean;
  error: Error | null;
}
```

- `copy(text)` writes via `navigator.clipboard.writeText` when
  available. On success: sets `copied=true`, clears `error`,
  schedules a `setTimeout` that flips `copied=false` after
  `resetMs`. The returned promise resolves to
  `{ ok: true, error: null }`.
- On rejection (or fallback failure): `copied` stays `false`,
  `error` is populated with the captured `Error`, and the promise
  resolves to `{ ok: false, error }`. The hook does **not** re-throw
  -- callers branch on `result.ok` to drive their own UI feedback.
- When `navigator.clipboard` is unavailable (SSR, older browsers),
  the hook falls back to a temporary off-screen `<textarea>` +
  `document.execCommand('copy')`, wrapped in try / catch. Both the
  textarea insertion and the removal happen inside the same call,
  so a failing copy never leaves an orphan node in the DOM.
- The unmount effect clears the pulse timer so a long `resetMs`
  plus a quick navigation cannot fire `setCopied(false)` after the
  component is gone.

## Why a `{ ok, error }` return shape

`useCopyToClipboard` exposes `error` as React state for components
that render an inline error badge, but a callback that needs to
show a toast on the same tick the user clicks **cannot read the
new state value** until the next render. Returning `{ ok, error }`
from the awaited promise gives the caller (e.g. `use-morning.ts`'s
`showToast` flow) the rejection reason synchronously, without a
ref dance.

## Callsite adoption

`grep -rn 'navigator.clipboard.writeText' web/src/` enumerated
three production callsites (the other matches are test files /
matchers):

| File | Before | After |
|------|--------|-------|
| `lib/use-copy-pulse.ts` | inline `if (navigator.clipboard) writeText` + `setTimeout` | delegates to `useCopyToClipboard(durationMs)`; public `{ copied, copy }` shape unchanged |
| `lib/use-morning.ts` | inline `try { writeText } catch` -> `showToast` | `await copy(report.content)`; branches on `res.ok` to fire success / failure toast |
| `lib/use-action-items-export.ts` | silent `writeText(md).catch(() => {})` (no feedback) | `void copy(md)`; exposes new `copied: boolean` so `MeetingsActionItemsPanel` can render a `✓ Copied` pulse |

`MeetingsActionItemsPanel.tsx` flips its MD copy button label
between `⧉ MD` and `✓ Copied` based on the new `copied` flag and
also stamps a `data-copied="true"` attribute so a future visual
regression / e2e selector can latch on without scraping label
text. The mocked `useActionItemsExport` in
`MeetingsActionItemsPanel.test.tsx` returns the old shape -- the
component reads the new field via destructuring, so a missing
`copied` resolves to `undefined` which is falsy, leaving the
existing 28 tests byte-identical.

### Not converted

## Tests

Hook (`use-copy-to-clipboard.test.tsx`, 7 cases, vitest + RTL +
`vi.useFakeTimers`):

1. `copy()` calls `navigator.clipboard.writeText` with the given
   text and resolves `{ ok: true }`.
2. `copied` flips to `true` after a successful copy.
3. `copied` auto-resets to `false` exactly at `resetMs`
   (`vi.advanceTimersByTime(resetMs)`).
4. Rejection from `writeText` populates `error` with the
   captured `Error`; `copied` stays `false`.
5. Fallback path: when `navigator.clipboard` is undefined, the
   hook calls `document.execCommand('copy')` (stubbed) and
   reports success.
6. Fallback returning `false` is captured as an `Error` so the
   caller sees `{ ok: false, error: Error }`.
7. Unmount with a pending pulse timer is a no-op: advancing
   timers post-unmount does not throw.

## Summary

Add a new `<ChartRadialBar>` UI primitive in
`web/src/components/ui/chart-radial-bar.tsx` -- pure-SVG
concentric radial bar chart. Each series gets a dedicated
ring at its own radius; an arc inside the ring is drawn
proportional to the value (`0 -> max`). A background
"track" arc behind each bar provides completion context.
The center of the chart hosts an optional label slot for
an aggregate value or legend node. Hover / focus on an arc
opens a tooltip with the series label + value + % of max.
Per-series click handler.

## API

```ts
interface ChartRadialBarSeries {
  id: string;
  label: string;
  value: number;
  color?: string;
}

interface ChartRadialBarProps {
  series: readonly ChartRadialBarSeries[];
  width?: number;                  // default 320
  height?: number;                 // default 320
  maxValue?: number;
  startAngle?: number;             // default -90 (12 o'clock)
  endAngle?: number;               // default 270 (full circle)
  barWidth?: number;               // default 14
  barGap?: number;                 // default 4
  trackColor?: string;             // default '#e2e8f0'
  showCenterLabel?: boolean;       // default true
  centerLabel?: ReactNode;
  showLabels?: boolean;            // default true
  showValues?: boolean;            // default true
  showTooltip?: boolean;           // default true
  showAxisTicks?: boolean;         // default true
  tickCount?: number;              // default 4
  animate?: boolean;               // default true
  className?: string;
  ariaLabel?: string;              // default 'Radial bar chart'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  formatPercent?: (v: number) => string;
  onSeriesClick?: (args: {
    series: ChartRadialBarSeries;
    index: number;
    value: number;
    ratio: number;
  }) => void;
}
```

## Behaviour

- Renders a `<div role="region">` wrapping `<svg
  role="img">` carrying the same `aria-label`.
- Ring geometry: each series occupies `barWidth` of
  radial thickness; rings stack outward-to-inward so the
  first series sits at the outermost ring. `barGap`
  separates consecutive rings.
- Arc geometry: each arc sweeps from `startAngle` (in
  degrees, default `-90` = 12 o'clock) through
  `endAngle` (default `270`, completing a full circle).
  The filled arc spans `startAngle ... startAngle +
  (endAngle - startAngle) * ratio`. Background "track"
  arcs paint the full angular span in `trackColor` at
  reduced opacity so the unused portion is visible.
- Direction logic: positive values fill clockwise (sweep
  flag 1); a 0 / non-finite value collapses the value
  arc entirely (only the track renders) so the chart
  stays defined.
- Centre slot: when `centerLabel` is provided and
  `showCenterLabel=true`, a 128x64 `<foreignObject>` at
  the chart centre hosts the React node so adopters can
  show an aggregate value, mini-icon, or legend.
- Axis ticks: small radial tick marks just outside the
  outer ring at evenly-spaced angular positions
  (`getRadialTickPositions`). `tickCount` controls the
  granularity. `showAxisTicks=false` suppresses.
- Labels: per-series text label rendered along the
  starting edge of each ring (near `startAngle - 4`).
  `showLabels=false` suppresses; `showValues=false`
  drops the value suffix from the label.
- Hover / focus: highlights the arc (stroke + full
  opacity) and opens a tooltip with the series label,
  formatted value, and `% of max`. Tooltip suppressed
  via `showTooltip=false`.
- Mount animation: `motion-safe:animate-fade-in` per
  series group. `animate=false` removes the class.
- Default ARIA description: `"Radial bar chart with N
  series. <label> <value> (<pct>), ..."` --
  `ariaDescription` overrides; empty series produces
  `"No data"`.

## Pure helpers (exported)

```ts
DEFAULT_CHART_RADIAL_BAR_WIDTH        = 320
DEFAULT_CHART_RADIAL_BAR_HEIGHT       = 320
DEFAULT_CHART_RADIAL_BAR_START_ANGLE  = -90
DEFAULT_CHART_RADIAL_BAR_END_ANGLE    = 270
DEFAULT_CHART_RADIAL_BAR_BAR_WIDTH    = 14
DEFAULT_CHART_RADIAL_BAR_BAR_GAP      = 4
DEFAULT_CHART_RADIAL_BAR_TRACK_COLOR  = '#e2e8f0'
DEFAULT_CHART_RADIAL_BAR_TICK_COUNT   = 4

getRadialBarMax(series, override?): number
getRadialBarRatio(value, max): number
polarToCartesian(cx, cy, radius, angleDegrees): { x, y }
buildRadialArcPath(cx, cy, innerRadius, outerRadius,
                   startAngle, endAngle): string
getRadialTickPositions(startAngle, endAngle, count?): number[]
describeRadialBarChart(series, max, formatValue?,
                       formatPercent?): string
getDefaultRadialBarColor(index): string
```

- `getRadialBarMax` honours `override > 0` first; then
  the largest finite series value; falls back to 1.
- `getRadialBarRatio` clamps `value / max` into
  `[0, 1]`; non-finite, non-positive, zero-max inputs
  collapse to 0.
- `polarToCartesian` uses SVG conventions (0deg =
  East, -90deg = North, +90deg = South).
- `buildRadialArcPath` emits a closed annular arc
  segment with two `A` curves (outer + inner) and one
  `L`. `largeArc` flag set when `|sweep| > 180`. Returns
  `""` for zero outer radius or zero sweep.
- `getRadialTickPositions` returns evenly-spaced
  angles inclusive of endpoints (min count 2); returns
  `[]` for non-finite angles.
- `getDefaultRadialBarColor` cycles an 8-colour palette
  by index (modulo); negative indices fall back to the
  first colour.
- `describeRadialBarChart` produces the default ARIA
  summary; honours `formatValue` + `formatPercent`.

## Out of scope

- Negative / signed values (host normalises into
  `[0, max]` first).
- Animated value transitions on data change (snap v1;
  mount fade only).
- Stacked radial bars in a single ring (one ring per
  series; host stacks multiple instances when needed).
- Custom angular tick labels (only tick marks render
  by default; host renders its own labels with the
  exported `polarToCartesian`).

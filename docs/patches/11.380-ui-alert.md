## Goal

Dispatch:

- Build `web/src/components/ui/alert.tsx` --
  info / success / warning / error variants
  with icon, title, description, optional
  dismiss button, ARIA alert role.

`<Alert>` already shipped most of the
contract:

| dispatch item | shipped previously |
| --- | --- |
| info / success / warning / error variants | + neutral |
| title slot | `title` prop |
| description slot | `children` |
| optional dismiss button | `dismissible` + `onDismiss` |
| ARIA alert role | `role` auto-resolves (error -> alert/assertive, else status/polite) |

The dispatch's "with icon" hint was met
syntactically (the `icon` prop existed) but
required the caller to wire the icon
themselves. This patch closes that gap with
per-variant auto-icons + adds a size scale
and ergonomic data hooks.

## What changed

### Per-variant auto-icons

Each signal-bearing variant renders a
leading icon by default when `icon` is not
explicitly passed:

| variant | auto-icon (lucide) |
| --- | --- |
| `info` | `Info` |
| `success` | `CheckCircle2` |
| `warning` | `AlertTriangle` |
| `error` | `XCircle` |
| `neutral` | none (icon-less) |

- Caller-supplied `icon={<Custom />}` wins
  over the auto-icon (legacy contract).
- `icon={false}` opts out of the auto-icon
  entirely (new escape hatch).
- The icon palette tracks the variant
  colour via `text-current` inheritance.

This mirrors the Badge (11.229) auto-icon
contract so the two primitives feel
cohesive when a row pairs them.

### Size scale

```ts
type AlertSize = 'sm' | 'md' | 'lg';
```

| size | padding | text | gap | dismiss button |
| --- | --- | --- | --- | --- |
| `sm` | `p-2` | `text-xs` | `gap-2` | `h-5 w-5` |
| `md` (default) | `p-3` | `text-sm` | `gap-3` | `h-6 w-6` |
| `lg` | `p-4` | `text-base` | `gap-4` | `h-7 w-7` |

Default `md` matches the legacy 11.149
layout byte-for-byte; every existing call
site stays identical.

### Data attributes

- Root: `data-section="alert"`,
  `data-variant`, `data-size`.
- Inner: `data-section="alert-icon"`,
  `-title`, `-description`, `-action`,
  `-dismiss`.

Downstream tests + storyshots can branch on
attributes without parsing class lists.

### AlertBanner

`alert-banner.tsx` now destructures the
optional `size` prop and forwards it via
conditional spread so `<AlertBanner>`
inherits the new size scale through the
underlying Alert (which it already wraps).
Byte-identical default behaviour.

## API additions

```ts
interface AlertProps {
  // ... existing
  icon?: ReactNode | false;  // false opts out of auto-icon
  size?: 'sm' | 'md' | 'lg';
}
```

## Pairs with existing primitives

- `<AlertBanner>` (11.275) -- persistent
  banner wrapper. Now inherits the size
  scale.
- `<Badge>` (11.229) -- short signal
  pill. Auto-icon contract aligns so a
  row pairing Alert + Badge feels
  cohesive.
- `<Tooltip>` -- inline assistive tooltip;
  reach for Alert when the message is
  block-level and persistent.

  variant. Adopters that need a custom
  icon set `icon={<...>}` per instance.
- Inline action callouts beyond the
  existing `action` slot.
- Animated mount / dismiss transitions.
- Fixing the pre-existing
  `alert-banner.tsx` TS rest-spread error.
- Fixing the pre-existing
  `UIDemoRoute.tsx` demo page drift.

## Summary

Adds the `ChartLineFisherOversoldCross` primitive
(`web/src/components/ui/chart-line-fisher-oversold-cross.tsx`): a
pure SVG dual-panel chart that visualizes John Ehlers' Fisher
Transform crossings through the canonical oversold line -2.5. The
top panel plots the underlying close; the bottom oscillator panel
plots the Fisher line on an auto-fitted vertical range with a
horizontal threshold guide at -2.5, discrete cross markers at every
Fisher / threshold crossover, and overlay chevrons on the price
panel that mark the bar where the Gaussian-normalised momentum
pushed into (bearish, crossed down through -2.5) or out of (bullish,
crossed up through -2.5) the oversold zone.

Mirror complement of `chart-line-fisher-overbought-cross` (TODO
11.968): identical Ehlers pipeline, opposite threshold semantics.
Together with `chart-line-fisher-zero-cross` and the existing
`chart-line-fisher` / `chart-line-fisher-cross` /
`chart-line-fisher-cross-pct` / `chart-line-fisher-stoch` family,
this completes the threshold-crossing layer for the Fisher
Transform indicator (overbought + oversold + zero / baseline).

## Pipeline

Same close-only Fisher Transform pipeline as the overbought / zero
variants:

1. Rolling min/max on a sliding `length` window:
   - `hi[i] = max(close[i-length+1 .. i])`
   - `lo[i] = min(close[i-length+1 .. i])`
2. Normalize the current close into a `[-1, 1]` channel:
   - `raw[i] = (hi == lo) ? 0 : 2 * (close - lo) / (hi - lo) - 1`
3. Recursive smoother on the channel with weights `0.33 / 0.67`,
   clamped to `[-0.999, 0.999]`:
   - `x[i] = clamp(0.33 * raw[i] + 0.67 * x[i-1], -0.999, 0.999)`
4. Inverse hyperbolic tangent (Fisher transform) and another
   recursive half-EMA blend:
   - `f[i] = 0.5 * ln((1 + x[i]) / (1 - x[i])) + 0.5 * f[i-1]`
5. Regime classifier (strict `>=`):
   - `f >= -2.5` -> bullish (above oversold zone)
   - `f <  -2.5` -> bearish (in oversold zone)
6. Cross detector emits a `bullish` cross when
   `f[i-1] <= -2.5 && f[i] > -2.5` and a `bearish` cross when
   `f[i-1] >= -2.5 && f[i] < -2.5`. Strict inequality on the
   current side means the boundary value -2.5 does not double-fire.

## Defaults

- `length = 10` (Ehlers' canonical window)
- `threshold = -2.5` (canonical oversold line)
- Warmup boundary: bar `length - 1 = 9` under defaults, so the
  first 9 samples are null and the remaining 31 (out of 40 bars)
  are populated.

## Bit-exact anchors

- CONST `close = K` for all `K in {0, 1, 42, 100, 1234}` ->
  `fisher[i] === 0` literal (NOT `-0`) for every `i >= 9`. Regime
  classifies as bullish because 0 >= -2.5 -- the exact mirror of
  the overbought variant (where 0 < 2.5 gave bearish) and shares
  the bullish classification with the zero-cross variant.
- CONST K full-run counters under defaults, 40 bars:
  `noneCount = 9`, `bullishCount = 31`, `bearishCount = 0`,
  `crosses.length = 0`.
- LINEAR UP `close = i` -> the channel converges to the +clamp
  0.999, fisher grows to ~ 7.6, well above -2.5 the entire run.
  Counters under 60 bars: `bullishCount = 51`,
  `bearishCount = 0`, `crosses.length = 0`.
- LINEAR DOWN `close = -i` -> the channel converges to -0.999,
  fisher to ~ -7.6. At some bar after warmup the fisher crosses
  down through -2.5 -- emitting exactly one bearish cross -- then
  settles bearish thereafter.
- `pricePath` and `fisherPath` are byte-identical across repeated
  layout calls, with `.toFixed(2)` precision and `M ... L ...`
  command shapes. The fisher path uses a single `M` for post-warmup
  data.
- Cross detector boundary: strict inequality (`> T`, `< T`).
- Layout auto-fit: when fisher collapses to a single value (CONST
  case, fisher = 0), the panel expands DOWN to include
  threshold = -2.5 with 10% padding both sides -> oscMin = -2.75,
  oscMax = 0.25 (mirror of the overbought variant which expanded
  UP to include +2.5).

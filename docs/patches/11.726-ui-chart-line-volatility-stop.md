## Summary

`web/src/components/ui/chart-line-volatility-stop.tsx` ships
`<ChartLineVolatilityStop>`, a pure-SVG single-panel chart with a
**Welles Wilder Volatility Stop** overlay flipping a stop on an ATR
cross. The close line and the trailing stop share one panel; the
stop renders as position-coloured segments (green long, red short)
with one marker per defined bar and a highlighted marker on each
flip.

## Computation

The True Range of a single bar is the Wilder definition:

```
TR(i) = max( high - low, |high - prev_close|, |low - prev_close| )
```

For the very first bar (no previous close), TR collapses to
`high - low`.

The Average True Range uses Wilder's smoothing -- an initial SMA seed
of the first `period` true ranges, then

```
ATR[i] = ((period - 1) * ATR[i-1] + TR[i]) / period
```

The position starts `none` for the warm-up. At the first defined-ATR
bar it initialises to `long` with `stop = high - multiplier * ATR`.
While long the stop trails up only; while short the stop trails down
only. A close that crosses the stop flips the position and seeds a
new stop on the opposite side at the same ATR offset.

## Worked anchors

The ATR is a transcendental rational sequence (the Wilder smoothing
produces ratios like 2/3, 7/9, 23/27, ...) so generic distance
values cannot be expected to match bit-exact. The implementation
has three exact-style anchors that survive the floating-point
arithmetic by construction.

- **Constant-series anchor** -- `CONST_FLAT [5 x 10]` with
  `high = low = close = 5`. Every TR is exactly 0 (zero arithmetic
  is exact in IEEE 754), every ATR is exactly 0 (`((p-1) * 0 + 0)/p
  = 0`), every stop equals the close (`5 - 2 * 0 = 5`), and no flip
  fires (`close < stop` is `5 < 5` which is false). All defined
  bars are long, the long count is `n - (period - 1)`, the flip
  count is 0. Asserted with `toBe(5)` for every defined stop and
  `toBe(0)` for the flip count.
- **Rising-series anchor** -- `RISING [1..10]` with
  `high = low = close = i`. TR is 1 from bar 1 onward. The stop
  trails up but the close rises faster, so `close > stop` at every
  bar and no flip fires. All defined bars are long, the long count
  is 8 (bars 2..9), the flip count is 0.
- **Falling-series worked anchor** -- `FALLING [10..1]` with
  `high = low = close = v`, `period = 3`, `multiplier = 2`. The
  hand-traced ATR sequence is `ATR[2] = 2/3, ATR[3] = 7/9,
  ATR[4] = 23/27, ATR[5] = 73/81, ...`. Bar 2 initialises long with
  `stop = 8 - 2 * (2/3) = 20/3 = 6.667`. Bar 3 close 7 > stop, no
  trail update (the candidate `7 - 14/9 = 49/9 = 5.444` is below
  the prior stop), stay long. Bar 4 close 6 < stop 20/3 -> FLIP at
  bar 4. New short, new stop `6 + 2 * 23/27 = 208/27 = 7.704`. Bars
  5..9 stay short, trailing down. Asserted: flip count = 1, flip
  bar = 4, position after flip = short for bars 4..9, long count =
  2 (bars 2..3), short count = 6 (bars 4..9). The worked initial
  long stop `20/3` is asserted to 12 places with `toBeCloseTo`.

## Distinct from neighbouring primitives

- vs `<ChartLineMovingAvg>` and similar smoothers -- the smoothers
  ride alongside the price; the Volatility Stop is a DIRECTIONAL
  trailing stop that FLIPS regime on a cross.
- vs `<ChartLineBollinger>` / channel primitives -- those are
  symmetric bands around a central line that do NOT trail; the
  Volatility Stop trails monotonically in one direction until the
  cross.
- vs `<ChartLineKama>` / adaptive smoothers -- the smoothing adapts
  but the line still rides alongside; the Volatility Stop's stop
  sits OFFSET from the price by the ATR.
- vs `<ChartLineMonteCarlo>` -- the Monte Carlo Fan is a forward-
  looking probabilistic forecast; the Volatility Stop is a
  backward-looking deterministic trailing stop.

## API

`data` -- a single `ChartLineVolatilityStopPoint[]`
(`{x, high, low, close}`). `period` (default 10, integer >= 2).
`multiplier` (default 3, strictly positive finite). Standard
sizing / colour / a11y props plus `strokeWidth`, `dotRadius`,
`tickCount`. Toggle flags: `showAxis`, `showGrid`, `showDots`
(default false), `showStop`, `showMarkers`, `showFlips`,
`showTooltip`, `showConfigBadge`, `showLegend`, `animate`.
Controlled and uncontrolled visibility via `hiddenSeries`
(`'price' | 'stop'`). Callbacks: `onPointClick({point})`,
`onSeriesToggle({seriesId, hidden})`. Format functions:
`formatValue`, `formatX`. ARIA: `ariaLabel`, `ariaDescription`.

## Pure helpers

- `DEFAULT_CHART_LINE_VOLATILITY_STOP_*` -- defaults.
- `getLineVolatilityStopFinitePoints` -- keep only points with
  finite x/high/low/close.
- `normalizeLineVolatilityStopPeriod(period, fallback)` -- integer
  >= 2; else fallback.
- `normalizeLineVolatilityStopMultiplier(multiplier, fallback)` --
  finite > 0; else fallback.
- `computeLineVolatilityStopTrueRange(curr, prev)` -- the scalar
  true range; `high - low` for the first bar.
- `computeLineVolatilityStopATR(bars, period)` -- per-bar ATR with
  Wilder's smoothing; warm-up null.
- `computeLineVolatilityStop(bars, period, multiplier)` -- per-bar
  `{position, stop, flip, atr}`; the warm-up bars are
  `{'none', null, false, null}`.
- `runLineVolatilityStop(points, options?)` --
  `{series, period, multiplier, atr, bars, samples, stopFinal,
  positionFinal, longCount, shortCount, flipCount, ok}`.
- `computeLineVolatilityStopLayout({data, period, multiplier, ...})`
  -- single panel with the close line, one segment per pair of
  consecutive same-position bars (no segment across a flip), one
  marker per defined-position bar, and a value domain that spans
  the close range and the stop.
- `describeLineVolatilityStopChart(data, options?)` -- ARIA summary
  mentioning the indicator, the period, the multiplier, the trail-
  in-direction behaviour, and the flip count.

## a11y / data hooks

Root `<div role="region">` with `aria-label` + `aria-describedby`; the
SVG is `role="img"` with the same label; an sr-only
`data-section="chart-line-volatility-stop-aria-desc"` span carries the
description. The close line, every price dot and every marker carry
`role="graphics-symbol"` and `tabIndex={0}`. Every node carries a
`data-section`. Root mirrors `data-period`, `data-multiplier`,
`data-stop-final`, `data-position-final`, `data-long-count`,
`data-short-count`, `data-flip-count`, `data-total-points`,
`data-animate`, `data-empty`. Segments expose `data-position`;
markers expose `data-position`, `data-flip` and `data-stop`. Flip
markers render larger with a contrasting stroke when `showFlips`.

## Summary

New `ChartLineBandPass` UI primitive in
`web/src/components/ui/chart-line-band-pass.tsx` -- a pure-SVG two-panel
line chart with an **Ehlers Bandpass Filter** panel isolating the
dominant cycle band of the price. The top panel plots the price; the
bottom panel plots the bandpass and its trigger with a zero line.

## The computation

John Ehlers' Bandpass Filter is a two-pole resonant filter tuned to a
chosen cycle. It is a close-only indicator, so `ChartLineBandPassPoint`
carries `{x, value}`.

- `computeLineBandPassCoefficients(period, bandwidth)` -- turns a centre
  `period` and a fractional `bandwidth` into the three filter constants:
  `beta = cos(2*pi / period)`,
  `gamma = 1 / cos(2*pi*bandwidth / period)` and
  `alpha = gamma - sqrt(gamma^2 - 1)`. When the discriminant
  `gamma^2 - 1` is non-positive (a degenerate band) `alpha` falls back to
  1, which zeroes the drive term so the filter stays at rest.
- `computeLineBandPass(values, period, bandwidth)` -- drives the **two-bar
  price difference** through the tuned resonance:
  `bp[i] = 0.5*(1-alpha)*(price[i] - price[i-2]) + beta*(1+alpha)*bp[i-1] - alpha*bp[i-2]`,
  seeding the first two bars at zero. The feedback turns the difference
  into a clean oscillation around zero at the chosen cycle, rejecting the
  trend below the band and the noise above it. The trigger is the
  bandpass delayed one bar.
- `classifyLineBandPassZone(bp)` -- marks a bar **up** above zero,
  **down** below, **flat** at zero.

## Worked fixture

The `cos` / `sqrt` coefficients make the bandpass transcendental for a
general series, but one anchor is exact.

A **constant series** leaves every two-bar price difference
(`price[i] - price[i-2]`) at zero. With the first two bars seeded at zero
the recursion `0.5*(1-alpha)*0 + beta*(1+alpha)*0 - alpha*0` stays
exactly at zero for every later bar, so the bandpass is exactly 0 across
the whole series -- the test asserts every reading is 0 and the run is
wholly `flat`. The coefficients are checked structurally: `beta` against
its defining identity `cos(2*pi / period)`, `gamma >= 1` for a valid
band, `alpha` inside the open unit interval, and the bandwidth
normalization. A two-cycle sinusoid fixture confirms the bandpass rings
with both signs and the trigger lags it exactly one bar.

## API

`data` (a `ChartLineBandPassPoint[]`), `period` (default 20),
`bandwidth` (default 0.3), `pricePanelRatio` / `gap`,
sizing/format/a11y props, `strokeWidth` / `dotRadius`, the colour props,
the toggle flags (`showAxis` / `showGrid` / `showDots` default false /
`showBandpass` / `showTrigger` / `showZeroLine` / `showMarkers` /
`showTooltip` / `showConfigBadge` / `showLegend` / `animate`), controlled
+ uncontrolled `hiddenSeries` (keys `price` / `bandpass` / `trigger`),
`formatValue` / `formatX`, `onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineBandPassFinitePoints`,
`normalizeLineBandPassPeriod` (coerces the period to an integer of at
least 2), `normalizeLineBandPassBandwidth` (coerces the bandwidth into
the open unit interval), `computeLineBandPassCoefficients`,
`computeLineBandPass`, `classifyLineBandPassZone`, `runLineBandPass`,
`computeLineBandPassLayout`, `describeLineBandPassChart`, plus the
`DEFAULT_CHART_LINE_BAND_PASS_*` constants and the
`ChartLineBandPassZone` / `ChartLineBandPassCoefficients` types. Every
exported name carries a `LineBandPass` infix so the `export *` barrel
line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen
`data-section="chart-line-band-pass-aria-desc"` description span; the
price line, the bandpass line, the trigger line and every dot / marker
are `role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-period`, `data-bandwidth`,
`data-bp-final`, `data-up-count`, `data-down-count`, `data-flat-count`,
`data-total-points` and `data-animate`.

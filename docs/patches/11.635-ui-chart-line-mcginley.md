## Summary

New `ChartLineMcGinley` UI primitive at
`web/src/components/ui/chart-line-mcginley.tsx` -- a pure-SVG line
chart with a McGinley Dynamic adaptive moving average overlay
that self-adjusts to speed.

## Indicator math

The McGinley Dynamic, John R. McGinley's adaptive moving average,
seeds with the first price and folds each later bar in:

```
MD[0] = price[0]
MD[i] = MD[i-1] + (price[i] - MD[i-1]) / (N * (price[i] / MD[i-1])^4)
```

The `(price / MD_prev)^4` divisor is the adaptive term. In an up
market `price / MD_prev > 1`, so the divisor grows and the
adjustment shrinks -- the average **slows** and lags less
aggressively. In a down market the ratio is below one, the
divisor shrinks and the adjustment grows -- the average **speeds
up** to track the fall. The fourth power makes the response
strongly non-linear. The line is defined from index 0 with no
warm-up; a flat series holds exactly at its constant; a
non-positive previous value re-seeds on the current price to
avoid a divide-by-zero.

### Worked fixture

`period N = 10`, price series `[10, 16, 12, 9, 13]`. The first
move 10 -> 16 has divisor `10 * (16/10)^4 = 10 * 6.5536 =
65.536`, so `MD[1] = 10 + 6/65.536 = 10.091552734375` exactly.
The series runs up then drops sharply to 9: the down step
adapts faster (a smaller divisor) than the up steps, and the
price ends above the McGinley line on 3 bars, below on 1. Each
McGinley value is a fractional step strictly between the
previous value and the current price.

## API

- `getLineMcGinleyFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineMcGinleyPeriod(period, fallback)` -- coerce to a
  positive integer; non-finite or sub-1 falls back, fractional
  floors.
- `computeLineMcGinley(values, period)` -- the McGinley series, a
  plain `number[]` (no warm-up).
- `runLineMcGinley(points, options?)` -- sort finite points by
  `x`, compute the series, return `{ series, period, md,
  samples, mdFinal, mdMin, mdMax, aboveCount, belowCount, ok }`.
  Each sample is classified `above` / `below` / `on` the
  McGinley line. `ok` is false for fewer than two points.
- `computeLineMcGinleyLayout(options)` -- project the price path
  and the McGinley path on one shared panel.
- `describeLineMcGinleyChart(data, options?)` -- ARIA
  description.

## Component

`ChartLineMcGinley` is a `forwardRef` region (`role="region"`)
with an ARIA-linked description, a `role="img"` SVG, axis ticks
and grid, a config badge, a two-series legend (Price / McGinley)
with toggle, a hover/focus tooltip exposing the value, the
McGinley reading and the position, and a
`motion-safe:animate-fade-in` entrance. It consumes
`{ x, value }` points. Every node carries a `data-section`
attribute.

## Distinctness

- **vs the fixed-weight moving averages (chart-line-smma /
  chart-line-wma / an EMA)** -- those fold price in at a constant
  weight. The McGinley divisor changes every bar with the
  price-to-average ratio raised to the fourth power, so the line
  accelerates into declines and decelerates into rallies.
- **vs chart-line-kama / chart-line-vidya** -- those adapt their
  smoothing from an efficiency ratio or volatility band.
  McGinley adapts purely from the instantaneous price/average
  ratio, with no separate look-back window.
- **vs chart-line-lsma** -- the LSMA is a regression endpoint;
  the McGinley Dynamic is a single recursive step with an
  adaptive divisor and no warm-up.

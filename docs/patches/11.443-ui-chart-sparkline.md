## Summary

Add a new `<ChartSparkline>` UI primitive in
`web/src/components/ui/chart-sparkline.tsx` -- minimal inline
trend line for table cells / stat cards. Pure SVG, no axes,
no tooltip layer. Optional area fill, last-point dot, and
high / low markers. The accessible name is configurable; the
default is `"Trend"` and an auto-generated description
summarises the first / last / min / max values for screen
readers.

## API

```ts
interface ChartSparklineProps {
  data: readonly number[];
  width?: number;                      // default 80
  height?: number;                     // default 24
  color?: string;                      // default '#3b82f6'
  fill?: boolean;                      // default false
  fillOpacity?: number;                // default 0.2
  strokeWidth?: number;                // default 1.5
  showLastDot?: boolean;               // default false
  showHighLow?: boolean;               // default false
  smooth?: boolean;                    // default false
  className?: string;
  ariaLabel?: string;                  // default 'Trend'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  baseline?: number;                   // area-fill baseline
}
```

## Behaviour

- Renders a `<span role="img">` with an inline SVG sized to
  the configured `width` x `height` (default 80 x 24). The
  span collapses to its inline-flow context so adopters can
  drop the chart inside `<td>` / `<span>` / stat-card slots
  without breaking layout.
- The default ARIA description is auto-generated as
  `"Trend from <first> to <last>, min <min>, max <max>."` -- a
  custom `formatValue` formatter shapes the numbers inside
  the description.
- `smooth=true` builds the line via
  Catmull-Rom-to-cubic-bezier; smooth with fewer than 3
  points degrades to a straight `M / L` polyline.
- `fill=true` paints a closed area below the line. The
  baseline defaults to the bottom of the chart; passing
  `baseline` (in data-space) drops the closure line at that
  value so adopters can render diverging or baseline-
  anchored fills.
- `showLastDot` renders a small dot on the final point;
  `showHighLow` adds high + low markers (low is suppressed
  when high and low resolve to the same index, e.g. a
  constant series).
- Empty `data` renders an empty SVG + a `"No data"` ARIA
  description.

## Pure helpers (exported)

```ts
DEFAULT_SPARKLINE_WIDTH = 80
DEFAULT_SPARKLINE_HEIGHT = 24
DEFAULT_SPARKLINE_STROKE_WIDTH = 1.5
DEFAULT_SPARKLINE_FILL_OPACITY = 0.2
DEFAULT_SPARKLINE_COLOR = '#3b82f6'
getSparklineBounds(data): { min, max }
getSparklinePoints(data, width, height, bounds?): SparklinePoint[]
buildSparklinePath(points, smooth?): string
buildSparklineAreaPath(points, baselineY, smooth?): string
findSparklineExtremes(data): { highIndex, lowIndex }
```

- `getSparklineBounds` expands a constant series by
  `+/- 0.5` so the line never collapses to a single
  horizontal pixel and stays visible.
- `getSparklinePoints` snaps a single point to the
  geometric centre so a one-element series still renders.
- `buildSparklinePath` / `buildSparklineAreaPath` share a
  single Catmull-Rom smoother; the area variant closes the
  loop with a `Z` segment after dropping to the supplied
  baseline.
- `findSparklineExtremes` skips non-finite values and
  returns the first index achieving the high / low so the
  marker rendering is deterministic.

## ARIA + data attributes

- Root: `<span role="img">` + `aria-label`.
- `chart-sparkline-aria-desc` is a `sr-only` description
  region with the auto-summary (override via
  `ariaDescription`).
- SVG: `aria-hidden="true"` (decorative; the role lives on
  the root span).
- `data-section` on every node:
  - `chart-sparkline` (root span)
  - `chart-sparkline-aria-desc`
  - `chart-sparkline-svg`
  - `chart-sparkline-area`
  - `chart-sparkline-line`
  - `chart-sparkline-high`
  - `chart-sparkline-low`
  - `chart-sparkline-last`
- Root mirrors `data-point-count`, `data-fill`,
  `data-smooth`. High / low / last circles mirror
  `data-index`.

## Pairs with existing primitives

- `<ChartLine>` (11.439) for a full line chart with axes +
  tooltip; the sparkline is the inline cell-sized cousin.
- `<ChartArea>` (11.441) for the area-fill version with
  axes + multi-series.
- `<DataTable>` cells often embed `<ChartSparkline>` in a
  trailing summary column.

## Out of scope

- Multi-series sparklines (host renders multiple
  instances side by side or composes with `<ChartLine>`).
- Click / hover tooltips inside the cell (the primitive
  is intentionally decorative -- host wraps with a
  `<Tooltip>` if context-on-hover is needed).
- Threshold reference lines (host adds a custom SVG layer
  beneath the sparkline when needed).
- Server-side data subscription (host owns the array).

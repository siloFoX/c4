# 11.168 -- UI: form validation helpers + useForm hook (v1.11.186)

## Goal

Pull form validation logic out of individual page files into a shared
primitive + hook pair that any future page can drop in. Integrate with
the existing `Input` / `Textarea` `error` ReactNode slot so the inline
error styling stays consistent.

## Surface

### `web/src/lib/form-validation.ts`

```ts
export type ValidationResult = { error?: string };
export type Validator<T> = (value: T) => ValidationResult;

export function required(msg?: string): Validator<unknown>;
export function minLength(n: number, msg?: string): Validator<string>;
export function maxLength(n: number, msg?: string): Validator<string>;
export function pattern(re: RegExp, msg?: string): Validator<string>;
export function email(msg?: string): Validator<string>;
export function custom<T>(predicate: (v: T) => boolean, msg: string): Validator<T>;
export function compose<T>(...validators: Validator<T>[]): Validator<T>;
```

- `required` rejects `''` / `null` / `undefined`. `0` and `false` pass.
- `minLength` / `maxLength` are tolerant of non-string input (treat as
  pass) so a composed string validator does not throw if the field is
  briefly typed as another type.
- `email` uses `^[^\s@.]+(?:\.[^\s@.]+)*@[^\s@.]+(?:\.[^\s@.]+)+$` --
  rejects empty, missing `@`, consecutive dots, no TLD separator.
- `compose` short-circuits on the first error and returns
  `{ error: undefined }` when every validator passes.

### `web/src/hooks/use-form.ts`

```ts
export interface UseFormConfig<TFields extends Record<string, unknown>> {
  initialValues: TFields;
  validators?: { [K in keyof TFields]?: Validator<TFields[K]> };
  onSubmit?: (values: TFields) => void | Promise<void>;
}

export function useForm<TFields extends Record<string, unknown>>(
  config: UseFormConfig<TFields>,
): {
  values: TFields;
  errors: Partial<Record<keyof TFields, string>>;
  touched: Partial<Record<keyof TFields, boolean>>;
  setValue<K extends keyof TFields>(field: K, value: TFields[K]): void;
  setTouched(field: keyof TFields, touched: boolean): void;
  handleSubmit(e?: { preventDefault?: () => void }): void;
  isValid: boolean;
  reset(): void;
};
```

Behaviour:

- `errors[field]` is populated only when the field is touched OR after
  any `handleSubmit` call.
- `setValue(field, value)` re-validates that field when it is already
  touched / submitted; otherwise it just updates `values`.
- `setTouched(field, true)` validates the field with the current value
  immediately. `setTouched(field, false)` just unmarks touched; it does
  not clear the on-screen error (callers can `reset()` for that).
- `handleSubmit(e?)` calls `e?.preventDefault?.()`, marks every field
  touched, runs the full validator sweep, and only calls `onSubmit` when
  the sweep produces zero errors. The hook does not currently await the
  promise -- async errors are the caller's responsibility.
- `isValid` is computed from the full validator sweep across `values`,
  not from the visible `errors` map -- it stays accurate before any
  field has been touched so a submit button can stay disabled.
- `reset()` restores `initialValues`, clears errors + touched, and
  resets the submitted flag.

## Adoption

- `pages/Plan.tsx`: `task` field migrated to `useForm`. `onChange`
  routes through `setValue`, `onBlur` through `setTouched`,
  `errors.task` flows into the `Textarea` `error` slot. The Send
  button calls `handleSubmit` (to surface the inline error on empty
  input) and then still invokes `dispatchPlan` -- the existing toast
  / set-error path in `usePlanDispatch` is preserved so the test that
  asserts `dispatchPlan` fires on click is unchanged.
- `pages/Risk.tsx`: raw `<textarea>` for the command input replaced
  with the `Textarea` primitive so the `error` slot is available.
  `command` is now `commandForm.values.command`; both the Check and
  Sandbox preview buttons run `commandForm.handleSubmit()` before
  invoking their existing handlers, so the inline error appears
  immediately when the field is empty (existing `disabled={!command.trim()}`
  gating is unchanged).
- `pages/Profiles.tsx`: skipped. The page renders a read-only profile
  list plus a SearchBar filter; there is no editable profile name
  input today, so per the task's "fit naturally" rule there is
  nothing to wire.

## Tests

`web/src/lib/form-validation.test.ts` (10 assertions across 9 cases):

1. `required` rejects `''` / `null` / `undefined`; accepts non-empty
   plus `0` / `false`
2. `required` uses the custom message when provided
3. `minLength(3)` rejects below, accepts at + above
4. `maxLength(3)` rejects above, accepts at + below
5. `pattern(/^[a-z]+$/)` rejects upper, accepts lower
6. `email()` accepts valid forms, rejects empty / no-at / double-dot
7. `custom(predicate, msg)` uses the predicate + the supplied message
8. `compose(required, minLength)` returns the first failing error in
   order
9. `compose(required, minLength)` returns `{ error: undefined }` when
   every validator passes

`web/src/hooks/use-form.test.ts` (9 cases via `renderHook` + `act`):

1. initial values populated; errors + touched empty
2. `setValue` updates values
3. errors empty until touched or submit (isValid still false when
   required field is empty)
4. setting touched then setValue populates / clears error per
   validator
5. `handleSubmit` calls `onSubmit` with the values when valid
6. `handleSubmit` does NOT call `onSubmit` when invalid; populates
   errors for every failing field and marks every field touched
7. `isValid` flips from false to true as the field becomes valid
8. `reset()` restores initial values, clears touched + errors
9. `handleSubmit({ preventDefault })` calls `preventDefault`

pattern already in use by `web/src/lib/use-attach-form.test.ts` and
peers.)

## Out of scope

- Other pages -- adoption is intentionally limited to the two pages
  that have an existing required-field need with a primitive that
  carries the `error` slot.
- Async validators / debounced validation.
- Field-level `onChange` deduplication or shallow-equal value
  short-circuit.
- `docs/autonomous-queue-v10.md` (manager handles queue).
- `package-lock.json` (untouched).

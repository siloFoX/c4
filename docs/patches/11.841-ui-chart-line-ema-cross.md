# Patch 11.841 -- ChartLineEmaCross primitive (v1.11.859)

## Summary

Add `<ChartLineEmaCross />`, a pure-SVG single-panel chart that
overlays a fast EMA and a slow EMA on a price line, with scatter
markers at every detected fast-over-slow crossover event. Parallel
to 11.840 chart-line-sma-cross, swapping the rolling SMA for the
SMA-seeded EMA.

```
fastEma[i] = EMA(close, fastLength)[i]
slowEma[i] = EMA(close, slowLength)[i]
up cross   = prev fastEma <= prev slowEma  AND  curr fastEma > curr slowEma
down cross = prev fastEma >= prev slowEma  AND  curr fastEma < curr slowEma
```

## EMA helper precision

Carries the `min === max` seed precision fix used in 11.831
(TRIX), 11.832 (fractal KC), 11.836 (EMA pct). When the seed window
is constant the EMA is seeded with the exact value instead of
`sum / length`. The CONST short-circuit
`next = v === ema ? v : alpha*v + (1-alpha)*ema` then preserves the
constant through the recursive smoothing, so CONST inputs make both
EMAs bit-exactly equal -- no spurious crosses.

| Knob              | Value     |
|-------------------|-----------|
| fastLength        | 12        |
| slowLength        | 26        |
| priceColor        | `#2563eb` |
| fastColor         | `#f97316` (orange) |

| bearishColor      | `#dc2626` (red) for down-cross markers |
| markerRadius      | 5         |

Defaults match the canonical MACD 12/26 EMA pair. Different from
11.840 (sma-cross uses 10/50 golden-cross defaults).

## Bit-exact anchors

- `CONST close = K`: both EMAs collapse to `K` exactly via the
  min/max seed and CONST short-circuit, so `upCrossCount =
  downCrossCount = 0`. Verified across `(K, fast, slow)` sweeps
  including the MACD `(12, 26)` pair.
- `LINEAR UP close = i + 1`: each EMA tracks the input with a
  steady-state offset proportional to its length; with
  `slowLength > fastLength`, the fast EMA leads, so `fast > slow`
  for every post-warmup bar. Zero crosses. Verified across
  `(2, 4)`, `(5, 10)`, `(12, 26)` pairs.
- `STEP close = K1 for i < N, K2 for i >= N` with `K1 < K2`: the
  fast EMA accelerates first. For `K1 = 1, K2 = 5, fastLength = 2,
  slowLength = 4, N = 4`, the up-cross fires at exactly `i = 4`
  (same deterministic index as the SMA-cross case, because the
  inequality `fastEma > slowEma` holds the same way). The symmetric
  `STEP K1 = 5, K2 = 1` produces exactly one down-cross.

## Surface

Single-panel primitive surface matching 11.840:

- `ChartLineEmaCrossPoint = { x, close }`
- `runLineEmaCross(data, options?) -> Run` with
  `ok = series.length >= max(fastLength, slowLength)`.
- `computeLineEmaCrossLayout({ data, width?, height?, padding?, fastLength?, slowLength? }) -> Layout`.
  Single panel; y-axis auto-fits across `close / fast / slow`.
- `describeLineEmaCrossChart(data, opts?) -> string` for the ARIA
  description.
- `<ChartLineEmaCross />` `forwardRef<HTMLDivElement>` with three
  legend toggles (`price`, `fast`, `slow`), config badge with live
  cross counts, scatter markers at each crossover event, and a
  tooltip surfacing `close / fast / slow / relation / cross`. Live
  cross counts also exposed as `data-up-cross-count` /
  `data-down-cross-count` attributes for selector-driven assertions.

# 11.191 -- UI: BulkActionToolbar primitive

`web/package.json` 1.11.208 -> 1.11.209.

## What

A floating multi-select toolbar primitive for list / table views. Shows
up only when at least one row is selected; surfaces a row of action
buttons (with optional destructive tone + leading icon) and an
optional clear-selection control. Three first-party adoptions ship in
the same patch.

## New files

- `web/src/components/ui/bulk-action-toolbar.tsx` -- the primitive
  (forwardRef, ~100 lines).
- `web/src/components/ui/bulk-action-toolbar.test.tsx` -- vitest + RTL
  coverage (16 cases).

## API

```ts
export interface BulkAction {
  id: string;
  label: ReactNode;
  icon?: ReactNode;
  onClick: () => void;
  tone?: 'default' | 'danger';
  disabled?: boolean;
}

export interface BulkActionToolbarProps
  extends Omit<HTMLAttributes<HTMLDivElement>, 'role'> {
  selectedCount: number;            // visibility gate: > 0 only
  actions: BulkAction[];
  onClearSelection?: () => void;    // renders X icon-button when set
  position?: 'top' | 'bottom';      // default 'bottom'
  ariaLabel?: string;               // forwarded to aria-label
}
```

## Behaviour

- `selectedCount <= 0` -> returns `null` (no DOM).
- Label: `'1 item selected'` (singular) / `'<N> selected'` (plural).
- `position='bottom'` (default): `fixed bottom-4 left-1/2 -translate-x-1/2 z-40`.
- `position='top'`: `sticky top-0 z-30 mx-auto`.
- Action buttons use `<Button size="sm">`; `tone='danger'` maps to
  `variant='destructive'`. `icon` is rendered inside an
  `aria-hidden` span to the left of `label`.
- Clear button (when `onClearSelection` is provided) is a tight
  `<IconButton>` with the lucide-react `X` glyph; `aria-label="Clear selection"`.
- Entrance animation: `animate-in fade-in slide-in-from-bottom-2 duration-200`
  (or `slide-in-from-top-2` for top); suppressed when
  `useReducedMotion()` returns `true`.
- `role="toolbar"` on the root; `aria-label` defaults to
  `'Bulk actions'`.
- `forwardRef` to the root `<div>`.

## Adoption sites

1. `WorkerList` (`web/src/components/WorkerList.tsx`): adds a
   `Select` / `Deselect` toggle to the existing right-click
   `<ContextMenu>` per row; selected rows get a `ring-2 ring-primary`
   highlight; floating `BulkActionToolbar` exposes a `Kill selected`
   action (danger tone, `Trash2` icon) wired to a placeholder handler
   that clears the selection. Same placeholder semantics as the
   existing `Kill` context-menu item.
2. `SessionsListSection`
   (`web/src/components/SessionsListSection.tsx`): same context-menu
   `Select` toggle pattern; `Close selected` action (danger, `X`
   icon). Placeholder action -- no backend wired.
3. `HistoryView` (`web/src/components/HistoryView.tsx`):
   `shift+click` / `meta+click` / `ctrl+click` on a worker row in
   the sidebar (or right-click) toggles membership; selected rows
   get a `ring-2 ring-primary` highlight; `Delete selected` action
   (danger, `Trash2` icon). Placeholder -- no backend wired.

The chosen toggle mechanism avoids invasive per-row checkboxes,
following the task's "long-press / right-click toggle" fallback
guidance.

## Tests

`bulk-action-toolbar.test.tsx` covers:

- not rendered when `selectedCount=0`
- rendered when `selectedCount>=1`
- shows `1 item selected` (singular) and `<N> selected` (plural)
- renders action buttons
- action button click invokes `onClick`
- `tone='danger'` action receives `bg-destructive`
- clear button renders only when `onClearSelection` is provided
- clear button click invokes `onClearSelection`
- default `position='bottom'` applies `fixed bottom-4`
- `position='top'` applies `sticky top-0`
- caller `className` is merged with built-in classes
- `role='toolbar'` with the `aria-label` override
- `forwardRef` resolves to the root `<div>`
- `disabled` flag on an action propagates to the button element

## Not in scope

- Real backend wiring for `Kill selected` / `Close selected` /
  `Delete selected`. Each adoption keeps a clearly-marked placeholder.
- Per-row checkboxes (deliberately deferred -- the existing
  `<Card>`/`<ListItem>` rows are interactive and a leading checkbox
  would steal clicks).
- Keyboard shortcuts for bulk-select (future patch).

## Goal

Dispatch:

- Build `web/src/components/ui/drawer-resize.tsx`
  -- resizable side panel with drag handle,
  min/max width constraints, persist size
  to localStorage.

No existing drawer-resize primitive in the
codebase. This patch ships one from
scratch.

## API

```ts
interface DrawerResizeProps {
  children: ReactNode;
  side?: 'left' | 'right';     // default 'right'
  width?: number;              // controlled
  defaultWidth?: number;       // uncontrolled, default 280
  onWidthChange?: (next: number) => void;
  minWidth?: number;           // default 200
  maxWidth?: number;           // default 600
  storageKey?: string;         // omit to disable persistence
  keyboardStep?: number;       // default 16
  pageStep?: number;           // default 80
  handleAriaLabel?: string;    // default 'Resize panel'
  className?: string;
  handleClassName?: string;
}
```

```tsx
<DrawerResize
  side="left"
  defaultWidth={320}
  minWidth={240}
  maxWidth={480}
  storageKey="workers-sidebar"
>
  <SidebarBody />
</DrawerResize>
```

## Behaviour

### Pointer drag

- `pointerdown` (primary button only) on
  the handle captures the start position
  via `setPointerCapture`.
- `pointermove` computes `dx`, applies
  the side-aware delta, and commits a
  new width (no localStorage write
  during move).
- `pointerup` / `pointercancel` ends the
  drag, removes the dragging visual,
  and writes the final width to
  localStorage when `storageKey` is set.

### Side semantics

Right-side panel: handle docks to the
panel's LEFT edge. Dragging the handle
LEFT (toward the viewport edge) GROWS
the panel.

Left-side panel: handle docks to the
panel's RIGHT edge. Dragging the handle
RIGHT (toward the viewport edge) GROWS
the panel.

### Clamping

Every commit (pointer + keyboard) runs
through `clampDrawerWidth(value, min,
max)`. Stored values are clamped on
mount so a stale localStorage entry
never opens the panel outside the
declared bounds.

### Keyboard

Matches the user's mental model of
"drag the handle left / right":

| panel side | ArrowLeft | ArrowRight | PageUp | PageDown | Home | End |
| --- | --- | --- | --- | --- | --- | --- |
| right | +step | -step | +pageStep | -pageStep | min | max |
| left | -step | +step | +pageStep | -pageStep | min | max |

Default `keyboardStep=16`, `pageStep=80`.

### ARIA

Handle:
- `role="separator"`.
- `aria-orientation="vertical"`.
- `aria-valuemin / valuenow / valuemax`.
- `aria-label` (default "Resize panel";
  configurable via `handleAriaLabel`).

The handle is focusable (`tabIndex={0}`)
so screen-reader users can adjust the
width via keyboard.

## Pure helper

```ts
export function clampDrawerWidth(
  value: number,
  min: number,
  max: number,
): number;
```

Returns `min` for NaN / inverted (min >
max) bounds. Exported for tests + alternate
hosts that want the same clamp logic.

## localStorage persistence

- Key: `c4:drawer-resize:<storageKey>`.
- Initial mount reads the stored value
  (clamped to `[min, max]`) when
  `storageKey` is set. Falls back to
  `defaultWidth` when the key is absent
  or the stored value is non-numeric.
- Every commit (keyboard + pointerup +
  controlled `onWidthChange`) writes
  back.
- All access is wrapped in try/catch so
  private-mode / quota errors silently
  fall through.

## Data attributes

| attr | location |
| --- | --- |
| `data-section="drawer-resize"` | root |
| `data-side="left" | "right"` | root + handle |
| `data-width` (px) | root |
| `data-dragging="true" | "false"` | root + handle |
| `data-section="drawer-resize-handle"` | handle |
| `data-section="drawer-resize-body"` | children wrapper |

## Pairs with existing primitives

- `<Drawer>` (11.279) -- modal side
  panel with focus trap + slide
  animation. Use Drawer for transient
  overlays; DrawerResize for persistent
  layout sidebars.
- `<SplitPane>` (existing) -- two-pane
  resizer where both sides own
  content; reach for SplitPane when
  the resize is between siblings, not
  between a fixed sidebar + main.
- `<ScrollArea>` (11.382) -- composable
  scroll surface; pair when the panel
  body needs custom scroll chrome.

## Out of scope

- Per-page adoption of DrawerResize.
  Additive; no existing surface is
  forced onto the new primitive.
- Auto-collapse to a 0px width
  (mini-sidebar). Belongs in a
  separate CollapsibleSidebar
  primitive.
- Drag from anywhere within the panel.
  The handle is the canonical
  affordance; ambient drag would
  conflict with text selection +
  scroll.
- Touch-specific drag thresholds. The
  pointer event API covers touch +
  mouse uniformly; per-device tuning
  belongs in a follow-on.
- Vertical resize (top / bottom panel).
  Contract is horizontal-only;
  vertical resize is `<SplitPane>`
  territory.
- Cross-tab storage sync. The stored
  value is read once on mount;
  multi-tab consistency would need a
  `storage` event listener (matches
  the useTheme cross-tab pattern but
  out of scope here).

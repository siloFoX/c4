## Preview thumbnails

`FileDrop` gains a `previewThumbnails` prop
(default `true`). When enabled:

- Every staged file whose MIME type starts with
  `image/` renders an inline 32x32 thumbnail
  next to the row (in place of the canonical
  `FileText` glyph).
- The thumbnail `<img>` ships with
  `decoding="async"` + `loading="lazy"` per the
  11.350 lazy-image contract.
- `URL.createObjectURL` is the source; URLs are
  cached per identity key
  (`name|size|lastModified`) so re-renders do
  not allocate fresh blob URLs.
- The cache is reconciled on every render: URLs
  for files that no longer appear in the staged
  list are revoked, and an unmount cleanup
  revokes anything still in the map. No blob
  leaks.
- Non-image files keep the `FileText` icon.
- Set `previewThumbnails={false}` to disable
  entirely (every row uses the icon).
- `data-section="file-drop-thumbnail"` on the
  `<img>` lets adopters target the surface in
  CSS / e2e.

## Upload helper (`lib/file-upload.ts`)

`fetch()` does not surface upload-progress
events, so XHR remains the canonical pattern.
The helper wraps the boilerplate:

```ts
uploadFile({
  url,
  file,
  field?,          // FormData field name (default 'file')
  method?,         // POST | PUT | PATCH (default POST)
  headers?,        // extra request headers
  fields?,         // extra FormData entries
  onProgress?,     // { loaded, total, progress }
  signal?,         // AbortSignal
  timeoutMs?,      // 0 disables
  parseResponse?,  // default JSON.parse (text fallback)
}): Promise<UploadResult>

uploadFiles({
  url, files,
  onProgress?,     // adds { index, file }
  onResult?,       // per-file completion callback
  stopOnError?,    // default false
  ...uploadFile rest
}): Promise<UploadResult[]>

validateFile(file, { accept?, maxSize? }):
  { ok: true } | { ok: false, reason: 'unaccepted-type' | 'too-large' }
```

### Result shape

```ts
interface UploadResult<T = unknown> {
  ok: boolean;
  status: number;
  data: T | null;
  error: string | null;
}
```

`error` flavours:

| value | trigger |
| --- | --- |
| `null` | success (2xx + parser succeeded) |
| `'aborted'` | AbortSignal fired or `xhr.abort()` |
| `'timeout'` | XHR timeout event |
| `'network-error'` | XHR error event (transport failure) |
| `'status <N>'` | non-2xx response |
| `<parser error>` | custom `parseResponse` threw |

The promise never rejects -- callers read the
shape and route through `result.ok`.

### Progress events

```ts
interface UploadProgressEvent {
  loaded: number;
  total: number | null;
  progress: number | null;
}
```

`progress` is `0..1` (clamped) when the response
length is computable; `null` otherwise (server
emitting chunked / unknown-length payloads).

`uploadFiles` decorates each event with `index`
and `file` so a host can drive a multi-row
progress UI off one callback.

### Abort

Pass an `AbortSignal`. Already-aborted signals
resolve immediately with `error: 'aborted'`. The
listener is removed on settle so the signal does
not leak references.

### Validation

`validateFile` mirrors `FileDrop`'s internal
`accept` matcher (`.ext` / `mime/*` / exact
MIME) plus optional `maxSize`. Exposed for
pre-pick validation (e.g. dropping the file
before staging it).

## Tests

`file-upload.test.ts` -- 19 cases:

- `uploadFile`: POST + FormData send, progress
  ratio computation, `lengthComputable=false`
  emits `null`, 5xx surfaces `status N` error,
  network error, abort signal + pre-aborted
  signal, header + extra-field forwarding,
  timeout event, custom `parseResponse`.
- `uploadFiles`: sequential dispatch, per-file
  progress carries `index`, `stopOnError`
  exits early.
- `validateFile`: no-accept default, size
  limit, extension match + mismatch,
  `image/*` wildcard, exact MIME, `maxSize:
  null` skip.

`file-drop.test.tsx` -- 31 existing cases still
pass; the thumbnail logic does not break any
existing assertion.

Total: 50/50 pass under vitest 4.1.5. TypeScript
clean for the touched files.

## Out of scope

- **Per-page adoption beyond the Snapshots
  dropzone.** Templates / Profiles imports keep
  their current flow; a follow-up patch wires
  `uploadFile` through.
- **Chunked / resumable uploads.** Single POST
  per file; the chunk pipeline is a follow-up
  if a >2GB surface appears.
- **Server-side antivirus / scanning hooks.**
  Daemon-side concern; the helper trusts the
  response.
- **Drag-over visual polish.** The existing
  `data-active=true` selector already drives
  the border colour change.
- **Native `image/*` fallback rasterisation**
  for SVG inputs that misreport size. Browsers
  decode SVG via the same `<img>` path; if a
  malformed SVG fails the decode the wrapper
  silently retains the `FileText` glyph (no
  thumbnail).

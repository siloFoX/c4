# 11.293 -- Card link mode + disabled state + data selectors (v1.11.311)

Component-scope-only enhancement of the existing `Card` primitive.
No daemon-side change and no `c4` CLI surface change. The default
surface (no href, no disabled, no interactive) is preserved
byte-for-byte modulo the new data-section attribute set; the
existing snapshot baselines were rebuilt to include the new attrs.

## Why

Two pain points across the recent card adoption surfaces:

1. The dispatch's intended sites (Workers cards, Sessions cards,
   Templates cards) currently render their per-row Card with an
   inline `onClick` AND a manual `tabIndex={0}` + role="button"
   set, OR wrap the entire row in an `<a>` outside the Card. The
   second pattern fragments the hover-lift / focus-visible-ring
   affordances because the link affordances live on the parent
   anchor instead of the Card surface itself.
2. Disabled cards (e.g., an already-attached Session row) had
   no first-class disabled state; callers inlined
   `opacity-60 cursor-not-allowed` on `className` AND wrote
   their own `aria-disabled` + `onClick` guards.

Patch 11.293 ships the link-mode + disabled-state contract
inside the primitive so every Card surface reaches for the same
vocabulary.

## What changed

### `web/src/components/ui/card.tsx`

- New `href?: string` prop. When set, the Card renders as an
  `<a>` element instead of a `<div>` (the Linkable-Card
  pattern). Interactive affordances (hover-lift, focus-visible
  ring, active press) apply automatically; the caller does not
  need to also set `interactive`.
- New `target?` + `rel?` props -- standard anchor attributes
  pass through when `href` is set (useful for external link
  cards: `target="_blank" rel="noreferrer"`).
- New `disabled?: boolean` prop. When true:
  - Drops the interactive hover-lift / active-press /
    cursor-pointer affordances.
  - Adds `cursor-not-allowed opacity-60` styling.
  - Suppresses the `onClick` handler at the wrapper level so
    disabled cards do not fire dispatched actions.
  - Sets `aria-disabled="true"` for SR users.
  - Link mode (href set) drops the href so the browser does
    not navigate on click, AND sets `tabIndex={-1}` so
    keyboard users skip it.
- Hoisted shared `INTERACTIVE_CLASS` + new `DISABLED_CLASS`
  constants out of the `forwardRef` body so the link-mode
  anchor branch can reach for the same affordance set as the
  div branch.
- New data-attribute selectors:
  - `data-section="card"` on every Card render.
  - `data-interactive="true|false"` reflects the resolved
    interactive state.
  - `data-disabled="true|false"` reflects the disabled prop.
  - `data-mode="link"` only on the link-mode render path.
- exactOptionalPropertyTypes-safe conditional spreads for the
  anchor's `target`, `rel`, `role`, `tabIndex` props so a
  caller passing `undefined` does not violate the
  `AnchorHTMLAttributes` shape.

### Adopted

- Primitive enhancement only this commit. The dispatch sites
  inherit the new defaults + data-section attributes without
  any caller change.

## Deferred (dispatch follow-ups)

- Workers cards (open detail): thread `href` or
  `interactive + onClick` onto the per-row Card so the
  primitive handles hover-lift + focus ring uniformly.
- Sessions cards (attach): same story -- the row-level Card
  can opt into the disabled state for already-attached
  sessions.
- Templates cards (open editor): the per-template card row
  is the canonical adoption site once the editor surface
  lands.

## Notes

- The `INTERACTIVE_CLASS` constant is hoisted to module
  scope (rather than declared inside `forwardRef`) so the
  div + anchor render paths share one source of truth.
- Link-mode disabled does not unmount the anchor element on
  purpose -- the anchor stays in the DOM (so parent-row
  layouts that expect a single child node continue to work)
  but reads as inert (no href, aria-disabled, tabIndex=-1).

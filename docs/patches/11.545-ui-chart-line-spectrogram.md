# 11.545 -- web/src/components/ui/chart-line-spectrogram.tsx

Pure-SVG **STFT spectrogram** chart. Pairs a time-domain line
on top with a 2D frequency-vs-time heatmap below, where each
column is one Short-Time Fourier Transform frame, each row is
a frequency bin, and the cell color encodes the magnitude. A
dominant-frequency track overlays the heatmap as a dashed
line showing how the dominant frequency evolves across frames.

## Distinct from related primitives

- `<ChartLineFft>` (11.540): single 1D spectrum of the whole
  signal (no windowing). Spectrogram is a SEQUENCE of
  windowed spectra over sliding windows.
- `<ChartLineFftWindow>` (11.544): single windowed 1D
  spectrum of the whole signal. Spectrogram applies the
  same windowing PER WINDOW POSITION and stacks the
  results into a 2D heatmap.
- `<ChartLineDecompose>` (11.543): time-domain seasonal
  split. Spectrogram is frequency-domain and TIME-VARYING.

The canonical librosa / matplotlib / Audacity spectrogram
output -- used for visualising audio, vibration, EEG, or
any signal where the dominant frequency shifts over time.

## STFT algorithm

For each window position `start = 0, hop, 2*hop, ...`:

1. Extract samples `y[start..start+W-1]` from the input
   (optionally pre-detrended by subtracting the global
   mean).
2. Multiply element-wise by the window coefficients
   `w[0..W-1]`.
3. Compute the half-spectrum DFT (length `W/2 + 1`).
4. Take per-bin magnitude `|X[k]|`.

The resulting 2D array has dimensions `frames x (W/2+1)`
where:
- frames = `floor((N - W) / hop) + 1`
- frequency at bin k is `k / (W * sampleStep)` where
  `sampleStep` is the average x-spacing
- bin 0 (DC) is dropped from the heatmap by default
  (`excludeDc=true`)

  scale for the heatmap.
- `dominantColor` -- dominant-track + badge color.
- Toggle flags: `showAxis`, `showGrid`, `showDots`,
  `showLegend`, `showTooltip`, `showDominantBadge`,
  `showDominantTrack`, `showLine`, `showWindowToggle`,
  `showColorScale`, `animate`.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX` / `formatFrequency` /
  `formatMagnitude`.
- `onCellClick({cell})` / `onPointClick({point})`.

## Pure helpers

- `LINE_SPECTROGRAM_WINDOW_MODES` -- `['rectangular',
  'hann', 'hamming', 'blackman']`.
- `getLineSpectrogramFinitePoints(points)`.
- `normaliseLineSpectrogramWindowSize` -- clamps to `>= 2`;
  floors fractional; non-finite -> default 32.
- `normaliseLineSpectrogramHopSize(value, windowSize)` --
  clamps to `[1, windowSize]`; default = floor(windowSize *
  0.5).
- `normaliseLineSpectrogramWindowMode` -- identity for
  valid; default `'hann'`.
- `normaliseLineSpectrogramPanelRatio` -- clamps `[0.1,
  0.6]`.
- `computeLineSpectrogramWindowCoefficients(N, mode)` --
  canonical window weights. Same formulas as 11.544.
- `getLineSpectrogramScaleColor(t, low, mid, high)` --
  3-stop linear interpolation in RGB space. `t` clamped to
  `[0, 1]`; non-finite -> `low`.
- `computeLineSpectrogram(points, {windowSize?, hopSize?,
  windowMode?, detrend?, excludeDc?})` -- canonical STFT
  pipeline. Returns `{ok, frames, windowSize, hopSize,
  windowMode, binCount, maxMagnitude, frequencies,
  totalSamples, sampleStep}`. Each frame carries
  `{frameIndex, centerSampleIndex, centerX, startX, endX,
  magnitudes, dominantBin, dominantFrequency,
  dominantMagnitude}`. Input shorter than window ->
  ok=false. Sorts ascending; drops non-finite; excludes DC
  from dominant search by default.
- `computeLineSpectrogramLayout` -- projects line path +
  heatmap cells + dominant-track polyline.
- `describeLineSpectrogramChart` -- screen-reader
  description.

  magnitude + window mode.

## Tooltip

Hovering a cell shows the frame number, bin, frequency,
bold magnitude, and (when dominant) a coloured "dominant"
row. Hovering a time dot shows x and bold y.

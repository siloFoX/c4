## Summary

`ChartLineAtrTrail` is a pure-SVG single-panel React primitive
that overlays a Chandelier-style long-side ATR trailing stop on
the close, with a monotonic ratchet rule:

```
TR[i]        = max(high - low, |high - prevClose|, |low - prevClose|)
ATR[i]       = SMA(TR, length)[i]
highest[i]   = max(close, [i - length + 1, i])
candidate[i] = highest[i] - multiplier * ATR[i]
trail[i]     = candidate[i]                    (first valid bar)
             = max(trail[i - 1], candidate[i]) (ratchet up)
```

Defaults: `length = 14`, `multiplier = 2`. Bars before
`i = length - 1` are warmup nulls.

The ratchet rule never lets the trail descend: once raised, it
stays put or rises. This is the long-side trailing-stop
convention; pairs with a corresponding short-side trail in a
stop-and-reverse system (not in scope here).

## Bit-exact anchors

- **CONST_FLAT** (`high = low = close = K`): every TR is zero,
  ATR is zero, `highest = K`, `candidate = K - 0 = K`, and the
  trail equals `K` exactly past warmup. The trail coincides with
  the close. Verified across `K` including 0 and negatives,
  across `length` and `multiplier` combinations.
- **CONST_BAR with dyadic TR** (`high = K + r`, `low = K - r`,
  `close = K + r` etc. with `2r` a power of 2): every TR is the
  dyadic value, ATR equals it bit-exact, and the trail equals
  `C - multiplier * TR` bit-exact (the ratchet has no effect
  because the candidate is constant). The test verifies a `(H,
  L, C) = (11, 9, 10)` anchor: `trail = 10 - 2*2 = 6` exactly
  past warmup.

The ratchet rule is verified explicitly with a rising-then-
falling close series and a noisy random walk, asserting that
`trail[i] >= trail[i - 1]` for every valid `i`.

## Zone classifier

- `breakout` -- close > trail AND trail rose vs. previous bar
- `above` -- close > trail (trail did not rise)
- `at` -- close == trail
- `broken` -- close < trail (potential exit signal)
- `none` -- trail is null (warmup)

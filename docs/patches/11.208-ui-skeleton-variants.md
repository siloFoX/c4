# 11.208 -- UI: Skeleton variants

Tag: v1.11.226
Branch: `c4/auto-ui-skeleton-variants`
Worktree: `/root/c4-worktree-auto-w192`

## Goal

Give the web UI a small, typed set of compound loading-state shapes so
the dozens of inline `animate-pulse rounded-md bg-muted` divs scattered
across the dashboard can be replaced with named, accessible components
that share a single source of truth for spacing, rounding, and a11y.

## Scope

`web/src/components/ui/skeleton.tsx` is extended with four new
sub-components attached to the existing `Skeleton` primitive:

| Sub-component        | Purpose                                        | Defaults                                    |
| -------------------- | ---------------------------------------------- | ------------------------------------------- |
| `Skeleton.Text`      | Single-line or multi-line text placeholder.    | `width='100%'`, `height='1em'`, `lines=1`   |
| `Skeleton.Avatar`    | Circular avatar placeholder for list rows.     | `size='md'` (24/32/48 px for sm/md/lg)      |
| `Skeleton.Card`      | Bordered panel: header bar + 2 text lines.     | -                                           |
| `Skeleton.Table`     | Header row + N body rows with C cells each.    | `rows=5`, `cols=3`                          |

Each is also exported by name (`SkeletonText`, `SkeletonAvatar`,
`SkeletonCard`, `SkeletonTable`) for callers that prefer flat imports.

The pre-existing `Skeleton` primitive (variants `text` / `row` /
`card` / `avatar` / `rect` / `line` / `circle` / `page`) and the
v1.11.174 composable shape exports (`TextLine`, `Rect`, `Circle`,
`AvatarShape`, `StatCardShape`, `TableRowShape`) remain byte-identical.
The new sub-components are additive, not replacements.

## Why a TypeScript `namespace` merge

The task spec asks for `Skeleton.Text` / `Skeleton.Avatar` / etc.
attached to `Skeleton`. Two patterns can express that:

1. Runtime `Object.assign(Skeleton, { Text, Avatar, ... })` plus an
   intersection cast for the type side.
2. A TypeScript `namespace Skeleton { export const Text = ... }`
   merged into the function declaration.

Option 2 is the chosen approach because it keeps both the runtime
value and the static type in sync without a `as` cast and without
weakening `Skeleton`'s function signature. The
`@typescript-eslint/no-namespace` rule is disabled locally on that
single namespace block -- this is exactly the declaration-merging
pattern the rule documents as legitimate.

## A11y

Every sub-component sets `role="status"` + `aria-hidden="true"` on
its outer wrapper. The default supports the common case of a top-level
loading placeholder (screen readers announce one status region). When
the sub-component is nested inside another `role=status` region (as
happens in `PageFrame.LoadingSkeleton` and `Auto.tsx`'s
`BlockSkeleton`), the call site passes `role="presentation"` so
`getByRole('status')` in adopter tests continues to return a single
match. The `{...rest}` spread is intentionally last so caller props
win over the defaults.

## Adopted callsites

Three ad-hoc loading placeholders are swapped to the new variants:

1. **`web/src/pages/PageFrame.tsx`** -- `LoadingSkeleton`'s inner bars
   become `<Skeleton.Text height="2rem" role="presentation" />`. The
   outer wrapper keeps `role="status"` + `aria-live="polite"` + the
   localized `sr-only` `pageFrame.loading` text so every page that
   renders `<LoadingSkeleton rows={N} />` (Cleanup, Validation,
   Scribe, Templates, TokenUsage, Plan) continues to find the loading
   region via `getByRole('status')`.
2. **`web/src/pages/Auto.tsx`** -- `BlockSkeleton`'s inner bars
   become `<Skeleton.Text height="2.5rem" role="presentation" />`.
   The outer `aria-label="Loading"` status wrapper is preserved.
3. **`web/src/components/ui/stat-card.tsx`** -- the inline-block
   loading badge becomes `<Skeleton width="5rem" height="2rem"
   aria-hidden={false} aria-label={`${label} loading`} className="mt-2
   inline-block bg-muted/70" />`. The base `Skeleton` already supplies
   `role="status"`, so `stat-card.test.tsx`'s
   `getByRole('status', { name: '<label> loading' })` keeps matching.

### Sites intentionally left alone

- `web/src/components/ui/progress.tsx` -- the `animate-pulse` stripe
  is an indeterminate progress indicator, not a loading placeholder.
- `web/src/components/ui/image.tsx` -- the `showPulse` pulse is the
  image-decode fade-in animation; replacing it with a `Skeleton`
  would change the semantics.
- `web/src/components/MeetingsList.tsx` -- the row-level shimmer
  carries its own structure already validated by
  `MeetingsList.test.tsx`; swapping requires a deeper refactor that
  is out of scope for this patch.

## Tests

`web/src/components/ui/skeleton.test.tsx` gains 14 cases on top of
the 42 prior cases (Skeleton base + v1.11.135 variants + v1.11.174
shape variants), bringing the file to 56 / 56 passing.

The 14 new cases cover, at minimum:

1. Base `Skeleton` renders with `animate-pulse` + `role=status`.
2. `Skeleton.Text` default 100% width / 1em height.
3. `Skeleton.Text` accepts string + numeric `width`.
4. `Skeleton.Text lines=3` renders 3 line elements (final shortened).
5. `Skeleton.Text lines=1` collapses to a single bar with no wrapper
   line nodes.
6. `Skeleton.Avatar size='sm'` renders a 24 px circle.
7. `Skeleton.Avatar size='md'` renders a 32 px circle.
8. `Skeleton.Avatar size='lg'` renders a 48 px circle.
9. `Skeleton.Avatar` defaults to `md` (32 px).
10. `Skeleton.Card` renders 1 header + 2 text rows inside a bordered
    panel.
11. `Skeleton.Table rows=5 cols=3` renders 1 header + 5 body rows
    with 3 cells per row.
12. `Skeleton.Table` defaults: rows=5, cols=3, 18 total cells.
13. Custom `className` passes through on every sub-component
    wrapper.
14. Every sub-component carries `role=status` + `aria-hidden=true`
    on the wrapper, and caller-supplied `role` overrides the
    default.

## Summary

Added `<ChartLineCciCross />` -- a pure-SVG, dual-panel React/TS
primitive that renders a close line in the top panel and the
Commodity Channel Index (CCI) on the bottom panel. Six threshold
events fire at the momentum extreme transitions:

- `enter100`  -- CCI newly exceeds +100 (overbought entry)
- `exit100`   -- CCI newly drops below +100
- `enterN100` -- CCI newly drops below -100 (oversold entry)
- `exitN100`  -- CCI newly rises above -100
- `zeroUp`    -- CCI crosses 0 upward (bullish bias flip)
- `zeroDown`  -- CCI crosses 0 downward (bearish bias flip)

Defaults: `length = 14`, cciColor `#7c3aed` (violet), upper line
`#dc2626` (red, at +100), lower line `#16a34a` (green, at -100),
zero line `#475569` (slate), enter markers `#dc2626`, exit markers
`#0ea5e9` (sky), zero-up marker green, zero-down marker `#f97316`
(orange).

## Formula

```
typical = (high + low + close) / 3
sma     = SMA(typical, n)
meanDev = (1/n) * sum_{w in window} |typical[w] - sma|
CCI     = meanDev > 0 ? (typical - sma) / (0.015 * meanDev) : null
```

Cross detection uses a prev-value + strict-inequality guard.
Priority order in `detectLineCciCrossEvents`: `+100` transitions
first, then `-100`, then zero. Each bar reports at most one event.

## Bit-exact anchor

- **CONST h = l = close = K**: typical = K everywhere, SMA = K,
  meanDev = 0 -- the divide-by-zero guard leaves CCI permanently
  `null`. Zero events, regime `none` everywhere.

## Anchored CCI value

LINEAR UP with `n = 3` lands the classic CCI = +100 boundary value
in IEEE 754:

```
typical[i] = i + 1
sma[i]     = i  (i.e. (i+1 + i + i-1) / 3 = i)
diff       = 1
meanDev    = (1 + 0 + 1) / 3 = 2/3
CCI        = 1 / (0.015 * 2/3) == 1 / (0.015 * 2/3)
```

The test asserts `cci[i] === 1 / (0.015 * (2/3))` so both sides
evaluate to the same double (about `100.00000000000001`). Because
of that 1-ULP drift the LINEAR UP regime classifies as
`overbought` (cci > 100), and LINEAR DOWN mirrors as `oversold`.
In both cases CCI is constant after warmup, so no events fire
(the cross detector skips the `null -> value` warmup transition).

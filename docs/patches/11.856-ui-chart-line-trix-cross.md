## Summary

Added `<ChartLineTrixCross />` -- a pure-SVG, dual-panel React/TS
primitive that renders a close line in the top panel and the
TRIX (triple-smoothed rate of change) plus its signal-line EMA in
the bottom panel. Markers fire on every TRIX-vs-signal crossover
-- the canonical triple-smoothed momentum turning point.

Cross events:

- `up` cross   -> regime `accelerating-up`
- `down` cross -> regime `accelerating-down`

Defaults: `length = 14`, `signalLength = 9`, trixColor `#0ea5e9`
(sky), signalColor `#f97316` (orange), bullish marker `#16a34a`,
bearish marker `#dc2626`, zero line `#475569`.

## Formula

```
ema1   = EMA(close, length)
ema2   = EMA(ema1,  length)
ema3   = EMA(ema2,  length)
TRIX[i] = ema3[i-1] === 0 ? null
                          : 100 * (ema3[i] - ema3[i-1]) / ema3[i-1]
signal = EMA(TRIX, signalLength)
```

Cross detection uses the standard prev-relation + strict-
inequality guard.

## Bit-exact anchor

The EMA helper carries the SMA-seeded `min === max` precision fix
plus the `next = v === smoothed ? v : ...` CONST short-circuit:

- **CONST close = K (K > 0)**: every EMA in the chain collapses
  to K via the precision fix. ema3 stays K -> `TRIX = 100*0/K =
  0`, signal = EMA(0) = 0. Relation `equal` forever, regime
  `aligned`, zero crosses. Verified across `K in {1, 10, 100}`.

## Soft anchors (sign + cross counts)

- **LINEAR UP close = i+1**: the EMA chain lags but advances at
  the same rate, so `ema3[i] - ema3[i-1]` is a slowly-decaying
  positive constant -> TRIX > 0. Signal = EMA(TRIX) trails TRIX
  in the same direction; signs match and they never cross. Zero
  crosses.
- **LINEAR DOWN close = N-i**: mirror image -> TRIX < 0,
  signal < 0, no crossover. Zero crosses.

All three anchors share the same zero-cross outcome across
`length in {3, 4, 5}`.

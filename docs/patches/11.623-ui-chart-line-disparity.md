## Summary

New `ChartLineDisparity` UI primitive at
`web/src/components/ui/chart-line-disparity.tsx` -- a pure-SVG line
chart with a Disparity Index panel rating the price distance from
its moving average as a percent. The top panel renders the price
line; the bottom panel renders the Disparity Index on a symmetric
data-driven y-axis with a centred dashed zero line and sign-coloured
markers.

## Indicator math

Steve Nison's Disparity Index:

```
SMA[i]       = mean of the trailing window of `period` values
disparity[i] = 100 * (price[i] - SMA[i]) / SMA[i]
```

The index rates how far the price sits from its own simple moving
average as a percentage. It swings around zero -- a positive reading
means the price is above its moving average, a negative reading
means below -- and the size of the swing shows how stretched the
price has become.

The index is defined from index `period - 1` onward (where the SMA
is defined). A zero moving average reads 0 rather than dividing by
zero, and negative-zero is normalized.

### Worked fixture

`values = [6, 9, 15, 6, 9, 15]`, `period 3`. Every three-bar window
sums to 30, so the period-3 SMA is exactly 10 throughout, and the
Disparity Index is `100 * (price - 10) / 10 = 10*price - 100`:

```
sma       = [null, null, 10, 10, 10, 10]
disparity = [null, null, 50, -40, -10, 50]
```

`disparityFinal = 50`, `disparityMin = -40`, `disparityMax = 50`,
two positive readings and two negative. Every value is exact, so the
tests assert the series with `toEqual`.

## API

- `getLineDisparityFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineDisparityPeriod(period, fallback)` -- coerce to a
  positive integer; non-finite or sub-1 falls back, fractional
  floors.
- `computeLineDisparitySma(values, period)` -- the simple moving
  average series.
- `computeLineDisparity(values, period)` -- returns
  `{ sma, disparity }`; both null through the warm-up.
- `runLineDisparity(points, options?)` -- sort finite points by
  `x`, compute the series, return `{ series, period, sma,
  disparity, samples, disparityFinal, disparityMin, disparityMax,
  positiveCount, negativeCount, ok }`. Each sample carries a
  `positive` / `negative` / `zero` sign. `ok` is false for fewer
  than two points.
- `computeLineDisparityLayout(options)` -- stack a price panel
  above a Disparity panel on a symmetric data-driven y-domain.
  Returns panels, ticks, the price path, the Disparity path,
  sign-coloured markers, and the centred zero line `y`.
- `describeLineDisparityChart(data, options?)` -- ARIA description.

## Component

`ChartLineDisparity` is a `forwardRef` region (`role="region"`) with
an ARIA-linked description, a `role="img"` SVG, panel labels, a
config badge, a two-series legend (Price / Disparity) with toggle, a
dashed zero centerline, the Disparity line, markers coloured by sign
(positive green, negative red), a hover/focus tooltip exposing the
value, moving average, Disparity Index and sign, and a
`motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs chart-line-roc** -- the Rate of Change measures the
  percentage change from a *past price* N bars ago. The Disparity
  Index measures the percentage gap from the *moving average* -- a
  different reference point.
- **vs the moving-average overlays** (chart-line-ema,
  chart-line-sma, chart-line-dema, chart-line-tema, ...) -- those
  plot the average itself as a line drawn over the price. The
  Disparity Index plots the price's percentage *deviation* from
  that average as a separate, zero-centred oscillator panel.
- **vs chart-line-ppo** -- the PPO is the percentage difference of
  *two EMAs of the price*. The Disparity Index is the percentage
  difference of the *price itself* from one moving average.

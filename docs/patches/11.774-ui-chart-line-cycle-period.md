# 11.774 chart-line-cycle-period (v1.11.792)

## Summary

Pure-SVG dual-panel React/TypeScript chart primitive with a
**Cycle Period** oscillator panel beneath the close. The period
is measured by counting strict zero crossings of the detrended
close (close minus its rolling SMA) over the lookback window.
Added as the `<ChartLineCyclePeriod />` UI primitive in
`web/src/components/ui/`.

## Algorithm

For each bar with `lookback` (default 30), `smoothLength`
(default 4), and clamp range `[minPeriod, maxPeriod]`
(defaults `[2, 50]`):

```
detrended[i] = close[i] - SMA(close, smoothLength)[i]
crossings[i] = number of strict sign changes in
               detrended[i - lookback + 1..i]
period[i]    = (crossings > 0)
               ? clamp(2 * lookback / crossings, minP, maxP)
               : maxPeriod
```

A strict zero crossing requires consecutive samples on opposite
sides of zero (zeros do not count). Bars before
`i = lookback + smoothLength - 1` are `null` (warmup).

## Anchors

- **CONST_FLAT (close == K)**: SMA = K (bit-exact when K is
  integer or simple dyadic), `detrended = 0` at every bar
  (bit-exact for K = 0; within ULPs for other K but still
  exactly 0 when SMA rounds back to K). No strict crossings
  occur, so the period clamps to `maxPeriod` **bit-exact**.
  Verified across `(K, lookback, maxPeriod) in
  {0, 1, 5, 10, 100, -3} x {20, 30, 40} x {25, 50, 100}` (54
  combos).
- **Rising trend (no oscillation)**: detrended is a constant
  positive lag; no crossings -> period = maxPeriod past warmup.
- **Sinusoid of period P**: the estimated period falls in a
  reasonable band around P after the warmup.
- **Period bounds**: `minPeriod <= period <= maxPeriod` for any
  finite input.
- **Invalid bounds**: when `maxPeriod <= minPeriod`, the
  implementation bumps `maxPeriod` to `minPeriod + 1`.

## Summary

Add a new `<ChartCluster>` UI primitive in
`web/src/components/ui/chart-cluster.tsx` -- pure-SVG
**cluster dendrogram** (sibling of `<ChartDendrogram>`
11.486). Distinct from the plain dendrogram in three
ways:

1. **Distance-aware depth axis**: each node carries an
   optional `distance` (the merge distance in
   hierarchical clustering); the depth axis encodes
   distance rather than uniform depth steps. Leaves
   sit at distance 0; internal nodes sit at their
   merge distance.
2. **Cluster cut line**: optional `cutDistance` prop
   draws a horizontal threshold line; every internal
   node at or below the cut becomes a *cluster root*
   and its descendants share a colour from the cluster
   palette.
3. **Distance axis with ticks**: a vertical axis on
   the left side shows tick marks at evenly-spaced
   distance values so adopters can read merge heights
   directly.

## Behaviour

- Renders `<div role="region">` wrapping
  `<svg role="img">` with the same `aria-label`.
- Distance resolution
  (`getClusterNodeDistance`): explicit
  `distance >= 0` always wins. For leaves without an
  explicit distance, the default is 0 (the canonical
  hclust leaf). For internal nodes without an
  explicit distance, the fallback is
  `(maxDepth - depth) / maxDepth` so the tree
  visually mimics a uniform-spacing dendrogram in
  the absence of true cluster heights.
- Layout (`computeClusterLayout`):
  - Leaves laid evenly along the x axis at
    `leafIndex * (innerW / (leafCount - 1))`.
  - Each internal node's x is the *mean* of its
    children's x positions.
  - y is derived from distance: leaves at
    `padY + innerH - 0 = padY + innerH`; internal
    nodes at `padY + innerH - (distance /
    maxDistance) * innerH`.
  - Links are right-angle elbows: horizontal from
    parent x to child x at parent's y, then
    vertical down to child's y. This is the
    canonical cluster dendrogram style and
    visually distinguishes vertical drops at merge
    heights.
- Cluster cut (`cutDistance`):
  - If set and `> 0`, walks the tree and marks
    every internal node with `distance <=
    cutDistance` as a cluster root. Descendants of
    that root share the cluster id.
  - Each cluster id gets its own colour from
    `DEFAULT_CHART_CLUSTER_PALETTE` (modulo wrap).
  - Nodes above the cut keep their default leaf /
    internal colour. A dashed horizontal cut line +
    label render at y(cutDistance) when
    `showCutLine=true`.
- Distance axis (`showDistanceAxis=true`, default):
  vertical axis on the left at
  `padX + axisReserve` with evenly-spaced ticks
  from 0 to `maxDistance`. Tick labels formatted by
  `formatDistance`.
- Leaf labels render rotated 45 deg below each
  leaf so long labels don't overlap on a dense
  axis; toggleable via `showLeafLabels`. Internal
  labels are hidden by default (`showInternalLabels`).
- Hover/focus on a node opens a tooltip with the
  label, distance, leaf / leafCount, and the
  cluster id when one is assigned. Hover/focus on a
  link opens a link tooltip.
- Mount animation:
  `motion-safe:animate-fade-in` per node + link.
- Default ARIA description: `"Cluster dendrogram
  with N nodes, L leaves, max distance D[, cut at
  K]."` Null root -> `"No data"`.

  the leaf count, the max distance, the axis ticks,
  and the ordered list of cluster root ids
  (`clusterIds`).

# 11.1079 -- UI: chart-line-volume-trend-cross primitive (v1.11.1097)

## Scope

Add a pure-SVG dual-panel `ChartLineVolumeTrendCross`
component that renders the close in the top panel and a
volume-weighted moving average (VWMA) overlaid against a
simple moving average (SMA) of the same period in the
bottom panel, marking bullish (VWMA crosses up through
SMA -- volume confirms uptrend; buying volume
concentrated at higher prices) / bearish (VWMA crosses
down through SMA -- volume confirms downtrend; selling
volume concentrated at lower prices) volume-weighted
trend crossover trigger events with bias coloring derived
from the (VWMA - SMA) slope at the trigger bar.

The VWMA-vs-SMA gap reveals whether volume is concentrated
at higher prices (VWMA > SMA) or lower prices (VWMA < SMA)
within the window. A cross marks "trend confirmation by
volume regime change" -- the volume-weighted price has
overtaken (or fallen behind) the unweighted average.

## Volume trend cross pipeline

```
VWMA[i] = sum(close[j] * volume[j]) / sum(volume[j])
          over j in [i-period+1 .. i]
          (null when window volume sum is 0)
SMA[i]  = sum(close[j]) / period

bullish (VWMA-cross up) :
  prev VWMA <= prev SMA && cur VWMA > cur SMA
bearish (VWMA-cross down) :
  prev VWMA >= prev SMA && cur VWMA < cur SMA

regime : 'bullish' when VWMA >= SMA
         'bearish' when VWMA <  SMA
         'none'    when either is null
bias   : (VWMA - SMA)[i] vs prev -> up/down/flat/none
```

Default `period = 20`.

## Sibling family (volume-input family)

- **chart-line-volume-spike-cross v1.11.1064** -- volume
  vs its moving-average spike threshold (the first
  volume-input primitive)
- **chart-line-vwap-cross / vwap-cross-sig** -- price vs
  cumulative session VWAP
- **chart-line-vwma** -- raw VWMA overlay
- **chart-line-volume-trend-cross v1.11.1097** (this
  primitive) -- VWMA vs SMA cross (volume-weighted trend
  confirmation)

Distinct from the VWAP-cross family: VWAP is a cumulative
(session-anchored) volume-weighted average, whereas the
VWMA here is a rolling `period`-window volume-weighted
average compared against the matching rolling SMA. The
VWMA-vs-SMA cross isolates the volume-weighting effect
(both use the same window) so the cross is purely a
function of how volume is distributed across prices
within that window.

Warmup is `period` for the first detectable cross: VWMA
and SMA both seed at i = period - 1, and cross detection
needs the previous bar, so the first potential cross
lands at i = period.

## Bit-exact anchors

All anchors use close + volume input.

- **CONST** `close = K`, `volume = V` (V > 0): VWMA =
  sum(K*V) / sum(V) = K. SMA = K. VWMA === SMA every bar
  -> no cross. regime `bullish` (via >=). 0 crosses.
  Verified across K in {0, 1, 50, 200, 1234}.
- **CONST, zero volume** `close = K`, `volume = 0`:
  window volume sum is 0 -> VWMA = null (SMA still =
  K). regime `none`. 0 crosses. Verifies the
  divide-by-zero guard.
- **LINEAR UP, CONST volume** `close = i`, `volume = V`:
  with constant volume the weights cancel, so VWMA = SMA
  = mean(i-period+1 .. i) **exactly**. VWMA === SMA every
  bar -> no cross. regime `bullish` (via >=). 0 crosses.
  This anchors the constant-volume identity VWMA === SMA,
  which holds because sum(c*V)/sum(V) = V*sum(c)/(V*n) =
  sum(c)/n = SMA for any constant V.
- **LINEAR UP, RISING volume** `close = i`, `volume =
  i + 1`: higher prices carry proportionally more
  volume, so VWMA > SMA (weighted toward the higher
  recent prices). The gap is positive and stable -> no
  cross. regime `bullish`. 0 crosses.
- **VOLUME REGIME SHIFT**: alternating price (100 / 110)
  with volume concentrated on the LOW bars for the first
  half (VWMA < SMA, bearish) then on the HIGH bars for
  the second half (VWMA > SMA, bullish) -> a bullish
  cross as the window slides into the second regime. At
  least one cross fires, and both bullish and bearish
  regime bars are present.

The constant-volume identity (VWMA === SMA) is the
defining property: the signal is SILENT in flat-volume
regimes and fires only when volume VARIES and its
distribution across prices shifts. This makes
volume-trend-cross fundamentally a volume-dispersion
detector, not a price detector.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 3-state regime classifier follows the cross-sig family
  convention (bullish on `===` via `>=`).
- Zero-volume guard: when the window volume sum is 0,
  VWMA returns null (SMA still computes). This prevents
  NaN on illiquid / synthetic flat-volume input.
- The constant-volume identity (VWMA === SMA) is exact
  in floating point because the algorithm factors the
  constant volume out symmetrically -- no accumulated
  rounding divergence. The LINEAR-UP-const-vol anchor
  verifies this to 9 decimal places.
- Second member of the volume-input family (after
  volume-spike-cross). Unlike the spike-cross which
  thresholds volume magnitude, this primitive measures
  volume DISTRIBUTION across prices -- a complementary
  volume signal.
- The signal is genuinely silent on flat-volume input
  (a feature: no false signals when volume carries no
  information), and only activates on volume regime
  changes, matching the dispatch intent ("trend
  confirmation by volume regime change events").

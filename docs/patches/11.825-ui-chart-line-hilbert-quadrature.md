# Patch 11.825 -- ChartLineHilbertQuadrature primitive (v1.11.843)

## Summary

Add `<ChartLineHilbertQuadrature />`, a pure-SVG dual-panel line chart
plotting the Hilbert Transform Quadrature component beneath a price
line. The quadrature is approximated by FIR-detrending the close
(subtracting the equal-weight SMA over the cycle length) and shifting
the result by the quarter-cycle delay `floor(length / 4)`, yielding a
90-degree phase-shifted view of the detrended price.

## Formula

```
sma[i]     = mean(close[i - length + 1 .. i])
detrend[i] = close[i] - sma[i]
quad[i]    = detrend[i - shift]   where shift = floor(length / 4)
```

- Combined warmup: `length - 1 + shift` indices are `null`.
- Null propagation: any `null` SMA or detrend slot propagates through.
- `-0` is normalized to `+0` on every arithmetic result so Object.is
  comparisons remain clean across negative inputs.

## Defaults

| Knob       | Value     |
|------------|-----------|
| length     | 20        |
| threshold  | 0         |
| quadColor  | `#0891b2` (cyan) |

Length defaults to 20 (a common Ehlers cycle assumption); the shift is
`floor(20 / 4) = 5`. Threshold = 0 means zone classification is
strictly above / below / equal to zero, with zero-crossings driving
the up/down cross detector.

## Bit-exact anchors

The integration sweep iterates `(K, length)` and asserts:

- `CONST close = K` -> `sma = K` -> `detrend = 0` -> `quad = 0`
  post-warmup. Subtraction of identical values is exact in IEEE 754.
- `LINEAR UP close = i + 1`, length L: SMA reduces to
  `(i + 1) - (L - 1) / 2`, so detrend = `(L - 1) / 2`. For
  L in {4, 8, 16, 20} the constant is dyadic (1.5, 3.5, 7.5, 9.5) and
  exact under IEEE 754. The quarter-cycle delay just lags the
  constant by `shift` indices, preserving the value bit-exactly.
- `LINEAR DOWN close = N - i`: detrend = `-(L - 1) / 2`, same exact
  dyadic constant negated. Quad inherits the value.

These three anchors are stable across every `length` ≥ 4 because the
closed-form SMA of an arithmetic progression cancels the per-index
linear term exactly, leaving a constant that the rolling-mean sum can
represent without rounding.

## Surface

Standard primitive surface:

- `ChartLineHilbertQuadraturePoint = { x, close }`
- `runLineHilbertQuadrature(data, options?) -> Run` with
  `ok = data.length >= length + shift` post-warmup gate.
- `computeLineHilbertQuadratureLayout({ data, width?, height?, padding?, panelGap?, length?, threshold? }) -> Layout`.
  Quad axis auto-fits but always includes `0`.
- `describeLineHilbertQuadratureChart(data, opts?) -> string` for the
  ARIA description.
- `<ChartLineHilbertQuadrature />` `forwardRef<HTMLDivElement>` with
  legend toggle (`price`, `quadrature`), config badge, zero line,
  optional threshold band, markers on cross events, and a tooltip
  surfacing `close / sma / detrend / quadrature / zone / cross`.

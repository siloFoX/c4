# Patch 11.215 -- UI empty-state illustrations refresh (v1.11.233)

Component-scope-only refresh of the web empty-state illustration set.
No daemon-side change, no `c4` CLI surface change.

## Summary

- Standardised the illustration size surface with a named-token
  convention -- `size='sm' | 'md' | 'lg'` resolves to 64 / 96 / 128 px
  respectively. Legacy numeric `size={N}` keeps working unchanged so
  the four v1.11.84 hero illustrations (`EmptyQueueIllustration`,
  `NoWorkersIllustration`, `WelcomeOnboardingIllustration`,
  `AllDoneIllustration`) retain their default 160 px size and their
  full existing test suite passes untouched.
- Documented the tone convention in `illustrations/index.tsx`: outlines
  paint through `currentColor` (consumer drives via
  `text-muted-foreground`) and highlights use `hsl(var(--primary) / 0.3)`
  -- the SVG-equivalent of the `text-primary/30` utility -- via the new
  exported `HIGHLIGHT_FILL` constant. The legacy hero set keeps its
  lighter 0.15 accent to avoid an unintended contrast shift.
- Added two new illustrations under
  `web/src/components/illustrations/index.tsx`:
  - **`SearchEmpty`** -- a magnifying glass tilted over a stack of
    three blank list rows. For search-with-no-results states.
  - **`SessionsEmpty`** -- three stacked terminal cards with a dashed
    "no connection" slash across them. For sessions-not-attached
    states.
  Both default to the `md` (96 px) token and honour `size`,
  `className`, and `aria-hidden` exactly like the v1.11.84 set.

## Adoption sites

- `HistoryView.tsx`: the sidebar list now renders `<SearchEmpty />`
  plus the existing `history.empty` copy when the search input is
  non-empty and produces zero filtered rows. The pre-existing
  "no history yet" branch (`summary.length === 0`) is unchanged.
- `SessionsEmptyAttachBanner.tsx`: the attached-sessions empty banner
  now ships the `<SessionsEmpty size="sm" />` illustration inline
  alongside the existing Info icon + CTA button. The illustration is
  decorative (`aria-hidden`) and `hidden sm:block`, so the compact
  mobile layout is unchanged.

## Tests

- `web/src/components/illustrations/new-illustrations.test.tsx` (new,
  ~95 lines) covers:
  - `SearchEmpty` + `SessionsEmpty` render an `<svg>` with the shared
    `0 0 240 180` viewBox.
  - The named tokens `sm` / `md` / `lg` map to `width`/`height`
    `64` / `96` / `128` and the `md` token is the default when `size`
    is omitted.
  - Numeric `size={N}` still passes through verbatim.
  - Decorative-by-default (`aria-hidden="true"`, no `role`) and the
    flip to `role="img"` + `aria-label` when `aria-hidden={false}`.
  - `className` forwards to the root `<svg>`.
  - Shape contracts: `SearchEmpty` ships >= 1 `<circle>` (the lens)
    and >= 3 `<rect>` rows; `SessionsEmpty` ships >= 3 `<rect>` cards
    and >= 1 dashed `<line>` (the no-connection slash).
  - The `resolveIllustrationSize` resolver passes through numbers,
    maps tokens, and respects the fallback.
- `web/src/components/illustrations/illustrations.test.tsx` is
  untouched -- the v1.11.84 hero suite continues to pass.
- `web/src/components/SessionsEmptyAttachBanner.test.tsx` updates the
  decorative-icon-count assertion from `2` to `3` to account for the
  newly adopted `SessionsEmpty` svg (still `aria-hidden`).

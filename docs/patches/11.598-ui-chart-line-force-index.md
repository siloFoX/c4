## Summary

New `ChartLineForceIndex` UI primitive at
`web/src/components/ui/chart-line-force-index.tsx` -- a pure-SVG line
chart with an Elder Force Index panel combining price change and
volume. The top panel renders the price line; the bottom panel
renders the smoothed Force Index line and a faint raw Force Index
line on a symmetric y-axis with a dashed zero centerline and
sign-coloured markers.

Like `chart-line-mass-index`, this primitive consumes a multi-field
point -- here `{ x, price, volume }` -- since the Force Index is
intrinsically a price-and-volume study.

## Indicator math

Alexander Elder's Force Index, with an EMA period (default 13):

```
rawForce[i]   = (price[i] - price[i-1]) * volume[i]
forceIndex    = EMA(rawForce, emaPeriod)
```

`computeLineForceIndexRaw(prices, volumes)` multiplies each bar's
price change by its volume; index 0 has no prior price so it reads
`null`. A negative-zero raw force (a zero price change times a
negative volume) is normalised to positive zero.

`computeLineForceIndexEma(src, period)` is an EMA that tolerates the
leading `null` of the raw series: it seeds with the simple mean of
the first `period` defined values, then folds each later defined
value in at weight `2 / (period + 1)`.

The smoothed Force Index measures the power behind a move -- it
swings around zero, positive when buyers dominate and negative when
sellers do.

### Worked fixture

`prices = [10, 12, 16, 16, 14, 14]`, `volumes = [100, ...] (all 100)`,
`emaPeriod = 2`:

```
rawForce   = [null, 200, 400, 0, -200, 0]
forceIndex = [null, null, 300, 100, -100, -100/3]   (EMA mult 2/3, SMA seed at index 2)
```

`forceFinal = -100/3`, `rawFinal = 0`, `forceMin = -100`,
`forceMax = 300`, two readings above and two below zero.

## API

- `getLineForceIndexFinitePoints(points)` -- drop points with a
  non-finite `x`, `price` or `volume`.
- `normalizeLineForceIndexPeriod(period, fallback)` -- positive
  integer; non-finite / sub-1 falls back, fractional floors.
- `computeLineForceIndexEma(src, period)` -- the null-tolerant EMA.
- `computeLineForceIndexRaw(prices, volumes)` -- the raw Force Index
  series (`(number | null)[]`).
- `computeLineForceIndex(prices, volumes, emaPeriod)` -- returns
  `{ rawForce, forceIndex }`.
- `runLineForceIndex(points, options?)` -- sort finite points by `x`,
  compute the series, return `{ series, emaPeriod, rawForce,
  forceIndex, samples, forceFinal, rawFinal, forceMin, forceMax,
  positiveCount, negativeCount, ok }`. Each sample carries a
  `positive` / `negative` / `zero` sign from the smoothed Force
  Index. `ok` is false for fewer than two points.
- `computeLineForceIndexLayout(options)` -- stack a price panel above
  a Force Index panel with a symmetric y-bound covering the raw and
  the smoothed readings. Returns panels, ticks, the price path, the
  raw path, the smoothed path, sign-coloured markers, and the zero
  centerline `y`.
- `describeLineForceIndexChart(data, options?)` -- ARIA description.

## Component

`ChartLineForceIndex` is a `forwardRef` region (`role="region"`) with
an ARIA-linked description, a `role="img"` SVG, panel labels, a
config badge (EMA period), a three-series legend (Price / Force Index
/ Raw Force) with toggle, a dashed zero centerline, a faint raw Force
Index line, the smoothed Force Index line, markers coloured by sign,
a hover/focus tooltip exposing the price, volume, raw force, smoothed
force, and sign, and a `motion-safe:animate-fade-in` entrance. Every
node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-obv** -- On-Balance Volume is a *cumulative running
  total*: it adds the whole volume on up bars and subtracts it on
  down bars, so its magnitude grows without bound and it weights only
  the *direction* of each move. The Force Index is not cumulative and
  weights by the *extent* of each price change, then EMA-smooths the
  result.
- **vs chart-line-mfi** -- the Money Flow Index is a *bounded 0-100*
  volume-weighted oscillator (a ratio of positive to negative money
  flow). The Force Index is *unbounded* and centred on zero.
- **vs chart-line-ppo / chart-line-coppock** -- those are price-only
  momentum studies. The Force Index is the volume-weighted member of
  the family: only it multiplies the price change by volume.

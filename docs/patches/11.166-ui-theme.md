# 11.166 -- UI theme toggle (v1.11.184)

## Summary

Adds `useTheme()` + `<ThemeToggle>` so the web UI honours an explicit
user preference (light / dark) or follows the OS via
`prefers-color-scheme`. Wired into `AppHeader` next to the existing
Help / Shortcuts / Language actions.

## Files

- `web/src/hooks/use-theme.ts` -- hook (Theme / ResolvedTheme types,
  `useTheme()` returning `{ theme, setTheme, resolvedTheme }`).
- `web/src/components/ThemeToggle.tsx` -- cycle (default) or group
  variant button(s) using lucide Sun / Moon / Monitor icons.
- `web/src/hooks/use-theme.test.ts` -- 9 vitest cases (default, init
  from storage, invalid fallback, setTheme dark/light, persistence,
  system matchMedia tracking, explicit theme ignores OS changes).
- `web/src/components/ThemeToggle.test.tsx` -- 7 vitest cases (cycle
  aria-label, cycle order, mount icon, className merge, group
  3-button render, group click sets theme, group aria-pressed).
- `web/src/components/layout/AppHeader.tsx` -- adds `<ThemeToggle
  size="sm" />` in the actions slot before the Language button.

## Storage key

`c4:theme` in `window.localStorage`. Valid values: `'system'`,
`'light'`, `'dark'`. Any other value (or an absent / unreadable
entry) resolves to `'system'`. `setTheme` writes on every call
including `'system'` so an explicit choice survives subsequent
visits.

## DOM contract

On every resolved-theme change the hook:

1. Adds the `dark` class to `document.documentElement` when resolved
   theme is `'dark'`, removes it when `'light'`.
2. Sets `data-theme="<resolved>"` on `document.documentElement` for
   stylesheets that prefer attribute selectors over class selectors.

Tailwind config already had `darkMode: ['class']` so existing `dark:`
utilities pick up the class automatically -- no `tailwind.config.js`
edit was needed.

## Integration note

The toggle is rendered in `AppHeader.tsx` actions slot, between the
Shortcuts button and the Language button, using the default `cycle`
variant + `size='sm'` so it matches the surrounding `IconButton`
sizing. The group variant is exported for future preference panels
that want all three options visible at once.

## SSR

All `window` / `document` access is guarded so the module can be
imported in a non-browser environment without throwing. Initial
state falls back to `'system'` + `'light'` when no `window` is
present.

# 11.527 -- web/src/components/ui/chart-line-velocity.tsx

Pure-SVG multi-series line chart with **arrow markers at every
point** indicating velocity direction (up / down / right) and
optionally angled along the local tangent. Each arrow's color
reflects its direction (green for up, red for down, slate for
flat), making the visual flow of the data immediately obvious.

Two arrow modes:

- `'tangent'` (default): arrow tip at the point, pointing along the
  pixel-space slope from the previous point. The arrow visually
  follows the line.
- `'sign'`: arrow tip at the point, pointing straight up / down /
  right based on the sign of `dy`. Cleaner glyphs at the cost of
  losing the local-slope cue.

## Distinct from other rate / marker primitives

- `<ChartLineRate>` (11.521): plots the derivative as a separate
  line chart. Velocity decorates the original line with directional
  arrows.
- `<ChartLineMinMax>` (11.517): triangle markers at the absolute
  extremes only. Velocity puts an arrow at every point.
- `<ChartLineMarker>` (11.506): configurable static markers
  (circle / square / triangle / etc) per point. Velocity arrows
  *encode direction* dynamically from the data.
- `<ChartLineEvent>` (11.515): event icons on a dedicated track at
  specific timestamps. Velocity is per-point and inline with the
  line.
- `<ChartLineCumulative>` (11.520): running sum transform.
- `<ChartLineAnomaly>` (11.524): z-score outliers.

The canonical "where is this metric heading" visualisation -- adds
a glanceable directional cue to a standard line chart without
needing to read x or y values.

## Public surface

- `<ChartLineVelocity series={...} arrowMode="tangent" />`.
- Per-series `{id, label, data, color?, positiveColor?,
  negativeColor?, zeroColor?, arrowMode?}`. `data` entries are raw
  `{x, y}`.
- `arrowMode` chart-level default; per-series override always wins.
- `arrowSize` (default `8`) tunes the chevron stroke length.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showArrows`, `showLine`, `animate`.
- Standard sizing + format + a11y props.
- `formatMagnitude(n)` for the tooltip magnitude row (falls back to
  `formatValue`).
- Controlled + uncontrolled `hiddenSeries`.
- `onPointClick({series, point})`.

## Pure helpers

- `getLineVelocityDefaultColor(index)` -- 10-color palette cycle.
- `getLineVelocityFinitePoints(points)` -- finite-only filter.
- `computeVelocity(prev, curr)` -- returns `{dx, dy, magnitude,
  angle, direction}` or `null` on missing / non-finite inputs.
  `angle` is in DATA space (math convention: 0 is +x, +pi/2 is
  +y). `direction = 'up' | 'down' | 'right'` based on the sign of
  `dy`.
- `buildLineVelocityArrowPath(tipX, tipY, angleRadians, size,
  wingDegrees?)` -- canonical chevron path builder. Emits two
  stroke segments (no fill needed) from the tip backward at
  `+/- wingDegrees` (default `150 deg`) from the reverse direction.
  Non-finite or `size <= 0` returns `''`.
- `computeLineVelocityLayout({series, arrowMode?, arrowSize?,
  hiddenSeries?, ...})` -- returns `{series, xTicks, yTicks, xMin,
  xMax, yMin, yMax, innerWidth, innerHeight, totalPoints,
  visibleSeriesCount}`. Each layout series carries `{id, label,
  index, color, positiveColor, negativeColor, zeroColor,
  arrowMode, points, path, stats, finiteCount, totalCount}`.
  Per-point `{index, x, y, px, py, velocity, arrowAngle,
  arrowPath, arrowColor}`. Stats: `{finiteCount, upCount,
  downCount, flatCount, maxMagnitude, meanMagnitude}`.
- `describeLineVelocityChart(series, arrowMode?, hidden?,
  formatValue?)` -- builds the screen-reader description.

## Arrow rendering

In SVG pixel space the y axis is inverted (down is +y visually), so
the tangent mode computes the arrow angle from the **pixel-space**
segment delta (`atan2(dyp, dxp)`) — this makes the arrow visually
align with the slope of the line on screen. The sign mode uses
pixel angles `-PI/2` (up), `+PI/2` (down), and `0` (right).

The first point of each series has no previous neighbour, so it
gets no arrow (`arrowPath: ''`, `arrowAngle: null`). Tooltips for
the first point show `velocity: start of series`.

## Tooltip

Hovering a dot shows the series label, x, y (bold), and a coloured
`velocity:` row showing direction + magnitude. The first point
shows `velocity: start of series`.

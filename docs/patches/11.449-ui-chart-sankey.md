## Summary

Add a new `<ChartSankey>` UI primitive in
`web/src/components/ui/chart-sankey.tsx` -- pure-SVG
Sankey diagram. Nodes arrange in vertical columns (layers)
according to longest path from a source node; flows
between nodes paint as smooth cubic bezier ribbons whose
width is proportional to flow value. Hovering a node
highlights its upstream + downstream paths so adopters can
read conversion / energy / cost flow at a glance. Per-node
+ per-link colour overrides; per-link click + per-node
click callbacks.

## API

```ts
interface ChartSankeyNode {
  id: string;
  label: string;
  color?: string;
  category?: string;
}

interface ChartSankeyLink {
  source: string;     // node id
  target: string;     // node id
  value: number;
  color?: string;
}

interface ChartSankeyProps {
  nodes: readonly ChartSankeyNode[];
  links: readonly ChartSankeyLink[];
  width?: number;                    // default 600
  height?: number;                   // default 320
  nodeWidth?: number;                // default 14
  nodePadding?: number;              // default 10
  showLabels?: boolean;              // default true
  showTooltip?: boolean;             // default true
  showValues?: boolean;              // default true (in labels)
  animate?: boolean;                 // default true
  className?: string;
  ariaLabel?: string;                // default 'Sankey diagram'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  onNodeClick?: (args: { node; value }) => void;
  onLinkClick?: (args: { link; source; target }) => void;
  highlightOnHover?: boolean;        // default true
}
```

## Behaviour

- Layer assignment: `getSankeyNodeLayers` runs a
  topological pass. Sources (no incoming links) sit at
  layer 0; every other node sits at
  `max(layer(u) + 1)` over its incoming links. Any
  remaining nodes after one full pass (i.e., cycles)
  park at the next layer so the diagram still renders.
- Node geometry: each node's height is proportional to
  `max(sum-of-inflow, sum-of-outflow)`. Padding budget
  (`nodePadding` between stacked nodes) is subtracted
  from the column height before scaling so the heights
  always fit. All columns share the same scale
  (`maxLayerTotal`), giving a visually balanced layout
  across layers.
- Link geometry: each link is a closed cubic bezier
  ribbon (`buildSankeyLinkPath`) between
  `(x1, sourceY)` and `(x2, targetY)` with constant
  thickness. The thickness is the larger of the
  source-side and target-side projections of the link's
  value so the ribbon never disappears at the narrow
  side.
- Hover highlight: hovering a node sets `data-hovered`
  on that node and `data-highlighted` on every upstream
  and downstream node + every link that touches them.
  Other elements fade to a low fill-opacity (0.12 link,
  0.35 node) so the highlighted path reads first. Set
  `highlightOnHover=false` to render flat opacity.
- Tooltips: hovering a node opens the node tooltip
  (label + value); hovering a link (without an active
  node hover) opens the link tooltip with
  `source -> target` plus the formatted value.
- Default ARIA description: `"Sankey with N nodes, M
  flows. <node> <value>, ..."` -- `ariaDescription`
  overrides; empty data produces `"No data"`.
- Mount animation: `motion-safe:animate-fade-in` per
  node group + per link. `animate=false` removes the
  class.

## Pure helpers (exported)

```ts
DEFAULT_CHART_SANKEY_WIDTH         = 600
DEFAULT_CHART_SANKEY_HEIGHT        = 320
DEFAULT_CHART_SANKEY_NODE_WIDTH    = 14
DEFAULT_CHART_SANKEY_NODE_PADDING  = 10
DEFAULT_CHART_SANKEY_NODE_COLOR    = '#475569'
DEFAULT_CHART_SANKEY_LINK_COLOR    = '#94a3b8'

getSankeyNodeLayers(nodes, links): Map<string, number>
getSankeyNodeValue(nodeId, links): number
findUpstreamNodes(nodeId, links): Set<string>
findDownstreamNodes(nodeId, links): Set<string>
buildSankeyLinkPath(x1, x2, sourceY, targetY, thickness): string
describeSankeyChart(nodes, links, formatValue?): string
```

- `getSankeyNodeLayers` is cycle-safe (parks unresolved
  nodes after the highest known layer).
- `getSankeyNodeValue` returns
  `max(sum-inflow, sum-outflow)`; non-finite or non-
  positive link values are ignored.
- `findUpstreamNodes` / `findDownstreamNodes` walk the
  link graph defensively (visited-set guards against
  cycles); the queried node is excluded from the result.
- `buildSankeyLinkPath` emits a closed cubic bezier
  ribbon with one `M`, two `C` curves, one `L`, and a
  `Z`.
- `describeSankeyChart` produces the default ARIA
  summary string.

## ARIA + data attributes

- Root: `<div role="region">` + `aria-label`.
- SVG: `role="img"` + same `aria-label`.
- sr-only span:
  `data-section="chart-sankey-aria-desc"` carries the
  auto-summary or `ariaDescription` override.
- Each node rect: `role="graphics-symbol"` +
  `tabIndex={0}`. Each link path: `role="graphics-symbol"`
  + `tabIndex={0}`. Focus opens the tooltip via
  `onFocus`.
- `data-section` on every node:
  - `chart-sankey` (root)
  - `chart-sankey-aria-desc`
  - `chart-sankey-svg`
  - `chart-sankey-node` (per-node group)
  - `chart-sankey-node-rect`
  - `chart-sankey-label`
  - `chart-sankey-link` (per-link path)
  - `chart-sankey-node-tooltip` + `-tooltip-label` /
    `-tooltip-value`
  - `chart-sankey-link-tooltip` + `-link-tooltip-label`
    / `-link-tooltip-value`
- Root mirrors `data-node-count`, `data-link-count`,
  `data-animate`. Node groups mirror `data-node-id`,
  `data-layer`, `data-hovered`, `data-highlighted`.
  Links mirror `data-source-id`, `data-target-id`,
  `data-link-index`, `data-hovered`, `data-highlighted`.

## Tests (51 cases)

  layer 2 sinks, empty input, cycle-safe parking.
- `getSankeyNodeValue`: source sum, sink sum, mid
  max(in,out), isolated node = 0, non-finite +
  non-positive link values ignored.
- `findUpstreamNodes`: full ancestry walk, empty for
  sources, excludes self, cycle safety.
- `findDownstreamNodes`: full descendant walk, empty
  for sinks, excludes self.
- `buildSankeyLinkPath`: closed path with 2 C curves.
- `describeSankeyChart`: "No data" for empty, node +

## Out of scope

- Iterative crossing-minimisation between layers (basic
  declaration-order layout for v1; host can sort the
  `nodes` array to influence stacking).
- Drag-to-reorder nodes (host owns the array; render
  is read-only).
- Animated value transitions on data change (snap v1;
  mount fade only).
- Negative / signed flows (host normalises before
  passing; the primitive drops non-positive link
  values).

# Patch 11.742 -- UI: chart-line-relative-vigor primitive

Released as `v1.11.760` on 2026-05-26.

## Scope

Add `<ChartLineRelativeVigor>` -- a pure-SVG two-panel chart with a
John Ehlers **Relative Vigor Index** panel reading the smoothed
close-minus-open over the smoothed high-minus-low ratio.

## Definition

Inputs:

- `data: ChartLineRelativeVigorPoint[]` where
  `ChartLineRelativeVigorPoint = { x; open; high; low; close }`.
  Bars with `high < low` are dropped.
- `period: number` -- the SMA lookback after the SWMA pass.
  Default `10`. Normalized to an integer `>= 2`.
- `threshold: number` -- the +/- strong-zone boundary. Default `0.5`.

Algorithm (per bar `i`):

```
num_raw[i] = close[i] - open[i]
den_raw[i] = high[i]  - low[i]

SWMA(x, i) = (x[i] + 2 * x[i - 1] + 2 * x[i - 2] + x[i - 3]) / 6

num_smooth[i] = SWMA(num_raw, i)
den_smooth[i] = SWMA(den_raw, i)

RVI[i]    = SMA(num_smooth, period) / SMA(den_smooth, period)
Signal[i] = SWMA(RVI, i)
```

The SWMA needs three prior bars (defined at `i >= 3`); the SMA pass
needs `period - 1` more. So the RVI track is defined at `i >=
period + 2`. The Signal track needs three more bars on top.

## Bit-exact algebraic anchors

**SWMA-of-constant**: integer weights `1 / 2 / 2 / 1` sum to `6`,
so `SWMA(c, c, c, c) = (c + 2c + 2c + c) / 6 = 6c / 6 = c` exactly.
This is the key lemma.

**SMA-of-constant**: `SMA(c, c, ..., c) = period * c / period = c`
exactly.

**CONSTANT_UP** (`open = 10, close = 20, high = 20, low = 10` at
every bar):

- `num_raw = 20 - 10 = 10` (constant), `den_raw = 20 - 10 = 10`
  (constant).
- `num_smooth = den_smooth = 10` bit-exact (SWMA of constant).
- After the SMA pass, both averages are `10` bit-exact.
- `RVI = 10 / 10 = 1` bit-exact at every defined bar.
- `Signal = SWMA(1) = 1` bit-exact once warm.

**CONSTANT_DOWN** (`open = 20, close = 10`): `num_raw = -10`, same
denominator, so `RVI = -1` bit-exact.

**CONST_FLAT** (`open == high == low == close == K`): `num_raw =
den_raw = 0`. SWMA of zero is zero, SMA of zero is zero. The
denominator falls below `CHART_LINE_RELATIVE_VIGOR_EPSILON` and the
bar is null at every position.

**SWMA worked anchor** `[1, 2, 3, 4]`:

```
SWMA[3] = (4 + 2 * 3 + 2 * 2 + 1) / 6 = (4 + 6 + 4 + 1) / 6 = 15 / 6 = 2.5
```

Bit-exact dyadic.

**SMA integer-mean anchor**: `SMA([2, 4, 6, 8], 4) = 20 / 4 = 5`
bit-exact.

**Translation invariance**: shifting every OHLC field by a
constant `k` leaves `close - open` and `high - low` unchanged, so
the RVI and Signal are unchanged bit-exact for any `k`.

## Component

`forwardRef<HTMLDivElement, ChartLineRelativeVigorProps>` with
displayName. Renders:

- A region-role outer `<div data-section="chart-line-relative-vigor">`.
- An `<svg role="img">` with an `aria-describedby` description.
- Top panel: close line, optional dots.
- Bottom panel: RVI line, dashed Signal line, zone-coloured
  markers (strong-up / up / flat / down / strong-down / none),
  zero reference line, dashed `+/- threshold` reference lines.
- Legend toggle for `price` / `rvi` / `signal`, plus `showRvi` /
  `showSignal` props and an `onPointClick` handler.
- ARS design-system tokens for colour and motion (`motion-safe:
  animate-fade-in`).

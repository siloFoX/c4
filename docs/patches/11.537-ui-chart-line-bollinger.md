# 11.537 -- web/src/components/ui/chart-line-bollinger.tsx

Pure-SVG **Bollinger Bands** technical indicator (John
Bollinger, 1980s). Each series renders against three coordinated
**rolling** rules driven by a sliding window of size `window`
(default 20): the middle band is the rolling SMA (dashed) and the
upper / lower bands are at `SMA +- kSigma * rolling_std` (default
k = 2 sigma, the canonical Bollinger configuration). The region
between upper and lower is shaded with the band color, and points
that pierce the bands ("breakouts") are tinted green when above
or red when below at a slightly larger radius. A breakout badge
in the top-left tells operators how many points sit outside the
envelope.

## Distinct from other reference / outlier primitives

- `<ChartLineConfidence>` (11.502): per-point upper / lower
  bounds supplied EXTERNALLY (e.g. from a model). Bollinger
  DERIVES the bounds statistically from a ROLLING window over
  the same series.
- `<ChartLineMovingAvg>` (11.513): SMA overlay only. Bollinger
  adds the `+/-kSigma` envelope around the SMA.
- `<ChartLinePercentile>` (11.514): distribution percentile
  bands per x bucket -- requires MULTI-SAMPLE input (`{x,
  values[]}`). Bollinger uses single-sample input with rolling
  stats.
- `<ChartLineAnomaly>` (11.524): per-point z-score outlier
  highlighting (optional rolling window). Bollinger surfaces
  the BANDS as a visible envelope and tags every breakout as
  part of a continuous signal; anomaly emphasizes isolated
  outliers.
- `<ChartLineControl>` (11.534): SPC chart with GLOBAL mean +/-
  k*sigma (single set of stats over the whole series).
  Bollinger is ROLLING -- the bands track LOCAL volatility, so
  the same threshold means different things in different
  regimes.

The canonical trader's volatility indicator -- showing where
"normal" lies given recent history, so breakouts mean a regime
shift, not just deviation from a global mean.

## Pure helpers

- `getLineBollingerDefaultColor(index)` -- 10-color palette
  cycle; NaN / negative -> color 0.
- `getLineBollingerFinitePoints(points)` -- finite-only filter.
- `normaliseLineBollingerWindow(value)` -- clamps to `>= 2`;
  floors fractional; non-finite -> default 20.
- `normaliseLineBollingerKSigma(value)` -- clamps to `> 0`;
  non-finite / non-positive -> default 2.
- `computeRollingMean(values, window)` -- per-index rolling
  mean over the trailing `window` samples; first `window-1`
  entries null; non-finite entries excluded from the sum.
- `computeRollingStd(values, window, means?)` -- per-index
  rolling **population** standard deviation; first `window-1`
  entries null. Reuses pre-computed means when supplied.
- `classifyLineBollingerState(y, upper, lower)` -- `'inside' |
  'above' | 'below'`. Bands null -> inside. Non-finite y ->
  inside.
- `computeLineBollingerBands(points, {window?, kSigma?})` --
  canonical Bollinger helper. Sorts ascending; drops
  non-finite; returns per-point `{index, x, y, middle, upper,
  lower, sigma, bandwidth, percentB, state}` where
  `bandwidth = (upper - lower) / |middle|` (null when middle
  is 0 or non-finite) and `percentB = (y - lower) / (upper -
  lower)` (null when bands collapse). The first `window-1`
  entries have null bands.
- `computeLineBollingerLayout({series, ...})` -- returns
  `{series, xTicks, yTicks, xMin, xMax, yMin, yMax,
  innerWidth, innerHeight, totalPoints, visibleSeriesCount}`.
  Y range expanded to cover bands too. Per-series carries
  `{id, label, color, bandColor, window, kSigma, points, path,
  middlePath, upperPath, lowerPath, bandPath, finiteCount,
  totalCount, bandValidCount, aboveCount, belowCount,
  insideCount, latestPercentB, latestBandwidth, latestState}`.
  Per-point carries projected `{...sample, px, py, middlePy,
  upperPy, lowerPy}`. Paths are built as **contiguous-run
  segmented** SVG path strings -- breaks across null gaps
  preserve correct line continuity through windows that aren't
  yet filled or that have non-finite samples.
- `describeLineBollingerChart(series, options?)` -- screen-
  reader description with per-series window / k / latest %B
  summary.

## Single-spike breakout math

For a sliding window of size N with `k` sigma bands, a single
outlier within the window breaks out iff `sqrt(N-1) > k`. With
the canonical k=2 this means N > 5. The component still works
for any window >= 2 but the envelope is wider relative to the
data; breakouts in small windows are usually structural
(repeated extremes), not single spikes.

## Tooltip

Hovering a dot shows the series label in its colour, x, bold y,
and three reference rows (`middle: <m>`, `upper: <u>`, `lower:
<l>`; each showing `n/a` when the window is unfilled), then a
coloured `inside bands` / `breakout above` / `breakout below`
row with the point's %B value.

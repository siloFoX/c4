# Patch 11.752 -- UI: chart-line-adx-trend primitive

Released as `v1.11.770` on 2026-05-26.

## Scope

Add `<ChartLineAdxTrend>` -- a pure-SVG two-panel chart with a
Wilder **ADX trend-filter** panel (SMA-smoothed variant) reading
the smoothed directional movement difference over the
directional movement sum, scaled to `100`.

## Definition

Inputs:

- `data: ChartLineAdxTrendPoint[]` where
  `ChartLineAdxTrendPoint = { x; high; low; close }`. Bars with
  `high < low` are dropped.
- `period: number` -- the SMA length for both passes. Default
  `14`. Normalized to an integer `>= 2`.
- `threshold: number` -- the trend boundary; strong starts at
  `2 * threshold`. Default `25`. Normalized to a finite in `(0,
  100]`.

Algorithm (per bar `i >= 1`):

```
upMove   = high[i] - high[i - 1]
downMove = low[i - 1] - low[i]
plusDM   = (upMove   > downMove && upMove   > 0) ? upMove   : 0
minusDM  = (downMove > upMove   && downMove > 0) ? downMove : 0
TR       = max(high - low, abs(high - prevClose),
               abs(low  - prevClose))

plusS, minusS, trS = SMA(plusDM, period), SMA(minusDM, period),
                     SMA(TR, period)

+DI = 100 * plusS  / trS
-DI = 100 * minusS / trS
DX  = 100 * abs(+DI - -DI) / (+DI + -DI)
ADX = SMA(DX, period)
```

A bar is null when `trS == 0` (no movement) or `+DI + -DI == 0`.

## Bit-exact algebraic anchors

**CONST_FLAT** (`high == low == close == K`): every `upMove`,
`downMove`, and `TR` is zero; all DMs and TR are zero; `trS = 0`
-> bar null at every position. ADX is null everywhere.

**RISING** (`high == low == close == i + 10`): for every `i >=
1`, `upMove = +1, downMove = -1`. `plusDM = 1, minusDM = 0`.
`TR = max(0, |+1|, |0|) = 1` (since high - low = 0 and prev
close = i + 9, so |i + 10 - (i + 9)| = 1 and |i + 10 - (i + 9)|
= 1). Note: with `high == low == close == i + 10`, the prev
close is `i + 9`. `hl = 0`, `hc = |high - prevClose| = |i + 10
- (i + 9)| = 1`, `lc = |low - prevClose| = |i + 10 - (i + 9)| =
1`. So `TR = 1`.

After SMA with `period = 4`:
- `plusS = SMA(1, 1, 1, 1) = 1`,
- `minusS = SMA(0, 0, 0, 0) = 0`,
- `trS = SMA(1, 1, 1, 1) = 1`.

So `+DI = 100 * 1 / 1 = 100`, `-DI = 100 * 0 / 1 = 0`, `DX = 100
* |100 - 0| / (100 + 0) = 100`. The second SMA `SMA(DX, 4)`
returns `100` once it has filled. `ADX = 100` bit-exact at every
defined bar from bar 7 onward.

**FALLING** (`high == low == close == 19 - i`): mirror. `+DI =
0, -DI = 100, DX = 100, ADX = 100` bit-exact.

**ALTERNATING**: high cycles `[10, 11]`, low cycles `[9, 10]`,
close == high. The series of `plusDM` and `minusDM` is balanced
once the SMA window has averaged enough cycles -- `plusS ==
minusS` -> `+DI == -DI` -> `DX = 0` -> `ADX = 0` bit-exact in
steady state (from bar 11 onward on the 20-bar fixture).

**Translation invariance**: shifting every OHLC field by `+k`
leaves every `upMove`, `downMove`, `hl`, `hc`, `lc` unchanged
(differences are invariant). So `ADX` is unchanged bit-exact.

## Component

`forwardRef<HTMLDivElement, ChartLineAdxTrendProps>` with
displayName. Renders:

- A region-role outer `<div data-section="chart-line-adx-trend">`.
- An `<svg role="img">` with an `aria-describedby` description.
- Top panel: close line, optional dots.
- Bottom panel: ADX line, optional dashed `+DI` / `-DI`
  overlays, zone-coloured markers (strong / trend / weak / none),
  zero reference line, dashed `threshold` and `2 * threshold`
  reference lines.
- Legend toggle for `price` / `adx` / `plusDi` / `minusDi`, plus
  `showAdx` / `showPlusDi` / `showMinusDi` props and an
  `onPointClick` handler.
- ARS design-system tokens for colour and motion (`motion-safe:
  animate-fade-in`).

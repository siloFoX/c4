# Patch 11.955 - UI: chart-line-kc-percent-zero-cross primitive

Version: 1.11.973

## Summary

Add `web/src/components/ui/chart-line-kc-percent-zero-cross.tsx` -- a
pure-SVG dual panel line chart primitive that visualizes price alongside
Chester Keltner's %K position-within-channel oscillator, with explicit
cross markers and overlay arrows at every %K 0.5-midline crossover.
Keltner %K normalises the close to its position within fixed-multiple
Keltner Channels centered on the rolling EMA, with channel width scaled
by a close-only ATR (`|close - close_prev|`-based EMA of true range).
Midline (0.5) crossovers mark position-within-channel centerline regime
transition events.

## Pipeline

1. Middle: `middle_i = EMA(close, length)` SMA-seeded at index
   `length - 1`.
2. Close-only true range: `tr_i = |close_i - close_{i-1}|` (i >= 1).
3. ATR: `atr_i = EMA(tr, length)` SMA-seeded at index `length` (since
   `tr[0]` is null).
4. Bands: `upper = middle + mult * atr`, `lower = middle - mult * atr`.
5. Percent K:
   - `range = upper - lower = 2 * mult * atr`.
   - `range == 0` -> `pctK = 0.5` (CONST neutral fallback).
   - else -> `pctK = (close - lower) / range`.
6. `classifyLineKcPercentZeroCrossRegime(pctK, T)`: null -> none,
   pctK >= T -> bullish, pctK < T -> bearish.
7. `detectLineKcPercentZeroCrossCrosses(series, pctK, T)`: strict
   `prev <= T && cur > T` for bullish, strict `prev >= T && cur < T`
   for bearish. Boundary equality does not fire.

## Bit-exact anchors

| Pattern                  | %K (after warmup)                   | Regime  |
|--------------------------|-------------------------------------|---------|
| CONST close = K          | 0.5 (range = 0 fallback)            | bullish |
| LINEAR UP close = i (length=5, mult=2)   | 1.0 (close touches upper) | bullish |
| LINEAR DOWN close = -i (length=5, mult=2)| 0.0 (close touches lower) | bearish |

CONST anchor tested bit-exactly across K in `{0, 1, 42, 100, 1234}`.
LINEAR anchors: at steady state for arithmetic close, EMA lag is
`(length - 1) / 2`, so `close - middle = (length - 1) / 2`. With
`tr = 1` constant and `atr = 1`, `pctK = 0.5 + ((length - 1) / 2) /
(2 * mult)`. For length=5, mult=2 this evaluates to exactly 1.0
(close sits on the upper band) and the LINEAR DOWN symmetric case to
0.0.

## Layout

* Default dimensions: 720 x 460.
* Two panels: price panel on top (~55% of plot area), %K panel below.
* %K panel auto-fits to the observed `[min, max]` range with 5% padding
  on each end, always including both the `[0, 1]` channel envelope and
  the threshold in the displayed range.
* Threshold band rendered as a horizontal rule at `y = thresholdY` in
  the %K panel.
* Cross markers drawn as circles on the %K panel at `(cx, cyOsc)` and
  as arrows on the price panel at `(cx, cyPrice)`.
* SVG path strings use `.toFixed(2)` for deterministic snapshot tests.

## Accessibility

* Wrapping `<div role="region" aria-label="KC Percent K Zero Cross
  chart">`.
* `<svg role="img">` with an aria-described-by linked `<desc>` sr-only
  text.
* Each cross marker: `role="graphics-symbol"` + `tabIndex={0}` +
  descriptive `aria-label` ("bullish cross at x=...").
* Legend toggles: `<button role="switch" aria-pressed="...">`.
* Motion-safe fade-in via `motion-safe:animate-fade-in` (opt-out with
  `animate={false}`).

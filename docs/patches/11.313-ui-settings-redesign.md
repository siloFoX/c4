# 11.313 -- Settings page redesign (v1.11.331)

Refactors `web/src/pages/Settings.tsx` to use the
canonical RadioGroup primitive (v1.11.306) and adds
consistent data-section markers for e2e targeting.
Component-scope only, no daemon-side change and no `c4`
CLI surface change.

## Why

The Settings page already received most of the
"redesign" treatment in earlier patches:

- v1.11.199: Tabs for sections (the 7-tab consolidation
  layer).
- v1.11.269: SectionDivider between the consolidated
  banner and the tab strip.
- v1.11.275: AlertBanner for the "dedicated pages
  remain authoritative" advisory.
- v1.11.290: Accordion for the General tab's nested
  groups.

The two remaining pain points were:

1. **Locale picker uses the legacy `<Radio>` loop.**
   The v1.11.306 RadioGroup primitive ships
   arrow-key navigation, roving tabindex, and a
   canonical `data-section="radio-group"` e2e selector
   shape. The Settings page's locale picker was the
   last place still using the manual
   `<Radio name="..." onChange={...}>` loop pattern.

2. **No consistent e2e wrapper.** Per-test selectors
   reach inside the page via per-element data-testids
   (`settings-general-config-link`,
   `settings-locale-radio-ko`, etc), but there is no
   single root selector that says "this is the
   Settings surface" -- e2e tests that want to
   bound their assertions to the Settings page have
   to enumerate all the per-tab containers.

Patch 11.313 fixes both with the smallest possible
diff:

- Swap the manual Radio loop for `<RadioGroup>`.
- Wrap the whole page in
  `<div data-section="settings-page" className="contents">`
  so the wrapper does not disturb layout.
- Wrap the Locale panel body in
  `<div data-settings-panel="locale">` so per-panel
  selectors can scope without grabbing the entire
  Panel header.

## What changed

### `web/src/pages/Settings.tsx`

- Import: replaced `Radio` with `RadioGroup` from
  the ui barrel.
- LocalePanel body refactored from the manual loop:
  ```tsx
  // Before
  <div role="radiogroup" aria-label="Locale" ...>
    {LOCALES.map((loc) => (
      <Radio
        key={loc}
        name="settings-locale"
        value={loc}
        checked={locale === loc}
        onChange={() => handleChange(loc)}
        label={loc === 'en' ? 'English' : 'Korean'}
        data-testid={`settings-locale-radio-${loc}`}
      />
    ))}
  </div>;
  ```
  to the canonical primitive:
  ```tsx
  // After
  <RadioGroup
    name="settings-locale"
    value={locale}
    onChange={handleChange}
    ariaLabel="Locale"
    items={[
      { value: 'en', label: 'English' },
      { value: 'ko', label: 'Korean' },
    ]}
    data-testid="settings-locale-radiogroup"
  />;
  ```
  The `handleChange` signature widened from
  `(next: Locale)` to `(next: string)` (and casts
  to `Locale` inside) because the RadioGroup's
  generic value type is its callback's input
  type.
- New wrapper element on the page root:
  `<div data-section="settings-page" className="contents">`.
  Tailwind's `contents` class removes the element
  from the layout while preserving its children,
  so the wrapper is visually transparent.
- New wrapper element on the Locale panel body:
  `<div data-settings-panel="locale">`.

### `web/src/pages/Settings.test.tsx`

- Updated the existing "persists Locale selection
  to localStorage" test to use the new
  `[role="radio"][data-radio-value="<locale>"]`
  selectors instead of the legacy per-locale
  `data-testid` hooks (the new RadioGroup does
  not stamp per-item test ids).
- 3 new cases:
  - Page root carries
    `data-section="settings-page"`.
  - Locale panel uses the canonical RadioGroup:
    `[data-settings-panel="locale"] [data-section="radio-group"]`
    is present and >=2 `role="radio"` items render.
  - Locale RadioGroup carries
    `data-testid="settings-locale-radiogroup"`
    for e2e.

## Deferred (dispatch follow-ups)

The dispatch listed several items that were already
delivered in prior patches or that would balloon the
scope of this commit:

- **Tabs for sections.** Already shipped in
  v1.11.199.
- **FormField wrappers.** Deferred. The existing
  Panel + HelpTip shape carries the label/hint
  semantics today; threading FormField through every
  panel would require re-shaping the Panel children
  contract (FormField requires a single
  ReactElement child whose id+aria-describedby it
  injects). A targeted FormField adoption could
  land in a follow-up patch.
- **RadioGroup for theme/density.** Deferred. The
  Theme and Density panels already group their
  options via the v1.11.158 ToggleGroup variants of
  `ThemeToggle` and `DensityToggle`; swapping them
  to RadioGroup would change the visual surface
  (button-style toggles vs radio dots) and needs a
  separate UX pass.
- **Switch for toggles.** Vacuously applicable --
  the Settings page has no boolean toggles, only
  selectors. The Feature Flags tab embedding
  already uses Switch (via the embedded
  FeatureFlags page).

## Notes

- The Tailwind `contents` class on the page
  wrapper is what keeps the wrapper visually
  transparent. Without it, the new `<div>` would
  start a new layout context inside the PageFrame
  card body and could break the spacing between
  the PageHeader, AlertBanner, SectionDivider, and
  Tabs.
- The legacy `data-testid="settings-locale-radio-<loc>"`
  per-item selectors are no longer emitted. Any
  external e2e tests that relied on those selectors
  need to migrate to the standard RadioGroup
  selectors (`[role="radio"][data-radio-value="<loc>"]`).
  The vitest suite has been updated to use the new
  pattern.
- The page-level `data-section="settings-page"`
  wrapper is the same pattern used by the
  UIDemoRoute (v1.11.325) and matches the broader
  convention in `web/src/components/ui/*.tsx` of
  exposing a `data-section="<name>"` attribute on
  the primitive's root element.
- The new `data-settings-panel="locale"` is a
  per-panel marker; the other 6 panels can adopt
  the same `data-settings-panel="<name>"` shape in
  follow-up patches when their bodies need scoped
  selectors. For this commit only the Locale panel
  carries it because that is the only one whose
  body changed.

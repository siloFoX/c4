## Goal

Dispatch:

- Build `web/src/components/ui/virtual-table.tsx`
  -- VirtualizedList + DataTable composite for
  10k+ row tables. Header sticky, horizontal
  scroll sync, row selection support.

Existing primitives in the codebase already
provide the building blocks:

- `<VirtualizedList>` (11.333 / v1.11.351)
  -- row-virtualization scroller with
  computeVisibleRange + overscan window +
  optional scroll restoration.
- `lib/data-table-state.ts` (11.355 /
  v1.11.373) -- multi-column sort
  (`applyMultiSort`, `toggleSort`) +
  type-aware filters (`applyFilters`,
  `ColumnFilter`).
- `<DataTable>` (11.355) -- non-virtual
  table primitive that composes both
  helpers.

`<DataTable>` is the canonical surface
for < 500 row tables, but it does NOT
virtualize: every row mounts at once and
big tables (the 10k+ snapshots inspector,
audit log, runs) drop frames. `<VirtualTable>`
is the composite for those big-N surfaces.

## Architecture

```
+---------------------------------------------------------+
| <div role="table" data-section="virtual-table">         |
|   overflow-x-auto  (horizontal scroll lives here)       |
|                                                          |
|   +--------------------------------------------------+  |
|   | <div role="row" data-section="virtual-table-header">
|   |   position: sticky; top: 0; z-10                 |  |
|   |   grid-template-columns: ${gridTemplate}         |  |
|   | </div>                                            |  |
|   +--------------------------------------------------+  |
|                                                          |
|   +--------------------------------------------------+  |
|   | <VirtualizedList ...>                            |  |
|   |   height: ${height}                              |  |
|   |   overflow-y-auto (body's own scroll)            |  |
|   |   renderRow:                                     |  |
|   |     <div role="row" grid + ${gridTemplate}>      |  |
|   |       <cells> ...                                 |  |
|   |     </div>                                        |  |
|   | </VirtualizedList>                               |  |
|   +--------------------------------------------------+  |
| </div>                                                  |
+---------------------------------------------------------+
```

- The outer `<div>` owns horizontal scroll;
  the header + the virtualized body share
  the wrapper so a horizontal scroll moves
  both together.
- The header is `position: sticky; top: 0;
  z-10` (within the wrapping container) so
  it pins as the body scrolls vertically.
- Each body row uses the SAME
  `gridTemplateColumns` string as the
  header, so cells stay perfectly aligned
  even at extreme horizontal scroll.

## API

```tsx
import {
  VirtualTable,
  type VirtualTableColumn,
} from './components/ui/virtual-table';

interface Row { id: string; name: string; age: number; }

const columns: VirtualTableColumn<Row>[] = [
  { key: 'name', label: 'Name', sortable: true, width: '200px' },
  { key: 'age',  label: 'Age',  sortable: true, width: '80px', align: 'right' },
];

<VirtualTable
  columns={columns}
  rows={rows}
  rowKey={(r) => r.id}
  rowHeight={36}
  height={480}
  // Optional sort + filter pass-through
  sortBy={sortBy}
  onSortByChange={setSortBy}
  filters={filters}
  // Optional selection
  selectable
  selectedIds={selectedIds}
  onSelectionChange={setSelectedIds}
  ariaLabel="Users"
  stickyHeader
/>
```

### Column shape

```ts
interface VirtualTableColumn<T> {
  key: string;
  label: ReactNode;
  width?: string;            // any grid track: '120px' | '1fr' | 'minmax(80px, 1fr)'
  align?: 'left' | 'center' | 'right';
  sortable?: boolean;
  render?: (row: T, index: number) => ReactNode;
  accessor?: (row: T) => Comparable;
  filterAccessor?: (row: T) => Comparable;
  className?: string;
}
```

Width defaults to `'1fr'`. When `selectable`
is on, a leading `36px` track is prepended
automatically (via `buildGridTemplate`).

### Sort UX

- Plain click on a sortable header
  toggles asc -> desc -> clear (matches
  `<DataTable>` semantics).
- Shift-click APPENDS to the sort list
  (multi-column sort).
- Active sort column shows the direction
  glyph (`↑` for asc, `↓` for desc) and
  the index priority badge when the sort
  list has more than one column.
- The header cell exposes `aria-sort` per
  the WAI-ARIA grid pattern.

### Selection UX

- `selectable=false` (default) drops the
  entire selection chrome.
- Per-row checkbox toggles the row key
  in/out of the `selectedIds: Set<string>`.
- Select-all in the header toggles every
  VISIBLE row (post-filter + post-sort).
- Tri-state: shows indeterminate via the
  native `indeterminate` property +
  `data-indeterminate="true"` when only
  some-but-not-all visible rows are
  selected.
- `data-row-selected` on the body row
  mirrors selection state for CSS hooks.

## Pure helpers

Two helpers are exported alongside the
component:

```ts
export function buildGridTemplate(
  columns: ReadonlyArray<{ width?: string }>,
  selectable: boolean,
): string;

export function applyTableTransforms<T>(args: {
  rows: readonly T[];
  filters?: Record<string, ColumnFilter>;
  sortBy?: readonly SortDescriptor[];
  filterAccessor: FilterAccessor<T>;
  sortAccessor: SortAccessor<T>;
}): T[];
```

`applyTableTransforms` runs filters FIRST
then sort (the canonical order: the sort
operates over the filtered subset, so big
filter+sort tables stay responsive). Both
helpers are pure and synchronous.

## Pairs with existing primitives

- `<VirtualizedList>` (11.333) -- the body
  virtualizer.
- `lib/data-table-state.ts` (11.355) -- the
  sort + filter helpers.
- `<DataTable>` (11.355) -- non-virtual
  table primitive. Reach for that when
  `< 500` rows. Reach for `<VirtualTable>`
  when `> 10k` rows OR when the host
  requires a sticky header.
- `<Pagination>` -- complementary primitive.

## Out of scope

- **Pagination integration.** Adopters
  paginate before passing `rows` to the
  table.
- **Column resize.** Width is declarative
  via `column.width`; drag-to-resize is a
  follow-on with its own pointer story.
- **Frozen columns / row headers** (sticky
  first column). The current grid
  template treats all columns as
  scrollable.
- **Cell editing.** Read-only cells only.
- **Inline filter row** beneath the
  header. `<DataTable>` ships an in-band
  filter row; `<VirtualTable>` keeps the
  filter UI external (host drives the
  `filters` prop).
- **Variable row height.** The
  virtualizer assumes uniform row height
  (`rowHeight`); variable heights need a
  measurement / window-rebuild cycle.

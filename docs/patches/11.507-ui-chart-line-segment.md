# TODO 11.507 - chart-line-segment primitive

Status: SHIPPED v1.11.525 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartLineSegment>`, a pure-SVG multi-series line chart that
**colours each line segment based on threshold matching**. Each
series carries a sorted `thresholds: [{value, color, label?}]` list;
each segment between two adjacent points is classified into the
highest-value threshold whose value the segment crosses. Consecutive
segments sharing the same colour merge into a single `<path>` so
the DOM stays small even for long series with infrequent transitions.

## API

```tsx
<ChartLineSegment
  series={[
    {
      id: 'cpu',
      label: 'CPU',
      color: '#94a3b8',           // fallback when no threshold matches
      classifyBy: 'max',
      thresholds: [
        { value: 80, color: '#dc2626', label: 'critical' },
        { value: 50, color: '#f59e0b', label: 'warning' },
      ],
      data: [
        { x: 0, y: 30 }, { x: 1, y: 55 }, { x: 2, y: 85 },
        { x: 3, y: 90 }, { x: 4, y: 40 },
      ],
    },
  ]}
/>
```

## Distinct from prior chart-* primitives

- `<ChartLineDashed>` (11.499) - colours one stroke and varies dash
  patterns per segment; chart-line-segment varies the *colour* per
  segment based on threshold classification.
- `<ChartLineThreshold>` (11.503) - paints horizontal threshold
  rules + zone shading; chart-line-segment paints the line itself
  in the threshold colour.
- `<ChartLineMarker>` (11.506) - per-point marker shape encoding.
- All other line variants (`<ChartLineMulti>`, `<ChartLineConfidence>`,
  etc.) focus on different dimensions.

## Classification contract

### Per-segment representative value

`classifyLineSegmentValue(startY, endY, by)` reduces a segment to
a single value:

| `classifyBy` | result          |
| ------------ | --------------- |
| `'max'` (default) | `max(startY, endY)` |
| `'min'`      | `min(startY, endY)` |
| `'avg'`      | `(startY + endY) / 2` |
| `'start'`    | `startY`        |
| `'end'`      | `endY`          |

Non-finite inputs collapse to 0.

### Colour picking

`pickLineSegmentColor(value, thresholds, fallback)`:

1. Drops thresholds with non-finite values or non-string colours.
2. Sorts the rest in descending order by `value`.
3. Walks top to bottom; the first threshold whose value is
   `<= effectiveValue` wins. Exact equality counts as crossed.
4. Empty / no-match -> the `fallback` colour (`series.color` or the
   palette default).
5. Non-finite input value -> always fallback.

The picked threshold's label + value are recorded on the segment so
the tooltip and aria-label can surface "you crossed `critical` at
80".

### Segment merging

`buildLineSegmentPath(points, thresholds, classifyBy, fallback)`
walks consecutive points, classifies each segment, and merges
consecutive same-colour segments into a single SVG sub-path. A
colour transition closes the current sub-path and opens a new one
that starts at the transition point so the line stays visually
continuous (no gap). Empty / single point -> `[]`. Two points -> 1
segment.

## Layout

`computeLineSegmentLayout(input)` builds per-series:

- `fallbackColor` from `series.color ?? palette[index]`.
- `classifyBy` from `series.classifyBy ?? 'max'`.
- `thresholds` (copied list).
- `points[]` with pixel coordinates.
- `segments[]` from `buildLineSegmentPath`.

Bounds, ticks, hidden filter follow the standard line-chart pattern.

## Interactivity

- Hover or focus on a dot dims every other series stroke to ~30%;
  tooltip shows series label, formatted x, formatted y, and (when
  the point's segment crosses a threshold) a coloured `threshold:`
  row identifying the band.
- Per-dot `onPointClick({series, point})` callback.
- Per-segment `onSegmentClick({series, segment})` callback for
  adopters who want to react to band hits at the segment level.
- Legend toggles series visibility (controlled + uncontrolled).
- Legend also renders one tiny coloured swatch per threshold so the
  threshold colour-to-label mapping is part of the legend.
- Mount fade via `motion-safe:animate-fade-in` (suppress via
  `animate=false`).

## Exported helpers

- `DEFAULT_CHART_LINE_SEGMENT_*` numeric / string defaults.
- `DEFAULT_CHART_LINE_SEGMENT_PALETTE` (10-colour palette).
- `getLineSegmentDefaultColor(index)` - palette wrap.
- `getLineSegmentFinitePoints(points)` - drops non-finite.
- `getLineSegmentBounds(series, hidden?)` - per-axis bounds.
- `getLineSegmentTicks(min, max, count?)` - evenly-spaced ticks.
- `classifyLineSegmentValue(startY, endY, by)` - canonical
  classify-by-segment reduction.
- `pickLineSegmentColor(value, thresholds, fallback)` - canonical
  threshold matcher returning `{ color, thresholdLabel,
  thresholdValue }`.
- `buildLineSegmentPath(points, thresholds, classifyBy, fallback)`
  - canonical merged-segment builder.
- `computeLineSegmentLayout(input)` - per-series layout + per-point
  + per-segment data.
- `describeLineSegmentChart(series, hidden?, formatValue?)` - ARIA
  summary mentioning threshold band count.

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only
  `data-section="chart-line-segment-aria-desc"` span.
- SVG `role="img"` with the same `aria-label`.
- Each segment path is `role="graphics-symbol"` with `tabIndex={0}`
  and `aria-label="<series> segment <startIndex> to <endIndex>
  (<thresholdLabel>)"`.
- Each dot is `role="graphics-symbol"` with `tabIndex={0}` and
  `aria-label="<series>: x=<x>, y=<y>"`.

## Data attributes

- Root: `data-section="chart-line-segment"`, `data-series-count`,
  `data-visible-series-count`, `data-total-points`,
  `data-total-segments`, `data-animate`.
- Series group: `data-section="chart-line-segment-series-group"`,
  `data-series-id`, `data-series-index`,
  `data-series-fallback-color`, `data-series-classify-by`,
  `data-series-segment-count`, `data-series-threshold-count`,
  `data-series-point-count`, `data-series-finite-count`,
  `data-hovered`.
- Segment path: `data-section="chart-line-segment-path"`,
  `data-series-id`, `data-segment-index`, `data-segment-start`,
  `data-segment-end`, `data-segment-color`, `data-segment-value`,
  `data-segment-threshold-label`, `data-segment-threshold-value`.
- Dot: standard pattern.
- Tooltip: `-tooltip-threshold` row appears only when the segment
  has a matching threshold.
- Legend: standard series items PLUS
  `data-section="chart-line-segment-legend-threshold"` per
  threshold so the colour-to-label mapping is visible.

## Out of scope

- Per-point colour (only per-segment via threshold classification).
- Gradient transitions across the threshold (hard colour cut at the
  point that crosses; adopters who want a smooth gradient use SVG
  `<linearGradient>` directly).
- Threshold animation on value crossing - snap; mount fade only.
- Log-scale axes - linear only.
- Crossing-point interpolation (when one endpoint is above and the
  other below a threshold, the segment's colour reflects the
  classifyBy result; v1 does not subdivide the segment at the
  exact crossing x).

## Goal

Dispatch:

- Build `web/src/components/ui/slider.tsx`
  -- single-value + range slider, keyboard
  navigation (arrow keys + page up/down),
  discrete steps, value tooltip, ARIA slider
  role.

No existing slider in the codebase. This
patch creates the file from scratch with
two canonical primitives:

- `<Slider>` -- single-value slider.
- `<RangeSlider>` -- two-thumb range
  selector.

Both implement the WAI-ARIA slider pattern
with full keyboard navigation. Horizontal
orientation only in v1; vertical can be added
on a follow-on (the geometry is mirror-
symmetric).

## API

```tsx
import { Slider, RangeSlider } from './components/ui/slider';

<Slider
  value={42}
  onChange={setVolume}
  min={0}
  max={100}
  step={1}
  pageStep={10}
  showTooltip
  formatValue={(v) => `${v}%`}
  aria-label="Volume"
/>

<RangeSlider
  values={[20, 80]}
  onChange={setBounds}
  min={0}
  max={100}
  step={1}
  ariaLabelLow="Low"
  ariaLabelHigh="High"
/>
```

### Keyboard contract

| key | Slider | RangeSlider (focused thumb) |
| --- | --- | --- |
| ArrowRight / ArrowUp | +step | +step |
| ArrowLeft / ArrowDown | -step | -step |
| PageUp | +pageStep | +pageStep |
| PageDown | -pageStep | -pageStep |
| Home | jump to min | jump to min (or pair-bound) |
| End | jump to max | jump to max |

`pageStep` defaults to
`max(step, (max-min) / 10)` so a 0-100 range
with `step=1` page-steps by 10.

RangeSlider clamps each thumb so the low
cannot exceed the high (and vice versa).

### ARIA

```
role="slider"
aria-valuemin=<min>
aria-valuemax=<max>
aria-valuenow=<value>
aria-valuetext=<formatValue(value)>
aria-orientation="horizontal"
aria-label=<ariaLabel | 'Minimum' | 'Maximum'>
```

RangeSlider's low thumb exposes
`aria-valuemax=<high>` (the per-thumb upper
bound) and the high thumb exposes
`aria-valuemin=<low>` so screen readers
announce the constraint correctly.

### Step snapping

```ts
export function clampToStep(
  value: number,
  min: number,
  max: number,
  step: number,
): number
```

Pure helper exported for tests + custom
callers. Clamps to `[min, max]` then snaps to
the nearest `step` increment anchored at
`min`. Returns `min` for NaN; passes through
unchanged when `step <= 0`.

The Slider snaps in both directions:
- the visible `aria-valuenow` is always the
  snapped value, so `value={7}` with
  `step={5}` renders as 5 immediately;
- `onChange` always receives the snapped
  value, never the raw arrow-key delta.

### Tooltip

`showTooltip=true` (default) renders a small
floating label above each thumb showing the
formatted value:

- Slider: `data-section="slider-tooltip"`.
- RangeSlider: `-tooltip-low` /
  `-tooltip-high` per thumb.

The default formatter is `String(value)`. A
helper map exports common formatters:

```ts
sliderFormatters.none           // (v) => `${v}`
sliderFormatters.withPercent    // (v) => `${v}%`
sliderFormatters.fixed1         // (v) => v.toFixed(1)
sliderFormatters.fixed2         // (v) => v.toFixed(2)
```

### Controlled / uncontrolled

| Props | Mode |
| --- | --- |
| `value` + `onChange` | controlled |
| `defaultValue` only | uncontrolled, internal state |
| `defaultValue` + `onChange` | uncontrolled, observable |
| `value` + `defaultValue` | controlled (`value` wins) |
| neither | uncontrolled, starts at `min` |

Same five-row table for RangeSlider with
`values` / `defaultValues`. Internal state
seeds once on mount.

### Disabled

`disabled={true}` adds `aria-disabled="true"`,
flips `tabIndex` to `-1`, drops the cursor
affordance to `not-allowed`, and makes every
keyboard handler a no-op.

### No-refire optimisation

If a keystroke would resolve to the SAME
snapped value as the current value (e.g.,
ArrowRight at `value=max`), the component
does NOT fire `onChange`. RangeSlider applies
the same rule per-tuple (`[lo, hi]` equality).

## Pairs with existing primitives

- `<NumberInput>` -- numeric text input for
  explicit value entry. Pair when a value is
  precise (volume in db, latency in ms).
- `<Progress>` (linear) -- visualizes
  a non-interactive value. Unchanged.

## Out of scope

- **Pointer drag-to-set**. v1 is keyboard-
  only. Drag will need pointermove +
  pointercancel handlers across the track
  surface plus a translateX commit on
  release. Deferred to its own patch with
  the touch-vs-pointer story called out.
- **Vertical orientation**. Mirror-
  symmetric geometry; deferred.
- **Marks / tick labels**. The discrete
  step contract already enforces snapping
  on the value side; visual tick rendering
  belongs in a separate styling patch.
- **Inverted track** (high-on-left). Layout
  flip; can be added with an `inverted` prop.
- **Multi-thumb (>2)**. RangeSlider is
  fixed-arity at 2; a multi-thumb slider
  is a different primitive.

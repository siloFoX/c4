# 8.18 - Claude session JSONL viewer

## Summary

Builds a parser + daemon endpoints + Web UI that turn a Claude Code
`~/.claude/projects/<encoded-cwd>/<session-id>.jsonl` transcript into a
browsable, claude.ai-style chat view. Foundation for 8.17 (external
session import) - the external importer consumes the same
`session-parser` contract documented here.

## Module layout

```
src/session-parser.js              # pure parser (no deps)
src/daemon.js                      # /api/sessions* route wiring
web/src/components/ConversationView.tsx  # chat-style turn renderer
web/src/components/SessionsView.tsx      # 2-pane list + detail
web/src/components/layout/TopTabs.tsx    # "Sessions" tab
web/src/App.tsx                    # topView route
tests/fixtures/session.jsonl       # one-of-every-role fixture
tests/session-parser.test.js       # 32 assertions / 8 suites
```

### `src/session-parser.js`

Pure module, zero runtime dependencies. Public surface:

| Export | Shape | Purpose |
|---|---|---|
| `parseJsonl(path)` | `Conversation` | read + normalize a whole transcript |
| `parseJsonlStream(path)` | `AsyncIterable<Turn>` | tail/live-parse without buffering |
| `listSessions(rootDir)` | `SessionSummary[]` | enumerate transcripts under a root |
| `groupSessionsByProject(sessions)` | `SessionGroup[]` | group list output by project dir |
| `defaultProjectsRoot()` | `string` | resolve `$CLAUDE_PROJECTS_DIR` or `~/.claude/projects` |
| `parseLine(line, warnings)` | `Turn[]` | used by the SSE tail endpoint for per-line emit |
| `eventToTurns(event, warnings)` | `Turn[]` | lower-level fan-out of one JSONL event |
| `pairToolTurns(turns)` | mutates in place | backfill `tool_use.toolResult` by `tool_use_id` |
| `decodeProjectDir(name)` | `string \| null` | reverse `-root-c4` -> `/root/c4` (best effort) |
| `tailSnippet(path)` | `string` | cheap last-assistant-text preview for list view |

### `Conversation` shape

```ts
interface Conversation {
  sessionId: string;
  projectPath: string | null;   // prefer cwd from first event, fall back to decoded dir
  createdAt: string | null;     // first ISO timestamp observed
  updatedAt: string | null;     // last ISO timestamp observed
  model: string | null;         // most recent assistant.message.model
  totalInputTokens: number;
  totalOutputTokens: number;
  turns: Turn[];                // ordered, post-paired
  warnings: string[];           // malformed lines + unexpected shapes
}
```

### `Turn` shape

```ts
type TurnRole = 'user' | 'assistant' | 'thinking' | 'tool_use' | 'tool_result' | 'system';

interface Turn {
  id: string;                  // block id, or uuid[:index] when block lacks one
  role: TurnRole;
  createdAt: string | null;    // ISO from event.timestamp
  durationMs: number | null;   // reserved - computed by downstream consumers
  model: string | null;        // assistant + thinking + tool_use only
  tokens: { input, output, cacheRead, cacheCreate };  // attached to the first block of a message
  content: string;             // markdown text for assistant, plain for user
  toolName: string | null;
  toolArgs: unknown;           // raw `input` passthrough
  toolUseId: string | null;    // pair key for tool_use / tool_result
  toolResult: unknown;         // filled post-hoc by pairToolTurns
  thinkingText: string | null;
  attachments: unknown[];      // toolUseResult.file when present
  raw: unknown;                // original event for debugging
}
```

## JSONL schema assumptions

The parser targets the transcript shape produced by Claude Code CLI
>= 2.1 (what `~/.claude/projects/<encoded-cwd>/<session-id>.jsonl` looks
like today). Each line is one JSON object. The event shape we rely on:

1. **Metadata events** (no user-visible content): `type` in
   `{permission-mode, file-history-snapshot, summary}`. Rendered as a
   single `system` turn so operators notice instead of silently losing
   context.
2. **User text events**: `type: "user"` with `message.content` as a
   string. Becomes one `user` turn.
3. **User tool-result events**: `type: "user"` with `message.content` as
   an array whose blocks have `type: "tool_result"` + `tool_use_id` +
   `content`. Fans out to one `tool_result` turn per block. Content can
   be a string, an array of `{type: text, text}` blocks, or a structured
   object; the parser normalizes all three into a display string while
   keeping the raw value available on `toolResult`.
4. **Assistant events**: `type: "assistant"` with `message.content` an
   array of blocks, each of type `thinking`, `text`, or `tool_use`. Each
   block becomes its own turn (so the UI can render thinking separately
   from the response body). `message.usage` is counted once per message -
   attached only to the first emitted block so `totalInputTokens` /
   `totalOutputTokens` stay truthful after fan-out.
5. **Tool pairing**: after reading the whole file, the parser walks
   `tool_use` turns into a map by id and copies the matching
   `tool_result.content` onto `tool_use.toolResult`. The `tool_result`
   turn is kept in the stream too - the UI hides it when a pair exists
   to avoid double-rendering.
6. **Top-level context fields** (`sessionId`, `cwd`, `timestamp`,
   `version`, `gitBranch`) are hoisted into the `Conversation` summary
   on the first event that carries them.
7. **Project dir encoding**: Claude Code encodes the cwd by replacing
   each path separator with `-`, so `/root/c4` -> `-root-c4`. The
   decoder reverses `-` -> `/` best effort and the per-event `cwd`
   overrides the decoded value whenever the file actually contains one.

Malformed lines are collected in `Conversation.warnings` instead of
throwing - one corrupt line in a multi-megabyte transcript must not
break a whole view.

## Daemon endpoints

All three live under the existing auth middleware from 8.14 (JWT with
`?token=` SSE fallback). Admin / single-user default sees every
session; multi-user gating is deferred until RBAC grows a
`session.read` action.

```
GET  /api/sessions
GET  /api/sessions/<session-id>
GET  /api/sessions/<session-id>/stream
```

### `GET /api/sessions`

- Query `q=<text>` case-insensitive filter across projectPath,
  projectDir, sessionId, and lastAssistantSnippet.
- Returns `{ rootDir, sessions: SessionSummary[], groups: SessionGroup[], total }`.
- Sort order: `updatedAt` desc within each group and across groups.
- Override root with `config.sessions.projectsDir`.

### `GET /api/sessions/:id`

- Returns a parsed `Conversation`.
- 404 when no JSONL file under `rootDir` matches the session id.

### `GET /api/sessions/:id/stream`

- SSE. Emits `event: conversation` with the full snapshot first, then
  one `event: turn` per newly parsed Turn as the file grows.
- Uses `fs.watch` where available; falls back to 2-second stat polling
  when `watch` throws (network filesystems, Windows edge cases).
- Tracks a byte offset so re-parsing of the whole file never happens.
- 30-second `: keepalive` heartbeat so proxies do not drop the
  connection mid-session.

## Web UI

Top-level **Sessions** tab (between History and Chat). Two-pane layout:

- Left: `SessionsView.tsx` lists groups keyed by project. Each row shows
  a shortened session id, turn count badge, last assistant snippet, and
  a relative timestamp. A search box filters both groups and sessions.
- Right: `ConversationView.tsx` renders the parsed `Conversation`:
  - `user` -> right-aligned `bg-primary/10` bubbles
  - `assistant` -> left-aligned full width, minimal markdown
    (paragraphs, headings, fenced code, inline code, bold/italic,
    lists, blockquotes, links) via a dependency-free renderer
  - `thinking` -> collapsible muted card, preview truncated to 120 chars
  - `tool_use` -> expandable card showing the tool name badge + input
    JSON + optional paired result
  - `tool_result` -> expandable code block; hidden when already paired
    to a `tool_use` row
  - `system` -> small centered chip
- Auto-scroll only when the user is within 24px of the bottom; a
  "Jump to latest" button appears when they scroll up.
- Dark mode falls out of existing `bg-card` / `bg-muted/40` /
  `text-muted-foreground` tokens; layout collapses to a stacked flex
  column below `md` so the left pane does not pin the detail out.

## Interface stability

8.17 (external session import) will consume this parser directly. The
stable surface is the `Conversation` / `Turn` JSON shape above and the
`parseJsonl` / `parseJsonlStream` / `listSessions` exports. Any future
break to those needs a major-version bump on the parser and a matching
change in the 8.17 worker.

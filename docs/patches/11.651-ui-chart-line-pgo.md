## Summary

New `ChartLinePgo` UI primitive at
`web/src/components/ui/chart-line-pgo.tsx` -- a pure-SVG
two-panel chart with a Pretty Good Oscillator panel normalizing
price distance from a moving average by the average true range.

## Indicator math

The Pretty Good Oscillator (Mark Johnson) reads how far the
close has moved from its moving average, in units of Average
True Range.

`ChartLinePgoPoint` is `{ x, value, high, low }` -- `value` is
the bar close.

- `computeLinePgoTrueRange(highs, lows, closes)` -- the per-bar
  true range. The first bar has no prior close so its true range
  is `high - low`; every later bar is the widest of the bar
  range and the two gaps from the prior close:

```
TR[i] = max(high[i] - low[i],
            abs(high[i] - close[i-1]),
            abs(low[i] - close[i-1]))
```

- `computeLinePgoAtr(highs, lows, closes, period)` -- the
  Average True Range, the `period`-bar simple moving average of
  the true range.

- `computeLinePgo(highs, lows, closes, period)` -- the Pretty
  Good Oscillator:

```
PGO[i] = (close[i] - SMA(close, period)) / ATR(period)
```

  the displacement of the close from its moving average, divided
  by the ATR, so it reads the price move in ATR units. A
  positive PGO marks the close above its moving average, a
  negative PGO below. A window whose ATR is zero (a perfectly
  flat market) has an undefined PGO and is null, as are bars
  before the window is full.

### Worked fixture

The PGO is a ratio of plain sums -- no transcendentals -- so the
whole pipeline is exact. The fixture (`period 2`) is hand-tuned
so every bar has a true range of exactly 4, hence every ATR is
4 and every PGO divides cleanly by 4:

```
close   [10, 12, 16, 14, 16, 20, 18, 22]
pgo     [ ., 0.25, 0.5, -0.25, 0.25, 0.5, -0.25, 0.5]
```

A flat close on a non-zero range reads zero (the close sits on
its own moving average); a perfectly flat market reads null
(the ATR is zero).

## API

- `getLinePgoFinitePoints(points)` -- drop points with a
  non-finite `x`, `value`, `high` or `low`.
- `normalizeLinePgoPeriod(period, fallback)` -- coerce to an
  integer of at least 2.
- `computeLinePgoTrueRange(highs, lows, closes)` -- the per-bar
  true range.
- `computeLinePgoAtr(highs, lows, closes, period)` -- the
  Average True Range, null through the warm-up.
- `computeLinePgo(highs, lows, closes, period)` -- the Pretty
  Good Oscillator series, null through the warm-up and for
  zero-ATR markets.
- `runLinePgo(points, options?)` -- sort finite points by `x`,
  run the pipeline, return `{ series, period, pgo, samples,
  pgoFinal, pgoMin, pgoMax, positiveCount, negativeCount, ok }`.
  Each sample is classified `positive` / `negative` / `zero`.
  `ok` is false for fewer than two points.
- `computeLinePgoLayout(options)` -- stack the price panel above
  a zero-centred PGO panel; project the price and PGO paths plus
  markers and the zero line.
- `describeLinePgoChart(data, options?)` -- ARIA description.

## Component

`ChartLinePgo` is a `forwardRef` region (`role="region"`) with
an ARIA-linked description, a `role="img"` SVG, two stacked
panels (price on top, PGO below) with their own axis ticks and
grid, panel labels, a config badge showing the period, a
two-series legend (Price / PGO) with toggle, a hover/focus
tooltip exposing the close, high, low, PGO and sign, and a
`motion-safe:animate-fade-in` entrance. It consumes
`{ x, value, high, low }` points. The PGO panel carries a dashed
zero line and sign-coloured markers (green positive, red
negative, grey zero). Every node carries a `data-section`
attribute.

## Distinctness

- **vs a raw distance-from-MA oscillator** -- a plain
  `close - SMA` is in price units and is not comparable across
  instruments. The PGO divides by the ATR, so the reading is
  scale-free: a PGO of 1 means "one ATR above the mean"
  regardless of the instrument's volatility.
- **vs chart-line-cog** -- the CoG measures the centroid of the
  price window. The PGO measures the displacement of the close
  from a plain moving average, normalized by volatility.
- It requires `high` and `low` fields on each data point
  (alongside the plotted `value`, the close) to build the true
  range.

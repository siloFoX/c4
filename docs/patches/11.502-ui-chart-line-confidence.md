# TODO 11.502 - chart-line-confidence primitive

Status: SHIPPED v1.11.520 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartLineConfidence>`, a pure-SVG multi-series line chart with
**confidence band overlay**: each series carries optional per-point
`yLower` / `yUpper` bounds; the primitive paints a filled polygon
between the two bound curves and overlays the central line on top.
Common pattern in forecasting (point estimate + prediction interval),
metric dashboards (median + p5/p95), and statistical models.

## API

```tsx
<ChartLineConfidence
  series={[
    {
      id: 'forecast',
      label: 'Forecast',
      data: [
        { x: 0, y: 100, yLower: 95, yUpper: 105 },
        { x: 1, y: 105, yLower: 96, yUpper: 114 },
        { x: 2, y: 110, yLower: 98, yUpper: 122 },
        { x: 3, y: 115, yLower: 100, yUpper: 130 },
      ],
    },
  ]}
/>
```

## Distinct from prior chart-* primitives

- `<ChartLine>` - single y value per point; no band overlay.
- `<ChartLineStep>` (11.497) - step interpolation; no band.
- `<ChartLineSmooth>` (11.498) - Bezier / monotone curves; no band.
- `<ChartLineDashed>` (11.499) - dash patterns; no band.
- `<ChartLineMulti>` (11.500) - synchronised crosshair tooltip; no
  band.
- `<ChartLineZoom>` (11.501) - brush-to-zoom; no band.
- `<ChartErrorBars>` - vertical bars per point; chart-line-confidence
  paints a continuous filled polygon.
- `<ChartArea>` - filled area from baseline to single y; this
  primitive fills between TWO curves (yLower and yUpper).

## Band contract

### Per-point shape

`ChartLineConfidencePoint = { x: number; y: number; yLower?: number;
yUpper?: number }`. A point participates in the band only when:

1. `x` and `y` are finite (otherwise the whole point is dropped from
   the layout).
2. `yLower` and `yUpper` are both finite.
3. `yLower <= yUpper` (inverted bounds are rejected -- the test
   `hasFiniteBand` enforces this).

`hasBand=true` is recorded per layout point so adopters can style
the corresponding dot differently.

### Segment building

`buildLineConfidenceBandSegments(points)`:

1. Walk the points in input order.
2. Push consecutive points where `hasBand=true` into the current run.
3. When the run breaks (invalid band point, or end of array), flush
   the run as one closed polygon if it has >= 2 points (a single
   point can't form a polygon).
4. Each polygon is `M low[0] L low[1] ... L low[n-1] L up[n-1] ... L
   up[0] Z` so the path traces the lower bound left-to-right then
   the upper bound right-to-left.

A series with all valid bounds produces exactly one polygon. A gap
in the band emits one polygon per contiguous run.

### Series-level toggle

`series.showBand !== false` enables the band (default true). Setting
`showBand: false` makes the series render as a plain line (useful
when one series in a chart has bounds and another doesn't).

## Layout

- `getLineConfidenceBounds` expands the y range to include `yLower`
  and `yUpper` for every band-valid visible point. Pure
  forecast-style series with no bounds still contribute (just `y`).
- Each band path is wrapped under the series `<g>` BEFORE the line
  path so the band fill sits behind the line stroke.
- Hover or focus on a dot dims every other series stroke + band to
  ~30% so the focused series visually dominates.

## Interactivity

- Hover or focus on a dot opens a tooltip showing series label,
  `x` (via `formatX`), `y` (via `formatValue`), and (when the dot
  has a band) a `band:` row with `yLower` to `yUpper`.
- Per-dot `onPointClick({series, point})` callback.
- Legend buttons toggle series visibility (controlled +
  uncontrolled). Each legend swatch is a tiny band rectangle plus a
  line so adopters see both styles at a glance.
- Mount fade via `motion-safe:animate-fade-in` (suppress via
  `animate=false`).

## Exported helpers

- `DEFAULT_CHART_LINE_CONFIDENCE_*` numeric / string defaults.
- `DEFAULT_CHART_LINE_CONFIDENCE_PALETTE` (10-colour palette).
- `getLineConfidenceDefaultColor(index)` - palette wrap.
- `getLineConfidenceFinitePoints(points)` - drops points with
  non-finite `x` or `y`.
- `getLineConfidenceBounds(series, hidden?)` - per-axis bounds with
  band expansion when valid.
- `getLineConfidenceTicks(min, max, count?)` - evenly-spaced ticks.
- `buildLineConfidenceLinePath(points)` - `M x y L x y ...` central
  line.
- `buildLineConfidenceBandSegments(points)` - canonical closed
  polygon builder; one polygon per contiguous run of `hasBand`
  points; single-point runs are discarded.
- `computeLineConfidenceLayout(input)` - per-series layout with
  `bandSegments` array and per-point `hasBand` flag.
- `describeLineConfidenceChart(series, hidden?, formatValue?)`
  - ARIA summary "Line chart with confidence band: N series and K
  points, B with bands.".

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only
  `data-section="chart-line-confidence-aria-desc"` span.
- SVG `role="img"` with the same `aria-label`.
- Each line path is `role="graphics-symbol"` with `tabIndex={0}`
  and `aria-label="<series>: line with K points and B band points"`.
- Each dot is `role="graphics-symbol"` with `tabIndex={0}` and a
  band-aware label.

TypeScript clean for touched files.

## Out of scope

- Multiple stacked confidence levels per series (e.g. p5/p95 +
  p25/p75) - one band per series in v1; host composes multiple
  instances.
- Curved band edges (Bezier / monotone) - linear segments only.
  Combine with `<ChartLineSmooth>` for smooth lines and pre-compute
  band points densely.
- Animated band transitions - snap; mount fade only.
- Log-scale axes - linear only.
- Brush-to-zoom or synchronised crosshair tooltip - use
  `<ChartLineZoom>` or `<ChartLineMulti>` respectively for those
  interactions.

## What changed

### Section headings

```ts
interface ContextMenuItem {
  // ... existing
  sectionHeading?: boolean;
}
```

When `sectionHeading: true`, the item renders
as a non-interactive uppercase label row
(`role="presentation"`) above the items that
follow. Useful for grouping items by purpose:

```tsx
const items = [
  { id: 'h-edit', label: 'Edit', sectionHeading: true },
  { id: 'cut', label: 'Cut' },
  { id: 'copy', label: 'Copy' },
  { id: 'sep', label: '', separator: true },
  { id: 'h-view', label: 'View', sectionHeading: true },
  { id: 'zoom', label: 'Zoom in' },
];
```

Section heading rows are NOT selectable
via keyboard nav -- `firstSelectable` /
`findNextSelectable` / `lastSelectable` skip
them.

### Sub-menu support

```ts
interface ContextMenuItem {
  // ... existing
  items?: ContextMenuItem[];
}
```

When a parent item has `items`, it renders
with a trailing ChevronRight glyph. The
parent button gets `aria-haspopup="menu"`
and `aria-expanded` mirroring the sub-menu
visibility.

```tsx
const items = [
  { id: 'open', label: 'Open' },
  {
    id: 'more',
    label: 'More',
    items: [
      { id: 'a', label: 'Reload' },
      { id: 'b', label: 'Duplicate' },
      { id: 'sep', label: '', separator: true },
      { id: 'c', label: 'Delete', danger: true },
    ],
  },
];
```

Behaviour:

- **Hover open** -- mouseenter on a parent
  auto-opens the sub-menu; mouseenter on a
  non-parent closes the active sub-menu.
- **ArrowRight open** -- ArrowRight on a
  highlighted parent opens the sub-menu.
- **Enter on parent opens sub-menu** (does
  NOT fire `onSelect` -- a parent is an
  opener, not a leaf).
- **ArrowLeft close** -- closes the
  sub-menu and returns highlight to the
  parent.
- **Enter on sub-menu item** fires
  `onSelect` and closes ALL menus
  (root + sub).
- **Escape inside sub-menu** closes only
  the sub-menu (focus-trap Escape closes
  the whole tree from the root).
- **Click outside** dismisses both root
  AND sub-menu.

### Home / End

`Home` jumps to the first selectable item;
`End` jumps to the last selectable item.
Both work inside the parent menu AND the
sub-menu.

## Render contract

The sub-menu mounts as a sibling of the
root menu inside the same portal Fragment.
Position is computed from the parent item's
`getBoundingClientRect().right` /
`.top` so the sub-menu sits flush with the
parent panel's right edge.

## Data attributes

| attr | location |
| --- | --- |
| `data-section="context-menu"` | root portal panel |
| `data-section="context-menu-submenu"` | sub-menu panel |
| `data-parent="<parent-id>"` | sub-menu panel |
| `data-section="context-menu-item"` | every item li |
| `data-context-menu-item="<id>"` | every item li |
| `data-section="context-menu-separator"` | separator li |
| `data-section="context-menu-section-heading"` | heading li |
| `data-section="context-menu-submenu-chevron"` | trailing chevron svg |

## Pairs with existing primitives

- `<DropdownMenu>` (11.362) -- click-
  triggered menu; same item shape but
  the trigger is a button, not a
  right-click target.
- `<HoverCard>` (11.385) -- preview
  popover; reach for HoverCard when the
  surface needs rich layout, not a
  list of menu items.
- `<Popover>` (11.275) -- generic
  click-triggered floating panel.

## Out of scope

- Per-page adoption of `sectionHeading`
  or `items`. Additive; every existing
  call site stays byte-identical at the
  legacy item shape.
- Nested-nested sub-menus (level 3+).
  Current implementation supports one
  level of nesting; deeper trees need a
  recursive panel render with a shared
  open-path stack.
- ArrowLeft on the root menu (no parent
  to return to). The contract only
  handles ArrowLeft when a sub-menu is
  active.
- Type-ahead (jump to item starting with
  typed letter). Belongs in a separate
  patch with its own debounce story.
- Custom sub-menu positioning. Defaults
  to right-of-parent; browser-side
  overflow handles flips. A `placement`
  prop is a follow-on.
- Sticky / pinned sub-menu (one that
  stays open after pointerleave).
  Belongs in a separate "menu pin"
  primitive.

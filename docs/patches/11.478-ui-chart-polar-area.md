## Summary

Add a new `<ChartPolarArea>` UI primitive in
`web/src/components/ui/chart-polar-area.tsx` -- pure-SVG
Nightingale rose / polar area chart. Every wedge gets an
**equal angular slice** of the full circle (`2π / N`);
**radius encodes value**. The default radius scaling is
`sqrt` (so wedge area is proportional to the value, the
honest Nightingale-style encoding); a `linear` mode is
available when the host needs radius itself proportional
to value. Distinct from `<ChartPie>` (11.440) and
`<ChartDonut>` (11.469) (where angle encodes value and
radius is constant) and from `<ChartRadialBar>`
(11.452) (which paints separate concentric rings per
series rather than one wedge per category).

## Behaviour

- Renders `<div role="region">` wrapping
  `<svg role="img">` with the same `aria-label`.
- Layout: `computePolarAreaLayout` carves the full
  circle into one wedge per visible item. Each wedge
  occupies the same angular slice (`2π / N`, minus
  pad-angle per wedge when `padAngle > 0`). The
  outer radius of each wedge encodes `value`:
  - `radiusMode='sqrt'` (default): wedge area is
    proportional to value (the honest Nightingale
    encoding, since wedge area = (angle/2) * r^2).
  - `radiusMode='linear'`: radius itself is
    proportional to value (over-emphasises larger
    values visually; provided for adopters who
    insist).
- `maxValue` prop overrides the auto-derived peak
  so multiple `<ChartPolarArea>` instances can
  share a comparable radial scale.
- Non-positive / non-finite values render no wedge
  (radius = 0).
- Hover/focus on a wedge dims every other wedge to
  opacity 0.3 (focused wedge dominates) and opens
  a tooltip with label, formatted value, and the
  percentage ratio against `maxValue`.
- Axis rings paint at every tick value (one per
  evenly-spaced step from 0..maxValue); axis labels
  paint at the top of every ring (the 0 ring is
  skipped to avoid clutter).
- Optional in-wedge labels at the centroid (62%
  radius) when `showWedgeLabels=true` and the
  rendered radius is large enough to host text.
- Legend (when present): focusable
  `<button aria-pressed>` per wedge. Controlled via
  `hiddenWedges` + `onHiddenWedgesChange`, or
  uncontrolled via `defaultHiddenWedges`. Toggling
  fires `onWedgeToggle({wedge, hidden})` with the
  next state.
- Mount animation:
  `motion-safe:animate-fade-in` per wedge.
  `animate=false` removes the class.
- Default ARIA description: `"Polar area chart
  (<mode> radius) with N wedges, peak value
  <peak>."` Empty / all-hidden / non-finite-only
  returns `"No data"`; `ariaDescription` overrides.

- `getPolarAreaRadius`: returns 0 for non-positive
  value or non-positive max; clamps ratio above 1
  to the outer radius.
- `computePolarAreaLayout` returns wedge entries of
  shape `{ id, label, index, value, color,
  startAngle, endAngle, midAngle, outerRadius,
  ratio, path, midLabelX, midLabelY }`. Empty
  inputs / all-hidden / non-positive radius / zero
  max yield an empty wedges array.

## Pairs with existing primitives

- `<ChartPie>` (11.440) / `<ChartDonut>` (11.469)
  -- angle-encodes-value siblings with constant
  radius.
- `<ChartRadialBar>` (11.452) -- concentric-rings
  alternative where each series is its own ring.
- `<ChartRadar>` (11.445) -- polygon-on-axes
  alternative.

## Out of scope

- Stacked polar wedges (multi-series ring stacked
  per wedge) -- one wedge per item v1.
- Animated transitions on data / radiusMode change
  -- snap v1; mount fade only.
- Custom kernel for the angular smoothing -- no
  smoothing v1; angles are exact.
- Polar tick marks per angle (use `<ChartRadar>`
  axes for tick labels around the perimeter).
- Negative values (Nightingale rose requires
  positive areas; non-positive values skip).

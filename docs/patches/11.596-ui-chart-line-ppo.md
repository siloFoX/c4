## Summary

New `ChartLinePpo` UI primitive at
`web/src/components/ui/chart-line-ppo.tsx` -- a pure-SVG line chart
with a Percentage Price Oscillator panel rendering the PPO line, the
signal line, and the histogram. The value line occupies the top
panel; the bottom panel renders the PPO line, a dashed signal line,
and sign-coloured histogram bars anchored on a dashed zero centerline.

## Indicator math

The Percentage Price Oscillator is the percentage-normalised analog
of MACD. With a fast period, a slow period and a signal period
(defaults 12 / 26 / 9):

```
fastEMA = EMA(values, fastPeriod)
slowEMA = EMA(values, slowPeriod)
PPO     = slowEMA === 0 ? 0 : 100 * (fastEMA - slowEMA) / slowEMA
signal  = EMA(PPO, signalPeriod)
histogram = PPO - signal
```

`computeLinePpoEma(src, period)` is an EMA that tolerates leading
`null` placeholders (the PPO line has them): it collects the defined
values in order, seeds with the simple mean of the first `period` of
them placed at that value's index, then folds each later defined
value in at weight `2 / (period + 1)`. Indices before the seed read
`null`.

A negative-zero PPO (a zero numerator over a negative slow EMA) is
normalised to positive zero.

### Worked fixture

`values = [4, 4, 4, 12, 12, 12, 12, 12]`, `fastPeriod = 1`,
`slowPeriod = 3`, `signalPeriod = 3`:

```
fastEMA = [4, 4, 4, 12,  12,    12,     12,      12]      (period 1 = the series)
slowEMA = [.,  .,  4,  8,  10,    11,   11.5,   11.75]    (mult 0.5, SMA seed at i=2)
PPO     = [.,  .,  0, 50,  20, 100/11, 50/11.5, 25/11.75]
signal  = [.,  .,  .,  ., 70/3,    ...]                   (EMA of PPO, mult 0.5)
histogram = PPO - signal, defined from index 4 onward (all negative here)
```

`signal[4] = mean(PPO[2], PPO[3], PPO[4]) = mean(0, 50, 20) = 70 / 3`;
`histogram[4] = 20 - 70/3 = -10/3`.

## API

- `getLinePpoFinitePoints(points)` -- drop points with a non-finite
  `x` or `value`.
- `normalizeLinePpoPeriod(period, fallback)` -- positive integer;
  non-finite / sub-1 falls back, fractional floors.
- `computeLinePpoEma(src, period)` -- the null-tolerant EMA series.
- `computeLinePpo(values, fastPeriod, slowPeriod, signalPeriod)` --
  returns `{ ppo, signal, histogram }`.
- `runLinePpo(points, options?)` -- sort finite points by `x`, compute
  the three series, return `{ series, fastPeriod, slowPeriod,
  signalPeriod, ppo, signal, histogram, samples, ppoFinal,
  signalFinal, histogramFinal, ppoMin, ppoMax, positiveCount,
  negativeCount, ok }`. Each sample carries a `positive` / `negative`
  / `zero` histogram sign. `ok` is false for fewer than two points.
- `computeLinePpoLayout(options)` -- stack a value panel above a PPO
  panel with a symmetric y-bound. Returns panels, ticks, the value
  path, the PPO path, the signal path, zero-anchored histogram bars,
  sign-coloured markers, and the zero centerline `y`.
- `describeLinePpoChart(data, options?)` -- ARIA description.

## Component

`ChartLinePpo` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge (EMA and signal periods), a four-series legend (Value / PPO /
Signal / Histogram) with toggle, a dashed zero centerline,
zero-anchored histogram bars, the PPO line, a dashed signal line,
markers on the PPO line coloured by histogram sign, a hover/focus
tooltip exposing the value, PPO, signal, histogram, and sign, and a
`motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs chart-line-macd** -- MACD plots the *raw* difference of two
  EMAs (`fastEMA - slowEMA`) in absolute price units. The PPO divides
  that same difference by the slow EMA and scales by 100, so the
  reading is a *percentage*. That normalisation -- absent from MACD --
  is what makes the PPO comparable across instruments trading at
  different price levels and across time as a single instrument's
  price drifts. The signal line and histogram share MACD's shape but
  are built on the percentage PPO line, not the absolute MACD line.
- **vs chart-line-coppock / chart-line-trix** -- those smooth ROC
  legs; the PPO is a ratio of two EMAs of the *price itself*.

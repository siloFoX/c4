# 11.1009 -- UI: chart-line-dpo-divergence-cross primitive (v1.11.1027)

## Scope

Add a pure-SVG dual-panel `ChartLineDpoDivergenceCross`
component that renders the close in the top panel and the
Detrended Price Oscillator (DPO) line in the bottom panel,
marking discrete price-versus-DPO direction disagreement events
as bullish / bearish divergence cross transitions. Divergence
variant of the detrended momentum family; pairs the canonical
DPO formula already used by `chart-line-dpo-cross-sig`,
`chart-line-dpo-overbought-cross`, and
`chart-line-dpo-oversold-cross` (shifted close minus rolling
SMA) with the same 5-state regime model already used by the
other divergence primitives.

## DPO pipeline (Vitali Apollonio)

```
shift   = floor(length / 2) + 1
sma[i]  = SMA(close[i-length+1..i], length)
dpo[i]  = close[i - shift] - sma[i]
```

The shift looks back to remove the bulk of the SMA's centred
lag, so the residual tracks short-term momentum around the
trend. Same formula as the existing DPO primitives in the
codebase (`chart-line-dpo-cross-sig` etc) -- only the
classification layer changes here.

Warmup is `length - 1`: the SMA needs `length` valid closes to
seed, and once `i >= length - 1` the shifted close lookup
`close[i - shift]` is also in range because
`shift = floor(length/2) + 1 < length` for any `length >= 2`.

## Divergence regime model

5-state classifier exactly like the other divergence primitives:

```
priceUp = close[i] > close[i - divergenceWindow]
dpoUp   = dpo[i]   > dpo[i - divergenceWindow]
state   = aligned-bullish    if priceUp && dpoUp
        | aligned-bearish    if !priceUp && !dpoUp
        | divergent-bullish  if !priceUp && dpoUp   (price down, DPO up)
        | divergent-bearish  if priceUp && !dpoUp   (price up, DPO down)
        | none               if either input is null
bullish-cross : prev != 'divergent-bullish' && cur == 'divergent-bullish'
bearish-cross : prev != 'divergent-bearish' && cur == 'divergent-bearish'
```

Defaults: `length = 20` (Vitali Apollonio's canonical DPO
window), `divergenceWindow = 5`. Crosses suppress when either
`prev` or `cur` is `none`, so warmup transitions never fire
spuriously. Strict inequality guarantees boundary equality
(DPO parked at a constant offset) falls into the
aligned-bearish state instead of double-firing.

## Bit-exact anchors

| input              | dpo (post-warmup)    | regime (post i=24)  | crosses |
| ------------------ | -------------------- | ------------------- | ------- |
| CONST close = K    | 0                    | aligned-bearish     | 0       |
| LINEAR UP, close=i | -1.5 (constant)      | divergent-bearish   | 0       |
| LINEAR DOWN, close=-i | +1.5 (constant)   | aligned-bearish     | 0       |

CONST K verified across K = 0..1234 -- both `close[i - shift]`
and `sma[i]` collapse to K so `dpo = K - K = 0` and priceUp =
false (`close[i] === close[i-5]`, strict `>`).

LINEAR UP exercises the structural bias of the DPO formula:
the SMA of `[i-19..i]` has the average `i - 9.5` (centred lag
of `(length - 1) / 2 = 9.5`), while `close[i - shift] =
close[i - 11] = i - 11`. The difference is therefore the
constant `(i - 11) - (i - 9.5) = -1.5` for every post-warmup
sample. priceUp = true, dpoUp = false (constant -> not strictly
greater) -> regime `divergent-bearish` for the full 56 valid
samples on the 80-bar fixture. The cross count stays at 0
because the regime is entered from `none`.

LINEAR DOWN is the algebraic mirror: the SMA sits at `-i + 9.5`
and `close[i - 11] = -i + 11`, giving the constant `+1.5`.
priceUp = false, dpoUp = false -> regime `aligned-bearish`.

(Note: the constant-positive / constant-negative DPO bias on
pure trending input is the canonical signature of the DPO
formula's centred-SMA construction -- it does not represent a
real divergence signal in production, only an algebraic
anchor for testing the regime classifier.)

### Implementation note

The project's runtime budget is zero
new deps. Pulling Slate / Lexical /
ProseMirror as a runtime dependency
is out of scope. Instead, the editor
ships a Slate-FLAVOURED contract
(controlled HTML string `value`,
toolbar commands, paste sanitizer)
over native `contentEditable` +
`document.execCommand`. Migration to
a structured document model is a
follow-on; the public surface stays
portable.

## API

```ts
interface RichTextEditorProps {
  value: string;
  onChange?: (next: string) => void;
  placeholder?: string;
  readOnly?: boolean;
  ariaLabel?: string;
  className?: string;
  allowedTags?: string[];
  allowedAttrs?: Record<string, string[]>;
  showToolbar?: boolean;

  onPaste?: (sanitized: string, raw: string) => void;
  // ...HTMLAttributes<HTMLDivElement>
    (minus onChange + onPaste)
}
```

## Behaviour

### Controlled HTML

The host owns the HTML string via
`value`. The editor mirrors that
into the contentEditable
`innerHTML` whenever the prop
differs from the current DOM. On
every input the editor emits the
new innerHTML via `onChange`.

The mirror-only-when-different
guard preserves caret position
during typing -- naive controlled
contentEditable would bounce the
cursor.

### Toolbar

Six buttons drive
`document.execCommand`:

| label | command |
| --- | --- |
| Bold | bold |
| Italic | italic |
| Underline | underline |
| Bullet list | insertUnorderedList |
| Numbered list | insertOrderedList |
| Link | createLink |

After each command, the new
`innerHTML` propagates via
`onChange`.

Disabled when `readOnly` is true.

### Link insertion

The Link button calls
`onLinkRequest()` if supplied,
otherwise falls back to
`window.prompt('Link URL')`. The
returned URL passes through
`safeRichTextHref`:

| input | result |
| --- | --- |
| empty | null (no-op) |
| `http(s)://...` | passthrough |
| `mailto:` / `tel:` | passthrough |
| relative path / fragment | passthrough |
| `javascript:` / `data:` / `vbscript:` / `file:` | null (no-op) |
| unknown scheme | null (no-op) |

A `null` result means the
operation is silently dropped (no
execCommand call, no innerHTML
change).

### Paste sanitization

`onPaste` intercepts the
clipboard:

1. Read `text/html` (fall back to
   `text/plain`).
2. Run through
   `sanitizeHtmlForRichText`:
   - Tags NOT in `allowedTags`
     are inline-stripped (children
     survive, wrapper removed).
     `<script>` / `<iframe>` /
     `<style>` collapse.
   - Attributes NOT in
     `allowedAttrs[tag]` are
     dropped. `onclick`,
     `onload`, arbitrary
     `style`, etc. evaporate.
   - `<a href="...">` runs
     through `safeRichTextHref`;
     rejected URLs have the href
     omitted. Survivors gain a
     defensive
     `rel="noopener noreferrer"`
     + `target="_blank"`.
3. Insert the sanitized HTML via
   `execCommand('insertHTML', ...)`.
4. Fire `onPaste(sanitized,
   raw)` so the host can record
   metrics / diff the cleanup.

### Read-only

- `contentEditable` flips to
  `false`.
- `aria-readonly="true"`.
- Toolbar buttons disable.
- Paste handler short-circuits
  (event still fires but no
  insertion).
- `data-read-only="true"`.

### Placeholder

The placeholder string serializes
to `data-placeholder` on the
content element. The component
also flips a `data-empty="true"`
attribute on the root when the
value is empty (`""`, `"<br>"`,
or `"<p></p>"`). Hosts can render
the placeholder via:

```css
[data-section="rich-text-editor-content"][data-empty="true"]::before {
  content: attr(data-placeholder);
  color: var(--muted-foreground);
}
```

## ARIA + data attributes

Root (wrapper):

- `data-section="rich-text-editor"`
- `data-read-only`
  ('true' / 'false')
- `data-empty` ('true' / 'false')

Toolbar:

- `role="toolbar"` +
  `aria-label="Formatting"`
- `data-section="rich-text-editor-toolbar"`
- Per button:
  `data-section="rich-text-editor-toolbar-button"`
  + `data-command`

Content:

- `role="textbox"` +
  `aria-label` +
  `aria-multiline="true"` +
  `aria-readonly` (set when
  readOnly is true)
- `data-section="rich-text-editor-content"`
- `data-placeholder`

## Out of scope (deferred)

- True Slate / Lexical /
  ProseMirror backend. The
  primitive uses
  `document.execCommand` (the
  canonical native API). Migration
  to a structured document model
  is a larger follow-on; the
  surface contract is portable.
- Mention / hashtag autocomplete
  popovers. Host layer.
- Image / file uploads. Text +
  inline formatting only for v1.
- Tables. Out of scope for v1.
- Undo / redo on top of native
  browser undo. Belongs in a
  follow-on once the document
  model is structured.
- Collaborative editing (OT /
  CRDT). Out of scope.
- Adoption sweep through
  existing comment / chat
  composers. RichTextEditor is
  the canonical surface for new
  adopters; existing ad-hoc
  textareas ship until a
  migration dispatch.

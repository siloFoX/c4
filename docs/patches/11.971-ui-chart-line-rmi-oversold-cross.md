## Summary

Adds the `ChartLineRmiOversoldCross` primitive
(`web/src/components/ui/chart-line-rmi-oversold-cross.tsx`): a
pure SVG dual-panel chart that visualizes Relative Momentum Index
(RMI) crossings through the canonical oversold line 30. Mirror of
`chart-line-rmi-overbought-cross` (11.970): identical RSI-with-
lookback Wilder pipeline, opposite threshold semantics (30 vs 70).
Together with `chart-line-rmi-zero-cross` (11.948) and
`chart-line-rmi-mid-cross` (11.967), this completes the canonical
threshold-crossing layer for the RMI indicator family
(overbought + oversold + centerline + zero).

## Pipeline

Same RMI pipeline shared with the rmi-zero / mid / overbought
siblings:

1. `mom[i] = close[i] - close[i - lookback]` (i >= lookback)
2. `gain[i] = max(0, mom)`, `loss[i] = max(0, -mom)`
3. `avgGain = Wilder(gain, length)`,
   `avgLoss = Wilder(loss, length)` -- SMA-seeded at
   `lookback + length - 1`, then
   `prev = (prev * (n - 1) + value) / n`.
4. RMI from the smoothed ratio with explicit fallbacks:
   - `avgLoss == 0 && avgGain == 0` -> RMI = 50 (neutral).
   - `avgLoss == 0 && avgGain > 0`  -> RMI = 100.
   - `avgGain == 0`                  -> RMI = 0.
   - else                            -> `100 - 100 / (1 + g / l)`.
5. Regime classifier (strict `>=`):
   - `RMI >= 30` -> bullish (above oversold zone)
   - `RMI <  30` -> bearish (in oversold zone)
6. Cross detector emits a `bullish` cross when
   `RMI[i-1] <= 30 && RMI[i] > 30` and a `bearish` cross when
   `RMI[i-1] >= 30 && RMI[i] < 30`. Strict inequality on the
   current side means the boundary value 30 does not double-fire.

## Defaults

- `length = 14`
- `lookback = 5` (Altman's canonical settings)
- `threshold = 30` (canonical oversold line)
- Warmup boundary: bar `length + lookback - 1 = 18`, so the first
  18 samples (40-bar run) are null and the remaining 22 are
  populated.

## Bit-exact anchors

- CONST `close = K` for all `K in {0, 1, 42, 100, 1234}` ->
  `rmi[i] === 50` literal for every `i >= 18`. Regime classifies
  as bullish because 50 >= 30 -- mirror of the overbought variant
  where 50 < 70 gave bearish.
- LINEAR UP `close = i` -> `rmi[i] === 100` constant from i = 18
  onward (no losses, avgLoss = 0, avgGain > 0). All bullish
  (100 >= 30), but 0 crosses because RMI jumps directly from null
  to 100 with no `prev <= threshold && cur > threshold`
  transition.
- LINEAR DOWN `close = -i` -> `rmi[i] === 0` constant. All bearish
  (0 < 30), 0 crosses (RMI jumps from null to 0, prev-null guard).
- CONST K full-run counters under defaults, 40 bars:
  `noneCount = 18`, `bullishCount = 22`, `bearishCount = 0`,
  `crosses.length = 0`.
- `pricePath` and `rmiPath` are byte-identical across repeated
  layout calls, with `.toFixed(2)` precision and `M ... L ...`
  command shapes. The rmi path uses a single `M` for post-warmup
  data.
- `thresholdY` at 30 sits BELOW the panel midpoint in SVG y-space
  (i.e., a larger y value) -- exact mirror of the overbought
  variant where threshold=70 sat above the midpoint.
- Cross detector boundary: strict inequality (`> T`, `< T`).

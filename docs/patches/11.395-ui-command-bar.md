  -- bottom-anchored quick action bar

  contextual actions, batch operations,
  ARIA toolbar role.

Pairs with `<VirtualTable>` (11.375) +
any host that owns its own selection
state.

## API

```ts
interface CommandBarAction {
  id: string;
  label: string;
  onClick: () => void;
  icon?: ReactNode;
  variant?: 'default' | 'primary' | 'destructive';
  disabled?: boolean;
  shortcut?: string;
  ariaLabel?: string;
}

interface CommandBarSeparator {
  id: string;
  type: 'separator';
}

type CommandBarItem = CommandBarAction | CommandBarSeparator;

interface CommandBarProps {
  selectedCount: number;
  actions: CommandBarItem[];
  onClearSelection?: () => void;
  selectionLabel?: (count: number) => ReactNode;
  visible?: boolean;
  position?: 'bottom' | 'top' | 'static';
  align?: 'left' | 'center' | 'right';
  className?: string;
  ariaLabel?: string;
  showClearButton?: boolean;
  clearLabel?: string;
  motionSafe?: boolean;
}
```

## Behaviour

### Visibility

| `visible` | `selectedCount` | rendered? |
| --- | --- | --- |
| `undefined` | `> 0` | yes |
| `undefined` | `<= 0` | no |
| `true` | any | yes |
| `false` | any | no |

When not rendered the component returns
`null` (no DOM node, no fixed-positioned
ghost).

### Toolbar contract (WAI-ARIA)

- Root: `<div role="toolbar"
  aria-orientation="horizontal"
  aria-label>`.
- Single-tabstop pattern -- exactly one
  button carries `tabIndex=0` at a time;
  the others carry `tabIndex=-1`. The
  active tabstop moves with the focus,
  not on every render.
- Arrow keys:
  - `ArrowRight` -> next focusable
    action.
  - `ArrowLeft` -> previous focusable
    (wraps).
  - `Home` -> first focusable.
  - `End` -> last focusable.
  - Disabled actions + separators are
    skipped.
- `Escape` -> `onClearSelection()` if
  the handler is supplied. No-op
  otherwise.

### Action variants

`variant` selects from a fixed palette
(default / primary / destructive). Each
maps to the canonical token-based
class (border, bg, hover). The host
cannot pass arbitrary class -- this is
intentional. Variants are the contract.

### Positioning

| position | layout |
| --- | --- |
| `bottom` (default) | `fixed left-1/2 bottom-4 -translate-x-1/2 z-40` |
| `top` | `fixed left-1/2 top-4 -translate-x-1/2 z-40` |
| `static` | inline; `align` controls `mr-auto` / `mx-auto` / `ml-auto` |

In static mode the host is responsible
for the surrounding layout.

### Selection label

`defaultSelectionLabel(count)`:
- `count <= 0` -> "No selection"
- `count === 1` -> "1 selected"
- otherwise -> "<n> selected"

Override via `selectionLabel: (count) =>
ReactNode` for i18n or custom phrasing.

### Clear button

Renders when `onClearSelection` is
supplied AND `showClearButton !==
false`. Click + `Escape` both trigger
the same callback. `clearLabel`
overrides the default "Clear
selection" string -- which is also
used as the button's `aria-label`.

### Disabled actions

- `disabled: true` ->
  `disabled={true}` on the native
  `<button>`, `data-disabled="true"`,
  click handler stripped, focus cycle
  skips the action.
- The CSS class adds
  `disabled:cursor-not-allowed
  disabled:opacity-50` so visually
  disabled state matches assistive
  tech state.

### Motion

`motionSafe` (default true) adds the
existing Tailwind motion-safe ladder:
`motion-safe:animate-in
motion-safe:fade-in
motion-safe:slide-in-from-bottom-4`.
Disabled by passing `motionSafe={false}`
for storybook captures or environments
that opt out of animations.

## Data attributes

| location | attrs |
| --- | --- |
| root | `role="toolbar"`, `aria-orientation`, `aria-label`, `data-section="command-bar"`, `data-position`, `data-align`, `data-selected-count` |
| selection label | `data-section="command-bar-selection-label"` |
| actions wrapper | `data-section="command-bar-actions"` |
| action button | `data-section="command-bar-action"`, `data-action-id`, `data-action-variant`, `data-disabled` |
| action icon | `data-section="command-bar-action-icon"` |
| action label | `data-section="command-bar-action-label"` |
| action shortcut | `data-section="command-bar-action-shortcut"` |
| separator | `role="separator"`, `aria-orientation="vertical"`, `data-section="command-bar-separator"` |
| clear button | `data-section="command-bar-clear"`, dynamic `aria-label` |

## Pairs with existing primitives

- `<VirtualTable>` (11.375) -- the
  canonical batch-selection surface.
  CommandBar appears below the table
  while rows are selected.
- `<Toolbar>` -- this primitive is the
  selection-scoped variant; a general-
  purpose page-level toolbar is a
  separate dispatch.
- `<Menubar>` (11.387) -- top-of-page
  application menu (different
  affordance -- menus vs. quick
  actions).

## Out of scope (deferred)

- Drag-to-reorder. The `actions`
  array order is canonical.
- Overflow menu when too many
  actions to fit horizontally.
  Belongs in a follow-on once a real
  adopter hits the limit.
- Per-page adoption sweep.
- Animated count-up when selection
  grows. The label updates instantly.
- Multi-selection-state mode (union
  vs. intersection). Single count
  + flat action list is canonical.
- `aria-keyshortcuts` mapping from
  the `shortcut` field. The `<kbd>`
  is decorative; binding the
  shortcut to a global hotkey is
  the caller's responsibility.

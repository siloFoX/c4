## Summary

New `ChartLineAlma` UI primitive at
`web/src/components/ui/chart-line-alma.tsx` -- a pure-SVG line chart
with an Arnaud Legoux Moving Average overlay using a gaussian-weighted
window. Like `chart-line-hma` and `chart-line-zlema` it is a
**single-panel** chart: the ALMA line is drawn directly over the
price line on shared axes.

## Indicator math

Arnaud Legoux's Moving Average, with a period (default 9), an offset
(default 0.85) and a sigma (default 6):

```
m       = offset * (period - 1)
s       = period / sigma
w[k]    = exp(-(k - m)^2 / (2 * s^2))     for k in 0..period-1
ALMA[i] = sum(w[k] * value[i - (period-1) + k]) / sum(w)
```

`computeLineAlmaWeights(period, offset, sigma)` builds the Gaussian
window weights -- a bell curve centred at `m`. An offset near 1
pushes the peak toward the recent end of the window (cutting lag); a
larger sigma narrows the curve (stronger noise filtering).
`computeLineAlma` averages each window with those weights, normalised
by their sum. Because every weight is positive, each ALMA reading
stays inside its window's value range.

### Worked fixture

`values = [0, 6, 12, 18, 24]` (a step-6 linear ramp), `period = 3`,
`offset = 0.5`, `sigma = 2`:

With a centred offset the weights are symmetric `[w, 1, w]`, so the
ALMA of a linear ramp is the window centre exactly:

```
weights ~= [0.8007, 1, 0.8007]
alma    ~= [null, null, 6, 12, 18]
```

The price (`12 / 18 / 24`) runs above the lagging ALMA
(`6 / 12 / 18`) -- three `above` bars in the uptrend.

## API

- `getLineAlmaFinitePoints(points)` -- drop points with a non-finite
  `x` or `value`.
- `normalizeLineAlmaPeriod(period, fallback)` -- positive integer;
  non-finite / sub-1 falls back, fractional floors.
- `computeLineAlmaWeights(period, offset, sigma)` -- the Gaussian
  window weights (`number[]`).
- `computeLineAlma(values, period, offset, sigma)` -- returns
  `{ weights, alma }`.
- `runLineAlma(points, options?)` -- sort finite points by `x`,
  compute the ALMA, return `{ series, period, offset, sigma,
  weights, alma, samples, almaFinal, almaMin, almaMax, aboveCount,
  belowCount, ok }`. Each sample carries an `above` / `below` / `on`
  position. `ok` is false for fewer than two points.
- `computeLineAlmaLayout(options)` -- fit one panel whose y-domain
  spans both the price and the ALMA. Returns the panel, ticks, the
  price path, the ALMA path, the price dots, and the ALMA markers.
- `describeLineAlmaChart(data, options?)` -- ARIA description.

## Component

`ChartLineAlma` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, a single panel with axis
ticks and grid, a config badge (period, offset and sigma), a
two-series legend (Price / ALMA) with toggle, the price line, the
ALMA line drawn over it, markers on the ALMA line, optional price
dots, a hover/focus tooltip exposing the value, ALMA, and position,
and a `motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs chart-line-hma** -- the HMA cancels lag by nesting three
  weighted moving averages (`2*WMA(half) - WMA(full)` then a
  sqrt-length WMA). The ALMA uses *one* window and instead reshapes
  the weight curve -- a Gaussian whose peak is slid toward the recent
  end by the offset.
- **vs chart-line-zlema** -- the ZLEMA shifts the price forward by
  its momentum and then EMA-smooths it. The ALMA never shifts the
  price; it tunes responsiveness purely through the offset of the
  Gaussian weights.
- **vs chart-line-ewma / chart-line-moving-avg** -- those weight a
  window exponentially or uniformly with the peak fixed at the most
  recent bar. The ALMA's peak is freely positioned, and the curve is
  Gaussian rather than exponential or flat.

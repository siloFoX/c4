## Summary

`web/src/components/ui/chart-line-gauss.tsx` ships `<ChartLineGauss>`,
a pure-SVG single-panel chart with an **Ehlers Gaussian Filter**
overlay from an N-pole cascaded low-pass smoother of selectable poles.
The price line and the filter line share one panel; the filter line is
segment-coloured by its own slope (up / down / flat).

## Computation

For a lookback `period` and a pole count `poles`, the smoothing
constant comes from the Ehlers Gaussian formula:

```
beta  = (1 - cos(2 * pi / period)) / (sqrt(2)^(2 / poles) - 1)
alpha = -beta + sqrt(beta^2 + 2 * beta)
```

The filter is then N stages of a one-pole EMA cascaded in series:

```
stage_k[i] = stage_k[i-1] + alpha * (input_k[i] - stage_k[i-1])
```

where the first stage's input is the price and each later stage's
input is the previous stage's output. Stages are seeded from the first
value so the filter is defined from bar 0.

The defining algebraic invariant: each one-pole stage has unity DC
gain (its coefficients sum to one), so the whole N-pole cascade passes
a constant series through unchanged. This holds for any period and any
pole count -- the exact anchor that survives every parameter choice.

## Worked anchors

- **CONST_FLAT** -- values `[5 x 10]`. For any period and any poles,
  every stage is seeded at 5 and `alpha * (5 - 5) = 0`; the cascade
  output is `[5, 5, ..., 5]`. Asserted exactly with `toEqual` across
  16 combinations (`period in {2, 4, 10, 20}` and
  `poles in {1, 2, 4, 8}`).
- **SINGLE_BAR** -- input `[7]`. The seed is 7; the single iteration
  preserves it. Output is `[7]` exact.
- **TWO_VALUE** -- input `[5, 10]` at period 4, poles 2. The seed is
  5; the first output is 5 exact. The second output flows through two
  stages: `stage_1 = 5 + alpha * (10 - 5) = 5 + 5 * alpha`; then
  `stage_2 = 5 + alpha * (stage_1 - 5) = 5 + 5 * alpha^2`. Asserted
  with `toBeCloseTo(5 + 5 * alpha * alpha, 10)`.
- **RISING** / **FALLING** -- the filter rises / falls strictly across
  a strictly rising / falling input.

## Distinct from neighbouring primitives

- vs `<ChartLineMovingAvg>` / a plain SMA -- the SMA is a single
  flat-weighted average; the Gaussian Filter is an N-stage CASCADE
  with alpha tuned so the cumulative frequency response approximates
  a Gaussian.
- vs `<ChartLineEma>` -- the EMA is the one-pole case (`poles = 1`)
  with a different alpha derivation; the Gaussian Filter generalises
  it to N poles with the Ehlers compensation: more poles means more
  lag from the cascade, so alpha is tuned UP to keep the cutoff near
  the same frequency.
- vs `<ChartLineCoral>` / Tillson T3 -- the Coral is a SIX-pole
  cascade recombined with NEGATIVE-weighted Tillson coefficients to
  inject lead and cut lag. The Gaussian Filter is the pure low-pass
  cousin: every coefficient is non-negative (each one-pole stage),
  so the response is smooth but lagging rather than lead-corrected.
- vs `<ChartLineBandPass>` / `<ChartLineHighpass>` -- those isolate a
  cycle band or a high-frequency residual; the Gaussian Filter keeps
  the low-frequency trend and rejects the cycle band.

## API

`data` -- a single `ChartLineGaussPoint[]` (`{x, value}`). `period`
(default 14, coerced to an integer >= 2; sub-2 falls back, fractional
floors). `poles` (default 4, coerced to an integer clamped to
`[1, 8]`). Standard sizing / colour / a11y props plus `strokeWidth`,
`dotRadius`, `tickCount`. Toggle flags: `showAxis`, `showGrid`,
`showDots` (default false), `showGauss`, `showMarkers`, `showTooltip`,
`showConfigBadge`, `showLegend`, `animate`. Controlled and
uncontrolled visibility via `hiddenSeries` (`'price' | 'gauss'`).
Callbacks: `onPointClick({point})`, `onSeriesToggle({seriesId, hidden})`.
Format functions: `formatValue`, `formatX`. ARIA: `ariaLabel`,
`ariaDescription`.

## Pure helpers

- `DEFAULT_CHART_LINE_GAUSS_*` -- defaults (period 14, poles 4, etc.).
- `getLineGaussFinitePoints` -- keep only points with a finite x and
  value.
- `normalizeLineGaussPeriod(period, fallback)` -- integer >= 2; else
  fallback.
- `normalizeLineGaussPoles(poles, fallback)` -- integer clamped to
  `[1, 8]`; else fallback.
- `computeLineGaussAlpha(period, poles)` -- the Ehlers Gaussian
  smoothing constant. Non-finite or out-of-range inputs fall back to
  defaults.
- `computeLineGauss(values, period, poles)` -- the N-pole cascaded
  EMA. Non-array input returns `[]`. Stages are seeded from the first
  value.
- `classifyLineGaussSlope(gauss, prevGauss)` --
  `'up' | 'down' | 'flat' | 'none'` by slope.
- `runLineGauss(points, options?)` --
  `{series, period, poles, alpha, gauss, samples, gaussFinal,
  upCount, downCount, flatCount, ok}`.
- `computeLineGaussLayout({data, period, poles, ...})` -- single
  panel with the price line, one segment between each pair of bars
  (slope-coloured), one marker per bar, and a value domain that
  spans both the price and the filter. Degenerate canvas or fewer
  than 2 points -> `ok = false`.
- `describeLineGaussChart(data, options?)` -- ARIA summary
  mentioning the indicator, the N-pole cascade, the period, the pole
  count and the slope counts.

## a11y / data hooks

Root `<div role="region">` with `aria-label` + `aria-describedby`; the
SVG is `role="img"` with the same label; an sr-only
`data-section="chart-line-gauss-aria-desc"` span carries the
description. The price line, every price dot and every marker carry
`role="graphics-symbol"` and `tabIndex={0}`. Every node carries a
`data-section`. Root mirrors `data-period`, `data-poles`,
`data-alpha`, `data-gauss-final`, `data-up-count`, `data-down-count`,
`data-flat-count`, `data-total-points`, `data-animate`, `data-empty`.
Segments and markers expose `data-slope`; markers also expose
`data-gauss`.

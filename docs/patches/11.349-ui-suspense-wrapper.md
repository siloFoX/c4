## Composite

```
<UIErrorBoundary
  fallback={errorFallback ?? <ErrorState ... />}
  resetKeys={resetKeys}
  onError={onError}
>
  <Suspense fallback={skeletonFallback}>
    {children}
  </Suspense>
</UIErrorBoundary>
```

Two existing primitives compose: `UIErrorBoundary`
(11.334) for runtime errors and `Suspense` for
lazy-loaded / data-fetching subtrees. The default
fallbacks reuse the existing design language:
`Skeleton` rows for the loading state,
`ErrorState` for the failure state.

## Public surface

```tsx
<SuspenseWrapper
  name="history detail"          // drives default labels
  skeletonRows={3}               // default fallback row count
  fallback={undefined}           // override default skeleton (pass null to disable)
  errorFallback={undefined}      // override default error UI
  onError={(err, info) => ...}   // telemetry hook
  resetKeys={[topView]}          // reset-on-route-change
  showAfterMs={120}              // anti-flash gate
  minDisplayMs={300}             // reserved (forward-compat)
  className="..."                // default skeleton container class
  data-testid="..."              // default skeleton container testid
>
  {children}
</SuspenseWrapper>
```

### Anti-flash gate

The default skeleton fallback paints nothing for
the first `showAfterMs` (default 120ms). After
that point the `Skeleton` rows render inside a
`role="status"` container with an aria-live
"polite" attribute and an aria-label that routes
through `name` (`Loading <name>` / `Loading`).

Sub-120ms suspense resolutions never paint a
skeleton -- aligned with the
`use-loading-skeleton` hook contract.

`minDisplayMs` is reserved -- the Suspense
primitive unmounts the fallback the moment a
child resolves. A future patch can plumb a
Promise-resolved delay through the gate without a
breaking-change to the public surface.

### Default error fallback

When the wrapped subtree throws,
`UIErrorBoundary` catches and renders the
canonical `ErrorState` with a "Could not load
<name>" title (or the default "Something went
wrong" when no name is set) and a Retry button
wired to the boundary's reset.

`errorFallback` accepts the same `ReactNode |
(error, retry) => ReactNode` shape as
UIErrorBoundary; pass a factory for retry
control.

### resetKeys

Forwarded to UIErrorBoundary. Pass route keys
(`[topView]`) so navigating away from a crashed
subtree recovers automatically.

## Out of scope

- **Per-page adoption** at History / Snapshots /
  Audit. The primitive ships standalone; the
  dispatch language "Standardize async boundary
  pattern across History / Snapshots / Audit"
  is best served by a follow-up per-page
  adoption patch so each migration can pair with
  the page's own loading-state refactor.
- **`minDisplayMs` Promise-resolved enforcement.**
  The gate prevents flash-of-skeleton; a real
  minimum-display ladder would require a
  Suspense-cache wrapper not yet in scope.
- **Playwright visual coverage.** Unit tests
  assert behaviour; a browser harness is a
  separate patch.
- **Telemetry-only fallback** (no UI). Use the
  existing `errorFallback={null}` pattern and
  pipe through `onError`.

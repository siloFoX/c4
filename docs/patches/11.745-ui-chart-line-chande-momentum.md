# Patch 11.745 -- UI: chart-line-chande-momentum primitive

Released as `v1.11.763` on 2026-05-26.

## Scope

Add `<ChartLineChandeMomentum>` -- a pure-SVG two-panel chart with
a Tushar Chande **Momentum Oscillator** panel reading the
imbalance of up-closes against down-closes across the lookback,
scaled to `[-100, +100]`.

## Definition

Inputs:

- `data: ChartLineChandeMomentumPoint[]` where
  `ChartLineChandeMomentumPoint = { x; close }`. Bars with
  non-finite `x` / `close` are dropped.
- `period: number` -- the lookback. Default `14`. Normalized to
  an integer `>= 2`.
- `threshold: number` -- the `+/-` overbought / oversold band.
  Default `50`. Normalized to a finite in `(0, 100]`.

Algorithm (per bar `i >= period`):

```
For j in [i - period + 1, i]:
  delta[j]    = close[j] - close[j - 1]
  upPart[j]   = delta[j] > 0 ?  delta[j] : 0
  downPart[j] = delta[j] < 0 ? -delta[j] : 0

upSum   = sum(upPart   over window)
downSum = sum(downPart over window)

CMO[i]  = 100 * (upSum - downSum) / (upSum + downSum)
```

The first `period` bars are null (we need `period + 1` closes to
fill the window of deltas). A zero net move (every delta zero ->
`upSum + downSum = 0`) also nulls the bar.

## Bit-exact algebraic anchors

**CONST_FLAT** (`close == K`): every delta is `K - K = 0`, so
`upSum + downSum = 0`. The bar is null at every position (no
scale to divide by).

**Monotone rising** (`close[j+1] > close[j]` integer ramp): every
delta is `+1`, so `downSum = 0`, `upSum = period`, and

```
CMO = 100 * (period - 0) / (period + 0) = 100
```

bit-exact at every defined bar.

**Monotone falling**: every delta is `-1`, so `upSum = 0`,
`downSum = period`, and `CMO = -100` bit-exact.

**Alternating** (`close == [10, 11, 10, 11, ...]` period 4):
deltas alternate `+1, -1, +1, -1`, so over any window of 4
consecutive deltas we have two of each. `upSum = downSum = 2`,
and `CMO = 100 * (2 - 2) / 4 = 0` bit-exact.

**Asymmetric anchor** (`close = [10, 11, 12, 13, 12]` period 4 at
bar 4): the window of deltas covers bars 1..4 = `[+1, +1, +1,
-1]`. `upSum = 3`, `downSum = 1`. `CMO = 100 * (3 - 1) / (3 + 1)
= 100 * 2 / 4 = 50` bit-exact.

**Translation invariance**: shifting every close by a constant
`k` leaves every `delta = close[j] - close[j-1]` unchanged, so
the CMO is unchanged bit-exact.

**Scale invariance**: multiplying every close by a positive
constant `s` scales every delta by `s`, but also scales every
`upPart` and `downPart` by `s`. The numerator and the denominator
both pick up the same factor, so the ratio is unchanged
bit-exact for any positive integer `s`.

**Antisymmetry**: reversing the closes (RISING vs FALLING) flips
every delta's sign, swapping `upSum` and `downSum`. Numerator
flips sign; denominator unchanged. So `CMO(RISING) =
-CMO(FALLING)` bit-exact.

## Component

`forwardRef<HTMLDivElement, ChartLineChandeMomentumProps>` with
displayName. Renders:

- A region-role outer `<div data-section="chart-line-chande-momentum">`.
- An `<svg role="img">` with an `aria-describedby` description.
- Top panel: close line, optional dots.
- Bottom panel: CMO line, zone-coloured markers (overbought /
  positive / flat / negative / oversold / none), zero line,
  dashed `+/- threshold` lines.
- Legend toggle for `price` / `cmo`, plus a `showCmo` prop and
  an `onPointClick` handler.
- ARS design-system tokens for colour and motion (`motion-safe:
  animate-fade-in`).

- CONST_FLAT identity: CMO null at every bar.
- Monotone rising / falling: CMO = +100 / -100 bit-exact.
- Alternating: CMO = 0 bit-exact.
- Asymmetric anchor: CMO[4] = 50 bit-exact.
- Translation invariance: shift `+1000` leaves CMO unchanged.
- Scale invariance: factor `*100` leaves CMO unchanged.
- Antisymmetry: `CMO(RISING) = -CMO(FALLING)` bit-exact.
- CMO bounded in `[-100, +100]` on the wave fixture.
- Layout / component / ARIA / data-section hooks all covered.

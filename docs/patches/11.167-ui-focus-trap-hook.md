# 11.167 -- UI: focus-trap reusable hook (v1.11.185)

## Goal

Extract the inline focus-trap logic duplicated between `Dialog` and `Popover`
into a single reusable hook so future modal-style primitives
(`DropdownMenu`, future `Drawer`, command palettes, etc.) can opt in without
re-implementing the same Tab / Shift+Tab / Escape / initial-focus /
return-focus machinery.

## Surface

`web/src/hooks/use-focus-trap.ts`

```ts
export interface UseFocusTrapOptions {
  active?: boolean;                                  // default true
  initialFocusRef?: React.RefObject<HTMLElement>;    // explicit initial focus
  restoreFocusOnUnmount?: boolean;                   // default true
  onEscape?: () => void;                             // optional Escape handler
}

export function useFocusTrap<T extends HTMLElement>(
  containerRef: React.RefObject<T>,
  options?: UseFocusTrapOptions,
): void;
```

Behaviour:

- On activation (`active` truthy + container mounted), captures
  `document.activeElement` as `previouslyFocused`.
- Initial focus priority: `initialFocusRef.current` ->
  first focusable inside container -> container itself.
- Tab / Shift+Tab wrap through the focusable descendants using the
  canonical selector
  `a[href], area[href], input:not([disabled]), select:not([disabled]),
  textarea:not([disabled]), button:not([disabled]), iframe,
  [tabindex]:not([tabindex="-1"]):not([disabled]),
  summary:not(:disabled), [contenteditable=true]:not([disabled])`.
  Empty container falls back to focusing the container.
- Escape invokes `onEscape()` when provided; the hook does not close
  anything itself.
- On deactivation / unmount, restores `previouslyFocused.focus()` when
  `restoreFocusOnUnmount` is true (default).
- SSR-safe: `typeof document === 'undefined'` short-circuits the effect;
  also no-ops when the container ref is null.

## Refactor

- `web/src/components/ui/dialog.tsx`: replaced three inline `useEffect`s
  (escape, initial focus, tab cycle) with a single `useFocusTrap` call
  bound to the card ref. `onEscape` is wired to the `onClose` prop.
- `web/src/components/ui/popover.tsx`: same swap on the panel ref.
  `onEscape` is `closeOnEsc ? () => setOpen(false) : undefined` so the
  prop gate is preserved. `restoreFocusOnUnmount: false` keeps the
  popover-specific rule of returning focus to the trigger; a local
  `prevOpenRef` effect handles that since the generic hook only knows
  how to restore the captured `previouslyFocused`, which is not always
  the trigger in controlled-open mode.

## Tests

`web/src/hooks/use-focus-trap.test.tsx` covers (10 cases):

1. focuses first focusable on mount when active
2. Tab from last wraps to first
3. Shift+Tab from first wraps to last
4. inactive (`active=false`) does not trap focus
5. `initialFocusRef` takes priority over the first focusable
6. Escape calls `onEscape`
7. restores focus to previously focused element on unmount
8. `restoreFocusOnUnmount=false` does not restore
9. empty container focuses container itself
10. SSR-safe smoke (hook does not throw with a never-bound ref)

`web/src/components/ui/dialog.test.tsx` and
`web/src/components/ui/popover.test.tsx` are unchanged -- the refactor
preserves their public observable behaviour.

## Out of scope

- Other UI primitives (`dropdown-menu`, `tabs`, `tooltip`, ...) -- they
  do not currently trap focus and the refactor is intentionally limited
  to the two components with literal duplication.
- Touch / pointer focus handling beyond what the existing primitives
  did.
- `docs/autonomous-queue-v10.md` (manager handles queue).
- `package-lock.json` (untouched).

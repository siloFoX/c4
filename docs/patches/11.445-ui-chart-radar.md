## Summary

Add a new `<ChartRadar>` UI primitive in
`web/src/components/ui/chart-radar.tsx` -- pure-SVG radar /
spider chart. Each axis is a labelled dimension; each
series draws a filled polygon connecting per-axis values.
The polar grid is rendered as N concentric polygons plus
one radial spoke per axis. Hovering a data point opens a
tooltip with the series, axis, and formatted value. Legend
renders to the right (default) or below the chart. Default
ARIA description auto-summarises every series across every
axis so screen readers can describe the dataset without
extra wiring.

## API

```ts
interface ChartRadarAxis {
  id: string;
  label: string;
  max?: number;        // per-axis max for normalisation
}

interface ChartRadarSeries {
  id: string;
  label: string;
  data: readonly number[];   // length === axes.length
  color?: string;
  fillOpacity?: number;
}

interface ChartRadarProps {
  axes: readonly ChartRadarAxis[];
  series: readonly ChartRadarSeries[];
  size?: number;                  // default 320
  padding?: number;               // default 48
  levels?: number;                // default 4
  maxValue?: number;
  showGrid?: boolean;             // default true
  showAxisLabels?: boolean;       // default true
  showLegend?: boolean;           // default true
  showTooltip?: boolean;          // default true
  showPoints?: boolean;           // default true
  animate?: boolean;              // default true
  className?: string;
  ariaLabel?: string;             // default 'Radar chart'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  onPointClick?: (args: {
    series: ChartRadarSeries;
    axis: ChartRadarAxis;
    value: number;
    seriesIndex: number;
    axisIndex: number;
  }) => void;
  legendPlacement?: 'right' | 'bottom';
  pointSize?: number;             // default 3
  strokeWidth?: number;           // default 1.5
}
```

## Behaviour

- Renders a `<div role="region">` wrapping a square `<svg>`
  sized `size x size` (320 x 320 by default). The SVG also
  carries `role="img"` + the same `aria-label` so screen
  readers find both the region and the chart-as-image.
- The first axis sits at 12 o'clock; subsequent axes go
  clockwise. Axis labels render just outside the chart
  radius with `textAnchor` chosen from the axis angle so
  they never overlap the polygon edge.
- Grid: `levels` concentric polygons (default 4) outline
  the polar grid; the outermost level draws darker so it
  reads as a chart border. One radial spoke per axis
  connects centre to chart edge.
- Each series fills its polygon with `series.color`
  (defaults from `getDefaultBarColor`) at
  `series.fillOpacity` (default 0.2) and strokes the outline
  at `strokeWidth` (default 1.5).
- When `showPoints=true` (default), each data point gets a
  filled circle with a white border for visibility against
  the polygon fill. Hover / focus enlarges the dot by 1 px
  and surfaces a tooltip with series + axis + value rows.
  `onPointClick` receives `{ series, axis, value, seriesIndex,
  axisIndex }` so adopters can dispatch drill-downs.
- Tooltip uses `role="tooltip"` and is positioned next to
  the hovered point.
- ARIA description auto-summary -- the sr-only span emits
  `"Radar with N axes. <series>: <axis> <value>, ..."` for
  every series. `ariaDescription` overrides it; empty axes
  or series produce `"No data"`.
- `legendPlacement="bottom"` stacks legend under the
  canvas; default keeps it on the right.

## Pure helpers (exported)

```ts
DEFAULT_CHART_RADAR_SIZE         = 320
DEFAULT_CHART_RADAR_PADDING      = 48
DEFAULT_CHART_RADAR_LEVELS       = 4
DEFAULT_CHART_RADAR_FILL_OPACITY = 0.2
DEFAULT_CHART_RADAR_POINT_SIZE   = 3
DEFAULT_CHART_RADAR_STROKE_WIDTH = 1.5

getRadarAxisAngle(axisIndex, total): number
getRadarPoint(value, max, angle, cx, cy, radius): { x; y }
getRadarMax(axes, series, override?): number
buildRadarGridLevelPoints(axisCount, ratio, cx, cy, radius): string
buildRadarSeriesPoints(series, axes, max, cx, cy, radius): string
findRadarPointHit(positions, sx, sy, maxDistance?):
  { seriesIndex; axisIndex; distance } | null
describeRadarChart(axes, series, formatValue?): string
```

- `getRadarAxisAngle` returns `-pi/2` for axis 0 and
  spaces the rest clockwise; total=0 short-circuits to 0
  to keep the helper composable.
- `getRadarPoint` clamps `value / max` into `[0, 1]` so
  non-finite, negative, or over-max inputs never escape
  the chart bounds; max <= 0 collapses to the centre.
- `getRadarMax` honours `maxOverride` first, then per-axis
  `max`, then the data max; falls back to 1 when nothing
  finite is found so empty datasets still render.
- `buildRadarGridLevelPoints` and `buildRadarSeriesPoints`
  share a fixed format (`x.xx,y.yy` tokens, space-joined)
  so SVG `<polygon points>` accepts them directly.
- `findRadarPointHit` is exported for downstream
  keyboard / marquee select (the component does not need
  it today because each point has its own pointer events).
- `describeRadarChart` produces the default ARIA summary
  and honours `formatValue`.

## ARIA + data attributes

- Root: `<div role="region">` + `aria-label`.
- SVG: `role="img"` + same `aria-label`.
- sr-only span: `data-section="chart-radar-aria-desc"`
  carries the auto-summary or `ariaDescription` override.
- Each point: `role="graphics-symbol"` + `tabIndex={0}` so
  keyboard focus opens the tooltip via `onFocus`.
- `data-section` on every node:
  - `chart-radar` (root)
  - `chart-radar-canvas` (wrapper)
  - `chart-radar-aria-desc`
  - `chart-radar-svg`
  - `chart-radar-grid-level` (one per grid ring)
  - `chart-radar-spoke` (one per axis)
  - `chart-radar-axis-label` (one per axis)
  - `chart-radar-series` (per-series group)
  - `chart-radar-polygon` (per-series fill)
  - `chart-radar-point` (per data point circle)
  - `chart-radar-tooltip` + `-series` / `-axis` / `-value`
  - `chart-radar-legend` + `-item` / `-swatch` / `-label`
- Root mirrors `data-axis-count`, `data-series-count`,
  `data-animate`. Series groups mirror `data-series-id`,
  `data-series-color`. Points mirror `data-series-id`,
  `data-axis-id`, `data-axis-index`, `data-hovered`.

## Tests (58 cases)

- `getRadarAxisAngle`: axis 0 at 12 o'clock; clockwise
  ordering; total=0 short-circuit.
- `getRadarPoint`: full value at edge; zero at centre;
  negatives clamp; over-max clamps; non-finite + max<=0
  collapse to centre.
- `getRadarMax`: override (positive / non-positive),
  per-axis max, data max, empty fallback to 1.
- `buildRadarGridLevelPoints`: zero axes empty; one point
  per axis; ratio clamping (lower + upper).
- `buildRadarSeriesPoints`: one point per axis, per-axis
  max wins, missing data slot defaults to 0.
- `findRadarPointHit`: no hit, closest hit, custom
  `maxDistance`.
- `describeRadarChart`: empty -> "No data"; summary;
  formatValue shapes numbers.
- Constants: defaults exported and positive.
- Component:
  - region + aria-label (default + custom)
  - one polygon per series
  - one point per axis per series with `showPoints=true`
  - `showPoints=false` suppression
  - grid levels + spokes rendered by default
  - custom `levels` count
  - `showGrid=false` suppresses both rings + spokes
  - axis labels render by default + `showAxisLabels=false`
    suppression
  - legend default-on + suppression + bottom placement
  - default palette + custom series colour
  - tooltip opens on hover, closes on mouse-leave
  - tooltip value formatting via `formatValue`
  - `showTooltip=false` suppression
  - `onPointClick` invoked with full payload
  - default cursor when `onPointClick` is missing
  - `data-hovered` mirrors hover state
  - role=graphics-symbol + aria-label per point
  - `animate` mirrored to `data-animate`
  - SVG mirrors width / height / viewBox from `size`
  - auto-generated ARIA description
  - `ariaDescription` override
  - empty series -> "No data" description

## Out of scope

- Multi-axis logarithmic scale (host normalises before
  feeding the chart).
- Polygon clipping / shading between series (host
  composes multiple radars side by side instead).
- Drag-to-reorder axes (host owns axis order).
- Animated polygon morphing on data change (snap-only for
  v1; `motion-safe:animate-fade-in` covers mount only).

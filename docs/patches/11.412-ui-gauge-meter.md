## Gauge

### API

```ts
interface GaugeThreshold {
  from: number;
  to: number;
  color?: string;
  label?: string;
}

interface GaugeProps {
  value: number;
  min?: number;           // default 0
  max?: number;           // default 100
  thresholds?: GaugeThreshold[];
  size?: number;          // px diameter; default 120
  thickness?: number;     // arc width; default 10
  showValue?: boolean;    // default true
  formatValue?: (value: number) => ReactNode;
  ariaLabel?: string;     // default 'Gauge'
  className?: string;
  trackColor?: string;
  progressColor?: string;
}
```

### Behaviour

- Sweep: **270 degrees** (start
  135deg, end 405deg = 45deg + 360
  full turn). The needle traces a
  U-shape with a 90-degree gap at
  the bottom -- the canonical
  "speedometer" silhouette.
- Track arc painted in
  `trackColor` (default 20%
  grey).
- Threshold band arcs layered
  beneath the progress arc at
  35% opacity. The progress arc
  picks the matched threshold
  colour (via
  `findThresholdColor`) and
  paints over the band, so the
  current value shows in the
  band colour at full opacity.
- Optional centred value label
  (default the rounded value as
  text; `formatValue` overrides
  to ReactNode).

### Pure helpers (exported)

```ts
export function clampGaugeValue(
  value: number, min: number, max: number,
): number;

export function valueToFraction(
  value: number, min: number, max: number,
): number;

export function findThresholdColor(
  value: number,
  thresholds: GaugeThreshold[] | undefined,
  fallback?: string,
): string | undefined;

export function describeArc(
  cx: number, cy: number, r: number,
  startAngleDeg: number, endAngleDeg: number,
): string;

export const GAUGE_START_ANGLE_DEG: number; // 135
export const GAUGE_END_ANGLE_DEG: number;   // 405
export const GAUGE_SWEEP_DEG: number;       // 270
```

`describeArc` generates an SVG
arc path with the correct
large-arc flag (1 for sweeps >
180 degrees, else 0).

### Edge cases

- `max <= min` (zero or
  inverted range) -> aria
  values stay sane by falling
  back to `[min, min + 100]`
  on the component.
- `NaN` value -> clamps to
  min.
- `+Infinity` -> clamps to
  max.
- `-Infinity` -> clamps to
  min.
- Threshold lookup uses
  inclusive
  `value >= from && value <= to`
  -- boundary values land on
  the lower band.

## Meter

### API

```ts
interface MeterSegment {
  from: number;
  to: number;
  color?: string;
  label?: string;
}

interface MeterProps {
  value: number;
  min?: number;             // default 0
  max?: number;             // default 100
  segments?: MeterSegment[];
  size?: 'sm' | 'md' | 'lg';     // default 'md'
  orientation?: 'horizontal' | 'vertical';   // default 'horizontal'
  showValue?: boolean;        // default false
  formatValue?: (value: number) => ReactNode;
  ariaLabel?: string;          // default 'Meter'
  className?: string;
  trackColor?: string;
  fillColor?: string;
}
```

### Behaviour

- Track `<div>` background uses a
  segmented `linear-gradient`
  when `segments` is supplied;
  each segment paints a hard
  stop at its declared colour
  so the bar reads as a row of
  coloured bands.
- Indicator `<div>` overlays the
  track. Horizontal:
  `width = (value/max) * 100%`,
  bottom-aligned at the left
  edge. Vertical:
  `height = ...`, bottom-aligned
  upward. Indicator colour
  matches the active segment's
  colour (via
  `findMeterSegment`); falls
  back to `fillColor`.
- Size variants:
  - sm: 1.5 (6px)
  - md (default): 2 (8px)
  - lg: 3 (12px)
- Orientation:
  - horizontal (default): bar
    grows left-to-right.
  - vertical: bar grows
    bottom-to-top. `aria-
    orientation="vertical"`.
- Optional value badge to the
  right (or below, for
  vertical) of the bar.

### Pure helpers (exported)

```ts
export function findMeterSegment(
  value: number,
  segments: MeterSegment[] | undefined,
): MeterSegment | null;

export function buildMeterStops(
  segments: MeterSegment[] | undefined,
  min: number,
  max: number,
  fallback: string,
): string;
```

`buildMeterStops` returns the
CSS gradient stops list (without
the wrapping
`linear-gradient(...)`) so
callers can compose into any
direction. Returns the
`fallback` colour string when
`segments` is empty or
`max <= min`.

Reuses `clampGaugeValue` +
`valueToFraction` from
`gauge.tsx`.

### Edge cases

- `max <= min` (zero or
  inverted range) -> falls
  back to `[min, min + 100]`.
- Clamping (NaN -> min,
  +Infinity -> max, -Infinity
  -> min) inherits from
  `clampGaugeValue`.

### ARIA + data attributes

Root (`<div role="progressbar">`,
`aria-label`):

- aria-orientation
- aria-valuemin / aria-valuemax
  / aria-valuenow /
  aria-valuetext
- `data-section="meter"`
- `data-orientation`
- `data-size`
- `data-value`
- `data-fraction`
- `data-segment` (label or
  colour of matched segment)

Sub-nodes:

- `data-section="meter-track"`
- `data-section="meter-indicator"`
  + `data-fill-color`
- `data-section="meter-value"`

## Pairs with existing primitives

- `<Progress>` -- linear task-
  progress bar (single colour
  + indeterminate state).
  Meter is the multi-segment
  status variant.
- `<StatCard>` (11.124 /
  11.406) -- common host for
  Gauge as a hero metric or
  Meter as the comparison bar
  below the value.
- `<SkeletonSet>` (11.402) --
  loading silhouette before
  Gauge / Meter values
  arrive.
- ThemeCustomizer (11.394) --
  both read token-based
  classes for fallback
  colours.

## Out of scope (deferred)

- Animated needle / fill
  transitions. Both primitives
  snap to the new value; CSS
  transitions belong in a
  follow-on motion layer.
- Multiple values on one
  meter / gauge (comparison
  fill). Single value for v1.
- Tick marks / labels along
  the arc or bar. Adopters
  layer their own.
- Custom sweep angles for
  Gauge (semicircle / full
  circle). 270-degree sweep
  is canonical for v1;
  configurable sweep is a
  follow-on.
- Vertical gauge variant.
  Gauge is always radial;
  Meter covers the vertical
  case.
- Touch / mouse drag to set
  the value. Both are
  read-only; use ColorPicker
  or Slider for input.

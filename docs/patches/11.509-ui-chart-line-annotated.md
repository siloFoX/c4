# TODO 11.509 - chart-line-annotated primitive

Status: SHIPPED v1.11.527 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartLineAnnotated>`, a pure-SVG multi-series line chart with
**vertical annotation lines + text labels at specific x values**.
The canonical "annotated event" pattern for monitoring dashboards:
metric over time, with vertical reference lines marking deploys,
incidents, releases, holidays, or any other moment-in-time event.
Each annotation owns its own colour, label position, and dash
pattern.

## API

```tsx
<ChartLineAnnotated
  series={[
    {
      id: 'rps',
      label: 'Requests/s',
      data: [
        { x: 0, y: 100 }, { x: 1, y: 110 }, { x: 2, y: 130 },
        { x: 3, y: 200 }, { x: 4, y: 220 },
      ],
    },
  ]}
  annotations={[
    { id: 'deploy', x: 2, label: 'deploy v2.3', color: '#16a34a' },
    { id: 'incident', x: 3.5, label: 'incident',
      color: '#dc2626', position: 'bottom' },
  ]}
/>
```

## Distinct from prior chart-* primitives

- `<ChartLineDiscontinuous>` (11.508) - null/undefined gap markers.
- `<ChartLineSegment>` (11.507) - per-segment threshold colour.
- `<ChartLineThreshold>` (11.503) - horizontal threshold rules + zone
  shading (chart-line-annotated paints vertical event rules with
  text labels instead).
- All other line variants focus on different dimensions.

## Annotation contract

### Annotation spec

`ChartLineAnnotatedAnnotation = { id, x, label, color?, position?,
dashArray? }`:

- `x` (required, finite): the x value where the annotation lives.
- `label` (required): text rendered above / below / inside the plot
  at that x.
- `color` (default `'#64748b'`): used for both the vertical line
  stroke and the label fill.
- `position` (default `'top'`): `'top'` puts the label above the
  plot, `'bottom'` puts it below, `'inline'` puts it just inside
  the top of the inner plot area.
- `dashArray` (default `'4 3'`): SVG dasharray for the vertical
  rule.

Annotations with non-string id or non-finite x are dropped silently
via `getLineAnnotatedFiniteAnnotations`.

### Layout

`computeLineAnnotatedLayout` calls `getLineAnnotatedBounds` which
expands the x range to include every finite annotation x so the
vertical lines always sit inside the plot. Each annotation records:

- `px` (pixel x position).
- `inRange` (true when annotation x is within the active x bounds;
  rendering skips out-of-range annotations).
- `labelX`, `labelY`, `labelAnchor` resolved via
  `resolveLineAnnotatedLabel(px, position, padding, innerHeight)`.

The canonical helper `resolveLineAnnotatedLabel`:

| `position` | labelY                                      | anchor |
| ---------- | ------------------------------------------- | ------ |
| `'top'`    | `max(0, padding - 6)`                       | middle |
| `'inline'` | `padding + 12`                              | middle |
| `'bottom'` | `padding + max(0, innerHeight) + 18`        | middle |

### Click + ARIA

- Each annotation line is `role="graphics-symbol"` with
  `tabIndex={0}` and `aria-label="Annotation <label> at x <x>"`.
- Click on the line fires `onAnnotationClick({annotation})` so
  adopters can react (open a release-notes drawer, scroll to the
  incident, etc.).
- Annotations and series live in independent svg `<g>` groups so the
  vertical rules render *on top of* the line stroke, giving the
  annotation visual priority at intersections.

## Interactivity

- Hover or focus on a dot dims other series to ~30% and opens a
  tooltip with series label + formatted x + formatted y.
- Per-dot `onPointClick({series, point})` callback.
- Per-annotation `onAnnotationClick({annotation})` callback.
- Legend toggles series visibility (controlled + uncontrolled).
- Mount fade via `motion-safe:animate-fade-in` (suppress via
  `animate=false`).

## Exported helpers

- `DEFAULT_CHART_LINE_ANNOTATED_*` numeric / string defaults.
- `DEFAULT_CHART_LINE_ANNOTATED_PALETTE` (10-colour palette).
- `getLineAnnotatedDefaultColor(index)` - palette wrap.
- `getLineAnnotatedFinitePoints(points)` - drops non-finite.
- `getLineAnnotatedFiniteAnnotations(annotations)` - drops malformed
  annotations; preserves input order.
- `getLineAnnotatedBounds(series, annotations, hidden?)` - per-axis
  bounds with annotation x expansion.
- `getLineAnnotatedTicks(min, max, count?)` - evenly-spaced ticks.
- `buildLineAnnotatedPath(points)` - `M x y L x y ...` line builder.
- `resolveLineAnnotatedLabel(px, position, padding, innerHeight)`
  - canonical label coordinate resolver.
- `computeLineAnnotatedLayout(input)` - per-series + per-annotation
  layout.
- `describeLineAnnotatedChart(series, annotations?, hidden?,
  formatValue?)` - ARIA summary listing annotations.

## ARIA

- Root `<div role="region">` with `aria-label` and
  `aria-describedby` -> sr-only
  `data-section="chart-line-annotated-aria-desc"` span.
- SVG `role="img"` with the same `aria-label`.
- Each line path is `role="graphics-symbol"` with `tabIndex={0}`.
- Each annotation line is `role="graphics-symbol"` with
  `tabIndex={0}` and a label-aware aria-label.
- Each dot is `role="graphics-symbol"` with `tabIndex={0}`.

## Data attributes

- Root: `data-section="chart-line-annotated"`,
  `data-series-count`, `data-visible-series-count`,
  `data-total-points`, `data-annotation-count`, `data-animate`.
- Series group + path + dot: standard pattern.
- Annotation group: `data-section="chart-line-annotated-annotation"`,
  `data-annotation-id`, `data-annotation-index`, `data-annotation-x`,
  `data-annotation-px`, `data-annotation-color`,
  `data-annotation-position`.
- Annotation line + label: standard pattern with
  `data-annotation-id`.
- Tooltip, legend, axes, grid: standard pattern.

## Out of scope

- Range annotations (start..end x band). v1 is point annotations
  only; adopters use `<ChartLineThreshold>` + custom zone or
  compose multiple annotations.
- Auto-staggering of labels when annotations are close together
  (v1 paints them all at the same y per position; adopters
  vary `position` manually).
- Annotation symbols (icons / shapes at the top of the line).
- Animated reveal at annotation x positions (snap; mount fade only).
- Log-scale axes - linear only.

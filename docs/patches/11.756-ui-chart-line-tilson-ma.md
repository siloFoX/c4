# Patch 11.756 -- UI: chart-line-tilson-ma primitive

Released as `v1.11.774` on 2026-05-26.

## Scope

Add `<ChartLineTilsonMa>` -- a pure-SVG single-panel chart with a
Tillson **T3** moving-average overlay built from six chained EMAs
and combined by volume-factor-weighted coefficients.

## Definition

Inputs:

- `data: ChartLineTilsonMaPoint[]` where
  `ChartLineTilsonMaPoint = { x; close }`. Bars with non-finite
  fields are dropped.
- `period: number` -- the EMA length. Default `8`. Normalized to
  an integer `>= 2`.
- `volumeFactor: number` -- `a` in the coefficient formulas.
  Default `0.7`. Normalized to a finite in `[0, 1]`.

Algorithm:

```
EMA1 = EMA(close, period)
EMA2 = EMA(EMA1, period)
EMA3 = EMA(EMA2, period)
EMA4 = EMA(EMA3, period)
EMA5 = EMA(EMA4, period)
EMA6 = EMA(EMA5, period)
c1 = -a^3
c2 =  3 * a^2 + 3 * a^3
c3 = -3 * a   - 6 * a^2 - 3 * a^3
c4 =  1 + 3 * a + 3 * a^2 + a^3
T3 = c1 * EMA6 + c2 * EMA5 + c3 * EMA4 + c4 * EMA3
```

EMAs seed at the first finite input (`EMA[0] = input[0]`) with
`alpha = 2 / (length + 1)`.

## Bit-exact algebraic anchors

**EMA-of-constant lemma**: `EMA[0] = c`, `EMA[i] = EMA[i-1] +
alpha * (c - EMA[i-1]) = EMA[i-1]`. So `EMA(c, _) = c` exactly
at every bar.

**Coefficient sum identity**: `c1 + c2 + c3 + c4 = -a^3 + 3*a^2
+ 3*a^3 - 3*a - 6*a^2 - 3*a^3 + 1 + 3*a + 3*a^2 + a^3 = 1`
(algebraic identity).

**CONST_FLAT with a = 0**: `c1 = c2 = c3 = 0`, `c4 = 1`. The T3
reduces to `EMA3`, and EMA-of-constant says `EMA3 = K`. So `T3 =
K` bit-exact at every bar.

**CONST_FLAT with a = 1**: integer coefficients `c1 = -1, c2 =
6, c3 = -12, c4 = 8` -- sum is `-1 + 6 - 12 + 8 = 1` in integer
arithmetic, which is exact in floating point. With `EMA3 = EMA4
= EMA5 = EMA6 = K`, `T3 = -K + 6K - 12K + 8K = K` bit-exact.

**Coefficient worked anchors**:

- `a = 0.5`: `a^2 = 0.25, a^3 = 0.125` -- all dyadic. `{c1, c2,
  c3, c4} = {-0.125, 1.125, -3.375, 3.375}` bit-exact.

**Translation invariance**: shifting every close by `+k` shifts
every EMA by `+k` (EMA is affine). So `T3(close + k) = c1*(EMA6
+ k) + c2*(EMA5 + k) + c3*(EMA4 + k) + c4*(EMA3 + k) = T3(close)
+ (c1 + c2 + c3 + c4) * k = T3(close) + k`. Bit-exact at integer
shifts modulo the standard floating-point sum rounding (tests
use `toBeCloseTo(_, 10)` for non-trivial volume factors).

## Acceptance

- CONST_FLAT identity (`a = 0`): T3 = EMA3 = K bit-exact.
- CONST_FLAT identity (`a = 1`): T3 = K bit-exact via integer
  coefficients summing to 1.
- Coefficient anchors: `a = 0` -> `{0, 0, 0, 1}`; `a = 1` ->
  `{-1, 6, -12, 8}`; `a = 0.5` -> `{-0.125, 1.125, -3.375,
  3.375}` bit-exact dyadic.
- EMA-of-constant lemma bit-exact.
- Default `a = 0.7`: T3 = K to 10 decimal places on CONST_FLAT.
- Translation invariance: `+1000` shifts T3 by 1000 to 10dp.
- Layout / component / ARIA / data-section hooks all covered.

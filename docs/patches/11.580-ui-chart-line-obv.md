## Summary

New `ChartLineObv` UI primitive at
`web/src/components/ui/chart-line-obv.tsx` -- a pure-SVG line chart
with an On-Balance Volume (OBV) cumulative sub-panel computed from
paired price and volume series. The price line occupies the top
panel; the bottom panel plots the OBV cumulative line with a dashed
zero baseline and direction-coloured markers. Both panels share one
x-axis.

## Indicator math

### Close direction

`computeLineObvDirections(prices)` returns the per-period close
direction. Index 0 has no prior close and reads `flat`; later indices
read `up` / `down` / `flat` from the sign of `price[i] - price[i-1]`.

### On-Balance Volume (Joe Granville)

`computeLineObv(prices, volumes)` is the running cumulative total of
volume. OBV starts at 0; each period adjusts the running total by the
period's volume according to the close direction:

```
obv[0] = 0
obv[i] = obv[i-1] + volume[i]   when price[i] > price[i-1]  (up)
obv[i] = obv[i-1] - volume[i]   when price[i] < price[i-1]  (down)
obv[i] = obv[i-1]               when price[i] = price[i-1]  (flat)
```

The OBV is defined at every index -- there is no warm-up window.

### Worked fixture

`prices = [10, 13, 11, 14, 12, 9]`,
`volumes = [100, 200, 150, 400, 250, 300]`:

```
obv[0] = 0
obv[1] = 0   + 200  (13 > 10 up)   = 200
obv[2] = 200 - 150  (11 < 13 down) = 50
obv[3] = 50  + 400  (14 > 11 up)   = 450
obv[4] = 450 - 250  (12 < 14 down) = 200
obv[5] = 200 - 300  (9 < 12 down)  = -100
obv         = [0, 200, 50, 450, 200, -100]
directions  = [flat, up, down, up, down, down]
obvFinal = -100   obvMin = -100   obvMax = 450
```

## API

- `getLineObvFinitePoints(points)` -- drop points with a non-finite
  `x`, `price`, or `volume`.
- `computeLineObvDirections(prices)` -- per-period close direction.
- `computeLineObv(prices, volumes)` -- cumulative OBV series.
- `runLineObv(points)` -- sort finite points by `x`, compute OBV +
  directions, return `{ series, obv, directions, samples, obvFinal,
  obvMin, obvMax, ok }`. `ok` is false for fewer than two points.
- `computeLineObvLayout(options)` -- stack a price panel above an OBV
  panel sharing one x-axis. The OBV panel y-range spans 0 through the
  OBV extent so the dashed zero baseline is always placed; each OBV
  marker carries its close direction. Returns panels, ticks, the price
  path, OBV path, OBV markers, and the projected zero baseline `y`.
- `describeLineObvChart(data, options?)` -- ARIA description string.

## Component

`ChartLineObv` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge (final + peak OBV), a two-series legend (Price / OBV) with
toggle, a dashed zero baseline in the OBV panel, OBV markers coloured
by close direction (up green / down red / flat grey), a hover tooltip
exposing the price, volume, OBV, and direction for the hovered period,
and a `motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs chart-line-vwap** -- VWAP is a volume-weighted average *price*:
  a price-domain line (`cumulative price*volume / cumulative volume`)
  overlaid on the price. chart-line-obv is a volume-domain momentum
  line -- it signs each period's volume by the price-change direction
  and accumulates, producing a running total in volume units shown in
  its own sub-panel.
- **vs chart-line-cumulative** -- the generic cumulative chart is a
  running sum of a single series. OBV is a coupled two-series
  accumulation: the volume series is signed by the price series'
  direction before being summed.
- **vs chart-line-macd / -rsi / -stochastic / -atr** -- those
  sub-panels plot momentum, range-position, or volatility statistics;
  chart-line-obv plots a cumulative signed-volume total.

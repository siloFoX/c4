## Summary

Adds the `ChartLineCmoMidCross` primitive
(`web/src/components/ui/chart-line-cmo-mid-cross.tsx`): a pure SVG
dual-panel chart that visualizes Chande Momentum Oscillator (CMO)
centerline-cross events through the 0 midline for trend confirmation.
The top panel renders the close with bullish (up) / bearish (down)
chevron overlays at every CMO midline cross; the bottom panel renders
the CMO line on a fixed -100..100 oscillator with the 0 reference band
and discrete cross markers.

Companion to `chart-line-cmo-zero-cross` (TODO 11.946): same closing
delta-based CMO pipeline, identical default threshold (0), differently
framed ARIA / description (centerline trend confirmation rather than
zero-line momentum baseline). Aligns with the broader mid-cross family
(`chart-line-rsi-mid-cross`, `chart-line-stoch-mid-cross`,
`chart-line-cci-mid-cross`, `chart-line-mfi-mid-cross`,
`chart-line-stoch-rsi-mid-cross`, `chart-line-williams-r-mid-cross`,
`chart-line-uo-mid-cross`) which each pin the oscillator-specific
centerline.

## Pipeline

CMO is the classic Chande Momentum Oscillator over a length window:

1. `delta[i] = close[i] - close[i-1]`
2. `gain[i] = max(delta, 0)`, `loss[i] = max(-delta, 0)`
3. `sumGain = sum(gain, length)`, `sumLoss = sum(loss, length)`
4. `cmo[i] = (sumGain - sumLoss) / (sumGain + sumLoss) * 100`
   - When `sumGain + sumLoss == 0` (no movement in the window), short
     circuit to `cmo[i] = 0`. The result is bit-exact +0, never -0
     (verified via `Object.is`).
5. Regime classifier (strict `>=`):
   - `cmo >= 0` -> bullish
   - `cmo <  0` -> bearish
6. Cross detector emits a `bullish` cross when
   `cmo[i-1] <= 0 && cmo[i] > 0`, and a `bearish` cross when
   `cmo[i-1] >= 0 && cmo[i] < 0`. Strict inequality on the current
   side ensures the boundary value 0 does not double-fire.

## Defaults

- `length = 14` (canonical CMO window)
- `threshold = 0` (midline / centerline)
- Warmup boundary: bar 14 under the default length, so under SMALL
  (length = 14) the first 14 samples are null and the remaining 16
  are populated.

## Bit-exact anchors

The tests pin the following determinism contracts:

- CONST `close = K` for all `K in {0, 1, 42, 100, 1234}` ->
  `cmo[i] === 0` literal (NOT `-0`) for every `i >= 14`. Regime
  classifies as bullish because 0 sits on the threshold (`cmo >=
  threshold`).
- LINEAR UP `close = i` -> `cmo[i] === 100` constant from i = 14
  onward (no losses, sumLoss = 0, sumGain = length).
- LINEAR DOWN `close = -i` -> `cmo[i] === -100` constant (no gains).
- Balanced alternation `close = i % 2 ? 11 : 10` -> `cmo[i] === 0`
  (sumGain == sumLoss).
- CONST K full-run counters under length = 14, 30 bars:
  `noneCount = 14`, `bullishCount = 16`, `bearishCount = 0`,
  `crosses.length = 0`.
- `pricePath` and `cmoPath` are byte-identical across repeated layout
  calls, with `.toFixed(2)` precision and `M ... L ...` command
  shapes. The cmo path uses a single `M` (no null-gap re-starts) for
  warmup data because the post-warmup samples are all finite.
- Cross detector boundary: strict inequality (`> T`, `< T`), so
  equality with the threshold does not fire.

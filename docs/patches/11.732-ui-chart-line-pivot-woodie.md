## Summary

`web/src/components/ui/chart-line-pivot-woodie.tsx` ships
`<ChartLinePivotWoodie>`, a pure-SVG single-panel chart with **Woodie
pivot** level overlays. Woodie's variant of the classic floor pivot
weights the close TWICE in the pivot point so the levels lean toward
the prior close. The close line shares one panel with five step-
function level lines (R2 / R1 / PP / S1 / S2); each defined bar
carries the five levels as bar-wide horizontal stub segments
projected from the prior bar.

## Computation

From the PRIOR bar's `(high, low, close)`:

```
range = high - low
PP    = (high + low + 2 * close) / 4
R1    = 2 * PP - low
S1    = 2 * PP - high
R2    = PP + range
S2    = PP - range
```

The first bar has no prior reference and is left null on every level.

## Worked anchors

The Woodie formula is integer-rational when the prior bar is integer.
The implementation has three exact-style anchors.

- **Constant-series anchor** -- prev `H = L = C = 5`. `range = 0`,
  `PP = (5 + 5 + 10) / 4 = 5`. Every R/S level is 5 bit-exact.
- **Worked anchor A** -- prev `H = 12, L = 8, C = 10`. `range = 4`,
  `PP = (12 + 8 + 20) / 4 = 40 / 4 = 10`. `R1 = 2*10 - 8 = 12`,
  `S1 = 2*10 - 12 = 8`, `R2 = 10 + 4 = 14`, `S2 = 10 - 4 = 6`.
  All bit-exact integer.
- **Worked anchor B** -- prev `H = 14, L = 6, C = 12`. `range = 8`,
  `PP = (14 + 6 + 24) / 4 = 44 / 4 = 11`. `R1 = 22 - 6 = 16`,
  `S1 = 22 - 14 = 8`, `R2 = 11 + 8 = 19`, `S2 = 11 - 8 = 3`. All
  bit-exact integer.

The strict order `R2 > R1 > PP > S1 > S2` is tested directly. Two
algebraic identities are bit-exact for integer fixtures:

- `R2 + S2 = 2 * PP` (the R2/S2 pair is symmetric around PP).
- `R1 + S1 = 4 * PP - H - L` (the R1/S1 identity from `2*PP - L`
  and `2*PP - H`).

Plus `R2 - PP = range` and `PP - S2 = range` (the R2/S2 range
identities).

## Distinct from neighbouring primitives

- vs `<ChartLinePivotFibonacci>` -- the Fibonacci pivot uses
  `pp = (H + L + C) / 3` (close weighted once) plus Fibonacci-ratio
  offsets of the range; the Woodie pivot doubles the close weight
  in PP so the levels track the close more closely.
- vs `<ChartLinePivotDemark>` -- the DeMark pivot's X seed is
  DIRECTION-DEPENDENT (bullish doubles H, bearish doubles L); the
  Woodie pivot is direction-AGNOSTIC, always doubling the close
  regardless of `C vs O`.
- vs the Classic floor pivot -- the Classic uses `pp = (H + L + C)
  / 3`; the Woodie variant uses `(H + L + 2C) / 4` so a strong
  close pulls the pivot harder.

## API

`data` -- a single `ChartLinePivotWoodiePoint[]`
(`{x, high, low, close}`). Standard sizing / colour / a11y props
plus `strokeWidth`, `dotRadius`, `tickCount`. Toggle flags:
`showAxis`, `showGrid`, `showDots` (default false), `showPp`,
`showLevel1`, `showLevel2`, `showMarkers`, `showTooltip`,
`showConfigBadge`, `showLegend`, `animate`. Controlled and
uncontrolled visibility via `hiddenSeries`
(`'price' | 'pp' | 'r1' | 's1' | 'r2' | 's2'`). Callbacks:
`onPointClick({point})`, `onSeriesToggle({seriesId, hidden})`.
Format functions: `formatValue`, `formatX`. ARIA: `ariaLabel`,
`ariaDescription`.

## Pure helpers

- `DEFAULT_CHART_LINE_PIVOT_WOODIE_*` -- defaults.
- `getLinePivotWoodieFinitePoints` -- keep only points with finite
  x / high / low / close and `high >= low`.
- `computeLinePivotWoodieLevels(prev)` -- the five levels for a bar
  given the prior; nulls when prev is null or non-finite.
- `computeLinePivotWoodie(bars)` -- per-bar levels; bar 0 is all
  nulls.
- `classifyLinePivotWoodieZone(close, levels)` -- `'above-r2' |
  'r1-to-r2' | 'pp-to-r1' | 's1-to-pp' | 's2-to-s1' | 'below-s2' |
  'none'`.
- `runLinePivotWoodie(points)` --
  `{series, levels, samples, ppFinal, aboveCount, belowCount,
  betweenCount, ok}`.
- `computeLinePivotWoodieLayout({data, ...})` -- single panel with
  the close line, five level stub segments per defined bar, one
  marker per defined-zone bar coloured by zone band.
- `describeLinePivotWoodieChart(data)` -- ARIA summary mentioning
  the close double-weight, the pivot formula, the warm-up first
  bar, and the count of bars above / below / between.

## a11y / data hooks

Root `<div role="region">` with `aria-label` + `aria-describedby`; the
SVG is `role="img"` with the same label; an sr-only
`data-section="chart-line-pivot-woodie-aria-desc"` span carries the
description. The close line, every price dot and every marker carry
`role="graphics-symbol"` and `tabIndex={0}`. Every node carries a
`data-section`. Root mirrors `data-pp-final`, `data-above-count`,
`data-below-count`, `data-between-count`, `data-total-points`,
`data-animate`, `data-empty`. Segments expose `data-series-id` and
`data-value`; markers expose `data-zone` and `data-close`.

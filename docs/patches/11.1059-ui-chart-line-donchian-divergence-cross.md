# 11.1059 -- UI: chart-line-donchian-divergence-cross primitive (v1.11.1077)

## Scope

Add a pure-SVG dual-panel
`ChartLineDonchianDivergenceCross` component that renders
the close in the top panel and the Donchian Channel
(rolling HH / LL envelope with geometric midline) in the
bottom panel, marking price-vs-channel direction
disagreement events as range-break-reversal warnings with
bias coloring derived from the Donchian midline slope at
the trigger bar.

Richard Donchian's 1950s Donchian Channel locates price
within the rolling `[HH, LL]` high-low range as a literal
range envelope. The midline is the geometric centre of
the channel. This primitive compares the midline slope
against price direction to surface range-break-reversal
warnings.

Unlike EMA / SMA midline siblings, the Donchian midline
is a *range* indicator -- it follows the HH/LL extremes,
not a smoothed average. This makes it especially sensitive
to volatility regime changes (new highs / new lows
immediately shift the midline) but more rigid in
congestion zones (when no new HH/LL prints, the midline
is exactly flat).

## Donchian divergence-cross pipeline

```
upperBand[i] = max(high[j], j in [i - period + 1, i])
lowerBand[i] = min(low[j],  j in [i - period + 1, i])
midline[i]   = (upperBand[i] + lowerBand[i]) / 2

priceUp     : close[i] > close[i-1]
priceDown   : close[i] < close[i-1]
midlineUp   : midline[i] > midline[i-1]
midlineDown : midline[i] < midline[i-1]

regime ->
  'aligned-bullish'   when priceUp   && midlineUp
  'aligned-bearish'   when priceDown && midlineDown
  'divergent-bullish' when priceDown && midlineUp
                       (selling pressure but Donchian range
                        still ratcheting up -- bullish
                        range-break-reversal warning)
  'divergent-bearish' when priceUp   && midlineDown
                       (rally but Donchian range
                        ratcheting down -- bearish
                        range-break-reversal warning)
  'none'              when either side flat or null

bullish (divergence trigger up) :
  prev regime !== 'divergent-bullish' &&
  cur regime === 'divergent-bullish'
bearish (divergence trigger down) :
  prev regime !== 'divergent-bearish' &&
  cur regime === 'divergent-bearish'

bias        : midline[i] vs midline[i-1] -> up / down /
              flat / none
```

Default `period = 20` -- canonical Donchian 4-week
breakout tuning.

Sibling to chart-line-keltner-divergence-cross v1.11.1072
(EMA + ATR envelope) and chart-line-bollinger-divergence-
cross v1.11.1073 (SMA + stdev envelope) -- three midline-
plus-volatility-bands variants with different envelope
semantics. Donchian uses *range extremes*, Keltner uses
*ATR*, Bollinger uses *stdev* -- three distinct volatility
measures bracketing the same family.

Warmup is `period = 20` for the default tuning: HH / LL
seed at i = period - 1 = 19, midline valid from i = 19,
direction detection needs the previous bar's midline, so
the first regime classification lands at i = period =
20.

## Bit-exact anchors

All anchors use HL input.

- **CONST band** `high = K + 1`, `low = K - 1`, `close
  = K`: HH = K + 1, LL = K - 1, midline = K (constant).
  Both price and midline are flat -> regime `none` for
  every valid bar. 0 crosses. Verified across K in
  {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close
  = i`: HH = i + 1 (latest). LL = i - 20 (oldest at j =
  i - 19: (i - 19) - 1 = i - 20). midline = ((i + 1) +
  (i - 20)) / 2 = (2i - 19) / 2 = i - 9.5 (slope +1).
  upperBand - lowerBand = 21 (channel width). priceUp +
  midlineUp -> aligned-bullish for every valid bar. 0
  crosses.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`,
  `close = -i`: HH = -i + 20 (oldest at j = i - 19:
  -(i - 19) + 1 = -i + 20). LL = -i - 1 (latest).
  midline = ((-i + 20) + (-i - 1)) / 2 = -i + 9.5
  (slope -1, mirror). priceDown + midlineDown ->
  aligned-bearish. 0 crosses.

Note that the Donchian midline on LINEAR UP at i - 9.5 is
*exactly* the same value as the Bollinger SMA midline at
i - 9.5 and the Keltner EMA midline at i - 9.5 (with
period=20 each) -- all three converge to the same
centroid on a LINEAR input. The differences appear with
volatility, where each midline has distinct dynamics.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 5-state regime classifier follows the divergence-cross
  family convention.
- Donchian's HH/LL midline is the most rigid of the
  three envelope-divergence variants: in congestion
  (no new HH/LL prints) the midline is exactly flat,
  producing many `none` regimes. By contrast Keltner
  (EMA-driven) and Bollinger (SMA-driven) midlines
  rarely sit exactly flat. This is a documented
  intrinsic behaviour.
- HL input boundary filter requires `high >= low` --
  malformed bars are dropped at the input boundary.
  Note that Donchian uses only HL (not HLC), but the
  primitive accepts HLC for family consistency.
- On steady LINEAR input the midline produces a constant
  slope and zero divergence crosses, which is correct
  canonical behaviour.
- Complementary to:
  - `chart-line-donchian-mid-cross-sig` (sibling in the
    mid-cross-sig family -- Donchian midline vs SMA
    signal)
  - `chart-line-keltner-divergence-cross` v1.11.1072 --
    EMA + ATR envelope variant
  - `chart-line-bollinger-divergence-cross` v1.11.1073
    -- SMA + stdev envelope variant
  Together these three primitives bracket the modern
  midline-plus-volatility-envelope family with three
  different volatility measures.

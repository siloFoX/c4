## Summary

Add `web/src/components/ui/chart-line-dpo-oversold-cross.tsx` --
a pure-SVG dual-panel React/TypeScript chart primitive that
renders the close in the top panel and the close-only DPO line
in the bottom panel, marking discrete bullish (DPO crosses up
through oversold threshold -1, detrended-momentum oversold exit)
/ bearish (DPO crosses down through oversold threshold -1,
oversold entry) DPO oversold-crossover events. Mirror of
`chart-line-dpo-overbought-cross` (11.993) with default
threshold flipped from +1 to -1.

## Math anchor (bit-exact)

```
shift     = floor(length / 2) + 1
sma[i]    = SMA(close, length)[i]
dpo[i]    = close[i - shift] - sma[i]
bullish   : prev dpo <= -1 && cur dpo >  -1
bearish   : prev dpo >= -1 && cur dpo <  -1
```

- **CONST close = K**: dpo = 0 constant. 0 >= -1 -> regime
  `bullish` (mirror flip of overbought-cross's `bearish` at the
  same dpo=0 reading). 0 crosses. Verified across K = 0..1234.
- **LINEAR UP close = i**: dpo = -1.5 constant. -1.5 < -1 ->
  regime `bearish` (in oversold zone). 0 crosses. LINEAR UP
  reads `oversold` on DPO because the look-back close sits 1.5
  units below the centered SMA -- canonical DPO behavior on
  monotonic uptrends after the cycle saturates.
- **LINEAR DOWN close = -i**: dpo = 1.5 constant. 1.5 >= -1 ->
  regime `bullish`. 0 crosses.

For 50 bars at defaults (length=20, shift=11, threshold=-1):
- dpo null for i = 0..18 (19 entries)
- post-warmup regime count = 31 (i = 19..49)

Strict inequality on `cur` (`cur > -1` for bullish, `cur < -1`
for bearish) means boundary equality (dpo == -1) never double-
fires.

Layout auto-fits oscMin / oscMax to include the threshold band
with 10% padding -- on CONST K=50 the panel expands DOWN to
include the threshold (oscMin=-1.1, oscMax=0.1); on LINEAR UP
the panel collapses around -1.5 (oscMin=-1.55, oscMax=-0.95).

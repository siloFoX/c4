# 11.551 -- web/src/components/ui/chart-line-autocorrelation.tsx

Pure-SVG line chart with an **autocorrelation function
side panel**. The main panel renders the input time series;
the side panel shows the autocorrelation function (ACF) at
each lag from `0` to `maxLag`, with a 95% confidence band
under the white-noise null hypothesis. The lag with the
highest `|ACF|` (for `lag >= 1`) is highlighted as the
*dominant* lag.

## Math

The biased autocorrelation estimator (the convention used
in R's `acf()`, numpy's `np.correlate`, and statsmodels):

```
ybar = mean(y)

           sum_{i=0}^{N-1-k} (y_i - ybar) * (y_{i+k} - ybar)
ACF(k) = ------------------------------------------------------
           sum_{i=0}^{N-1}    (y_i - ybar)^2
```

Properties:
- `ACF(0) = 1` always (the numerator equals the denominator
  at zero lag).
- `ACF(k)` is in `[-1, 1]` under the biased estimator.
- A *positive* ACF at lag `k` means the series tends to look
  similar to itself shifted by `k`; a *negative* ACF means
  it looks like the mirror.
- For a sine wave with period `P`, the ACF approximates
  `cos(2*pi*k/P) * (N - k) / N`, so peaks (with alternating
  sign) appear at multiples of `P/2`.

### Confidence bound

Under the white-noise null hypothesis, the ACF at any lag
`k > 0` is approximately normally distributed with standard
deviation `1/sqrt(N)`. The 95% confidence bound is then:

```
bound = z / sqrt(N)    where z = 1.96 for 95% CI
```

Lags exceeding `|ACF(k)| > bound` are considered
statistically significant; this is the standard rule-of-thumb
"the spike pokes out of the dashed band" test you see in R's
`acf()` plot and any time-series textbook.

## Distinct from related primitives

- `<ChartLineCorrelation>` (Pearson r between TWO series at
  a single point). Autocorrelation is a series correlated
  with **itself** at many lags.
- `<ChartLineFft>` (11.540) and `<ChartLineFftWindow>`
  (11.544): frequency-domain spectrum. The ACF is the
  inverse Fourier transform of the power spectrum
  (Wiener-Khinchin theorem) but presented in *lag domain*,
  which is more interpretable for "what is the most likely
  period?"
- `<ChartLineSpectrogram>` (11.545): STFT heatmap of
  frequency over time. Autocorrelation collapses across
  time and looks at lag structure.
- `<ChartLineDecompose>` (11.543): seasonal split into
  trend + seasonal + residual. Autocorrelation is the input
  diagnostic you would use to *find* the seasonal period
  that decompose then uses.
- `<ChartLineChangepoint>` (11.546): regime shift detector.
  Different question (when did the variance change?).
- `<ChartLineLoess>` (11.549) / `<ChartLineSavgol>`
  (11.547) / `<ChartLineEwma>` (11.539) / `<ChartLineKalman>`
  (11.538) / `<ChartLineHampel>` (11.548): smoothing /
  filtering primitives. Autocorrelation is a *diagnostic*
  used to decide if smoothing is warranted in the first
  place (high ACF at lag 1 = autocorrelated noise to filter).

The canonical Box-Jenkins ACF plot used to identify
autoregressive (AR) order, moving-average (MA) order,
seasonal periods, and to test residuals after model fitting.

## Layout

Two side-by-side SVG panels:
- **Time panel** (left, ~55% width): raw time series as a
  line with optional dots.
- **ACF panel** (right, ~45% width): one stick + dot per
  lag from `0` to `maxLag`. Sticks anchor at `ACF = 0` and
  extend to the lag value. Confidence band drawn as a
  translucent rectangle plus two dashed reference lines at
  `±bound`. The dominant lag has a dashed vertical marker
  through the panel.

## Colour rules

- Lag is **positive-significant** (`ACF(k) > bound`) ->
  green (`#16a34a`).
- Lag is **negative-significant** (`ACF(k) < -bound`) ->
  red (`#dc2626`).
- Lag is **insignificant** (`|ACF(k)| <= bound`) -> slate
  (`#94a3b8`).
- Lag is **dominant** (largest `|ACF|` for `lag >= 1`) ->
  violet (`#7c3aed`) overrides the significance colour.

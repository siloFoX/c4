# Patch 11.924 -- UI: chart-line-stoch-oversold-cross primitive

## Summary

Added `<ChartLineStochOversoldCross />` -- pure-SVG dual-panel
React/TS primitive that renders the close (top panel) with
bullish (exit upward) / bearish (entry downward) arrow overlays
at every Stochastic %K threshold cross trigger, and the close-
only %K line (bottom panel) on a fixed 0-100 oscillator with
20 / 50 reference bands. Single-threshold cross variant of the
Stochastic family that flags the discrete %K level-20 entry
and exit events distinct from the canonical %K / %D crossover.
Mirror of 11.923 `chart-line-stoch-overbought-cross` using the
oversold level instead of the overbought level.

## Algorithm

```
range[i] = max(close[i-n+1..i]) - min(close[i-n+1..i])
rawK[i]  = range > 0
            ? ((close - lowestClose) / range) * 100
            : 50
k[i]     = SMA(rawK, kSmoothing)
bullish : prev <= threshold && cur > threshold  (exit upward)
bearish : prev >= threshold && cur < threshold  (entry downward)
```

Defaults `length = 14` (canonical Stochastic window),
`kSmoothing = 3` (slow %K smoothing), `threshold = 20`
(oversold level). Regime classifier `bearish` (k < threshold,
oversold), `neutral` (k >= threshold), `none` (k null).

## Bit-exact anchor

CONST close = K: range = 0 every bar -> rawK = 50 neutral
fallback. SMA of 50s = 50 via the `min === max` short-circuit.
50 > 20, so regime is `neutral` and the threshold is never
crossed. cross count = 0. Verified across K = 0, 1, 5, 17,
100, 1234.

LINEAR DOWN yields %K = 0 (close at bottom of range) -> 0 <
20 -> regime `bearish`. LINEAR UP yields %K = 100 -> regime
`neutral`.

  crosses.

# 11.320 -- UI: Queue page redesign

- TODO id: 11.320
- Version: 1.11.338
- Date: 2026-05-18
- Scope: `web/src/pages/Queue.tsx`, `web/src/pages/Queue.test.tsx`

## Goal

Bring the autonomous queue editor up to the same UX bar as
the recent page hero set (Snapshots, Templates, Workers).
Specifically:

- Status filter Tabs strip with live count chips.
- Debounced SearchBar narrowing rows by id / title / detail.
- HTML5 drag-and-drop drop-target indicator so the operator
  can see exactly where a dragged row will land.
- Success-path Toast on `POST /api/autonomous/queue` so the
  optimistic mutation gets explicit confirmation.

Component-scope only. No daemon endpoint change, no CLI
surface change.

## Behaviour

### Tabs

Tabs strip renders five options: `All`, `todo`, `doing`,
`done`, `partial`. The active option restricts the visible
rows to that status (or all rows under `All`). Each option
label carries a count chip with the per-status total.

Tab semantics follow the existing `Tabs` primitive: arrow
keys cycle the active tab, Enter / click activates it, and
`aria-label="Filter queue by status"` is announced to AT.

### SearchBar

Below or beside the Tabs, a `SearchBar` (`size="sm"`,
`debounceMs=200`) filters rows by case-insensitive
substring across `id`, `title`, and `detail`. Matches
compose with the active tab: with the `doing` tab and the
search query `"cap"`, only the doing rows whose
id / title / detail contain `cap` survive.

### Drop-target indicator

`onDragOver(event, row.id)` sets `dropTargetId` to the
hovered row's id (only when `dragId && dragId !== targetId`
to avoid self-highlight). The row renders
`data-drop-target="true"` plus the Tailwind class
`border-t-2 border-primary` so the user sees a visible
highlight above the landing slot.

`onDragLeave` clears the indicator only when the cursor
actually leaves the row (relatedTarget contains check).
This avoids flicker as the cursor crosses inner `<td>`
boundaries. `onDrop` and `onDragEnd` also clear the
indicator alongside the existing `dragId` reset.

### Success toast

`commit(nextRows)` now calls
`showToast('Queue saved.', 'success')` on the resolved
path. Failure paths still surface the message through the
inline `Alert` (`saveError`) for higher visibility -- the
toast covers the success path only.

A new Toast slot at the bottom of the page renders the
toast in a fixed-position container, matching the pattern
used by Snapshots / Profiles / Templates.

## Data attributes

| attr | element | purpose |
| --- | --- | --- |
| `data-section="queue-toolbar"` | toolbar wrapper | e2e selector for the Tabs + SearchBar block |
| `data-testid="queue-status-tabs"` | Tabs wrapper | direct query for the strip |
| `data-testid="queue-search"` | SearchBar input | direct query for the input |
| `data-testid="queue-tab-count-{key}"` | per-tab count chip | per-status total query |
| `data-testid="queue-empty-filter"` | placeholder row | filter-miss assertion |
| `data-drop-target="true"` | `<tr>` under cursor | drop-target indicator query |

## Deferred

ConfirmDialog for delete -- the queue page does not expose
a delete action and the daemon has no
`DELETE /api/autonomous/queue` endpoint. Folding in a
destructive confirm dialog without a delete affordance to
wire it to would be dead UI. Will revisit once a delete
flow lands.

## Tests

`web/src/pages/Queue.test.tsx` adds eight cases on top of
the existing thirteen:

1. `filters table rows by status when a status tab is clicked`
2. `renders per-status count chips on the Tabs strip`
3. `filters rows by debounced search across id/title/detail`
4. `composes the status tab and search filter`
5. `shows the "no rows match" placeholder when the filter eliminates everything`
6. `marks the row under the cursor with data-drop-target during a drag`
7. `shows a success toast after a successful save`
8. `does not show a success toast when the save fails`

Cases 3 and 4 use `vi.useFakeTimers({ shouldAdvanceTime: true })`
with `userEvent.setup({ advanceTimers: ... })` so the 200ms
debounce window is stepped past without sleeping.

All 21 cases pass against vitest 4.1.5 + jsdom 29.1.1.

## Adoption sites

The Queue page is the only adopter -- no other page
embeds the autonomous queue editor. The new primitives
(SearchBar with `debounceMs`, Tabs with count-chip labels,
data-drop-target rows) come from the existing ui barrel
and were already adopted by Snapshots / Workers / Sessions
in earlier patches.

# 11.528 -- web/src/components/ui/chart-line-acceleration.tsx

Pure-SVG multi-series line chart that **plots the discrete second
derivative** (acceleration / "rate of rate change") of an input
series. For every triple of consecutive finite samples sorted by x
ascending the chart computes:

```
v_left  = (y_b - y_a) / (x_b - x_a)
v_right = (y_c - y_b) / (x_c - x_b)
accel   = (v_right - v_left) / ((x_c - x_a) / 2)
```

(the standard non-uniform finite-difference second derivative
anchored at the middle index). The result is rendered as a line
against a horizontal zero-acceleration baseline; sign-shaded regions
paint the area between the acceleration line and zero with distinct
**concavity** colours (orange for positive / concave up, teal for
negative / concave down). Segments that cross zero are split exactly
at the inflection point.

## Distinct from other rate / transform primitives

- `<ChartLineRate>` (11.521): **first** derivative (rate). This is
  the second derivative.
- `<ChartLineVelocity>` (11.527): decorates the original line with
  directional arrows showing velocity direction. Acceleration plots
  a transformed time series.
- `<ChartLineCumulative>` (11.520), `<ChartLineMovingAvg>` (11.513),
  `<ChartLineTrend>` (11.512): different transforms entirely.
- `<ChartLineMinMax>` (11.517), `<ChartLineAnomaly>` (11.524):
  point-level highlights vs whole-series transform.

The canonical "is the growth itself accelerating or decelerating?"
visualisation — inflection-point detection, second-order momentum
diagnostics, "where does the curve change concavity".

## Public surface

- `<ChartLineAcceleration series={...} mode="center" />`.
- Per-series `{id, label, data, color?, mode?}`. `data` entries are
  raw `{x, y}`. Per-series `mode` overrides the chart-level default.
- Three anchor modes:
  - `'center'` (default): anchor at the middle x of each triple.
  - `'left'`: anchor at the leftmost x.
  - `'right'`: anchor at the rightmost x.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showSignFill`, `showZeroLine`, `animate`.
- `positiveColor` (default amber `#f59e0b`, "accelerating") /
  `negativeColor` (default cyan `#0891b2`, "decelerating") /
  `zeroColor` for the baseline.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX` / `formatAcceleration` (sign-aware
  fallback).
- Controlled + uncontrolled `hiddenSeries`.
- Callbacks: `onSampleClick({series, sample})`,
  `onRegionClick({series, region})`.

## Pure helpers

- `getLineAccelerationDefaultColor(index)` -- 10-color palette
  cycle.
- `getLineAccelerationFinitePoints(points)` -- finite-only filter.
- `computeAcceleration(points, mode?)` -- canonical second-
  derivative helper. Returns `(n - 2)` samples for `n` finite
  inputs sorted by x ascending. Drops triples where any segment
  `dx === 0`. Output's `index` is position in the OUTPUT array;
  `leftIndex` / `midIndex` / `rightIndex` reference the ORIGINAL
  input array. Less than 3 finite samples / non-array -> `[]`.
- `findLineAccelerationZeroCrossing(x1, a1, x2, a2)` -- crossing x
  via absolute-value proportion; returns `null` on same-sign /
  endpoint-on-zero / degenerate / non-finite.
- `computeLineAccelerationStats(samples)` -- returns
  `{finiteCount, maxAcceleration, minAcceleration,
  averageAcceleration, positiveCount, negativeCount, zeroCount,
  totalAbsoluteArea}`. Empty -> all zeros.
- `computeLineAccelerationLayout({series, mode?, hiddenSeries?,
  showSignFill?, positiveColor?, negativeColor?, ...})` -- returns
  `{series, xTicks, yTicks, xMin, xMax, yMin, yMax, zeroY,
  innerWidth, innerHeight, totalSamples, visibleSeriesCount}`.
  Sign-fill regions are closed `M ... L ... L ... Z` polygons
  between the acceleration line and the y=0 baseline. Y range is
  always expanded to include `0`.
- `describeLineAccelerationChart(series, mode?, hidden?,
  formatValue?)` -- builds the screen-reader description.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Acceleration line + dots are `role="graphics-symbol"` +
  `tabIndex={0}`; zero baseline is `role="graphics-symbol"`. Dot
  aria-label includes acceleration value + concavity tag.
- Every node carries `data-section="chart-line-acceleration-..."`
  for test selectors.
- Series groups expose `data-series-max-acceleration`,
  `data-series-min-acceleration`,
  `data-series-average-acceleration`,
  `data-series-positive-count`, `data-series-negative-count`,
  `data-series-zero-count`, `data-series-mode`,
  `data-series-color`, `data-series-finite-count`.
- Dots expose `data-acceleration-sign` (`'positive' | 'negative' |
  'zero'`), `data-left-index`, `data-mid-index`,
  `data-right-index`.
- Regions expose `data-region-is-positive`, `data-region-start-x`,
  `data-region-end-x`, `data-region-fill-color`, `data-region-area`.
- Root mirrors `data-mode`.

## Tooltip

Hovering a dot shows the series label, x, a coloured
`acceleration:` row (tinted by sign) with a `(concave up)` or
`(concave down)` suffix, and a `mode: <kind> (from #i - #j - #k)`
row indicating the three original-array indices that produced the
sample.

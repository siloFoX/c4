# 11.1057 -- UI: chart-line-adx-pos-neg-divergence primitive (v1.11.1075)

## Scope

Add a pure-SVG dual-panel `ChartLineAdxPosNegDivergence`
component that renders the close in the top panel and the
Average Directional Index +DI and -DI directional
indicators in the bottom panel, marking +DI-vs--DI
direction disagreement events as trend strength divergence
triggers.

Unlike the price-vs-indicator divergence-cross family, this
primitive compares the **two directional indicators
against each other**: the +DI direction (bullish trend
strength) against the -DI direction (bearish trend
strength). When the two diverge in direction, the trend-
strength balance is shifting -- one side is strengthening
while the other weakens. This is the canonical Wilder
trend-confirmation signal.

## ADX +DI / -DI divergence pipeline

```
upMove[i]    = high[i] - high[i-1]
downMove[i]  = low[i-1] - low[i]
+DM[i]       = (upMove > downMove && upMove > 0)
               ? upMove : 0
-DM[i]       = (downMove > upMove && downMove > 0)
               ? downMove : 0
tr[i]        = max(high - low,
                   |high - close[i-1]|,
                   |low  - close[i-1]|)
smPlus[i]    = SMA(+DM, period)
smMinus[i]   = SMA(-DM, period)
smTr[i]      = SMA(tr,  period)
plusDI[i]    = (smTr > 0) ? 100 * smPlus / smTr : 0
minusDI[i]   = (smTr > 0) ? 100 * smMinus / smTr : 0

plusUp      : plusDI[i]  > plusDI[i-1]
plusDown    : plusDI[i]  < plusDI[i-1]
minusUp     : minusDI[i] > minusDI[i-1]
minusDown   : minusDI[i] < minusDI[i-1]

regime ->
  'aligned-bullish'   when plusUp   && minusUp
                       (both strengthening -- indecisive
                        market with both sides active)
  'aligned-bearish'   when plusDown && minusDown
                       (both weakening -- range collapse /
                        consolidation)
  'divergent-bullish' when plusUp   && minusDown
                       (bullish strength rising while
                        bearish falls -- canonical bull
                        confirmation)
  'divergent-bearish' when plusDown && minusUp
                       (bullish strength falling while
                        bearish rises -- canonical bear
                        confirmation)
  'none'              when either side flat or null

bullish (divergence trigger up) :
  prev regime !== 'divergent-bullish' &&
  cur regime === 'divergent-bullish'
bearish (divergence trigger down) :
  prev regime !== 'divergent-bearish' &&
  cur regime === 'divergent-bearish'

bias        : (plusDI - minusDI) at trigger bar vs prev ->
              up / down / flat / none
```

Default `period = 14` -- canonical Wilder ADX tuning.

Uses **SMA-based smoothing** (matching the
chart-line-adx-trend-cross v1.11.1047 and adx-* family
convention) rather than Wilder's exponential smoothing,
for bit-exact integer DI values on linear input. The two
ADX smoothing variants in the codebase are a documented
family-level tuning choice.

Zero-guard on `smTr > 0` prevents 0/0 NaN when ATR is
zero (e.g. CONST band where every TR is the same constant
2 but DM is 0; the guard returns DI = 0).

Warmup is `period = 14` for the default tuning: DM and TR
seed at i = 1, SMA window-fill at i = period = 14, and
direction detection needs the previous bar's +DI/-DI, so
the first regime classification lands at i = period + 1 =
15.

## Bit-exact anchors

All anchors use HLC input.

- **CONST band** `high = K + 1`, `low = K - 1`, `close
  = K`: upMove = 0, downMove = 0 -> +DM = -DM = 0
  (`upMove > 0` fails). TR = 2. smPlus = smMinus = 0,
  smTr = 2. plusDI = 100 * 0 / 2 = 0, minusDI = 0.
  Both flat -> regime `none` for every valid bar. 0
  crosses. Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close
  = i`: upMove = +1, downMove = -1. upMove > downMove
  AND upMove > 0 -> +DM = 1. -DM = 0 (downMove fails
  `> 0` check). TR = 2. smPlus = 1, smMinus = 0, smTr =
  2. plusDI = 100 * 1 / 2 = 50, minusDI = 0
  (constants). Both flat -> regime `none`. 0 crosses.
  The +DI = 50 reflects the canonical Wilder reading on
  a steady linear uptrend: bullish directional movement
  is exactly half the true range.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`,
  `close = -i`: mirror -> upMove = -1, downMove = +1.
  +DM = 0, -DM = 1. plusDI = 0, minusDI = 50
  (constants). Both flat -> regime `none`. 0 crosses.

All three anchors produce zero divergence crosses -- the
regime stays in `none` because both +DI and -DI directions
are flat. Divergence crosses only appear when one of the
indicators changes direction while the other does not,
which requires curvature in the price series.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 5-state regime classifier follows the divergence-cross
  family convention.
- **Distinct semantics from price-vs-indicator divergence
  variants**: this primitive compares the two directional
  indicators against each other rather than against
  price. The "divergent-bullish" regime here means +DI
  rising + -DI falling (bull confirmation), not
  priceDown + indicatorUp as in the price-vs-indicator
  siblings. This is documented in the regime
  classifier and aria description.
- SMA-based smoothing (not Wilder exponential) for bit-
  exact testability. Different from canonical Wilder
  formulation -- documented as a family-level tuning
  choice.
- Zero-guard on `smTr > 0` prevents NaN on CONST input
  -- returns 0 for both DI values. Matches pandas-ta
  and ta-lib behaviour.
- On steady LINEAR input both +DI and -DI saturate at
  constants (50/0 for LINEAR UP, 0/50 for LINEAR DOWN)
  with flat slopes -- no divergence crosses fire. This
  is correct canonical behaviour: trend-strength
  divergence requires the *balance* of bullish vs
  bearish pressure to shift, which only happens on
  curvature in the price series.
- Complementary to chart-line-adx-trend-cross
  v1.11.1047 (which compares ADX itself against the
  20/25 thresholds for trend emerging confirmation) and
  the divergence-cross family (which compares price vs
  individual indicator direction). This primitive
  occupies a different signal niche: it watches the
  *internal* balance between the two directional
  indicators.

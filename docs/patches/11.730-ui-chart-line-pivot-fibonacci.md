## Summary

`web/src/components/ui/chart-line-pivot-fibonacci.tsx` ships
`<ChartLinePivotFibonacci>`, a pure-SVG single-panel chart with
**Fibonacci pivot level** overlays at 0.382 / 0.618 / 1.0 of the
prior bar range. The close line shares one panel with seven step-
function level lines (R3 / R2 / R1 / PP / S1 / S2 / S3); each defined
bar carries the seven levels as bar-wide horizontal stub segments,
projected from the PRIOR bar's `(high, low, close)`.

## Computation

Each bar carries a pivot point and three pairs of resistance /
support levels offset by Fibonacci ratios of the PRIOR bar's range:

```
range = prevHigh - prevLow
pp    = (prevHigh + prevLow + prevClose) / 3

R1 = pp + level1 * range     S1 = pp - level1 * range
R2 = pp + level2 * range     S2 = pp - level2 * range
R3 = pp + level3 * range     S3 = pp - level3 * range
```

Default ratios: `level1 = 0.382`, `level2 = 0.618`, `level3 = 1.0`.

The first bar has no prior and is left null on every level.

## Worked anchors

The pivot formula is integer-rational when the prior bar is integer.
The implementation has three exact-style anchors.

- **Constant-series anchor** -- prev `H = L = C = 5`. `range = 0`,
  `pp = 5`. Every R/S level is `5 + ratio * 0 = 5` (multiplication
  by 0 is bit-exact in IEEE 754). Every level reads exactly 5.
- **Worked integer anchor** -- prev `H = 12, L = 6, C = 9`.
  `range = 6` exact, `pp = 27 / 3 = 9` exact. At the 1.0 ratio:
  `R3 = 9 + 6 = 15`, `S3 = 9 - 6 = 3`, both bit-exact (integer
  add / subtract). At 0.382 and 0.618 the products are
  transcendental in binary so the levels are asserted with
  `toBeCloseTo(_, 12)`.
- **Mirror identity** -- `Sk = 2 * PP - Rk` for k = 1, 2, 3 within
  ULP (the S and R legs are symmetric around the pivot).
- **Strict order** -- `R3 > R2 > R1 > PP > S1 > S2 > S3` for any
  positive `range`.

## Distinct from neighbouring primitives

- vs `<ChartLineBollinger>` / `<ChartLineKeltner>` -- those are
  symmetric bands around a rolling MEAN; the Fibonacci pivot
  projects forward from a SINGLE PRIOR bar's HLC, refreshing every
  bar.
- vs `<ChartLineVolatilityStop>` -- the Volatility Stop is a
  DIRECTIONAL trailing stop that flips on a cross; the Fibonacci
  pivot is a static set of horizontal levels per bar.
- vs other pivot variants (Camarilla, Woodie, Classic) -- those use
  different offset coefficients on the same prior-bar reference;
  the Fibonacci variant uses the golden-ratio fractions 0.382 /
  0.618 plus a 1.0 outer level.

## API

`data` -- a single `ChartLinePivotFibonacciPoint[]`
(`{x, high, low, close}`). `ratios` (partial; default
`{level1: 0.382, level2: 0.618, level3: 1}`; per-key fallback).
Standard sizing / colour / a11y props plus `strokeWidth`,
`dotRadius`, `tickCount`. Toggle flags: `showAxis`, `showGrid`,
`showDots` (default false), `showPp`, `showLevel1`, `showLevel2`,
`showLevel3`, `showMarkers`, `showTooltip`, `showConfigBadge`,
`showLegend`, `animate`. Controlled and uncontrolled visibility via
`hiddenSeries`
(`'price' | 'pp' | 'r1' | 's1' | 'r2' | 's2' | 'r3' | 's3'`).
Callbacks: `onPointClick({point})`,
`onSeriesToggle({seriesId, hidden})`. Format functions:
`formatValue`, `formatX`. ARIA: `ariaLabel`, `ariaDescription`.

## Pure helpers

- `DEFAULT_CHART_LINE_PIVOT_FIBONACCI_*` -- defaults.
- `getLinePivotFibonacciFinitePoints` -- keep only points with
  finite x / high / low / close and `high >= low`.
- `normalizeLinePivotFibonacciRatios(ratios, fallback)` -- per-key
  fallback for non-finite or negative ratios; accepts zero.
- `computeLinePivotFibonacciLevels(prev, ratios)` -- the seven
  levels for a single bar; nulls when prev is null or non-finite.
- `computeLinePivotFibonacci(bars, ratios)` -- per-bar levels;
  bar 0 is all nulls.
- `classifyLinePivotFibonacciZone(close, levels)` --
  `'above-r2' | 'r1-to-r2' | 'pp-to-r1' | 's1-to-pp' |
  's2-to-s1' | 'below-s2' | 'none'`.
- `runLinePivotFibonacci(points, options?)` --
  `{series, ratios, levels, samples, ppFinal, aboveCount,
  belowCount, betweenCount, ok}`.
- `computeLinePivotFibonacciLayout({data, ratios, ...})` -- single
  panel with the close line, seven level stub segments per defined
  bar (one segment per level per bar), one marker per defined-zone
  bar coloured by zone band.
- `describeLinePivotFibonacciChart(data, options?)` -- ARIA summary
  mentioning the ratios, the pivot formula, the warm-up first bar,
  the count of bars above / below / between, and the final pivot.

## a11y / data hooks

Root `<div role="region">` with `aria-label` + `aria-describedby`; the
SVG is `role="img"` with the same label; an sr-only
`data-section="chart-line-pivot-fibonacci-aria-desc"` span carries
the description. The close line, every price dot and every marker
carry `role="graphics-symbol"` and `tabIndex={0}`. Every node carries
a `data-section`. Root mirrors `data-ratio-level1`,
`data-ratio-level2`, `data-ratio-level3`, `data-pp-final`,
`data-above-count`, `data-below-count`, `data-between-count`,
`data-total-points`, `data-animate`, `data-empty`. Segments expose
`data-series-id` and `data-value`; markers expose `data-zone` and
`data-close`.

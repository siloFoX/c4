# 11.559 -- web/src/components/ui/chart-line-tsne.tsx

Pure-SVG line chart with a **2D embedding side panel**. The
main panel (left) draws a multivariate time series -- one
line per channel; the side panel (right) projects every
shared time point into a 2D scatter via a **simple PCA-based
projection**. This is the placeholder for a t-SNE-style
nonlinear embedding visual -- a fast, deterministic *linear*
stand-in.

## Math

Each time point (an x value present in *every* channel) is
treated as a `D`-dimensional observation, `D` = the channel
count. The `N x D` observation matrix is projected to 2D by
classical PCA:

1. **Centre** each column on its mean.
2. Form the `D x D` **covariance** matrix.
3. Extract the **top-2 principal components** by **power
   iteration with deflation**: PC1 is the dominant
   eigenvector of the covariance; the covariance is deflated
   by `lambda1 * pc1 pc1^T` and PC2 is iterated in the
   subspace orthogonal to PC1.
4. **Project** each centred observation onto PC1 / PC2 ->
   `(e1, e2)`.

The eigenvectors are **sign-fixed** (the largest-magnitude
component is forced positive) so the embedding is fully
deterministic and reproducible -- no random seed, no
iteration jitter. Power iteration runs from a fixed
distinct-component seed, so two runs on the same data give
identical output.

`explainedVariance[k] = eigenvalue_k / totalVariance`, where
`totalVariance` is the trace of the covariance matrix; the
fraction captured by the 2D embedding is
`explainedVariance[0] + explainedVariance[1]`.

`ok = false` when there are fewer than 2 channels or fewer
than 2 shared time points (no embedding possible); the main
line chart still renders.

### Verified properties (by test)

- For `y = x` data (a 2-channel matrix whose rows all lie on
  the `(1,1)` line) PC1 aligns with `(1,1)/sqrt(2)`, captures
  100% of the variance, and the PC2 embedding coordinate is
  zero for every point.
- For 3-channel data where channel B = 2*A and C is constant,
  the covariance is rank 1: PC1 is `(1,2,0)/sqrt(5)` with
  eigenvalue 10, capturing 100% of the variance.
- The principal components are unit vectors and mutually
  orthogonal.
- Non-finite rows are dropped before the fit.

## Why PCA as a placeholder

Real t-SNE is an *iterative, stochastic, nonlinear*
embedding -- expensive and non-deterministic. For a chart
primitive a linear PCA projection is the standard
lightweight stand-in: it is `O(N D^2)`, deterministic, and
needs no training loop, while still giving the
"high-dimensional state -> 2D scatter trajectory" visual
that an embedding panel is for. The component is named
`chart-line-tsne` to mark its intent as the embedding
primitive; the projection is documented as PCA-based.

## Public surface

- `<ChartLineTsne series={...} />`.
- `series: ChartLineTsneSeries[]` --
  `{id, label, data, color?}`. Each series is one channel /
  dimension.
- `mainPanelRatio` (default 0.58; clamped to `[0.2, 0.8]`).
- `embeddingColor` -- colour of the embedding dots /
  trajectory.
- Toggle flags: `showAxis`, `showGrid`, `showDots`,
  `showLegend`, `showTooltip`, `showBadge`, `showTrajectory`,
  `showEmbeddingDots`, `animate`.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX` / `formatEmbedding`.
- `xLabel` / `yLabel`.
- `onPointClick({channel, point})` (main panel),
  `onEmbeddingPointClick({point})` (embedding panel).

## Pure helpers

- `getLineTsneDefaultColor(index)`.
- `getLineTsneFinitePoints(points)`.
- `buildLineTsneMatrix(series)` -- pairs all channels by
  exact x match into the `N x D` observation matrix; one row
  per x present in every channel.
- `computePcaProjection(matrix, maxIterations?)` -- the PCA
  projection: `{ok, sampleCount, dimensions, mean,
  components, eigenvalues, totalVariance, explainedVariance,
  embedding}`. Drops non-finite rows.
- `runLineTsne(series, maxIterations?)` -- `buildLineTsneMatrix`
  + `computePcaProjection` together.
- `computeLineTsneLayout({...})` -- the dual-panel layout:
  the main multi-line chart, the embedding scatter +
  trajectory path, tick arrays, the PCA result.
- `describeLineTsneChart(series)`.

## Layout

Two side-by-side panels sharing no axes:
- **Main panel** (left, ~58%): one line per channel over the
  time axis, on a shared y-axis.
- **Embedding panel** (right, ~42%): a 2D scatter of the
  embedded time points. PC1 on x, PC2 on y, a dashed origin
  crosshair at the data centroid, and a faint **trajectory
  path** connecting consecutive time points. The dots ramp
  in opacity from the first time point (0.35) to the last
  (1.0) so the direction of travel through embedding space
  is visible.

## Tooltip

Hovering a main-panel dot shows the channel label, x, y.
Hovering an embedding point shows the time x, PC1 and PC2.

## Config badge

Top-left badge shows `PCA d=<channels> n=<samples>
var=<explained variance %>` (or `embedding n/a` when the
embedding is unavailable).

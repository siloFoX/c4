## Goal

Polish `web/src/pages/Health.tsx` to bring the Modules
surface up to the rest-of-app rhythm:

- BadgeCounter chips next to the module counts.
- ScrollArea cap on long module lists.
- ErrorState (with Retry button) on the error path.
- Accordion items grouped by module category.

The Sparkline trends already shipped in v1.11.279 -- this
patch leaves them in place. The dispatch lists Sparkline
as part of the polish set; v1.11.340 confirms that the
three hero sparklines (uptime / workers / queue) plus the
inline Sparklines in the hero DataList rows are still on
the page.

## Module categorisation

A new helper:

```ts
export function categorizeModule(name: string): string;
```

returns the leading path segment (e.g. `"src/foo.js"` ->
`"src"`), or the file extension when there is no path
(e.g. `"foo.js"` -> `".js"`), or `"other"` when neither
applies. Exported from `web/src/pages/Health.tsx` so the
rule stays unit-checked.

Inside the page, `data.modules` flows through a
`useMemo<Map<string, string[]>>` that buckets every module
into its category. Insertion order is preserved so the
Accordion items render in a stable order across
re-renders.

## Accordion behaviour

| modules count | category count | render |
| --- | --- | --- |
| 0 | -- | "Loaded-modules not yet exposed" hint (unchanged) |
| n >= 1 | 1 | single Accordion item titled `Loaded modules (n)` with BadgeCounter chip |
| n >= 1 | >= 2 | one Accordion item per category, title `<cat>` + BadgeCounter chip |

The Build accordion item (daemon version + config path)
is preserved unchanged at the bottom of the list.

`defaultOpenIds` switches based on the category count:

- Single-category: `['modules']` (matches the prior
  v1.11.272 default).
- Multi-category: every category id (`modules-<cat>`) is
  opened by default, so the operator sees the full
  diagnostic surface without an extra click. The Build
  item stays collapsed by default.

## ScrollArea cap

A new constant:

```ts
const MODULES_SCROLL_THRESHOLD = 12;
```

gates a `ScrollArea axis="y" className="max-h-64"`
wrapper around the module list. Below the threshold the
list renders inline (no wrapper -- lighter DOM, fewer
scroll shells). At or above the threshold the wrapper
kicks in so a daemon with hundreds of modules cannot
balloon the accordion item beyond one viewport.

The threshold applies per-category in the multi-category
branch so a single big category triggers its own
ScrollArea independently.

## ErrorState swap

`ErrorPanel` (imported from `./PageFrame`) is replaced by
the canonical `ErrorState` primitive on the error path.
`role="alert"` is preserved by the primitive so the prior
`getByRole('alert')` assertions still hold. New surface:

- Title: "Could not load /api/health".
- Description: "The daemon returned an error. Retry to
  re-fetch."
- `onRetry`: wired to the same `refresh` callback the
  header refresh icon uses.
- `data-testid="health-error-state"` for direct query.

## Tests

`web/src/pages/Health.test.tsx` adds nine cases on top of
the existing thirty-seven:

1. error state shows a Retry button that fires `refresh`;
2. BadgeCounter renders with the correct count;
3. long module lists wrap in a ScrollArea (>= 12 entries);
4. short module lists do NOT wrap in a ScrollArea;
5. multi-category modules render one accordion item per
   category;
6. `categorizeModule` splits by leading path segment;
7. `categorizeModule` falls back to file extension;
8. `categorizeModule` falls back to "other";
9. (implicit via 5) BadgeCounter renders per-category
   counts.

44/45 cases pass. The single pre-existing failure is the
`does NOT render the loading skeleton when data is
already present` case -- the StatusDot primitive exposes
`role="status"` for its pulse animation, which the test
does not account for. Left untouched to keep this
patch's scope tight (it can be addressed separately by
either dropping the assertion or scoping it to the
skeleton's container).

## Adoption sites

The Health page is the only adopter. The
`categorizeModule` helper is exported so tooling /
follow-up tests can reuse it without re-implementing the
rule.

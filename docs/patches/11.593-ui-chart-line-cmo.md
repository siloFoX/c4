## Summary

New `ChartLineCmo` UI primitive at
`web/src/components/ui/chart-line-cmo.tsx` -- a pure-SVG line chart
with a Chande Momentum Oscillator panel on a +/-100 scale. The value
line occupies the top panel; the bottom panel renders the CMO on a
fixed -100 to +100 scale with a dashed zero centerline, +50/-50 level
lines, and shaded overbought/oversold zones.

## Indicator math

`computeLineCmoChanges(values)` splits each period's change into a
gain (the positive part) and a loss (the magnitude of the negative
part); index 0 reads null.

`computeLineCmo(values, period)` is Tushar Chande's Momentum
Oscillator. Over a trailing window of `period` changes the gains and
losses are summed with a *plain sum* -- no Wilder smoothing:

```
CMO = 100 * (sumUp - sumDown) / (sumUp + sumDown)
```

The result runs -100 (all losses) to +100 (all gains), centred on
zero. A window with no movement reads 0. CMO is defined from index
`period` onward.

### Worked fixture

`values = [10, 18, 17, 18, 10, 21]`, `period = 3`:

```
changes -> up / down: +8/0, -1/1, +1/0, -8/8, +11/0
i=3 window 1..3: up 9, down 1  -> 100*(9-1)/10  = 80
i=4 window 2..4: up 1, down 9  -> 100*(1-9)/10  = -80
i=5 window 3..5: up 12, down 8 -> 100*(12-8)/20 = 20
cmo = [.,.,.,80, -80, 20]
```

`cmo[3] = 80` is overbought; `cmo[4] = -80` is oversold;
`overboughtCount = 1`, `oversoldCount = 1`, `cmoFinal = 20`.

## API

- `getLineCmoFinitePoints(points)` -- drop points with a non-finite
  `x` or `value`.
- `normalizeLineCmoPeriod(period, fallback)` -- positive integer;
  non-finite / sub-1 falls back, fractional floors.
- `computeLineCmoChanges(values)` -- the `{ up, down }` gain/loss
  pair.
- `computeLineCmo(values, period)` -- the CMO series.
- `runLineCmo(points, options?)` -- sort finite points by `x`, compute
  the CMO series, classify each reading as overbought / oversold /
  neutral, return `{ series, period, overbought, oversold, cmo,
  samples, cmoFinal, overboughtCount, oversoldCount, ok }`. `ok` is
  false for fewer than two points.
- `computeLineCmoLayout(options)` -- stack a value panel above a fixed
  -100 to +100 CMO panel. Returns panels, ticks, the value path, CMO
  path, zone-tagged markers, the zero centerline, the level
  positions, and the two shaded zone rectangles.
- `describeLineCmoChart(data, options?)` -- ARIA description string.

## Component

`ChartLineCmo` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge (period + total extreme count), a two-series legend (Value /
CMO) with toggle, a dashed zero centerline, dashed +50/-50 level
lines, two shaded extreme zone rectangles, zone-coloured CMO markers,
a hover tooltip exposing the value, CMO, and zone, and a
`motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs chart-line-rsi** -- RSI uses Wilder-smoothed average gains and
  losses on a 0 to 100 scale centred on 50. CMO sums the *raw* gains
  and losses over the window with *no smoothing* and reports their net
  on a -100 to +100 scale centred on zero.
- **vs chart-line-tsi** -- TSI double-smooths the momentum (two
  successive EMAs). CMO is the un-smoothed, single-window-sum momentum
  ratio: `(sumUp - sumDown) / (sumUp + sumDown)`.

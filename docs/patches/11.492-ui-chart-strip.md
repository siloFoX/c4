# TODO 11.492 - chart-strip primitive

Status: SHIPPED v1.11.510 (mgr-direct on c4/c4-mgr-auto)

## Summary

Adds `<ChartStrip>`, a pure-SVG one-dimensional strip plot (also known as
a 1D scatter or strip chart). Each data point is a small mark placed
along a single value axis. Multiple groups stack into parallel lanes.
Optional deterministic jitter spreads coincident values in the
orthogonal direction so dense regions remain readable.

## API

```tsx
<ChartStrip
  groups={[
    { id: 'a', label: 'Control', values: [12, 14, 15, 15, 17, 18, 18, 21] },
    { id: 'b', label: 'Treatment', values: [10, 11, 13, 14, 14, 16, 17] },
  ]}
  orientation="horizontal"
  mark="tick"
  jitter={0.6}
  jitterSeed={42}
  valueLabel="ms"
  onPointClick={({ group, point }) => console.log(group.id, point.value)}
/>
```

## Exported helpers

- `DEFAULT_CHART_STRIP_*` numeric / string defaults
- `DEFAULT_CHART_STRIP_PALETTE` (10-colour palette)
- `getStripDefaultColor(index)` - palette index with modulo wrap
- `getStripFiniteValues(values)` - drops non-finite values
- `getStripBounds(groups, hidden?)` - min/max across visible finite values;
  empty -> `(0..1)`, collapsed range expands +/- 0.5
- `getStripTicks(min, max, count?)` - evenly-spaced ticks with normalised
  position `[0, 1]`; collapsed -> single tick at `min`
- `getStripJitter(seed, groupIndex, pointIndex)` - deterministic offset in
  `[-1, 1]` using a mulberry32-style mixer; identical input -> identical
  output so points never reshuffle across renders
- `computeStripLayout(input)` - canonical layout returning
  `{ groups, ticks, valueMin, valueMax, orientation, mark, innerWidth,
  innerHeight, laneHeight }`; empty when canvas is degenerate or every
  group is hidden
- `describeStripChart(groups, hidden?, formatValue?)` - ARIA summary
  `"Strip plot of N values across M groups, range <min> to <max>."`

## Distinct from prior chart-* primitives

- `<ChartScatter>` - 2D scatter; both x and y encode values.
- `<ChartBubble>` (11.470) - 2D scatter with size encoding.
- `<ChartDotPlot>` - stacks dots vertically by bin to show frequency
  (Wilkinson dot plot); strip leaves raw points along the axis.
- `<ChartBoxplot>` - five-number summary; strip plots the raw data.
- `<ChartHistogram>` (11.473) - binned counts; strip plots every value.
- `<ChartViolin>` - KDE-shaped density envelope; strip is point-level.
- `<ChartRidge>` (11.474) - stacked density curves across lanes; strip
  is stacked raw-point lanes.
- `<ChartCalendarHeatmap>` (11.471) - day grid coloured by value; strip
  is a 1D axis instead of a calendar grid.

## Layout

### Value axis

- Range auto-computed from finite values across visible groups; collapsed
  range expands by `+/- 0.5`; empty -> `(0..1)`.
- `valueMin` / `valueMax` provide an explicit range; values outside the
  range are clamped to the nearest endpoint (so the point still appears
  at the boundary instead of being dropped).
- `horizontal` orientation maps the value axis to x (left -> right);
  `vertical` maps it to y (bottom -> top, i.e. larger values toward the
  top of the SVG).

### Lanes

- One lane per visible group.
- Total inner height (or width, for vertical) is divided into N equal
  lanes separated by `laneGap` pixels.
- Lane labels render at the lane's leading edge.

### Jitter

- `getStripJitter(seed, groupIndex, pointIndex)` returns a deterministic
  value in `[-1, 1]`.
- Final jitter offset = `getStripJitter(seed, gi, pi) * laneHeight/2 * jitter`.
- `jitter = 0` (default) means no orthogonal offset.
- Deterministic so points stay in place across renders; never shuffles.

### Marks

- `'tick'` - vertical (horizontal orientation) or horizontal (vertical
  orientation) line segment of length `markSize` (default 14).
- `'dot'` - circle of radius `markSize` (default 3).

## Interactivity

- Hover or focus on a mark dims every other mark to opacity 0.35 and
  opens a tooltip with the group label and formatted value.
- Click on a mark fires `onPointClick({group, point})`.
- Legend buttons toggle the visibility of a group (uncontrolled by
  default; bind `hiddenGroups` + `onHiddenGroupsChange` for controlled).
- Mount fade via `motion-safe:animate-fade-in` on the root; suppress
  with `animate=false`.

## ARIA

- Root `<div role="region">` with `aria-label` and `aria-describedby`
  pointing to an `sr-only` `data-section="chart-strip-aria-desc"` span
  containing the auto-generated summary.
- SVG `role="img"` with the same `aria-label`.
- Each mark is `role="graphics-symbol"` with `tabIndex={0}` and
  `aria-label="<group label>: <formatted value>"`.

## Data attributes

- Root: `data-section="chart-strip"`, `data-orientation`, `data-mark`,
  `data-jitter`, `data-group-count`, `data-visible-group-count`,
  `data-finite-count`, `data-animate`.
- Group: `data-section="chart-strip-group"`, `data-group-id`,

- Mark: `data-section="chart-strip-mark"`, `data-mark-kind`,

  children.
- Legend: `data-section="chart-strip-legend"`, `-legend-item`
  (`data-group-id`, `data-hidden`), `-legend-button`, `-legend-swatch`,
  `-legend-label`.
- Axes / grid: `-axes`, `-axis`, `-ticks`, `-tick`, `-tick-label`,
  `-grid`, `-grid-line`, `-value-label`, `-lane-labels`, `-lane-label`.

## Out of scope

- Half-eye / mean-bar overlays - host stacks a `<ChartBoxplot>` if it
  wants a five-number summary; strip remains point-level only.
- Beeswarm-style non-overlapping layout - v1 uses random jitter, not a
  force-directed swarm. A future `<ChartBeeswarm>` could share the
  group/lane API.
- Highlighting individual points by external selector (e.g.
  `selectedPointIds`); only hover/focus drives the highlight in v1.
- Log-scale value axis - linear only; host can pre-log values.
- Pre-aggregated input (e.g. `{value, count}` tuples) - v1 always
  consumes one value per point.

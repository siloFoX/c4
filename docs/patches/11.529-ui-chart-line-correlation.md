# 11.529 -- web/src/components/ui/chart-line-correlation.tsx

Pure-SVG **dual-axis line chart** that pairs a `primary` series on
the left y-axis with a `secondary` series on the right y-axis, each
normalised to its own independent y range, and computes the
**Pearson correlation coefficient** `r` between the two over their
shared x samples. The r value is surfaced as a coloured badge in
the top-left corner with magnitude + qualitative strength
(strong/moderate/weak/none) + direction (positive/negative/neutral).

The canonical "two metrics on different scales — do they correlate?"
view: temperature vs ice cream sales, ad spend vs revenue, traffic
vs errors, etc.

## Distinct from other multi-series primitives

- `<ChartLineComparison>` (11.511): two curves on the **same** y
  scale with difference shading between them. Correlation chart
  uses **separate** y axes so vastly different units can sit on the
  same chart, plus surfaces the statistical correlation as a badge.
- `<ChartLineMulti>` (11.500): multi-series with synchronised
  crosshair tooltips, same y scale.
- `<ChartLineResidual>` (11.525): observed minus predicted at the
  same x. Correlation pairs two independent series.
- `<ChartLineTrend>` (11.512): single-series regression line.
  Correlation is the bilateral statistic between two series.
- `<ChartLineAnomaly>` (11.524): z-score outliers in one series.
- `<ChartLineBaseline>` (11.510), `<ChartLineThreshold>` (11.503),
  `<ChartLineBand>` (11.516), etc.: horizontal references, not
  bilateral statistics.

## Public surface

- `<ChartLineCorrelation primary={...} secondary={...} />`.
- `primary` / `secondary`: `{id, label, data, color?}` -- each can
  be `null` for a single-series view.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showCorrelationBadge`, `showRightAxis`, `animate`.
- Per-axis bounds: `xMin/xMax`, `primaryYMin`/`primaryYMax`,
  `secondaryYMin`/`secondaryYMax`.
- Strength thresholds: `strongThreshold` (default 0.7),
  `moderateThreshold` (0.4), `weakThreshold` (0.2).
- `positiveColor`, `negativeColor`, `neutralColor` for the badge.
- Standard sizing + format + a11y props.
- `formatR(r)` for the badge value (sign-aware fallback).
- `onPointClick({series, point})`.

## Pure helpers

- `getLineCorrelationDefaultColor(index)` -- 10-color palette
  cycle.
- `getLineCorrelationFinitePoints(points)` -- finite-only filter.
- `pairLineCorrelationByX(a, b)` -- returns one
  `ChartLineCorrelationPair` per shared x value where both series
  have a finite sample. `indexA` / `indexB` reference ORIGINAL
  input arrays. Output sorted by x ascending.
- `computePearsonCorrelation(pairs)` -- canonical formula
  `r = Σ (ya - mean_a)(yb - mean_b) / sqrt(Σ (ya - mean_a)^2 *
  Σ (yb - mean_b)^2)`. Returns `{r, ok, meanA, meanB}`. `ok=false`
  when fewer than 2 pairs or either side has zero variance
  (constant signal). Clamps `r` to `[-1, 1]`.
- `classifyLineCorrelationStrength(r, strong?, moderate?, weak?)`
  -- `'strong' | 'moderate' | 'weak' | 'none'`. Non-finite `r` ->
  `'none'`. Non-finite thresholds fall back to defaults.
- `classifyLineCorrelationDirection(r)` -- `'positive' |
  'negative' | 'neutral'`. Non-finite -> `'neutral'`.
- `computeLineCorrelation(primary, secondary, ...thresholds?)` --
  one-call wrapper returning `{r, ok, pairCount, strength,
  direction, meanA, meanB}`.
- `computeLineCorrelationLayout({primary, secondary, ...})` --
  returns `{primary, secondary, correlation, xTicks, yLeftTicks,
  yRightTicks, xMin, xMax, innerWidth, innerHeight, totalPoints}`.
  Each layout series carries `{id, label, axis ('left'|'right'),
  color, points, path, finiteCount, totalCount, yMin, yMax}`.
- `describeLineCorrelationChart(primary, secondary, formatValue?)`
  -- builds the screen-reader description.

## Dual-axis rendering

The left y-axis is coloured by the primary series's color (so the
ticks visually associate with the left curve), and the right y-axis
is coloured by the secondary. Each tick set uses the **independent**
y range of its own series, so a series ranging 60-80 on the left can
share the chart with a series ranging 100-320 on the right without
either being squished.

The correlation badge sits in the top-left corner with a coloured
border + text matching the direction (green for positive, red for
negative, slate for neutral / no correlation). When `r` is not
computable the badge shows `r = n/a` and `no correlation`.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Per-line path + per-dot are `role="graphics-symbol"` +
  `tabIndex={0}`. Dot aria-label includes the axis side.
- Every node carries `data-section="chart-line-correlation-..."`
  for test selectors.
- Series groups expose `data-series-axis` (`'left' | 'right'`),
  `data-series-y-min`, `data-series-y-max`,
  `data-series-color`, `data-series-finite-count`.
- Root mirrors `data-r`, `data-pair-count`, `data-strength`,
  `data-direction`, `data-correlation-ok`.
- Badge mirrors the same `data-r` / `data-strength` /
  `data-direction` / `data-pair-count` / `data-correlation-ok`.
- Ticks expose `data-axis` (`'x' | 'y-left' | 'y-right'`).

## Tooltip

Hovering a dot shows the series label coloured by its line, an
`(axis side)` suffix, `x:` and bold `y:` rows.

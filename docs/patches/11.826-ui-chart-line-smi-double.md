# Patch 11.826 -- ChartLineSmiDouble primitive (v1.11.844)

## Summary

Add `<ChartLineSmiDouble />`, a pure-SVG dual-panel line chart that
renders Blau's Stochastic Momentum Index beneath a price line. The
SMI is the close re-centred on the lookback midpoint and divided by
half the lookback range, then double-EMA-smoothed and scaled to
`[-100, 100]`. The double-smoothing dampens the raw stochastic's
whipsaw while preserving the centred-zero crossing semantics used by
typical SMI overbought / oversold thresholds (`+/-40`).

## Formula

```
highestHigh[i] = max(high[i - length + 1 .. i])
lowestLow[i]   = min(low[i  - length + 1 .. i])
midpoint[i]    = (highestHigh[i] + lowestLow[i]) / 2
range[i]       = highestHigh[i] - lowestLow[i]
centered[i]    = close[i] - midpoint[i]
numerEMA[i]    = EMA(EMA(centered, smoothLength1), smoothLength2)[i]
denomEMA[i]    = EMA(EMA(range,    smoothLength1), smoothLength2)[i]
smi[i]         = denomEMA[i] === 0
                   ? null
                   : (numerEMA[i] / (denomEMA[i] / 2)) * 100
```

- Combined warmup `length - 1 + smoothLength1 - 1 + smoothLength2 - 1`
  is `null`.
- The EMA helper uses an SMA-seeded EMA with the CONST short-circuit
  `next = v === prev ? v : alpha*v + (1-alpha)*prev` to avoid 1-ULP
  drift on non-dyadic `alpha` for constant streams.
- `-0` is normalized to `+0` on every arithmetic result.

## Defaults

| Knob          | Value     |
|---------------|-----------|
| length        | 10        |
| smoothLength1 | 3         |
| smoothLength2 | 3         |
| overbought    | +40       |
| oversold      | -40       |
| smiColor      | `#db2777` (pink/magenta) |

The pink magenta keeps the SMI panel visually distinct from the
indigo/purple stochastic primitives shipped earlier in this batch.

## Bit-exact anchors

Integration sweep iterates `(K, length, smoothLength1, smoothLength2)`
and asserts:

- `CONST high = low = close = K` -> `range = 0` and `centered = 0`
  for every window, so `denomEMA = 0` -> divide-by-zero guard
  returns `null` everywhere post-warmup.
- `LINEAR UP high = low = close = i + 1`: rolling extrema give
  `centered = (length - 1) / 2`, `range = length - 1`. Both stream
  constants survive both EMA passes (CONST short-circuit on the
  SMA-seeded EMA). The ratio reduces to
  `((L-1)/2) / ((L-1)/2) * 100 = 100` bit-exact.
- `LINEAR DOWN high = low = close = N - i`: symmetric, `smi = -100`
  bit-exact.

Verified across `L in {4, 5, 7}` and `(S1, S2) in {2, 3}^2`.

## Surface

Standard primitive surface:

- `ChartLineSmiDoublePoint = { x, high, low, close }`
- `runLineSmiDouble(data, options?) -> Run` with
  `ok = series.length >= length + smoothLength1 + smoothLength2 - 2`.
- `computeLineSmiDoubleLayout({ data, width?, height?, padding?, panelGap?, length?, smoothLength1?, smoothLength2?, overbought?, oversold? }) -> Layout`.
  SMI axis fixed to `[-100, 100]`.
- `describeLineSmiDoubleChart(data, opts?) -> string` for the ARIA
  description.
- `<ChartLineSmiDouble />` `forwardRef<HTMLDivElement>` with legend
  toggle (`price`, `smi`), config badge, threshold + midline overlays,
  markers on threshold crosses, and a tooltip surfacing `close / hh /
  ll / centered / range / smi / zone`.

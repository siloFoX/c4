## Summary

New `ChartLineMfi` UI primitive at
`web/src/components/ui/chart-line-mfi.tsx` -- a pure-SVG line chart
with a Money Flow Index panel computed from paired price and volume
series. The price line occupies the top panel; the bottom panel
renders the MFI oscillator on a 0-100 scale with dashed 80/20 level
lines and shaded overbought/oversold zones. Both panels share one
x-axis.

## Indicator math

`computeLineMfi(prices, volumes, period)` is the Money Flow Index, a
volume-weighted momentum oscillator. Each period's raw money flow is
`price * volume`; the flow counts as positive when the price rose
from the prior period and negative when it fell. Over a trailing
window of `period` flows:

```
positiveFlow = sum of raw flows on up closes in the window
negativeFlow = sum of raw flows on down closes in the window
MFI = 100 * positiveFlow / (positiveFlow + negativeFlow)
```

A window with no directional flow reads 50. MFI is defined from index
`period` onward; earlier indices read `null`.

### Worked fixture

`prices = [10,20,10,20,10,20]`, `volumes = [100,400,100,50,800,50]`,
`period = 3`:

```
rawFlow = price * volume = [1000, 8000, 1000, 1000, 8000, 1000]
posMF   = [0, 8000, 0,    1000, 0,    1000]
negMF   = [0, 0,    1000, 0,    8000, 0]

i=3 window 1..3: pos 9000, neg 1000 -> 100*9000/10000 = 90
i=4 window 2..4: pos 1000, neg 9000 -> 100*1000/10000 = 10
i=5 window 3..5: pos 2000, neg 8000 -> 100*2000/10000 = 20

mfi = [.,.,.,90, 10, 20]
```

`mfi[3] = 90` is overbought; `mfi[4] = 10` is oversold;
`overboughtCount = 1`, `oversoldCount = 1`, `mfiFinal = 20`.

## API

- `getLineMfiFinitePoints(points)` -- drop points with a non-finite
  `x`, `price`, or `volume`.
- `normalizeLineMfiPeriod(period, fallback)` -- coerce a period to a
  positive integer; non-finite or sub-1 falls back, fractional floors.
- `computeLineMfi(prices, volumes, period)` -- the MFI series.
- `runLineMfi(points, options?)` -- sort finite points by `x`, compute
  the MFI series, classify each reading as overbought / oversold /
  neutral, return `{ series, period, overbought, oversold, mfi,
  samples, mfiFinal, overboughtCount, oversoldCount, ok }`. `ok` is
  false for fewer than two points.
- `computeLineMfiLayout(options)` -- stack a price panel above a fixed
  0-100 MFI panel. Returns panels, ticks, the price path, MFI path,
  zone-tagged markers, the level positions, and the two shaded zone
  rectangles.
- `describeLineMfiChart(data, options?)` -- ARIA description string.

## Component

`ChartLineMfi` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge (period + total extreme count), a two-series legend (Price /
MFI) with toggle, dashed 80/20 level lines, two shaded extreme zone
rectangles, zone-coloured MFI markers, a hover tooltip exposing the
price, volume, MFI, and zone, and a `motion-safe:animate-fade-in`
entrance. Every node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-rsi** -- RSI measures momentum from price changes
  alone. MFI weights each move by the money flow (price times volume)
  behind it, so a move on heavy volume shifts the oscillator far more
  than the same move on thin volume. The MFI conventional zones are
  80/20 rather than RSI's 70/30.
- **vs chart-line-obv** -- OBV is an unbounded cumulative signed-volume
  line with no window. MFI is a bounded 0-100 *windowed* oscillator.
- **vs chart-line-vwap** -- VWAP is a price-domain volume-weighted
  average overlaid on the price. MFI is a momentum oscillator in a
  dedicated sub-panel, not a price overlay.

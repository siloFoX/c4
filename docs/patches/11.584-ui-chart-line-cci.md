## Summary

New `ChartLineCci` UI primitive at
`web/src/components/ui/chart-line-cci.tsx` -- a pure-SVG line chart
with a Commodity Channel Index oscillator panel and shaded extreme
zones. The value line occupies the top panel; the bottom panel renders
the CCI oscillator on a symmetric y-axis with a dashed zero
centerline, dashed +/-100 level lines, and shaded overbought/oversold
zones beyond those levels. Both panels share one x-axis.

## Indicator math

`computeLineCci(values, period)` is Donald Lambert's Commodity Channel
Index. For each index from `period - 1` onward the window of `period`
values is reduced:

```
SMA          = mean(window)
meanDeviation = mean(|w - SMA|  for w in window)
CCI          = (value - SMA) / (0.015 * meanDeviation)
```

The 0.015 constant scales the result so roughly 70-80% of readings
fall within +/-100. A flat window (zero mean deviation) reads 0.
Indices before the window fills read `null`.

### Worked fixture

`values = [10, 12, 14, 13, 20, 8, 11]`, `period = 4`:

```
i=3 window [10,12,14,13]: SMA 12.25, meanDev 1.25
    -> (13-12.25)/(0.015*1.25)  = 0.75/0.01875  = 40
i=4 window [12,14,13,20]: SMA 14.75, meanDev 2.625
    -> (20-14.75)/(0.015*2.625) = 5.25/0.039375 = 133.3333
i=5 window [14,13,20,8] : SMA 13.75, meanDev 3.25
    -> (8-13.75)/(0.015*3.25)   = -5.75/0.04875 = -117.9487
i=6 window [13,20,8,11] : SMA 13,    meanDev 3.5
    -> (11-13)/(0.015*3.5)      = -2/0.0525     = -38.0952

cci = [.,.,.,40, 133.3333, -117.9487, -38.0952]
```

`cci[4] > +100` is overbought; `cci[5] < -100` is oversold;
`overboughtCount = 1`, `oversoldCount = 1`, `cciFinal ~ -38.0952`.

## API

- `getLineCciFinitePoints(points)` -- drop points with a non-finite
  `x` or `value`.
- `normalizeLineCciPeriod(period, fallback)` -- coerce a period to a
  positive integer; non-finite or sub-1 falls back, fractional floors.
- `computeLineCci(values, period, constant?)` -- the CCI series.
- `runLineCci(points, options?)` -- sort finite points by `x`, compute
  the CCI series, classify each reading as overbought / oversold /
  neutral, return `{ series, period, overbought, oversold, cci,
  samples, cciFinal, cciMin, cciMax, overboughtCount, oversoldCount,
  ok }`. `ok` is false for fewer than two points.
- `computeLineCciLayout(options)` -- stack a value panel above a CCI
  panel with a symmetric y-bound (the larger of the +/-100 band and
  the data extent). Returns panels, ticks, the value path, CCI path,
  zone-tagged markers, the zero / level positions, and the two shaded
  zone rectangles.
- `describeLineCciChart(data, options?)` -- ARIA description string.

## Component

`ChartLineCci` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge (period + total extreme count), a two-series legend (Value /
CCI) with toggle, a dashed zero centerline, dashed +/-100 level lines,
two shaded extreme zone rectangles, zone-coloured CCI markers, a hover
tooltip exposing the value, CCI, and zone, and a
`motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs chart-line-zscore** -- the z-score standardizes a value against
  the rolling *standard deviation*. CCI normalizes against the *mean
  absolute deviation* and applies Lambert's 0.015 constant -- a
  different dispersion measure that yields the conventional +/-100
  banding rather than a unit-variance score.
- **vs chart-line-rsi / chart-line-stochastic** -- those oscillators
  are bounded to 0-100. CCI is unbounded: it can run well past +/-100,
  which is why the panel uses a dynamic symmetric y-bound and the
  extreme zones are shaded regions beyond the +/-100 level lines
  rather than fixed bands.

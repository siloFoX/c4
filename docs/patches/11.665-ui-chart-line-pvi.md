## Summary

New `ChartLinePvi` UI primitive at
`web/src/components/ui/chart-line-pvi.tsx` -- a pure-SVG
two-panel chart with a Positive Volume Index panel tracking
cumulative change on higher-volume bars.

## Indicator math

The Positive Volume Index is the mirror of the Negative Volume
Index. It is a cumulative index that moves only on the loud
bars -- the premise is that the crowd trades on high-volume
days, so the PVI isolates what the price does on those days.

`computeLinePviIndex(closes, volumes, base)` seeds the index at
`base` and, for each later bar:

```
PVI[i] = PVI[i-1] * close[i] / close[i-1]   if volume[i] > volume[i-1]
PVI[i] = PVI[i-1]                           otherwise
```

So on a higher-volume bar the index compounds the price's
percent change; on a lower-volume bar (or an unchanged-volume
bar) it holds flat. An PVI above its starting base means the
price has gained ground on the busy days.

### Exactness

The index is a chain of ratio multiplications, so it is exact
when the closes are powers of two -- every ratio
`close[i]/close[i-1]` is then dyadic, and the index, seeded at
1000, stays a dyadic multiple of 1000. The fixture closes
`[256, 512, 512, 1024, 1024, 512, 512, 256, 256, 512]` paired
with the volumes `[20, 50, 30, 70, 40, 80, 30, 60, 25, 90]` give
the exact index `[1000, 2000, 2000, 4000, 4000, 2000, 2000,
1000, 1000, 2000]`, asserted with `toEqual`.

## API

- `getLinePviFinitePoints(points)` -- drop points with a
  non-finite `x`, `value` or `volume`, or a negative `volume`.
- `normalizeLinePviBase(base)` -- coerce the base to a positive
  number, falling back to 1000.
- `computeLinePviIndex(closes, volumes, base)` -- the cumulative
  index.
- `runLinePvi(points, options?)` -- sort finite points by `x`,
  run the pipeline, return `{ series, base, pvi, samples,
  pviFinal, upCount, downCount, flatCount, ok }`. Each sample is
  classified `up` / `down` / `flat` by the index movement. `ok`
  is false for fewer than two points.
- `computeLinePviLayout(options)` -- stack the price panel above
  a PVI panel; project the price path, the PVI path, one marker
  per bar, and the base reference line.
- `describeLinePviChart(data, options?)` -- ARIA description.

## Component

`ChartLinePvi` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, two stacked panels
(price on top, PVI below) with their own axis ticks and grid,
panel labels, a config badge showing the base, a two-series
legend (Price / PVI) with toggle, a hover/focus tooltip exposing
the volume, the PVI and the movement state, and a
`motion-safe:animate-fade-in` entrance. It consumes
`{ x, value, volume }` points. The PVI panel draws the index line
over a dashed base reference line, with each marker coloured by
its movement -- green up, red down, fuchsia flat. Every node
carries a `data-section` attribute.

## Distinctness

- **vs chart-line-nvi** -- the Negative Volume Index is the
  mirror: it moves on the quiet (falling-volume) bars. The PVI
  moves on the loud (rising-volume) bars.
- **vs chart-line-obv / chart-line-vzo** -- those accumulate
  volume itself. The PVI ignores volume size entirely and only
  uses it as a gate to decide which bars compound the price
  change.
- **vs chart-line-cumulative** -- a cumulative return runs on
  every bar. The PVI runs on a selected subset of bars and
  freezes on the rest.

## New prop

```ts
isDateDisabled?: (date: Date) => boolean;
```

Available on both `<DatePicker>` and
`<DateRangePicker>`. The predicate
receives a start-of-day `Date` and
returns `true` to mark the cell
unavailable.

## Pure helper (exported)

```ts
export function isDayAllowed(
  d: Date,
  min?: Date,
  max?: Date,
  isDateDisabled?: (date: Date) => boolean,
): boolean;
```

Combined gate -- a date is allowed
only when it passes ALL three:

| min | max | predicate | result |
| --- | --- | --- | --- |
| (none) | (none) | (none) | true |
| < min | (any) | (any) | false |
| (any) | > max | (any) | false |
| within | within | returns true | false |
| within | within | returns false | true |
| within | within | throws | false |

Predicate throws are swallowed -- a
bad caller-supplied function cannot
crash the calendar render. The cell
is treated as disabled when the
predicate raises.

## Cell + interaction behaviour

- Predicate-disabled cells render
  identically to min/max disabled
  cells: native `disabled` +
  `aria-disabled="true"` +
  `opacity-40` + pointer-events
  stripped.
- Click on a disabled cell is a
  no-op (no `onChange` fired).
- Enter on a predicate-disabled
  focused cell is a no-op. Arrow
  navigation still moves focus
  through the cell so the user
  can see what the constraint
  excluded, but they cannot
  select it.
- DateRangePicker honours the
  predicate in BOTH calendar
  panes simultaneously.

## Out of scope (deferred)

- Memoization across cells. The
  predicate is called per cell
  on each render. If a caller
  hits a hot-loop, wrap the
  predicate with `useMemo`.
- Async predicate (e.g., "check
  if this slot is booked").
  Synchronous for v1; adopters
  that need async should fetch
  the disabled-set upfront and
  close over it in the
  predicate.
- Multi-date selection. Single
  date + range only.
- Year / decade picker
  jump-views. Prev/Next year
  buttons handle the common
  case; a year-picker view is
  a follow-on.
- Per-cell tooltip on disabled
  cells. Host layer.

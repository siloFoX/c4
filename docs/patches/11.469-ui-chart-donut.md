## Summary

Add a new `<ChartDonut>` UI primitive in
`web/src/components/ui/chart-donut.tsx` -- pure-SVG donut
chart (variant of pie with a center hole). Each slice is
an annular arc; the hollow center hosts a configurable
label group (primary value + secondary label). Distinct
from `<ChartPie>` (11.440): donut has a first-class center
slot with default total + label, configurable
thicknessRatio, optional padAngle to space arcs, and
cornerRadius for rounded arc ends. On hover each arc
opens a tooltip with label / value / percent. Legend has
a focusable `<button aria-pressed>` per slice that
toggles its visibility; hidden slices reproportion the
ring.

## API

```ts
interface ChartDonutSlice {
  id: string;
  label: string;
  value: number;
  color?: string;
}

interface ChartDonutCenterContent {
  primary?: ReactNode;
  secondary?: ReactNode;
}

interface ChartDonutProps {
  slices: readonly ChartDonutSlice[];
  width?: number;             // default 320
  height?: number;            // default 320
  padding?: number;           // default 24
  thicknessRatio?: number;    // default 0.32 (0..1)
  padAngle?: number;          // default 0.012 (rad)
  cornerRadius?: number;      // default 0
  startAngle?: number;        // default -pi/2 (top)
  hiddenSlices?: readonly string[];
  defaultHiddenSlices?: readonly string[];
  onHiddenSlicesChange?: (hidden: string[]) => void;
  showLegend?: boolean;       // default true
  showTooltip?: boolean;      // default true
  showCenter?: boolean;       // default true
  showLabels?: boolean;       // default false (in-arc)
  showCenterTotal?: boolean;  // default true
  animate?: boolean;          // default true
  centerContent?: ChartDonutCenterContent;
  centerLabel?: ReactNode;
  centerValue?: ReactNode;
  className?: string;
  ariaLabel?: string;         // default 'Donut chart'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  formatPercent?: (v: number) => string;
  legendPlacement?: 'right' | 'bottom'; // default 'bottom'
  onArcClick?: (args: { slice: ChartDonutSlice; arc: ChartDonutArc }) => void;
  onSliceToggle?: (args: { slice: ChartDonutSlice; hidden: boolean }) => void;
  style?: CSSProperties;
}
```

## Behaviour

- Renders `<div role="region">` wrapping
  `<svg role="img">` with the same `aria-label`.
- Center slot: when `showCenter !== false`, an in-SVG
  group draws a primary line + a smaller secondary
  line in the donut hole. Resolution order for the
  primary line: `centerValue` > `centerContent.primary`
  > formatted visible total (suppressed via
  `showCenterTotal=false`). Resolution order for the
  secondary line: `centerLabel` >
  `centerContent.secondary`. If neither lane resolves
  the entire group is omitted.
- Layout: `computeDonutArcs` sums visible-positive
  values, drops the rest, then assigns each visible
  slice an angular sweep proportional to its share of
  the visible total. `padAngle` is deducted between
  consecutive arcs (only when more than one slice
  is visible). `thicknessRatio` is clamped to `[0, 1]`;
  inner radius = `outerRadius * (1 - thicknessRatio)`.
- Hover / focus on an arc sets the hovered id; the
  tooltip surfaces label, formatted value, and
  formatted percent.
- Legend: each entry is a focusable `<button>` with
  `aria-pressed`; controlled via `hiddenSlices` +
  `onHiddenSlicesChange`, or uncontrolled via
  `defaultHiddenSlices`. Toggling fires
  `onSliceToggle({ slice, hidden })` with the next
  state.
- Mount animation:
  `motion-safe:animate-fade-in` per arc.
  `animate=false` removes the class.
- Default ARIA description: `"Donut chart with N
  slices, total <total>. <label> <value> (P%) ..."`
  -- empty / all-hidden input returns `"No data"`;
  `ariaDescription` overrides.

## Pure helpers (exported)

```ts
DEFAULT_CHART_DONUT_WIDTH            = 320
DEFAULT_CHART_DONUT_HEIGHT           = 320
DEFAULT_CHART_DONUT_PADDING          = 24
DEFAULT_CHART_DONUT_THICKNESS_RATIO  = 0.32
DEFAULT_CHART_DONUT_PAD_ANGLE        = 0.012
DEFAULT_CHART_DONUT_CORNER_RADIUS    = 0
DEFAULT_CHART_DONUT_START_ANGLE      = -Math.PI / 2
DEFAULT_CHART_DONUT_COLORS           = [10-color palette]

getDonutDefaultColor(index): string
polarToCartesian(cx, cy, radius, angle): {x, y}
getDonutTotal(slices): number
getDonutVisibleTotal(slices, hidden): number
computeDonutArcs({slices, hidden, cx, cy, outerRadius,
                  thicknessRatio, startAngle, padAngle,
                  cornerRadius}): ChartDonutArc[]
describeDonutChart(slices, hidden, formatValue?): string
```

- `getDonutDefaultColor` wraps the palette modulo;
  non-finite / negative index falls back to color 0.
- `getDonutTotal` / `getDonutVisibleTotal` clamp
  non-positive + non-finite slice values to 0.
- `computeDonutArcs` is the canonical layout pass.
  Visible arcs are reproportioned across the full
  ring (minus pad angles). Arc fractions sum to 1.
  Each arc carries `{ id, label, value, color,
  index, startAngle, endAngle, midAngle,
  innerRadius, outerRadius, path, fraction }`.
- `describeDonutChart` returns the auto ARIA summary.

  `data-section="chart-donut-aria-desc"`.

  `tabIndex={0}`. aria-label = `"<label>: <value>

## Out of scope

- Drag-to-reorder slices around the ring -- host
  owns slice order.
- Animated reflow on data change (snap v1; mount
  fade only).
- Drill-down (clicking an arc swaps to its sub-data)
  -- host handles via `onArcClick`.
- Multi-ring concentric donuts -- v1 is single ring;
  host nests `<ChartDonut>` if needed.

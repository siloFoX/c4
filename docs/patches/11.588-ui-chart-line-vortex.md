## Summary

New `ChartLineVortex` UI primitive at
`web/src/components/ui/chart-line-vortex.tsx` -- a pure-SVG line chart
with a Vortex Indicator panel plotting the positive and negative
vortex movement lines. The value line occupies the top panel; the
bottom panel renders the VI+ and VI- lines on a 0 to 1 scale with a
dashed 0.5 crossover midline and amber crossover markers. Both panels
share one x-axis.

## Indicator math

`computeLineVortexMovement(values)` is the per-period vortex movement
for a single-value series. The positive vortex movement is the
up-part of the period's change, the negative is the down-part, and
the true range is their sum (the absolute change):

```
vmPlus[i]    = max(value[i] - value[i-1], 0)
vmMinus[i]   = max(value[i-1] - value[i], 0)
trueRange[i] = |value[i] - value[i-1]| = vmPlus[i] + vmMinus[i]
```

`computeLineVortex(values, period)` is the Vortex Indicator. Over a
trailing window of `period` movements, via simple window sums:

```
VI+ = sum(VM+) / sum(TR)
VI- = sum(VM-) / sum(TR)
```

With the single-value adaptation `TR = VM+ + VM-`, the two lines sum
to 1 and cross at 0.5 as the trend turns. A flat window (zero true
range) reads 0.5 for both. VI is defined from index `period` onward;
earlier indices read `null`.

### Worked fixture

`values = [10, 14, 10, 12, 8, 12]`, `period = 3`:

```
vmPlus    = [., 4, 0, 2, 0, 4]
vmMinus   = [., 0, 4, 0, 4, 0]
trueRange = [., 4, 4, 2, 4, 4]

i=3 window 1..3: VI+ 6/10 = 0.6, VI- 4/10 = 0.4  (up)
i=4 window 2..4: VI+ 2/10 = 0.2, VI- 8/10 = 0.8  (down)
i=5 window 3..5: VI+ 6/10 = 0.6, VI- 4/10 = 0.4  (up)
```

The trend runs up -> down -> up, so VI+/VI- cross twice
(`crossUpCount = 1`, `crossDownCount = 1`); `trendFinal = up`.

## API

- `getLineVortexFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineVortexPeriod(period, fallback)` -- coerce a period to
  a positive integer; non-finite or sub-1 falls back, fractional
  floors.
- `computeLineVortexMovement(values)` -- the `{ vmPlus, vmMinus,
  trueRange }` triple.
- `computeLineVortex(values, period)` -- the `{ viPlus, viMinus }`
  series.
- `runLineVortex(points, options?)` -- sort finite points by `x`,
  compute the VI series, classify each sample's trend, detect
  crossings, return `{ series, period, viPlus, viMinus, samples,
  viPlusFinal, viMinusFinal, trendFinal, crossUpCount, crossDownCount,
  ok }`. `ok` is false for fewer than two points.
- `computeLineVortexLayout(options)` -- stack a value panel above a
  fixed 0 to 1 Vortex panel. Returns panels, ticks, the value path,
  VI+ / VI- paths, markers (crossings flagged), and the midline `y`.
- `describeLineVortexChart(data, options?)` -- ARIA description.

## Component

`ChartLineVortex` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge (period + final trend), a three-series legend (Value / VI+ /
VI-) with toggle, a dashed 0.5 crossover midline, trend-coloured
markers with amber crossover highlighting, a hover tooltip exposing
the value, VI+, VI-, and trend, and a `motion-safe:animate-fade-in`
entrance. Every node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-adx** -- ADX Wilder-smooths the directional
  movement and then derives a separate ADX trend-*strength* line. The
  Vortex Indicator uses plain window *sums* (no Wilder smoothing) and
  is just the two VI lines, read by their *crossover* rather than by a
  strength magnitude. There is no third derived line.

## Summary

New `ChartLineFisher` UI primitive at
`web/src/components/ui/chart-line-fisher.tsx` -- a pure-SVG line chart
with a Fisher Transform panel sharpening turning points alongside its
trigger line. The value line occupies the top panel; the bottom panel
renders the Fisher line and its one-bar-lagged trigger line on a
symmetric y-axis with a dashed zero centerline.

## Indicator math

`computeLineFisher(values, period)` is John Ehlers' Fisher Transform.
For each index from `period - 1` onward:

```
rawNorm  = 2 * (value - windowLow) / (windowHigh - windowLow) - 1
smoothed = 0.33 * rawNorm + 0.67 * prevSmoothed   (seeded at the first index)
smoothed = clamp(smoothed, -0.999, 0.999)
fisher   = 0.5 * ln((1 + smoothed) / (1 - smoothed)) + 0.5 * prevFisher
```

The `0.5 * ln((1 + v) / (1 - v))` term is the inverse hyperbolic
tangent: it steepens sharply as `v` approaches `+/-1`, so price
turning points become pronounced peaks rather than gentle curves --
the "sharpening" the Fisher Transform is known for. The clamp keeps
the log finite. The trigger line is the Fisher lagged by one bar.

### Worked fixture

`values = [10, 30, 20, 40, 30, 50]`, `period = 3`:

```
rawNorm  = [., ., 0, 1, 0, 1]
smoothed = [., ., 0, 0.33, 0.2211, 0.478137]
fisher   = [., ., 0, 0.342828, 0.396226, 0.718681]
trigger  = [., ., ., 0, 0.342828, 0.396226]   (Fisher lagged 1)
```

## API

- `getLineFisherFinitePoints(points)` -- drop points with a non-finite
  `x` or `value`.
- `normalizeLineFisherPeriod(period, fallback)` -- positive integer;
  non-finite / sub-1 falls back, fractional floors.
- `computeLineFisher(values, period)` -- the `{ fisher, trigger }`
  series.
- `runLineFisher(points, options?)` -- sort finite points by `x`,
  compute the Fisher and trigger series, return `{ series, period,
  fisher, trigger, samples, fisherFinal, triggerFinal, fisherMin,
  fisherMax, ok }`. `ok` is false for fewer than two points.
- `computeLineFisherLayout(options)` -- stack a value panel above a
  Fisher panel with a symmetric y-bound. Returns panels, ticks, the
  value path, Fisher path, trigger path, markers, and the zero
  centerline `y`.
- `describeLineFisherChart(data, options?)` -- ARIA description.

## Component

`ChartLineFisher` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge (period), a three-series legend (Value / Fisher / Trigger) with
toggle, a dashed zero centerline, the Fisher line, a dashed trigger
line, markers on the Fisher line, a hover tooltip exposing the value,
Fisher, and trigger, and a `motion-safe:animate-fade-in` entrance.
Every node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-stochastic / chart-line-williams-r** -- those
  oscillators report the *raw position* of the value within its
  high-low range. The Fisher Transform begins with that same range
  normalization but then applies the *arctanh transform*. That
  non-linear step -- absent from the plain range oscillators -- is
  what sharpens the extremes: a small move near the range boundary
  produces a large Fisher move, so turning points stand out as
  spikes. The Fisher output is also unbounded and centred on zero,
  unlike the bounded 0-100 / 0..-100 range oscillators.

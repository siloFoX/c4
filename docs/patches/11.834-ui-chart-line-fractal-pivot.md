# Patch 11.834 -- ChartLineFractalPivot primitive (v1.11.852)

## Summary

Add `<ChartLineFractalPivot />`, a pure-SVG single-panel chart that
marks confirmed Bill Williams n-bar swing-high and swing-low fractal
pivots on a price line. Unlike the related fractal primitives in
this batch:

- `chart-line-fractal-bb` (11.815): SMA + standard-deviation band
  anchored to fractals.
- `chart-line-fractal-kc` (11.832): Keltner-style mid + ATR band
  anchored to fractals.
- `chart-line-fractal-channel` (11.833): rolling max/min of fractal
  pivot values.
- `chart-line-fractal-pivot` (11.834, this patch): pure marker
  overlay -- no band, just the pivot points.

```
isUpperFractal[i] = high[i] > high[i +/- 1 .. i +/- n]
isLowerFractal[i] = low[i]  < low [i +/- 1 .. i +/- n]
```

A fractal at index `i` is "confirmed" `n` bars later (at index
`i + n`) because the right-side neighbours must be observed before
the strict-max / strict-min test can be evaluated. The primitive
renders one scatter marker per confirmed pivot at the bar's `high`
(upper) or `low` (lower), optionally with a price label.

## Defaults

| Knob              | Value     |
|-------------------|-----------|
| fractalLookback   | 2         |
| markerRadius      | 5         |
| priceColor        | `#2563eb` |
| upperPivotColor   | `#ef4444` |
| lowerPivotColor   | `#22c55e` |
| showLabels        | `false`   |

## Bit-exact anchors

- `CONST high = low = K`: strict-max / strict-min tests fail on flat
  sequences, so `upperPivotCount = lowerPivotCount = 0`.
- `LINEAR UP / LINEAR DOWN`: monotonic series also yield no fractals.
- `SAWTOOTH` with peaks at fixed `peakK` and troughs at fixed
  `troughL`, period 4 (peak/mid/trough/mid/peak/...): upper pivots
  fire at indices `{2, 6, 10, ...}`, lower pivots at `{4, 8, 12,
  ...}`. For 20 bars the pivot counts are deterministic
  (`upper = lower = 4`); `pivotValue` equals `peakK` at upper pivots
  and `troughL` at lower pivots, bit-exact (IEEE 754 reads back the
  same constants we wrote in). Verified across `(peakK, troughL,
  lookback)` sweeps.

## Surface

Single-panel primitive surface:

- `ChartLineFractalPivotPoint = { x, high, low, close }`
- `runLineFractalPivot(data, options?) -> Run` with
  `ok = series.length >= 2 * fractalLookback + 1`.
- `computeLineFractalPivotLayout({ data, width?, height?, padding?, fractalLookback? }) -> Layout`.
  Single panel; y-axis auto-fits across `high/low`.
- `describeLineFractalPivotChart(data, opts?) -> string` for the ARIA
  description.
- `<ChartLineFractalPivot />` `forwardRef<HTMLDivElement>` with three
  legend toggles (`price`, `upper-pivots`, `lower-pivots`), config
  badge with live pivot counts, optional price labels above/below
  markers, and a tooltip surfacing
  `close / high / low / pivotKind / pivotValue`. Live pivot counts
  also exposed as `data-upper-pivots` / `data-lower-pivots`
  attributes for selector-driven assertions.

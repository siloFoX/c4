## Summary

New `ChartLineTrendTrigger` UI primitive in
`web/src/components/ui/chart-line-trend-trigger.tsx` -- a pure-SVG
two-panel line chart with a **Trend Trigger Factor** panel comparing
buying power against selling power across two lookbacks. The top panel
plots the bar midpoint; the bottom panel plots the TTF with two
horizontal +/- trigger reference lines.

## The computation

M. H. Pee's Trend Trigger Factor pits buying power against selling power
across two adjacent lookback windows. `ChartLineTrendTriggerPoint`
carries `{x, high, low}`.

- `computeLineTrendTrigger(bars, period)` -- per bar past the warm-up
  (i >= 2*period - 1):
  - `recentHigh / recentLow` = max / min of bars `[i-period+1 .. i]`.
  - `priorHigh / priorLow`  = max / min of bars `[i-2*period+1 .. i-period]`.
  - `BP = recentHigh - priorLow`, `SP = priorHigh - recentLow`.
  - `TTF = 100 * (BP - SP) / ((BP + SP) / 2)`, null when the denominator
    is zero (a flat market).
- `classifyLineTrendTriggerZone(ttf, triggerLevel)` -- marks a bar
  **up** at or above the trigger level, **down** at or below the
  negative trigger level, **neutral** between them, **none** for null.

## Worked fixture

The TTF is pure rational arithmetic (max / min / subtraction / scaled
ratio), so an integer fixture computes to an exact integer TTF.

A **strictly rising ramp** (high = low = close = 1, 2, ..., 10) at
period 3 gives at every defined bar `BP = recentHigh - priorLow = 5`
(the full extent across both windows) and `SP = priorHigh - recentLow =
-1` (the prior window's max sits one bar below the recent window's
min). The TTF is `100 * (5 - (-1)) / ((5 + -1) / 2) = 100 * 6 / 2 =
300` -- asserted with `toEqual([null, null, null, null, null, 300, 300,
300, 300, 300])`. A **strictly falling ramp** mirrors it to -300. A
**constant series** has every BP and SP equal to zero, so the
denominator is zero and the TTF is null on every defined bar. The
zone classifier is checked at the trigger level, past it and between
the levels.

## API

`data` (a `ChartLineTrendTriggerPoint[]`), `period` (default 15),
`triggerLevel` (default 100), `pricePanelRatio` / `gap`,
sizing/format/a11y props, `strokeWidth` / `dotRadius`, the colour
props, the toggle flags (`showAxis` / `showGrid` / `showDots` default
false / `showTtf` / `showZeroLine` / `showTriggers` / `showMarkers` /
`showTooltip` / `showConfigBadge` / `showLegend` / `animate`),
controlled + uncontrolled `hiddenSeries` (keys `price` / `ttf`),
`formatValue` / `formatX`, `onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineTrendTriggerFinitePoints`,
`normalizeLineTrendTriggerPeriod`, `normalizeLineTrendTriggerLevel`,
`computeLineTrendTrigger`, `classifyLineTrendTriggerZone`,
`runLineTrendTrigger`, `computeLineTrendTriggerLayout`,
`describeLineTrendTriggerChart`, plus the
`DEFAULT_CHART_LINE_TREND_TRIGGER_*` constants and the
`ChartLineTrendTriggerZone` / `ChartLineTrendTriggerComputed` types.
Every exported name carries a `LineTrendTrigger` infix so the
`export *` barrel line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen
`data-section="chart-line-trend-trigger-aria-desc"` description span;
the midpoint line, the TTF line and every dot / marker are
`role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-period`, `data-trigger-level`,
`data-ttf-final`, `data-up-count`, `data-down-count`,
`data-neutral-count`, `data-total-points` and `data-animate`.

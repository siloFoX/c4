## Summary

New `ChartLineLsma` UI primitive at
`web/src/components/ui/chart-line-lsma.tsx` -- a pure-SVG line
chart with a Least Squares Moving Average overlay tracing the
endpoint of a rolling linear regression.

## Indicator math

For each window of the last `period` prices, fit an ordinary
least-squares straight line `y = a + b*x` over the relative
x-indices `0 .. period-1`, then read the line at its newest end
`x = period - 1`:

```
meanX = (period - 1) / 2
meanY = mean of the window
Sxx   = period * (period^2 - 1) / 12      (sum of (x - meanX)^2)
Sxy   = sum(x*y) - meanX * sum(y)         (sum of (x-meanX)(y-meanY))
slope = Sxy / Sxx                          (the regression slope b)
LSMA  = meanY + slope * meanX
```

The endpoint `a + b*(period-1)` simplifies to `meanY + slope*meanX`
because `(period-1) - meanX === meanX`. Because the regression
line is read at its **endpoint** rather than averaged across the
whole window, the LSMA tracks price more closely and turns sooner
than a same-length simple moving average, while the linear fit
filters single-bar noise. The LSMA is defined from index
`period - 1` onward. `computeLineLsmaSlope` exposes `slope` --
the local linear trend per bar. For `period === 1` the slope is
undefined (`Sxx` is zero) and defaults to 0, so the endpoint is
the lone value (the identity). For `period === 2` two points fix
an exact line, so the endpoint is always the latest value.

### Worked fixture

`values = [2, 4, 6, 14, 10, 12]`, `period 3`. For a window
`[y0, y1, y2]` the endpoint is `mean + slope` with
`slope = (y2 - y0) / 2`:

```
W@2 = [2,4,6]    -> mean 4,  slope (6-2)/2=2,   lsma 6    (linear -> on)
W@3 = [4,6,14]   -> mean 8,  slope (14-4)/2=5,  lsma 13   (concave up -> above)
W@4 = [6,14,10]  -> mean 10, slope (10-6)/2=2,  lsma 12   (concave down -> below)
W@5 = [14,10,12] -> mean 12, slope (12-14)/2=-1, lsma 11  (concave up -> above)
lsma  = [null, null, 6, 13, 12, 11]
slope = [null, null, 2, 5,  2,  -1]
```

The window `W@2 = [2,4,6]` is perfectly linear, so its regression
endpoint reproduces the price exactly (`lsma 6 == price 6`).
Every value is exact, so the tests assert the series with
`toEqual`. The price-versus-LSMA position works out to
`[on, on, on, above, below, above]` -- aboveCount 2, belowCount 1,
lsmaFinal 11, lsmaMin 6, lsmaMax 13, slopeFinal -1.

A second fixture `RAMP = [1,3,5,7,9,11]` (`y = 1 + 2x`) is a
perfectly linear ramp: `computeLineLsma(RAMP, 4)` reproduces the
series from index 3 (`[null,null,null,7,9,11]`) and the slope is
a constant 2.

## API

- `getLineLsmaFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineLsmaPeriod(period, fallback)` -- coerce to a
  positive integer; non-finite or sub-1 falls back, fractional
  floors.
- `computeLineLsma(values, period)` -- the LSMA series (the
  rolling regression endpoint), null through the warm-up.
- `computeLineLsmaSlope(values, period)` -- the regression slope
  per window, null through the warm-up.
- `runLineLsma(points, options?)` -- sort finite points by `x`,
  compute the series, return `{ series, period, lsma, slope,
  samples, lsmaFinal, lsmaMin, lsmaMax, slopeFinal, aboveCount,
  belowCount, ok }`. Each sample carries the price's `above` /
  `below` / `on` position versus the LSMA plus its window
  `slope`. `ok` is false for fewer than two points.
- `computeLineLsmaLayout(options)` -- project the price and LSMA
  onto one shared panel. Returns the panel, ticks, the price
  path, the LSMA path, and LSMA markers.
- `describeLineLsmaChart(data, options?)` -- ARIA description.

## Component

`ChartLineLsma` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, axis ticks and grid, a
config badge, a two-series legend (Price / LSMA) with toggle, a
hover/focus tooltip exposing the value, LSMA, slope and position,
and a `motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs chart-line-smma / a simple or weighted moving average** --
  the SMA, SMMA and WMA all weight a window into an *average*. The
  LSMA is the *endpoint of a regression line*: it extrapolates the
  fitted linear trend to the current bar, which removes the
  half-window centering lag a plain average carries.
- **vs a plain exponential moving average** -- an EMA folds price
  in at a fixed decay weight. The LSMA fits a fresh straight line
  per window; on a steady trend the LSMA sits exactly on price
  (the regression IS the trend) whereas an EMA still lags.
- **vs chart-line-dema / chart-line-tema** -- those cascade EMAs
  to cancel lag. The LSMA removes lag a different way: by
  projecting a least-squares regression to its endpoint.
- **slope** -- the LSMA is the only member of the moving-average
  family here that also yields a per-bar regression slope, which
  the component surfaces in the tooltip and as `data-slope-final`.

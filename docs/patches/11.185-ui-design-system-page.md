# 11.185 - UI: Design System docs page (v1.11.203)

Inline live demo page that catalogues every UI primitive in
`web/src/components/ui`. Each primitive gets a section with name,
description, a Variants block rendering live instances, and a Code
block with a JSX snippet so engineers can copy/paste the canonical
usage.

Files:
- `web/src/pages/DesignSystem.tsx` (page)
- `web/src/pages/DesignSystem.test.tsx` (vitest + RTL, 6 cases)
- `web/src/pages/registry.ts` (new `design-system` feature, config category, Palette icon)
- `web/src/i18n/en.json` + `web/src/i18n/ko.json` (`feature.designSystem.label` / `.description`)
- `web/package.json` (1.11.202 -> 1.11.203)

## Layout

The page is wrapped in the standard `<PageFrame>` with the title
"Design System" and a reference Alert pointing at the ARPS palette
source-of-truth (`arps-design-system-v1/USAGE.md`).

Primitives are grouped into six categories rendered inside a
`<CollapsibleGroup exclusive={false}>` so the operator can keep
multiple categories open at once:

| Category   | Primitives                                                                                  |
|------------|---------------------------------------------------------------------------------------------|
| Inputs     | Input, Textarea, Select, Switch, NumberInput, SearchBar, TagInput, DatePicker(+Range), Rating |
| Buttons    | Button (variant x size matrix + disabled)                                                   |
| Display    | Chip, Badge, Card/Panel, StatCard, Avatar/AvatarShape, Kbd, Separator, DataList, Timeline, Progress, Spinner (placeholder) |
| Feedback   | Alert (5 variants), EmptyState                                                              |
| Layout     | ScrollArea, HScroll                                                                         |
| Navigation | Breadcrumbs, Stepper, Tooltip, Popover, Dialog, DropdownMenu, FileTree                      |

## Demo block shape

Each `<Demo>` block renders:
- `<h3>` with the primitive name
- one-line description
- "Variants" subsection: a dashed-border preview area with real live
  instances (e.g. for Button, the full 5 variants x 3 sizes matrix
  plus a disabled instance)
- "Code" subsection: a `<CodeBlock language="tsx">` with the canonical
  JSX snippet

A `<MissingPrimitive name="Spinner">` fallback renders a placeholder
note for any primitive that is referenced in the spec but not yet
exported from `components/ui/index.ts`. Today only Spinner is missing.

## Nav integration

The Sidebar in this repo is the worker list, not a Settings group, so
top-level nav lives in `web/src/pages/registry.ts` (the source of truth
for the Features tab in `<AppHeader>`). A new entry is appended:

```ts
{
  id: 'design-system',
  labelKey: 'feature.designSystem.label',
  descriptionKey: 'feature.designSystem.description',
  category: 'config',
  Icon: Palette,
  load: () => import('./DesignSystem'),
}
```

The page is reachable from the Features tab in the top-level
navigation (a `topView` option already wired in `App.tsx`).

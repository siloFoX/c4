## Goal

Dispatch:

- Add `web/src/styles/print.css` -- print-friendly
  layout for History detail, Snapshots list, Audit
  log. Hide nav / header / footer.

`print.css` already exists from 11.196 and already
hides `nav` / `.sidebar` / `.topbar` / `.toast` /
`.dropdown` / `.tooltip` / `.popover` / `.drawer` /
`.command-palette` / `[role="dialog"][data-overlay]`
etc. So the dispatch is realistically asking for
**page-specific** print rules layered on top of the
existing chrome-hiding rules.

This patch ships three `data-print-section` hooks
plus the matching @media-print rules so the three
target pages produce clean printable output.

## Section hooks

| hook                  | element                                       |
| --------------------- | --------------------------------------------- |
| `history-detail`      | `HistoryDetailPane` outer `<Card>`            |
| `snapshots-list`      | `Snapshots.tsx` table-wrapping `<div>`        |
| `audit-log`           | `SpecialistsAuditPanel` outer `<div>`         |

Each hook is a single root element. Multiple hooks
can coexist on a single page (none of the three
share a page today, but the rules are
side-effect-free under the `@media print` gate).

## Print rules (added inside the existing @media print block)

### Generic per-section

```css
[data-print-section] {
  page-break-before: always;
  break-before: page;
  width: 100% !important;
  max-width: none !important;
  border: 0 !important;
  box-shadow: none !important;
  background: #ffffff !important;
  color: #000000 !important;
  padding: 0 !important;
  margin: 0 !important;
}
[data-print-section]:first-of-type {
  page-break-before: auto;
  break-before: auto;
}
```

The `:first-of-type` reset stops the very first
print section on the page from inserting a leading
blank page.

### Screen-only controls inside a section

```css
[data-print-section] [role="tablist"],
[data-print-section] [role="navigation"],
[data-print-section] .pagination,
[data-print-section] [data-pagination],
[data-print-section] button,
[data-print-section] [role="button"]:not([data-print-keep]),
[data-print-section] [data-testid$="-button"],
[data-print-section] [data-testid="copy-button"],
[data-print-section] [data-testid$="-filter"],
[data-print-section] [data-print-hide] {
  display: none !important;
}
```

Operators printing a report want the data, not the
controls. Tab strips, pagination, action buttons,
copy buttons, and filter bars all hide. The
`[data-print-keep]` escape hatch lets a future
component opt back in if needed.

### history-detail

```css
[data-print-section="history-detail"] [data-testid="breadcrumb"],
[data-print-section="history-detail"] [data-testid="avatar"] {
  display: none !important;
}
[data-print-section="history-detail"] section[data-section] {
  page-break-inside: avoid;
  break-inside: avoid;
  margin: 0 0 0.8rem 0 !important;
}
```

The breadcrumb is "History / <worker>" -- in print
the page title already conveys this. The avatar
tile is screen chrome (color + ring) and reproduces
poorly on paper. Each `section[data-section]`
inside the detail Card (Task / Output / Metrics
tabs are rendered as sections in the underlying
Tabs primitive) gets `break-inside: avoid` so
related rows do not split across pages.

### snapshots-list

```css
[data-print-section="snapshots-list"] {
  overflow: visible !important;
}
[data-print-section="snapshots-list"] table {
  table-layout: fixed;
  width: 100% !important;
  font-size: 10pt;
}
[data-print-section="snapshots-list"] th,
[data-print-section="snapshots-list"] td {
  word-break: break-word;
  padding: 4pt 6pt !important;
}
[data-print-section="snapshots-list"] th[scope="col"]:last-child,
[data-print-section="snapshots-list"] td:last-child {
  display: none !important;
}
```

`overflow: visible` overrides the screen wrapper's
`overflow-x-auto` so the table is not clipped on a
portrait sheet. `table-layout: fixed` + word-break
keeps the columns from spilling. The Actions
column is dropped entirely (its only content is
buttons, already hidden by the generic rule).

### audit-log

```css
[data-print-section="audit-log"] {
  background: #ffffff !important;
}
[data-print-section="audit-log"] [aria-expanded] {
  display: none !important;
}
[data-print-section="audit-log"] [data-collapsed] {
  display: block !important;
}
[data-print-section="audit-log"] [data-window-filter],
[data-print-section="audit-log"] [data-export-row] {
  display: none !important;
}
[data-print-section="audit-log"] ol,
[data-print-section="audit-log"] ul {
  list-style: none;
  padding: 0;
  margin: 0;
}
[data-print-section="audit-log"] li {
  page-break-inside: avoid;
  break-inside: avoid;
  border-bottom: 1px solid #cccccc;
  padding: 4pt 0;
}
```

The audit panel's open/closed state is screen-only;
print always expands. The `[data-collapsed]`
escape hatch lets nested screen-only collapse UI
opt into the expanded layout for print without
extra wiring. The window-filter + export buttons
hide; each timeline `<li>` gets `break-inside:
avoid` so a single audit event never splits
across pages.

## Reference

ARPS design tokens (`/root/c4/arps-design-system-v1/tokens.css`)
are not applied to the print sheet -- per the
11.196 contract, print uses black-on-white with
Georgia 12pt for legibility on paper. The section
hooks map to the same surface boundaries the
design system defines for screen (the Card root
for history detail, the table wrapper for
snapshots, the panel root for audit log), so
adopting future design-system primitives only
requires moving the `data-print-section` attribute
along with the root element.

## Out of scope

- **Vite extraction** of the print sheet into a
  separate stylesheet with `media="print"` so
  screen renders never download it. The current
  import pattern via `main.tsx` is intentional
  (the `@media print` gate keeps the screen cost
  to one parse). A follow-up patch could split
  the stylesheet if first-paint regresses.
- **Print previews** via puppeteer / playwright.
  The unit tests assert structural invariants
  rather than rendered visuals; a future patch
  could add a browser-test screenshot for the
  three section hooks.
- **Locale-aware date formatting in print.** The
  headers retain whatever the screen renders.
  TODO 11.346 (locale-aware number / date
  formatters) covers the call-site migration.
- **Header / footer @page rules.** Per @page
  blocks for top-of-page title + bottom-of-page
  page number are widely supported but
  inconsistent across browsers; out of scope for
  this patch.
- **Color-preserving badges in print.** The
  11.196 `.keep-color` opt-in already covers
  this; this patch does not extend the opt-in
  surface.

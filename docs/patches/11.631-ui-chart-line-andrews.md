chart with an Andrews Pitchfork drawn from three pivot points.

## Indicator math

An Andrews Pitchfork is built from three pivot points
`p1`, `p2`, `p3`:

```
midpoint M = ( (p2.x+p3.x)/2 , (p2.value+p3.value)/2 )
median slope = (M.value - p1.value) / (M.x - p1.x)
```

The **median line** runs from `p1` through `M`. The **upper** and
**lower** tines are two lines parallel to the median (same
slope) passing through `p2` and `p3`. A line through `(ax, av)`
with slope `s` has intercept `av - s*ax`; the tine with the
larger intercept is the upper line. Because `M` is the average
of `p2` and `p3`, the median's intercept is exactly the average
of the two tine intercepts -- the median sits midway between the
tines. `computeLineAndrewsFork` returns `null` when a pivot
coordinate is not finite, or when `M.x === p1.x` (a vertical,
slope-less median).

### Worked fixture

`p1 (0,10)`, `p2 (3,42)`, `p3 (7,18)`:

```
M = ( (3+7)/2 , (42+18)/2 ) = (5, 30)
median slope = (30 - 10) / (5 - 0) = 4
intercepts -> p2 tine: 42 - 4*3 = 30   (upper)
              median:  10 - 4*0 = 10
              p3 tine: 18 - 4*7 = -10  (lower)
```

The median intercept 10 is exactly `(30 + -10) / 2`. The median
line is `value = 10 + 4x`; the price fixture
`[12,13,20,21,26,33,32,40,41]` straddles it -- 4 bars above,
4 below, 1 on. Projected across `x 0..8` the three lines run
from start values `[30,10,-10]` to end values `[62,42,22]`.

## API

- `getLineAndrewsFinitePoints(points)` -- drop price points with
  a non-finite `x` or `value`.
- `computeLineAndrewsFork(pivots)` -- the pitchfork from a
  `{ p1, p2, p3 }` object: `{ slope, midpoint, lines }` where
  `lines` is `[upper, median, lower]`, each with an `id`,
  `label`, `slope`, anchor coordinates and `intercept`. `null`
  for a non-finite pivot or a vertical median.
- `runLineAndrews(points, options?)` -- sort finite points by
  `x`, build the fork from `options.pivots`, return
  `{ series, pivots, fork, samples, aboveCount, belowCount, ok }`.
  Each sample is classified `above` / `below` / `on` the median.
  `ok` is false for fewer than two points or invalid pivots.
- `computeLineAndrewsLayout(options)` -- project the price path,
  the three fork lines across the full x range, and a marker per
  pivot.
- `describeLineAndrewsChart(data, options?)` -- ARIA description.

## Component

`ChartLineAndrews` is a `forwardRef` region (`role="region"`)
with an ARIA-linked description, a `role="img"` SVG, axis ticks
and grid, a config badge, a three-series legend (Price /
Pitchfork / Pivots) with toggle, a hover/focus tooltip on the
price dots, and a `motion-safe:animate-fade-in` entrance. It
consumes `{ x, value }` points plus a `{ p1, p2, p3 }` pivots
object. The median line is drawn solid and emphasized; the two
tines are dashed; each pivot is a labelled diamond marker. The
fork lines are clipped to the panel with an SVG `clipPath` so a
tine leaving the panel does not stretch the price axis. Every
node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-gann** -- the Gann fan radiates nine rays from
  a single anchor at fixed angular ratios. The pitchfork's three
  lines are mutually *parallel*, and their slope and offsets are
  derived entirely from the geometry of the three chosen pivots.
- **vs chart-line-pivot-points** -- pivot points are horizontal
  levels from a prior-period HLC. The pitchfork lines are sloped
  and anchored to three explicit chart points.
- **vs the moving-average overlays** -- those average a sliding
  window of the price series. The pitchfork is a static
  geometric construction independent of the in-period price.

# Patch 11.950 - UI: chart-line-vfi-zero-cross primitive

Version: 1.11.968

## Summary

Add `web/src/components/ui/chart-line-vfi-zero-cross.tsx` -- a pure-SVG dual
panel line chart primitive that visualizes price alongside Markos Katsanos'
Volume Flow Index (VFI) panel, with explicit cross markers and overlay
arrows at every VFI zero crossover. VFI weights each bar's signed money
flow by its volume, filters small price moves with a standard-deviation
cutoff, caps outlier-volume bars at `maxVolumeCoef * vol_avg`, then
SMA-smooths the resulting normalised flow. Zero crossovers mark
volume-weighted price momentum baseline transition events: a sharper
trigger than raw close-to-close change because volume confirms the move.

## Input shape

The primitive requires **volume-aware input** -- specifically
`{x, close, volume}` -- because the indicator's defining feature is its
volume weighting:

```typescript
{ x: number, close: number, volume: number }
```

`getLineVfiZeroCrossFinitePoints` drops any point whose `x`, `close`, or
`volume` is non-finite.

## Pipeline

1. Log-return per bar (i >= 1, both closes > 0):
   `inter[i] = ln(close[i] / close[i-1])`.
2. Rolling population stdev over `length` rets: `sd[i]`.
3. Cutoff = `coef * sd[i]` (default `coef = 0.2`).
4. Rolling SMA over `length` volumes: `vol_avg[i]`. Vol cap =
   `vol_avg[i] * maxVolumeCoef` (default `maxVolumeCoef = 2.5`).
   `vol_capped[i] = min(volume[i], cap)`.
5. Money flow:
   - `inter[i] >  cutoff[i]` -> `mf[i] = +vol_capped[i]`
   - `inter[i] < -cutoff[i]` -> `mf[i] = -vol_capped[i]`
   - else -> `mf[i] = 0`
6. Normalised ratio: `ratio[i] = mf[i] / vol_avg[i]`
   (vol_avg = 0 -> null).
7. VFI: `vfi[i] = SMA(ratio, length)` at `i >= 2 * length - 1`.
8. `classifyLineVfiZeroCrossRegime(vfi, T)`: null -> none, vfi >= T ->
   bullish, vfi < T -> bearish.
9. `detectLineVfiZeroCrossCrosses(series, vfi, T)`: strict
   `prev <= T && cur > T` for bullish, strict `prev >= T && cur < T`
   for bearish. Boundary equality does not fire.

## Bit-exact anchors

| Pattern                       | VFI (after warmup) | Regime  |
|-------------------------------|--------------------|---------|
| CONST close = K (V const)     | 0                  | bullish |
| GEOMETRIC UP close = K * 1.01^i  | +1              | bullish |
| GEOMETRIC DOWN close = K * 0.99^i| -1              | bearish |

The CONST anchor is tested bit-exactly across K in `{1, 42, 100, 1234}`
(K must be positive for log to be defined). The GEOMETRIC anchors
exploit the fact that a constant ratio `r` yields constant
`inter = ln(r)`, so stdev = 0, cutoff = 0, and every bar resolves to
`mf = +/- vol_capped`, giving `ratio = +/- 1` and `vfi = +/- 1`.

## Layout

* Default dimensions: 720 x 460.
* Two panels: price panel on top (~55% of plot area), VFI panel below.
* VFI panel auto-fits to the observed `[min, max]` range with 10%
  padding on each end, always including the threshold in the displayed
  range.
* Threshold band rendered as a horizontal rule at `y = thresholdY` in
  the VFI panel.
* Cross markers drawn as circles on the VFI panel at `(cx, cyOsc)` and
  as arrows on the price panel at `(cx, cyPrice)`.
* SVG path strings use `.toFixed(2)` for deterministic snapshot tests.

## Accessibility

* Wrapping `<div role="region" aria-label="VFI Zero Cross chart">`.
* `<svg role="img">` with an aria-described-by linked `<desc>` sr-only
  text.
* Each cross marker: `role="graphics-symbol"` + `tabIndex={0}` +
  descriptive `aria-label` ("bullish cross at x=...").
* Legend toggles: `<button role="switch" aria-pressed="...">`.
* Motion-safe fade-in via `motion-safe:animate-fade-in` (opt-out with
  `animate={false}`).

## Tests

* `normalizeLineVfiZeroCrossCoef` -- accepts finite >= 0, fallback on
  negative / non-finite.
* `normalizeLineVfiZeroCrossThreshold` -- accepts any finite, fallback
  on non-finite.

* `applyLineVfiZeroCrossStdev` -- CONST -> 0, alternating +/-1 -> 1.

  `{1, 42, 100, 1234}`, GEOMETRIC UP -> +1, GEOMETRIC DOWN -> -1, null
  warmup before `2 * length - 1`, custom length=5, zero close drops the
  log step gracefully.
* `classifyLineVfiZeroCrossRegime` -- null / boundary / below behaviour.
* `detectLineVfiZeroCrossCrosses` -- bullish / bearish, sweep, null
  gaps, boundary-not-crossed, no-cross stays.

  all-bearish, decline-then-rise bullish cross,

  input, sort by x, sample shape (carries volume + vfi + regime).

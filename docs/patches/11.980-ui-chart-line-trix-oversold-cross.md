## Summary

Adds the `ChartLineTrixOversoldCross` primitive
(`web/src/components/ui/chart-line-trix-oversold-cross.tsx`): a
pure SVG dual-panel chart that visualizes TRIX (triple-smoothed
exponential moving average rate of change) crossings through the
canonical oversold line -0.1 for triple-smoothed momentum oversold
trigger events. Mirror of `chart-line-trix-overbought-cross`
(TODO 11.979): identical TRIX pipeline, threshold flipped from
+0.1 to -0.1. Joins `chart-line-trix-zero-cross` (11.941) to round
out the canonical TRIX threshold layer (overbought + oversold +
zero).

## Pipeline

Same close-only TRIX pipeline as the zero / overbought variants:

1. `ema1 = EMA(close, length)` (SMA-seeded with CONST short-circuit)
2. `ema2 = EMA(ema1, length)`
3. `ema3 = EMA(ema2, length)`
4. TRIX from the rate of change of the third smoothing:
   - `trix[i] = (ema3[i] - ema3[i-1]) / |ema3[i-1]| * 100`
   - When `ema3[i-1] == 0`, short circuit to `trix = 0` (bit-exact
     +0; verified via `Object.is`).
5. Regime classifier (strict `>=`):
   - `trix >= -0.1` -> bullish (above oversold zone)
   - `trix <  -0.1` -> bearish (in oversold zone)
6. Cross detector emits a `bullish` cross when
   `trix[i-1] <= -0.1 && trix[i] > -0.1` (exit from oversold) and
   a `bearish` cross when `trix[i-1] >= -0.1 && trix[i] < -0.1`
   (entry into oversold). Strict inequality on the current side
   ensures the boundary value -0.1 does not double-fire.

## Defaults

- `length = 15` (canonical TRIX window)
- `threshold = -0.1` (canonical oversold line for TRIX %)
- Warmup boundary: bar `3 * length - 2 = 43`, so for a 60-bar run
  the first 43 samples are null and the remaining 17 are populated
  (ema1 valid at 14, ema2 at 28, ema3 at 42, trix at 43).

## Bit-exact anchors

- CONST `close = K` for all `K in {0, 1, 42, 100, 1234}` ->
  `trix[i] === 0` literal (NOT `-0`) for every `i >= 43`. Regime
  classifies as bullish because 0 >= -0.1 -- mirror of the
  overbought variant (where 0 < 0.1 gave bearish).
- LINEAR UP `close = i + 100` -> trix stays positive across the
  test window, well above -0.1 throughout (all bullish, 0
  crosses).
- LINEAR DOWN `close = 200 - i` -> trix stays well below -0.1 the
  entire run (all bearish, 0 crosses because trix jumps from null
  directly to a value well below the threshold).
- CONST K full-run counters under defaults, 60 bars:
  `noneCount = 43`, `bullishCount = 17`, `bearishCount = 0`,
  `crosses.length = 0`.
- `pricePath` and `trixPath` are byte-identical across repeated
  layout calls, with `.toFixed(2)` precision and `M ... L ...`
  command shapes. The trix path uses a single `M` for post-warmup
  data.
- Cross detector boundary: strict inequality (`> T`, `< T`).
- Layout auto-fit: when trix collapses to 0 (CONST case), the
  panel expands DOWN to include threshold = -0.1 with 10% padding
  both sides -> oscMin = -0.11, oscMax = 0.01 -- exact mirror of
  the overbought variant which expanded UP.

  numeric range of the indicator.

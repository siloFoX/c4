# Patch 11.837 -- ChartLineSmaPct primitive (v1.11.855)

## Summary

Add `<ChartLineSmaPct />`, a pure-SVG dual-panel line chart with an
SMA Percent-Change oscillator beneath a price line that also
overlays the underlying SMA on the top panel. Parallel to 11.836
chart-line-ema-pct, swapping the SMA-seeded EMA for a pure rolling
mean.

```
sma[i]    = mean(close[i - length + 1 .. i])
smaPct[i] = sma[i] === 0 ? null : (close[i] - sma[i]) / sma[i] * 100
```

- Warmup: `i < length - 1` -> `null`.
- `null` whenever `sma[i]` is zero (divide-by-zero guard).
- `-0` normalized to `+0` on every arithmetic result.

At the SMA seed index `i = L - 1`, this primitive and the EMA-pct
primitive compute the same value because the EMA helper seeds with
the SMA; downstream they diverge as the EMA recurses and the SMA
keeps rolling.

## SMA helper precision

The rolling-mean helper performs a `winMin === winMax` check across
each window. When all values in the window are identical the rolling
mean returns that value directly instead of `sum / length`, avoiding
1-ULP drift when `length` is not a power of 2 and the constant is
non-dyadic.

## Bit-exact anchors

- `CONST close = K, K != 0`: rolling mean is `K` exactly (min/max
  short-circuit), so `smaPct = (K - K) / K * 100 = 0` bit-exact
  post-warmup. Verified across `(K, length)` sweeps.
- `CONST close = 0`: `sma = 0`, divide-by-zero guard -> `null`.
- `LINEAR UP close = i + 1` at `i = L - 1`: SMA seed
  `sma = (1 + 2 + ... + L) / L = (L + 1) / 2`, `close = L`, so
  `smaPct = (L - (L+1)/2) / ((L+1)/2) * 100 = (L-1)/(L+1) * 100`.
  For `L in {3, 7, 15}` dyadic constants
  `{50, 75, 87.5}` are exact in IEEE 754.
- `LINEAR UP close = i + 1, L = 3`: for `i >= 2` the SMA simplifies
  to `i`, so `smaPct = (i + 1 - i) / i * 100 = 100 / i`. Bit-exact
  at `i in {2, 4, 5, 8, 10, 20, 25}` where `100 / i` is dyadic or
  a simple ratio representable without rounding -> tested values
  `{50, 25, 20, 12.5, 10, 5, 4}`.

## Surface

Standard primitive surface mirroring 11.836:

- `ChartLineSmaPctPoint = { x, close }`
- `runLineSmaPct(data, options?) -> Run` with
  `ok = series.length >= length`.
- `computeLineSmaPctLayout({ data, width?, height?, padding?, panelGap?, length?, bullishThreshold?, bearishThreshold? }) -> Layout`.
  Top panel renders close + SMA overlay; bottom panel renders the
  smaPct oscillator. Bottom y-axis auto-fits but always includes 0.
- `describeLineSmaPctChart(data, opts?) -> string` for the ARIA
  description.
- `<ChartLineSmaPct />` `forwardRef<HTMLDivElement>` with three
  legend toggles (`price`, `sma`, `pct`), config badge, optional
  threshold band + zero-line overlay, markers on cross events, and
  a tooltip surfacing `close / sma / delta / smaPct / zone / cross`.

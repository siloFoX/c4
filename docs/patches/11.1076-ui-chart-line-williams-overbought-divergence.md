# 11.1076 -- UI: chart-line-williams-overbought-divergence primitive (v1.11.1094)

## Scope

Add a pure-SVG dual-panel
`ChartLineWilliamsOverboughtDivergence` component that
renders the close in the top panel and Larry Williams'
Percent R (%R) in the bottom panel, marking bullish
(price down while %R up at overbought levels --
contrarian dip-buy warning, less common) / bearish
(price up while %R down at overbought levels -- **top
reversal warning**) price-vs-%R direction disagreement
(divergence) trigger events with bias coloring derived
from the %R slope at the divergence-entry bar. **Crosses
are gated to the overbought zone**: only fire when %R is
at or above the overboughtLevel (default -20).

Generalises the CCI zone-gated divergence pair
(chart-line-cci-overbought-divergence v1.11.1092 /
chart-line-cci-oversold-divergence v1.11.1093) to the
bounded [-100, 0] Williams %R oscillator. Williams %R is
INVERTED relative to typical oscillators: values near 0
mean the close is at the period high (overbought / top),
values near -100 mean the close is at the period low
(oversold / bottom).

## Williams %R overbought-divergence pipeline

```
HH[i]  = max(high[i-period+1..i])
LL[i]  = min(low[i-period+1..i])
%R[i]  = (HH[i] - close[i]) / (HH[i] - LL[i]) * -100
         (null when HH === LL)

regime :
  'aligned-bullish'   when priceUp   && wrUp
  'aligned-bearish'   when priceDown && wrDown
  'divergent-bullish' when priceDown && wrUp
  'divergent-bearish' when priceUp   && wrDown
                      (PRIMARY: top reversal warning
                       when %R is in overbought zone)
  'none'              otherwise

bullish cross (at overbought) :
  prev regime !== 'divergent-bullish' &&
  cur regime === 'divergent-bullish' &&
  cur %R >= overboughtLevel
bearish cross (at overbought) :
  prev regime !== 'divergent-bearish' &&
  cur regime === 'divergent-bearish' &&
  cur %R >= overboughtLevel

bias : %R[i] vs %R[i-1] -> up/down/flat/none
```

Default `period = 14`, `overboughtLevel = -20` --
canonical Larry Williams (1973) %R tuning + classical
overbought threshold. Overbought zone is -20..0 (close
near the period high); oversold zone is -100..-80 (close
near the period low); -50 is the midline.

## Sibling family

The zone-gated divergence family (now spanning two
indicators):

- **chart-line-cci-overbought-divergence v1.11.1092** --
  CCI top reversal warning, CCI >= +100 gate
- **chart-line-cci-oversold-divergence v1.11.1093** --
  CCI bottom reversal warning, CCI <= -100 gate
- **chart-line-williams-overbought-divergence
  v1.11.1094** (this primitive) -- Williams %R top
  reversal warning, %R >= -20 gate

Other Williams %R siblings:

- chart-line-williams-r -- raw %R oscillator
- chart-line-williams-r-cross / williams-r-cross-sig --
  %R level / signal crosses
- chart-line-williams-mid-cross-sig -- %R vs -50 mid
- chart-line-williams-divergence-cross -- ungated %R
  divergence

Distinct from chart-line-williams-divergence-cross (the
ungated sibling): this primitive filters crosses to only
those that occur while %R is in the overbought zone
(>= -20). Fewer false positives, narrower coverage --
specifically targets the classical "top reversal
warning" use case.

Warmup is `period = 14` for the first %R value. Cross
detection needs the previous bar's regime, so the first
potential cross lands at i = period + 1 = 15.

## Bit-exact anchors

All anchors use HLC input.

- **CONST band** `high = K + 1`, `low = K - 1`,
  `close = K`: HH = K + 1, LL = K - 1, close = K.
  %R = ((K+1) - K) / ((K+1) - (K-1)) * -100 = 1/2 *
  -100 = **-50** (constant midpoint). NOT in the
  overbought zone (-50 < -20). priceFlat, wrFlat ->
  regime `none`. 0 crosses, overboughtCount = 0.
  Verified across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`,
  `close = i`: HH = i + 1, LL = i - period, close = i.
  %R = 1 / (period + 1) * -100. For period = 14:
  **-6.667** (constant). IN the overbought zone
  (-6.667 >= -20; close sits near the period high).
  priceUp, wrFlat -> regime `none`. The gate passes but
  the flat %R produces no divergent state. 0 crosses,
  overboughtCount > 0.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`,
  `close = -i`: HH = -i + period, LL = -i - 1,
  close = -i. %R = period / (period + 1) * -100. For
  period = 14: **-93.333** (constant). NOT in the
  overbought zone (-93.333 < -20; close sits near the
  period low = oversold). priceDown, wrFlat -> regime
  `none`. The gate filters any crosses. 0 crosses,
  overboughtCount = 0.

The bounded [-100, 0] range lets the chart panel
hard-lock to that span with no auto-scaling, and the
CONST/LINEAR anchors all produce clean constant %R
values that exercise the overbought-zone gate from both
sides: LINEAR UP sits inside the zone (close near the
high), CONST and LINEAR DOWN sit outside it.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 5-state regime classifier follows the divergence-
  cross family convention exactly.
- Overbought-zone gate (`curWr >= overboughtLevel`) is
  the key specialisation, mirrored from the CCI
  overbought sibling. Verified by the explicit
  detector-level test that builds the same divergent-
  bearish regime sequence with %R values both above and
  below the threshold.
- Williams %R is INVERTED relative to typical
  oscillators: overbought is near 0 (the top of the
  range), oversold is near -100 (the bottom). The
  `overboughtLevel >= ` gate semantics are correct for
  this orientation (a HIGHER %R, closer to 0, is more
  overbought).
- Zero-range guard returns null when HH === LL (a
  perfectly flat window). Matches pandas-ta / ta-lib
  behaviour.
- The bounded [-100, 0] range is mathematically
  guaranteed (numerator HH - close is in [0, HH - LL],
  so the ratio is in [0, 1] and the * -100 maps it to
  [-100, 0]). The panel hard-locks accordingly.
- Third member of the zone-gated divergence family and
  the FIRST to generalise the pattern beyond CCI --
  confirming the template extends cleanly to any
  bounded oscillator with overbought/oversold zones.
  Sets up obvious future siblings:
  williams-oversold-divergence (mirror, bottom
  reversal), rsi-overbought/oversold-divergence,
  stochastic-overbought/oversold-divergence.

## Cross-tab sync

`useTheme()` now subscribes to the `storage`
event keyed on `THEME_KEY` (`c4.theme`):

```ts
useEffect(() => {
  if (typeof window === 'undefined') return;
  const onStorage = (e: StorageEvent): void => {
    if (e.key !== THEME_KEY) return;
    if (e.newValue === null) {
      setTheme(readTheme());
      return;
    }
    if (isValidTheme(e.newValue)) {
      setTheme(e.newValue);
    }
  };
  window.addEventListener('storage', onStorage);
  return () => window.removeEventListener('storage', onStorage);
}, []);
```

- The browser dispatches `storage` to other
  tabs whenever `localStorage.setItem(...)` /
  `removeItem(...)` / `clear()` runs.
- The hook mirrors `e.newValue` into local
  state; the existing
  `useEffect([theme])` re-applies the
  `<html class="dark">` flip + writes back to
  storage (idempotent, no loop -- the storage
  event is only emitted in OTHER tabs).
- `e.newValue === null` is the `clear()`
  signal; the hook reverts to `readTheme()`'s
  canonical default so the tab does not freeze
  on the stale value.
- Invalid values are silently ignored (an
  extension or manual edit cannot push a
  malformed theme).

Per-page primitives (sidebar mode, detail mode,
top view) already flow through a shared App-level
storage listener; only theme needed the
hook-local treatment because its side effect
(`<html class="dark">`) lives outside React
state.

## Contrast helpers

Pure-math module at `lib/contrast.ts`. No DOM,
no React. Used today by the test harness;
intended for future runtime contrast inspectors
+ design-time audits.

| function | purpose |
| --- | --- |
| `parseColor(input)` | hex / HSL triplet / RGB object -> `{r,g,b}` or null |
| `relativeLuminance(color)` | per-WCAG 2.1, lineariser + canonical weights |
| `contrastRatio(fg, bg)` | `(L1 + 0.05) / (L2 + 0.05)` with max/min |
| `meetsAaContrast(fg, bg, isLargeText?)` | AA threshold (`4.5` / `3`) |
| `meetsAaaContrast(fg, bg, isLargeText?)` | AAA threshold (`7` / `4.5`) |
| `evaluateContrast(fg, bg)` | one-shot report bundle |

Exports `AA_NORMAL` / `AA_LARGE` / `AAA_NORMAL` /
`AAA_LARGE` so adopters reference one canonical
source.

### HSL parser

The c4 design tokens are HSL triplets in
`tokens.css` (e.g. `--text-primary: 220 15%
96%`). The parser accepts `'h s% l%'` with
whitespace tolerance + hue normalisation so a
caller can pass the raw token value through
`var()`-stripped:

```ts
parseColor('220 18% 8%')
// -> ARPS --surface-canvas as { r, g, b }
```

The 8-digit hex variant (`#rrggbbaa`) strips
alpha because the contrast contract is undefined
for translucent foregrounds. Adopters that need
alpha-aware contrast must composite first.

## Tests

`contrast.test.ts` -- 23 cases covering parser
shapes, NaN propagation on invalid input,
black/white pinning at ratio 21 / 1, symmetry,
WCAG reference value tolerance, AA / AAA
threshold crossover, threshold constants.

`use-theme.test.ts` -- 5 new cases:

- Mirrors a theme written by another tab via
  the storage event.
- Ignores storage events keyed on something
  else.
- Ignores storage events carrying an invalid
  theme value.
- Falls back to the canonical default when
  `newValue === null` (localStorage.clear).
- Removes the storage listener on unmount.

40/40 total tests pass under vitest 4.1.5
(23 contrast + 16 use-theme + 1 displayName).
TypeScript clean for the source files;
pre-existing strict-tuple-access warnings in
the legacy portion of `use-theme.test.ts` are
unchanged and out of scope.

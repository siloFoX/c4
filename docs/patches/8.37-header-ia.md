# 8.37 — Header IA + Manager / Worker grouping

Branch: `dgx-spark/header-ia`

## What ships

Two IA fixes the user flagged on 2026-04-20: (a) the C4 logo was
mounted inside the Workers sidebar header instead of the AppHeader,
and (b) the Workers sidebar lumped manager workers together with
regular workers in a flat list.

- The logo + wordmark now live in the AppHeader's left slot
  (claude.ai / Linear / VS Code convention). The Workers sidebar
  drops the inline logo and just labels the section.
- The Workers sidebar partitions workers into **Managers** and
  **Workers** buckets, each with a collapsible header (lucide
  `Crown` / `Wrench`), a count badge, and a primary / muted accent.
- Per-group expand/collapse state persists via two new localStorage
  keys (`c4.workerList.managers.open` / `c4.workerList.workers.open`).
- Manager rows wear a left primary accent border so the role
  distinction stays visible even after a user collapses one bucket.
- Daemon's `GET /api/list` now folds the `tierWorkerMap` into each
  worker so the Web UI can group without a follow-up round-trip and
  without guessing from name patterns.

## Surface area

### `src/daemon.js`

- `/list` route walks `manager.list().workers` and writes
  `w.tier = tierWorkerMap.get(w.name) || 'worker'` onto every entry
  before responding. Pre-existing `tier` callers (CLI `c4 list`) keep
  working unchanged.

### `web/src/types.ts`

- `Worker` interface gains an optional `tier?: 'manager' | 'worker' |
  string`. Pre-8.37 daemons may omit the field; the UI defaults to
  `'worker'` on read.

### `web/src/components/WorkerList.tsx`

- New `groupOf(w)` helper returns `'manager'` when `w.tier ===
  'manager'`, otherwise falls back to a name-pattern heuristic
  (`c4-mgr-*`, `auto-mgr-*`, `*-mgr-*`) for compatibility with
  pre-8.37 daemons.
- New `GroupHeader` component: chevron + lucide icon (Crown for
  managers, Wrench for workers) + label + count badge. Clickable
  with `aria-expanded` + `aria-controls` for screen reader hints.
- Workers list is rendered as two stacked groups. Each group only
  renders when its bucket is non-empty so users with a single-tier
  workforce don't see a permanently empty header.
- Per-group sorting (`localeCompare` by name) keeps order stable
  across SSE bumps.
- Manager rows wear a left primary accent (`border-l-2
  border-l-primary/40`); regular worker rows are unstyled.
- Per-group expand/collapse state hydrates from localStorage on
  mount and writes through on toggle. New keys:
  `c4.workerList.managers.open` and `c4.workerList.workers.open`
  (each stored as `'1'` / `'0'`).

### `web/src/components/layout/AppHeader.tsx`

- Logo + wordmark rendered in the left slot next to the mobile
  hamburger.

### `web/src/components/layout/Sidebar.tsx`

- `<img src="/logo.svg" />` removed from the sidebar header — the
  section header now reads just "Workers".

## Tests

`tests/header-ia.test.js` — 14 source-grep assertions across 5
suites:

1. AppHeader renders `/logo.svg` and keeps the C4 Dashboard wordmark.
2. Sidebar no longer renders `/logo.svg` and keeps the Workers label.
3. `Worker` type carries the optional `tier` field.
4. Daemon `/list` route folds `tierWorkerMap` into each worker.
5. `WorkerList` exposes `groupOf` + name-pattern fallback +
   `GroupHeader` (Crown / Wrench) + per-group open state +
   localeCompare sort + manager primary accent + empty-bucket guard
   + aria-expanded / aria-controls.

Full suite: 116 / 117 pass — the lone failure is the pre-existing
`file-transfer` test PR #24 fixes (origin's hardcoded
`/root/project` path). `npm --prefix web run build` holds the 47 TS
error baseline.

## Test plan (manual)

1. `c4 daemon start && cd web && npm run dev`
2. Open the Workers tab. AppHeader's left slot now shows the C4
   logo next to the wordmark; the sidebar header just reads
   "Workers".
3. Spawn a manager via `c4 auto "..."` (or `c4 new c4-mgr-test
   --tier manager`). Spawn a couple of regular workers.
4. The sidebar shows two groups: **Managers** with a Crown icon and
   primary accent, **Workers** with a Wrench icon and muted accent.
   Each group header carries a count badge.
5. Manager rows show a left primary border; worker rows don't.
6. Click each group header — section toggles. Reload — state
   persists.
7. `c4 close <manager>` until the Managers bucket is empty —
   the Managers section disappears entirely.

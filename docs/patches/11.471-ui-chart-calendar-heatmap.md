## Summary

Add a new `<ChartCalendarHeatmap>` UI primitive in
`web/src/components/ui/chart-calendar-heatmap.tsx` --
pure-SVG GitHub-style year calendar heatmap. Each day is
a small colored square; columns are weeks; rows are
days-of-the-week. Color encodes activity intensity via a
5-step palette plus an "empty" color for zero-value days.
Month labels render above the grid at the first column
that touches each new month; a sparse day-of-week label
axis renders on the left (3 labels by default to match
the GitHub look). Distinct from `<ChartHeatmap>` (11.442)
which is a generic 2-D matrix heatmap -- this primitive
is *time-aware*: it knows about ISO dates, weeks, and
months, so a caller can hand it `[{date, value}]` and get
a year layout for free.

## API

```ts
type ChartCalendarHeatmapWeekStart = 0 | 1;

interface ChartCalendarHeatmapValue {
  date: string;  // YYYY-MM-DD
  value: number;
}

interface ChartCalendarHeatmapProps {
  values: readonly ChartCalendarHeatmapValue[];
  startDate?: string;
  endDate?: string;
  year?: number;
  weekStart?: ChartCalendarHeatmapWeekStart; // 0 = Sun, 1 = Mon
  cellSize?: number;          // default 12
  cellGap?: number;           // default 3
  topPad?: number;            // default 18
  leftPad?: number;           // default 28
  rightPad?: number;          // default 8
  bottomPad?: number;         // default 22
  palette?: readonly string[];     // default 5-step green
  emptyColor?: string;             // default '#e5e7eb'
  maxValue?: number;
  showMonthLabels?: boolean;       // default true
  showDowLabels?: boolean;         // default true
  showLegend?: boolean;            // default true
  showTooltip?: boolean;           // default true
  animate?: boolean;               // default true
  monthLabels?: readonly string[]; // default Jan...Dec
  dowLabels?: readonly string[];   // default Sun...Sat
  className?: string;
  ariaLabel?: string;              // default 'Calendar heatmap'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  formatDate?: (date: string) => string;
  onCellClick?: (cell: ChartCalendarHeatmapCell) => void;
  style?: CSSProperties;
}
```

## Behaviour

- Range resolution: if both `startDate` and `endDate`
  are present, they are used verbatim; otherwise the
  `year` prop (or the current UTC year) materialises
  Jan 1 -> Dec 31 via `getYearRange`.
- Layout: `computeCalendarHeatmap` pads the start of the
  grid back to the previous `weekStart` so that every
  day of the requested range falls on the correct row.
  Cells outside the requested range (the pre-pad days
  in the first column) are not emitted.
- Color: 5-step palette by default. Zero / non-finite /
  negative values paint with the `emptyColor`. Non-zero
  values map to `ceil(ratio * paletteLength)` so the
  smallest positive value still gets palette[0] rather
  than the empty color. Ratios over 1 clamp to the top
  step.
- Tooltip: hover or focus on a cell opens a tooltip
  with the formatted date + formatted value.
- ARIA / keyboard: every cell is
  `role="graphics-symbol"` + `tabIndex={0}` so the
  whole year is keyboard-tabbable.
- Mount animation:
  `motion-safe:animate-fade-in` per cell.
  `animate=false` removes the class.
- Default ARIA description: `"Calendar heatmap from
  <start> to <end>. N active days, total <sum>, peak
  <peak>."` -- invalid range returns `"No data"`;
  `ariaDescription` overrides.

```ts
toISODate(d): string
parseISODate(s): Date | null
addDays(d, n): Date
getYearRange(year): { start, end }
getCalendarHeatmapColor(value, max, palette?, emptyColor?): { color, level }
computeCalendarHeatmap(input): {
  weeks, cells, monthLabels, width, height, max,
  weekStart, cellSize, cellGap, topPad, leftPad
}
describeCalendarHeatmap(values, startDate, endDate, formatValue?): string
```

- `parseISODate` rejects malformed + impossible dates
  (e.g. `2024-02-30`).
- `getCalendarHeatmapColor` returns `{ color, level }`;
  level 0 = empty; 1..palette.length = palette step.
- `computeCalendarHeatmap` returns `{ cells, weeks,
  monthLabels, width, height, max }`; invalid /
  inverted range -> empty result.

## Pairs with existing primitives

- `<ChartHeatmap>` (11.442) -- generic 2-D matrix
  heatmap; siblings; calendar-heatmap is the
  time-aware variant.
- `<ChartSparkline>` (11.443) -- compact daily
  sibling for a single value series.

## Out of scope

- Multi-year stacking in one chart -- host stacks
  instances (one per year) instead.
- Localised month / day names -- host passes
  `monthLabels` / `dowLabels` props.
- Click-drag range selection -- v1 supports per-cell
  click + hover only; host owns drag selection.
- Animated reflow on data change (snap v1; mount
  fade only).

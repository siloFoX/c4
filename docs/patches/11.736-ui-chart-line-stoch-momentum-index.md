## Summary

`web/src/components/ui/chart-line-stoch-momentum-index.tsx` ships
`<ChartLineStochMomentumIndex>`, a pure-SVG dual-panel chart with a
**Stochastic Momentum Index** panel (William Blau) from the
double-smoothed midpoint distance scaled by the half-range. The top
panel plots the close; the bottom panel plots the SMI in a fixed
`[-100, 100]` band with a zero line and `+/-threshold`
overbought / oversold reference lines.

## Computation

For each bar `i` with a filled lookback `period`:

```
HH        = max(high over [i - period + 1 .. i])
LL        = min(low  over [i - period + 1 .. i])
midpoint  = (HH + LL) / 2
halfRange = (HH - LL) / 2
distance  = close - midpoint
```

The distance and the halfRange are then smoothed twice with EMAs of
lengths `smooth1` and `smooth2`. The SMI is

```
SMI = 100 * smoothed(distance) / smoothed(halfRange)
```

A zero smoothed halfRange (constant series, no range) leaves the
bar null. Bounded `[-100, 100]`: +100 at a new high, -100 at a new
low, 0 at the midpoint.

## Worked anchors

Three exact-style anchors survive the EMA arithmetic by construction.

- **Constant-series anchor** -- `H = L = C = 5` at every bar.
  `halfRange = 0` everywhere; the smoothed halfRange is also zero;
  the divisor is below the epsilon so the SMI is null at every
  defined bar.
- **CENTERED anchor** -- `H = 10, L = 0, C = 5` at every bar. At
  every defined bar `distance = 5 - 5 = 0` and `halfRange = 5`.
  The EMA of a constant zero is zero (zero arithmetic exact); the
  EMA of a constant 5 is 5 (EMA-of-constant identity). So `SMI =
  100 * 0 / 5 = 0` bit-exact at every defined bar.
- **ALL-HIGH anchor** -- `H = 10 + i, L = 0, C = 10 + i` at every
  bar. The current bar is always the highest, so `HH = close`,
  `midpoint = (close + LL) / 2`, `distance = close - midpoint =
  (close - LL) / 2 = halfRange`. With `distance = halfRange` at
  every bar, the EMAs of the two are identical, so the ratio is
  exactly 1 and `SMI = 100` bit-exact.
- **ALL-LOW anchor** -- mirror: `distance = -halfRange` at every
  bar, `SMI = -100` bit-exact.

The `[-100, 100]` bound on a wave fixture and the threshold
classifier (at-or-above the threshold -> overbought, at-or-below
the negative threshold -> oversold) are tested directly.

## Distinct from neighbouring primitives

- vs `<ChartLineStochastic>` -- the standard Stochastic measures
  `100 * (close - LL) / (HH - LL)` in `[0, 100]`; the SMI measures
  the close-to-MIDPOINT distance over the HALF-range, double-
  smoothed, in `[-100, 100]`. The SMI is centred on zero with
  symmetric overbought / oversold zones.
- vs `<ChartLineRsi>` -- the RSI is a bar-to-bar change rate; the
  SMI is a level-versus-range reading like the Stochastic but
  centred and smoothed.
- vs `<ChartLineErgodic>` -- the Ergodic Oscillator is a triple-
  smoothed momentum ratio (close-to-close momentum / its absolute
  value); the SMI is a level-position reading.

## API

`data` -- a single `ChartLineStochMomentumIndexPoint[]`
(`{x, high, low, close}`). `period` (default 10, integer >= 1).
`smooth1` / `smooth2` (defaults 3, integer >= 1). `threshold`
(default 40, in `(0, 100]`). Standard sizing / colour / a11y props
plus `pricePanelRatio` (default 0.55, clamped to `[0.3, 0.8]`),
`gap`, `strokeWidth`, `dotRadius`, `tickCount`. Toggle flags:
`showAxis`, `showGrid`, `showDots` (default false), `showSmi`,
`showZeroLine`, `showThresholdLines`, `showMarkers`, `showTooltip`,
`showConfigBadge`, `showLegend`, `animate`. Controlled and
uncontrolled visibility via `hiddenSeries` (`'price' | 'smi'`).
Callbacks: `onPointClick({point})`,
`onSeriesToggle({seriesId, hidden})`. Format functions:
`formatValue`, `formatX`. ARIA: `ariaLabel`, `ariaDescription`.

## Pure helpers

- `DEFAULT_CHART_LINE_STOCH_MOMENTUM_INDEX_*` -- defaults.
- `CHART_LINE_STOCH_MOMENTUM_INDEX_EPSILON` -- 1e-12 (the
  smoothed-halfRange singular guard).
- `getLineStochMomentumIndexFinitePoints` -- keep only points with
  finite OHLC and `high >= low`.
- `normalizeLineStochMomentumIndexPeriod(period, fallback)` --
  integer >= 1; else fallback.
- `normalizeLineStochMomentumIndexSmooth(smooth, fallback)` --
  integer >= 1; else fallback.
- `normalizeLineStochMomentumIndexThreshold(threshold, fallback)`
  -- finite in `(0, 100]`; else fallback.
- `computeLineStochMomentumIndexEma(values, length)` -- EMA over a
  nullable series; `alpha = 2 / (length + 1)`; null inputs
  propagate.
- `computeLineStochMomentumIndexLegs(bars, period)` --
  `{distance, halfRange}` per bar; warm-up null.
- `computeLineStochMomentumIndex(bars, period, smooth1, smooth2)`
  -- the double-smoothed SMI per bar; null when smoothed
  halfRange is below the epsilon.
- `classifyLineStochMomentumIndexZone(value, threshold)` --
  `'overbought' | 'neutral' | 'oversold' | 'none'`.
- `runLineStochMomentumIndex(points, options?)` --
  `{series, period, smooth1, smooth2, threshold, smi, samples,
  smiFinal, overboughtCount, oversoldCount, neutralCount, ok}`.
- `computeLineStochMomentumIndexLayout({data, period, smooth1,
  smooth2, threshold, ...})` -- two panels with the close on top,
  the SMI below in a fixed `[-110, 110]` band, one marker per
  defined-SMI bar coloured by zone, plus `zeroY`,
  `upperThresholdY`, `lowerThresholdY` for the reference lines.
- `describeLineStochMomentumIndexChart(data, options?)` -- ARIA
  summary mentioning the indicator, the period, the smoothings,
  the bound and the zone counts.

## a11y / data hooks

Root `<div role="region">` with `aria-label` + `aria-describedby`; the
SVG is `role="img"` with the same label; an sr-only
`data-section="chart-line-stoch-momentum-index-aria-desc"` span
carries the description. The close line, every price dot, the SMI
line and every marker carry `role="graphics-symbol"` and
`tabIndex={0}`. Every node carries a `data-section`. Root mirrors
`data-period`, `data-smooth1`, `data-smooth2`, `data-threshold`,
`data-smi-final`, `data-overbought-count`, `data-oversold-count`,
`data-neutral-count`, `data-total-points`, `data-animate`,
`data-empty`. Grid / axis / panel-label nodes carry
`data-panel="price"|"smi"`. Threshold lines carry
`data-direction="upper"|"lower"`. Markers expose `data-zone` and
`data-smi`.

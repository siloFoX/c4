## Goal

Dispatch:

- Build `web/src/lib/toaster.ts` --
  programmatic toast API
  (`toast.success` / `.error` / `.info`
  / `.warning` / `.promise`) consumed by
  the Toast primitive, with queue
  management, auto-dismiss, action
  button support.

The existing `<ToastProvider>`
(`components/ui/toast.tsx`, 11.280) is
hook-based -- callers need a React
context to push a toast. This patch
ships the parallel non-react surface
that any module can import directly.

## API

```ts
import { toast } from '@/lib/toaster';

toast.success('Saved');
toast.error('Failed to save', {
  durationMs: 10_000,
  action: { label: 'Retry', onClick: retry },
});
toast.info('Queued');
toast.warning('Disk almost full');

const id = toast.info('Loading...', { durationMs: Infinity });
toast.dismiss(id);

await toast.promise(savePayload(), {
  loading: 'Saving...',
  success: (saved) => `Saved ${saved.id}`,
  error: (err) => `Failed: ${(err as Error).message}`,
});
```

### Surface

```ts
export interface ToastApi {
  success(message: ReactNode, opts?: ToastOptions): number;
  error(message: ReactNode, opts?: ToastOptions): number;
  info(message: ReactNode, opts?: ToastOptions): number;
  warning(message: ReactNode, opts?: ToastOptions): number;
  promise<T>(p: Promise<T>, messages: ToastPromiseMessages<T>): Promise<T>;
  dismiss(id: number): void;
  clear(): void;
}

export interface ToastOptions {
  durationMs?: number;
  action?: ToastAction;
  description?: ReactNode;
  id?: number;
}

export interface ToastAction {
  label: string;
  onClick: () => void;
}

export interface ToastPromiseMessages<T> {
  loading: ReactNode;
  success: ReactNode | ((data: T) => ReactNode);
  error: ReactNode | ((err: unknown) => ReactNode);
  durationMs?: number;
}
```

## Behaviour

### Queue management

Entries live in a module-level array.
`push(kind, message, opts?)` returns a
numeric id assigned monotonically from
`now() * 1000 + counter`, so rapid calls
within the same millisecond never
collide.

```ts
const id = toast.info('Loading');
// later
toast.success('Done', { id }); // replace in place
```

ID reuse is the primitive `promise()`
uses to morph the loading entry into
its final success / error variant
without re-stacking.

### Auto-dismiss

Resolved at push:

| `opts.durationMs`           | timer scheduled? |
| --- | --- |
| `undefined` + kind=error    | yes (8000ms) |
| `undefined` + other kinds   | yes (`defaultDurationMs`, default 5000) |
| finite positive number      | yes (use that) |
| `Infinity`                  | no (sticky) |
| `0` / negative              | no (sticky) |

Sticky entries persist until `dismiss`
/ `clear`. The error default is
intentionally longer so a busy operator
does not miss a fail-state.

Timer host is injectable for tests:

```ts
const store = createToaster({
  now: () => 12345,
  setTimer: (cb, ms) => setTimeout(cb, ms),
  clearTimer: (h) => clearTimeout(h),
});
```

### Visible limit

`visibleLimit` defaults to 100; floor 1.
When `push` would exceed the cap, the
oldest entries drop and their timers
cancel (FIFO overflow).

### Subscribers

```ts
const unsubscribe = subscribeToaster((entries) => {
  // fired on subscribe + every mutation
});
```

The host receives the new entries array
on every change. Hosts mounted /
unmounted (HMR, dialog-scoped provider)
unsubscribe cleanly without resetting
the store.

### `toast.promise(p, messages)`

```
1. push 'info' loading toast, sticky
2. await p
3a. resolve -> push 'success' with same id
3b. reject  -> push 'error'   with same id
4. return data / rethrow
```

Sonner-compatible. The host renders the
same DOM node morphing through the
three kinds; nothing re-mounts.

## Out of scope (deferred)

- Rendering host. A `<Toaster>`
  component that subscribes via
  `subscribeToaster` and renders into
  a portal belongs in a follow-on (the
  visual layer in 11.280 already
  covers the canonical look; this
  patch is data-layer only).
- Position config (top-right /
  bottom-center). Belongs in the
  render host.
- `toast.custom(node)` for arbitrary
  render slot. Belongs in the render
  host signature.
- Cross-tab sync. Sonner / hot-toast
  do not sync across tabs by default;
  per-tab is canonical.
- React-router clear on navigation.
  Host can opt in cheaply.
- Mass migration of legacy
  `useToast()` call sites.

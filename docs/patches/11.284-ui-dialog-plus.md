# 11.284 -- Dialog primitive: variants + body scroll lock (v1.11.302)

Component-scope-only enhancement of the existing `Dialog`
primitive. No daemon-side change and no `c4` CLI surface change.
The default surface is preserved byte-for-byte; new affordances
opt in via props.

## Why

Three pain points across the modal layer:

1. Operators interacting with destructive modals (delete a
   worker, drop a snapshot) had no visual distinction at the
   primitive layer -- callers either rendered their own
   destructive-flavoured border via `className` (fragmenting the
   surface vocabulary) or used the bespoke `ConfirmDialog`
   component (which reimplemented the backdrop / panel /
   focus-trap loop instead of inheriting from Dialog).
2. The underlying page was free to scroll behind an open modal,
   creating the "tap outside, scroll past the modal" footgun on
   mobile. Patch 11.284 locks `document.body.style.overflow`
   while open so the operator's scroll gestures stay inside the
   modal.
3. The header had a single `title` slot. Modals with body
   context (a confirmation paragraph, a destructive preview
   summary) had to render that copy as the first child, missing
   the `aria-describedby` plumbing that SR users rely on to hear
   the body context after the title.

## What changed

### `web/src/components/ui/dialog.tsx`

- New `variant: 'default' | 'destructive' | 'confirmation'` prop.
  `default` keeps `border-border` byte-for-byte. `destructive`
  swaps in `border-destructive/40` + an `AlertTriangle` glyph in
  destructive palette. `confirmation` uses `border-warning/40` +
  a `HelpCircle` glyph in warning palette.
- New `icon?: ReactNode | false` prop. `undefined` falls back to
  the variant default. `false` suppresses the auto-icon. Any
  ReactNode replaces verbatim.
- New `description?: ReactNode` prop. Header subtitle slot wired
  through `aria-describedby` so SR users hear the body context
  after the title. When supplied without a title, mounts
  standalone with its own id.
- New `lockBodyScroll?: boolean` prop (default true). Toggles
  `document.body.style.overflow="hidden"` while open. The
  previous overflow value is saved and restored on cleanup so a
  host page with its own overflow lock is left untouched.
- New `closeOnBackdropClick?: boolean` prop (default true). Opt
  out so destructive flows require the explicit Cancel button.
- New `closeOnEsc?: boolean` prop (default true). Pairs with the
  existing `useFocusTrap` onEscape hook.
- Data-attribute selectors:
  `data-section="dialog-backdrop"` on the scrim,
  `data-section="dialog"` + `data-variant` on the panel,
  `data-section="dialog-body"` on the children wrapper,
  `data-section="dialog-footer"` on the footer row.

### Adopted

- Primitive enhancement only this commit. Existing adopters
  (`AttachModal`, `KeyboardShortcutsModal`, `NewChatModal`,
  `Dialog` demos) inherit the new defaults -- body scroll lock
  is on now, and the additional data selectors are available
  for e2e without any caller change.

## Deferred (dispatch follow-ups)

- `ConfirmDialog` migration: the existing component owns its
  own preview block + per-button refs + initial-focus selector
  that do not map cleanly to the new Dialog slots today.
  Migrating means either threading those props through the
  primitive (bloating the surface) or rewriting ConfirmDialog
  as a thin composition over the new
  `<Dialog variant="destructive" description>` + footer slot.
  Tracked as a follow-up so the migration can land with its
  own test refresh.

## Tests

- `web/src/components/ui/dialog.test.tsx` -- 11 new vitest
  cases:
  - default variant uses `border-border`.
  - destructive variant uses `border-destructive/40` + renders
    the AlertTriangle svg.
  - confirmation variant uses `border-warning/40` + renders
    the HelpCircle svg.
  - `icon={false}` suppresses the auto-icon.
  - `description` renders + plumbs `aria-describedby`.
  - Body scroll lock toggles `document.body.style.overflow`
    and restores the previous value on close.
  - `lockBodyScroll={false}` leaves overflow untouched.
  - `closeOnBackdropClick={false}` ignores backdrop clicks.
  - Default `closeOnBackdropClick` closes on backdrop click.
  - `data-section` selectors on backdrop + panel.
  - `data-section` selectors on body + footer.
  All 27 (16 original + 11 new) green.
- Dependent suites: AttachModal + KeyboardShortcutsModal
  unchanged. NewChatModal carries one pre-existing focus
  failure unrelated to this commit.

## Versions

- 1.11.301 -> 1.11.302 across root + web package.json + both
  lockfiles. CHANGELOG.md entry under `## [1.11.302]`.

## Notes

- Body scroll lock cleanup runs synchronously on close /
  unmount so a host page can react to the overflow restore in
  the same paint cycle.
- The destructive icon + warning icon are sized identically
  (h-4 w-4) so a future variant swap does not reflow the
  header.

## Summary

Add a new `<ChartMarimekko>` UI primitive in
`web/src/components/ui/chart-marimekko.tsx` -- pure-SVG
Marimekko chart (variable-width stacked bars). Each
column's *width* is proportional to its share of the
grand total; within each column, every visible series
stacks vertically with cell heights proportional to the
series's share of *that column's* total. Optional
percentage labels render inside cells whose pixel area
exceeds `labelMinArea`. Distinct from `<ChartStackedBar>`
(11.468) which uses uniform column widths, and from
`<ChartMosaic>` (11.457) which is similar but typically
treats the data as a contingency table -- Marimekko
emphasises composition across categories where the x
axis is itself a value dimension.

## Behaviour

- Renders `<div role="region">` wrapping
  `<svg role="img">` with the same `aria-label`.
- Column widths: `columnWidth = (columnTotal /
  grandTotal) * widthBudget`, where `widthBudget =
  innerW - (N - 1) * columnGap`. The leftmost column
  starts at `padding` and consecutive columns leave a
  `columnGap` of empty space between them.
- Cell heights: within a column, cells stack
  top-to-bottom with height proportional to `cellValue
  / columnTotal`. `cellGap` is split half above and
  half below the cell so neighbours have a thin
  divider.
- Non-positive / non-finite values are skipped from
  the layout entirely (they contribute neither width
  nor height nor a cell).
- Hidden series (controlled or uncontrolled) drop
  their cells everywhere; column widths reproportion
  themselves to the remaining visible totals.
- Percentage labels paint at the cell center when
  `showPercentageLabels=true` AND the cell's pixel
  area >= `labelMinArea`. Cells too small for a
  readable label automatically hide their text.
- Hover/focus on a cell sets the hovered key
  (`columnId::seriesId`) and opens a tooltip with the
  column + series labels, the formatted value, the
  share within the column, and the share of the
  overall grand total.
- Mount animation: `motion-safe:animate-fade-in` per
  cell group. `animate=false` removes the class.
- Default ARIA description: `"Marimekko chart with
  C columns and V visible series. Total <grand>."`
  Empty input / zero grand total returns `"No data"`;
  `ariaDescription` overrides.

```ts
describeMarimekkoChart(columns, series, hiddenSeries, formatValue?): string
```

- `computeMarimekkoLayout` returns per-column
  `{ id, index, label, total, share, x, y, width,
  height, cells }` where each cell is
  `{ columnId, columnIndex, seriesId, seriesIndex,
  value, columnTotal, columnShare, cellShare,
  overallShare, x, y, width, height, color, label,
  seriesLabel, columnLabel }`.
- Layout returns `[]` for non-positive inner
  dimensions or when the visible grand total is zero.

## Pairs with existing primitives

- `<ChartStackedBar>` (11.468) -- uniform-width
  stacked sibling.
- `<ChartMosaic>` (11.457) -- contingency-table
  alternative with variable widths AND heights
  derived from row/column independence.
- `<ChartTreemap>` (11.444) -- nested rectangle
  composition alternative.

## Out of scope

- True mosaic / contingency table variant with row
  bands that span columns -- v1 is a pure column-
  centric layout. Use `<ChartMosaic>` when both
  axes need to vary.
- Animated transitions on data / hidden-series
  changes -- snap v1; mount fade only.
- 100%-stacked variant where every column has the
  same width (call `<ChartStackedBar>` instead).
- Negative values (Marimekko requires positive
  shares; non-positive values are skipped).

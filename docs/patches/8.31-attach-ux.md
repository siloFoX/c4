# 8.31 - Sessions attach UX guidance + onboarding

## Summary

After 8.17 shipped `c4 attach` and the Sessions tab gained an "Attach new..."
button, operators reported the workflow was opaque: the button said what it
did but not why, the modal asked for a JSONL path with no preview of what
was available, post-attach rows had one tiny trash icon with no visible
"view the conversation" action, and nothing explained how an attached
session differs from a live worker. 8.31 adds the missing surface without
changing the `/api/attach*` contract or the `~/.c4/attached.json` shape.

Everything lives in `web/src/components/SessionsView.tsx` — no new runtime
deps, no new backend routes. The existing wiring (`apiGet /api/attach/list`,
`apiPost /api/attach`, `apiDelete /api/attach/:name`, `ConversationView`
snapshotUrl) is preserved verbatim; `tests/session-attach.test.js` still
passes unchanged.

## Changes

### 1. Empty-state banner explaining attach

When `filteredAttached.length === 0` the Attached sub-section used to render
the bare string `No attached sessions. Use "Attach new..." to import an
external JSONL.`. 8.31 replaces it with an `EmptyAttachBanner` component
that renders an `Info` icon, a **"What is attach?"** headline, the full
sentence `Import external Claude Code sessions (~/.claude/projects/*.jsonl)
to view conversation history in c4 Web UI.`, and a primary
`Attach your first session` button that opens the modal. The banner sits
inside the sidebar card so the user sees it immediately on first visit.

Strings are exported module constants (`EMPTY_ATTACH_BANNER_TITLE`,
`EMPTY_ATTACH_BANNER_BODY`) so the source-grep test file locks them down.

### 2. Attach modal with available-session preview + post-attach help

The `AttachModal` component grows an `available: SessionSummary[]` prop
that `SessionsView` threads in from the already-fetched `/api/sessions`
payload (no new HTTP call). The modal now renders:

- An intro line clarifying that attach is pointer-only.
- An **Available sessions** preview list (top 10) showing per-entry
  project path, relative "last update" time, turn-count badge, a
  shortened session UUID, and the last assistant snippet. Each row ships
  a `Use this id` button that auto-fills the UUID field.
- The existing manual `JSONL path or session UUID` + `Alias (optional)`
  inputs.
- A dashed **After attach you can:** help card listing the three
  capabilities an operator unlocks: `view full conversation timeline`,
  `search messages across sessions`, `resume the session via claude
  --resume`.
- Cancel / Attach buttons (unchanged).

The modal widens from `max-w-md` to `max-w-2xl` to accommodate the
preview block.

### 3. Post-attach row actions

Each attached row in the sidebar used to carry a single trash icon on the
right edge. 8.31 adds an `AttachedRowActions` panel directly beneath the
row with three clearly-labeled buttons:

| Button | Icon | Action |
|---|---|---|
| View conversation | `Eye` | `setSelection({ kind: 'attached', name })` |
| Resume in terminal | `Terminal` | expands an inline code block showing `claude --resume <sessionId>` + a copy button |
| Detach | `Trash2` | `handleDetach(name)` — unchanged |

Every button has a contextual `aria-label` (`View conversation for <name>`,
`Resume <name> in terminal`, `Detach <name>`). The resume command uses
`navigator.clipboard.writeText` with a graceful no-op fallback when the
clipboard API is unavailable; a transient `copied` indicator surfaces for
1.5 s after a successful copy.

### 4. Attached vs Live comparison card

A new `ComparisonCard` component renders a four-row table contrasting
attached sessions and live workers along the dimensions that most often
confuse operators:

| | Attached | Live worker |
|---|---|---|
| Mode | Read-only view | Interactive PTY |
| Source | JSONL transcript | Live pty stream |
| Updates | Re-parse on refresh | Real-time SSE |
| Resume | `claude --resume <id>` | Already running |

The card renders in two places so the operator sees it regardless of
where they landed:

- As part of the **empty right-pane card** when no session is selected.
- As a self-ending side-card **below the conversation view** when an
  attached row is active.

It does not render for live-worker sessions (`kind: 'session'`) — those
are read-only session history and the distinction does not apply.

### 5. First-visit 3-step onboarding tour

A dismissable `Tour` overlay fires on the first visit to the Sessions
tab. It renders in the bottom-right corner (top-right on desktop) and
walks through three steps:

1. **Welcome to Sessions** — browsing past conversations.
2. **Attach external sessions** — the "Attach new..." entry point.
3. **View or resume** — read the timeline or copy the resume command.

The step counter sits in the header (`1/3`, `2/3`, `3/3`). Each step has
a `Next` button (advances `step`) and a `Skip tour` button (dismisses
early); the final step swaps `Next` for `Done`. Both dismissal paths set
`localStorage['sessions-tour-v1'] = 'done'`.

Both the read site (`localStorage.getItem`) and the write site
(`localStorage.setItem`) are wrapped in `try/catch` so private-browsing
throws cannot crash the page. A `useRef` guards against re-checking the
flag on every re-render.

No third-party tour library — this is a ~40 line component built on the
same `Card` primitives the rest of the Sessions tab already uses.

## Not in scope

- **i18n**: the existing component renders English strings directly; we
  keep that pattern. When a translation layer lands, the exported
  constants at the top of `SessionsView.tsx` are the drop-in extraction
  points.
- **Clipboard fallback UI**: if `navigator.clipboard.writeText` throws
  or is absent, the copy silently no-ops. The resume command is still
  visible in the inline code block so the operator can select + copy
  manually.
- **Tour re-trigger**: there is no "replay tour" affordance in 8.31.
  Operators who want to see it again can clear `sessions-tour-v1`
  from localStorage. A Settings toggle is a follow-up.
- **Attached-session auth ACL** (still 8.17 P3): every authenticated
  viewer sees every attached session.

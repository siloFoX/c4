## API

```ts
interface DetailListItem {
  id: string | number;
  label: ReactNode;
  value: ReactNode;
  copyValue?: string;
  copyable?: boolean;
  ariaLabel?: string;
}

interface DetailListProps {
  items: DetailListItem[];
  orientation?: 'horizontal' | 'vertical';
    // default 'vertical'
  size?: 'sm' | 'md' | 'lg';
    // default 'md'
  className?: string;
  ariaLabel?: string;
    // default 'Details'
  showCopyOnHover?: boolean;
    // default true
  copyLabel?: (item) => string;
  onCopy?: (item, text: string) => void;
  copyFeedbackMs?: number;
    // default 1500
  emptyLabel?: ReactNode;
    // default '(no details)'
}
```

## Behaviour

### Semantics

Native `<dl>` root with `<dt>` /
`<dd>` per row, plus
`role="list"` / `role="listitem"`
annotations for AT defensive
guards. The dt-dd shape is
preserved so screen readers
announce "definition list, 3
items, term: Name, definition:
arps-1, ...".

### Orientations

| orientation | layout |
| --- | --- |
| `vertical` (default) | label on top (small uppercase tracked muted), value below |
| `horizontal` | label left ~1/3 width on `md+`, value right-aligned; collapses to vertical on mobile (`< md`) |

### Copy-on-hover

When an item is copyable, a small
button slides in on hover / focus
(opacity 0 -> 100). The button is
ALWAYS in the DOM (so the row
height does not jump) and ALWAYS
focusable from the keyboard
(focus-visible reveals it).

Click writes the resolved text
via `navigator.clipboard.writeText`
AND fires `onCopy(item, text)`.
The callback fires regardless of
clipboard support so tests +
non-https hosts can observe.

Copy success flips the icon to a
checkmark (`Check` from
lucide-react) for `copyFeedbackMs`
(default 1500), then back to the
copy glyph. The row also gains
`data-copied="true"` during the
window for downstream styling.

### Copyable resolution

| item.copyable | item.copyValue | item.value | copyable? |
| --- | --- | --- | --- |
| `true` | (any) | (any) | yes |
| `false` | (any) | (any) | no |
| undefined | string | (any) | yes (uses copyValue) |
| undefined | undefined | string | yes (uses value) |
| undefined | undefined | number | yes (coerced) |
| undefined | undefined | bigint | yes (coerced) |
| undefined | undefined | ReactNode | no |

`copyable: true` with no
copyValue + non-string-coercible
value will still hide the button
because the pure helper
`getDetailListCopyText` returns
null (no text to write). Adopters
should pair `copyable: true` with
`copyValue`.

### Empty list

When `items.length === 0`:

- dl carries `data-empty="true"`.
- `data-item-count` is omitted.
- `emptyLabel` renders inside the
  dl in a small muted span with
  `data-section="detail-list-empty"`.
- `aria-label` still applies so
  AT users hear the section name.

## Pure helpers (exported)

```ts
export function getDetailListCopyText(
  item: DetailListItem,
): string | null;

export function isDetailListItemCopyable(
  item: DetailListItem,
): boolean;
```

`getDetailListCopyText`:
- copyValue wins
- string / number / bigint
  values coerce
- ReactNode returns null

`isDetailListItemCopyable`:
- copyable=true wins
- copyable=false wins
- otherwise: text-presence
  fallback

## ARIA + data attributes

Root (`<dl>`):

- `role="list"`
- `aria-label`
- `data-section="detail-list"`
- `data-orientation`
- `data-size`
- `data-empty` ('true' / 'false')
- `data-item-count`
  (omitted when empty)

Per row:

- `role="listitem"`
- `data-section="detail-list-row"`
- `data-detail-id`
- `data-orientation`
- `data-copyable`
- `data-copied`

Sub-nodes:

- `<dt>`:
  `data-section="detail-list-label"`
- `<dd>`:
  `data-section="detail-list-value"`
- inner span:
  `data-section="detail-list-value-text"`
- copy button:
  `data-section="detail-list-copy"`
  + `data-detail-id`
- copy glyph:
  `data-section="detail-list-copy-icon"`
- checkmark glyph:
  `data-section="detail-list-copy-check"`
- empty state:
  `data-section="detail-list-empty"`

## Out of scope (deferred)

- Inline edit on a value cell
  (click value -> input).
  Belongs in a follow-on
  primitive
  (`<EditableDetailList>` or
  per-row `renderValue` slot
  with the host owning the
  input).
- Per-row badges / icons next
  to the label. Currently the
  label accepts ReactNode so
  callers can inline; a
  structured badge slot can be
  a later ergonomic
  improvement.
- Sort / reorder. List order
  is source-array order;
  reorder semantics are out of
  scope.
- Group / section headers
  between rows. Belongs in a
  follow-on if a real adopter
  hits the limit.
- Custom copy icon. Currently
  fixed to lucide-react Copy /
  Check; a future API can
  accept `copyIcon` /
  `copiedIcon` ReactNode
  overrides.

# 11.1100 -- UX FIX: gallery ChartLineMomentum render crash (v1.11.1118)

## Scope

The functional audit caught a runtime crash: the gallery's
ChartLineMomentum tile threw "Cannot read properties of undefined
(reading 'reduce')" and tripped the per-tile UIErrorBoundary.

## Root cause

ChartLineMomentum reads a `series` prop (not `data`). The
ChartLineGallery GalleryTile passed only `data={GALLERY_SAMPLE}`, so
`series` was undefined. `computeLineMomentumLayout` guards its input,
but the component's `allTotalPoints` useMemo called `series.reduce(...)`
directly (and the JSX uses `series.length` / `series.map`), so
`undefined.reduce` threw during render.

## Two-part fix

1. **ChartLineGallery passes a series-shaped sample too.** Added
   `GALLERY_SERIES` (one series whose points carry every field incl.
   `y` = close) and pass `series={GALLERY_SERIES}` alongside
   `data={GALLERY_SAMPLE}`, so every tile gets its sample regardless of
   which prop it reads. ChartLineMomentum now renders meaningfully.
2. **chart-line components default a missing `series` prop to `[]`.**
   - ChartLineMomentum: `series = EMPTY_MOMENTUM_SERIES` (stable empty
     const) at the destructure, so a missing series can never crash.
   - Siblings: grepped `web/src/components/ui/chart-line-*.tsx` for the
     same unguarded `series` prop usage. The components destructure
     `series,` (no default) and use `series.length` / `series.map` /
     `series.reduce` directly. Added `series = []` to every file where
     the destructured `series,` appears exactly once (unambiguously the
     props destructure -- files with multiple/ambiguous occurrences were
     skipped to avoid touching object literals). This is behaviour-
     neutral when a series is passed (the gallery passes one) and only
     prevents the undefined-prop crash. The whole bulk edit was
     validated with `tsc --noEmit` (clean).

## Verification (mandatory visual)

`web/e2e/gallery-no-crash.spec.ts` (new) opens the gallery, waits for
every lazy tile to mount (no skeleton left), and asserts zero uncaught
page errors AND zero tiles showing the UIErrorBoundary "Chart failed"
fallback (enumerating any crashed tile's data-chart-id). Result:
`1 passed` -- all 41 registry tiles render. Screenshot
`test-results/gallery-no-crash-1440.png`.

## Risk

- The gallery now passes `series` in addition to `data` (extra ignored
  prop for `data`-reading charts; the real sample for `series`-reading
  charts).
- The `series = []` defaults are behaviour-neutral when a series is
  provided and only fire on a missing prop. tsc-validated. The
  ambiguous (multi-occurrence) files were left unchanged.

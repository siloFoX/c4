# 11.159 -- UI: Collapsible disclosure primitive (v1.11.177)

## Component

- `web/src/components/ui/collapsible.tsx` -- single-section disclosure
  primitive plus an accordion wrapper.
  - Exports: `Collapsible` (forwardRef to `HTMLElement` -- outer
    `<section>`), `CollapsibleGroup`, `CollapsibleProps`,
    `CollapsibleGroupProps`.
  - `Collapsible` props: `{ title, defaultOpen?, open?, onOpenChange?,
    description?, className?, children }`.
  - `CollapsibleGroup` props: `{ exclusive? (default true),
    defaultOpenId?, className?, children }`.
  - Header is a real `<button>` with `aria-expanded` + `aria-controls`
    so Enter / Space toggle natively. The panel is a sibling `<div>`
    that uses `hidden` + `aria-hidden` when closed so focus does not
    traverse into hidden controls.
  - Chevron right (lucide-react `ChevronRight`) rotates 90deg when
    open under Tailwind `transition-all duration-200`; the rotation
    is dropped under `prefers-reduced-motion` via the `motion-reduce:`
    prefix on both the transition and the rotate.
  - Group state is propagated via a single React context
    (`CollapsibleGroupContext`) exposing `isOpen(id)` + `setOpen(id,
    next)`. Each child Collapsible registers an internal id via
    `useId()`. `defaultOpen` on a child seeds the group's state on
    mount so the seed survives toggling siblings.
  - Three modes resolve open state in order: controlled `open` prop
    -> parent group -> local uncontrolled state.
- `web/src/components/ui/index.ts` -- adds `export * from
  './collapsible'`.
- `web/src/components/ui/collapsible.test.tsx` -- vitest + RTL
  coverage (13 cases, listed below).

## Adoption sites (3)

1. `web/src/components/ErrorBoundary.tsx` -- the stack-trace panel
   migrated from a native `<details>`/`<summary>` to a `<Collapsible
   title="Stack trace">` (closed by default). The two adopting tests
   in `ErrorBoundary.test.tsx` were rewritten to assert against the
   `aria-expanded` button + `role="region"` panel instead of the
   removed `<details>` element.
2. `web/src/pages/Health.tsx` -- the Loaded-modules panel (previously
   an always-open `<Panel>` with the module ul) is now wrapped in a
   `<Collapsible>` so the long sub-system list does not dominate the
   page. The empty-state hint branch is unchanged. The existing
   `screen.getByText('Loaded modules (N)')` health tests continue to
   pass because the header text now lives on the Collapsible button.
3. `web/src/components/RiskRuleCatalogPanel.tsx` -- the per-severity
   critical/high/medium filter groups are wrapped in a
   `<CollapsibleGroup exclusive={false}>` so the operator can pin
   one severity open while scrolling another. Each severity becomes a
   `<Collapsible>` whose title is the existing severity Badge. The
   `Fieldset` + filter input outside the group is unchanged.

## Tests

- `web/src/components/ui/collapsible.test.tsx`: 13 cases -- title text
  renders, closed-by-default panel is `hidden`, `defaultOpen=true`
  exposes the body in the a11y tree, click toggles open then closed,
  controlled `open` prop drives state across rerenders, `onOpenChange`
  fires with the next state on each click, `description` renders
  inside the header row, accordion mode closes the previous section
  when a new one opens, accordion mode toggles the active section
  closed on a repeat click, `exclusive=false` allows multiple sections
  open simultaneously, child `defaultOpen` seeds the group, caller
  className merges onto the outer `<section>`, `forwardRef` returns
  a live `HTMLElement` whose tagName is `section`.

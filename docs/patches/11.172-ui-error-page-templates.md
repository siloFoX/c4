# Patch 11.172 -- UI: error/empty page templates (v1.11.190)

Three page-level template components in `web/src/components/templates/`
covering the recurring "full-height centered card with icon + title +
description + CTA(s)" surface for error / not-found / unauthorized
states. Templates compose existing `ui/*` primitives (Alert, Button)
and lucide-react icons; no new dependencies.

Adds:

- `web/src/components/templates/ErrorPage.tsx` -- generic error fallback.
- `web/src/components/templates/NotFoundPage.tsx` -- 404 page.
- `web/src/components/templates/UnauthorizedPage.tsx` -- 401 page.
- `web/src/components/templates/index.ts` -- barrel re-export.
- `web/src/components/templates/ErrorPage.test.tsx` -- RTL coverage (10 cases).
- `web/src/components/templates/NotFoundPage.test.tsx` -- RTL coverage (5 cases).
- `web/src/components/templates/UnauthorizedPage.test.tsx` -- RTL coverage (7 cases).

## Component contracts

### `<ErrorPage>`

Props:

- `error?: Error | unknown` -- when provided, the resolved message
  renders inside a `variant='error'` `<Alert role='alert'>`. The
  resolver accepts `Error`, string, or any object with a `message`
  string property; anything else is treated as "no message" and the
  Alert is omitted.
- `resetError?: () => void` -- when provided, a primary `<Button>`
  labelled `Try again` fires it. Omitted -> no Try again button, and
  the Go home anchor promotes from `outline` to `default` variant so
  the page always has exactly one visually primary action.
- `homeHref?: string` -- defaults to `/`.
- `title?: string` -- defaults to `Something went wrong`.
- `description?: string` -- defaults to
  `An unexpected error occurred. Try again, or head back home.`.
- `forwardRef<HTMLDivElement>` on the wrapper.
- Wrapper `role='alert'` so screen readers surface the page even when
  no inner Alert is rendered.

### `<NotFoundPage>`

Props:

- `homeHref?: string` -- defaults to `/`.
- `title?: string` -- defaults to `Page not found`.
- `description?: string` -- defaults to
  `The page you are looking for has moved or no longer exists.`.
- Single CTA: `Go home` anchor styled with `buttonVariants({ variant: 'default' })`.
- `forwardRef<HTMLDivElement>` on the wrapper.

### `<UnauthorizedPage>`

Props:

- `signInHref?: string` -- defaults to `/login`.
- `homeHref?: string` -- defaults to `/`.
- `title?: string` -- defaults to `Unauthorized`.
- `description?: string` -- defaults to
  `You do not have permission to view this page.`.
- Two CTAs: `Sign in` (primary, `buttonVariants({ variant: 'default' })`)
  and `Go home` (secondary, `outline` variant).
- `forwardRef<HTMLDivElement>` on the wrapper.

All three use `cn()` for class merging and lucide-react icons
(`AlertCircle` / `FileQuestion` / `Lock`). The Go home / Sign in
affordances render as styled `<a>` anchors so a real `href` is on the
DOM (not a click handler) -- enabling middle-click / open-in-new-tab
without component-level wiring and giving tests an `href` to assert
against directly.

## Adoption notes

- **Barrel.** A new `web/src/components/templates/index.ts` barrel
  re-exports all three. The existing `components/index.ts` barrel is
  not modified; consumers import from `components/templates` directly.
- **Router wiring.** `web/src/App.tsx` is not modified in this patch.
  The top-view switcher's `topView` value is exhaustively constrained
  by the `TopView` union in `components/layout/TopTabs.ts` and
  filtered through `TOP_VIEW_VALUES` in `lib/preferences.ts` -- there
  is no reachable "unknown topView" path to wire NotFoundPage into.
  The component is exported and ready for adoption the moment a
  generic router (or a `topView === 'notFound'` extension) lands.
- **ErrorBoundary.** `components/ErrorBoundary.tsx` already ships a
  bespoke fallback with stack trace + GitHub deep link + copy-stack
  affordances. ErrorPage is offered as a lighter-weight alternative
  for feature-level boundaries; the top-level ErrorBoundary stays as-is
  to preserve the operator-targeted debug experience.
- **Unauthorized wiring.** `lib/use-auth-state.ts` exposes
  `loading` / `anon` / `authed` / `disabled` states but does not
  surface a 401-after-auth state distinctly from `anon`; rather than
  invent a new state machine entry, this patch leaves UnauthorizedPage
  unwired. Adopt it when a feature route gates on a per-resource 403
  / 401 (e.g., admin-only Settings tabs).

npx vitest run \
  web/src/components/templates/ErrorPage.test.tsx \
  web/src/components/templates/NotFoundPage.test.tsx \

home defaults to `/` / custom title + description override defaults /
string-shaped errors render their message.

custom title + description override defaults.

UnauthorizedPage (7): default title `Unauthorized` / default
description / Sign in honours signInHref / Sign in defaults to
`/login` / Go home honours homeHref / Go home defaults to `/` /
custom title + description override defaults.

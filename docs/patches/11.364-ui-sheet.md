## Goal

Dispatch:

- Build `web/src/components/ui/sheet.tsx` --
  side-anchored modal panel
  (left/right/top/bottom) with slide
  animation, focus trap, backdrop. Adopt at
  mobile nav, filter panels.

`<Drawer>` already ships the contract (shipped
in 11.279 / v1.11.297):

- side anchor (left / right / top / bottom) ✓
- slide animation (transition-transform +
  reduced-motion gate) ✓
- focus trap (`useFocusTrap`) ✓
- backdrop with `closeOnBackdropClick` ✓
- Escape close ✓
- portal mount + ARIA ✓

The shadcn ecosystem calls this primitive
`Sheet`. This patch ships the canonical name
alias plus a `size` preset
(`sm` / `md` / `lg` / `full`) so adopters get a
consistent rhythm across the modal surface.

## API

```tsx
<Sheet
  open={isOpen}
  onOpenChange={setIsOpen}
  side="right"          // 'left' | 'right' | 'top' | 'bottom'
  size="md"             // 'sm' | 'md' | 'lg' | 'full'
  title="Filters"
  description="Narrow the visible rows"
>
  {body}
</Sheet>
```

### Size presets

| size | width (left/right) | height (top/bottom) |
| --- | --- | --- |
| `sm` | `256px` | `25%` |
| `md` | `320px` | `50%` |
| `lg` | `480px` | `75%` |
| `full` | `720px` (clamped to viewport by Drawer maxWidth) | `95vh` |

`md` matches the legacy `Drawer` defaults
(`'320px'` / `'50%'`) byte-identical so swapping
`<Drawer>` for `<Sheet>` with no `size` is a
no-op.

Explicit `width` / `height` props win over the
size preset.

The Drawer always applies `maxWidth: 100%` /
`maxHeight: 100%` so `size="full"` collapses
gracefully on narrow viewports.

### Side anchor

| side | docks | scroll axis |
| --- | --- | --- |
| `left` | left edge | width |
| `right` | right edge | width |
| `top` | top edge | height |
| `bottom` | bottom edge | height |

For top / bottom, the `size` preset resolves to
the `height` axis; `width` is not set inline
(the underlying Drawer handles the cross-axis
full-bleed).

## Tests

`sheet.test.tsx` -- 15 cases:

- Renders nothing when open=false.
- Renders body when open=true.
- Defaults side="right".
- Forwards side="left" / "top" / "bottom".
- size="md" -> 320px on right.
- size="sm" -> 256px on left.
- size="lg" -> 480px on right.
- size="full" -> 720px + maxWidth: 100%.
- top side picks the height preset
  (size="md" -> 50%); width inline NOT set.
- bottom side="lg" -> 75% height.
- Explicit width wins over size preset.
- Explicit height wins over size preset.
- Title + close button render; close fires
  onOpenChange(false).
- closeOnBackdropClick=false suppresses the
  backdrop close.
- displayName exposed.

15/15 pass under vitest 4.1.5. TypeScript clean.

## Pairs with existing primitives

- `<Drawer>` (11.279) -- unchanged. Sheet is a
  thin wrapper.
- `useFocusTrap` -- inherited via Drawer.
- `useReducedMotion` -- inherited.
- `<Dialog>` size scale (11.363) -- the
  sm/md/lg/full presets match the Dialog
  scale so modal-vs-sheet decisions stay
  visually continuous.

## Out of scope

- Per-page adoption swap (Drawer -> Sheet).
  The alias is additive; existing call sites
  stay byte-identical.
- An `xs` size below `sm`.
- Touch swipe-to-close.
- Responsive size cascade. The Drawer's
  `maxWidth: 100%` / `maxHeight: 100%` clamps
  already handle the narrow-viewport case.

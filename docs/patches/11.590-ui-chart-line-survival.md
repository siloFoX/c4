## Summary

New `ChartLineSurvival` UI primitive at
`web/src/components/ui/chart-line-survival.tsx` -- a pure-SVG
Kaplan-Meier survival curve with a descending step line and optional
censoring ticks. The curve plots the estimated survival probability
against time on a single 0 to 1 panel, with a dashed median-survival
reference line.

## Indicator math

`computeKaplanMeier(observations)` is the Kaplan-Meier survival
estimate. Each observation is a `time` plus an `event` flag (`true` =
the event occurred, `false` = censored). Observations are grouped by
distinct time; at each time:

```
atRisk n  = subjects still at risk just before the time
deaths d  = events at the time
S         = S * (1 - d / n)   when d > 0
```

Censored observations do not lower `S` but reduce the at-risk count
`n` for later times. The result is a descending step function.
`computeLineSurvivalMedian(steps)` reports the earliest time the
estimate falls to 0.5 or below (null if it never does).

### Worked fixture

4 subjects: `t=1 event`, `t=2 event`, `t=3 censored`, `t=4 event`:

```
t=1 d=1 n=4 -> S = 1    * (1 - 1/4) = 0.75
t=2 d=1 n=3 -> S = 0.75 * (1 - 1/3) = 0.5
t=3 d=0 n=2 -> S unchanged 0.5 (one censoring tick)
t=4 d=1 n=1 -> S = 0.5  * (1 - 1/1) = 0

steps survival = [0.75, 0.5, 0.5, 0]
```

`survivalFinal = 0`, `eventCount = 3`, `censorCount = 1`,
`medianSurvival = 2` (the first step with `S <= 0.5`).

## API

- `getLineSurvivalFiniteObservations(observations)` -- drop
  observations with a non-finite `time`; coerce `event` to a strict
  boolean (only `event === true` counts as an event).
- `computeLineSurvivalMedian(steps)` -- the median survival time.
- `computeKaplanMeier(observations)` -- the Kaplan-Meier result
  `{ steps, censorTicks, total, survivalFinal, eventCount,
  censorCount, medianSurvival }`. Sorts internally.
- `runLineSurvival(observations)` -- drop non-finite observations,
  sort by time, compute the estimate, return `{ observations, steps,
  censorTicks, total, survivalFinal, eventCount, censorCount,
  medianSurvival, ok }`. `ok` is false for fewer than two
  observations.
- `computeLineSurvivalLayout(options)` -- build the descending
  staircase path, the step markers, the censoring tick positions, and
  the projected median reference line.
- `describeLineSurvivalChart(data)` -- ARIA description string.

## Component

`ChartLineSurvival` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, a config badge (subject
count + median survival), a two-series legend (Survival / Censored)
with toggle, the descending step path, censoring ticks (short
vertical marks on the curve), step markers, a dashed median reference
line, a hover tooltip exposing the time, survival, at-risk count,
events, and censored count for the hovered step, and a
`motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

This primitive is a biostatistics chart, not a financial indicator.
It is distinct from every `chart-line-*` oscillator in the library:
its input is time-to-event data carrying a censoring flag, and its
curve is a right-continuous step function that only ever descends,
never recovering. The Kaplan-Meier estimator -- the displaced
multiplicative survival estimate with explicit handling of censored
observations -- has no counterpart among the moving-average,
momentum, or volatility charts.

# 11.1077 -- UI: chart-line-williams-oversold-divergence primitive (v1.11.1095)

## Scope

Add a pure-SVG dual-panel
`ChartLineWilliamsOversoldDivergence` component that
renders the close in the top panel and Larry Williams'
Percent R (%R) in the bottom panel, marking bullish
(price down while %R up at oversold levels -- **bottom
reversal warning**) / bearish (price up while %R down at
oversold levels -- contrarian rally exhaustion, less
common) price-vs-%R direction disagreement (divergence)
trigger events with bias coloring derived from the %R
slope at the divergence-entry bar. **Crosses are gated
to the oversold zone**: only fire when %R is at or below
the oversoldLevel (default -80).

Direct mirror of `chart-line-williams-overbought-divergence`
v1.11.1094 -- same zone-gated divergence pattern but
focused on bottom reversal warnings. Completes the
Williams %R zone-gated divergence pair, which itself
generalises the CCI overbought/oversold pair (v1.11.1092
/ v1.11.1093) to the bounded [-100, 0] Williams %R
oscillator.

## Williams %R oversold-divergence pipeline

```
HH[i]  = max(high[i-period+1..i])
LL[i]  = min(low[i-period+1..i])
%R[i]  = (HH[i] - close[i]) / (HH[i] - LL[i]) * -100
         (null when HH === LL)

regime :
  'aligned-bullish'   when priceUp   && wrUp
  'aligned-bearish'   when priceDown && wrDown
  'divergent-bullish' when priceDown && wrUp
                      (PRIMARY: bottom reversal warning
                       when %R is in oversold zone)
  'divergent-bearish' when priceUp   && wrDown
  'none'              otherwise

bullish cross (at oversold) :
  prev regime !== 'divergent-bullish' &&
  cur regime === 'divergent-bullish' &&
  cur %R <= oversoldLevel
bearish cross (at oversold) :
  prev regime !== 'divergent-bearish' &&
  cur regime === 'divergent-bearish' &&
  cur %R <= oversoldLevel

bias : %R[i] vs %R[i-1] -> up/down/flat/none
```

Default `period = 14`, `oversoldLevel = -80` -- canonical
Larry Williams (1973) %R tuning + classical oversold
threshold. Williams %R overbought zone is -20..0 (close
near the high); oversold zone is -100..-80 (close near
the low); -50 is the midline.

## Sibling family

The zone-gated divergence family (now four members
across two indicators):

- **chart-line-cci-overbought-divergence v1.11.1092** --
  CCI top reversal, CCI >= +100 gate
- **chart-line-cci-oversold-divergence v1.11.1093** --
  CCI bottom reversal, CCI <= -100 gate
- **chart-line-williams-overbought-divergence
  v1.11.1094** -- %R top reversal, %R >= -20 gate
- **chart-line-williams-oversold-divergence v1.11.1095**
  (this primitive) -- %R bottom reversal, %R <= -80 gate

Other Williams %R siblings: chart-line-williams-r (raw),
williams-r-cross / williams-r-cross-sig (level / signal),
williams-mid-cross-sig (-50 mid), williams-divergence-
cross (ungated).

Distinct from chart-line-williams-divergence-cross (the
ungated sibling): this primitive filters crosses to only
those that occur while %R is in the oversold zone
(<= -80).

Warmup is `period = 14` for the first %R value. Cross
detection needs the previous bar's regime, so the first
potential cross lands at i = period + 1 = 15.

## Bit-exact anchors

All anchors use HLC input. The exact inverse of the
overbought sibling's anchor behaviour.

- **CONST band** `high = K + 1`, `low = K - 1`,
  `close = K`: %R = **-50** (constant midpoint). NOT in
  the oversold zone (-50 > -80). priceFlat, wrFlat ->
  regime `none`. 0 crosses, oversoldCount = 0. Verified
  across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`,
  `close = i`: %R = 1 / (period + 1) * -100. For
  period = 14: **-6.667** (constant). NOT in the
  oversold zone (-6.667 > -80; close near the period
  high = overbought). The gate filters any divergent
  crosses. 0 crosses, oversoldCount = 0.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`,
  `close = -i`: %R = period / (period + 1) * -100. For
  period = 14: **-93.333** (constant). IN the oversold
  zone (-93.333 <= -80; close near the period low).
  priceDown, wrFlat -> regime `none`. The gate passes
  but the flat %R produces no divergent state. 0
  crosses, oversoldCount > 0.

LINEAR DOWN sits inside the oversold zone (close near
the period low), while CONST and LINEAR UP sit outside
it -- the exact inverse of the overbought sibling, where
LINEAR UP was the in-zone anchor.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 5-state regime classifier follows the divergence-cross
  family convention exactly.
- Oversold-zone gate (`curWr <= oversoldLevel`) is the
  key specialisation, mirrored from the overbought
  sibling with the inequality flipped. Verified by the
  explicit detector-level test that builds the same
  divergent-bullish regime sequence with %R values both
  below and above the threshold.
- Williams %R is INVERTED relative to typical
  oscillators: oversold is near -100 (the bottom of the
  range). The `oversoldLevel <= ` gate semantics are
  correct for this orientation (a LOWER %R, closer to
  -100, is more oversold).
- Zero-range guard returns null when HH === LL (a
  perfectly flat window). Matches pandas-ta / ta-lib
  behaviour.
- The bounded [-100, 0] range is mathematically
  guaranteed; the panel hard-locks accordingly.
- Completes the Williams %R zone-gated divergence pair
  (overbought top reversal v1.11.1094 + oversold bottom
  reversal). Together with the CCI pair, the zone-gated
  divergence template now spans two oscillators (one
  unbounded centered at 0, one bounded [-100, 0]),
  demonstrating the pattern generalises across
  oscillator families. Obvious next siblings:
  rsi-overbought/oversold-divergence (bounded [0, 100]),
  stochastic-overbought/oversold-divergence.

# 11.765 chart-line-swing-index (v1.11.783)

## Summary

Pure-SVG dual-panel React/TypeScript chart primitive with a
**Wilder Swing Index (SI)** oscillator panel beneath the close.
The Swing Index measures directional bar pressure scaled by the
larger of true range and price gap. Added as the
`<ChartLineSwingIndex />` UI primitive in
`web/src/components/ui/`.

## Algorithm

For each bar with prior OHLC = `(prevO, prevH, prevL, prevC)`
and current OHLC = `(O, H, L, C)`:

```
A = |H - prevC|
B = |L - prevC|
C = |H - L|
K = max(A, B)
N = (C - prevC) + (C - O) / 2 + (prevC - prevO) / 4
R = if A is largest of {A, B, C}:
      A - B / 2 + |prevC - prevO| / 4
    elif B is largest:
      B - A / 2 + |prevC - prevO| / 4
    else:
      C + |prevC - prevO| / 4
SI = 50 * (N / R) * (K / T)
```

`T` is the **limit move** (futures-specific constant). For
non-futures applications the default is `T = 1`. The seed bar
(no prior reference) is `null`. When `R == 0` (bar with no range
and no gap) the bar is `null`. When `K == 0` (close did not move
above or below the prior close range) `SI` is `0` by
construction.

## Anchors

- **Seed bar (`i == 0`)**: `SI = null` (no prior reference).
- **ALL_FLAT (every bar `O = H = L = C = K`)**: `R = 0` at every
  bar -> `SI = null` for all bars.
- **`K = 0` (current high = low = prevC)**: `K / T = 0`, so
  `SI = 0` bit-exact regardless of `N` and `R`.
- **Worked dyadic anchor**:
  prev `{o: 2, h: 4, l: 2, c: 2}`,
  curr `{o: 2, h: 4, l: 2, c: 4}`, `T = 1`.
  - `A = 2, B = 0, C = 2` -> else branch -> `R = 2`.
  - `K = max(2, 0) = 2`.
  - `N = (4 - 2) + (4 - 2) / 2 + (2 - 2) / 4 = 3`.
  - `SI = 50 * (3 / 2) * (2 / 1) = 150` **bit-exact**.
- **Antisymmetry**:
  prev `{2, 4, 2, 4}`, curr `{4, 4, 2, 2}` (mirror image)
  yields `SI = -100` (case-B branch).
- **Limit move scaling**: `T = 2` halves the SI to `75` bit-exact
  (since `K / T` is the only term that changes).
- **Case-A branch**:
  prev `{5, 10, 5, 5}`, curr `{5, 20, 8, 20}` yields
  `SI = 50 * (22.5 / 13.5) * 15 = 1250` (within ULP of 1250).
- **Case-B branch**:
  prev `{10, 10, 10, 10}`, curr `{10, 9, 2, 2}` yields
  `SI = -640` bit-exact.

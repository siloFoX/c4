## Summary

Adds the `ChartLineTrixOverboughtCross` primitive
(`web/src/components/ui/chart-line-trix-overbought-cross.tsx`): a
pure SVG dual-panel chart that visualizes TRIX (triple-smoothed
exponential moving average rate of change) crossings through the
canonical overbought line 0.1 for triple-smoothed momentum
overbought trigger events. Companion to
`chart-line-trix-zero-cross` (TODO 11.941): shares the same
triple-EMA pipeline (with the zero-divide guard returning 0), only
the default threshold flips from 0 to +0.1 -- the canonical
pre-noise level analysts use to distinguish bullish-trend onset
from baseline drift.

## Pipeline

Same close-only TRIX pipeline as the zero-cross variant:

1. `ema1 = EMA(close, length)` (SMA-seeded with CONST short-circuit)
2. `ema2 = EMA(ema1, length)`
3. `ema3 = EMA(ema2, length)`
4. TRIX from the rate of change of the third smoothing:
   - `trix[i] = (ema3[i] - ema3[i-1]) / |ema3[i-1]| * 100`
   - When `ema3[i-1] == 0`, short circuit to `trix = 0` (bit-exact
     +0; verified via `Object.is`) to avoid 0/0 NaN.
5. Regime classifier (strict `>=`):
   - `trix >= 0.1` -> bullish (in overbought zone)
   - `trix <  0.1` -> bearish (below overbought)
6. Cross detector emits a `bullish` cross when
   `trix[i-1] <= 0.1 && trix[i] > 0.1` (entry) and a `bearish`
   cross when `trix[i-1] >= 0.1 && trix[i] < 0.1` (exit). Strict
   inequality on the current side ensures the boundary value 0.1
   does not double-fire.

## Defaults

- `length = 15` (canonical TRIX window)
- `threshold = 0.1` (canonical overbought line for TRIX %)
- Warmup boundary: bar `3 * length - 2 = 43`, so for a 60-bar run
  the first 43 samples are null and the remaining 17 are
  populated (ema1 valid at 14, ema2 at 28, ema3 at 42, trix at
  43).

## Bit-exact anchors

- CONST `close = K` for all `K in {0, 1, 42, 100, 1234}` ->
  `trix[i] === 0` literal (NOT `-0`) for every `i >= 43`. Regime
  classifies as bearish because 0 < 0.1 -- opposite of the
  zero-cross variant where 0 sat on the threshold 0 and read
  bullish.
- LINEAR UP `close = i + 100` -> ema3 grows ~1 per bar, trix
  remains positive and well above 0.1 across the 100-bar test
  window, so all samples are bullish, 0 crosses (no `prev <= T &&
  cur > T` transient since trix stays above 0.1 throughout).
- LINEAR DOWN `close = 200 - i` -> trix is negative the entire
  run, all bearish, 0 crosses.
- CONST K full-run counters under defaults, 60 bars:
  `noneCount = 43`, `bullishCount = 0`, `bearishCount = 17`,
  `crosses.length = 0`.
- `pricePath` and `trixPath` are byte-identical across repeated
  layout calls, with `.toFixed(2)` precision and `M ... L ...`
  command shapes. The trix path uses a single `M` for post-warmup
  data.
- Cross detector boundary: strict inequality (`> T`, `< T`).
- Layout auto-fit: when trix collapses to 0 (CONST case), the
  panel expands UP to include threshold = 0.1 with 10% padding
  both sides -> oscMin = -0.01, oscMax = 0.11.

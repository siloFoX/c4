## Summary

Add a new `<ChartSlope>` UI primitive in
`web/src/components/ui/chart-slope.tsx` -- pure-SVG
slope / parallel-line chart for before-after comparison
across two time points. Each item gets a connecting line
from its `before` value on the left axis to its `after`
value on the right axis. Direction colors (up / down /
flat) make rate-of-change visually obvious at a glance.
Distinct from `<ChartLine>` (11.439) which paints a
continuous series over many x positions, and from
`<ChartBullet>` (11.442) which compares a single value
to a target range.

## API

```ts
type ChartSlopeDirection = 'up' | 'down' | 'flat';

interface ChartSlopeItem {
  id: string;
  label: string;
  before: number;
  after: number;
  color?: string;
}

interface ChartSlopeProps {
  items: readonly ChartSlopeItem[];
  beforeLabel?: string;          // default 'Before'
  afterLabel?: string;           // default 'After'
  width?: number;                // default 560
  height?: number;               // default 320
  padding?: number;              // default 40
  labelGap?: number;             // default 8
  pointRadius?: number;          // default 4
  tickCount?: number;            // default 5
  yMin?: number;
  yMax?: number;
  epsilon?: number;              // default 0 (flat band)
  upColor?: string;              // default '#16a34a'
  downColor?: string;            // default '#dc2626'
  flatColor?: string;            // default '#64748b'
  showLeftLabels?: boolean;      // default true
  showRightLabels?: boolean;     // default true
  showLeftValues?: boolean;      // default true
  showRightValues?: boolean;     // default true
  showAxes?: boolean;            // default true
  showAxisTicks?: boolean;       // default false
  showGrid?: boolean;            // default false
  showTooltip?: boolean;         // default true
  showColumnHeaders?: boolean;   // default true
  animate?: boolean;             // default true
  className?: string;
  ariaLabel?: string;            // default 'Slope chart'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  formatLabel?: (label: string, item: ChartSlopeLayoutItem) => string;
  onItemClick?: (args: {
    item: ChartSlopeItem;
    layout: ChartSlopeLayoutItem;
  }) => void;
  style?: CSSProperties;
}
```

## Behaviour

- Renders `<div role="region">` wrapping
  `<svg role="img">` with the same `aria-label`.
- Direction (`getSlopeDirection`): when `after -
  before > epsilon` the slope is `up`, when `before -
  after > epsilon` it's `down`, else `flat`.
  `epsilon=0` (default) means any non-zero delta
  registers a direction; `epsilon > 0` widens the
  flat band so tiny noise is reported as flat.
  Non-finite values always resolve to `flat`.
- Color: per-item `color` always wins; otherwise the
  primitive picks `upColor` / `downColor` /
  `flatColor` based on the direction. The default
  palette is the conventional green / red / grey.
- Bounds: auto-derived via `getSlopeBounds` across
  both endpoints. Collapsed range expands by +/-0.5.
  `yMin` / `yMax` props override.
- Items with non-finite before OR after still appear
  in the layout but flagged `isValid=false` and
  skipped from the painted line. They still
  contribute zero counts to `data-up-count` etc.
- Hover/focus on an item sets the hovered id and dims
  every other item to opacity 0.3 so the focused
  slope dominates visually. Tooltip surfaces label,
  formatted before + after, signed delta, and the
  direction.
- Mount animation:
  `motion-safe:animate-fade-in` per item group.
  `animate=false` removes the class.
- Default ARIA description: `"Slope chart from
  <before> to <after> comparing N items: U up, D down,
  F flat. Value range <yMin> to <yMax>."` Empty / no
  valid pairs -> `"No data"`; `ariaDescription`
  overrides.

## Pure helpers (exported)

```ts
DEFAULT_CHART_SLOPE_WIDTH         = 560
DEFAULT_CHART_SLOPE_HEIGHT        = 320
DEFAULT_CHART_SLOPE_PADDING       = 40
DEFAULT_CHART_SLOPE_LABEL_GAP     = 8
DEFAULT_CHART_SLOPE_POINT_RADIUS  = 4
DEFAULT_CHART_SLOPE_TICK_COUNT    = 5
DEFAULT_CHART_SLOPE_UP_COLOR      = '#16a34a'
DEFAULT_CHART_SLOPE_DOWN_COLOR    = '#dc2626'
DEFAULT_CHART_SLOPE_FLAT_COLOR    = '#64748b'

getSlopeDirection(before, after, epsilon?): 'up'|'down'|'flat'
getSlopePctChange(before, after): number
getSlopeDirectionColor(direction, up, down, flat): string
getSlopeBounds(items): { yMin, yMax }
getSlopeTicks(min, max, count?): number[]
computeSlopeLayout(input): ChartSlopeLayoutItem[]
describeSlopeChart(items, beforeLabel, afterLabel, formatValue?): string
```

- `computeSlopeLayout` returns
  `{ id, label, index, color, direction, before,
  after, delta, pctChange, leftX, leftY, rightX,
  rightY, isValid }[]` -- non-positive inner ->
  `[]`.
- `getSlopePctChange` returns `(after - before) /
  |before|`. Zero-before with positive after ->
  `+Infinity`, with negative after -> `-Infinity`,
  both zero -> 0. Non-finite either -> 0.

## ARIA + data attributes

- Root: `<div role="region">` + `aria-label`.
- SVG: `role="img"` + same `aria-label`.
- sr-only span:
  `data-section="chart-slope-aria-desc"`.
- Each line: `role="graphics-symbol"` +
  `tabIndex={0}`. aria-label =
  `"<label>: <before> -> <after> (<direction>)"`.
- `data-section` on every node:
  chart-slope / -canvas / -aria-desc / -svg /
  -headers / -header / -grid / -grid-line / -axes /
  -axis / -ticks / -tick / -tick-label / -items /
  -item / -line / -point / -value / -label /
  -tooltip / -tooltip-label / -tooltip-before /
  -tooltip-after / -tooltip-delta.
- Root mirrors `data-item-count`,
  `data-visible-count`, `data-up-count`,
  `data-down-count`, `data-flat-count`,
  `data-animate`.
- Item groups mirror `data-item-id`,

  `data-item-delta`, `data-item-pct-change`,
  `data-item-color`, `data-hovered`.
- Endpoints mirror `data-side="left"` /
  `data-side="right"`.

## Pairs with existing primitives

- `<ChartLine>` (11.439) -- continuous multi-point
  sibling.
- `<ChartBullet>` (11.442) -- target-vs-actual
  alternative.
- `<ChartWaterfall>` (11.450) -- step-by-step
  delta alternative.
- `<ChartArrow>` -- single-arrow before/after
  alternative.

## Out of scope

- More than two time points (use `<ChartLine>` or
  `<ChartParallel>` (11.448) for multi-axis
  parallel coordinates).
- Tukey-style highlight tracking per item (host
  composes a parent `<ChartSlope>` for the focused
  subset).
- Animated transitions on data change (snap v1;
  mount fade only).
- Stacked / grouped slopes (one slope per item v1).

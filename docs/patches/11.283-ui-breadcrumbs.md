# 11.283 -- Breadcrumbs primitive: truncate-middle + onClick (v1.11.301)

Component-scope-only enhancement. No daemon-side change and no `c4`
CLI surface change. The existing `Breadcrumbs` primitive
(`web/src/components/ui/breadcrumbs.tsx`) keeps its current item
shape + ChevronRight separator + maxItems ellipsis collapsing
byte-for-byte; the new affordances opt in via props.

## Why

Long worker / branch names + 3-level hierarchies (History > worker
> task) were pushing CardTitles off the right edge on narrow
viewports. The existing breadcrumb shipped without any
truncate-middle policy, so callers had to either:

- Pre-truncate the label themselves and lose the full text on hover.
- Wrap the breadcrumb in a horizontal scroll container, which
  fragments the page-hierarchy reading mode.

Patch 11.283 puts the truncate-middle policy inside the primitive so
every consumer gets the same `<head>...<tail>` shape with the full
label preserved via `title={...}` for hover disclosure. The
`onClick` addition rounds out SPA navigation -- the previous shape
only spoke `href`, which forced router-driven consumers to either
synthesize an `href`-then-`preventDefault` dance or drop the
breadcrumb entirely.

## What changed

### `web/src/components/ui/breadcrumbs.tsx`

- New `maxLabelLength?: number` prop. When a string label exceeds
  this many characters, the rendered text collapses to
  `<head>...<tail>` so both ends of the path segment stay visible.
  The full label is preserved via `title={...}` on the wrapping
  span so a hover still surfaces what got clipped. ReactNode
  labels (non-string) are passed through verbatim because the
  primitive does not know how to slice them safely.
- New exported helper `truncateMiddle(input, maxLength)`. Other
  primitives can reuse the policy (e.g., a future
  `<PathBreadcrumb>` that pre-formats filesystem paths). Edge
  cases: input <= maxLength returns unchanged, maxLength <= 0
  returns unchanged, maxLength below the ellipsis budget (3 chars)
  returns just `'...'`.
- New `BreadcrumbItem.onClick?: (e) => void` prop. SPA navigation
  hook. When set, the item renders as a `<button>` instead of `<a>`
  and the click handler fires instead of the browser-default href
  navigation. Coexists with `href` -- `onClick` wins (the button
  takes precedence so router-driven state updates do not need
  `preventDefault`).
- `data-section` selectors added across the layer:
  - `data-section="breadcrumbs"` on the outer `<nav>`.
  - `data-section="breadcrumb-item"` per `<li>` + `data-breadcrumb-current="true|false"`.
  - `data-section="breadcrumb-separator"` on each separator `<li>`.
  - `data-section="breadcrumb-ellipsis"` on the collapsed-paths affordance.
  - `data-section="breadcrumb-truncated"` on the truncate-middle wrapper.
- Pre-existing `noUncheckedIndexedAccess` hit on `items[0]` inside
  `buildSlots()` was annotated with the non-null assertion -- the
  codepath is gated by an `items.length > maxItems` check above,
  so the index is always live there.

### `web/src/components/HistoryDetailPane.tsx`

- Worker detail header now leads with a `Breadcrumbs` context row
  showing `History / <worker-name>`. The breadcrumb mounts above
  the Avatar + CardTitle row so the operator can see the page
  hierarchy without leaving the detail pane.
- `maxLabelLength={24}` ensures long worker names truncate the
  middle without pushing the CardTitle off the right edge; the
  full name is preserved on hover via the `title` attribute.

### `web/src/components/HistoryDetailPane.test.tsx`

- Updated the "renders the worker name as the card title" case to
  expect the doubled echo (Breadcrumbs row + CardTitle row both
  render `detail.name`). Asserts `getAllByText(name).length >= 2`.

## Summary

New `ChartLineKama` UI primitive at
`web/src/components/ui/chart-line-kama.tsx` -- a pure-SVG line chart
with a Kaufman Adaptive Moving Average overlay drawn on the same
axes as the price. KAMA speeds up to track clean trends and slows
down to ignore choppy noise.

## Indicator math

Perry Kaufman's Adaptive Moving Average:

```
ER[i] = |value[i] - value[i-erPeriod]| / sum(|value[j] - value[j-1]|)
fastSC = 2 / (fastPeriod + 1)
slowSC = 2 / (slowPeriod + 1)
SC[i]  = (ER[i] * (fastSC - slowSC) + slowSC)^2
KAMA[i] = KAMA[i-1] + SC[i] * (value[i] - KAMA[i-1])
```

The Efficiency Ratio is the net price change over the window
divided by the sum of the bar-to-bar moves. It runs 0 for pure
noise (the moves cancel out, so the net change is small relative to
the path travelled) to 1 for a clean one-directional trend (the net
change equals the path). That ratio scales a per-bar smoothing
constant between a fast and a slow bound; squaring it makes the
average linger near the slow constant unless the trend is genuinely
efficient. The KAMA then folds each bar in at its own SC.

The KAMA is seeded at index `erPeriod - 1` with that bar's price and
is recursive from index `erPeriod` onward (ER needs `erPeriod` bars
of lookback). A flat window reads an efficiency ratio of 0 rather
than dividing by zero.

### Worked fixture

`values = [10, 14, 10, 22, 30]`, `erPeriod 2`, `fastPeriod 2`,
`slowPeriod 30`:

```
ER[2] = |10-10| / (|10-14|+|14-10|) =  0/8  = 0    (pure noise)
ER[3] = |22-14| / (|22-10|+|10-14|) =  8/16 = 0.5
ER[4] = |30-10| / (|30-22|+|22-10|) = 20/20 = 1    (clean trend)
ER    = [null, null, 0, 0.5, 1]

fastSC = 2/3, slowSC = 2/31
KAMA seed: KAMA[1] = value[1] = 14
KAMA[2] = 14 + (2/31)^2  * (10-14)        ~= 13.983351
KAMA[3] = KAMA[2] + (34/93)^2 * (22-KAMA[2]) ~= 15.054833
KAMA[4] = KAMA[3] + (2/3)^2   * (30-KAMA[3]) ~= 21.697129
```

The efficiency ratios are exact (0, 0.5, 1); the recursive KAMA
values accumulate floating-point error because `2/3` and `2/31` are
inexact, so tests assert them with `toBeCloseTo` and keep the seed
(14) as an exact `toBe`.

## API

- `getLineKamaFinitePoints(points)` -- drop points with a non-finite
  `x` or `value`.
- `normalizeLineKamaPeriod(period, fallback)` -- coerce to a
  positive integer; non-finite or sub-1 falls back, fractional
  floors.
- `computeLineKamaEfficiencyRatio(values, erPeriod)` -- the
  efficiency ratio series, null through the warm-up.
- `computeLineKama(values, erPeriod, fastPeriod, slowPeriod)` --
  returns `{ er, sc, kama }`.
- `runLineKama(points, options?)` -- sort finite points by `x`,
  compute the series, return `{ series, erPeriod, fastPeriod,
  slowPeriod, er, sc, kama, samples, kamaFinal, kamaMin, kamaMax,
  aboveCount, belowCount, ok }`. Each sample carries the price's
  `above` / `below` / `on` position versus the KAMA. `ok` is false
  for fewer than two points.
- `computeLineKamaLayout(options)` -- project the price and KAMA
  onto one shared panel. Returns the panel, ticks, the price path,
  the KAMA path, and KAMA markers.
- `describeLineKamaChart(data, options?)` -- ARIA description.

## Component

`ChartLineKama` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, axis ticks and grid, a
config badge calling out the efficiency-ratio period and the
fast/slow bounds, a two-series legend (Price / KAMA) with toggle, a
hover/focus tooltip exposing the value, KAMA, efficiency ratio and
position, and a `motion-safe:animate-fade-in` entrance. Every node
carries a `data-section` attribute.

## Distinctness

- **vs chart-line-zlema / chart-line-hma / chart-line-alma** -- those
  moving-average overlays reduce lag with a *fixed* weighting scheme
  (a de-lagged EMA, a weighted Hull blend, a gaussian-weighted
  window). The KAMA instead *varies its own responsiveness* every
  bar: the smoothing constant is recomputed each bar from the
  efficiency ratio, so the average is fast in a trend and slow in
  noise. No other primitive computes a signal-to-noise efficiency
  ratio.
- **vs a plain EMA** -- an EMA folds each bar in at one fixed weight
  `2/(period+1)`. The KAMA's weight is the *squared, ratio-scaled*
  smoothing constant, which collapses toward the slow bound whenever
  the market is choppy.

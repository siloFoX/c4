## `lib/format-locale.ts`

### Locale tag mapping

```ts
type LocaleTag = 'en-US' | 'ko-KR';

const TAG_BY_LOCALE: Record<Locale, LocaleTag> = {
  en: 'en-US',
  ko: 'ko-KR',
};

export function localeTag(locale = getLocale()): LocaleTag;
```

The c4 locale enum (`'en' | 'ko'`) maps to BCP-47
tags so `Intl.*` constructors pick the right
numbering / grouping / calendar.

### Functions

| function | purpose |
| --- | --- |
| `formatNumberLocale(value, locale?, { fractionDigits, notation, signDisplay })` | grouped number; `'-'` on null/NaN/Infinity. |
| `formatIntegerLocale(value, locale?)` | 0-digit number (default for counts). |
| `formatDecimalLocale(value, digits?, locale?)` | n-digit number (default 2). |
| `formatCompactLocale(value, locale?)` | `Intl.NumberFormat({notation:'compact'})`. `15K` / `2.5M`. |
| `formatPercentLocale(value, digits?, locale?)` | 0..1 ratio -> percent. |
| `formatCurrencyLocale(value, currency, locale?)` | per-locale currency placement. |
| `formatBytesLocale(bytes, locale?)` | picks unit, formats through locale number formatter. |
| `formatDateLocale(input, preset?, locale?)` | presets: `short` / `medium` / `long` / `iso-date` / `date-time` / `time`. |
| `formatDateTimeLocale`, `formatTimeLocale`, `formatDateRangeLocale` | preset shortcuts + ISO-date range fallback. |

`Date` / `string` / `number` (ms epoch) all
coerce to a `Date`; invalid input yields `'-'`.

### Memoisation

`Intl.NumberFormat` / `Intl.DateTimeFormat`
constructors are surprisingly expensive (locale
tag resolution + options parsing). The module
keeps a per-shape cache keyed by
`${tag}|${JSON.stringify(options)}`:

```ts
const NUMBER_CACHE = new Map<string, Intl.NumberFormat>();
const DATETIME_CACHE = new Map<string, Intl.DateTimeFormat>();
```

`resetFormatLocaleCachesForTests` clears both for
unit-test isolation.

### React hook

```ts
const fmt = useLocalizedFormatters();
// fmt.locale === 'en' | 'ko'
// fmt.tag === 'en-US' | 'ko-KR'
// fmt.integer(value) / fmt.decimal(value, 2) / ...
// fmt.bytes(bytes), fmt.date(input, preset), fmt.dateRange(a, b, preset)
```

Returns a memoised bundle bound to the current
Settings locale (via `useLocale()`). Re-renders
the caller on locale flip; returns identity-stable
closures while locale is stable.

## Adoption

### `web/src/components/HistoryView.tsx`

```tsx
// (v1.11.364) Locale-aware count formatters.
const fmt = useLocalizedFormatters();
// ...
<span>{fmt.integer(summary.length)} {summary.length === 1 ? 'worker' : 'workers'}</span>

// SidebarVirtualizedList
const fmt = useLocalizedFormatters();
// ...
{tFormat(
  w.taskCount === 1
    ? 'history.taskCount.singular'
    : 'history.taskCount.plural',
  { count: fmt.integer(w.taskCount) },
)}
```

The sidebar virtualised list calls the hook
locally (the row renderer is a separate React
component) so the table respects a locale flip
without a full HistoryView remount.

### `web/src/pages/Health.tsx`

`formatNumber` from `lib/format` dropped from the
import list. Every numeric stat (Workers / Queue
trend / Active / Idle / Queue depth / Lost
workers) flows through `fmt.integer`. Duration is
unchanged -- it uses `formatDuration` which is
locale-agnostic by design (the segment labels are
`d`/`h`/`m`/`s`).

### `web/src/pages/TokenUsage.tsx`

`formatNumber` import dropped. Every call site
swapped to `fmt.integer` (totals, per-period
breakdown rows, per-task table columns, sparkline
`lastValueFormatter` reference). The `PerTaskTable`
function calls the hook locally for the per-row
columns.

## Goal

The dispatch lists:

- Test every page at mobile (375), tablet (768),
  desktop (1280), wide (1920) breakpoints.
- Fix any overflow, broken layout, or missing
  collapse behavior.
- Add a smoke test (vitest + jsdom) that renders each
  page at all four widths and asserts no horizontal
  overflow.

This patch lands the smoke helper + harness + two
demonstration page tests. jsdom does NOT compute
layout, so geometry-level overflow checks
(`scrollWidth > clientWidth`) cannot run in this
environment. The helper relies on signals that ARE
evaluable in jsdom: the `matchMedia` flip + an
inline-style overflow scan.

A future Playwright-based follow-up can sample real
geometry against the same four breakpoints; this
patch is the vitest-runnable lever the dispatch
specifically asks for.

## Helper

`web/src/test-utils/responsive.ts` exposes:

```ts
export const RESPONSIVE_BREAKPOINTS: readonly ResponsiveBreakpoint[];

export function installResponsiveHarness(): ResponsiveHarness;
export function findInlineOverflow(root, viewportWidth): OverflowFinding[];
export function runResponsiveSmoke({ render, breakpoints? }): ResponsiveSmokeReport[];
export function formatResponsiveReports(reports): string;
export function expectAllResponsiveOk(reports): void;
```

### `installResponsiveHarness()`

Installs a `matchMedia` mock that evaluates
`(min-width:Npx)` and `(max-width:Npx)` queries
against a tracked "current width". `setWidth(N)` flips
the value, fires every registered MQL change listener
(so consumers that `addEventListener` once observe
every flip), updates `window.innerWidth`, and
dispatches a synthetic `resize` event so size-aware
hooks re-run. `restore()` puts the previous
`matchMedia` + `innerWidth` descriptor back.

The harness caches MQLs by query string so two
`window.matchMedia('(min-width: 768px)')` calls return
the same object -- listeners stay attached across
re-renders.

### `findInlineOverflow(root, viewportWidth)`

Scans the rendered subtree for elements whose inline
`style="width: <N>px"` or `style="min-width: <N>px"`
declaration exceeds the viewport. Returns one
`OverflowFinding` per violation:

```ts
{ selector: 'span#foo', width: 600, cssProperty: 'width', cssValue: '600px' }
```

Utility-class widths (`w-64`, `max-w-md`) are
intentionally NOT evaluated because jsdom does not
load the Tailwind stylesheet; their resolved values
are unknown in this environment. The inline-style
scan still catches the common foot-gun of hardcoding
a pixel value that busts a 375-pixel-wide phone.

Non-pixel values (`width: auto`, `width: 100%`,
`width: 20em`, `width: calc(...)`) are skipped. The
boundary case where `width === viewportWidth` is NOT
flagged (only strictly greater).

### `runResponsiveSmoke({ render, breakpoints? })`

Iterates over `RESPONSIVE_BREAKPOINTS` (or the
caller's override), installs the harness, invokes
`render(width, name)` at each step, and collects:

```ts
{ width, name, ok, inlineOverflows, errorMessage? }
```

Render exceptions are captured into `errorMessage`
rather than thrown -- a single failing width does not
poison the rest of the sweep, so the report shows
every breakpoint's result.

The helper restores the harness on completion via a
`try { ... } finally { harness.restore(); }` block.

### `expectAllResponsiveOk(reports)`

Throws a human-readable Error when any report
failed; no-ops when every report is `ok`.

## Adoption pattern

```tsx
const reports = runResponsiveSmoke({
  render: () => {
    const { container, unmount } = render(<MyPage />);
    const snapshot = container.cloneNode(true) as Element;
    unmount();
    return snapshot;
  },
});
expectAllResponsiveOk(reports);
```

The `cloneNode(true) + unmount()` pattern snapshots
the rendered DOM before the React tree is torn down,
so the next iteration starts from a clean DOM state.
The harness owns the matchMedia + innerWidth state;
the test owns the React tree lifecycle.

## Out of scope

- **Pixel-perfect geometry assertion.** jsdom does
  not compute layout. A Playwright follow-up can
  re-enable `expect(element.getBoundingClientRect().
  width).toBeLessThanOrEqual(viewportWidth)` against
  a real browser render.
- **Per-page responsive smoke for every page.** The
  helper is ready to drop into any existing page
  test. Adding the same 5-line block to Queue.tsx /
  Workers.tsx / Sessions.tsx / Snapshots.tsx etc. is
  a per-page exercise that can land incrementally.
  The two demonstration adopters in this patch
  validate the helper contract and serve as a
  copy-paste template.
- **Existing-layout fixes.** No page-level layout
  bugs surfaced from the UIDemoRoute / FeatureFlags
  smoke pass at any of the 4 widths; no per-page
  layout fixes are bundled here.

## Summary

`ChartLineMomentumRoc` is a pure-SVG dual-panel React primitive
that plots the close on the top panel and the Rate-of-Change (ROC)
momentum oscillator on the bottom panel:

```
ROC[i] = (close[i] - close[i - length]) / close[i - length] * 100
```

Defaults: `length = 10`. Bars before `i = length` are warmup nulls
(need a valid lookback close). When `close[i - length] == 0` ROC is
`null` (divide-by-zero guard). `-0` from `0 / negative_K` is
normalized to `+0` so the bit-exact zero anchor holds for negative
K too.

## Bit-exact anchors

- **CONST close** (`close = K` constant, `K != 0`):
  `close[i] = close[i - length] = K`, so the numerator is exactly
  zero and `ROC = 0 / K * 100 = 0` past warmup. The integration
  sweep verifies this across positive and negative K and several
  lengths.

- **GEOMETRIC close = 2^i** (`close[i] = 2 ** i`): every close is
  an exact power of two (representable in IEEE 754). For any
  `i >= length`:

      close[i] - close[i - length]
        = 2^i - 2^(i - length)
        = 2^(i - length) * (2^length - 1)
      close[i - length] = 2^(i - length)
      ROC = (2^length - 1) * 100   (exact integer)

  The integration test sweeps `length` in `{2, 3, 4, 5}` (small
  enough to keep `2^length` representable in 53-bit floats).

- `close = 0` past warmup is the singular case: the lookback close
  is zero so ROC is `null`. The test verifies all-null output.

## Zone classifier

- `strong-up` -- ROC >= 10
- `above` -- 0 < ROC < 10
- `at` -- ROC == 0
- `below` -- -10 < ROC < 0
- `strong-down` -- ROC <= -10
- `none` -- ROC is null (warmup or singular)

## Out of scope

- No new dependencies.
- No canvas, no third-party chart libs.
- This primitive uses the percentage Rate-of-Change formulation
  (TradingView / Schwager convention). A momentum-as-difference
  variant (`close - close[i - length]`) could ship in a future
  dispatch.

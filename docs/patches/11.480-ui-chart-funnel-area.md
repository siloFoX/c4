## Summary

Add a new `<ChartFunnelArea>` UI primitive in
`web/src/components/ui/chart-funnel-area.tsx` -- pure-SVG
**area-funnel** variant where each stage's **vertical
height is proportional to its value share** and the
funnel walls taper smoothly from the full width at the
top to a configurable neck-ratio width at the bottom.
Distinct from `<ChartFunnel>` (11.443) which sizes each
trapezoid's *width* by value with equal heights; in
ChartFunnelArea every stage is the same width-band but
its area (height x mean width) is proportional to value,
matching the common "stage by share" reading common in
analytics dashboards.

## API

```ts
interface ChartFunnelAreaStage {
  id: string;
  label: string;
  value: number;
  color?: string;

interface ChartFunnelAreaProps {
  stages: readonly ChartFunnelAreaStage[];

  showLabels?: boolean;       // default true
  showValues?: boolean;       // default true

  formatPercent?: (p: number) => string;

  onStageClick?: (args: {

  }) => void;
  style?: CSSProperties;
}
```

## Behaviour

- Renders `<div role="region">` wrapping
  `<svg role="img">` with the same `aria-label`.
- Layout: `computeFunnelAreaLayout` walks each stage:
  - Height is `(value / total) * heightBudget`
    (heightBudget = `innerH - (N-1)*stageGap`).
  - Width tapers linearly with cumulative share:
    `width = topWidthFull + (bottomWidthFull -
    topWidthFull) * cumulativeShare`, where
    `bottomWidthFull = innerW * neckRatio`. Each
    stage carries the top/bottom widths at its
    boundaries so the previous stage's bottom width
    equals the next stage's top width (continuous
    funnel walls).
  - `neckRatio` is clamped to `[0, 1]` (0 collapses
    to a point; 1 is a straight column).
- Conversion rate per stage `i > 0` is
  `value[i] / value[i-1]` (clamped to 0 when
  previous value is <= 0 or non-finite). The first
  stage's conversion rate is 1; its drop rate is 0.
- Optional in-trapezoid label + value text renders
  at the centroid of each stage when the stage
  height >= `labelMinHeight`. Optional conversion
  badge renders to the right of each stage `i > 0`
  when `showConversionRates=true`.
- Non-positive / non-finite values still occupy a
  slot but render with height 0 so the funnel
  geometry stays continuous.
- Hover/focus on a stage dims the others to opacity
  0.4 (focused stage dominates) and opens a tooltip
  with label, value, share against total, and the
  conversion rate (for stages `i > 0`).
- Mount animation:
  `motion-safe:animate-fade-in` per stage group.
  `animate=false` removes the class.
- Default ARIA description: `"Area funnel with N
  stages, total <total>. Top stage <first.label>
  <first.value>, end stage <last.label> <last.value>
  (<conversion> conversion)."` Empty / zero -> `"No
  data"`; `ariaDescription` overrides.

```ts
computeFunnelAreaLayout(input): ChartFunnelAreaLayoutStage[]
describeFunnelAreaChart(stages, formatValue?): string
```

- `computeFunnelAreaLayout` returns
  `{ id, label, index, value, share, cumulativeShare,
  color, yTop, yBottom, topWidth, bottomWidth,
  centerX, height, path, conversionRate, dropRate }[]`.
  Empty / zero total / non-positive inner -> `[]`.
- `getFunnelAreaConversionRate(current, previous)`
  returns 0 when previous <= 0 or either argument is
  non-finite.

## Pairs with existing primitives

- `<ChartFunnel>` (11.443) -- width-encoded funnel
  sibling (equal heights, widths proportional).
- `<ChartWaterfall>` (11.450) -- delta-based
  alternative for stepwise totals.
- `<ChartSankey>` (11.448) -- multi-stage flow
  alternative with arbitrary edges.

## Out of scope

- Multi-segment funnels with branching paths --
  v1 is single-track; host composes
  `<ChartSankey>` when branching is needed.
- Animated transitions between value changes --
  snap v1; mount fade only.
- Per-stage drilldowns -- host handles via
  `onStageClick`.
- Reversed (bottom-wide / top-narrow) variant --
  v1 always tapers top-to-bottom; host flips by
  reversing the stages array.

# 11.538 -- web/src/components/ui/chart-line-kalman.tsx

Pure-SVG **1D Kalman filter** smoothing overlay. Each series
takes noisy observations `{x, y}`, runs a one-dimensional random-
walk Kalman filter forward, and renders three coordinated
elements on the same axes: the dimmed raw observations line, the
smoothed estimate line in the series color, and an uncertainty
band at the posterior `+/- kSigma * sqrt(P_k)`. Per-point dots
are tinted along a gradient from `gainLowColor` (low gain --
filter trusting its model) to `gainHighColor` (high gain --
filter trusting the measurement). A Kalman-gain badge in the
top-left calls out the final gain of the series with the
strongest absolute gain.

## Filter equations

For each step k from initial state (x_0, P_0):

- **Predict**: `x_pred = x_{k-1}`, `P_pred = P_{k-1} + Q`
- **Gain**:    `K_k = P_pred / (P_pred + R)`
- **Update**:  `x_k = x_pred + K_k * (z_k - x_pred)`,
                `P_k = (1 - K_k) * P_pred`

Where `Q` = process noise variance, `R` = measurement noise
variance, `P_0` = initial covariance, `x_0` = initial estimate
(defaults to the first observation).

## Distinct from related primitives

- `<ChartLineMovingAvg>` (11.513): linear SMA smoothing with a
  fixed window. Kalman is **recursive Bayesian filtering** with
  a posterior covariance that adapts each step.
- `<ChartLineConfidence>` (11.502): per-point upper / lower
  bounds supplied EXTERNALLY. Kalman's band is **derived from
  the filter's own posterior variance** `P_k`.
- `<ChartLineBollinger>` (11.537): rolling SMA +/- k * rolling
  std over a fixed window. Kalman is recursive and depends on
  all past samples through the Q/R model, not a windowed view.
- `<ChartLineControl>` (11.534): global mean +/- k*sigma (SPC).
  Kalman bounds are **per-step adaptive**, shrinking as the
  filter gains certainty.
- `<ChartLineAnomaly>` (11.524): per-point z-score outliers.
  Kalman doesn't flag outliers -- it tracks the optimal
  Bayesian estimate; the innovation (obs - predicted) is
  exposed per-point so callers can build their own outlier
  rules.
- `<ChartLineForecast>` (11.519): historical solid + forecast
  dashed split on time axis. Kalman smooths observed data
  without splitting the timeline.

The canonical recursive-Bayesian smoother used in sensor
fusion, signal tracking, and any context where the question is
"what is the underlying state given noisy observations?".

## Public surface

- `<ChartLineKalman series={...} />`.
- `series: ChartLineKalmanSeries[]` -- `{id, label, data,
  color?, processNoise?, measurementNoise?, initialEstimate?,
  initialVariance?, kSigma?}`. Per-series Q/R/P0/x0 overrides
  always beat chart-level defaults.
- `processNoise` Q (default `0.01`) -- variance of the random-
  walk innovation at each step.
- `measurementNoise` R (default `1`) -- variance of the
  measurement noise. The ratio Q/R controls how reactive the
  filter is.
- `initialEstimate` x_0 -- starting estimate. Defaults to the
  first observation (per-series).
- `initialVariance` P_0 (default `1`) -- starting posterior
  variance.
- `kSigma` (default `2`) -- multiplier for the uncertainty band.
- `gainHighColor` / `gainLowColor` -- per-point dot gradient
  endpoints.
- `obsColor` -- raw observation line color.
- `bandColor` -- uncertainty band stroke + fill.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showGainBadge`, `showBand`, `showObservations`,
  `animate`.
- Controlled + uncontrolled visibility via `hiddenSeries` /
  `defaultHiddenSeries` / `onHiddenSeriesChange`.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX` / `formatGain`.
- `onPointClick({series, point})`.

## Pure helpers

- `getLineKalmanDefaultColor(index)` -- 10-color palette cycle.
- `getLineKalmanFinitePoints(points)` -- finite-only filter.
- `normaliseLineKalmanNoise(value, fallback)` -- non-finite /
  negative -> fallback; zero allowed.
- `normaliseLineKalmanKSigma(value)` -- clamps to > 0; non-
  finite / non-positive -> default 2.
- `runLineKalmanFilter(points, options?)` -- canonical filter
  pass. Sorts ascending; drops non-finite; returns per-point
  `{index, x, observation, predicted, predictedVariance,
  estimate, variance, gain, innovation, upper, lower}`. The
  filter assumes a random-walk process model
  (`x_k = x_{k-1} + w`). Empty / null -> [].
- `computeLineKalmanLayout({series, ...})` -- returns
  `{series, xTicks, yTicks, xMin, xMax, yMin, yMax,
  innerWidth, innerHeight, totalPoints, visibleSeriesCount}`.
  Y range expanded to cover the upper / lower band. Per-series
  carries `{id, label, color, processNoise, measurementNoise,
  initialVariance, initialEstimate, kSigma, points, obsPath,
  estPath, bandPath, finiteCount, totalCount, meanGain,
  maxGain, minGain, finalVariance, finalEstimate, finalGain,
  rmseObservation}`. Per-point carries projected `{..., px,
  obsPy, estPy, upperPy, lowerPy}`.
- `describeLineKalmanChart(series, options?)` -- screen-reader
  description with per-series Q / R / final estimate / final
  gain.

## Filter behaviour

- With **Q = 0** (no process noise), the posterior variance
  collapses asymptotically toward 0 and the Kalman gain
  approaches 0 -- the filter becomes the running average and
  ignores new observations.
- With **Q > 0**, the variance reaches a non-zero steady state
  and the gain stabilises at a fixed value depending on the
  Q/R ratio -- the filter remains responsive to new data
  forever.
- The **innovation** `obs - predicted` is exposed per-point
  so callers can build outlier detection or model-quality
  rules on top of the filter output.
- When `initialEstimate` is unspecified, the filter seeds with
  the first observation so `innovation[0] = 0` and the first
  estimate equals the first observation regardless of the gain.

## Tooltip

Hovering a dot shows the series label in its colour, x, an
`obs:` row, a bold `est:` row, a `var:` row, a gain-coloured
`gain:` row, and an `innov:` row (the per-point innovation).

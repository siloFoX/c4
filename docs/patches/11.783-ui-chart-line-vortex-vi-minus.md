## Summary

`ChartLineVortexViMinus` is the companion oscillator to
`ChartLineVortexViPlus` (TODO 11.782, shipped in v1.11.800). It plots
the close on the top panel and a Vortex Indicator -VI oscillator on
the bottom panel:

```
VM-[i] = |low[i] - high[i - 1]|       (i >= 1; null at i = 0)
TR[i]  = max(high[i] - low[i],
             |high[i] - close[i - 1]|,
             |low[i] - close[i - 1]|)  (bar 0 falls back to high-low)
-VI[i] = sum(VM-, [i - length + 1, i]) /
         sum(TR,  [i - length + 1, i])
```

Defaults: `length = 14`. Bar 0 has no prior high, so VM-[0] is `null`
and the rolling sum sets -VI to `null` for bars `0 .. length - 1`. When
the TR sum is zero (singular: completely flat bars) -VI is `null`.

## Bit-exact anchor

For **CONST_BAR** (high `H`, low `L`, close `C` constant at every bar,
with `H > L` and `L <= C <= H`):

- `VM-[i] = |L - H| = H - L` for every `i >= 1`.
- `TR[i] = max(H - L, |H - C|, |L - C|) = H - L` for every `i >= 1`.
- Therefore for any bar `i >= length` the rolling sums both equal
  `length * (H - L)`, so:

      -VI = (length * (H - L)) / (length * (H - L)) = 1

  **bit-exact** -- integer accumulation, exact cancellation.

For **CONST_FLAT** (`H = L = C`): every TR is zero, the rolling sum is
zero, and the component emits `null` rather than dividing by zero.

## Difference from +VI (11.782)

+VI uses `|high[i] - low[i - 1]|` -- the **upward** vortex movement.
-VI uses `|low[i] - high[i - 1]|` -- the **downward** vortex movement.
In a constant H/L/C regime the two formulas collapse to the same
quantity (`H - L`) and both oscillators read exactly 1. They diverge
when bar-to-bar gaps appear; a trader typically crosses +VI vs -VI to
read trend direction.

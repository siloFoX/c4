# Patch 11.747 -- UI: chart-line-kaufman-efficiency primitive

Released as `v1.11.765` on 2026-05-26.

## Scope

Add `<ChartLineKaufmanEfficiency>` -- a pure-SVG two-panel chart
with a Perry Kaufman **Efficiency Ratio** panel reading the net
price change over the sum of absolute single-bar changes across
the lookback.

## Definition

Inputs:

- `data: ChartLineKaufmanEfficiencyPoint[]` where
  `ChartLineKaufmanEfficiencyPoint = { x; close }`. Bars with
  non-finite `x` / `close` are dropped.
- `period: number` -- the lookback. Default `10`. Normalized to
  an integer `>= 2`.
- `threshold: number` -- the low/medium boundary; high starts at
  `2 * threshold`. Default `0.3`. Normalized to a finite in `(0,
  1)`.

Algorithm (per bar `i >= period`):

```
net    = abs(close[i] - close[i - period])
volSum = sum(abs(close[j] - close[j - 1])) for j in [i - period + 1, i]
ER[i]  = net / volSum
```

Bounded `[0, 1]`. The first `period` bars are null. A bar with a
zero volSum (every delta in the window is zero) is also null.

## Bit-exact algebraic anchors

**CONST_FLAT** (`close == K`): every delta is `K - K = 0`. Both
`net` and `volSum` are zero, so the bar is null at every position.

**Monotone rising** (`close[j+1] - close[j] = +1`): `net =
abs((j + period) - j) = period`. Each `|delta_j| = 1`, so `volSum
= period`. `ER = period / period = 1` bit-exact.

**Monotone falling** (`close[j+1] - close[j] = -1`): `net = abs(-1
* period) = period`. `volSum = period`. `ER = 1` bit-exact.

**Alternating** (`close = [10, 11, 10, 11, 10, ...]` period 4):
at bar 4, `close[4] = 10 = close[0]`, so `net = 0`. `volSum =
|1| + |-1| + |1| + |-1| = 4`. `ER = 0 / 4 = 0` bit-exact at
every defined bar (the window repeats the same alternation).

**Asymmetric anchor** (`close = [10, 11, 12, 11, 12]` period 4):
at bar 4, `net = |12 - 10| = 2`, `volSum = |1| + |1| + |-1| + |1|
= 4`. `ER[4] = 2 / 4 = 0.5` bit-exact.

**Translation invariance**: shifting every close by `+k` leaves
every `close[j] - close[j - 1]` unchanged, so `net`, `volSum`, and
their ratio are unchanged bit-exact.

**Scale invariance**: multiplying every close by `s` scales `net`
and `volSum` by the same `s`, so the ratio is unchanged bit-exact
for any positive integer `s`.

## Component

`forwardRef<HTMLDivElement, ChartLineKaufmanEfficiencyProps>`
with displayName. Renders:

- A region-role outer `<div data-section="chart-line-kaufman-efficiency">`.
- An `<svg role="img">` with an `aria-describedby` description.
- Top panel: close line, optional dots.
- Bottom panel: ER line, zone-coloured markers (high / medium /
  low / none), zero and one reference lines, dashed `threshold`
  reference line.
- Legend toggle for `price` / `er`, plus a `showEr` prop and an
  `onPointClick` handler.
- ARS design-system tokens for colour and motion (`motion-safe:
  animate-fade-in`).

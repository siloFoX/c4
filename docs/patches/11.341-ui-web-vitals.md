## Goal

The dispatch lists:

- Build `web/src/lib/web-vitals.ts` -- reports LCP /
  FID / CLS / INP via the `web-vitals` library to
  `console.log` in dev, optional reporter prop in
  prod.
- Wire into `App.tsx`.

This patch ships the lib + 11 unit tests + the App.tsx
wiring. **FID is omitted** because `web-vitals` v5
dropped `onFID` -- INP replaces it as the canonical
input-responsiveness Core Web Vital. The companion
test documents the absence so future operators do
not search for the missing handler.

## Public API

```ts
export function initWebVitals(options?: {
  onReport?: (report: WebVitalsReport) => void;
  forceDevLog?: boolean;
  forceQuiet?: boolean;
}): void;

export interface WebVitalsReport {
  name: 'LCP' | 'INP' | 'CLS' | 'TTFB';
  value: number;
  id: string;
  rating: 'good' | 'needs-improvement' | 'poor';
  delta: number;
  navigationType: string;
}

export function __resetWebVitalsForTest(): void; // test-only
```

## Sink behaviour

| condition | behaviour |
| --- | --- |
| `import.meta.env.DEV === true` (default) | `console.log("[web-vitals] LCP=1234.00 (good)")` per metric |
| `forceDevLog === true` | log even in prod |
| `forceQuiet === true` | suppress log entirely |
| `onReport` set | always invoked, regardless of log state |

Reporter throws are swallowed -- a buggy RUM sink
cannot crash the page render. The init function is
idempotent (a second call is a no-op via an internal
flag); HMR / test-driven re-mounts do not double-
register listeners.

## Metrics covered

| metric | meaning |
| --- | --- |
| `LCP` | Largest Contentful Paint -- core CWV, "is the page rendered?" |
| `INP` | Interaction to Next Paint -- core CWV, "how responsive?" |
| `CLS` | Cumulative Layout Shift -- core CWV, "is layout stable?" |
| `TTFB` | Time to First Byte -- not a CWV but the most useful network-side signal |

FID is omitted (see "Goal" above).

## App.tsx wiring

```tsx
useEffect(() => {
  initWebVitals();
}, []);
```

The hook lives at the top of the App component so it
fires once on mount, before any user interaction. The
`web-vitals` library handles `visibilitychange`
transitions internally so CLS / INP capture the
delayed-final-value semantics correctly.

## Adoption pattern (production reporter)

```tsx
useEffect(() => {
  initWebVitals({
    onReport: (report) => {
      void fetch('/api/rum', {
        method: 'POST',
        headers: { 'content-type': 'application/json' },
        body: JSON.stringify(report),
      });
    },
  });
}, []);
```

The reporter receives the canonical `WebVitalsReport`
shape so the remote sink can pivot on
`name` / `rating` / `value` without knowing about the
`web-vitals` library's internal Metric type.

## Tests

`web-vitals.test.ts` -- 11 unit cases with
`vi.mock('web-vitals')` that captures the registered
callbacks so each test fires a synthetic metric
without waiting for a real paint:

1. Handlers registered for LCP / INP / CLS / TTFB.
2. Idempotent (second init is a no-op).
3. `onReport` invoked for every metric.
4. Report payload shape matches `WebVitalsReport`.
5. Reporter throws swallowed.
6. `forceDevLog=true` produces a console.log entry.
7. `forceQuiet=true` suppresses the console.log.
8. `forceQuiet` does NOT suppress the reporter.
9. Runs with no options without throwing.
10. Dev-log message shape:
    `[web-vitals] <name>=<value.toFixed(2)> (<rating>)`.
11. `__resetWebVitalsForTest()` allows
    re-initialisation in subsequent tests.

11/11 pass against vitest 4.1.5 + jsdom 29.1.1.

## Bundle impact

`web-vitals` v5 ships ~3 KB gzipped. The vendor chunk
gains a similarly-small entry and remains comfortably
under the v1.11.358 800 KB vendor budget. No budget
bump needed.

## Out of scope

- **Remote RUM endpoint.** The `onReport` hook is
  documented; the production reporter wires it when
  the deployment has a metrics destination.
- **FID polyfill.** FID is deprecated; INP is the
  modern replacement.
- **Page-by-page metrics segmentation.** The
  library reports per-page totals. A future patch
  could pivot the report payload by `route` (e.g.,
  using `topView`) before posting to the remote
  sink.

## Summary

New `ChartLineSsl` UI primitive in
`web/src/components/ui/chart-line-ssl.tsx` -- a pure-SVG single-panel line
chart with an **SSL Channel** overlay flipping between the high and low
moving averages on a close cross. The close line and the two SSL lines
share one panel.

## The computation

The SSL Channel runs a moving average over the bar highs and another over
the bar lows. It needs high, low and close per bar, so `ChartLineSslPoint`
carries `{x, high, low, close}`.

- `computeLineSslSma(values, period)` -- the simple moving average run over
  the highs and over the lows.
- `computeLineSslHlv(closes, smaHigh, smaLow)` -- the direction state
  machine: +1 once the close crosses above the high moving average, -1
  once it crosses below the low moving average, carried between crosses; a
  bar without both moving averages, or before any cross, is 0.
- `computeLineSsl(bars, period)` -- composes the pipeline: in an uptrend
  the SSL Up line rides the high average and the SSL Down line rides the
  low average; in a downtrend they swap. The crossover of the two lines
  marks the trend change.

Each defined bar is classified by the direction: **up** or **down**.

## Worked fixture

The moving average is exact for integer windows that divide cleanly, and
the direction state machine and the line swap are exact comparisons and
selections -- so the whole SSL pipeline is exact.

The fixture is 7 OHLC bars read with period 2 (every high and low even, so
the period-2 averages are exact integers). The close crosses above the
high average at bar 1, holds, crosses below the low average at bar 3,
holds, then crosses back above at bar 5. The direction state
`[0,1,1,-1,-1,1,1]`, the SSL Up line `[.,35,45,19,6,45,65]` and the SSL
Down line `[.,15,25,47,37,12,30]` are all asserted with `toEqual`. The
direction state machine is also tested directly: it flips up on a close
above the high average, down on a close below the low average, carries
between crosses, stays 0 before any cross and does not flip when the close
only equals the average.

## API

`data` (a `ChartLineSslPoint[]`), `period` (default 10, coerced to an
integer >= 1), sizing/format/a11y props, `strokeWidth` / `dotRadius`, the
colour props, the toggle flags (`showAxis` / `showGrid` / `showDots`
default false / `showSslUp` / `showSslDown` / `showMarkers` / `showTooltip`
/ `showConfigBadge` / `showLegend` / `animate`), controlled + uncontrolled
`hiddenSeries` (keys `price` / `sslUp` / `sslDown`), `formatValue` /
`formatX`, `onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineSslFinitePoints`, `normalizeLineSslPeriod`,
`computeLineSslSma`, `computeLineSslHlv`, `computeLineSsl`,
`classifyLineSslZone`, `runLineSsl`, `computeLineSslLayout`,
`describeLineSslChart`, plus the `DEFAULT_CHART_LINE_SSL_*` constants and
the `ChartLineSslZone` type. Every exported name carries a `LineSsl` infix
so the `export *` barrel line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen `data-section="chart-line-ssl-aria-desc"`
description span; the close line, both SSL lines and every dot / marker
are `role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-period`, `data-hlv-final`,
`data-up-count`, `data-down-count`, `data-total-points` and
`data-animate`.

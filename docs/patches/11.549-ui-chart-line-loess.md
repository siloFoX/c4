# 11.549 -- web/src/components/ui/chart-line-loess.tsx

Pure-SVG line chart with a **LOESS** (locally weighted
scatterplot smoothing) overlay. For each query point `x_i`,
the smoother:

1. Finds the `k = ceil(bandwidth * N)` nearest neighbours of
   `x_i` by absolute distance (with `k >= degree + 1` so the
   fit has enough degrees of freedom).
2. Determines the maximum neighbour distance `d_max`.
3. Computes a **tricube weight** for each neighbour:
   `w_j = (1 - (d_j / d_max)^3)^3` for `d_j < d_max`, else 0.
4. Fits a weighted polynomial of degree `p` (0, 1, or 2) via
   weighted least squares (Gauss-Jordan on the normal
   equations).
5. Evaluates the polynomial at `x_i` -- that's the smoothed
   value.

Edges have no special treatment because LOESS naturally
includes neighbours from one side at the edges of the
domain.

## Why LOESS over Savitzky-Golay or SMA?

LOESS is more flexible than SG and far more flexible than
SMA in two ways:

- **k-NN neighbourhood (not fixed window)**: when data is
  non-uniformly spaced, LOESS still uses a CONSTANT NUMBER
  of neighbours per fit, so the smoother adapts to the
  local sample density. SG uses a fixed-index window, which
  breaks under non-uniform spacing.
- **Tricube weights (not uniform weights)**: closer
  neighbours contribute more, so the fit at `x_i` is biased
  toward the local behaviour rather than averaging across
  the whole window. SG gives equal weight to every sample
  in the window.

Verified by test: degree-1 LOESS reproduces a linear input
EXACTLY, degree-2 LOESS reproduces a quadratic input
EXACTLY, and constant input passes through unchanged at
all degrees.

## Distinct from related smoothers

- `<ChartLineSavgol>` (11.547): polynomial fit on a FIXED
  window with EQUAL weights. LOESS uses VARIABLE k-NN
  neighbourhood and TRICUBE WEIGHTS.
- `<ChartLineMovingAvg>` (11.513): SMA == uniform-weight,
  degree-0 fit on a fixed window. LOESS handles
  non-uniform spacing.
- `<ChartLineEwma>` (11.539): exponential decay, recursive
  single-parameter smoother. LOESS is per-point local
  regression.
- `<ChartLineKalman>` (11.538): Bayesian filter with Q/R
  noise model. LOESS is deterministic local regression.
- `<ChartLineHampel>` (11.548): robust outlier filter,
  replaces outliers with window median. LOESS smooths
  through all points (use a robust variant for outlier
  resistance).
- `<ChartLineBollinger>` (11.537): rolling SMA + envelope.
  LOESS has no envelope.
- `<ChartLineDecompose>` (11.543): seasonal split with
  centered moving average. Different goal.

The canonical R `loess` / scipy LOESS reference smoother
used in exploratory data analysis where the relationship
between x and y is unknown and should be discovered from
the data ("let the data speak").

## Public surface

- `<ChartLineLoess series={...} />`.
- `series: ChartLineLoessSeries[]` -- `{id, label, data,
  color?, bandwidth?, degree?}`. Per-series overrides
  always beat chart-level.
- `bandwidth` (default 0.3; clamped to (0, 1]) -- fraction
  of points to use as the neighbourhood (R `loess` calls
  this `span`). 0.3 = 30% of points.
- `degree` (default 2; clamped to {0, 1, 2}) -- polynomial
  degree of the local fit.
- `rawColor`, `residualPosColor`, `residualNegColor`.
- Toggle flags: `showAxis`, `showGrid`, `showDots`,
  `showLegend`, `showTooltip`, `showConfigBadge`,
  `showRaw`, `showResidualSticks`, `animate`.
- Controlled + uncontrolled visibility.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX` / `formatBandwidth`.
- `onPointClick({series, point})`.

## Pure helpers

- `getLineLoessDefaultColor(index)` -- 10-color palette
  cycle.
- `getLineLoessFinitePoints(points)` -- finite-only
  filter.
- `normaliseLineLoessBandwidth(value)` -- clamps to
  `(0, 1]`; non-finite -> default 0.3.
- `normaliseLineLoessDegree(value)` -- clamps to {0, 1, 2};
  floors fractional; non-finite -> default 2.
- `computeLineLoessTricubeWeight(distance, dMax)` --
  canonical tricube weight `(1 - (d/dMax)^3)^3`. Returns 0
  when `d >= dMax`, when `dMax <= 0`, when `d < 0`, or when
  either argument is non-finite. **Verified canonical
  value at d/dMax = 0.5 is (7/8)^3 = 0.6699...**
- `fitLineLoessWeightedPolynomialAtCenter({xs, ys, weights,
  degree, centerX})` -- canonical weighted polynomial fit;
  returns the polynomial value at `centerX` (computed in
  centred coordinates so the output is just `beta_0`).
  Returns null for empty inputs or zero total weight.
- `classifyLineLoessResidualSign(residual)` -- `'positive'
  | 'negative' | 'zero'`.
- `runLineLoess(points, options?)` -- canonical pipeline.
  Sorts ascending; drops non-finite; per-sample
  `{index, x, raw, smoothed, residual, residualSign,
  neighborhoodSize, neighborhoodWidth}`. The neighbourhood
  count is clamped to at least `degree + 1` so the fit has
  enough degrees of freedom.
- `computeLineLoessLayout({series, ...})` -- projects raw
  + smoothed paths + residual segments. Y range covers
  both raw and smoothed.
- `describeLineLoessChart(series, options?)` -- screen-
  reader description.

## Tooltip

Hovering a dot shows the series label, x, raw, bold
smoothed (or n/a at the edges if the fit failed), residual
(with `+`/`-` prefix), and the LOESS config
`α=<bandwidth>, d=<degree>, k=<neighbour count>`.

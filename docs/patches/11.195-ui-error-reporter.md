# 11.195 -- UI: local error reporting sink

Patch level: 1.11.213.

Adds a strictly local, in-memory error reporting sink for the web UI
plus an operator-facing inspector page. No third-party SDK, no
network traffic.

## What ships

### `web/src/lib/error-reporter.ts`

Module singleton with:

- `ErrorRecord` type:
  `{ id, timestamp, source: 'window' | 'unhandledrejection' | 'react' | 'manual', message, stack?, componentStack?, url? }`.
  `id` prefers `crypto.randomUUID()` and falls back to a monotonic
  counter so tests stay deterministic in environments without crypto.
- A ring buffer capped at the last `50` records, FIFO-evicted on
  overflow. `getAll()` returns the snapshot newest-FIRST (index 0 is
  the most recent record).
- `install()` -- idempotent. Wires `window.addEventListener('error',
  ...)` and `window.addEventListener('unhandledrejection', ...)`,
  plus chains `window.onerror` so any previously-installed handler
  still fires. The internal `installed` flag guards against
  double-wiring; calling `install()` N times still only registers one
  listener of each kind.
- `report(record)` -- manual entry, used by `ErrorBoundary`.
- `clear()` -- empties the buffer, notifies subscribers.
- `subscribe(listener) -> unsubscribe` pattern. `useErrorRecords()`
  React hook wires `useEffect` + `subscribe` and returns the current
  snapshot.
- `downloadJson(filename?)` -- builds an `application/json` Blob from
  `getAll()`, attaches a temporary `<a download>` anchor, clicks it,
  and revokes the object URL on the next tick. Skips entirely when
  `window` / `document` are undefined (SSR-safe).
- `_resetForTests()` -- internal helper used by the vitest suite to
  unwire listeners + clear the buffer between cases.

Message / stack / componentStack are truncated at `MAX_FIELD_LEN`
(8192 chars) so a runaway stack cannot bloat the buffer.

### `web/src/components/ErrorBoundary.tsx` (modified)

Existing class boundary now also calls
`reportError({ source: 'react', message, stack, componentStack })`
inside `componentDidCatch`. The console.error fallback and the
existing fallback UI (Reload / Copy stack / Open GitHub issue / Try
again) are preserved. The report call is wrapped in `try/catch` so
the sink can never throw back into the render path.

### `web/src/pages/ErrorReports.tsx`

Operator-facing inspector under the Diagnostics category:

- Header chip with the live record count.
- Row layout: source `<Chip>` (tone mapped per source: window =
  danger, unhandledrejection = warning, react = primary, manual =
  neutral), localized timestamp, monospace short id, truncated
  message (`100` chars). Clicking the message toggles a detail block
  that shows the full message + `stack` + `componentStack` + `url`.
- Action bar (right side): `Test error` (creates a synthetic
  `manual` record so operators can verify the wiring without forcing
  a real crash), `Download JSON` (blob download), `Clear all`.
- `<EmptyState>` when the buffer is empty.

### `web/src/pages/registry.ts`

New `error-reports` feature entry under the `diagnostics` category
with the `AlertTriangle` lucide icon and the standard
`load: () => import('./ErrorReports')` lazy thunk.

i18n keys `feature.errorReports.label` /
`feature.errorReports.description` added to `en.json` and `ko.json`.

## Tests

`web/src/lib/error-reporter.test.ts` (vitest + jsdom). 8 cases:

1. `report()` appends to the buffer; `getAll()` returns newest-first.
2. Buffer caps at `50` with FIFO eviction (oldest dropped after 60
   reports).
3. `clear()` empties the buffer and notifies subscribers.
4. `subscribe()` fires on `report()`; `unsubscribe()` stops further
   notifications.
5. `install()` is idempotent -- three calls register exactly one
   `error` and one `unhandledrejection` listener.
6. `window.onerror` chain captures the synthetic error and still
   delegates to the previously-installed handler.
7. `addEventListener('error', ...)` handler captures a synthetic
   `ErrorEvent`.
8. `downloadJson()` creates a Blob (`application/json`) and triggers
   an anchor click; `URL.createObjectURL` / `URL.revokeObjectURL` /
   `HTMLAnchorElement.click` are spied + restored.

Run:

```bash
env -C /root/c4-worktree-auto-w179/web \
  /root/c4/web/node_modules/.bin/vitest run src/lib/error-reporter.test.ts
```

All 8 cases pass.

## Non-goals

- No remote sink. Errors stay in the page session; refresh discards
  the buffer.
- `install()` is exported but not invoked automatically -- the
  feature is opt-in. `ErrorBoundary` already reports `react`-source
  errors without needing `install()`.
- No new dependency: no sentry, no bugsnag, no logging service.

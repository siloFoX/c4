# 11.523 -- web/src/components/ui/chart-line-spark-bar.tsx

Pure-SVG **hybrid sparkline** that pairs a compact line chart with
an overlaid bar track for dense, inline use. The canvas is split
vertically into a top line track and a bottom bar track separated
by `trackGap` (default 2px). The line uses one y dimension
(`point.value`); the bars use a second y dimension (`point.bar`),
each normalised independently to fit its own track. Sparkline-sized
defaults (width 160, height 48, padding 4) so the component fits
inline with text or inside table cells.

## Distinct from the full-size chart primitives

- `<ChartLineCumulative>` (11.520), `<ChartLineRate>` (11.521),
  `<ChartLineNormalize>` (11.522): full-size charts with axes,
  legend, larger padding. This primitive is the **sparkline form
  factor** -- compact, axis-light, designed to ship dozens at once
  in a dashboard or table.
- `<ChartLineMovingAvg>` (11.513), `<ChartLineTrend>` (11.512),
  `<ChartLineAreaStacked>` (11.505): single-dimension transforms /
  overlays.
- `<ChartLineMarker>` (11.506): markers on the line, not paired
  bars on a second track.
- `<ChartLineForecast>` (11.519): historical/forecast split, not a
  bar overlay.

The classic "stock close + volume" sparkline pattern: a thin line
of price + a row of volume bars underneath, both squeezed into a
few pixels of vertical real estate.

## Public surface

- `<ChartLineSparkBar series={...} />`.
- Per-series `{id, label, data, lineColor?, barColor?,
  positiveBarColor?, negativeBarColor?}`. `data` entries are
  `{x, value, bar}` -- one entry drives the line y AND the bar
  magnitude at the same x.
- `barRatio` (default 0.55) -- share of the inner canvas allocated
  to the bar track; clamped to `[0.05, 0.95]`.
- `signedBars` (default false) -- when true, positive bars use
  `positiveBarColor` (green) and negative bars use `negativeBarColor`
  (red).
- Toggle flags: `showLine`, `showBars`, `showMinMaxDots`,
  `showLastValuePill`, `showLastValueDot`, `showTooltip`,
  `showLegend` (default FALSE for the sparkline form factor),
  `animate`.
- Bounds overrides: `xMin/xMax/valueMin/valueMax/barMin/barMax` --
  separate y bounds for the line vs the bars so adopters can keep
  each track stable across re-renders.
- Sparkline-specific knobs: `barGap` (default 1) controls the gap
  between bar slots; `trackGap` (default 2) controls the vertical
  gap between line + bar tracks.
- Standard a11y + format props (`formatValue`, `formatX`,
  `formatBar`).
- Controlled + uncontrolled `hiddenSeries`.
- `onPointClick({series, point})`.

## Pure helpers

- `getLineSparkBarDefaultColor(index)` -- 10-color palette cycle.
- `getLineSparkBarFinitePoints(points)` -- finite-only filter
  requiring **all three** fields (`x`, `value`, `bar`) finite.
- `computeLineSparkBarBounds(series, hidden?)` -- joint bounds with
  these guarantees:
  - `barMin` is clamped to `0` when all observed bars are
    non-negative (positive-only tracks ground at the bottom of
    their zone).
  - `barMax` is clamped to `0` when all observed bars are
    non-positive.
  - Equal-bounds get expanded by `+/-0.5`.
  - Empty / no-finite -> unit defaults `(0..1)`.
- `computeLineSparkBarLayout({series, hiddenSeries?, width, height,
  padding, barRatio?, barGap?, trackGap?, ...})` -- returns
  `{series, bounds, innerWidth, innerHeight, barTrackTop,
  barTrackBottom, lineTrackTop, lineTrackBottom, barZeroY,
  totalPoints, visibleSeriesCount}`. Each layout series carries
  `{id, label, index (stable original), lineColor, barColor,
  positiveBarColor, negativeBarColor, points, path, finalValue,
  minValue, maxValue, totalBar, finiteCount, totalCount, minIndex,
  maxIndex}`. Per-point `{index, x, value, bar, px, py, barX,
  barY, barWidth, barHeight, barSign, isMin, isMax}` with the bar
  rect signed around `barZeroY` (positive bars grow upward from
  zero, negative bars grow downward).
- `describeLineSparkBarChart(series, hidden?, formatValue?)` --
  builds the screen-reader description.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Per-line path + per-hit-circle are `role="graphics-symbol"` +
  `tabIndex={0}`. Hit aria-label includes value + bar.
- Every node carries `data-section="chart-line-spark-bar-..."` for
  test selectors.
- Series groups expose `data-series-final-value`,
  `data-series-min-value`, `data-series-max-value`,
  `data-series-total-bar`, `data-series-line-color`,
  `data-series-bar-color`, `data-series-finite-count`.
- Bars expose `data-bar-sign` (`'positive' | 'negative' | 'zero'`).
- Root mirrors `data-signed-bars`.

## Interaction model

The sparkline form factor has tiny dots; pointer hits land more
reliably on invisible **hit-circle** overlays centered at each
point (radius derived from `dotRadius` and `barWidth`). Hover
opens a small tooltip with x / value / bar rows. Click forwards to
`onPointClick`. Hits are kept at the end of the SVG so they sit on
top of bars and lines for hit-testing.

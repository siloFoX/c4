# Patch 11.759 -- UI: chart-line-wma-cross primitive

Released as `v1.11.777` on 2026-05-26.

## Scope

Add `<ChartLineWmaCross>` -- a pure-SVG single-panel chart with a
Weighted-MA fast-over-slow **crossover** overlay marking up-cross
and down-cross events.

## Definition

Inputs:

- `data: ChartLineWmaCrossPoint[]` where `ChartLineWmaCrossPoint
  = { x; close }`. Bars with non-finite fields are dropped.
- `fastLength: number` -- the fast WMA length. Default `9`.
  Normalized to an integer `>= 2`.
- `slowLength: number` -- the slow WMA length. Default `21`.
  Normalized to an integer `>= 2`.

Algorithm (for each lookback `L`):

```
WMA[i] = sum_{j=1..L} (j * close[i - L + j]) / sum_{j=1..L} j
```

The most recent bar has weight `L`, the oldest has weight `1`.
The denominator `sum_{j=1..L} j = L * (L + 1) / 2`.

Crosses (per bar `i >= 1`):

```
upCross   at fast[i-1] <= slow[i-1] && fast[i] > slow[i]
downCross at fast[i-1] >= slow[i-1] && fast[i] < slow[i]
```

## Bit-exact algebraic anchors

**CONST_FLAT** (`close == K`): WMA of constant `K` is the
weighted mean of `K`'s, which equals `K`:

```
WMA = sum_{j=1..L} (j * K) / sum_{j=1..L} j = K * sum / sum = K
```

bit-exact at every defined bar, at any length.

**Integer ramp** (`close = c + (j - 1)` over the window, where
`c` is the bar at `j = 1` -- the oldest in the window):

```
WMA = sum_{j=1..L} j * (c + j - 1) / sum_{j=1..L} j
    = (c * sum_{j=1..L} j + sum_{j=1..L} j * (j - 1))
      / sum_{j=1..L} j
    = c + sum_{j=1..L} j * (j - 1) / sum_{j=1..L} j
    = c + (L * (L + 1) * (L - 1) / 3) / (L * (L + 1) / 2)
    = c + 2 * (L - 1) / 3
```

For `L = 4`, this is `c + 2 * 3 / 3 = c + 2` bit-exact integer.
On the `RISING_INT` fixture (`close = i + 10`), at bar `i` the
oldest window value is `c = i - 4 + 1 + 10 = i + 7`, so `WMA[i]
= c + 2 = i + 9` bit-exact at every defined bar.

**Worked anchor** (`close = [0, 1, 2, 3]`, period 4):

```
WMA[3] = (1 * 0 + 2 * 1 + 3 * 2 + 4 * 3) / (1 + 2 + 3 + 4)
       = 20 / 10 = 2
```

bit-exact integer.

**Period-2 worked anchor** (`close = [10, 12]`, period 2):

```
WMA[1] = (1 * 10 + 2 * 12) / (1 + 2) = 34 / 3
```

bit-exact rational (verified with `toBe(34 / 3)` -- JavaScript's
`/` produces the same IEEE 754 division).

**Translation invariance**: shifting every close by `+k` shifts
the weighted mean by exactly `+k` (linearity of the WMA).

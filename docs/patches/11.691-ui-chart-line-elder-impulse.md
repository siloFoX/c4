## Summary

New `ChartLineElderImpulse` UI primitive in
`web/src/components/ui/chart-line-elder-impulse.tsx` -- a pure-SVG
single-panel line chart with an **Elder Impulse** overlay colouring each
bar from the EMA slope and the MACD histogram.

## The computation

Alexander Elder's Impulse System colours each bar from two momentum
readings: the slope of an EMA of the close, and the slope of the MACD
histogram. It is a close-only indicator, so `ChartLineElderImpulsePoint`
carries `{x, value}`.

- `computeLineElderImpulseEma(values, period)` -- an EMA seeded from the
  first value (`EMA[0] = value[0]`).
- `computeLineElderImpulse(values, options)` -- builds the impulse EMA
  (default period 13), the MACD line (fast EMA 12 minus slow EMA 26), the
  signal line (9-EMA of the MACD) and the histogram (MACD minus signal),
  then classifies each bar.
- `classifyLineElderImpulse(emaSlope, histSlope)` -- the colouring rule:
  **green** when both slopes rise, **red** when both fall, **blue** when
  they disagree or either is flat, **none** when a slope is undefined.

Because the EMA is seeded from the first value, the EMA and the MACD
pipeline are defined from the first bar and the impulse class is defined
from the second bar onward (the first bar has no prior slope, so it is
`none`).

## Worked fixture

The classification is a pure function of two slope signs and is tested
exhaustively (green / red / blue / flat / null cases).

For the integrated pipeline a **flat series** gives exact results: the EMA
stays flat, the fast EMA equals the slow EMA so the MACD is exactly zero,
the signal is zero and the histogram is exactly zero. Every slope is zero,
so every bar after the first is **blue**. `[50,50,50,50,50,50]` ->
`ema = [50,50,50,50,50,50]`, `macd = [0,0,0,0,0,0]`,
`hist = [0,0,0,0,0,0]`, `impulse = [none,blue,blue,blue,blue,blue]` -- all
asserted with `toEqual`.

The EMA helper is checked exactly with a dyadic alpha: period 3 gives
`alpha = 2/4 = 0.5`, so `[4,8,16]` -> `[4,6,11]`.

Rising and falling series are checked structurally: a strictly increasing
series has a strictly positive EMA slope on every bar (a weighted average
of strictly smaller prior values is always below the current value), so no
bar can be red -- the test asserts `redCount === 0` and `greenCount >= 1`
for an accelerating series. A strictly decreasing series is the mirror:
`greenCount === 0`, `redCount >= 1`.

## API

`data` (a `ChartLineElderImpulsePoint[]`), `emaPeriod` (default 13),
`fastPeriod` (default 12), `slowPeriod` (default 26), `signalPeriod`
(default 9), sizing/format/a11y props, `strokeWidth` / `dotRadius`, the
colour props, the toggle flags (`showAxis` / `showGrid` / `showEma` /
`showMarkers` / `showTooltip` / `showConfigBadge` / `showLegend` /
`animate`), controlled + uncontrolled `hiddenSeries` (keys `price` /
`ema`), `formatValue` / `formatX`, `onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineElderImpulseFinitePoints`,
`normalizeLineElderImpulsePeriod`, `computeLineElderImpulseEma`,
`classifyLineElderImpulse`, `computeLineElderImpulse`,
`runLineElderImpulse`, `computeLineElderImpulseLayout`,
`describeLineElderImpulseChart`, plus the
`DEFAULT_CHART_LINE_ELDER_IMPULSE_*` constants and the
`ChartLineElderImpulseClass` type. Every exported name carries a
`LineElderImpulse` infix so the `export *` barrel line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen
`data-section="chart-line-elder-impulse-aria-desc"` description span; the
EMA line and every marker are `role="graphics-symbol" tabIndex={0}`. Every
node carries a `data-section`; the root mirrors `data-ema-period`,
`data-fast-period`, `data-slow-period`, `data-signal-period`,
`data-green-count`, `data-red-count`, `data-blue-count`,
`data-total-points`, `data-impulse-final` and `data-animate`.

## Summary

Add a new `<ChartStream>` UI primitive in
`web/src/components/ui/chart-stream.tsx` -- pure-SVG stream
graph (a stacked-area variant with a centered baseline). At
every x position the chart sums the per-series values and
centers the stack vertically around a horizontal baseline,
so the silhouette grows symmetrically up + down. Each
series fills its own band; hovering a band highlights it
and dims the others. Smooth interpolation uses a
Catmull-Rom-to-cubic-Bezier pass on both upper + lower
edges. Per-series click + tooltip with running totals.

## API

```ts
interface ChartStreamSeries {
  id: string;
  label: string;
  data: readonly number[];
  color?: string;
}

interface ChartStreamProps {
  series: readonly ChartStreamSeries[];
  xLabels?: readonly string[];
  width?: number;                 // default 540
  height?: number;                // default 240
  padding?: number;               // default 24
  smooth?: boolean;               // default true
  showLegend?: boolean;           // default true
  showTooltip?: boolean;          // default true
  showAxisLabels?: boolean;       // default true
  highlightOnHover?: boolean;     // default true
  animate?: boolean;              // default true
  className?: string;
  ariaLabel?: string;             // default 'Stream graph'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  formatXLabel?: (label: string, index: number) => string;
  legendPlacement?: 'right' | 'bottom';   // default 'bottom'
  onSeriesClick?: (args: {
    series: ChartStreamSeries;
    index: number;
    total: number;
  }) => void;
}
```

## Behaviour

- Renders a `<div role="region">` wrapping `<svg
  role="img">` carrying the same `aria-label`.
- Centered baseline: at each x, the stack height is
  `total * scale` where `scale = innerHeight / max(total)`
  across the series. The stack is positioned around
  `centerY = padding + innerHeight / 2` so half the
  stack rises above and half drops below -- the
  classic stream-graph silhouette.
- Series order matters: the first series sits in the
  middle of the stack (closest to the baseline) and
  later series stack outward symmetrically. Layer N's
  upper edge meets layer N-1's lower edge so the
  bands tile cleanly.
- Non-finite / negative values are clamped to 0 so the
  stream never inverts.
- Smooth interpolation: `smooth=true` (default) uses a
  Catmull-Rom-to-cubic-Bezier pass on both the upper
  and lower edges. Two-point bands fall back to the
  straight `M / L / Z` form to avoid degenerate
  control points. `smooth=false` keeps the polylines
  straight.
- Hover highlight: hovering a band sets `data-hovered`
  and (when `highlightOnHover=true`) dims the other
  bands to 0.25 fill-opacity so the focused band reads
  first.
- Tooltip: hover opens a tooltip with the series
  label, value at the hovered x, optional `xLabels[i]`,
  and the column total at that x.
- `onSeriesClick` receives `{ series, index, total }`
  so adopters can dispatch drill-downs.
- Mount animation: `motion-safe:animate-fade-in` per
  band on mount. `animate=false` removes the class.
- Default ARIA description summary: `"Stream graph
  with N layers, M samples. <label> sum <value>, ..."` --
  `ariaDescription` overrides; empty series produces
  `"No data"`.
- Legend renders below by default (stream graphs are
  wide); `legendPlacement="right"` switches to the
  vertical legend used elsewhere in the chart family.

## Pure helpers (exported)

```ts
DEFAULT_CHART_STREAM_WIDTH    = 540
DEFAULT_CHART_STREAM_HEIGHT   = 240
DEFAULT_CHART_STREAM_PADDING  = 24

getStreamTotalAt(series, index): number
getStreamMax(series): number
computeStreamLayout(series, innerWidth, innerHeight,
                    paddingX, paddingY): {
  series: ChartStreamSeries;
  points: StreamLayoutPoint[];
}[]
buildStreamPath(points, smooth): string
describeStreamChart(series, formatValue?): string
```

- `getStreamTotalAt` sums series values at index;
  ignores non-finite + non-positive values.
- `getStreamMax` returns the largest column-sum across
  the series; falls back to 1 for empty / zero-length /
  all-non-finite data so the chart still renders.
- `computeStreamLayout` is the canonical layout pass.
  Returns one `{ series, points }` entry per input
  series. Points include `{ x, upperY, lowerY }` so
  adopters can paint custom overlays (e.g., annotation
  pins) at the same vertical positions.
- `buildStreamPath` emits a closed band path. With
  smooth = false, it emits `M ... L ... Z` for the
  upper edge, then the reversed lower edge, then `Z`.
  With smooth = true and >= 3 points, both edges use
  Catmull-Rom-to-cubic-Bezier; < 3 points fall back to
  the polyline form.
- `describeStreamChart` produces the default ARIA
  summary; honours `formatValue`.

## ARIA + data attributes

- Root: `<div role="region">` + `aria-label`.
- SVG: `role="img"` + same `aria-label`.
- sr-only span:
  `data-section="chart-stream-aria-desc"` carries the
  auto-summary or `ariaDescription` override.
- Each band path: `role="graphics-symbol"` +
  `tabIndex={0}`. Aria-label = series label so screen
  readers can describe the band.
- `data-section` on every node:
  - `chart-stream` (root)
  - `chart-stream-canvas`
  - `chart-stream-aria-desc`
  - `chart-stream-svg`
  - `chart-stream-layer` (per-series group)
  - `chart-stream-band` (per-series path)
  - `chart-stream-xlabel`
  - `chart-stream-tooltip` + `-tooltip-label` /
    `-tooltip-value` / `-tooltip-x` /
    `-tooltip-total`
  - `chart-stream-legend` + `-legend-item` /
    `-legend-swatch` / `-legend-label`
- Root mirrors `data-layer-count`, `data-sample-count`,
  `data-smooth`, `data-animate`. Layer groups mirror
  `data-series-id`, `data-series-index`,
  `data-series-color`, `data-hovered`.

## Tests (50 cases)

  lower edge meets layer N+1's upper edge, stack

## Out of scope

- Wiggle / minimum-curvature baseline (Byron + Wattenberg
  2008); v1 uses the centered baseline only. Host can
  pre-process data if a different baseline is needed.
- Negative / signed values (host normalises into
  positives; non-positive values clamp to 0 during
  layout).
- Animated layer transitions on data change (snap v1;
  mount fade only).
- Time-axis ticks (host renders own time-axis below
  the canvas when needed).

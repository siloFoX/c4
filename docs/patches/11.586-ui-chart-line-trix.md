## Summary

New `ChartLineTrix` UI primitive at
`web/src/components/ui/chart-line-trix.tsx` -- a pure-SVG line chart
with a TRIX triple-smoothed exponential moving average oscillator
panel. The value line occupies the top panel; the bottom panel
renders the TRIX line and its signal line on a symmetric y-axis with
a dashed zero centerline. Both panels share one x-axis.

## Indicator math

`computeLineTrixEma(values, period)` is an exponential moving average
seeded with the first value, rolling forward with the smoothing factor
`k = 2/(period+1)`: `ema[i] = value[i]*k + ema[i-1]*(1-k)`.

`computeLineTrix(values, period)` is Jack Hutson's TRIX. The series is
smoothed by three successive EMAs of the same period, and TRIX is the
1-period percentage rate of change of that triple-smoothed line:

```
ema1 = EMA(values, period)
ema2 = EMA(ema1,  period)
ema3 = EMA(ema2,  period)
TRIX[i] = 100 * (ema3[i] - ema3[i-1]) / ema3[i-1]
```

Index 0 has no prior value and reads `null`; an all-zero series reads
TRIX 0 without dividing by zero. `computeLineTrixSignal(trix,
signalPeriod)` is an EMA of the TRIX series, seeded at the first
defined TRIX value.

### Worked fixture

`values = [10, 20, 30, 40, 50, 30, 10]`, `period = 3` (so `k = 0.5`):

```
ema1 = [10, 15,    22.5,   31.25,  40.625,  35.3125,   22.65625  ]
ema2 = [10, 12.5,  17.5,   24.375, 32.5,    33.90625,  28.28125  ]
ema3 = [10, 11.25, 14.375, 19.375, 25.9375, 29.921875, 29.1015625]
trix = [.,  12.5,  27.7778,34.7826,33.8710, 15.3614,   -2.7416   ]
```

The triple-smoothed line peaks then turns down, so TRIX crosses from
positive into negative at the final bar (`trix[6] ~ -2.7416`).

## API

- `getLineTrixFinitePoints(points)` -- drop points with a non-finite
  `x` or `value`.
- `normalizeLineTrixPeriod(period, fallback)` -- coerce a period to a
  positive integer; non-finite or sub-1 falls back, fractional floors.
- `computeLineTrixEma(values, period)` -- a single seeded EMA.
- `computeLineTrix(values, period)` -- the `{ ema1, ema2, ema3, trix }`
  series.
- `computeLineTrixSignal(trix, signalPeriod)` -- the TRIX signal EMA.
- `runLineTrix(points, options?)` -- sort finite points by `x`,
  compute the TRIX and signal series, return `{ series, period,
  signalPeriod, ema3, trix, signal, samples, trixFinal, signalFinal,
  trixMin, trixMax, ok }`. `ok` is false for fewer than two points.
- `computeLineTrixLayout(options)` -- stack a value panel above a TRIX
  panel with a symmetric y-bound covering the TRIX and signal extent.
  Returns panels, ticks, the value path, TRIX path, signal path,
  markers, and the zero centerline position.
- `describeLineTrixChart(data, options?)` -- ARIA description string.

## Component

`ChartLineTrix` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, panel labels, a config
badge (period + signal period), a three-series legend (Value / TRIX /
Signal) with toggle, a dashed zero centerline, the TRIX line, a dashed
signal line, markers on the TRIX line, a hover tooltip exposing the
value, ema3, TRIX, and signal, and a `motion-safe:animate-fade-in`
entrance. Every node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-macd** -- MACD is the difference of two EMAs of
  *different* periods. TRIX applies three EMAs of the *same* period
  and then takes a rate of change, so it filters short-term noise far
  more aggressively and reads as a smoothed momentum oscillator
  centred on zero.
- **vs chart-line-ewma** -- chart-line-ewma plots a single smoothing
  EMA overlaid on the data. chart-line-trix plots the percentage rate
  of change of a *triple*-smoothed EMA in a dedicated sub-panel -- a
  derived oscillator, not a smoothing overlay.

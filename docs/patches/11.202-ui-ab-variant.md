# 11.202 -- UI: A/B variant harness

Status: shipped in v1.11.220 (2026-05-14).
Branch: `c4/auto-ui-ab-variant`.

## What

Component-scoped A/B variant harness for the web UI. Provides
deterministic per-session bucketing, a localStorage override channel
for QA, a React hook for subtree subscription, and a single example
adoption on the Health dashboard hero block.

## Surface

```ts
type VariantId = 'A' | 'B';

interface ExperimentDef {
  name: string;
  splits?: number[]; // default [0.5, 0.5]
}

hashString(input: string): number;                              // FNV-1a 32-bit
assignVariant(name: string, sessionId: string, splits?): 'A'|'B';
getSessionId(): string;                                         // sessionStorage 'c4:session-id'
getVariant(name: string, splits?): 'A' | 'B';                   // override-aware
setVariant(name: string, variant: 'A' | 'B'): void;             // persists + dispatches event
clearVariants(): void;
useABVariant(name: string, splits?): [VariantId, (v) => void];  // React hook
```

Storage keys:

- `sessionStorage['c4:session-id']` -- per-tab stable session id (UUID).
- `localStorage['c4:ab-variants']` -- `{[experiment]: 'A' | 'B'}` operator
  override map.

Event: `window` dispatches `ab-variant-changed` with `CustomEvent` detail
`{ experiment, variant }`.

## Assignment

`assignVariant()` hashes `name + ':' + sessionId` with FNV-1a 32-bit,
buckets the result over the normalised `splits` array, and returns the
variant id at that bucket. Splits are renormalised so any non-negative
weight vector that sums to a positive number resolves to a valid
distribution. The hash is deterministic, so the same `(name, sessionId)`
pair always yields the same variant.

## Override channel

`localStorage['c4:ab-variants']` is the operator escape hatch -- writing
`{ 'health-hero': 'B' }` pins the Health hero variant for QA without
touching the daemon. Malformed JSON / non-`'A' | 'B'` values fall back to
the deterministic assignment so a corrupt row cannot strand a visitor.

## SSR safety

Every `window`, `sessionStorage`, and `localStorage` touch is wrapped
in a `typeof window !== 'undefined'` guard. `getSessionId()` returns
`'ssr'` in non-browser environments, and assignment falls back to the
default 50/50 split when no override is available.

## Health page hookup

`web/src/pages/Health.tsx` calls `useABVariant('health-hero')` and
forwards the assigned variant as a `data-ab-variant` attribute on the
hero `DashboardGrid` (`<DashboardGrid data-stat-card-trends gap="sm"
data-ab-variant={heroVariant}>`). Both variants render the same UI
today; the attribute is the contract that downstream styling and
Playwright selectors will pivot on. A follow-up can either:

1. Branch on `heroVariant === 'B'` in the JSX to swap the three stat
   cards for a single-row condensed summary, or
2. Add a Tailwind / CSS rule keyed off
   `[data-ab-variant='B'] .stat-card { ... }` so the structural change
   stays purely cosmetic.

The hookup point is intentionally minimal in this patch so the harness
ships independently of any specific visual experiment.

## Tests

`web/src/lib/ab-variant.test.ts` (12 vitest cases):

1. `hashString` is deterministic for the same input.
2. `assignVariant` returns A or B deterministically per `sessionId`.
3. 50/50 split distributes within +/- 10% over 1000 sessions.
4. 80/20 split honours the weights (700-900 A / 100-300 B).
5. `getSessionId` persists across calls in `sessionStorage`.
6. `setVariant` persists the override to `localStorage`.
7. `getVariant` returns the override when present, ignoring the
   deterministic assignment.
8. `clearVariants` empties the override store.
9. `useABVariant` reflects `setVariant` via the
   `ab-variant-changed` `CustomEvent`.
10. `useABVariant`'s `set()` setter persists and updates the hook
    state.
11. Malformed JSON in `localStorage` falls back to the assigned
    variant.
12. Non-object payload in `localStorage` falls back to the assigned
    variant.

Test command:

```
env -C /root/c4-worktree-auto-w186/web \
  /root/c4/web/node_modules/.bin/vitest run src/lib/ab-variant.test.ts
```

Result: 12/12 passing.

## Out of scope

- Server-side assignment / sticky bucketing across devices. The
  harness is browser-local; a daemon-side counterpart can layer on top
  later by surfacing an assignment cookie.
- Variant-specific Health UI. Today both variants render the same
  hero block; the structural change is left to a follow-up so the
  harness can ship and be reviewed in isolation.
- Analytics. There is no impression / conversion logging in this
  patch -- consumers wire that into their own existing telemetry.

## Summary

`ChartLineVolatilityQuality` is a pure-SVG dual-panel React
primitive that plots the close on the top panel and a Volatility
Quality Index (VQI) oscillator on the bottom panel. VQI compares
the EMA-smoothed true range to the rolling ATR:

```
TR[i]    = max(high - low, |high - prevClose|, |low - prevClose|)
ATR[i]   = SMA(TR, length)[i]
EMA_TR[i] = EMA(TR, length)[i]
VQI[i]   = EMA_TR[i] / ATR[i]
```

Defaults: `length = 14`. Bars before `i = length - 1` are warmup
(`VQI = null`) because ATR requires `length` TR samples. When
`ATR == 0` (singular: completely flat bars) VQI is `null`.

A reading above unity (1.0) indicates expanding volatility (the
EMA-smoothed TR is rising above the longer-baseline ATR). Below
unity indicates contracting volatility.

## Bit-exact anchors

For **CONST_BAR** (constant `H`, `L`, `C` with `H > L` and
`L <= C <= H`):

- Every TR is `max(H - L, |H - C|, |L - C|) = H - L` (the gap
  legs are bounded by the range when `C` lies in `[L, H]`).
- For dyadic `H - L` (e.g. 2, 4, 8) the EMA-of-constant identity
  `alpha*K + (1 - alpha)*K = K` holds bit-exact in IEEE 754
  because the multiplications carry no rounding. So
  `EMA_TR = ATR = H - L` exactly, and `VQI = 1` exactly past
  warmup.
- For non-dyadic `H - L` (e.g. 10, 99) the EMA can drift by 1
  ULP, but the ratio still rounds to `1` within 12 decimal
  places. The integration sweep validates this with
  `toBeCloseTo(1, 12)`.

For **CONST_FLAT** (`H = L = C`): every TR is zero, ATR is zero,
and VQI is `null` (divide-by-zero guard).

## Zone classifier

- `expanding` -- VQI >= 1.1 (sharp uptick in TR)
- `above` -- 1.0 < VQI < 1.1
- `at` -- VQI == 1.0
- `below` -- 0.9 < VQI < 1.0
- `contracting` -- VQI <= 0.9 (sharp downtick in TR)
- `none` -- VQI is null (warmup or singular)

## Out of scope

- No new dependencies.
- No canvas, no third-party chart libs.
- This primitive uses the simple ratio formulation
  (EMA_TR / ATR). Hawkeye Volatility Quality, which adds a signed
  component based on close-position within the bar, is a separate
  variant and is not in scope here.

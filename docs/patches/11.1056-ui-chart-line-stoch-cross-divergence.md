# 11.1056 -- UI: chart-line-stoch-cross-divergence primitive (v1.11.1074)

## Scope

Add a pure-SVG dual-panel `ChartLineStochCrossDivergence`
component that renders the close in the top panel and the
slow Stochastic oscillator (smoothed %K and SMA %D signal
line) in the bottom panel, marking bullish / bearish
Stochastic-cross-with-divergence events.

A **cross-divergence** trigger is a compound signal that
fires only when BOTH:

1. %K crosses %D (the canonical smoothed-momentum
   crossover signal), AND
2. the price-vs-%K regime at the cross bar is **divergent**
   (priceDown + %K-up for bullish, or priceUp + %K-down
   for bearish).

This compound filter rejects "aligned" crossovers (where
price and %K move in the same direction at the cross bar)
which often represent late-stage trend continuation
rather than reversals. Filtering to divergent crosses
tends to surface higher-quality reversal signals -- the
sort of signal that combines cross-sig and divergence-cross
information into one trigger.

## Stochastic cross-divergence pipeline

```
hh[i]        = max(high[j], j in [i - period + 1, i])
ll[i]        = min(low[j],  j in [i - period + 1, i])
rawK[i]      = (hh[i] === ll[i])
               ? 50   (flat-window guard -> midline)
               : 100 * (close[i] - ll[i]) /
                       (hh[i] - ll[i])
smoothK[i]   = SMA(rawK,    smoothK)
d[i]         = SMA(smoothK, smoothD)

priceUp     : close[i] > close[i-1]
priceDown   : close[i] < close[i-1]
kUp         : smoothK[i] > smoothK[i-1]
kDown       : smoothK[i] < smoothK[i-1]

regime ->
  'aligned-bullish'   when priceUp   && kUp
  'aligned-bearish'   when priceDown && kDown
  'divergent-bullish' when priceDown && kUp
  'divergent-bearish' when priceUp   && kDown
  'none'              when either side flat or null

k-over-d cross  :
  prevK <= prevD && curK > curD   -> raw bullish
  prevK >= prevD && curK < curD   -> raw bearish

bullish (cross-divergence trigger up) :
  raw bullish cross && regime === 'divergent-bullish'
bearish (cross-divergence trigger down) :
  raw bearish cross && regime === 'divergent-bearish'
```

Default `period = 14`, `smoothK = 3`, `smoothD = 3` --
canonical "slow stochastic" tuning. George Lane's 1950s
Stochastic oscillator locates the close within the rolling
`[HH, LL]` high-low range as a 0..100 percentage. The slow
stochastic smooths %K with an SMA (giving "smoothed %K")
and then takes another SMA over that to form the %D
signal line. 50 is the natural centerline; > 80 / < 20
are conventional overbought / oversold zones.

Warmup is `period + smoothK + smoothD - 2 = 18` for the
default tuning: rawK valid from i = period - 1 = 13,
smoothK valid from i = 13 + smoothK - 1 = 15, %D valid
from i = 15 + smoothD - 1 = 17. Cross detector needs prev
%K and prev %D, so the first potential cross lands at i =
18.

## Bit-exact anchors

All anchors use HLC input.

- **CONST band** `high = K + 1`, `low = K - 1`, `close
  = K`: hh = K + 1, ll = K - 1. range = 2. rawK = 100 *
  (K - (K - 1)) / 2 = 50 (midline). smoothK = 50, %D =
  50. %K === %D every bar -> no cross. price flat ->
  regime `none` -> no divergence. 0 crosses. Verified
  across K in {0, 1, 50, 200, 1234}.
- **LINEAR UP** `high = i + 1`, `low = i - 1`, `close
  = i`: hh = i + 1, ll = i - 14. range = 15. rawK = 100
  * (i - (i - 14)) / 15 = 1400 / 15 = 280 / 3 ~=
  93.33 (deep overbought). smoothK = %D = same. No
  cross (K === D). priceUp + Kflat -> regime none
  (flat-direction guard). 0 crosses.
- **LINEAR DOWN** `high = -i + 1`, `low = -i - 1`,
  `close = -i`: hh = -i + 14, ll = -i - 1. range = 15.
  rawK = 100 * (-i - (-i - 1)) / 15 = 100 / 15 = 20 /
  3 ~= 6.67 (deep oversold). No cross. priceDown +
  Kflat -> regime none. 0 crosses.

All three anchors produce zero divergence crosses -- the
regime stays in `none` and %K never crosses %D. Crosses
only fire when BOTH the smoothed-momentum crossover and
the price-vs-%K divergence regime align, which requires
curvature in both the close and the stochastic.

## Risk

- Pure-SVG, deterministic `.toFixed(2)`, no canvas, no
  chart lib dependency.
- 5-state regime classifier follows the divergence-cross
  family convention, combined with a K-over-D cross
  detector (cross-sig family convention) -- this is a
  hybrid that combines both families' signals.
- Compound filter (cross AND divergence) is more
  selective than either signal alone. Some traders may
  prefer the looser "cross OR divergence" interpretation
  -- this primitive uses the strict AND form per the
  task description "crossover divergence variant".
- Flat-window guard on `hh === ll` returns midline 50
  instead of 0/0 NaN. Matches pandas-ta and other
  reference implementations.
- On steady LINEAR input %K saturates near the zone
  extremes (93.33 overbought, 6.67 oversold) and %K ===
  %D so neither cross nor divergent regime fires --
  correct canonical behaviour. Real-world price data
  with trend-direction changes produces the typical
  Stochastic cross-divergence patterns.
- Stochastic is hard-bounded to `[0, 100]` so the layout
  panel can be locked, providing stable threshold
  reference rendering.
- Distinct from the mid-cross-sig family
  (Stochastic-vs-50-centerline-via-signal) and the
  divergence-cross family (Stochastic-direction-vs-
  price). This compound signal targets the intersection
  of both -- a more selective momentum-reversal trigger.

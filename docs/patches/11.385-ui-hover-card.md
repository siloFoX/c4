## Goal

Dispatch:

- Build `web/src/components/ui/hover-card.tsx`
  -- rich popover on hover (not focus),
  open/close delay tuning, portal
  rendering, smart anchor positioning.

No existing hover-card primitive in the
codebase. This patch ships one from
scratch.

## Architecture

```
+-----------------------------+
| trigger ReactElement        |  <- cloned with mouse/touch/key handlers
+-----------------------------+
       |
       v  (mouseenter -> openDelay timer)
+-----------------------------+
| <createPortal>              |  <- mounts under #hover-card-root
|   panel role="tooltip"      |
|   position: fixed top/left  |
|   z-index: 50               |
+-----------------------------+
       ^
       | mouseenter cancels pending close
       | mouseleave schedules close
```

The trigger is `cloneElement`-ed so the
caller's existing handlers are preserved.
Panel mounts in a fresh portal node
(`hover-card-root`), auto-created on first
use via the existing
`lib/portal-root.ts` helper.

## API

```ts
interface HoverCardProps {
  trigger: ReactElement;
  content: ReactNode;
  open?: boolean;
  defaultOpen?: boolean;
  onOpenChange?: (open: boolean) => void;
  placement?: 'top' | 'bottom' | 'left' | 'right';  // default 'bottom'
  align?: 'start' | 'center' | 'end';                // default 'center'
  offset?: number;                                    // default 8 (px)
  openDelay?: number;                                 // default 300 (ms)
  closeDelay?: number;                                // default 200 (ms)
  className?: string;
  'data-testid'?: string;
}
```

```tsx
<HoverCard
  trigger={<a href="/profile/alice">Alice</a>}
  content={
    <div>
      <strong>Alice</strong>
      <p>Lead engineer, fleet ops</p>
    </div>
  }
  placement="bottom"
  align="start"
  openDelay={300}
  closeDelay={200}
/>
```

## Behaviour

- **Hover-only trigger.** `mouseenter` ->
  open after `openDelay`. `mouseleave`
  -> close after `closeDelay`. `focus`
  does NOT open the card (focus is
  reserved for the `<Popover>` primitive).
- **Cancellable timers.** If the user
  leaves the trigger before `openDelay`
  elapses, the pending open is
  cancelled. If they re-enter the
  trigger OR the panel before
  `closeDelay` elapses, the pending close
  is cancelled.
- **Touch fallback.** `touchstart` on
  the trigger acts like `mouseenter` so
  a tap-and-hold reveals the card.
- **Keyboard.** `Escape` closes a
  stale-open card -- on the trigger or
  at the document level.
- **Smart anchor positioning.** Panel
  flips to the opposite side when the
  preferred placement would overflow
  the viewport. Same flip-on-overflow
  math as the existing `<Popover>`
  primitive.
- **Portal rendering.** Panel mounts
  under the auto-created
  `hover-card-root` portal so it
  escapes parent overflow / transform
  containers.
- **ARIA.** Trigger gets
  `aria-describedby` pointing at the
  panel when open. Panel uses
  `role="tooltip"`.

## Pure helper

`computeHoverCardPosition(trigger, panel,
viewport, placement, align, offset)` is
exported alongside the component for
tests + parallel primitives. Returns
`{ top, left, placement }`. Placement
auto-flips when the preferred side
overflows.

## Data attributes

- Trigger:
  `data-hover-card-trigger="true"` +
  `data-hover-card-open="true|false"`.
- Panel: `data-section="hover-card-panel"`
  + `data-hover-card-placement` mirrors
  the resolved placement.

## Pairs with existing primitives

- `<Tooltip>` -- short text-only
  hover / focus surface. Use Tooltip for
  inline accessibility labels + hints;
  reach for HoverCard when the preview
  needs rich layout.
- `<Popover>` (11.275) -- click-
  triggered rich surface with focus
  trap. Reach for Popover when the
  trigger is a click target; reach for
  HoverCard when the trigger is a pure
  preview affordance.
- `lib/portal-root.ts` -- shared
  portal node helper, auto-creates
  `hover-card-root` div in
  `document.body`.

## Out of scope

- Per-page adoption of HoverCard. The
  primitive is additive; existing
  Tooltip / Popover sites stay
  unchanged.
- Sub-card / nested HoverCard. Panel
  mouseenter/leave logic is
  single-level only; nested previews
  would need a shared open-state
  registry.
- Focus-triggered open. By design --
  use `<Popover>` for focus-trigger
  surfaces.
- Arrow / pointer glyph on the panel.
  Flat surface; an arrow bump would
  conflict with rich content.
- Long-press / haptic feedback on
  touch. Current touchstart shortcut
  covers the baseline.
- Server-side rendering of the panel.
  Panel only mounts client-side
  (portal requires `document`).

# Patch 11.945 - UI: chart-line-dpo-zero-cross primitive

Version: 1.11.963

## Summary

Add `web/src/components/ui/chart-line-dpo-zero-cross.tsx` -- a pure-SVG dual
panel line chart primitive that visualizes price alongside a Detrended Price
Oscillator (DPO) panel, with explicit cross markers and overlay arrows at every
DPO zero crossover. The detrended price oscillator removes the centered
moving-average trend component from price, so DPO sits around zero when the
short-term close matches the historical centered trend. Zero crossovers mark
detrended momentum regime transitions: cross-up = bullish (close pulled above
the historical mean), cross-down = bearish (close pulled below).

## Look-back inversion (important property)

The standard DPO formula displaces the SMA back in time by `shift = floor(N/2) + 1`
bars and compares the shifted close to the *current* SMA value:

```
dpo[i] = close[i - shift] - sma[i]
```

Because the SMA at index `i` is a centered-future-leaning average relative to
the shifted close, monotone trends produce a *constant but opposite-sign* DPO:

| Pattern                  | DPO sign (length=20) | Regime  |
|--------------------------|----------------------|---------|
| LINEAR UP (close = i)    | -1.5 (constant)      | bearish |
| LINEAR DOWN (close = -i) | +1.5 (constant)      | bullish |
| CONST K (close = K)      | 0 (constant)         | bullish |

This is documented in the tests as the canonical bit-exact anchor for the
primitive.

## Pipeline

1. `applyLineDpoZeroCrossSma(close, length)` -- SMA with CONST short-circuit
   when `min === max` in the window.
2. `shift = floor(length / 2) + 1`.
3. `dpo[i] = close[i - shift] - sma[i]` for `i >= length - 1 + shift`, else null.
4. `classifyLineDpoZeroCrossRegime(dpo, T)`: null -> none, dpo >= T -> bullish,
   dpo < T -> bearish.
5. `detectLineDpoZeroCrossCrosses(series, dpo, T)`: strict `prev <= T && cur > T`
   for bullish, strict `prev >= T && cur < T` for bearish. Boundary equality
   does not fire.

## Layout

* Default dimensions: 760 x 360.
* Two panels: price panel on top (~58% of plot area), DPO panel below.
* DPO panel auto-fits to the observed `[min, max]` range with 10% padding
  on each end, but always includes the threshold in the displayed range.
* Threshold band rendered as a horizontal rule at `y = thresholdY` in the
  DPO panel.
* Cross markers drawn as circles on the DPO panel at `(cx, cyOsc)` and as
  arrows on the price panel at `(cx, cyPrice)`.
* SVG path strings use `.toFixed(2)` for deterministic snapshot tests.

## Accessibility

* Wrapping `<div role="region" aria-label="DPO Zero Cross chart">`.
* `<svg role="img">` with an aria-described-by linked `<desc>` sr-only text.
* Each cross marker: `role="graphics-symbol"` + `tabIndex={0}` + descriptive
  `aria-label` ("bullish DPO zero crossover at x=...").
* Legend toggles: `<button role="switch" aria-pressed="...">`.
* Motion-safe fade-in via `motion-safe:animate-fade-in` (opt-out with
  `animate={false}`).

## Tests

68 cases in `chart-line-dpo-zero-cross.test.tsx`:

* `getLineDpoZeroCrossFinitePoints` -- finite-filter, null/undefined guards.
* `normalizeLineDpoZeroCrossLength` -- floor, fallback on invalid.
* `normalizeLineDpoZeroCrossThreshold` -- accepts any finite, fallback on
  non-finite.
* `applyLineDpoZeroCrossSma` -- CONST short-circuit, +0 preservation, len=1.
* `computeLineDpoZeroCross` -- CONST K bit-exact anchor (K in
  `{0, 1, 42, 100, 1234}`), LINEAR UP -> -1.5, LINEAR DOWN -> +1.5, shift
  formula across length values.
* `classifyLineDpoZeroCrossRegime` -- null/boundary/below behaviour.
* `detectLineDpoZeroCrossCrosses` -- bullish/bearish, null gaps,
  boundary-not-crossed.
* `runLineDpoZeroCross` -- CONST counters, LINEAR all-bearish / all-bullish
  (look-back inversion), rise+decline crossing pattern, threshold
  normalization, empty input.
* `computeLineDpoZeroCrossLayout` -- dimensions, thresholdY ordering, empty
  input, osc range auto-fit, path precision, deterministic markers.
* `describeLineDpoZeroCrossChart` -- "No data" and parameterized format.
* React render -- region/role=img/aria-desc, badge, legend toggles, controlled

  animate flag, hover tooltip, band/axis/grid/legend visibility flags, cross

## Summary

New `ChartLineHeikinAshi` UI primitive at
`web/src/components/ui/chart-line-heikin-ashi.tsx` -- a pure-SVG
line chart overlaying a Heikin-Ashi smoothed close series that
filters noise from the raw close line.

## Indicator math

The Heikin-Ashi ("average bar") transform converts raw OHLC bars
into smoothed bars:

```
haClose = (open + high + low + close) / 4
haOpen  = first bar : (open + close) / 2
          later bar : (haOpen[prev] + haClose[prev]) / 2
haHigh  = max(high, haOpen, haClose)
haLow   = min(low,  haOpen, haClose)
```

`haClose` blends all four OHLC fields, so a single-bar spike is
damped to a quarter of its weight. `haOpen` is **recursive** --
it carries the previous bar forward, so a noisy close is averaged
down rather than swinging the line. That recursion is what
filters noise out of the raw close. A bar is **bullish** when its
`haClose` sits above its `haOpen`, **bearish** when below, and a
**doji** when equal.

### Worked fixture

Five OHLC bars chosen so the recursive `haOpen` lands on
integers:

```
bar 0  haClose (10+20+10+12)/4 = 13   haOpen (10+12)/2 = 11   bullish
bar 1  haClose (14+22+10+18)/4 = 16   haOpen (11+13)/2 = 12   bullish
bar 2  haClose (18+26+14+22)/4 = 20   haOpen (12+16)/2 = 14   bullish
bar 3  haClose (16+18+4+6)/4   = 11   haOpen (14+20)/2 = 17   bearish
bar 4  haClose (14+16+12+14)/4 = 14   haOpen (17+11)/2 = 14   doji
```

The raw close drops to 6 at bar 3, but the Heikin-Ashi close only
falls to 11 -- the spike is filtered. 3 bullish, 1 bearish,
1 doji; the final Heikin-Ashi close is 14.

## API

- `getLineHeikinAshiFinitePoints(points)` -- drop bars with a
  non-finite `x`, `open`, `high`, `low` or `close`.
- `computeLineHeikinAshi(bars)` -- the Heikin-Ashi transform,
  returning `{ haOpen, haHigh, haLow, haClose }` per bar.
- `runLineHeikinAshi(points)` -- sort finite bars by `x`, run the
  transform, return `{ series, bars, samples, haCloseSeries,
  bullishCount, bearishCount, dojiCount, haCloseFinal, ok }`.
  Each sample is classified `bullish` / `bearish` / `doji`. `ok`
  is false for fewer than two bars.
- `computeLineHeikinAshiLayout(options)` -- project the raw close
  path and the Heikin-Ashi close path on one shared panel, with a
  marker per bar.
- `describeLineHeikinAshiChart(data)` -- ARIA description.

## Component

`ChartLineHeikinAshi` is a `forwardRef` region (`role="region"`)
with an ARIA-linked description, a `role="img"` SVG, axis ticks
and grid, a config badge showing the latest bar's trend, a
two-series legend (Close / HA Close) with toggle, a hover/focus
tooltip exposing the raw close, the Heikin-Ashi close and the
trend, and a `motion-safe:animate-fade-in` entrance. It consumes
`{ x, open, high, low, close }` bars. The raw close is a plain
line; the Heikin-Ashi close is a second line whose markers are
coloured by trend (bullish / bearish / doji). Every node carries
a `data-section` attribute.

## Distinctness

- **vs the moving-average overlays (chart-line-smma /
  chart-line-lsma / ...)** -- those weight a sliding window of a
  single value series. Heikin-Ashi blends all four OHLC fields of
  each bar and carries a recursive open forward; it is a bar
  transform, not a windowed average.
- **vs chart-line-fractal** -- the fractal detects discrete swing
  pivots. Heikin-Ashi produces a continuous smoothed series and a
  per-bar bullish / bearish / doji trend.
- The component takes full OHLC bars rather than a single
  `{ x, value }` series, and exposes the trend as the canonical
  Heikin-Ashi signal.

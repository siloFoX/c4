# 11.519 -- web/src/components/ui/chart-line-forecast.tsx

Pure-SVG multi-series line chart that **splits each series into a
historical solid line and a forecast dashed extension** at a
configurable `forecastFrom` cutoff. A vertical dashed reference line
marks the cutoff x with a "Forecast" label. Forecast samples may
carry optional `yLower` / `yUpper` bounds; when present a translucent
confidence-band polygon is painted under the dashed extension.

## Distinct from other line + reference primitives

- `<ChartLineConfidence>` (11.502): per-point band around an entire
  series. `<ChartLineForecast>` ships a band but **only on the
  forecast half** of each series, paired with the historical/forecast
  styling split.
- `<ChartLineThreshold>` (11.503), `<ChartLineBaseline>` (11.510),
  `<ChartLineBand>` (11.516), `<ChartLineTarget>` (11.518):
  horizontal references. Forecast is a **vertical** time-axis split.
- `<ChartLineDashed>` (11.499): per-segment dash styling. Forecast is
  a strict before/after split with a single seam point.
- `<ChartLineComparison>` (11.511): two-curve comparison. Forecast is
  a single curve split into two phases.
- `<ChartLineAnnotated>` (11.509): vertical annotation rules with
  labels. Forecast adds **a path styling split** at the rule, not
  just an annotation.

The canonical "observed metric + predicted metric" view for capacity
planning, financial projections, traffic forecasting, etc.

## Public surface

- `<ChartLineForecast series={...} forecastFrom={4} />`.
- Per-series `{id, label, data, color?, forecastFrom?,
  forecastDashArray?}`. `data` entries are
  `{x, y, yLower?, yUpper?}`.
- `forecastFrom` defaults are chart-level; per-series override always
  wins.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showBand`, `showCutoff`, `showCutoffLabel`,
  `animate`.
- `forecastDashArray` (`'6 4'`), `cutoffDashArray` (`'4 4'`),
  `cutoffColor` (`'#0f172a'`), `forecastOpacity` (`0.85`),
  `bandOpacity` (`0.18`).
- Standard sizing + format + a11y props.
- Controlled + uncontrolled `hiddenSeries`.
- `onPointClick({series, point})` -- point carries `phase`
  (`'historical' | 'forecast'`).

## Pure helpers

- `getLineForecastDefaultColor(index)` -- 10-color palette cycle.
- `getLineForecastFinitePoints(points)` -- finite-only filter.
- `isForecastPoint(point, cutoff)` -- true when `x >= cutoff`.
  Non-finite cutoff or point -> false.
- `splitLineForecastPoints(points, cutoff)` -- splits an x-sorted
  finite series into `{historical, forecast, join}`. When the cutoff
  lands strictly inside a segment, a synthetic **join point** at the
  interpolated `(cutoff, y)` (and interpolated `yLower` / `yUpper`
  when both endpoints carry bounds) is added as the LAST point of
  `historical` and the FIRST point of `forecast` so the two paths
  render seamlessly. Non-finite cutoff puts everything in historical
  (no split).
- `buildLineForecastPath(points)` -- standard `M ... L ...` builder
  that skips non-finite px / py.
- `buildLineForecastBandPath(points)` -- closed polygon walking
  upper then lower, skipping points where either bound is missing.
  Returns `''` when fewer than 2 paired bounds exist.
- `computeLineForecastLayout({series, forecastFrom?, hiddenSeries?,
  showBand?, ...})` -- returns `{series, xTicks, yTicks, xMin, xMax,
  yMin, yMax, defaultForecastFromX, innerWidth, innerHeight,
  totalPoints, visibleSeriesCount}`. Each layout series carries
  `{id, label, index (stable original), color, forecastDashArray,
  points, historicalPath, forecastPath, bandPath, forecastFrom,
  forecastFromX, historicalCount, forecastCount, finiteCount,
  totalCount}`. Per-point `{index, x, y, px, py, phase}`.
- `describeLineForecastChart(series, forecastFrom?, hidden?,
  formatValue?)` -- builds the screen-reader description with per-
  series historical/forecast counts and the cutoff value.

## Join interpolation

For consecutive `(p_i, p_{i+1})` where `p_i.x < cutoff < p_{i+1}.x`:

```
t      = (cutoff - p_i.x) / (p_{i+1}.x - p_i.x)
y_join = p_i.y + (p_{i+1}.y - p_i.y) * t
yLower_join = p_i.yLower + (p_{i+1}.yLower - p_i.yLower) * t   (when both finite)
yUpper_join = p_i.yUpper + (p_{i+1}.yUpper - p_i.yUpper) * t   (when both finite)
```

The join point has `originalIndex: null` and `isJoin: true` so
adopters can distinguish it from real samples when iterating.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Historical path + forecast path + cutoff line + per-dot are
  `role="graphics-symbol"` + `tabIndex={0}` (cutoff line is
  `graphics-symbol` only). Dot aria-label includes the phase.
- Every node carries `data-section="chart-line-forecast-..."` for
  test selectors.
- Series groups expose `data-series-historical-count`,
  `data-series-forecast-count`, `data-series-forecast-from`,
  `data-series-color`, `data-series-finite-count`.
- Dots expose `data-phase` (`'historical' | 'forecast'`).
- Root mirrors `data-forecast-from`.

## Tooltip

Hovering a dot shows the series label, x, y, and a `phase:` row.
Tooltips are suppressed when `showTooltip=false`.

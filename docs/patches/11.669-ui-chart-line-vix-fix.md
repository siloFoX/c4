two-panel chart with a Williams VIX Fix panel estimating implied
volatility from the recent highest close.

## Indicator math

Larry Williams' VIX Fix is a synthetic volatility gauge that
approximates what a fear index would show, computed from price
alone -- no options data.

`computeLineVixFixHighest(closes, period)` is the highest close
over the trailing `period` bars.
`computeLineVixFix(closes, period)` is the VIX Fix itself:

```
WVF[i] = 100 * (highestClose - close) / highestClose
```

It measures how far the close has fallen below its recent peak,
as a percentage. When the price makes a new high the highest
close equals the close and the WVF is zero; when the price drops
sharply below its recent peak the WVF spikes up. A non-positive
highest close is null.

The chart classifies each bar against a `threshold`: a WVF above
the threshold is a `spike` (volatility flare-up), at or below is
`calm`.

### Exactness

The VIX Fix is a ratio of integers -- no transcendentals -- so it
is exact when the closes are integers and the highest close
divides cleanly. The fixture places a close of 100 inside every
window, so the highest close is always 100 and `WVF = 100 -
close` lands on exact integers: the series `[., ., ., 30, 0, 15,
25, 0, 40, 0]` is asserted with `toEqual`.

## API

- `getLineVixFixFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineVixFixPeriod(period, fallback)` -- coerce to a
  positive integer.
- `computeLineVixFixHighest(closes, period)` -- the rolling
  highest close.
- `computeLineVixFix(closes, period)` -- the VIX Fix series.
- `runLineVixFix(points, options?)` -- sort finite points by
  `x`, run the pipeline, return `{ series, period, threshold,
  highest, vixFix, samples, vixFixFinal, spikeCount, calmCount,
  ok }`. Each sample is classified `spike` / `calm` / `none`.
  `ok` is false for fewer than two points.
- `computeLineVixFixLayout(options)` -- stack the price panel
  above a VIX Fix panel on a non-negative scale; project the
  price path, the VIX Fix path, one marker per defined bar, and
  the threshold reference line.
- `describeLineVixFixChart(data, options?)` -- ARIA description.

## Component

`ChartLineVixFix` is a `forwardRef` region (`role="region"`) with
an ARIA-linked description, a `role="img"` SVG, two stacked
panels (price on top, VIX Fix below) with their own axis ticks
and grid, panel labels, a config badge showing the period, a
three-series legend (Price / VIX Fix / Threshold) with toggle, a
hover/focus tooltip exposing the VIX Fix and the zone, and a
`motion-safe:animate-fade-in` entrance. It consumes `{ x, value }`
points. The VIX Fix panel is a non-negative scale, draws a dashed
threshold reference line, and colours each marker by its zone --
red for a spike, cyan for calm. Every node carries a
`data-section` attribute.

## Distinctness

- **vs the volatility-band indicators (bollinger / keltner /
  bandwidth)** -- those wrap a band around the price. The VIX
  Fix is a one-sided drawdown gauge, never negative.
- **vs chart-line-stddev** -- a rolling standard deviation
  measures dispersion both ways. The VIX Fix measures only the
  drop from the recent peak, so it spikes on fear and rests at
  zero in a rising market.
- **vs chart-line-atr** -- the average true range needs the
  bar's high and low. The VIX Fix needs only the close.

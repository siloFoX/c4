# 11.525 -- web/src/components/ui/chart-line-residual.tsx

Pure-SVG multi-series **residual plot**: takes paired observed and
predicted values per x and plots `residual = observed - predicted`
against a horizontal zero-reference line. Stems drop from each
residual point to the zero line so over- vs under-prediction reads
at a glance; the residual line itself overlays for trend
visualisation. Per-series diagnostic stats (mean, MAE, RMSE, max
|r|, bias direction) are exposed via the layout, the legend, and
the auto-generated ARIA description.

## Distinct from other diagnostic / reference primitives

- `<ChartLineRate>` (11.521): first derivative (delta per
  interval). Residual is observed minus predicted at the SAME x.
- `<ChartLineComparison>` (11.511): difference shading between
  two curves. Residual collapses each pair into one
  observed-minus-predicted value plotted against zero.
- `<ChartLineTarget>` (11.518): vs a flat target. Residual is vs
  a per-point predicted value.
- `<ChartLineTrend>` (11.512): regression line. Residual plot is
  the standard regression diagnostic for assessing fit.
- `<ChartLineAnomaly>` (11.524): z-score outliers on a single
  series. Residual works on paired observed/predicted samples.
- `<ChartLineBaseline>` (11.510): per-point classification vs flat
  baseline. Residual is the dual data (observed, predicted) form.

The canonical regression-diagnostic / model-evaluation
visualisation: "is the model systematically over- or
under-predicting? Where? By how much?"

## Public surface

- `<ChartLineResidual series={...} />`.
- Per-series `{id, label, data, color?, positiveColor?,
  negativeColor?}`. `data` entries are
  `{x, observed, predicted}`. Residual = `observed - predicted`.
- `biasEpsilon` (default 0) widens the equality band for the
  `bias` classification.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showZeroLine`, `showLine`, `showStems`,
  `animate`.
- `positiveColor` (default green) for under-prediction (`r > 0`),
  `negativeColor` (default red) for over-prediction (`r < 0`).
- Standard sizing + format + a11y props.
- `formatValue` / `formatX` / `formatResidual` (sign-aware
  fallback).
- Controlled + uncontrolled `hiddenSeries`.
- `onPointClick({series, point})`.

## Pure helpers

- `getLineResidualDefaultColor(index)` -- 10-color palette cycle.
- `getLineResidualFinitePoints(points)` -- finite-only filter
  (requires all three fields finite).
- `computeResidual(observed, predicted)` -- `observed -
  predicted`; non-finite returns `0`.
- `classifyLineResidualSign(residual, epsilon?)` -- returns
  `'positive' | 'negative' | 'zero'`. Non-finite -> `'zero'`.
  Non-finite / negative epsilon falls back to `0`.
- `computeLineResidualStats(points, biasEpsilon?)` -- returns
  `{finiteCount, meanResidual, mae, rmse, maxAbs, maxResidual,
  minResidual, bias, positiveCount, negativeCount, zeroCount}`.
  `bias` is `'positive' | 'negative' | 'unbiased'` after epsilon
  band. RMSE = `sqrt(Σr² / n)` (population, since the residuals
  are observed errors, not samples). Drops non-finite samples;
  empty / non-array -> all zeros and `bias: 'unbiased'`.
- `computeLineResidualLayout({series, hiddenSeries?, biasEpsilon?,
  ...})` -- returns `{series, xTicks, yTicks, xMin, xMax, yMin,
  yMax, zeroY, innerWidth, innerHeight, totalPoints,
  visibleSeriesCount}`. Each layout series carries `{id, label,
  index (stable original), color, positiveColor, negativeColor,
  points, path, stats, finiteCount, totalCount}`. Per-point
  `{index, x, observed, predicted, residual, px, py, stemY1,
  stemY2, sign}` with `stemY2` always landing on the zero
  reference. Y range is always expanded to include 0.
- `describeLineResidualChart(series, hidden?, formatValue?,
  biasEpsilon?)` -- builds the screen-reader description.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Line paths + dots are `role="graphics-symbol"` + `tabIndex={0}`;
  zero baseline is `role="graphics-symbol"`. Dot aria-label
  includes observed + predicted + residual.
- Every node carries `data-section="chart-line-residual-..."` for
  test selectors.
- Series groups expose `data-series-mean-residual`,
  `data-series-mae`, `data-series-rmse`, `data-series-max-abs`,
  `data-series-bias`, `data-series-positive-count`,
  `data-series-negative-count`, `data-series-zero-count`,
  `data-series-color`, `data-series-finite-count`.
- Dots expose `data-observed`, `data-predicted`, `data-residual`,
  `data-sign`.
- Stems expose `data-sign` so adopters can drive per-sign
  highlights.

## Tooltip

Hovering a dot shows the series label, x, observed, predicted,
and a coloured `residual:` row with a suffix indicating
`(under-predicted)` or `(over-predicted)` when non-zero.

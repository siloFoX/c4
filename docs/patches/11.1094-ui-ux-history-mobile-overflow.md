# 11.1094 -- UX FIX: History list overflows on mobile (v1.11.1112)

## Scope

At 375 the History sidebar's actions toolbar (column-picker / export /
scribe) and the per-row CLOSED status badges clipped at the right edge.

## Root cause

The master-detail root (`flex flex-col md:flex-row`) and the aside
(`w-full md:w-80`) already collapse to one column below md, so the
collapse itself was fine. The two real overflows were inside the
full-width mobile aside:

1. The sidebar `CardHeader` was `flex-row justify-between` (title left,
   toolbar right) with no wrap. The toolbar's intrinsic width plus the
   title exceeded the aside, so the toolbar ran ~52px past the right
   edge at 375 (measured: right edge 427 vs viewport 375).
2. Each row's name `<span className="truncate">` had no `min-w-0`, so a
   long worker name kept its full content width (flex min-width:auto)
   and pushed the StatusPill (the CLOSED badge) off the right edge
   instead of letting the name truncate.

## Change (src/components/HistoryView.tsx)

- Sidebar `CardHeader`: `flex-row items-center justify-between` ->
  `flex-col gap-2 sm:flex-row sm:items-center sm:justify-between`, so
  the toolbar stacks below the title below sm and returns to the
  single row at >= sm (where the aside is the fixed md:w-80 column).
- Sidebar actions `Toolbar`: added `flex-wrap` so its controls wrap
  within the row as a final safety on the narrowest widths.
- Row name span: added `min-w-0` so it truncates and the status pill
  stays on screen.

Layout-only; reuses existing token-backed classes. No change to the
detail pane or the master-detail collapse (already correct).

## Verification (mandatory)

`web/e2e/history-mobile-overflow.spec.ts` (new) boots into the History
view with a long-named CLOSED worker and, at 375 / 768 / 1440, asserts:
(a) no horizontal page overflow (documentElement.scrollWidth <=
viewport), (b) the sidebar toolbar's right edge is within the viewport,
and (c) the CLOSED row's status pill's right edge is within the
viewport. It screenshots the 375 layout.

Red/green confirmed: before the fix the 375 case failed with the
toolbar right edge at 427 (> 375); after, `3 passed`. The 375
screenshot shows the Columns/Export/Scribe toolbar on its own row fully
visible and both rows' badges (BUSY, CLOSED) on screen with the long
name truncated.

## Risk

- Pure layout change to the sidebar header + one row span. At >= sm the
  header renders as before (verified at 768 / 1440 -- no regression).
  No behaviour change.

## Goal

Dispatch:

- Build `web/src/components/ui/timeline.tsx`
  -- vertical timeline with timestamp /
  title / description slots, custom dot/icon,
  variants (default / compact / dense).

`<Timeline>` (11.149 / v1.11.167) already
covered most of the contract:

| dispatch item | shipped at |
| --- | --- |
| vertical timeline | always vertical `<ol>` |
| timestamp / title / description slots | per-item fields |
| custom dot/icon | `icon` field on `TimelineItem` |
| tones | `tone: 'neutral' \| 'primary' \| 'success' \| 'warning' \| 'danger'` |
| groupByDay | optional bucketing by ISO date |
| connector | drawn for every row except last in a group |

The remaining dispatched gap was the
**variants (default / compact / dense)**
density scale. This patch adds it.

## Variants

| variant | row padding | dot size | title text | description |
| --- | --- | --- | --- | --- |
| `default` (legacy) | `pb-4` + `pl-6` | `h-4 w-4` | `text-sm` | block below title |
| `compact` | `pb-2` + `pl-5` | `h-3 w-3` | `text-xs` | block below title |
| `dense` | `pb-1` + `pl-4` | `h-2 w-2` | `text-xs` (inline) | hidden |

### `default`

Legacy 11.149 layout byte-for-byte: every
existing call site stays identical when
the variant prop is omitted.

### `compact`

Halves the vertical rhythm + dot size for
sidebars / nested panels where a 4-line
default row reads heavy. Title text drops
to `text-xs`; description block stays
visible.

### `dense`

Collapses the row to a single
baseline-aligned line:

```
.    Sep 12, 10:30   Worker w42 started build
.    Sep 12, 10:31   Worker w42 connected upstream
.    Sep 12, 10:32   Build complete
```

- timestamp + title are siblings inside one
  `<div class="flex items-baseline gap-2">`;
- description block is omitted entirely;
- in-dot icon is suppressed (the 8px dot
  is too small to host a glyph).

Use for huge activity logs (audit pane,
build history strip) where every pixel
counts.

## Data attributes

- `data-variant` mirrors the prop on both
  the root `<ol>` AND every `<li>` so CSS
  hooks + downstream tests can branch.
- New per-row inner sections:
  `data-timeline-title`,
  `data-timeline-timestamp`,
  `data-timeline-description`. The legacy
  class-only branching is replaced with
  these attribute hooks so tests can
  assert by attribute.

## Grouped mode

`variant` threads through `groupByDay` --
every per-day bucket renders its rows
with the same density.

## Pairs with existing primitives

- `<Stepper>` -- wizard-style sequential
  timeline with progress semantics.
- `<DataTable>` -- tabular alternative
  when timeline rows are uniform fields
  rather than narrative.
- `<Accordion>` -- collapsible per-day
  buckets; pair with `groupByDay=false`
  + a custom Accordion structure when the
  set is huge.

## Out of scope

- Per-page adoption swap (`compact` /
  `dense` rollout). All three variants are
  additive; every existing call site stays
  byte-identical at `default`.
- Horizontal timeline orientation.
  Vertical-only by convention; horizontal
  feels are better served by `<Stepper
  orientation="horizontal">`.
- Per-row variant override. Variant is
  applied at the timeline level; mixed
  density inside one timeline is rare and
  belongs in a separate primitive.
- Animated row stagger. The connector +
  dot rendering is static; reveal motion
  belongs in a per-surface tuning patch.
- Variable connector lengths between
  rows. The connector is positioned by
  `top: 1.5/2/2.5; bottom: 0` so it
  always spans the row gap. Adjustable
  per-row "gap" markers are a follow-on.

## Summary

Add a new `<ChartTreemapSquarified>` UI primitive in
`web/src/components/ui/chart-treemap-squarified.tsx` --
pure-SVG **squarified** treemap. Implements the Bruls,
Huijsen & van Wijk (2000) layout: values are sorted
descending and greedily packed into rows along the
**short side** of the remaining free rectangle so the
worst-case cell aspect ratio is minimised at every
step. Distinct from `<ChartTreemap>` (11.444) which
typically uses the slice-and-dice layout where cells
inherit the parent's strong aspect ratio (one tall
column with thin slivers). The squarified algorithm
trades a small amount of value-order locality for
much more readable rectangles.

## Behaviour

- Renders `<div role="region">` wrapping
  `<svg role="img">` with the same `aria-label`.
- Layout: `computeTreemapSquarifiedLayout` is the
  canonical Bruls/Huijsen/van Wijk algorithm:
  1. Drop non-positive / non-finite values, sort
     remaining by value descending.
  2. Normalise areas so they sum to `innerW *
     innerH`.
  3. Greedy row-packing: for each item, try adding
     it to the current row. Compute the worst
     aspect ratio of the row with vs without the
     item; keep the row when adding makes it
     better, otherwise lay out the row and start a
     new row.
  4. Rows always lay along the **short side** of
     the remaining free rectangle.
  5. Each cell carries an `aspectRatio` field so
     adopters can inspect per-cell quality.
- `cellGap` is split half above and half below each
  cell so neighbours have a thin divider.
- Total cell area equals `innerW * innerH` when
  `cellGap=0`; cell areas are proportional to
  `value / total`.
- Non-positive / non-finite values are dropped from
  the layout entirely.
- Hover/focus on a cell sets the hovered id and
  opens a tooltip with the cell's label, formatted
  value, share against the grand total, and the
  computed aspect ratio.
- Mount animation:
  `motion-safe:animate-fade-in` per cell group.
  `animate=false` removes the class.
- Default ARIA description: `"Squarified treemap
  with N cells, total <total>. Largest cell <label>
  <value>."` -- empty / zero -> `"No data"`.

- `sortTreemapSquarifiedDesc` preserves the original
  index so click + ref payloads stay correlated even
  after sort.
- Layout cell shape:
  `{ id, label, index, value, share, color, x, y,
  width, height, aspectRatio }`.
- `computeTreemapSquarifiedLayout` returns `[]` for
  non-positive inner dimensions, empty items, or
  zero total value.

## Pairs with existing primitives

- `<ChartTreemap>` (11.444) -- slice-and-dice
  sibling (cheaper layout, worse aspect ratios for
  skewed value sets).
- `<ChartMosaic>` (11.460) -- contingency-table
  alternative with width + height encoding two
  different dimensions.
- `<ChartMarimekko>` (11.475) -- variable-width
  stacked bars (1D categorical x + stacked series).
- `<ChartDonut>` (11.469) / `<ChartPie>` (11.440)
  -- share-of-total alternatives without the
  bounding rectangle.

## Out of scope

- Nested / hierarchical treemap with parent groups
  -- v1 is flat. Host composes nested instances or
  uses `<ChartTreemap>` if hierarchy is needed.
- Animated transitions on data change -- snap v1;
  mount fade only.
- Alternative quality metrics (e.g., distance-to-
  square via `(w-h)^2 / max(w,h)^2`) -- v1 uses
  the canonical `max(w/h, h/w)` ratio.
- Per-cell border / corner radius -- v1 paints
  rectangles only.

## New props

```ts
comparison?: {
  value: number | string;
  label?: string;
};
sparklineSlot?: ReactNode;
```

## Behaviour

### Comparison

`comparison` renders a small
"<label> <value>" line between the
primary metric and the trend
indicator. Layout:

- `value` accepts number | string.
  Use number for previous-period
  totals; use string for tagged
  labels like "Q1: 220".
- `label` overrides the default
  "vs." prefix. Common patterns:
  `target`, `prev`, `last week`,
  `baseline`.
- Hidden when `comparison` is
  undefined -- the legacy layout
  is byte-identical.
- Block carries
  `data-section="stat-card-comparison"`
  + `data-stat-comparison-value`
  attrs for tests + analytics.

### Sparkline slot

`sparklineSlot` lets callers
inject a third-party chart body
in place of the built-in
polyline:

- When `sparklineSlot` is
  provided, the slot renders
  inside a wrapper marked
  `data-section="stat-card-sparkline-slot"`
  and the built-in svg is
  suppressed.
- The wrapper applies the same
  margin + width as the built-in
  svg (`mt-2 w-full`) so layout
  parity holds.
- The legacy `sparkline:
  number[]` prop still works when
  `sparklineSlot` is undefined.

### Trend direction helper

Exported pure helper:

```ts
export type StatCardTrendDirection = 'up' | 'down' | 'flat';

export function getStatCardTrendDirection(
  value: number,
): StatCardTrendDirection;
```

Behaviour:
- positive -> 'up'
- negative -> 'down'
- 0 / NaN / +-Infinity -> 'flat'

Used internally to drive the
class palette + the new
`data-stat-trend-direction`
attribute on the trend block.
Exported so adopters that
manage trend state can reuse the
canonical mapping.

## Out of scope (deferred)

- Numeric formatting (commas,
  currency, units). Callers
  format the string themselves
  and pass via `value`.
- Trend animation (rolling
  counter for the trend %).
  Out of scope for v1.
- Comparison percentage delta
  computation. Caller computes
  the value to display.
- Multi-metric mode (two
  side-by-side values). Belongs
  in a different primitive.
- Per-page adoption sweep. The
  Auto page hero stats already
  use StatCard; the new props
  are opt-in.

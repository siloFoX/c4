# 11.541 -- web/src/components/ui/chart-line-resample.tsx

Pure-SVG line chart with a runtime **downsample / upsample
toggle** for handling dense (or sparse) data. Renders the raw
series as a dimmed reference + the resampled series in the
series color on top, plus a 5-button mode toggle (`LTTB`,
`Stride`, `Mean`, `Min/Max`, `Upsample`) so operators can pick
the algorithm that best preserves the visual structure of
their data.

## Algorithms

- **LTTB** (Largest Triangle Three Buckets, default): the
  canonical visual downsampling algorithm. Splits the data
  into `target-2` equal-length buckets and picks the sample
  from each bucket that forms the largest triangle with the
  previously kept sample (`A`) and the average point of the
  NEXT bucket (`C`). First and last samples are always kept.
  Best-in-class shape preservation for line charts.
- **Stride**: keeps every k-th sample at `k = (N - 1) / (T -
  1)`. Simple and predictable but loses extrema between
  strides.
- **Mean**: bucket average. Smooths the data; useful when
  noise reduction is wanted as a side-effect of downsampling.
- **Min/Max**: keeps both the min and the max sample from
  each bucket. Best for OHLC / range data where extrema
  matter and exact x-positions are useful.
- **Linear upsample**: linearly interpolates between samples
  to produce `target` evenly-spaced output points along the
  x range. Used when the source is sparser than `target` and
  visual smoothness is wanted.

All algorithms preserve `originalIndex` so callers can map
resampled points back to source rows; LTTB / Stride / Min/Max
keep ORIGINAL samples and report `synthesized: false`, while
Mean and Linear-upsample emit synthesized samples and report
`synthesized: true`.

## Distinct from related primitives

- `<ChartLineMovingAvg>` (11.513): SMA smoothing -- preserves
  count, modifies values. Resample CHANGES the count.
- `<ChartLineEwma>` (11.539): exponential smoothing --
  same count.
- `<ChartLineKalman>` (11.538): recursive Bayesian filter --
  same count.
- `<ChartLineBollinger>` (11.537): adds an envelope -- same
  count.
- `<ChartLineZoom>` (catalog earlier): viewport-zooms on the
  same data without changing the underlying sample count.
  Resample changes the count.
- `<ChartLineFft>` (11.540): frequency-domain transform.
  Resample stays in the time domain.

The canonical "this is too many points to render" / "I need
to interpolate this sparse series" primitive used in
observability dashboards (Prometheus / Grafana LTTB),
financial charts, and any context where the question is
"keep the shape, change the point count".

## Public surface

- `<ChartLineResample series={...} />`.
- `series: ChartLineResampleSeries[]` -- `{id, label, data,
  color?, mode?, target?}`. Per-series mode / target always
  beats chart-level.
- `mode` (controlled) / `defaultMode` (uncontrolled; default
  `'lttb'`) -- canonical mode toggle.
- `onModeChange(mode)` -- fires when the mode toggle is
  clicked.
- `target` (default `100`; clamped to `>= 2`) -- desired
  output sample count.
- `rawColor` -- dimmed raw line color.
- `rawOpacity` (default `0.3`).
- Toggle flags: `showAxis`, `showGrid`, `showDots`,
  `showLegend`, `showTooltip`, `showReductionBadge`,
  `showRaw`, `showModeToggle`, `animate`.
- Controlled + uncontrolled visibility via `hiddenSeries` /
  `defaultHiddenSeries` / `onHiddenSeriesChange`.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX` / `formatRatio`.
- `onPointClick({series, point})`.

## Pure helpers

- `LINE_RESAMPLE_MODES` -- `['lttb', 'stride', 'mean',
  'minmax', 'linear-upsample']`.
- `getLineResampleDefaultColor(index)` -- 10-color palette
  cycle.
- `getLineResampleFinitePoints(points)` -- finite-only
  filter; preserves `originalIndex` from the input array.
- `normaliseLineResampleTarget(value)` -- clamps to `>= 2`;
  floors fractional; non-finite -> default 100.
- `normaliseLineResampleMode(value)` -- identity for valid
  modes; otherwise default `'lttb'`.
- `lineResampleStride(points, target)` -- every-Nth.
- `lineResampleMean(points, target)` -- bucket average.
  Synthesized.
- `lineResampleMinMax(points, target)` -- min + max per
  bucket; orders by x for path continuity. Original
  samples.
- `lineResampleLttb(points, target)` -- canonical LTTB.
  Original samples. Target < 3 falls back to stride. First
  and last always kept.
- `lineResampleLinearUpsample(points, target)` -- linear
  interpolation; passthrough when input >= target. Endpoints
  not synthesized; interior new samples are.
- `applyLineResample(points, mode, target)` -- dispatcher.
- `computeLineResampleLayout({series, ...})` -- runs the
  selected algorithm per series and projects raw + resampled
  paths. Returns `{series, xTicks, yTicks, xMin, xMax, yMin,
  yMax, innerWidth, innerHeight, totalRawPoints,
  totalResampledPoints, visibleSeriesCount}`. Per-series
  carries `{id, label, color, mode, target, rawPoints,
  resampledPoints, rawPath, resampledPath, rawCount,
  resampledCount, finiteCount, totalCount, reductionRatio}`.
  Per-point carries `{...sample, index, px, py}` including
  `synthesized` and `originalIndex`.
- `describeLineResampleChart(series, options?)` -- screen-
  reader description with per-series mode + ratio.

## Mode toggle (UI control)

The `showModeToggle` panel is a `role="radiogroup"` with one
`role="radio"` button per mode. Each button exposes
`aria-checked`, `data-mode`, and `data-active` so callers can
write assertions or react to mode changes. The active button
is highlighted with a dark background; clicking dispatches
`onModeChange(mode)` and (in uncontrolled mode) updates
internal state.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Raw path + resampled path + every dot is
  `role="graphics-symbol"` + `tabIndex={0}`. Raw aria-label
  includes the raw count; resampled aria-label includes the
  mode + resampled / raw counts.
- Mode toggle is `role="radiogroup"` with per-button
  `role="radio"` + `aria-checked`.
- Every node carries `data-section="chart-line-resample-..."`
  for test selectors.
- Series groups expose `data-series-id`, `data-series-color`,
  `data-series-mode`, `data-series-target`,
  `data-series-raw-count`, `data-series-resampled-count`,
  `data-series-reduction-ratio`, `data-series-finite-count`.
- Raw path has `data-kind="raw"`; resampled
  `data-kind="resampled"`.
- Dots expose `data-original-index`, `data-synthesized`.
- Root mirrors `data-series-count`,
  `data-visible-series-count`, `data-total-raw-points`,
  `data-total-resampled-points`, `data-mode`, `data-target`,
  `data-reduction-ratio`, `data-animate`.
- Badge mirrors the same mode + counts + ratio.

## Tooltip

Hovering a dot shows the series label in its colour, x, bold
y, and a "source" row indicating either `original #<i>`
(when the dot came from a kept input sample) or `synthesized
(<mode>)` (when it's a bucket average or interpolated
sample).

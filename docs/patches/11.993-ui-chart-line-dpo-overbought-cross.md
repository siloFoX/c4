## Summary

Add `web/src/components/ui/chart-line-dpo-overbought-cross.tsx`
-- a pure-SVG dual-panel React/TypeScript chart primitive that
renders the close in the top panel and the close-only
Detrended Price Oscillator (DPO) line in the bottom panel,
marking discrete bullish (DPO crosses up through overbought
threshold +1) / bearish (DPO crosses down through overbought
threshold +1) DPO overbought-crossover events. Overbought
threshold variant of the DPO family that flags discrete crossing
events at a configurable positive upper band -- the canonical
threshold analysts tune to the volatility scale of the
underlying close series for detrended-momentum overbought
signals.

## Math anchor (bit-exact)

```
shift     = floor(length / 2) + 1
sma[i]    = SMA(close, length)[i]
dpo[i]    = close[i - shift] - sma[i]
bullish   : prev dpo <= 1 && cur dpo >  1
bearish   : prev dpo >= 1 && cur dpo <  1
```

- **CONST close = K**: close[i-shift] = K, sma = K -> dpo = 0
  constant. dpo = 0 < 1 -> regime `bearish` (not overbought).
  0 crosses. Verified across K = 0..1234.
- **LINEAR UP close = i**: close[i-shift] = i - 11, sma[i] = i
  - 9.5 -> dpo = -1.5 constant. -1.5 < 1 -> regime `bearish`.
  0 crosses. Negative DPO reflects "look-back close sits below
  centered SMA" on a clean linear ramp.
- **LINEAR DOWN close = -i**: close[i-shift] = -(i-11), sma =
  -(i - 9.5) -> dpo = 1.5 constant. 1.5 >= 1 -> regime
  `bullish` (in overbought zone). 0 crosses. The DPO look-back
  inversion makes LINEAR DOWN read as `overbought` -- canonical
  DPO behavior on monotonic downtrends after the cycle
  saturates (look-back close, which is higher than current
  close, sits above the centered SMA).

For 50 bars at defaults (length=20, shift=11, threshold=1):
- dpo null for i = 0..18 (19 entries)
- post-warmup regime count = 31 (i = 19..49)

Strict inequality on `cur` (`cur > 1` for bullish, `cur < 1`
for bearish) means boundary equality (dpo == 1) never double-
fires.

Layout auto-fits oscMin / oscMax to include the threshold band
with 10% padding -- on CONST K=50 the panel expands UP to
include the threshold (oscMin=-0.1, oscMax=1.1); on LINEAR DOWN
the panel collapses around 1.5 (oscMin=0.95, oscMax=1.55).

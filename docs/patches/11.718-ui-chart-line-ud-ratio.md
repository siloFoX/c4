## Summary

New `ChartLineUdRatio` UI primitive in
`web/src/components/ui/chart-line-ud-ratio.tsx` -- a pure-SVG two-panel
line chart with an **Up/Down Ratio** panel from the ratio of total
up-bar change to total down-bar change. The top panel plots the close;
the bottom panel plots the UD ratio inside a fixed 0..100 band with a
horizontal 50 midline.

## The computation

The UD Ratio reads the bull share of total bar-to-bar change across the
lookback. `ChartLineUdRatioPoint` carries `{x, value}`.

- `computeLineUdRatio(values, period)` -- for each bar past the first,
  splits the price change into an `upGain` (the positive change) and a
  `downLoss` (the absolute negative change), then sums the two over the
  lookback and returns
  `udRatio = 100 * sum(upGain) / (sum(upGain) + sum(downLoss))`. A
  window with zero total change (a flat market) yields null.
- `classifyLineUdRatioZone(udRatio)` -- marks a bar **up** above the
  50 midline, **down** below, **flat** at the midline, **none** for
  null.

## Worked fixture

The whole pipeline is integer arithmetic with no transcendentals, so
integer fixtures compute to exact integer values.

A **strictly rising series** (`[1..10]`) makes every change a positive
up gain; the down loss is zero on every bar, the rolling ratio is
`100 * sum / sum = 100`. A **strictly falling series** mirrors it to 0.
A **constant series** has every change equal to zero, so the
denominator is zero and the UD ratio is null on every defined bar.
A **mixed fixture** (`[10, 14, 12, 16, 14, 18, 16, 20]`, deltas
`+4, -2, +4, -2, +4, -2, +4`) at period 3 gives the exact integer ratio
series `[null, null, null, 80, 50, 80, 50, 80]` (asserted with
`toEqual`) -- the alternating delta pattern makes every rolling window
sum either `{4+0+4, 0+2+0} = {8, 2} -> 80` or `{0+4+0, 2+0+2} = {4, 4}
-> 50`. The zone classifier is checked at and on either side of the
midline.

## API

`data` (a `ChartLineUdRatioPoint[]`), `period` (default 14),
`pricePanelRatio` / `gap`, sizing/format/a11y props, `strokeWidth` /
`dotRadius`, the colour props, the toggle flags (`showAxis` / `showGrid`
/ `showDots` default false / `showUdRatio` / `showMidline` /
`showMarkers` / `showTooltip` / `showConfigBadge` / `showLegend` /
`animate`), controlled + uncontrolled `hiddenSeries` (keys `price` /
`udRatio`), `formatValue` / `formatX`, `onPointClick` / `onSeriesToggle`.

Exported helpers: `getLineUdRatioFinitePoints`,
`normalizeLineUdRatioPeriod` (coerces the period to an integer of at
least 1), `computeLineUdRatio`, `classifyLineUdRatioZone`,
`runLineUdRatio`, `computeLineUdRatioLayout`,
`describeLineUdRatioChart`, plus the `DEFAULT_CHART_LINE_UD_RATIO_*`
constants, `CHART_LINE_UD_RATIO_MIDLINE` and the `ChartLineUdRatioZone`
/ `ChartLineUdRatioComputed` types. Every exported name carries a
`LineUdRatio` infix so the `export *` barrel line cannot collide.

## Accessibility

Root `<div role="region">` with `aria-label` + `aria-describedby`; SVG
`role="img"`; an off-screen
`data-section="chart-line-ud-ratio-aria-desc"` description span; the
close line, the UD line and every dot / marker are
`role="graphics-symbol" tabIndex={0}`. Every node carries a
`data-section`; the root mirrors `data-period`, `data-ud-ratio-final`,
`data-up-count`, `data-down-count`, `data-flat-count`,
`data-total-points` and `data-animate`.

# 11.526 -- web/src/components/ui/chart-line-rolling-min-max.tsx

Pure-SVG multi-series line chart that **shades the rolling min/max
envelope** of the data over a sliding window. For each finite sample
the chart computes the min and max of y values within a window of
size `W` (default 5) using one of three modes (`trailing` /
`centered` / `edge`) and paints a translucent band between them.
Dashed envelope lines trace the min/max edges; the raw line overlays
through the middle so the noise vs envelope reading is intuitive.

## Distinct from other envelope / aggregate primitives

- `<ChartLineMinMax>` (11.517): **absolute** min/max over the entire
  series (two markers + horizontal lines). Rolling min/max tracks
  these locally as a sliding band.
- `<ChartLineMovingAvg>` (11.513): windowed **average**, not min/max.
  Same window-mode API but different aggregation.
- `<ChartLinePercentile>` (11.514): percentile bands from **multiple
  samples at each x**. Rolling min/max derives the band from a
  **sliding window across x** of single-sample-per-x data.
- `<ChartLineConfidence>` (11.502): per-point bounds supplied
  **externally**. Rolling derives bounds from the data itself.
- `<ChartLineAnomaly>` (11.524): z-score outliers from global mean +
  stddev. Rolling is a local non-parametric envelope.
- `<ChartLineBaseline>` (11.510), `<ChartLineBand>` (11.516),
  `<ChartLineTarget>` (11.518): horizontal references from outside
  the data.

The canonical "high-low channel" / "Donchian channel" visualisation
used in finance, sensor monitoring, and any time series where you
want to see local volatility envelopes.

## Public surface

- `<ChartLineRollingMinMax series={...} window={5} mode="trailing" />`.
- Per-series `{id, label, data, color?, bandColor?, window?, mode?}`.
  `data` entries are raw `{x, y}`. Per-series `window` and `mode`
  override chart-level defaults.
- Three modes (mirroring `<ChartLineMovingAvg>` 11.513): `'trailing'`
  (default), `'centered'`, `'edge'`.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showBand`, `showEnvelope`, `showLine`, `animate`.
- `envelopeStrokeWidth` (1), `envelopeOpacity` (0.55),
  `envelopeDashArray` ('4 3'), `bandOpacity` (0.16).
- Standard sizing + format + a11y props.
- Controlled + uncontrolled `hiddenSeries`.
- `onPointClick({series, point})`.

## Pure helpers

- `getLineRollingMinMaxDefaultColor(index)` -- 10-color palette
  cycle.
- `getLineRollingMinMaxFinitePoints(points)` -- finite-only filter.
- `normaliseLineRollingMinMaxWindow(window)` -- positive integer >=
  1; non-finite / <1 / fractional / non-numeric clamp/floor to 1.
- `computeRollingMinMax(values, window, mode?)` -- canonical helper.
  Returns same-length `RollingMinMaxEntry[]` (`{min, max}` each
  `number | null`). Non-finite values are skipped within a window
  (window where every value is non-finite produces `{null, null}`).
  Modes:
  - **`trailing`** (default): `[i - W + 1, i]`, valid `i >= W - 1`.
  - **`centered`**: `[i - half, i + (W - 1 - half)]` where `half =
    floor((W - 1) / 2)`, valid only when both ends fit.
  - **`edge`**: same as trailing but the leading positions take a
    partial window so every index is valid.

  Window > length under `trailing` / `centered` -> all nulls.
- `buildLineRollingMinMaxBandPaths(points)` -- builds one closed
  polygon per contiguous run of `{px, pyMin, pyMax}` triples. Skips
  single-entry runs (no area). Returns an array of `d` strings so
  adopters can render multiple separate band regions when the
  envelope is broken by nulls.
- `buildLineRollingMinMaxEnvelopePath(points)` -- standard `M..L..`
  builder that breaks on null / non-finite py (gap rendering).
- `computeLineRollingMinMaxLayout({series, window?, mode?,
  hiddenSeries?, showBand?, ...})` -- returns `{series, xTicks,
  yTicks, xMin, xMax, yMin, yMax, innerWidth, innerHeight,
  totalPoints, visibleSeriesCount}`. Each layout series carries
  `{id, label, index (stable original), color, bandColor, window,
  mode, points, path, minPath, maxPath, bandPaths, validCount,
  finiteCount, totalCount}`. Per-point `{index, x, y, px, py, min,
  max, pyMin, pyMax}`. y range expanded to include all observed
  rolling min/max samples.
- `describeLineRollingMinMaxChart(series, window?, mode?, hidden?)`
  -- builds the screen-reader description.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Raw line + dots are `role="graphics-symbol"` + `tabIndex={0}`;
  envelope paths + band polygons are `role="graphics-symbol"` only.
- Every node carries `data-section="chart-line-rolling-min-max-..."`
  for test selectors.
- Series groups expose `data-series-window`, `data-series-mode`,
  `data-series-valid-count`, `data-series-color`,
  `data-series-band-color`, `data-series-finite-count`.
- Dots expose `data-rolling-min`, `data-rolling-max` (empty string
  when window unfilled).
- Envelopes expose `data-envelope-kind` (`'min' | 'max'`).
- Root mirrors `data-window`, `data-mode`.

## Tooltip

Hovering a dot shows the series label, x, y (bold), and
`rolling max (W):` / `rolling min (W):` rows with the value or
`n/a` when the window cannot be filled.

# 11.154 -- UI: Stepper multi-step wizard primitive (v1.11.172)

## Summary

Add a `<Stepper>` UI primitive at `web/src/components/ui/stepper.tsx` for
multi-step / wizard-style flow indicators. Numbered circular badges with
state-driven color (complete / current / pending), connector line
between steps (solid-primary when both ends are complete, muted
otherwise), vertical orientation renders descriptions under labels,
optional onStepClick gated by `allowFuture`.

## API

```tsx
<Stepper
  steps={[
    { id: 'a', label: 'Select worker' },
    { id: 'b', label: 'Describe task' },
    { id: 'c', label: 'Dispatch' },
    { id: 'd', label: 'Result' },
  ]}
  currentIndex={2}
  orientation="horizontal"
  size="sm"
  onStepClick={(i) => goTo(i)}
  allowFuture={false}
/>
```

- `steps`: `{ id, label, description? }[]`
- `currentIndex`: 0-based index of the current step
- `orientation`: `'horizontal'` (default) or `'vertical'`
- `onStepClick(i)`: optional click handler -- when provided, complete +
  current rows are clickable; future rows become clickable only when
  `allowFuture` is `true`
- `allowFuture`: default `false`
- `size`: `'sm' | 'md'` (default `'md'`)
- `className`: passes through to the root `<ol>`

Renders `role="list"` on the `<ol>`, `role="listitem"` per step, and
`aria-current="step"` on the current row. Each badge is a `<button>`
that disables itself when not clickable, so keyboard users skip
unreachable steps.

## Adoption sites

1. `web/src/pages/Plan.tsx` -- horizontal Stepper above the form
   summarising the planning flow stages
   (`Select worker -> Describe task -> Dispatch plan -> View plan`),
   `currentIndex` derived from local state (selection / task text /
   dispatching flag / plan content).
2. `web/src/pages/Config.tsx` -- new `DispatchLifecycleDemo` panel
   appended below the import section, showing the autonomous dispatch
   lifecycle (`dispatch -> work -> verify -> merge -> complete`) as a
   contextual demo of the primitive.

## Tests

`web/src/components/ui/stepper.test.tsx` covers 12 cases: label
rendering, numbered indicators for pending/current, check icon on
complete, `aria-current="step"` on current only, vertical renders
descriptions, horizontal omits descriptions, click gating for
`allowFuture=false` and `allowFuture=true`, className merge, forwardRef,
connector colour by neighbour-completion, last-step connector omitted.

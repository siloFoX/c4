This is a higher-level composition
that bundles the existing per-shape
`<Skeleton.Card>` / `.List` / `.Table`
/ `.Text` primitives (11.135 / 11.174
/ 11.208 / 11.255) into the four
canonical page silhouettes operators
see while loading.

## API

```ts
type SkeletonSetVariant =
  | 'card'
  | 'list'
  | 'table'
  | 'detail-page';

interface SkeletonSetProps {
  variant: SkeletonSetVariant;
  count?: number;        // card
  rows?: number;          // list / table
  cols?: number;          // table
  showAvatar?: boolean;   // list
  linesPerRow?: number;   // list
  className?: string;
  ariaLabel?: string;
  // ...HTMLAttributes<HTMLDivElement>
}
```

## Variants

| variant | default | configurable |
| --- | --- | --- |
| `card` | 3 cards in a responsive grid (1/2/3 cols on phone/sm/lg) | `count` |
| `list` | 6 rows, 2 text lines each, no avatar | `rows`, `showAvatar`, `linesPerRow` |
| `table` | 5 body rows + header, 4 cols | `rows`, `cols` |
| `detail-page` | title + subtitle header over 2-col body (main: paragraph + card; side: 2 cards) | (no per-variant knobs) |

## Behaviour

### Shimmer + reduced motion

The component reads
`useReducedMotion()` (media query
`(prefers-reduced-motion: reduce)`)
and routes the animation through
the shared `loading-motion`
contract:

- when not reduced: `animate-pulse`
  utility on every shape.
- when reduced: the animation class
  drops; the root carries
  `data-motion-reduced="true"` so
  tests + downstream styling can
  branch.

### Sanity guards

- `count`, `rows`, `cols` clamp to
  `>= 0` and floor to integer
  (negative / non-finite -> 0).
- `linesPerRow` clamps to `>= 1`.

### Composition

- card variant -> grid of
  `<Skeleton.Card>` placeholders.
- list variant -> `<Skeleton.List>`
  delegate.
- table variant -> `<Skeleton.Table>`
  delegate.
- detail-page variant -> header
  bars (rendered locally) + main
  column (`<Skeleton.Text lines=3>`
  + `<Skeleton.Card>`) + side
  column (two `<Skeleton.Card>`).

## ARIA + data attributes

Root (`role="status"`,
`aria-busy="true"`, `aria-label`):

- `data-section="skeleton-set"`
- `data-variant`
- `data-motion-reduced`
  ('true' / 'false')

Variant containers:

- `data-section="skeleton-set-cards"`
  + `data-card-count` (card)
- `data-section="skeleton-set-list"`
  delegates to `<Skeleton.List>`
- `data-section="skeleton-set-table"`
  delegates to `<Skeleton.Table>`
- `data-section="skeleton-set-detail"`
  (detail-page wrapper)
- `data-section="skeleton-set-detail-header"`
- `data-section="skeleton-set-detail-body"`
- `data-section="skeleton-set-detail-main"`
- `data-section="skeleton-set-detail-side"`
- `data-skeleton-set-card-index`
  (card variant)
- `data-skeleton-detail`
  ("title" / "subtitle") on
  detail header bars

Default `aria-label`:
`"Loading <variant>"`.

## Pairs with existing primitives

- `<Skeleton>` + `Skeleton.Card` /
  `.List` / `.Table` / `.Text` /
  `.Avatar` (11.135 / 11.174 /
  11.208 / 11.255) -- building
  blocks.
- `<SuspenseWrapper>` (11.158) --
  often hosts SkeletonSet as a
  fallback for a whole page.
- ThemeCustomizer (11.394) --
  shimmer bg-muted resolves via
  token so placeholders auto-
  theme.

## Out of scope (deferred)

- Per-card content slot (avatar /
  thumbnail variants). Skeleton.
  Card is the only card shape for
  v1; richer per-card variants
  belong in a follow-on.
- Form skeleton variant. Field
  silhouettes do not map onto the
  card / list / table / detail
  axis; a separate variant or
  helper is needed.
- Stat dashboard variant (rows of
  stat tiles). StatCardShape
  covers single tiles; bundling
  them into a "dashboard"
  composition is a separate
  primitive.
- Adoption sweep through existing
  loading states. SuspenseWrapper +
  per-page skeletons still ship
  their own; SkeletonSet is the
  canonical surface for new
  adopters.

# 11.178 -- UI: NotificationBanner primitive (v1.11.196)

## Summary

Add a `<NotificationBanner>` UI primitive at
`web/src/components/ui/notification-banner.tsx` for sticky top-of-page
system messages (paused dispatcher, halted loop, reviewer escalation,
maintenance notices). Wires the primitive to the live autonomous
dispatcher state via a small `AutonomousStatusBanner` wrapper mounted
in `App.tsx`.

## API

```tsx
<NotificationBanner
  variant="warn"                 // 'info' | 'warn' | 'critical' (default 'info')
  title="Autonomous loop paused"
  description="Paused manually."
  icon={<AlertTriangle />}        // optional; defaults per variant
  action={<Button size="sm">Resume</Button>}
  onDismiss={() => setHidden(true)}
  dismissLabel="Dismiss"          // default 'Dismiss'
  sticky                          // default true -> sticky top-0 z-40
  className="text-xs"
/>
```

Variant -> token mapping (consumes ARPS design-system color tokens
`info` / `warning` / `destructive` already in tailwind.config.js):

| Variant    | Background        | Border              | Text                          | Default icon  | Role     |
| ---------- | ----------------- | ------------------- | ----------------------------- | ------------- | -------- |
| `info`     | `bg-info/10`      | `border-info`       | `text-info-foreground`        | `Info`        | `status` |
| `warn`     | `bg-warning/10`   | `border-warning`    | `text-warning-foreground`     | `AlertTriangle` | `status` |
| `critical` | `bg-destructive/10` | `border-destructive` | `text-destructive-foreground` | `AlertOctagon` | `alert`  |

`role='alert'` carries `aria-live='assertive'`; the other two carry
`aria-live='polite'`. The wrapper is `forwardRef<HTMLDivElement>` so
callers can attach refs for scroll-into-view or measurement.

Dismiss button is only rendered when `onDismiss` is provided. The
shadow (`shadow-sm`) is always applied so the banner reads as
elevated whether or not the page is scrolled. Class merging uses the
shared `cn()` helper.

## Adoption

`web/src/components/AutonomousStatusBanner.tsx` polls
`/api/autonomous/status` every 15s and maps the dispatcher payload
(`{ enabled, paused, pauseReason, consecutiveHalts, circuitThreshold,
pendingEscalations }`) onto a banner:

- `enabled === false` -> no banner.
- `consecutiveHalts >= circuitThreshold` -> `variant='critical'`,
  "Autonomous loop halted".
- `pendingEscalations > 0` -> `variant='critical'`,
  "Reviewer attention required".
- `paused === true` -> `variant='warn'` with `pauseReason` as the
  description.

The wrapper is mounted in `App.tsx` directly below `MetricsBar` so the
banner spans the full viewport above the page content.

## Tests

`web/src/components/ui/notification-banner.test.tsx` covers 16 cases:
title text, description, default info classes, warn classes, critical
classes + `role='alert'`, info/warn `role='status'`, default icon per
variant, custom icon override, action slot, onDismiss callback,
dismiss button absent when `onDismiss` is missing, custom
`dismissLabel`, sticky=true classes, sticky=false omits them,
className merge, and forwardRef.

## Out of scope

- No automatic enter/exit animation -- callers can wrap with the
  existing motion helpers if needed.
- Stacking multiple banners is not modelled here; the autonomous
  wrapper picks the single most urgent state per render.
- No persistence of the dismissed state across reloads -- callers
  manage their own `dismissed` flag if desired.

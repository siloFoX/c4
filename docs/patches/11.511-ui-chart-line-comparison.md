# 11.511 -- web/src/components/ui/chart-line-comparison.tsx

Pure-SVG line chart for **comparing two series via difference shading**.
The chart paints the polygon strip between a `primary` curve and a
`secondary` curve and colours each segment by **which curve sits on
top** at that span: `primaryAboveColor` when `primary > secondary`,
`primaryBelowColor` when `primary < secondary`. When the curves cross
inside a segment, the segment is split exactly at the crossing point
and the two halves are painted in opposite directions.

## Distinct from other line primitives

- `<ChartLineConfidence>` (11.502) draws an upper/lower **band around a
  single series** with per-point bounds.
- `<ChartLineThreshold>` (11.503) draws **horizontal reference lines**
  plus zone shading above/below those references.
- `<ChartLineAreaGradient>` (11.504) fills the area **below one line
  toward a flat baseline** with a vertical gradient.
- `<ChartLineAreaStacked>` (11.505) **stacks multiple series** on top
  of each other (no comparison concept).
- `<ChartLineBaseline>` (11.510) classifies one curve's samples against
  a **flat horizontal baseline**.

`<ChartLineComparison>` is the only one whose shaded polygon is bounded
by **two curves** rather than a curve + a constant or a curve + a
per-point band. The shading colour therefore encodes a direction
(primary higher vs secondary higher) that flips at every crossing.

## Public surface

- `<ChartLineComparison primary={...} secondary={...} />` -- both
  `primary` and `secondary` are nullable. Either or both may be `null`
  to render an empty chart.
- Per-series `color` override always beats the direction-default
  palette (primary blue, secondary red).
- `primaryAboveColor` / `primaryBelowColor` override the region fill.
- Optional `xMin/xMax/yMin/yMax`, `tickCount`, `padding`,
  `strokeWidth`, `dotRadius`, `lineOpacity`, `fillOpacity`,
  `gridColor`, `axisColor`.
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showRegions`, `animate`.
- `formatValue` / `formatX` for tick + tooltip formatting.
- `xLabel` / `yLabel` axis-label slots.
- `onPointClick({track, point})` and `onRegionClick({region})`.

## Pure helpers

- `getLineComparisonDefaultColor(index)` -- 10-color palette cycle.
- `getLineComparisonFinitePoints(points)` -- finite-only filter.
- `buildLineComparisonXUnion(primary, secondary)` -- sorted unique x
  values across both series.
- `buildLineComparisonYLookup(series)` -- `Map<x, y>` of finite samples.
- `interpolateLineComparisonY(x, x1, y1, x2, y2)` -- linear
  interpolation between two pixel-space points.
- `findLineComparisonCrossing(x1, p1, s1, x2, p2, s2)` -- returns the
  x at which the segments cross, or `null` if no crossing exists or
  inputs are degenerate.
- `computeLineComparisonLayout({primary, secondary, width, height,
  padding, ...})` -- returns `{primary, secondary, regions, stats,
  xTicks, yTicks, xUnion, xMin, xMax, yMin, yMax, innerWidth,
  innerHeight}`.
- `describeLineComparisonChart(primary, secondary, formatValue?)` --
  builds the screen-reader description.

`stats` aggregates `primaryHigherCount`, `secondaryHigherCount`,
`equalCount`, `maxPrimaryGap`, `maxSecondaryGap`, `crossingCount`, and
`xUnionCount` across the shared x samples.

## Region build algorithm

The chart walks the sorted x union pairwise:

1. Skip the segment if either series lacks a finite sample at one of
   the two endpoints.
2. If both endpoints agree on which curve is higher (no crossing),
   emit a single quadrilateral polygon `(x1, p1) -> (x2, p2) -> (x2,
   s2) -> (x1, s1) -> Z`.
3. If the endpoints disagree (the curves cross inside the segment),
   compute the crossing x via the absolute-delta proportion
   `x* = x1 + (x2 - x1) * |p1 - s1| / (|p1 - s1| + |p2 - s2|)` and the
   shared y by linear interpolation, then emit **two** polygons that
   each respect their own direction.

A degenerate `|p1 - s1| + |p2 - s2|` of zero, or a zero delta on either
side, is treated as **no crossing**.

## ARIA + test selectors

- Root `role="region"` with auto description (`describeLineComparisonChart`).
- SVG `role="img"`.
- Both line paths and every dot expose `role="graphics-symbol"` and
  `tabIndex=0`.
- Every node carries `data-section="chart-line-comparison-..."` for
  test selectors.
- Track groups expose `data-track="primary" | "secondary"`,
  `data-series-id`, `data-series-color`, `data-series-finite-count`.
- Regions expose `data-region-index`, `data-region-start`,
  `data-region-end`, `data-region-primary-higher`,
  `data-region-fill-color`, `data-region-area`.

## Tooltip

Hovering a dot shows a four-line tooltip:

- track label,
- `x: <fmtX(x)>`,
- `y: <fmtValue(y)>`,
- `vs <other label>: +/-<fmtValue(delta)>` (only when the other curve
  has a matching x sample).

- Default constants are positive, valid hex, palette has 10 entries.
- `getLineComparisonDefaultColor`: cycle, NaN fallback, fractional
  floor.
- `getLineComparisonFinitePoints`: keep finite, drop NaN/+-Inf, return
  [] for non-array input.

  when empty.

  NaN-x fallback, NaN-y1 returns 0, extrapolation.

  null, exact-zero delta returns null, degenerate segment returns
  null, non-finite returns null, biased crossing.

  expansion, primaryAbove/Below color overrides on regions, area =
  |gap|*|dx|, ticks honor stepCount.

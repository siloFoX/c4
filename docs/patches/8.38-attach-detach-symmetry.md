# 8.38 — Attach role detection + two-step detach

Branch: `dgx-spark/attach-detach-symmetry`

## What ships

Two parts of TODO 8.38: role-aware attach (so manager / worker /
planner / executor / reviewer / generic can be told apart) and a
two-step detach with explicit "your terminal keeps running" copy so
the operator never wonders whether the underlying claude process is
about to die.

The Detach surface itself was already shipped in 8.31 — what was
missing was the *role* signal and the *confirmation strip*.

## Surface area

### `src/session-attach.js`

- New `detectAgentRole(jsonlPath)` helper. Returns one of `'manager'
  | 'worker' | 'planner' | 'executor' | 'reviewer' | 'generic'`.
  Detection order:
  1. **Path heuristic** — if the JSONL lives under `c4-mgr-*` or
     `auto-mgr-*`, return `'manager'` (no file read).
  2. **Content sniff** — read the first 64 KiB and match against
     `[Role: Manager]` / `[역할: Manager]` / `Auto-spawned by c4 auto`
     / `halt-prevention` / `approval protocol` for managers; analogous
     prefixes for planner / executor / reviewer.
  3. **Path fallback** — `c4-worktree-*` or `/c4/` paths default to
     `'worker'`.
  4. Anything else: `'generic'`.
  Returns `'generic'` (never throws) on missing files / read errors.
- New `ROLE_VALUES` export (the canonical role enum).
- `normalizeRecord` now carries `role` through (defaults to
  `'generic'` if absent / unknown).
- `AttachStore.add` sniffs the role at attach-time and records it on
  the persisted entry so `/api/attach/list` doesn't have to re-read
  the JSONL on every refresh.
- `AttachStore._load` heals legacy records that landed before role
  detection existed: if a record's role is missing or `'generic'`,
  the store re-sniffs and persists the upgrade back to disk on the
  next list call.

### `web/src/components/SessionsView.tsx`

- New `AttachedRole` discriminated union mirrors the daemon enum.
- `AttachedSession` exposes optional `role`.
- New `attachedRoleStyle(role)` maps each role to a token-backed
  badge class — manager gets the primary accent, planner / executor /
  reviewer share neutral secondary, worker is muted, generic is
  fully muted.
- `AttachedRowActions` now renders a small role badge + a "read-only
  mirror" hint above the action buttons, so operators see the role
  at a glance and know detach won't kill anything live.
- Detach button is two-step: first click expands an inline
  destructive-styled strip with "Your terminal session keeps running
  — only the read-only mirror is dropped" + Cancel / Detach session
  buttons. `aria-expanded`-on-the-trigger + `role="alert"` on the
  strip drive screen reader hints.

## Tests

`tests/attach-detach-symmetry.test.js` — 26 assertions across 5
suites:

1. `detectAgentRole` — generic on falsy input, Korean / English role
   prefixes, auto-spawn signal, planner / executor / reviewer
   prefixes, c4-mgr-* / auto-mgr-* / c4-worktree-* path heuristics,
   missing-file safety.
2. `AttachStore` — `add()` writes role on new attachments;
   `_load()` heals legacy records and persists the upgrade;
   `ROLE_VALUES` export carries the canonical enum.
3. `SessionsView` (source-grep) — `AttachedRole` union, `role` field
   on `AttachedSession`, `attachedRoleStyle` covering each role,
   role badge + "read-only mirror" hint, two-step detach with
   "Your terminal session keeps running" copy, detach button toggles
   confirmation rather than firing onDetach directly, **review-fix
   `aria-controls` reference** (regression guard so the
   expand/collapse trigger always points at a real element when
   expanded and drops the reference when collapsed).
4. **Behavioural `attachedRoleStyle`** — manager → primary,
   planner / executor / reviewer share the secondary palette,
   worker → muted/60, generic / undefined / unknown roles → neutral
   muted (no accidental promotion). Source mirror.
5. **`AttachStore.add` preserves explicit non-generic role** —
   manual `role='reviewer'` wins over a JSONL prelude that says
   manager (the `role === 'generic'` re-sniff guard fires only
   when caller didn't pin a role); invalid roles outside
   `ROLE_VALUES` get coerced to generic and re-sniffed (so a
   typo'd `role='admin'` falls back to detection on a planner
   transcript).

Full suite: 116 / 117 pass. The lone failure is the pre-existing
`file-transfer` test PR #24 fixes (origin's hardcoded
`/root/project` path). `npm --prefix web run build` holds the 47 TS
error baseline.

## Test plan (manual)

1. `c4 daemon start && cd web && npm run dev`
2. Spawn a manager (`c4 auto "..."`) and a regular worker. Note the
   worker JSONL paths.
3. From the Sessions tab → Attach new... attach the manager JSONL by
   path. The attached row now shows a `MANAGER` badge in primary
   accent + "read-only mirror" hint.
4. Attach the worker JSONL. Row shows `WORKER` badge in muted accent.
5. Attach an unrelated `~/.claude/projects/.../*.jsonl` (a non-c4
   session). Row shows `GENERIC` badge.
6. Click Detach on any attached row → inline destructive-styled
   confirmation strip slides in. Click Cancel → strip collapses.
   Click Detach session → row removes from the list.
7. Verify the original `claude` terminal (where the JSONL is being
   written) keeps running and isn't affected by detach.
8. Manually delete the `role` field from a record in
   `~/.c4/attached.json`, restart the daemon, hit /api/attach/list —
   the field re-fills with the sniffed role.

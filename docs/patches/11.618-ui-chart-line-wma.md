## Summary

New `ChartLineWma` UI primitive at
`web/src/components/ui/chart-line-wma.tsx` -- a pure-SVG line chart
with a linearly Weighted Moving Average overlay drawn on the same
axes as the price.

## Indicator math

The linearly Weighted Moving Average. Over a trailing window of
`period` values:

```
WMA[i] = sum(weight_k * value_k) / (period * (period + 1) / 2)
```

where the oldest value in the window carries weight 1, the next
weight 2, ..., and the newest weight `period`. The weighted sum is
divided by the triangular weight total `period * (period + 1) / 2`.

Because the newest value carries the heaviest weight, the WMA reacts
to the latest price faster than a simple equal-weight moving
average. It is a **non-recursive fixed-window** average -- unlike
the EMA-based moving averages it carries no running state -- and is
defined from index `period - 1` onward.

### Worked fixture

`values = [6, 6, 12, 12, 6, 6]`, `period 3`. The weight total is
`3 * (3 + 1) / 2 = 6` and the window values divide cleanly, so the
WMA stays bit-exact:

```
WMA[i] = (1*oldest + 2*middle + 3*newest) / 6
WMA[2] = (1*6  + 2*6  + 3*12) / 6 = 54/6 =  9
WMA[3] = (1*6  + 2*12 + 3*12) / 6 = 66/6 = 11
WMA[4] = (1*12 + 2*12 + 3*6 ) / 6 = 54/6 =  9
WMA[5] = (1*12 + 2*6  + 3*6 ) / 6 = 42/6 =  7
WMA = [null, null, 9, 11, 9, 7]
```

The price climbs above the WMA on the way up and drops below it on
the way down. Every value is exact, so the tests assert the series
with `toEqual`. A separate case shows the recency weighting: the
window `[1, 1, 4]` yields `(1 + 2 + 12) / 6 = 2.5`, pulled above the
simple mean `(1 + 1 + 4) / 3 = 2` because the newest value 4 carries
the heaviest weight.

## API

- `getLineWmaFinitePoints(points)` -- drop points with a non-finite
  `x` or `value`.
- `normalizeLineWmaPeriod(period, fallback)` -- coerce to a positive
  integer; non-finite or sub-1 falls back, fractional floors.
- `computeLineWma(values, period)` -- the WMA series, null through
  the warm-up.
- `runLineWma(points, options?)` -- sort finite points by `x`,
  compute the series, return `{ series, period, wma, samples,
  wmaFinal, wmaMin, wmaMax, aboveCount, belowCount, ok }`. Each
  sample carries the price's `above` / `below` / `on` position
  versus the WMA. `ok` is false for fewer than two points.
- `computeLineWmaLayout(options)` -- project the price and WMA onto
  one shared panel. Returns the panel, ticks, the price path, the
  WMA path, and WMA markers.
- `describeLineWmaChart(data, options?)` -- ARIA description.

## Component

`ChartLineWma` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, axis ticks and grid, a
config badge, a two-series legend (Price / WMA) with toggle, a
hover/focus tooltip exposing the value, WMA and position, and a
`motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs a simple moving average** -- the SMA weights every value in
  the window equally. The WMA weights them by a *linear ramp*, so
  the newest bar moves the average `period` times as much as the
  oldest.
- **vs chart-line-alma** -- the ALMA weights the window with a
  *gaussian* curve centred by an offset parameter. The WMA uses a
  plain *linear* ramp peaking at the newest bar.
- **vs chart-line-dema / chart-line-tema** -- those combine
  recursive *exponential* moving averages to cancel lag. The WMA is
  a single *non-recursive* fixed-window average and carries no
  running state. (The Hull moving average is itself composed from
  several WMAs -- the WMA is the building block, not a competitor.)

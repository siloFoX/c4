## Goal

Dispatch:

- Build `web/src/components/ui/data-table.tsx` --
  column header sort indicators, multi-column
  sort, type-aware filters
  (text/select/range/date), URL state sync.

The existing `<Table>` primitive at
`components/ui/table.tsx` (11.130 ships single-
column sort) + `useTableSort` hook
(11.240 ships localStorage sort persistence) cover
the single-column surface. This patch adds the
multi-column / typed-filter / URL-sync layer as a
new `DataTable` composite that wraps the existing
helpers without disturbing them.

## State helpers (`lib/data-table-state.ts`)

Pure functions, no React, no DOM. All exports
testable in isolation.

### Sort

```ts
type SortDescriptor = { key: string; dir: 'asc' | 'desc' };

applyMultiSort<T>(rows, sortBy, accessor?): T[]
toggleSort(current, columnKey, { shiftKey? }): SortDescriptor[]
```

- Stable sort via decorate-sort-undecorate (ES2019
  guarantees stability but the explicit
  tiebreaker is defence in depth).
- Nulls sort last regardless of direction. The
  sign flip from `dir: 'desc'` skips
  null-vs-non-null comparisons.
- `toggleSort` cycles asc -> desc -> clear on plain
  click. Shift+click appends a new column or
  flips an existing one in place; shift+click on
  a desc column removes it.

### Filters

```ts
type ColumnFilter =
  | { type: 'text'; value: string }
  | { type: 'select'; values: readonly string[] }
  | { type: 'range'; min?: number; max?: number }
  | { type: 'date'; from?: string; to?: string };

applyFilters<T>(rows, filters, accessor?): T[]
```

- Text: case-insensitive substring; empty value
  is a no-op.
- Select: row passes when its value is one of
  the entries; empty list is a no-op.
- Range: inclusive `[min, max]` on numeric
  fields.
- Date: inclusive ISO date range. The `to`
  bound is bumped to end-of-day so the
  ISO-truncated `2026-05-18` covers events at
  23:59.
- All filters AND-combine. Missing field values
  fail the range/date checks.

### URL sync

```ts
serializeTableState({ sortBy, filters, paramPrefix? }): { params }
parseTableState(params, paramPrefix?): { sortBy, filters }
useUrlTableState({ paramPrefix?, defaultSort?, defaultFilters?, read?, onWrite? })
```

Format:

```
sort=col1:asc,col2:desc
f.<col>=text:foo
f.<col>=select:open|done
f.<col>=range:1..100
f.<col>=date:2026-05-01..2026-05-18
```

`paramPrefix` namespaces multiple tables on one
route (e.g. `users-` -> `users-sort`,
`users-f.role`).

The hook reads the URL on mount, writes back on
every state change via
`window.history.replaceState`. The `read` /
`onWrite` knobs let tests drive without a
browser; the same knobs let a future React
Router adoption plug in `useSearchParams()`.

Unrelated URL params are preserved so a table
state update does not nuke auth / pagination
query state.

## DataTable component (`components/ui/data-table.tsx`)

```tsx
<DataTable
  columns={[
    { key: 'name', label: 'Name', sortable: true, filter: { type: 'text' } },
    { key: 'age',  label: 'Age',  sortable: true, filter: { type: 'range', step: 1 } },
    {
      key: 'status',
      label: 'Status',
      sortable: true,
      filter: { type: 'select', options: [{ value: 'open' }, { value: 'closed' }] },
    },
  ]}
  rows={rows}
  sortBy={sortBy}
  onSortByChange={setSortBy}
  filters={filters}
  onFilterChange={(col, filter) => setFilters({...})}
/>
```

### Behaviour

- Sortable headers are buttons that cycle
  asc -> desc -> clear. Shift+click appends to
  the sort list (multi-column).
- Active sort columns show an arrow icon (up
  for asc, down for desc) plus a priority
  badge (1, 2, 3, ...) when more than one
  column is active. The badge hides when a
  single column is sorted.
- `aria-sort` set on the active header
  (`ascending` / `descending` / `none`).
- The filter row renders when at least one
  column declares a `filter`. Each column
  picks the input shape that matches its
  type.
- The `select` filter supports `multi: true`
  (renders a `<select multiple>` checkbox
  list) and single-select default (renders a
  `<select>` with an `All` blank option).
- Per-column `accessor` lets a row store the
  comparable value elsewhere; same accessor
  wires sort + filter so they agree.
- Per-column `render` for custom cells; the
  default coerces `row[key]` to a string.
- Renders the empty-state message when no rows
  pass the filter.
- Exported from `components/ui/index.ts`.

### What it does not do

- Pagination: out of scope. Adopters wrap with
  the existing `Pagination` primitive.
- Server-side sort / filter: helpers run
  client-side; an adapter can swap them out.
- Editable cells: out of scope.
- Column reorder / hide: out of scope (the
  existing `ColumnPicker` primitive covers
  hide).

## Tests

`data-table-state.test.ts` -- 31 cases:

- `applyMultiSort`: empty sortBy, single asc /
  desc, secondary tiebreaker, nulls-last in
  both directions, stable for tied rows.
- `toggleSort`: cycle (asc -> desc -> clear),
  replace on plain click, append on shift,
  flip on shift, remove on shift+desc.
- `applyFilters`: empty -> all rows, text
  case-insensitive substring, empty text no-op,
  select multi-value, empty select no-op,
  range bounds + open-ended, date inclusive
  `to`, date excludes missing fields, AND of
  multiple filters.
- `serializeTableState` + `parseTableState`:
  round-trip multi-column sort, round-trip
  every filter type, omit empty filters,
  paramPrefix, URLSearchParams shape, malformed
  sort segments dropped.
- `useUrlTableState`: defaults on empty URL,
  parses initial URL, writes back on state
  change, setFilter(null) removes, reset
  restores defaults, preserves unrelated URL
  params.

`data-table.test.tsx` -- 21 cases:

- Renders header + filter row + data rows
  (and empty message), `aria-sort` flag on
  active column.
- Sort click cycles (asc / desc / clear).
- Shift+click appends a secondary sort + shows
  priority badges.
- Single-column sort hides the priority badge.
- All four filter types narrow the visible
  rows (text, range, select). Clearing the
  text filter restores all rows.
- Per-column `accessor` + `render` overrides.
- Filter row omits when no column declares a
  filter.
- onSortByChange / onFilterChange callbacks
  emit the right shape.

52/52 pass under vitest 4.1.5. TypeScript clean
for the new source files. Pre-existing
strict-tuple warning in `skeleton.tsx` is
unchanged and out of scope.

## Pairs with existing primitives

- `components/ui/table.tsx` (11.130) + `useTableSort`
  (11.240) unchanged; cover the single-column
  surface. Adopters who need the new features
  switch to `<DataTable>`; existing call sites
  keep working.
- `lib/format-locale.ts` (11.346) -- cells that
  render numbers / dates should run through the
  locale formatters.
- The URL state hook composes cleanly with the
  scroll-restore hook (11.348) -- both write to
  the URL bar via `history.replaceState`; they
  do not collide because they touch different
  param prefixes.

## Out of scope

- Per-page adoption. Snapshots / TokenUsage /
  Workers already use the legacy `<Table>` +
  `useTableSort`; each migration pairs better
  with its page's own loading-state refactor.
- React Router integration for the URL hook.
- Inline editing.
- Sticky / pinned columns.
- Virtualised rendering (use `VirtualizedList`
  for large datasets; the canonical
  unification is a follow-up).

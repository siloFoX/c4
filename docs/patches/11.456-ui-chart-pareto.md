## Summary

Add a new `<ChartPareto>` UI primitive in
`web/src/components/ui/chart-pareto.tsx` -- pure-SVG
Pareto chart. Categories sort by value descending and
render as bars on the left axis; a cumulative percentage
line overlays on the right axis (0..100%). A dashed
threshold reference line (default 80%) marks the Pareto
principle boundary; bars at or below that line are tagged
as "vital few" via `data-vital-few`. Per-bar tooltip
shows label + value + cumulative percent. Per-bar click
handler.

## API

```ts
interface ChartParetoCategory {
  id: string;
  label: string;
  value: number;
  color?: string;
}

interface ChartParetoProps {
  categories: readonly ChartParetoCategory[];
  width?: number;                  // default 600
  height?: number;                 // default 320
  padding?: number;                // default 40
  barGap?: number;                 // default 6
  paretoThreshold?: number;        // default 0.8
  showBars?: boolean;              // default true
  showLine?: boolean;              // default true
  showThreshold?: boolean;         // default true
  showLabels?: boolean;            // default true
  showLeftAxis?: boolean;          // default true
  showRightAxis?: boolean;         // default true
  showTooltip?: boolean;           // default true
  animate?: boolean;               // default true
  className?: string;
  ariaLabel?: string;              // default 'Pareto chart'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  formatPercent?: (v: number) => string;
  defaultBarColor?: string;        // default '#2563eb'
  lineColor?: string;              // default '#dc2626'
  thresholdColor?: string;         // default '#f59e0b'
  tickCount?: number;              // default 5
  onBarClick?: (args: {
    category: ChartParetoCategory;
    index: number;
    cumulativePercent: number;
  }) => void;
}
```

## Behaviour

- Renders a `<div role="region">` wrapping `<svg
  role="img">` carrying the same `aria-label`.
- Sorting: `sortParetoCategoriesDesc` drops categories
  with non-finite / non-positive values and sorts the
  rest by `value` descending. The component renders
  this sorted list -- so the bars are always in Pareto
  order regardless of input order.
- Bar geometry: bars stack horizontally with
  `barGap` between them. Bar height scales by
  `value / max`. Bar color: per-category `color` wins;
  otherwise `defaultBarColor`.
- Cumulative line: each bar position emits a dot at
  `(barCenterX, yForPercent(cumulativePercent))`,
  joined by a polyline. The polyline scale uses the
  right axis (0..100%). Line markers are small filled
  circles at each point.
- Threshold reference: a dashed horizontal line at the
  configured `paretoThreshold` (clamped to [0, 1]) with
  a small label showing the percent. `showThreshold=
  false` suppresses; per-thresholdColor overrides.
- Vital few: bars whose cumulative percent crosses the
  threshold first carry `data-vital-few="true"`; later
  bars carry `data-vital-few="false"`. The tooltip also
  surfaces a "vital few" hint for the marked bars.
- Dual axes: left = value axis (0 -> max); right =
  percent axis (0 -> 100%). Either side can be
  individually suppressed via `showLeftAxis=false` /
  `showRightAxis=false`.
- Tooltip: hover opens a tooltip with the category
  label, value, cumulative percent, and the optional
  vital-few hint.
- Mount animation: `motion-safe:animate-fade-in` per
  bar group. `animate=false` removes the class.
- Default ARIA description: `"Pareto chart with N
  categories. Total <total>. Vital few (>=
  <threshold%>): <labels>."` --
  `ariaDescription` overrides; empty / all-non-positive
  produces `"No data"`.

## Pure helpers (exported)

```ts
DEFAULT_CHART_PARETO_WIDTH            = 600
DEFAULT_CHART_PARETO_HEIGHT           = 320
DEFAULT_CHART_PARETO_PADDING          = 40
DEFAULT_CHART_PARETO_BAR_GAP          = 6
DEFAULT_CHART_PARETO_THRESHOLD        = 0.8
DEFAULT_CHART_PARETO_TICK_COUNT       = 5
DEFAULT_CHART_PARETO_BAR_COLOR        = '#2563eb'
DEFAULT_CHART_PARETO_LINE_COLOR       = '#dc2626'
DEFAULT_CHART_PARETO_THRESHOLD_COLOR  = '#f59e0b'

sortParetoCategoriesDesc(categories): ChartParetoCategory[]
getParetoTotal(categories): number
getParetoCumulative(categories): number[]
getParetoCumulativePercent(categories): number[]
findParetoThresholdIndex(cumulativePercent, threshold): number
getParetoTicks(max, count?): number[]
describeParetoChart(categories, threshold, formatValue?,
                    formatPercent?): string
```

- `sortParetoCategoriesDesc` is non-mutating; drops
  non-positive / non-finite values.
- `getParetoTotal` sums positive finite values.
- `getParetoCumulative` preserves the input array
  length (non-positive values do not advance the
  running total).
- `getParetoCumulativePercent` clamps every value to
  `[0, 1]`; all zeros when total is zero.
- `findParetoThresholdIndex` clamps the threshold to
  `[0, 1]`; returns the first index where the
  cumulative array crosses; `-1` when nothing crosses
  or `threshold` is non-finite.
- `getParetoTicks` is the standard chart-tick helper
  (evenly-spaced; `[0]` for non-positive max; min
  count 2).
- `describeParetoChart` produces the default ARIA
  summary; honours formatters.

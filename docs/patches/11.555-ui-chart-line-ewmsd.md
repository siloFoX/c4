# 11.555 -- web/src/components/ui/chart-line-ewmsd.tsx

Pure-SVG line chart with an **EWMA mean** line and an
**exponentially-weighted standard deviation band** overlay.
The raw observations render as a dimmed reference line; the
EWMA mean renders as a solid coloured line; a translucent
`mean +/- k * EWM-std` band fills the volatility envelope
around it; points that fall outside the band are flagged as
exceedances.

## Math

The chart uses Finch's **incremental exponentially-weighted
mean and variance** recursion. For each new value `y_t`:

```
delta_t = y_t - mean_{t-1}
mean_t  = mean_{t-1} + alpha * delta_t
var_t   = (1 - alpha) * (var_{t-1} + alpha * delta_t^2)
```

- The mean recursion is exactly the standard EWMA
  `mean_t = alpha * y_t + (1 - alpha) * mean_{t-1}`.
- `var_t` is the **exponentially-weighted variance**: every
  past squared deviation contributes with a geometrically
  decaying weight, so the EWM standard deviation
  `sqrt(var_t)` tracks *recent* volatility.
- Seed: `mean_0 = y_0`, `var_0 = 0` (a single point has no
  spread).

The band at index `t` is `[mean_t - k*std_t, mean_t +
k*std_t]` where `k` is the band-width multiplier (default 2,
a roughly 95% envelope for Gaussian data). A point `y_t` is
an **exceedance** when it falls strictly outside the band --
`'above'` when `y_t > upper`, `'below'` when `y_t < lower`.

### Verified properties (by test)

- A constant series has `var = std = 0` throughout, so the
  band collapses onto the mean.
- A step series `[0,0,0,10]` with `alpha = 0.5` yields, at
  the jump, `mean = 5`, `var = 0.5 * 0.5 * 100 = 25`,
  `std = 5`.
- The `EXCEED_UP` fixture (`[10,10,10,10,10,100]`,
  `alpha = 0.1`) yields exactly `mean = 19`, `var = 729`,
  `std = 27` at the spike.
- The variance is always non-negative.
- Index 0 is always within the band (`std = 0`).
- A spike exceeds the band only when
  `alpha < 1/(1 + k^2)` -- with `k = 2` that is
  `alpha < 0.2`, so the calm-then-spike fixture with
  `alpha = 0.1` produces a genuine exceedance while the
  same data with the default `alpha = 0.3` does not.

## Distinct from related primitives

- `<ChartLineEwma>` (11.539): plots only the EWMA mean line
  -- a single exponential smoother with **no variance and
  no band**. EWMSD adds the second moment: the
  exponentially-weighted standard deviation and the
  volatility envelope around the mean.
- `<ChartLineBollinger>` (11.537): Bollinger Bands use a
  **rolling** (fixed-window) simple moving average plus a
  **rolling sample standard deviation**. The window has a
  hard edge -- a value `N` steps old contributes fully,
  `N + 1` steps old not at all. EWMSD has **infinite memory
  with geometric decay** -- every past point contributes,
  weighted by `alpha * (1 - alpha)^age`, so the band reacts
  faster to a volatility regime change and has no abrupt
  window-drop artifacts.
- `<ChartLineControl>` (11.534): a Shewhart SPC chart with
  **global, constant** control limits at `mean +- k*sigma`
  computed once over the whole series. EWMSD's band is
  **adaptive** -- it breathes with local volatility, so a
  calm stretch has a tight band and a turbulent stretch a
  wide one.
- `<ChartLineConfidence>` (11.502): renders externally
  supplied per-point bounds. EWMSD derives its band from
  the data via the EWM recursion.
- `<ChartLineCusum>` (11.554): a cumulative-sum drift
  detector for the mean. EWMSD models the spread, not the
  cumulative mean shift.

The volatility-adaptive cousin of Bollinger Bands -- the
RiskMetrics-style exponentially-weighted view used when
recent volatility should dominate the envelope.

## Public surface

- `<ChartLineEwmsd series={...} />`.
- `series: ChartLineEwmsdSeries[]` --
  `{id, label, data, color?, alpha?, bandMultiplier?}`.
  Per-series `alpha` / `bandMultiplier` overrides beat
  chart-level.
- `alpha` (default 0.3) -- EWMA smoothing factor; clamped to
  `(0, 1]` (values below 0.01 raised to 0.01).
- `bandMultiplier` (default 2) -- band width `k`; clamped to
  `>= 0`.
- `rawColor`, `exceedanceColor`.
- `bandOpacity` (default 0.15).
- Toggle flags: `showAxis`, `showGrid`, `showDots`,
  `showLegend`, `showTooltip`, `showConfigBadge`, `showRaw`,
  `showMean`, `showBand`, `showExceedances`, `animate`.
- Controlled + uncontrolled visibility.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX` / `formatCoefficient`.
- `onPointClick({series, point})`.

## Pure helpers

- `getLineEwmsdDefaultColor(index)`.
- `getLineEwmsdFinitePoints(points)`.
- `normaliseLineEwmsdAlpha(value)` -- clamps to `(0, 1]`,
  default 0.3.
- `normaliseLineEwmsdBandMultiplier(value)` -- `>= 0`,
  default 2.
- `classifyLineEwmsdExceedance(value, lower, upper)` --
  `'above' | 'below' | 'within'` (strict comparison).
- `computeLineEwmsd(values, alpha)` -- the incremental
  EWM mean/variance recursion; returns per-index
  `{mean, variance, std}`. Drops non-finite values.
- `runLineEwmsd(points, options?)` -- canonical pipeline.
  Sorts ascending, drops non-finite, runs the recursion,
  builds the band and the exceedance flags, returns
  per-sample `{index, x, raw, mean, variance, std, upper,
  lower, exceedance}` plus exceedance counts and
  `finalMean` / `finalStd` / `maxStd`.
- `computeLineEwmsdLayout({...})` -- projects raw + mean
  paths, the band fill + edge paths, and the exceedance
  list.
- `describeLineEwmsdChart(series, options?)`.

## Tooltip

Hovering a dot shows the series label, x, raw value, bold
`EWMA mean`, `EWM std`, the `band: [lower, upper]` range,
and -- when the point is outside the band -- a coloured
`band exceedance (<above|below>)` row.

## Config badge

Top-left badge shows
`EWMSD a=<alpha> k=<multiplier> sd=<finalStd> exc=<count>`.

## Summary

Add a new `<ChartChord>` UI primitive in
`web/src/components/ui/chart-chord.tsx` -- pure-SVG chord
diagram. Categories sit as arcs around a ring; ribbons
inside the ring connect pairs whose mutual flow value sets
each ribbon's width at both endpoints. Ribbon fill colour
encodes the source category. Hovering a ribbon highlights
it and dims the others; hovering a category highlights
every ribbon that touches it. Per-category and per-ribbon
click handlers fire with full payloads.

## API

```ts
interface ChartChordCategory {
  id: string;
  label: string;
  color?: string;
}

interface ChartChordFlow {
  source: string;
  target: string;
  value: number;
}

interface ChartChordProps {
  categories: readonly ChartChordCategory[];
  flows: readonly ChartChordFlow[];
  width?: number;                  // default 380
  height?: number;                 // default 380
  padding?: number;                // default 36
  arcWidth?: number;               // default 16
  arcGapDegrees?: number;          // default 2
  ribbonOpacity?: number;          // default 0.55
  showLabels?: boolean;            // default true
  showTooltip?: boolean;           // default true
  animate?: boolean;               // default true
  highlightOnHover?: boolean;      // default true
  className?: string;
  ariaLabel?: string;              // default 'Chord diagram'
  ariaDescription?: string;
  formatValue?: (v: number) => string;
  onCategoryClick?: (args: {
    category: ChartChordCategory;
    total: number;
  }) => void;
  onFlowClick?: (args: {
    flow: ChartChordFlow;
    source: ChartChordCategory;
    target: ChartChordCategory;
  }) => void;

```

## Behaviour

- Renders a `<div role="region">` wrapping `<svg
  role="img">` carrying the same `aria-label`.
- Layout: `buildChordLayout` computes per-category
  totals via `getChordCategoryTotals` (a flow with
  source `s` and target `t` adds to both `s` and `t`,
  except for self-loops where it only adds once),
  drops zero-total categories, and allocates angular
  space proportionally with `arcGapDegrees` gaps
  between arcs. Within each arc, sub-arcs are
  allocated for every flow touching the category in
  deterministic order (sorted by `source`, then
  `target`).
- Arcs: each category renders as an annular arc from
  the inner radius (`outer - arcWidth`) to the outer
  radius. The first arc starts at 12 o'clock and the
  sweep proceeds clockwise.
- Ribbons: each flow renders as a closed cubic-bezier
  ribbon whose endpoints sit on the inner edge of the
  ring at the sub-arc ranges allocated for that flow.
  The control point of both bezier curves is the
  chart centre so the ribbon curves toward the
  middle.
- Ribbon colour matches the source category. The
  ribbon fill-opacity defaults to `ribbonOpacity`
  (0.55); when a category or another ribbon is
  hovered, non-highlighted ribbons fade to 0.08 and
  highlighted ones lift to >=0.85.
- Per-category click + tooltip; per-ribbon click +
  tooltip. The component prefers the category
  tooltip when both are active (e.g., when a ribbon
  hover happens while a category is also under the
  cursor).
- Mount animation:
  `motion-safe:animate-fade-in` per arc and per
  ribbon. `animate=false` removes the class.
- Default ARIA description: `"Chord diagram with N
  categories, M flows. <label> <total>, ..."` --
  `ariaDescription` overrides; empty input produces
  `"No data"`.

## Pure helpers (exported)

```ts
DEFAULT_CHART_CHORD_WIDTH            = 380
DEFAULT_CHART_CHORD_HEIGHT           = 380
DEFAULT_CHART_CHORD_PADDING          = 36
DEFAULT_CHART_CHORD_ARC_WIDTH        = 16
DEFAULT_CHART_CHORD_ARC_GAP          = 2
DEFAULT_CHART_CHORD_RIBBON_OPACITY   = 0.55
DEFAULT_CHART_CHORD_CATEGORY_COLOR   = '#475569'

getChordCategoryTotals(categories, flows):
  Map<string, number>
chordPolarToCartesian(cx, cy, radius, angle):
  { x, y }
buildChordArcPath(cx, cy, innerRadius, outerRadius,
                  startAngle, endAngle): string
buildChordRibbonPath(cx, cy, innerRadius, sourceStart,
                     sourceEnd, targetStart, targetEnd):
  string
buildChordLayout(categories, flows, arcGapDegrees):
  { arcs, ribbons }
describeChordChart(categories, flows, formatValue?): string
```

- `getChordCategoryTotals` drops non-finite /
  non-positive flows and unknown source / target
  ids; initialises totals at 0 for every supplied
  category so the map is always defined.
- `chordPolarToCartesian` is the standard SVG
  polar-to-cartesian helper.
- `buildChordArcPath` emits a closed annular arc
  (`M / A / L / A / Z`); returns `""` for collapsed
  angles or `inner >= outer`. Uses largeArc flag
  for sweeps > pi.
- `buildChordRibbonPath` emits a closed ribbon
  (`M / A / Q / A / Q / Z`); returns `""` for zero
  inner radius or fully collapsed arcs.
- `buildChordLayout` is the canonical layout pass.
  Zero-total categories are excluded; flows that
  reference excluded categories are dropped.
- `describeChordChart` produces the default ARIA
  summary.

## Tests (52 cases)

- `getChordCategoryTotals`: undirected sums,
  self-loop counted once, drops non-finite /
  non-positive, drops unknown ids, initialises 0 for
  every category.
- `chordPolarToCartesian`: centre at radius 0,
  angle 0 east, -pi/2 above centre.
- `buildChordArcPath`: closed annular path with 2 A
  + 1 L, collapsed angles -> "", inner >= outer ->
  "", largeArc flag for sweeps > pi.
- `buildChordRibbonPath`: closed bezier ribbon with
  2 Q + 2 A, zero inner -> "", collapsed arcs -> "".
- `buildChordLayout`: one arc per active category,
  skips zero-total, empty when no flows, arc sizes
  proportional to totals, one ribbon per positive
  flow, sub-arcs within parent arc bounds, drops
  flows with unknown source / target.
- `describeChordChart`: empty "No data", summary,
  formatter shaping.
- Constants exported.
- Component:
  - region + role + custom aria-label
  - one arc per category, one ribbon per flow
  - arc metadata mirrored (color / total)
  - default color when category has no color
  - ribbon metadata mirrored
  - ribbon color matches source category
  - labels render + suppression
  - category tooltip on arc hover
  - ribbon tooltip on ribbon hover
  - tooltip hides on mouse-leave
  - showTooltip=false suppresses tooltips
  - highlight connected ribbons on category hover
  - no dim when highlightOnHover=false
  - onCategoryClick payload (category + total)
  - onFlowClick payload (flow + source + target)
  - role=graphics-symbol + aria-label per arc +
    ribbon
  - data-animate mirrors prop
  - SVG mirrors width / height / viewBox
  - auto ARIA description rendered

  - empty data renders without crashing

## Out of scope

- Asymmetric ribbons (separate flow magnitudes per
  direction) -- v1 sums both directions into one
  arc total; host can split into two ribbons if
  needed by feeding source-target and target-source
  as two separate flow rows.
- Drag-to-reorder categories around the ring (host
  re-renders with a new category order).
- Animated transitions on data change (snap v1;
  mount fade only).
- Hierarchical chord (group categories into super-
  arcs) -- host wraps with custom SVG layer when
  needed.

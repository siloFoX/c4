# 11.174 -- UI: TagInput primitive

Scope: new `web/src/components/ui/tag-input.tsx` + its test file,
barrel export in `web/src/components/ui/index.ts`, two adoption
sites (`SpecialistsTagEditor`, `pages/Templates.tsx`), plus the
`web/package.json` version bump to 1.11.192.

## What changed

1. **`<TagInput>` primitive.** Multi-tag input field. Renders the
   controlled `value: string[]` as a row of `<Chip>` instances
   (existing primitive, dismiss button wired to remove the tag)
   alongside a native add-tag input at the end. Public surface:

   - `value: string[]` (controlled tag list).
   - `onChange: (tags: string[]) => void`.
   - `placeholder?: string` (default `'Add tag...'`).
   - `maxTags?: number` (no cap when omitted; the add-tag input
     disappears once `value.length >= maxTags`).
   - `normalize?: (raw: string) => string` (defaults to
     `raw.trim()`; runs on Enter / comma commit and on edit-mode
     commit).
   - `dedupe?: boolean` (default `true`; rejects a candidate whose
     normalized form already exists in `value`).
   - `disabled?: boolean` (hides the add-tag input and drops the
     chip dismiss buttons + click-to-edit).
   - `className?: string` (merged onto the wrapper).
   - `inputClassName?: string` (merged onto both the add-tag and
     edit-mode inputs).
   - `ariaLabel?: string` (`aria-label` on the wrapper, which
     carries `role="group"` so screen readers announce a single
     labelled region).
   - Extra `HTMLAttributes<HTMLDivElement>` are spread onto the
     wrapper.

2. **Behaviour.** Enter or comma commits the draft; pasting text
   that contains commas or newlines splits into multiple tags and
   leaves the trailing fragment in the input; Backspace on an
   empty input removes the trailing tag; clicking a chip swaps it
   for an inline edit input (Enter commits, Escape cancels, blur
   commits; an empty commit removes the slot; a duplicate commit
   under the active normalize removes the slot rather than
   creating a parallel entry). `forwardRef` points at the wrapper
   div; classes flow through the shared `cn()` helper.

3. **Adoption.**

   - `web/src/components/SpecialistsTagEditor.tsx`. The edit
     branch dropped the legacy `<Input>` + `useSpecialistTagEditor`
     hook combo (the hook's CSV `+foo,bar` add / `-foo,bar`
     remove prefix grammar does not map onto a tag-array UI) and
     now embeds `<TagInput>` directly. The component owns the
     local `string[]` editor state, pre-fills from the current
     tags on open, and calls `apiPatch` with
     `{ tags, mode: 'replace' }` on Apply. Apply is gated when
     the editor is empty so a stray click cannot wipe the tag
     list.
   - `web/src/pages/Templates.tsx`. The template list itself is
     read-only against the daemon (no add / remove endpoints,
     edit / remove buttons trigger a toast). The natural editing
     surface is `ImportTemplateForm`, which gains an optional
     **Tags** field bound to `<TagInput>` with a lowercase
     `normalize`; the value is held in local state so the form
     can attach it once a daemon-side import endpoint lands.

## Tests

`web/src/components/ui/tag-input.test.tsx` (16 cases):

- renders existing tags as chips
- Enter adds a new tag
- comma separator adds a new tag
- Backspace on an empty input removes the last tag
- Backspace on a non-empty input does NOT remove a tag
- clicking a chip enters edit mode (input replaces chip)
- edit mode Enter commits the change
- edit mode Escape cancels the change
- pasting CSV adds multiple tags
- dedupe rejects duplicates with a lowercase normalize
- maxTags caps additions (input disappears at the cap)
- disabled hides the input and the chip dismiss buttons
- chip dismiss button removes the tag
- className merges on the wrapper
- ariaLabel is applied to the wrapper
- onChange fires with the new tag list

`web/src/components/SpecialistsTagEditor.test.tsx` was rewritten
to match the post-migration component: the hook mock was removed
and `apiPatch` is mocked instead. New coverage includes
view / edit branch toggling, pre-population of the editor from
the current tags, the disabled-Apply guard for an empty editor,
the PATCH payload shape (`{ tags, mode: 'replace' }`), the
onError path on PATCH rejection, chip-dismiss removal inside the
editor, Enter-to-add inside the editor, and the existing locale
flip / button label flip cases.

## Out of scope

- No changes to other UI primitives. `Chip`, `Input`, `Label`,
  `Button`, `FileInput`, `HScroll`, `ListItem`, `Panel`,
  `SearchBar`, `Tooltip`, `Pagination`, `EmptyState` consumed
  as-is.
- No changes to `docs/autonomous-queue-v10.md`.
- `package-lock.json` untouched.
- `useSpecialistTagEditor` hook left in `web/src/lib/`; the
  Specialists view is the only caller and the migration removed
  that call site, but the file is preserved in case a future
  CSV-driven flow wants to revive the prefix grammar.

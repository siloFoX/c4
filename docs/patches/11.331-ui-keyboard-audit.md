## Goal

The dispatch lists:

- Walk every page and assert Tab / Shift+Tab / Esc /
  Enter / Space all work as expected.
- Fix any focus traps, missing keyboard handlers, or
  skipped elements.
- Document the full keyboard map in `docs/patches/`.

This patch lands the audit helper, the integration
audit pass over the storybook gallery, and the
keyboard map document below. The audit found zero
"div-as-button" gaps in the components/ui tree --
every interactive primitive is either natively
focusable or carries the proper tabindex + role.


The synthetic `fire*` helpers are useful for tests
that exercise Escape-to-close (Drawer, Dialog,
ConfirmDialog), Enter-activates-button, and

## Canonical keyboard map

### Global navigation

| Keys | Action |
| --- | --- |
| `?` or `Shift+/` | Open the in-app help modal. |
| `H` | Open the help drawer. |
| `T` | Toggle the onboarding tour. |
| `Ctrl+B` / `Cmd+B` | Toggle the sidebar. |
| `Ctrl+\\` / `Cmd+\\` | Collapse / expand the sidebar rail (desktop only). |
| `Cmd+K` / `Ctrl+K` | Open the Command Palette (v1.11.295). |
| `g g` | Go to top of the page. |
| `g h` | Go to home. |
| `g w` | Go to Workers. |

Global keys are registered via the
`useKeyboardShortcut` hook (or its successors) and
respect text-entry surfaces: when focus is inside
`<input>`, `<textarea>`, or a `contenteditable`
element, the global shortcut is skipped so the
keystroke reaches the input.

### Per-surface keys

| Surface | Key | Effect |
| --- | --- | --- |
| Dialog / ConfirmDialog | `Esc` | Close (cancel). |
| Dialog / ConfirmDialog | `Tab` | Cycle through focusable elements (focus trap, `useFocusTrap`). |
| Dialog / ConfirmDialog | `Shift+Tab` | Reverse cycle. |
| Drawer | `Esc` | Close. |
| Tooltip | `Esc` | Hide (when triggered via focus). |
| Popover | `Esc` | Close + return focus to trigger. |
| Tabs (pill / line) | `ArrowLeft` / `ArrowRight` | Move active tab (roving tabindex). |
| Tabs | `Home` / `End` | Jump to first / last tab. |
| Accordion | `ArrowDown` / `ArrowUp` | Move between headers. |
| Accordion | `Home` / `End` | Jump to first / last header. |
| Accordion | `Enter` / `Space` | Toggle the header. |
| SegmentedControl | `ArrowLeft` / `ArrowRight` | Move active option. |
| RadioGroup | `ArrowLeft` / `ArrowRight` / `ArrowUp` / `ArrowDown` | Move selection. |
| Combobox / SearchBar (with suggestions) | `ArrowDown` | Open / move down. |
| Combobox / SearchBar | `ArrowUp` | Move up. |
| Combobox / SearchBar | `Enter` | Select the focused suggestion. |
| Combobox / SearchBar | `Esc` | Close the suggestion dropdown. |
| Switch | `Space` / `Enter` | Toggle. |
| Chat composer | `Enter` | Send. |
| Chat composer | `Shift+Enter` | Insert newline. |
| Terminal | `Ctrl+F` | Search. |

### Activation keys on buttons

Every native `<button>` activates on both `Enter` and
`Space`. Custom button-like elements
(`<div role="button" tabindex="0">`) must register a
`keyDown` handler that filters for both keys. The
audit scanner flags any `[data-clickable="true"]`
element that does NOT carry `tabindex` + a
button-like `role`.

### Focus management primitives

- `useFocusTrap` (`web/src/hooks/use-focus-trap.ts`)
  -- traps focus inside a container, cycles on Tab /
  Shift+Tab. Used by Dialog, Drawer, ConfirmDialog,
  CommandPalette.
- `useFocusCycle` (`web/src/hooks/use-focus-cycle.ts`)
  -- cycles focus across a fixed item list with
  ArrowLeft / ArrowRight (horizontal) or ArrowUp /
  ArrowDown (vertical) and Home / End. Used by
  Tabs, SegmentedControl, RadioGroup, Accordion.
- `useRovingTabindex` (`web/src/hooks/use-roving-tabindex.ts`)
  -- sets `tabindex="0"` on the active item and
  `tabindex="-1"` on the rest so only one item is in
  the document tab order. Used by Tabs, Accordion,
  SegmentedControl, ActionMenu.

## Audit pass

`web/src/pages/UIDemoRoute.keyboard.test.tsx`
runs `auditKeyboardNav()` against the storybook
gallery (every primitive variant rendered in one
place). Two assertions:

1. zero `div-as-button` gaps;
2. `> 10` tabbable nodes in the inventory.

Both pass as of v1.11.349. The audit specifically
checks for elements marked
`data-clickable="true"` without the proper
tabindex + role; this is the canonical foot-gun the
audit catches.

## What is NOT in this patch

- **Real-browser focus traversal.** jsdom does NOT
  advance `document.activeElement` in response to a
  Tab keydown. The helper enumerates the tabbable
  inventory and validates affordances, but cannot
  prove the OS-level focus algorithm visits them in
  order. A Playwright follow-up can re-validate the
  geometry side.
- **Per-page keyboard audits.** UIDemoRoute is the
  first adopter; per-page audits for Queue, Workers,
  Sessions, Notifications, etc. can land
  incrementally using the new helper.
- **Custom div-as-button remediation.** The audit
  found zero offenders; no remediation work was
  needed. If a future patch introduces a clickable
  `<div>` without tabindex + role, the integration
  audit will fail loudly with the offending element's
  selector.

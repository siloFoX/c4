# Patch 11.753 -- UI: chart-line-adx-di-plus primitive

Released as `v1.11.771` on 2026-05-26.

## Scope

Add `<ChartLineAdxDiPlus>` -- a pure-SVG two-panel chart with the
Wilder **+DI** directional indicator (SMA-smoothed variant)
reading the smoothed positive directional movement over the
true range, scaled to `100`.

## Definition

Inputs:

- `data: ChartLineAdxDiPlusPoint[]` where
  `ChartLineAdxDiPlusPoint = { x; high; low; close }`. Bars with
  `high < low` are dropped.
- `period: number` -- the SMA length. Default `14`. Normalized
  to an integer `>= 2`.
- `threshold: number` -- the bull boundary; strong starts at `2
  * threshold`. Default `25`. Normalized to a finite in `(0,
  100]`.

Algorithm (per bar `i >= 1`):

```
upMove   = high[i] - high[i - 1]
downMove = low[i - 1] - low[i]
plusDM   = (upMove > downMove && upMove > 0) ? upMove : 0
TR       = max(high - low, abs(high - prevClose),
               abs(low  - prevClose))

plusS = SMA(plusDM, period)
trS   = SMA(TR, period)

+DI[i] = 100 * plusS / trS
```

A bar is null when `trS == 0`.

## Bit-exact algebraic anchors

**CONST_FLAT** (`high == low == close == K`): every `TR` is zero
-> `trS = 0` -> `+DI` null at every bar.

**RISING** (`high == low == close == i + 10`): for every `i >=
1`, `upMove = +1`, `downMove = -1` -> `plusDM = 1`. `TR = max(0,
|+1|, |+1|) = 1`. SMA over any `period` -> `plusS = 1, trS = 1`
-> `+DI = 100 * 1 / 1 = 100` bit-exact.

**RISING_HALF** (`high == close == i + 10, low == i + 8`): for
every `i >= 1`, `upMove = +1`, `downMove = (i - 1 + 8) - (i + 8)
= -1` -> `plusDM = 1`. `TR`: `hl = 2, hc = |i + 10 - (i + 9)| =
1, lc = |i + 8 - (i + 9)| = 1`. `TR = max(2, 1, 1) = 2`. `plusS
= 1, trS = 2` -> `+DI = 100 * 1 / 2 = 50` bit-exact.

**FALLING** (`high == low == close == 19 - i`): `upMove = -1,
downMove = +1` -> `plusDM = 0`. `TR = 1`. `plusS = 0, trS = 1`
-> `+DI = 0` bit-exact.

**Translation invariance**: shifting every OHLC field by `+k`
leaves every `upMove`, `downMove`, `hl`, `hc`, `lc` unchanged
(differences are invariant). So `+DI` is unchanged bit-exact.

## Component

`forwardRef<HTMLDivElement, ChartLineAdxDiPlusProps>` with
displayName. Renders:

- A region-role outer `<div data-section="chart-line-adx-di-plus">`.
- An `<svg role="img">` with an `aria-describedby` description.
- Top panel: close line, optional dots.
- Bottom panel: +DI line, zone-coloured markers (strong / bull
  / weak / none), zero reference line, dashed `threshold` and
  `2 * threshold` reference lines.
- Legend toggle for `price` / `diPlus`, plus a `showDiPlus` prop
  and an `onPointClick` handler.
- ARS design-system tokens for colour and motion (`motion-safe:
  animate-fade-in`).

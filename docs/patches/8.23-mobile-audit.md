# TODO 8.23 - Mobile UI audit (puppeteer device emulation)

**Status:** done
**Branch:** `c4/mobile-audit`
**Scope:** Extend the 8.22 visual-regression pass in
`tools/ux/explore.mjs` with puppeteer device emulation for iPhone 13,
iPhone SE, Galaxy S20, and iPad Mini in portrait + landscape, plus
mobile-specific checks (horizontal overflow, 44x44 touch targets,
14 px min font, hover-only affordances, soft-keyboard probe) and a
separate `mobile` section in the audit report. 8.21 and 8.22 scope is
deliberately left alone - this patch adds new surface only.

## Summary

A second visual pass in `tools/ux/explore.mjs` runs after the existing
desktop/tablet sweep: four `KnownDevices` descriptors emulate each
phone/tablet in portrait *and* landscape (eight configurations),
cycles through the same eight `VISUAL_PAGES` routes, writes stable
PNGs under `patches/ui-audit-<date>/mobile/`, diffs each against a
persistent baseline at `patches/ui-audit-baseline/mobile/`, and
collects in-page mobile-specific metrics: horizontal overflow, touch
targets smaller than 44x44 CSS px, text with resolved `font-size`
below 14 px, hover-only affordances that change
visibility/display/opacity, and a soft-keyboard `visualViewport`
probe on `/`, `/workflows`, `/settings` for iPhone 13 and Galaxy S20
portrait only. Clipping reuses the 8.22 `detectClipping` helper
verbatim but stores results under `mobile.clipping`. All results land
under a new `mobile` key in the audit report, leaving the 8.22
`visual` key untouched.

A `--skip-mobile` CLI flag short-circuits the mobile pass so the 8.22
desktop pass can still run stand-alone during dev iteration, and the
mobile pass runs *after* the desktop pass so a mobile failure cannot
swallow the functional report.

## Run

```
node tools/ux/explore.mjs --iteration <n>              # desktop + mobile
node tools/ux/explore.mjs --iteration <n> --skip-mobile # desktop only
```

Credentials stay at `/tmp/c4-silofox-cred` (username `silofox`). The
existing functional click-through runs first, then the 8.22 visual
pass, then (unless `--skip-mobile`) the new mobile pass. All three
funnel into the same `ui-audit-report.json`.

## Device matrix

```js
const MOBILE_DEVICES = [
  { id: 'iphone-13',  device: KnownDevices['iPhone 13']  },
  { id: 'iphone-se',  device: KnownDevices['iPhone SE']  },
  { id: 'galaxy-s20', device: KnownDevices['Galaxy S20'] },
  { id: 'ipad-mini',  device: KnownDevices['iPad Mini']  },
];

const ORIENTATIONS = ['portrait', 'landscape'];
```

Landscape swaps `device.viewport.width` and `device.viewport.height`
on `page.setViewport` after `page.emulate(device)`, and flips
`isMobile: true`, `hasTouch: true`, `isLandscape: true` so the layout
sees the device touch + orientation signals alongside the swapped
dimensions.

4 devices x 2 orientations x 8 pages = 64 screenshots per mobile run.
The sweep shares a single browser instance and a single page - we
cycle `page.emulate()` + `page.setViewport()` + `page.reload()`
between iterations rather than re-launching Chrome eight times
(spec P4).

## Mobile-specific checks

Each runs after the screenshot for each (device, orientation, page)
cell and records up to the capped sample alongside a count in the
report:

### 1. Horizontal overflow (`mobile.overflow`)

Re-uses the 8.22 `detectOverflow` helper verbatim:
`getBoundingClientRect().right > window.innerWidth + 1`. Capped at
20 samples per cell. Horizontal scroll on mobile is a P0 layout bug;
keeping it in a separate report key makes it easy to filter.

### 2. Touch target size (`mobile.touchTargets`)

```js
const SELECTOR =
  'button, a[href], [role="button"], input, [role="link"], [tabindex]:not([tabindex="-1"])';
```

Walks every match, skips hidden elements (`el.offsetParent === null`),
records any whose bounding box is narrower than 44 CSS px *or*
shorter than 44 CSS px (iOS HIG + Android guidelines). Up to 30
samples per cell with tag / class / width / height / 40-char text
snippet.

### 3. Minimum font size (`mobile.smallFonts`)

`TreeWalker(document.body, SHOW_TEXT)` walks visible text nodes,
skips whitespace-only nodes, reads `parseFloat(getComputedStyle
(parent).fontSize)` and flags anything `< 14`. Up to 20 samples per
cell with tag / class / text (80-char cap) / resolved size.

### 4. Hover-only affordances (`mobile.hoverOnly`)

Walks `document.styleSheets`, catches CORS-blocked sheets with a
silent `try/catch`, looks for selectors containing `:hover` whose
`cssText` also matches `/visibility|display|opacity/i`. Up to 20
matches, best-effort - advisory field only (spec 8.23 P2 step 4).

### 5. Soft-keyboard probe (`mobile.softKeyboard`)

For `/`, `/workflows`, `/settings` on **iPhone 13 portrait** and
**Galaxy S20 portrait** only: reads
`window.visualViewport.height` before focusing the first
`<input>` / `<textarea>` and again after, records both heights and
an `obscured: bool` flag set true when the focused element's
bounding-box bottom sits below the shrunken viewport. Pages with no
input skip silently (entry is omitted from the report). Limited to
two portrait devices per family to cut runtime.

### 6. Long-text truncation (`mobile.clipping`)

Re-uses the 8.22 `detectClipping` helper verbatim
(`scrollWidth > clientWidth` on `text-overflow: ellipsis` /
`overflow: hidden`). Stored under `mobile.clipping` so mobile
operators can see mobile-specific clipping without scanning the
desktop list.

## `--skip-mobile` flag

Minimal parsing per spec 8.23 P4:
`process.argv.includes('--skip-mobile')`. When set, the entire
`runMobileAudit` block is bypassed but the 8.22 visual pass and the
functional click-through still land as before. Reason: the mobile
pass takes ~2 min on a typical dev box and you often don't need it
while iterating on 8.22 scope.

## Performance notes

- Single `puppeteer.launch` inside `runMobileAudit` shared across
  all 4 devices x 2 orientations x 8 pages = 64 screenshots.
- `page.emulate(device)` + conditional `page.setViewport(...swap)`
  between iterations, followed by `page.reload({ waitUntil:
  'networkidle2' })` so React re-measures against the new viewport
  + user agent.
- Login runs exactly once (at a neutral 1440x900 desktop viewport)
  before the first `emulate` call, then localStorage auth carries
  through every reload.

## Decisions / deviations from the spec

- **Spec says `import ... from 'puppeteer'`, code uses
  `puppeteer-core`.** The 8.20-era baseline has always used
  `puppeteer-core` (the chrome binary lives at
  `/root/.cache/puppeteer/chrome/...` and is driven manually).
  `KnownDevices` exports from `puppeteer-core@^23` at the same
  path, so the spec's import shape works verbatim with an
  `puppeteer-core` identifier swap. Kept `puppeteer-core` per the
  8.22 convention to avoid churning `tools/ux/package.json`.
- **Soft-keyboard probe returns `null` when the page has no
  input.** Rather than recording a placeholder entry we simply omit
  the cell from `mobile.softKeyboard` - matches spec intent ("Skip
  if the page has no input").
- **Login happens once at a desktop viewport before the mobile
  sweep.** Spec text says "share the same browser instance across
  devices" but is silent on where login sits; running login on the
  actual phone emulation adds ~8 flaky form-submits per run for no
  signal since auth is a pure API flow. One desktop-viewport login
  + token in localStorage is carried through every subsequent
  `page.reload()`.
- **Mobile diff overlays land in `patches/ui-audit-<date>/diffs/`
  with a `mobile-` filename prefix** instead of a separate
  `mobile-diffs/` folder, so operators who already have the 8.22
  diff viewer muscle memory don't have to relearn the layout.

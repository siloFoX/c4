# 8.42 (partial) — Composer special-keys hidden on desktop

Branch: `dgx-spark/composer-mobile-keys`

## What ships

Step 1 of TODO 8.42 (composer cleanup). The WorkerDetail composer's
"Keys" row (Esc / Ctrl-C / Ctrl-D / Tab / arrows) is now hidden at
the `md:` breakpoint. Desktop users have a physical keyboard that
already produces these inputs natively; the buttons existed for
mobile soft-keyboard users where Esc / Ctrl-C aren't reachable
without scrolling the keyboard layout.

## Surface area

`web/src/components/WorkerDetail.tsx` — single class change:

```diff
-<div className="flex flex-wrap items-center gap-2">
+<div className="flex flex-wrap items-center gap-2 md:hidden">
```

with a comment explaining the rationale so a future refactor doesn't
silently revert it.

## Tests

`tests/composer-mobile-keys.test.js` — 2 source-grep assertions:

1. The "Keys" row container carries the `md:hidden` class.
2. The leading comment cites TODO 8.42 + the soft-keyboard reason
   so the contract isn't lost across renames.

`npm test` full suite holds the 116 / 117 pre-existing baseline (the
lone failure is the `file-transfer` test PR #24 fixes).
`npm --prefix web run build` holds the 47 TS error baseline.

## Test plan (manual)

1. `c4 daemon start && cd web && npm run dev`
2. Open the WorkerDetail tab on desktop (>= 768px). The composer's
   bottom row (the one starting with "Keys" + Esc / Ctrl-C / etc)
   is gone.
3. Resize the browser below 768px. The row reappears so soft-
   keyboard users can still send Ctrl-C / Esc.
4. The actual keyboard interaction (Ctrl-C in the textarea, etc) is
   unaffected — those are handled by the textarea's keydown
   listener, not the buttons.

## Not in scope (rest of 8.42)

The full TODO 8.42 spec is broader:

  claude.ai style)

  dropdown next to the composer

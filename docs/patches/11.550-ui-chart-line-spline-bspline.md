# 11.550 -- web/src/components/ui/chart-line-spline-bspline.tsx

Pure-SVG line chart with a **cubic B-spline interpolation
overlay**. Distinct from Catmull-Rom and monotone-cubic
splines: a B-spline curve does **not** pass through its
interior control points but instead *approximates* them with
a smooth blended curve, while passing exactly through the
first and last raw points when the curve is *clamped*
(default).

## Math

For each cubic B-spline segment with 4 consecutive control
points `(P_0, P_1, P_2, P_3)` and parameter `t in [0, 1]`,
the canonical uniform basis is:

```
B(t) = (1/6) * [
  (1 - t)^3                * P_0
  + (3t^3 - 6t^2 + 4)      * P_1
  + (-3t^3 + 3t^2 + 3t + 1) * P_2
  + t^3                    * P_3
]
```

This is the standard uniform cubic B-spline basis. Each
basis function is a piecewise cubic. The four basis values
sum to `6` (so dividing by 6 gives partition of unity), which
is what makes the curve reproduce constants exactly.

Each segment can be converted to a cubic Bezier with control
points:

```
b_0 = (P_0 + 4*P_1 + P_2) / 6
b_1 = (2*P_1 + P_2) / 3
b_2 = (P_1 + 2*P_2) / 3
b_3 = (P_1 + 4*P_2 + P_3) / 6
```

The endpoints `b_0` and `b_3` are on the curve at `t = 0`
and `t = 1` respectively. We emit one SVG `C` command per
B-spline segment.

## Clamped vs. open

When `clamp = true` (the default), the first and last raw
control points are each duplicated twice at the start and
end of the control sequence. The first B-spline segment then
uses control points `(P_0, P_0, P_0, P_1)`, and its endpoint
at `t = 0` is `(P_0 + 4*P_0 + P_0) / 6 = P_0`. Similarly the
last segment ends exactly at the last raw point. This is the
clamped (knot-multiplicity = degree + 1) form of the curve.

When `clamp = false`, the curve uses only the raw control
points. The curve does not pass through the endpoints; it
begins at `(P_0 + 4*P_1 + P_2) / 6` and ends at
`(P_{n-3} + 4*P_{n-2} + P_{n-1}) / 6`. This form has the
**linear precision** property: for a linear input the curve
exactly reproduces the line in its interior segments.

## Why B-spline over Catmull-Rom / monotone-cubic / SG / LOESS?

- `<ChartLineSmooth>` (Catmull-Rom + monotone-cubic):
  *interpolating* splines that pass through **every** control
  point. B-spline is *approximating* and does not pass
  through interior controls -- it's smoother in the presence
  of noise because each control point only has local
  influence (4 segments) and is "filtered" by the basis.
- `<ChartLineSavgol>` (11.547): polynomial fit on a fixed
  window with equal weights. Discrete smoother, no curve
  in between samples.
- `<ChartLineLoess>` (11.549): local weighted regression
  per query point. Statistical smoother for noisy data.
- `<ChartLineMovingAvg>` (11.513): simple moving average,
  uniform-weight window.
- `<ChartLineEwma>` (11.539): exponential decay smoother.
- `<ChartLineKalman>` (11.538): Bayesian state-space filter.
- `<ChartLineHampel>` (11.548): robust outlier filter
  (replaces outliers with the median of a sliding window).
- `<ChartLineDecompose>` (11.543): trend + seasonal split.

B-spline's distinctive value: a **C^2-continuous** curve
(continuous second derivative) suitable for visualising data
that should appear smooth but does not need to interpolate
every point exactly. It is the canonical de Boor / NURBS
curve form used in CAD/CAM and computer graphics.

## Public surface

- `<ChartLineSplineBspline series={...} />`.
- `series: ChartLineSplineBsplineSeries[]` --
  `{id, label, data, color?, clamp?, samplesPerSegment?}`.
- `clamp` (default `true`) -- whether to duplicate the first
  and last control points so the curve passes through them.
- `samplesPerSegment` (default `16`; clamped to `[2, 256]`)
  -- how densely to sample each B-spline segment when
  computing the per-control-point smoothed value (via
  linear interpolation between dense samples).
- `rawColor`, `residualPosColor`, `residualNegColor`,
  `controlColor`.
- Toggle flags: `showAxis`, `showGrid`, `showDots`,
  `showLegend`, `showTooltip`, `showConfigBadge`,
  `showRaw`, `showControlPolygon`, `showResidualSticks`,
  `animate`.
- Controlled + uncontrolled visibility.
- Standard sizing + format + a11y props.
- `formatValue` / `formatX`.
- `onPointClick({series, point})`.

  parametric samples along the curve.

  -- linear interpolation between curve samples to find the
  smoothed y at a given x.
- `runLineBSpline(points, options?)` -- canonical pipeline.
- `computeLineSplineBsplineLayout({...})`.
- `describeLineSplineBsplineChart(series, options?)`.

# 11.520 -- web/src/components/ui/chart-line-cumulative.tsx

Pure-SVG multi-series line chart that **plots the running cumulative
sum** of per-period increments. Each `{x, value}` sample is an
increment; the chart computes `cumulative_i = baseline + sum(value_0
.. value_i)` and renders that as the line. Optional faint increment
bars below the line show the per-period delta; optional horizontal
target line + crossing detection drives a "progress to target" %
shown in the legend.

## Distinct from other line + reference primitives

- `<ChartLineMovingAvg>` (11.513): windowed average -- does NOT
  accumulate.
- `<ChartLineTrend>` (11.512): regression line -- not a running sum.
- `<ChartLineAreaStacked>` (11.505): stacks SERIES, not a running sum
  inside one series.
- `<ChartLineBaseline>` (11.510) / `<ChartLineTarget>` (11.518):
  classification or shading against a reference. Cumulative is a
  TRANSFORM of the data, not a comparison.
- `<ChartLineForecast>` (11.519): historical/forecast split on the
  time axis -- different concern.

The canonical "year-to-date" / "running total" / "progress to goal"
visualisation for sales pipelines, fundraisers, build artefact size,
cumulative downloads, etc.

## Public surface

- `<ChartLineCumulative series={...} target={1000} />`.
- Per-series `{id, label, data, color?, baseline?}`. `data` entries
  are `{x, value}` (the per-period increment, not the running total).
  `baseline` defaults to `0`; per-series override always wins.
- Optional `target?` to flag the progress goal. When supplied, the
  layout finds the first index where cumulative crosses target and
  exposes `targetCrossingX` (linearly interpolated x within the
  segment).
- Toggle flags: `showAxis`, `showGrid`, `showDots`, `showLegend`,
  `showTooltip`, `showIncrements`, `showBaseline`, `showTarget`,
  `showTargetLabel`, `animate`.
- `incrementWidth` (`6`) tunes the bar width.
- Standard sizing + format + a11y props + `formatValue` /
  `formatX` / `formatPercent`.
- Controlled + uncontrolled `hiddenSeries`.
- `onPointClick({series, point})`.

## Pure helpers

- `getLineCumulativeDefaultColor(index)` -- 10-color palette cycle.
- `getLineCumulativeFinitePoints(points)` -- finite-only filter.
- `computeRunningCumulative(values, baseline?)` -- returns a
  same-length array of running sums. **Non-finite values are treated
  as 0** (they don't reset the sum). Non-array -> `[]`. Non-finite
  baseline falls back to `0`.
- `buildLineCumulativeRunningPoints(points, baseline?)` -- sorts
  finite samples by x ascending and returns
  `{index (original), x, value, cumulative}` per sample. Non-array
  -> `[]`.
- `computeLineCumulativeStats(running, baseline, target?)` -- returns
  `{finiteCount, totalValue, total, maxIncrement, minIncrement,
  baseline, reachedTarget, targetCrossingX, targetCrossingIndex,
  percentToTarget}`. Crossings strictly inside a segment use linear
  interpolation in x: `x* = prev.x + (curr.x - prev.x) * (target -
  prev.cumulative) / (curr.cumulative - prev.cumulative)`.
  `percentToTarget` is clamped to `[0, 1]`.
- `computeLineCumulativeLayout({series, target?, hiddenSeries?,
  ...})` -- returns `{series, xTicks, yTicks, xMin, xMax, yMin, yMax,
  baselineY, targetY, target, innerWidth, innerHeight, totalPoints,
  visibleSeriesCount}`. Each layout series carries `{id, label,
  index (stable original), color, baseline, points, path, stats,
  finiteCount, totalCount}`. Per-point `{index, x, value, cumulative,
  px, py, incrementY, incrementHeight}` (the increment rect sits
  between cumulative and baseline, signed by `value`).
- `describeLineCumulativeChart(series, target?, hidden?,
  formatValue?)` -- builds the screen-reader description.

## ARIA + test selectors

- Root `role="region"` + `aria-label` + `aria-describedby`.
- SVG `role="img"`.
- Per-line path + per-dot are `role="graphics-symbol"` +
  `tabIndex={0}`; target line is `role="graphics-symbol"`. Dot
  aria-label includes value + cumulative.
- Every node carries `data-section="chart-line-cumulative-..."` for
  test selectors.
- Series groups expose `data-series-total`, `data-series-baseline`,
  `data-series-percent-to-target`, `data-series-reached-target`,
  `data-series-target-crossing-x`.
- Increments expose `data-increment-sign` (`'positive' | 'negative'`).
- Root mirrors `data-target` (empty when no target).

## Tooltip

Hovering a dot shows the series label, x, `value: +/-<v>` (signed),
and `cumulative: <v>` (bold).

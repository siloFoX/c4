## Summary

New `ChartLineVidya` UI primitive at
`web/src/components/ui/chart-line-vidya.tsx` -- a pure-SVG line chart
with a Variable Index Dynamic Average overlay whose smoothing adapts
to volatility.

## Indicator math

Tushar Chande's Variable Index Dynamic Average:

```
CMO   = computeLineVidyaCmo(values, cmoPeriod)   -- runs -100..+100
k     = |CMO| / 100                              -- volatility index, 0..1
alpha = 2 / (period + 1)                         -- base smoothing constant
VIDYA[i] = alpha*k*value[i] + (1 - alpha*k)*VIDYA[i-1]
```

The VIDYA is an exponential moving average whose smoothing constant
is scaled, bar by bar, by a **volatility index** `k`. That index is
the absolute Chande Momentum Oscillator divided by 100, so it runs 0
to 1:

- A strong trend pushes the CMO toward `+/-100`, so `k` approaches 1
  and the effective smoothing constant approaches `alpha` -- the
  average tracks the price closely.
- A directionless market pushes the CMO toward 0, so `k` approaches
  0 and the effective constant approaches 0 -- the average
  **freezes**, holding its prior value.

The CMO is defined from index `cmoPeriod` onward. The VIDYA is
seeded at index `cmoPeriod - 1` with that bar's price and is
recursive from index `cmoPeriod`.

### Worked fixture

`values = [10, 14, 18, 14, 18, 14, 10]`, `period 3`, `cmoPeriod 2`.
The base smoothing constant is exactly `2 / (3 + 1) = 0.5`, and the
2-change CMO windows come out to clean `100 / 0 / -100`, so the
volatility index is `1 / 0 / 1` and the whole pipeline stays
bit-exact:

```
changes = [_, +4, +4, -4, +4, -4, -4]
cmo  = [_, _, 100, 0, 0, 0, -100]
k    = [_, _, 1, 0, 0, 0, 1]
VIDYA seeded at index 1 with value[1] = 14, then recursive:
  VIDYA[2] = 0.5*1*18 + 0.5*14 = 16
  VIDYA[3] = 0.5*0*14 + 1*16   = 16   (k = 0 -> frozen)
  VIDYA[4] = 16,  VIDYA[5] = 16       (k = 0 -> frozen)
  VIDYA[6] = 0.5*1*10 + 0.5*16 = 13
vidya = [null, 14, 16, 16, 16, 16, 13]
```

The fixture exercises the defining behaviour: at indices 3-5 the
price oscillates so the CMO is 0, `k` is 0, and the VIDYA freezes at
16; at the trending bars (`k = 1`) it tracks. Every value is exact,
so the tests assert the series with `toEqual`.

## API

- `getLineVidyaFinitePoints(points)` -- drop points with a
  non-finite `x` or `value`.
- `normalizeLineVidyaPeriod(period, fallback)` -- coerce to a
  positive integer; non-finite or sub-1 falls back, fractional
  floors.
- `computeLineVidyaCmo(values, cmoPeriod)` -- the Chande Momentum
  Oscillator series, the volatility gauge that drives the VIDYA.
- `computeLineVidya(values, period, cmoPeriod)` -- returns
  `{ cmo, k, vidya }`.
- `runLineVidya(points, options?)` -- sort finite points by `x`,
  compute the series, return `{ series, period, cmoPeriod, cmo, k,
  vidya, samples, vidyaFinal, vidyaMin, vidyaMax, aboveCount,
  belowCount, ok }`. Each sample carries the price's `above` /
  `below` / `on` position versus the VIDYA. `ok` is false for fewer
  than two points.
- `computeLineVidyaLayout(options)` -- project the price and VIDYA
  onto one shared panel. Returns the panel, ticks, the price path,
  the VIDYA path, and VIDYA markers.
- `describeLineVidyaChart(data, options?)` -- ARIA description.

## Component

`ChartLineVidya` is a `forwardRef` region (`role="region"`) with an
ARIA-linked description, a `role="img"` SVG, axis ticks and grid, a
config badge calling out the period and the CMO period, a
two-series legend (Price / VIDYA) with toggle, a hover/focus tooltip
exposing the value, VIDYA, volatility index and position, and a
`motion-safe:animate-fade-in` entrance. Every node carries a
`data-section` attribute.

## Distinctness

- **vs chart-line-kama** -- both are adaptive recursive averages.
  The KAMA scales its smoothing by an *efficiency ratio* and its
  squared smoothing constant never reaches zero (a slow bound). The
  VIDYA scales by the *absolute Chande Momentum Oscillator*, so a
  directionless market drives the index to 0 and the line fully
  *freezes* -- something the KAMA cannot do.
- **vs chart-line-ema / dema / tema / wma / trima** -- those use a
  *fixed* smoothing scheme. The VIDYA's smoothing varies bar to bar
  with the volatility index.
- **vs chart-line-cmo** -- that primitive *plots* the CMO as an
  oscillator panel. The VIDYA *consumes* the CMO internally as a
  volatility gauge to scale an exponential average -- it never plots
  the oscillator itself.

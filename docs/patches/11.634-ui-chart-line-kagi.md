## Summary

line transform that reverses direction only on a reversal
threshold and thickens on trend confirmation.

## Indicator math

A Kagi line is a continuous zigzag of vertical segments that
ignores time and reacts only to price. Two rules drive it:

**Reversal.** The line extends with the price while it advances.
It reverses direction only when the price moves back by at least
`reversalAmount` from the current segment extreme:

```
direction up:   p > extreme           -> extend the extreme
                extreme - p >= rev    -> reverse down, new vertex p
direction down: p < extreme           -> extend the extreme
                p - extreme >= rev    -> reverse up,   new vertex p
```

`computeLineKagiTurns` returns the vertex sequence -- the start,
then each reversal price; the last entry is the live extreme.

**Thickness.** The line is thin (yin) or thick (yang), a
persistent state. `computeLineKagiSegments` flips it to **thick**
when an up segment pushes past the vertex two turns back (the
prior shoulder), and back to **thin** when a down segment breaks
below the vertex two turns back (the prior waist). A segment that
confirms nothing carries the previous thickness; the first
segment is thin.

### Worked fixture

`reversalAmount 3`, price series `[10, 14, 13, 11, 16, 12, 20]`:

```
10 ->14   start an up line
->13 ->11 the 11 is a 3-unit pullback -> reverse down
->16      a 5-unit rebound           -> reverse up
->12      a 4-unit pullback          -> reverse down
->20      an 8-unit rebound          -> reverse up
turns = [10, 14, 11, 16, 12, 20]
```

Segments and thickness: `up thin` (10-14), `down thin` (14-11),
`up thick` (11-16, breaks the prior shoulder 14), `down thick`
(16-12, the pullback does not break the prior waist 11, so the
thickness carries), `up thick` (12-20). 3 up, 2 down; 2 thin,
3 thick.

## API

- `getLineKagiFinitePoints(points)` -- drop price points with a
  non-finite `x` or `value`.
- `normalizeLineKagiReversal(amount, fallback)` -- coerce the
  reversal amount to a positive finite number.
- `computeLineKagiTurns(values, reversalAmount)` -- the
  turning-point vertices.
- `computeLineKagiSegments(turns)` -- the segments
  `{ index, fromValue, toValue, direction, thickness }`.
- `runLineKagi(points, options?)` -- sort finite points by `x`,
  run the transform, return `{ series, reversalAmount, turns,
  segments, upCount, downCount, thinCount, thickCount, ok }`.
  `ok` is false when the price never reverses.
- `computeLineKagiLayout(options)` -- lay the segments out one
  per column and project each to a vertical-plus-connector path.
- `describeLineKagiChart(data, options?)` -- ARIA description.

## Component

`ChartLineKagi` is a `forwardRef` region (`role="region"`) with
an ARIA-linked description, a `role="img"` SVG, axis ticks and
grid, a config badge showing the reversal amount, a two-series
legend (Yin / Yang) with toggle, a hover/focus tooltip exposing
the segment index, direction, thickness and price range, and a
`motion-safe:animate-fade-in` entrance. It consumes
`{ x, value }` points plus a `reversalAmount`. Each segment is an
SVG path -- yin segments thin and red, yang segments thick and
green -- laid out one per column with a right-angled connector.
Every node carries a `data-section` attribute.

## Distinctness

- **vs chart-line-renko** -- Renko fixes the brick size and steps
  once per full price move. A Kagi segment has a *variable*
  length and the line reverses only on a counter-move of the
  reversal amount, so a long trend is one long segment rather
  than many bricks.
- **vs the time-based line charts** -- the Kagi x-axis is a
  segment ordinal, not time; flat price stretches add nothing.
- **vs chart-line-heikin-ashi** -- Heikin-Ashi smooths every bar;
  Kagi re-samples the series onto reversal-driven vertices and
  carries a thin/thick trend state.
